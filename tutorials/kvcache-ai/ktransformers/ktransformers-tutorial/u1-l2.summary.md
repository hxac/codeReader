本讲拆解 KTransformers 仓库的目录结构与代码组织，分三个最小模块：顶层目录、`kt-kernel` 子目录、归档目录 `archive/`。核心结论：顶层是「文档 + shim + 运行时子项目」的组合，运行时代码全部集中在 `kt-kernel/`；顶层的 `ktransformers.py`、`setup.py` 只是门面，通过 `install_requires` 转发安装 `kt-kernel`，故 `pip install ktransformers` 实际装的是 `kt-kernel`。`kt-kernel/` 内部按语言分层：Python 包在 `python/`、C++ CPU 算子在 `operators/`、CUDA GPU 算子在 `cuda/`、CPU 线程运行时在 `cpu_backend/`，C++↔Python 桥梁为 `ext_bindings.cpp`；微调（SFT）在 `python/sft/`，`kt` 命令在 `python/cli/`。关键术语：shim（垫片）、`package-dir` 包目录映射、`kt-kernel`（发布名/连字符）与 `kt_kernel`（导入名/下划线）的区分。`archive/` 是旧一体化框架，已不参与新流程，遇其路径需谨慎参考。三类代码快速定位：推理 Python 包 → `python/`，C++ 算子 → `operators/`，CUDA 算子 → `cuda/`。
