本讲总览 kt-kernel 两条安装路径：`pip install kt-kernel` 的 PyPI 预编译 wheel（官方推荐、无需编译工具链），以及 `./install.sh` 源码编译（用于 AMD/ARM 后端、自定义 CUDA、可移植二进制或本机极致优化）。官方 wheel 有两大省心特性：一个包内置多个 CPU 变体（AMX/AVX512 各子集/AVX2），变体选择推迟到 import 时按本机指令集自动完成；静态 CUDA 运行时，无需装 toolkit、无 GPU 时自动禁用。源码安装分两层：根目录 install.sh 作编排者，串联 submodule→deps→sglang→kt-kernel；kt-kernel/install.sh 作构建者，含自动探测（-march=native）与 --manual 两模式。安装后用 `kt version` 与 `import kt_kernel` 双重验证，读取 `__version__`、`__cpu_variant__`（import 时动态探测）并判断 CUDA，异构推理须确认装的是 sglang-kt fork 而非官方 sglang。`__cpu_variant__` 运行时动态决定、`__version__` 优先取包元数据再回退 version.py，体现单源真相与运行时自适应设计，其检测细节留给后续 u2-l2、u2-l3。
