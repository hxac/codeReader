本讲回答 MoE 推理中 GPU 隐藏状态如何搬到 CPU 计算并与 GPU 重叠。主角是缓冲管家 KExpertsCPUBuffer 与 BaseMoEWrapper 的三个异步前向方法。关键术语：pinned memory（页锁定内存，支撑 non_blocking 异步拷贝）、双缓冲 buffer_depth=2（相邻层靠 slot=layer_idx%2 交替落不同槽避免输出覆盖）、capture_bs/temp_bs（按批大小长期或临时缓存缓冲组）、CUDA 流与 cudaLaunchHostFunc（把 CPU 函数排进流，统一流顺序）。核心结论：submit_forward 非阻塞，先用 non_blocking 把数据拷到 pinned 缓冲，再用 submit_with_cuda_stream 把 CPU 前向挂流上立即返回；sync_forward 用 cudaLaunchHostFunc 把同步点排进同一条流，靠 allow_pending 容忍延迟专家未完成即放行，使 CPU/GPU 重叠；forward 是 submit+sync 的同步快捷写法。延迟专家的选择策略与流水线收益留待后续讲义。
