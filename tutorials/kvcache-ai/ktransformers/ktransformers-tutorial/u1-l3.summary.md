本讲聚焦仓库根目录的「门面」层，解释 `pip install ktransformers` 真正安装的是什么。核心结论：顶层 `ktransformers` 是一个 shim（垫片）包，自身几乎不含逻辑，通过 `setup.py` 的 `install_requires=[f"kt-kernel=={_v}"]` 将安装请求转发给真正的运行时 `kt-kernel`，二者以严格相等的版本号锁死对齐。讲义覆盖三个最小模块：shim 模块 `ktransformers.py` 提供 `__version__`（优先读 `importlib.metadata`，回退读 `version.py`）与惰性探测函数 `has_sft_support()`（尝试 `import kt_kernel.sft`，成功即代表 SFT 可用）；extras 依赖中 `[sft]` 拉取 `transformers-kt`、`accelerate-kt`，`[sglang]` 拉取 `sglang-kt`；版本管理遵循单源真相，全仓库仅 `version.py` 一处定义版本号，被顶层 `setup.py`、`ktransformers.py` 与 `kt-kernel/setup.py` 三处共享读取。关键术语：shim 包、发布名与导入名（`kt-kernel`/`kt_kernel`）、extras、`importlib.metadata`、单源真相、`py-modules`。
