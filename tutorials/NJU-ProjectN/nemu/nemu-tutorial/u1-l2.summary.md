本讲讲解 NEMU 如何从配置变成可执行二进制，建立「Kconfig 描述配置、Makefile 驱动构建」的两段式体系。引入术语：配置项（CONFIG_* 开关，取 y/n 或字符串）、配置产物。核心模块：Kconfig 用 choice/config/depends on/source 描述 ISA、引擎、运行模式、构建目标四组互斥开关及调试选项；make menuconfig 经 mconf 交互、conf --syncconfig 同步，产出 .config（人读）、auto.conf（Make 用）、autoconf.h（C 用）三份文件，一份配置同时驱动构建与源码；Makefile 用 remove_quote 提取 GUEST_ISA/ENGINE 拼出二进制名 NAME 并注入 -D__GUEST_ISA__，filelist 机制以 DIRS-/SRCS-$(CONFIG_*) 收集源文件、filter-out 黑名单细粒度排除（如 AM 模式排除 sdb/alarm）；build.mk 用模式规则编译链接，产物按 NAME 隔离，fixdep 保证改配置后增量重编，native.mk 包装并提供 run/gdb。
