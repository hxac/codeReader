本讲精读 `src/cpu/cpu-exec.c`，拆解 NEMU 的 CPU 执行主循环，聚焦「驱动与调度」这一维度，把单条指令当作黑盒（其内部译码留给后续讲义）。核心内容是 `cpu_exec → execute → exec_once` 三层调用链：`cpu_exec(n)` 是引擎唯一入口，`n` 为 `uint64_t`，`c` 命令传 `-1` 即「跑到停」；它承担状态守卫（拦截已终止态）与收尾归类（打印 `HIT GOOD/BAD TRAP`）。引入五种执行状态 `NEMU_RUNNING/STOP/END/ABORT/QUIT` 及承载它们的 `nemu_state` 结构体（含 `halt_pc`、`halt_ret`）。`execute` 循环 n 次，每步依次单步执行、`g_nr_guest_inst++`（客机指令计数唯一累加点）、`trace_and_difftest`、状态检查 `break`、`device_update`。`exec_once` 是 ISA 无关骨架，经 `isa_exec_once(s)` 委托 ISA 实现，并通过三个 PC——`pc`（当前地址）、`snpc`（顺序下一地址，兼记指令长度 `ilen = snpc - pc`）、`dnpc`（动态下一地址，跳转目标）——完成流转，最后 `cpu.pc = dnpc` 提交。`trace_and_difftest` 是每步一次的三合一钩子，集中了 itrace 写日志、`g_print_step` 屏幕打印、`difftest_step` 差分比对三个正交插入点；其中 `MAX_INST_TO_PRINT`（=10）经 `g_print_step = (n < 10)` 控制 `si` 单步打印以避免 `c` 刷屏。`statistic` 在终止态打印 host 耗时、客机指令数与模拟频率。关键术语 `Decode` 结构体（栈上复用，承载解码中间信息）、`ITRACE_COND`、`device_update` 节流等供后续讲义承接。
