承接 u2-l6 词法分析，本讲回答「给定 token 流如何算出值」：用递归下降做分治求值。`eval(p, q)` 在区间上识别叶子，或找主运算符切成左右两段递归求值后再做该步运算。围绕四个最小模块展开：一是递归下降 `eval`，叶子含 `strtoul` 解析十/十六进制数、`isa_reg_str2val` 查寄存器、`vaddr_read` 做内存解引用及一元负号；二是 `check_parentheses`，用计数器判定区间是否被一对外层括号整体包围并校验配平；三是主运算符查找，遵循「深度为 0、优先级最低、同优先级取最右」以保左结合，并以「前驱是否为值」区分一元/二元的 `-` 与 `*`；四是 `gen-expr` 差分测试，以 gcc 为参考实现，随机生成合法 C 表达式逐条比对 `unsigned` 结果，高效暴露优先级与结合性 bug。求值统一用 `word_t`（riscv32 下即 `uint32_t`），除零等错误经出参 `success` 报告。成果落地为 SDB 的 `p EXPR` 命令，并为 u2-l8 监视点「每步重算表达式」奠定基石。
