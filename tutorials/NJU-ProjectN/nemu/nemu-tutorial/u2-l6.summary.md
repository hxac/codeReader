本讲为 SDB 引入表达式求值的第一步——词法分析（lexing）：把用户输入的无结构字符串切成一串带类型的 token（记号），每类 token 至少记录类型（type）与字面值（lexeme）。本讲只做「切词」，不做求值，求值（递归下降）留待下一讲 u2-l7。

内容集中在一个文件 src/monitor/sdb/expr.c，按数据流向拆为四个最小模块：Token 结构与类型枚举、rules 规则表、init_regex 预编译、make_token 主匹配循环。核心设计要点：其一，token 类型编码上，单字符运算符（如 `+` `(`）直接复用 ASCII 码作类型值，多字符类别（TK_NOTYPE、TK_EQ、TK_DECIMAL、TK_HEX、TK_REG 等）从 256 起自定义枚举，二者天然不冲突，便于后续统一 switch 求值。其二，表驱动设计：rules[] 把「正则字符串 + token 类型」写成数据，扫描逻辑通用不变，新增 token 只需「加表项 + 加 case」，ARRLEN 宏自动计算条目数。其三，规则顺序即优先级，regexec 自上而下取首个匹配，故十六进制 0x… 规则必须写在十进制 [0-9]+ 之前。其四，init_regex 在启动时用 regcomp 将每条正则编译为 regex_t 一次复用，正则写错立即 panic。其五，make_token 通过传入 e+position 并校验 regmatch_t 的 rm_so==0 实现锚定匹配，TK_NOTYPE 空白丢弃，其余记入 tokens[]，匹配失败时打印 ^ 指示并返回 false。这些 token 流是后续递归下降求值、isa_reg_str2val 寄存器解析与 vaddr_read 内存解引用的输入基础。
