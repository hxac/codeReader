本讲讲解 NEMU 的差分测试机制：以可信模拟器（REF，如 spike/QEMU/KVM）为参考答案，让被测的 NEMU（DUT）与之同跑一份程序，每条指令后比对寄存器，不一致即定位 bug。NEMU 用 dlopen 加载编译成动态库的 REF，靠 dlsym 取函数指针驱动，从而解耦、可换 REF。init_difftest 负责加载 REF 并同步镜像与寄存器初值（顺序 init→memcpy→regcpy，最后必须 regcpy）；difftest_step 每步让 REF 单步执行、取回寄存器交 checkregs 与 ISA 侧 isa_difftest_checkregs 比对，失败则 NEMU_ABORT。两类对齐问题：difftest_skip_ref 处理 REF 无法复现的行为（MMIO 设备副作用、nemu_trap、invalid_inst），把 DUT 寄存器拷给 REF 并跳过比对；difftest_skip_dut 用 skip_dut_nr_inst 应对 QEMU 指令打包（以 ref.pc==npc 判定追上）。REF 须导出 5 个 __EXPORT 函数，DIFFTEST_REG_SIZE 须与 CPU_state 二进制布局一致（riscv32 为 132 字节），构建由 difftest.mk 产出 .so 并经 --diff= 传入，开关在 Kconfig。
