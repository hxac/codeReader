本讲是整套手册的第一篇，先把 perf-book 读「对」。它的正式书名是 *The Rust Performance Book*（《Rust 性能手册》），由 Nicholas Nethercote 主笔、2020 年 11 月首发，托管于 GitHub 仓库 `nnethercote/perf-book`。其特殊性在于「源码即 Markdown」：它是一本用 **mdBook** 把 `src/` 下的 Markdown 渲染成网页的在线技术书，而非可运行库；故「读源码」即「读书稿」。本书明确面向**中高级 Rust 用户**，对初学者是「无益的干扰」，这提示后续讲义默认读者已具备相当 Rust 经验。它关心的**四个性能维度**为运行速度、内存使用、二进制体积（均为运行期）与编译时间（构建期）。写作取向上**故意简洁、重广度轻深度（breadth over depth）**，便于快读、深度靠外链，且**不是通用 profiling/优化指南**，内容偏编译器开发、远离科学计算。`README.md` 提供「查看/构建/预览/测试」四类操作信息（`mdbook build`/`serve`/`test`）以及「偏好 issue 而非 PR、拒收生成式 AI 内容」的态度声明；`src/introduction.md` 则提供范围、风格与读者定位。掌握「定位—范围—风格」这套参照系，是后续构建运行、基准测试与代码优化讲义的前提。
