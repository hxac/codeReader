本讲解决「如何把 perf-book 跑起来」。最小模块为四环：安装、命令、配置、自动化。关键术语：mdBook 是把 Markdown 渲染成在线书的 Rust 外部工具（独立可执行程序，非仓库内容），用 `cargo install mdbook` 安装，命令名为全小写 `mdbook`。三条核心命令分工为：`mdbook build` 读 `src/` 生成静态站点到 `book/`；`mdbook serve` 起本地服务器（`localhost:3000`）并监听文件自动热更新；`mdbook test` 抽取并编译书稿代码块以验证 Rust 示例正确。`book.toml` 是总配置：`[book]` 定书名/作者/源目录、`[build]` 设 `create-missing=false`（缺失文件报错而非补建）、`[rust]` 定 edition、`[output.html]` 管主题与编辑链接。CI（`.github/workflows/ci.yml`）用与本地相同的 `mdbook build/test` 把关，仅在 push 到 master 且为本仓库时部署到 GitHub Pages。核心结论：「运行本书」是构建 Markdown 书而非运行 Rust 程序，书中的性能技巧是内容而非待跑代码。后续讲义将进入 `SUMMARY.md` 章节组织与正式性能优化工作流。
