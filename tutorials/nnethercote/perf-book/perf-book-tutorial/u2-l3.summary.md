本讲覆盖 perf-book 第 3 章「Build Configuration」。核心结论：构建配置能不改代码大幅改变性能，但每个选项都在「编译时间 / 运行速度 / 内存 / 二进制体积 / 可调试性 / 可剖析性」这一特征向量上权衡，无全优解。

最小模块四块：① dev 与 release 两类 profile，用 release 是最易被忽视却收益最高的单项改动（相对 dev 常达 10–100× 加速）；② 提速四件套 codegen-units=1、LTO、`-C target-cpu=native`、PGO（插桩采集→二次编译），代价多为更慢编译；③ 缩体积三件套 opt-level="z"/"s"、panic="abort"、strip="symbols"，其中 codegen-units=1 与 lto="fat" 兼提速与缩体积；④ 经 `#[global_allocator]` 替换全局分配器（jemalloc/mimalloc）影响运行期，换链接器（lld/mold/wild）只影响编译时间，是本章唯一几乎无代价的改动。关键术语：profile、特征向量、codegen-units、LTO（false≠"off"）、PGO、opt-level、panic=abort、strip、global_allocator、链接器。铁律：所有改动每次只改一项再测量，把书里的推荐当候选清单、由实测定去留。
