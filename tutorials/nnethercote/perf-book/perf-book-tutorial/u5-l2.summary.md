本讲精读 perf-book 的 I/O 章，含四个最小模块。

锁：`println!` 每次都锁 stdout，循环里反复调用会累积锁开销；改为循环前 `stdout().lock()`、循环内 `writeln!(lock, ...)`，把 n 次锁摊还成一次，stdin/stderr 同理。

缓冲：Rust 文件 I/O 默认不缓冲，`BufReader`/`BufWriter` 在内存攒批以最小化系统调用次数；读写存在非对称性——写两侧都实现 `Write`、加缓冲几乎不改代码故易被忘，读一侧须从 `Read` 切到 `BufRead`、代码会变样故很少漏。显式 `flush()?` 的价值在暴露刷盘错误（drop 时本会自动 flush）。

逐行读的分配取舍：`BufRead::lines()` 每行分配一个 `String`；用 workhorse `String` + `read_line` + `clear()`（承接 u3-l2）可把分配压到「至多几次、甚至一次」，前提是循环体能吃 `&str`。

按字节读：读入 `String` 带 UTF-8 校验开销，只关心字节值（如处理 ASCII）时可用 `read_until` 或 `bstr`/`linereader` 按原始字节跳过校验。

核心结论：锁与缓冲是正交的两个维度、可组合（`stdout().lock()` 再包一层 `BufWriter`，同时省锁、省系统调用）；一切写法改动只是候选优化，收益依赖负载、须靠基准测试确认。
