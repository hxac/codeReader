本讲聚焦「改源码」缩短 Rust 编译时间，与 u2-l3 的「构建配置」路线互补。核心直觉：编译时间大头是 rustc 前端把 crate 翻译成 IR、以及 LLVM 后端优化 IR，优化即让这两块工作量变小。

引入的关键术语：crate（编译单元）、IR（中间表示）、单态化 monomorphization（泛型按每种具体类型各生成一份代码）、声明宏与过程宏（编译期展开成真实代码）。

四个最小模块：(1) `cargo build --timings` 生成甘特图，诊断 crate 依赖图的并行度，定位「串行化编译」、应拆分的大 crate；(2) nightly `-Zmacro-stats` 按宏列出代码生成量，过程宏最值得关注，生成量与手写代码相当者应删、换或改；(3) `cargo llvm-lines` 按函数列出 LLVM IR 行数，泛型函数因单态化被实例化几十上百次而最易上榜；(4) 核心优化 `std::fs::read` 模式——把泛型函数里类型无关的逻辑抽成只实例化一次的非泛型 inner fn，使 N 份重复塌缩成一份。

核心结论：这类改动收益「通常很小、偶尔很大」，还能顺带减小二进制体积，一切以前后测量（`--timings` / `llvm-lines`）为准，遵循先测量再优化的纪律。
