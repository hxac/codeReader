本讲承接 u1-l2，把视角从「本地能跑」提升到「书稿正确、自动部署、可扩展」，覆盖四个最小模块。一是 mdbook test：复用 rustdoc 的文档测试（doctest）能力，抽全书 Rust 代码块编译运行，把「书」变成「可测试文档」；`rust` 块被测试，`rust,ignore` 显式跳过，`# ` 开头的行渲染时隐藏但仍参与编译。二是 CI 流水线（GitHub Actions，ci.yml）：单 job 走 checkout → 安装 mdBook → `mdbook build`（结构闸门，create-missing=false 守幽灵条目）→ `mdbook test`（正确性闸门）→ deploy，且与本地命令同构。三是 deploy 的三重条件门控（`push` + `refs/heads/master` + 仓库为 `nnethercote/perf-book`），把发布严格限定在正本仓库的 master 推送，PR 与 fork 只验证不部署。四是 mdBook 的可插拔管线：preprocessor（预处理）→ renderer/output backend（HTML/EPUB/PDF 等输出后端）→ theme（主题）；perf-book 极简，仅启用内置 HTML renderer，被注释的 `[output.epub]` 展示了启用新后端所需的三处联动改动。关键术语：CI/CD、GitHub Actions、文档测试、条件门控、preprocessor/renderer/theme、site-url。核心结论：源码即 Markdown，build/test 守正确性，发布动作默认关闭、显式开启。
