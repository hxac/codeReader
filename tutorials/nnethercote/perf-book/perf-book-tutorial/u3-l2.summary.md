本讲承接 u3-l1，聚焦最常见的分配大户 `Vec`，建立「测量 → 决策 → 优化 → 防回归」闭环，四个最小模块层层递进。第一，准倍增长（quasi-doubling）：标准库容量序列为 0→4→8→16→32……，0 直接跳到 4 是为了在真实程序中少分配；翻倍让 `push` 摊还 O(1)，但带来「重配频率指数下降、浪费容量指数上升」的权衡。诊断手段是在热点 `push` 处用 `eprintln!` 打印长度、用 `counts` 出长度分布。第二，已知规模时用 `Vec::with_capacity`/`reserve`/`reserve_exact` 一次命中容量，跳过中间扩容；`shrink_to_fit` 可省内存但自身可能触发重配。第三，大量短向量场景引入 `SmallVec<[T; N]>`（前 N 个内联、超出才上堆，降低分配率但不保证更快）与 `ArrayVec<T, N>`（精确知上限时永不上堆、略快）。第四，循环里反复用完即丢的场景，引入 workhorse 集合 + `clear()` 复用容量（按行读文件用 `read_line` 而非 `lines()`）。最后用 `dhat-rs` 的 heap usage testing 把分配量写成回归断言，使分配回归成为 CI 可拦截的硬错误。关键术语：准倍增长、摊还 O(1)、`with_capacity`、`SmallVec`/`ArrayVec`、workhorse 集合、`clear()` 保留容量、heap usage testing。
