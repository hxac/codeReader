本讲承接协同仿真，回答海量压测程序从何而来：Nyuzi 采用约束随机（constrained random）自动生成汇编，核心是「随机求数量、约束求质量」。三类关键约束贯穿始终——前向短分支（只跳 1–6 条、末尾 nop 兜底，杜绝死循环且避免流水线冲刷掩盖冒险，占比仅约 1%）；固定内存指针寄存器配合每线程私有写段（地址始终合法，密集触发缓存命中/缺失/别名写回，并回避模拟器不建模 store buffer 的盲区）；小寄存器池（运算仅用 s3–s8，依 P_RAW=1/n 将相邻指令 RAW 冒险命中率放大约 5 倍）。指令按累积概率派发，二元算术约 50%、访存约 20%。另一条验证线是 csmith：生成随机 C 程序，比对宿主 gcc 与 Nyuzi 模拟器 checksum 验证编译器，因 32/64 位字宽不匹配须禁用 long long 与 packed struct，仍存假阴性。二者构成「汇编验硬件 RTL、csmith 验编译器」的双层随机验证结构。
