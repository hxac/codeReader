本讲分三模块：LL/SC 语义、同步状态（store 队列与 L2 协作）、membar 排序。`load_sync`（LL）读值并在 L2 登记缓存行，`store_sync`（SC）条件写回：成功写 1、失败写 0 且不动内存；二者同属 `MEM_SYNC` 访存类型、靠 [29] 位区分 load/store。成功判定为「链接有效且行号相等」，监视粒度为 64 字节缓存行，副作用是伪共享。链接状态存于共享 L2、每线程一个 `{core,id}` 槽，因唯有 L2 能观察全部核的写而成为裁决冲突的唯一正确位置。硬件用「两遍协议」实现：L1 将同步访存当 cache miss 发往 L2，第一遍登记/裁决并挂起线程，响应唤醒后第二遍重发同一条指令取结果，靠 `dd_load_sync_pending` 与 `restarted_sync_request` 区分两遍；`l1_store_queue` 每线程一条表项，负责入队、发送、锁存 `sync_success`、唤醒。反活锁设计令失败的 `store_sync` 不失效链接。`membar`（`CACHE_MEMBAR`）复用 store 回滚机制，等此前所有 store 落地、自身不入队，以最小硬件代价建立内存可见性顺序。
