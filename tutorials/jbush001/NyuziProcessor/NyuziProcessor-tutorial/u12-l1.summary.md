本讲解析 Nyuzi 微内核 software/kernel 的三大机制。启动加载：线程 0 在物理地址建页表、登记陷阱向量 trap_entry 与 tlb_miss_handler，用 eret 当跳板同时打开 MMU、跳到高虚拟地址并切内核栈进入 kernel_main，再由 exec_program 把用户 ELF 装入独立地址空间、经 jump_to_user_mode 弹入 user 模式。陷阱入口：trap_entry 是唯一向量，借 CR_SCRATCHPAD0/1 解决保存寄存器前的鸡生蛋问题，按来源 user/supervisor 决定是否切内核栈，存标量寄存器（不存向量）成陷阱帧，调 C 派发器 handle_trap 后用 eret 原子返回。系统调用派发：syscall imm 触发 TT_SYSCALL，立即数进 CR_SYSCALL_INDEX，handle_syscall 用 switch 分派；参数与返回值借 s0..s5 零拷贝、返回时 frame->pc+=4 跳过指令；user_copy 借 fault_handler 把缺页转成 -EFAULT，防止用户非法指针拖垮内核。贯穿全讲的 eret 兼做陷阱返回与模式切换跳板，handle_trap 按 CR_TRAP_CAUSE&0xf 分派缺页、系统调用、中断或 bad_fault。
