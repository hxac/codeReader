本讲（u2-l2，ISA 第二讲）讲解 Nyuzi 寄存器中的数如何被「算」出来，核心是贯穿硬件、模拟器、工具链三方的 6 位操作编码表 `alu_op_t`（对应硬件 `defines.svh`、模拟器 `instruction-set.h` 的 `arithmetic_op`、LLVM 工具链，数值同构以支持协同仿真）。覆盖四个最小模块：整数算术、逻辑与移位、比较操作、浮点算术。

整数算术（`add_i`/`sub_i`/`mull_i`/`mulh_u`/`mulh_i`）按补码自然回绕、溢出不触发异常；乘法拆成取低 32 位与取高 32 位两条指令以拼出 64 位积。逻辑与移位含按位运算、`shl`/`shr`/`ashr`（移位量取 operand2 低 5 位）、`clz`/`ctz`（查表单周期，输入为 0 时结果为 32）、`move`/`sext`；需注意一元操作作用于 operand2。

关键结论：比较指令为「set 型」，结果直接写目标寄存器且恒为 0 或 1，分有符号 `_i`、无符号 `_u`、浮点 `_f` 三套语义，编译器按 C 类型选择，无 flags 寄存器；比较以减法为基础，借助 `borrow`、`overflow==negative` 判据修正有符号关系。

另一关键概念是指令分流：单周期整数流水线 `int_execute_stage` 处理简单整数运算并对 16 通道并行实例化；而浮点加减乘（`add_f`/`sub_f`/`mul_f`）、整数乘法、`ftoi` 走独立 5 级浮点流水线——整数乘法复用浮点乘法器是反直觉点；`reciprocal` 用 ROM 查表单周期完成、仅约 6 位精度，用于软件实现除法（Nyuzi 无硬件除法）。模拟器 `processor.c` 的 `scalar_arithmetic_op` 是功能参考实现，`-v` 跟踪只输出 PC 与寄存器写回值，需结合反汇编对账。
