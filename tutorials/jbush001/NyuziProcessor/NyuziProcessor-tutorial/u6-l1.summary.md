本讲解析 Nyuzi 单核流水线的 L1 数据缓存（L1D）。它采用**虚拟索引/物理标签（VI/PT）**组织：用虚拟地址的 `set_idx` 索引标签 SRAM（与 DTLB 查询并行，省一拍），用物理 tag 比对命中。避免**别名**的根因是约束「组索引位＋行偏移位 ≤ 页偏移位（12 位）」，故 `L1D_SETS ≤ 64`，由仿真断言守护。L1D 占两拍：**标签级** `dcache_tag_stage` 算地址、查 DTLB、并行读各路 tag/valid 并响应 snoop；**数据级** `dcache_data_stage` 判命中、读数据、检测 fault（对齐/页缺失/越权）并产生缺失回滚。替换用**伪 LRU 树**（路数限 1/2/4/8，fill 优先于 access 以防活锁）。写操作经每线程 **store 队列**，支持写合并、load-after-store 旁路与 sync 等待。多核一致性靠 **snoop**：L2 广播物理地址，各核监听并更新/作废命中行，单核下也用于消除同义词（synonym）。缓存行恰为 64 字节等于向量寄存器宽度，缺失回填与线程挂起留给后续 L1-L2 接口讲义。
