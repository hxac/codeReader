本讲义讲解如何把 Nyuzi 源码从「源文件」变成「能跑的程序」，分三个最小模块：依赖安装、工具链脚本 `scripts/setup_tools.sh`、CMake 构建。依赖安装分系统包（apt-get 装 cmake、bison、emacs、SDL2 等，其中 Emacs 用于展开 verilog-mode AUTO 宏）与 Python 包（pillow），并指出新版 cmake 会破坏 LLVM 工具链构建（issue #204，需回退旧版）。`setup_tools.sh` 拉取两个 git 子模块后，分别编译安装 Verilator 与基于 LLVM 的 NyuziToolchain（`clang`、`elf2hex`、`lldb` 等），装入 `/usr/local/`。根 `CMakeLists.txt` 通过四个 `add_subdirectory`（tools/software/hardware/tests）组织构建树，并用 `NYUZI_COMPILER_ROOT` 指向工具链。核心结论：执行 `cmake . && make` 后，两件产物都落在 `bin/`——`nyuzi_emulator` 是 C 写的指令集模拟器，速度快但非周期精确，用于软件开发与调试；`nyuzi_vsim` 由 Verilator 编译 SystemVerilog RTL 而成，周期精确，含流水线与缓存，用于验证硬件。`run_emulator`/`run_verilator` 运行脚本由 `cmake/nyuzi.cmake` 在构建时自动生成。关键术语：周期精确仿真器、指令集模拟器、工具链、子模块、原地构建。
