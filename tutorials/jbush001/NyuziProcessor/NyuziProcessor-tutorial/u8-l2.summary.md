本讲讲解 Nyuzi 模拟器如何用宿主机 C 代码仿真外设。核心是内存映射 I/O（MMIO）：物理地址 ≥0xffff0000 不走内存数组，而经 processor.c 判为设备区，交给 device.c 的 read/write_device_register 用 switch 分发；设备访问只接受 32 位字（MEM_LONG），块/scatter 访问设备会崩溃。四个最小模块：帧缓冲仅凭 REG_VGA_BASE/ENABLE 取地址与开关，周期性把模拟内存用 SDL 贴成窗口，忽略 VGA 时序微码；虚拟 SD 卡以宿主文件为后端，transfer_sdmmc_byte 逐字节驱动 SPI 状态机，把命令翻译成 lseek/read/write，对软件透明；UART 的 TX 写 REG_SERIAL_OUTPUT 直接 putc 到 stdout，使 printf 落地终端，RX 经 stdin 轮询与环形缓冲喂回软件。关键约定：软件侧 registers.h 与模拟器侧 device.h 共用同一字节地址（REG_UART_TX 与 REG_SERIAL_OUTPUT 同为 0xffff0048），故同份软件在模拟器与 FPGA 行为一致；外设事件经 raise_interrupt/clear_interrupt 转为 INT_* 位掩码中断，衔接 u7-l2。整体为功能级仿真，不复刻硬件时序。
