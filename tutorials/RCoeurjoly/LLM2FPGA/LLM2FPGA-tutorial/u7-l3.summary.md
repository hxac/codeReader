本讲是手册收束篇，回答「项目下一步做什么」。前置结论来自 u6-l4：TinyStories-1M 经降级链可综合，但比目标芯片 XC7K480T（298,600 个 CLB LUT）大约 141 倍（需约 42,123,250 LUT），且该芯片已是支持的最大 FPGA，nextpnr-xilinx 因 OOM 跑不完布局布线。项目剩余三个 Task 中，Task 4（上板验证）以「设计装得下」为前提，Task 5（TinyStories 家族 scaling 分析）为 Task 6 提供基线，Task 6（资源削减）是当前主角。核心工程判断是跳过 Task 4、先做 Task 6：141 倍是数量级问题，必须结构性削减而非调参。Task 6a 将优化策略分三层：模型层（量化 int8/int4）、MLIR 层（换掉 Handshake 方言、通用 MLIR 优化）、RTL 层（eqmap、板载内存/DDR3 卸载）。其中「换掉 Handshake 方言」由 3e 报告点名为最大负担之一，同时命中握手缓冲与片上存储两个瓶颈，预期收益与难度均最高；量化从源头收窄位宽；DDR3 卸载承接 u6-l2 的 externalize。关键术语包括降级链、Handshake 方言（valid/ready 弹性数据流）、CLB LUT、141 倍超配。Task 6b 要求每条策略记录 LUT/FF/BRAM/DSP、Fmax、编译时间/峰值内存的 delta，且必须在不依赖 blackbox 或 stub 的前提下跑通全链。
