本讲承接 u5-l1，把开源后端推到可烧录比特流，核心是三段式 JSON→FASM→bit：先用 Yosys 的 synth_xilinx 把 SystemVerilog 综合成已映射到 Xilinx 7 系原语的网表 JSON（matmul 走 svFilelist 模式、TinyStories 走 modelIl 模式）；再用 nextpnr-xilinx 布局布线出 FASM；最后经 prjxray 的 fasm2frames 与 xc7frames2bit 拼成 .bit。这四段都由 runCommand 包成可缓存派生，串成一条依赖链。关键结论有二：其一，映射 JSON 是分叉点，既喂 nextpnr 做物理实现，又供下一讲 u5-l3 做资源核算；其二，只有 matmul 走得到 .bit，141 倍超配的 TinyStories 会在 nextpnr 因 OOM 失败，终点停在 utilization 报告。板级自测外壳用 SystemVerilog 常量函数在综合期把期望点积折叠成常量 816，与 PyTorch 黄金参考交叉验证，上电后自行播种输入、触发计算、比对结果并用 LED 报告 pass/fail。
