本讲第一次把 LLM2FPGA 真正「跑」起来：执行 gate 命令 `nix build .#tiny-stories-1m-baseline-float-selftest-all-memory-utilization -L`，把 TinyStories-1M 浮点基线沿降级链（PyTorch→torch-MLIR→CIRCT→SystemVerilog→Yosys RTLIL）走通到 Yosys，并在当前目录留下 `result` 符号链接，指向 `/nix/store/` 里一个以 `-utilization` 结尾的目录型派生。三个最小模块：一是 `nix build` 与 `result` 软指针机制；二是目标命名约定——五段连字符分别编码模型（tiny-stories-1m）、浮点基线（baseline-float，未量化）、自测外壳（selftest，顶层 `tiny_stories_selftest_top`）、外部化超大存储（all-memory，阈值 128 kbit 的 Handshake 存储被 blackbox）和只产报告（utilization，不出比特流）；三是三类产物 `summary.txt`（人读）/`summary.json`（脚本读）/`stat.json`（叶单元原料），由 `write_utilization_report.py` 递归统计 mapped JSON 生成。核心结论：关键指标 `clb_luts` 用量约 42,123,250，目标芯片 XC7K480T 容量仅 298,600，超配约 141 倍，装不下；且这份报告是 Yosys 估算而非 nextpnr 布局布线报告——因 nextpnr-xilinx 在该路线 OOM，且设计比芯片大 141 倍。由此引出后续方向是 Task 6 资源最小化（如 int8/int4 量化），而非 Task 4 上板。
