本讲义给出 LLM2FPGA 仓库的「目录地图」，核心结论是：目录划分即「降级流水线的站段划分」。沿 PyTorch → torch-MLIR → CIRCT → SystemVerilog → Yosys 降级链，`src/`、`TinyStories/` 是 PyTorch 前端模型与适配器；`scripts/`（含 `pipeline/`，每脚本包一站）负责前端导出与整条降级；`rtl/` 是降级产出的手写支撑（如浮点原语），区别于 `fpga/` 的上板胶水与约束；`sim/` 以 PyTorch 当黄金参考做语义验证；`nix/` 编排可缓存派生；`patches/` 修补上游工具；入口 `flake.nix` 位于根目录而非 `nix/`。关键约定：`.org` 为权威源（canonical source），`.md` 由 `nix run .#docs-md`（flake.nix 中 docs-md app，用 pandoc 转换 README.org、deliverables/、docs/ 三处）自动生成，因此永不手改 `.md`。`deliverables/` 命名遵循 `<任务号><子任务字母>-<主题>`，与 `project-plan_v2.md` 的 Task 1–6 一一对应，目前主要装 Task 1–3 产出，项目正从「证明可降级但超配」迈向「削减资源」。`LLM2FPGA-genAI-logs/` 是 git submodule，仅作日志归档，不参与编译。
