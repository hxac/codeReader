本讲是 Yosys 综合段的入口：sv_to_il.sh 用 yosys-slang 插件（先 plugin -i slang.so 再调 read_slang）把 CIRCT 导出的 SystemVerilog 读进 Yosys，落地成 RTLIL（`.il`）交下游综合。选用 slang 而非自带 read_verilog，是因为只有 slang 能完整解析 SystemVerilog（logic、always_comb 等），且 slang 与 Yosys 全开源，满足项目工具链约束。

read_slang 有两种模式，由环境变量 YOSYS_SLANG_PER_FILE_EXTERNS 切换：matmul 走单次 elaborate（一条命令带 --top main，slang 从顶层一次展开全设计，简单稳健）；tiny-stories-1m 走逐文件 extern（每文件一条 --extern-modules，把找不到定义的模块当 blackbox 占位，峰值内存可控）。逐文件模式下文件按早（`000_*` 基础声明、黑盒 stub、浮点原语）、中（其余模块）、晚（main.sv 顶层）三批读入，保证「定义先于使用」、顶层最后读。

收尾用 hierarchy -check -top main 校验层次、stat 打印过程性摘要（注意不是最终 FPGA 资源报告）、write_rtlil 落盘；--no-proc 故意跳过 proc/flatten 等重活，推给下游 synth_xilinx。run_yosys_script 通过临时关 errexit 抓退出码，区分 OOM（退出码 137/9，提示换大内存或削减模型）与普通失败（修源码），给出不同处置。
