LLM2FPGA 旨在用**全开源 EDA 工具链**把**开源 LLM**（从 TinyStories-1M 起步）部署到 FPGA 上做本地推理，填补「没有全开源方案」的空白。核心约束是流程每一步都必须开源——这否决了主流 HLS 路线（所有 HLS 方案都依赖闭源的 Vitis HLS，开源替代 Panda Bambu 兼容性差），从而选定一条**编译器链降级路线**：`PyTorch → Torch-MLIR → CIRCT → SystemVerilog → Yosys`。其中 PyTorch 模型是整条链的**唯一真相源**，用于逐阶段语义验证。关键术语包括 **lowering（降级）**、**dialect（方言）**、Handshake 弹性数据流方言、CLB LUT 等。该路线有三大预设风险：算子不被支持、降级时工具崩溃、最小 LLM 装不下。当前阶段（Task 3）证明 TinyStories-1M 能成功降级并综合，但资源**超配约 141 倍**（设计需约 4212 万 CLB LUT，目标芯片 XC7K480T 容量仅 298,600，nextpnr-xilinx 因此 OOM）。据此工程判断，下一步是 **Task 6（资源最小化）**而非 Task 4（上板），可探索方向包括更直接使用板载内存、换掉 Handshake 方言、量化等。
