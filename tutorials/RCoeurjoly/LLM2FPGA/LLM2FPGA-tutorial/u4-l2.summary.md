本讲讲解用 Verilator 仿真验证降级链生成的硬件（DUT，即 matmul 的 SystemVerilog）是否与 PyTorch 黄金参考语义等价。核心三块：一是 valid/ready 弹性数据流握手——testbench 在 start 通道（in3）作发送方，等 ready 再撤销 valid 完成单脉冲触发，输出通道则把 ready 常置 1 取结果，matmul 结果从 in2_st0 通道输出（ESI 的 store 命名）。二是层次化引用播种——因输入被内化为内部 Handshake 存储而未暴露成端口，testbench 用 dut.handshake_memory0 点分路径在复位期直接写入，属仿真特权、综合不可用。三是 flake.nix 的 Verilator 派生：simMain 用 --binary --timing 编译可执行文件（--timing 是支持延迟语句的硬性开关），matmulSvSim 跑并解析 PASS 行成 JSON（grep 不到即 exit 1，化作 CI 硬门禁），matmulSvWave 用 --trace 产 VCD 波形。PASS 行中 expected 来自 PyTorch、got 来自 DUT，二者相等即完成语义等价闭环。
