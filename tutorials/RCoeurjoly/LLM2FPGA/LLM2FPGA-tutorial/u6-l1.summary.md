本讲介绍 TinyStories-1M 自测外壳 `tiny_stories_selftest_top` 的自动生成器 `gen_tiny_stories_selftest_top.py`。它用「正则 + 方向继承状态机」把构建产物 `main.sv`（其端口表会随降级链变化，故不在 git 中）的端口表解析成 `Port(name, direction, decl, bits)` 三元组，遇到 packed struct 时位宽返回 `None` 以触发 fail-fast。随后按命名约定启发式识别四类 ESI 通道——start（编号最大的成对 `in<N>_valid/ready`）、done（编号最小的 `out<N>_valid/ready`）、store（最小 `in<N>_st0*` 前缀）、load（`in<N>_ld0_*` 四端口齐全）——分别用 `always_ff` 生成 boot 复位、start 单脉冲、load 地址请求-数据响应、store 接载荷-回 done token 五段握手，并以「done 在超时前拉起」作为降级 pass 判据（因 141 倍超配跑不完推理，不做数值比对）。核心结论：手写版 `rtl/tiny_stories_selftest_top.sv` 与生成器输出结构等价（仅 `load_pending`/`in16_pending` 命名差异），当前 `flake.nix` 实际消费手写版，生成器是「端口变化时自动重派生外壳」方向的设计凭据。
