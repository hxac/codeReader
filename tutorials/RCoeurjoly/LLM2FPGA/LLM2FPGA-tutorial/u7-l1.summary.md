本讲（u7-l1）讲如何把自有模型接入降级链，核心结论是「入口与引擎分离」：加新模型只改 `nix/models.nix`（入口）和 Python adapter，引擎 `pipeline.nix` 不动。最小知识是「两步法」：①写 adapter，须实现 `build_model(model_path)` 与 `example_inputs()`（可选 `EXPORT_STRICT`）；②在 `models.nix` 加一条 `registerModel { ... }`，内部经 `mkTorchInput`→`mkPipeline` 返回 `{ pipeline; }`。adapter 契约由 `compile-pytorch.py` 强制执行，`example_inputs` 的数量/形状/dtype 必须与 `forward` 形参一一对应。`registerModel` 三个开关按问题维度独立设置：`allowHwExterns` 与 `fpPrimsSv` 成对开（浮点 extern，服务 SV 导出站）；`slangPerFileExternModules`（设计太大导致 slang OOM，服务 Yosys 前端站）。开关经 `optionalString` 注入环境变量并进入派生 hash 以隔离缓存，matmul 全关、TinyStories 全开。可复现性的命门是 Python 环境、`torchMlirPythonPath` 拼 `PYTHONPATH`、权重快照（`fetchurl`+`linkFarm`+`local_files_only=True` 禁联网）三样的绑定。
