本讲是单元 4 首讲，只生产等价性验证基准，仿真消费侧留给下一讲。核心问题：模型降级为硬件后，凭什么相信硬件算出的数与原始 PyTorch 一致？

关键术语：黄金参考（独立于被测设计 DUT 的可信实现）、唯一真相源（PyTorch）、fail-fast（快速失败）、代码生成。

三个最小模块：其一，PyTorch 黄金参考——参考脚本与降级链适配器共用同一模型加载器，对固化输入 a、b 跑点积得标量 expected（示例 816），保证与 DUT 同源。其二，测试向量校验——对 dtype、shape、真实元素数三重校验实现 fail-fast 的纵深防御。其三，生成 tb_data.sv——用 Python 拼字符串产出存储器声明、expected 与 initial 块，全走 stdout，由 Nix 派生重定向成文件，供 testbench 经 include 消费。

结论：黄金参考须独立于 DUT，其 Nix 派生只依赖模型源码与仿真脚本、不碰降级产物，故降级链坏了参考仍正确；冒烟测试刻意选 int32，以解耦结构正确性与浮点近似误差，实现逐位严格比对。
