本讲承接 u3-l1 的「只配置不执行」查询模型，专门讲解行级过滤。过滤有两条入口：`only_if("SQL字符串")` 本地与远程表都支持，`only_if_expr(Expr)` 借助 `crate::expr` 的 `col`/`lit` 构建类型安全表达式但**仅限本地表**，二者最终都存进 `QueryRequest.filter` 的 `QueryFilter` 枚举（`Sql`/`Datafusion`/`Substrait`）。核心结论是：LanceDB 核心本身不解析 SQL，本地表把字符串原样交给 Lance 的 `scanner.filter(sql)`、远程表通过 HTTP 发给服务器，统一由底层 DataFusion 解析，因此非法 SQL 要到 `execute()` 才报错。`expr/sql.rs` 的 `expr_to_sql_string` 做反向转换（`Expr`→SQL），服务远程表的 `only_if_expr` 与 `Select::Expr` 投影，并用自定义 `LanceSqlDialect`（反引号引号、二进制字面量占位符慢路径）保证可回解析。最后，过滤与向量搜索的顺序由 `QueryRequest.prefilter` 切换：默认预过滤（准确、可凑满 `limit`，延迟略高），`postfilter()` 切后过滤（延迟低但结果数可能少于 `limit`，可用 `refine_factor` 补救）。
