本讲专讲向量索引 IVF 家族（index (vector) 模块）。IVF（倒排文件）用 k-means 把向量空间划成 num_partitions 个分区，每分区记一个质心；查询只搜离 query 最近的 nprobes 个分区，把搜索成本从 O(N) 降到约 nprobes×N/num_partitions。IVF 只管分桶，桶内存什么决定精度与体积，由此衍生四兄弟：IvfFlat 存原始向量（最准最大）、IvfSq 标量量化（约 4×）、IvfPq 乘积量化（约 32×~64×，是 Index::Auto 对向量列的默认）、IvfRq 残差量化。关键术语含质心、召回率、量化、refine 重排。参数分两组——建索引侧 num_partitions 与 PQ 的 num_sub_vectors/num_bits 固化进索引；查询侧 nprobes（默认 20）/refine_factor 每次可调，是上线后调召回的主力。重要约束：训练度量必须等于查询度量。LanceDB 核心只做参数搬运，真正的 k-means 与量化算法在 lance-index 依赖库里。
