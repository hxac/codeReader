本讲围绕官方最小示例 `simple.rs`，串起 LanceDB 的完整使用链路：connect → create_table → create_index → query → delete。核心模块与结论：连接是入口，`connect(uri)` 返回 `ConnectBuilder`，`.execute()` 按 URI 前缀分流到本地 `ListingDatabase` 或远程 `RemoteDatabase`，对外统一为 `Connection`。建表时「首批数据 Schema 即表 Schema」，向量列约定为 `FixedSizeList<Float32>`，`create_table`/`add` 均接收 `Scannable` 泛型以同时支持整块与流式写入。`Index` 枚举分标量、向量两类，`Index::Auto` 按列类型自动选型，无索引也能搜索仅退化为暴力搜索。查询由 `QueryBase`（通用配置）与 `ExecutableQuery`（执行返回流）两个 trait 组织，`nearest_to()` 把普通 `Query` 升级为 `VectorQuery` 并附加 `_distance` 打分列（默认 L2 距离）。删除区分 `delete`（行级、SQL 谓词、底层逻辑删除）与 `drop_table`（表级）。贯穿全程的是 Builder 模式：凡执行 IO 的操作都先返回 Builder，`.execute().await` 才生效。后续讲义将分别展开 Connection 配置、Table 三层抽象（Table/BaseTable/NativeTable）、Scannable 流式写入与完整检索链路。
