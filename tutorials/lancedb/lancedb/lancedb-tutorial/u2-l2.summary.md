本讲承接 u2-l1，讲解 LanceDB 的三层 Table 抽象：对外轻量句柄 Table、统一契约 trait BaseTable、本地实现 NativeTable 与远程实现 RemoteTable。

Table 持有 `Arc<dyn BaseTable>`，方法几乎都是一行转发（委托），并夹带 `database`、`embedding_registry` 等上下文，克隆仅为 Arc 引用计数、极廉价。BaseTable 用「默认实现返回 NotSupported」容纳可选能力，用 `as_any()` 配合 `downcast_ref` 支持向下转型，使契约可宽于具体实现。NativeTable 持有 `DatasetConsistencyWrapper`（封装底层 Lance `Dataset`），把契约翻译为 Lance 调用：`schema()` 将 Lance schema 转为 Arrow schema，`count_rows()` 透传 `Filter::Sql`（`Filter::Datafusion` 暂返回 NotSupported）。

关键术语：句柄、trait 对象 `Arc<dyn BaseTable>`、委托、向下转型、`DatasetConsistencyWrapper`、`Filter` 枚举（Sql/Datafusion）。核心结论是本地（函数调用直达 Lance）与远程（HTTP 往返）共用同一套 Table API、对用户完全透明；表由 `Connection::open_table` 经 `OpenTableBuilder.execute()` 由数据库后端决定生成 NativeTable 还是 RemoteTable，再包成统一的 `Table`。
