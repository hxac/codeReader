本讲义围绕 `rust/lancedb/src/connection.rs` 这一个文件，讲解 LanceDB 连接模块（connection）。核心结论：`connect(uri)` 只是入口语法糖，返回 `ConnectBuilder` 构建器；真正建连发生在 `.execute().await`，体现 LanceDB 贯穿的「构建器 + execute」风格。

引入的关键术语：`ConnectBuilder`（持有 `ConnectRequest` 与 `embedding_registry`）、`ConnectRequest`（含 `uri`、`options`、`read_consistency_interval`、`manifest_enabled`）、`Connection` 句柄——它轻量、可 Clone，内部持有 `Arc<dyn Database>`，几乎所有方法都委托给底层 trait 对象，使本地与远程后端对外接口一致。

URI 分流是连接机制关键：`execute()` 按 URI 前缀分流，以 `db` 开头走远程 `RemoteDatabase`（需 region + api_key，缺失则 `Error::InvalidInput`；未开 remote feature 用 `db://` 则 `Error::Runtime`），否则走本地/对象存储的 `ListingDatabase`，`manifest_enabled=true` 另切命名空间分支。配置方面，`api_key`/`region`/`host_override` 是写进 `request.options`（键名由 `remote/db.rs` 常量约定）的语法糖，`storage_options` 是通用键值透传通道；远程连接支持环境变量兜底，`read_consistency_interval` 控制读取新鲜度。`table_names()` 同样返回构建器 `TableNamesBuilder`。后续讲义 u2-l2 将承接 `Connection::open_table` 与 `Table` 句柄。
