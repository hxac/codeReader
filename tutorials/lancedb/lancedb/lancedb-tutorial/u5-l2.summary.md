本讲围绕 table 模块四个数据与结构修改操作。`update` 采用构建器风格，以 `only_if`（WHERE）+ `column`（SET）做条件批量改写，须至少一列；`delete` 直接接收谓词（SQL 字符串或 DataFusion Expr），删除为逻辑性、即便 0 行命中也会升版本，物理回收依赖 optimize；`schema_evolution` 含 add/alter/drop columns 与 update_field_metadata，每次提交新版本，列类型（NewColumnTransform/ColumnAlteration）直接复用 Lance，cast 能力受其限制（如 Int32→Float64 不支持）；`merge_insert` 以连接键 on 配合 when_matched/when_not_matched/when_not_matched_by_source 三种情况实现 upsert，条件用 target./source. 前缀区分新旧数据，另可选 MemWAL LSM 高吞吐写路径。核心结论：四操作共享「ensure_mutable 守卫→快照→委托 Lance→升版本」骨架，LanceDB 核心主要做参数搬运与版本回写，真正算法在 Lance crate，且本地与远程后端接口一致。关键术语：版本（version）、谓词（Predicate）、构建器（Builder）、逻辑删除、upsert、MergeResult。
