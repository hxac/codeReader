本讲只覆盖最小模块「arrow」（数据模型与接入抽象）。核心结论：LanceDB 全盘采用 Apache Arrow 作为数据模型——schema 用 arrow_schema::Schema、数据用 RecordBatch，且首批数据的 schema 即表 schema，无需单独声明。引入的关键术语有列式存储、Schema、Field、RecordBatch、FixedSizeList。向量列约定为 FixedSizeList<Float16/Float32>，维度编码进类型；LanceDB 扫描 schema 自动识别向量列（外层 FixedSizeList 且子字段为浮点），并直接从类型取出维度。通用数据接入靠两条窄抽象：同步的 RecordBatchReader/IntoArrow 与异步的 RecordBatchStream/IntoArrowStream，二者均借 blanket 实现让标准 Arrow 类型免适配，故任意 Arrow 数据源可直接写入。选择 Arrow 的根本收益是零拷贝跨语言与生态复用（DataFusion/Polars/Lance 列式对齐）。
