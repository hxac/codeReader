LanceDB 是「一核多绑定」的 Cargo workspace，成员为核心库 `rust/lancedb`、Python 绑定（PyO3，产物 `_lancedb`）、Node 绑定（napi-rs）；Java 绑定用 Maven，不在本 workspace 内。workspace 通过 `[workspace.dependencies]` 集中管理 Lance/Arrow/DataFusion 等版本，并用 git tag 锁定 Lance；绑定 crate 以 `path` + `default-features = false` 引用核心库，使其版本（如核心 0.31 与 Python 0.34）可独立演进、改核心即生效。核心 crate 声明 `default = []`，即默认不开启任何 feature。feature 分五类：存储后端（`aws`/`gcs`/`azure`/`oss`/`huggingface`/`dynamodb`）、远程后端（`remote`）、嵌入集成（`openai`/`bedrock`/`sentence-transformers`）、性能格式（`fp16kernels`/`polars`）、测试（`s3-test`）。其机制是「`optional = true` 依赖 + 跨 crate feature 透传」：如 `remote` 仅在开启时才拉入 `reqwest`/`http`，并向 `lance-namespace-impls` 传递 `rest`；`aws` 则沿依赖图把对象存储实现下推到 `lance`/`object_store`。核心结论：远程后端做成可选 feature，让纯本地用户免编译 HTTP 客户端与云 SDK；而 Python/Node 绑定默认开启 `remote`/`aws`/`gcs` 等以开箱即用。常用命令包括 `cargo check/test --features remote` 与 `cargo tree`，后者可直观对比开 feature 前后依赖差异。`[[example]]` 的 `required-features` 则限定示例编译条件。
