本讲精读 `cloud.py`，讲清 `openllm deploy` 把模型部署到 BentoCloud 的完整链路。`cloud.deploy` 仅是薄编排函数，真正执行的是 `bentoml deploy` 子进程。核心有三：其一，环境变量来源分 `bento.yaml` 声明、`os.environ`、CLI `--env` 三类，按 last-wins 拼入命令，最终优先级为 CLI > os.environ > bento.yaml；命令行 `--env` 为云端 Bento 注入运行时变量，而返回的 `EnvVars(BENTOML_HOME)` 属本地子进程环境，两条通道不可混。其二，`ensure_cloud_context` 借 `bentoml cloud current-context` 检测登录，未登录时按 `INTERACTIVE` 上下文变量分叉——交互分支用 questionary 引导 `bentoml cloud login`，非交互分支打印指引后退出。其三，`get_cloud_machine_spec` 把云端 `list-instance-types` 译成统一的 `DeploymentTarget`，`gpu_type` 经 `ACCELERATOR_SPECS` 转成带显存的 `Accelerator`，列表长度即 GPU 数。另因 `BENTOML_HOME` 被重定向到仓库内 `bento.repo.path/bentoml` 目录，必须把本地 `.yatai.yaml` 云凭证拷过去，子进程才能既找到 Bento 产物又完成云端认证。关键术语：BentoCloud、context、`.yatai.yaml`、`bentoml_tag`、`BENTOML_HOME`、last-wins、`INTERACTIVE`、`ACCELERATOR_SPECS`。
