本讲拆解 `src/openllm/common.py` 的「配置 + 上下文 + 输出」三件套，奠定 OpenLLM 的运行时状态层。其一，路径常量 `OPENLLM_HOME`（默认 `~/.openllm`，可被同名环境变量覆盖）派生出 `REPO_DIR/TEMP_DIR/VENV_DIR`，三者因写在模块顶层而「导入即建目录」，`CONFIG_FILE` 则只在 `save_config` 时落盘。其二，`Config` 是描述用户偏好（`repos` + `default_repo`）的 Pydantic 模型；`load_config` 对「文件不存在 / 合法 / 损坏」三态安全降级，`save_config` 用 `indent=2` 全量覆盖写。其三，`ContextVar` 是自制的**栈式**上下文变量：`get` 读栈顶、`set` 是压栈而非覆盖、`patch` 是「压栈+自动弹栈」的 `with` 管理器，实例化出 `VERBOSE_LEVEL`（默认 0）与 `INTERACTIVE`（默认 False）。其四，`output` 按 `level > VERBOSE_LEVEL.get()` 过滤消息，非字符串内容经 `pyaml` 美化为 YAML，详细度分档 0/10/20 与各数据模型 `tolist()` 一致。核心结论：CLI 通过 `VERBOSE_LEVEL.set(--verbose)` 这条共享变量，成为指挥层与输出层之间的通信纽带。子进程执行（`run_command` 等）留给下一讲 u2-l2。
