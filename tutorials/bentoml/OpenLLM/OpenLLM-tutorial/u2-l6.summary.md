本讲解析 venv.py：OpenLLM 为每个 Bento 在 `~/.openllm/venv` 下用 uv 单独建虚拟环境，按依赖哈希缓存复用。核心概念 VenvSpec 是环境规格模型，经归一化（依赖排序、去注释空行）后用确定性 md5 算出哈希作为目录名，实现「同一份依赖只装一次」且跨进程稳定。`_resolve_bento_venv_spec`（带 lru_cache）从 requirements.lock.txt 与 bento.yaml 读出规格并合并 `--env` 透传变量；`_ensure_venv` 用 uv venv 建环境、先单独装 bentoml 再装其余依赖，以 DONE 文件标记完整：有目录无 DONE 视为脏先删，命中缓存静默返回，失败必清理并以 typer.Exit(1) 退出。`check_venv` 复用同一哈希做只读就绪探测。公开入口 ensure_venv 串联上述步骤，把装好依赖的 venv 目录交回 local.py 去跑 bentoml serve。
