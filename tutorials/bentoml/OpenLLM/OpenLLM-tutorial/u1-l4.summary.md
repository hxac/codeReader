本讲精读 `openllm hello` 这条交互式引导命令的完整链路，不涉及真正的模型加载与服务启动。其编排固定为：更新仓库 → 打开 INTERACTIVE 开关 → 探测本机硬件 → 扫描所有 Bento → 依次选择模型名、版本、动作并执行。

涉及三类核心数据对象：`BentoInfo`（可运行的模型版本，含 tag）、`DeploymentTarget`（运行目标，本机或云端）、`Accelerator`（GPU，含显存）。核心判定函数 `can_run(bento, target)` 返回一个 float 分数，**分数大于 0 即本地可运行**；需要 GPU 但显存或数量不足时精确返回 0.0。`get_local_machine_spec` 用 `lru_cache` 缓存，对 macOS 与无 GPU 机器均有兜底，故无 GPU 也能走到选择步骤。

本讲引入的惯用手法包括：`questionary`（交互选择）与 `tabulate`（表格渲染）拼成的「表格即选项」、`_select_bento_name` 按 (repo, name) 累加分数的「乐观勾选」、由 `score` 正负驱动 run/serve/deploy 可选性（本地跑不动时仅 deploy 可选），以及各动作在 `finally` 中打印等价非交互命令、引导用户「毕业」到直接使用 `serve`/`run`/`deploy` 的设计。`CHECKED` 常量（值为 `'Yes'`）用于在表格中表示「打勾」。
