本讲精读 `repo.py`，它把「可用模型目录」外包给 git 仓库，每个模型仓库即一个含 Bento 的 git 仓库。核心机制有三：其一，`parse_repo_url` 用 `GIT_HTTP_RE`/`GIT_SSH_RE` 两条正则统一拆解 HTTP 与 SSH 两种 URL，清洗 `.git`/`@branch`，并把 `server/owner/repo/branch` 确定性映射到 `REPO_DIR` 四级本地缓存目录；非法 URL 抛 `ValueError`，供 `add` 校验。其二，`Config.repos`（`name→url` 字典，落盘 `config.json`）是登记表，`add`/`remove` 只动登记表，`update` 才删旧克隆新：`_clone_repo` 优先 `git clone --depth=1`、失败回退纯 Python 的 `dulwich`，末尾用 `glob('*/*/*/*')` 按 URL 元组清理孤儿缓存。其三，`ensure_repo_updated` 采用非对称策略——「从未更新」非交互下硬退出、「超 3 天过期」仅黄色提醒放行，由 `list_bento` 隐式调用作为新鲜度闸门。`_complete_alias` 在每次更新后把 `bento.yaml` 的 `aliases` 物化成「文件名=别名、内容=版本号」的普通文件，为下一讲 `model.py` 的 glob 扫描铺路，构成 `RepoInfo.path → BentoInfo.path → serve cwd` 链路起点。
