本讲讲清 OpenLLM 的安装入口、源码布局与运行时落盘位置。安装有两条路：用户 `pip install openllm`、贡献者克隆后 `pip install -e .`，二者都靠 `pyproject.toml` 的 `[project.scripts]`（`openllm = "openllm.__main__:app"`）生成入口命令。项目采用 src layout，构建后端 hatch/hatch-vcs 从 git 标签推导版本；`--version` 走 `importlib.metadata` 读已安装包元数据，而非构建期生成的 `_version.py`。`src/openllm/` 下九个模块「一文件一职责」，`__main__.py` 为 CLI 总入口，用 `add_typer` 挂载 `repo`/`model`/`clean` 子命令组。运行时工作目录 `OPENLLM_HOME`（默认 `~/.openllm`）下的 `repos`/`temp`/`venv` 在导入 `common` 时即创建，而 `config.json` 仅在 `repo add`/`repo remove` 调用 `save_config` 时落盘。
