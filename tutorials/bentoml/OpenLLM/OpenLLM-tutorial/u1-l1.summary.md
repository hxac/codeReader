本讲是 OpenLLM 学习手册的第一篇,只从 `README.md` 和 `pyproject.toml` 两个文件入手,覆盖三个最小模块。第一模块「项目简介与能力边界」:OpenLLM 的核心定位是把任意开源大模型「一条命令」变成对外提供 OpenAI 兼容接口的服务,对外提供 `serve`(本地起服务+Chat UI)、`run`(终端对话)、`deploy`(部署到 BentoCloud)、`hello`(交互引导)四类顶层命令,以及 `repo`/`model`/`clean` 管理子命令;关键结论是它本质是「编排者/胶水层」,自身不训练模型、不存储模型权重,运行时才拉取权重与元信息。第二模块「依赖与技术栈」:讲清五层分工——OpenLLM 是指挥官,BentoML 负责服务化、vLLM 负责推理、uv 负责备虚拟环境、OpenAI SDK 定义对话协议;另用 Typer/questionary/tabulate 构建 CLI、nvidia-ml-py 探测 GPU、dulwich 管理 git 形式的模型仓库;其中 `bentoml`、`openai` 被锁死版本体现强依赖。第三模块「模型与鉴权」:模型以「模型仓库(git 仓库,默认连官方 `bentoml/openllm-models`)」组织,可 `repo add` 扩展;权重托管在 Hugging Face,gated 模型(如 Llama 系列)需先申请访问权限并设置 `HF_TOKEN` 环境变量才能下载,云端部署则用 `--env HF_TOKEN` 透传。引入的关键术语包括:OpenAI 兼容 API、编排者、Bento、模型仓库、gated 模型、`HF_TOKEN`、`OPENLLM_HOME`(后续讲义将承接),下一讲将走进 `src/openllm` 内部模块布局与运行时目录。
