本讲讲解 hdl-modules 的 `resync` 模块，处理时钟域跨越（CDC）的基础同步构件。核心动机是亚稳态无法消除，只能靠同步器把平均无故障时间（MTBF）做到可接受——统一拓扑为「两级 `async_reg` 寄存器 + 布局到同一 slice + `dont_touch`」，并配合 `.tcl` 作用域约束（`set_max_delay`/`set_bus_skew`/`set_false_path`）限定路径延迟。四类信号各配一实体：`resync_level` 同步准静态电平，`enable_input_register` 兼具抗毛刺与确定性延迟两职，但绝不能用于脉冲（会被静默丢弃）；`resync_pulse` 用「脉冲↔电平翻转 + 边沿检测」传单拍脉冲，反馈门控保证一簇脉冲至少产出一个但不保证数量精确；`resync_counter` 用格雷码安全同步「每次只 ±1」的多比特计数器，断言「汉明距离 ≤ 1」守护此前提；`resync_cycles` 复用 `resync_counter` 实现「拍数守恒」。关键术语包括亚稳态、MTBF、`async_reg`、`dont_touch`、脉冲过载与反馈门控、格雷码与 bus skew。本讲是后续 `asynchronous_fifo`（读写指针同步）与 AXI CDC 的前置。
