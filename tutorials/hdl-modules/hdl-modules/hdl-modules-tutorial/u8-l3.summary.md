本讲解 hdl-modules 如何把资源占用做成回归：对每个有意义的 generic 组合综合出一个 netlist（网表，只跑综合、不跑实现），断言其资源数精确等于已知正确值，偏离即构建失败，把面积/时序退化当 bug 防。

核心是 module_*.py 的 get_build_projects() 钩子，它只声明一组 TsfpgaExampleVivadoNetlistProject 工程，真正综合由 tools/build_fpga.py --netlist-builds 经 tsfpga 统一执行。每个工程挂 build_result_checker，由「资源类型 + 算子」组成：TotalLuts（LUT）、Ffs（触发器）、Ramb36/Ramb18（两种块 RAM）、MaximumLogicLevel（最大逻辑级数，守时序），均配 EqualTo（精确等于）以捕捉资源缓慢爬升；资源数依赖器件，故 fifo 钉死 part=xc7z020clg400-1。

关键术语含 netlist 构建、资源回归、逻辑级数，以及 fifo_netlist_build_wrapper——只接出 barebone 裸端口的最小顶层 wrapper，使最小 FIFO 资源数干净可重复；测某端口代价时则直接用实体当顶层。FIFO 案例实证了 u4-l1 论断：enable_last 几乎免费，packet/drop/peek 计数器逐级增 FF，输出寄存器单独开近乎免费但叠加 packet_mode 时逻辑级数 6→9；异步 FIFO 开 packet/drop 反因省掉 read_level 同步而资源下降。
