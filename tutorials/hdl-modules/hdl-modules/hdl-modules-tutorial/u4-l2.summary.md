本讲把同步 FIFO 扩展为双时钟的异步 FIFO，核心模块 `asynchronous_fifo` 复用 `resync_counter`。读写指针经「二进制→格雷码→两级 async_reg 同步链→二进制」跨域，靠格雷码相邻仅差 1 位保证采样安全，故 RAM 深度必须是 2 的幂（`is_power_of_two` 断言）。跨域指针皆为陈旧值，工程上令其朝安全方向偏差：`write_level` 偏高防溢出、`read_level` 偏低防空读，即「方向性安全（directional safety）」，二者有抖动、非确定性，用户应优先用 `almost_full`/`almost_empty` 单比特标志。packet 模式不同步写地址，改同步 `num_lasts_written`（包尾计数）判整包可读，并关闭 `read_level`（恒 0），多耗一个 `resync_counter`。`fifo_wrapper` 用 `depth=0` 与 `use_asynchronous_fifo` 两个 generic，一个实体在直通/同步/异步三种模式间切换，端口集为同步、异步 FIFO 之并集。约束分工明确：指针路径用 `set_bus_skew` 加 `set_max_delay -datapath_only`（有界，保 MTBF），数据路径用 `false_path`（无界，靠指针协议保证读写不冲突、故仅 LUTRAM 实现时命中，并用 CDC-26 waiver 豁免）。
