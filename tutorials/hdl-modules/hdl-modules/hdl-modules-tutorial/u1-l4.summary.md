本讲解释 hdl-modules 如何用 Python 工具链发现 VHDL 模块并接入仿真与综合。`hdl_modules/__init__.py` 的 `get_hdl_modules()` 是对外入口，它调用 tsfpga 的 `get_modules` 扫描 `modules/` 目录得到一组 Module 对象，并以 `library_name_has_lib_suffix=False` 固化「库名即模块名」约定（关键导入延迟到函数体内，以兼容未装 tsfpga 的用户）。每个模块目录必须有一个 `module_<名>.py`，其中定义一个继承 tsfpga `BaseModule` 的 `Module` 类，文件名与类名均为硬约定。`BaseModule` 暴露两个核心钩子：仿真钩子 `setup_vunit`，由 `tools/simulate.py` 在 `add_modules` 阶段回调，用嵌套循环加 `self.add_vunit_config` 枚举出 generic 矩阵批量登记测试；综合钩子 `get_build_projects`，由 `tools/build_fpga.py` 收集工程时回调，返回 netlist 构建并用 `build_result_checkers`（如 `EqualTo`）把 LUT/FF 等资源占用纳入回归断言。两钩子由不同入口在不同时机触发、互不依赖。常用三件套为 `self.library_name`、`self.add_vunit_config`、`self.netlist_build_name`；common 模块则把两类逻辑分别拆成 `_setup_*_tests` 与 `_get_*_build_projects` 私有方法做分发。核心结论：VHDL 描述电路，`module_*.py` 描述「这些电路怎么被测、怎么被综合」。
