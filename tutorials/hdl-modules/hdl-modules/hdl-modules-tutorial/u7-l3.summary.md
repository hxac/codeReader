本讲以 `dma_axi_write_simple` 为例，打通「toml 寄存器清单 → hdl-registers 代码生成 → 手写 C++ 驱动」整条链。toml 是寄存器接口的单一信息源，用 `mode`（`r`/`w`/`r_w`/`r_wpulse`）编码「谁写给谁」；DMA 共 7 个寄存器，其中 `interrupt_status` 为 `r_wpulse`（读后写 1 清零），含 `write_done` 等 5 个位字段。hdl-registers 把 toml 投影成 4 个 VHDL 制品（寄存器包、record 包、AXI-Lite slave 壳、仿真读写包）和 3 个 C++ 制品（接口头、头、实现），生成物不入库、构建时自动生成。手写 VHDL 用 `regs_up`/`regs_down` 记录把 DMA core 与生成的 slave 壳相连；手写 `DmaNoCopy` 驱动架在生成接口之上，用 `volatile` 指针和 `(written-read)%buffer_size` 公式实现零拷贝，靠 `outstanding`（已交未用完）与 `done`（已写回 `buffer_read_address`）两游标维持不变量 `done ≤ outstanding ≤ written`。中断经 `interrupt_status` 上报，`check_status` 读后立即写 1 清零并以位掩码断言无错误，兼容中断与轮询两种工作流。
