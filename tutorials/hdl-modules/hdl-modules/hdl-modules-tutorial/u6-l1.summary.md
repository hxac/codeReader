本讲聚焦通用寄存器文件，介绍 `register_file/` 下三个构建块。`register_file_pkg` 是寄存器生态的「字典」，用 `register_mode_t` 枚举定义五种访问模式——`r`（软件读 fabric 值）、`w`（写给硬件）、`r_w`（读写回环）、`wpulse`（单拍写脉冲）、`r_wpulse`（读硬件值加写脉冲），并由 `is_read_mode` 等四个查询函数翻译为布尔属性，还定义 `register_definition_vec_t` 清单类型与地址位数算法。`axi_lite_register_file` 仅凭 `registers` 与 `default_values` 两份 generic 即参数化出完整 AXI-Lite slave：读、写各一套独立握手状态机，默认回 `SLVERR`、命中且模式匹配才 `OKAY`；通过 `regs_up`（fabric→总线）、`regs_down`（总线→fabric，只动 `utilized_width` 位）搬运数据，并以 `reg_was_read`/`reg_was_written` 访问脉冲供副作用触发，写脉冲靠「每拍先复位到 default」实现。`interrupt_register` 以「清优先、源置位、否则保持」的粘滞 `status` 配 `mask` 门控归约出单比特 `trigger`，并支持常驻源清后立即重触发的边沿语义。三者天然组合：`status`/`mask`/`clear` 恰对应 `r`/`r_w`/`wpulse` 三种寄存器，构成完整 CPU 中断接口；真实工程的寄存器清单多由 `hdl-registers` 从 toml 生成。
