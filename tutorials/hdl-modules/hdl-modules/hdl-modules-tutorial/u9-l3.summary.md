本讲是第 9 单元收尾，覆盖四个最小模块。其一，发版工程：`tools/tag_release.py` 把发版编成五步幂等脚本——`verify_new_version_number` 做干净仓库/日志非空/版本递增/tag 不重四重防呆校验，再借 tsfpga 的 `VersionNumberHandler` 改版本号、搬运 `unreleased.rst` 为发布说明、提交打 `vX.Y.Z` tag、最后 bump 回 `-dev`。版本号唯一信息源是 `hdl_modules/__init__.py` 的 `__version__`，比较用 `packaging.version.parse` 而非字符串比较。其二，lint 三件套位于 `test/lint/`，共享 tsfpga 的 `find_git_files` 只扫「已 git 跟踪文件」：版权头检五种后缀、文件格式检六项文本卫生（编码/行尾/Tab/回车/尾随空格/行宽）、Python lint 调两次 `ruff`。其三，`pyproject.toml` 采「`select=["ALL"]` 默认最严 + `ignore` 与 `per-file-ignores` 显式放行」策略，isort 自定义 `tools_pythonpath` 分区承接引导机制。其四，`contributing.rst` 规定变更日志持续维护与发版五步走，并对推 tag、命令行 fast-forward 推 main 给出带 WARNING 的人工清单。lint 随 `pytest` 进 CI，不过即 PR 不能合并。
