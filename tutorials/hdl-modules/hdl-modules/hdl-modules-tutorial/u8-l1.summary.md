本讲介绍 hdl-modules 的总线功能模型（BFM）：全部位于 `modules/bfm/sim/`、只仿真不可综合，在测试台中扮演 AXI 主/从设备以替代真实 CPU 或外设。BFM 有两条驱动路线：VC 包装路线（`axi_master`、`axi_lite_master`、`axi_slave`）把 VUnit 验证组件（VC）的标量信号桥接到项目 record 接口、逐通道挂 `axi_stream_protocol_checker` 自检，经 `bus_handle`+`net` 调用，适合单拍寄存器访问；队列驱动路线（`axi_read_master`、`axi_stream_master`）用 `queue_t`+`integer_array` 喂作业与期望数据，BFM 自行消费、发事务并比对结果，适合大规模随机突发。核心结论是随机背压决定覆盖率：`stall_configuration_t` 借 OSVVM `random_stall` 在握手前按概率插入随机 stall，经 `handshake_master`/`handshake_slave` 接到 valid/ready，逼出 ready/valid 时序角落 bug，种子由 VUnit 管理可复现且不改变事务结果；从设备把事务落到 VUnit `memory_t`，构成「同份数据既写入 memory 又作期望值」的自检闭环，测试经 `module_bfm.py` 的 generic 矩阵登记入 CI。
