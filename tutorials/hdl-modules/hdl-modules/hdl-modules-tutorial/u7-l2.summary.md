本讲剖析 dma_axi_write_simple：把 AXI-Stream 流写进 DDR 的 S2MM（流到内存映射）DMA 核心。它以资源极省、AXI 吞吐极高为目标，把包长、对齐、位宽比固化为编译期约束，故不支持 scatter-gather、窄传输或写半包。数据通路为可选 width_conversion → AXI 写状态机 → ring_buffer_write_simple 三级：状态机保证 well-behaved（有数据才发 AW、用最长突发、BREADY 常高），长包由精化期函数 get_num_axi_bursts_per_packet 切成多段满长突发，单拍包走 handshake_merger 与 handshake_splitter 特化路径，代价是每包固定 stall 一拍。buffer_start/end/written/read 四地址构成 FPGA 写、CPU 读的环形缓冲契约：硬件推进 written、CPU 推进 read，半开区间下 written 永不等于 end。核心寄存器面为 record，由顶层 dma_axi_write_simple_axi_lite 套上 hdl-registers 生成的 AXI-Lite 寄存器文件壳供实例化。write_done、write_error 与三个对齐错误经 interrupt_register 粘滞并掩码聚合成单比特 interrupt，write_done 还可经 event_aggregator 按事件数或拍数降频，降低中断率。
