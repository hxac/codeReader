`hard_fifo` 把 Xilinx UltraScale+ 的 `FIFO36E2` 硬核原语封装成 AXI-Stream 式 ready/valid 接口，与 u4-l1 的推断式 `fifo` 端口一致，可作纯数据缓冲的直接替换。核心差别在于：推断式 FIFO 用 LUT/FF 实现读写指针与满空比较等控制逻辑，而硬核 FIFO 把这些都固化在硅片内，控制逻辑几乎为零（CI 断言仅 3 LUT、1 FF、1 块 BRAM、逻辑级数 2），代价是器件绑定、粒度固定、需 unisim 才能仿真、且不暴露 `level`/`almost_*` 与 packet 一族特性。

模块分三层：`hard_fifo_pkg` 用精化期纯函数 `get_fifo_width`/`get_fifo_depth` 把任意用户位宽量化到 {4,9,18,36,72} 档、深度由 1024×36/width 公式（4 位宽特例 8192）决定；`hard_fifo`（单时钟）与 `asynchronous_hard_fifo`（双时钟）是区分单/双时钟、选择原语的薄壳；`fifo36e2_wrapper` 才是真正与原语对接的核心，完成奇偶端口拼位、16 级 SRL 写时钟域自复位、用 impure 函数按 generic 配置 `CLOCK_DOMAINS`/`REGISTER_MODE`，并用 FWFT 下的四行组合逻辑把 `full/empty/wren/rden` 翻译成 ready/valid（关键套路：`read_valid = not empty`、`write_ready = not(full or reset)`）。

异步版的关键设计取舍：**不使用** u4-l2 的 `resync_counter` 格雷码方案，CDC 完全由原语内部完成，故本模块没有 `scoped_constraints/`、无需用户侧 `set_max_delay`/`set_false_path`。这是硬核 FIFO 在异步场景最省心之处，也锁死了它只能在 Xilinx UltraScale+ 上使用。
