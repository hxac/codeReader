本讲精读 `fifo` 实体（单时钟同步 FIFO；异步 FIFO 见 u4-l2）。两侧为 AXI-Stream 式 ready/valid 握手：`write_ready='0'` 即满、`read_valid='0'` 即空，并提供 `level`/`almost_full`/`almost_empty` 状态旁路（非默认阈值会多耗逻辑）。核心是环形 RAM：读写指针用「多一位 MSB」区分满与空，`level` 用 `_next` 指针的模运算保持「永远正确」，同步 FIFO 用二进制指针即可（异步 FIFO 须改格雷码）。四个 generic 层层依赖：`enable_last`（RAM 字宽+1，近乎免费）← `enable_packet_mode`（用 `num_lasts_in_fifo` 计数实现「整包才可见」）← `enable_drop_packet`（指针回退丢包）/`enable_peek_mode`（双读指针反复读）；依赖用 `assert severity failure` 固化，不开的 `generate` 块零资源消失。`enable_output_register` 复用 `handshake_pipeline`，把寄存器吸进 BRAM 硬输出寄存器而几乎不增 FF（需 `depth=2^k+1`）。netlist 资源回归量化了每开一个 generic 的 LUT/FF 代价。
