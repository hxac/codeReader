本讲承接 u5-l1 认证体系，聚焦 genai-bench 存储侧，拆三个最小模块。一是抽象基类 BaseStorage：以读写删等六个抽象方法定义多云存储统一契约，与负责身份的 StorageAuthProvider 成对解耦。二是存储工厂 StorageFactory.create_storage：先比对 provider 与 auth.get_storage_type() 做一致性校验，再按 provider 惰性导入实现类；校验与惰性导入均分工厂层与构造器层两道，构成纵深防御。三是结果上传链路：--upload-results 开关经校验后，由 UnifiedAuthFactory.create_storage_auth 造认证、StorageFactory 造存储，upload_folder 递归遍历实验目录、按 prefix 拼装对象名搬上云端。关键术语含对象存储的桶与键、横向关注点、ABC 与工厂、惰性导入，以及 OCI 独有的三层封装（OCIObjectStorage→OSDataStore→ObjectURI），支持 namespace 寻址与大文件分片。模型认证与存储认证相互独立，故支持"用 A 家模型、存到 B 家云"的跨云组合。
