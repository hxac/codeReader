本讲讲解 `genai_bench/analysis/experiment_loader.py` 这一「纯读」加载层，负责把实验目录的 JSON 读成内存对象，不跑压测、不算指标。核心直觉是：一次实验等同于一张 scenario×concurrency 的二维网格；磁盘上有两类结构不同的 JSON——`experiment_metadata.json`（整次实验的「身份证」，对应 `ExperimentMetadata`）和单个 run 的聚合结果文件（对应 `AggregatedMetrics` 加逐请求明细 `individual_request_metrics`）。加载器用 `load_experiment_metadata` 与 `apply_filter_to_metadata` 读元数据并过滤，过滤分三种语义：不存在的键直接丢弃、`traffic_scenario` 做交集「收窄」（非全有全无）、其它标量字段做相等比较。`load_run_data` 用正则识别 run 文件并按文件级过滤跳过无关文件，把数据组织成两层嵌套字典 `scenario(str) → concurrency(int) → {aggregated_metrics, individual_request_metrics}`（类型别名 `ExperimentMetrics`）。`load_one_experiment` 负责总编排并做一致性检查：缺失场景打 warning 并从元数据移除、缺失并发档位只打 warning 但保留已有数据，最终返回干净的嵌套结构。`load_multiple_experiments` 是遍历子目录逐个加载的薄封装，用 `metadata and run_data` 剔除空壳。这层产出的索引结构正是后续 u6-l2（Excel 报告）与 u6-l4（绘图）的统一输入。
