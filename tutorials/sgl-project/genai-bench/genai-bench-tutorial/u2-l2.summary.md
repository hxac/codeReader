本讲讲解 genai-bench「场景（Scenario）」的语法与解析。任务定性、场景定量：场景用微型语言 `<前缀>(参数)[/(参数)]` 描述每个请求的输入输出规模。文本分布类 N/D/U 经 `sample()` 返回 `(输入,输出)` token 数（用 `max` 兜底下界），模态类 I/A 返回尺寸或字符数，E、R 及特殊 `dataset` 各有专属返回。三大机制：每类用 `validation_pattern` 自带正则，`Scenario.validate` 做「查表+正则」双重校验，能区分未知类型与格式不符；注册表 `_registry` 借 `__init_subclass__` 实现「定义即注册」，键为 `scenario_type.value`，只认实际存在的子类（故枚举里的 V 因无子类而非法）；`from_string` 按「取前缀→校验→查表→裁参→委派 `parse`」五步路由，与 `to_string` 互逆，共同构成场景跨进程传递的序列化协议。
