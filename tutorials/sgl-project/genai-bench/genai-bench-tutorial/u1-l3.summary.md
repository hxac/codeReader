本讲绘制 genai-bench 的目录与模块全景。仓库根由核心源码包 genai_bench/、镜像测试 tests/、文档 docs/、示例 examples/、构建与 CI 配置组成。核心包下 11 个子包沿「输入→执行→指标→产出」数据流组织：data、scenarios、sampling 组装请求；user 提供各后端 Locust 虚拟用户；metrics 计算并聚合指标；analysis 产出 Excel 与绘图；auth、storage 为横切云能力；cli、distributed、ui、logging 为基础设施；protocol.py 是全局共享的 Pydantic 数据契约。核心结论：包入口 __init__.py 首行执行 gevent monkey patch 以支撑 Locust 协作式并发；云能力经惰性导入按需开启；cli.py 顶部 import 揭示其直接依赖，而 user、scenarios、metrics 经校验回调中的 API_BACKEND_USER_MAP 及 sampling、distributed 间接接入。benchmark/excel/plot 三命令挂于 cli group，后二者在 report.py 中复用 analysis 子系统。关键术语：子包、数据流编排、直接依赖与间接依赖、惰性导入。
