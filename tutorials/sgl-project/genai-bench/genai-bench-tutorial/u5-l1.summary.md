本讲建立 genai-bench 认证体系的整体认知，最小模块为两套抽象接口与一个统一工厂。认证被拆成互不污染的两套：ModelAuthProvider（管调模型时的 HTTP 头）与 StorageAuthProvider（管给存储客户端的配置/凭据），均基于 abc.ABC 用抽象方法强制契约。唯一对外入口是统一工厂 UnifiedAuthFactory，由两个静态方法按 provider 字符串分发，未知值抛 ValueError 并列出合法取值。工厂内并存两种构造策略：aws-bedrock/azure-openai/gcp-vertex 及存储侧 aws/azure/gcp/github 直接实现新接口；openai/together/oci 走适配器路线，因其背后是继承旧基类 AuthProvider 的稳定老类。适配器精髓为「实现新接口、内部转发老对象」，本质是组合优于继承。OCI 最特殊：同一份 IAM 凭据被模型与存储两个 adapter 分别适配成签名器视图与完整 provider 视图，即「一份凭据，两种视图」。CLI 经 auth_backend_map 别名归一化后调用工厂，调用点为 benchmark 主流程与上传结果。
