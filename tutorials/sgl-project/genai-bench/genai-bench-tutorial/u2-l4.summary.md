本讲聚焦 genai-bench 的采样器（Sampler）层，负责把「场景」与「数据集」两类素材装配成可发送的 `UserRequest`，是连接 u2-l1～l3 各零件的关键一步。覆盖三个最小模块：一是 `Sampler` 基类的注册机制——用类级注册表 `modality_registry` 配合 `__init_subclass__` 实现「定义即注册」，再由 `create(task)` 工厂按 `<input>-to-<output>` 任务字符串拆分、查表、`supports_task` 校验后实例化；输入模态决定采样器类，输出模态作为运行时参数。二是 `TextSampler.sample()` 按 `output_modality` 分发到 chat/embeddings/rerank/image/speech 五个私有方法，并贯穿「场景模式」（由 `scenario.sample()` 取目标 token 数、`ignore_eos=True` 强制生成到 max_tokens 测吞吐）与「数据集模式」（直接取原文、`ignore_eos=False` 自然停止）的二分。三是 prefix 缓存机制：`--prefix-len` 全局生成一次并复用以最大化 KV-cache 命中，`--prefix-ratio` 按请求动态生成；两者均用确定种子保证多 worker 一致，并在前缀与后缀间插入随机分隔符以稳定 token 边界，切换场景时需调用 `reset_prefix_cache()`。核心结论：场景负责定量、数据集提供素材、采样器负责揉合成请求对象。引入的关键术语包括 `modality_registry`、`create()` 工厂、`output_modality`、场景模式/数据集模式、`ignore_eos`、`_shared_prefix` 及 prefix 缓存；产出物为 `UserChatRequest` 等 `UserRequest` 子类，交由后续 u3-l1 的 Locust 用户发送。
