本讲聚焦 genai-bench 的全局数据契约 protocol.py：它以 Pydantic v2 定义一组数据模型，让采样器、User、指标、分析等本不相干的子系统，靠同一套类交换数据，实现「字段即文档、构造即校验、对象可无损序列化」。核心结论：协议即契约，一个模块的输出天然是另一个模块的输入。内容覆盖三个最小模块：①请求模型族，以 UserRequest 为基类，按聊天、嵌入、重排、文生图等任务派生子类，各追加专属字段，采样器是生产者；②响应模型族，以 UserResponse 为基类，承载 status_code 与 start_time、time_at_first_token、end_time 三元时间戳及 token 计数，是 TTFT、吞吐等指标计算的输入；③ExperimentMetadata，一整次实验的宏观「身份证」，用 conint(ge=1) 约束并发、Literal 枚举迭代类型，经 model_dump_json 写出、构造读回形成闭环。指标如何计算留待 U4。
