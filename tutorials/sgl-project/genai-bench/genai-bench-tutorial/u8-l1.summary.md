本讲是 capstone 顶点讲义，把 genai-bench 各子系统缝合为一条完整数据流。核心是把 `cli.py` 的 `benchmark` 函数拆成七段流水线：初始化 UI/日志 → 认证 → 数据与采样 → 实验目录与元数据 → 环境与分布式 → 双层循环 → 报告 → 可选上传。关键结论是「磁盘即检查点」：元数据在发请求前先写盘，报告阶段用 `load_one_experiment` 从磁盘重读，使 `excel`/`plot` 子命令能对任意旧实验重出报告。心脏是 scenario × iteration 双层循环，每个 [场景, 并发/batch] 组合即一次 run（`total_runs = 场景数 × 档位数`），经 start → `manage_run_time`（到时或到量）→ stop → aggregate → save → clear 闭环。引入术语：run、iteration_type、`get_run_params`（batch 模式并发恒为 1）、别名归一化认证、worker 提前 return 的三态落地。
