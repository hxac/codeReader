本讲解析 genai-bench 的 CLI 入口与命令体系。`genai-bench` 命令由 `pyproject.toml` 的 `[project.scripts]` 入口点映射到 `genai_bench.cli.cli:cli`；该函数经 `@click.group()` 装饰成为命令组，只承载 `--version` 等公共选项、负责分发，函数体为空。group 下挂三个子命令：`benchmark`（压测主流程，定义于 cli.py）、`excel` 与 `plot`（出 Excel 与绘图，定义于 report.py），三者均在文件末尾经 `cli.add_command()` 注册。引入的关键术语：entry point 入口点、click 的 group/command/option 与 pass_context、选项组装饰器（把海量参数按主题分组，click 按选项名以关键字参数传入、顺序无关）、`ctx.obj`（跨回调数据口袋，校验回调 validate_api_backend 据此把选中的 user_class/user_task 传给 benchmark 函数体）。核心结论：benchmark 依赖 cli 层的选项组与校验，excel/plot 则是 analysis 子系统的薄封装——仅用 loader 读结果，再调 create_workbook 或 plot_experiment_data_flexible 出报告，可对任意旧实验重出而不必重跑压测。
