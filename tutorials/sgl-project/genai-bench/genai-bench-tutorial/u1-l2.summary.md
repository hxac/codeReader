本讲覆盖「安装→跑第一个基准→看产出物」最短路径。安装上，genai-bench 是标准 Python 包（需 3.10–3.12），命令经 pyproject.toml 的 [project.scripts] 入口点注册（无 __main__.py）；云厂商能力靠 [aws]/[azure]/[gcp]/[multi-cloud] 可选依赖按需开启。benchmark 子命令挂在 cli 组下，README 快速开始含占位符且漏掉必填的 --model-tokenizer；默认行为有「惊喜」：不指定场景/并发时 text-to-text 实跑 5×9=45 次 run（total_runs=场景数×并发档位数），--max-time-per-run 单位是分钟。建议显式收窄为 1 场景×1 并发以快速试跑。产出物分阶段落盘：发请求前先写 experiment_metadata.json，再按 run 写 JSON（场景串经 sanitize_string 清洗，如 D(100,100)→D100_100），最后生成 _summary.xlsx 与 plot PNG；run 异常时留 debug_for_run_*.json。实验目录默认在当前工作目录，命名含 {api_backend}_{task}_{model}_{timestamp}。
