`OpenAIUser` 是 OpenAI 兼容协议后端的实现类，用 `supported_tasks` 字典将 chat、embeddings、rerank、images、speech 五类任务映射到方法与固定 endpoint（chat 与 speech 走流式，其余非流式）。每个任务方法只负责组装 payload 并委托统一的发送中枢 `send_request`；后者通过 `parse_strategy` 回调把"发"与"解析"解耦，四个 `parse_*_response` 靠占位参数对齐签名，并用异常兜底把网络错误翻译成带状态码的 `UserResponse`。核心解析器 `parse_chat_response` 按 SSE 协议逐行处理：剥 `data:` 前缀、遇 `[DONE]` 结束、从 `delta.content` 累积 `generated_text`，并在首个 content chunk 处记录 `time_at_first_token`（即 TTFT 的来源），还兼容两种末尾 usage 格式与"流中途出错"。token 数优先取服务端 `usage`（含 `reasoning_tokens`），缺失时用 `sampler.get_token_length` 做 tokenizer 回退估算，并以 `warning_once` 去重告警；推理模型的 `reasoning`/`reasoning_content` 字段单独累积为 `reasoning_tokens`。最终产出 `UserChatResponse`，含三个时间戳与 token 计数，供下游指标计算承接。
