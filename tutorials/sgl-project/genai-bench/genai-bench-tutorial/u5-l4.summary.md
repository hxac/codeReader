本讲专讲 OCI（甲骨文云）的认证与对象存储，承接 u5-l1「一份凭据两种视图」与 u5-l3「OCI 存储多一层封装」两个结论。核心模块有三：其一，OCI 收敛出四种 auth_type——user_principal（配置文件+API Key）、instance_principal（实例元数据临时凭据）、security_token（私钥+令牌文件）、instance_obo_user（OBO token+region），均继承 AuthProvider，实现 get_config 与 get_credentials，由 AuthFactory.create_oci_auth 白名单路由、惰性缓存。关键术语「signer（签名器）」：OCI 对每个请求签名而非用请求头 Bearer token，故 get_credentials 返回签名器对象而非字符串。其二，同一份 IAM 凭据经两个适配器分头适配——OCIModelAuthAdapter 返回签名器（模型端点够用，get_headers 返空、用类名嗅探推断 auth 类型），OCIStorageAuthAdapter 返回整个 provider（因构造客户端需 config+signer 双入参），即「一份凭据，两种视图」。其三，对象存储三层封装：OCIObjectStorage（实现 BaseStorage 做地址翻译）→OSDataStore（封装 ObjectStorageClient，处理 namespace 查询、128MB 分片阈值与流式下载）→ObjectURI（Pydantic 模型，解析 oci://n/.../b/.../o/... 地址）。要点：namespace 为寻址必需项、缺失则向服务查询；128MB 以上走并发分片；oci SDK 是核心依赖而非可选依赖。
