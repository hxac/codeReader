本讲是 genai-bench 学习手册的第一篇，从零建立对项目的整体认知，覆盖三个最小模块：项目定位、技术栈与依赖、版本与发布。核心结论是：genai-bench 是一个面向 LLM 服务（LLM serving system）的 token 级基准测试（benchmarking）工具，专门针对流式输出场景设计——由于 LLM 响应是流式（streaming）逐 token 吐出，仅用请求延迟无法区分"久才开口"与"开口慢"，因此需要 token 级指标，如 TTFT（首 token 延迟）、TPOT（每输出 token 耗时）、吞吐量（throughput）。项目提供三大核心能力：CLI 工具、实时 UI 仪表盘、实验分析器，外加富日志这一工程细节。它是 Python 工程，用 pyproject.toml 配合 hatchling 构建，支持 Python 3.10–3.12，核心依赖 19 个（如 locust 驱动并发、click 做 CLI、pydantic 统一数据契约、rich 做 UI、transformers 计 token）。版本号采用单一数据源设计：pyproject.toml 定义、version.py 用 importlib.metadata 运行时读取、CLI 经 --version 暴露。关键运行前提是 __init__.py 必须最先执行 gevent.monkey.patch_all()，以配合 Locust 协程式并发。本讲仅停留在根目录与包入口，子系统的目录全景留给 u1-l3。
