本讲承接 u6-l3 的声明式 PlotConfig，讲清「怎么画出来」。核心模块 FlexiblePlotGenerator 消费 PlotConfig 与 run_data_list，按 group_key 三态分发：none 每场景一图（单分组，最适配多字段图）、traffic_scenario 所有场景叠在一张图（多分组）、自定义键则在每个场景内按元数据字段（如 server_version）再分组。最关键的设计是「多字段 × 多分组」会自动降级为只画首字段的单线并告警，故多字段 preset 须配 group-key none。底层原语 plot_graph（并发轴等距化、TTFT 自动对数轴）、plot_error_rates（HTTP 状态码堆叠柱）及三个数据切片器位于 plot_report.py，其 plot_experiment_data 为待删除的 2×4 硬编码遗留 fallback。genai-bench plot 是薄封装流水线：加载配置（preset 优先于 file，否则默认）→ 自动判别单/多实验数据 → validate_plot_config_with_data 校验 → 灵活绘图，异常即回退遗留实现。
