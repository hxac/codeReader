本讲讲 Excel 报告生成，核心是总装入口 `create_workbook`：输入为已加载好的 `experiment_metadata` 与 `run_data`（一张场景×并发网格），按 `percentile`（默认 `mean`）与 `metrics_time_unit`（默认 `s`）拼出多 sheet 的 `Workbook` 并落盘，全程不碰原始 JSON、不重跑压测。关键机制是 `StatField.__getitem__` 把字符串下标转发到属性访问，使 `percentile` 能按名字贯穿所有表挑出一列统计量。一份报告含五张表，粒度由粗到细：Summary（每场景一行，靠阈值判定选「能达标的最大并发」，chat 默认 `output_inference_speed > 10`，配合 `is_within_relative_difference` 吞吐/RPS 差异 <5% 即认为到瓶颈停止上探，不达标写 `N/A`）、Appendix（每 [场景,并发] 一行的详细指标）、Experiment Metadata（元数据键值对）、Aggregated Metrics for Each Run（整轮字段+统计 JSON）、Individual Request Metrics（逐请求明细）。两道横切后处理：`_group_empty_columns` 折叠宽表里全空列并隐藏，`_rename_headers` 在 TTS 任务把 `TTFT` 改为 `TTFB`。`SCENARIO_MAP` 仅把场景字符串翻译成可读名（需手工维护，`reorder_scenarios` 让标准场景排前），当前代码不含任何定价/成本计算。`genai-bench excel` 子命令是其薄封装，`--experiment-folder`/`--excel-name` 必填，输出落在实验目录内。下一讲 u6-l3 讲声明式绘图配置作为另一条出报告路径。
