本讲拆解 genai-bench 指标数据的存储与单位换算，含三个最小模块。其一为指标数据模型分层：四层金字塔自底向上是单请求的 `RequestLevelMetrics`、单个字段统计容器 `StatField`、一组 `StatField` 组成的 `MetricStats`、整轮 run 摘要 `AggregatedMetrics`；其中 `MetricStats` 的字段名与 `RequestLevelMetrics` 的数值字段一一对应（排除 `error_*`），形成统计镜像。其二为序列化与校验：`AggregatedMetrics` 自定义 `model_dump`/`model_validate`，借 `MetricStats.to_dict`/`from_dict` 让 `stats` 在 Pydantic 对象与纯 dict 间无损往返；`RequestLevelMetrics.validate_metrics`（`mode="before"`）保证成功请求的核心指标不许为 `None`。其三为时间单位转换：`TimeUnitConverter` 用白名单 `LATENCY_FIELDS={ttft,tpot,e2e_latency,output_latency}` 只换算这四个时间量，token 数、吞吐、计数、比率原样保留；开关字符串 `metrics_time_unit` 从 CLI 经 `ExperimentMetadata` 贯穿保存、实时 UI、分析报告三个出口，报告侧无需重跑即可换单位重出。
