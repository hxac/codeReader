本讲拆解 genai-bench 实时仪表盘子系统 ui/，聚焦三个最小模块。其一，工厂 create_dashboard 读环境变量 ENABLE_UI（默认开、白名单判真），在真渲染 RichLiveDashboard 与全 no-op 的 MinimalDashboard 间二选一；两者实现同一接口、类型统一为 Dashboard，即 Null Object（空对象）模式，使调用方零分支，连 dashboard.live 也靠动态构造的 no-op 上下文管理器兼容。其二，指标面板与横向直方图每条成功请求刷新一次（数据来自 get_live_metrics 的 stats 快照），散点图则每个 run 落一点（数据来自 get_ui_scatter_plot_metrics，按 scenario 累积、reset 清空）。其三，handle_single_request 是 master 收到每条请求指标的唯一渲染入口，取时间进度与请求数进度的较大值推进当前 run 进度条，失败请求（error_code 非空）只推进度、不刷指标面板。核心结论：所有"图"都是字符画、不依赖图形库；延迟在显示层按 metrics_time_unit 临时换算，吞吐永不换算（只动延迟）；渲染只发生在 master 进程，关 UI 不影响 JSON/Excel 等实验产物。
