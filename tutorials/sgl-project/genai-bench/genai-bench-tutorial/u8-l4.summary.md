本讲收尾整本手册，按"写代码→守代码→发代码"组织三个最小模块。测试上，tests/ 镜像源码包结构，conftest.py 用 autouse fixture 在每测试后还原 OpenAIUser 类属性、清空 warning_once 缓存，杜绝跨测试状态泄漏。CI 上，ci.yml 分 test（3 版本矩阵加 93% 覆盖率门槛）、lint（ruff 加 mypy）、build（依赖前两者）三段，pre-commit 把检查前置到 commit。发布上，release.yml 发版前再跑测试，核心是 Git tag 与 pyproject.toml 的双源版本校验，通过后发 PyPI 并推多架构镜像到 ghcr.io。贯穿全程的是版本号单一数据源：pyproject 定义→写入包元数据→version.py 用 importlib.metadata 读出→发版时由 Git tag 比对，形成可信闭环。
