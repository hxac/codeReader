本讲聚焦 sglang-grpc 服务从开机到关停的工程化细节。服务引导：`run_grpc_server` 把标准库 `TcpListener` 经 `from_std`（依赖 `start_server` 设的非阻塞模式）移交 Tokio，组装 `SglangServiceImpl`，用 `serve_with_incoming_shutdown` 开机。消息大小上限：tonic 默认 4 MiB 对 LLM 场景不足，故设常量 `DEFAULT_GRPC_MAX_MESSAGE_SIZE`（64 MiB），并由环境变量 `SGLANG_TONIC_PAYLOAD` 经 `resolve_max_message_size` 覆盖——合法正数原样采纳，0、非数字、未设置则回退默认并告警。优雅关停：靠 `Arc<Notify>` 跨线程打通，服务线程 `notified().await`，Python 句柄 `GrpcServerHandle::shutdown` 须先 `notify_one()` 再 `join()`，顺序不可颠倒且幂等；`is_alive` 基于 `JoinHandle::is_finished`，只回答线程存活，就绪探测留给 health_check。
