本讲聚焦 sglang-grpc 桥接层的背压设计：核心自由函数 try_send_chunk 以非阻塞 try_send 投递响应 chunk，三分支产出三态枚举 ChunkSendStatus（Ready/Pending/Closed），使持有 GIL 的回调线程不为慢客户端等待。通道满时由 register_pending_send 占「单停泊位」并 tokio_handle.spawn 异步重发，再经 mark_send_ready/notify_ready 把「可续产」信号边沿传回 Python。关键术语：单停泊位不变量 parked(rid)≤1，借 pending_sends（HashSet）的二值语义实现，违反即关停并立 TerminalError::ChannelFull；就绪通知恰好一次，靠 ready_callbacks（回调本体）与 ready_signals（错过事件回放）双表保证「排空与注册任意时序下不丢不重」；终端 chunk（Finished/Error）排空后不再触发就绪，与生产者契约对齐。所有异常终止汇入 close_channel_with_error：清四表、立 terminal_errors 案、best-effort 通知 Python abort，消费侧再翻译为 gRPC Status。
