本讲义讲解 gRPC 请求提前结束（客户端取消、超时、abort_all）时，Rust 如何把「停止生产」信号可靠传回 Python，避免 Python 在无人消费的通道上空烧 GPU。核心模块有四：RequestAbortGuard 是 RAII guard，用 armed 布尔位把 abort 绑定到对象生命周期，disarm/abort_now/Drop 三路径共享 one-shot 语义；spawn_abort 经 spawn_blocking 把同步 GIL 调用挪出 Tokio worker，避免阻塞与死锁；PyBridge::abort 区分单请求 abort（按 was_active 决定是否立案底并调 Python）与批量 abort_all（drain+clear 四表、必调 Python）；remove_channel_refs_locked 是 channels/pending_sends/ready_callbacks/ready_signals 四表统一清理原语，故意不碰 terminal_errors，案底管理（abort 立/remove_channel 清/close_channel_with_error 改写）交给调用方。贯穿纪律是先在临界区写完 BridgeState、释锁后再持 GIL，避免锁序环；abort_all 因具跨租户 DoS 能力而触发安全告警。
