本讲聚焦一元（unary）RPC 的收尾与响应解析，含四个最小模块：一元收银台 `recv_terminal_chunk_for_request`、结构化一元 `text_embed`/`embed`、复用 embed 管线的 `classify`、JSON 控制型一元 `recv_json_response` 与 `list_models`。核心契约：一元 RPC 只期望 `Finished`/`Error` 终止 chunk，中途收到 `Data` 即协议违例，报 `INTERNAL` 并经 `RequestAbortGuard` 把取消传回 Python；该收银台被多个一元方法共用，统一处理超时、通道关闭（`closed_stream_status`）与取消传播，故调用方的 `Data` 分支仅为满足穷尽性的死代码。`classify` 传 `req_type="embed"`、用 `build_classify_dict`，因 Python 侧分类/奖励模型与 embedding 共用 `EmbeddingReqInput` 管线，结果向量落入 `embedding` 字段。JSON 一元分两型：通道型（`recv_json_response` 从终止 chunk 的 `json_bytes` 解 UTF-8 串）与同步直调型（如 `bridge.list_models()`，无通道、无取消语义）。关键模式：proto 必填字段用 `unwrap_or` 兜底，`optional` 字段用 `.get().and_then().map()` 映射到 `None`，以区分「缺失」与「空值」，`ModelCard` 四字段为范本。
