本讲讲清桥接层 Python→Rust 侧的回调机制。两个 PyO3 `#[pyclass]` 回调由工厂在持 GIL 时现场构造：`ChunkCallback` 收结构化 dict，服务原生数据型 RPC；`JsonChunkCallback` 收不透明 bytes，服务 OpenAI 透传与控制型。工厂把请求 id、共享账本 Arc、Python 与 Tokio 句柄四样能力塞进对象，再擦除成可调用的 PyObject 交给 Python。两类 `__call__` 骨架相同：按 id 定位 mpsc 通道、放锁、error 短路，按 `finished` 选 Data 或 Finished 变体，交 `try_send_chunk` 入通道。差异在提取：dict 回调容错填 text/output_ids/embedding 进 ResponseData；bytes 回调只塞 json_bytes，meta_info 仅含 status_code。核心设计是 meta_info 的每个值（含字符串）都做 JSON 编码，以适配 proto 的 map<string,string>，换来客户端统一 json.loads 无损还原类型的契约。最后介绍 set_on_ready 借 ready_callbacks、ready_signals 两表实现"晚注册也能补发"的背压就绪信号注册，完整停泊流程留待 u3-l1。
