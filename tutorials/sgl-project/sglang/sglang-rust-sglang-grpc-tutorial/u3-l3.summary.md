本讲聚焦 sglang-grpc 服务端把内部错误翻译成 gRPC Status 的错误映射层，核心函数均在 server.rs。前提是服务端错误有两个独立来源：跨 GIL 调 Python 产生的 PyErr，与桥接层响应流出错产生的 TerminalError。pyerr_to_status 采用两分法——PyValueError/PyTypeError 归 INVALID_ARGUMENT（客户端错误），其余归 INTERNAL（服务端故障），判别须在 Python::with_gil 内用 is_instance_of 完成。TerminalError 是三变体枚举（ChannelFull、ClientDisconnected、Aborted，各携带 rid），由 close_channel_with_error 先以「案底」写入 BridgeState::terminal_errors，消费侧收到 Ok(None) 后用 take_terminal_error（取出即删）取出，再由 terminal_error_status 映射：ChannelFull → RESOURCE_EXHAUSTED，ClientDisconnected/Aborted → CANCELLED。recv_chunk_with_timeout 用 tokio::time::timeout 产出 DEADLINE_EXCEEDED，超时经 abort_guard.abort_now 把取消传回 Python。closed_stream_status 对 Ok(None) 二分：有案底则翻译且 should_abort=false，无案底则兜底 INTERNAL 且 should_abort=true；返回 (Status, bool) 将「翻译错误码」与「决定是否中止 Python」解耦。各映射函数均为纯函数式、不跨 await、临界区极短。
