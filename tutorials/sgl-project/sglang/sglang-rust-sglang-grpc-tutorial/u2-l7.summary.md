本讲补上「Rust → Python」一侧：proto 请求如何被翻译成 Python `GenerateReqInput` 可用的字典。核心是一条**三段式管道**：proto 消息 → `req_dict`（`HashMap<String, serde_json::Value>`）→ Python `dict`。serde_json 中间层使 `build_*_dict` 构建函数纯 Rust、无 GIL、可单测，并把 proto 类型挡在桥接层之外；最后由 `json_map_to_pydict` 持 GIL 把草稿印成成品，与 u2-l6 的 `py_value_to_json_string`（反方向）配成双向桥。

`sampling_params_to_map` 给出字段插入三策略：`optional` 标量用 `if let Some(v)`、`repeated` 字段用 `!x.is_empty()`、`optional string` 用 `if let Some(ref v)`，原则是「只插显式设置的字段」以免污染 Python 默认值。`build_text_generate_dict` 与 `build_generate_dict` 高度同构，差异仅在输入载体（`text` ↔ `input_ids`）与 text 版独有的 `return_text_in_logprobs`，末尾皆盖 `received_time` 服务端时间戳。`insert_disaggregated_params`（三键整块）与 `trace_headers_to_json`（单键 `Option`）展示「整块条件插入」；`json_value_to_py` 的 `Number` 分支以「先 `i64` 后 `f64` 后 `None`」保证整数不丢精度。所有函数均 `pub(crate)`，经 `utils/mod.rs` 再导出，Python 端只见成品 dict。
