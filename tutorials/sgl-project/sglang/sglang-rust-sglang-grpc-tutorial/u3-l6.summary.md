本讲是 sglang-grpc 进阶层收尾篇，从"怎么跑"转向"怎么验证与扩展"，覆盖四块：测试组织、串行环境变量测试、纯函数单元测试、两类扩展实践。

测试用 `#[cfg(test)] mod tests` 紧贴实现，混用"文件分离"（`mod tests;` 解析到同名子目录）与"内联"两种写法，靠 `use super::*` 访问模块私有项。改进程级环境变量的测试（如 `resolve_max_message_size_honors_env_var`）因环境变量是全局可变状态，必须把同一变量用例串行进单个函数；edition 2024 下 `set_var`/`remove_var` 标记为 `unsafe`。被测函数刻意做成无 GIL、无 Tokio 的纯函数（`TerminalError::message`、`json_encode_string`、`build_*_dict`），常做往返解码断言，保持快而确定。

扩展点：分词器后端由 crate 私有 trait `TokenizerBackend` 加 `Box<dyn>` 多派发，`RustTokenizer` 对外仅暴露稳定接口，`load_backend` 是探测链唯一扩展位；新增 RPC 清单为 proto 与 server.rs 必改、请求字典/桥接视 RPC 类型而定、错误映射可复用、build.rs 免改（契约驱动自动重生，漏实现即编译失败）。本 crate 为 cdylib Python 扩展，单测随 sglang wheel 构建。
