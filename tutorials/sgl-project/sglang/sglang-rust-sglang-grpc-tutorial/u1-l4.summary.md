本讲义精读 `rust/sglang-grpc/src/lib.rs`，讲清 crate 对 Python 暴露的唯一启动入口 `start_server` 的完整生命周期。核心流程：解析绑定地址、参数下限归一化、标准库 `TcpListener` 同步绑端口（占用即在启动期抛 `PyRuntimeError`）、一次性持 GIL 提取分词器信息、建多线程 Tokio 运行时并构造 `PyBridge`、在独立 OS 线程 `block_on` 起 tonic 服务。关键概念：`GrpcServerHandle` 用 `Arc<Notify>` + `Option<JoinHandle>` 做优雅关停与存活探测，`shutdown()` 须先 `notify_one()` 后 `join()`；`TokenizerInfo` 为一次性 GIL 摘出的纯 Rust 结构，仅 `tokenizer_manager` 缺失致命，余者 best-effort 降级；`try_get_attr` 家族做容错取值；参数归一化规则为 worker 至少 1，通道容量与超时为 0 时回退默认 64/300。三线程职责分明：Python 主线程、`sglang-grpc` 服务线程、`sglang-grpc-tokio` worker 池。本讲为后续 u2、u3 的地基。
