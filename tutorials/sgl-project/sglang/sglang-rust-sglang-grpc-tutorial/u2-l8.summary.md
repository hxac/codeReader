本讲讲解 sglang-grpc 中高频但轻量的 RPC `tokenize`/`detokenize` 的实现。核心模块为 Rust 原生分词器：对外包装 `RustTokenizer`、crate 私有后端抽象 trait `TokenizerBackend`、唯一实现 `HuggingFaceTokenizerBackend`（包装 HuggingFace `tokenizers` crate）。构造函数 `from_tokenizer_path` 设三道 `None` 关卡——找不到 `tokenizer.json`、`tokenizer_mode="slow"`、后端加载失败——任一不通过即回退 Python。贯穿主线是「原生优先、Python 回退」：启动期一次性持 GIL 抽取 `TokenizerInfo` 构造 `RustTokenizer` 装入 `PyBridge`，运行期 `Some` 走纯 Rust 无 GIL 计算，`None` 用 `spawn_blocking` 包住 `Python::with_gil` 调 `runtime_handle` 同名方法。两路径产出语义一致的 `TokenizeResponse`，其中 `max_model_len` 取自启动期摘出的 `context_len`（原生 encode 不做截断）；回退路径的 `.await??` 双 `?` 分别处理 `JoinError` 与 `PyErr`，后者经 `pyerr_to_status` 映射为 gRPC 状态码。
