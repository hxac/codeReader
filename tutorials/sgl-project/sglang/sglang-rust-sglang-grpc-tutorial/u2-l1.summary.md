本讲是进阶层首篇，讲清「proto 契约 → tonic 代码生成 → Rust 引入」这条链路。契约本体是 `proto/sglang/runtime/v1/sglang.proto`，声明 `package sglang.runtime.v1` 与 `service SglangService`，共 25 个 rpc，分三类：SGLang 原生（16，强类型消息）、OpenAI 透传（6，`bytes json_body` 不透明字节）、Admin/Ops（3）；其中 4 个带 `stream` 关键字为服务端流式，其余为一元。代码生成由 `build.rs` 的 `tonic_build::configure()` 完成：`.build_server(true).build_client(false)` 只产服务端骨架；`--experimental_allow_proto3_optional` 让 proto3 的 `optional` 字段生效；`rerun-if-changed` 只盯 proto 以加速增量构建。生成代码经 `src/lib.rs` 的 `tonic::include_proto!("sglang.runtime.v1")` 包入 `pub mod proto`，参数必须与 `package` 逐字一致。关键命名映射：`message Foo` → `proto::Foo`；`service Bar` → `proto::bar_server::Bar`（无 `bar_client`）；流式 rpc 在 trait 里多一个 `type MStream` 关联类型。核心结论：新增最简一元 rpc 只需改 proto 与 server.rs，build.rs 因「全文件编译 + 自动重跑」而免改。本讲不涉及方法实现，那是 u2-l2 之后主题。
