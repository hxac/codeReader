本讲给出 server.rs 中 SglangServiceImpl 的全景地图。它是实现 tonic 生成的 SglangService trait 的「薄壳」，仅持有 bridge（Arc<PyBridge>）与 response_timeout 两个字段，请求级状态全在 bridge 内部；类型别名 StreamResult<T> 统一描述所有流式响应。trait 实现块共 25 个 async fn，分 SGLang 原生(16)、OpenAI 透传(6)、Admin(3) 三类：4 个流式 RPC 需声明关联类型 type XStream 并返回 Response<Self::XStream>，21 个一元 RPC 无关联类型、直接返回单条消息。数据型 RPC 共享三种公共提交模式——submit_request（结构化字典）、submit_openai（OpenAI JSON 字节透传）、submit_json 系列（控制查询回 JSON 串），另有 7 个走直连不创建通道。引入的关键术语：rid（请求唯一标识）、ResponseChunk（通道响应片段）、openai_streaming_rpc/openai_unary_rpc（辅助方法，非 trait 方法）。后续 u2-l3 精读流式、u2-l4 精读一元 JSON、u2-l5 精读桥接通道。
