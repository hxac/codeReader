本讲定位 `benchmark/` 目录：它是 SGLang 的评测与压测脚本集合，自身不含推理引擎，而是去驱动一个运行中的服务。核心结论有三。其一，按「量什么」这一维度，目录可归为四类——精度评测（答对率等质量分数，如 mmlu、gsm8k）、在线服务吞吐（token/s、QPS、延迟分布，需先起服务）、内核微基准（不启服务、用 torchrun 计时单算子，如 kernels/ 下各项）、基础设施微基准（衡量引擎内部机制，如 hicache、scheduler）；前两类脚本常叫 bench_sglang.py 且依赖服务，后两类多出现在 kernels/ 下用 torchrun 启动。其二，几乎所有脚本都是「客户端」，经 HTTP（默认 127.0.0.1:30000 的 /generate）连到由 launch_server 启动的「服务端」，吞吐等指标由客户端按墙钟时间自行计算，这也使脚本可切换不同后端。其三，README 是「可复现的操作手册」，按 Server 块、Client 块给出命令；精度与吞吐脚本的产物是结果，内核调优脚本的产物是配置 JSON 会被运行时加载。关键术语：客户端/服务端、launch_server、四分类、微基准、README 驱动。
