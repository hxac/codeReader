本讲讲解 SGLang 的最小吞吐基准 `benchmark_batch.py`：它用合成随机 token 当 prompt 压测运行中的服务。执行链路为：用 `ProcessPoolExecutor` 多进程预生成随机 prompt，将昂贵的分词耗时排除在计时之外；经 `RuntimeEndpoint` 连接服务（构造时打 `/get_model_info` 做健康检查）；用 `requests.post` 直接打 `/generate`，`text` 字段一次送一整批 prompt；最后用 `time.perf_counter` 统计延迟与吞吐。负载形状由四个旋钮控制：`NUM_REQUESTS`、`BATCH_SIZE`、`NUM_TOKENS`、`GEN_TOKENS`。调大 `NUM_TOKENS` 且 `GEN_TOKENS=0` 即纯 prefill 负载，反之切换为重 decode 负载。核心结论：本脚本是顺序压测，衡量的是连续批的延迟与吞吐，而非并发饱和吞吐，后者要到 u5-l1 的 `bench_serving`。配套的 `benchmark_tokenizer.py` 量化分词器逐条与批量处理的开销，解释了分词预处理为何要并行化。
