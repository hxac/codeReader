本讲追踪 `sglang serve` 语言模型启动链路 `serve → prepare_server_args → run_server → http_server.launch_server`，`ServerArgs` 是贯穿全程的唯一配置对象。三处核心机制：`A[T, Arg(...)]`（`A = typing.Annotated`）让数据类字段一处声明即自动生成 CLI 参数与默认值，扫描引擎为 `add_cli_args_from_dataclass`；`run_server` 是薄分发器，按 `encoder_only`/`smg_grpc_mode`/`use_ray`/默认 HTTP 四种模式选择入口，普通服务走最后的 `else`；`__post_init__` 是有序的 `_handle_*` 分发器，负责补默认值与校验，`chunked_prefill_size`、`mem_fraction_static` 的真实默认值在此按 GPU 显存动态算出。终点 `launch_server` 由 `Engine._launch_subprocesses` 拉起 TokenizerManager/Scheduler/Detokenizer 三进程并经 ZMQ 连成环（承接 u2-l1 的多进程拓扑），再以 `_setup_and_run_http_server` 套上 FastAPI+uvicorn 监听 `127.0.0.1:30000`。
