本讲区分 SGLang 的两种使用入口：HTTP 服务（`sglang serve`）与进程内 `sgl.Engine`。核心结论是两者共享同一套引擎——即由 `Engine._launch_subprocesses` 拉起的 TokenizerManager（主进程）与 Scheduler、DetokenizerManager（子进程），三者经 ZMQ+msgspec 做进程间通信；区别仅在于主进程是否多一层 FastAPI/uvicorn 外壳。`launch_server.run_server` 是模式分发器，默认走 `http_server.launch_server`，另有 Ray/gRPC/encode 分离等形态。`Engine` 构造参数与 `ServerArgs` 一一对应；`generate` 是同步包装异步内核，`async_generate` 为原生异步，适合离线批量与 RL 后端。`EngineBase` 是两种入口共享的抽象基类，定义 `generate`/`flush_cache`/`update_weights_from_tensor`/`shutdown` 等统一接口，这是两边 API 同构的根源。进程内用法须带 `if __name__ == "__main__":` 守卫，否则 spawn 子进程会无限递归。
