本讲承接 u4-l1 的 RadixCache，讲解 SGLang 管理 KV 缓存的**两级内存池**。第一级 `ReqToTokenPool` 用一张 `(size+1, max_context_len)` 的 int32 二维表，把每条在途请求的每个 token 映射到物理槽号，第 0 行/槽保留为填充，供 CUDA Graph 假请求安全落地。第二级 `TokenToKVPool` 按物理槽号存放真实 K/V：`MHATokenToKVPool` 形状为 `(slots, head_num, head_dim)`，每层独立 K/V 缓冲；`MLATokenToKVPool`（DeepSeek MLA）则用 `(slots, 1, kv_lora_rank+qk_rope_head_dim)` 的单一融合缓冲，靠低秩大幅省显存。`FP4`/`MXFP8` 等量化池仅重写缓冲与读写，不改索引体系。桥函数 `alloc_for_extend`/`alloc_for_decode` 同时申请两级槽位，产出关键字段 `out_cache_loc`（新 token 的物理槽号清单），它经 `ScheduleBatch` 借给 `ForwardBatch`，最终在 `RadixAttention.forward` 中指导注意力后端把新 K/V 写入正确物理行。两级映射对 MHA/MLA/量化池完全统一，`out_cache_loc` 语义一致。
