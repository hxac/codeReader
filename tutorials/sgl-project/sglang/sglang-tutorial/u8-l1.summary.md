本讲是「分布式与并行」单元首篇，承接 u5-l1，专门拆解一次前向中跨 GPU 的通信环。核心结论：张量并行（TP）通过 Colwise（按输出维切分，前向无需通信）与 Rowwise（按输入维切分，末尾需 all-reduce 求和）切分权重，使大模型跨多卡，代价是每个 Transformer block 末尾必有一次 all-reduce。模块地图分四层：`parallel_state.py` 是分布式「户籍管理处」，`initialize_model_parallel` 按 tp_size 切进程组，`GroupCoordinator` 封装一个通信子群并挂载多路通信器，`all_reduce` 在 ca_comm/qr_comm/pymscclpp/symm_mem/pynccl 间按形状择优；`communication_op.py` 是薄包装，把「定位组+调方法」封成语义函数供模型层调用。自定义 all-reduce（ca_comm）专为小消息低延迟，decode 阶段尤受益。融合 collective 把 all-reduce+加残差+RMSNorm 融进单 kernel，本轮新增的 `fused_allreduce_rmsnorm_quant_per_group` 进一步并入 per-group FP8 量化，仅 ROCm/aiter/gfx95 可用，设计范式是「逐层 return None、逐层回退」，非 AMD 或 ca_comm 缺失即回退。`TpModelWorker` 是调度器与 `ModelRunner` 的桥梁，本身不 all-reduce 但驱动整条前向，并在 world group 广播随机种子保证各卡采样一致。关键术语：rank/world_size、进程组、GroupCoordinator、LayerCommunicator、ca_comm、融合 collective、TpModelWorker，供后续 DP/PP/EP 讲义承接。
