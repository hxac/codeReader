本讲建立 SGLang 的全局视图。它是一个面向 LLM 与多模态模型的**高性能推理服务框架（serving framework）**，区别于训练框架，目标是从单卡到大规模集群都做到低延迟、高吞吐。仓库分为两大子包：前端语言 `sglang.lang`（结构化生成 DSL）与后端运行时 `sglang.srt`（真正干活的进程）。README 的 `Fast Runtime` 即特性总目录：**RadixAttention 前缀缓存**（基数树自动复用公共前缀 KV）、**零开销调度器**（CPU 调度与 GPU 计算重叠）、**Prefill-Decode 分离部署**、**投机解码**、**连续批处理（continuous batching）**、**paged attention**、**TP/PP/EP/DP 并行**、**结构化输出**、**chunked prefill**、**量化**、**多 LoRA 批处理**——每项几乎都能在 `python/sglang/srt/` 下找到对应目录（managers 调度、mem_cache 缓存、speculative、disaggregation、distributed、constrained、quantization、lora）。一条请求依次经 TokenizerManager 路由、Scheduler 调度、RadixCache 命中、ModelRunner 前向、Sampler 解码。SGLang 还定位为 **RL/后训练 rollout 后端骨干**，故内置权重热更新等接口。依赖层（`pyproject.toml`）揭示技术栈：`pyzmq+msgspec` 暗示多进程 ZMQ 通信、`flashinfer/sglang-kernel` 提供算子、`xgrammar/outlines/llguidance` 支撑结构化输出、`fastapi/uvicorn` 提供 OpenAI 兼容 HTTP，入口命令 `sglang serve` 来自 `[project.scripts]`。
