本讲承接 u1-l1，目标是把 SGLang 跑起来并发出首个请求。核心结论：安装强依赖 GPU 栈（torch/flashinfer/sgl-kernel），`sglang` 命令由 `pyproject.toml` 的 `[project.scripts]` 映射到 `sglang.cli.main:main`，extras 含 diffusion/http2/tracing。`sglang serve` 分发链路：`main()` 最薄分发（懒加载 + `parse_known_args` 透传）→ `serve()` 按 `--model-type` 分流语言模型/扩散模型 → `prepare_server_args` 构造 `ServerArgs` → `launch_server.run_server` 拉起 FastAPI+uvicorn 服务，默认端口 30000；`serve()` 用 `try/finally + kill_process_tree` 回收派生的多子进程。访问方式三选一：curl、openai SDK、前端 DSL。引入的关键术语：控制台脚本、argparse 子命令、懒加载、`ServerArgs`、OpenAI 兼容 API（`/v1/chat/completions`）、前端 DSL（`@sgl.function`/`sgl.gen`/`ProgramState`）、`Runtime`（自启服务子进程）与 `RuntimeEndpoint`（连接外部已运行服务）两种 backend。多进程请求流转（TokenizerManager→Scheduler→Detokenizer）留待 u2。
