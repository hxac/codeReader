本讲是 SGLang 的「地图课」，覆盖四个最小模块：仓库顶层目录布局、`python/sglang` 包内 `lang` 与 `srt` 的分工、顶层 `__init__.py` 的公开 API 门面、`srt` 运行时子系统目录布局。核心结论是：SGLang 是 monorepo（核心代码在 `python/`，算子在 `sgl-kernel/`，另有 `rust/`、`sgl-model-gateway/` 等多语言工程）；`python/sglang` 分为前端 DSL（`lang/`，偏编排）与后端运行时（`srt/`，偏计算与性能）。引入的关键术语包括：前端 DSL 与后端运行时 `srt`（SGLang RunTime）、公开 API 门面 `__init__.py`、`LazyImport` 延迟导入、`Engine`/`ServerArgs`/`RuntimeEndpoint`。需注意 `sglang.Engine` 经门面二次赋值后绑定到运行时的 `srt.entrypoints.engine.Engine`。`srt/` 按请求生命周期归类为 `entrypoints/`→`managers/`→`mem_cache/`→`model_executor/`+`models/`+`layers/`→`sampling/`+`constrained/`，并横向贯穿 `distributed/`、`disaggregation/`、`speculative/`、`lora/`、`observability/`、`connector/` 等能力，分别对应后续单元。
