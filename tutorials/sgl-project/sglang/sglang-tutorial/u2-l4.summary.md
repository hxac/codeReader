本讲打开 u2-l3 留下的黑盒：HTTP 请求如何进入 SGLang。OpenAI 兼容层（`entrypoints/openai/`）本身不做推理，而是一层「翻译 + 编排层」，把 OpenAI 端点适配成内部统一的 `GenerateReqInput`，再把 token 流翻译回 OpenAI 协议。它含四个最小模块：一是路由函数仅做无脑转发，所有 handler 共享基类 `OpenAIServingBase` 的模板方法 `handle_request`（校验→转换→按流式分流）；二是 `protocol.py` 用 Pydantic 定义请求/响应契约，核心 `to_sampling_params()` 实现「用户传值 > 模型 generation_config > OpenAI 默认」的三级回退；三是 `OpenAIServingChat` 把对话渲染成 prompt、抽取多模态与扩展字段，组装成 `GenerateReqInput` 交棒给 `tokenizer_manager.generate_request` 进入三进程环；四是 `stream: true` 走 SSE（`data: {JSON}\n\n`），`sse_utils.build_sse_content` 用 msgspec 高性能编码。关键认知：`stream` 不影响 `SamplingParams`，只决定返回路径（SSE 流式 vs 完整 JSON）；流式 handler 会先拉首个 chunk，以便把校验错误在 HTTP 200 之前以 400 返回。
