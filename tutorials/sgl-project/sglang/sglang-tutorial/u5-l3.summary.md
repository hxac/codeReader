本讲解析「注意力后端」：SGLang 中一层可插拔的 attention 实现，让模型层与具体 kernel 解耦。模型层 RadixAttention 只依赖抽象基类 AttentionBackend，具体实现（FlashInfer/Triton/HPC-Ops）在启动时被注入，新增 kernel 不改模型代码。基类契约两部分：`forward` 按 `forward_mode` 分发到 `forward_extend`/`forward_decode`（子类只填这两者）；元数据 `init_forward_metadata` 拆成 `_out_graph`/`_in_graph` 以兼容 CUDA Graph 录制。注册机制：全局字典 ATTENTION_BACKENDS 由 `@register_attention_backend` 装饰器在导入时填充「名字→工厂函数」，工厂函数懒加载具体类以隔离导入失败；`--attention-backend` 经 build_attention_backends 查表产出后端对象，prefill/decode 可分别指定、不一致则包成 HybridAttnBackend。本轮新增的 hpc_ops 后端示范「分层约束校验」：注册级、构造级、层级三道关卡，含非 MLA、不支持投机解码、page_size 64、head_dim 128、GQA∈{4,8}（按单卡 TP 口径）、无滑动窗口等约束，原则是「能早失败的早失败」。
