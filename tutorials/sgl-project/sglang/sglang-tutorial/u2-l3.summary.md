本讲跟踪一条请求在 `TokenizerManager → Scheduler → DetokenizerManager` 三进程环上的端到端流转。核心结论：请求在环上演化四种消息——进程内不上线的 `GenerateReqInput`、真正上线的 `TokenizedGenerateReqInput`、Scheduler 回吐的 token id 批 `BatchTokenIDOutput`、Detokenizer 回吐的文本批 `BatchStrOutput`,取消另走 `AbortReq`。引入的关键概念:`rid`(贯穿三进程的唯一汇合键)、`rid_to_state`(TokenizerManager 入向 `generate_request` 与出向 `handle_loop` 的汇合点)、`ReqState`、增量解码及其 `DecodeStatus` 的 `surr_offset`/`read_offset` 偏移量、`finished_reasons is None` 表示仍在流式生成,以及 msgpack + `tag=True` 序列化与 `enc_hook`/`PickleWrapper` 兜底机制。Scheduler 内部在本讲作黑盒,下讲打开。
