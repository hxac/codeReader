本讲引入 HiCache，把 KV 缓存从 GPU 显存（L1）延伸到主机内存（L2）与磁盘/共享存储（L3），将「淘汰即丢弃」改为「降级（demote）后取回升级（promote）」，以省去重复 prefill、支撑更长上下文。HiRadixCache 继承 RadixCache，在节点上新增 host_value/backuped/evicted 等状态，以 evict、write_backup、load_back、prefetch_from_storage、match_prefix 编排三层间的搬运；写回方向为 L1→L2→L3，取回方向相反。L3 由 HiCacheStorage 抽象基类统一多种后端（如 HiCacheFile），可经 HiCacheController.attach_storage_backend 在运行期挂载（要求无在途请求）。主机侧 MHA 池用分阶段写回（staged write-back）经 GPU staging buffer 分批搬运以省显存带宽。本次更新把算子迁至 kernels.ops.kvcache.hicache，并为 AsymmetricMHATokenToKVPoolHost 新增分阶段写回——因 K、V 的 head_dim 不等导致步长不同，必须拆成两次独立单缓冲拷贝，用形状各异的 staging_k_buffer / staging_v_buffer。
