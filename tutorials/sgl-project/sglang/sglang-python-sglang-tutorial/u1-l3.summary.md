本讲义给出 sglang 的目录导航地图，不深入机制。核心结论：`python/sglang/` 顶层由入口脚本（`launch_server.py` 等）、子包（`lang/`、`srt/`、`cli/`、`kernels/`、`test/`）和公共 API 装配文件 `__init__.py` 组成。`import sglang` 执行的就是 `__init__.py`，其中 `Engine` 被定义两次，靠后的懒导入 `LazyImport("sglang.srt.entrypoints.engine","Engine")` 覆盖前端版，故 `sglang.Engine` 最终指向运行时引擎。`launch_server.run_server` 按 `server_args` 标志把服务分发到 HTTP/gRPC/Ray/PD 四种形态。`srt/`（SGLang Runtime）是项目主体，40+ 子目录按请求生命周期归为入口、调度、执行、存储、算子、扩展、基础设施七线，主链为 `entrypoints/`→`managers/`→`model_executor/`→`mem_cache/`。引入术语：包（含 `__init__.py` 的目录）、懒导入 `LazyImport`、配置中枢 `ServerArgs`、前端转发机制、`srt/` 子系统速查表。
