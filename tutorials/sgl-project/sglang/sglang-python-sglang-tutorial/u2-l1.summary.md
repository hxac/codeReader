本讲是第 2 单元（前端语言层 `lang/`）首篇，承接 u1-l4 关于请求入口的内容，主题是 SGLang 前端 DSL 的四个最小模块：`function`、`gen`、`select`、`image`。核心结论是：`gen`、`select`、`image`/`video` 返回的都不是立即执行结果，而是 `SglExpr` 表达式对象——这是前端「惰性求值」的根基，使程序可被拼接、分析、批处理与缓存。源码上，「造对象」（`lang/api.py`、`lang/ir.py`）与「执行对象」（`lang/interpreter.py`）是分离的两阶段。

关键术语：`SglExpr` 是所有指令的统一基类，靠 `node_id`/`prev_node`/`pid` 串成图，并用 `__add__` 拼接（字符串自动包成 `SglConstantText`）。`@sgl.function` 把普通函数包成 `SglFunction`，强约束第一个参数必须名为 `s`（即 `ProgramState`），只有 `.run()` 才真正触发执行。`gen` 有双重身份：带 `choices` 时退化为 `select`，否则构造 `SglGen`，采样参数收拢进 `SglSamplingParams`；`gen_int`/`gen_string` 仅是设了 `dtype` 的快捷方式，配合类型正则做约束生成。`select` 用似然挑候选，默认 `choices_method` 为 `token_length_normalized`，比较归一化平均对数概率取 argmax，结果写入变量供 `state["name"]` 读取。`image`/`video` 构造只存路径，读图与 base64 编码延迟到执行期。追踪（run vs trace）机制留待 u2-l2 承接。
