本讲承接 u2-l1，确立 SGLang 前端「先追踪成 IR，再解释执行」的两阶段模型。程序表示为一张由 SglExpr 节点组成的图，常量文本、gen、select 等均为子类，节点靠 prev_node 串成顺序链，gen/select 的参数收拢为 SglSamplingParams；SglFunction 只是「配方容器」，本身不持有 IR，图在追踪或执行时由 ProgramState 动态累积。Tracer 用空壳后端跑一遍函数体，仅把 s += expr 收集成节点而不联网，核心用途是提取公共前缀做前缀缓存。Interpreter 中的 StreamExecutor 持真实后端，遇 SglGen 调 backend.generate、遇 SglSelect 调 backend.select，结果写回 variables 并 set variable_event，靠后台线程加队列支持流式输出与变量就绪等待。分流枢纽 SglFunction.__call__ 依据线程级 TracingScope 决定走 run（解释）还是 trace（追踪），以正确处理嵌套 @function 调用。两阶段协作的直接收益是批处理时先 trace 公共前缀并缓存、再并发 run 各条入参，省掉重复 prefill。引入的关键术语 SglExpr、prev_node、SglSamplingParams、TracerProgramState、StreamExecutor、ProgramState、TracingScope、variable_event 供后续讲义承接。
