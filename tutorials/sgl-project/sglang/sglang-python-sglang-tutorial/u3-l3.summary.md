本讲讲解 SGLang 的单一配置源 ServerArgs：一个 8800+ 行的 @dataclass，承载所有服务参数，被主进程的 TokenizerManager 与子进程的 Scheduler、DetokenizerManager 共享读取。引入四个核心机制。第一是注解式参数声明：字段写作 `字段: A[类型, Arg(...), NS(...)] = 默认值`，其中 A 是 typing.Annotated 的别名、Arg 是承载 CLI 元数据的不可变数据类、NS 是无关 CLI 的命名空间标记；add_cli_args_from_dataclass 据此自动派生出 argparse 选项，做到「字段即 CLI」。第二是解析入口 prepare_server_args，经 ConfigArgumentMerger 把 YAML 转成 argv 并排在命令行之前，确立 CLI > YAML > Defaults 的优先级。第三是 __post_init__ 被刻意写成有序调度器，按依赖顺序调用数十个 _handle_* 完成派生与校验（如按显存推算 mem_fraction_static），末尾 materialize_declarations 固化配置。第四是 arg_groups 分组钩子目录：把 PD 分离、投机解码、HiSparse 等特性逻辑抽成 hook，模型架构驱动的调整集中到声明式覆盖注册表 overrides.py，弃用参数迁移交给 argparse_actions。核心铁律是固化后 ServerArgs 只读，__setattr__ 拒绝裸赋值，唯一合法改写入口是 override()。本讲的 NS 命名空间与 resolve-at-end 约定是下讲 RuntimeContext 的直接输入。
