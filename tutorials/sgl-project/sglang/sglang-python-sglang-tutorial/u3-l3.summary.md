本讲围绕 SGLang 的**单一配置源** `ServerArgs`（近 9000 行的 `@dataclass`，被三大管理器进程共享读取）展开，核心是「一行声明 → 自动 CLI → YAML 合并 → 派生校验 → 固化只读」的配置闭环。引入的关键术语：注解式声明 `A[类型, Arg(...), NS(...)]`（`A` 是 `typing.Annotated` 别名，`Arg` 承载 CLI 元数据，`NS` 是供 RuntimeContext 分组用的命名空间标记），由 `add_cli_args_from_dataclass` 自动派生为 argparse 选项。解析入口 `prepare_server_args` 经 `ConfigArgumentMerger` 确立 **CLI > YAML > Defaults** 优先级。`__post_init__` 是有序调度器，调用一串 `_handle_*` 做派生（如 `mem_fraction_static` 按显存推算）与校验，末尾 `materialize_declarations` 固化；此后 `__setattr__` 无条件拒改，唯一合法入口是 `override()`。`arg_groups/` 用分组钩子（PD/投机/HiSparse）、声明式覆盖注册表 `overrides.py` 与弃用动作 `argparse_actions.py` 控制文件膨胀。最后讲清 **per-arch 校验**：`_handle_model_specific_adjustments` 按 `hf_config.architectures[0]` 分发到 `<arch>_hook.py`（如 `deepseek_v4_hook` 的 CP 与 MegaMoE token 预算校验），新增的 `enable_cp_decode_attn_tp` 字段用架构白名单自我保护，mamba 字段补齐 `NS("exec.mamba")` 以满足命名空间覆盖 lint。
