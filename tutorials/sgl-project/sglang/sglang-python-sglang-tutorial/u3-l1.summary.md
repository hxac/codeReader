本讲追踪 `sglang serve` 从命令行到 HTTP 就绪的启动链。四个最小模块：prepare_server_args 用 argparse 加可选 --config 合并，把参数解析成单一配置源 ServerArgs，并在 __post_init__ 完成校验与派生；load_plugins 是基于 setuptools entry_points 的统一插件框架，幂等早调用，受 SGLANG_PLUGINS 白名单与 SGLANG_PLATFORM 排除集控制；run_server 为分发器，按 encoder_only/smg_grpc_mode/use_ray/默认四选一，默认形态由 Engine._launch_subprocesses 按 Scheduler 子进程→Detokenizer 子进程→主进程 TokenizerManager 顺序组装引擎；_setup_and_run_http_server 把对象存入 _global_state、装中间件后 uvicorn.run(app)，FastAPI 路由在模块导入时绑好。真正就绪需等 lifespan 后台线程跑完 warmup（轮询 /model_info + 发一条真实请求）置 ServerStatus.Up，故「端口已开但首请求偏慢」。三大管理器经 ZMQ 通信；进程拓扑与 ServerArgs 配置体系留待后续讲义展开。
