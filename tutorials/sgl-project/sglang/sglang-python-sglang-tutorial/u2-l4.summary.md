本讲覆盖前端 `lang/` 的两条线：**渲染线**与**决策线**。渲染线介绍角色原语 `system` / `user` / `assistant`：它们只构造「带角色标签」的惰性表达式（`SglRoleBegin` / `SglConstantText` / `SglRoleEnd`），本身不含任何格式信息。具体渲染格式由数据类 `ChatTemplate` 的 `role_prefix_and_suffix` 表（`{角色: (前缀, 后缀)}`）决定，从而把「角色语义」与「渲染格式」解耦，使同一份 `@function` 可跨模型复用。模板有两个共用同一张表的渲染入口：`get_prompt(messages)` 一次性拼出扁平字符串，解释器则用 `get_prefix_and_suffix` 边执行边拼 `text_`、同时维护结构化 `messages_`；`default_system_prompt` 会在首条非 system 消息时被自动补成 system。决策线讲 `select` 的本质：比较各候选在上下文下的 token 级 logprob 取最高者，返回 `ChoicesDecision(decision, meta_info)`，评分规则由可替换的 `choices_method` 决定。三种内置策略：默认 `token_length_normalized` 取平均对数概率（用 1/n 消除长度差异）、`greedy_token_selection` 逐 token 贪心淘汰（对前缀重叠更稳健）、`unconditional_likelihood_normalized` 减去无条件似然（PMI 风格，需额外一次前向）。关键协作约定是 `requires_unconditional_logprobs` 属性——策略声明需求、后端按需取数，避免所有策略都白白多算一次。
