SGLang 是面向大语言模型与多模态模型的高性能推理服务框架，追求单卡到集群规模的低延迟、高吞吐推理，也是主流 RL 框架的 rollout 后端。代码物理上分两层：前端语言层 `lang/`（用 `@function/gen/select` 等 DSL 描述复杂生成流程、编译成 IR 交给后端）与运行时层 `srt/`（负责加载模型、调度 batch、GPU 执行、管理 KV 缓存与分布式、PD 分离等）。关键区别在于：前端 `lang` 是可选的，多数用户直接用运行时入口 `sglang.Engine` 或起 HTTP 服务，仅当需要复杂生成控制流时才写前端 DSL。`import sglang` 执行的 `__init__.py` 即公共 API 清单，其中 `Engine` 这个名字被前端与运行时各赋值一次，最终生效的是运行时引擎 `sglang.srt.entrypoints.engine.Engine`。运行时引擎由 TokenizerManager（主进程）、Scheduler（子进程）、DetokenizerManager（子进程）三部分组成，进程间经 ZMQ 通信，这是后续服务端架构与调度核心两单元的主线。`global_config` 存放少量历史全局常量但已标记计划废弃，运行时配置正迁移到 `srt/environ.py` 的环境变量体系；`sglang.__version__` 由 `version.py` 的四级回退链动态决定，开发环境下出现 `0.0.0.dev0` 是正常的兜底值。
