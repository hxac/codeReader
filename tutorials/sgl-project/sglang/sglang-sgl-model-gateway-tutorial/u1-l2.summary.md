本讲聚焦如何把 sgl-model-gateway 从源码变成可运行产物。一次构建产出库 smg（rlib）与共用 src/main.rs 的三个二进制（sgl-model-gateway/smg/amg）。设计了三档 Cargo profile：release（opt-level="z"+fat LTO+codegen-units=1，体积最小）、ci（inherits release 后放宽、编译更快）、dev（opt-level=0+incremental、编译最快，依赖走 opt-level=2 子段以兼顾运行速度）。build.rs 在编译期通过 cargo:rustc-env 把版本号、git 状态、编译器版本注入，src/version.rs 用 env! 宏固化为常量，支撑 --version-verbose，rerun-if-changed 保证版本变化自动重注入。Python 绑定是 bindings/python 下的 cdylib 子 crate，以 path 依赖复用核心，maturin 打包成 wheel，abi3-py38 让一份 wheel 通吃 Python 3.8+，vendored-openssl 免除系统依赖。Makefile 统一工作流：bump-version 一次性同步散落各处的 5 处版本号，release-notes 按路径白名单生成网关专属 changelog。
