本讲讲解命令行参数如何变成内部配置对象这一翻译链，覆盖三个最小模块：用 clap 派生宏声明、按 help_heading 分组的 `CliArgs` 参数清单；CLI 与内部模型之间的翻译层 `CliArgs::to_router_config`；以及链式构建的 `RouterConfig::builder`。核心结论：`to_router_config` 按 backend→pd→regular 的优先级决定 `RoutingMode`（Regular／PrefillDecode／OpenAI），并用 `parse_policy`、`determine_connection_mode` 把字符串翻译成强类型的 `PolicyConfig` 与 `ConnectionMode`（Http／Grpc）；builder 内部直接持有一个 config 以避免字段重复，全部默认值来自 `RouterConfig::default()`，封口时集中读取 mTLS／TLS／MCP 文件并执行 `validate()`，翻译失败抛出 `ConfigError`。此外 `--prefill <url> [port]` 的可选第二值无法用 clap 表达，项目靠「预扫描 argv、过滤掉相关 token、再把干净参数交给 clap」三步绕过。本讲只讲构造过程，字段含义、`validate()` 校验细节与 `server::startup` 消费留待后续讲义承接。
