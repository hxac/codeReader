本讲建立 sgl-model-gateway（库名 `smg`）的源码模块地图。crate 根 `lib.rs` 仅 16 行，由 11 个自有模块加 5 个外部 crate 重导出构成。网关呈金字塔分层，依赖自上而下为 `config → core → policies → routers`：`config` 为最底层、几乎无内部依赖；`core` 是被各区域共享的抽象底座，含 `Worker`、`WorkerRegistry`、`JobQueue`、`steps`、`CircuitBreaker`、`HashRing` 等 14 个子模块；`policies` 是定义 `LoadBalancingPolicy` 的负载均衡策略层，依赖 core；`routers` 是数据面，以统一接口 `RouterTrait` 抽象 HTTP/gRPC/OpenAI 路由器，唯 `route_chat` 必须实现，其余方法默认返回 501。`observability`、`middleware` 为横切关注点，不属纵向链。装配层 `app_context.rs` 把各层组件捏合成共享状态 `AppContext`，`server.rs` 再经 `build_app`/`startup` 接到 axum；`main.rs` 保持极薄，仅直接引用 7 个模块，其余模块经 `server::startup` 间接使用。工程根目录的 `bindings`、`benches`、`e2e_test`、`examples` 为辅助目录，以库依赖形式复用 `smg`，不参与主二进制核心链路。
