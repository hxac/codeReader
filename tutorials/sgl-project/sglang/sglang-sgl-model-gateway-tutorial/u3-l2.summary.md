本讲义主角为 worker_registry.rs，定义 WorkerRegistry、HashRing、WorkerId。WorkerId 是 UUID v4 newtype，靠 url_to_id 实现「同 URL 复用 ID」，reserve_id_for_url 用 DashMap entry API 原子占位以规避竞态。WorkerRegistry 维护 workers、model_index、hash_rings、type_workers、connection_workers、url_to_id 六张并发表，按模型、类型、连接模式、URL 五维预建索引；针对「读极多写极少」，读路径以不可变快照 Arc<[Arc<dyn Worker>]> 加引用计数实现无锁读，写路径 copy-on-write 重建快照并同步重建哈希环、刷新索引。HashRing 用确定性 blake3 在 [0,2^64) 环上为每个 Worker 放 150 个虚拟节点（共享 Arc<str>）并排序，find_healthy_url 经二分定位、顺时针遍历、HashSet 去重，O(log n) 取首个健康 Worker，使增删 Worker 时迁移比例约 1/N，远优于 hash%N。环由 registry 经 get_hash_ring 预缓存、仅增删时重建，策略层（consistent_hashing/prefix_hash）借 SelectWorkerInfo.hash_ring 透传复用而不自建环；set_mesh_sync 以 RwLock<Option> 使同份代码在单机（None 即 no-op）与多机 Mesh 下通用。
