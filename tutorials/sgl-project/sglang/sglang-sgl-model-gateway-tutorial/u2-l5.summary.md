本讲聚焦启动收尾的纯装配函数 `build_app`：它接收在 `AppContext` 之上再打包 `Arc<dyn RouterTrait>` 的 `AppState`，将 `public/protected/admin/worker/mesh` 五大路由组 merge、套六层全局中间件（请求体限制、日志、指标、请求 ID、CORS）、挂 `sink_handler` fallback、`.with_state()` 注入状态，产出可直接 serve 的 `Router<()>`。认证按强度分层：`public` 无认证；`protected`/`mesh` 走单一 `AuthConfig.api_key` 的 `auth_middleware`（仅认 `Authorization: Bearer`、常量时间比较防时序侧信道）；`admin`/`worker` 走 `smg_auth` 控制面认证，未配置则降级为 `auth_middleware`。`protected` 组独有三层 `route_layer`（concurrency_limit→auth→wasm），按「先添加为外层」执行；认证用 `route_layer` 而非全局 `layer`，避免把 404 变成 401、泄露路由表信息。探针方面，`/liveness`、`/health` 恒 200；`/readiness` 按部署模式判定，PD 模式需 prefill 与 decode 双类 worker 均健康才算就绪，就绪返回 200、否则 503。
