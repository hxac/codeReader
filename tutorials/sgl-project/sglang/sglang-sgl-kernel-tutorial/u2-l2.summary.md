本讲承接 u2-l1（`.so` 加载完成），解答一行 `torch.ops.sgl_kernel.<name>.default(...)` 如何分派到 C++ 实现，覆盖三个最小模块：注册机制、schema 语法与可变语义、Python 包装转发。sgl-kernel 用 `TORCH_LIBRARY_FRAGMENT(sgl_kernel, m)` 打开可被多扩展文件重复追加的命名空间片段；`m.def` 以 schema 字符串声明算子契约，`m.impl` 把 C++ 函数绑定到 `torch::kCUDA`，`REGISTER_EXTENSION` 生成 Python 模块入口。CUDA 算子普遍用「显式 schema + 设备绑定」两步式，因显式 schema 是 `torch.compile` 正确追踪的前提；少数工具函数用一行式 catch-all（不绑设备）。schema 是一套小语言：`Tensor` 只读、`Tensor?` 可选、`Tensor!` 可变（原地写回），可叠加成 `Tensor!?`；`-> ()` 表示无返回、`-> Tensor` 表示返回新张量。核心结论：`-> ()` + `Tensor!` 是高性能推理算子主流写法——输出由调用者预分配、地址稳定，省去高频显存分配、支持 CUDA graph 捕获与缓冲复用；可变性由 `!` 声明，与 C++ 形参是引用还是按值句柄无关。Python 包装层承担「校验 + 输出预分配 +（可选 FlashInfer 回退，dynamo 编译期强制回退）+ 转发」，末行参数顺序与 schema 一一对应。本讲止于「分派到 C++ 函数门口」，kernel 体内部留给 u2-l3。
