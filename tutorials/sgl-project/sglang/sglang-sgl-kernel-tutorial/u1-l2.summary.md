本讲讲清 sgl-kernel 的目录职责与「六步内核流水线」这条全册主线索。源码分五大目录：csrc/（CUDA/C++ 实现与扩展注册）、include/（头文件声明）、python/（Python 包装与导出）、tests/（正确性）、benchmark/（性能），均按算子功能族（elementwise、gemm、attention、moe 等）分子目录组织。

核心模型是六步流水线：CUDA 实现 → 头文件声明 → torch 扩展注册 → Python 包装 → __init__ 导出 → CMake 构建加测试基准，与官方 README 贡献步骤一一对应，开发方向与调用方向相反。

引入的关键术语：算子、扩展（.so 动态库）、PyTorch Custom Op、TORCH_LIBRARY_FRAGMENT(sgl_kernel, m) 注册宏、m.def 写 schema（Tensor! 原地写回、-> () 空返回）、m.impl 绑设备、torch.ops.sgl_kernel.<算子名>.default 调用、CHECK_*/DISPATCH_* 校验与 dtype 分派宏、vendored FlashInfer 模板。

核心方法：定位任意算子，先在 common_extension.cc 用算子名 grep 到 m.impl 绑定的 C++ 函数名，再据此去 include/ 找声明、csrc/ 找实现；以 rmsnorm 全链路验证。
