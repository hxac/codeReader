本讲拆解 sgl-kernel 构建系统，分三模块。其一，构建后端用 scikit-build-core 而非 setuptools：pyproject.toml 借 [tool.scikit-build] 把 CMake 接入打包，wheel 只装 python/sgl_kernel 目录，运行期 torch 由用户钉版（dependencies 为空），扩展以 Stable ABI 编译。其二，核心是「一份 SOURCES、两个 CMake target」：common_ops_sm90_build 带 -use_fast_math、装到 sgl_kernel/sm90/，common_ops_sm100_build 用精确数学、装到 sgl_kernel/sm100/，二者 OUTPUT_NAME 同为 common_ops、靠目录区分，运行期按 compute capability 选载其一。gencode 指单个 .so 内为多架构生成机器码，勿与双产物混淆；另有 flash_ops 等独立扩展，第三方源由 FetchContent 现拉带校验。其三，make build 封装 uv build 与 pip 安装；资源控制有两层旋钮——MAX_JOBS 控翻译单元并行度，SGL_KERNEL_COMPILE_THREADS 控 NVCC 单文件线程，相乘近似总并发，防 OOM 时两者皆需调小；build.sh 是带 ccache 的 Docker 化发布构建。本讲承接六步流水线的构建步，下接 u2-l1 的架构自适应加载。
