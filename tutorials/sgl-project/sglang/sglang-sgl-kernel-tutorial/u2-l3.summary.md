本讲承接 u2-l2，推门进入 sgl-kernel CUDA 算子的 C++ 函数体，以 `fused_add_rms_norm_kernel.cu` 为范本，归纳出**「校验 → 分派 → 调用」三段式**结构。**校验段**用 `include/utils.h` 中的 `CHECK_INPUT`/`CHECK_DIM`/`CHECK_EQ`/`CHECK_SHAPE` 宏，本质是 `TORCH_CHECK` 叠加预处理器字符串化 `#x`，在 GPU 执行前拦住设备、连续性、维度、形状等错误并翻译为清晰的 Python `RuntimeError`。**分派段**用 `DISPATCH_PYTORCH_DTYPE_TO_CTYPE_FLOAT_FP16` 等宏，以运行时 `switch(scalar_type())` 配合每个 `case` 内 `using c_type = ...` 桥接运行时 dtype 与编译期 C++ 类型，复用同一模板并实例化出 float/half/bfloat16 多份机器码。**调用段**通过 `at::cuda::getCurrentCUDAStream()` 取队列，透传 `enable_pdl`（Hopper 的 PDL 特性）与行步长，转交 vendored 的 FlashInfer 模板 `norm::FusedAddRMSNorm`，再用 `TORCH_CHECK` 把 `cudaError_t` 翻译回 Python。核心结论：sgl-kernel 这层 C++ 函数本身不含 kernel 实现，只做包装与防御；函数返回 `void` 配合可变 `Tensor!`，闭合了 u2-l2 的 `-> ()` 可变语义。关键术语：三段式、CHECK 宏体系、dtype 编译期分派、`c_type` 占位名、`getCurrentCUDAStream`、`enable_pdl`/PDL、行步长 `stride(0)`、FlashInfer 模板透传。本讲埋下 ROCm 条件编译的多后端伏笔，留给 u3 展开。
