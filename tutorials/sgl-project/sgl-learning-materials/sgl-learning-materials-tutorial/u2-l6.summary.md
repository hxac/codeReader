本讲义讲两块多副本推理基础设施。一是 SGLang Router（缓存感知负载均衡器）：挡在多个 worker 副本前，依据 RadixAttention 的 KV Cache 前缀复用，把请求分给最可能已缓存该前缀的副本，减少重复 prefill，优于朴素 round-robin。二是分布式权重热更新（Update Weights From Distributed）：从分布式来源读取新权重并原子切换，跳过进程重启与磁盘冷读，把换权重从分钟级冷启动压到秒级且不停机。二者拼合正是大规模 RLHF 的关键——rollout 阶段「权重每步都变」靠热更新、「海量并行采样」靠 Router，常并存 policy 与 reference 等多角色模型。引入术语：worker/副本、cache-aware、冷启动、原子切换、rollout。本仓库无可运行代码，资料以 README 索引与幻灯片为主，须分清下划线、连字符两份 router 幻灯片，并以 v0.4 发布博客为权威锚点。
