本讲围绕 DeepSeek MLA 展开三个最小模块。其一讲 MLA（多头潜在注意力）原理：区别于 MHA、MQA、GQA 靠「砍头」或共享来瘦身，MLA 通过下投影把隐状态压成低维潜在向量并只缓存它，用时上投影还原，从而大幅缩小 KV Cache 又保精度；因 RoPE 逐维度相乘会干扰低秩压缩，故用「解耦 RoPE」将 K 拆为内容与位置两部分分别处理。其二讲 v0.3 的「7× Faster DeepSeek MLA」：属推理引擎侧工程优化而非改模型结构，比较口径是 SGLang 自身前后版本，不是 MLA 相对 MHA 的理论压缩比。其三把 MLA 当作「资料簇」范例：同主题散见于 meetup 仓库内 PDF、biweekly 外链幻灯片、Hyperbolic 线下聚会 PDF 及 v0.3 发布博客，应先 grep 检索、按日期排成时间线整体阅读，并留意部分幻灯片无配套录像。本讲引入的关键术语：MLA、MHA/MQA/GQA、KV Cache、RoPE、解耦 RoPE、潜在向量、资料簇、7× 提速、Model Optimizations。
