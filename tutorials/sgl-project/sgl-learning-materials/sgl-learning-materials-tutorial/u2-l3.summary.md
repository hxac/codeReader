本讲聚焦「让大模型又快又稳地生成合法结构化文本」这一主题。核心痛点是：受限解码（constrained decoding）通过在每步采样前用掩码（mask）屏蔽非法 token，保证输出符合 schema，但传统基于有限状态机（FSM）的做法被「一步一个 token」绑死，无法享受 prefill 的并行加速红利。SGLang 的破局解法是压缩有限状态机（Compressed FSM）配合跳跃式解码（jump-forward）：把 FSM 中只有一个出口的连续边压缩成「确定路径」，遇到确定段就直接 prefill 整段、跳到下一个分支点，仅在有分支处才逐 token 决策；该机制依赖 RadixAttention 复用 KV cache，并以重新分词（约 4% 开销）处理 tokenization 边界。XGrammar 是这条路线工程化后的独立结构化生成引擎，支持通用文法、跨硬件、近零开销（靠 token mask cache 与 CPU/GPU 开销重叠），已成为 vLLM、SGLang、TensorRT-LLM、MLC-LLM 的默认后端；其解码速度相比已有后端快 3–5×、端到端提速约 30%。读资料时需区分两组提速数字口径：压缩 FSM 博客的约 2× 延迟降低 / 2.5× 吞吐提升，与 XGrammar 的 3–5× / 30%，比较对象不同不可混用。本主题在本仓库构成一个「资料簇」，含两份 meetup 幻灯片、meetup 回顾博客、两篇 LMSYS 博客外链与一场 YouTube 录像。
