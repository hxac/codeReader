本讲义（u3-l3）从安全视角审视 SGLang 的 KV Cache 共享优化，拆为三个最小模块：机制回顾、攻击原理、缓解措施。核心结论是：RadixAttention 的前缀复用与 Cache-Aware Router 的按命中调度（资料入口见 README 第 62/100/118/126 行）带来巨大吞吐红利，但「命中比未命中快」这一时延不对称构成可观测的时延差，约等于 c·n（n 为前缀 token 数）。在多租户共享缓存前提下，攻击者无需读取 KV Cache，只需发请求并计时，即可将该时延差当作 oracle，通过逐 token 搜索还原他人或系统前缀，形成时序侧信道与 prompt 窃取风险。本仓库无运行时代码，核心资料为 slides/Possible_Timing_Side_Channel_Of_KV_Cache.pdf，标题中的 Possible 表明这是对潜在漏洞的可行性探讨而非落地工具。缓解总纲是抹平时延差或切断 oracle：租户隔离/分区、关闭跨租户复用、时延规整、速率限制与异常探测、架构上避免秘密入共享前缀，这些方向与吞吐直接对冲，需按租户是否互信权衡；真正的可配置开关在主仓库 sgl-project/sglang，衔接 u4-l3。引入并供后续承接的关键术语：KV Cache、RadixAttention、前缀复用、Cache-Aware Router、多租户、时序侧信道、prompt 窃取、时延规整。
