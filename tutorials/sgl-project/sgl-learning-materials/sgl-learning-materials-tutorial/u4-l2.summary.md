本讲义是专家层（u4）第二篇，核心价值在「编排」——把前序讲义散落在 README、slides、blogs 的资料，按读者目标重组为一条个人学习路线，几乎不含新知识。最小模块三块：(1) 按目标分组——把资料分进优化、部署、硬件、安全四个抽屉，用 grep 反查 README 行号定位资料簇；(2) 排阅读顺序——套用六原则：先广后深、概念先于机制、单机先于集群、红利先于风险、仓库内先于外链、录像优先于幻灯片；(3) 衔接外部资源——厘清本仓库作为「路标」的边界：跑起来去文档站、懂原理去论文或博客、看实现或可配置开关去主仓库 sgl-project/sglang。核心结论有二：一是同一份资料可多重归属，这是资料簇的特征而非分类错误；二是安全抽屉必须排在优化、部署之后，因为 RadixAttention 前缀复用与 Cache-Aware Router 的吞吐红利，正是 KV Cache 时序侧信道风险的根源——先懂红利才能懂风险。最终交付物是一份可执行的 4 周计划，每周锁定 2–3 份仓库内资料并写明周目标。需承接的关键术语：四个抽屉、资料簇、六条排序原则、外衔接通道、资料→用法→原理→代码递进链路。
