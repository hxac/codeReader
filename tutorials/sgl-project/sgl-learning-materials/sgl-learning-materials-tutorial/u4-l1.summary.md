本讲是专家层首篇，主题为如何向资料库贡献新内容。核心结论：本仓库无任何可执行代码，"贡献"等于两件事——把资产放进正确目录，再在 README 登记一条索引，二者缺一不可（仓库已存在"只放未登记"的真实反例）。

下列各项均为软约定（仓库无 CONTRIBUTING.md、无 .github/、无 PR 模板，需照抄现有文件来落实）：资产命名上，slides/ 主流为小写、事件前缀加下划线、以 PDF 为主（仅一个 .pptx 特例），取名前用 git ls-files 检索以避免近重名（如 sglang-router 与 sglang_router 一字之差）。README 条目统一格式为 [日期] [标题](链接)，日期用 YYYY-MM-DD 零填充，按事件用 ### 子区段归类、同段内日期从新到旧，### Other 兜底零散无日期资料；含空格的链接须做 %20 转义，且相对路径起点决定前缀写法（博客正文从 blogs/ 起写 docs/figs/…，README 从仓库根起写 blogs/…），配图统一放 blogs/docs/figs/。完整 PR 流程为 fork→分支→放资产→改 README→显式 git add（因 .gitignore 只忽略 .DS_Store，勿用 git add .）→commit→push→发 PR；贡献内容按 MIT 许可发布。
