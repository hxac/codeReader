本讲是手册收官篇，主题为「资料的边界与延伸资源」，由三个最小模块构成。4.1 守住边界：本仓库是资料聚合库而非运行时代码库，可用 `git ls-files` 按扩展名自证——全部资产为 25 PDF + 10 PNG + 2 MD + 1 PPTX + LICENSE + .gitignore，源码扩展名计数为零，根级仅 README/LICENSE/.gitignore 三块基石。4.2 打开中段通道：外部延伸入口分用法/原理/讲解/社区/组织五类，须合并 README 的 Documentation/Paper/Videos/Blog 四区段与博客页脚两来源，后者补全了 README 缺失的入口。4.3 抵达终点主仓库 `sgl-project/sglang`。核心结论有三：其一，文档站存在双域名（`sgl-project.github.io` 与 `sglang.readthedocs.io` 不一致，待本地验证）；其二，README 通篇不含任何 `github.com` 链接，主仓库地址仅出现在博客页脚，跳主仓库须主动迈出；其三，贯通「资料→用法→原理→代码」四层递进链路——资料即本仓库、用法即文档站、原理即论文/博客、代码即主仓库，各层入口不可混用。最终交付物为「延伸资源清单」。
