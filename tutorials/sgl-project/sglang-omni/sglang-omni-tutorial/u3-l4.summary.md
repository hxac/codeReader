本讲覆盖 SGLang-Omni 的进程拓扑求解与多进程 Runner，核心模块为 `topology.py`、`mp_runner.py`（`MultiProcessPipelineRunner`）、`stage_workers.py`、`runtime_config.py`。

核心结论：SGLang-Omni 把「进程共置」与「GPU 共置」拆成正交两维。进程拓扑只回答「哪些非 TP 阶段共进程」，不决定 GPU 放置——放置计划是其输入而非输出。求解用并查集：对声明相同 `process` 名、或同属一个 `fused_stages` 组的非 TP 阶段求传递闭包，每个等价类即一个 OS 进程；TP 阶段走一进程一 rank、独占进程、不参与合并。硬规则：每个非 TP 阶段必须显式声明 `process`。

关键术语：`StageLaunchConfig`（逻辑实例规格，role∈{single,leader,follower}，由 `owns_external_io` 区分是否拥有对外 IO）、`StageWorkerProcessSpec`（OS 进程载荷）；所有函数只以点分路径传递，子进程用 `import_string` 解析。`MultiProcessPipelineRunner.start()` 在父进程编译运行时并求解拓扑 → spawn → wait_ready → 向 Coordinator 注册 single/leader 端点 → 启动监控；子进程入口 `stage_process_main` 仅凭 spec 重建 Stage 而不重编译 pipeline，共进程多 stage 共享一个 asyncio 循环、无 per-stage 故障隔离。现状提醒：`apply_fusion` 当前是 no-op，`fused_stages` 仅在拓扑层让 stage 共进程，不在配置层重写 stage。
