本讲是 u2 单元「追踪请求主链路」的总览地图，覆盖三个最小模块。**分层职责**：SGLang-Omni 把请求链路显式切成六层——HTTP API → Client → Coordinator → Stage → Scheduler → ModelRunner，每层职责不重叠，核心直觉是「外层只翻译搬运，内层才计算」，HTTP/Client/Coordinator 基本不碰 GPU。**启动分界点**：`create_app` 只建 FastAPI 注册路由，`launch_server` 管完整生命周期（编译管线→启动运行时→建 Client→建 app→挂 profiling→跑 Uvicorn→关停），后者包含前者。**请求/响应路径**：请求正向有两次类型转换（`ChatCompletionRequest → GenerateRequest → OmniRequest`，分别在 HTTP 层与 Client 层）和一次提交（Coordinator 经控制平面送进入口 stage）；响应聚合发生在 Client 层（累积文本、拼接音频、base64 编码），不在 HTTP 层；一条管线可有多个终态（如 decode 与 code2wav），Coordinator 须等全部期望终态完成才结束请求。本讲还引入两个术语：控制平面（ZMQ+msgpack 传命令/状态）与 relay（数据平面搬运大张量），细节留待 u3。后续 u2-l2～l5 将自外向内逐层下钻。
