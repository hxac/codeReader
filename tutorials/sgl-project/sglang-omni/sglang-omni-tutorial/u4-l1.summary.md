本讲打开调度器黑盒的最简实现 SimpleScheduler，它是非 AR 阶段（preprocessing、encoder、聚合等无 KV 缓存、「来一个算一个」的阶段）的默认调度器。调度器对 Stage 承诺固定的五件套接口：inbox、outbox、start、stop、abort，只要满足这套契约，Stage 就能驱动任意调度器而不必关心其具体实现。消息分两类：进入调度器的 IncomingMessage（type ∈ new_request / stream_chunk / stream_done）与离开的 OutgoingMessage（type ∈ result / stream / error，多出 target / metadata 字段供下游路由）。SimpleScheduler 最简主循环即 inbox.get → compute_fn(data) → outbox.put，在专用线程阻塞运行，仅依赖这两个轻量 dataclass，完全不碰 types.py 里 OmniScheduler 专用的重类型；调度器只负责算完放进 outbox，结果路由归 Stage 的 _drain_outbox。可选的 batch_compute_fn 为 GPU 编码器做无状态批处理（与 max_concurrency 互斥、须返回等长结果列表）；abort 因非 AR 阶段无重资源可即时打标记并触发回调清理，区别于 AR 阶段的延迟回收。
