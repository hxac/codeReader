SGLang-Omni 是面向 omni、语音、TTS 模型的多阶段（multi-stage）推理服务运行时。其核心动机是：一次生成由性质迥异的子模块接力完成，强行统一会牺牲性能或徒增复杂度，故拆成异构阶段、各自接入最适合的调度器。

框架拥有五大职责：管线拓扑、阶段生命周期、跨阶段传输、模型族集成层、OpenAI 兼容 API 面。一次生成被建模为协同的七大阶段类别——preprocessing、encoders、autoregressive engines、talkers、decoders、vocoder、aggregators，Qwen3-Omni 的真实 stage 与之一一对应。阶段间构成带 fan-out / fan-in / stream 的有向图，一条管线可有多个 terminal 终态（文本收口 decode、语音收口 code2wav），由上层 Coordinator 合并。

与上游 SGLang 是「composing with」关系：Omni 负责拓扑、生命周期、传输、集成与 API，仅在适用处复用 SGLang 提供的高性能 AR 调度与执行；预处理、聚合、流式 vocoder 等非 AR 阶段由 Omni 自管。请求主链路为：HTTP API → Client → Coordinator → Stage → Scheduler → ModelRunner → model forward。被服务模型分 omni、TTS、ASR 三类，如 Qwen3-Omni、Higgs Audio v3、Qwen3-ASR。
