本讲义讲解内部 Client 层——Coordinator 之上不碰 GPU 的薄门面，只做协议无关的「请求翻译 + 结果聚合」。低级 generate 是唯一底座：用 uuid 兜底全局 request_id，把请求翻成三段式（inputs/params/metadata）的 OmniRequest 提交 Coordinator；三个高级方法只是它的不同消费——非流式累积、流式透传、speech 强制非流式。纪律：多模态媒体（images/audios/videos）落入 inputs，OpenAI 的 max_tokens 覆盖 max_new_tokens，metadata 补 model/output_modalities；非流式路径由 builder 把多 terminal（文本 decode 与音频 code2wav）合并成单条 GenerateChunk，再以文本 join、音频 np.concatenate、标量取末值拼成 CompletionResult。最后一公里在 audio.py：to_numpy 统一 float32、encode_audio 按 wav/pcm/mp3 等分派并带 WAV 回退、audio_to_base64 嵌进 JSON。核心认知：Client 是翻译+聚合层，真正算东西的是更内层的 Scheduler/ModelRunner。
