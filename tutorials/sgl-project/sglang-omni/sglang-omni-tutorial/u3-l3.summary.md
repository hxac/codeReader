本讲讲清 SGLang-Omni 数据平面（relay）与控制平面的分工：前者传大张量，后者传小命令，仅靠 DataReadyMessage.data_ref（携带 metadata）与回程 DataAckMessage 协作。核心抽象是 Relay（put_async/get_async/cleanup/close）与 RelayOperation：其 metadata 是发送方写给接收方的「取货单」，两端只靠这一个字典耦合。CUDA IPC 后端用发送侧有界 GPU 槽池——池子按 slot_size 切成等大 slot，张量按 ceil(size/slot_size) 占连续 slot，靠 interprocess CUDA Event 跨进程同步读写。资源生命周期严格一一对应：一次 put_async ⇄ 一个 object_id ⇄ 一条 DataAckMessage ⇄ 一次整段 release(offset,num_slots)，故 slot 粒度（仅用于池内碎片管理）与整段释放并不矛盾；ACK 超时走 fail_cb 而不释放，以免数据损坏。后端（cuda_ipc/shm/nixl/mooncake）的选择由 CommRouter 派生，留待后续讲义。
