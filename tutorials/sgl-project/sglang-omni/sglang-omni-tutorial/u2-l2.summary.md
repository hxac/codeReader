本讲下钻 SGLang-Omni 的 HTTP API 层，核心结论是：HTTP 层是纯翻译搬运层，不碰 GPU。FastAPI 工厂 create_app 只负责建 app、装中间件（CORS、上传体积限制）、把 client 注入 app.state 并挂载一串路由，而拉起管线、跑 Uvicorn 归 launch_server（承接 u2-l1）。路由分四类：OpenAI 标准 chat（/v1/chat/completions）、标准 audio（speech/transcriptions/models/health）、omni 扩展（generate/batch/ws/voices/realtime）与 admin 控制；/health 按运行时 running 返回 200/503，/v1/models 只回吐启动时定死的单模型名。请求转换 _build_chat_generate_request 将外部 ChatCompletionRequest 译成内部 GenerateRequest：做字段重命名与 stop 规范化，把 images/audios/videos 等多模态引用塞进 metadata，并用 explicit_generation_params（基于 Pydantic model_fields_set）区分「未传」与「传了默认值」，避免静默覆盖模型默认值；转换放 HTTP 层是为了把外部协议脏活封死在边界。流式 _chat_stream 遵守 SSE 四语义：首 chunk 仅发 role、文本与音频分离、末尾空 delta 加 finish_reason、最后 [DONE]，并按 requested_modalities 过滤，失败时不发 [DONE]；聚合与 base64 编码已下沉到 Client 层。关键术语 create_app、app.state、GenerateRequest、metadata、extra_params、explicit_generation_params、SSE，供 u2-l3 Client 层承接。
