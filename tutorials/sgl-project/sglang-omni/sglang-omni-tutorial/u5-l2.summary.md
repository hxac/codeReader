本讲将 stage 抽象与调度器落到 Qwen3-Omni。它有两套 stage 列表：text（6 阶段，单 terminal `decode`）与 speech（8 阶段，双 terminal `decode`+`code2wav`），由三个 `PipelineConfig` 子类（text/speech/speech-colocated）经同一 `_speech_stages` 工厂、仅凭默认参数区分形态。

Qwen3-Omni 是「双 AR」omni 模型：thinker（大 AR，读多模态输入产文本）与 talker_ar（小 AR，把 thinker 的 hidden state 转成语音 codec 码本，再交 code2wav vocoder 合成波形）。

核心认知是三类边的区分：`next`/`route_fn`（最终结果路由）、`stream_to`（每步流式 chunk）、`stream_done_to_fn`（流结束信号）。路由采用「静态全集+请求感知子集」：preprocessing 经 `route_fn` 按模态扇出（纯文本跳过 encoder），mm_aggregate 经 `wait_for_fn` 定 fan-in 等待集合，返回 `None` 表示暂挂。

thinker→talker 用流式+partial-start 重叠「思考」与「说话」以降低首音频时延：mm_aggregate 提前把 prompt+thinker_inputs 投影给 talker 建 prefill，talker 攒够 `partial_start_min_chunks` 即起步；chunked prefill 期间抑制流以防泄漏。拓扑分离式（thinker/talker 异卡、开 partial-start）与共置式（五 GPU 阶段同卡、关 partial-start）由 `Qwen3OmniPlacementPolicy` 校验，共置式须为五个 GPU 阶段声明 `total_gpu_memory_fraction`。
