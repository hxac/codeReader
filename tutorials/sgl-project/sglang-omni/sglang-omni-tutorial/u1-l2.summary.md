本讲义覆盖四个最小模块：Docker 安装路径、uv 与可编辑安装、依赖版本约束、CLI 入口 sgl-omni。核心结论是 SGLang-Omni 提供 Docker（推荐）与手动两条路径——官方镜像 lmsysorg/sglang-omni:dev 以 SGLang 镜像为基底额外编译 UCX 1.20.x，体现 u1-l1 所述「与 SGLang 组合」的关系；手动路径则需自行编译 UCX 与 flash-attn-4。无论哪条路径，Python 侧统一用 uv 建虚拟环境（推荐 3.12）并以可编辑模式安装，使源码改动即时生效。

引入的关键术语：pyproject.toml 把 torch==2.11.0、sglang==0.5.12.post1、transformers==5.6.0、flash-attn-4、kernels 锁成必须整体升级的紧耦合「栈」；跨阶段传输依赖 nixl-cu13 与 mooncake-transfer-engine-cuda13 必须点名 CUDA 13（cu130）专用 wheel，否则会 import 崩溃；可选依赖 [audar-tts] 按需加载，[tool.uv] 的 override-dependencies 用于兜底传递依赖冲突。CLI 入口 sgl-omni 由 [project.scripts] 声明、Typer 实现，含 serve、config 子命令，sgl-omni --help 是最直接的安装冒烟测试。
