本讲讲解 OmniScheduler 如何以「组合而非继承」复用上游 SGLang 的 AR 调度：经 `__getattr__` 委托加 `types.MethodType` 绑定，只借上游的 batch 选择、内存检查与结果处理算法，用空桩堵住 tokenizer/detokenizer 管线，对 Stage 仍保持与 SimpleScheduler 一致的五件套契约（inbox/outbox/start/stop/abort）。`create_sglang_infrastructure` 一次性建好七元组基础设施（model_worker、tree_cache、req_to_token_pool、KV 分配器、prefill/decode 管理器、model_config），prefill 与 decode 经 `on_retract` 回调缝成自修复闭环。`build_sglang_server_args` 是最薄翻译层，固化共享默认，模型专属语义走 adapter 回调注入。分工边界为：SGLang 管选 batch、算内存、管 KV、更新状态；omni 管跑哪个 model_runner、请求对象（`_omni_data`）与流式输出。最后 async decode 快路径下的 `_drop_stale_overrun` 须手动 reslice extend/mixed batch 的 per-token 字段，因为 SGLang 的 `filter_batch` 只懂 per-row 布局。
