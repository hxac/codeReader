本讲承接 Stage 抽象，讲清 SGLang-Omni 的「控制平面」：它与搬运大张量的数据平面（relay）显式分离，用 ZMQ + msgpack 传递小而快的命令状态，消息只带「指针」不带张量本体。点对点的 Submit / DataReady / DataAck / Complete / Stream / Shutdown / Admin / Profiler 走 PUSH/PULL，拓扑约定「接收方 bind、发送方 connect」，端点为 ipc://；只有 AbortMessage 走 PUB/SUB 广播——中止信号必须送达全体 stage（Coordinator 无法廉价定位请求当前位置，且 streaming/fan-out 时请求可能同时在多个 stage 活跃），PubSocket.bind 后 0.1s sleep 用于缓解 slow-joiner。线格式统一为 to_dict()（带 type 标签）→ msgpack.packb，接收端 unpackb → parse_message 按 type 分派。关键术语：控制平面与数据平面、bind/connect 惯例、DataReadyMessage.data_ref 的「双重身份」（直接 CUDA IPC 信封 vs relay DataRef 字典）、DataAckMessage 的 object_id 缓冲释放，供 u3-l3 数据平面讲义承接。
