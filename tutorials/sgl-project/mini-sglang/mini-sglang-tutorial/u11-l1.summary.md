本讲介绍 Mini-SGLang 离线推理入口 LLM 类与基准度量方法。LLM 直接继承 Scheduler，靠 offline_mode=True 使 I/O 层 SchedulerIOMixin 提前 return、跳过全部 ZMQ 队列创建，主循环 run_forever 一字不改地复用。离线通路只重写两个钩子：offline_receive_msg 从 pending_requests 取 UserMsg、按 prefill_budget 分波，并在空闲且无待处理时抛 RequestAllFinished 异常终结死循环；offline_send_result 把每 token 累积进 status_map。generate 捕获该异常后按 uid（等于下标的契约）聚合文本，返回 text 与 token_ids。离线基准度量聚合吞吐（总输出 token / 纯计算墙钟时间），ignore_eos=True 时用 max_tokens 预算否则数实际 token，warmup 用于隔离懒初始化。在线基准用 HTTP 客户端按 trace 时间戳回放请求，度量 TTFT/TPOT/E2E 分位，scale_traces 调节到达节奏模拟负载；两者吞吐不可横比。page_size 与 overlap 开关是离线吞吐的两个主要可调维度。
