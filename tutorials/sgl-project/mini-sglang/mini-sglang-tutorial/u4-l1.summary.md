本讲剖析 Scheduler 主循环：周而复始执行「收消息 → 调度下一批 → 前向 → 处理上一批结果」，入口为 run_forever（@torch.inference_mode 死循环）。核心是 Overlap Scheduling：用 scheduler stream（self.stream）跑 CPU 调度与元数据准备、engine stream 跑模型前向，把「算当前批」与「处理上一批」重叠，使每轮耗时从 T_cpu+T_gpu 降为约 max(T_cpu,T_gpu)。overlap_loop 把本轮 ForwardData（ForwardInput 与 ForwardOutput 成对）作为下一轮 last_data 接力，并靠两次反向同步（wait_stream 等元数据、copy_done.synchronize 等 token 拷回）保证正确性；ForwardInput 作为保活容器把张量生命周期延长到跨越 stream 边界，避免 IMA。normal_loop 是不缓存、不交错的串行回退，由开关 MINISGL_DISABLE_OVERLAP_SCHEDULING=1 强制启用，便于调试对照。
