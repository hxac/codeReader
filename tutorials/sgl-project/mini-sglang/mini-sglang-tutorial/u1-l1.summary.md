Mini-SGLang 是上游 SGLang 的精简实现，以约 5000 行 Python 同时作为高性能 LLM 推理引擎与透明可读的教学参考，处在应用与裸 GPU 之间，负责请求排队、显存管理、多卡切分与算子调用。本讲确立五大核心特性：Radix Cache（基数树复用共享前缀 KV）、Chunked Prefill（长 prompt 切块防 OOM）、Overlap Scheduling（CPU 调度与 GPU 计算重叠以隐藏延迟）、Tensor Parallelism（张量并行）、Optimized Kernels（集成 FlashAttention/FlashInfer）。前三者重在省时省显存藏延迟，后两者重在堆算力，大多默认开启并有 --cache、--attn、--tp 等 CLI 开关。引入的关键术语包括 LLM 推理、KV Cache、Prefill/Decode 两阶段、张量并行与 CUDA kernel。项目要求 Python ≥3.10，高性能依赖外部 CUDA 库 sgl_kernel、flashinfer-python、apache-tvm-ffi、quack-kernels，辅以 torch、transformers、fastapi、pyzmq、msgpack；支持 dense 架构模型 Llama-3、Qwen-3（含 MoE）、Qwen-2.5，平台仅限 Linux + NVIDIA GPU；源码集中于 python/minisgl/ 下 14 个按职责划分的子包。
