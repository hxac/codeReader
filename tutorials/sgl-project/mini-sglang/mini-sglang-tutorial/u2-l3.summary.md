本讲聚焦 `python/minisgl/message/` 包，解决进程间消息如何编码与解码这一单一问题。核心内容：将消息按目的地分为 **backend / frontend / tokenizer 三族**（基类 `BaseBackendMsg`/`BaseFrontendMsg`/`BaseTokenizerMsg` 及打包类 `BatchXxxMsg`），三族沿请求生命周期经 `uid` 串成环——`TokenizeMsg`、`UserMsg`、`DetokenizeMsg`、`UserReply` 依次流转。序列化引擎是 `message/utils.py` 的 `serialize_type`/`deserialize_type`：靠 `__dict__` 反射递归压平 dataclass 为纯字典（dict/list/标量/bytes），新增字段免改代码，支持任意深度嵌套；用 `__type__` 键作类名路标。`torch.Tensor` 先转 numpy 再 `tobytes()` 降级为 `buffer`+`dtype`，**只支持 1D**（多维重型张量走 NCCL）。反序列化靠 `cls_map = globals()` 按类名查类重建，由此带来隐形契约：被嵌套引用的类须在 decoder 所在模块 import 到。encoder/decoder 作为回调注入 `ZmqPushQueue`/`ZmqPullQueue`，与 msgpack、ZMQ 串成链路，`use_bin_type=True` 与 `raw=False` 须成对使用；多 rank 广播用 `get_raw`/`put_raw` 搬原始字节省去重复编解码。
