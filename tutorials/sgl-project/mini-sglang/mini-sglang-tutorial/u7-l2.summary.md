本讲打开 u7-l1 留下的 wrapper.run 黑盒，讲清 FlashInfer 后端如何用 paged attention 算一个 batch 的注意力。FIMetadata 把 batch 翻译成 CSR 描述符：cu_seqlens_q/k 作前缀和（即 indptr），indices 是逐 token 槽位指针，硬约束 page_size 等于 1，与池存储 page_table 的下标对齐。prepare_metadata 读 Req 的 extend_len/device_len/cached_len 构造 q-indptr，分 decode（走 arange）、纯 prefill（复用 k-indptr）、extend（独立 cumsum）三分支，纯 CPU 拼数组可与上一批前向重叠。forward 遵循懒 plan 仅第一层、store_kv 先写池、wrapper.run 后读池的定序，用 last_event 防止异步 H2D 覆盖复用的 host 缓冲；plan 一次、run 每层一次，把规划开销摊薄到 1/num_layers。CUDA Graph 须用专用 CUDAGraphBatchDecodeWithPagedKVCacheWrapper，把 plan 结果写进地址固定、内容可变的固定 GPU buffer 方可录制，prepare_for_replay 每步换上对应 bs 的 wrapper 并重新 plan。
