本讲实现 `BasePrefixCache` 的真正实现 `RadixPrefixCache`：用一棵**基数树（radix tree）**把多条请求共享的 token 前缀压缩存储。核心数据结构是 `RadixTreeNode`，每个节点保存一段 token 序列（`_key`）及其对应的 KV 池槽位下标（`_value`），孩子表按「该段 token 的第一个 page」索引。

匹配与插入共用 `_tree_walk`：每次按 page 对齐推进，用 `align_down(common_len, page_size)` 对齐命中长度，遇到「只命中节点一部分」就调 `split_at` 把节点分裂成命中前缀节点与未命中后缀节点。`ref_count` 把节点分成两桶——大于 0 为受保护（protected）、等于 0 为可淘汰（evictable），分别由 `evictable_size`/`protected_size` 计数；`lock_handle` 沿路径上溯到 root 逐节点调整，只改尺寸计数器、不动树结构。淘汰 `evict` 用 `heapq` 最小堆按 `timestamp` 做 LRU，只摘 `ref_count==0` 的叶子节点，并在父亲变为新叶子时级联入堆，返回被回收的 KV 槽位下标。基数树是纯 CPU 的「索引层」，不持有 KV 张量，真正的数据在池 `MHAKVCache` 里，淘汰即把下标还给 `CacheManager` 的 `free_slots`。
