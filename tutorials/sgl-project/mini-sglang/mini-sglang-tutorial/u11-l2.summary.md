本讲梳理 Mini-SGLang 的测试体系，回答「改了代码怎么知道没改坏」。`tests/` 分三类：序列化往返（`misc/`，CPU）、端到端与 cache 完整性（`core/`）、kernel 正确性（`kernel/`），运行代价从纯 CPU 到需多卡 GPU 递增；其中 `test_cache_allocate.py` 是唯一不依赖 GPU 的单元测试，是本讲重点。核心术语有：**不变量**（invariant，性质恒成立）、**测试发现**（pytest 自动收集 `test_*`）、**序列化往返**（编码再解码验证无损）、**端到端/冒烟测试**（spawn 真实 Scheduler 子进程，主进程用 ZMQ 直接驱动、绕过 tokenizer/api）。最关键的被测方法是 `CacheManager.check_integrity`，它检测两个不变量：**页计数守恒**（空闲页 + 已缓存页 = 总页）与**页对齐**（`free_slots` 元素为 `page_size` 倍数）；该方法仅在 `run_when_idle` 里被调用，即 scheduler 空闲进入 blocking 接收时——此时无在途计算、账目稳定，是低成本高信噪比的清点时机。配置上，`pyproject.toml` 的 `[tool.pytest.ini_options]` 控制发现规则并强制覆盖率统计；项目并存「pytest 风格」与靠 `call_if_main` 的「脚本风格」两种测试，是否需要 GPU 决定运行方式。
