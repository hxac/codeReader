本讲聚焦主进程中的 FastAPI 前端 api_server.py——用户唯一直接打交道的 HTTP 入口。它只做协议翻译、不做推理：把 OpenAI/chat/generate 请求统一翻成 TokenizeMsg，把增量 UserReply 包回 OpenAI 格式；`/v1/models`、`/health` 这类路由不进流水线。核心是 FrontendManager：用全局自增 uid 配两张表——ack_map（收件箱，缓存增量回复）与 event_map（asyncio.Event 门铃）。listen 死循环按 uid 分拣回复、先 append 再 set，wait_for_ack 先 clear 再 drain 批量取走，二者配合使多次 set 合并成一次唤醒却不丢任何消息，破解了异步、多用户交错、增量分批到达的回复配对难题；其中 Event 只是唤醒信号、列表才是真正缓冲。同一 wait_for_ack 又分出两条返回路径：流式经 stream_chat_completions 边到边发 SSE（首块带 role、末尾补 finish_reason=stop 与 [DONE]），非流式累加 full_content 返回单个 JSON（usage 固定为 0）。客户端断连时，流式路径包 stream_with_cancellation，每块前用 request.is_disconnected() 探活，断连即抛 CancelledError 并异步 abort_user 发 AbortMsg(uid) 通知 scheduler 停算；非流式路径当前未接入断连检测。启动入口为 run_api_server。关键术语 TokenizeMsg/UserReply/AbortMsg、new_user/send_one/listen/wait_for_ack 供后续讲义承接。
