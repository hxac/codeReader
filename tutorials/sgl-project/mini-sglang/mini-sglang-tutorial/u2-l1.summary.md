本讲精读 `python/minisgl/core.py`，定义贯穿调度器、引擎、注意力后端、KV cache 与模型层的四个 dataclass，作为全系统的公共词汇表。`SamplingParams` 是携带用户采样意图的被动数据袋，派生属性 `is_greedy` 让引擎快速走 argmax 快路径。`Req` 是一条请求在调度/引擎侧的全部分身，本质是一部长度计数器：核心字段 `cached_len`（已缓存）、`device_len`（逻辑游标）、`max_device_len`（上限）恒满足不变量 `0<=cached_len<device_len<=max_device_len`，派生量 `extend_len`（本轮新算 token 数，prefill 等于 prompt 长度、decode 恒为 1）与 `remain_len` 决定工作量与剩余预算；`complete_one` 先推进游标、`append_host` 再补齐 `input_ids`，形成「游标先走一步、数据随后补齐」的节奏，是 overlap scheduling 的微观基础。`Batch` 是多个 Req 的打包并带 `phase` 标签（`"prefill"`/`"decode"`），区分两种计算形状；其 `input_ids`/`positions`/`out_loc`/`padded_reqs` 由调度器填、`attn_metadata` 由注意力后端填。`Context` 身兼两职：进程级共享设施（`page_table`/`attn_backend`/`moe_backend`/`kv_cache`）持有者与「当前 batch」临时挂载点（`forward_batch` 上下文管理器），并通过模块级单例 `_GLOBAL_CTX` 以「一写多读」拓扑串起全进程；两个断言（全局上下文只能设一次、`forward_batch` 不能嵌套）守护张量并行正确性。这些字段名将在 u4 调度器、u5 引擎、u6 KV cache 讲义中反复出现。
