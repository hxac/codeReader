本讲承接 u8 的单卡模型层，讲张量并行（TP）如何把权重拆到多卡。核心数学：列并行沿输出维（dim0）切，每卡算出不重叠的输出块、前向无需通信；行并行沿输入维（dim1）切，每卡只得部分和、前向后必须 all_reduce 求和。「先列后行」串联可把每段通信压成一次 all_reduce，每个 decoder layer 恰好 2 次。关键术语：rank/size、集合通信（all_reduce 求和、all_gather 拼接）、GQA（query 头多、KV 头少）。

最小模块：进程级单例 DistributedInfo 登记 rank/size（set_tp_info 只能调一次），div_even 按卡均分维度（allow_replicate 模式使 GQA 下 KV 头数少于卡数时复制而非切分）；TP Linear 族五个成员均继承公共基类 _LinearTPImpl，区别只在构造时 local 权重形状的算法与前向是否通信——列并行族（LinearReplicated/LinearColParallelMerged/LinearQKVMerged）不通信，行并行族（LinearOProj/LinearRowParallel）在 forward 里调 all_reduce；首部 VocabParallelEmbedding 用 all_reduce（屏蔽+求和复原），尾部 ParallelLMHead 用 all_gather 拼全词表 logits 需 permute 重排 rank 顺序。DistributedCommunicator 用插件栈抽象通信后端，永远调用栈顶 plugins[-1]；enable_pynccl_distributed 追加 PyNCCLDistributedImpl 即从 torch.distributed 切到自研 PyNCCL（默认开启，底层仍为 NCCL，靠 gloo CPU 组广播 UID 协商句柄），其更适合 CUDA Graph 回放，是 overlap scheduling 的通信拼图。
