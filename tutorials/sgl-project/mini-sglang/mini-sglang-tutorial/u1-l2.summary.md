本讲把 Mini-SGLang 从「特性」落到「能跑」。命令行入口是 `python -m minisgl`（项目未注册可执行命令），它执行 `__main__.py`，后者只调用 `launch_server`。`launch_server` 是核心编排器，职责为四步：`parse_args` 解析并归一化参数（唯一必填 `--model-path`，`auto` dtype 读 HF 配置，`--tensor-parallel-size` 转为 `tp_info`）；以 `spawn` 启动子进程——每个 GPU 一个 scheduler 进程，外加 tokenizer/detokenizer 进程，CPU 与 GPU 工作分离；通过 `ack_queue` 阻塞等待收齐 `num_tokenizers + 2` 条就绪消息后才开服，保证不接未就绪请求。默认监听 `127.0.0.1:1919`，提供 OpenAI 兼容的 `/v1/chat/completions`，可用 `curl` 拿到非流式 JSON。`--shell`（即 `--shell-mode`，靠前缀匹配生效）复用同一 `FrontendManager` 通路，强制单请求、单 batch、静默输出，在终端交互，`/reset` 仅清空本地 `history` 列表而不直接回收 KV cache。Dockerfile 采用多阶段构建，固定 `ENTRYPOINT ["python","-m","minisgl"]`，预设 HF/tvm-ffi/flashinfer 缓存目录，运行时需 `--host 0.0.0.0` 才能被宿主机访问。关键术语 `tp_info`、`ServerArgs`、`spawn`、`ack_queue`、`FrontendManager` 将在后续进程架构讲义中展开。
