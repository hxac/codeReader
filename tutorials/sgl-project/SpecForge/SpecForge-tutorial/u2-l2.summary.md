本讲解析 SpecForge run config 的结构。一次 run 的配置由七段（model/data/training/tracking/profiling/runtime/deployment）加两个顶层字段（run_id/output_dir）组成，其中只有 model 与 data 必填，其余五段全省略即用默认值。所有配置类继承 StrictConfigModel（extra="forbid"），未知字段直接报错而非被静默忽略。关键术语：数据模式 online/offline 是 Config.mode 推导属性，由 data.hidden_states_path 是否为空决定，需与显式字段 deployment.mode 区分；data 段的 train_data_path/prompts_path/hidden_states_path 三选一，由 _exactly_one_source 强制。training.strategy 在 schema 层只是普通字符串，最终经 builtin_algorithm_registry() 与 registry.resolve(name) 按名查表，匹配到含 spec（契约）与 providers（可执行端口）的 AlgorithmRegistration；内置算法名为 eagle3/peagle/dflash/domino/dspark。另，role 默认按 deployment.mode 推导（disaggregated→auto，否则→all）；draft_checkpoint_path（仅权重热启动）与 resume_from（完整续训）互斥。
