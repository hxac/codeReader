本讲回答「改了代码后如何确信没把别处搞坏」，构建 SpecForge 的测试与质量门禁体系，含三块。其一，测试网格全用标准库 unittest、由 `unittest discover` 发现，`tests/test_*` 目录镜像源码包（改某源码即知对应测试目录），通用范式是「工厂函数造最小样本→断言不变量」；策略类测试多落在 test_runtime 而非 test_training。其二，端到端门禁（e2e gate）以单样本 overfit 到 loss≈0、精度 1.0 来证明 disaggregated 全链路数值正确，复用唯一训练入口，靠 `_e2e_common.sh` 管进程组生命周期与 cleanup trap，并支持 dry-run 零成本体检；叶子检查器 check_overfit_metrics.py 方法无关、可复用。其三，两道质量闸：本地 pre-commit 自动修复（含 no-commit-to-branch 守护主干），CI 的 lint.yaml 与 GPU test.yaml（实捕 gate + 全量 unittest）只验证不修复。关键结论：新增算法至少补 test_algorithms（契约/注册）、test_application（组合根）、test_runtime（策略）三类测试。
