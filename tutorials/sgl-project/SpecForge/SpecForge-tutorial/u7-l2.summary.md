本讲放大四平面分工中的「控制面」，只依赖标准库、刻意不导入 torch，核心是 `specforge/runtime/control_plane/` 子包。最小模块有四：`DataFlowController` 是只搬元数据、绝不碰张量的调度器，提供 prompt 注入/租赁/重试、`SampleRef` 提交去重、durable ack 三类原语，无运行循环、不调用 rollout 或 trainer，每个接受 record 的方法都跑 `assert_no_tensors`。`MetadataStore` 是可替换账本接口，三种后端对应三种拓扑：`NoOpMetadataStore`（colocated/在线 producer，不记账）、`InMemoryMetadataStore`（进程内记账、非权威 DP rank）、`SQLiteMetadataStore`（在线 consumer 唯一持久权威，WAL + synchronous=FULL 保证 ack 能扛断电）。`DPAckController` 继承 controller，把 `ack_train_refs` 改造成一次 DP collective：gather 全秩 id 并集后只让 rank0 单一权威写一次账本，再广播结果，清理也做第二次全秩 gather。核心契约两条：freshness 契约要求每个新 consumer 用全新 SQLite 路径（拒绝复用 db/wal/shm）；恢复契约为 consumer-only，跳过已 ack 前缀、重放未 ack 尾部。最关键的硬约束是「durable ack step 必须等于 checkpoint step」——ack 与 checkpoint 是两份独立持久事实，错位即 fail closed，绝不拿旧 optimizer 状态重放已消费 ref。本讲引入的关键术语包括：控制面/数据面边界、元数据原语、账本（ledger）、去重新鲜度、durable ack、optimizer marker、rank0 单一写权威、quantum、freshness/恢复契约。后续讲义将承接去「发货」侧：u7-l3 数据平面 feature store 与传输、u7-l4 在线引用分发与流式队列。
