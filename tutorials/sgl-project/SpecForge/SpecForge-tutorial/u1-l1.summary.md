SpecForge 是 SGLang 团队出品的投机解码草稿模型训练框架，定位为「训练 + 导出」：用目标模型当老师训练草稿模型，再用 `specforge export` 物化成 SGLang 可直接加载的服务目录，加速推理交由 SGLang 承担。它有三个核心承诺：持续维护开箱即用、训练产物直接兼容 SGLang 无需移植、一套运行时统一支撑本地离线与服务端在线分离训练及多种并行拓扑，硬件上覆盖 CUDA/ROCm/Ascend。所有方法共用同一个类型化训练入口 `specforge train --config`，没有方法专属的 Python 入口，靠配置区分；当前支持 6 种草稿方法（EAGLE3 主力、P-EAGLE、EAGLE3.1、DFlash、Domino、DSpark），不支持的「方法 × 拓扑」组合在校验或装配阶段直接 fail-fast 报错。SpecBundle 是团队联合产业伙伴发布的生产级草稿模型集合（即「成品货架」），解决缺基础设施、缺高质量模型、训练规模不足三大痛点，配合 SGLang 可获最高约 4x 加速。关键术语：草稿方法与草稿模型、目标模型、接受率、统一运行时、再生数据集。
