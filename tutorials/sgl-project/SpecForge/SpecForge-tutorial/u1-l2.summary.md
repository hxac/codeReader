本讲把 SpecForge 从「认知」落到「能跑」，围绕三个最小模块展开。其一，依赖与入口：SpecForge 是 Python ≥ 3.11 的包（名 `specforge`，版本 `0.2.0` 由 `version.txt` 动态读取），`pyproject.toml` 是单一事实来源；核心依赖中 `torch==2.11.0`、`transformers==5.8.1`、`sglang==0.5.14` 三者钉死版本构成接口对齐「铁三角」，另有 `data/dev/fa/liger` 四个可选扩展；`[project.scripts]` 注册 `specforge` 命令指向 `specforge.cli:main`，内含 `train`、`export`、`benchmark` 三个子命令。其二，安装方式：官方推荐从源码安装（`uv venv -p 3.11` 后 `uv pip install -v . --prerelease=allow`），亦可 `pip install specforge`。其三，三类加速器差异只在装环境：CUDA 标准安装、ROCm 需先装含 ROCm 专属 wheel index 的 `requirements-rocm.txt`、NPU 需先装厂商 PyTorch 与 `torch_npu`；装好后三类硬件共用同一入口与 YAML，ROCm 复用 `torch.cuda` API、NPU 由运行时自动切换到 HCCL。本讲引入的关键术语包括 pyproject/extras/console script、铁三角三依赖、源码安装与 `--prerelease`、`requirements-rocm.txt`、`torch_npu`/HCCL 等，供后续讲义承接。
