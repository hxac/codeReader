本讲含三模块：命令行覆盖、配置加载校验、计划预览。覆盖用尾部参数 section.field=value 临时改值——apply_overrides 先把已校验 Config 拍平为字典，再逐层校验点分路径存在性（中间层须为 dict、末层 key 须已存在，否则抛 ValueError），标量原样写回、由末尾 model_validate 统一做强转与跨字段校验。未知字段走 fail-fast：YAML 端由 StrictConfigModel 的 extra=forbid 拦截，覆盖端由 apply_overrides 拦截；load_config 经 from_file、migrate_legacy_config 到 model_validate 是唯一加载链，旧部署字段仅在此边界翻译。--plan 复用真实训练前三步（load_config、resolve_run、build_launch_plan），打印 plan.render 后直接返回、不起进程不占 GPU，是零开销的配置与拓扑体检工具，并脱敏密钥环境变量、把覆盖原样转发进子进程 argv。plan.kind 取 worker、command、supervisor、managed_supervisor 四值，其判定细节与 resolve_run 的算法注册体系留待后续讲义展开。
