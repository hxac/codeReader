本讲核心是 `DraftTrainStrategy` 抽象基类——训练进程里唯一知道「batch 如何算 loss」的地方。`TrainerCore.train_step` 刻意保持「无分支」，只统一调 `strategy.forward_loss(batch, ctx)`，算法差异全收敛进策略插件。基类暴露五职责：`required_features`（声明所需张量）、`validate_batch`（前向守卫）、`forward_loss`（训练/评估通用算损失）、`checkpoint_state_filter`（存盘过滤草稿权重）、`trainable_module`（暴露可训练模块）；产物与训练进度由值对象 `StepOutput`、`StepContext` 承载。五个策略 EAGLE3/P-EAGLE/DFlash/DSpark/Domino 在所需特征、目标投影、loss 形态三维度上不同：EAGLE3 家族用 `_prepare_eagle_target` 投影目标（在线直用 logits、离线用冻结 `target_head` 重做位移与投影），其 loss 是 TTT 多步 `plosses` 按 `ploss_decay` 几何衰减求和；DFlash 家族用硬标签出单标量。存盘过滤各异——DFlash 家族只剥 `draft_model.` 前缀，EAGLE3 额外剔除冻结 embedding，P-EAGLE 因训练自身 embedding 完整保留。`StepContext` 只服务 Domino：`linear_lambda_base` 在前 `decay_ratio` 比例步数内将 `lambda_base` 从 `lambda_start` 线性衰减到 0，无 `total_steps` 时安全退化为 0。
