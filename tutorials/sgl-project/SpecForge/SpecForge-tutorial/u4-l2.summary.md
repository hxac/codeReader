本讲承接 u4-l1 的纯契约 `AlgorithmSpec`，回答「会动的代码放哪」：核心是 `AlgorithmRegistration` 的**双半绑定**——把纯契约 `spec` 与可执行端口 `providers` 绑成同一对象，避免维护两张会漂移的表。`providers` 字段类型刻意标注为 `object`，让 `registry.py` 保持只依赖 contracts 的叶子模块，规避循环导入。绑定经两道校验：`make_registration` 校验契约与 provider 的 `(mode, modality)` 键集合完全相等（parity），`__post_init__` 校验名字一致。

`AlgorithmRegistry` 是不可变（`frozen=True`）目录，构造时做类型校验、按名去重、按名排序，故 `names` 永远确定性字典序；`with_registration` 返回新对象而非原地修改。`resolve(name)` 线性扫描查表，未命中抛 `KeyError` 并列出全部合法算法名，组合根再转成 `ValueError`。`builtin_algorithm_registry()` 是工厂函数（非模块级单例），注册 eagle3/peagle/dflash/domino/dspark 五个算法，构造过程只存函数引用、懒加载重依赖、不 import torch，使 `--plan` 等无 GPU 场景也能安全完成解析。
