本讲是 SpecForge 主力草稿方法 EAGLE3 的原理总纲，统一目标是最大化接受率。它围绕三大特性展开：其一是**特征式草拟**——区别于「同家族小模型」token 进 token 出的路线，EAGLE3 让草稿模型吃目标模型的**隐藏状态**当输入、输出 token 概率（logits）供目标模型验证，草稿模型常仅 1 层，靠富输入而非大模型取胜。其二是**多层隐藏状态拼接**——从目标模型三个不同深度层（默认 `[1, L//2-1, L-4]`，即捕获层）各抽一份隐藏状态，拼成 `3d` 经线性投影压回 `d`，再拼 token embedding 得 `2d` 输入，提供多分辨率信息。其三是**训练时测试（TTT）**——训练循环让草稿连走默认 7 步、下一步吃自己上一步的输出，使训练贴近推理以缓解 teacher forcing 带来的**误差累积**。**动态草稿树**则属 SGLang 推理期机制，SpecForge 只负责训出 logits 足够准的草稿。本讲确立的特征空间、隐藏状态、捕获层、接受率、TTT、误差累积等术语，将贯穿后续 u4 算法契约、u5 特征捕获、u6 损失各讲。
