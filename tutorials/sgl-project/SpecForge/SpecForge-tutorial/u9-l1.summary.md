本讲讲解 SpecForge 训练进度的检查点与恢复，覆盖三个最小模块。一是检查点的磁盘布局与轮转：每个检查点是 `{run_id}-step{N}/` 目录，含 rank0 共享负载与每 rank 分片；保存采用 fork/回卷语义（存 step S 先删所有 ≥ S 的目录），轮转按 `max_checkpoints` 留最新 N 个，但 best 与当前步永远受保护；多 rank 保存为 collective，任一 rank 失败则全员抛错。二是两种刻意分开且互斥的操作：`training.resume_from` 恢复权重、优化器、计数器、数据位置与 RNG 以续训；`model.draft_checkpoint_path` 仅恢复草稿权重以热启动新 run。resume 须校验 strategy 与 world_size 一致，草稿权重在 FSDP wrap 前加载、优化器分片在 wrap 后加载。三是在线 disaggregated 的 consumer-only 恢复契约：复用留存 SQLite 账本、channel/inboxes 与 Mooncake 对象，durable ack step 必须等于 checkpoint step，错位即 fail-closed，producer 不重启。引入的关键术语有 global_step、durable ack、rewind/rotation、FSDP 分片与 warm start。
