本讲聚焦训练执行期的「包裹/反向/步进/累积边界」一环。`FSDPTrainingBackend` 是 SpecForge 目前唯一的训练后端，把模型包裹、反向、optimizer step、状态字典四件事收口，使 `TrainerCore` 保持无分支。其 `prepare_model` 按 `sharding_strategy` 在 FSDP（默认 `SHARD_GRAD_OP`、bf16 混合精度，可按 transformer 块自动包裹）与 DDP（`NO_SHARD`）间分流，在包裹**之后**才对草稿子模块创建优化器，冻结目标头经 `ignored_modules` 排除分片。

梯度累积由两点配合：`loss` 除以 `accumulation_steps` 把每个 micro-batch 的梯度缩到 1/N；`no_sync()` 让前 N-1 步只在本地累加梯度，仅在边界步触发一次跨卡 reduce，把通信降到 1/N。梯度范数与裁剪由 `BF16Optimizer` 跨 FSDP 进程组 all-reduce 平方和求得，DDP/未包裹时关闭归约，`backend.step()` 只是薄封装。

`optimizer_stepped = (_micro % accumulation_steps == 0)` 在反向之前唯一算出，是驱动 ack、检查点、评测、日志的单一权威边界信号；训练流自然结束必须落在 optimizer 边界，否则 `_fit` 抛 `RuntimeError`，宁可失败也不做半个 step 或谎报检查点成功。
