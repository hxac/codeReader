本讲围绕 SpecForge 唯一入口 `cli.py`，讲命令解析与 worker 进程生命周期。`main` 用 argparse subparser 分发 `train/export/benchmark` 三子命令；`train` 分支七步为 `load_config → resolve_run → build_launch_plan → (--plan?) → (plan.kind=="worker"?) → _train / run_commands`。`--role` 的 `auto/all/producer/consumer/both` 五值由 `launch_plan` 裁决，决定一个 worker「只捕获特征」还是「初始化分布式并训练」：`plan.kind` 为 worker 时进程内直跑 `_train`，否则充当 supervisor 走 `run_commands`。`_train` 按 role 分两路：producer 不初始化 CUDA、直接进组合根 `build_application_run`；trainer（all/consumer）先 `_bootstrap_single_process_env` 补齐 rendezvous 五变量、`init_distributed` 建进程组，用 `try/finally` 保证 `destroy_distributed` 执行并对 world_size 校验两次。`_config_for_role` 用 `model_dump + model_validate` 做角色投影，不改原配置（写 role、清空 managed_local、producer 关 profiling）。`_worker_signal_unwind` 把终止信号翻译成 `BaseException`（`_WorkerTermination`）使训练的 `finally` 清理正常执行，退出码 `128+signum`。引入术语：worker、supervisor、rendezvous、LaunchPlan、角色投影、组合根。
