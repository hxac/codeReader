本讲说明 SpecForge 的唯一训练入口 `specforge train --config <yaml>`：所有草稿方法（eagle3/dflash/domino/dspark/peagle）共用此命令，仅凭 `training.strategy` 区分，无方法专属脚本；`--plan` 可在不启动训练、不占 GPU 时预览进程计划。核心结论是分清两条正交轴线：数据模式（online/offline，由 `data` 段是否填 `hidden_states_path` 决定）与部署模式（local_colocated/disaggregated，由 `deployment.mode` 决定），且 online 强制 disaggregated + `target_backend=sglang`。`data` 段三个字段（train_data_path/prompts_path/hidden_states_path）必须且仅填一个。一份 run config 含 model/data/training/tracking/profiling/runtime/deployment 七段，外加 run_id、output_dir 两个顶层字段。训练进程数由 `nnodes*nproc_per_node` 决定，`world_size>1` 时以 torchrun 拉起多个 consumer rank；producer 仅负责特征捕获、不算 trainer，故 online 训练对应一个 supervisor 计划（含 producer 与 consumer 两条命令）。
