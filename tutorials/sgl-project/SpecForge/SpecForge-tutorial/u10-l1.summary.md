本讲讲解在 SpecForge 中新增草稿架构（draft architecture）。核心结论：草稿架构与训练算法是两条正交的扩展轴；新增架构只需「新建文件 + `@register_draft` 装饰器 + 在 `draft/__init__.py` 加一行 import」，无需修改 `modeling/auto.py`。

关键术语与机制：`DRAFT_REGISTRY`（架构轴注册字典，整条轴的唯一事实来源）、`@register_draft`（类装饰器，默认以类名为 key，做 `config_class` 必填与架构名唯一两道校验）、`config_class`（两用桥梁，同一注册条目同时提供模型类与配置类，支持子类继承）、`AutoDraftModel`/`AutoDraftModelConfig`（数据驱动总线，只凭 `config.architectures[0]` 查表解析，不硬编码架构名）、`Eagle3DraftModel`（抽象基类，定下 4 个抽象方法）。

两轴交汇于装配阶段 `_load_draft`：用算法要求的架构名 `resolve_draft` 取 `expected_type` 再做 `isinstance` 校验，实现「解耦但安全」；注册依赖 import 时机触发，`resolve_draft_config` 保证 config 里的架构名与算法要求严格一致。新增训练损失或流程属另一条轴（`AlgorithmSpec` 三件套），由 u10-l2 承接。
