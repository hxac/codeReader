本讲聚焦 SpecForge 四平面中的「推理面（inference plane）」——它只存在于在线 disaggregated 训练，目标模型作为独立的 patched SGLang 外部服务运行。核心不变量是：目标模型张量永不流经 controller 或 producer 进程。讲清三个最小模块：RolloutWorker 是算法无关的轻量编排器，按「lease → produce_refs/generate_features → commit」三段式把 prompt 变成已提交的 SampleRef，从不把张量交给 controller；SGLangServerCaptureAdapter 是零拷贝捕获引擎，服务端 prefill 后直接把张量写入 Mooncake 共享存储，HTTP 只回 key/shape/dtype 元数据，producer 据此拼出纯指针 SampleRef；CaptureConfig 契约与 verify_capture_specs 由算法流式 FeatureContract 推导，做缺特征、aux 层 id、aux 宽度、目标维度四道形状检查，把形状错误从「训练崩」提前到「捕获当场炸」。引入关键术语 SampleRef、零拷贝、Mooncake、adopt/abort、ServerCaptureSchema、target_repr（hidden_state/logits/pruned_logits）、FeatureSpec，供后续讲义承接。
