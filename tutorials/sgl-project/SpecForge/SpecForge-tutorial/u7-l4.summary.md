本讲聚焦 SpecForge 在线 disaggregated 训练中 producer 与 consumer 跨进程、纯元数据协同的三个最小模块。核心结论是：消费记账必须有单一权威。`RefDistributor` 住在 DP rank 0，是 producer 引用通道的唯一读者、durable SQLite 账本的唯一写者；它把去重后的 `SampleRef` 按 **quantum 窗口**（`dispatch_quantum = dp_size × batch_size × accumulation_steps`）拆成 `accumulation_steps` 轮、每轮每 rank `batch_size` 条轮询分发到各 rank 私有 inbox，保证 DP lockstep。关键不变量是「只在整窗口已本地 commit 时才开窗」，使 EOF 必落在窗口边界。**quantum 握手**：consumer 捕获前发布 quantum sidecar，producer 要求高/低水位都 ≥ quantum 否则 fail-fast，杜绝死锁。通道链为 `StreamingRefChannel`（跨进程追加写 JSONL + sidecar 计数器）→ `InboxChannel`（`.failed` 毒丸失败）→ `StreamingRefQueue`（阻塞 get、前缀 ack）。`DPAckController` 把 optimizer 边界 ack 变成两段 collective：`all_gather` 去重并集后 rank0 单一权威写一笔 SQLite 并广播结果，再各 rank 删自己分片特征并汇集清理错误；背压 `published - consumed` 中 consumed 仅在 durable ack 后推进——dispatch 不算消费。EOF 不足一个 quantum 的零头走 drop-last 式收尾，账本留 committed-but-unacknowledged，故带零头的成功 attempt 不可 resume。
