本讲是「分布式与并行」单元第一讲，核心源码在 `specforge/distributed.py`，只讲「初始化与拓扑建立」这一件事，TP/SP 的具体用法留给下一讲。

`init_distributed(timeout, tp_size, sp_ulysses_size, sp_ring_size)` 是训练面进入分布式的唯一入口，**只对 trainer 角色调用，producer 绕过**。它做四件事：选后端（CUDA/ROCm→nccl、NPU→hccl、CPU→gloo，靠 `get_device_type()` 自动探测，ROCm 复用 `torch.cuda` 故仍走 nccl）、绑本机 LOCAL_RANK 卡、调 `dist.init_process_group` 用 `env://` 会合、再用并行度切子组建网格。

它一次建出**两张设备网格**：主网格 `(dp_size, tp_size)`（维名 dp/tp）和草稿网格 `(draft_dp_size, sp_size)`，其中 `sp_size = sp_ulysses_size × sp_ring_size`。沿各维取出**五类子组**——TP、DP、draft_dp、draft_sp，外加 yunchang 切出的 Ulysses（头维切分）与 Ring（序列段环形传递）两个 SP 子组，全部存进模块级全局并配 `get_*` 访问器。

核心硬约束是两条整除：world_size 必须同时被 tp_size 和 sp_ulysses_size×sp_ring_size 整除，由 `init_distributed` 断言与 `schema.validate_world_size` 双重把关。关键算子包括无通信本地分片 `shard_tensor`（TP 权重加载）、`gather_tensor` 及带反向的 `Gather`（反向乘 sp_world_size 做梯度缩放，使 SP 与 DP 梯度尺度等价）。引入术语：进程组、设备网格、TP/DP/SP、USP（Ulysses×Ring 复合序列并行）、draft_dp/draft_sp 组，为下一讲 USP 与 ring attention 承接。
