本讲是 SpecForge 的物理直觉基础，解释投机解码为何能加速大模型推理，含三个最小模块：延迟瓶颈分析、草稿/目标协作、验证与拒绝采样。核心结论：逐 token 解码延迟主要被显存带宽（memory-bound）卡住，这是投机解码成立的物理前提——因验证同样 memory-bound，并行验证 N 个 token 的耗时近似于生成单个 token。工作流分 prefill（目标模型对 prompt 预填充）、drafting（草稿模型自回归连猜 N 个候选 token）、verification（目标模型并行验证，遇首个被拒 token 即终止本轮，从最后一个被接受处重新开始）三阶段。引入术语：目标模型、草稿模型、接受率、拒绝采样、残差分布、接受概率 r(x)=min(1, p(x)/q(x))。最终保证：目标模型按拒绝采样决定接受并在被拒时从残差分布补采，使输出分布与单独使用目标模型完全一致，即加速但不改变结果；草稿质量直接决定接受率与加速比，这正是 SpecForge 专门训练草稿模型的根本原因。
