本讲补全 SpecForge 训练数据的准备环节，含三个模块。其一，训练入口只认两种 schema：对话格式（`id`+`conversations`，消息带 `role`/`content`）与预格式化文本格式（`id`+`text`），均须有 `id`；关键结论是「预格式化不免除 `chat_template`」——设 `is_preformatted: true` 时模板用途从渲染转为定位 assistant 区间、切 loss mask，此点 u5-l2 承接。其二，`scripts/prepare_data.py` 用 19 个预设将各类开源数据集归一化为统一 JSONL，写盘幂等，`--split-eval` 以 `seed=42` 确定性切分，重依赖懒加载。其三，数据再生（regenerate）保留 user、用目标模型重写 assistant，使训练数据对齐目标分布、提升草稿模型接受率，属锦上添花而非必需；靠多 SGLang server 并发线性加速，支持 `--resume` 与 `--reasoning none/save/disable` 三模式，产物分 `_regen`/`_error`/`_skipped` 三类，校验强制 user/assistant 交替、内容非空、无 `<think>` 残留。本讲产物供 `data.train_data_path` 使用，模板渲染、loss mask、离线特征留待 u5-l2、u5-l3。
