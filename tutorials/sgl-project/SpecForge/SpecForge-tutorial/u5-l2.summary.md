本讲承接 u5-l1，说明原始对话如何变成训练可消费的 (input_ids, loss_mask)。三层分工：parse.py 出 tensor（算法层）、preprocessing.py 做 HF Dataset map（编排层）、prompt_builder.py 产出无 tensor 的控制面任务并嗅探已分词/原始两条路径（装配层）。chat_template 在 SpecForge 里身兼渲染与定位两职，由极简数据类 ChatTemplate 与全局 TEMPLATE_REGISTRY 承载，行为交给 Parser（按 parser_type 分发 General/Thinking/GLM/Harmony）。loss mask 是与 input_ids 等长的 0/1 张量，仅 assistant 回复为 1，难点是字符区间到 token 区间的映射，GeneralParser 用「前缀重编码计数」法解决。核心结论：is_preformatted=true 只免除渲染、不免除定位，loss mask 仍依赖模板拼出的正则圈选 assistant 区间，故模板仍是必填项；train_only_last_turn 仅监督最后一轮，ignore_token 清零空 think 块，minimum_valid_tokens 按有效 token 数过滤样本。
