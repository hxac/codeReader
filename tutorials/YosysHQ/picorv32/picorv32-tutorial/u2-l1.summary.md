本讲回答「`firmware.hex` 如何从源码生成」，覆盖三个核心模块：工具链、链接脚本、二进制转 hex。PicoRV32 需要纯 RV32I[M][C] 交叉工具链——强调「纯」是因为发行版自带的库含 PicoRV32 不支持的 A/F/D 扩展；由四个 make 目标分别装入 `/opt/riscv32i[m][c]`，默认 `TOOLCHAIN_PREFIX` 指向 `riscv32i`。固件构建链为：`.c/.S` 经 gcc 链接成 `.elf`、`objcopy` 抠成 `.bin`、`makehex.py` 翻成 `.hex`。`sections.lds` 把所有节塞进从地址 0 开始的 96 KB 区间，并强制 `start.o` 排最前，确保复位向量落在 CPU 复位取指的地址 0。引入的关键术语包括交叉工具链、`-march`/`-mabi`、裸机程序、ELF/BIN/HEX 三格式、链接脚本、小端字节序与 `$readmemh`；最终 `.hex` 由测试台读入 128 KB 内存模型。
