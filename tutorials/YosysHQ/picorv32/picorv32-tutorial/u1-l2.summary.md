本讲是读源码前的热身，不涉及 CPU 内部实现，只回答「目录里有什么、怎么编译、怎么跑」。仓库核心 RTL 仅 `picorv32.v` 一个文件，内含 CPU 主体、AXI 与 Wishbone 两种总线变体、AXI 适配器及 PCPI 乘除法核，拷贝即用；其余目录分别承载固件、指令测试、基准、示例 SoC 与工具脚本。构建入口是根 `Makefile`，其 `test_*` 目标族遵循「运行规则 + 编译规则」两层结构，用 iverilog 把测试台（testbench）编译成 `.vvp` 再用 vvp 执行。核心结论：`make test` 实际驱动经 `picorv32_wrapper` 包装的 `picorv32_axi`，而非裸 `picorv32`（裸核仅在 `testbench_ez.v` 中实例化）；`make test_axi` 与 `make test` 共用同一 `.vvp`，仅多传 `+axi_test` plusarg；`make test_ez` 是唯一不依赖 RISC-V 工具链的入口。`.elf`/`.hex`/`.vvp` 均为 make 产物，可被 clean 清除。`picorv32.core` 是 FuseSoC（CAPI=2）打包文件，用 filesets/targets/parameters 以工具无关方式重新声明同一份 RTL 与参数，相当于「FuseSoC 版 Makefile」；其中 `COMPRESSED_ISA` 属编译期宏（vlogdefine），`axi_test`、`vcd` 等属运行时开关（plusarg）。
