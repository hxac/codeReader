本讲介绍 pyosys——libyosys 的 Python 绑定：import 时自动 yosys_setup()、退出时自动 yosys_shutdown()，与 u9-l2 的 C++ 库方式共享同一套生命周期与驱动函数。核心覆盖三个最小模块。其一，脚本驱动：Design() 显式建一个设计，run_pass(cmd, design) 发字符串命令最终汇入 Pass::call，与 shell/.ys 同源，却能同时持有多个互不干扰的设计。其二，遍历 RTLIL：generator.py 把 kernel/rtlil.h 的类机械翻译成 pybind11 绑定，Design/Module/Wire/Cell/SigSpec 可像 C++ Pass 一样读写，hashlib 的 dict/pool 被包成 Python 友好对象，标 ref_only 的 Cell/Wire 只能引用、须经 module 工厂方法接纳，且修改设计库会使旧引用失效。其三，Python pass：子类化 ys.Pass 实现 execute(args, design)，实例化即触发 init_register() 完成去中心化注册，但 pyosys 仅 Pass 与 Monitor 两类经 trampoline 支持虚函数覆写，故自定义命令只限这两类。三种扩展方式（C++ 插件 .so、C++ 库嵌入、Python 绑定）注册与驱动机制同构。
