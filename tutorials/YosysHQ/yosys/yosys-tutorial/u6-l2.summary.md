本讲讲解 `proc` 如何把行为级 `always` 块编译成门级网表，是核心综合流程的第二讲。关键起点是：`read_verilog` 并不直接生成 `$dff`/`$mux`，而是把 `always` 的语义封装为 `RTLIL::Process` 对象。Process 只有两个核心成员：决策树 `root_case`（描述「值」）和敏感事件列表 `syncs`（描述「时机」）。决策树由 `CaseRule`（分支，含 actions 和嵌套 switches）与 `SwitchRule`（判断）互相嵌套而成；敏感事件用 `SyncRule` 配合 `SyncType` 枚举描述，包括电平敏感 `ST0/ST1`、边沿敏感 `STp/STn`、恒有效 `STa`、全局时钟 `STg` 等。

`proc` 是一条编排型 `Pass`，自身不做算法，按固定顺序串联子 pass：`proc_clean → rmdead → prune → init → arst → rom → mux → dlatch → dff → memwr → clean → opt_expr`。其中三个核心子 pass 分工明确：`proc_mux` 通过 `SigSnippets` 切分被赋值位、逆序遍历决策树，把 `if`/`case` 翻译成 `$mux`/`$pmux` 网络（互不重叠的并行分支并入 `$pmux`，比较逻辑用 `$eq`/`$reduce_or`）；`proc_dff` 处理边沿事件，按有无时钟、异步复位、复位值是否常数生成 `$ff`/`$dff`/`$adff`/`$aldff`/`$dffsr`；`proc_dlatch` 处理 `STa` 事件，通过在 mux 网络中检测组合反馈环识别锁存器，生成 `$dlatch`/`$adlatch`，纯组合部分改写成普通连线。

核心结论：顺序敏感（`proc_arst` 必须先于 `proc_mux`、`proc_dlatch` 必须先于 `proc_dff`）；整条链跑完后 Process 被删除，设计变为纯门级网表，即「process 消失」。本讲产物将交给下一讲 `opt` 进一步优化。
