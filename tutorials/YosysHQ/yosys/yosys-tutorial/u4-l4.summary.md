本讲把 driver 调度、日志系统与帮助生成三块拼成「从敲下命令到看到输出」的完整链路。日志方面，`log()` 一族函数都汇聚到核心函数 `logv_string`，把同一段文本广播到 `log_files`（控制台与 `-l` 文件）、`log_streams`（供 tee/help 截获）和 scratchpad 三类目的地；`log_push`/`log_pop` 维护一个 `header_count` 栈，`log_header` 据此生成 `1.`/`2.1.` 式层级标题编号；警告/错误走单管线，`-w` 降级、`-e` 升级为致命、`-W` 把普通日志转警告，`log_error` 标注 `[[noreturn]]` 最终 `_Exit(1)`。帮助方面，`help` 是一条普通 pass，`HelpPass::execute` 命中命令时调用该 pass 的 `help()` 实时生成文本；`PrettyHelp` 配合 `ContentListing` 以 RAII 全局栈式上下文叠加结构化层，既能 80 列折行渲染，也能 `to_json` 供 docs 站点。调度方面，`driver.cc` 的 `main()` 按「解析 → yosys_setup → 加载插件 → read -define → run_frontend → run_pass → shell 或 run_backend → 耗时统计」固定顺序编排，用单个布尔 `run_shell` 决定结尾进交互还是写文件；具体动作由 `yosys.cc` 的 `run_frontend`/`run_pass`/`run_backend`/`shell` 承担，按扩展名自动猜前后端。三个模块在 `Pass::call` 处交汇：`pre/post_execute` 维护的 `call_counter` 与 `runtime_ns` 喂给结尾的 `Time spent:` 统计，`execute`/`help` 里的 `log()` 走同一日志管道。
