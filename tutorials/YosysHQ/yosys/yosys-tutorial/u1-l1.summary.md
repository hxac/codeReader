本讲是 Yosys 全局导论，覆盖三个最小模块：项目定位、文档结构、许可证与依赖。核心结论是：Yosys 是一个开源 RTL 综合框架（framework），把行为级/RTL 的 Verilog、SystemVerilog 翻译成门级或物理门级网表，不做布局布线。其核心理念是「组合 pass（变换算法）完成任意综合任务」，可用 C++ 扩展新 pass。一次综合的数据流为前端（读 HDL，如 read_verilog）→ 一串 pass 变换内部表示 RTLIL（proc/opt/techmap/abc 等）→ 后端（写出网表）；synth 与 prep 是两条内置默认脚本（ScriptPass），分别面向通用门级综合与 SMT 形式验证的字级综合。需辨析职责：Yosys 做综合，Verilator/iverilog 做仿真，官方建议综合前先用 Verilator lint，三者互补。文档以 .rst 存于 docs/source/，根目录 index.rst 的 toctree 分 introduction/getting_started/using_yosys/yosys_internals 四大块，综合详解在 using_yosys/synthesis/，并分粗粒度与细粒度两阶段。Yosys 主体采宽松 ISC 许可证（可闭源商用）；SystemVerilog 支持依赖 sv-elab + slang（git submodule），abc/、libs/ 等第三方组件各有许可。本讲引入并供后续承接的关键术语：HDL、RTL、综合、pass、前端/后端、RTLIL、techmap、synth/prep、网表、ISC 许可证、sv-elab/slang。
