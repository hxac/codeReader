本讲讲解 Yosys 工艺映射的最后一步：把高层参数化 `$` 单元（如 `$and`、`$mul`）下沉为单位宽门级原语（如 `$_AND_`）。核心有两条映射路径。`techmap` 是一台不懂任何 RTLIL 语义的「模板替换机器」：它加载映射库（默认 `+/techmap.v`），按 cell 类型匹配模板模块，把模板内容（线、子单元、连接）内联进当前模块并删除原 cell，循环至不动点；映射规则本身用普通 Verilog 模块编写，新增映射无需改 C++，体现了数据驱动的设计哲学。模板靠属性与特殊线网驱动 techmap：`techmap_celltype` 声明匹配范围，`techmap_simplemap`/`techmap_maccmap`/`techmap_wrap` 声明委托对象，`_TECHMAP_DO_*` 让行为级模板自动被 `proc`/`opt` 门级化，`_TECHMAP_FAIL_` 让模板有条件退出。需区分三个易混文件：`techmap.v` 是「实现库」（怎么造门），`simlib.v` 是高层 `$` 单元的仿真模型，`simcells.v` 是门级 `$_` 单元的仿真模型且是 `abc` 的输入约定，映射只读 techmap.v。`simplemap` 是写死在 C++ 里的快速通道，用逐位分解把位/逻辑/比较/多路/寄存器等机械单元直接拆成 `$_` 门，它既是独立命令也被 techmap 复用，但完全不处理算术单元。关键结论：`$mul`/`$add` 等算术单元只能走 techmap，其下沉链为 `techmap_wrap "alumacc"` → `$macc` → `techmap_maccmap`(maccmap) → `$and`/`$fa` → `$_` 门。在标准 `synth` 中，`alumacc` 位于 coarse 阶段，`techmap` 位于 fine 阶段（紧跟 `memory_map`）。本讲为下一讲 abc9 与 liberty 的「门级→工艺单元」映射奠定基础。
