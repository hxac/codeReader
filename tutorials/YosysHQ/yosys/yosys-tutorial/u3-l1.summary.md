本讲讲解在 Pass 中用 C++ 构造合法 RTLIL 网表的编程接口。核心规则：Wire/Cell 必须经 Module 接纳才合法，应使用工厂方法 `addWire/addCell` 创建，私有 `add()` 负责把对象塞入字典、写入反向指针 `module` 并断言名字唯一（wire/cell/memory/process 共享同一命名空间）。须严格区分两种连接：`module->connect(lhs,rhs)` 是 module 级 assign 赋值，存入 `Module::connections_`；`cell->setPort(name,sig)` 是给门的管脚接线，存入 `Cell::connections_`，门本身已表达逻辑关系故通常无需再 connect。Cell 由两套字典构成：`connections_`（端口名→SigSpec）与 `parameters`（参数名→Const），分别用 `hasPort/getPort/setPort` 与 `hasParam/getParam/setParam` 读写；`getParam` 找不到时会回退查 design 中同 type 模块的 `parameter_default_values`；`fixup_parameters` 可据端口宽度自动补齐参数；`NEW_ID` 用于生成唯一 `$` 自动名。属性系统 `AttrObject` 提供 `attributes` 表与 `set_bool_attribute` 等接口（设 false 等价于删除），Module/Wire/Cell 皆因继承 `NamedObject←AttrObject` 而具备属性能力。
