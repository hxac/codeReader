本讲是「高级内部机制」收尾篇，串讲三大工程化主题。**FSM 提取**：`fsm` 是编排型 Pass，按 `fsm_detect → fsm_extract → fsm_opt/opt_clean → fsm_recode → fsm_info → fsm_map` 串联；只识别 `$dff`/`$adff`，故需先 `opt -nosdff -nodffe`。核心是用 `ConstEval` 符号仿真沿 mux 树找状态码与控制输入，对未定义控制位做 0/1 二叉展开枚举迁移表，存入 `FsmData`（`copy_to_cell` 压平为 `$fsm` 单元参数）。抽象层优化包括删不可达状态、找 don't-care，重编码阈值 < 32 选独热否则二进制，最后 `fsm_map` 物化回 `$dff`+`$pmux`+`$eq`。**线程模型**：RTLIL 非线程安全，并行受 `Multithreading` 守卫约束——并发段只算不改，`autoidx++` 与新建 `IdString` 被禁。并行原语集中在 `kernel/threading`（`ParallelDispatchThreadPool`、`ShardedVector`、`ConcurrentQueue`），采用 fork-join + 屏障同步；线程数经 `YOSYS_MAX_THREADS`（运行时）与 `YOSYS_DISABLE_THREADS→YOSYS_ENABLE_THREADS`（编译期）两层控制，`opt_merge` 是真实范例。**测试体系**分两套：单元测试（GoogleTest + CTest，`tests/unit`，`test-unit` 目标）与功能测试（Makefile + iverilog，`test-vanilla` 目标）；部分功能测试目录由脚本随机生成。关键术语：FSM、状态编码（binary/one-hot）、符号仿真、屏障同步、守卫对象、分片容器。
