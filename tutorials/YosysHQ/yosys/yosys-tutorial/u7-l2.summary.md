本讲介绍 cxxrtl——Yosys 的 C++ 仿真后端。它本质是代码生成器而非序列化器：把 RTLIL 编译成可运行的 C++ 源码，再由普通编译器产出软件仿真模型。`write_cxxrtl` 先在 `prepare_design` 中跑 `hierarchy`/`flatten`/`proc` 把设计拍平成可顺序求值的平面网表，再由 `dump_design` 翻译成 C++。核心难点是求值顺序：FlowGraph 建数据流依赖图，Scheduler 用反馈弧启发式排序，WireType 把连线分为缓冲成员、局部、内联等类；RTLIL 对象按 Node::Type 分发，组合单元翻成内联表达式，触发器翻成 `posedge`/`negedge` 守卫赋值。生成代码依赖公共 runtime：`value<>` 为任意位宽右值，`wire<>` 为 curr/next 双缓冲左值，`module` 基类以 `eval()`/`commit()`/`step()` 构成 delta-cycle 仿真契约。`-O`（删线提速）与 `-g`（补可见性）正交；C API 稳定，C++ 接口不稳定。
