本讲回答两个问题：如何把 Yosys 源码构建成可执行程序，以及敲下命令后程序内部发生了什么。涵盖三个最小模块。其一是 CMake 构建：Yosys 强制 out-of-tree 构建（`cmake -B build`），使用 C++20；通过 `YOSYS_WITHOUT_*`/`YOSYS_WITH_*` 选项结合 `condition()` 宏，把「用户意愿 ∧ 依赖探测」合并为 `YOSYS_ENABLE_*` 编译宏，决定各特性是否编入，最终产出 `yosys` 可执行程序与 `libyosys` 库两个目标。其二是 `kernel/driver.cc` 中 `main()` 的调度流程：用 cxxopts 解析命令行，按「读前端文件 → 跑脚本 → 跑 `-p` 命令 → 进 shell 或写后端」的顺序调用 `run_frontend`/`run_pass`/`run_backend`/`shell`；关键变量 `run_shell` 决定结尾是交互还是写文件，`-V` 等纯查询选项在 `yosys_setup()` 之前就 exit，`-S` 是 `synth` 的快捷方式，`auto` 前端/后端按扩展名猜测。其三是 `yosys_setup()`/`yosys_shutdown()` 生命周期：预填 `IdString`、注册所有 pass、创建全局 `yosys_design`（一个 `RTLIL::Design`）。核心机制是 pass 的自动注册——每个 pass 定义为全局静态实例，构造函数把自己头插进 `first_queued_pass` 链表，`Pass::init_register()` 遍历后写入 `pass_register` 表，因此新增 pass 无需修改中心注册代码。引入的关键术语：CMake、out-of-tree、`YOSYS_ENABLE_*`、`libyosys`、cxxopts、`run_shell`、`yosys_setup`/`yosys_shutdown`、`pass_register`、`first_queued_pass`、`yosys_design`。
