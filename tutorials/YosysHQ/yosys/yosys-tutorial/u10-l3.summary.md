本讲覆盖 Yosys 模式匹配生成器 pmgen，分三模块：① pmgen 生成器——一个仅依赖标准库的 Python 脚本，把声明式 `.pmg` 编译成 header-only 的 `*_pm.h`（含 `setup()`/`run_*()`/`block_*()` 匹配器类），算法是带回溯的递归子图同构搜索；② `.pmg` 语言——将一条规则拆为「声明式查找 + 命令式动作」，`match` 块用 `select`/`index`（预建索引、O(1) 查表）/`filter`（逐候选慢路径）描述目标子图，`code` 块用普通 C++ 加 `accept`/`reject`/`branch` 控制回溯与改写，关键术语含 state（匹配过程携带、自动回溯保存）、`port()`/`param()`/`nusers()`，且匹配经 SigMap 归一化按规范信号而非字面 wire；③ peepopt 是「瘦宿主 + 数据驱动规则」的真实消费者，自身仅有不动点循环，7 条规则全在 `.pmg` 中，经 CMake 的 `pmgen_command` 在构建期生成并 include，加规则只需加一个 `.pmg` 与一行 `pm.run_*()`。入门样本 muldiv（`(A*B)/B→A`）演示了 state 传递、`branch` 枚举乘数顺序、`autoremove`+`accept` 改写网表的完整套路。核心结论：pmgen 真实用户是 peepopt 全套规则与各厂商 DSP/SRL/进位链推断 pass；`alumacc` 并不使用 pmgen（手写 C++），需注意区分。
