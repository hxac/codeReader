本讲讲解把 Yosys 作为共享库 `libyosys` 嵌入自己 C++ 程序的方式，区别于 u9-l1 的插件 `.so`（yosys 当宿主、你加命令）。核心区别在「谁拥有 `main()`」：库方式下你的程序当宿主，全权掌控生命周期。

最小驱动模式：在自己的 `main()` 中依次调用 `yosys_setup()`（初始化，幂等，注册所有 Pass、`new` 全局 `yosys_design`）→ 若干 `run_pass()`/`run_frontend()`/`run_backend()`/`shell()` → `yosys_shutdown()`（清理并销毁 design）。关键结论：这套「驱动接口」就是命令行 `yosys` 程序 `main()` 内部用的同一组函数——`run_pass` 本质把字符串交给 `Pass::call`，与 shell 敲命令同路径；`run_frontend`/`run_backend` 的 `"auto"` 按扩展名猜前后端。因此库与 CLI 能力完全等价。

构建上 `libyosys` 默认为共享库但默认不安装，需用 `yosys-config --cxxflags --ldflags ... -lyosys` 或 CMake 的 `Yosys::libyosys` ALIAS 链接。公开符号均在 `namespace Yosys`（`YOSYS_NAMESPACE`）。高级用法可在 `setup` 后经 `yosys_get_design()` 直接读写 RTLIL，使用 `ConstEval`（模块求值）、`scopeinfo`（源码层级索引 API）等，由 `evaldemo.cc`/`scopeinfo_example.cc` 示范。后续 u9-l3 将对应到 Python 绑定 `pyosys`。
