本讲讲解 Yosys 的「选择（selection）」机制，即综合脚本中每条命令如何确定其作用对象。围绕三个最小模块展开。其一，数据结构 `RTLIL::Selection`：用「全选 / 全选非黑盒 / 显式枚举」三档布尔标志加 `selected_modules`、`selected_members` 两张表，紧凑表达任意对象子集，默认构造即为「全选非黑盒」（黑盒库单元默认排除）；Pass 通过 `selected_module`、`selected_whole_module`、`selected_member` 查询，并由 `optimize()` 保持表示最简。其二，选择栈与作用域：当前选择存于 `Design::selection_stack` 栈顶，`selection()` 取之；`Pass::call` 在每条命令前后守护栈深度不变，使命令尾巴上的选择参数经 `extra_args → handle_extra_select_args → push_selection` 成为「临时覆盖、用完即弹」；`cd` / `select -module` 通过 `selected_active_module` 切换模块作用域。其三，`select` 命令语法是一台栈机器：模式（`t:`/`w:`/`a:` 等前缀、模块/成员以 `/` 分隔）压栈，`%` 操作符（`%u`/`%i`/`%d`/`%ci`/`%co` 等）做集合运算与图遍历，`@name`、`%` 引用命名选择或上一轮选择，结束求并集得结果。核心结论：消费端 Pass 不直接读 `Selection`，而用 `Module::selected_cells()` 等便捷函数经 `design->selected()` 过滤，故 `stat`、`delete` 等天然支持选择。
