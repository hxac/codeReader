本讲是「扩展 Yosys」首讲，确立「一条 Pass 如何被定义、注册、调用」的最小闭环，覆盖三个模块：一是 Pass 骨架——派生 `Pass` 子类、实现纯虚 `execute(args, design)`、重写 `help()`、声明全局静态实例；注册全自动完成（构造时入 `first_queued_pass` 链表，由 `init_register()` 搬进 `pass_register`，命令名须全局唯一）。二是遍历 RTLIL——for 扫 `args` 解析 `-选项` 后交 `extra_args`，由它报出拼错的选项并把尾部选择表达式压入选择栈（临时覆盖、用完即弹）；再经 `design->selected_modules()`→`mod->cells()`→`cell->type` 读网表，输出走 `log()`/`log_id()`/`log_header()`。三是插件化——`yosys-config --build` 编译出与 yosys ABI 一致的 `.so`，`-m`（或 `plugin -i`）在 `yosys_setup()` 之后经 `load_plugin` 用 `dlopen` 打开，触发其中全局对象构造并再次 `init_register()`，使插件命令与内置命令地位平等。核心结论：新增一条命令只需「派生 + execute + 全局实例 + 编译加载」，去中心化注册让框架无需感知新命令，并为后续 u9-l2（C++ 库接口）与 u9-l3（Python 绑定）奠基。
