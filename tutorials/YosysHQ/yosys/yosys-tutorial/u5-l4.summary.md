本讲走完 Verilog 前端最后一段——AST 到 RTLIL 的生成，聚焦三个最小模块。`AST::process()` 是前端最终入口，对每个模块按「dump_ast1→simplify→dump_ast2→两趟 genRTLIL」编排。`simplify()` 只改写 AST、不碰 RTLIL，返回 bool 由 `while` 跑到不动点（fixpoint），以 stage 分层（stage 0 调度 stage1→mem2reg→stage 2），完成名字解析（设 `id2ast` 指针指向声明子树）、宽度推导（detectSignWidth）、常量折叠（const_fold，仅两侧皆为 AST_CONSTANT 才触发）、generate 展开与 mem2reg 等改写，统一经 apply_newNode 收口替换节点。`genRTLIL()` 只读 AST 产出 RTLIL，是按 type 分发的大 switch，分两趟扫描（先 WIRE/MEMORY 再其余节点，最后 INITIAL）保证被引用的线先存在；表达式节点返回 SigSpec，运算符经工厂函数 binop2rtlil/uniop2rtlil/mux2rtlil 映射成 `$` 单元（AST_BIT_AND→$and、AST_TERNARY→$mux），常量返回字面值、标识符返回 `id2ast` 指向的 Wire，always 块则交 ProcessGenerator 生成 RTLIL::Process。关键术语：Clifford's Device（用 if(0)+fall-through 让多 case 共享代码段）、`$类型$文件:行$序号` 单元命名。核心结论：simplify 与 genRTLIL 职责严格分离——先把树整理成可机械翻译的形态、再生成网表，由此收口整个 read_verilog 流水线。
