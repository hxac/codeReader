本讲钻进 RTLIL::Module 内部，认识构成网表的三大基本零件：Wire（线网）、Cell（单元）、SigSpec（信号说明）。

Wire 是一根带位宽的连线，只描述几何属性（width、start_offset、upto）与端口方向（port_input、port_output），不描述驱动关系——寄存器也只是一根 Wire，时序信息由驱动它的 $dff 单元承担。它继承自 NamedObject，故既有 name 又有 attributes 表。RTLIL 统一规定 LSB 为索引 0，用 upto 与 start_offset 还原 HDL 的升序/偏移下标。

Cell 是被实例化的器件，靠 type 说明种类（如 $and、$mux、$dff，或 \子模块名、黑盒单元），靠 parameters 携带位宽等参数。核心在于它不直接持有 Wire 指针，而是用 connections_ 字典以「端口名 → SigSpec」表达连接，因此一个端口可接到切片与常数的拼接，断开重连也只需改字典。

SigSpec 是描述「一段信号」的通用语言，能涵盖整根线、线切片、常数及拼接，故 Cell 端口才用它而非 Wire*。它内部由 SigChunk（块）与 SigBit（位）组成，层级为 SigSpec = 一串 SigChunk = 展开后的 SigBit；并用 CHUNK / BITS 双重表示做快慢路径优化。位状态有 S0/S1/Sx/Sz/Sa/Sm 六种。

三者关系：Module 拥有若干 Wire 与 Cell，Cell 经 SigSpec 间接引用 Wire。write_rtlil 文本里的 wire / cell / connect \端口 信号 即此三者的一一映射。
