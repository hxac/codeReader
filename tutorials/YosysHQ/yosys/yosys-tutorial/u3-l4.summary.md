本讲是 u3（RTLIL 核心数据结构）的收尾，讲解 Yosys 的「内部单元库」——相当于 RTLIL 这门汇编语言的指令集。它覆盖三个最小模块。其一，内部单元命名：所有前端产出、Pass 改写、后端消费的统一目标分两类——参数化高层单元 `$and $mux $dff $adff $mem`（综合早中期，位宽可变）与单位宽标准门 `$_AND_ $_DFF_P_`（工艺映射后，极性编进名字）。典型端口约定为二元运算 `A/B→Y`、多路器 `A/B/S→Y`、触发器 `CLK/D(,ARST)→Q`。其二，constids.inc 与 ID() 宏：一份按严格 ASCII 序排列的 X-macro「知名标识符」清单，被四次 `#include` 展开为 StaticId 枚举（稳定整数下标）、`ID::` 命名空间下的编译期常量、支持二分查找的有序 IdTable、以及启动时预填的全局字符串槽；`ID(...)` 宏在编译期二分，命中知名 id 则零开销返回 `constexpr`，否则退化为运行时构造。其三，单元类型登记：旧表 `CellTypes`（运行时 dict）与新表 `StaticCellTypes`/`NewCellTypes`（编译期按下标索引的定长数组 + 位集分类 `is_ff/is_evaluable` 等），经全局 `yosys_celltypes` 暴露并由 `RTLIL::Cell::known/input/output` 委托，使任意 Pass 都能零开销查询端口方向。核心结论：内部单元库 = constids.inc（名字清单）+ newcelltypes.h（端口清单），二者经 ID 宏与整数下标缝合为全局查询服务；新增单元只需在清单插一行、加一条 `setup_type`，无需改中心注册代码。
