本讲跨过 RTLIL 文本，进入 Yosys 的 C++ 内存模型，建立「设计在内存里长什么样」的骨架。`RTLIL::Design` 是内存模型的根，用 `modules_`（`dict<IdString, Module*>`）按名拥有所有模块并统一负责析构，`module(name)` 是 O(1) 查找，`top_module()` 按「带 `top` 属性优先、否则选区唯一模块」判定，无法确定时返回 `nullptr`。`RTLIL::Module` 对应一个 Verilog 模块，经 `NamedObject ← AttrObject` 继承而「有名、有属性」，内部用 `wires_`/`cells_` 字典存线网与单元、用 `connections_` 向量存 `assign`，并持有 `design` 反向指针；`add()` 同时建立归属与反向指针。Design 还承载设计级横切状态：`selection_stack`（选择栈，`selection()` 取栈顶，弹空自动补全选）、`scratchpad`（字符串键值表，pass 间传数据，如 `stat` 写结果）、`verilog_defines`（宏表，影响下次 `read_verilog`）。这些状态挂在 Design 而非 Module，是 pass 签名统一为 `execute(args, design)` 的基础。关键术语：Design、Module、IdString、selection、scratchpad、blackbox/top 属性、ObjRange、反向指针。下一讲将打开 Module 内部的 Wire、Cell、SigSpec。
