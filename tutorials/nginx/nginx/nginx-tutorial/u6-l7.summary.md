本讲剖析 nginx 变量系统 ngx_http_variables，揭示 `$name` 从配置文本到运行期字符串值的完整机制。核心是挂在 cmcf 上的三张表：配置期 variables_keys（全部已注册变量的名字哈希）、variables（被引用变量的索引数组）、prefix_variables（如 arg_、http_ 前缀变量），运行期由 variables_keys 转建出只读的 variables_hash。取值有两条路径：按 index（`ngx_http_get_indexed_variable`，配置期把 `$name` 编译成 index，最常用）与按名（`ngx_http_get_variable`，运行期哈希查找，命中 INDEXED 标志则转发到按 index 查找以共享缓存）。变量值存于每请求的 r->variables[index]，靠 ngx_variable_value_t 的 valid/not_found 位实现惰性求值与缓存（含 not_found 缓存），NOCACHEABLE 标志强制每次重算。get_handler 有三种套路：offsetof 零拷贝指向 r 内字段、计算后写请求池、前缀驱动剥离前缀再查列表。map、geo 等「复杂变量」复用同一注册接口，注册目标变量并装上自定义 get_handler，映射值含变量时用 valid=0 标记延迟到运行期二次求值。
