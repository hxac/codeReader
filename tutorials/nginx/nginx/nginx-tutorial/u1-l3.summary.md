本讲建立 nginx 源码全景图。核心结论是源码自底向上分五层：OS 抽象层（`src/os/unix`、`src/os/win32`，封装系统调用差异）、基础设施层（`src/core`，自研标准库：内存池、字符串、容器、buf、cycle、模块系统，含程序入口 `main()`）、事件层（`src/event`，事件框架＋epoll/kqueue 等多路复用后端＋SSL＋QUIC）、协议层（`src/http`、`src/stream`、`src/mail` 三个平行同构子系统，命名规则一致 `ngx_<层>_*`）。依赖方向单向不可倒置，上层 include 下层。关键术语：`auto/sources` 列「总要编译」的 core/event/os 清单，`auto/modules` 按 configure 开关动态组装可选协议层，二者最终被「拍平」成运行时一维数组 `ngx_modules[]` 供 `main()` 遍历初始化。由此确立导航规则：配置块名（events/http/stream/mail）与 src 子目录一一对应，看模块名前缀即可定位文件。
