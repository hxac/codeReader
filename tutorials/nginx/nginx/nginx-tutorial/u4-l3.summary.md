本讲解决 nginx 多 worker（各有独立地址空间）如何共享同一份动态状态的问题，由四个最小模块构成。共享内存：用 `mmap MAP_SHARED` 申请映射到各进程同一物理页的内存；nginx 要求模块在配置阶段用 name+tag+size 经 `ngx_shared_memory_add` 登记，到 `ngx_init_cycle` 才由 `ngx_shm_alloc` 真正分配，登记与分配分离以支持 reload 复用与冲突检测。slab 分配器：把裸共享内存按 small/exact/big/page 四层切分，固定大小对象命中同桶、位图 O(1) 分配、支持精确 free 与相邻页合并，专为共享内存里频繁增删而生，与进程私有的 `ngx_pool_t` 截然不同。shmtx 进程间自旋锁：一条原子变量（0 表空闲、pid 表持有者）配 CAS 互斥，辅以指数退避自旋与信号量回退；trylock 非阻塞用于 accept 互斥，lock 阻塞用于保护共享数据。rwlock 读写锁用一条原子变量同时编码写锁（全 1）与读者计数，服务读多写少的 upstream peer 状态。核心结论：`limit_req_zone` 这类跨 worker 有状态功能正是「共享内存 + slab + shmtx + 红黑树」四件套的组合，构成 limit_req、缓存元数据、upstream zone 等一大类功能的共同骨架。
