本讲拆解 nginx 如何把纯文本 nginx.conf 转为内存中可驱动服务器的配置结构，覆盖词法、分发、指令描述符三个最小模块。解析分三层：词法器 ngx_conf_read_token 是逐字符状态机，返回 NGX_OK/BLOCK_START/BLOCK_DONE/FILE_DONE/ERROR 五种码；分发器 ngx_conf_handler 暴力遍历所有模块的指令表做两轮匹配，校验模块类型、作用域（cmd->type & cf->cmd_type 为零即报 "not allowed here"）、结尾符与参数个数；执行由 set 回调完成。主驱动 ngx_conf_parse 可重入：块指令的 set 回调递归调用 ngx_conf_parse(cf, NULL)，靠同一 4KB 文件缓冲区与栈式保存/恢复 cf 上下文实现块嵌套，而非语法树。关键术语包括指令描述符 ngx_command_t，其 type 位掩码分三组——参数个数（TAKE1..7/FLAG/ANY/1MORE）、flags（BLOCK）、作用域（核心 MAIN_CONF、HTTP 层 MAIN/SRV/LOC/UPS_CONF）；cf->module_type 与 cf->cmd_type 是当前块的"身份标签"，进块切换、出块恢复，是强制指令作用域的根本机制。通用 ngx_conf_set_*_slot 函数用 (char*)conf + cmd->offset 反射式定位字段，复用 ngx_atoi/ngx_parse_size/ngx_parse_time，靠 NGX_CONF_UNSET 哨兵实现"同作用域不可重复"与"merge 阶段继承默认值"。HTTP 配置为 main/srv/loc 三层并行的指针数组（ngx_http_conf_ctx_t）。解析仅在启动或 reload 时执行一次，因此暴力分发对运行时性能无影响。
