本讲是 u6-l6 过滤器链的延伸专题，剖析 `ngx_http_xslt_filter_module`：它在 postconfiguration 把自己 push 进 header/body 两条 filter 链，header filter 按内容类型与是否配置 `xslt_stylesheet` 决定放行或扣留响应头，body filter 用 libxml2 的 **push parser** 跨块增量解析上游 XML，收齐后做整体 XSLT 转换再下发。核心难点在于 XXE（XML 外部实体）防护：解析器默认会替你加载外部资源，可致信息泄露、SSRF、DoS。nginx 采用三层防护——`XML_PARSE_NONET` 禁网络加载、`entityDecl` 回调剥离 `systemId` 禁本地文件、关闭文档内 catalog 防重定向。其中剥离逻辑通过把外部实体改写为空的内部实体来保持文档 well-formed。新增的 `xml_external_entities` 指令（默认 off，FLAG 类型，main/srv/loc 作用域）是第二层总开关，经「offsetof 反射赋值 + create_conf 哨兵 + merge_conf 三段式继承」的标准配置生命周期落地。引入的关键术语：push parser、SAX 回调表、`_private` 反查、systemId/publicId、内部 DTD 子集、三层防护模型。
