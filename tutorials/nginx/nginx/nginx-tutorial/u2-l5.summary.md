本讲解析 nginx 时间缓存与日志两子系统。时间缓存：事件循环每轮经 ngx_time_update，把墙上秒、单调毫秒（全局 ngx_current_msec，定时器基准）与多种预格式化时间字符串写入 64 个轮转 slot，读者无锁读取、写新读旧规避撕裂，同秒内仅更毫秒。日志：变参宏 ngx_log_error 以 log_level 阈值门控、未达标连参数都不求值，再由 ngx_log_error_core 在栈上拼「时间戳＋级别＋pid#tid＋连接号＋消息＋errno＋上下文」沿日志链写出，链按阈值降序、遇不够即停。error_log 指令选目标（文件/stderr/syslog/memory，后两者经 writer 回调）与级别，二者正交；日志直拷缓存时间戳，故 ngx_time_init 须先于日志。所引关键术语（ngx_current_msec、阈值门控、日志链降序、多后端 writer）为 u5-l4 定时器与 u10 日志讲的前置。
