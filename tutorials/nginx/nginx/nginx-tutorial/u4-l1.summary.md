本讲贯通 nginx 的 master/worker 进程循环。核心结论：master 是「信号驱动的状态机」——它长期睡在 sigsuspend 上，信号 handler 只置 sig_atomic_t 标志位（HUP→ngx_reconfigure、USR2→ngx_change_binary、QUIT→ngx_quit、TERM→ngx_terminate、CHLD→ngx_reap 等），主循环醒来后按标志位分发，自身不处理任何业务连接。worker 由 ngx_start_worker_processes 经 ngx_spawn_process 派生，登记进全局进程表 ngx_processes[]；派生类型（RESPAWN / JUST_RESPAWN / NORESPAWN）决定是否自动重启及 reload 时是否受 just_spawn 保护。worker 在 ngx_worker_process_init 里完成降权、CPU 亲和性、解屏蔽信号、调用各模块 init_process 回调、注册 channel 读事件，随后进入以 ngx_process_events_and_timers 为核心的事件循环。master 与 worker 间用 socketpair channel（传 NGX_CMD_QUIT 等命令）和 kill 信号两种途径通信。三大运维操作：reload 是「先启新 worker 再优雅退旧 worker、共享监听 fd、配置失败回退旧 cycle」；graceful-stop 关监听端口后让 worker 优雅退出；upgrade 经 ngx_exec_new_binary 拉起带继承 fd 的新 master，双 master 并存，新二进制崩溃则由 ngx_reap_children 改回 oldbin.pid 自动回退。关键术语：进程身份常量（SINGLE/MASTER/SIGNALLER/WORKER/HELPER）、sigprocmask、respawn、channel、init_process、继承套接字。
