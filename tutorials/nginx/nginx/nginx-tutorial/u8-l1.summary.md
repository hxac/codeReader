本讲讲解 nginx 如何把 OpenSSL 的阻塞式 API 改造成非阻塞、事件驱动形式，是 HTTP/2、HTTP/3、证书缓存讲义的地基。核心采用两层结构：配置层的 `ngx_ssl_t`/`SSL_CTX`（每 server 一份，持有证书与策略）与连接层的 `ngx_ssl_connection_t`/`SSL`（每连接独占），以 `ex_data` 作为 OpenSSL 回调反查 nginx 连接的桥梁。四个核心函数构成主轴：`ngx_ssl_create_connection` 只做 `SSL_new` 绑定与状态标记；`ngx_ssl_handshake` 通过把读写事件 handler 都设为 `ngx_ssl_handshake_handler`，在 epoll 多轮唤醒下反复调 `SSL_do_handshake` 完成多 RTT 握手，成功后立即把 `c->recv/send` 换成加密版，HTTP 层透明无感；`ngx_ssl_recv/send_chain` 用循环 `SSL_read` 处理分片、用 `saved_read/write_handler` 在「读内部需写」时临时接管事件，并以 16K 缓冲合并小 buf 降低 TLS 记录开销。最后覆盖会话复用（`ssl_session_cache` 共享内存与 session ticket）与 SNI（握手期间经回调 `ngx_http_ssl_servername` 切换 `SSL_CTX`，实现一端口多证书），并埋下 ALPN 协商 `h2` 的伏笔。
