本讲回答「超时如何实现」与「就绪事件按何顺序处理」两个问题，含三个最小模块。其一，定时器红黑树：全 worker 共一棵树，节点 key 为绝对毫秒时间戳，最小者最早到期；`ngx_event_find_timer` 返回最近到期剩余时间作为 `epoll_wait` 的阻塞上限，`ngx_event_expire_timers` 循环触发已到期者并置 `ev->timedout=1`，使 handler 区分超时唤醒与 I/O 就绪；插入比较用 `ngx_rbtree_insert_timer_value` 配有符号差值，扛 49 天回绕。其二，`add/del_timer` 为内联函数，懒优化 `NGX_TIMER_LAZY_DELAY=300ms` 在频繁续期（如慢速上传）时跳过删旧插新，以可控精度损失换性能。其三，posted 队列核心是 accept/普通两级：epoll 据 `rev->accept` 分流，accept 类（监听套接字）在持锁期先 drain，普通类放锁后才处理，使 accept 锁持有极短、避免一个慢请求独占锁导致的连接饿死；`ngx_posted_next_events` 是写缓冲满时推迟到下一轮的延迟写优化。退出阶段 `ngx_event_no_timers_left` 据 `cancelable` 标志判断剩余定时器能否抛弃以安全退出。关键状态位：`timer_set`、`timedout`、`posted`、`NGX_POST_EVENTS`。主循环顺序（accept 队列→放锁→expire 定时器→普通队列）将在 u5-l5 详讲。
