本讲承接事件模型与 epoll 后端，讲清一条连接从 accept 到回收的完整生命周期。引入两个核心抽象：ngx_connection_t 统一表示已建立连接与监听套接字，ngx_listening_t 持有监听地址与 handler 回调。worker 启动时一次性预分配 connection_n 个连接及读写事件，用 data 字段复用为链表指针串成空闲链表，ngx_get_connection 摘头、ngx_free_connection 头插，取放 O(1) 且零系统 malloc，每次取连接翻转 instance 位以检测过期事件。ngx_event_accept 是监听读事件的 handler，循环 accept4 取出就绪连接、为每条连接建专属内存池（accept 路径上唯一的系统 malloc），新连接写事件直接置就绪，末尾调 ls->handler（HTTP 即 ngx_http_init_connection）交接协议层，并用 ngx_accept_disabled = connection_n/8 - free_connection_n 这一本地量实现 worker 间连接负载均衡；EMFILE/ENFILE 时摘掉监听事件以防 ET 死循环。ngx_close_connection 按「摘定时器→摘 epoll→摘 posted 队列→标 closed→归还结构体→关 fd」顺序执行，closed 位与 instance 位共同防御 fd 复用导致的过期事件；资源紧张时 ngx_drain_connections 从可复用连接队列尾部取若干空闲 keepalive 连接，置 c->close=1 触发协议层优雅关闭以腾出连接结构体。
