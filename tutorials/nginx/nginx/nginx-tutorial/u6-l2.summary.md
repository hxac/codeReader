本讲从运行时视角拆解 HTTP 请求从连接 accept 到 keepalive 回归的完整生命周期，含三个核心模块：ngx_http_init_connection 只搭脚手架不读数据、挂 wait_request_handler 并起首字节定时器；ngx_http_create_request 诞生贯穿全程的 ngx_http_request_t（开私有内存池、接 main/srv/loc 三层 conf、置 main 与 count 初值）；ngx_http_finalize_request 按返回码语义收尾分派。最关键的是两层 handler 机制：连接层 c->read/write->handler 由 epoll 直接调，请求层 r->read/write_event_handler 由连接层经 ngx_http_request_handler 这个路由器间接调，请求靠改写自身 handler 来重新编程读写行为。连接级 ngx_http_connection_t（hc）长存、请求级 r 短命是 keepalive 的钥匙——响应完成后 set_keepalive 销毁 r、把 c->data 换回 hc、切到 keepalive_handler 并起定时器，下个请求字节到来时再 create_request 闭环回到解析阶段；count 引用计数则守护请求在异步操作完成前不被释放。本讲把解析器当作黑盒，状态机细节与请求体读取留待后续讲义展开。
