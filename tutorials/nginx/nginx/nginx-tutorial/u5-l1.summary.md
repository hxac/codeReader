本讲是第五单元「事件驱动核心」的地基，覆盖 `src/event/ngx_event.{h,c}`，确立事件机制的整体骨架，不深入单个后端。核心结论有三：其一，`ngx_event_t` 是 nginx 调度原子单位，靠 `data` 指归属连接、靠一串 1 位标志位表状态、靠 `handler` 记回调，内嵌 `timer` 红黑树节点与 `queue` posted 队列节点。四个关键位的置位场景须记牢：`active` 由后端 add/del 维护、`ready` 由后端 process_events 置 1 而 I/O 包装置 0、`eof`/`error` 不在事件分发时置而在 `recv` 返回 0 或返回 -1 且非 EAGAIN 时置。其二，`ngx_event_actions_t` 是 10 槽函数指针表，统一屏蔽 epoll/kqueue/poll 等后端差异，上层用 `ngx_add_event` 等宏间接调用；与之正交的 `ngx_event_flags` 声明后端能力（边缘/水平触发等）。其三，`events {}` 初始化分三层：`ngx_events_block` 进块解析并分 `ctx_index`、`ngx_event_core_init_conf` 选后端存入 `ecf->use`、`ngx_event_init_conf` 做跨层硬校验；真正跑起来是 worker 启动时的 `ngx_event_process_init`，调后端 init、预分配连接与事件数组、给监听 fd 挂 `ngx_event_accept`。「选后端」与「初始化后端」刻意分置配置层与运行时层，因每个 worker 需独立 epoll fd。后续讲义将承接 epoll 细节、accept 与连接管理、定时器与 posted 队列、主循环。
