本讲解析 nginx 的 HTTP/2 实现，覆盖五个最小模块。连接建立经 TLS ALPN 协商 `h2` 并校验连接前言后，由 `ngx_http_v2_init` 创建连接级上下文 h2c，以函数指针表 `ngx_http_v2_frame_states[]` 按 9 字节帧头类型做 O(1) 分发，逐帧状态机沿用 HTTP/1 的可重入设计。多路复用中，一条连接上由单调递增、不可复用的奇数 Stream ID 标识并发流；nginx 用 fake connection 把每条流包成独立 `ngx_connection_t`，使标准 HTTP 框架（phases、过滤器、请求体）零改动复用，HPACK 解码完成后交回标准处理。HPACK 以静态表（61 项）、动态表（4096 字节环形缓冲）与 Huffman 压缩头部，变长整数限 4 个续字节防恶意大数，首字节高位决定表示类型。流量控制为连接级与流级双层信用窗口：接收侧低于 1/4 上限即补满以减少 WINDOW_UPDATE，发送侧耗尽则按优先级挂入 waiting 队列，收到 WINDOW_UPDATE 后唤醒。v2 过滤器作为 filter chain 适配器，将 `headers_out` 编码为 HEADERS 帧、响应体切为 DATA 帧并按 rank/rel_weight 优先级调度，收发两侧均设洪水保护。HTTP/2 仍基于 TCP，仅解决 HTTP 层队头阻塞，TCP 层队头阻塞留给 HTTP/3 与 QUIC。
