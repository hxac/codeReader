本讲义讲解 nginx 如何从源码构建与运行，覆盖三个最小模块：构建总入口 auto/configure、编译开关 auto/options、模块清单与产物生成链（auto/sources、auto/modules、auto/make）。

核心结论：nginx 构建系统是纯手写 shell 脚本而非 autotools。auto/configure 是总指挥，按固定顺序 source 各子脚本，全程在同一 shell 进程共享同一组 shell 变量；每个编译开关对应一个 shell 变量（如 --with-http_v2_module 把 HTTP_V2 从默认 NO 翻成 YES）。auto/sources 只列「总要编」的核心与事件基础清单（CORE_MODULES、EVENT_MODULES 及各 OS 后端），可选模块源文件由 auto/modules 按开关动态追加并生成 objs/ngx_modules.c（内含 ngx_modules[] 指针数组）；auto/make 据此生成 objs/Makefile，最终链接出 objs/nginx。验证某模块是否编入有两条证据：nginx -V 回显对应开关、ngx_modules.c 含其模块指针。

引入的关键术语与概念（供后续讲义承接）：configure 阶段、auto/ 脚本目录、objs/ 产物目录、ngx_modules.c 及其中的 ngx_modules[] 数组（运行时模块系统的入口表，启动时被遍历初始化）、ngx_module_link 的三种取值（YES 静态、DYNAMIC 动态编成 .so、ADDON 第三方附加）。
