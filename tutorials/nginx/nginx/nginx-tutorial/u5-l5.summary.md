本讲打开 worker 事件循环的心跳函数 `ngx_process_events_and_timers`：一次调用分五阶段——算超时、抢 accept 锁、处理延迟写队列、阻塞等事件、善后。善后顺序固定为「accept 事件队列 → 释放 accept 锁 → 到期定时器 → 普通事件队列」，目的是让 accept 锁持有时间最短、优先接收新连接避免饿死。关键术语：accept 互斥锁（共享内存自旋锁 `ngx_accept_mutex`，保证同一时刻仅一个 worker 把监听 fd 挂上 epoll，从源头消除惊群）；`ngx_accept_disabled = connection_n/8 - free_connection_n`（忙 worker 主动让出抢锁的无通信负载均衡）；`NGX_POST_EVENTS`（持锁时后端投递事件而非就地处理）；`NGX_UPDATE_TIME`（阻塞返回后刷新时间缓存）；`ngx_posted_next_events`（写缓冲满的延迟写优化）。定时器由红黑树驱动：`find_timer` 定 epoll_wait 阻塞上限，`expire_timers` 触发到期者并置 `ev->timedout=1` 区分超时唤醒与 I/O 就绪。
