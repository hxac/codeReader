nginx 响应输出经两条对称过滤器链：header filter 处理响应头、body filter 处理响应体，分别由 `ngx_http_send_header`、`ngx_http_output_filter` 触发，各以一个全局链头指针（`ngx_http_top_header_filter` / `ngx_http_top_body_filter`）为入口。链靠各模块私有 static next 指针隐式串联——filter 在 `postconfiguration` 阶段用「`next = top; top = self`」两行把自身 push 进链，因而是栈式注册、最终调用顺序与注册顺序相反，整体排布由构建期 `auto/modules` 的 `ngx_module_order` 决定。核心节点：`write_filter` 是 body 链终点，负责攒包（`postpone_output`）、限速（`limit_rate`）、调 `c->send_chain` 真正写 socket，未发完则置 buffered 返回 `NGX_AGAIN`，是背压源头；`copy_filter` 借通用框架 `ngx_output_chain` 在文件 buf 与内存 buf 间桥接，当下游 filter 声明 `need_in_memory`（如 gzip）时被迫把文件读进内存、关掉 sendfile；`header_filter` 是 header 链终点，先算总长再把结构化的 `headers_out` 序列化成 HTTP 文本、直接交给 write_filter 发出；`postpone_filter` 用 `c->data` 与 `r->postponed` 为子请求排序输出，无子请求时走快路径透明透传。
