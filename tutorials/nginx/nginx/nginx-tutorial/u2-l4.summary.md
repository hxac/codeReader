本讲讲解 nginx 数据搬运的核心结构与输出框架。`ngx_buf_t` 是数据流原子单位，凭内存区间、文件区间加一组位域标志，一身兼表内存数据、文件数据与控制信号三种身份。`ngx_chain_t` 是 buf 加 next 的单链表，节点借内存池自由链复用、浅拷贝共享缓冲，默认零拷贝。`ngx_output_chain` 是通用过滤式输出框架，以 in、free、busy 三链驱动：无需拷贝时直通，否则把文件读进临时内存 buf 再下发，按 tag 回收已发完节点。默认写出口 `ngx_chain_writer` 调连接的 send_chain 写 socket。返回值 NGX_AGAIN 是贯穿全程的背压信号，意为数据滞留、稍后再来。本套结构是后续 HTTP 过滤器链与 upstream 缓冲的基础。
