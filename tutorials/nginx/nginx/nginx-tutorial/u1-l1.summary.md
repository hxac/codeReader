本讲是手册首篇，不涉及 C 代码，只为后续阅读建立整体印象。核心结论：nginx 不只是 Web 服务器，而是 Web 服务器、负载均衡器、反向代理、API 网关与内容缓存五合一的多角色程序，这解释了源码中静态文件、upstream、proxy_cache、limit_req 等子系统为何并存。本讲依据 README 覆盖 Modules（模块化）与 Runtime（进程模型）两个最小模块，并以 Configurations（配置驱动）作为通向后续讲义的桥梁。引入的关键术语有：模块，分为静态模块与动态模块（后者以独立 .so 经配置中的 load_module 指令运行时加载，二者区别在分发与加载方式而非功能强弱）；指令（directive，配置文件中的参数）；配置驱动（能力由编译进来的模块决定、行为由配置决定，解析入口为 ngx_conf_parse）；master/worker 进程模型（master 负责读取评估配置并维护 worker，worker 以事件驱动处理实际请求，数量由 worker_processes 控制，可设 auto）；共享内存（worker 间同步状态，限流等跨 worker 功能依赖它）。配套命令 nginx -v 查版本、nginx -V 查编译启用的静态模块、nginx -t 校验配置；当前版本号定义于 src/core/nginx.h 的 NGINX_VERSION 宏。
