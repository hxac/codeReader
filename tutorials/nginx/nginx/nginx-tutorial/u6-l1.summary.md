本讲讲清 `http{}` 块背后的配置框架。nginx 用 `ngx_http_conf_ctx_t` 把 HTTP 配置组织成 main/srv/loc 三层指针数组，按模块 `ctx_index` 索引：`main_conf` 全局唯一被所有 server/location 共享，`srv_conf` 每 server 一份，`loc_conf` 每 location 一份（http、server 层各留一份空 loc_conf 作 merge 基准）。每个 HTTP 模块经 `ngx_http_module_t` 暴露 8 回调：`create_main/srv/loc_conf` 建空壳（字段置 `NGX_CONF_UNSET` 哨兵）、`init_main_conf` 补默认、`merge_srv/loc_conf` 沿「http→server→location」继承（子未设取父、父也未设取硬编码默认）、`pre/postconfiguration` 做解析前后钩子。须区分两个核心：`ngx_http_module`（CORE 模块）仅注册 `http{}` 块、其 set 回调即 `ngx_http_block`；`ngx_http_core_module`（HTTP 模块）是框架骨架，产出总仓库 `ngx_http_core_main_conf_t`(cmcf)，注册 `server`/`location`/`listen` 等结构定义性指令。`ngx_http_block` 是装配线：建数组→create_conf→preconfiguration→递归 `ngx_conf_parse` 建子层 ctx→init_main_conf+merge_servers→建 location 查找树→init_phases→postconfiguration→拍平 phase_engine→optimize_servers。三层 ctx 靠「块指令 set 回调再递归 `ngx_conf_parse`」构建，merge 发生在解析之后，靠临时改写 ctx 的 srv/loc_conf 切换父子来源再递归 `ngx_http_merge_locations`。
