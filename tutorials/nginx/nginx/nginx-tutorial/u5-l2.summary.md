本讲在 u5-l1 建立的事件骨架基础上，钻进 Linux 默认事件后端 epoll。它通过 `ngx_epoll_module_ctx.actions` 把自身函数注册进全局 `ngx_event_actions` 表，上层用 `ngx_add_event`/`ngx_process_events` 等宏间接调用，实现与具体后端解耦。核心拆为五个最小模块：模块注册与 actions 对接、`ngx_epoll_init` 初始化、事件表示与 `data.ptr` 编码、`add_event`/`add_connection` 注册、`process_events` 主循环。引入的关键术语包括：`ngx_event_actions_t` 接口表、`ngx_event_flags` 能力位（`NGX_USE_CLEAR_EVENT`=边缘触发、`NGX_USE_GREEDY_EVENT`=读到 EAGAIN、`NGX_USE_EPOLL_EVENT`）、`instance` 位（检测 fd 复用导致的过期事件）、`epoll_ctl` 的 ADD/MOD 合并读写、`NGX_CLOSE_EVENT` 捷径、`NGX_POST_EVENTS` 与 posted 队列。核心结论：epoll 把就绪集维护在内核红黑树，wait 期只取就绪项 O(k)，相比 poll 的全量扫描 O(n) 在海量连接下更高效；ET 模式下用 `rev->available=-1` 落实「读到 EAGAIN」契约。
