本讲承接 u8-l1，回答证书如何缓存与 OCSP Stapling 如何工作两个问题。

nginx 有两套同构（共用红黑树 + LRU 队列 + create/free/ref 回调表）但独立的 SSL 对象缓存：配置期全局缓存（入口 `ngx_ssl_cache_fetch`，挂 `ngx_openssl_cache_module`）服务静态 `ssl_certificate`，靠 reload 时按文件 mtime/uniq 复用旧 cycle 已解析对象、无 LRU；运行期按 server 缓存（入口 `ngx_ssl_cache_connection_fetch`，由 `ssl_certificate_cache` 指令创建，带 max/valid/inactive）服务变量化证书，带 LRU 淘汰。含变量的 `ssl_certificate` 在配置阶段编译为 `complex_value` 并注册 cert_cb，握手时 `ngx_http_ssl_certificate` 求值出路径、经 `ngx_ssl_connection_certificate` 用 `SSL_use_certificate` 把证书装到当前连接，结果进运行期缓存；静态证书则用 `X509_set_ex_data` 贴文件名作反查钥匙。

OCSP Stapling 以 X509 指针为键，把每张证书的 `staple` 节点挂在配置层 `ssl->staple_rbtree`；初始化时找 issuer 与 responder URL（从证书 AIA 扩展或 `ssl_stapling_responder` 指令取）；握手回调 `ngx_ssl_certificate_status_callback` 附带有效响应并懒触发后台异步 OCSP 客户端（自带独立内存池，resolver→connect→write→read→verify 全异步）刷新，下次刷新时刻由公式 max(min(valid-300, now+3600), now+300) 决定。已知局限：变量化动态证书不在 `ssl->certs` 数组中，难以享受 stapling。
