本讲承接词法分析，讲解 njs 解析器如何把扁平的 token 流组织成抽象语法树（AST）。解析器是**状态机驱动的递归下降解析器**：每条语法规则对应一个状态函数，用显式栈（`njs_parser_after` 压入「续体」、`njs_parser_stack_pop` 弹回）替代 C 的函数递归，主循环即「取 token → 调当前状态函数」。优先级通过一条下降阶梯实现（`expression → assignment → … → additive → multiplicative → … → primary`），每层均为「入口下降 + `_match` 回来消费运算符」的二段式，越靠近 `primary` 优先级越高、节点越深、越早求值。AST 节点 `njs_parser_node_t` 的核心字段包括 `token_type`（语义）、`u` 联合（按类型复用附加数据）、`left/right`（孩子）、`dest`（指向消费本节点结果之父节点的反向指针）、`scope` 与 `index`；二元运算组装成左结合二叉树，多条语句则串成左倾的 `NJS_TOKEN_STATEMENT` 链表挂于 `scope->top`。节点的 `index` 是连接编译期 AST 与运行期存储的桥梁，为 32 位整数按 `value(24)|type(4)|var_type(4)` 位编码，运行期由 `njs_scope_value` 解码为 `vm->levels[type][value]`；NAME 节点经 `njs_variable_resolve` 取变量槽位、运算结果经 `njs_scope_temp_index` 取临时槽位，最终被字节码生成器写入指令操作数，为下一讲（变量与作用域、生成器、字节码）做好准备。
