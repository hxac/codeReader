本讲剖析 njs 包装 QuickJS 的核心文件 src/qjs.c，拆为三块。其一，上下文创建入口 `qjs_new_context(rt, addons)`：用 `JS_NewContextRaw` 建空壳，逐个 `JS_AddIntrinsic*` 点齐语言特性，再注册模块、补 njs 全局对象、删 eval/Function 做安全裁剪；CLI 与 NGINX 共用此函数，仅第二个参数 `addons` 是否为 NULL 之别。其二，模块注册类型 `qjs_module_t{name, init}`，其 `init` 返回 `JSModuleDef *`（ES 模块对象），与内置引擎的 `njs_module_t{preinit, init}`（返回状态码）本质不同；注册分两条对等通道：构建期生成的全局清单 `qjs_modules[]`（通用模块）与运行期嵌入者传入的 `addons`（NGINX 的 r/s/ngx 等宿主模块）。其三，`qjs_add_intrinsic_*` 替 QuickJS 补 btoa/atob、TextEncoder/TextDecoder、njs 等全局；类 id 枚举 `QJS_CORE_CLASS_ID_*` 集中定义且从 64 起（避开 QuickJS 内建类 1..63）；`process` 由 qjs.c 供给、宿主挂载，非此函数产物。兼容方面，`quickjs_compat.h` 配合 `NJS_HAVE_QUICKJS_*` 宏让一份代码在 Bellard QuickJS 与 QuickJS-NG 上均可编译。关键运行期钩子 `njs.engine === 'QuickJS'` 供脚本做能力分支，为后续 u6-l4 引擎选择所用。
