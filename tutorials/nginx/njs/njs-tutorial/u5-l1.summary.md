本讲聚焦 njs 内置引擎的对象模型，拆解三个最小模块：**对象结构与双哈希**、**属性查找链**、**写时复制**。核心结构 `njs_object_t` 同时持两张 flathsh——只读共享的 `shared_hash`（压缩内建方法的内存占用）与实例私有可变的 `hash`（存自身属性），并由 `__proto__` 串成原型链；属性名经 atom_id 驻留为整数，`unique_find` 直接做整数匹配。属性读取经三层调用链：`njs_value_property`（快/慢路径分派）→ `njs_property_query`（按类型装箱、取原型对象）→ `njs_object_property_query`（沿 `__proto__` 逐层查 `hash` 再查 `shared_hash`，平均 O(d)）。关键术语包括 `own` 标志（控制是否仅查自身）与 `NJS_WHITEOUT`（delete 留下的墓碑，供 SET 复用槽位）。最核心的优化是写时复制：`shared_hash` 命中时由 `njs_prop_private_copy` 把共享属性惰性物化进当前层的私有 `hash`，并对函数/对象值深拷贝，实现「写自身、不污染共享模板」的属性遮蔽语义。
