本讲讲解 njs 的异步 HTTP 客户端 ngx.fetch()：API 形状对齐 Web Fetch，提供 Request/Response/Headers 三类宿主对象，但底层走 nginx 原生网络栈。核心认知是 fetch 立即返回 Promise 挂起，resolver→connect→SSL→读 status line/headers/body（含 chunked）全由 nginx 事件循环驱动，收齐后经 jobs 队列结算，JS 自身不做阻塞 IO。「双引擎＝双份代码」在此体现为 ngx_js_fetch.c（njs）与 ngx_qjs_fetch.c（qjs）两份 JS 包装共享同一份引擎无关的 ngx_js_http.c 连接与解析状态机；该状态机靠 http->process 函数指针在 status line→headers→body 三阶段间自行迁移，机制与策略分离，读循环只管收数据并分派。js_fetch_* 指令存于 loc_conf（默认 timeout 60s/buffer 16K/body 1M/keepalive 关，可被 init 单次覆盖），响应体走明文（按 Content-Length 或对端关闭）与 chunked 两条路径。关键约束：用主机名 fetch 必须在所在 server/location 配 resolver，否则报 no resolver defined。
