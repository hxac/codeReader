本讲义讲解 njs 中 Buffer/TypedArray/ArrayBuffer 的双引擎实现。内置引擎里二进制数据是「后端 `njs_array_buffer_t`（union 字节指针）+ 视图 `njs_typed_array_t`（offset/byte_length/type）」两层结构，Buffer 本质就是换了原型链的 `Uint8Array`，真实字节起点为 offset×element_size，detach 仅判底层指针是否为空。QuickJS 侧不重造 TypedArray，而是注册空壳 Buffer 类（类 id 从 64 起）、让 Buffer 原型继承 `Uint8Array` 原型，并用 `qjs_typed_array_data`「取值即校验」地取字节。本讲还用四个真实修复归纳二进制操作的边界陷阱：concat 的 TOCTOU、readInt/writeInt 漏判 0 的边界值遗漏、fill 零长度死循环、同类型拷贝忽略视图偏移，并强调双引擎两侧须同步修改、配 test262 风格回归用例。
