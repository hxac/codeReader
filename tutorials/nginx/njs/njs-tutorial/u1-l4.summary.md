本讲只围绕 `external/njs_shell.c` 一个文件，拆解 njs CLI 从命令行到启动引擎的主线。三个最小模块：一是 `main` 入口，它把 `argv` 经 `njs_options_parse` 加工成结构体 `njs_opts_t`，再交给 `njs_main` 选择运行模式（交互式 REPL、`-c` 内联命令、文件/stdin）；默认行为是交互式，由 `opts.interactive=1` 与 `isatty(stdin)` 共同决定。二是选项解析 `njs_options_parse`，用一个大 `switch` 逐字符匹配 `-c`/`-d`/`-e`/`-n`/`-p` 等选项并写入对应字段，需参数的用 `++i` 取下一段，`-c` 取到脚本后 `goto done` 跳出循环。三是环境变量与引擎分发：五个 `NJS_*` 环境变量在主循环前读入作默认值，命令行可覆盖；`-n njs|QuickJS` 把字符串映射成 `engine` 枚举。核心结论是双引擎抽象：`-a`/`-d`/`-g`/`-o`/`-s`/`-u` 为内置 njs 引擎专属，与 `-n QuickJS` 同用会被兼容性校验拒绝；`njs_create_engine` 用函数指针表对两套引擎做统一封装，上层只调指针不感知差异。关键术语 `njs_opts_t`、`engine` 枚举、`njs_create_engine`、函数指针表将供后续 VM 生命周期讲义承接。
