本讲承接 u8-l1，讲 ngx_http_js_module 如何让 JavaScript 参与 HTTP 请求处理，含三个最小模块。指令与 phase 绑定：js_access 注册进 access 阶段，js_content 设为 location 内容生成器，header/body filter 在 postconfiguration 接管过滤链头，js_set 注册惰性求值的变量取值回调、不绑固定阶段；filter 与 js_set 禁止异步，仅 access/content/periodic 支持挂起恢复。per-request VM：ngx_http_js_init_vm 在请求池分配 ctx，用 engine->clone 克隆模板 VM 并绑定 r，注册池清理钩子，幂等且整请求只 clone 一次。r 对象形状由原型表声明，内置引擎用 proto_id 加 njs_external_t、QuickJS 用 JS 类，对外 API 一致；content handler 默认 status 为 500，须以 r.return/finish/decline/internalRedirect 收尾，r.done 仅 body_filter 有效。
