本讲拆解 njs 引擎异常机制的实现，分三个模块。**异常指令**：try/catch/finally/throw 被编译成带跳转偏移的字节码指令（TRY_START、THROW、TRY_END、CATCH、FINALLY 及 break/continue/return 跳板），catch 地址登记在调用帧的 `frame->exception.catch`，嵌套 try 用 `njs_exception_t` 链表保存外层；throw 时跳到 `error:` 标签，沿 `top_frame->previous` 调用帧链回溯寻找最近的 catch。**异常值存储**：被抛出的值挂在 VM 的单值字段 `vm->exception` 上，由 `njs_vm_throw` 写入、`njs_vm_exception` 破坏性读出、`njs_vm_exception_get` 供宿主读取；异常是跨帧的单值，回溯途中穿越的帧被逐层弹出。finally 用 `exit_value` 槽编码区分正常结束、break、continue、return 四种退出。**Error 家族**：用连续枚举段 `NJS_OBJ_TYPE_ERROR…AGGREGATE_ERROR` 编号，`njs_vm_error2` 把「相对编号 + NJS_OBJ_TYPE_ERROR」转成绝对编号，经 `njs_vm_proto` 取原型物化错误对象并直接写进 `vm->exception`（造错即抛错）；仅当抛出真正的 Error 对象时才由 `njs_error_stack_attach` 附调用栈。
