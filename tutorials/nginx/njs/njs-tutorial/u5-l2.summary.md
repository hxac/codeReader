本讲剖析 njs 内置引擎三个最基础内建类型的内部表示。字符串以堆分配的 `njs_string_t` 头部承载，区分 `size`（字节数）与 `length`（字符数），字节内容、末尾 `'\0'` 与可选的 UTF-8 offset map 一次性连续分配；offset map 仅在非 ASCII 且字符数超 32 时惰性构建，用于把字符下标快速映射到字节偏移。数组有快数组（连续 `start[]` 缓冲）与慢数组（退化为 `object.hash` 哈希表）两条路径，由 `fast_array` 标志位区分，length 超 32768、`Object.freeze`、属性描述符等场景会触发 `njs_array_convert_to_slow_array` 降级；`Array.prototype.slice` 的快慢双路径是 bug 高发区（如 commit `66c6cdf9` 修复的稀疏大数组结果错乱）。数字以 `double` 直接存于 payload，`NaN`/`Infinity` 复用预定义 atom，普通数走移植自 QuickJS 的 `njs_dtoa` 生成最短可往返十进制串，进制转换复用同一 `njs_dtoa2` 仅改 `radix` 与禁用指数。三者共同体现「快/慢路径分叉、atom 驻留、惰性初始化」的 VM 设计哲学。
