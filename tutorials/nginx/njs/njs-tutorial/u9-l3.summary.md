本讲覆盖 HTTP 请求处理三类「读输入」能力。`r.subrequest()` 发起 NGINX 内部子请求（零网络开销、响应收进内存）：无回调返回 Promise、有回调返回 `undefined`，`detached` 与回调互斥，完成回调把子请求对象包成 reply 送回 JS。四个 `readRequest*()` 方法（Text/ArrayBuffer/JSON/Form）的异步性由 `body_read_state` 状态机管理（IDLE/DEFERRED/IN_PROGRESS + FORM 位）：JS 仅置 DEFERRED，由 phase handler 在调用返回后才发起读体并据此选择 `NGX_AGAIN`/`NGX_DONE`，故仅 access/content/periodic 支持异步；读到的请求体与表单结果各自缓存复用。引擎无关的 `ngx_js_form.c` 解析 urlencoded 与 multipart 产出 `ngx_js_form_t`；文件上传字段只保留文件名、丢弃内容，JS 侧呈现为 `{name}` 对象；`maxKeys`（默认 128）与 `NGX_JS_FORM_MAX_*` 边界常量共同防御恶意输入。三者统一体现「JS 异步＝NGINX 事件循环驱动＋jobs 队列结算」模型。
