本讲进入 njs 内核 `src/` 源码，拆解 `njs_vm_t`（虚拟机实例）从创建到销毁的完整生命周期，以公共 API 为线索，覆盖三个最小模块。

模块一是「创建与选项」：嵌入 njs 固定写法为先 `njs_vm_opt_init` 清零 `njs_vm_opt_t` 并补默认值，再 `njs_vm_create` 真正建机。`njs_vm_create` 内部建内存池、按值拷贝选项、按需复用或新建 `shared`（共享只读内建资源），并经模块 `preinit`、`njs_vm_protos_init`、`init` 物化原型。关键术语包括选项开关 `init`、`interactive`、`sandbox`、`unsafe`、`module`、`shared`、`external`、`addons`，以及返回码约定 `NJS_OK/ERROR/AGAIN/DECLINED`。

模块二是「编译与执行」：刻意拆分以实现「编译一次、克隆多次执行」。`njs_vm_compile` 走「解析→AST→字节码」流水线，产物挂到 `vm->start` 与 `vm->global_scope`；`njs_vm_start` 从 `vm->start` 进入解释器主循环跑全局代码。另有 `njs_vm_call`/`njs_vm_invoke` 调用指定函数，以及 `njs_vm_enqueue_job`/`njs_vm_execute_pending_job` 驱动 Promise 作业队列。

模块三是「克隆与隔离」：`njs_vm_clone` 通过浅拷贝复用模板的 `shared` 与字节码，却重建私有 runtime、atom 表、全局 levels 与 `external`，实现「每请求隔离」；`interactive` 模式会拒绝克隆。销毁极简——`njs_vm_destroy` 只销毁内存池，体现池式整体生命周期，故销毁前须先排空 jobs、跑完 exit 钩子。
