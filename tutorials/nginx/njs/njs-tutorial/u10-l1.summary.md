本讲梳理 njs 分层互补的测试体系，共四层：C 单元测试、test262 风格 JS 测试、nginx 集成测试与辅助套件。`make unit_test` 把「JS 源码串→期望输出/异常串」喂进引擎，用 `njs_strstr_starts_with` 前缀匹配判定，并在 `njs_process_test` 里循环 `njs_vm_execute_pending_job` 排空作业队列以支持异步用例；`make lib_test` 则单独压测红黑树、扁平哈希表等内部 C 数据结构。第二层是 `test/js/*.t.js`，借用 test262 的 YAML frontmatter（`includes`/`flags`/`negative`），由 `test/test262` 脚本经 `test/prepare` 把 `test/harness/*.js` 拼到用例前，普通用例「不打印即通过」，异步用例靠 `$DONE` 打印 `Test262:AsyncTestComplete` 收尾。第三层 `nginx/t/*.t` 是基于 `Test::Nginx` 的 Perl 脚本，起真 nginx、打真 HTTP 请求验证指令绑定与 `r`/`s` 对象。贯穿各层的是双引擎测试：test262 用 `--binary='build/njs -n QuickJS -m'`，nginx 用 `TEST_NGINX_GLOBALS_HTTP='js_engine qjs;'` 切换 QuickJS，差异项经 `NJS_SKIP_LIST` 显式跳过。此外 `shell_test`（Expect）测交互式 REPL，`benchmark` 测性能基线。
