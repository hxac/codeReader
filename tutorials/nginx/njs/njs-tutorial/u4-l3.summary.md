本讲解析 njs 如何执行一次函数调用，分三块：函数表示、调用帧链、调用指令。

njs 用同一个 njs_function_t 承载两种函数，靠 native 位区分：为 0 时 u.lambda 指向字节码产物（含 start 入口、nlocal、closures），为 1 时 u.native 为 C 函数指针。调用上下文由调用帧链管理，top_frame 经 previous 串成调用栈；njs_frame_t 以 njs_native_frame_t 为首字段嵌套，实现「lambda 帧是 native 帧子类」的多态；lambda 帧用「指针数组+值数组」双区布局，被闭包捕获的槽位改指向堆副本，帧分配受 spare_stack_size 总预算保护，深递归抛 RangeError 而非 C 崩溃。

核心结论：一次调用由三件套协作——FUNCTION_FRAME/METHOD_FRAME 建帧（后者多带 this）、PUT_ARG 逐个写实参、FUNCTION_CALL 保存调用者 pc 并经 njs_function_frame_invoke 分流；native 调用直接执行 C 函数自行弹帧、不进解释器，lambda 调用切换 LOCAL/CLOSURE 指针后递归进入 njs_vmcode_interpreter，故调用栈深度即 C 递归深度；RETURN 完成构造器返回值修正、njs_vm_scopes_restore 弹帧、释放帧三步后退出本层解释器。本讲建立的调用帧链是后续异常回溯与 async/await 续体恢复的基石。
