本讲拆解 njs 执行核心 njs_vmcode_interpreter，它实现取指→分发→执行→推进 PC 的循环，是一个由 CASE 宏划分上百分支的巨型函数。分三个最小模块：(1) 分发主循环：pc 是指向字节码的 u_char*，ret 表示本条指令长度或跳转偏移，BREAK 宏完成「pc += ret 后重新分发」；退出仅有两条路——STOP/RETURN 的 return NJS_OK（正常结束），或 error: 标签沿调用帧链回溯寻找最近 try/catch、未捕获才 return NJS_ERROR。(2) computed goto：由编译期宏 NJS_HAVE_COMPUTED_GOTO 切换，生产默认走 switch_tbl[] 标签地址表、goto *switch_tbl[op] 间接跳转，每个 opcode 独占一条跳转指令，分支预测优于集中式 switch。(3) 操作数访问：32 位 njs_index_t 按 value(24)|type(4)|var_type(4) 编码，njs_scope_value 两次移位即定位到 vm->levels[type][value]（LOCAL/CLOSURE/GLOBAL/STATIC 四级存储）槽位；njs_vmcode_operand 经 njs_scope_valid_value 取值并兼任 TDZ 守卫，对未初始化的 let/const 抛 ReferenceError。本讲与 u3-l3、u3-l5 闭环，覆盖变量从声明到执行的完整链路，并为 u4-l2 作用域寻址、u4-l3 调用帧、u4-l4 异常、u4-l5 异步铺垫。
