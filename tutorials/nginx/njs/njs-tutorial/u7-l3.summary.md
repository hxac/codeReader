本讲承接 fs 模块，覆盖 querystring、xml、zlib 三个外部扩展模块。querystring 是无外部依赖的纯 C 模块，提供 parse/stringify/escape/unescape（含 decode/encode 别名），核心为 sep/eq 切分、重复键折叠成数组、基于 hex 查找表与 escape 位图的百分号编解码。xml 模块包装 libxml2，对外暴露 XMLDoc/XMLNode/XMLAttr 三类对象，用 $tag/$tags/$attr/$attrs/$text/$name/$ns/$parent 等前缀语法把属性访问映射到 libxml2 树操作。zlib 模块包装系统 zlib 库，deflate/inflate 四个同步方法靠 magic8/raw 标志共用同一 C 函数，并把 zalloc/zfree 重定向到 njs 内存池使压缩临时内存随 VM 自动回收。本讲贯穿两条机制：一是「双引擎=双份代码」，每个模块成对提供 njs_*_module.c 与 qjs_*_module.c，且同一优化因对象模型不同而手段各异（内置引擎用外部原型 + 内存池 cleanup 链，QuickJS 用 JS 类 + exotic methods + 引用计数）；二是「可选依赖构建期探测」，libxml2、zlib 由 auto/options 默认开启、经 auto/libxml2 与 auto/zlib 多路径探测，再由 auto/modules 以「用户开关 ∧ 探测成功」双条件决定是否编入，querystring 因无依赖而无条件编入。
