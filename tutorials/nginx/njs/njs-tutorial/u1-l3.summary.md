本讲承接 u1-l2，把构建系统真正「跑起来」，覆盖三块内容。其一，构建分两步：`./configure` 只解析选项、探测环境并生成 `build/Makefile`（依次加载 `auto/options`、`auto/quickjs`、`auto/make` 等），`make njs` 才真正编译链接出 `build/njs`；改 `configure` 选项后须先 `make clean`。其二，CLI 有三种运行方式：`-c` 一行命令、`file.js` 文件、无参数的交互式 REPL；关键认知是 CLI 脱离 NGINX 独立运行，没有 `r`/`s` 请求与会话对象，只用于验证 JS 语法与语言行为。其三，QuickJS 是可选链接的外部库，默认仅尝试探测（`NJS_TRY_QUICKJS=YES`），`auto/quickjs` 用回退策略寻找 `libquickjs`，找到才定义 `NJS_HAVE_QUICKJS` 并把库追加进 `NJS_LIB_AUX_LIBS`，`auto/make` 据此把 `libqjs.a` 链入 `build/njs`；用 `--cc-opt`/`--ld-opt` 暴露自编译的 QuickJS，`--with-quickjs` 则升级为强制。运行期可用 `-n njs`/`-n QuickJS` 或 `NJS_ENGINE` 环境变量切换引擎，默认为内置 njs 引擎。
