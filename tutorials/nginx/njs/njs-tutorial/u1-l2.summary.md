本讲梳理 njs 的工程骨架。顶层目录分层：src/ 引擎内核（含 QuickJS 包装层），external/ 扩展模块与 CLI 入口，nginx/ NGINX 集成，ts/ 类型定义，auto/ 构建系统。核心结论「双引擎＝双份代码」：扩展模块均成对提供 njs_*、qjs_* 两份实现，产物分 libnjs.a、libqjs.a。configure 依次 source auto/*.sh 做特性检测与清单收集，由 auto/make 生成 build/Makefile。njs 以 shell 变量当清单：auto/sources 固定列内核 .c，auto/modules、auto/qjs_modules 登记扩展模块，经 auto/module 追加源文件并按依赖条件注册；auto/make 还动态生成 njs_modules.c、qjs_modules.c 供引擎启动初始化扩展模块。
