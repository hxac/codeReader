本讲讲解 atom（字符串/符号驻留）机制。njs 把属性名、标识符、已知符号驻留成 32 位 atom_id，使属性查找退化为整数比较，字节码也只存 id 而非字符串。atom_id 用最高位 0x80000000 做位编码：置位即 number atom，低 31 位就是整数本身（njs_number_atom/njs_atom_number/njs_atom_is_number 等宏），小整数下标因此走纯算术快路径；最高位为 0 则是 named atom，指向哈希表条目。克隆模型下设两张 flathsh 表：模板 VM 的 atom_hash_shared 装填全部预定义原子，克隆 VM 经浅拷贝只读复用共享表，并另起私有 atom_hash；atom_hash_current 指针指向当前写入表，shared_atom_count 作分界线，查找先私有后共享，njs_atom_to_value 按分界线分流取值。预定义原子由 X-Macro 清单 njs_atom_defs.h 统一驱动，在头文件展开为 NJS_ATOM_* 枚举、在源文件展开为静态值数组 njs_atom[]（预填 token_type/token_id 供词法器识别关键字）；unknown（id=0）充当「未原子化」哨兵，触发 njs_atom_atomize_key 按需驻留。关键术语：atom_id、number atom、named atom、atom_hash_shared/atom_hash、shared_atom_count、atom_id_generator、NJS_ATOM_SIZE。
