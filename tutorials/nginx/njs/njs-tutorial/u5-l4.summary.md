本讲讲解内置 njs 引擎如何从 C 代码构造宿主对象（外部对象，如 `console`、`r`、`ngx`），分四点。一是声明式描述符 `njs_external_t`：用一张静态表声明对象形状，`flags` 低 2 位区分属性、方法、子对象三类成员，`name` 可为字符串或已知 symbol。二是注册三件套：`njs_vm_external_prototype` 把描述符编译成槽位数组并返回整数句柄 `proto_id`；`njs_vm_external_create` 按 proto_id 造实例、把 C 指针用 `njs_make_tag` 打标签存入；`njs_vm_external` 在回调里靠标签校验后反查指针，构成 JS 对象与 C 结构体的桥。三是属性处理器 `njs_prop_handler_t`：靠 `retval`/`setval` 是否为 NULL 区分 GET/SET/DELETE，`magic16`/`magic32` 充当字段类型与偏移使一个处理器复用多属性。四是原生函数：复用 `njs_function_t`，靠 `native` 位区分 C 函数与字节码 lambda，`njs_vm_function_alloc` 用于独立全局函数，`magic8` 让多个 JS 方法（如 `console.log/info/warn/error`）共用一个 C 实现。外部对象的 `external_shared_hash` 即 u5-l1 共享哈希机制的特化；本讲机制仅限内置引擎，QuickJS 侧另有 `qjs_*` 包装。
