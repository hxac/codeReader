本讲解决 nginx 多 worker + 按请求克隆 VM 模型下普通 JS 变量无法跨请求、跨 worker 共享状态的问题：`ngx.shared` 建于 master 预先 mmap 的 MAP_SHARED 共享内存之上，是跨 worker 通信的标准方式。讲义覆盖三个最小模块：zone 配置、共享内存布局、字典 API。`js_shared_dict_zone` 在配置期只做参数解析与 shm 登记，fork worker 前于 `ngx_js_dict_init_zone` 建树；一块 zone = slab 池 + 跨 worker 共享头 `ngx_js_dict_sh_t`（主树按 key、过期树按 TTL、读写锁）+ 每 worker 一份的描述符 `ngx_js_dict_t`，数据节点 `ngx_js_dict_node_t` 同时挂在两棵红黑树上。TTL 走惰性过期 `ngx_js_dict_expire`，`evict` zone 空间不足时按最久未用驱逐 `ngx_js_dict_evict`，因续期（删-改-插）赋予其 LRU 语义，故 evict 必须搭配 timeout。API 分两层：引擎相关绑定层（njs 的 `njs_js_ext_*`、QuickJS 的 `ngx_qjs_ext_*`）做类型校验并取出 shm_zone，引擎无关核心层 `ngx_js_dict_*` 加锁改树；set/add/replace 共用 `ngx_js_dict_set` 靠 flags 分叉，incr 仅限 number 字典且为跨 worker 原子计数原语。读写同步靠共享头里的 `ngx_rwlock`。
