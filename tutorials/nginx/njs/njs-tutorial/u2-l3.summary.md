本讲讲解 njs 内核两件基础设施。内存池 `njs_mp_t` 采用 cluster→page→chunk 三级分配：小块切 chunk（位图管理，至多 32/页）、中等占整页、大块走系统 memalign 并登记进按地址排序的红黑树，`free` 时用指针反查块；池随 VM 创建、随 `njs_mp_destroy` 一次性释放——先跑 cleanup 处理器链回收非内存资源（如正则上下文），再遍历红黑树释放全部块。扁平哈希表 `njs_flathsh_t` 把桶数组、描述符、元素数组打包进一段连续内存，用数组下标而非指针串联拉链链表，故可整体搬迁、保留插入序，按 3/2 扩容、按空洞比例缩容；通过 `njs_flathsh_proto_t` 的 test/alloc/free 三回调把哈希算法与键语义、分配器解耦。核心结论：`njs_vm_s.mem_pool` 是 VM 几乎所有运行时分配的唯一所有者，对象属性哈希表内部内存经 proto 向池申请，因此 `njs_vm_destroy` 仅调一次 `njs_mp_destroy` 即零泄漏回收，克隆 VM 各持独立池以隔离多请求状态。关键术语：三级分配、chunk 位图、红黑树块索引、cleanup 链、proto 三回调、装载因子扩缩容、unique_insert/unique_find（仅比 key_hash 即 atom_id）。
