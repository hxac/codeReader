本讲是 njs 全景入门，确立三个核心认知。第一，njs 本质是一个深度集成进 NGINX 的 JavaScript 引擎，用 JS 语法扩展 NGINX，典型场景是访问控制、改写响应头、异步内容处理。它有两种交付形态：嵌入 NGINX 的两个动态模块（处理 HTTP 的 `ngx_http_js_module` 与处理 TCP/UDP 流的 `ngx_stream_js_module`），以及脱离 NGINX 独立运行的命令行工具 CLI（`build/njs`，用于调试语法，但没有 `r`/`s` 等 NGINX 对象）。第二，njs 内置两套可互换引擎，用 `js_engine` 指令切换：官方推荐的 QuickJS（ES2023，新代码首选）与自 1.0.0 起弃用的内置 njs 引擎（ES5.1 strict 子集）。`class`、`Map/Set`、解构、生成器、`BigInt` 等现代语法仅在 QuickJS 可用。注意「双引擎 = 双份代码」，扩展模块常同时维护 `njs_*.c` 与 `qjs_*.c`。第三，njs 是指令驱动的运行模型：JS 不会自启动，必须由 `js_content`、`js_access`、`js_set` 等指令绑到请求处理的某个阶段（如 access、content），由 NGINX 触发执行，并注入请求对象 `r` 或会话对象 `s`。本讲通过 `js_path`/`js_import`/`js_content` 构成的 Hello World 串起这些概念，为后续单元进入 `src/` 内核源码打下基础。
