本讲聚焦 FIFO 读侧时序与多 FIFO 组合两大进阶主题。核心区分两种读模式：normal 模式下 `r_data` 寄存输出，读请求后数据晚一拍出现；FWFT（首字直通 / show-ahead）模式下 `r_data` 组合输出，永远等于当前队头，读请求仅用于弹掉队头、延迟为零。`fifo_single_clock_reg_v2` 用触发器阵列做存储体，借字符串参数 `FWFT_MODE` 在组合读 `data[r_ptr]` 与寄存读 `data_buf` 间二选一；而块 RAM 版（u4-l2）因同步读只能 normal。

预读有两条路线：`preview_fifo` 把一个 FIFO 拆成两个交替的子 FIFO（依赖 Altera `scfifo`，仅限 Quartus），输出端同时露出相邻两字 `od0`/`od1`；`read_ahead_buf` 是套在 FWFT FIFO 读口的薄壳，用"幽灵读"在 `fifo_empty` 下降沿抢先取队头进缓冲，配合 `soft_latch` 锁存，让用户在单个 `always_ff` 里同拍发请求并取数，代价是有效深度 +1、空标志撤销多 1 拍。

多 FIFO 聚合有两个对称模块：`fifo_combiner` 任一非空即读，经仲裁器（`ENCODER_MODE` 选轮询/优先级）多路选一汇聚到单输出；`fifo_operator` 全部非空才读，对多路数据施加运算（默认按位或的 `operator` 函数可改）。二者均用 `FWFT_MODE` 适配上游时序，normal 模式需把索引/数据打一拍对齐，下游宜接 `almost_full` 防溢出。
