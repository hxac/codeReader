本讲围绕三种仲裁器（arbiter），核心都是从多位请求总线每拍挑出一个获胜者，统一输出 `od_valid`、`od_filt`（one-hot）、`od_bin`（二进制下标）。`priority_enc` 是纯组合的固定优先级编码器（MSB 优先），复用 `reverse_vector`+`leave_one_hot`+`pos2bin` 三个积木拼出，确定性强但低位会饿死。`round_robin_enc`（朴素版）用自由轮转计数器当优先指针，平均公平，但单个稀疏请求要等满 WIDTH 拍才被服务一次。`round_robin_performance_enc`（性能版）用双倍位宽拼接加掩码再对 WIDTH 取模，把「带回绕的找下一个置位位」压成一次组合运算，有请求就每拍授权，指针仅在授权时推进。三者构成确定性对公平、面积对吞吐的取舍谱系，按请求到达模式选型；下游必须先查 `od_valid` 再采信 `od_bin`，否则会把无效态误判为授权第 0 位。
