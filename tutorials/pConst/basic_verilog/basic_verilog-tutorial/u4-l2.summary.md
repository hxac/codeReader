本讲围绕仓库模块 `fifo_single_clock_ram` 讲清单时钟域、基于双口 RAM 的标准 FIFO 设计，核心是把线性 RAM 当成首尾相连的环来复用。它覆盖四个最小模块：**环形指针**（写指针 `w_ptr`、读指针 `r_ptr` 各按 `inc_ptr`「到 `DEPTH-1` 回零」递增，实现先进先出）；**满空判断**（采用计数器流派，用元素计数 `cnt` 直接译码，`empty=(cnt==0)`、`full=(cnt==DEPTH)`，位宽 `DEPTH_W=clogb2(DEPTH)+1` 多一位保证能表示「满」值）；**同时读写仲裁**（`unique case({w_req,r_req})` 处理四种请求，`2'b11` 时空则只写、满则只读、否则既读又写且 `cnt` 不变）；**overflow/underflow 保护**（用 `w_req_f`/`r_req_f` 过滤非法请求驱动 RAM 使能，用组合 `fail` 当拍报告「满写/空读」，两层防护由同一组 `empty`/`full` 驱动而完全一致）。关键结论：`DEPTH` 须为 2 的幂；本模块只实现 normal 模式，块 RAM 同步读使 `r_data` 比请求晚一拍，这与 FWFT（首字直通）变体的区别留待 u4-l3 承接；存储体委托给 `true_dual_port_write_first` 双口 RAM。
