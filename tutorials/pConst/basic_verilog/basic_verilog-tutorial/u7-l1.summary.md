本讲是专家层第一讲，主题不是某个具体 RTL 模块，而是「怎么写 testbench 验证模块」，围绕三个最小模块展开：随机激励、异步时钟注入、自校验，核心理念是让 bug 自己报错而非靠人眼盯波形。随机激励用不可预测的输入覆盖人脑想不到的边界，提供两种随机源：仿真专用的 `$urandom`/`$urandom_range`（每拍现取）与可综合的 `c_rand` LCG 模块（例化成可共享总线），配合固定种子 `$urandom(SEED)` 实现可复现性。异步时钟注入故意造一个频率与主时钟不成整数比的时钟，并用 `always @(*)` + 内嵌随机延迟注入抖动（封装版为 `sim_clk_gen`），专测跨时钟域 bug；`forever` 周期性复位则测 DUT 恢复能力。自校验用黄金模型连续比对 DUT 输出，失配即 `$error` 并锁存进一根 `success` 标志，分「复制比对」（如 FIFO 并排厂商实现）与「计数守恒」（如 cdc_strobe 统计脉冲数）两套套路；并强调要故意改坏黄金模型来验证检查器本身有效。
