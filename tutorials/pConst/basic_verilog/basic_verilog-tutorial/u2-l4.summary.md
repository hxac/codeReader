本讲讲解位宽计算，覆盖三个最小模块：`clogb2` 函数、`$clog2` 系统函数、以及"地址位宽 vs 计数位宽"的 +1 分界。仓库自带的 `clogb2(n)` 数的是"把 n 右移到 0 所需次数"，即表示数值 n 本身的位数，等于 ⌊log₂n⌋+1；它诞生于 Verilog-2001 年代，需通过 `include "clogb2.svh"` 引入。`$clog2(n)` 是 IEEE 1800 内建系统函数，等于 ⌈log₂n⌉，回答"n 个表项需要几 bit 地址"，无需 include。核心等式为 `clogb2(n) == $clog2(n+1)`，两者仅在 2 的幂次边界差 1。关键结论：寻址 DEPTH 个表项（取值 0..DEPTH−1）用 `$clog2(DEPTH)`；装下 0..DEPTH 的计数器用 `$clog2(DEPTH+1)`。一句话防错：`clogb2(DEPTH)` 是计数位宽而非地址位宽。作者明确建议新设计弃用 `clogb2`，改用语义清晰的 `$clog2`。
