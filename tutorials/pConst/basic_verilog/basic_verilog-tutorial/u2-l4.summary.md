本讲解决「给定存储深度 DEPTH，地址指针与元素计数器各需多少位」这一 FIFO/RAM 位宽问题。引入两个关键术语：手写函数 clogb2（Verilog-2001 时代的「移位计数」实现，等于 ⌈log₂(N+1)⌉，表示 0~N 需几位）与语言内置系统函数 $clog2（等于 ⌈log₂ N⌉，区分 N 个值需几位）；同传 8 时前者为 4、后者为 3，入参含义差一位。核心结论是两条等价式：$clog2(DEPTH)==clogb2(DEPTH-1) 用于地址位宽，$clog2(DEPTH+1)==clogb2(DEPTH) 用于计数器位宽。地址指针只取到 DEPTH-1，而计数器须能取到 DEPTH 以表示「满」状态，故计数器比地址多一位，这正是 FIFO 设计中最常见的 off-by-one 错误源。仓库 FIFO 实际写为 clogb2(DEPTH)+1，比严格值多一位，安全但略浪费。新设计应直接用 $clog2；且 clogb2.svh 必须 include 在某个模块内部。本讲为后续 RAM/ROM 模板（u4-l1）与单时钟 FIFO（u4-l2）的地基。
