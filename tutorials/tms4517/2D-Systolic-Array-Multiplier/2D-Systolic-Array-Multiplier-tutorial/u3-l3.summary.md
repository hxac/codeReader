本讲将视角从仿真切换到综合，围绕三个最小模块展开。其一，Quartus 工程由 `.qpf`（工程身份）与 `.qsf`（配置清单）构成，`.qsf` 通过 `FAMILY`、`DEVICE`、`TOP_LEVEL_ENTITY`、`SYSTEMVERILOG_FILE` 等赋值把源码锁定到具体芯片，本设计目标为 Cyclone V 5CSEMA5F31C6、顶层 `topSystolicArray`，报告写入 `output_files`。其二，资源利用的核心结论是 **DSP 块数恰好等于 PE 数 N²**，源于每个 PE 的 8×8 乘法独占一颗 DSP；引脚数满足 48N²+4，引脚是放大规模的首要瓶颈。其三，同一份 RTL 在不同工具下可综合性不同：RTL 采用 SV2009 风格的内联 `for (genvar ...)` 与 packed 多维数组，在 Quartus（仅支持 SV2005 子集）下需改写为单独 `genvar` 声明并加 `generate/endgenerate`，在 Yosys 开源流程下需把 packed 数组拆平。本讲引入的关键概念包括综合、网表、ALM、DSP 块、资源利用报告与 SV2005 兼容性。
