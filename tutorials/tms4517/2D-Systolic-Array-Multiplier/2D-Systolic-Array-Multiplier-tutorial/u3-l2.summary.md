本讲为「回头读」型讲义，把前序各讲散落的 SystemVerilog 编码手法收拢为可复用工程范式，围绕三个最小模块展开。第一，参数化设计：顶层用 `parameter int unsigned N = 4` 声明规模，经 `#(.N(N))` 传给阵列，再由 `generate` 铺出 N² 个 PE；叶子节点 PE 不带 N，保持通用。所有随规模变化之量均为 `localparam` 派生：`MULT_CYCLES = 3N-2`、`MULT_CYCLES_W = $clog2(MULT_CYCLES+1)`、`PAD = 8(N-1)`。改规模只需改 RTL 一行加 TB 宏一行，其余全派生。第二，编译期检查：用 `localparam bit N_VALID = &{N>2, N<257}`（拼接+归约与）配精化期 `if(!N_VALID) $error(...)`，在精化阶段（非运行时、零硬件成本）拦非法规模；注意代码护栏 `N<257` 比 README 文档范围 `N<17` 宽松，以代码为准。第三，packed 多维数组（如 `logic [N-1:0][N-1:0][7:0]`）是结构基石：最左维对应最高位，整体为连续位，故支持整体移位/拼接（行/列变换）、逐元素下标访问及 Verilator 字拆分。此外贯穿全仓库的编码约定包括每个文件 `` `default_nettype none ``…`` `resetall `` 包夹、`var logic`、`always_ff`/`always_comb`、命名 generate 块、`'0`/`'1` 填充字面量、`WID'(value)` 显式转型、`.i_clk` 隐式命名连接与 `lint_off` 精准抑制，共同把人为错误前移到编译期。关键接缝：RTL 参数 N 与 TB 宏 N 必须手动同步，但 TB 里 `numArrays`/`j<4` 中的 `4` 是「字节/字」比值、由元素位宽决定，与 N 无关，不可误改。
