本讲解析 topSystolicArray.sv 的控制段，含三个最小模块。一是参数护栏：派生 1 位常量 N_VALID=(N>2)&(N<257)，在精化（elaboration）阶段用 $error 拦截非法规模 N，不占任何运行期硬件。二是控制计数器：派生常量 MULT_CYCLES=3N-2（N=4 时为 10），源自二维脉动阵列的经典延迟公式（错峰进网+逐列传播+逐元素累加）；存放位宽 MULT_CYCLES_W=$clog2(MULT_CYCLES+1)，其中 +1 用于容纳 counter_q+1。counter_q 仅在 doProcess_d==1 时自增、否则归 0，与 doProcess 互相驱动但被寄存器打断，无组合环路。三是 doProcess 门控与 validResult 生成：doProcess_d 按「validInput 启动 > counter==MULT_CYCLES+1 结束 > 否则保持」三态优先级生成，综合成优先编码器，使新请求可打断旧状态。核心结论是 o_validResult 为天然单拍脉冲——由 counter_q==MULT_CYCLES 触发置 1，但因计数器每拍自增，置 1 条件仅成立一拍；doProcess 故意多撑一拍（至 MULT_CYCLES+1）以盖住有效采样窗口，避免在 validResult 亮起那拍把 MAC 清零。整机一次乘法流程：i_validInput 启动 → 计数 3N-2 拍 → o_validResult 单拍脉冲供使用者采样 o_c → doProcess 关闭、counter_q 回 0，整机归静。
