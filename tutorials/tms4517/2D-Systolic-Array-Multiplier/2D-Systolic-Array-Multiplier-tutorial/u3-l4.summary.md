本讲义是学习手册的收口篇，从「读」设计转向「改」设计，含三个递进的最小模块。其一，安全改规模：把阵列规模 N 从 4 改成任意值只需两行真源头改动——RTL `topSystolicArray.sv` 的 `parameter N` 与 TB 宏 `#define N`；而 `MULT_CYCLES`(=3N-2)、`PAD`(=8(N-1))、packed 端口宽度、TB 的 `numArrays`(=(N²+3)/4) 全部由 `localparam` 自动派生。关键陷阱：TB 打包循环内层的 `4` 是「32 位字装 4 字节」的字节比值，与 N 无关，绝不能改；RTL 参数与 TB 宏是分属 SV/C++ 名字空间的两个人工接缝。其二，有符号改造：把无符号 8 位 MAC 改成 INT8 补码乘加，最小硬性改动是让乘法两端 `i_a`/`i_b`/`mult` 都声明 `signed`，乘积自动符号扩展到 32 位；累加器位宽 32 无需加宽（补码加法位级等价无符号加法且不溢出）；C++ 侧须先把 `int8_t` 经 `static_cast<uint8_t>` 剥成裸比特再左移，否则整型提升带来的符号扩展会污染整个 32 位字。其三，SIMD wrapper 扩展方向：用「并/串转换 + 单拍启动 + 单拍锁存」把 48N²+4 引脚的宽端口包成窄存储器接口，代价是加载/读回成新瓶颈需 DMA 摊平。核心术语：INT8、补码、符号扩展、localparam 自动派生、人工接缝、整型提升、SIMD wrapper、单向触发握手（无 ready 反压）。
