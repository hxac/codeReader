本讲只读 README.md，建立全局直觉，不涉 RTL。项目是用 SystemVerilog 写的二维脉动阵列（2D Systolic Array）矩阵乘法器：输入两个 N×N 方阵（2<N<17，默认 N=4），由一块 N×N 个 PE 组成的阵列算出 C=A×B；依赖仅 Verilator 与 GNU Make，运行入口为 `cd tb && make all`，改规模需同步修改 RTL 参数 N 与测试台宏 N。讲义覆盖三个最小模块：项目定位与 TL;DR、脉动数据流思想、输入输出位宽与精度取舍。引入的关键术语：PE（处理单元，每拍做一次乘加、仅与邻居通信）、MAC（乘加运算）、TPU（Google 矩阵乘法单元，本项目的工业原型）、错峰 skew（让数据逐拍在正确时刻到达正确 PE）。核心结论：脉动阵列靠大量 PE 并行、数据在 PE 间流动复用与局部邻居通信，把矩阵乘法从串行 O(N³) 压缩到 O(N) 量级，即 3N−2 个周期完成；输入选 8 位契合神经网络 INT8 量化、面积小速度快，输出选 32 位保证最大 N=16 时累加不溢出（理论上 20 位即够）。整体数据流分两步：先把输入矩阵做行/列变换（元素反转、高位补零、右移）实现错峰，再让 row 矩阵第一列接水平输入、column 矩阵第一列接垂直输入，每拍左移一个元素喂入阵列。这些位宽与数据流约定将贯穿后续 PE（8×8→16 乘法、32 位累加）、阵列互联与顶层接口讲义。
