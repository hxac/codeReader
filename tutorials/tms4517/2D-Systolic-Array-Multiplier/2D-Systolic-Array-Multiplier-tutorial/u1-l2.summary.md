本讲解决「如何把这个脉动阵列项目跑起来」，覆盖三个最小模块。一是仓库结构：`rtl/`（SystemVerilog 源码）、`tb/`（C++ 测试台与 Makefile）、`FPGA/`（Quartus 工程）、`images/`，整体只依赖 Verilator 与 GNU Make。二是 `tb/Makefile` 的四阶段流水线 verilate→build→sim→waves：verilate 把 SV 翻译成 C++（`--trace` 用于产生波形），build 编译出可执行程序，sim 运行产出 `waveform.vcd`，waves 用 gtkwave 查看。三是 clean/lint 与生成物管理。关键术语：RTL、仿真、Verilator 的「翻译成 C++」机制、GNU Make 的目标依赖与时间戳增量构建。核心结论：`make all` 因首依赖 `clean` 每次全量重跑；`make lint` 用 `--lint-only` 提供最快自检；`+verilator+rand+reset+2` 配合 `--x-initial unique` 给未初始化寄存器确定性随机初值以暴露 X 传播问题；`.gitignore` 忽略 `obj_dir/`、`waveform.vcd` 等全部生成物。默认矩阵规模 N=4（4×4），为后续讲义通用参数。
