本讲义讲解 ready/valid 弹性流水线接口与脉冲接口之间的双向转换，核心动机是：带反馈回路、算一个结果要 N 拍的**迭代型模块**（initiation interval 大于 1，如累加器）天然适合脉冲接口，再用两个转换器包装成 ready/valid 与流水线世界对接。两个最小模块为：`Pipeline_to_Pulse` 把 ready/valid 输入握手翻译成单周期脉冲喂给内核，最绕的 `ready_in` 用**两个 Pulse_Latch** 分别记忆「是否握过手」和「内核是否就绪」，化解「初始就绪」与「clear 归零」的矛盾，并旁路 `module_ready` 省一拍延迟；`Pulse_to_Pipeline` 把内核的 done 脉冲翻译成 ready/valid 输出，`module_ready = output_handshake_done`（结果被缓冲收下即回送），且**必须插输出缓冲**（Skid/Half/FIFO 三选一）切断 `ready_out → module_ready` 的反向组合路径。两个转换器经一条 `module_ready` 回送链路把内核「夹」在中间，耦合为一次「进—算—出」事务（范例见 `Averager_Powers_of_Two`）。核心纪律是：内核**至少一级寄存器、不得组合直通**，否则回送环构成组合环路（u9-l2 铁律）。两者只在单时钟域内工作，但暴露的脉冲接口正是日后插 `CDC_Pulse_Synchronizer` 跨域的天然钩子。
