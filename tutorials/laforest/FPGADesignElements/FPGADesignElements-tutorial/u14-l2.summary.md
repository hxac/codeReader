本讲覆盖 CDC（时钟域穿越）单元的三块最小模块。**Weinstein Flancter** 是跨域事件标志：用两个寄存器的相对差表达标志值——set 域置位使两者「不同」、reset 域清位使两者「相同」，无需异步复位、也不依赖对方时钟；其 `bit_out` 异步于两域，使用前须经位同步器。单个 Flancter 只标记「事件有无」，级联成逐位结构才能计数快事件，约束是 set 与 reset 绝不可重叠。**CDC FIFO Buffer** 由双时钟双口 RAM、两个 `Counter_Binary` 地址计数器、两个 `Register_Toggle` 绕回位和两个 `CDC_Word_Synchronizer` 构成，支持任意深度与循环缓冲。核心机关在于「空」与「满」的地址值完全相同，必须靠**绕回位**区分（相同为空、不同为满）；读写地址永不相超，故指针同步滞后只影响峰值容量。本讲的核心难题是**多位指针跨域**：逐位同步会撕裂数值。经典 Cummings 方案用 **Gray 码**（相邻值只变一位）；本书两个 FIFO 却不用 Gray 码，而改用「二进制指针 + 绕回位 + `CDC_Word_Synchronizer` 整字锁存握手」，只同步一个 toggle，保证整字相干到达，殊途同归。**CDC FIFO Repacker** 进一步以**最小公倍数（LCM）缓冲深度**、head/tail 双计数器和变址部分位选，把一种位宽的 ready/valid 流无缝重打包成另一种位宽并跨域，存储用寄存器而非 BRAM 以支持任意位宽子集。三者共享同一哲学：CDC 正确性靠结构纪律（互斥、整字相干、一起复位）保证，而非靠仿真。
