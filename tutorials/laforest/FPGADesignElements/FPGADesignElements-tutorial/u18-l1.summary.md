本讲从「读者」转向「贡献者」视角，讲清本书自身的生产与发布工具链。`v2h.py` 以「行注释当 Markdown、其余当代码」实现注释即文档，依赖「一模块一文件、文件名=模块名」约定，并把代码中已有对应 `.html` 的模块名自动转成跨页超链接。样板自动化三件套：`generate_file_skeleton.py` 固化开篇规矩（`` `default_nettype none ``、参数默认 0、`initial` 初始化）；`generate_instance.py` 是只向前扫描的「填空式」实例生成器，仅认本书固定排版；`verilinter` 用 Verilator+Icarus 双 linter 加三条 grep，把 reg 必须初始化、wire 不可赋初值、必备 `` `default_nettype none `` 这三条纪律变成可执行检查。FuseSoC 的 `.core`（CAPI=2）使整本书可作为一个 HDL 包被外部工程引用，由 `generate_fusesoc_core_file` 扫描目录自动重生、增删文件自动同步。`CONTRIBUTING.md` 四步流程（写 `.v` → 跑 `v2h.py` → 手工改 `index.html` → 发 PR）将上述工具串成一条贡献流水线，其中 `index.html` 的目录链接为手工维护。关键术语：注释即文档、Markdown、lint、CAPI=2、FuseSoC。
