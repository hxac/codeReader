本讲建立「参数化 + 复制构造常量 + clog2」三件套。先区分两种命名常量：`parameter` 在模块 `#(...)` 头部声明、可被实例化改写，是端口位宽的唯一来源；`localparam` 只能在模块体内定义、不可改写，用于持有由其他参数算出的内部常量，且不能出现在端口定义里（那是 SystemVerilog 才允许）。本书硬规矩是所有 `parameter` 默认值必须为 0：忘设参数会使 `[WIDTH-1:0]` 退化为非法的 `[-1:0]`，在 elaboration（精化）阶段吵闹地失败，而非静默错位运行。其次，Verilog-2001 禁止用 parameter/localparam 作字面量位宽说明符（故 `WORD_WIDTH'b0` 非法，`'b0` 在宽位下扩展行为也不一致），改用复制 `{N{1'b0}}` 与拼接 `{}` 构造位宽严格匹配的常量，让位宽告警有意义；窄值扩宽需用拼接显式做零扩展或符号扩展。最后引入 `clog2(N)=⌈log₂N⌉`（置于 `clog2_function.vh`，用 `` `ifndef `` 包成幂等供 `` `include ``），给出编址 N 个项目所需的位数；若要表示到 N 本身则需 `clog2(N)+1`，并注意 N<2 会撞上零位宽非法问题。涉及的样板模块为 `Register.v`、`Counter_Binary.v` 与 `Pipeline_Stall_Smoother.v`。
