本讲将 ready/valid 握手追溯到更根本的会合（rendez-vous）：相关方各自拉高 OK 信号，在同一拍同时见到 OK_IN&&OK_OUT 才一起前进，对等、无 master/slave、不必传数据。会合的硬件原语是 Muller C 元素，用 set（全 1 置 1）与 hold（非全 0 且当前为 1 则保持）实现「全员一致才翻转、否则锁存」；同步版把输入与输出一起送进流水线寄存器，使反馈回路可被重定时。落到 ready/valid 上即 Pipeline_Synchronizer_Lazy：以「先 Join 再 Fork、丢弃多余副本」将 N 条流水线锁步，输入侧 Join 要求全员 valid，输出侧 Fork 要求全员 ready。Lazy 指无缓冲纯组合，面积小、延迟低，但 input_data_ready 是组合路径，须防组合环，必要时改用缓冲版。C 元素是有状态会合，Lazy 同步器是无状态组合会合；本讲为后续 Fork/Join/Merge 家族奠定统一的会合视角。
