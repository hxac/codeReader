本讲把 ready/valid 握手扩展到 Merge 仲裁合流与 Gate 条件放行。核心反差：Merge 是「仲裁合流」，输出宽度等于单路（不拼接）、按仲裁逐笔放行，与 Join 的「会合拼接」（AND 归约、全部到齐、输出更宽）相对。三类 Merge 共享「独热选路＋mux 取数＋demux 回 ready」骨架：One-Hot 由外部 selector 选路、缓冲输出侧；Priority 用 Arbiter_Priority 固定优先；Round-Robin 用 Arbiter_Round_Robin 公平轮询、缓冲输入侧、切路有一拍空拍，须把 grant 与原始 valid 相与屏蔽，以免换状态那拍多传一笔错数据。未被授权路 ready 为 0，被反压冻结于 Skid Buffer。Pipeline_Gate 是纯组合条件门控，核心结论是必须同时门控 valid 与 ready——只堵一个会丢数据或吃垃圾；它用一个 Annuller 把控制位（可选连同数据）拼成一字统一清零。Pipeline_Credit_Gate 在 Gate 上加饱和信用计数器，把开关时机从外部 enable 内化为「信用是否为零」，实现信用制流控（信用数即允许在途的最大笔数）；用一句 (handshake_done != add_credit_pulse) 同时表达加、减、不动，且累加器不可加流水级。
