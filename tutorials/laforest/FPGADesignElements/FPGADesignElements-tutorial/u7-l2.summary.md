本讲承接 u7-l1，讲如何突破单块 BRAM「最多两三口」的限制，覆盖四个模块。其一，`RAM_Multiported_LE` 用触发器加查找逻辑实现任意数量读写口，存储体即寄存器故能一拍整体清零，但不宜做大。其二，写冲突靠 `Population_Count` 数命中个数（命中数 ≥2 即冲突），由 `ON_WRITE_CONFLICT` 在 PRIORITY、ROUNDROBIN、DISCARD、布尔归约四类策略中择一，被卷入的写口下一拍拉高 `write_conflict`。其三，`RAM_1WnR_Replicated` 以「存储复制 n 份、写广播到全部副本」换出 n 个独立读口，内部零逻辑，仅 `generate` 出 n 个 `RAM_Simple_Dual_Port`，是构建块库复用的范本。其四，内存初始化二选一：`USE_INIT_FILE=0` 用 `initial` 加 `for` 循环写同一 `INIT_VALUE`，置 1 则用 `$readmemh` 从「每行一个裸十六进制字」的文件加载，每行字符数为 ceil(WORD_WIDTH/4)。关键术语包括端口拼接（多口数据/地址/使能按口编号拼成宽向量、用 `base +: width` 切位段，位宽＝每口位宽×口数）、1WnR/nWnR、整体清零、`$readmemh`、`RAM_generate_empty_init_file.py`。
