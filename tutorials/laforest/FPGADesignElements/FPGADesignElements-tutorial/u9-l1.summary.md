本讲确立 ready/valid 握手接口这套模块间通信的通用语，含三个最小模块：（1）接口定义：由 source 驱动并指向 destination 的 `valid`/`data` 与 destination 驱动、指回 source 的 `ready` 三信号连接，均同步时钟上升沿，连接恒由 source 到 destination，接口内不得有组合环路（source 禁 `ready→valid`、destination 禁 `valid→ready`）。（2）握手过程：同一拍 valid 与 ready 同高即完成，标准式 `handshake_complete = (ready == 1'b1) && (valid == 1'b1)`；三条铁律为——valid 须保持到完成（防活锁）、source 不得等 ready 才拉 valid 而 destination 可等 valid 才拉 ready（防死锁）、影响接口的内部状态只在完成拍改变（防丢数据）；单周期握手最优，两周期合法。（3）AXI 不根本：AXI 每通道本即 ready/valid 接口，后者又只是「会合同步（rendez-vous，OK_IN/OK_OUT）+ 单向数据流」的实现，故本书停在 ready/valid 层、用 Skid Buffer/Fork/Join/Arbiter 自建接口，并以 source/destination 取代 master/slave。下讲 u9-l2 将把这些规则落到 `Pipeline_Skid_Buffer.v`。
