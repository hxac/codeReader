本讲承接 u8-l1 的加减法器与 u6-l1 的 Register 家族，把二者拼成时序构件**计数器**。核心结论：`Counter_Binary` 不是一句 `count <= count + 1`，而是「1 个 `Adder_Subtractor_Binary` 算下一拍值 + 4 个 `Register` 存储」的组合，模块自身零算术、零触发器，只做连线与控制选择。这种「算/存分离」使换计数方案（BCD、LFSR）只换加法器、不动控制逻辑，也把进位链推断的位宽技巧藏进加法器。

计数器经 `count` 反馈接加法器 `A`、`INCREMENT` 接 `B` 形成反馈环，方向由 `up_down`(即 `add_sub`) 决定。三套操作优先级为 **`clear` > `load` > `run`**，由 `Register` 的「最后赋值胜出」实现。`INCREMENT` 参数让计数器天然支持「每拍计 N 个」；标志位（`carry_out`/`overflow`）只在 `run` 时更新、`load`/`clear` 时清零。级联时把前级 `carry_out` 与后级 `run` 相「与」。

第二件事是可复用函数：`clog2(N)=⌈log₂N⌉` 返回「编址 N 个项目所需位数」。因 Verilog-2001 函数只能定义在模块体内（无 `package`），故约定把纯计算函数放进 `.vh` 文件，在模块体开头用 `` `include `` 引入，并以 `` `ifndef `` 守卫保证幂等。**位数陷阱**：若计数值可能取到 N（0..N 共 N+1 个状态），需用 `clog2(N)+1` 位。`clog2` 算位宽 → `Counter_Binary` 用此位宽是标准链路，全书 `abs`/`gcd`/`min`/`max` 等函数同此组织。
