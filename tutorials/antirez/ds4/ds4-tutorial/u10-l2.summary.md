本讲覆盖 ds4-agent 工具系统的三个最小模块：DSML 流式解析、工具执行、终端可视化。模型以 DeepSeek V4 原生 DSML 文本格式（`<｜DSML｜invoke>` 配 `<｜DSML｜parameter>`）直接生成工具调用，agent 用五态状态机（SEARCH→STRUCTURAL→PARAM_VALUE）逐字节增量解析成 `agent_tool_call`；参数值靠显式闭合标签定界，`string="true/false"` 区分原始文本与 JSON 字面量。引入的关键术语包括：七种 param_kind（NORMAL/PATH/OFFSET/CONTENT/DIFF_OLD/DIFF_NEW/BASH_COMMAND，按「工具名+参数名」分类，仅驱动可视化与颜色而非执行路径）；11 个内建工具（read/more/write/list/edit/search/google_search/visit_page/bash/bash_status/bash_stop），统一契约为返回文本、错误以 `Tool error:` 开头并仍塞进 transcript。核心结论：工具调用自生成起即是 KV 里的 token，作 `tool` 角色消息进入 transcript，因此 agent 天然不需要 u7 服务器的精确 DSML 回放与规范化；可视化层与解析共享同一字节流，却只画给人看、不碰 transcript/KV，非交互模式降级为纯文本。
