本讲承接 u5-l1/u5-l2，讲 CUDA 与 ROCm/HIP 两个编译期 GPU 后端如何落地张量常驻执行模型。三个最小模块：统一抽象上，`ds4_gpu.h` 是 Metal/CUDA/ROCm 共同遵守的 C ABI 合同，引擎核心只调这组符号，后端为编译期互斥单选，由 Makefile 切换 `CORE_OBJS` 第四个对象与链接器。CUDA 后端集中于 `ds4_cuda.cu`（nvcc 编译），用 `cudaHostRegisterMapped` 做权重零拷贝，用 `should_use_managed_kv_cache` 决定 KV 等长寿大分配是否走托管内存，Q4 专家表为空操作桩。ROCm 后端用伞状 `ds4_rocm.cu` 拼接二十多个 `rocm/*.cuh`，靠 `ds4_rocm.h` 宏垫片把 `cuda*` 名字改写为 `hip*`，几乎逐字复用 CUDA 运行时；另有 hipBLASLt、rocWMMA、`__dp4a` 映射到 `v_dot4_i32_i8` 等专属件，并在 Strix Halo 等 UMA 机器上改走分阶段拷贝、不 host-register 的权重路径。核心结论：两后端共享代码靠 HIP 模仿 CUDA API 形状加预处理期垫片，而非运行期插件。
