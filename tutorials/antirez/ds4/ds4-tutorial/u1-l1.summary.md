本讲（ds4 学习手册首篇）在进入 C 源码前先确立 DwarfStar 的定位与设计哲学，覆盖三个最小模块。其一，项目动机与定位：ds4 是为 DeepSeek V4 Flash／PRO 量身打造的、完全自包含的原生推理引擎，刻意做窄，不做通用 GGUF 运行器或套壳。其核心取舍是「窄而精」（一次只死磕一个模型，机会主义式可整体替换）、「官方向量校验」（用官方实现 logits 作基准对齐）、「本地高端机器优先」（96／128GB 起步），并描画出 A 引擎+API、B 专用 GGUF、C agent 测试校验、D 专用 agent 的四件套愿景。关键术语为「KV 缓存即一等磁盘公民」——压缩 KV 配合快 SSD 使其值得被磁盘持久化、复用与淘汰，代码落点是 ds4_kvstore.c。其二，与 llama.cpp／GGML 的关系：ds4 不链接 GGML，架构与工程（推理图、MLA 注意力、MoE 调度、后端、服务器、agent、磁盘 KV 格式）全自建，仅在 MIT 协议下源码级继承 GGUF 量化布局与 CPU 点积逻辑，并在 LICENSE 保留 GGML 版权；「格式与数学借鉴，架构与工程自建」。其三，质量状态：引擎、服务器、GGUF 为 beta，ds4-agent 为 alpha，新近的分布式推理与 SSD 流式尤其需谨慎；贡献底线是「正确性优先于速度」，须保护 Metal、SSD 流式、分布式、CUDA 四条命脉路径；出问题用 --dump-tokens、--dump-logprobs、--trace 三件套自检。
