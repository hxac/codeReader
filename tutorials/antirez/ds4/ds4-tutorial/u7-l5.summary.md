本讲讲解 ds4-server 对无状态「重发整段对话」请求如何逐级复用活 KV checkpoint：按 memory-token（逐 token 比对，common==old_pos 且仅纯追加才命中）→ memory-text（把活 checkpoint 解码成文本做字节前缀比对，命中时遵循「活 graph 是权威」，保留活 token、只对新增文本重新分词，避免 BPE 跨边界合并错位）→ 磁盘文本前缀 → 冷 prefill 逐级尝试，越靠前越便宜。另有一独立入口 canonicalize_tool_checkpoint，在工具调用生成后用 rewrite_from_common 把活 checkpoint 改写成下一轮将渲染的字节（规范化）；其判定函数 rewrite_requires_rebuild 的全部逻辑即 common<live_len：末尾追加安全、活末端之内改写必须重建，根因是压缩 KV 的 compressor frontier 不可廉价回退；REBUILD_NEEDED 时不改 session，先回退磁盘旧快照只 replay 后缀、再退到整段重建。核心结论：前缀复用只奖励末尾追加、惩罚中间改写。
