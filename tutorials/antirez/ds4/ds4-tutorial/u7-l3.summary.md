本讲承接 u7-l2，往下钻一层讲 `stream:true` 请求下 ds4-server 往连接上写什么。核心是三个最小模块：SSE 事件生命周期、thinking 流式形态、include_usage。ds4 维护两套半流式实现——OpenAI Chat（连续 `chat.completion.chunk`）、Responses（带 `type` 与 `sequence_number` 的生命周期事件）、Anthropic（`event:` 命名事件 + content block 生命周期），由 `request_uses_*_stream` 判定，均遵循「开场事件→逐 token 增量→收尾事件+usage」三段式骨架。关键概念是三套同构的 `THINKING/TEXT/TOOL/SUPPRESS` 状态机，靠 `emit_pos` 游标推进、只发安全区间，能把同一段 `<think>` 翻译成 OpenAI 的 `reasoning_content` 字段、Anthropic 的 `thinking` 内容块、Responses 的 `reasoning` 输出项；Responses 推理默认 opt-in 且未闭合时标 `incomplete`。`stream_options.include_usage` 为 OpenAI 风格专属，由 `sse_usage_chunk` 在 `[DONE]` 前发 `choices:[]` 统计帧，透传 `cached_tokens`/`cache_write_tokens`。两条工程底线：prefill 期间提前发头并每 5s 发心跳防超时，`utf8_stream_safe_len` 非绝不切断多字节字符以保住客户端回放逐字节一致与后续 KV 前缀复用。
