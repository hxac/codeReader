`ds4-eval` 是 ds4 自带的能力评测工具，但它不是 leaderboard 跑分器，而是一套**能力回归套件（capability regression suite）**——核心工程问题不是“模型得几分”，而是“改了内核/量化/提示/KV 缓存/工具流后，模型能否用同一条推理路径解一组有代表性的硬题”，因此它追求分数的稳定性与逐题可见性，而非满分。本讲覆盖三个最小模块：**题集与难度**——92 道题硬编码进二进制，分 GPQA Diamond(25)、SuperGPQA(25)、AIME 2025(25)、COMPSEC(17)，前 75 题交替渐进式排列，题型完全由 `source`/`choice` 字段派生，无显式 type；**抽取与评分**——`find_case_answer` 与 `answer_matches` 各自三分（多选字母、整数、COMPSEC 行号），共享“先跳 `<think>`、优先强 `Answer:` 标记、兜底松散回退”策略，关键在避免假阴性污染回归信号；**TUI 与 regrade**——纯 ANSI 双栏分屏（不依赖 ncurses），`run_one_case` 是 prefill→采样循环（含 think-close 预算控制器）→抽取评分的三段式。引入的关键术语：trace 轨迹、regrade 重判、q1..q4 确定性闸门、`--self-test-extractors`/`--regrade-trace` 两条加载模型前的离线审计路径。
