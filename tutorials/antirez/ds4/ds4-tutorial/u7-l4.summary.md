DSML 是 DeepSeek V4 的工具调用文本格式，以全角竖线包裹的 `｜DSML｜` 标记（如 `<｜DSML｜tool_calls>`）表达工具调用，模型直接以 token 流生成，省去 JSON 转换、与普通回答共用生成路径与 KV 缓存。参数区分 `string="true"`（原始文本，仅保护闭标签转义）与 `string="false"`（JSON 字面量）。服务器在系统提示中「教」模型写 DSML，再由 `parse_generated_message_ex` 解析回结构化对象并投影为 OpenAI/Anthropic 协议。

本讲核心矛盾是无状态 API 重发工具调用时的 KV 前缀失配：客户端以 JSON 回发历史，重渲染出的字节很难与模型当时采样的 DSML 逐字节相等，导致 `ds4_session_common_prefix` 在工具调用处断开、后缀需重 prefill。解法是「精确回放」：为每个调用分配不可猜测的 tool id，用 rax 基数树维护 `by_id`（id→条目，主查询）与 `by_block`（字节→块，去重），加 LRU 链表淘汰，默认 100000 id/512 MiB，并持久化到磁盘 KV 的 KTM 段跨重启恢复。下一轮 id 命中时，`append_dsml_tool_calls_text` 逐字节抄写 `raw_dsml`，使渲染字节等于活 KV 字节、前缀复用不断。失灵时（id 被淘汰/重启未恢复/`--disable-exact-dsml-tool-replay`/`raw_dsml` 为空）则走规范化后备：`canonicalize_tool_checkpoint` 主动把活 checkpoint 改写成下一轮将渲染的字节，改不动则回退磁盘快照重建。此外，生成中对 DSML 语法 token 强制贪婪采样、对参数负载保留正常采样，以保证可解析性。
