本讲拆解「Agent 联网」的端到端链路，含三个最小模块。一是 web 回调接口：`ds4_web` 是「无主见」的浏览器子系统（不透明类型），以 `confirm`/`log`/`cancel` 三个回调把开浏览器许可、日志去向、可否中断交调用方注入，体现依赖反转，对外仅 `create`/`free`/`google_search`/`visit_page` 四个函数。二是搜索与访问实现：`google_search` 与 `visit_page` 共用 CDP（Chrome DevTools Protocol）管线 `web_run_page_js`，差别仅在 URL、抽取 JS、是否滚动；子系统不用 headless，而是手写 HTTP/WebSocket/CDP 客户端，拉起可见的真实 Chrome（持久 profile，更不易被风控），`web_ensure_browser` 提供懒启动与一次性许可闩 `browser_allowed`，取消靠 50ms 切片轮询 `cancel`，`ds4_web_free` 故意不杀 Chrome。三是 agent 集成：三个回调 `privdata` 均为 `agent_worker`；`confirm` 因 worker 不能读键盘而用条件变量跨线程握手问 UI（非交互模式直接拒），`log` 接 trace，`cancel` 转发统一中断谓词；`visit_page` 用「头部 100 行/8KiB 摘要＋临时文件」裁剪输出避免撑爆 KV。核心结论：web 工具被严格绑定到交互模式与人工确认，结果以 `tool` 消息进 transcript，天然享受 KV-as-session，无需服务器精确回放。
