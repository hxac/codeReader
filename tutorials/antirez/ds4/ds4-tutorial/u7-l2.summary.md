本讲专讲 ds4-server 的 HTTP 端点层，含端点路由、参数映射、模型别名三个模块。路由在 `client_main` 内用线性 `method+path` 字符串匹配分发：GET `/v1/models` 直接回 JSON，四个 POST（chat/responses/messages/completions）各调一个 `parse_*_request`，把请求解析进统一 `request`，并用正交标签 `(req_kind, api_style)` 区分方言（completions 为 `(REQ_COMPLETION, API_OPENAI)`，其余三类为 `REQ_CHAT`，api 分别为 `API_OPENAI`/`API_RESPONSES`/`API_ANTHROPIC`）。参数映射靠手写 JSON 遍历读 key 填字段，未知 key 用 `json_skip_value` 跳过，故额外字段不报错；各方言在必填字段、系统提示位置、工具形态上差异最大，Responses 还显式拒绝 `tool_choice=required` 与 `previous_response_id`/`conversation` 非空。`model` 字段为装饰性：flash/pro 是不切换 GGUF 的兼容别名，deepseek-chat/reasoner 在客户端未显式控制思考时隐式开关思考。四种方言最终汇流到同一份 prompt token 序列，graph worker 完全不感知客户端用了哪种 API。
