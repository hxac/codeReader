本讲介绍 ds4 离线模型构建工具链的入口 `gguf-tools/deepseek4-quantize`，一个**纯 C、模型专用、不链接 GGML** 的 safetensors→GGUF 量化器，覆盖三个最小模块：

第一，**模板 + HF 重生成**。工具采用双输入设计：`--template`（已存在的 ds4 GGUF）贡献**骨架**——KV 元数据（分词器、架构常量、`deepseek4.expert_count`）、张量顺序与逻辑形状，其旧量化字节被丢弃；`--hf`（safetensors 目录）贡献**血肉**——每个张量的真实权重字节（FP8 E4M3 / FP4 / F16 等）。两者靠 `top_map`/`layer_map` 这对人写命名翻译表衔接（GGUF 扁平名 ↔ HF 嵌套名）。流程是把 HF 字节反量化回 float32，再按本 run 配方重新量化、流式写出。

第二，**张量族机制**（`quant_policy` + `policy_type`）。把张量按语义分族——routed 专家三段（w1 gate / w2 down / w3 up）、attention 投影、shared、output、embedding、dense——每族用 `--routed-w1` 等开关单独指定类型，按固定优先级判定。可实际写出的量化类型**仅四种**：`q8_0`、`q2_K`、`q4_K`、`iq2_xxs`（`can_quantize=true`）。

第三，**校验入口**：`--dry-run`（只打印计划、不读 HF）、`--compare-tensor`（重生单张量并 FNV-1a64 字节比对）、`--imatrix-strict`（拒绝合成兜底），外加 `--overwrite` 安全闸。imatrix 分三态：外给 `.dat`、权重平方和合成兜底、严格报错，溯源写入输出元数据。

承接关系：量化块字节布局见 u3-l4，imatrix 采集与质量打分留 u11-l2，整条 prefill 的 logit 级回归留 u11-l3。
