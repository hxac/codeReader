本讲解析 ds4-agent：单进程双线程（UI 线程管终端、worker 线程独占活 ds4_session 与 KV），推理在进程内、无 socket 边界；核心哲学是 KV-as-session——会话即一个 .kv 文件（KVC 格式），保存=序列化活 KV，恢复=反序列化回内存并跳过 prefill。核心结论：每个推进 KV 的 token 都被同一行代码同步追加进 transcript，活 KV 与 transcript 恒等，故 KV mismatch 按构造不可能，无需 u7 服务器的精确 DSML 回放与规范化机制。前缀复用仍保留（common_prefix 测量、sync 只 prefill 后缀），但只追加不改写、几乎总命中。垂直设计表现为默认值专为 DeepSeek V4 调、工具调用原生 DSML 无 JSON 转换、低延迟受 prefill 约束。关键术语：transcript（唯一真相）、sysprompt.kv（系统提示固定检查点）、stripped 会话（删 payload 留文本、重开需 prefill 重建）、会话身份=标题与创建时间的 SHA（稳定文件名），加载时按 model_id 与 quant_bits 校验。内建工具与 web 搜索留待后续讲义展开。
