本讲讲清「文本如何变成模型 token 序列」这道桥梁，覆盖三个最小模块。其一是词表加载与字节级 BPE：ds4 从 GGUF 读出 token 字符串表与合并规则表，建 `token_to_id`、`merge_rank` 两张哈希表，并按名字查出 7 个特殊 token id；运行时经预切分、字节编码、按 rank 贪心合并产出 id。其二是聊天模板与 thinking 模式：`encode_chat_prompt` 按 `BOS → (Max 前缀) → 系统提示 → [User] → 正文 → [Assistant] → <think>/<\/think>` 拼接，控制标记按 id 直插、正文才走 BPE；none/high/max 三档在分词阶段就改写序列，上下文不足时 Max 降级为 high。其三是 DSML 与渲染回扫：以 BOS 开头的整段文本由 `tokenize_rendered_chat` 逐字符扫描，特殊标记（含全角竖线的 `｜DSML｜`）被原子识别不进 BPE；`--dump-tokens` 走回扫路径、推理前退出，是排查分词的首选工具。关键术语：字节级 BPE、merge_rank、聊天模板、thinking 三档、渲染回扫、DSML 特殊 token、dump-tokens。
