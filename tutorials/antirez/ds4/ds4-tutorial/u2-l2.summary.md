本讲剖析 ds4 CLI 前端（代码集中在 ds4_cli.c）从 main 到吐出首个 token 的主流程。配置分两层：cli_config 是总容器（内嵌 engine/gen，dist 用堆指针延迟构造），cli_generation_options 汇集提示、采样参数与思考模式等生成旋钮，由 parse_options 从 argv 一次性填充。main 以 cfg.gen.prompt 是否为 NULL 分流：非空走一次性 run_generation，空走交互 run_repl；其前还有 dump-tokens、inspect、perplexity、分布式 worker 等短路特例。run_generation 再按 temperature 分发：默认 1.0 走采样路径 run_sampled_generation，--temp 0 才落到更快的 ds4_engine_generate_argmax。采样路径为 create session → session_sync(prefill) → while(sample→eval) 三段式；REPL 的 run_chat_turn 用 session_common_prefix 做前缀复用，仅对新增后缀 prefill。本讲还引入协作式中断：Ctrl+C 仅置 volatile sig_atomic_t 标志 cli_interrupted，循环在每个 token 之间检查以保 KV 停在合法前缀，处理器只在 run_repl 安装。
