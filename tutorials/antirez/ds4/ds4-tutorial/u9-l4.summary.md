本讲承接层切分，讲解多机如何串成推理流水线。协议分两种连接：控制连接跑 HELLO 注册 worker 的身份与层段，数据连接跑 WORK/RESULT 传重数据。coordinator 用递归回溯把各 worker 层段首尾相接，拼成覆盖全部层、output head 有归属的 route；route_blob 随每个 WORK 帧下发，使中间 worker 直接转发下一跳，coordinator 不中转。prefill 可流水线加速：算 chunk N+1 的同时 worker 算 chunk N，flow window 控制在飞 chunk 数；生成严格自回归，每 token 一圈跨机往返，必慢于单机。每个 worker 维护滚动 FNV-1a token-prefix hash 作 KV 状态指纹，干活前核对 prefix_hash 拒绝漂移。故障分两类：传输失败丢路由、等兼容 worker 重连重组；逻辑失败（KV/hash 不匹配）保路由原地重放；两者最后都用 RESET_SESSION 重放整段 transcript 重建 KV。协议无加密无鉴权、非 release-stable。
