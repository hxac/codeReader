本讲是 ds4 的「起跑线」，覆盖下载与两种运行姿态，不涉及推理内核。模块一：`download_model.sh <target>` 用语义名（如 `q2-imatrix`）屏蔽真实文件名，普通 GGUF 用 `curl -C -` 断点续传到 `.part` 再 `mv`，中途断开重跑同命令即可续传；PRO 大文件（约 430GB）强制走官方 `hf` CLI，且不接管 curl 留下的 `.part`；主模型下完后软链到默认路径 `ds4flash.gguf`（`ds4`/`ds4-server` 默认模型路径），分布式分片与 mtp 因非单文件主模型而不更新此软链。模块二：`ds4_cli.c` 的 `main` 按 `prompt==NULL` 分流——有 `-p` 走 `run_generation`（默认贪婪路径 `ds4_engine_generate_argmax`），无 `-p` 走 `run_repl`。关键术语 **prefill**（一次性填 KV 缓存）决定首 token 延迟（≈提示 token 数 ÷ prefill t/s），**decode**（自回归生成）决定生成速度，二者独立。模块三：REPL 用 `repl_chat`（`transcript` 已渲染聊天转录本 + 一个活 `ds4_session`）实现多轮上下文复用，斜杠命令 `/think`、`/ctx N`、`/power N`、`/read FILE` 可运行时调整；切换 Think Max 或 `/ctx` 会改 transcript 结构或缓存容量，导致 KV 失效重建。排查问题可先用 `--inspect` 只加载不推理。
