本讲只盯 `ds4.h` 与 `ds4.c` 的生命周期函数，确立 ds4 引擎的两条公共边界。`ds4_engine` 代表已加载模型，进程级、基本只读，持有权重、词表与 GPU；`ds4_session` 代表一条可变推理时间线，对话级、可反复创建释放，持有 KV 缓存与 logits，并通过反向引用共享 engine 的权重。两者都声明为不透明指针，使 CLI、服务器、agent 等前端不依赖张量内部结构，实现"窄头"。打开引擎靠配置包 `ds4_engine_options`，集中了路径、后端、推理控制、SSD 流式、分布式、功耗、检查七大类字段，其中分布式选项嵌套为 `ds4_distributed_options` 子结构体。`ds4_engine_open` 几乎逐字段消费配置包，处理方式分四种：直接拷贝、带钳位拷贝、触发副作用（mmap 加载 GGUF、GPU 初始化、获取进程单例锁）、分支跳过（`inspect_only` 在 GPU 初始化前提前返回，实现 `--inspect`）。生命周期主线为 `open → summary → close`：open 昂贵、summary 仅委托 `model_summary`、close 按逆序释放并对 NULL 安全。open 用二级指针出参、int 返回状态，失败时内部已自清理。本讲确立的边界与术语（engine/session、配置包、单例锁、sync 的复用/增量/重建语义）将由 u2-l2 的 CLI 主流程与 u2-l3 的 session 同步算法承接。
