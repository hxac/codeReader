本讲解析 DeepSeek V4 单层 KV 缓存的三个最小模块及其各自解决的问题。其一为 **raw 滑动窗口**：每层都有、固定 128 行的环形缓冲，写满后整体前移一格，存最近 token 的精确 KV（fp16 落盘），内存与上下文无关，承担近期上下文的 dense 注意力。其二为 **compressor（压缩器）**：把每 `ratio` 个 token 经逐维 softmax 池化流式压成 1 行 512 维压缩 KV，攒满一整块才吐行；正在攒的半窗口状态叫 **compressor frontier（压缩前沿）**，必须随行计数器一起序列化、不可廉价回退，压缩行永不淘汰，承担长期记忆的存储压缩。其三为 **ratio-4 indexer（索引器）**：仅 ratio-4 层出现，用一套 128 维小压缩行给所有压缩行打廉价分、选 top-k（Flash 512 / PRO 1024）生成掩码，把每 token 的压缩注意力打分行数从随上下文增长封顶为常数。按层 ratio 来自 GGUF 元数据（Flash 第 0/1 层为 0，之后偶数 4 奇数 128）并严格校验。核心结论一句话：**compressor 让你存得下长上下文，indexer 让你在长上下文里算得动**，再加固定容量的 raw 窗口，共同支撑 1M 上下文在本地可行（实测约 26GB）。
