本讲讲 ds4 分布式推理的架构与层切分，覆盖三个模块。其一，角色划分：只有 coordinator 与 worker 两种角色，coordinator 持分词器、prompt 与采样器且必须拥有从第 0 层起的一段（因为 token 序列只在它手里，需先做 token_embd 查表），worker 只负责中间或末尾一段层并各自持有自己的 KV 切片；worker 之间直连转发 hidden state，coordinator 不当中转。分布式是编译进引擎的「后端」而非独立前端，前端仍调用普通 `ds4_session_*`，由 coordinator 侧的 `ds4_dist_session` 沿路由透明分发。其二，`--layers A:B` 为闭区间，`A:output` 表示「到末层加 output head」，`output` 在解析时存成哨兵 `UINT32_MAX`，加载时由 `dist_resolved_layer_end` 折算成真实末层；output head 归属是核心取舍——默认末位 worker 持有（`N:output`，prefill 友好），慢链路可用 `N:42` 让 coordinator 持 output head、每 token 回传更小的 hidden state 而非高维 logits，代价是 coordinator 多扛权重与计算。其三，切分复用 u3-l2 的同一份 `weights_bind`：`ds4_dist_prepare_engine_options` 把 `--layers` 翻译成 `load_slice/load_layer_start/load_layer_end/load_output` 字段族，引擎打开时层循环与 mmap 区间都收窄到本段层，token_embd 只在 coordinator（start==0）绑定。路由就绪条件为 coordinator 从 0 起、一串 worker 首尾相接覆盖到末层、且 output head 有归属。协议帧与流水线留待下一讲。
