本讲讲解 `ds4_session` 的同步与 KV 复用，含三个最小模块：前缀匹配与增量评估、rewrite 判定、会话控制原语。session 用 `checkpoint`（KV 当前对应的 token 序列）与 `checkpoint_valid`（是否可信）刻画状态。`ds4_session_sync` 是总开关：当 checkpoint 为 prompt 的严格前缀时只评估后缀（后缀≥4 走续 prefill 图，否则逐 token decode），否则整段重建，且重建前先作废状态以保证失败安全；`ds4_session_common_prefix` 供前端测量前缀长度。rewrite 判定 `ds4_session_rewrite_requires_rebuild` 以 `common < live_len` 表达「末尾追加安全、中间改写须重建」，根因是压缩 KV 前沿无法廉价回退，REBUILD_NEEDED 时服务器优先回退磁盘快照。控制原语 `invalidate/rewind/eval/pos` 对 `checkpoint_valid` 处理不同，决定下次 sync 走向；协作式取消返回 `INTERRUPTED`，使中断停在合法前缀、可原地续算。
