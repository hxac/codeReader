本讲讲 ds4 的 Metal 后端（ds4_metal.m 这层 Objective-C 胶水）如何把 Metal 设备、.metal 内核与 C 推理图粘合，含四个最小模块。运行时把 device/queue/library/pipeline 做成进程级全局懒初始化，由 ds4_gpu_init 固定顺序创建、逐级失败回滚。metal/*.metal 不参与 make 编译，而在进程启动时被拼接成大源码，由 newLibraryWithSource 运行时编译，并用一组 DS4_METAL_* 宏焊死敏感数值路径以保证采样可复现。命令走批处理：多个 dispatch 共享 encoder 录进同一 command buffer，flush 提交并新开以隐藏延迟、end 提交并等待；激活/KV/scratch 为 StorageModeShared 常驻张量，权重经 newBufferWithBytesNoCopy 零拷贝包装并按 mmap 区段缓存。推理采用 layer-major 图调度：一个 chunk = 一个 command buffer = 全部 43 层（attention+ffn）+ output head，层间靠双缓冲残差交替；长 prompt 按 DS4_METAL_PREFILL_CHUNK（默认 4096/8192）切分并对齐到边界，与 KV checkpoint 边界一致。代表内核为 flash attention（C++ 模板用 host_name 特化出 f16_dk512_dv512 等入口）与 dsv4_kv（把官方 FP8 KV 语义、compressor frontier 更新焊进专用内核），二者在一次 prefill 的某层 attention 编码里被绑定进同一 command buffer。本讲不展开 MLA/MoE 数学与 CUDA/ROCm 后端。
