DSV4 session payload 是 KVC 文件中间层、由 ds4.c 独占格式，字节布局为：13 个小端 u32 自描述头部（magic "DSV4"/version 2、图布局与运行时容量）+ checkpoint token 序列 + logits + 两个行计数数组 + 逐层 raw/compressed/indexer KV 与 compressor frontier。

三个核心结论：(1) 头部做身份与图布局校验，层/head/vocab 须与编译期常量逐位匹配，但 prefill_cap 被显式忽略（它只是调度旋钮、非持久布局），raw_window 必须匹配；(2) logits 紧随 checkpoint tokens 写盘，使加载后可直接采样、省掉一次额外 decode，MTP draft 状态不持久化、加载后作废重建；(3) 逐层张量按 live 行数而非容量计量，故 payload 随已保存 token 缩放——raw 只存窗口内活行（按逻辑序写以跨物理 ring 可移植）、compressed/indexer 从行 0 起存全部活行、frontier 必须随行计数一起写盘。

引入的关键术语：payload、live row count、compressor frontier、checkpoint_valid、staging（先写临时文件量出 payload_bytes 供外层做预算/淘汰）、失败安全加载（先影子构建、成功才提交、末尾 remaining==0 校验，读侧配额由外层 KVC 头的 payload_bytes 框定）。
