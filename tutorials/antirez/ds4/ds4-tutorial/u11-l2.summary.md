本讲回答两个工程问题：量化用的 imatrix（列重要性）从哪来、如何客观衡量量化保真度。收集端 `ds4_engine_collect_imatrix` 复用 Metal prefill 图，读回已物化的 `batch_ffn_norm`/`batch_routed_mid`/`batch_router_selected`，对每个路由命中的 routed MoE 专家累加输入平方和（gate/up 共用累加器、down 独立），归一化后写成 llama.cpp 兼容的 `.dat`（每条目打包 n_expert 个专家向量），全程只读旁路、不改推理数学、仅 Metal 可用。消费端 `deepseek4-quantize` 读 `.dat` 指导量化；类型 `requires_imatrix`（仅 IQ2_XXS）且缺失时用权重列平方和作合成兜底（权宜之计，统计的是权重而非激活），`--imatrix-strict` 禁止兜底。打分端 quality-testing 三件套：`collect_official.py` 确定性抓官方续写、`score_official` 用教师强制逐 token 读 logprob 算目标 token NLL、`compare_scores.py` token 加权对比，NLL 负向变化说明更接近官方。本讲测分布保真度，与 ds4-eval 的能力下限互补。
