本讲是磁盘 KV 缓存的策略层，回答两个问题：何时写 `.kv` 检查点、预算吃紧时删哪个。它在四个时机写盘，每个时机带一个**原因码（reason code）**，写入 KVC 固定头第 5 字节：**cold**（首段长 prompt prefill 完、生成前，存稳定前缀）、**continued**（命中对齐前沿，约每 10240 token 留重启点）、**evict**（加载磁盘快照替换活 session 前，救活当前会话）、**shutdown**（干净退出前）。四种写入在 kvstore 层汇流到同一函数，区别仅在 reason。cold/continued 的落点先**裁掉 32 尾 token**（防 BPE 跨界合并导致前缀失配），再**向下对齐到 2048 网格**，使压缩行最终化与一次冷全量 prefill 逐位一致；continued step 把 10000 向上对齐到 10240，与 cold 共享同一张 2048 网格；cold 还优先落在「第一个 assistant 标记前的最后一个 user 标记」这个**聊天锚点**。淘汰用**评分最低者优先删**：`score = (衰减命中+1)·tokens/file_size`，命中计数按 **6 小时半衰期**衰减；cold/evict/shutdown 这类 **anchor** 原因享 2 倍软保护；被新检查点取代的 **continued 路标**压到 5%–50%，是廉价牺牲品。预算带 1% 安全裕量，写前后各判一次，启动时也跑一次淘汰。
