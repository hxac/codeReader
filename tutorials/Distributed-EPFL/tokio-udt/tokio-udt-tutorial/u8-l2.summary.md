本讲覆盖 UDT 连接生命周期的终点，三个最小模块：close+linger、Shutdown 包与状态迁移、garbage_collect_sockets。close() 顺序：幂等守卫→读 linger_timeout（None 即 ZERO、不逗留）→ linger 循环→从 mux.snd_queue 摘除并按需清 mux.listener→仍为 Connected 才 best-effort 发 Shutdown→置 Closing→notify_all。linger 三退出条件：状态离开 Connected、发送缓冲空、超时。核心设计分工：teardown（发 Shutdown、置 Closing）由 socket 自管，因 close() 仅持 &self、够不到全局写锁；从注册表删除外包给后台 GC。Shutdown 是对称信令，收端只置 Closing、不调本地 close()。GC 两阶段：阶段一处理 Broken（从 listener 队列摘除后 spawn 一个 close()，不 await 以免占住写锁）；阶段二处理 Closing（先收 id 再删、置 Closed）。cleanup_worker 每秒一轮。
