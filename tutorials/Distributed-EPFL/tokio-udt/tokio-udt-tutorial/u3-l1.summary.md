本讲聚焦 tokio-udt 的进程级「总账本」设计，是最小模块 `src/udt.rs` 的核心讲解，为进阶单元（u3）的地基。tokio-udt 用 `OnceCell<RwLock<Udt>>` 实现全局单例 `UDT_INSTANCE`：`Udt::get()` 返回 `&'static` 读写锁引用，初始化闭包内顺带 `tokio::spawn` 一个后台 GC worker，且因 `spawn` 仅排队而不会递归死锁。`Udt` 内部维护三张 `BTreeMap` 注册表：`sockets`（本地 socket id → `SocketRef`，即 `Arc<UdtSocket>`）、`multiplexers`（mux id → `Arc<UdtMultiplexer>`）、`peers`（`(对端 socket id, isn)` → 本地 socket id 集合，用于握手去重），外加随机起点、`wrapping_sub` 递减分配的 `next_socket_id`。核心方法 `new_socket`/`get_socket`/`update_mux`/`new_connection` 分别完成建表登记、带 `Closed` 存活判定的查找、mux 复用或新建（客户端 `connect` 默认 `bind_addr=None` 故不复用）、服务端握手建连（新连接经 `with_listen_socket` 复用 listener 的 mux，一个 listener 通常只对应一个 UDP socket）。后台 `cleanup_worker` 每秒取写锁执行 `garbage_collect_sockets`：对 `Broken` 连接 spawn 异步 `close()` 以缩短持锁时间，对 `Closing` 连接直接摘除并置 `Closed`。用户全程不直接接触该单例，由 `UdtListener`/`UdtConnection`/收发 worker 透明使用。关键术语 `SocketRef`、`UdtStatus`、mux 复用、`peers` 去重将承接至 u3-l2 状态机与 u3-l3 多路复用器。
