本讲是全系列第一篇，不动手写协议代码，仅通过 README、Cargo.toml、lib.rs 三个门面文件建立项目总览。核心结论有三：其一，UDT 是构建在 UDP 之上的高性能可靠传输协议，旨在克服 TCP 在高速广域网上的「效率」与「公平性」问题，同时提供可靠数据流（streaming）与消息服务（messaging）两类服务；基于 UDP 的原因是 TCP 拥塞控制固化在内核中，应用层难以替换。其二，tokio-udt 是 UDT 的 Rust + Tokio 实现，版本 0.1.0-alpha.8（API 尚不稳定）、edition 2021、许可证 AGPL-3.0（强 copyleft，需做合规评估）；技术栈以 tokio 为核心，辅以 socket2/nix、bytes、sha2（握手 cookie）、once_cell、rand，Linux 下额外引入 tokio-timerfd 做高精度定时。其三，对外公共 API 仅 5 个类型：UdtConfiguration、UdtConnection（实现 AsyncRead/AsyncWrite）、UdtListener（bind+accept）、RateControl、SeqNumber；README 示例只用到 UdtListener 与 UdtConnection。lib.rs 另声明 17 个私有子模块，构成后续逐模块拆解的目录骨架。
