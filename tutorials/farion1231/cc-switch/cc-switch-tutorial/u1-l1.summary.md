CC Switch 是基于 Tauri 2 的跨平台桌面应用，定位为「七种 AI CLI 工具的统一配置管家」。它解决的痛点是：各工具配置格式互不相同（JSON/TOML/.env/YAML/JSON5），手动切换供应商既繁琐又易损坏文件。本讲覆盖四个最小模块：项目定位、支持的七种工具、核心功能矩阵、数据存储位置。

支持的七种工具为 Claude Code、Claude Desktop、Codex、Gemini CLI、OpenCode、OpenClaw、Hermes，各有独立配置目录与格式，其中仅 Claude Code 支持热切换，其余切换后需重启。核心功能包括供应商管理、本地代理与故障转移、MCP/Prompts/Skills 管理、用量统计、会话管理及平台集成（托盘/Deep Link/云同步），各功能域分别对应手册后续单元。

核心设计理念为 SSOT（单一事实源）：可同步数据集中存于 SQLite 数据库 `cc-switch.db`，设备级偏好（语言、主题等）放 `settings.json` 且不参与同步，并以原子写入保护配置。关键术语：Provider（供应商）、Live 文件（真实配置文件）、SSOT、双层存储、双向同步、原子写入、回填（backfill）；核心数据流为「数据库（SSOT）→ 切换写入各工具 Live 文件 → 回填读回手改内容」。
