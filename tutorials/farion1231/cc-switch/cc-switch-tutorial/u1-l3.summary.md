本讲是「动手」首讲，覆盖四个最小模块：①工具链要求——Node.js 18+、pnpm 8+、Tauri CLI 2.8+（随 `@tauri-apps/cli` 安装），Rust 最低版本以 `Cargo.toml` 的 `rust-version = "1.85"` 为准（机器校验，权威于 README）。②开发命令——前端 `pnpm dev/build/typecheck/format:check/test:unit`，其中 `dev`、`build` 转发给 `tauri`；Rust 在 `src-tauri/` 下用 `cargo fmt/clippy/test`，`--features test-hooks` 按需启用测试专用钩子。③前端 `src/` 按「层 + 领域」组织：`components/`、`hooks/`、`lib/`(api/query)、`config/`(预设)、`types/`，同一领域在多层成对出现。④后端 `src-tauri/src/` 将各 CLI 工具配置写入器放根级、业务逻辑进 `commands/ → services/ → database/`，`commands/mod.rs` 是命令层权威汇总入口。核心约定是「按领域组织」：从 UI 一路追到后端命令的链路靠它贯通；文档与实际目录不一致时（如 `locales/` vs `i18n/`）以实际目录为准。
