本讲承接「项目定位」，回答 cc-switch「用什么技术做、代码怎么组织」，只建立全局地图，不深入单模块实现。它覆盖四个最小模块：前端技术栈、后端技术栈、分层架构图、核心设计模式。技术栈上，前端基于 React 18 / Vite / TailwindCSS / shadcn-ui / TanStack Query，外加 CodeMirror、dnd-kit、i18next；后端基于 Tauri 2.8 / Rust，核心依赖 tokio（异步）、rusqlite（SQLite）、serde 配合 toml/yaml/json5 等多格式库（适配七种 CLI 工具的异构配置）、axum/hyper（内置本地代理）。核心结论：前后端各自分层，经 Tauri IPC 拼接——前端 Components → Hooks → Query → API，后端 Commands → Services → DAO → Database，同一领域在三层成对出现；一次「切换供应商」操作即沿此路径从 UI 下沉到数据库再原子写回 Live 文件。本讲引入的关键术语：Tauri IPC（invoke/listen）、SSOT 单一事实源、双层存储（可同步数据进 SQLite、设备级偏好进 settings.json 且不参与同步）、双向同步（数据库 ↔ Live 文件回读 backfill）、原子写入（临时文件 + rename）、并发安全（Mutex 包裹数据库连接）。这些模式贯穿后续每一讲，是理解全项目的钥匙。
