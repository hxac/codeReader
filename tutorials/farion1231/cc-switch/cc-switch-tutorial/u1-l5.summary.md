本讲梳理后端启动主线。极薄的入口 main.rs 仅做平台环境修正（隐藏 Windows 控制台、修 Linux 渲染），随后转交库入口 run()。run() 是 Tauri Builder 的四段式骨架：崩溃钩子兜底、插件链（single_instance、deep_link、store、window_state 等）、setup 钩子、命令白名单 invoke_handler，最后生成应用进入事件循环。setup 钩子是启动最重的一段，按序完成日志初始化、数据库初始化与迁移、provider 官方预设种子、MCP/Prompts 导入、托盘与后台 worker，再经 app.manage 注入全局状态，遵循失败可重试、按表独立导入、重活异步化的纪律。本版本新增「数据库版本过新」恢复早退：Database::init 前用 stored_user_version_exceeds_supported 只读探版本，若磁盘版本高于 SCHEMA_VERSION（当前为 11），即写 InitErrorPayload(kind=db_version_too_new)、强制显示主窗口并提前返回成功，前端据 get_init_error 渲染应用内升级恢复界面，既不写坏数据库也不闪退。需承接的关键术语：Tauri Builder、setup 钩子、invoke_handler、AppState（db/proxy_service/usage_cache 三字段）、init_status 静态信箱、check_app_update_available。
