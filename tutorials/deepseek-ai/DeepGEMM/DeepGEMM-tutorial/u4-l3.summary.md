本讲打通 JIT 系统的「最后一公里」：把磁盘上的 `kernel.cubin` 加载成可启动的内核句柄并推上 GPU。核心模块有四：其一，`KernelRuntime` 经 `load_kernel` 将 cubin 加载为 `LibraryHandle` + `KernelHandle`，默认走 Driver API，靠 `cuLibraryEnumerateKernels` 枚举并断言「恰好一个 kernel」，从而绕开 mangled name，旧驱动则回退 `cuobjdump` 按名查找；`lazy_cu*` 符号通过 dlopen/dlsym 懒加载，使 `_C` 扩展不硬绑定 driver 版本。其二，`construct_launch_config` 把 grid/block/smem/stream 四件套打包成 `LaunchConfigHandle`，先 `cuFuncSetAttribute` 抬高共享内存上限，再按需挂 cluster、PDL 两个启动属性（`attrs` 必须为 `static`）。其三，关键结论：`cluster_dim` 由启发式 layout 配置决定，`enable_pdl` 由全局开关 `device_runtime->get_pdl()` 无条件覆盖（默认关闭）。其四，`LaunchRuntime<Derived>` 用 CRTP 通过 `generate`/`launch` 公共方法加 `generate_impl`/`launch_impl` 钩子统一十几个 kernel 的流程，宿主侧以 `generate → build → launch` 三行胶水完成从配置到启动。引入的术语 cubin、句柄、LaunchArgs、cluster、PDL、CRTP 供后续讲义承接。
