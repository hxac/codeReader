本讲（u9-l2）承接 u9-l1 的非分页 MQA 评分内核，讲解解码阶段的分页版本 `fp8_fp4_paged_mqa_logits`，覆盖三个最小模块：

一是**分页 KV 寻址与 fused 布局**：解码阶段各请求上下文长度不一、显存碎片化，故把 KV 切成固定大小 `block_kv` 的「页」离散存放，用 `block_table` 把逻辑页偏移映射到物理块号。`fused_kv_cache` 将每个 token 的数值区与逐 token 缩放因子（SF）紧凑放于同一物理块，宿主用 `from_blob` 零拷贝拆成 `kv_cache` 与 `kv_cache_sf` 两个视图；分页 KV 用 **3D TMA 描述符**（外维即物理块号），区别于非分页的 2D 描述符。

二是**调度元数据与跨 SM 负载均衡**：用户须先调 `get_paged_mqa_logits_metadata`，其内启动一个轻量 metadata kernel，按 `work = num_kv_splits * num_q_tokens` 做前缀和并均摊到每个 SM，产出 `[num_sms+1, 2]` 的 `schedule_meta`（每行一对 `(q_token_idx, kv_split_idx)`，末行为「全部结束」哨兵）。主 kernel grid 恰为 `num_sms`，每个 CTA 读自己的 `[start, end)` 半开区间动态领任务。

三是 **split-K 无归约并行**：长上下文下以 `SPLIT_KV=256`、`splits_per_chunk=16` 切 KV 轴。关键结论是——因输出逐 `(i,j)` 独立，切 KV 轴等价于切输出列轴，每个 split 写到不重叠列段 `(base+split_idx)*SPLIT_KV + thread`，多个 SM 并行处理同一请求的不同 KV 段而**无需任何跨 SM 归约**，区别于传统需部分和归约的 split-K GEMM。本讲还引入 **varlen**（`indices` 等值连跑拍平变长序列，仅 SM100、`next_n==1`）与 warp 内一次预取最多 32 个页号、`__shfl_sync` 广播的页查表优化。
