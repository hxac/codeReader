本讲覆盖 DeepGEMM 启发式系统的「可调旋钮层」，核心载体是 `HeuristicsRuntime` 进程级单例。三个旋钮管理「形状特化即投资」这一权衡：特化维度越多、单 kernel 越快，但需编译的形状组合越多。

其一，**形状特化**受双重控制：局部参数 `compiled_dims`（字符串如 `"nk"`，非 `0` 维度烤成编译期常量、`0` 留运行时；默认因算子而异，稠密 nt/nn 与 m-grouped 取 `"nk"`、tn/tt 与 k_grouped 取 `"mn"`）与全局开关 `set_ignore_compile_dims`（优先级最高，一键清空所有特化），二者均经 `get_compiled_dim` 裁决，设备侧以 `SHAPE_* != 0 ? SHAPE_* : shape_*` 覆盖。

其二，**block 对齐** `set_block_size_multiple_of` 经 `std::lcm(base, x)` 放大候选步长，兼顾用户对齐与硬件基础步长（SM90 为 16、SM100 为 32），默认 `1` 零干扰。

其三，**contiguous 对齐** `mk_alignment_for_contiguous_layout`（默认保守值 128）决定 m-grouped 每段 M/K 对齐粒度并直接成为 `block_m` 候选；其理论下界由 `get_theoretical_mk_alignment_for_contiguous_layout` 推导——SM90 固定 128，SM100 在候选集 {64,…,224}（步长 32）中取不小于 `expected_m` 的最小值，且断言须为 32 倍数。三者统一遵循「先算理论值、再 set」的约定，为 u6（内核入口）与 u7（分组布局）承接 `SHAPE_*`、`mk_alignment` 等概念。
