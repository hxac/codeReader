本讲为入门单元收尾，把 DeepGEMM 当黑盒跑通第一次调用，覆盖三块。接口全貌：所有能力挂在 `deep_gemm` 顶层命名空间，按配置旋钮、cuBLASLt 基准、FP8/FP4/BF16 GEMM 及其分组版、Einsum、MQA、Mega MoE、testing·utils 分类，命名遵循 `<精度>_<算子>_<布局>` 约定。`_C` 初始化：`import deep_gemm` 的末步自动调用 `_C.init(library_root, cuda_home)`，把设备头文件目录与 CUDA 目录交给 C++ 侧的 JIT 编译器、运行时加载器与头文件解析器，import 成功即等价于初始化完成。最小 FP8 GEMM 调用：引入 NT 布局、分块量化与缩放因子（SF，按 `gran_k` 切块求 amax 除以 448 得到）、`per_token_cast_to_fp8` 等术语；关键约定是输入为 `(tensor, sf)` 元组、输出需预分配、SF 随架构变化（SM90 用 FP32、SM100 用打包 UE8M0，由 `disable_ue8m0_cast` 开关控制），SF 到 TMA 布局的变换由 API 内部完成；数值校验统一用 `calc_diff`（1 减余弦相似度，FP8xFP8 阈值 0.001），参考输出以 FP32 计算以排除参考误差。
