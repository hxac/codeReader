本讲是单元 7「分组 GEMM（MoE）」的起点，讲解 M 轴分组的 **contiguous 连续布局**：MoE 中多个变长 expert 的 token 按 M 轴首尾拼接进单个 `[M,K]` 张量，权重保留为 `[G,N,K]`，从而把一组形状各异的 GEMM 压成**一次 kernel launch**，前提是所有 expert 的 N、K 固定不变。对应 API 为 `m_grouped_fp8_fp4_gemm_nt_contiguous`，沿用 NT 原生 kernel、SF 变换、架构派发（SM90 走 1d2d，SM100 走 1d1d）四步范式。

核心机制是 **`grouped_layout`**：非 psum 模式用 `[M]` 数组逐行标记 expert 号（padding 行标 `-1`），psum 模式用 `[num_groups]` 前缀和存累计结束偏移。设备侧 `Scheduler` 在 `MGroupedContiguous` 分支靠采样 tile 首行 `grouped_layout[m_block*BLOCK_M]` 反查 expert，`is_computation_valid` 跳过 padding，`is_tma_multicast_valid` 保证相邻 CTA 同属一个 expert。

关键前提是 **mk 对齐要求**（`mk_alignment_for_contiguous_layout`，SM90 固定 128、SM100 可低至 32），它保证 BLOCK_M 大小的 tile 不跨 expert 边界。**psum 布局**用紧凑数组消除 padding 间隙，并通过 `psum_sfa_layout` 让缩放因子打包跳过 gap 行以省显存，代价是设备侧按 expert 扫段并需 `expected_m_for_psum_layout` 编译期提示。本讲为后续 u7-l2（masked 布局）与 u7-l3（K 轴分组 psum）奠基。
