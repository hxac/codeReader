本讲专讲 Mega MoE 调用 mega-kernel 前的**权重静态变换**，由 `transform_weights_for_mega_moe` 统一完成，含两个最小模块：一是 **gate/up 交错**（`_interleave_weights`，粒度 8），把 SwiGLU 的 `[gate|up]` 拼接重排成 `[gate₀..₇, up₀..₇, …]` 小块交替，使相邻两块恰为同一对 (gate, up)，这是把 SwiGLU 就地融合进 L1 GEMM epilogue 的物理前提（数值不变、仅重排 N 维，并强制 contiguous 以满足 TMA）。二是 **UTCCP SF 转置**（`_transpose_sf_for_utccp`），在每 128 元素段内做 4×32 转置，置换式为 `(idx%32)*4 + idx//32`，与设备侧 `transform_sf_token_idx` 逐位一致，使 UTCCP 硬件指令能把权重 SF 正确搬进 TMEM 供 scaled UMMA 读取。两条路径差异为：BF16 仅对 L1 交错、L2 不变；FP8×FP4 对 L1 权重与 SF 都处理（SF 先交错再转置，不可交换），对 L2 仅转置 SF、权重不变——根因是 SwiGLU 只发生在 L1 之后，而 SF 转置只取决于是否用 UTCCP。关键术语：gate/up 交错、UTCCP、TMEM、打包 UE8M0、MN-major。
