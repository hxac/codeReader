本讲首次下钻到设备 kernel 内部，聚焦 SM90 FP8 稠密 GEMM 内核 `sm90_fp8_gemm_1d1d_impl`。内核含 17 个由宿主 JIT 填充的模板参数，`__launch_bounds__(线程总和, 1)` 表明因寄存器占用极高每个 SM 仅驻留 1 个 CTA。所谓 1D1D 指 A/B 的 TMA 加载盒不切分，恰好搬满一个 block。核心采用「1 个 TMA warp-group + 若干 math warp-group」分工：TMA 线程发异步拷贝喂数据，math 线程发 WGMMA 驱动 tensor core 算乘加，靠 `warp_idx` 与 `kNumMathThreads/32` 比较分裂线程。共享内存用 `extern __shared__` 加纯偏移量划分为 D / A / B / SFA / SFB（各乘 kNumStages）与 2×kNumStages 个 mbarrier，经 `PatternVisitor` 封装成「第 i 级地址」的下标访问，构成环形多缓冲队列。关键结论 `BLOCK_K==128` 由「缩放因子 SF 每 128 个 K 通道一个」粒度（`sf_k_idx == k_block_idx`）与 WGMMA 的 K=32 共同钉死。生产者-消费者以 `full_barriers`/`empty_barriers` 双缓冲握手，`get_pipeline` 用 `iter_idx % kNumStages` 与 phase 翻转驱动软件流水线。math 侧把硬件原始 MMA 结果 `accum` 与缩放后累加 `final_accum` 分离，正确实现逐块缩放 GEMM。引入的关键术语供后续承接：TMA/math warp-group、WGMMA、双缓冲 mbarrier、软件流水线、`kNumStages`、`PatternVisitor`、compiled_dims 形状特化、逐块缩放 SF。
