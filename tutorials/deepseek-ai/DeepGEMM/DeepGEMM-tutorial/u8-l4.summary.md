本讲收尾 Mega MoE 单元，剖析单内核 `sm100_fp8_fp4_mega_moe` 如何把 dispatch、Linear1/SwiGLU、Linear2、combine 几个阶段塞进同一个 CTA，实现 NVLink 通信与 tensor core 计算的真正重叠。核心结论有三：①内核按 warp group 划分 dispatch、GEMM、epilogue 三条支路，靠共享内存 mbarrier 与 Workspace 的 full/empty 计数表做生产者-消费者握手，环形缓冲使 num_ring_tokens 这样的小工作区即可承载任意总 token，这是重叠的物理来源；②`layout::Workspace` 是对称内存首部的「控制平面」，集中存放 grid sync 与 NVLink barrier 计数器、expert 收发数、四张环形 full/empty 计数表与 combine 回写元数据，数据缓冲链顺序接在其后，设备与宿主两份布局必须镜像一致；③`grid_sync` 以「最后到达者翻转 tag 位」的全局原子技巧实现可复用的跨 SM 汇合，`nvlink_barrier` = grid_sync 加仅 SM 0 发的跨 rank 信号，因时间重叠，dispatch 与 epilogue 须用不同 grid sync index。本讲引入的关键术语：warp group 流水线、full/empty 计数、控制平面与数据平面、`grid_sync`/`nvlink_barrier`、tag 位翻转。
