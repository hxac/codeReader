本讲下钻 DeepGEMM JIT 缓存的正确性与并发安全，覆盖三件事。其一，缓存键由 `name`/`signature`/`flags`/`code` 四元组经 `get_hex_digest`（双状态 FNV-1a + splitmix64，128 位指纹）压成摘要当目录名，叠加内存层 `KernelRuntimeCache` 与磁盘层 `$HOME/.deep_gemm` 两层查找。其二，`IncludeParser` 递归哈希所有 `<deep_gemm/*>` 头文件得 `include_hash`，以注释渗入 `code` 使头文件改动触发重编；`nullopt` 兼任循环检测与记忆化，`exclude_code` 避免 code 重复计入。其三，编译先在 `tmp/<uuid>` 私有目录完成，`fsync_dir` 自底向上刷盘后原子 `rename` 上线，输家用 `safe_remove_all` 清理并复用赢家产物，由此支撑 `check_validity`「目录存在则 cu 与 cubin 必同在」的不变量。
