本讲（u3-l2）讲解 DualPipe 引擎在 `step()` 执行前的状态初始化与两个计算原语，紧承 u3-l1 的拓扑确定。

`_reset_states` 是八步调度的状态地基：先清空 `WeightGradStore`，再建立 `input_chunks / output_chunks / output_grad_chunks / input_grad_chunks` 四类缓冲，它们按 `[phase][chunk_id]` 二维组织——外层 phase 选方向，内层 chunk_id 选微批次；并初始化 6 组进度计数器（`current_f/b_chunk_id`、`current_send/recv_f/b_chunk_id`），分别供计算、发送、接收各用一套游标，采用「先读后自增」方式取用。

四类缓冲对应一个 chunk 张量的四个生命阶段：前向原料、前向产物、下游回传梯度、算出的输入梯度。

`_forward_compute_chunk` 取出下一 chunk 输入、跑 `self.module[phase]`，终点 stage 算 loss 存入 `loss_chunks`，否则把输出存入 `output_chunks`。

`_backward_compute_chunk` 的核心分叉是：终点 stage 用 `loss.backward()`（标量 loss 作种子），中间 stage 用 `run_backward(outputs, output_grads)`（下游回传梯度作种子）。反向原语严格遵循「取出 → 用完 → 置 None」生命周期，将 `output_chunks`、`output_grad_chunks`、`input_chunks` 对应槽清空，并把输入梯度写入 `input_grad_chunks` 等待发给上游——这是 DualPipe 控制激活显存的关键。`enable_zb` 仅负责权重梯度的立即算或延后入队，开启时反向后调 `WeightGradStore.flush()`。

通信收发与张量排入缓冲由 u3-l4 的通信原语承担，八步调度则在 u3-l5 展开。
