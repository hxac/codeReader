本讲让 DualPipe 从概念走向「可运行」，覆盖安装、启动、初始化、训练与校验四个最小模块，并讲清 DualPipe 与 DualPipeV 的差异。依赖方面：DualPipe 需 PyTorch 2.0+，但 `setup.py` 未把 PyTorch 写入 `install_requires`，需自行安装；版本号由 git 短哈希或时间戳生成。启动方面：示例用 `torch.multiprocessing.spawn` 拉起多进程，本讲中「进程数 = 流水线阶段数 `pp_size` = GPU 数」三者恒等；DualPipe 入口对 GPU 数「向下取偶、步长 −2」（至少 2 张卡），DualPipeV「步长 −1」任意数量。初始化链路：`dist.init_process_group(backend='nccl', init_method='env://')` 建进程组，再绑 GPU、统一随机种子 233、设 `CUBLAS_WORKSPACE_CONFIG` 保证数值确定性。关键术语 rank、world_size、进程组、NCCL、P2P 通信由此引入。校验是示例精髓：先用单进程 `ref_step` 生成参考结果，再跑分布式 `step`，用 loss/输出 `torch.equal`、梯度 `cal_diff<1e-13` 三件套逐项核对。核心差异：DualPipe 从首末两端各喂一半数据，每设备持两个 stage；DualPipeV 仅从首端喂入、末端做 V 型转折，stage 数为两倍。本讲还初次接触包导出符号 `DualPipe`、`DualPipeV`、`set_p2p_tensor_shapes`、`set_p2p_tensor_dtype`、`WeightGradStore`，为后续讲义承接。
