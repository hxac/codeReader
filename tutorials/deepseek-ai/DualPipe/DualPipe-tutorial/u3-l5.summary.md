本讲是 DualPipe 引擎剖析单元的收尾篇，把前几讲拆解的 rank 拓扑、状态缓冲、计算/通信原语与组合操作装配成最终调度引擎 `DualPipe.step()`，其骨架为「入口校验 → scatter 切半 → 八步调度 → loss/outputs 聚合 → 复位返回」。

八步循环次数由三个核心量驱动：`num_half_ranks`、`half_rank`（关于流水线中点对称的位置量）、`half_num_chunks`，满足 `step_1 + step_2 + step_4 = half_num_chunks` 的负载均衡恒等式。八步依次为灌水（nF0）、双向启动（nF0F1）、三重叠过渡（nB1W1F1，首次启用零气泡）、稳态全重叠主步（nF0B1F1B0，面积最大，middle rank 在 i==0 特判不重叠以进一步缩气泡）、排水（nB1B0/nWB0）与收尾排空（nW）。

零气泡（zero bubble）靠 `WeightGradStore` 把权重梯度 `put` 延后、用 `_weight_chunk` 的 `pop` 填进气泡，step 8 全部排空并以 `assert funcs_queue.empty()` 收尾。最终 loss/outputs 只在 first/last rank 聚合返回，索引 `output_chunks[is_first_rank]` 精准取到该 rank 作为终点 stage 所在 phase 的输出。
