本讲聚焦 DualPipe 与 DualPipeV 的选型对比，含两个最小模块：精读 README 对比表、设备数差异的源码根因。核心结论：相同 PP 阶段数下，两者气泡公式 \((PP/2{-}1)(F\&B{+}B{-}3W)\)、每设备参数（\(2\times\)）、每设备激活（\(PP{+}1\)）三列完全相同，唯一区别是所需设备数——DualPipe 需 \(PP\) 个，DualPipeV 仅 \(PP/2\) 个。根因在拓扑：DualPipe 是两条独立流的交叉（首末两端喂数据，每设备被各用一次）；DualPipeV 是一条流的折叠（仅首端喂，末 rank 经 detach 折返成 V 型顶点，折返点两 stage 共驻一台设备零通信），省下一半设备。引入术语：对比表四列（气泡/参数/激活/设备数）、V 型转折与 detach 衔接、半量（half_rank）vs 全量（rank）调度量、偶数设备约束（DualPipe 强制偶数，DualPipeV 任意）。选型建议：设备效率优先选 DualPipeV，确需双向独立喂数据才选 DualPipe。
