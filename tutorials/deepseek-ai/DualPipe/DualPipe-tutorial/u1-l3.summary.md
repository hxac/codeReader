本讲从文件层面梳理 DualPipe 仓库的结构、对外 API 与内部组织。仓库分为「包代码 `dualpipe/`」与「外围文件」（`setup.py`、`README.md`、`examples/`、`images/`），只有 `dualpipe/` 因 `setup.py` 的 `packages=["dualpipe"]` 被安装，示例与配图只属仓库、不进 site-packages。`__init__.py` 作为门面（facade），用重新导出（re-export）和 `__all__` 集中暴露五个公共 API：`DualPipe`（来自 `dualpipe.py`）、`DualPipeV`（来自 `dualpipev.py`）、`WeightGradStore`（来自 `utils.py`）、`set_p2p_tensor_shapes` 与 `set_p2p_tensor_dtype`（均来自 `comm.py`），可用 `__module__` 自省验证。包内依赖呈三层单向结构：叶子层（`comm.py`、`utils.py`，仅依赖 PyTorch）→ 引擎层（`dualpipe.py`、`dualpipev.py`，共用底层且互不依赖）→ 门面层（`__init__.py`），无循环依赖。此依赖图同时确定了后续源码的自底向上阅读顺序：先 `comm.py`、`utils.py`，再 `dualpipe.py`，最后 `dualpipev.py`。`utils.py` 中多个工具函数仅有 `WeightGradStore` 被提升为公共导出，体现「公共 API 精简」原则。
