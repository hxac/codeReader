`dualpipe/comm.py` 是 DualPipe 的叶子通信层，只依赖 PyTorch，专责相邻 GPU 之间的点对点（P2P）张量收发。本讲覆盖三个最小模块：①全局约定——模块级变量 `TENSOR_SHAPES`/`TENSOR_DTYPE` 记录统一的张量形状与 dtype，由公共 API `set_p2p_tensor_shapes`/`set_p2p_tensor_dtype` 在启动时写入，引擎 `step()` 入口会断言二者已设置；②接收缓冲区工厂 `build_from_tensor_shapes`，用 `torch.empty` 按全局形状在 CUDA 上分配未清零缓冲区，且 `requires_grad=True` 以参与反向求导；③累积函数 `append_irecv`/`append_isend`，它们不立即收发，而是把 `dist.P2POp` 追加进传入的 `ops` 列表，并用 `get_global_rank` 把组内 rank 翻译成全局 rank 以兼容子组场景；接收现造缓冲区并返回，发送用调用方传入的张量。核心结论：通信层只提供「攒请求」工具，「提交」由引擎层用 `dist.batch_isend_irecv` 一次性完成——这种「累积→批量提交」的两段式设计，使多条收发能彼此重叠、与计算重叠，是 DualPipe 压缩流水线气泡的底层基石。关键术语 P2POp、batch_isend_irecv、组内 rank/全局 rank、comm_ops 将在 u3-l4 通信原语与 u3-l5 八步调度讲义中承接。
