本讲针对 u5-l2 揭示的 FP8 sparse 解码 dequantization-bound（反量化约 50 cycle/token 慢于 MMA 的 34 cycle），讲解破局技术 **crossover**。其前提是 MQA 下 128 个 query 头共享同一份 KV，故 2 个 CTA 可各反量化半个 KV block（`CLUSTER_SIZE = NUM_HEADS/64`，仅 128 头时为 2，由 `if constexpr` 编译期钉死）再互换，使单 CTA 反量化降至约 25 cycle，瓶颈翻转为 MMA-bound。硬件基础是 Hopper 的 **CTA cluster** 与 **Distributed Shared Memory（DSM）**：对端 smem 地址由 `PEER_ADDR_MASK = 1<<24` 异或得到，用 `st.async.shared::cluster.mbarrier::complete_tx::bytes` 异步写入，不经 L2。同步原语为 **cluster transaction barrier**（`bar_k_remote_ready`），即到达计数加字节计数：消费端 `arrive_and_expect_tx(N)` 预登记、生产端 `st.async` 自动扣字节，两者齐备才放行。producer 对同一份反量化结果做本地 store 与 DSM 双写，反量化只算一次。结论：crossover 不省 smem、不省寄存器，省的是 dequant cycle，代价是多占一个 SM 与集群同步开销。
