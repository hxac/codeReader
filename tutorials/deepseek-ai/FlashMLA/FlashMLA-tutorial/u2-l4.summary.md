本讲解析 sparse 路径（decode/prefill）的 `ImplBase` 派发框架与 Feature 声明机制，位于 `csrc/api/common.h` 后半部分，配合 `sparse_decode.h`、`sparse_fwd.h` 的实现类使用。核心结论：将每个 kernel 实现抽象为「支持的能力清单」，派发即做子集校验 `需求集合 R ⊆ 实现支持集合 S`。关键术语包括：`ImplBase<RunArgT, FeatureT>` 类模板基类（`run` 为 public 入口，`run_`、`get_supported_features` 为 protected 纯虚函数，靠访问控制强制「先校验再执行」）；`DECLARE_SUPPORTED_FEATURES` 宏（一行声明 `static constexpr` 数组并 override 能力清单）；`check_if_all_features_are_supported`（只判定，供 prefill 的「优先/兜底」选择）与 `..._and_abort`（失败时打印需求/支持/差集三段诊断并抛 `TORCH_CHECK` 异常）；以及基于 `__PRETTY_FUNCTION__` 的 enum 名字反射（`get_static_enum_name`/`get_enum_max`/`get_dynamic_enum_name`，将运行时整数变回可读字符串，强依赖枚举从 0 连续）。完整派发流程为「构造 required_features → 按架构/形状选实现 → `impl->run`」，框架作为选择逻辑之外的安全网。
