本讲义讲解 SM100（Blackwell）上 sparse prefill 的 `phase1` kernel，覆盖三条实现路径及其分工。普通路径按 query 头数分两条：head64 为单 CTA、全 Q 进 TMEM；head128 为 2-CTA cluster，因 TMEM 容量不足将 Q 沿特征维拆成 tQ(TMEM)/sQ(smem)，P 用 TS+SS 两种 MMA 累加。算法骨架沿用 online softmax 三段流水（Copy|MMA|Scale&Exp），但累加器 O、部分 Q、P 全部搬入 TMEM，以 UMMA/UTCCP 与 2x1SM cluster MMA 取代 Hopper 的 WGMMA。第三条是 small_topk 变体，用更细的 B_TOPK=64 与更深流水缓冲（prefill 4 / decode 3）专门服务小 topk（接口阈值 topk≤1280），仅支持 D_QK=512。其最巧妙之处是用模板参数 `SparseAttnFwdMode` 把 prefill 与 decode 编进同一份代码：prefill 用 CLC 分发、吃 bf16 KV、写最终 out；decode 读 DecodingSchedMeta、吃 FP8 KV、写 o_accum。关键 Blackwell 概念包括 TMEM、UMMA、UTCCP、TMEM Allocator、2-CTA cluster、2-SM multicast TMA、CLC；dual gemm、online softmax 的 rescale 公式、base-2 内部/base-e 输出约定仍延续自前讲。公共逻辑（索引装载与掩码、P 归约、O rescale、求 max/S）抽入 common_subroutine.h，供 head64 与 small_topk 复用。接口层以 FwdFeatures 子集校验加 topk 阈值在普通 head128 与 small_topk 间路由，decode 侧 Decode_Sm100_Head128_Impl 直接复用 small_topk kernel。
