本讲聚焦 FP8 稀疏解码的 C++ 接口函数 `sparse_attn_decode_interface`，它是 Python 端 `flash_mla_with_kvcache(..., indices=...)` 与底层 SM90/SM100 kernel 之间的「校验＋编排＋派发」中枢。核心模块四块：接口函数对张量形状、dtype、layout 及 FP8 字节布局（V3.2 每块 656 字节、MODEL1 584 字节）层层校验，并把输入形态翻译成一个 `DecodeFeatures` 需求集合；`DecodeImplBase` 派发框架落地为四个实现类，由显式路由表按 `arch × h_q × d_qk` 选中，再以 `ImplBase::run` 的 feature 子集校验作安全网。关键结论：SM100f 上无原生支持 `HEAD_DIM_575` 的 head128 kernel，故 V3.2 形状（`h_q=128, d_qk=576`）退回 `Decode_Sm100_Head64x2_Impl`，将 128 头切两段、平移 q/out/o_accum/lse/lse_accum/attn_sink 指针各跑一次 head64 kernel。本讲引入的关键术语供后续承接：`DecodeFeatures`、`DecodeImplBase`、`SparseAttnDecodeParams`、四个实现类、需求 feature 集合、head64x2 折中、`extra_kv`⇔`extra_indices` 配对、`topk_length` 的 `[b]` 掩码语义，以及每个可选能力编译期特化为无分支 kernel 以保护 dequantization-bound 热路径的设计。
