本讲讲解 FlashMLA 的测试脚手架，覆盖 `tests/` 下四个核心模块。`lib.py` 是数据工厂，用 `TestParam`/`Testcase`/`KVScope` 把形状与开关翻译成带边界条件的随机张量，`_randperm_batch` 故意掺入 `-1`/`s_kv`/`INT32_MAX` 等无效索引候选来逼出 kernel 边界 bug，decode 路径用 `KVScope.quant_and_dequant_` 量化再反量化以剥离 FP8 量化误差。`ref.py` 是 sparse 路径的数值黄金参考，`_merge_two_lse` 用 `torch.logsumexp` 合并 lse 与 attn_sink（prefill 改算 out 的 lse、decode 直接乘缩放因子，两者数学等价且对外 lse 不含 sink），lonely query 一律 `out=0, lse=+inf`；dense decode 的参考 `reference_torch` 写在测试文件内而非 `ref.py`。用例分 correctness/corner/performance 三类，前两类只验正确性、第三类测 TFlops/GB/s。判定核心是 `kk.check_is_allclose` 的三重容差：异常值（inf/nan）逐位掩码匹配 + abs/rel 双门 OR + cos_diff 全局兜底；容差与 bf16 精度挂钩（out 用 `2.01/128`、lse 用 `8.01/65536`），lse 因被指数放大而用紧容差。性能测量靠 `bench_kineto` 的 CUPTI 采样与 L2 刷缓存。
