本讲承接 u5-l1，量化 FP8 KV cache 喂给 Tensor Core 前的反量化开销。H800 无 fp8→bf16 直转指令，反量化被拆成 4 步转换链（fp8→half→fp32→bf16→×scale），其中 C++ 的 (float4) 强转在硬件上下放为两条 cvt。按「指令吞吐倒数」估算，单 token（512 个 fp8）反量化约 50 cycle，瓶颈在 fp32→bf16（占总开销 64%），而 64 头 MMA 仅需约 34 cycle。反量化跑在 CUDA Core、MMA 跑在 Tensor Core，两者同 SM 可并行但有数据依赖，max(50,34)=50，Tensor Core 每 token 空等约 16 cycle，故 kernel 为 dequantization-bound。引入的关键术语：四步转换链、dequantization-bound、HFMA2、tile 级 scale。破局思路「少反量化一半」即下一讲 u5-l3 的 crossover——利用 MQA 下两个 CTA 共享同一份 KV，各反量化一半再经 DSM 互换，把 50 cycle 砍到 25 < 34，从而把瓶颈转回 MMA-bound。
