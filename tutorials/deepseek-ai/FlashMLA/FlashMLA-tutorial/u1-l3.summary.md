本讲建立 FlashMLA 仓库的代码空间感。顶层分用户壳（`flash_mla/`、`setup.py`）与算力实现（`csrc/`），辅以 `tests/`、`benchmark/`、`docs/` 保障质量。核心是 `csrc/` 按「架构（sm90/sm100/smxx）× 阶段（decode/prefill）× 稀疏性（dense/sparse）」三维嵌套组织，读路径即可判断 kernel 归属。支持矩阵的不对称直接映射到目录：dense decode 仅 sm90、dense prefill 仅 sm100、sparse 两架构都有；SM100 head128 解码无独立实现，复用 small_topk prefill kernel。关键术语：`csrc/api/` 用 5 个 pybind 绑定（如 dense_decode_fwd）连接 Python 与全部 kernel；`instantiations/` 子目录承载模板实例化，仅 `.cu` 文件被 setup.py 编译，`.h`/`.cuh`/`config.h` 分工明确；`smxx/` 是两架构共用的解码辅助 kernel（tile scheduler 元数据生成与 Split-KV combine 归并）；`kerutils/` 为 header-only 工具库；`tests/ref.py` 是最权威的 kernel 行为参考实现。
