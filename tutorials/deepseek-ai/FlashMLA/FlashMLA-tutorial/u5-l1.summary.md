本讲定义 FlashMLA 在 FP8+sparse 解码下 KV cache 的显存字节布局与量化方法。MLA 把每个 576 维 token 拆为前 512 维 NoPE 与后 64 维 RoPE：仅 NoPE 段做 tile 级量化，每 128 维配一个 scale_inv，按 2 的幂向上取整（UE8M0）以保证 fp8_e4m3 不溢出 448 上限；RoPE 段对精度敏感，原样保留 bf16 不量化。由此引入 V32（656 字节、4 个 fp32 scale、段落序为 NoPE→scale→RoPE）与 MODEL1（584 字节、7 个 e8m0 scale、NoPE→RoPE→scale、含 block 对齐填充）两种布局，差异集中在 d_qk、tile 粒度、scale 编码与段落顺序。ModelType 由 d_qk 运行时推断、驱动 kernel 模板特化；sparse 路径强制 FP8 且要求整个 block 连续。tests/quant.py 的 quantize/dequantize_k_cache 为权威参考实现，本讲不展开 kernel 内反量化与 crossover。
