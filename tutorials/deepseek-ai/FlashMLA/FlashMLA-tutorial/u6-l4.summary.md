本讲讲 FlashMLA sparse prefill 的 C++ 接口层 `sparse_attn_prefill_interface`，它是 Python 入口与底层 kernel 间的「校验+装配+派发」中枢，分四个最小模块：(1) 接口校验：架构门禁限 SM90a/SM100f，再做张量五道校验（NDIM/device/dtype/shape/末维连续）与 `SparseAttnFwdParams` 装配（注意 `sm_scale` 预乘 `LOG_2_E` 换算 base-2）；(2) `required_features` 构造：将运行时张量属性翻译为需求能力集合，源自 `FwdFeatures` 枚举的 7 个能力位，其中 `SINK_LSE` 被各实现声明却从不被本接口需求；(3) 派发：经 `FwdImplBase` 做「需求集合⊆支持集合」的子集校验，落地为 SM90 全覆盖、SM100 head64、head128 普通、head128 small_topk 四个实现类；(4) SM100 head128 唯一的二选一：`use_small_topk = (topk≤1280 ∧ R⊆S_small) ∨ (R⊄S_regular)`，兜底项因普通实现是 small_topk 的严格超集而恒假，仅作 feature 漂移防御网。核心结论：small_topk 仅支持 `d_qk=512`、专为小 topk 优化，且经 `SparseAttnFwdMode` 模板开关与 SM100 head128 decode 同源复用。
