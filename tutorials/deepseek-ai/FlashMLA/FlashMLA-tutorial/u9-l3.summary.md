本讲是 FlashMLA 收官实战篇，将前述零件串成一条 sparse decode 端到端链路：数据构造 → FP8 量化与 indices 烘焙 → 调用 kernel → 正确性校验 → 性能测量，分别由 lib.py、quant.py、flash_mla_interface.py、ref.py 与 test 文件(+kernelkit) 承担。引入的关键术语：lib.py 作数据工厂，生成协同约束张量并掺入 -1/越界/NaN 边界逼出 bug；quant.py 用 V32 的 656 字节 FP8 布局，以「量化再反量化」把量化误差从正确性判定里剥离；indices_in_kvcache 把分页映射烘焙成物理索引，使 sparse 路径不再需要 block_table；FlashMLASchedMeta 首次填充、后续复用并断言形状一致；check_is_allclose 用三重容差判定正确性，bench_kineto 按 kernel 名拆分测速。核心结论：必须先拿答案再测速以防假阳性；FLOPs 按 attended token、访存按去重 retrieved token；手算算术强度约 351 > 平衡点 258，判定 compute-bound，与 README 报告的 410 TFlops 吻合。
