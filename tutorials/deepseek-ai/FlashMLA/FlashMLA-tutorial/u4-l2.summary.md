本讲专注 decode 三段式的第三段——combine kernel，它读取主 kernel 写入的两个 float32 累加缓冲（段内已归一化输出 `o_accum`、base-2 局部 lse `lse_accum`），把单个 batch 拥有的若干 split 归并成一份最终输出，并顺带完成 float32→bf16 的精度下转换。combine 是 SM90/SM100 两代架构共用的 kernel，由 `CombineParams` 承载输入输出契约，dense 与 sparse 解码路径共用。

核心要点有三。其一，grid 划分为 `(b*s_q, 1, ceil_div(h_q, BLOCK_SIZE_M))`，遵循「一个 warp 负责一个 head」的设计：每个 CTA 含 8 个 warp（`BLOCK_SIZE_M=8`），32 个 lane 平摊 head_dim_v=512 的向量。其二，跨 split 合并的数学基石是 online softmax 的 rescale 公式 L=log₂Σ 2^ℓₛ、O=Σ 2^(ℓₛ−L)·oₛ，与整体 softmax 等价，kernel 用 `__shfl_xor_sync` 蝶形归约实现数值稳定（先减 max）的 max/sum；kernel 内部统一用 base-2，对外返回的 lse 转为 base-e。其三，`my_num_splits==1` 时直接 return 是正确性必需而非优化，源于主 kernel 的 `is_no_split` 双写契约：未切分的请求由主 kernel 直写最终 out/lse，combine 无事可做。

此外引入 sparse 解码独有的 `attn_sink` 机制：它把内部 lse 抬高为含 sink 的 L′，使缩放因子变小、输出按比例缩小，但返回的 lse 在 sink 修改前已写入、不含 sink。PDL 配合 `cudaGridDependencySynchronize` 使 combine 与主 kernel 重叠执行。本讲为后续 u4-l3（tile scheduler 元数据分配）与 Unit 5（sparse 解码）的前置。
