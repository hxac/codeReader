本讲讲解 FlashMLA 底层 header-only 工具库 csrc/kerutils/（别名 ku=kerutils），夹在 PyTorch/CUDA 与各代 attention kernel 之间，分 host 与 device 两条线。host 线（host.h、torch_tensors.h）：KU_CHECK_* 校验宏经 _check_optional_tensor 用 if constexpr 同时处理 Tensor 与 optional<Tensor>，可选为空即判合法；get_optional_tensor_ptr 把可选张量转 nullptr-able 指针，是 params「nullptr=禁用」约定的来源；另有 KU_CUDA_CHECK/KU_ASSERT（抛 KUException）、make_tensor_map 构造 TMA 描述符（stride 以字节为单位）。device 线按 sm80→sm90→sm100 三层组织，靠 __CUDA_ARCH__ 与 KERUTILS_ENABLE_SM*（带 A 为精确架构、无 A 为及以上）编译期裁剪：sm80 基线 cache policy/get_sm_id，sm90 加 DSM（get_peer_addr 异或 1<<24）与 WGMMA，sm100 加 TMEM/UMMA/tma_gather4；device.cuh、kerutils.cuh 为聚合头。另有更全局的 defines.h（bf16/fp8/transac_bar_t、bf16x8 打包向量）与 utils.h（CHECK_CUDA/FLASH_ASSERT、FLASH_DEVICE_ASSERT 用 trap、ceil_div、RingBufferState 的 phase 翻转），与 kerutils 部分功能重叠、长期并存。
