本讲精读 `csrc/params.h`：它用一组 POD 纯数据结构作为接口层与 kernel 层之间的数据契约，把 kernel 启动所需的指针、stride（统一以元素为单位）、split-KV 缓冲、tile 调度元数据与 stream 打包成对象，从接口函数透传到 kernel。

六个核心结构中，`DecodingSchedMeta` 是 tile 调度产物（32 字节、8 个 int），被多结构以 `tile_scheduler_metadata_ptr` 共享，先写一次再读两次；`DenseAttnDecodeParams` 与 `SparseAttnDecodeParams` 解码双结构同构，关键差异为 stride 类型（int64 vs int）、KV dtype（同 Q vs FP8）、sparse 专属的 `indices`/`topk`/`extra_*`/`model_type`；`CombineParams` 归并各 split 的 `o_accum`/`lse_accum`，`GetDecodeSchedMetaParams` 产出调度元数据（后两者 dense/sparse 共用，靠 `-1`/`nullptr` 开关区分）；`SparseAttnFwdParams` 服务 prefill（无 batch、无 split-KV、多 `max_logits` 输出）。

引入的关键术语与概念：`head_dim_k=576`、`head_dim_v=512` 的 MLA 硬约束；`ModelType`（V32/MODEL1，由 `d_qk` 决定，编码 FP8 字节布局）；`SparseAttnFwdMode`（Prefill/DecodeWithSplitKV）配合 `is_decode_v` 与 `SparseFwdArgT` 类型别名，让双模 kernel 一份代码兼容两种入参结构；可空张量统一用 `nullptr` 表示禁用。本讲只读结构定义与装配，kernel 如何消费这些字段留待 u3-u6。
