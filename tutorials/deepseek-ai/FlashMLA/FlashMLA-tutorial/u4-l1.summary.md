本讲承接 u3-l4 的三段式编排，拆解 decode 路径中主 kernel 与 combine 之间的数据契约——split-KV 缓冲。decode 阶段 query 极少（s_q=1）而 KV 很长，单条序列无法喂饱 SM，故将长 KV 横向切给多个 SM partition（num_sm_parts）并行处理，即 split-KV（Flash-Decoding）。引入两块 float32 accumulate 缓冲 lse_accum、o_accum 暂存各 split 的局部结果，其大小由安全上界 total_num_splits = batch_size + num_sm_parts 决定；num_splits[batch_size+1] 为前缀和，给 combine 提供每个请求占用的 split 区间。主 kernel 双写：单 split 走 is_no_split 直写最终 out/lse（base-e），多 split 写 accumulate 缓冲（base-2、段内已归一化的 O）。核心数学是局部 lse 经 rescale 合并为全局 lse，与整体 softmax 代数等价；combine 对单 split 早退，两条路径对调用方一致。
