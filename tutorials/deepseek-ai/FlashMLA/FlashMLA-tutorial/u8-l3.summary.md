本讲回答「kernel 有多快、离理论极限多远」，含四个模块。一是多实现 benchmark：bench_flash_mla.py 用 FUNC_TABLE 统一 torch/flash_mla/flash_infer/triton 四种实现签名，先 assert_close 校验、再 triton.testing.do_bench 测时落 CSV，由 visualize.py 画图。二是理论指标：由 FLOPs 与访存字节公式除耗时得 TFLOPS、GB/s，MLA 因 K/V 同源不单算 V 项；sparse 解码改用去重 token 数与 FP8 字节数。三是 kernelkit 两套测时：bench_kineto 基于 CUPTI 按 kernel 名拆分并刷 L2、用 get_e2e_time 串联多 kernel；bench_by_cuda_events 用 cuda event 交错多候选做公平 A/B；二者均检测 nsys/ncu 避免冲突。四是 NVCC flag：--use_fast_math 提速标量运算、--register-usage-level 平衡占用率与 spill、--warn-on-spills 是稳定性护栏、-lineinfo 服务 ncu 定位。核心结论：seesaw/细粒度 TMA 流水/cache hint/PDL 把 dense decode 推至约 660 TFlops，crossover+DSM 解放 FP8 sparse 反量化瓶颈达 410+ TFlops，所有优化效果最终通过 TFLOPS/GB/s 与 get_e2e_time 量化验证，形成「改一处、测一处」的调优闭环。
