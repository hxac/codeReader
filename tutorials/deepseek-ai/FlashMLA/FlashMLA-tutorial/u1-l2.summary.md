本讲解决「如何把 FlashMLA 编译装起来」。核心结论：安装一次性产出两样东西——纯 Python 包 `flash_mla`（做形状校验与参数拼装）与 CUDA 扩展 `flash_mla.cuda`（经 PyTorch 的 `CUDAExtension` 把 `csrc/` 下的 `.cu/.cpp` 编译成 `.so`，由 pybind 桥接给 Python）。四块内容：构建链路、CUDAExtension 源文件清单与 cxx/nvcc 双套编译参数、NVCC 版本检测与 arch gencode（本讲核心）、feature flag 机制。关键概念：CUTLASS 作为 git 子模块嵌在 `csrc/cutlass`，clone 后是空目录、必须 `submodule update --init`，`setup.py` 顶部还会自动补跑一次作双保险。`-gencode` 决定为哪代 GPU 生成代码：`sm_90a`（Hopper，后缀 `a` 启用 WGMMA/TMA 专有指令）与 `sm_100f`（Blackwell）互相独立，可同时编译进同一产物。核心约束：`sm_100f` 目标只在 CUDA 12.9+ 的 NVCC 才被支持，故 NVCC ≤ 12.8 时 `get_arch_flags` 会断言、强制用户设 `FLASH_MLA_DISABLE_SM100` 才放行，把环境不匹配暴露成显式错误。三类 flag：`FLASH_MLA_DISABLE_FP16` 是 `-D` 宏（同时进 cxx/nvcc）；`SM90/SM100` 是 gencode 开关（只进 nvcc）；`NVCC_THREADS` 控编译并行度。
