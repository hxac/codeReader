本讲用纯理论回答：为什么 MLA 解码阶段是 compute-bound 而非 memory-bound。它通过三个最小模块展开：先推导 FLOPs 与访存量，再算算术强度，最后用 Roofline 判定 bound。核心公式为：FLOPs = 2·h_q·s_q·s_k·(d_k+d_v)，因 MLA 中 K、V 同源（V 仅取 K 的前 d_v 维），K 只需加载一次，故 bytes ≈ 2·s_k·d_k，访存被长 KV 主导。两者相除得算术强度 I ≈ h_q·s_q·(d_k+d_v)/d_k ≈ 2·h_q·s_q，它只依赖 h_q·s_q，与 batch 和序列长度 s_k 基本无关。引入的关键术语包括：compute-bound / memory-bound、算术强度（arithmetic intensity）、Roofline 模型、平衡点 I* = P/B。以 H800 SXM5 降频后峰值 P≈865 TFlops、带宽 B=3.35 TB/s 得平衡点 I*≈258 FLOP/byte，由此得出分界点 h_q·s_q ≈ 128：大于为 compute-bound，小于为 memory-bound。DeepSeek 解码不做 Tensor Parallel（h_q=128）且关闭 MTP（s_q=1），h_q·s_q=128 恰踩分界点，按 compute-bound 优化，这也是后续 seesaw 调度要重叠 CUDA Core 与 Tensor Core、喂饱 Tensor Core 的理论前提。
