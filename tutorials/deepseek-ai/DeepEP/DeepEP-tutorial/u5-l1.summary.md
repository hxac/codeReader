本讲拆解 DeepEP V2 单节点直接模式（`num_scaleout_ranks == 1`，纯 NVLink 节点内通信）下的 dispatch 内核 `dispatch_impl`，把一个 CTA 的 warp 分成两组协作的 warp。**notify warps**（默认 4 个，须为 4 的倍数）负责计数与跨 rank 归约：先在共享内存里逐 token atomicAdd 统计每个 rank/专家收到几个，再做全 grid 归约（SM 序号压高位、计数压低位）与跨 rank 交换，用 `encode_decode_positive` 把计数编码成正数以复用 0 作「未到」哨兵；末尾经前缀和产出 `psum_num_recv_tokens_per_scaleup_rank`（warp 0，inclusive）与 `psum_num_recv_tokens_per_expert`（warp 1，exclusive），并写 `num_unaligned_recv_tokens_per_expert`。**dispatch warps** 负责搬数据：每 warp 在共享内存有独立 TMA 暂存区，按步长跨 SM 与 warp 认领 token，TMA load 隐状态、`__ldg` 读标量路由信息，经 `atomicAdd` 领对端槽位并回写 `dst_buffer_slot_idx`，再用 `gin.get_sym_ptr` 翻译对称指针、`tma_store_1d` 直达对端（不可达则经 send_buffer 中转 + RDMA）。核心结论是 `num_dispatch_warps = min((num_smem_bytes − notify_smem)/per_warp_tma_bytes, 32 − num_notify_warps)`，受共享内存容量与每 CTA 最多 32 warp 双重约束，大 hidden 时共享内存成瓶颈。该模式为后续 u5-l2 hybrid 两级通信奠定模板基础。
