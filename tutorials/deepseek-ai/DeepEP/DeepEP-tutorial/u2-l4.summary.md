本讲聚焦 DeepEP 在 Python 接口层与 C++ host 层之间的通信-计算重叠机制。核心是双流模型：每个 ElasticBuffer 绑定一条进程级单例的高优先级通信流 comm_stream，用户算子仍在 compute stream（即当前 PyTorch 流）上，两者靠 CUDA event 建立最小依赖，从而把总耗时从「通信+计算」降到接近 max(通信,计算)。

关键概念包括：dispatch/combine 的三段式 host 流控制——prologue（切换分配流、决定 comm 等谁）→ 主 kernel → epilogue（决定回吐 event 还是阻塞等待）；以及四个流控开关，其中 async_with_compute_stream 是重叠总开关，previous_event（由 buffer.capture() 在计算流 record 得到）表示「计算先跑、通信等它」，并强制要求 allocate_on_comm_stream（对应 host 断言）。C++ 的 EventHandle 是裸 torch::Event 封装，负责 record/wait 与 stream_wait 跨流同步；Python 的 EventOverlap 在其上提供 current_stream_wait、register_hook_after_wait（支撑确定性排序）与 with 语法。record_stream 负责多流显存安全。标准重叠模式为：dispatch(async=True) → 插入与结果无关的独立计算 → event.current_stream_wait() → 才使用 recv_x。本讲不涉及 GPU kernel 内部，留待 U5/U6。
