本讲讲解 DeepEP V2 中独立于 dispatch/combine 之外的流水线并行（Pipeline Parallelism, PP）通信原语 `pp_send`/`pp_recv`，用于在把 transformer 层切分到不同 rank（流水线级）时，沿环形拓扑在相邻级间传递激活与梯度。核心约束：PP 只与环形邻居 prev/next（`(r±1) % N`）通信，该约束在 host 层由 `EP_HOST_ASSERT` 硬性强制，N=2 时两邻居重合属合法退化。`pp_set_config` 先 `barrier` 刷新旧操作，再记录张量上限（32 字节对齐）与最大在途数。缓冲区遵循 **4 倍因子**：2 方向（prev/next）× 2 角色（send/recv），每角色下再用取模轮转 `slot_idx = count % num_max_inflight_tensors` 形成流水线深度。send/recv 为 cooperative GPU 内核，三段式执行：`tma_copy` 本地双缓冲搬运 → `grid.sync` → SM 0 发起 `gin.put`（跨 rank RDMA 写对称地址）与 `gin.signal`；send/recv 通过「数据就绪」与「槽位释放」两对 NCCL Gin 信号完成无锁握手。死锁保护依赖 GPU 侧 `timeout_while` 计时 + `ptx::trap()`，超时阈值由「秒 × 时钟频率」烘焙为编译期 `kNumTimeoutCycles`，超时打印方向性文案后崩溃。关键术语：环形拓扑、4 倍因子、在途数、取模轮转、对称内存、cooperative 内核、grid 同步、信号握手。
