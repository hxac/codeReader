本讲讲解 DeepEP 纯 GPU 侧跨 rank barrier 同步机制。因 kernel 链靠 PDL 在 GPU 侧串接、全程不回 CPU，无法依赖 `ncclGroupEnd` 或 `cudaStreamSynchronize`，改由 cooperative kernel `barrier_impl` 用共享信号内存完成 release/acquire 同步——返回后本 rank 的所有写对所有 rank 可见。节点内 NVLink scaleup 用对称内存信号（`get_sym_ptr` 直接寻址）配 sense-reversing 算法：counter 低 2 位编码 phase/sign，双 signal 槽交替，`red.release.sys`/`ld.acquire.sys` 配对保证内存序，信号区占 workspace 最前 16 字节且须初值清零；跨节点 RDMA scaleout 用 NCCL Gin 的 `SignalInc` 计数信号加 shadow counter 复用槽位，发信号前必须 flush 全部 QP，否则对端会读到未落地数据。多节点 hybrid 分 sequential（串行 scaleout→scaleup，1 SM，scaleup 阶段顺带 flush 残留 RDMA）与 parallel（2 SM 并发）两模式，仅多节点 hybrid 下才有性能差异。host 层 `ElasticBuffer::barrier(use_comm_stream, with_cpu_sync, sequential)` 是统一围栏入口，被 `destroy`、`engram_write`、`pp_set_config` 用作数据流切换与资源释放的一致性保证；超时经 `timeout_while` + `trap()` 把死锁暴露成可定位崩溃。
