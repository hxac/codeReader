本讲建立 DeepEP JIT 子系统整体地图并打通一次内核编译的端到端主流程。子系统含六大组件：Compiler/NVCCCompiler 负责编译、KernelRuntime 持有已加载内核、KernelRuntimeCache 提供进程内缓存、LaunchRuntime 以 CRTP 标准化「生成—启动」两段式、DeviceRuntime 缓存 GPU 属性、IncludeParser 算头文件哈希，多为 LazyInit 惰性单例。init_jit 不编译，仅把 Python 探测的库根、CUDA home、NCCL root 钉入静态成员，编译推迟到首次 build。运行时 JIT 化解「想编译期化十多个模板参数换最优代码、但参数运行时才确定」的矛盾，靠按需生成与两级缓存复用避免安装期组合爆炸；代码生成用「取模板特化地址强转 void*」强迫 nvcc 实例化特定内核。build 主流程为：内容寻址定位缓存目录、进程内缓存命中即返回、否则编译到唯一临时目录、fsync_dir、整个目录原子 rename 到最终路径、再用 cuobjdump 从 cubin 提取唯一符号加载；目录 rename、fsync、safe_remove_all 共同保证多 rank 并发编译下的分布式文件系统安全。
