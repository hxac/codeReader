本讲讲解 DeepEP V2 中独立于 dispatch/combine 的实验性原语 **AGRS（All-Gather Reduce-Scatter）**，但当前仓库只实现了 all-gather 路径。AGRS 面向**节点内 NVLink** 对称内存（强制 `num_nvl_ranks == num_ranks` 且 `num_ranks > 1`），且**没有任何 GPU kernel**——发送用 `cudaMemcpyBatchAsync`、同步用 `cuStreamBatchMemOp`，全程由 host 在 comm stream 上拼出，这是它与 JIT kernel 路线的根本差异。核心是 **session + slot** 批量模型：一个 session 内线性复用缓冲区（`agrs_buffer_offset` 累加）与信号槽（`agrs_buffer_slot_idx` 每次 all_gather 加 1，上限 `kNumMaxInflightAGRS=32`）；生命周期由 `agrs_set_config` / `create_agrs_session` / `destroy_agrs_session` 三段构成，单调递增的 `agrs_session_idx` 作信号值配 GEQ 等待，实现 release/acquire barrier，安全支持跨 session 复用。`agrs_get_inplace_tensor` 用 `from_blob` 建零拷贝视图（不改成员 offset），配合指针落点检测跳过自拷贝；batched 模式一次调用多张量仅占一个 slot。`all_gather` 的正确性靠流序保证：发送 → per-slot all-to-all 信号握手（`batched_write_and_wait`）→ `EventHandle` 让计算流等；返回的 `out` 是急切视图，必须调 wait hook 后才能读。关键术语含对称指针 `get_sym_ptr`、`batched_write_and_wait`、wait hook。
