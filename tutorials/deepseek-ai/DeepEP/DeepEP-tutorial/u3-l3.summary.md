本讲义解析 DeepEP V2 如何用「带宽建模 + 解析式公式」决定 dispatch/combine 通信内核所需的 SM 与 QP 数量，取代 V1 的 auto-tuning，做到无需预热运行。核心模块有三：SM 建模、expected top-k 计算、QP 分配。SM 建模基于「带宽均衡」——以 epilogue 读总量 V 为归一化单位，累加 `sm_read/sm_write/rdma_traffic/nvlink_traffic` 四个流量分数，选出流量/带宽比最大的瓶颈链路，令「HBM 搬运时间 = 链路传输时间」，解出 `num_sms = (bounded_gbs/bounded_traffic)·max(sm_read/sm_read_gbs, sm_write/sm_write_gbs)`，V 与真实 token 数被约掉，再经 1.25 倍系数、对齐偶数等调整。expected top-k 用组合数公式 `G·(1 - C(E-E/G,K)/C(E,K))` 算「均衡门控下 token 的期望跨 rank 数」，仅对 balanced gate 成立，group-limited gate 不适用。QP 分配在 direct 模式用 `min(num_sms,8)+1` 省门铃开销、hybrid 模式用 `num_sms*16+1` 给每 channel 独立 QP，最终被构造期 `num_allocated_qps`（direct 17 / hybrid 65 或 129，受 fast RDMA atomic 影响）封顶。`prefer_overlap_with_compute` 是省 SM 与满性能的总开关。
