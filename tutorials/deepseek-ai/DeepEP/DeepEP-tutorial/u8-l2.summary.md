本讲覆盖 DeepEP V2 拿到 ProcessGroup 后的两件底层事。其一，NCCL communicator 获取走三级回退——缓存命中、复用 PyTorch 的 `_comm_ptr()`、用 unique id 自建，由 `get_nccl_comm_handle` 调度；轻量句柄 `NCCLCommHandle` 的 `managed` 字段决定销毁归属（复用归 PyTorch，自建由 DeepEP 用 `ncclCommAbort` 销毁），进程级缓存为模块字典 `_storage`。`force_new_comm`（带 CPU buffer、即 `num_cpu_bytes>0` 时为真）与开关 `EP_REUSE_NCCL_COMM` 共控来源：前者强制自建并以纳秒时间戳作 key，目的是隔离含 CPU 段的弹性窗口与重型 Gin 改造、独立掌握生命周期；后者置 0 可全局禁用复用，是排查风险的逃生开关。其二，拓扑数值不自检而问 NCCL：轻量入口读 `ncclTeamWorld/ncclTeamLsa` 得物理域 `num_rdma/num_nvl_ranks`，重量入口 `NCCLSymmetricMemoryContext` 读 Gin `dev_comm.lsaSize`，再按 `allow_hybrid_mode` 投影成逻辑域 `num_scaleout/num_scaleup_ranks`，并由 `is_scaleup_nvlink = (num_scaleup_ranks == num_nvl_ranks)` 决定对称内存走 hybrid 还是普通弹性分配。
