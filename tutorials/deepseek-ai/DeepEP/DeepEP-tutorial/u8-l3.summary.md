本讲梳理 DeepEP 环境变量体系，按 General/Networking/JIT/Build 四大类给出权威清单与代表变量，Python 侧统一用 `os.environ.get`、C++ 侧用模板函数 `get_env<T>` 读取同一批变量（注意 Python 字符串 `'0'` 为真的陷阱）。核心是 DeepEP 独有的「持久化变量」两级默认值机制：`EP_JIT_CACHE_DIR`、`EP_JIT_PRINT_COMPILER_COMMAND`、`EP_NUM_TOPK_IDX_BITS`、`EP_NCCL_ROOT_DIR` 在 `pip install` 时由 `setup.py` 动态烘焙进 `envs.py`，`import deep_ep` 时仅在用户未设时注入 `os.environ`，C++ 侧靠进程环境变量天然共享，优先级为运行期 export > 构建期烘焙 > 代码硬默认。JIT 调试类变量（`EP_JIT_DEBUG` 看命令行与 PTXAS 日志、`EP_JIT_DUMP_SASS/DUMP_PTX` 落地汇编产物、`EP_JIT_PTXAS_CHECK` 断言 local memory 溢出）用于排查运行时编译问题。最后讲解 RDMA 的 Service Level(SL) 与 Virtual Lane(VL)：通过 `EP_OVERRIDE_RDMA_SL` 把 SL 透传给 NCCL Gin 的 `ginTrafficClass`，经交换机 SL→VL 映射实现 EP 流量与其他流量的物理通道级隔离。
