本讲覆盖 DeepEP combine 的 reduce epilogue 与 multiple reduction 两个最小模块。combine 被拆成「省 SM 的主 kernel」与「满 SM 的 reduce epilogue」两个 GPU kernel，host 依次下发、靠 PDL（cudaGridDependencySynchronize）在 GPU 侧串接，无需 CPU 同步；主 kernel 返回 reduce_buffer 指针供 epilogue 读取。epilogue 的核心 combine_reduce 对每个 token 从接收缓冲区读出多份副本并加权累加：用 match/deduplicate 按源 rank 去重选出 master lane，compute_topk_slots 压成紧凑槽位数组，FP32 累加后用 TMA 一次写回 combined_x；无 bias 且有效 topk≤2 时走 BF16 快路径 enable_hadd_bypass，其余走 FP32 慢路径。allow_multiple_reduction 开关权衡「单次精确归约（传输多）」与「多次归约（源端/节点内先预归约、传输少、多一次舍入）」，经 use_rank_layout、主 kernel 的 no_local_reduce 分支、epilogue 去重键三分支联动生效，hybrid 多节点下形成「先 scaleup 内归约、再 scaleout 全局」的两级归约。主 kernel 与 epilogue 共享同一 combine_reduce，分做源端预归约与最终归约。0/1/2 个 bias 与多份副本并入同一 FP32 累加器；combined_topk_weights 经 match 找 master lane 从 buffer 读出对应源 rank 的权重。
