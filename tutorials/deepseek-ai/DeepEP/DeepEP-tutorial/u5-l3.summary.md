本讲讲解 dispatch 链路为何拆成「主 kernel（省 SM 做跨 rank 通信）+ copy epilogue（满 SM 做纯本地拷贝/重排）」两个内核，并靠 PDL 机制（`cudaGridDependencySynchronize`）在 GPU 侧串接、无需 CPU 介入。epilogue 用两套 `BufferLayout`（smem 暂存区 + 全局 buffer）配合 TMA load/store，把打包好的 `[hidden|SF|metadata|mbarrier]` token 拆包写到 `recv_x` 等用户张量。重点区分两种接收布局：**非 expand（compact）** 按到达顺序排列、保留 `recv_topk_idx`；**expand** 按专家分组连续排列、`recv_topk_idx` 返回 `None`，直接切片即可喂 per-expert GEMM。关键变量 `dst_tensor_idx` 在非 expand 取 `i`，expand 用 `atomicAdd(psum[expert],1)` 抢槽。最微妙处是同一张量 `psum_num_recv_tokens_per_expert` 在两模式下语义不同：非 expand 切 inclusive 段返回对齐边界，expand 切 exclusive 段当原子计数器并被 in-place 改写。最后引入 `expert_alignment` 对齐与 `do_zero_padding` 零填充，二者均依赖 `do_expand`，由 epilogue 第二阶段用波浪扫描清零对齐缝隙。
