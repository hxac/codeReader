本讲是 U4 JIT 单元的收尾，讲清 DeepEP 如何用统一启动框架把 CUDA 内核启动配置标准化。核心模块有四：一是 `LaunchRuntime<Derived>`（CRTP 基类），其 `launch()` 静态方法从 `args.launch_args` 取描述符、构造 `CUlaunchConfig`、回调派生类 `launch_impl`，实现零虚函数开销的编译期多态；派生类只需写 `generate_impl`/`launch_impl`，新增内核零样板。二是 `LaunchArgs`——六字段纯描述符（`grid_dim`/`num_threads`/`smem_size`/`cluster_dim`/`cooperative`/`pdl_enabled`），由 `construct_launch_config` 翻译为 Driver API 属性数组（默认走 Driver API 分支，支持 cooperative/cluster/PDL），`launch_kernel` 再调 `cuLaunchKernelEx` 下发。三是 `DeviceRuntime`，借 `LazyInit` 惰性探测并缓存 `cudaDeviceProp`，提供 `get_num_sms`/`get_num_smem_bytes`/`get_clock_rate`/`get_arch`，其中时钟频率（kHz→Hz）把超时秒数换算成 GPU 周期数烘焙为编译期常量。四是 `IncludeParser`，递归解析 `<deep_ep/*>` 头文件并求哈希，注入生成代码首行，使内容寻址缓存对头文件改动敏感。范例：dispatch 主内核以 `cluster_dim=2-(num_sms%2)` 对齐下游计算内核、`cooperative=true` 借「一 SM 一 block」实现 grid 级同步；epilogue 则开 PDL。这些术语（cluster、cooperative launch、PDL、超时周期、include 哈希）为 U5 内核链路深入做铺垫。
