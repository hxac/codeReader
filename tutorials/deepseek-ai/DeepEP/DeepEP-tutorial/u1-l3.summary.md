本讲梳理 DeepEP 的运行依赖与安装期构建流程。依赖需 Hopper（SM90）GPU、CUDA 12.3+、PyTorch 2.10+、NCCL 2.30.4+，节点内 NVLink、节点间 RDMA，及仅遗留方法用的 NVSHMEM。构建主入口 setup.py（无 CMake）借 CUDAExtension 与 BuildExtension，把 csrc/* 源文件、include 目录、链接库与编译标志拼成安装期编译，产出 deep_ep._C.so；而 deep_ep/include/impls/*.cuh 仅作 package_data 随包发布，留待运行时 JIT。两条核心结论：持久化环境变量（EP_JIT_CACHE_DIR 等）构建期烘焙进动态生成的 envs.py，import 时作默认值注入 os.environ 但不覆盖用户已设值，即"构建期烘焙、运行期可覆盖"的两级默认值；get_nccl_lib_name 优先取未版本化符号链接、否则回退带版本 SONAME，以 -l: 精确链接化解 pip wheel 只发带版本 SO 的痛点。find_pkgs.py 靠环境变量优先、再扫 Python 包元数据定位库根目录，setup.py 用 importlib.util 加载它以绕过尚未就绪的 deep_ep.__init__。
