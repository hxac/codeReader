DeepEP（DeepEveryParallel）是面向 MoE 训练与推理的高性能通信库，核心职责是专家并行（EP）所需的 dispatch/combine all-to-all 内核，追求零或极少的 SM 占用。dispatch 把 token 从原始 rank 发到目标专家所在 rank，combine 把专家输出送回原 rank 并按权重归约，二者通过携带路由元数据的 EPHandle 串联、互为反向。物理链路分两类：NVLink 用于节点内（intranode）通信，RDMA 用于节点间（internode）通信，`EP N x M` 描述每节点 N rank、共 M 节点的拓扑。V2 相对 V1 有四大变化：以 header-only 的 NCCL Gin 后端取代 NVSHMEM、全 JIT 编译、解析式 SM/QP 计算取代 auto-tuning、统一 ElasticBuffer 接口（最大 EP2048）；峰值性能最高 1.3 倍、SM 占用最多降到 1/4。需注意性能表中的瓶颈带宽是"逻辑带宽"，包含本 rank 内部流量，不等于 RDMA 网线纯物理带宽。本讲结论均来自 README，尚未深入 CUDA 源码。
