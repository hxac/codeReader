本讲把 Dispatch 从单节点纯 NVLink 扩展到多节点混合网络，引入 **channel**（通道）抽象：把一次 all-to-all 拆成两级——scaleout（RDMA 跨节点）与 scaleup（NVLink 节点内转发），以匹配集群的两级带宽拓扑，并对应 `num_ranks = num_scaleout_ranks × num_scaleup_ranks` 的域划分。`hybrid_dispatch_impl` 内核把一个 CTA 的 warp 分为三类：**notify**（先节点内再跨节点两级归约 token 计数）、**scaleout**（`gin.put` 经 IBGDA 发 RDMA，靠 signaled tail 周期推送尾指针与 forward 流水线同步，纯节点内目标走 local bypass 零 RDMA）、**forward**（轮询 tail，用 NVLink 把收到的 token 转发到目标 GPU）。「a warp is a channel」——channel 数由共享内存、combine 布局、`prefer_overlap_with_compute` 共同决定，既作流水线深度又作 QP 负载均衡粒度。本讲还交付三个 cached handle 元数据张量——`dst_buffer_slot_idx`、`token_metadata_at_forward`、`channel_linked_list`，形状围绕 `num_channels` 与 `num_max_tokens_per_channel` 展开；其中 `channel_linked_list` 以尾指针＋0 号 head 哨兵把零散到达的 token 串成链，供 combine 反向「重放 dispatch」枚举。combine 方向、PTX/TMA 原语、cached handle 全貌留待后续讲义展开。
