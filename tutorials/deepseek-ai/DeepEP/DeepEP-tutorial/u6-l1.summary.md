本讲阐述 combine 是 dispatch 的逆过程：不重新算路由，而是重放 EPHandle 中 dispatch 写好的元数据，把专家输出推回原 rank 并按 topk_weights 加权归约。反向路由主表 recv_src_metadata 每行编码 token 的源 rank、源位置与 top-k 槽位，接收规模由 psum_num_recv_tokens_per_scaleup_rank 给出，无 CPU sync 时内核自行从最坏上界兜底。直接模式内核 combine_impl 逐 token 解码源地址，NVLink 可达则 TMA store 经对称指针直达对端，否则经 send_buffer 加 gin.put 走 RDMA，首尾用 gpu_barrier 保证可见性。输入强制 BF16 与 16 字节对齐；launch_combine 按共享内存最大化定 warp 数，hybrid 模式要求 SM 数与 dispatch 一致，返回的 reduce_buffer 供 reduce epilogue 归约；do_expand 决定 topk_weights 为 1D 或 2D。hybrid combine 复用 dispatch 产出的 token_metadata_at_forward 与 channel_linked_list，scaleup 与 forward 两类 warp 分别做节点内、跨节点反向路由，靠 signaled tail 同步。归约细节与确定性排序分别留待 u6-l2、u6-l3。
