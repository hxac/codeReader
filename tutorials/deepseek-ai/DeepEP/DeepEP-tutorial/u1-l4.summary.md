本讲是入门单元收尾，聚焦两个最小模块：`tests/elastic/test_ep.py` 与 `deep_ep/utils/envs.py`。核心结论：在单机 8 卡上跑通一次 dispatch+combine，并读懂性能输出。

引入的关键术语：rank/world_size（全局进程号与进程总数）、local_rank（节点内 GPU 序号）、NCCL backend、MASTER_ADDR 握手；SO=scaleout（RDMA 跨节点流量）、SU=scaleup（NVLink 节点内流量）、逻辑带宽（本 rank 视角下经过内核的总字节，非物理带宽）。

核心工具函数：`init_dist`（从环境变量 `MASTER_ADDR/PORT/WORLD_SIZE/RANK` 推算 world_size、rank 并建立 NCCL 集群，返回 ProcessGroup）、`init_seed`（种子=全局种子+rank，保证各 rank 数据不同且可复现）、`dist_print`（`once_in_node`+`barrier` 避免多进程输出交错）。

主线为 `__main__`→`torch.multiprocessing.spawn(test_loop)`→`test_dispatch_combine`，由 `ElasticBuffer` 提供 dispatch/combine；dispatch 返回的 `handle`（EPHandle）承载路由元数据，combine 完全依赖它反向路由。性能数字由 `bench_kineto`（用 `_sleep`+`barrier` 抹平 launch 不均）测出。

关键判断：单机时 `num_scaleout_ranks==1`，SO 带宽恒为 0，只有多节点才能看到非零 RDMA；单机应对照 README 的 NVLink 行而非 RDMA 行。README 提到的 `tests/utils/envs.py` 是笔误，真实位置在 `deep_ep/utils/envs.py`。
