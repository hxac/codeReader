本讲沿 `StorageOperator::batchRead` 主线拆解 3FS storage 读数据路径，覆盖三个最小模块。**读路由**：CRAQ 的「写全读任何」使读可打链上任意 target，`batchRead` 在批次开头取一次 `TargetMap` 快照，对每个 `ReadIO` 用 `getByChainId(chainId, chainVer)` 解析本地 target 并校验 `upToDate`（public=SERVING 且 local=UPTODATE），全程单机闭环、不跨网络转发。**RDMA buffer**：`BufferPool` 启动期预注册小池（4MB×1024）与大池（64MB×64），运行期 `tryAllocate` 快路径 + `allocate` 慢路径借出，并用信号量限流；直接 I/O（O_DIRECT）要求 4096 对齐（`kAIOAlignSize`），靠向两侧扩展 head/tail 再读后裁剪实现。**异步批量读**：`BatchReadJob` 聚合若干 `AioReadJob`，`AioReadWorker` 多线程（默认 32）跑 collect→submit→reap 事件循环，整批按 `batch_read_job_split_size` 切子批次，libaio 与 io_uring 双实现（后者用 fixed file/buffer 预注册提速），完成计数到齐后 `Baton` 唤醒。核心结论：零拷贝——数据从 SSD 进本地已注册 RDMA buffer，再以 RDMA Write 单边推回客户端预留 buffer，RPC 回包仅含 `IOResult` 元数据；一致性靠 `aioPrepareRead`/`aioFinishRead` 前后两次校验 chunk 版本（`commitVer==updateVer`）防止读到读一半被改的脏数据。关键术语：`ReadIO`/`BatchReadReq`/`IOResult`、`VersionedChainId`、`TargetMap`、`BufferPool`、`AioReadWorker`、`AioReadJob`。
