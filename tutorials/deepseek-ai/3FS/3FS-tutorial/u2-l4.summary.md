本讲承接 u2-l2（RPC 与 serde），讲解 `src/common/net/` 下 TCP/RDMA 传输基础设施，覆盖五个最小模块：①服务端分层——`net::Server` 以最多 4 个 `ServiceGroup`（每组装载 `Listener` 建链 + `IOWorker` 读写 + `Processor` 解包派发 + `serde::Services` 业务），按负载特征拆线程池，默认走 RDMA、可降级 TCP；②事件循环——`EventLoop` 用 `epoll`（边缘触发）+ `eventfd` 汇聚多连接事件，延迟删除保证安全；③统一传输层——`Transport` 持有抽象基类 `Socket`（`TcpSocket`/`IBSocket` 子类），用一组原子标志位驱动读写状态机，`doRead` 凑齐消息交 `Processor`、`doWrite` 批量写，使上层对底层传输无感；④`IBSocket` 零拷贝收发——`fd()` 返回 CQ 的 completion channel fd 以复用 epoll，poll 采用「取事件→武装→再 poll」三段式防丢 WC，小消息走 `SEND/RECV`（带 `ImmData` ACK 信用流控），大块数据走 `RDMA Read/Write`（链尾 WR 带 signal，一批一个 WC）；⑤`RDMABuf`/`RDMABufPool` 将「钉住 + `ibv_reg_mr` 注册」的内存池化，`toRemoteBuf` 把本地缓冲变成可发给对端的 `RDMARemoteBuf`（`addr` + `rkey`，按设备索引），`createFromUserBuffer` 支持用户内存零拷贝注册。核心结论：RDMA 建链本身不走 RDMA，需先经 TCP 旁路交换 `IBConnectReq/Rsp`（QP 号、LID/GID、MTU）再 `ibv_modify_qp` 推到 READY。
