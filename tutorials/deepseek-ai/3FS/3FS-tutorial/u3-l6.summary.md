本讲拆解 mgmtd 如何把集群视图与运行时配置分发到每个使用者，分四个最小模块。**路由信息分发**采用「304 式」按需拉取：客户端带 `routingInfoVersion` 询问，mgmtd 仅在版本不同时回传完整 `flat::RoutingInfo`，否则回包留空；客户端用 `atomic_shared_ptr` 无锁整体替换视图并通知 `StorageClient` 等监听者。**节点 tag 与 enable/disable**：`Disable` tag 与 `DISABLED` 状态一体两面，`TrafficZone` 用于客户端读流量隔离，二者随 `NodeInfo` 进入路由视图，经版本推进被全员感知，影响 `selectServingTargets` 的 target 选择。**配置托管**按 `NodeType` 分桶、版本只追加，靠「反转版本 + 大端序」的 FDB key 编码让最新版排在前缀扫描首位，并搭心跳回包便车下发，应用成功才推进本地 `configVersion`。**版本号推进**区分两种：`routingInfoVersion` 整视图共用，用「脏标记 + 周期 5s bump」合并成簇变更、避免写放大，且严格「先落 FDB 再改内存」以保崩溃切换后版本单调不退；`ConfigVersion` 按 NodeType 独立、`SetConfig` 时立即推进。引入的关键术语有 `routingInfoVersion`、`ConfigVersion`、`routingInfoChanged`、`BumpRoutingInfoVersion`、`flat::RoutingInfo` 及 `Disable`/`TrafficZone` 两个特殊 tag。
