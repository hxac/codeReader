本讲梳理 mgmtd 中集群拓扑的核心数据结构：ChainTable（链表，持有一串 ChainId 与链清单顺序）、Chain（一条 CRAQ 复制链，含链内 ChainTargetInfo 列表与偏好读顺序）、Target（ChainTargetInfo 仅含 targetId+publicState 的精简视图，完整 TargetInfo 含 public/local 双状态、nodeId、diskIndex、usedSize）。三者通过 flat::RoutingInfo 的四张映射表（NodeMap/ChainTableMap/ChainMap/TargetMap）以 id 作「逻辑外键」组织，可整体序列化下发。关键设计：ChainTable 只存 ChainId，链与链表多对多，同链可被多表复用。版本语义采用三级单调递增版本号——routingInfoVersion 管整视图、chainTableVersion 管链清单/顺序、chainVersion 管链内成员；storage 在数据面逐请求校验 chainVersion、拒绝过期写。ChainRef=(chainTableId, chainTableVersion, chainIndex) 借助按版本有序的 std::map 钉住文件创建时的历史链表版本，getChainId 用 1-based 取模实现 round-robin 选链。多张 ChainTable 按 id 共存以服务不同放置需求，互斥隔离由建表脚本构造时保证，结构层不强制。本讲只讲数据结构与版本语义，状态机转换留给 u3-l5。
