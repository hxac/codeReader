chunk engine 是 3FS storage 服务新一代落盘后端的 Rust 实现，是「一个 target 在单机上的物理化身」。它由三层构成：**meta** 层用 RocksDB 持久化 chunk 元数据（位置、长度、校验和、版本号）；**alloc** 层用位图管理 SSD 物理块分配；**file** 层把块组织成 cluster 等数据文件。`Engine` 结构体的所有字段都是 `Arc`，可被多线程廉价克隆共享。

它通过 **cxx** 库做 FFI 桥接：`#[cxx::bridge]` 声明 C++/Rust 共享类型（如请求输出合一的 `UpdateReq`、元数据视图 `RawMeta`）和 `extern "Rust"` 方法，`build.rs` 生成 C++ 头文件，`Cargo.toml` 声明 staticlib 供 C++ 链接。适配层（带 `_raw` 后缀的 `update_raw_chunk` 等）把 Rust 的 `Result` 翻译成 C 风格的「裸指针 + `out_error_code` + error 字符串」，错误码与 C++ `StorageCode` 数字对齐。跨语言所有权靠 `Box::into_raw`/`Arc::into_raw` 交出、靠对应 `release`/`commit` 函数归还，C++ 侧以 `rust::Box<Engine>` 为真正拥有者。

线程安全靠四件套封装在 Rust 内部：**Arc 引用守护 chunk 生命周期**（读拿引用、写产新块再原子替换，故读写无冲突）、**LockMap 256 分片锁**（per-chunk 串行）、**writing_list**（防同一 chunk 重复写）、**RocksDB WriteBatch**（新元数据与旧块释放原子落盘）。`get/update_chunk/commit_chunk` 三接口分别编排读、写、提交，并强制 CRAQ 双版本不变量（commitVer≤updateVer≤commitVer+1）。C++ 侧 `StorageTarget` 经 `useChunkEngine()` 开关在 Rust 引擎与旧 C++ `ChunkStore` 间二选一，只把引擎当线程安全黑盒使用。
