本讲覆盖数据放置算法与链表生成的三个模块:BIBD建模、链表生成、链导入集群。核心结论:链表设计的首要目标是故障时恢复流量均衡,而非「能存」——朴素链表会把 CRAQ 重定向读流量砸在少数节点拖垮吞吐,故 3FS 将节点作「点」、链作「区组」建模为平衡不完全区组设计(BIBD),要求每对节点均匀共链。data_placement.py 用整数规划(Pyomo+HiGHS)求关联矩阵:决策变量为「节点是否在链」,约束含每链恰 k 节点、每节点 r 个 target、节点对恢复流量落在窗口 [λ-lb, λ+ub],二次项以 qlinearize 线性化,不可行或超时则自动放宽 lb/ub 重试。gen_chain_table.py 把矩阵映射到真实节点/盘并按位段编码 target/chain ID,区分 CR(每链 k 个 target)与 EC(每链 1 个 target),产出链定义、链表、建 target 三件套。导入集群严格三步:create-target → upload-chains(setChains) → upload-chain-table(setChainTable),链表 id 须与 init-cluster 写入根 Layout 的 chainTableId 对齐。引入的关键术语:BIBD 参数 (v,b,r,k,λ)、关联矩阵、qlinearize、lb/ub 松弛、CR/EC 链结构、chainTableVersion(每次 setChainTable 单调递增)。
