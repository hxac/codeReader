本讲承接 u7-l3，把零拷贝 USRBIO 的 C 接口搬进 Python，供训练 Dataloader 使用。3FS 的 Python 交付物分三层：编译型扩展 hf3fs_py_usrbio、纯 Python 易用包 hf3fs_fuse、走内核 ioctl 的管理 CLI hf3fs_utils。核心是 pybind11 绑定层 usrbio_binding.cc（唯一参与编译，binding.cc 已失效），它把 hf3fs_iovwrap / iorcreate4 / prep_io / submit_ios / wait_for_ios 翻译成 Python 的 iovec / ioring，每个 C 调用做两件固定的事：把负 errno 翻成 OSError、在 prepare/submit/wait 等阻塞点用 gil_scoped_release 释放 GIL。iovec 借 buffer protocol 与指针平移切片实现零拷贝；ioring 的 prepare/submit/wait 对应 SQ/CQ 环，wait 用 CQE 的 index 找回 iov 并回填 result/userdata。setup.py 借自定义 CMakeBuild 在编译期调 CMake 编出扩展并链接 hf3fs_api_shared，setup_hf3fs_utils.py 则是纯 Python。三条用法铁律：fd 用前 register_fd、关前 deregister_fd；同一 ioring 不可多线程并发 prepare（hf3fs_prep_io 非线程安全，需每线程一个 ioring）；USRBIO 写后长度最终一致，须 force_fsync 才见准确 size。
