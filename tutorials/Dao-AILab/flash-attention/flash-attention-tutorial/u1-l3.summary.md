本讲走通「装好 → 跑通 → 不踩输入约束坑」三步。FA4 发行包名是 `flash-attn-4`，属纯 Python 包，`pip install` 不调用 nvcc、不编译 kernel，故安装极快；kernel 由 CuTeDSL 在运行时 JIT 编译，因此第一次调用 `flash_attn_func` 慢、之后命中缓存即秒返回。安装有三式：发行版 `pip install flash-attn-4`、CUDA 13（B200）用 `cu13` 扩展、改源码用 `pip install -e "flash_attn/cute[dev]"`。对外入口仅 `flash_attn_func` 与 `flash_attn_varlen_func`，均从 `flash_attn.cute` 导入，内部经 `FlashAttnFunc.apply`（torch.autograd.Function）分发。最小调用只需 `q/k/v` 加 `causal=True`，但务必注意 `flash_attn_func` 始终返回元组 `(out, lse)`——`lse` 即 log-sum-exp，形状 `(batch, num_heads, seqlen_q)`、float32，供反向与后续 SplitKV 合并使用。输入硬性约束三条：布局固定 `(batch, seqlen, nheads, head_dim)`、最后一维连续（不满足由 `maybe_contiguous` 静默修复）、head_dim 满足 16 字节对齐（fp16/bf16 即 8 的倍数，由 `_validate_head_dims` 校验，否则 AssertionError）；`softmax_scale` 缺省为 `1/√head_dim`。FA4 为精确注意力，与参考实现的误差仅来自 fp16 舍入。
