本讲覆盖 FA4 的 Top-k KV Gather 稀疏注意力，由 `CpasyncGatherKVManager` 实现，仅接在 MLA 吸收前向 kernel 上。外部打分器为每个 query 选出最相关的 k 个 KV token（存入 `gather_kv_indices`），kernel 只对它们做精确注意力，算力与带宽按 k/seqlen_k 比例下降。gather 走 cp.async 三步法：`load_index_topk` 取行号、`compute_X_ptr` 算 gmem 指针、`load_X` 把散落行 gather 成连续 smem tile；因行号任意散布只能用 cp.async 而非 TMA。非法行（越界/占位/因果边界）由 `compute_bitmask` 生成 32 列 uint32 位图，消费侧 `apply_mask_sm100` 把对应分数置 -inf，故结果精确而非近似。关键约束：仅 qv 路径、`gather_kv_length%128==0`、与分页 KV 互斥、`is_topk_gather` 进 compile_key 触发重编译。与块稀疏对比：Top-k 是 token 级、内容相关、动态选取；块稀疏是块级、结构相关、静态选取。主干 FLOPs 与 KV 带宽随 k 线性下降，top-25% 时理论降至约 25%，实际加速受 gather 开销与打分器质量影响。
