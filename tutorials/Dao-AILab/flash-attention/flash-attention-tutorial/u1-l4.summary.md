本讲是入门层收尾，阐明 FA2 与 FA4 为何能共用 `flash_attn` 包名却互不冲突。核心机制是**命名空间包**：FA2 的 `flash_attn/__init__.py` 用 `__path__ = extend_path(__path__, __name__)` 把 `flash_attn` 升级为命名空间包，扫描 `sys.path` 将分散在多个目录的代码拼成逻辑包；FA4 的 wheel 只提供 `flash_attn/cute/` 子目录、把 `__init__.py` 的「定义权」让给 FA2，二者文件层不重叠故不互相覆盖。由此 `from flash_attn import flash_attn_func` 得到 **FA2**（C++/CUDA、安装期 `nvcc` 编译的 `flash_attn_2_cuda` 扩展、默认只返回 `out`），而 `from flash_attn.cute import flash_attn_func` 得到 **FA4**（Python+CuTeDSL、首次调用 JIT 编译、恒返回 `(out, lse)` 元组）。关键术语包括：`extend_path`、命名空间包、`flash_attn_2_cuda`、`_get_device_arch`（FA4 在 Python 层按 SM80/90/100/110/120 显式分发到不同 kernel 类，可被 `FLASH_ATTENTION_ARCH` 环境变量覆盖）。选型结论：Blackwell 必选 FA4；Hopper 看功能（需要 `score_mod`/`mask_mod`/SplitKV/paged KV/MLA 选 FA4，否则 FA2 更成熟）；Ampere 两者皆可。
