本讲聚焦 FA4 kernel 的编译期特化机制。FA4 用两条途径把值烘焙成编译期常量：显式 `cutlass.Constexpr` 参数注解，以及经 `self.*` 捕获进 `@cute.kernel`/`@cute.jit` 的 Python `int`/`bool`（由 `StaticTypes` 元组界定）。`const_expr(x)` 在函数体里强制编译期求值，`if const_expr(...)` 于编译期删除无用分支，生成更精简的特化产物。`score_mod`/`mask_mod` 是用户 `@cute.jit` 回调，以 `Constexpr` 形式在编译期内联进 kernel（经 `call_score_mod`/`call_mask_mod`），运行时不存在函数调用，故不能用普通 Python 函数。宿主侧用 `compile_key` 元组统一管理特化：`causal`、`head_dim`、`tile_m/n`、`pack_gqa`、`arch` 等以裸值进键，`score_mod`/`mask_mod` 以源码哈希（含闭包捕获值）进键，键不等即重编译。判定口诀：凡改变生成代码（控制流、tile/类型/线程配置、被内联回调的源码或闭包值）的参数变化触发特化与重编译；纯运行期数值（如 `softmax_scale`、窗口大小数值）与输入形状则不重编译。
