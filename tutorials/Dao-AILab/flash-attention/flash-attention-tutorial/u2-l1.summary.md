本讲解析 FA4 仅有的两个对外入口 `flash_attn_func`（定长）与 `flash_attn_varlen_func`（变长/分页超集），二者数学等价，都是对 `FlashAttnFunc` / `FlashAttnVarlenFunc` 的薄包装，经「用户函数→autograd Function→`_flash_attn_fwd`」三层链挂入自动微分。引入的关键概念：LSE（log-sum-exp，online softmax 的天然产物）、`compile_key`（决定是否重编译的缓存键）、掩码四元组归一化（`_resolve_causal_local_window`）、tile 自动选择（`FwdConfig`/`_tile_size_fwd_sm90`）。核心结论：公共 API 不暴露 tile 与线程数；`softcap` 折叠成 `score_mod`、与用户 `score_mod` 互斥；返回值恒为 `(out, lse)`，`out` 用 `head_dim_v`，`lse` 为 float32；纯推理加 `return_lse=False` 时 `lse` 为 `None`，且「是否算 LSE」进 `compile_key` 会编译不同 kernel；`causal`、dtype、hdim、tile 等改值会重编译，而 `softmax_scale`、窗口具体大小不进键则不重编译。`exp(lse)` 可精确还原 softmax 归一化分母。
