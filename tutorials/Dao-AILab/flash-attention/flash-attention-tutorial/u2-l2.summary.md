本讲聚焦 FA4 在 `interface.py` Python 层的架构分发与 tile 配置决策，不进 kernel 内部。三模块：(1) `_get_device_arch()` 探测整数 arch（80/90/100/120），按 `arch//10` 分发到五个前向 kernel 类；`FLASH_ATTENTION_ARCH` 可覆盖硬件，但因 `@lru_cache` 须在首次调用前设；它与 `CUTE_DSL_ARCH`（PTX 编译目标）可解耦。(2) 头维校验 `_validate_head_dims` 是编译前早失败护栏，要求整除 alignment（fp16=8）：SM90 允许 8~256，SM100/110 限 8~128 加 DeepSeek/MLA/hd256 特例，SM80/120 跳过。(3) tile 配置 `FwdConfig`（m/n_block_size、mma_pv_is_rs、intra_wg_overlap），`_tile_size_fwd_sm90` 按 head_dim 分档、按 causal/local 缩 tile_n；num_threads SM80/120=128、SM90/100=384。核心结论：arch、tile、线程数及两布尔标志均入 compile_key 触发重编译，但只改实现不改数学结果。
