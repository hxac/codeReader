本讲义聚焦反向计算中独立于主循环的两个轻量 kernel：`FlashAttentionBackwardPreprocess`（预处理）与 `FlashAttentionBackwardPostprocess`（后处理），构成「预处理 → 主循环 → 后处理」三阶段。预处理按 m_block 并行，产出三件「料」：行修正项 `D = (O⊙dO).rowsum`（可选减 dLSE 得 D′）、供硬件 exp2 复用的 base-2 缓冲 `lse_log2 = LSE·log₂e`、以及清零的 fp32 全局累加器 `dQaccum`——清零是主循环跨 thread block `atomic_add` 正确性的前提。后处理则把 fp32 累加器收敛为最终 fp16 梯度，并将延迟的 `softmax_scale` 与类型转换融合于一步（`rdQ.store((acc.load()*scale).to(dtype))`）。引入的关键术语：dQaccum、dpsum/lse_log2、延迟缩放、D′ 行修正、PDL（`griddepcontrol_wait/launch_dependents`，与前向尾部重叠）。结论要点：dV 后处理 scale=1.0（P 已含 scale），dK/dQ 需补 softmax_scale；MHA 下 dK/dV 由主循环直接写出，GQA 才走后处理；fp32 与 fp16 共用一块 smem 节省显存。
