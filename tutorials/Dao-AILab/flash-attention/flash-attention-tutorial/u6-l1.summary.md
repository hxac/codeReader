本讲以 FA4 的 `FlashAttentionForwardSm80`（Ampere/SM120 基线前向 kernel）为对象，把前几讲的积木拼成完整数据流。核心策略是「Q 常驻、K/V 流水」：Q tile 在 prologue 一次性经 cp.async 从 gmem 搬入单槽 smem，内层循环反复 ldmatrix 读入寄存器；K/V 各占 `num_stages` 份环形 smem，靠 cp.async 计数器（`commit_group`/`wait_group`）让搬运与计算重叠。主循环从 `n_block_max-1` 倒序遍历，拆成首迭代/掩码/无掩码三段以编译期特化去谓词；每个 n block 内做两段 GEMM：`gemm(Q,K)→acc_S`，经 `online_softmax` 更新 `row_max/row_sum` 并产出 `row_scale`，`rescale_O` 修正旧 O，再 `gemm_rs(P,V)→acc_O`。引入的关键术语包括 gmem/smem/rmem 三级存储、MMA（16×8×16，fp16 输入 fp32 累加）、acc_S/acc_O、LSE、pack_gqa（把 `qhead_per_kvhead` 折进 seqlen 维，使一块 KV 被多 Q 头复用，Q 经 `PackGQA.load_Q` 按行 gather 进 sQ）。收尾由 `finalize`（`1/row_sum` 且把 row_sum 改写成 LSE）与 `rescale_O` 归一化 O。关键架构结论：`use_tma_O` 边界从 `arch≥sm_90` 修正为 `sm_90≤arch<sm_120`，使 Ampere(80) 与 SM120(120) 都走非 TMA（cp.async）输出回写，仅 Hopper(90)/数据中心 Blackwell(100,110) 启用 TMA bulk store。本 kernel 也是 SM120 前向的基类。
