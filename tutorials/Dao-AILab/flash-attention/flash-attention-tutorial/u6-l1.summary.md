本讲以 FA4 的 `FlashAttentionForwardSm80`（Ampere 基线：无 TMA、无 warp-group MMA）为对象，拼装出完整的前向 kernel。核心是「Q 常驻、K/V 流水」：Q tile 在 prologue 经 cp.async 一次性搬到 smem 单槽 sQ 反复读；K/V 各有 num_stages 份在环形 smem 缓冲里轮转，使搬运与计算重叠。主循环倒序遍历 n_block（从 n_block_max-1 到 n_block_min），拆成首迭代（seqlen 掩码）/ 因果滑窗掩码 / 无掩码快路径三段，靠 const_expr 编译期特化减少谓词开销。单个 n block（compute_one_n_block）是两段 GEMM 夹一次 softmax：gemm(Q,K)→acc_S，online_softmax 更新 row_max/row_sum 并产出 row_scale，rescale_O 修正旧 O，rP=acc_S.to(dtype)，gemm_rs(P,V)→acc_O。Ampere 用 cp.async 计数器流水（commit_group/wait_group，K、V 各一组故乘 2）配合手写下标 smem_pipe_read/write，区别于 Hopper 的 mbarrier/PipelineStateSimple。finalize 算 1/row_sum 并把 row_sum 改写成 LSE，rescale_O 归一化 O；epilogue 经 rmem→smem→gmem（universal copy，非 TMA）写回 O，列 0 线程写 LSE。引入术语：gmem/smem/rmem 三级存储、MmaF16BF16Op、ldmatrix、TiledMma、acc_S/acc_O、online softmax 与 LSE、NamedBarrierFwd.Epilogue，为 u6-l2 Hopper TMA 对比铺垫基线。
