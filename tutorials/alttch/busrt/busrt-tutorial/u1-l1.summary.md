BUS/RT 是一个用 Rust + Tokio 编写的进程间通信（IPC）消息代理（broker），灵感来自 NATS、ZeroMQ、Nanomsg，是 EVA ICS v4 工业物联网平台的核心总线。它既可嵌入 Rust 程序（通过线程内异步通道通信），也可编译为独立服务 `busrtd` 常驻运行，并为高负载与超低延迟实时场景同时优化（`rt` feature 用 `parking_lot_rt` 替换含自旋锁的互斥锁以实现实时安全）。本讲义引入的核心术语：三种通信模式——点对点（one-to-one）、广播（one-to-many）、发布订阅（pub/sub，主题可用 `#`/`+` 通配符）；四种通道——线程内异步通道（仅 Rust 内嵌）、Unix socket、TCP、WebSocket（仅 broker 与 async client）；以及区分「通道（物理传输）」与「模式（逻辑分发语义）」这两个正交概念。项目高度模块化，理解它的钥匙是 `Cargo.toml` 的 `[features]` 表：`rpc`（RPC 调用层，依赖 serde、tokio-task-pool）、`broker`（服务端核心，依赖 submap、rustls、WebSocket 栈）、`ipc`（客户端）等 feature 各自经 `#[cfg(feature)]` 守卫打开不同模块与依赖，且 feature 间可传递（如 `broker-rpc` 含 `broker`+`rpc`+`rmp-serde`，`server` 引入 `broker-rpc`）。两个二进制 `busrtd`（`server` feature）与 `busrt`（`cli` feature）均构建于核心库之上。`src/lib.rs` 的模块声明区给出了「feature→模块」导航图。此外引出待后续承接的概念：QoS（`qos.no`/`qos.processed`）、跨语言二进制协议、Python/JS/Dart 绑定。
