本讲解析独立服务端二进制 busrtd：它只是给嵌入式 Broker 库套上「clap 参数＋日志＋daemonize＋信号处理＋多传输监听」的外壳，由 server feature 门控，区别于客户端 CLI busrt。核心结论与术语：参数全集中于 Opts（clap derive），其中 -B/--bind 为可重复的 Vec<String>，是「多绑定」的根，并透传给 Broker/ServerConfig，进程默认用 mimalloc 全局分配器。启动顺序固定：三选一日志→可选 daemonize→多线程 tokio 运行时→写 pid→挂 SIGINT/SIGTERM→Broker::create 与 init_default_core_rpc()（挂载 .broker 内置核心 RPC 客户端、set_queue_size）→绑定循环→主循环。关键特色是路径自动分发：fifo: 前缀走 spawn_fifo，/ 开头或 .sock/.socket/.ipc 结尾走 spawn_unix_server，其余走 spawn_tcp_server；FIFO 命令经核心 RPC 客户端解析，故依赖核心 RPC 已初始化。优雅终止由全局 AtomicBool SERVER_ACTIVE 协调：信号任务调 terminate()，删 pid/socket 文件、announce(BrokerEvent::shutdown()) 通知 .broker/warn、置标志 false 并 sleep 1 秒传播窗口，主循环随后退出；SIGINT 静默、SIGTERM 留日志。
