本讲把 BUS/RT 从「能编译」推进到「能运行」，覆盖四个最小模块。第一，二进制门控：项目产出服务端 `busrtd`（入口 `src/server.rs`）与 CLI 客户端 `busrt`（入口 `src/cli.rs`），分别由 `Cargo.toml` 的 `[[bin]]` 段用 `required-features = ["server"]`/`["cli"]` 门控，未启用对应 feature 则该二进制不编译。第二，feature 传递依赖：`server` 经 `broker-rpc` 自动带出 `broker` 与 `rpc`，使 `busrtd` 天然具备核心 RPC 能力。第三，一键启动：`test.sh` 用 `case` 封装 `server`/`cli` 两分支并预置三种 `-B` 绑定，`justfile` 的 `test` 目标对多种 feature 组合跑 clippy。第四，传输自动选择：`busrtd` 按 `-B` 写法判定传输——`fifo:` 前缀走命名管道，以 `/` 开头或特定后缀走 Unix socket，其余走 TCP，并用 `init_default_core_rpc()` 注册 `.broker` 内置方法。核心结论：`broker info`/`stats` 本质是 CLI 向代理自身 `.broker` 目标发起的 RPC，返回值经 rmp-serde 反序列化、prettytable 打印；退出时由 SIGINT/SIGTERM 触发，广播 shutdown 并清理 pid/socket 文件。关键术语：feature 门控、传递依赖、`-B` 绑定、传输自动选择、核心 RPC、`.broker` 目标、`BrokerInfo`/`BrokerStats`、QoS::Processed。
