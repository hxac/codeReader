本讲讲解 busrt CLI：由 cli feature 门控、连接代理而非充当服务端的调试与基准工具。命令行接口由 clap 描述为两层结构：顶层 Command 枚举分发 broker/listen/send/publish/rpc/benchmark 六个子命令，broker 与 rpc 各嵌套一层；全局 Opts 经 create_client 一比一映射到 ipc::Config（path/name/token/timeout 等旋钮）。核心结论：broker info/stats/test/client.list 本质是向代理内置 .broker 目标发起的点对点 RPC，结果反序列化后表格打印；send 目标含通配符时自动改走广播，listen 用先 exclude 再 subscribe 加 ping 保活维持循环；print_payload 按 JSON→MessagePack→HEX 三级解码载荷，RPC 参数经 str_to_params_map 解析为带类型 map 后序列化；wto! 宏为发送与确认各设一层超时。benchmark 基于 bma-benchmark，覆盖 RPC 与消息多场景、靠多 worker 并发测吞吐，可分离代理转发与 RPC 框架开销。
