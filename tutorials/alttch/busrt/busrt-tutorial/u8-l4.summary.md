本讲为手册终篇，以 Rust 实现为参照，讲解 BUS/RT 官方多语言绑定：Python 同步（busrt）、Python 异步（busrt_async）、JS/TS（busrt.ts），外加社区维护的 Dart 绑定。核心结论是「协议即契约，语言只是壳」：跨语言互通不依赖 FFI，而因每种绑定各自重新实现了同一套二进制线协议，故协议常量（GREETINGS=0xEB、PROTOCOL_VERSION、OP_*、9 字节全零 PING_FRAME、RESPONSE_OK=1）、帧布局（9 字节发送头 op_id|flags|len、6 字节接收头 kind|len|realtime、flags=op|(qos<<6)）、RPC 四态码 0x00/0x01/0x11/0x12（id=0 表示不回复、错误码按 -32000-code 换算、call_id 在 0xffff_ffff 处回绕）在三种语言里逐字节一致。同步与异步只是并发模型差异（threading / asyncio / Node 事件循环）而非协议差异，二者不可混用；JS 绑定最独特之处是用 Mutex 的「锁住—释放」模拟 Event 来等待 ACK。调用代理内置的 .broker 核心 RPC（如 test 方法）是验证跨语言 RPC 互通的最简方式。
