本讲在 u5-l1 帧协议之上讲解 RPC 客户端与服务端实现（源码位于 `src/rpc/async_client.rs`），覆盖四个核心模块。`RpcClient` 包裹底层 `AsyncClient` 并实现 `Rpc` trait，提供三种调用：`notify`（通知帧 `0x00`，无 method）、`call0`（请求帧 `0x01` 但 id 填全零、不登记等待）、`call`（带自增 `call_id`，登记 `CallMap` 的 oneshot 回复通道并等结果；id 回绕到 1 以避开「id=0 表示不需回复」）。`RpcHandlers` trait 含 `handle_call`（返回 `RpcResult`）、`handle_notification`、`handle_frame`（非 RPC 帧）三个回调，handler 会被并发启动多实例，共享状态须原子化；只调用不响应的客户端用 `DummyHandlers`（`RpcClient::new0`）。`processor()` 是事件循环心脏：取帧解析为 `RpcEvent` 后按种类分发，对 `id>0` 请求 spawn 处理并手工拼接回复帧（`0x11` 成功/`0x12` 错误，回复 QoS 镜像请求 realtime 位）回送，对 Reply 按 id 兑现 oneshot，找不到则记 orphaned。`Options` 提供 `blocking_notifications`/`blocking_frames`（顺序处理）与 `with_task_pool`（限流）；阻塞模式禁止在 handler 内调用本 client 否则死锁，请求处理一律异步 spawn；pinger 仅对有 timeout 的外部客户端启动。
