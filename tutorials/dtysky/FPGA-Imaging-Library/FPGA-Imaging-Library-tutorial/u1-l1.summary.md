本讲建立 FPGA-Imaging-Library（F-I-L）的项目全貌心智模型，由三个最小模块组成。**项目定位**：F-I-L 是面向 Xilinx FPGA 的开源图像处理 IP 核库，采用 LGPL 许可证；所有图像操作被封装成接口统一、可参数化、可复用的 IP 核，支持流水线与请求响应两种工作模式（由 `work_mode` 区分），可像搭积木般串联成流水线。**七大功能分类**构成能力地图：`Point`（点运算，如 Graying/Threshold）、`Geometry`（几何变换，如 Scale/Rotate）、`LocalFilter`（邻域滤波，如 MeanFilter/形态学）、`Generator`（产生坐标/行/窗口/帧的源头与节拍器）、`Connector`（位宽转换与通道多路复用的粘合剂）、`InOut`（Cam/VGA/IIC/BRAM 等真实外设接口）、`BoardInit_AXI`（经 AXI4-Lite 寄存器把算法 IP 接入 Zynq PS 软核的系统接入层）。典型链路为「InOut 采集 → Generator 驱动 → Point/Geometry/LocalFilter 处理 → Connector 整形 → InOut 输出」。**仿真闭环**遵循「软硬一致性」理念：Python 软件参考实现作为黄金模型，ModelSim 跑 RTL 功能仿真，自动比对二者结果；工具链为 Python 2.7+PIL、ModelSim 10.1+、Vivado 三类。每个 IP 有固定子目录约定，仿真统一为五步流程（含 `creat.py`/`covert.py` 等脚本），板上测试走 `TestOnBoard/` 下 Vivado+SDK 的 C 驱动链路。后续讲义将逐层放大这些细节。
