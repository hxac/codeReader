本讲以 `Point/ColorReversal` 为样板，揭示 F-I-L 每个 IP 都遵循同一套顶层目录约定，分硬件侧与软件仿真侧两部分。硬件侧 `HDL/` 下 `<IP>.srcs/` 按 fileset 分工：`sources_1` 存综合用 RTL 主模块、`sim_1` 存 testbench、`xgui` 存参数 GUI 的 Tcl 脚本、`component.xml` 是 IP-XACT 描述，负责把文件集、端口、参数登记成一个可参数化 IP，并用 `dependency` 表达式让端口位宽随 `color_channels * color_width` 动态变化。软件侧五块对应「软硬一致性」闭环：`ImageForTest`（测试图与 `conf.json`）、`SoftwareSim/sim.py`（软件黄金模型）、`HDLSimDataGen/create.py`（图片转 `.dat` 激励，每通道 10 位）、`FunSimForHDL/Run.do`（ModelSim 驱动出 `.res`）、`SimResCheck`（`convert.py` 把 `.res` 还原成 `.bmp`、`compare.py` 算 PSNR 打分）。核心结论是：统一目录与命名约定让脚本可跨 IP 复用、使软硬结果能自动配对比对；`.dat`/`.res` 等运行时产物由 `.gitignore` 排除不入库；README 中 `creat.py`/`covert.py` 为笔误，实际以 `create.py`/`convert.py` 为准。关键术语 RTL、testbench、`component.xml`、`.dat`/`.res`、PSNR、以及六端口约定、`work_mode` 枚举（Pipeline=0/ReqAck=1），将为下一讲工具链运行与后续接口讲义承接。
