本讲以全库最简点运算 `ColorReversal`（颜色取反）为例，把「统一接口 + 两种工作模式」契约落成可综合 RTL。核心拆为三块电路：其一，`generate if/else` 做**编译期**模式二选一——`work_mode=0` 保留流水线支（`posedge clk` 锁存 `~in_data`），`work_mode=1` 保留请求响应支（`posedge in_enable` 锁存），二者功能等价仅时序不同，故 `work_mode` 必须是编译期参数。其二，`out_ready` 由「三沿敏感列表（posedge clk / negedge rst_n / negedge in_enable）+ 联合复位」的 always 块产生，实现 `in_enable` 接入异步清零——上升沿不在敏感列表故上升晚一拍，下降沿在敏感列表故立刻清零；再由门控 `out_ready==0?0:reg_out_data` 保证无效时输出干净 0。其三，核心算法 `~in_data`，对 B 位像素满足 `~p=2^B-1-p`，8 位即 `255-p`，与软件黄金模型一致，是为「软硬一致性」。关键性质：按位取反**通道无关**（可整根总线一次做），而左移等**通道相关**运算须借助 `genvar i` 按通道分别做——这是承接后续点运算讲义的要点。
