本讲确立了 F-I-L 全库所有图像处理 IP 共享的对外契约。**统一参数**有三个：`work_mode`（0=流水线 Pipeline、1=请求响应 Req-ack，编译期选择不可运行时切换）、`color_channels`（颜色通道数，仅多通道 IP 才有）、`color_width`（每通道位宽，1~12）；数据总线总位宽恒为 W = C × B。**统一端口**有六个：`clk`、`rst_n`、`in_enable`、`in_data`、`out_ready`、`out_data`，其中 `in_enable`/`out_ready` 构成全库统一的握手对。核心结论是：`out_ready` 由带 `negedge in_enable` 三沿敏感列表的寄存器产生——其上升比 `in_enable` 晚一个时钟、下降则异步立刻清零；`out_data` 在 `out_ready=0` 时被门控为 0，给下游确定的无效占位。`in_enable` 由此具备双重身份：流水线模式里相当于帧开始的二次复位门，请求响应模式里是逐笔请求开关。两种模式的本质差别由 `generate if(work_mode==0)/else` 在综合阶段二选一决定——前者每拍锁存、吞吐 1 像素/时钟、适合连续流，后者只在 `in_enable` 上升沿锁存、适合节奏不定的逐笔握手；二者功能等价，仅时序形态不同。
