本讲解决「拿到 F-I-L 源码后如何跑起仿真」。核心结论：仿真由三种工具接力——Python 2.7+PIL 算软件黄金模型并做数据转换与打分，ModelSim 10.1+ 跑 RTL 功能仿真，Vivado 负责 IP 打包与器件库（ModelSim 须先编译进 Vivado 的 Xilinx 库才能仿真）。完整仿真分五步：配置图像、软件仿真、生成 dat、功能仿真、转换比对，依次由 sim.py、create.py、Run.do、convert.py、compare.py 完成。关键术语：黄金模型、testbench、.dat（8 位二进制输入激励，注意 create.py 里的魔数 10 实为 8 位 + '0b' 前缀）与 .res（十进制输出）、conf.json（其 conf 数组每组参数对应一个独立输出文件，无参数 IP 用 ["default"] 占位）；软件结果 `-soft.bmp` 与硬件结果 `-hdlfun.bmp` 最终用 PSNR 配对打分，完全一致记为 10^6，这些中间产物均不入库。
