本讲解决「如何把 VTR 源码编译成可运行工具」。核心是根目录 `Makefile` 作为**包装层**：它不编译源码，只把用户命令（`make`/`make vpr`/`make BUILD_TYPE=debug`）规范化并翻译成 CMake 参数，转发给 `build/` 下由 CMake 生成的 Makefile 真正编译。VTR 强制 out-of-source 构建、禁止 in-source，`vpr` 产物默认在 `build/vpr/vpr`，`make install` 后到 `build/bin/`。构建类型分主类型（`release`/`debug`/`RelWithDebInfo`，控制优化与调试级别）与后缀（`_strict` 警告即错误、`_pgo` 两阶段优化）；CMake 选项经 `CMAKE_PARAMS` 传入且**粘性**，改了不生效多为 build 缓存。加速靠并行 `-jN`、只编 `vpr`、开发期关 IPO、ccache。关键术语：包装层、BUILD_TYPE、CMAKE_PARAMS、构建类型、CMake 选项粘性、PGO、out-of-source。
