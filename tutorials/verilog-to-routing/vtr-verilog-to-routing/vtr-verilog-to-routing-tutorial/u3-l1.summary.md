本讲是「核心数据结构」单元首讲，剖析 VPR 所有网表的共同祖先——`Netlist` 模板基类。它用 Block/Port/Pin/Net 四类实体刻画电路：块是超图节点，网是连线（恰有一个驱动引脚恒在索引 0、若干接收引脚），端口是多位引脚的逻辑分组，引脚把块与网相连；四者经 `pin_net`/`net_pins`/`pin_block`/`port_pins` 等方法双向交叉引用。三大支撑设计：`vtr::StrongId` 借幽灵 tag 在编译期区分 ID 种类，靠 `explicit` 转换杜绝隐式误用，类型安全零运行时开销；`vtr::vector_map` 以 StrongId 为下标（要求 ID 连续递增），配合结构数组（SoA）布局换取内存紧凑与缓存友好；增删遵循「脏标记 + 批量压缩」，`remove_*()` 仅标无效，`compress()` 一次性重排并返回 `IdRemapper` 更新旧 ID。基类在 `netlist_fwd.h` 定义 `Parent*Id`，以 NVI 约定同子类分工，供后续 `AtomNetlist`（u3-l2）、`ClusteredNetlist`（u3-l3）派生特化，再由 VprContext（u3-l4）挂入全局状态。
