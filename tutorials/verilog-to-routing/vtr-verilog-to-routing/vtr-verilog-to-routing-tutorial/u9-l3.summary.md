本讲是「共享库、测试与工程实践」单元收尾，承接 u9-l2 单元测试，定位为「单元测试管 API 对错，回归测试管整套 CAD 流程的质量指标有没有变坏」。核心模块有四：回归任务体系由三层脚本（`run_reg_test.py`→`run_vtr_task.py`→`parse_vtr_task.py`）加两层配置（`task_list.txt`+`config.txt`）构成，一个 task 即电路×架构×脚本参数的笛卡尔积，黄金结果还会反喂成下次运行的运行提示；解析管道用 `name;file;regex` 模式将日志抓成 TSV，分全量 `parse_results.txt`（供校验）与精选 `qor_results.txt`（供均值）；黄金校验因启发式算法不可位级复现而采用容差判定，三类准则为 `Range`、`RangeAbs`、`Equal`，只强制校验准则声明的指标；QoR 对比用逐项比值与几何均值聚合整套 benchmark，对越小越好的指标 geomean<1.0 即改进，分套件级 `qor_geomean.txt` 与跨运行 Excel 比对两条路径。关键术语：QoR、黄金结果、通过准则、几何均值。
