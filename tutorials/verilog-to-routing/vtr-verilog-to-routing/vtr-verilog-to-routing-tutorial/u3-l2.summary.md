本讲承接 Netlist 基类，落到 VPR 的起点数据 AtomNetlist（原子级网表）。它在拓扑之上新增三类原子特有信息：原语模型绑定（LogicalModelId、t_model_ports*）、真值表（TruthTable）、网别名。真值表存于私有成员 block_truth_tables_，对 LUT 表达逻辑函数、对 FF/Latch 仅存初值，靠模型 ID 区分语义；块类型（INPAD/OUTPAD/BLOCK）由模型 ID 推导，不单独存储。BLIF 读入分两层：外层 read_and_process_circuit 做格式推断与后处理清洗，内层 read_blif 借外部库 blifparse 解析文本，由 BlifAllocCallback 回调建出块/端口/引脚/网；原语模型来自架构文件解析的 LogicalModels（含库模型 .names/.latch/.input/.output 与用户模型），.subckt 找不到即致命报错，体现「架构驱动」。AtomLookup 不存电路，只存原子实体与后续阶段实体（t_pb、ClusterBlockId、tnode）的映射，随打包、时序图构建逐步填表，与 AtomNetlist 共组 AtomContext 经 g_vpr_ctx 供全流程共享。
