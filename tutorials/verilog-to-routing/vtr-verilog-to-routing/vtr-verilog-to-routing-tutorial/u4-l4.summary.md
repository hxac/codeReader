本讲聚焦打包阶段的「门禁」——聚簇合法化与簇内布线。核心三块：其一，簇内布线图 `lb_type_rr_graph` 按逻辑块类型在 setup 阶段一次性建好，节点为 `pb_graph_pin` 加三个虚拟源/汇/反馈节点，类型仅 `LB_SOURCE`/`LB_SINK`/`LB_INTERMEDIATE` 三种，出边按父块**模式**分桶，刻画块内引脚连通性，区别于器件级 RR Graph。其二，`ClusterLegalizer::try_pack_molecule` 跑一条由便宜到贵的检查流水线：长链/布局/NoC 静态检查 → 引脚可行性过滤 → 簇内 Pathfinder 布线，支持 `SKIP_INTRA_LB_ROUTE`（先塞后检）与 `FULL`（边塞边检）两种策略。其三，`ClusterRouter` 用协商式 Pathfinder，逐轮抬高拥塞代价，以所有节点 `occ ≤ capacity` 为成功判据；失败原因由 `e_block_pack_status` 枚举归一，回退分分子级（清理现场、换根原语重试）与簇级（先 SKIP 后 FULL 重建）两层。合法化产物 `t_pb_route` 写入 `ClusteredNetlist` 的 `t_pb`，供布局阶段使用。
