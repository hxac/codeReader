本讲覆盖 VPR 的 Server 模式与两条并行布线支路。Server 模式让布线后常驻为 TCP 服务器，采用「IO 线程收发字节、主线程 Qt 定时器每 100ms 求解」双线程模型，Task 为唯一交接棒、两把 m_tasks_mutex 临界区隔开两条线程，整个模块由 NO_SERVER 宏在编译期门控；目前仅取/画关键路径两种命令，求解依赖布线阶段填入 ServerContext 的 timing_info 与 routing_delay_calc，未就绪则任务挂起。并行布线有两条正交、且当前实现下不叠加的轴：按网并行（--router_algorithm parallel）用 PartitionTree 把边界框不相交的网分组，再以 tbb::task_group 并行布不同树节点，每线程一份 SerialConnectionRouter 以回避共享可变状态，与串行等价且确定但拥塞时可能降 QoR；按连接并行（--enable_parallel_connection_router on）用 MultiQueueDAryHeap 把单连接 A* 堆拆成多个独立加锁子队列，push 随机入队、pop 从两个随机子队列取较优者即松弛 A*，靠缓存行对齐、原子 min、numIdle 终止检测与每节点自旋锁实现高并发低冲突。
