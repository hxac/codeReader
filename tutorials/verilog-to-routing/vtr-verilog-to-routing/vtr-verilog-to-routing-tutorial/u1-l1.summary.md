VTR 是开源 FPGA CAD 框架，核心价值是「开源 + 可换架构」，服务 FPGA 架构研究与 CAD 算法开发。它接收两个输入——电路的 Verilog 描述与目标 FPGA 架构的 XML 描述，产出速度（时序）与面积评估，可选生成 FASM。流程分三段：PARMYS（综合与部分映射，仓库内维护）→ ABC（逻辑优化与技术映射，外部不可改）→ VPR（打包/布局/布线/时序分析，主开发目标）。VPR 内部数据流为 AtomNetlist → Prepacker → Packer → ClusteredNetlist → Placer → Router，本质是网表被不断改写、逐层抽象。最关键的理念是「架构驱动」：目标 FPGA 由运行时 XML 决定，算法代码绝不硬编码架构假设；XML 启动时被解析进共享库 libarchfpga（核心头文件 arch_types.h）的数据结构，驱动后续全部阶段。由此引出调试直觉：VPR 行为反常，先怀疑架构 XML，再怀疑算法。
