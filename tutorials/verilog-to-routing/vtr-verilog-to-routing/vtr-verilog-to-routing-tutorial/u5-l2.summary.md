本讲讲解 VPR 模拟退火布局中的移动生成器——决定退火每步如何扰动布局的可插拔策略对象，核心接口为 propose_move()（提议移动、登记受影响块但不改全局状态）与 process_outcome()（接收反馈，默认空实现）。生成器分两层：叶子型（Uniform/Median/Centroid 及加权变体、CriticalUniform、FeasibleRegion、Manual）真正决定「挑哪个块、搬到哪」，差异集中在挑块策略（随机/关键块/块类型）与选址策略（均匀/中位数/质心/可行域），加权版以时序关键度为权重，融合线长与时序目标；聚合型（StaticMoveGenerator 固定概率、SimpleRLMoveGenerator 强化学习）持有全部叶子并决定本次用哪一种。SimpleRLMoveGenerator 用 k-臂赌徒 RL（ε-greedy 或 softmax）在线学习各移动 Q 值，按 Q(a)←Q(a)+α(reward−Q(a)) 增量更新，是 VPR「自学扰动策略」的核心；退火分 EARLY/LATE 两状态，温度衰减因子落在 (0.6, 0.85) 时切换，后期才引入 CriticalUniform/FeasibleRegion 等时序精修移动。关键澄清：simpleRL 是调度员、critical_uniform 是被调度的具体兵种，二者不在同一抽象层。本讲依赖 u5-l1 的退火框架，其 rlim、compressed grid 与 ΔC 增量计算留待 u5-l3 系统讲解。
