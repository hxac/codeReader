本讲讲 VTR 两层基础设施：底层 libvtrutil 是与 FPGA 无关的通用零件库，上层 vpr/src/util（vpr_utils）是懂 FPGA 语义的跨阶段胶水，后者站在前者肩膀上。关键概念：StrongId 用编译期 tag + explicit 构造，零运行时开销阻止 ID 类型混用，是 ID 容器的下标基础。vtr::vector 私有继承 std::vector、把下标重写为 StrongId；vtr::vector_map 额外提供 map 语义并用哨兵填空洞，二者都要求键从 0 连续。NdMatrix 用单块线性内存表达 N 维稠密矩阵，下标是 size_t（用于 DeviceGrid 等坐标网格，非 StrongId）；Cache 是单槽惰性缓存。日志/断言/错误分三层：VTR_LOG* 只打印不停机，VTR_ASSERT* 分四档且 release 可关，停机须用 VPR_THROW 抛 VprError，阶段类型由 e_vpr_error 枚举刻画。vpr_utils 跨阶段函数（physical_tile_type 三重载、is_inter_cluster_node）只读 g_vpr_ctx、彼此层层委托，是横切逻辑集中化的典范。
