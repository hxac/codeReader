本讲是「FPGA 架构描述与解析」的起点，聚焦架构 XML 如何被读进内存。它覆盖三个最小模块：①架构 XML 顶层结构——根节点 `<architecture>` 下按依赖顺序处理 `models`、`layout`、`device`、`switchlist`、`segmentlist`、`complexblocklist`、`tiles` 等元素；②解析入口 `xml_read_arch` 的签名与产物——输入 XML 文件路径，输出 `t_arch`（全局器件信息）、`physical_tile_types`（物理瓦片）和 `logical_block_type`（逻辑块）三组结构，产物直接写入 `DeviceContext`；③pugixml 外层封装 `pugiutil`——用 `ReqOpt`（REQUIRED/OPTIONAL）、`loc_data` 行号查询表和 `XmlError` 异常简化取节点/属性操作，错误经 `archfpga_throw` 转成带文件名行号的致命报错。核心结论是「架构驱动」：VPR 算法绝不硬编码架构假设，行为差异最终都追溯到这份 XML；物理瓦片与逻辑块是两个抽象层，由 `link_physical_logical_types` 绑定。`<switchlist>` 须先于 `<segmentlist>` 解析，`<switchblocklist>` 仅在开关盒类型为 `CUSTOM` 时必需。`t_pb_type` 层次、DeviceGrid 网格等结构含义留给后续讲义。
