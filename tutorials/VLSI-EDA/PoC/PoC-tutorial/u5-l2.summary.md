本讲讲 PoC 如何接入第三方验证生态，含三个最小模块。一、验证方法学对比：cocotb（Python cosim 驱动管脚）与 VUnit（Python 自动化回归）在外层调度；OSVVM（纯 VHDL 覆盖率/随机/记分板，渐进增强）与 UVVM（纯 VHDL 的 VVC 框架加总线 BFM/VIP，结构化多接口台）在内层提供验证原语。二、git submodule 集成：四库以 submodule 嵌入 `lib/`，权威来源是 `.gitmodules`，url 用相对路径（`../OSVVM.git`），相对 PoC origin 解析为同组织镜像（`VLSI-EDA/OSVVM` 等），与 README 记录的官方上游（`JimLewis/OSVVM`、`UVVM/UVVM_All`）不同；克隆须 `--recursive` 或 `init`+`update`。三、`.files` 清单：只有纯 VHDL 的 OSVVM/UVVM 有清单（cocotb/VUnit 是 Python 无 VHDL 清单）；cocotb 反以 `Cocotb_QuestaSim` 这个 `Tool` 取值出现在厂商原语清单里。骨架为版本门槛→按 `Tool` 选预编译库→`?{...}` 探测→找到则 `library` 挂接，否则 `vhdl` 自编译；OSVVM 有完整兜底，UVVM 仍为 TODO。核心结论：submodule 负责「把代码弄到本地」，`.files` 负责「把代码接进编译」，两者必须同时到位。
