本讲把 PoC 的「外层认知」细化为「空间地图」，只讲文件存放位置，不深入具体 IP 核实现。两个最小模块：其一为顶层目录用途——PoC 用 11 个顶层目录按「角色」组织文件：`src`（源码）、`tb`（测试台）、`lib`（第三方库，以 git submodule 形式嵌入，故克隆需 `--recursive`）、`ucf`（约束，支持 `.ucf`/`.xdc`/`.sdc`）、`netlist` 与 `xst`（综合输出与配置）、`py`/`sim`/`tcl`/`temp`/`tools`（基础设施）。关键约定：所有 VHDL 源码统一编译进名为 `PoC` 的库，不兼容文件用 `.v93.vhdl`/`.v08.vhdl` 标注版本。其二为命名空间子目录树——`src/` 内部按 15 个子命名空间分组，目录名基本等于 VHDL 命名空间名（但有 `dstruct` 对应 `PoC.dstructs` 这类例外），大命名空间内部再分层。每个核有三类伴生文件：源码 `.vhdl`、编译清单 `.files`（pyIPCMI 消费，通过条件 `include` 把 `lib/` 与 `src/common/` 串成依赖网）、命名空间级包 `<ns>.pkg.vhdl`。核心结论：`tb/` 是 `src/` 的镜像子集；顶层 `sim/`（波形视图）与 `src/sim/`（仿真辅助包源码）是两回事；`xst`/`netlist`/`ucf` 属综合链路，细节留待后续单元。本讲引入的关键术语：IP 核、测试台、综合、约束文件、命名空间、`.files` 编译清单、命名空间级包、模板文件 `my_config`/`my_project`（供下一讲承接）。
