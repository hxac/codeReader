本讲讲解 PoC 的命名空间包模式：每个命名空间都有一份固定名为 `<ns>.pkg.vhdl` 的「根包」，充当该命名空间对外的 API 目录页，集中声明 component 组件、type 枚举类型、function 辅助函数三类内容。核心结论有三：其一，命名空间包必须先于同名命名空间下任何核被编译——`.files` 清单先 include 公共包 `src/common/`，再编译 `<ns>.pkg.vhdl`，最后编译具体核，顺序错则核里的 `use PoC.<ns>.all` 会因找不到包而失败。其二，包的形态因地制宜：`arith.pkg` 三样俱全；`fifo.pkg` 是纯组件货架（无 type/function，故无 package body）；`mem.pkg` 仅含类型与函数，组件被下放到 `ocram`/`ocrom` 子包，形成「根包共享层 + 家族子包」的二级拆分。其三，跨命名空间复用只需一句 `use`，如 `fifo_cc_got` 写 `use PoC.ocram.all` 即可实例化 `mem` 下的 `ocram_sdp`。函数声明放在 `package`(spec)、实现放在 `package body`；枚举类型（如 `tArch`）常作 generic 的配置旋钮，全命名空间共享。关键术语：根包、子包、component 组件声明、`PoC` 库、`.files` 编译清单。
