本讲把 u4-l1 的仿真辅助包与 u3-l4 的 FIFO 接口组装成一个完整、可自检的 PoC 测试台，围绕三个最小模块展开。一是**测试台骨架**：固定四段式——文档头与许可证、`library/use` 子句（含 `sim_types`/`simulation`/`waveform` 等仿真专用包）、空实体 `<name>_tb`（无端口）、`architecture tb`。架构头放 `CLOCK_FREQ` 等常量与 `clk`/`rst` 信号，`begin` 后的 `simInitialize`、`simGenerateClock`、`simGenerateWaveform` 三连调用是含 `wait` 的过程，裸写在架构体里即等价隐式并发进程，负责自激时钟与复位脉冲。二是**DUT 例化与批量验证**：直接实体例化 + 命名绑定；对 `fifo_cc_got` 的 `DATA_REG`/`STATE_REG`/`OUTPUT_REG` 三个布尔 generic，用 `for c in 0 to 7 generate` 配合 `mod` 解码一次性展开 8 种组合，每组自带私有信号、DUT 与读写进程。三是**板级配置变体**：`tb/common/my_config_<board>.vhdl` 这族文件包名统一为 `my_config`、仅 `MY_BOARD` 取值不同，`my_config.files` 在**编译期**按 `BoardName` 挑一份编译进 `PoC` 库，区别于展开期 generate 分发。核心结论：自检式测试台通过 `simCreateTest`/`simRegisterProcess`/`simAssertion`/`simFinalizeTest` 把比对累计进共享的 `globalSimulationStatus`，不因首错中断，一次跑完全部失败清单；`config_tb` 则代表「无 DUT、纯断言」型测试台，换板需同时换变体与断言期望值。
