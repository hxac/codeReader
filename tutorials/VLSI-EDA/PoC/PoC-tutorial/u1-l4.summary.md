本讲拆解 PoC 官方规范 vhdl_coding.md，讲清代码该「长什么样」。三个最小模块：（1）命名与文件约定：源码统一用 `.vhdl` 后缀；实体采用蛇形命名并以命名空间为前缀（`<namespace>_<entity>`，如 `arith_addw`），单实体单文件，文件名即 `<entity>.vhdl`；文件名自带「地址」，可反推源码路径、测试台 `<entity>_tb.vhdl` 及命名空间包 `<ns>.pkg.vhdl`（其中集中声明各核的 component）。（2）架构命名：可综合实现统一用 `architecture rtl`；测试台实体加 `_tb` 后缀、架构名固定 `tb`，通常无端口，内部例化被测核作为 DUT，文档头标签写 `Testbench:`。（3）文档头与风格：每个文件须含编辑器配置行（强制用 Tab 缩进、宽 2，而非空格）与带分隔线（匹配 `/^--\s*={16,}$/`）的文档头加 Apache 2.0 许可证块，该格式会被 Sphinx 与 docs/Entity.template 自动解析；大小写约定为关键字与标准类型全小写、常量/generic 全大写；时序信号必须给初值（无关时用 don't-care `(others => '-')`），组合信号不给初值；实例化强制命名绑定、禁止位置绑定。
