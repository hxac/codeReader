本讲是第 5 单元（专家层）开篇，从「VHDL 写了什么」转向「PoC 项目如何被工具链驱动」。核心覆盖四条线索：一是 `poc.sh`/`poc.ps1` 两个平台相关入口脚本只是定位目录、记忆参数、委托加载、透传退出码的「薄壳」，不含任何业务逻辑，真正干活的是被它们 `source`/`Import-Module` 的子模块；二是 pyIPCMI 以 git submodule 形式挂在 `lib/pyIPCMI/`，必须 `--recursive` 克隆或手动 `update --init`，否则委托链断在第一步；三是 `.files` 是 pyIPCMI 自定义的工具中立编译清单语言（含 `vhdl`/`include`/`if`/`report`/`path` 语句），在编译前被求值，决定源码顺序与版本/条件筛选；四是工具链差异被抽象成正交条件变量 `ToolChain`/`Tool`/`VHDLVersion`/`BoardName`/`Environment`/`CONFIG.*`，同一组变量同时驱动编译期、展开期、综合期三层可移植性。关键术语：wrapper 脚本、git submodule、委托链、`.files` 清单、条件变量、上下文树。本讲止步于「委托边界」，pyIPCMI 内部 Python 源码留待 checkout 后承接。
