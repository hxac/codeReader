本讲解决「把 PoC 拉到本地、启动命令行、完成最小配置」三件事。第一，下载：PoC 以 git submodule 嵌入 pyIPCMI 及 OSVVM、UVVM、VUnit、cocotb 等第三方库，必须用 `git clone --recursive`，否则 `lib/pyIPCMI/` 为空、命令行无法启动；安装目录称为 `PoCRoot`。第二，入口脚本：`poc.sh`（Bash，Linux/macOS）与 `poc.ps1`（PowerShell，Windows）都是极薄的 wrapper script，只负责保存当前工作目录与解析脚本自身路径，再把控制权委托给 Python 基础设施 pyIPCMI；后者依赖仅 colorama、py-flags。第三，配置：仓库只提供 `my_config.vhdl.template` 与 `my_project.vhdl.template`，需复制去掉 `.template` 后缀、改掉 `"CHANGE THIS"`，得到本地私有配置文件。需填写四个常量——`my_config.vhdl` 的 `MY_BOARD`、`MY_DEVICE`（描述目标硬件）与 `my_project.vhdl` 的 `MY_PROJECT_DIR`、`MY_OPERATING_SYSTEM`（描述宿主环境）。核心结论：这四个常量最终被公共包 `config`（`src/common/config.vhdl`）经 `use PoC.my_config.all` / `use PoC.my_project.all` 读取，派生出厂商（`VENDOR`）与器件信息；当 `MY_DEVICE="None"` 时跳过器件常量、回退到由 `MY_BOARD`（如 `Custom`）派生。配置机制的内部派生细节留待后续讲义展开。
