本讲是 FlashInfer 仓库的「地图课」，不讲代码改动，只建立目录空间感。核心结论是仓库呈「三层蛋糕」分层：`include/` 为 header-only、框架无关的 CUDA kernel 模板（只认原始指针 `T*`）；`csrc/` 是 launcher 与 TVM-FFI 绑定的混合层；`flashinfer/` 为用户直接调用的 Python 包，内含 JIT 代码生成器。最重要的一条架构红线是 `include/` 绝不能 `#include` torch 头文件，所有框架相关脏活集中在 `csrc/`，从而使 kernel 可被多框架复用。本讲引入的关键术语包括：header-only 库、TVM-FFI 及其 `TensorView`（指针+形状+dtype 的轻量张量封装）、`TVM_FFI_DLL_EXPORT_TYPED_FUNC` 导出宏、注意力 wrapper 的 `plan`/`workspace_size`/`run` 三件套约定。配套讲清了 `3rdparty/` 下 `cutlass`、`spdlog`、`cccl`、`nixl` 四个 git 子模块的职责，及 `pyproject.toml` 将它们映射为 `flashinfer.data.*` 命名空间。最后以 decode 注意力串起四层调用栈：`flashinfer/decode.py` → `flashinfer/jit/attention/modules.py`（JIT 生成器）→ `csrc/batch_decode*.cu`（launcher + FFI 绑定）→ `include/flashinfer/attention/decode.cuh`，数据形态沿 `torch.Tensor → TensorView → T*` 裸指针演进，为后续 JIT 系统讲义铺垫。
