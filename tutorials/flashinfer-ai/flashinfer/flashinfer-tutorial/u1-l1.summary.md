FlashInfer 是面向 LLM 推理服务（而非训练）的 GPU kernel 库与 kernel 生成器，最鲜明的工程特征是默认 JIT 即时编译——修改 kernel 源码后无需重装包即可生效，且能按参数组合现场生成特化 kernel。它围绕三大核心能力展开：Attention（含 Paged/Ragged KV-Cache、MLA、Cascade、稀疏、POD）、GEMM（BF16/FP8/FP4、Grouped GEMM）与 MoE（融合 kernel、多路由、量化），外加采样、归一化等辅助算子。其关键设计是「多后端」：对同一算子提供 FlashAttention-2/3、cuDNN、CUTLASS、TensorRT-LLM 等多种实现，并按硬件、数据类型自动选优，以适配从 T4 到 B200 的差异负载。硬件覆盖 Turing(SM7.5) 至 Blackwell(SM12.x)，但「架构可用」不等于「特性全可用」——如 FlashAttention-3 需 SM9.0、BF16/FP4 GEMM 主要面向 Blackwell，需用 compute capability 对照支持矩阵判断。核心术语包括：kernel、compute capability、JIT、continuous batching、KV-Cache、后端选择。相比 PyTorch 自带 attention，FlashInfer 的价值在于专为服务场景优化（动态批处理 + KV-Cache 管理）、多后端选优、低精度与 CUDA Graph/torch.compile 兼容。本讲只读 README、CLAUDE.md、pyproject.toml 三个项目级文件，尚未进入具体代码目录。
