本讲解决 FlashInfer 的安装与验证，不涉及具体 kernel 代码。核心结论：FlashInfer 提供三个协作包——`flashinfer-python`（核心包，必装，JIT 为兜底机制，首次调用时现场编译/下载 kernel）、`flashinfer-cubin`（全架构预编译二进制，用于离线）、`flashinfer-jit-cache`（按 CUDA 版本预编译的 `.so` 缓存，用于加速启动）；后两者为可选加速包。源码开发安装用 `git clone --recursive`（拉取 `3rdparty/{cutlass,spdlog,cccl}` 子模块，缺则 kernel 编译找不到头文件）加 `pip install --no-build-isolation -e .`（避免 pip 从 PyPI 拉不兼容的 PyTorch/CUDA，并使构建钩子能装 moe_ep）。关键术语：**editable 安装**（软链接，改源码立即生效）、**自定义 PEP 517 构建后端** `build_backend.py`（包裹 setuptools，导入即生成版本元数据 `flashinfer/_build_meta.py`，并铺设 `flashinfer/data` 目录——editable 用软链接、wheel 用拷贝）、**单一事实源版本链** `version.txt → _build_meta.py → __version__`。验证入口为 `flashinfer show-config`：JIT 工作区按「版本/排序后架构」分层为 `{cached_ops,generated}`，cubin 目录按「cubin 包 → 环境变量 → 缓存目录/cubins」三档解析。本讲尚未真正触发一次 JIT 编译，为下一单元 JIT 编译系统的前置。
