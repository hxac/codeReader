本讲围绕 batch prefill 的两个并列 wrapper——`BatchPrefillWithPagedKVCacheWrapper`（KV 存于页池，需页表三件套 `paged_kv_indptr`/`paged_kv_indices`/`paged_kv_last_page_len`）与 `BatchPrefillWithRaggedKVCacheWrapper`（KV 拼接为一维张量，仅需 `kv_indptr`）。核心结论是「plan 统一、run 分叉」：两者复用同一份 JIT 模块，`plan` 汇入同一个 C++ 符号、产出同一份 15 个 int64 的 `PrefillPlanInfo`；只有 `run` 才分出 `paged_run` / `ragged_run`。关键技巧是 ragged 把 `page_size` 写死为 1，使「按页切 KV chunk」的逻辑同时服务两种存储。

变长批次靠前缀和数组 `qo_indptr` 描述变长 query；plan 内部经 `PrefillSplitQOKVIndptr` 同时切 Q（`cta_tile_q`）与 KV（split-kv），把批次展平为 `padded_batch_size` 个等粒度工作单元，必要时在 float workspace 分配 `tmp_v`/`tmp_s` 并用 merge kernel 合并部分结果。plan 只依赖批次结构、含动态调度，不可进 CUDA Graph，可被多层 Transformer 复用。

与 decode（每请求固定 1 个 query、memory-bound、只切 KV）相比，prefill 是 compute-bound、需切 Q，并多出 `qo_indptr`、`causal`/`custom_mask` 掩码、可拆分的 `head_dim_qk`/`head_dim_vo`。
