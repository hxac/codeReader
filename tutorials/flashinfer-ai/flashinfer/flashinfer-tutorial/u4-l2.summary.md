本讲介绍 Cascade 共享前缀注意力，解决多请求共享 system prompt 时 KV-Cache 重复存储与计算的问题。核心思想是把每个请求的 KV 拆成「全局共享段」与「各自独立段」，共享段只存一份、只算一次，主要省显存与 decode 带宽。当前推荐 API 是 `MultiLevelCascadeAttentionWrapper`：所有级 KV 存于统一 paged 表，内部持有一组 `BatchPrefillWithPagedKVCacheWrapper`；`plan` 逐级派发（`causal` 仅对最底层独立段生效），`run` 先算最底层、再逐层用 `merge_state_in_place` 原地合并；两个 `*SharedPrefix*` 老 wrapper 已弃用。合并的数学根基是 logsumexp（LSE）：对两个已归一化部分输出 \((O_A,l_A),(O_B,l_B)\)，可仅凭 LSE 加权 \(O=\alpha_AO_A+\alpha_BO_B\) 精确等价于整段 attention，无需重读 K/V。三个合并函数分别处理合两段、原地合两段（带 mask 供 CUDA Graph）、一次合多段（按段数自动选 kernel），数学源头是 `state.cuh` 中的 `state_t` 在线 softmax 三元组 `(o,m,d)`，并采用 base-2 的 `exp2/log2` 且 LSE 强制 float32。级数并非越多越好，每多一级都需付出一次合并 kernel 开销。
