本讲是 JIT 编译系统收尾篇。FlashInfer 用两级缓存摊薄首次编译开销：第一级为进程内 `@functools.cache`，键是编译期参数元组（dtype、head_dim、posenc、滑窗、logits_cap 等），命中即连磁盘都不查；kv_len、batch_size 等运行期参数不参与，故改它们不触发重编译。第二级为磁盘 `.so`，键由 version + sorted_arch + name 三者拼成路径，跨进程复用，存在性由 `is_compiled`/`is_aot`（仅 `.exists()`）判断。失效机制分摊到三处：参数、版本、架构变化靠换名字或换路径做目录级隔离；源码与 flags 变化靠 ninja 的 depfile + mtime + 命令指纹在同一目录内增量重编。关键进程边界规则：改源码后须重启 Python 进程才会加载新 `.so`，因进程内缓存仍持有旧模块对象；多进程并发首编由 `FileLock` 保护 build+load 以避免互删 `.so`。引入术语：两级缓存、缓存键、URI/name、编译期参数 vs 运行期参数、depfile、进程边界。
