本讲聚焦 Blackwell 平台的 FP4 GEMM（W4A16 路径，激活 BF16、权重 FP4）。FP4 采用 E2M1 格式：4 比特浮点，正半轴仅 8 个可表示值（最大 6.0），两个值打包进一个字节。两种量化配方：NVFP4（块大小 16 + E4M3 块缩放 + 全局缩放）与 MXFP4（块大小 32 + UE8M0 纯指数缩放），`mm_bf16_fp4` 走的是 NVFP4。块缩放默认以 **128×4 交织布局**存放，以匹配 Blackwell MMA 的访存轨迹；公共函数 `_unswizzle_sf_128x4` 先将其反交织回线性 `(N, K_sf)`，再由各后端二次加工：cuDNN 保持 uint8 权重与 E4M3 线性缩放，CuTe-DSL 则把权重重打包成 int32 瓦片、缩放转置并改成 S0E5M3——dtype（uint8 vs int32）是区分两后端最直接的信号。全局缩放 g=448×6/amax，因 cuDNN 反量化不含除以 g，调用时需传 alpha=1/g 才能与 BF16 参考对齐。核心 API 链为 `nvfp4_quantize → prepare_bf16_fp4_weights → mm_bf16_fp4`，`backend` 为必填项，仅支持 SM100/103/110/120/121。仓库另有一个 W4A4 的 `mm_fp4`（双 FP4），与 `mm_bf16_fp4` 共享「E2M1 + 交织块缩放」表示但实现不同。
