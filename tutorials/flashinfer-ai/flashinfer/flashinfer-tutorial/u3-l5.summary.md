本讲讲透注意力 wrapper 的 `backend` 参数如何被决定，分三个最小模块。第一，**后端派发**：注意力的 `backend="auto"` 只在 fa2/fa3 间二选一，由纯函数 `determine_attention_backend` 实现，设三道闸门——硬件闸门 `is_sm90a_supported`（要求 major==9 且 CUDA≥12.3）、参数闸门 `is_fa3_backend_supported`（排除自定义掩码、非 NONE 位置编码、fp16 qk reduction 等）、head_dim 闸门 `is_fa3_prefill_head_dim_supported`（仅 64/128/256 或 192×128），过则 fa3 否则 fa2。关键术语：**compute capability**（折算为 `major*10+minor` 整数）、`sm_90a` 特性超集、`PosEncodingMode` 枚举。核心结论：FA3 必须 SM90a，因其依赖 Hopper 的 `wgmma` 与 TMA 指令。第二，`@backend_requirement`+`@supported_compute_capability` 是 GEMM/MoE/norm 等新式 API 的声明式后端校验机制，提供 `is_backend_supported`/`is_compute_capability_supported` 能力查询与 auto 启发式排序；注意力 wrapper 仍走老式启发式。第三，**cuDNN 后端**不写 kernel 而是用 cuDNN frontend 构造 SDPA 计算图（`@cudnn.jit`+`@cudnn.graph_cache`），依赖外部 `cudnn` 包、只支持 NHD 布局、须用户显式 opt-in。贯穿结论：能力查询回答「能不能」，选择机制回答「用哪个」。
