本讲建立「MoE=门控路由+一组GEMM+加权合并」的直觉，覆盖三个最小模块。其一执行流程：一次前向为 gate→topk→dispatch→专家FFN（gemm1+激活+gemm2）→combine 五步；FlashInfer 把后三步融合成极少 kernel，其中 cutlass_fused_moe 只接收已算好的 topk_ids/topk_weights 做 dispatch+FFN+combine，与路由 kernel fused_topk_deepseek 解耦。其二统一 API 配置层：冻结 dataclass 把 MoEConfig 组合为 routing/quant/experts/activation/backend/execution 六子配置，支持 **config 解包、repr 往返、replace 派生；后端用 BackendOptions 有序候选加 supported(arch) 声明硬件门控，CutlassConfig 恒为兜底。其三 RoutingInputMode（FromLogits/PackedPrecomputed/UnpackedPrecomputed）决定路由是 kernel 现算还是外部预算，即 topk_ids/topk_weights 作输入还是输出，统一 API 将其隐式收敛进 MoEActivationPack。关键术语：门控/路由器、top-k、专家FFN、dispatch、combine、RoutingMethodType、QuantVariant、MoELayer。
