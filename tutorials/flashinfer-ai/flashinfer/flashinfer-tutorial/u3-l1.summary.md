本讲是「注意力基础」单元首篇，建立 LLM 推理注意力组织的概念框架，不涉及 CUDA kernel 内部。涵盖三个最小模块：

其一，注意力的三个阶段：prefill（query 为变长序列，compute-bound）、decode（每请求仅 1 个 query，memory-bound）、append（将新 token 的 K/V 写入 cache）。三者对应不同 wrapper：`BatchPrefillWithPagedKVCacheWrapper`、`BatchDecodeWithPagedKVCacheWrapper` 与 `page.py` 中的 `append_paged_kv_cache`。prefill 用前缀和数组 `qo_indptr` 描述变长 query，decode 无需该结构。

其二，动态批处理下 KV cache 必须分页（paged）或变长（ragged）存储，以解决朴素 `[batch, max_len, ...]` 张量的显存碎片、无法动态扩容与共享前缀重复存储问题。分页借鉴 OS 虚拟内存，以物理页池加页表实现按需分配与跨请求共享。核心是页表三件套：`paged_kv_indptr`、`paged_kv_indices`、`paged_kv_last_page_len`，序列长度可由 `get_seq_lens` 公式反推。KV 张量有 NHD/HND 两种布局，第一维为 `max_num_pages`。

其三，plan/run 两段式 API：plan 处理仅依赖批次结构的工作（split-k 划分、kernel 选择、辅助索引），结果供同一前向所有层复用；run 仅携带每层数据计算。plan 因含动态决策无法进 CUDA Graph 且批次结构变化时需重做，run 可进图。`end_forward` 已废弃。
