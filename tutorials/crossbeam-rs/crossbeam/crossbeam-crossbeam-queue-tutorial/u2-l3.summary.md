本讲聚焦 `src/array_queue.rs` 中的 `force_push`、`capacity`、`is_empty`、`is_full`、`len` 五个函数及其环形缓冲（ring buffer）语义。核心结论：`force_push` 与 `push` 共享 `push_or_else` 的 CAS 主循环，仅在「疑似满」分支分叉——`force_push` 用 `head.compare_exchange_weak(tail - one_lap, new_tail - one_lap)` 把「确认队列已满」与「取得覆盖权」合并为一步原子操作，成功则同时推进 head 与 tail、用 `replace` 取出最旧元素返回，被覆盖的旧值必以 `Some(old)` 交还调用者，绝不静默丢失。`capacity()` 直接返回 `buffer.len()`，恒定零成本；`is_empty`/`is_full` 用 `SeqCst` 双指针读取，返回瞬时快照，且 load 顺序刻意安排使「返回 true」倾向保守可信。`len()` 采用「读 tail → 读 head → 复查 tail」的一致性快照循环（类 seqlock），保证读到的 (tail, head) 是真实存在过的组合，再用 `hix`/`tix` 三分类加空/满二分算出环形元素数；`Drop` 在持有 `&mut self` 时原样复用了这套算术，无需复查循环。关键术语：环形缓冲、stamp 编码、一致性快照、快照语义。
