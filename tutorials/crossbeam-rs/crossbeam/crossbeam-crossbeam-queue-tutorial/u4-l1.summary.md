本讲是专家层首篇，专论 crossbeam-queue 中每个原子操作的 Ordering 选择理由，分四个最小模块。模块一「发布—订阅配对」：生产者「写值→Release 标记」、消费者「Acquire 标记→读值」，将数据的可见性与有序性绑定在同一原子变量上（ArrayQueue 的 slot.stamp、SegQueue 的 WRITE 位与块指针 next），关键术语 happens-before 即由此建立。模块二「SeqCst 与 Relaxed 取舍」：SeqCst 仅用于两类不可替代位置——推进 head/tail 游标的 CAS 成功序，以及 is_empty/is_full/len 的双指针一致性快照；Relaxed 用于循环起点值与 fence 后的重读。模块三「成功序与失败序」：ArrayQueue 失败序用 Relaxed，SegQueue 的 index CAS 失败序须用 Acquire，因失败可能伴随新块安装。模块四「fence(SeqCst) 二次确认」：三处 fence 钉住「先读游标 A、后读游标 B」的程序顺序并入全局全序，使满/空判定可信。核心结论：验证内存序错误须靠 miri 多种子或 Loom，x86 强模型会掩盖 bug。
