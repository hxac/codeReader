`crossbeam-queue` 是 crossbeam-rs 下一个职责单一的 crate，只提供可在多线程间共享的并发队列，对外仅导出 `ArrayQueue` 与 `SegQueue` 两个类型，二者均为 MPMC（多生产者多消费者）且采用无锁（lock-free）实现。`ArrayQueue` 是有界队列，构造时一次性预分配固定容量缓冲区，满了则 push 返回 false，另可通过 `force_push` 覆盖最旧元素充当环形缓冲，因运行期不再分配内存而略快；`SegQueue` 是无界队列，以分段链表（每段最多 31 个元素）实现，按需动态分配新分段，因此略慢但能吸收突发流量。crate 支持 `no_std`（依赖 alloc），导出受 `feature = "alloc"` 与 `target_has_atomic = "ptr"` 双重守卫约束。版本演进中的关键里程碑（0.2.2 引入 `MaybeUninit` 修复不健全性、0.3.0 移除错误枚举改返回 `Option`、0.3.5 增设 `force_push`、0.3.12 修复大元素栈溢出）为后续单元的算法、unsafe 安全性与 no_std 章节埋下伏笔。选型经验：需控内存、求低延迟选有界 `ArrayQueue`；不可丢数据、需应对突发流量选无界 `SegQueue`。
