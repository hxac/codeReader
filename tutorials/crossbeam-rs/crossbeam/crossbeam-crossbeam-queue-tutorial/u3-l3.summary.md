本讲聚焦 SegQueue 的 Block 内存回收与初始化，覆盖四个关联函数：`new`、`wait_next`、`destroy`、`destroy_mut`。核心结论：当某块的 31 个槽消费殆尽后，crossbeam 用 **DESTROY 位交接协议**安全回收——由读到末尾槽（槽 30）的消费者发起 `destroy`，通过 `fetch_or(DESTROY)` 与消费者的 `fetch_or(READ)` 两次原子操作做确定性「交接」，保证恰好一个线程完成 `Box::from_raw` 释放，既不重复释放也不泄漏。关键不变量：销毁首次发起者必是读到槽 30 的消费者，故循环刻意排除槽 30。引入的关键术语包括 DESTROY 协调协议、交接（handoff）、TOCTOU 竞态。`Block::new` 借 crate 私有的 `Global::allocate_zeroed` 直接在堆上分配全零内存，合法依据是 `AtomicPtr`(null)、`AtomicUsize`(0)、`MaybeUninit`(全零) 三类字段均允许零初值，同时避免大元素栈溢出；分配失败走 `handle_alloc_error` 终止进程而不 unwind。`wait_write`/`wait_next` 是带 `Backoff::snooze` 退让的自旋，分别等 WRITE 位与 next 指针，均以 Acquire 与生产者 Release 配对。`destroy_mut` 是独占引用下的快速释放特化版，省去全部协调。
