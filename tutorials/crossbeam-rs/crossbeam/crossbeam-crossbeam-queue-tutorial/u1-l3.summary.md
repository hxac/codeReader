本讲是「纯动手」快速上手讲，目标是把 crossbeam-queue 装得上、跑得动、写得出，不涉及无锁算法。覆盖三个最小模块：依赖与版本、测试套件、第一个并发程序。核心结论：在 `Cargo.toml` 写 `crossbeam-queue = "0.3"` 会按插入符要求解析到最新 `0.3.12`，MSRV 为 1.60；在仓库内用 `cargo test -p crossbeam-queue` 跑测试，可用名字子串、`--test <file>`、`--doc` 控制范围，测试专用依赖 `fastrand` 仅在 `[dev-dependencies]`。通过精读 `tests/` 下的冒烟测试建立 API 心智模型：`ArrayQueue::new(cap)` 有界，`push` 返回 `Result<(), T>`（满则 `Err(value)` 还回原值），独有 `capacity`/`is_full`/`force_push`；`SegQueue::new()` 无界，`push` 返回 `()` 永不失败；两者 `pop` 均返回 `Option<T>`，空则 `None`。引入的关键术语：MPMC/SPSC、冒烟测试、背压、生产者/消费者重试循环、`std::thread::scope`（子线程借用队列并在作用域结束前自动 join）。综合实践把 SPSC 升级为 MPMC，因多生产者不保全局序而改用计数向量校验「每个值恰好被消费一次」。
