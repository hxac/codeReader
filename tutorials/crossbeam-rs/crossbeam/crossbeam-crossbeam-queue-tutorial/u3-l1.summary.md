本讲只讲数据结构，不讲 push/pop。SegQueue 是单向块链表：每个 Block<T> 经 next 原子指针串联，内含 BLOCK_CAP=31 个 Slot<T>；Slot 由 UnsafeCell<MaybeUninit<T>>（值）与 AtomicUsize（状态）构成，状态用 WRITE=1、READ=2、DESTROY=4 三个正交比特位表达，可叠加。Block::new 以 allocate_zeroed 一次性零初始化整块，因 next、state、MaybeUninit 均容忍全 0。常量 LAP=32、BLOCK_CAP=31、SHIFT=1、HAS_NEXT=1 将全局 index 编码为：bit0 为 HAS_NEXT 元数据位、bit1–5 为块内偏移、bit6 起为块号 lap；其中 offset==31 作块边界缝，故每块真实槽正好 31 个，前进步长为 2。外壳 SegQueue<T> 持 head/tail 两个 CachePadded<Position<T>>（Position = index 加当前块裸指针缓存）与一个 PhantomData<T>；new() 的空队列两个 block 皆 null，首块由首次 push 现分配。push/pop 主链路见 u3-l2，块销毁与 DESTROY 协调见 u3-l3。
