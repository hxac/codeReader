本讲讲解 crossbeam-channel 的两个工具模块。utils.rs 提供三项工具：一是非毒 Mutex，对 std Mutex 做薄封装，lock() 用 PoisonError::into_inner 忽略毒化，避免「他线程持锁 panic」传染成整通道瘫痪（前提：临界区只做不可失败的指针/原子操作）；二是 shuffle，用线程本地 Xorshift32 PRNG 加 Lemire 快速取模实现无偏 Fisher–Yates 洗牌，是 select!（非 biased 模式）公平性的直接来源，select_biased! 通过跳过它实现有偏；三是 sleep_until(Option<Instant>)，以一个函数统一表达「永久阻塞(None)」与「睡到时刻(Some)」，并用 loop 容忍虚假唤醒，服务于 at/never flavor 与 select 空操作兜底。alloc_helper.rs 是软链接到 crossbeam-utils 的最小 Global 分配器封装，作为 unstable alloc::alloc::Global 的 stable 替代，返回 NonNull<u8>，仅供 list flavor 的 Block 堆分配使用；它必须对零大小类型 ZST 特殊处理（返回对齐合法的悬空指针、释放时跳过 dealloc），并用 without_provenance_mut 构造无 provenance 指针。两模块均受 #[cfg(feature="std")] 门控。
