本讲深入 `src/flavors/array.rs`，讲解 `bounded(cap>0)` 通道的底层实现——基于 Vyukov 无锁有界 MPMC 队列。核心设计是为每个槽位配一个 `stamp`（`usize` 打包「圈数 lap + 断开位 mark_bit + 槽位下标 index」），解决游标转圈后下标重复的歧义问题；`head`/`tail` 游标与槽位 stamp 用同套位编码（`mark_bit=(cap+1)` 向上取整到 2 的幂、`one_lap=2*mark_bit`），可直接比较。关键术语：stamp、lap、mark_bit、`Slot<T>`（stamp 为原子版本号、msg 为 `UnsafeCell<MaybeUninit<T>>`）、`ArrayToken`（携带槽位指针与目标 stamp，在抢占与搬运两步间传递）。收发拆成「抢占游标（`start_send`/`start_recv` 用 CAS 推进）」与「搬运数据（`write`/`read`）」两步，拆分是为配合 select 的「先抢占再决定完成」。内存序是正确性基石：`write`/`read` 用 `Release` 写 stamp、对端用 `Acquire` 读 stamp，建立 happens-before，既保证消息可见又防止生产者踩踏正在被读的槽。阻塞版 `send`/`recv` 走「自旋→register→防丢失唤醒复查→park→唤醒后重试」，对端在 `write`/`read` 末尾调 `SyncWaker::notify` 形成生产消费闭环。断开用 `tail` 上单一 mark_bit 表达：发送侧断开保留消息供接收者排空，接收侧断开立即 `discard_all_messages` 丢弃残留消息并唤醒所有阻塞发送者。工程细节包括 `CachePadded` 防伪共享、`Backoff` 自旋退避。
