本讲只覆盖两个文件：`src/channel.rs`（定义 `Sender`/`Receiver` 外壳与构造函数）和 `src/lib.rs`（文档示例与 `pub use` 导出），并点到 `src/flavors/mod.rs`。核心是 `crossbeam-channel` 作为 mpmc 通道（`Receiver` 可克隆、多接收者）的「怎么用」层。

引入的关键术语：通道、`Sender<T>`/`Receiver<T>`、阻塞、flavor（风味）、容量、会合（rendezvous）、背压（backpressure）。两个基础构造函数 `unbounded()` 和 `bounded(cap)` 均返回 `(Sender, Receiver)`，内部经 `counter::new` 计数化后塞进 `flavor` 字段。关键结论是「按容量选 flavor」的分流：`unbounded` 走 `flavors::list`（无界链表），`bounded(cap>0)` 走 `flavors::array`（预分配数组），`bounded(0)` 走 `flavors::zero`，由 `SenderFlavor`/`ReceiverFlavor` 枚举粘合。

`send`/`recv` 是阻塞式 API，源码呈现贯穿全文件的架构母题：「match flavor 转发 + 错误归一化」（底层复用带超时实现，`send` 用 `unreachable!()` 兜底不可能的 `Timeout`）。容量决定阻塞行为：无界不因满阻塞、`capacity()` 返回 `None`；有界提供背压；零容量不存消息，发送与接收必须同时在场（会合），单线程直接 `send` 会死锁，需多线程或 `select!`。后续讲义将承接 API 全貌（u1-l3）与 flavor 内部实现（u2）。
