本讲义覆盖 `crossbeam-channel` 的 `select!` / `select_biased!` 宏（`src/select_macro.rs`），从使用者视角讲解如何把多个通道操作（`recv` / `send`）组合在一个块里，等待其中任意一个「就绪」并执行它，其余操作不会真正执行。分支有三段式语法：`recv(r) -> msg` 绑定 `Result<T, RecvError>`，`send(s, v) -> res` 绑定 `Result<(), SendError<T>>`（失败时消息原样返还）。

三种分支形态对应单条收发 API 的三种阻塞模式：无 `default`（阻塞）、`default`（非阻塞）、`default(timeout)`（限时），底层分别调 `internal::select` / `try_select` / `select_timeout`，本质是同一个 `run_select` 配 `Timeout::Never/Now/At`。单 `recv` 分支会被宏优化为直接的 `recv()` / `try_recv()` / `recv_timeout()`，不走 `Select` 调度。

关键术语：**就绪**指不需阻塞即可完成（即使返回错误、通道断开也算就绪）；**公平/无偏**指 `select!` 在多个同时就绪的操作中随机选一个（通过仲裁前 `utils::shuffle` 打乱尝试顺序实现），**有偏**指 `select_biased!` 按列表顺序优先选靠前者。两个宏是同一套实现，唯一区别是 `_IS_BIASED` 布尔开关，共同委托内部宏 `crossbeam_channel_internal!`。本讲不展开宏的逐条规则（u3-l3）和 `Select` 动态 API 调度（u2-l10、u3-l1）。
