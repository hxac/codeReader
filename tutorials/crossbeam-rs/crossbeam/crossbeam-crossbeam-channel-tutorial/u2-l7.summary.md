本讲深入 `src/flavors/zero.rs`，拆解 `bounded(0)` 会合（rendezvous）通道的实现。核心结论：会合通道容量为 0、不缓冲任何消息，发送与接收必须「同时在场」，靠一次性 `Packet` 完成单次交接。最小模块包括：`Packet`（含 `on_stack`、`ready`、`msg` 三字段）、`Mutex<Inner>`（装两个裸 `Waker` 与 `is_disconnected` 标志）、配对枢纽 `Waker::try_select`、抢占搬运两步 `start_send/start_recv` 与 `write/read`、阻塞三段式 `register→notify→wait_until`。关键术语与概念：`ready` 是双向复用信号灯，由谁带消息进 packet 决定方向；`try_select` 只挑别的线程的 entry，一次完成 CAS 选中、`store_packet` 递包、`unpark` 唤醒，保证至多一个赢家；`token.zero.0` 用 null 表示断开。最重要的设计分野是栈 packet 与堆 packet：普通阻塞 send/recv 创建与使用在同一栈帧用栈 packet 零分配，select 路径因 register 与 accept 分离且需支持 cancel，必须用堆 packet。所有内存序靠 `Release`/`Acquire` 配对保证可见性与销毁安全。
