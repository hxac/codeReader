本讲建立 crossbeam-channel 的「架构地图」，含三个最小模块：六种 flavor 总览与构造函数分流、公共类型壳与「按 flavor 分发」、lib.rs 模块组织。

核心结论：该 crate 是「一套统一 API + 六种可替换 flavor」——array（有界数组）、list（无界链表）、zero（零容量会合）、at（定时）、tick（周期）、never（永不投递）。构造函数按容量分流：unbounded→list，bounded(cap>0)→array，bounded(0)→zero，after/at→at（after 溢出退化为 never），tick→tick，never→never。

关键术语：flavor（风味）、公共类型壳（Sender/Receiver 仅持一个 flavor 枚举字段，所有方法 match 后转发，即「按 flavor 分发」母题）、SenderFlavor（仅 3 变体，因 at/tick/never 无发送方）与 ReceiverFlavor（6 变体）、transmute_copy（At/Tick 把 Instant 换装为泛型 T，由构造函数钉死 T==Instant 而安全）。三种真实通道经 counter::new 引用计数、可克隆可 disconnect；at/tick 用 Arc、never 零大小，永不 disconnect。lib.rs 以 no_std + cfg(feature="std") 门控全部模块，末尾用 pub use 集中导出对外 API，#[doc(hidden)] 的 internal 模块是 select! 宏的后门。
