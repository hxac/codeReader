`crossbeam-channel` 是面向 mpmc（多生产者多消费者）消息传递的通道库，定位为标准库 `std::sync::mpsc` 的功能更强、性能更好的替代品，核心区别在于其 `Receiver` 可自由克隆、多个线程可同时接收；当前版本 `0.5.15`，MSRV 为 Rust 1.74。本讲作为整本手册起点，只覆盖项目「地图」与运行方式两件事，不涉及内部实现。关键术语：mpsc/mpmc、channel、`no_std`、flavor（风味）、feature 门控。crate 顶部声明 `#![no_std]`，但通过 `std` feature 门控几乎所有模块，「禁用 std 暂不支持」。`src/` 由 `channel`（对外 `Sender`/`Receiver` 壳）、`context`（线程阻塞唤醒上下文）、`counter`（引用计数）、`err`（错误类型）、`flavors`（array/list/zero/at/tick/never 六种通道实现）、`select`（动态 `Select`/`SelectHandle` 调度）、`select_macro`（`select!`/`select_biased!` 宏）、`utils`（非毒 `Mutex`、shuffle、`sleep_until`）、`waker`（阻塞者队列）、`alloc_helper`（分配器封装，软链接）组成。对外 API 经 `src/lib.rs` 末尾 `pub use` 集中导出：核心类型 `Sender`/`Receiver` 及迭代器、6 个构造函数 `unbounded/bounded/after/at/tick/never`、10 个错误类型、`Select`/`SelectedOperation`，以及 `#[macro_export]` 的两个宏；另有 `#[doc(hidden)]` 的 `internal` 模块作为宏展开后门。运行走 Cargo 标准流程：`cargo build`/`cargo test`，基准分 `benches/`（criterion 微基准）与 `benchmarks/`（与 Go/flume/std 等横向对比）两套。
