本讲只读 `src/base.rs`，是「读路径 → 删除标记 → 插入」三连的最后一块，覆盖 `random_height` 与 `insert_internal` 两个核心符号。`random_height` 用 xorshift 推进种子，再以 `trailing_zeros+1` 把均匀随机数映射成服从 p=1/2 几何分布的塔高（期望高度 2），撑起跳表 O(log n) 的期望复杂度，并用 CAS 维护只升不降的 `max_height` 起点。`insert_internal` 采用两阶段策略：阶段 A 用一次 level 0 CAS 挂入新节点（即插入的线性化点），阶段 B 自底向上补齐 `1..height` 层链接——高层只是可选的加速索引，任意中断都不破坏正确性。新节点初始引用计数为 2（level 0 链接 + 返回的 entry），塔每长一层 +1、每摘一层 -1，归零才回收。命中旧 key 时由 `replace` 闭包决定是否 `mark_tower` 旧节点并 `len-1` 实现净替换；`insert`/`compare_insert`/`get_or_insert`/`get_or_insert_with` 四者只是给 `insert_internal` 传不同 `replace` 与 `value` 参数的薄包装。阶段 B 靠 `search_bound` 与 `help_unlink` 协作式清理应对建塔中途被并发删除，是无锁活性的体现。
