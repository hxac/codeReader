本讲以 `crossbeam-skiplist` 的 `src/set.rs` 与 `comparator.rs` 为核心，讲解 `SkipSet` 如何零开销复用 `SkipMap` 并接入自定义排序。`SkipSet<T, C>` 本质是 `SkipMap<T, (), C>` 的一层 newtype：占位值 `()` 是零大小类型，节点不占内存，set 的全部方法都是单行委托，无独立并发逻辑；其 bounds 也直接继承 map 层——读操作需 `C: Comparator<T>`、写操作额外要 `T: Send + 'static`。`set::Entry` 通过 `value()` 内部调用 `self.inner.key()`、丢弃 `()`，使「set 的值即 map 的键」，并借 `Deref<Target = T>` 让 `*entry` 等价于取值；`FromIterator` 用 `get_or_insert` 幂等收集，`IntoIter` 消费时丢弃 unit。自定义排序经 `SkipSet::with_comparator` 注入：`Comparator<L, R>` 是 `Equivalator<L, R>` 的子 trait，默认实现 `BasicComparator`（ZST）行为同 `BTreeSet`。硬性契约要求 `compare` 与 `equivalent` 对相等判定必须一致，否则两步法查询会错位；最稳妥写法是让 `equivalent` 复用 `compare().is_eq()`，从结构上保证一致。
