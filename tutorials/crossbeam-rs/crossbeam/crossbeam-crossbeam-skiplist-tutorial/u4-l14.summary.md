本讲聚焦 `src/map.rs`，揭示 `SkipMap` 如何把底层 `base::SkipList`（要求显式传 `Collector` 与 `Guard`）封装成用户完全看不见 `Guard` 的高层 API。核心有四：一是 `SkipMap` 为 `base::SkipList` 的 newtype 包装，构造时固定用 `epoch::default_collector().clone()`，使方法内随手 `epoch::pin()` 拿到的 `Guard` 与 `inner.collector` 天然同源，`check_guard` 恒成立；二是统一套路——几乎每个公共方法首行都 `let guard = &epoch::pin();`，保护方法期间解引用指针的安全性，方法返回前丢弃；三是读操作（`front`/`get`/`lower_bound` 等）需经 `try_pin_loop` 把 guard-bound 的 `base::Entry` 经 `Entry::pin`/`try_increment` 升级为 refcount 的 `RefEntry`，pin 失败（节点被并发删）则重搜，写操作返回的 `RefEntry` 自带计数无需 loop；四是 `map::Entry = ManuallyDrop<base::RefEntry>`，`Drop` 借 `ManuallyDrop + ptr::read` 搬出 `RefEntry` 调 `release_with_pin(epoch::pin)`，实现「按需 pin」——仅当计数由 1 降至 0 时才真正 pin 并 `defer_unchecked(finalize)`，把 pin 摊销到最后一次 drop。本讲为 u4-l15（`SkipSet` 封装）打底。
