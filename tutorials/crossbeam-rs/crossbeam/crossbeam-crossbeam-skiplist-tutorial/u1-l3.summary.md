本讲梳理 crossbeam-skiplist 的工程骨架。crate 顶部声明 `#![no_std]`，靠 `extern crate` + Cargo feature 按 `core`→`alloc`→`std` 三档逐层引入标准库；feature 是级联关系：`default = ["std"]`，`std` 隐含 `alloc` 并把 feature 传递给 `crossbeam-epoch`/`crossbeam-utils`，依赖声明用 `default-features = false` 以保持 `no_std` 友好，纯 `core` 档暂不支持。三档之外还引入两个关键门控概念：`cfg` 条件编译与内置的 `target_has_atomic = "ptr"`，后者保证目标平台支持原子指针，是 `crossbeam-epoch` 回收机制的前提。六个子模块按门控分三类：`base`、`alloc_helper`（私有）挂 `alloc + target_has_atomic`；`map`、`set` 挂 `std`；`comparator`、`equivalent` 无门控、随时可见，依赖方向严格自底向上无环。类型上呈三层包装：`base::SkipList<K,V,C>` 是底层无锁算法原语，接受外部传入的 `Collector`，能在 `alloc` 档运行；`SkipMap` 内嵌 `base::SkipList` 并用 `epoch::default_collector()` 做人体工学封装，因而必须 `std`；`SkipSet` 是 `SkipMap<T,()>` 的零开销特化。MSRV 为 1.74。`alloc_helper::Global` 是手写分配器，替代未稳定的 `alloc::alloc::Global`。
