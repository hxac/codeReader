本讲是专家层首篇，回答 crossbeam-skiplist 能在多「裸」环境下运行。它将能力切成 core → alloc → std 三档，靠 Cargo feature 与 cfg 门控实现：crate 顶部声明 `#![no_std]`，按档 `extern crate alloc`/`std`；核心门是 `#[cfg(all(feature = "alloc", target_has_atomic = "ptr"))]`，map/set 再叠加 `feature = "std"`，而 comparator/equivalent 无门控。核心结论：`base::SkipList` 把 `Collector` 当构造参数、不需全局状态，故能在 no_std + alloc 跑；SkipMap/SkipSet 因调用 `epoch::default_collector()`（基于 thread_local，属 std）而必须依赖 std。关键模块是手写的 `alloc_helper::Global`：因 `alloc::alloc::Global`、`Layout::dangling`、`ptr::without_provenance_mut` 均未稳定（MSRV=1.74），它封装 `alloc/dealloc` 并处理 ZST，其中 `without_provenance_mut` 在 Miri 与非 Miri 下分叉实现。MSRV 为 1.74。下一讲 u5-l17 将系统分析 base.rs 的内存序。
