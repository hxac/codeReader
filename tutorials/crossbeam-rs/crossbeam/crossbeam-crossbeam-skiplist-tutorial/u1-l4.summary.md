本讲是入门单元收尾，只读工程脚手架与基准骨架，不涉及无锁算法。核心模块有三：一是构建约束——crate 的 MSRV 为 1.74，`cargo build`/`cargo test` 用 stable 即可，`default = ["std"]` 会级联打开 `alloc` 及依赖的 `std` feature；二是测试与基准组织——测试依赖写在 `[dev-dependencies]`（仅 `fastrand`），不污染下游，`benches/` 目录靠 Cargo 自动发现并套用默认 libtest harness，无需 `[[bench]]` 表；三是基准骨架——`cargo bench` 必须 nightly，因基准首行 `#![feature(test)]` 解锁 `#[bench]` 宏与 `black_box`，并用线性同余式确定性伪随机 `num.wrapping_mul(17).wrapping_add(255)` 生成 key 以保证公平对比。引入的关键术语包括 MSRV、nightly 工具链、dev-dependencies、隐式 harness（自动发现）、`black_box` 优化屏障、确定性 PRNG、同构基准（控制变量）。本讲还点出 base 层 `SkipList` 需显式传 `Guard`、高层 `SkipMap` 把 `Guard` 藏进方法这一分层差异。核心选型结论：单线程裸速度通常 `HashMap` > `BTreeMap` > `SkipMap`，跳表的真正价值在于「并发且有序」——lock-free，多线程吞吐高。本讲反复出现的 `epoch::pin()`、`Guard`、`release` 将在下一单元的内存布局与 epoch 回收讲义中解释。
