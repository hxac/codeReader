本讲讲解 crossbeam-skiplist `base.rs` 读路径的搜索算法。三个核心函数共享「自顶下降、水平推进」骨架：`search_position` 为 `insert`/`remove` 收集每层左右邻接点，封装于 `Position`（`found` 命中即 `right[0]`）；`search_bound` 用 `bound:Bound` 与 `upper_bound:bool` 两个开关统一 `get`/`front`/`back`/`lower_bound`/`upper_bound`，`false` 找第一个 ≥ bound（满足即 break），`true` 找最后一个 ≤ bound（不满足才 break、边走边记 result），`get` 再用 `equivalent` 二次确认；`next_node` 仅在 level 0 走到下一个活节点，失败时回退调用 `search_bound` 充当重启。三者复用 `help_unlink` 协作清理：发现 `succ.tag()==1` 便 CAS 把 pred→curr 改为 pred→succ.with_tag(0)，成功则 `curr.decrement` 触发 epoch 延迟回收。关键直觉是 Harris/Michael 逻辑删除——tag 打在指针上却描述源头节点：`curr.tag()==1` 表示立足的 pred 已死，须整体 `continue 'search` 重启，这是 lock-free 活性来源。`above_lower_bound`/`below_upper_bound` 两真值表把 `Bound` 译成 `compare` 的 `is_ge/is_gt/is_le/is_lt`（Included 带等号、Excluded 不带、Unbounded 恒真）。`mark_tower` 实现与插入/删除主链路留待后续讲义。
