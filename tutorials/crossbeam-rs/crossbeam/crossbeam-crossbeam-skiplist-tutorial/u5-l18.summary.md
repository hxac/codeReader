本讲义不再引入新算法，而厘清 crossbeam-skiplist 的两条承诺。其一是并发语义：严格区分**数据竞争**（违反内存安全的 UB，跳表靠 `&self`+原子操作且不暴露 `get_mut` 杜绝）与**竞态**（跨调用的逻辑错误，由用户负责），确立「单操作原子、多操作非原子」契约，删除的**线性化点**是 level 0 指针 tag 由 0 翻 1。其二是 unsafe 的末尾拼图：`is_removed` 仅用 `Relaxed` 读 level 0 tag，是尽力而为的观测，删除安全性依赖 `mark_tower` 的原子翻转而非它；`Drop for SkipList` 与 `IntoIter` 敢用 `epoch::unprotected()` 的前提是**独占访问**（`&mut self`/move `self`），安全源于该前置条件本身；`IntoIter` 用 `ptr::read` 搬走键值后只 `dealloc`，`Drop` 则 `finalize` 析构剩余节点，并以 `next.tag()==0` 跳过死节点。
