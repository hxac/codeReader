本讲承接项目全局观，聚焦 crossbeam-skiplist 的「怎么用」，覆盖三块最小模块与一个小基准，不涉及底层无锁算法。核心类型有二：`SkipMap<K, V>`（有序并发 map，对标 `BTreeMap`）与 `SkipSet<T>`（有序并发 set，对标 `BTreeSet`，内部复用 `SkipMap<T, ()>`）。两者的核心方法是 `insert / get / remove / contains(_key) / iter`，且都按 key 升序（字符串即字典序）遍历。

本讲确立三条关键约定，供后续讲义承接：其一，所有修改方法接收 `&self` 而非 `&mut self`，故声明变量无需 `mut`，多线程可同时调用；其二，`get` / `insert` / `front` / `back` 返回的不是值本身，而是 `Entry<'a, K, V>` 句柄，需再调 `.key()` / `.value()`（set 的 `value()` 即元素本身）才能取数据，且二者返回借用 `&K` / `&V` 而非拷贝；其三，「返回句柄」的本质是用引用计数替代对容器的借用，从而在无锁并发下避免 use-after-free——只要句柄活着，对应节点就不会被回收，即便该节点已被逻辑删除（`is_removed()` 为真）。`Entry` 可 `Clone`，`clone` 增计数、`drop` 减计数，归零后由 epoch 延迟回收。此外，`insert` 对已存在 key 采用「先删旧再插新」的替换语义，单操作原子但多操作非原子。

讲义还引入 `examples/simple.rs`：一个被注释的 100 万条插入与遍历计时基准，用伪随机递推生成 key，作为综合实践与下一讲基准对比的素材。
