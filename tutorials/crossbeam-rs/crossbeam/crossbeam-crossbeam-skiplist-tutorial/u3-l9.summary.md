本讲以 src/base.rs 为唯一模块，讲解 crossbeam-skiplist 的 Harris/Michael 风格无锁删除。删除必须分两步：逻辑删除（mark）在节点所有出边指针上 fetch_or(1) 打 tag 宣告已死，物理摘除（unlink）再由删除者或读线程 CAS 摘除，最后 decrement 触发 epoch 延迟回收。关键设计是 tag 打在「出边」却描述「源头节点」的死活，故读路径区分两类——curr.tag()==1 表示立足的前驱已死需整体重启搜索，succ.tag()==1 表示当前节点已死由 help_unlink 协助摘除。mark_tower 自顶向下逐层标记，以 level 0 的 0→1 翻转为删除的线性化点，恰有一个线程赢得删除权；is_removed 只看 level 0 且用 Relaxed，是尽力而为的观测，删除安全性不依赖它。SkipList::remove 为两步协议：先 try_acquire 抢引用计数保证返回节点不被回收，再 mark_tower 抢删除权，输了须 decrement 退计数以免泄漏。协作式清理（能摘就摘，摘不干净交给读路径兜底）保证 lock-free 活性。
