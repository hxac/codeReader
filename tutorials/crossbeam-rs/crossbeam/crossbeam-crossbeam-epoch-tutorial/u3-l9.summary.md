本讲聚焦 `Guard` 类型。它的全部状态只是一根 `*const Local` 裸指针，是「线程已 pin」的轻量凭证，不存 epoch 等信息；真正的 pin 状态住在 `Local`（含 `guard_count`、`handle_count`、`epoch` 等字段）。`Guard::drop` 把指针解引用为 `&Local` 调用 `unpin`，若指针为 null 则是 no-op。

pin 可重入：首个 guard 才真正 pin（写 local epoch + 屏障），其余 guard 仅增 `guard_count`，最后一个 guard 才真正 unpin；`is_pinned()` 即 `guard_count > 0`。`Guard` 因含裸指针而 `!Send + !Sync`，故 `guard_count` 用 `Cell` 而非原子。

`epoch::unprotected()` 返回 `local` 为 null 的 `&'static Guard`：不 pin、`defer` 立即执行、`flush` 为 no-op、`collector()` 返回 `None`，专用于无并发的构造/析构；它借 `GuardWrapper` 加 `unsafe impl Sync` 存入 static。本讲不展开 epoch 推进、屏障细节与回收主链路。
