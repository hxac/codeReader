本讲梳理 crossbeam-epoch 的代码物理形态与编译期裁剪机制。仓库由 `Cargo.toml`、`build.rs`、`src/lib.rs`（模块总装车间）、`src/sync/`（list、queue 子模块）与 tests/benches/examples 构成；atomic、collector、guard 等实现留待后续讲义。

引入三大特性开关：`default` 含 `std`，`std` 隐含 `alloc`，`alloc` 为标记位，`loom` 仅供并发测试、不受 semver 保障。核心结论：同时关 `std` 与 `alloc` 会使核心模块尽数裁剪、crate 变空壳，故不支持。

关键概念：核心模块统一受 `cfg(all(feature="alloc", target_has_atomic="ptr"))` 守卫，`pin`/`default_collector`/`is_pinned` 额外需 `std`；`lib.rs` 顶部以 `primitive` 抽象层屏蔽 loom 与真实环境差异，并为标准库 `UnsafeCell` 包一层对齐 loom API 的 wrapper；`build.rs` 读 `CARGO_CFG_SANITIZE` 探测线程消毒器，按需注入非公共的 `crossbeam_sanitize_thread` cfg。
