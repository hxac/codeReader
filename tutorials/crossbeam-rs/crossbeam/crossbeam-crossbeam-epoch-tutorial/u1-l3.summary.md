本讲义是 crossbeam-epoch 的快速上手篇，只讲「怎么用」三块最小模块：默认收集器的 `epoch::pin()`、`Atomic` 指针的 `load`/`swap`、以及 `Guard::defer_destroy` 延迟回收，串成「分配→读取→替换→延迟回收」的 EBR 最小闭环。

引入的关键术语与概念：**默认收集器**（进程级单例 `Collector` + 线程局部 `LocalHandle`）、**`Guard`**（pin 的 RAII 凭证，drop 即 unpin，且 pin 可重入）、三类指针 **`Atomic<T>`**（共享原子指针）、**`Owned<T>`**（独占所有权，类 `Box`）、**`Shared<'g, T>`**（借来的带 tag 指针，生命周期 `'g` 与 guard 绑定，由编译器防止 guard drop 后误用）。

核心结论：`load`/`swap` 强制要求传 `&Guard`；解引用 `Shared` 需 `unsafe`，且多线程下必须用 `Release`/`Acquire` 或 `SeqCst`，`Relaxed` 会构成数据竞争；`defer_destroy` 是 `unsafe` 的，契约为「对象已不再被其他线程触达」，闭包在宽限期（全局 epoch 前进 ≥ 2）后才执行；`Atomic` 无 `Drop`，当前对象须 `defer_destroy` 或 `into_owned()` 回收；`flush` 仅加速回收、不影响最终保证。本讲刻意回避 epoch 内部推进与内存屏障细节，留待 u3/u4/u5 单元展开。
