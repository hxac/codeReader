本讲只覆盖 `src/epoch.rs` 这一个文件，即 EBR 的「数据表示层」。核心结论：`Epoch` 把「代号」与「是否被 pin」压进一个机器字——最低位（LSB）作 pin 标志、高位作代号，未 pin 第 g 代为偶数 `2g`、已 pin 为奇数 `2g+1`。引入的关键概念包括：`starting()`（默认 0）、`pinned()`（`data|1`，幂等）、`unpinned()`（`data&!1`）、`is_pinned()`（读 LSB）、`successor()`（`wrapping_add(2)`，`+2` 而非 `+1` 是为不触碰 LSB，使 pin 标志跨代保留）。`wrapping_sub` 通过「抹掉 rhs 的 LSB → 回绕相减 → 按位解释为有符号 → 算术右移一位」算出带符号代差，支撑 `SealedBag::is_expired` 的「`global_epoch.wrapping_sub(sealed) >= 2`」宽限期判据。约定：全局 epoch 永远存未 pin（偶数），持 guard 期间本地 epoch 存已 pin（奇数），比较代号前须先 `unpinned()`。`AtomicEpoch` 是对底层原子整数的薄封装，仅提供 `load`/`store`/`compare_exchange`；通过条件编译，有 64 位原子时用 `AtomicU64`/`u64`/`i64`，否则退回 `AtomicUsize`/`usize`/`isize`，三别名须同步切换。
