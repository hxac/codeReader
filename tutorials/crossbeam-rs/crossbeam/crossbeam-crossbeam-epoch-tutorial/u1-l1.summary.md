本讲是 crossbeam-epoch 学习手册的开篇，覆盖三个最小模块：EBR 要解决的问题、epoch 与 grace period 的延迟回收直觉、crate 级文档导读。核心结论是：在无锁数据结构里，「逻辑上移除对象」与「物理上释放内存」必须分离——别的线程可能仍握着指向该对象的旧指针，立刻 `drop` 会导致悬垂指针与 use-after-free，因此必须延迟到确认无人再用时才回收。

引入并明确界定的关键术语包括：**EBR（epoch-based memory reclamation，基于纪元的内存回收）**、**全局 epoch（单调回绕递增的整数）**、**grace period（宽限期）**——精确定为全局 epoch 相对垃圾盖戳「前进 2 次」（`successor` 每次加 2），即在盖戳 epoch g 之后，全局前进满 2 次即可安全回收。此外还有 **pin/unpin**（线程访问共享数据前先钉住并快照当前 epoch、挂起新垃圾回收）、**Atomic**（共享原子指针）与 **Shared<'g,T>`（受 Guard 生命周期保护的读取结果）、**Guard::defer/defer_destroy**（延后销毁）。

定位结论：crate 核心是 `#![no_std]` 的，`no_std`+`alloc` 下除「默认收集器」外都可用，最便捷的 `pin()` 仅在 `std` 下提供（依赖线程局部存储）。pin 的不变量与 pin 屏障的严格实现细节留给后续单元承接。
