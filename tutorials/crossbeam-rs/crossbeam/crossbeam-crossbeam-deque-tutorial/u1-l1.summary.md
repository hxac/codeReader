`crossbeam-deque` 是一套源自 **Chase-Lev** 算法的无锁（lock-free）work-stealing 双端队列，主要服务于任务调度器。本讲从外部建立项目地图，不涉及无锁算法与内存序细节。核心是 work-stealing 拓扑与三种队列角色：**Worker**（每线程独占的本地队列，只能 `push`/`pop`，通过 `PhantomData<*mut ()>` 强制为 `!Send + !Sync`，只能单线程拥有）、**Stealer**（由 Worker 派生，可跨线程共享、可 Clone，供他人偷任务）、**Injector**（全局唯一、MPMC 的 FIFO 入口，用于外部注入任务）。调度循环为：本地 pop → Injector 批量偷 → 遍历 Stealer 偷 → 空闲休眠。偷操作有三种变体 `steal`/`steal_batch`/`steal_batch_and_pop`，且可能返回需重试的 `Steal::Retry`。项目元数据：版本 **0.8.6**、MSRV **1.74**、许可证 **MIT OR Apache-2.0**，依赖 `crossbeam-epoch`（epoch-based reclamation 内存回收）与 `crossbeam-utils`，`std` feature 默认开启且暂不可禁用。全部实现集中在单文件 `src/deque.rs`。API 骨架在 **0.7.0** 定型（`new_fifo`/`new_lifo` 取代旧 `fifo()`/`lifo()`、引入 `Injector`、`Steal::Success` 改名、`or_else`/`FromIterator` 组合子支撑 `find_task` 回退链），0.8.x 补充 `len()` 与 `_with_limit` 批量偷取，0.8.6 修复 Injector 大对象栈溢出。因安全公告 `GHSA-pqqp-xmhj-wgcw` 多版被 yank，写 `"0.8"` 会自动选到安全的 0.8.6。
