本讲为 `crossbeam-deque` 建立「源码地图与构建配置」导航，只盯 4 个外围文件，不深入算法（算法在 `src/deque.rs`，留待 u2/u4）。核心结论：crate 门面在 `src/lib.rs`——`#![no_std]` 声明加一段 crate 级文档，再以私有 `mod deque;` 挂载实现、用 `pub use` 仅把 `Worker / Stealer / Injector / Steal` 四个类型提升到 crate 根，实现细节藏在私有模块、封装干净。`no_std` 与 `std` feature 协作：`Cargo.toml` 中 `default = ["std"]`，开启时 `lib.rs` 显式 `extern crate alloc/std`，且所有 `mod` 与 `pub use` 都套 `#[cfg(feature = "std")]`，关掉 std 后 crate 退化为空壳，故「禁用 std 暂不支持」。首页的 `find_task` 文档示例既是 doctest 又浓缩 work-stealing 回退链：`local.pop → global.steal_batch_and_pop → stealers.steal`，引出 `Steal` 的 `or_else / is_retry / success` 组合子。`build.rs` 读 `CARGO_CFG_SANITIZE` 探测 ThreadSanitizer，命中 `thread` 时用 `cargo:rustc-cfg=crossbeam_sanitize_thread` 点亮条件编译开关（并用 `rustc-check-cfg` 消除警告），该开关点亮后代码如何变留到 u4-l3。`src/alloc_helper.rs` 是指向兄弟 crate `crossbeam-utils` 的符号链接，体现 workspace 内「单份源码、多 crate 共享」的复用技巧，仅于 std feature 下挂载。
