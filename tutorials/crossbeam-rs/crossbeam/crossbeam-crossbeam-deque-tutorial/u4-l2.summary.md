本讲专讲 crossbeam-deque 中无锁队列换 buffer 后旧内存的安全回收，最小模块为 `src/deque.rs`，核心依赖是 crossbeam-epoch（EBR，基于纪元的内存回收）。根本困境：`resize` 原子换掉共享 buffer 后不能立即 free，因为 stealer 可能正持有旧指针读槽位，会触发 use-after-free；加锁会毁掉无锁优势，故采用延迟回收。

本讲引入的关键术语：临界区（`epoch::pin()` 领 `Guard` 进出）、退役（retire）、退役闭包、最终存活 buffer。本 crate 只用 5 个 epoch API，对应三条互补释放路径：退役侧（`Worker::resize`）走 `pin` → `swap` 拿回旧 buffer → `defer_unchecked` 登记回收 → 大块（≥`FLUSH_THRESHOLD_BYTES=1024`）时 `flush` 尽早回收；保护侧（`Stealer::steal` 及 `steal_batch*`）用 `pin` 使加载到的指针在 guard 存活期不被释放，并用 `is_pinned()` 在重入时补发缺失的 `SeqCst fence`；绕过侧（`Inner::drop`）借 `&mut self`（`Arc` 计数归零、无并发）的独占保证，用 `epoch::unprotected()`（假手环）当场 `dealloc` 最终 buffer。结论：退役旧 buffer 走 epoch GC 延迟释放，析构时最终 buffer 走 drop 当场释放，两路径分工不重叠。承接 u2-l5、u4-l1，引出 u4-l3 的 tsan 兼容。
