本讲只讲 Injector 的数据结构与索引编码，把 push/steal 的并发算法当黑盒。核心结论：Injector 与 Worker/Stealer 实现截然不同——前者是 Block 单向链表加 head/tail 两个 Position，后者是单一固定容量环形 Buffer。引入的关键术语：索引编码常量 LAP(64)、BLOCK_CAP(63)、SHIFT(1)、HAS_NEXT(1)，把逻辑位置(>>SHIFT)、块内偏移(%LAP)、第几圈(/LAP)、是否有后继块(&HAS_NEXT)打包进一个 usize；每圈留 offset==63 作块边界哨兵，故每块实际可写 63 个任务。状态位 WRITE/READ/DESTROY 标记 Slot 生命周期，Slot 用 UnsafeCell<MaybeUninit<T>> 存任务，wait_write 以 Acquire 加载配对生产者 Release 置 WRITE 位并 Backoff 自旋。Block 提供 new(零分配整块)、wait_next(等 next 指针)、destroy(协作销毁避免 double-free)三方法。Position 缓存 (index, block) 二元组，Injector 借 PhantomData<T> 显式实现 Send+Sync 以支持跨线程共享，与单线程私有的 Worker 形成对比。
