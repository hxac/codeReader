本讲精读 `Injector::push` 的并发算法。push 采用「乐观锁预约槽位」：先 Acquire 读 `tail.index`，按 `offset = (index >> SHIFT) % LAP` 算块内偏移，再用 `compare_exchange_weak`（成功序 SeqCst、失败序 Acquire）把 index 推进一格，相邻成功 push 差值恒为 `1 << SHIFT = 2`，CAS 成功即独占该槽位，但任务体尚未写入。

`offset == BLOCK_CAP`（哨兵 63）标记本 block 已满、下一 block 正在安装，此时用 `Backoff::snooze` 长退避等待重读；CAS 失败则用更轻的 `Backoff::spin`。写最后一格前**预分配** `next_block`，使 CAS 成功后立即安装，最小化「哨兵窗口」。跨块安装经 `tail.block`、`tail.index`、`block.next` 三处 Release store 完成，把新 block 同时暴露给等待的 push 与偷取侧的 `Block::wait_next`，并跳过哨兵偏移落到下一 block 的 offset 0。

发布顺序：先以非原子写把任务体写入 `slot.task`，再 `slot.state.fetch_or(WRITE, Release)` 发布；偷取侧的 `Slot::wait_write`（Acquire）与之配对，建立 happens-before，杜绝读到未初始化内存。状态位 `WRITE/READ/DESTROY` 互不重叠，`READ`、`DESTROY` 留待 u3-l3 的 steal 与 Block 销毁。
