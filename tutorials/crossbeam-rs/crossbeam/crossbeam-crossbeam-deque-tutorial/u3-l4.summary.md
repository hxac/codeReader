本讲是 Injector 系列的收尾篇，覆盖批量偷取、长度查询、空判定与析构四个最小模块。批量偷取方法是「是否弹出一个任务 × 上限可否指定」的二维矩阵：`steal_batch` 与 `steal_batch_and_pop` 是转发壳（默认上限为 `MAX_BATCH` 与 `MAX_BATCH+1`），真实实现只有 `steal_batch_with_limit` 与 `steal_batch_with_limit_and_pop`，二者共享「加载 head → 算 advance → CAS 推进 → reserve → 拷贝 → 发布 → 销毁」骨架。关键概念是 `advance`（一次 CAS 认领的格数，遵循「偷约一半 `len.div_ceil(2)`」或「偷到块尾 `BLOCK_CAP - offset`」并受 `limit` 约束）与 `batch_size`（实际写入目的队列数，弹出版比 advance 少 1）。拷贝时源端按固定顺序读，FIFO 目的保持原序、LIFO 目的反转顺序；末尾用 Release fence 发布 back，并在到块尾或命中 DESTROY 位时触发 Block 的协作销毁。`len()` 用「读 tail → 读 head → 再读 tail 校验一致」的一致性循环避免撕裂读，并清 HAS_NEXT 位、修正哨兵位、按 lap 旋转，最终公式为 `tail - head - tail / LAP`。`is_empty` 只做 SeqCst 双加载判等无需重读；`Drop` 借 `&mut self` 用 `get_mut()` 跳过原子操作，遍历 head..tail 在真实槽位 `assume_init_drop`、在哨兵位释放 block，全程不走 epoch。
