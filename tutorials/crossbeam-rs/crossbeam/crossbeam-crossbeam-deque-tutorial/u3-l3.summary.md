本讲承接 Injector 的 Block/Slot 数据结构与 push 实现，转向消费者一侧讲解 Injector::steal 与 Block 销毁机制，分为四个最小模块：先 Acquire 读取 head 算出 offset，命中哨兵 63（下一 block 尚未安装）时用 snooze 重读；再用 HAS_NEXT 位懒缓存「head 与 tail 跨块」判定，仅在位为空时经 SeqCst fence 读 tail 判空，CAS 推进 head 失败则返回 Steal::Retry；读到块尾（offset 62）触发 wait_next 跨块切换，按「剥 HAS_NEXT、跳过哨兵、按需补回」算出 next_index；最后是核心的协作销毁——READ 位标记已读、DESTROY 位作为销毁责任令牌，Block::destroy 反向扫描并以双重 READ 检查交接接力棒，扫描到底才 drop，保证 block 恰好释放一次。
