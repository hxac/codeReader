本讲在 u7-l1 节点布局之上，讲解 crossbeam-skiplist 内核 `base::SkipList` 的四个最小模块。

导航上，`search_bound`（读）与 `search_position`（写，记录每层前驱 left[]、后继 right[]）共享「逐层下降 + 协作摘除」骨架：任何线程遍历中看到 tag=1 的标记节点，都经 `help_unlink` 顺手 CAS 摘除，失败则整体重来。插入分两阶段：level 0 CAS 必成（地基），高层 CAS 尽力补建（加速索引）；新节点初始引用计数为 2（Entry 句柄 + level 0 链接），每装一层 +1，体现「计数 = 层数 + 句柄数」。删除遵循 Harris/Michael 法：`mark_tower` 自顶向下打标记、以 level 0 tag 裁决唯一赢家，赢家逐层 unlink 并 `decrement`，输家仅 release 自己抢的计数。引用计数归零时不立即 free，而是经全文件唯二的 `defer_unchecked` 把 `Node::finalize` 推迟到 epoch 两代宽限期后执行，杜绝 use-after-free；独占路径（drop、panic 清理）则同步 `finalize`。`SkipMap`/`SkipSet` 把裸跳表包成「自动 `epoch::pin()`、返回引用计数 `Entry`」的友好接口。
