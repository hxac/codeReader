本讲讲解 `crossbeam_utils::CachePadded`，攻克并发中的「伪共享（false sharing）」：当多核频繁写**不同**却恰好落在**同一条缓存行**（通常 64 字节）上的原子变量时，MESI 缓存一致性协议会使该行在核间反复失效、乒乓弹射，吞吐因而崩塌；这与写同一变量的「真共享」不同，是可以消除的损耗。解药是「缓存行隔离」：把热点字段按缓存行对齐并填充，使其独占一条缓存行，邻居被挤到别的行去。

`CachePadded<T>` 靠编译期互斥的 `#[cfg_attr(target_arch=..., repr(align(N)))]` 选定对齐值 N——x86-64/aarch64/powerpc64 取 128（因 Intel 空间预取器成对抓取 64 字节行，悲观按 128 算），arm/mips 取 32，m68k 取 16，s390x 取 256，其余默认 64；对齐为 N 的类型其大小自动撑成 N 的倍数，padding 由布局隐式产生，结构体本身仅一个 `value: T`。它实现 `Deref`/`DerefMut` 透明访问，`Send`/`Sync`/`Clone` 忠实透传给 T。使用准则：只给被多核高频原子改写的少数控制字段（如队列头尾索引、全局计数器）加 padding，只读、线程私有、海量小元素都不该加。该原语是后续 AtomicCell 全局序列锁隔离、ArrayQueue 头尾索引隔离的物理前提。
