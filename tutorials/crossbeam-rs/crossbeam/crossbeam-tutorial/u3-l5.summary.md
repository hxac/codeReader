本讲深入 `crossbeam-channel` 的 `unbounded()` 通道底层实现 `flavors/list.rs`，核心是「分块链表 + 按需扩容 + 协作回收」。其存储结构为分块链表：每个堆上的 `Block` 容纳 `BLOCK_CAP=31` 条消息，块间用 `next` 指针串联，由 `head`/`tail` 两个 `CachePadded<Position>` 驱动收发。索引编码进单个 `usize`：最低位为 `MARK_BIT`（head 端表「还有下一块」、tail 端表「已断开」），其余位按 `LAP=32` 切成「圈+偏移」，使取模除法退化为位运算；保留 `LAP-1=31` 个槽是为了用 offset==31 作块边界哨兵。发送走两阶段 `start_send`/`write`：前者用 CAS 推进 `tail` 占槽并按需预分配、安装下一块实现无界扩容，后者写消息、置 `WRITE` 位、唤醒接收者；`ListToken{block, offset}` 在两阶段间传递占位信息。接收走对称的 `start_recv`/`read`，并通过 `DESTROY` 协议协作回收块——读完本块最后一槽者发起销毁，遇占用则留 `DESTROY` 拜托，最后离开者接力 `free`，保证每块恰好释放一次。销毁语义上，最后一个 receiver 离开时 `discard_all_messages` 即时丢弃全部消息释放整条链表以避免泄漏，而最后 sender 离开只置断开位唤醒接收者、不丢消息。与 array 的根本区别是：list 永不「满」（`is_full()` 恒 false、`capacity()` 返回 None），靠分配新块而非阻塞吸收发送。
