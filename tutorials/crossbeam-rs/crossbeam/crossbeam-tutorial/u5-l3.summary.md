本讲承接 u5-l2，进入 crossbeam-epoch 的使用入口，核心是 `Guard` 与延迟销毁机制。`Guard` 是线程处于 pin 状态的 RAII 凭证，唯一字段是指向参与者 `Local` 的裸指针，`Drop` 即 unpin；`null` 时为假守卫，操作退化为 no-op。可重入 pin 由线程局部的 `guard_count: Cell<usize>` 驱动，真正的「发布 epoch + `SeqCst` fence」只在 0→1 边界做一次，1→0 才真正 unpin。延迟销毁由三层 API 构成：安全的 `defer`（约束 `Send + 'static`）、`defer_destroy`、底层 `defer_unchecked`，后两者为 unsafe，因为「宽限期后对象已独占」是类型系统证不出的运行期不变量。闭包先进线程局部袋子（满 64 个盖章入全局队列），须等 `global_epoch - bag_epoch >= 2` 才由 `collect` 销毁。`Deferred` 用「函数指针 + 3 字缓冲」类型擦除任意 `FnOnce`，小闭包内联、大闭包装箱，最常见的「销毁一个 `Shared`」闭包正好内联。全局 `epoch::pin()` 走 `OnceLock` 单例收集器与线程局部句柄，`with_handle` 兜底保证即便在线程析构途中调用也永不 panic。
