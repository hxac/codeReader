本讲是 crossbeam-epoch 单元的入门篇，只精读 `lib.rs` 与 `default.rs`，回答「有何难题→epoch 如何解决→怎么用」三个问题。核心痛点是：无锁数据结构中删除方想释放节点时，读取方可能仍持有其指针，立即 free 会引发 use-after-free；GC 语言靠运行时兜底，Rust 需自建内存回收机制。epoch 方案分三阶段——标记（摘除对象连同当前全局 epoch 塞进垃圾袋 Bag）、延迟（等待推进）、销毁，关键安全判据是 `global_epoch - bag_epoch >= 2`，依据是「一个 pin 中的参与者最多见证一次 epoch 推进」；全局 epoch 仅在所有 pin 者都登记在当前代时才允许推进。公共 API 分三层：便捷层 `pin()`/`is_pinned()`/`default_collector()`（需 `std`，依赖 thread-local `LocalHandle` 与全局 `OnceLock` 收集器，`pin()` 可重入且永不 panic）、句柄层 `Collector`/`LocalHandle`、指针层 `Atomic`/`Owned`/`Shared`（需 `alloc` + 原子）。`pin()` 返回 RAII 凭证 `Guard`，`drop` 时 unpin；`Shared<'g,T>` 的生命周期 `'g` 绑定 `Guard`，从类型上约束读出指针不逃出本次 pin。`successor` 用 `wrapping_add(2)` 推进、`PINNINGS_BETWEEN_COLLECT=128` 等细节留待 u5-l2~u5-l5 展开。
