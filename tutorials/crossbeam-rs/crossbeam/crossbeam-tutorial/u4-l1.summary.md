本讲精读 `crossbeam-queue` 的 `ArrayQueue`——基于 Vyukov 算法、有界、MPMC、无锁的环形队列，容量构造时一次分配的 `Box<[Slot<T>]>`，较 `SegQueue` 略快，且可在 `no_std + alloc` 下使用。核心机制是**lap/index 编码**：把 `head`/`tail` 及每个 `Slot` 的 `stamp` 都打包进一个 `usize`，高位为代次 lap、低位为下标 index，`one_lap = (cap+1).next_power_of_two()` 使拆解只靠位运算；同一 index 在不同圈数值不同，从而**根除 ABA 问题**。每个 `Slot`（字段为 `AtomicUsize stamp` 与 `UnsafeCell<MaybeUninit<T>>`）的 stamp 是两态状态机：可写（`stamp == tail`）↔ 可读（`stamp == head+1`）。`push` 三步为 CAS 占 tail→写值→stamp 置 `tail+1`（Release）；`pop` 为 CAS 占 head→读值→stamp 重置为 `head+one_lap`（即下一圈才可写）；满判定为 `tail` 比 `head` 整整领先一圈，`force_push` 则挤掉最老元素。CAS 失败用 `Backoff::spin` 退避，状态迁移中用 `snooze` 让出。`head`/`tail` 各自 `CachePadded` 消除伪共享；因 `MaybeUninit` 不自动析构，`Drop` 需手动遍历存活槽位 `assume_init_drop()`。
