本讲剖析 crossbeam-utils 的 `thread::scope` 入口函数，它解决了 `std::thread::spawn` 闭包必须 `'static`、无法借用栈上局部变量的痛点。核心承诺是「scope 返回前一定 join 所有子线程」，故借用检查器允许子线程借用 `'env`（不变生命周期，由 `PhantomData<&'env mut &'env ()>` 实现）数据。`scope()` 执行分五段：`catch_unwind(f)` 捕获 `f` 的 panic → 创建局部 `AbortOnPanic` 守卫 → `drop(scope.wait_group)` 并 `wg.wait()` 等所有子 Scope 销毁 → join 残留线程收集 panic → `mem::forget(guard)` 后按 `match` 决定返回值。WaitGroup 充当「句柄列表已完整」的栅栏（含嵌套 spawn 的孙线程登记）；`AbortOnPanic` 在「已 spawn 未全 join」的危险窗口把任何 unwind 升级为 `abort()` 以防线程逃逸；多个子线程 panic 被汇总为 `Vec<Box<dyn Any + Send>>` 经 `Err` 返回。本讲仅覆盖 `scope()` 本身，`spawn`/`ScopedThreadBuilder`/`ScopedJoinHandle` 细节留至 u4-l2。
