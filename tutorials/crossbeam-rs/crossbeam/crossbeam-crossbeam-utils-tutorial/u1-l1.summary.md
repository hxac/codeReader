本讲是 crossbeam-utils 学习手册的开篇，只精读三个「门面」文件：README.md、Cargo.toml、src/lib.rs，建立全局认知，不涉及任何实现细节。核心结论：crossbeam-utils 是面向并发编程的「miscellaneous tools」工具集，各类型相对独立，公开类型分为三大类——Atomics（AtomicCell、AtomicConsume）、Thread synchronization（Parker、ShardedLock、WaitGroup）、Utilities（Backoff、CachePadded、scope）。元信息方面：版本 0.8.21、edition 2021、默认 MSRV 1.56；特性开关 default = ["std"]，atomic 为独立 feature（依赖 atomic-maybe-uninit，需 Rust 1.74）。关键设计取舍是 no_std 友好：lib.rs 用 `#![no_std]` 配合 `#[cfg(feature = ...)]` 精确门控模块可见性，使 AtomicCell/AtomicConsume/Backoff/CachePadded 可脱离 std 使用，而 Parker/ShardedLock/WaitGroup/scope 依赖 std。本讲引入的关键术语与概念包括：数据竞争、原子操作、no_std、MSRV、Feature 特性、feature 门控、false sharing、primitive 内部抽象层（loom 测试），以及「README 的 no_std 标记 ↔ lib.rs 的 cfg 门控」一一对应这把理解全 crate 可见性的钥匙。综合实践用 AtomicCell 写多线程计数器，为后续 thread::scope（u4-l1）埋下伏笔。
