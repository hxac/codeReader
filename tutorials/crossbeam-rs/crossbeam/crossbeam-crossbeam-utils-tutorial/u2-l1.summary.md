本讲是 atomic 模块首篇，只讲 AtomicCell 的对外契约，不涉及底层线程安全机制。其结构为 `#[repr(transparent)]` 包裹的 `UnsafeCell<MaybeUninit<T>>`：repr(transparent) 使其布局等同 T（是无锁路径的前提），UnsafeCell 提供内部可变性，MaybeUninit 把 drop 时机交给 AtomicCell 自身。它在 T: Send 时声明 Send + Sync（要求 Send 而非 Sync，因操作转移值而非共享引用），new/into_inner 均为 const fn。方法按 trait bound 分层：store/swap 无约束、take 要 Default、load 要 Copy、compare_exchange/fetch_update 要 Copy+Eq。内存序对外固定为 load=Acquire、store=Release、读改写=AcqRel，故不暴露 Ordering 参数。store 与 Drop 共用 needs_drop 分流：非 Copy 类型写入时经 swap 回收旧值、销毁时 drop_in_place 回收内部值，Copy 类型零开销跳过，MaybeUninit 防 double-drop。fetch_update 封装 CAS 循环，闭包可能多次调用但只生效一次，成功返回旧值保证等于 current。
