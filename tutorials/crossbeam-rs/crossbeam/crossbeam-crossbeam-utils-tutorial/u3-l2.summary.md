本讲专讲 `crossbeam-utils::sync::WaitGroup`——一个引用计数式同步原语，核心模块为 `src/sync/wait_group.rs`。其语义为：克隆即登记一个参与者，drop 或 wait 即注销，计数归零时 `notify_all` 唤醒所有等待者，典型用于 fork-join。内部状态 `Inner{cvar, lock, count}` 沿用上一讲 Parker 的「Condvar + Mutex + 原子计数」组合；`count` 与 `Arc` 引用计数数值相等但用途不同：前者需通过 `fetch_sub(Release)` 建立跨线程 happens-before 并据返回旧值做条件通知，故不能复用 `Arc`。`new()` 将 `count` 初始化为 1 以计入主线程自身；`wait` 用 `ManuallyDrop` + `ptr::read` 消费 self 避免重复减计数，慢路径用 `while` 循环复核 count 以防虚假唤醒。全讲难点是防丢失唤醒的关键顺序「先 `fetch_sub`、再 `lock`、最后 `notify_all`」：等待方在锁内决定入睡与归零方在锁内发通知互斥，与 Parker::unpark「先加锁再通知」是同一招。末尾对比 `std::sync::Barrier`：WaitGroup 动态登记、一次性、各线程自选是否阻塞；Barrier 需预知数量、可复用、全员到齐。本讲为后续 thread::scope 的直接前置。
