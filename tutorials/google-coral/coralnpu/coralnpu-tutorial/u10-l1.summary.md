本讲是 CoralNPU 软件层讲义，主题为 RVV（RISC-V 向量扩展）intrinsics 编程模型，不涉及 RTL。核心结论有三。其一，编程模型以 `vsetvli` 为枢纽：它把 SEW（元素位宽）、LMUL（寄存器分组）写入 `vtype` CSR，并算出本次实际处理元素数 `vl = min(AVL, VLMAX)`，使程序对 VLEN 透明、可移植。容量公式为 VLMAX = VLEN×LMUL/SEW，CoralNPU 默认 VLEN=128、支持 8/16/32 位元素。其二，软件 stripmining 写法是 `while(n>0){ vl=vsetvli; 处理 vl 个; n-=vl }`，范本是 `sw/opt/rvv_opt.h` 的 Memcpy/Memset，用动态 `vl`；这与 u7-l6 的硬件层 stripmining（一次派发→四次发射）属两个不同层面，不可混淆。其三，范例 `rvv_add_intrinsic.cc` 演示向量三段式（load→compute→store），以加宽加法 `vwadd` 体现强约束：输出位宽与 LMUL 均为输入的 2 倍；其写死 `vl=32` 仅为教学简化。配套测试基建 `rvv_type_util.py` 是 Python 黄金查表模型，含 dtype→SEW、(SEW,LMUL)→VLMAX 及自定义 vtype 位编码，反推 VLEN=128；`rvv_cpp_util_header_generator.py` 自动生成类型安全的 C++ 算子模板封装头供 cocotb 复用。intrinsics 由 multilib 工具链（rv32imf/Zve32x）的 `<riscv_vector.h>` 提供，无需额外依赖。
