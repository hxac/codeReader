本讲是 CoralNPU 仓库的「代码地图」，把约 1000 个文件按「根/硬件/软件/验证支撑」四个模块梳理。核心结论：仓库分层清晰，根目录的 `README.md`、`WORKSPACE`、`.bazelrc`、`.bazelversion` 定义项目身份与 Bazel 工作区（实操发现 README 写 Bazel 7.4.1 与 `.bazelversion` 钉死的 8.6.0 漂移，以配置为准）。硬件采用**双轨制**：`hdl/chisel/`（Scala）写标量核/SoC/总线/外设/Cache/TCM，`hdl/verilog/rvv/`（SystemVerilog）写向量与矩阵 MAC 后端，按处理器组件分工而非冗余；后者内部又分 `design/`（可综合 RTL）与 `sve/`（仿真验证环境）。软件侧三件套为 `toolchain/`（RISC-V 工具链、CRT、TCM 链接脚本 `coralnpu_tcm.ld.tpl`）、`sw/`（litert-micro ML 算子库）、`examples/`（入门示例）。验证规模庞大，`tests/`（cocotb/uvm/verilator 为主力，与 `hdl/` 量级相当），`hw_sim/` 提供 Verilator 仿真器 C++ 封装，`coralnpu_test_utils/` 提供测试接口与 AXI 从机。支撑设施含 `doc/`、`rules/`（Chisel/Verilog/cocotb 等 Bazel 自定义规则）、`third_party/`、`fpga/`。引入的关键术语：RTL、IP 核、CRT、TCM（ITCM/DTCM）、Bazel 包与目标标签；并推荐用 `git ls-files | cut | sort | uniq -c` 建立任意大仓库的全景认知。
