本讲介绍 CoralNPU 的 cocotb 测试框架，核心是用 Python 脚本化真实 SoC 中「主机 CPU」的角色，对 RTL（`CoreMiniAxi` DUT）重放驱动流程。测试台遵循固定生命周期：`init → reset → clock → load_elf → lookup_symbol+write 输入 → execute_from → wait_for_halted → read 输出`；程序无 `printf`，I/O 全靠主机与内核共享 DTCM，缓冲地址由 `lookup_symbol` 查 ELF 符号表得出。

主角类 `CoreMiniAxiInterface` 一身两角：向 `io_axi_slave_*` 端口注入命令/数据（外部主机），并用 4MB numpy 数组 `EXTMEM`（`0x20000000`）响应 `io_axi_master_*` 访存（外部存储服务器）；地址落 EXTMEM 直读写数组、不产生 AXI 流量，落 DTCM 等内部区才走真实突发事务。`load_elf` 默认走 backdoor（DPI 直写 SRAM，快），可经 `COCOTB_USE_FRONTDOOR=1` 切 frontdoor（真实 AXI，慢但贴近硬件）。`execute_from` 启动序列为「写 `PC_START`(0x30004) → 写 `RESET_CONTROL`(0x30000)=1 释放时钟门控 → =0 释放复位」；`wait_for_halted` 直接采样 `io_halted` 端口（CRT 的 `mpause`）。

镜像类 `AxiSlave` 是被动从机，响应 DUT 主动事务，用于 SoC 子系统测试模拟 DDR/外设；`has_memory=True` 用 dict 稀疏存储（写 OKAY），`False` 则读 `0xDEADBEEF`、写 `DECERR`。关键术语：RTL/DUT、cocotb、AXI manager/subordinate、DPI、backdoor/frontdoor、`EXTMEM`。
