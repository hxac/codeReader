本讲讲解 CoralNPU 两份独立可发射但当前未实例化进生产数据通路的 Cache 模块。L1I 为 8KB、4 路、256 位行、只读 cache：以 CAM 比较判命中，计数式伪 LRU 替换（每路唯一计数器，为 0 者淘汰），缺失经 AXI 单事务填充，刷写即整表单拍失效。L1D 为 16KB（双 bank × 8KB）、4 路、256 位行（外加逐字节 ECC 成 288 位 SRAM）：双 bank 按行低位交织使相邻行落不同 bank，借 addr+adrx 相邻行成对访问实现 next line prefetch、为 ML 引擎供半带宽并兼作标量/SIMD 对齐缓冲；带 dirty 位写回，脏行替换先经 AXI 写回旧行再读填新行。刷写经 IFlushIO/DFlushIO 分两路：fence.i 触发 I 侧整表失效 + 取指重定向，flushall/flushat 触发 D 侧 8 态 FSM 逐脏行写回并失效，期间内核 stall 到完成。关键事实：生产配置以 TCM 为主存，cache 仅为 cached 配置预留的 IP 块。
