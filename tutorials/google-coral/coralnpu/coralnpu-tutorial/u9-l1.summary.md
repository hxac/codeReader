本讲讲解 CoralNPU 如何实现 RISC-V External Debug 规范的子集。核心实体是 DebugModule——一个独立于 CPU 流水线的旁路模块：对外经 CoreCSR（AXI CSR 桥）暴露给调试器，对内用 haltreq/resumereq 控制标量核、用抽象命令读写核内资源。引入的关键术语包括：halt/resume 电平握手（halted 由核的 debug_mode 回采，resumereq 遇 resumeack 脉冲自动清零）；抽象命令 Access Register（写 command 寄存器触发，regno 区分 GPR/FPR/CSR，结果经 data0 中转，读 GPR 当拍组合完成）；DM 内部寄存器 dmcontrol/dmstatus/abstractcs/command；以及外部调试器经 req_addr/req_op/status 六寄存器走 poll-then-ack 轮询协议（仅写 req_op 才真正扣扳机）。错误回报为双轨：非法 regno 直接返回 rsp.op=FAILED，其余错误（未 halt、size 不支持、地址不在 TCM）经 abstractcs.cmderr 报告。需注意单步逻辑不在 Debug 模块内，而在 CSR 的 DCSR step 位。顶层 CoreAxi 用两路仲裁器把外部 AXI 调试器与内部 CoreCSR 汇入唯一的 DM；综合实践通过 pyOCD GDB server 让真实 GDB 驱动端到端调试。
