本讲承接 u8-l1，从固件视角讲解用 C 驱动 CoralNPU 的 DMA。核心是描述符（descriptor）：32 字节、须 32 字节对齐、放 SRAM/DDR 的结构体；len_flags 把传输长度（24 位）、beat 宽度（log2 字节数）、src_fixed/dst_fixed/poll_en 三标志位压进一个 32 位字，由 dma_make_len_flags 打包。标准编程序列仅四步：摆描述符→写 DESC_ADDR→写 CTRL=0x3（enable+start）→轮询 STATUS.done，v1 不支持中断。三种传输模式靠两个 fixed 位切换：Mem→Mem 双递增、Mem→Periph 目的固定（灌外设 FIFO）、Periph→Mem 源固定，beat 宽度不得超出目标粒度。描述符链接（next_desc）把 N 次 CPU 介入压成一次，done 仅在整链跑完后才置位。错误由 STATUS 的 error/error_code 区分 fetch/poll/read/write/abort，abort（CTRL=0x4）后留下 error_code=5；流控轮询（poll_en 配 poll_addr/mask/value）让 DMA 每 beat 前自查外设就绪，无需改外设即可适配慢速 SPI/I2C/UART。
