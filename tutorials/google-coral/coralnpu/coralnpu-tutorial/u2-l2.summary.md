本讲讲解「编写→编译→验证」一个 CoralNPU 裸机 C++ 程序的闭环，含三个最小模块。其一，程序为**三段式结构**：输入缓冲、输出缓冲加 `main()` 计算主体。CoralNPU 不用 `printf`/`scanf`，而采用「主机与内核共享 DTCM」的 I/O 模型——主机经 AXI 写入输入、启动内核、待其 halt 后读回输出，故缓冲须声明为全局变量以占据固定可查地址。其二，`__attribute__((section(".data")))` 把无初值的全局变量强制钉进 `.data` 段（否则进本项目里 NOLOAD 的 `.bss`），使其成为确定的可加载区；该段经链接脚本 `coralnpu_tcm.ld.tpl` 的 `> DTCM` 规则落入 DTCM，默认起址 `0x00010000`，区间 [0x00010000, 0x00018000)。其三，自定义构建规则 `coralnpu_v2_binary` 通过 platform transition 切到 RISC-V 裸机平台、自动链接 CRT 提供 `_start`、按 ITCM/DTCM 大小生成链接脚本，产出 `.elf/.bin/.vmem` 三件套，BUILD 仅需 `name`+`srcs`。可用 `readelf -S` 核验 `.data` 落点。本讲为下讲在 Verilator 仿真器上运行准备好一个可加载的 `.elf`。
