本讲从开源 Verilator 转向商业 VCS 与基于它的 UVM 验证平台。VCS 因需付费授权，在 Bazel 中默认被 tag 过滤（`-vcs`）关闭，须用 `--config=vcs`（将过滤改写为「只含 vcs」）配合 `VCS_HOME`/`LM_LICENSE_FILE` 等环境变量启用，常以 `CCACHE_DISABLE=1` 排错。`rules/vcs.bzl` 定义两条规则：`vcs_testbench_test`（纯 SV 单元测试台、`test=True`）与 `vcs_binary`（链 C++ 协同仿真库、生成 runner、`executable=True`），均自动打 `tags=["vcs"]`。

讲义介绍两个 UVM 平台：`tests/uvm/` 为全核平台，DUT 为 `RvvCoreMiniVerificationAxi`，由 AXI 主/从 agent、IRQ agent、被动 RVVI agent 与 cosim checker 组成，用 MPACT（可选 Spike）逐条退休指令协同比对，由 `run_uvm_regression.py` 驱动海量 ELF 回归；`rvv_backend_tb/` 为后端块级平台，采用标准「agent（rvs/lsu）+ scoreboard + coverage」三件套，以 SystemVerilog 参考模型 `rvv_behavior_model` 对退休、VRF、访存三类做逐条成对比对。

引入的关键术语有：Agent、Scoreboard、Coverage（covergroup）、协同仿真（Cosim）、RVVI 退休流、定向/随机/边界回归（三份 regress 清单）、代码覆盖率与功能覆盖率。核心结论：定向测试保证覆盖广度与已知 corner，随机测试靠参考模型兜底冲击隐蔽 bug，二者互补构成 UVM 方法学的精髓。
