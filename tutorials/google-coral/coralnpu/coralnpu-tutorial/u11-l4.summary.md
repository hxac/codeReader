本讲是第 11 单元（验证与 FPGA 构建）的收口，承接 u11-l1 的 Bazel+fusesoc 仿真机制，把它延伸到 Vivado 综合与比特流生成。覆盖的最小模块包括：`coralnpu_soc.core`（用 `depend` 列表拼装 Chisel 子系统＋OpenTitan uart/rom/adapter_sram＋`coralnpu_tlul`＋i2c_master＋ispyocto＋tlul2ahblite 桥）、`coralnpu_soc_pkg.core`（顶层常量包）、两个顶层 core——`chip_nexus.core`（走 Vivado、带 `pins_nexus.xdc` 引脚约束与 TCL 钩子、出比特流）与 `chip_verilator.core`（走 Verilator、挂 DPI 外设模型、出仿真可执行）、`fpga/BUILD`（综合流程真正入口）、`ddr4_stub` 与 `coralnpu_tlul`。核心结论有三点：其一，`get_bitstream.sh` 是下载器而非综合器，仅按 git SHA 从 GCP Artifact Registry 拉取 CI 预构建的 `chip_nexus.bin`；其二，真正综合藏在 Bazel 的 `fusesoc_build` 规则里，由 fusesoc 解析依赖图后调 Vivado 完成 synth_design→opt/place/route→write_bitstream。关键术语包括 fusesoc 的 `.core`（CAPI2）文件、virtual core 虚依赖与 `mapping:` 顶替机制、XDC 引脚约束、TCL 钩子、Xilinx MIG、比特流（bitstream），以及「保综合不保功能」的 stub 设计哲学（同名同端口空壳，ready 全拉 0，靠 `internal_exists` 在公开版 stub 与内部真实 DDR4 间切换）。设计哲学为「仿真优先、综合集中」。
