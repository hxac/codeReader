本讲承接 u2-l2，回答「全局数组经 `ct.load` 进入 kernel 后变成什么」——即 cuTile 的 tile 数据模型，含三个最小模块。其一，Tile 是 kernel 内部不可变、形状编译期已知、每维必须为 2 的幂的多维集合，与可读写、运行时 shape 的全局数组相对；该约束由后端 `require_constant_shape` 在编译期用 `x & (x-1)` 校验，非 2 的幂（如 3）会报错，`ct.cat` 也因此要求两输入同形。其二，scalar 即零维 tile（shape 为空元组 `()`）；Python 字面量在 tile code 里自动成为 loosely typed 常量标量，故 `a = 0; a.dtype` 返回 `int32`，且因零维而永远可广播。其三，形状广播与 NumPy 完全一致：末尾对齐、对应维相等或其一为 1、维度少者在左侧补 1，既不拷贝数据也不破坏 2 的幂约束。此外区分 element space（元素空间）与 tile space（瓦片空间）：瓦片索引 `(i,j)` 取出 `array[i*tm+x, j*tn+y]`。loosely typed 常量这条线留给下一讲 u2-l4 展开。
