本讲讲解 cuTile 基于 SQLite 的 cubin 磁盘缓存，目的是避免相同内核被重复交给 `tileiras` 编译，拆为四个最小模块：配置来源、缓存键、读写、淘汰。配置由环境变量 `CUDA_TILE_CACHE_DIR`（默认 `~/.cache/cutile-python`，设为 `off/0/none/""` 即禁用）与 `CUDA_TILE_CACHE_SIZE`（默认 2 GiB）在 `import cuda.tile` 之前一次性解析进 `TileContextConfig`。缓存键 `cache_key = SHA-256(_CACHE_VERSION ‖ compiler_version ‖ sm_arch ‖ opt_level|device_debug ‖ bytecode)`，其中 `opt_level/device_debug` 取**实际生效值**，每段带 4 字节大端长度前缀消除拼接歧义，`_CACHE_VERSION` 是维护者预留的全局失效开关。读写落在单文件 SQLite 库 `cache.db` 的四列 `key/blob/blob_size/atime` 上：`cache_store` 用 `INSERT OR IGNORE` 幂等写入，`cache_lookup` 命中后才刷新 `atime`（LRU 续命）；数据库损坏则删库重建、静默自愈，缓存全程「尽力而为」，不承担正确性责任。`evict_lru` 按 `atime` 升序对 `blob_size` 算前缀和，删除累计字节数 ≤ T−L 的最旧条目，当 T≤L 时自动退化为 no-op。命中缓存可直接跳过昂贵的 `tileiras` 子进程调用，这是缓存的核心收益。
