本讲是「运行时调度、导出与扩展」单元收尾，集中讲解 cuTile 的可观测性与调优工具，覆盖五个最小模块：`CUDA_TILE_LOGS`、`CUDA_TILE_DUMP_TILEIR`/`CUDA_TILE_DUMP_BYTECODE`、`enable_crash_dump`、`compiler_timeout`、`replace_hints`。核心机制在于调试开关有两条加载路径：进程级 `TileContextConfig`（在 `_context.py`，`import cuda.tile` 时一次性解析，如 `CUDA_TILE_LOGS`、超时、崩溃快照、缓存目录）与模块级常量（在 `_debug.py`，dump 系列直接 `os.environ.get`）；二者都在 import 时定型，因此环境变量须在导入前设置，仅 `ct.compiler_timeout` 是运行时上下文管理器（非线程安全）。关键术语：`CUDA_TILE_LOGS=CUTILEIR/TILEIR` 将优化后的 cuTile IR（`_transform_ir` 之后的 `ir.Block`）与经内部扩展反序列化的 TileIR MLIR 打印到 stderr；dump 把字节码（`.tileirbc`）与 MLIR（`.tileir`）落盘，依赖目前非公开的 `cuda.tile_internal._internal_cext`，缺失时优雅降级。`enable_crash_dump` 在 `tileiras` 抛 `TileCompilerError` 时打包匿名化字节码与 debug_info 成 zip，并让 `_IrKeeper` 保留全部 final IR。`compiler_timeout` 给编译子进程设硬超时，触发 `TileCompilerTimeoutError` 并提示缩小 tile。`replace_hints` 用 `dataclasses.replace` 派生新内核，是性能 hint（`num_ctas`/`occupancy`/`opt_level`/`num_worker_warps`，可配 `ByTarget`）调优与自动调优「物化 best config」的关键，新内核拥有独立 JIT 缓存；调优时需留意 `EXPERIMENTAL_CUDA_TILE_DEBUG_BUILD` 会强制 `-O0`。
