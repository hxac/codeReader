本讲讲解 cuTile 自动调优入口 exhaustive_search：它接收一组编译期配置（tile 尺寸、num_ctas 等）搜索空间，逐一编译计时，返回 TuningResult{best, successes, failures}，其中 successes 为按 mean_us 升序的 Measurement 序列。单配置计时由 _TimingCandidate 用 Welford 在线算法增量更新均值与 95% 置信区间一半 error_margin_us，命中相对误差<1% 等条件即判 converged，无需固定次数采样。搜索分三阶段：Phase 0 warmup 并据首个成功配置动态收紧启动超时；Phase 1 用 min-heap 凑齐 Top-K=5 收敛配置确立 cutoff；Phase 2 以 mean-error_margin<cutoff 剪枝慢配置。核心安全层 benchmark_with_timeout 把一次启动序列化为 IPC payload 经 _benchmark_worker 子进程执行，超时则杀子进程并抛 TileLaunchTimeoutError，使死循环内核不拖垮主进程；CUDA_TILE_BENCHMARK_DISABLE_SUBPROCESS 可禁用此机制（仅调试）。最后对照已废弃的 autotune_launch：它用两级缓存（kernel 指纹 + 张量 stride 指纹）加 reservoir 采样，但 torch.cuda.Event 固定次数计时且无超时隔离，正是被取代的主因。
