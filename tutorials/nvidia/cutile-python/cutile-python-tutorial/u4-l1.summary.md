本讲是「高级数据访问」单元首讲，打破「一个 block 处理一个输出 tile」的限制，引入四个最小模块：TiledView、num_tiles、num_blocks、persistent loop。核心概念 **TiledView（瓦片视图）** 把「全局数组 + 一种固定 tile_shape 切分方式」固化为对象，使 `tv.load(tile_index)`/`tv.store(tile_index, tile)` 在瓦片空间按索引读写多个 tile，无需重复传 shape；它不拷贝存储，只是对原 Array 的观察视角。**num_tiles** 有模块级（`ct.num_tiles(array,axis,shape)`，不必建视图）与视图方法两层，返回运行时 int32，计算式为 `ceil(dim/step)`，故瓦片索引 `t` 对应元素原点 `t·step`。**traversal_steps** 控制相邻瓦片原点步距：等于 tile_shape 无重叠无间隔；小于则重叠（滑动窗口/卷积），大于则有间隔（下采样），每分量须严格为正、需 CTK≥13.3。**persistent（常驻）内核** 通过 `ct.num_blocks(0)`（与 `bid` 对称，返回启动 block 总数）配合跨步循环 `for cur in range(bid, total_tiles, num_blocks)` 把 block 数限制为 NUM_SMS，让每个 block 轮流认领多个输出 tile（循环分块），从而解耦 block 数与输出瓦片数、减少调度开销。核心结论：traversal_steps（单 block 内数据遍历视角）与 persistent 循环（跨 block 调度视角）正交，常在 GEMM 中组合使用。
