本讲承接 u3-l1 的 load/store 范式，把隐含在每次 `ct.load`/`ct.store` 里的「瓦片结构」显式化为 `TiledView` 对象，并围绕四个核心问题展开。

第一，引入三层空间的概念：**元素空间**（数组真实数据）、**瓦片空间**（用某种瓦片形状切数组得到的网格）、**块空间**（`ct.launch` 启动的 grid）。`TiledView` 把「数组 + 瓦片形状 + 填充模式 + 步长」凝固成一个对象，通过 `tv.load(index)` / `tv.store(index, tile)` 按瓦片索引读写，与裸 API 完全等价，但便于在循环里反复访问同一数组。

第二，`num_tiles`（自由函数与方法两种入口）让内核自己推导循环上界，免去把范围作为 `Constant` 从 host 传入；其语义为向上取整除法，分母是 `traversal_steps` 而非 `tile_shape`，且计入部分越界的瓦片。

第三，`traversal_steps`（CTK 13.3 特性）控制相邻瓦片原点间距：小于 `tile_shape` 产生重叠（卷积/stencil），大于则留间隔（跨步采样）；它不要求为 2 的幂。

第四，**persistent 内核**用 `ct.num_blocks` 配合三参数 `range(bid, upper_bound, num_blocks)`，让 host 仅启动 SM 数量级个 block、每个 block 串行认领多块输出瓦片（cyclic 映射），从而把瓦片数与 block 数解耦、降低调度开销。`samples/MatMul.py` 给出 GEMM 的非 persistent 与 persistent 两版对照，关键差别仅是多一层认领瓦片的外循环。
