本讲围绕 cuTile 的编译期机制，引入三个最小模块。**Constant（常量嵌入）**：用 `ct.Constant[T]` 标注的参数在编译时被 `loosely_typed_const` 烘焙成 IR 字面量、从运行时签名消失，每个唯一取值单独编译一份 cubin，可触发常量折叠与对齐推理，但取值组合过多会引发**编译爆炸**；嵌入值在内核内表现为宽松类型常量，其算术结果类型仍由 u2-l4 的类型提升规则决定。**static_eval（编译期求值）**：在编译期用完整 Python 语义执行一段表达式，常量当普通 Python 对象、动态 tile/array 当只能查询 `.shape`/`.dtype` 的**代理对象（proxy）**，禁止运行时运算与 tile 函数调用，且必须直接按名字调用，可借此实现「编译期 if」或调用宿主函数。**static_assert（编译期断言）**：建立在 static_eval 之上，要求编译期常量布尔条件，失败时抛出 `TileStaticAssertionError`，把非法配置前移到编译阶段。三者共享「关键字式函数」机制，在 ast2hir 被特殊分派、跳过 Tile 翻译、走完整 Python 语法。
