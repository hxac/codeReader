本讲建立 cuTile 的执行模型直觉，拆成三个最小模块：**grid**（网格）、**block**（线程块）、**执行空间**。一次 kernel 启动会创建组织成 1D/2D/3D 的 grid，grid 尺寸用 `ct.cdiv(N, TILE_SIZE)` 向上取整计算，类型为 `Dim3`（1~3 元整数元组），由 `ct.launch(stream, grid, kernel, args)` 透传给 `cuLaunchKernel`。**block 是执行单元**，每个 block 完整执行一遍 kernel 函数体；标量运算在单线程上串行，tile 运算由整个 block 集体并行完成。关键区分是 block（执行单元）与 tile（数据单元）不可混淆。cuTile **只表达 block 级并行、不暴露单个线程**，block 在硬件上映射到 CTA（Cooperative Thread Array），集体运算由编译器映射到内部 warp、张量核、TMA。反直觉的同步规则是：block 内禁止显式同步（集体运算边界已隐式同步），block 之间允许同步。本讲引入的核心术语包括：grid、block、tile、`Dim3`、`ct.cdiv`、`ct.bid`、`ct.num_blocks`、**三种执行空间**（host code / SIMT code / tile code，其中 SIMT code 旧称 device code）、annotated function（已标注函数）。`function` 装饰器用 `host`/`tile` 参数标注可调用空间，`kernel` 通过 `__call__` 抛错强制只能用 `launch` 启动；`ct.cdiv` 跨 host 与 tile，`ct.launch` 仅 host，`ct.bid`/`ct.load` 仅 tile code。
