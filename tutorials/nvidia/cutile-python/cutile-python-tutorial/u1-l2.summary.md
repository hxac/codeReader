本讲承接 u1-l1 的编译全链路直觉，把 cuTile 在本地真正跑起来，覆盖四个最小模块。安装上区分两条路线：`cuda-tile[tileiras]` 把后端编译器 tileiras（连同 nvcc、nvvm，构成编译工具链，需主次版本一致）作为可选依赖装进 Python 环境，对应 extra 在 pyproject.toml 中展开为带 tileiras 子集的 CUDA Toolkit 元包；`cuda-tile` 则复用系统级 CUDA Toolkit 13.1+。从源码开发用 `pip install -e .` 可编辑安装，由 setup.py 的 BuildExtWithCmake 驱动 CMake 编译 C++ 扩展 `_cext`，并以符号链接把产物链入源码树，使改 C++ 只需 `make -C build` 增量重编。第一个内核 VectorAdd 确立了 cuTile 通用的 load–compute–store 三段式范式：`@ct.kernel` 标记的函数不在定义时执行，而在 host 端 `ct.launch` 时 JIT 运行；内核内用 `ct.bid(0)` 取当前 block 编号，`ct.load`/`ct.store` 以「tile 为单位」的 index 定位、shape 决定 tile 大小，tile 本身不可变。host 端用 `ct.cdiv` 做向上取整除法算出三维 grid（block 数量），再用 `ct.launch(stream, grid, kernel, args)` 在 CUDA 流上启动。tile_size 声明为 `ct.Constant[int]` 编译期常量，会被嵌入生成的 cubin，改值会触发重新编译。
