本讲拆解 Tile IR（_ir/ir.py）的核心数据结构，覆盖五个最小模块。IRContext 是「一个内核一份」的全局注册表，以 Var 名字为主键集中存放类型、常量、宽松类型、聚合值，并提供 make_var/make_temp/copy_type_information。Var 是轻量 SSA 值句柄，只持 name/loc/ctx，其余属性委托 IRContext 查表、可事后 set_* 补充。Operation 基类用 operand()/attribute()/nested_block() 三种声明式字段标注输入，由 __init_subclass__ 在类定义时自动归类并登记 opcode/terminator/memory_effect。Builder 基于 threading.local 做线程局部「当前构建器」，用上下文管理器压栈，add_operation 自动造临时 Var、设类型并 append，是发射 IR 的唯一入口。Block 是带 params（入口参数，承载 phi 语义）的指令容器，支持类列表增删改与递归 traverse；控制流体作 nested_block 挂在 Operation 上，形成树状 IR，Var 跨树传值，IRContext 为其共享后台。
