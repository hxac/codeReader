本讲是 cuTile 优化流水线后半段的收尾，聚焦四个 IR 变换 pass，执行顺序为 `rewrite_patterns → hoist_loop_invariants → unhoist_partition_views（仅字节码 < V_13_3）→ split_loops → DCE`，遵循「先局部算术融合、再循环变换、最后清理」的原则。

`hoist_loop_invariants` 实现循环不变量外提（LICM），用单遍线性算法同时刻画两个维度：块的可移动性三档（`IMMOVABLE`/`CAN_MOVE_WITH_LOOP`/`CAN_MOVE`，由 `STORE`/`Return`、`Continue`/`Break` 触发）与指令依赖深度（`max_outside_depth` 与 `must_stay`）。它把纯计算指令沿循环体向上外提，但只能提出「循环」、不能提出 `IfElse` 分支；关键细节是用外提后的 `target_depth` 登记结果变量。该 pass 必须排在 token 排序之后，以免破坏内存定序。`unhoist_partition_views` 是其善后，在旧字节码下把被外提的 `MakePartitionView` 克隆回消费者所在块。

`split_loops` 把「条件为归纳变量 vs 循环不变量」的单分支 `for` 循环（步长需为常量 1）拆成两段无分支循环，用 `min`/`max` 夹边界、中间携带变量接力，`_BRANCH_TO_KEEP` 决定各段保留哪个分支。`rewrite_patterns` 是微型「注册-匹配-安全校验-应用」模式匹配引擎，核心是 FMA 融合：把非受限浮点的 `mul`+`add/sub` 合成单条 `FusedMulAddOperation`（更快更准）；四道安全校验（尤其「新 op 不能引用被删结果」）使 `tmp=tx*tx; tmp+tmp` 这类自引用安全回退。
