本讲聚焦 cuTile 优化流水线，覆盖三个模块：`_transform_ir`（Pass 编排）、`eliminate_assign_ops`（消除赋值）与 `dead_code_elimination_pass`（DCE）。核心结论是：`hir2ir` 产出的裸 IR 忠于源码、含冗余临时值与桥接操作，须经有序 Pass 变换后才交给字节码生成器。流水线顺序为 `eliminate_assign → DCE → dataflow → add_divby → token_order → rewrite → hoist → unhoist? → split → DCE`；DCE 跑两次，dataflow 结果被 divby 与 token_order 共享，hoist 必须晚于 token_order 以免误外提 load。引入的关键术语：Pass（整树遍历变换）、副作用根、`MemoryEffect` 枚举（`NONE<LOAD<STORE`）、`Assign`（变量别名桥）、`Mapper`（变量重映射）、数据流图、可达性传播（工作表求不动点）、`$cf` 伪变量、携带值、`MakeDummy` 占位。要点：DCE 前必须先折叠 Assign，否则会把有用赋值链误判为死代码；DCE 以 `STORE` 与 `Return` 为根反向生长有用集合，闭包外即死代码；控制流体经 `$cf` 与四条建边规则参与可达性，被裁的携带值用 MakeDummy 占位以维持循环结构对齐。
