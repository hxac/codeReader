本讲讲解把 `@ct.kernel` 内核接入 JAX 计算图的全链路，含四个最小模块。其一，用户 API `cutile_call` 采用「一参数一角色」：从内核参数注解树 `parameter_annotations` 的叶子节点派生 role——输入 `'i'`、输出 `'o'`、输入输出 `'io'`、常量 `'c'`、运行时标量 `'s'`；`OutputPlaceholder` 声明输出，`InputOutput` 实现 in-place 别名。因只认叶子，tuple 参数被显式拒绝，编译固定用 `cutile_python_v1`。其二，`register_ffi`/`register_primitive` 在 import 时把经 `PyCapsule` 桥接的 C++ handler 注册为 XLA FFI target `"cutile_launch"` 与 primitive `cutile_call_ffi_p`，并挂 MLIR lowering。其三，lowering 按 role 派生 `ParameterConstraint`，`compile_kernel_cached` 编译 cubin（键 `(kernel, constraints)`，cubin_id=SHA-256(cubin)），把 cubin、buffer_ids、scalar_packed 作为属性塞进 custom call。其四，C++ `cutile_call_handler` 是 INSTANTIATE/INITIALIZE/EXECUTE 三阶段统一入口：INSTANTIATE 按 cubin_id find-or-load 进程级 `KernelEntry` 注册表（引用计数、slot 可复活），EXECUTE 由 `pack_buffer` 按 v1 把 buffer 展开成「指针+shape+strides」再 `cuLaunchKernel`（block 恒为 1）。核心结论：role 由注解决定而非运行时类型；cubin 加载一次、执行多次；`cutile_launch` 可进 `jax.export` 跨进程重跑。
