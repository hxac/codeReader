本讲讲解 cuTile 的两类集体运算。归约（`sum/prod/max/min`）沿 `axis` 把多元素合并为一个，`axis` 可取 `None`（全归约）、`int` 或 `tuple`，`keepdims` 控制是否保留被归约轴为 1；归约从**单位元**起算（sum→0、max→`-inf`）。`argmax/argmin` 是「值+下标」捆绑归约，返回 `int32` 下标、平局取小下标，不支持 tuple 轴。前缀扫描（`cumsum/cumprod`）输出与输入同形，`axis` 必须是整数（默认 0），`reverse=True` 给出后缀版本。核心结论：归约与扫描在底层都映射到**带 body 的 IR 操作** `tile_reduce`/`tile_scan`，body 是「合并两个标量」的纯函数；内置算子只是把 body 钉死，`ct.reduce`/`ct.scan` 允许自定义 `func` 及元组并行模式，但 body 禁含分支、循环、访存与嵌套。行归约内核的标准模式为「`ct.full` 建累加器→循环 `load` 累加→`ct.sum(axis=)` 压扁→`store`」，LayerNorm 是其范本。
