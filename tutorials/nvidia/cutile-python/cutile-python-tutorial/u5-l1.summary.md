本讲是编译前端单元首讲，拆开 `@ct.kernel` 黑盒，仅覆盖「装饰与对象构造」阶段，不展开 `compile_tile` 流水线与 `@impl` 注册系统。

三类装饰器用于标记执行空间：`function` 声明普通 tile 函数（默认仅 tile code，从 host 调用会被线程局部的 `DispatchMode` 拦截报错）；`stub` 在其上叠加 `_cutile_python_stub` 标记内置操作（如 `ct.load`）；`kernel` 则是一个**类**（继承 C++ 类型 `TileDispatcher`），装饰后返回对象而非函数，是 tile code 的唯一入口，禁止直接调用、必须由 `ct.launch` 启动。

`kernel.__init__` 一次性固化 `AnnotatedFunction` 与 `CompilerOptions`，并经 `super().__init__` 把三套参数掩码交给 C++ 层保存。`AnnotatedFunction` 用 `get_type_hints(include_extras=True)` 解析注解（含字符串注解），将 `Constant`/`IndexedWithInt64`/`ScalarInt64` 等元数据翻译成三套与参数一一对应的扁平布尔掩码（constant / int64_index / int64），供 C++ 层 `extract_cuda_args` 按位置高效查询。`CompilerOptions` 是冻结数据类，承载并自校验 `num_ctas`/`occupancy`/`opt_level`/`num_worker_warps` 四类性能 hint；`ByTarget` 让同一 hint 按不同 sm 架构取不同值。`_compile(signature, context)` 是装饰期固化信息与运行期签名的汇合点。注意 `.pyi` 中 `TileDispatcher.__init__` 存根已过时，以实现为准。
