本讲是「编译前端」单元首讲，聚焦 `@ct.kernel` 装饰阶段产出的对象模型，不讲编译流水线执行。覆盖五个最小模块：`kernel`/`function`/`stub` 三类装饰器、`CompilerOptions` 编译旋钮、`AnnotatedFunction` 注解解析、`ParameterAnnotationNode` 注解树、`TileDispatcher` 到 C++ 的桥接。

核心结论：`kernel` 是继承自 C++ `TileDispatcher` 的装饰器，产物是携带元数据的不可直接调用对象，须经 `ct.launch` 启动；它持久保存两个字段——`_annotated_function`（参数注解）与 `_compiler_options`（`num_ctas`/`occupancy`/`opt_level`/`num_worker_warps` 的冻结 dataclass）。

关键术语：`AnnotatedFunction` 用 `get_type_hints(include_extras=True)` 解析注解，归一成统一的 `ParameterAnnotationNode` 树，取代旧的三套扁平布尔掩码。树由三种节点构成：`LeafAnnotationNode`（标量/数组角色，`constant`/`scalar`/`array`/`list` 四字段互斥）、`HomogeneousTupleNode`（变长同质，`each`）、`HeterogeneousTupleNode`（定长异质，`items`），支持任意嵌套 tuple。常量性经 `outer_constant` 向下传播，区分「整组常量」与「部分常量」，决定哪些值进入 JIT 缓存键。`TileDispatcher.__init__` 现仅接收注解树，C++ 侧用 `KIND` 字符串分派镜像；tuple 与静态形状最终要求 `cutile_python_v2` 调用约定。
