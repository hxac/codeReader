本讲聚焦 `cuda.tile` 顶层门面模块 `__init__.py`：它从 `_cext`/`_datatype`/`_stub`/`_execution` 等子模块再导出全部公共 API，并以 `__all__` 声明公共契约。`ct.` 符号可归为装饰器、启动、数据类型、数据模型、内存操作、算术、归约、形状、索引、原子、常量与断言等类；其中来自 `_stub` 的操作类数量最大，真正的控制入口仅 `kernel`/`function`/`launch`/`compilation`/`tune`。核心是区分三类装饰器：`kernel` 是 tile 代码入口（继承 `TileDispatcher`，抽取参数注解掩码与 `CompilerOptions`，禁止直接调用，须用 `launch` 启动）；`function` 声明执行空间；`stub` 标记内置操作（签名在前端、实现在后端，且不在顶层 `__all__`），凭 `_cutile_python_stub` 标志由后端注册系统识别并对接 IR。
