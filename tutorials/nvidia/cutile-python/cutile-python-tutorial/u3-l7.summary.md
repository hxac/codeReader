本讲围绕「让内核参数更有表达力、让编译器拿到更多编译期信息」这一主线，引入 cuTile 两类新参数特性：tuple 参数与静态形状特化，共四个最小模块。

第一，统一注解树 `ParameterAnnotationNode` 取代旧的扁平布尔掩码，用叶子节点、同质元组（变长 `tuple[T, ...]`）和异质元组（定长 `tuple[A, B]`）三种节点表达任意嵌套的参数注解，是支持 tuple 的前提。

第二，`TupleConstraint` 把一组相关参数打包成单个 Python `tuple` 传入，内核内用 `pair[i]` 解包；只接受纯 tuple（拒绝 namedtuple），异质元组长度须匹配。`Constant` 与 tuple 有「整组常量」与「部分常量」两种组合，仅 Constant 元素进入 JIT 缓存键、改变取值才触发重新编译。

第三，`ArrayAnnotation.static_shape_dims`（对应 `ArrayConstraint.shape_constant`，支持负索引）把数组指定维度特化为编译期常量以换取对齐优化，代价是不同 shape 编译出不同 cubin，只适合取值有限的维度，否则编译爆炸。

第四，二者都强制要求新调用约定 `cutile_python_v2`，由 `_validate_constraint_support` 在 `KernelSignature` 构造期递归门控；mangled 符号名里 tuple 用 `T`、静态形状用 `s` 编码且可逆。
