本讲讲解 cuTile 内核从 Python 启动到 GPU 投放的完整链路与调用约定。核心结论有四：其一，`ct.launch` 的 JIT 链路为 `cuda_tile_launch → launch_impl → launch → prepare_launch → compile → cuLaunchKernelEx`，全程持锁保证并发安全；其二，`TileDispatcher` 现以单一参数注解树 `parameter_annotations` 初始化（取代旧的扁平布尔掩码），用 profile 缓存（按参数类型序列）与 kernel family 常量缓存（按 Constant 取值）两级缓存把热路径压到查表；其三，`DispatchMode`（`NormalMode`/`StaticEvalMode`）以线程局部存储阻止 `@ct.function` 标注的 tile 函数从 host 调用，`kernel` 不走它而直接在 `__call__` 报错；其四，调用约定 `CallingConvention` 规定二进制参数格式、支持的约束集合与 mangling，`extract_cuda_args` 据此把每个约束编码写入 arena，再以指针数组交给 `cuLaunchKernel`。`cutile_python_v1` 与 `v2` 参数顺序一致，v2 额外支持 tuple 参数与静态形状——JIT 由 `minimum_calling_convention` 自动升级到 v2，AOT 由 `KernelSignature` 构造期的 `_validate_constraint_support` 显式门控，version<2 即抛 `ValueError`。
