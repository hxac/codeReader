cuTile 的内存模型允许编译器与硬件为性能重排访存，跨 block 的访存顺序默认不保证，block 内部由集体运算边界隐式同步。定序靠两个维度协同：`MemoryOrder` 描述单次操作多强的顺序保证，取值 WEAK/RELAXED/ACQUIRE/RELEASE/ACQ_REL，其中 `load/store` 默认 WEAK、原子 RMW 默认 ACQ_REL；`MemoryScope` 描述对哪些 block 生效，取值 NONE/BLOCK/CLUSTER/DEVICE/SYS，其中 `load/store` 默认 NONE、原子默认 DEVICE。两条硬规则：原子操作不允许 WEAK；非 WEAK 的 order 必须配非 NONE 的 scope，否则编译期拒绝。同步为逐元素粒度。原子操作族 `ct.atomic_add/cas/xchg/max/min/and/or/xor` 沿用 gather/scatter 的元素下标约定、返回旧值，逐元素原子但整批非原子；另有 `RawArray.atomic_*_offset`（带 mask）与 `TiledView.atomic_store_*` 等价入口。cuTile 没有独立 `ct.fence()`，栅栏职责由配对的 RELEASE-store 与 ACQUIRE-load（或带 ACQ_REL 的原子）承担。直方图是其典型应用：用 `atomic_add` 把多 block 局部计数累加到全局 counts，逐元素原子保证不丢更新，结合律使无需顺序保证。
