本讲聚焦 cuTile 跨 block 共享数据。核心前提是内存模型允许编译器/硬件重排跨 block 访存，无显式同步就不能假设读写顺序，且同步以单个数组元素为粒度。引入两个排序旋钮：MemoryOrder（WEAK/RELAXED/ACQUIRE/RELEASE/ACQ_REL）定义排序语义，其中 acquire/release 配对建立可见性，RMW 默认 ACQ_REL，load 仅取读端序、store 仅取写端序；MemoryScope（NONE/BLOCK/CLUSTER/DEVICE/SYS）定义生效范围，原子默认 DEVICE，scope 仅在非 WEAK 时有意义。原子族 atomic_add/max/min/and/or/xor/xchg/cas（模块级）与 RawArrayMemory.atomic_*_offset（裸内存级）遵循 gather/scatter 索引约定，特性是逐元素原子、整体非原子、返回旧值，位运算要求 dtype 完全一致。fence 在当前 Python API 无独立原语，跨 block 排序由 order/scope 加底层 token 链实现。直方图内核是「多对一写入必须原子」的典型实践。
