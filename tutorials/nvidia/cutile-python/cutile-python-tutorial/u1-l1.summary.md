本讲只覆盖三个最小模块，建立对 cuTile Python 的全局直觉。**cuTile Python** 是面向 NVIDIA GPU 的 tile 级并行 DSL：用 `@ct.kernel` 写 kernel，遵循 load–compute–store 范式——host 端用 `ct.launch` 启动，kernel 内用 `ct.load` 把全局 array 的一块搬成 tile 做集体计算，再 `ct.store` 写回。关键数据概念是 array（全局显存、可变、带 strides）与 tile（kernel 内部不可变、每维为 2 的幂常量），这是理解全部 API 的钥匙。cuTile 抬高了抽象层级，表达 block 级并行、不暴露单个线程，这是它与传统 SIMT CUDA 的根本区别。**Tile IR** 是独立于 Python 的 block 级中间表示，是前端与后端之间的契约，可序列化为字节码用于优化 pass 与 AOT 导出。**tileiras** 是后端编译器，把 TileIR 字节码编译成 cubin。完整链路为：`AST → HIR → Tile IR →（优化 pass）→ 字节码 →（tileiras）cubin →（缓存）→ cuLaunchKernel`，整个过程在 `ct.launch` 时 JIT 完成，调用约定为 `cutile_python_v1`。
