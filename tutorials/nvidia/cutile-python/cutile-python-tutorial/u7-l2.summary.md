本讲钻进 TileIR 字节码的落盘格式，回答「`CodeBuilder.buf` 里的字节如何打包成 `.tileirbc`」。核心模块有四：一是顶层容器 `write_bytecode`，文件由 magic 头（`\x7fTileIR\x00` + 版本）、若干带 tag 的 section（Func→Global→Constant→Debug→Type→String）与 `EndOfBytecode` 哨兵构成，统一用 varint（LEB128）编码。二是版本模型 `BytecodeVersion`，以 `major*10000+minor*100+tag` 打包成 `IntEnum`，使「数值大小 == 版本新旧」，直接用 `>=` 做特性门控；最大支持版本由「空字节码探针实跑 `tileiras`」探测。三是通用表机制 `_write_table`（项数+偏移数组+数据区），Type/String/Constant 三张表共享，靠 `Table.__missing__` 自增 id 去重，`TypeTable` 进一步以编码字节串为键去重，I1/I32 预定义为 id 0/1。四是 `encode_*Op` 的固定模板（opcode→结果类型→flags→attributes→operands→`new_op`）与两套版本门控写法（新字段只在 X+ 写、旧版本断言取默认值；整条 op 只在 X+ 存在）。关键术语：section、tag、varint、`_write_table`、`BytecodeVersion`、特性门控、`TypeTable` 去重、`encode_*Op` 模板、`new_op`、`DYNAMIC_SHAPE`。
