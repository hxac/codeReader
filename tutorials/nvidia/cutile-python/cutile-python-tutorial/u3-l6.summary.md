本讲围绕高性能分块 GEMM（samples/MatMul.py）展开四个最小模块：①`ct.mma` 计算 (x@y)+acc、映射到张量核、结果 dtype 跟随 `acc` 且不做输入类型提升，是 K 维累加循环（本质 fold）的不二选择，累加器用 float32 保精度、初值为单位元 0；②`ct.matmul` 只算 x@y、会做类型提升、支持 1D/2D/3D，适合一次性整块乘与 batched 场景；③float32 输入须先 `astype(ct.tfloat32)` 才能走 tf32 张量核快通道，tf32 是 10 位尾数的受限浮点、精度近 fp16、需按架构设容差；④swizzle 重排 block→瓦片映射以提升 L2 复用，persistent kernel 用 `num_blocks` 个 block 串行处理多瓦片以减少调度开销，两者都不改变数值结果。host 端按 dtype 选 tile 尺寸并把 tm/tn/tk 作为 Constant 嵌入内核。本讲引入的关键术语：张量核（Tensor Core）、tfloat32（tf32）、mma/matmul、swizzle、persistent kernel、num_tiles/num_blocks。
