本讲引入 cuTile 中「按元素下标而非瓦片索引访问数组」的三类 API。`ct.gather`/`ct.scatter` 用**元素下标 tile**（索引元组长度等于数组秩、各分量按 NumPy 规则广播）做完全随机的逐元素读写，越界/自定义 `mask` 取逻辑 AND，gather 越界返回 `padding_value`、scatter 越界或被掩码处不写入；底层经 `_gather_scatter_pointer_and_mask` 折算为一个**散落指针 + 掩码**，落到 `LoadPointer`/`StorePointer`，无法走 TMA，且 scatter 对重复下标是**数据竞争**（要原子语义须用 `ct.atomic_*`）。`ct.load_advanced_indexing`/`ct.store_advanced_indexing` 解决「**恰好一个稀疏维**（1D 整数 tile）+ 其余**稠密维**用 `ct.Slice(start, length)`（length 须为 2 的幂且编译期常量）」的半结构化访问；底层构造 gather/scatter 视图后走结构化 `TileLoad`，**可能落到 TMA**，性能优于纯 gather，需 `BytecodeVersion.V_13_3`。选型直觉：规则整块用 `load/store`，完全随机用 `gather/scatter`，行列稀疏+其余连续用 advanced indexing。
