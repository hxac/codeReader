本讲承接 u8-l1 与 u5-l2，讲 cuTile 的 AOT（提前编译）导出。核心入口 `export_kernel` 只是 `compile_tile` 的薄封装：把 `output_format` 翻译成 `return_cubin`/`return_bytecode` 开关再写盘；AOT 与 JIT 走同一条编译流水线，区别仅在签名由用户显式给出还是框架自动推断。引入的关键术语：`KernelSignature`（参数约束 + 调用约定 + 符号名的不可变打包）与五种 `ParameterConstraint`（Scalar/Array/List/Tuple/Constant，裸值经 `_to_constraint` 归一为 Constant/Tuple）。核心结论有三：其一，调用约定门控放在 `KernelSignature.__init__`，按 `calling_convention.version` 递归校验，tuple 参数与静态形状都强制要求 `cutile_python_v2`，使 AOT、`from_kernel_args`、demangle 三条路径共用同一道检查；其二，`ArrayConstraint.shape_constant` 是内核注解侧 `static_shape_dims` 在 AOT 签名侧的对应，给定后会压制同维的 `shape_divisible_by`；其三，`mangle_kernel_name` 用前缀字母 + 轴位掩码分桶产出人可读符号（tuple 用 `T{count}`、静态形状用 `s` 谓词、浮点常量用 16 位 hex），`demangle_kernel_name` 递归下降可逆还原，mangle 内部 assert 往返一致，且 demangle 重建签名时也会触发门控。
