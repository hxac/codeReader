本讲拆解 `compile_tile`——cuTile 编译前端的总调度函数，吃进 `AnnotatedFunction` 与一组 `KernelSignature`，吐出含 cubin/字节码/final IR 的 `CompilationResult`。全函数被一把全局 `RLock` 串行化，主体为「线性向下 + 三处短路」：前端准备（补 mangled symbol、探测 `sm_arch`/`bytecode_version`）→ `get_function_hir` 一次性生成与 signature 无关的 HIR → `_IrKeeper` 懒生成 final IR → 后端 `_get_bytecode` 编码字节码 → `compile_cubin` 调外部 `tileiras` 产出 cubin → SQLite 磁盘缓存兜底。`return_cubin/return_bytecode/return_final_ir` 三开关决定在哪一步提前返回，分别服务 JIT、AOT、调试。关键设计：final IR 生成是被后端步骤「按需拉动」而非前端主动完成；`_IrKeeper` 用 `keep_all` 控制是否记忆（崩溃 dump 复用）。`_transform_ir` 是固定顺序的优化 pass 流水线（DCE、数据流分析、divby、token 排序、代码外提、循环分裂、模式重写），核心硬约束是「循环不变量外提必须在 token 排序之后」，数据流分析结果被多个 pass 共享。`cache_key` 含编译器版本、架构、优化等级、字节码与 device-debug，命中即跳过 tileiras。本讲只讲编排，各子模块（ast2hir、hir2ir、IR 核心、各 pass、字节码编码、tileiras、缓存）留待后续展开。
