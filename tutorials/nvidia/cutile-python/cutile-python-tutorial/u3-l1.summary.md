本讲是「编写内核」单元首讲，确立 cuTile 内核最核心的 **load–compute–store** 三段式范式，覆盖 `bid`、`load`、`store` 三个最小模块（即内核的「主循环骨架」）。`ct.bid(axis)` 返回当前 block 在 grid 第 `axis` 轴的坐标（int32 标量），是 tile code 里定位自己的唯一手段，通常直接用作 load/store 的 `index`，使「block 并行」与「tile 切分」对齐。`ct.load(array, index, shape, ...)` 把全局 `Array` 的一块搬成内核内不可变的 `Tile`；关键概念是 `index` 为 **tile space（瓦片）索引**而非元素下标，`shape` 是编译期常量且每维为 2 的幂，部分越界按 `padding_mode` 填充、整体越界未定义。`ct.store(array, index, tile, ...)` 是其逆操作：无 `shape`（从 `tile` 推断），部分越界写入被忽略、整体越界未定义，返回 `None`。host 端用 `ct.cdiv(N, TILE)` 算 grid、`ct.launch` 启动。三者均为 stub，真正实现由后端 IR 注册系统提供；中间的 compute（逐元素算术产生新 tile）留待 u3-l2 展开。
