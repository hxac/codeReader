本讲讲解 Tile IR 的类型系统。`Type` 根类用「聚合契约」四方法（`is_aggregate` / `aggregate_item_types` / `flatten_aggregate` / `make_aggregate_value`）统一描述类型的拆解与重组，`TensorLikeTy` 是带 dtype、shape 的中间基类。`TileTy` 以 `__new__` + 模块级缓存实现 flyweight，是叶子类型；`ArrayTy` 是聚合，展开为 `1+2*ndim` 个叶子 `(base_ptr, *shape, *strides)`，并借 shape 字段的 `None`/`int` 二义性承载静态形状特化；`TupleTy` 为异构定长（子类型即各元素），`ListTy` 为同质变长（运行时仅 `ptr<int64>` + `int32` 长度）。`flatten_block_parameters` 以「先拆后装」把聚合参数展开为可序列化的扁平 Var 充当 Block 入口参数，再用 `make_aggregate` 重建原聚合值供函数体使用，化解用户聚合参数与字节码扁平签名之间的矛盾。`TypingHooks.get_tensor_like_type` 是前端扩展接缝，标准实现 `_TileTypingHooks` 返回 `TileTy`，使 IR 与具体张量表示解耦。
