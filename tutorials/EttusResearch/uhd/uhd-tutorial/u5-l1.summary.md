本讲讲解 UHD 纯 C API。因 C ABI 是跨语言的「最小公倍数」，无法直接消费 C++ 异常与 STL，故用一层 C 外壳把 `multi_usrp` 高层门面暴露出去。其核心契约是「错误码 + 不透明句柄 + 出参指针」三件套：每个函数都返回 `uhd_error`，业务结果经出参指针写回。

引入的关键术语：不透明句柄 `uhd_usrp_handle`（前置声明 `struct uhd_usrp*`，内部字段藏于实现）；C 结构体 DTO 如 `uhd_stream_args_t`；生命周期四件套 `find`/`make`/`free`/`last_error`，句柄须先 make 再用、用完必 free，否则段错误。

错误处理：`error.h` 的 `uhd_error` 枚举复刻 `uhd::exception` 派生体系；宏 `UHD_SAFE_C` 与 `UHD_SAFE_C_SAVE_ERROR` 把 C++ 异常 catch 后翻译成整数码，并将 `what()` 存入「全局 + 句柄」两处缓冲——故无句柄函数（find/make）只能 `uhd_get_last_error` 取错，有句柄函数两者皆可；`error_from_uhd_exception` 用 `dynamic_cast` 精确归类异常，全局串以单例加锁保线程安全。

实现要点：`usrp_c.cpp` 采用句柄注册表——句柄只存单调递增的整数索引 `usrp_index`，真正的 `multi_usrp::sptr` 放在进程级 `std::map` 里，经 `USRP(h)` 宏查表，兼顾 ABI 隔离与悬空句柄安全；每个函数自带一把互斥锁串行化同类操作，表明 C API 面向单线程串行驱动而非高并发。
