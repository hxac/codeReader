本讲义给出 UHD 公共头文件全景，聚焦 host/include/uhd/。布局为「顶层总入口（device.hpp、stream.hpp、convert.hpp 等）+ 子目录（types/usrp/rfnoc/transport/cal/experts/features/utils）」，公开与否由 CMakeLists.txt 的 UHD_INSTALL 决定，C 头文件额外受 ENABLE_C_API 控制。四个最小模块：config（config.hpp/config.h 跨平台编译宏翻译层，核心 UHD_API 依 UHD_DLL_EXPORTS/UHD_STATIC_LIB 在导出与导入间切换）；version（version.hpp.in 为 CMake 模板，经 configure_file 生成 version.hpp，UHD_VERSION 整数按四段版本号编码，分编译期宏与运行期查询函数）；build_info（编译器、依赖版本、启用组件等构建期元数据查询，为 uhd_config_info 数据来源）。关键结论：UHD 平行维护 C++ API（无伞式头、按需 include）与 C API（伞式头 uhd.h、可选组件），风格为异常 vs 错误码 uhd_error、std::string vs 出参缓冲。引入术语：公共头文件、ABI、伞式头文件、UHD_API、UHD_VERSION_ABI_STRING、configure_file 模板，为 device.hpp、stream.hpp 与 C API 绑定的精读奠定导航基础。
