本讲拆解 RFNoC 块控制器的四个核心模块。`noc_block_base` 是所有块控制器的基类，双继承 `node_t`（属性系统）与 `register_iface_holder`（寄存器访问），统一管理块身份（`_noc_id`、`_block_id`）、端口数与 MTU 策略；其构造函数为 `protected`，只能由框架工厂调用，保证身份不冲突。`block_id_t` 是块实例的三段式软件寻址 `DEVICE/BLOCKNAME#COUNTER`，靠正则校验合法性，靠 `match()` 实现「缺省即通配」的模糊匹配，并可转成属性树路径。`blockdef` 定义了块说明书（端口/参数/寄存器）的抽象接口，但在现代主机运行时未启用——块的端口数实际来自 FPGA 全局寄存器空间，描述主要靠工具链 YAML 维护。`registry` 用自注册模式在 `main` 之前把 `(noc_id, device_id)` → 工厂函数登记进直接注册表单例，运行时 `factory::get_block_factory` 经三级回退（精确设备 → ANY_DEVICE → DEFAULT_NOC_ID）查表造块。关键连接点是注册宏的 `BLOCK_NAME`，它最终构成 block_id 的名字段；同一 NoC ID 须在 `defaults.hpp` 常量、注册宏、YAML 三处保持一致。Radio 块按设备族各注册一个实现（指定 `DEVICE_ID`、`MB_ACCESS=true`），通用 DSP 块则用 `ANY_DEVICE`。
