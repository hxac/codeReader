本讲建立 UHD 仓库的「目录地图」。UHD 并非单一二进制，而是横跨主机与设备端的软件集合，顶层有六大目录：`host/`（主机用户空间的 UHD 库 `libuhd`，C++ 共享库加命令行工具与示例，本手册主角，默认 Release 构建）；`mpm/`（设备端嵌入式 SoC 上的外设管理常驻进程，Python 为主，与主机驱动 `mpmd` 通过网络/PCIe 通信）；`firmware/`（设备端微处理器固件，如 fx3 USB PHY、OctoClock，负责上电早期底层初始化）；`fpga/`（FPGA 的 Verilog HDL 镜像，按代 `usrp1/2/3` 组织，做实时高速数字信号处理与 CHDR/RFNoC 数据通路）；`images/`（维护者把固件/FPGA 镜像打包成发布包的脚本，普通用户改用 `uhd_images_downloader` 下载）；`tools/`（外围调试工具如 CHDR 解析插件，明确不属于 UHD 本体）。核心结论：按运行位置区分主机/设备端是理解仓库的关键视角；完整调用链为「程序 → libuhd → 设备端 MPM + firmware + fpga → 射频」。引入的关键术语有 libuhd、MPM、mpmd、用户空间、CHDR、RFNoC、镜像代。后续讲义聚焦 `host/`，下一讲 u1-l3 进入构建系统。
