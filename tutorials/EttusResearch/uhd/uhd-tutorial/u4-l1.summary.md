本讲深入 UHD convert 子系统，揭示 cpu_format 与 otw_format 之间转换的实现机制。converter 是纯虚抽象基类，公开 conv 作为 NVI 门面、真正逻辑藏于私有 operator()，set_scalar 携带收发两侧的缩放系数（接收 1/32767、发送 32767）。转换由 id_type 四元组（输入格式、输入数、输出格式、输出数）唯一标识；注册表是两级字典 dict<id_type, dict<priority, function>>，get_converter 默认返回该 id 下最高优先级工厂，使「同 id 多实现、自动择优」对调用方完全透明。实现经 DECLARE_CONVERTER 宏配合 UHD_STATIC_BLOCK 自注册；优先级常量 PRIORITY_GENERAL=0、PRIORITY_TABLE=1、PRIORITY_SIMD=3（x86）或2（ARM）。SSE2/AVX2/NEON 向量化版与通用版注册到同一 id、不同优先级，CMake 按 CPU 指令集互斥编译，保证同 id 下唯一 SIMD 实现；尾部样本交由 convert_common.hpp 的标量函数收尾。独立字节表 get_bytes_per_item 按下划线截断前缀递归回退，使 sc16_chdr、sc16_item32_le 命中基类型字节数。消费者 rx/tx_streamer 在开流时字符串拼接 id（RFNoC 加 _chdr 后缀、老设备用 _item32_le/be）并实例化转换器。
