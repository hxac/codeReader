本讲讲解 UHD 的 extension（扩展）框架，让外部射频前端的控制逻辑能在不改动 libuhd 主干、不重编译库的前提下接入 USRP 控制链路。扩展是一个实现 `uhd::extension::extension` 基类、被运行期 `dlopen` 加载的独立共享库（一种 module）。其生命周期为：编译期用宏 `UHD_REGISTER_EXTENSION` 自注册 → 进程启动时 `load_modules` 静态块遍历 `UHD_MODULE_PATH`、用 `dlopen` 触发静态初始化、把工厂函数写入全局 registry → 用户以 `extension=<名字>` 参数创建 `multi_usrp` 时经 `get_extension_factory` 查表实例化 → 该通道的射频核心被 `dynamic_pointer_cast` 成扩展，此后所有频率/增益调用都走扩展。基类同时继承 `core_iface` 与 `power_reference_iface`，这是扩展能「冒充」射频控制器、被 multi_usrp 调用的前提。官方示例 `extension_example` 演示了三种钩子风格（转发、改写、抛 `not_implemented_error`），并配合 experts 框架实现「写属性即重算」的联动，构建需 `find_package(UHD)`、编成 SHARED 库并安装到模块目录。最后说明向 UHD 贡献代码须遵守 CODING.md（clang-format、include 顺序、commit 风格，行为修改与格式清理分开提交）与 CONTRIBUTING.md 的 CLA 法律流程。
