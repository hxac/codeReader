本讲讲解 UHD 主机端如何用 CMake 把源码编译成 `libuhd.so`，拆成四个最小模块：构建配置、版本管理、依赖检测、编译产物。核心主线是 `host/CMakeLists.txt` 的执行顺序：默认 Release → `project()` 声明项目 → 载入自定义 CMake 模块 → 检测依赖 → `LIBUHD_REGISTER_COMPONENT` 注册组件 → `add_subdirectory` 进入子目录 → 打印组件汇总表。版本管理说明四段式 `MAJOR.API.ABI.PATCH`（当前 `4.10.0.0`）在 CMake 中同时生成字符串 `UHD_VERSION`、ABI 串 `UHD_ABI_VERSION`（取前三段，决定 `libuhd.so` 的 SOVERSION）和整数宏 `UHD_VERSION`（公式 MAJOR×1e6+API×1e4+ABI×1e2+PATCH）。依赖检测分编译器（要求 C++20）、Boost 子库、Python 模块（mako/numpy 等）、硬件传输库（libusb/DPDK）四类，结果直接作为组件依赖，自动级联决定组件是否启用。编译产物方面，`libuhd` 是文件名层面称呼，CMake target 名是 `uhd`，下游导入名是 `UHD::uhd`，三者同一物。关键术语：组件机制、缓存变量、`configure_file` 数据链、`uhd_config_info --enabled-components`。
