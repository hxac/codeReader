本讲为 RFNoC 单元首讲，聚焦「会话入口」。RFNoC（RF Network on Chip，片上射频网络）将 FPGA 内部改造为可路由的块网络：块（block，如 Radio/DDC/FFT）经流端点（SEP）由 crossbar 互连，样本以 CHDR 包搬运，软件可动态描述块间连接而免重新综合 FPGA；块在硬件以 NoC ID 标识，软件以 block_id（如 0/Radio#0）寻址。`rfnoc_graph` 是 `uhd::device` 的超集会话，除持有设备外还提供块发现、图连接、流器创建、主板控制等能力，不可拷贝但支持 `shared_from_this`。`rfnoc_graph::make` 复用 `device::make` 打开设备，再以 `dynamic_pointer_cast<rfnoc_device>` 做 RFNoC 资格检查（非 RFNoC 设备抛 `key_error`），并用 `dev_to_graph` 缓存保证「一设备一图」；构造期依次初始化 I/O 服务管理器、主板控制器、图流管理器（GSM，校验 CHDR 位宽同构）、逐主板枚举块造控制器、SEP 表、静态连接表，末尾零时刻同步。RFNoC 编程骨架为：建图 → `get_block` 取类型化块控制器 → `create_rx_streamer` 造空流器 → `connect`（三类重载：块-块、TX 流器-块、块-RX 流器）→ `commit` 触发校验与属性传播 → `recv`/`send`。边分四类：STATIC（综合期焊死，输出以 `==>` 表示）、DYNAMIC（运行时经 crossbar 路由）、RX_STREAM/TX_STREAM（块与主机流器连接，以 `-->` 表示）；`connect` 内部分「物理连 + 逻辑连」两步。设备层呈 device → rfnoc_graph → multi_usrp_rfnoc 三层，各有 weak_ptr 缓存，rfnoc_graph 为其中间枢纽。
