本讲解析 UHD Python 绑定的「两层架构」：编译扩展 libpyuhd 作引擎忠实翻译 C++，纯 Python 包 uhd 作外壳负责重命名与增强，import uhd 拿到的外壳最终落到引擎。引擎以 pyuhd.cpp 的 PYBIND11_MODULE 宏为唯一聚合入口，经数十个 export_xxx() 函数挂到 types/usrp/rfnoc/chdr 等子模块上。核心 pybind11 包装手法包括：py::class_<T,Base,Holder> 声明类与基类继承、py::init 把工厂函数注册成构造函数、overload_cast 或函数指针强转消解重载、lambda 配合 return_value_policy::reference_internal 适配返回类型与生命周期、trampoline 类（PyTimekeeper）让 Python 能覆写 C++ 虚函数。外壳处理引擎名字分三种手段：重命名（rfnoc.py/types.py 把 C++ 下划线名转 PEP8 驼峰名）、继承增强（MultiUSRP 追加 recv_num_samps 等便利方法）、猴子补丁（chdr.py 给 ChdrPacket 动态加 get_payload）。扩展须守 Python 运行时两条铁律：导入时 init_numpy 初始化 numpy C API 以支持 zero-copy 收发、进入长操作前用 py::gil_scoped_release 释放 GIL（见 device_python.cpp 的 device::find）。CMake 用 add_library(pyuhd)+PREFIX"lib"+EXT_SUFFIX 拼出 libpyuhd 并拷入 uhd 目录，使扩展与包入口同目录。
