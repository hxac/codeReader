本讲聚焦 device::find / device::make 最核心的数据结构 device_addr_t。它是继承自 dict<string,string> 的无类型键值容器，支持 args 字符串（逗号分隔键值对、等号分隔键值）、std::map、逐个 set 三种构造方式。同一容器在不同阶段有三种语义：作 find 入参时是「提示 hint」，作返回值时是回填的「设备档案」，附带 recv_buff_size 等键时则成为「传输层参数」。device_filter_t 是 {ANY, USRP, CLOCK} 三值枚举，表示粗粒度设备大类，在注册表遍历时与实现自注册的 filter 匹配，排除非目标类型（如 OctoClock）。find 另有第二道过滤，用 hint 的 name/serial/type/product 四键精确匹配，其中 serial 经 serial_numbers_match 按十六进制数值容错比较；注意 device_addr_t 的 type 字符串与 device_filter_t 枚举是两个不同概念。设备哈希 hash_device_addr 通过排序键、黑名单临时键（claimed、skip_*）与 resource 特判，保证「同一设备→同一哈希」，用于 find 阶段去重和 make 阶段以 weak_ptr 缓存实现设备复用。多设备场景则用 separate_device_addr / combine_device_addrs 在带数字后缀的单地址与地址数组间转换。本讲为后续 multi_usrp 高层 API 铺垫了地址与发现的底层机制。
