本讲承接 u1-l1 的架构俯瞰，把视角落到工程仓库，讲透「项目骨架与构建方式」，不涉及模块内部实现。核心覆盖三个最小模块：目录层级与职责、文件清单组织、编译运行脚本。仓库顶层分四大区：`rtl/`（全部可综合 RTL，再分为跨标量/向量复用的通用 IP `shared/` 与向量数据通路专用的 `vector/`）、`sva/`（仅仿真的 X 检查与断言）、`vector_simulator/`（TB、驱动、构建脚本与示例）、`images/`。RTL 文件靠两份 filelist 组织：`files_rtl.f` 按子系统分组列出可综合设计（顺序暗合数据通路：参数定义 → 通用 IP → 存储底座 → `vector_top` → vRRM → vMU → vIS → vEX），`files_sim.f` 仅列主存模型、自动生成参数、驱动与 TB 顶层。`compile_vector_simulator.do` 流程为 `vlib work` 建库 → `vlog -f` 编译两份清单 → `vsim` 加载 `vector_sim_top` → `log`/波形 → `run -all`。关键术语：RTL/TB/DUT、filelist、`` `include ``、`+incdir`、`MODEL_TECH` 宏。核心结论：三个 `+incdir` 各司其职（`rtl/shared/` 解析 params/structs，`rtl/vector/` 解析 vstructs/vmacros，`sva/` 解析断言）；`sva/` 断言通过 `` `include `` 加 `` `ifdef MODEL_TECH `` 注入 RTL 模块内部，仅仿真生效、综合时剔除，故不进入 filelist。
