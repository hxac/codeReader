本讲解析向量数据通路的「公共词典」——三份被各模块反复 include 的头文件。`vstructs.sv` 定义各级流水间传递的 packed 结构体：入口 `to_vector`、重命名后的 `remapped_v_instr`（新增 `ticket/lock/*_iszero/mask_src`）、存储岔路的 `memory_remapped_v_instr`、执行级的 `to_vector_exec`（数据）与 `to_vector_exec_info`（控制，含 `head_uop/end_uop`）、以及 `vector_mem_req/resp`；字段随流水级「逐级累加再精简」，即结构体＝「谁填谁用」的接口契约。它还定义整数操作码枚举 `v_int_op_t`（7 位，`VADD=7'b0000001`，`0000000` 留作非法），其编码必须与 `v_int_alu` 的 `case` 一一对应。`vmacros.sv` 用 `define` 宏给控制位起名：四种 FU 编码（`MEM/FP/INT/FXP`=00/01/10/11）、存储位段（`LD_BIT`、寻址模式 unit-strided/strided/indexed、元素宽度 SZ_8/16/32）、归约码及转发点宏（`EX1..EX4` 与 `_F` flopped 变体，权衡冒险解除与主频）。`vex_pipe` 用 `` `INT_FU `` 路由、`params.sv` 用 `EX1/EX4_F` 配转发点即其真实用例。关键隐藏耦合：`vstructs.sv` 用本地假常量 `DUMMY_VECTOR_LANES` 而非 `VECTOR_LANES`（因不 include `params.sv`），改 lane 数须手工同步。标量侧 `structs.sv` 中 `decoded_instr.is_vector` 是标量→向量分流标志；标量核未公开，该翻译由测试台/生成器软件代替。本讲只定义「数据形状」，为后续 `vrrm/vis/vex/vmu` 各模块提供接口基础。
