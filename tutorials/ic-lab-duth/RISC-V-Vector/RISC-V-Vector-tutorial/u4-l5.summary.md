本讲深入 `sim_generator.py` 这个离线译码器，聚焦三个最小模块。**CSV 解析与操作数识别**：`remove_garbage` 按首字符 `#`/`/` 剔注释；`get_operand` 用首字符 `v`/`#` 判寄存器/立即数，对立即数返回寄存器位恒为 0、数值进数据位；`get_number_vdst` 用 set 去重统计目的寄存器个数，作为 maxvl 推导依据（store 的源寄存器也被计入，是隐藏陷阱）。**助记符→microop 映射**：两张并行字典 `get_microop`/`get_fu`，未命中即 `exit`；核心结论是**真操作码是 `{fu, microop}`**——`vld` 与 `vradd` 同为 `1000000` 靠 fu 区分；访存 microop 按 `vmacros.sv` 位域排布：`[6]` load/store、`[5:4]` 寻址模式（unit/strided/indexed）、`[3:2]` 元素宽度。**VL 展开与气泡注入**：`maxvl=⌊32/N_vregs⌋×LANES`、`REPEATS=⌈AVL/maxvl⌉`、末轮跑尾数 `AVL−(REPEATS−1)·maxvl`，对应硬件 `vl_therm` 掩码；每轮末尾插 `TOTAL_BUBBLE_CYCLES=4` 条哨兵 `reconfigure=1, fu=11, microop=1111111, vl=0, maxvl=0`，五字段合取精确命中驱动器 `instr_is_dummy`。产物总行数 `SIM_VECTOR_INSTRS` 写入 `autogenerated_params.sv` 决定驱动器深度。关键结论：**译码层可用≠执行层可用**，在字典加指令只让生成器认识，要让 DUT 真执行还需改 ALU/SVA。
