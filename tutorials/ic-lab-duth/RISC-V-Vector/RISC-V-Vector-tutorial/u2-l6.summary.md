本讲聚焦向量发射级 vis.sv：一条向量指令如何被硬件展开成多个 micro-op，以及逐元素写回如何被掩码门控。

vis 用计数器 current_exp_loop 把一条 VL 元素的指令展开成一串 micro-op，每拍发射一组（VECTOR_LANES 个元素），uop 数 = ⌈VL/LANES⌉，上限 max_expansion = maxvl >> $clog2(LANES)。终止由 vl_reached（已覆盖运行时实际 VL）与 maxvl_reached（达 maxvl 上限）二选一触发，相或成 expansion_finished；尾部不满一组用 vl_therm 温度计屏蔽无效 lane。每个 micro-op 的 dst/src 指针 = 架构基址 + exp_loop，正好踩在 vRRM 预分配的连续物理块（步长 vreg_hop）上。

归约（reduction）是特例：目的指针除末组外一律 +1、末组指回基址，让计分板覆盖整段寄存器范围，真正写回只有末组 0 号元素，跨组累加由 vEX 归约树用 head_uop/end_uop 完成。谓词掩码则让 mask_src 随 exp_loop 同步 stride、读出 v1 每元素 LSB，vis 按 use_mask（2'b10 用、2'b11 取反、其余不掩码）生成 data_to_exec[k].mask 送 vEX 逐元素门控写回。需注意 use_mask 当前声明为单 bit 而代码按 2 位比较，谓词分支可能不生效，是待本地验证的位宽疑点。
