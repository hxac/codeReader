本讲介绍 Zenoh 的 liveliness（存活检测）——一个由协议托管的「分布式存在感」服务：它搬运的是 token 的存在状态而非业务数据，payload 恒空。从统一入口 session.liveliness() 出发提供三个原语：declare_token 声明与 Session 绑定的存活令牌，满足「未 drop、进程在世、网络可达」三条件即存活，任一不满足则全网消失，Drop 时自动注销；declare_subscriber 订阅 token 变化，Put=上线、Delete=下线，history(true) 走 InterestMode::CurrentFuture 补发当前存活 token，默认 Future 只听将来；get 拉取当前快照，走 InterestMode::Current，到点投 ReplyError("Timeout") 使接收循环自然退出。底层复用 Interest + Declare 协议、以 InterestOptions::TOKENS 标志位区别于普通订阅，匹配本质仍是 key expression 的 intersects。三原语可拼成「声明—订阅—查询—下线」的服务发现闭环。
