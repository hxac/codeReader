本讲承接 Session，引入 Key Expression（KE）作为 Zenoh 的地址空间：一条 KE 表示「一批 key 的集合」，消息匹配完全由两端 KE 的集合关系决定，而非 IP 地址。覆盖三个最小模块：①三种存储形式——`keyexpr`（≈`str`，借用零开销）、`OwnedKeyExpr`（≈`Arc<str>`，可廉价克隆）、`KeyExpr`（≈`Cow<str>`，公开 API，可携带 Session 声明优化），三者都 `Deref` 到 `keyexpr`。②规范化（canonization）是强约束，保证「字符串相等即集合相等」：`new` 须传规范形，`autocanonize` 可就地零分配改写（如 `**/*`→`*/**`、连续 `**` 合并）；语法以 `/` 分隔非空 chunk，禁用 `#`、`?`、空 chunk 及首尾斜杠，通配符 `*` 匹配单层、`**` 匹配任意多层。③`intersects`/`includes` 判断相交与包含，分别由 `ClassicIntersector` 与 `LTRIncluder` 实现，`SetIntersectionLevel` 枚举刻画四种关系偏序；Zenoh 网络所有匹配本质都是这类集合判断。
