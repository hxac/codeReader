本讲展开内部 crate `zenoh-buffers`，讲解 Zenoh 在收发与编解码时如何尽量不拷贝字节，它是公开 API `ZBytes` 与 `Zenoh080` 线编码的共同地基。核心三模块：`ZSlice` 是 `Arc<dyn ZSliceBuffer>`（类型擦除）上 `[start,end)` 半开区间的可克隆字节窗口，克隆与 `subslice` 仅动引用计数、负载零拷贝；`ZBuf` 是「一串 `ZSlice`」（内部用 `SingleOrVec` 优化单片），代表多段不连续内存的逻辑拼接，遍历、判等与 `push` 均不拷贝，仅 `contiguous()`/`to_zslice()` 在 ≥2 片时拷成连续内存；`Reader`/`Writer` 是面向多片缓冲的统一读写抽象，方法返回 `Result<NonZeroUsize, DidntRead/DidntWrite>`，`read_zslice`/`read_zbuf` 提供零拷贝切片读，`BacktrackableReader/Writer` 的 `mark()`/`rewind()` 支持回溯，是 codec「先占位后回填」的基础。`ZSlice` 类型擦除的 `buf` 与 `ZSliceKind`（Raw/ShmPtr）是后续共享内存跨进程零拷贝的接入点。
