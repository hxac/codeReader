本讲精读内部 crate `zenoh-codec`，回答协议消息如何与字节相互转换。四个最小模块：三大 codec trait、`Zenoh080` 编码标记与 header 字节机制、zint 变长整数、batch/frame 分派。核心结论：Zenoh 不用 serde，而定义 `WCodec`（写）/`RCodec`（读）/`LCodec`（算长度）三个 trait，把「消息类型」与「缓冲类型」都当泛型，`self` 位携带编码标记，从而支持 `no_std` 与零拷贝。`Zenoh080` 是空结构体编码标记，派生出 Header/Condition/Length/Reliability/Bounded/Sliced 等包装类型携带上下文。线格式约定为「1 字节 header + 可选扩展」：低 5 位（mask 0x1F）是消息 id，高 3 位是 flag（Z=扩展、M=映射、N=名字等），由 `imsg::mid`/`imsg::has_flag` 解析。zint 即 LEB128 变长整数，小数 1 字节、大数最多 9 字节。codec 靠「读 header → 按 mid 分派 → 委托子 codec」逐层工作，Frame 打包多条 network 消息，Batch 用 mark/rewind 保证事务性，未知扩展用 skip 跳过以实现前向兼容。
