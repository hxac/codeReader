本讲绘制 Zenoh 仓库的 crate 全景地图。Zenoh 是标准 cargo workspace，根 Cargo.toml 用 members 列出约 40 个子 crate，并统一锁定版本 1.9.0、最低 Rust 1.75.0。crate 严格分四层：commons 地基（zenoh-protocol/codec/config/keyexpr/buffers 等）→ io 层（zenoh-link 及各具体链路、zenoh-transport）→ 主实现与公开 API 的 zenoh → 顶层产物 zenohd/zenoh-ext/plugins/examples；依赖只能自上而下以防循环。核心结论：仅 zenoh 与 zenoh-ext 对外稳定，commons/*、io/* 皆内部 crate，不应直接依赖。引入的关键术语有 workspace members、依赖分层、稳定边界与 feature 层层转发——transport_tcp、shared-memory、unstable 等由 zenoh 经 zenoh-transport、zenoh-link 逐级下发到具体可选依赖，实现按需编译；zenohd 作路由器薄壳，额外强制打开 internal/plugins/unstable 等内部 feature。此依赖图与 feature 转发图是后续内部架构讲义的导航地图。
