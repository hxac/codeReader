本讲是「Session 与 Key Expression」单元的第一讲，拆解「打开会话」这一步，含三个最小模块。其一 open/Session：`zenoh::open(config)` 返回 `OpenBuilder`，须 `.await`/`.wait()` 才 resolve 成 `ZResult<Session>`；`Session` 本质是 `Arc<SessionInner>`，克隆便宜、可跨线程共享，最后一个克隆 drop 时自动关闭，也可手动 `close()`。其二 Config：`zenoh::Config` 字段私有，只能用 `from_json5`/`insert_json5`/`get_json` 读写；启动前可改任意 key，启动后经 `session.config()` 返回的 `Notifier` 仅能改 `plugins/` 开头的 key。其三会话信息查询：经 `session.info()` 读 `zid()`/`routers_zid()`/`peers_zid()`，`transports()`/`links()` 需 unstable feature。关键术语：ZenohId（节点唯一标识，不固定则每次随机生成）、WhatAmI 三种角色 router/peer/client（默认 peer，由顶层 `mode` 决定）。核心结论：公开 API 没有读本端 whatami 的方法，本端角色只能靠配置 `mode` 观察，`Transport::whatami()` 反映的是对端角色。
