本讲承接《u3-l1 Pub/Sub 基础》，讲解 Zenoh 的服务质量（QoS）三个正交维度。核心结论：一条消息是否「可丢弃（droppable）」由公式 `droppable = !reliable || (cc==Drop)` 决定；只有 `Reliable + Block` 组合才不可丢弃，会在发送端队列拥塞时阻塞发布者形成背压。Reliability 仅是线路标记（可能用于选链路，如 Reliable 走 TCP），Zenoh 自身不重传。CongestionControl 是队列满时丢（Drop，默认）还是等（Block）。Priority 决定「谁先发」，公开 API 暴露 7 档（数值越小越高，默认 Data），协议层另有一档内部专用的 Control；传输层为每个优先级各维护独立发送队列、按高到低服务，每档再分 reliable/best_effort 两套通道。QoS 在 PublisherBuilder 上设置（congestion_control/priority/express 默认可用，reliability 需 unstable feature），随后该 publisher 每次 put 沿用同一套 QoS。配置层有两套全局机制：传输层 QoS 开关（关闭后只剩一条 Data 队列、priority 分流失效，且与 lowlatency 互斥）和按 key expression 覆盖（QoS overwrite，可在 wait() 前改写 cc/priority/express，reliability 暂不支持）。droppable、背压、优先级分道、QoS overwrite 是后续内核与协议讲义承接的关键术语。
