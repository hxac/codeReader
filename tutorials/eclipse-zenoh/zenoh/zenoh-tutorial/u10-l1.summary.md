本讲梳理内部协议 crate zenoh-protocol 的消息模型。消息分四层并逐层嵌套：zenoh 层是数据体（Put/Del/Query/Reply/Err），network 层是路由信封（Declare、Push、Request、Response、Interest 等），transport 层是线路帧（建连握手 Init/Open、KeepAlive、Frame、Join），scouting 层是发现协议（Scout/Hello，不经帧、用 UDP 裸发）。嵌套链 Put ⊂ Push ⊂ Frame 体现「每往上一层补一类元信息」：数据 key 在 network 层补、序列号与 reliability 在 transport 层补。core 模块是各层共享的「词法表」，定义原子类型 ZenohId、WhatAmI（比特位编码加 WhatAmIMatcher 掩码）、WireExpr（scope id + suffix + mapping，是 key expression 的线上形态）、Timestamp、Parameters、Encoding 及 Priority/Reliability/CongestionControl 三个 QoS 枚举。核心结论：声明式路由由 Declare（声明订阅者/可查询者等）与 Interest（声明对声明的兴趣，含四种 mode）在控制面塑造订阅表，Push/Request/Response 在数据面按表转发；只有 network 层消息直接携带 WireExpr，数据体本身不带 key，由此实现「数据与地址解耦」，便于同一条数据零拷贝转发到多个 face。所有消息采用固定 header 加可选 ext 扩展的 TLV 风格。
