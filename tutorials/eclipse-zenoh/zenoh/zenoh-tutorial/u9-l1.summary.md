本讲进入 Zenoh 协议栈最底层的 Link 层，讲解如何把一个 endpoint（如 `tcp/127.0.0.1:7447`）变成可读写的字节连接。核心抽象分单播、多播两套：`LinkManager`（有状态，持有监听 socket 与 accept 循环，负责创建/监听/删除连接，经 `flume` channel 把新链路上交 transport 层）与 `Link`（已建立连接的收发句柄，暴露 `get_mtu/is_reliable/is_streamed/write/read/close`）。二者差异源于「点对点连接 vs 无连接组通信」：多播 manager 只有 `new_link`，其 `read` 须返回来源 locator，且 `LinkMulticast` 自带 `Zenoh080` 编解码——一个数据报即一个完整传输消息。

`LocatorInspector` 是建连前的质询官，凭协议串判断链路是否可靠、是否多播（多播按 IP 是否落入 224.0.0.0/4 或 ff00::/8 判定）；可靠性遵循「locator metadata 覆盖优先、协议默认值兜底」。协议串经 `LinkKind` 枚举作 tag，由 `LinkManagerBuilderUnicast/Multicast::make` 用 `match` 分派到具体实现。结论上，TCP 天然可靠+字节流（MTU≈65535，生效再扣包头取 MSS 倍）；UDP 默认不可靠+数据报（MTU 受 OS 限制），但可经 metadata 标注后在 UDP 之上跑 QUIC 升级为可靠链路；多播仅 UDP 支持，采用「单播 socket 写 + 多播 socket 读」的双 socket 模型。MTU 等参数多为 `zconfigurable!` 运行时可调静态量。本讲为 u9-l2 传输层管理提供零件基础。
