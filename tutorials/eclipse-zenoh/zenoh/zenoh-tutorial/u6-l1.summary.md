本讲讲解 Zenoh 的节点发现机制 scouting，含三个最小模块：scout 发起、Hello 名片、Scout/HelloProto 协议消息。核心结论：scouting 不是收发数据的前提，只要在 Config 写死 connect/endpoints 即可直接建连；scouting 只是省去手写对端地址的便利机制，失败时静默不 panic。引入的关键术语：zenoh::scout(what, config) 返回 ScoutBuilder，经 callback/with 取数、resolve 后得到 Scout 句柄；WhatAmIMatcher 用位掩码过滤 router/peer/client 角色；公开 Hello 是对内部 HelloProto 的稳定封装，仅暴露 zid()/whatami()/locators()；网络层 Runtime::scout 用 UDP 多播发 Scout、被发现节点单播回 Hello，发送周期指数退避 1s→8s，提问方只处理 Hello。该机制为后续 liveliness 与 orchestrator 建连埋下伏笔。
