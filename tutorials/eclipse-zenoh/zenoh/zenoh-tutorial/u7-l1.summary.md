本讲是「用法→原理」的过渡站，拆解三个最小模块。**Session 与 SessionState**：公开 `Session` 是 `#[repr(transparent)]` 的 `Arc<SessionInner>` 薄壳，自管引用计数以实现「最后一个 drop 自动 close」；`SessionInner` 持有 `runtime` 与 `state: RwLock<SessionState>`，后者是登记所有 publishers/subscribers/queryables/resources 的实体仓库，并以 `Option<Arc<dyn Primitives>>` 持有出站出口。**net 模块与 Primitives 契约**：`net/` 是稳定 API 与内部实现的分界线，`Primitives` trait 把出站动作（`send_declare/send_push/send_request` 等）收口，API 层从不直接碰 socket。**Runtime 与 RuntimeState**：承载节点全部连接状态，持有 `zid`、`whatami`、`Gateway` 路由器、`TransportManager`、`HLC`、可热更新 `config`。核心结论是：`zenoh::open` 的执行链为 `RuntimeBuilder::build → Session::init → Runtime::start`（先接线再通电）；`init` 经 `runtime.new_primitives → Gateway::new_session` 开一张 `Face`，它双向使用——对 Session 是出站 `Primitives`，对路由器持有入站 `e_primitives` 回调。关键术语 `GenericRuntime`（兼容独占与 zenohd 多插件共享 Runtime 场景）、`Face`、`EPrimitives` 将由后续讲义承接。
