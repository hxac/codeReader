本讲聚焦 `zenoh/src/net/primitives/` 下三个文件，用两张「投递契约」缝合路由层与传输层。`Primitives`（入口，pub）把消息交进路由织网，由本地 API `Session` 与入站 `DeMux` 调用；`EPrimitives`（出口，crate 内部）把消息送出某张 face，返回 `bool` 表示是否真发出——据此在拦截器挡掉时补发 `ResponseFinal`/`DeclareFinal`，避免请求方挂死。二者方法几乎一一对应，差异集中在可见性、`RoutingContext` 信封、`bool` 返回值与 `consume`/`send_close`。`DeMux` 实现传输层回调 `TransportPeerEventHandler::handle_message`，按 `NetworkBodyMut` 类型把入站消息分用到 `Face` 的 `Primitives` 方法；`Mux`/`McastMux` 实现 `EPrimitives`，把消息包成 `NetworkMessageMut` 后经拦截器放行再 `schedule` 到 `TransportUnicast`/`TransportMulticast`。`Face` 双向使用，既是路由入口又持出口；一条 `Put` 的出入站复用同一个 `route_data`。`gateway.rs::new_transport_unicast` 把 `Mux`（出口）与 `DeMux`（入口）缝在同一张 `Face` 上，`new_session` 则以本地 `WeakSession` 作为本地 face 的出口。本讲引入并供后续承接的关键术语：`Primitives`/`EPrimitives`、`DeMux`、`Mux`/`McastMux`、`Face`/`FaceState`、`RoutingContext`、`route_data`、`TransportPeerEventHandler`、入站/出站拦截器及其兜底约定。
