本讲承接《u8-l1 路由骨架》，讲解 Zenoh 路由层的策略接口 HAT（分层自适应拓扑）。核心是把「一条消息转发给谁」从 Gateway 骨架中解耦为 `HatTrait`——它是 `HatBaseTrait + HatInterestTrait + HatPubSubTrait + HatQueriesTrait + HatTokenTrait` 五合一空 trait，是策略模式的落点，决策核心是 `compute_data_route`（对来源 face 无感知、结果带缓存版本语义）。Gateway 构建时按 `(region.bound(), 角色)` 装 HAT：North+Client→client HAT、South+Client→broker HAT、Peer→peer HAT、Router→router HAT，一个节点可同时持有多个 HAT。关键术语包括 region 的 North/South 朝向、`owned_faces`、`routes_version`。router HAT 维护全网链路状态图 `Network`，订阅按 router zid 记录，转发时查最短路径树 `trees[src].directions[sub]` 得唯一下一跳 face，靠树的无环性跨多跳无重复投递；`TreesComputationWorker` 把树计算异步化到后台每 100ms 批量重算，避免阻塞数据快路径，代价是约 100ms 最终一致延迟。peer HAT 用 `Gossip`/`Network` 仅做发现与去重，不算树，对所有带订阅者的直连 face 洪泛加兴趣剪枝。client HAT（朝北、断言单一上游）与 broker HAT（朝南、直连 client 间转发、不检查 src_region）是一对最简镜像，二者都不维护拓扑。
