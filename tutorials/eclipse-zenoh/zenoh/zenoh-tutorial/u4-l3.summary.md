本讲承接《u4-l1 Queryable》《u4-l2 Get》，专门讲清「一条查询带了多少附加信息」，核心结论是 **Selector = Key Expression + Parameters**。Selector 是形如 `key_expr?name=value;...` 的 URL 式字符串，解析时按**第一个** `?` 切两段，无 `?` 则参数为空；可经 `FromStr`/`TryFrom` 与字符串双向 round-trip。**Parameters** 是架在一段 `Cow<str>` 上的「类 HashMap 零拷贝视图」：`;` 分隔键值对、`=` 分隔键与值、`|` 拆多值；每对按第一个 `=` 切，无 `=` 则值为空串，空片段与末尾多余分隔符被忽略。读操作（`get`/`iter`/`values`）全程借用原串、零分配，仅写操作（`insert`/`remove`）才把 `Cow` 物化。参数分两类：**自定义参数**（应用定义、字母数字开头并建议加前缀）与**标准化参数**（下划线开头、由库自身解释，目前有 `_time`[unstable] 与 `_anyke`）。Queryable 端用 `query.parameters()` 取参并以 `get`/`iter`/`values` 消费，是查询条件真正落地处。引入术语：Selector、Parameters、三个分隔符、ZenohParameters trait、`_time`/`_anyke`、ReplyKeyExpr::Any。
