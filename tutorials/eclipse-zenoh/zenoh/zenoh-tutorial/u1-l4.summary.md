`zenoh/src/lib.rs` 是 crate 公开 API 的**门面**：它把私有实现层 `api/`（`pub(crate)`）与 `net/` 里的类型挑选后，用 `pub use` 重新组织成一个个 `pub mod` 门面模块呈现给用户，借此在不破坏用户代码的前提下自由重构内部，体现了 Zenoh 的「稳定边界」。顶层还集中了根级符号 `open`/`Session`/`Config`/`scout`/`Result`/`Error`，其中 `Result` 是 `zenoh_result::ZResult` 的别名。公开模块共 13 个，分四组：会话配置（`session`/`config`）、通信范式（`pubsub`/`query`）、数据地址（`key_expr`/`sample`/`bytes`/`time`）、机制支撑（`handlers`/`qos`/`scouting`/`liveliness`/`matching`），这张对照表是后续讲义的目录。几乎所有「创建实体」都走 **builder 模式**，统一由三大 trait 支撑：`Resolvable`（标记 + 目标类型 `To`）、`Wait`（同步 `wait()`）、`Resolve`（综合上界且带 `#[must_use]`），使 builder 既能 `.await` 又能 `.wait()`。可见性由 feature 分级：默认稳定、`unstable`（公开但可能变动，或可稳定）、`internal`（暴露实现细节、本质不稳定，供 binding 与插件使用），两者分别由 `#[zenoh_macros::unstable]`/`#[zenoh_macros::internal]` 展开成 `#[cfg(feature = ...)]`，后者额外 `#[doc(hidden)]`；`plugins` feature 更用 `compile_error!` 强制 `unstable+internal` 同开。
