本讲是 Zenoh 学习手册的第一站，只读三个文件（README、根 Cargo.toml、zenoh/src/lib.rs），不写代码，建立全局地图。核心结论：Zenoh 是一套统一的网络协议栈，用一套 API、一套协议把 Pub/Sub（data in motion，流动数据）、Store/Query（data at rest，静止数据）与 Compute（把计算搬到数据处）整合在一起，主打零开销；本仓库是 Zenoh 协议的 Rust 参考实现，其他语言多为绑定。使用视角上，用户调用 `zenoh::open(config)` 得到 Session，在其上声明实体，走两种通信范式之一（Pub/Sub 或 Query/Reply），底层由 Zenoh 协议自动处理路由与匹配，节点可组成 mesh/star/clique 拓扑，角色由 config 的 mode 决定（router/peer/client）。仓库分六大块：zenoh（主实现）、zenoh-ext（扩展库）、zenohd（路由器二进制）、plugins（插件）、commons/io（内部地基）、examples。关键稳定边界：仅 zenoh 与 zenoh-ext 提供稳定 API，commons、io（如 zenoh-protocol、zenoh-codec、zenoh-transport、zenoh-link-*）属内部实现，不保证稳定，写应用不应直接依赖，但读源码理解原理时它们是主角。所有内部 crate 共享统一版本号并互相严格锁定。关键术语：Session、Publisher/Subscriber、Querier/Queryable、Key Expression、Sample、ZBytes、Selector、Config/WhatAmI、scouting、builder 模式及其根基 Resolvable/Resolve/Wait 三 trait，以及 unstable/internal feature 门控机制。`lib.rs` 是公开 API 门面，把内部模块整理为 session、config、key_expr、pubsub、query、bytes、handlers、qos、scouting、liveliness、matching、time 等模块，后续每讲对应其中一两个。
