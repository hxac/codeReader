本讲进入 Zenoh 的查询/应答（Query/Reply）链路，即 data at rest 的「供给侧」。与 Pub/Sub 的 push 相对，它是 pull 模型：请求方问一次、应答方答一次，故 Queryable 天然适合暴露当前状态、历史与持久化数据，是 Zenoh 存储后端的底层形态。

讲义分四个模块：其一，Queryable 由 Session::declare_queryable(ke) 经 QueryableBuilder 配取数姿势后 .await 注册，句柄与 Subscriber 对称（Deref 到 handler、Drop 自动反声明）；入站查询由 handle_query 按 origin、complete、KE 相交三条件过滤后派发，命中者各得一份克隆的 Query。其二，Query 携带 key_expr（可含通配符）、parameters、payload、attachment，回 reply 须用具体 key 而非声明 KE。其三，三种应答 reply（Put）、reply_del（Delete）、reply_err（错误），前两者默认要求应答 key 与查询 KE 相交并沿用查询 QoS，应答结束后由 QueryInner::Drop 自动发 ResponseFinal。其四，应答在请求侧封装为 Reply＝Result<Sample, ReplyError>。

引入的关键术语：Queryable、Query、QueryableBuilder、QueryInner、三种 reply 方法、Reply、ReplyError、ReplyKeyExpr（accepts_replies/_anyke 控制是否允许不相交应答）、complete、Locality、intersects 匹配与 data at rest。发起查询的 get/Querier 留待后续讲义承接。
