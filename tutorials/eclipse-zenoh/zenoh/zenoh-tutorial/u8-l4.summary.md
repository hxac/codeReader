本讲回答 router/peer 如何发现全网拓扑并互联，覆盖三个最小模块：链路状态数据模型、Zenoh080Routing 线编码、router 的 Network 与 peer 的 Gossip 引擎。

核心数据模型 LinkState 是「节点名片」：含发送方本地编号 psid、单调递增版本号 sn、邻居列表 links（存邻居的 psid）及按选项位按需出现的 zid/whatami/locators/边权重/is_gateway，多条打包为 LinkStateList。关键技巧是 psid≠zid：较长的全局 zid 仅首次附带，接收方用 psid↔zid 映射还原成 LocalLinkState。Zenoh080Routing 是空结构体编码标记，序列化后塞进 OAM_LINKSTATE(0x0001) 消息搭车传输。

Network（router HAT）做完整链路状态：用 petgraph 维护拓扑图，按 sn 判新鲜度增删节点/边、删孤立节点、autoconnect、洪泛，再对每个根跑 Bellman-Ford 算最短路径树，directions[dst] 即「去 dst 的唯一下一跳」，树计算由 TreesComputationWorker 异步、每 100ms 合并重算。Gossip（peer HAT）共用同一套线协议但更轻，默认单跳传播，服务发现、autoconnect 与多网关去重，不算最短路径树。边权取两端声明权重较大者（默认 100）并加 zid 哈希确定性抖动；LinkInfo 仅本地派生观察，不上线。
