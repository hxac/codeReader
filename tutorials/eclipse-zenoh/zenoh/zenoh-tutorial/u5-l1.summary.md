本讲打开 `Sample` 的 `payload` 黑盒，讲解两个稳定类型 `ZBytes` 与 `Encoding`，承接《u3-l1》并铺垫《u5-l2 序列化》与《u12-l1 共享内存》。

`ZBytes` 是零拷贝字节容器，内部透明包装多片段 `ZBuf`（由若干 `ZSlice` 串成）。核心判据：取连续字节用 `to_bytes()` 返回 `Cow<[u8]>`，单片段零拷贝、多片段才拷贝；`slices()` 永不拷贝；`try_to_string()` 做 UTF-8 校验故可失败。构造规律：传借用（`&str`/`&[u8]`）会拷贝，传拥有（`String`/`Vec<u8>`）则移动不拷贝负载，`bytes::Bytes` 可零拷贝包裹。

`ZBytesReader` 实现 `Read`+`Seek` 供顺序/可回退读取；`ZBytesWriter` 实现 `Write`，双容器结构：`write_all` 拷贝字节进内部 `vec`，`append` 嫁接已有 `ZBytes` 的片段不拷贝内容，是实现零拷贝拼装的关键。

`Encoding` 是 MIME 风格的解读标签（`type/subtype;schema`），核心结论是 Zenoh 协议只搬运、不解释它，正确性由应用负责。常用编码内部映射为小整数 id 省带宽，自定义编码用 `0xFFFF` 标记并存入 `schema`；已知 id 无 schema 转字符串返回 `Cow::Borrowed` 零分配。`Publisher::put` 接受任意 `Into<ZBytes>` 并携带 `encoding`，订阅端从 `Sample` 读回相同的 `payload` 与 `encoding`。
