本讲承接《u4-l1》的 Query/Reply 供给侧，转入请求侧，覆盖三个最小模块：`Session::get`（一次性查询）、`Querier`（声明式可复用查询器）、`Reply` 与结果合并（Consolidation）。`Session::get(selector)` 返回 `SessionGetBuilder`，查询在 builder resolve 时由 `Session::query` 真正发出，`querier_id` 为 `None`，适合低频临时查询。`Querier` 通过 `declare_querier` 声明，固化的 key 经 `declare_querier_inner` 注册并向网络发送持久 `Interest`（`CurrentFuture`），后续每次 `get()` 只携带可变 payload/parameters，省去重复解析编码并支持匹配感知。核心术语：`Reply = Result<Sample, ReplyError>`，用 `result()` 区分成功与错误，错误应答绕过合并直接送达。两个正交维度——`QueryTarget`（BestMatching/All/AllComplete，决定问谁）与 `QueryConsolidation`/`ConsolidationMode`（None/Monotonic/Latest，决定合并方式）。默认 `BestMatching + AUTO` 解析为 `Latest`：按应答 key 与时间戳去重并攒到所有来源答完（`nb_final` 归零）才冲刷，故同 key 多个 Queryable 默认只露一条；要让各副本回答全可见，需 `target=All` 配 `consolidation=None`。查询参数含 `time_range` 时 AUTO 改解析为 `None`。取数循环 `recv_async` 靠 `nb_final` 归零或超时自然退出，无需手写 break。
