本讲建立仓库「阅读地图」。每章核心代码都在 `chXX/01_main-chapter-code/` 下，固定含主 notebook `chXX.ipynb`（正文逐行演进的入口）、汇总器 `previous_chapters.py`、summary 脚本（如 `gpt.py`、`gpt_train.py`）和 `exercise-solutions.ipynb`，顶层 README 目录表是这套约定的出处，`(summary)` 标记即汇总脚本。

核心机制 `previous_chapters.py` 是「精选成品汇总器」：只收后续章节会复用的稳定类/函数、不放 notebook 里的试错过程，并随章节逐层累积——ch04 版止步于 `MultiHeadAttention`，ch05 版又追加了 `GPTModel` 等第 4 章成品。当前章 notebook 用 `from previous_chapters import ...` 复用旧代码；本地缺该文件时，可改用 PyPI 包 `llms_from_scratch`（对应仓库 `pkg/` 目录）。

summary 脚本分两种复用模式：完全自包含（如 `ch04/gpt.py`，旧代码全部内联、单文件即可运行）与依赖模块（如 `ch05/gpt_train.py`，开头有 `from previous_chapters import`、须与汇总器同目录）。判断某脚本属于哪种，就看它开头是否出现这行 import。
