本讲承接 `u2-l1` 的词级分词器，讲解**字节对编码（BPE）**与工业级分词库 **tiktoken**。核心动机：词级分词遇到词表外词汇必须用 `<|unk|>` 兜底，而 BPE 初始词表覆盖全部 **256 个字节值**，任何生词都能拆成**子词（subword）**被稳定编码，因此**不需要 `<|unk|>`**。三个最小模块：①**BPE 原理**——反复合并当前序列中出现频率最高的相邻 token 对，记入合并表 `bpe_merges`，在「整词太粗、逐字符太细」间取得平衡；②**从零实现** `BPETokenizerSimple` 的 `train`/`encode`/`decode`，揭示合并规则如何攒出与套用，并引入 GPT-2 用字符 `Ġ` 表示空格的约定；③**tiktoken** 加载预训练 GPT-2 词表（50,257），`encode` 时须用 `allowed_special` 显式放行 `<|endoftext|>` 等特殊 token。词表大小是超参（GPT-2 为 50,257、`cl100k_base` 为 100,256、`o200k_base` 为 199,997）。本讲打通「文本→token ID」，所得 ID 序列即下一讲滑动窗口采样与嵌入层的输入。
