本讲是第 2 章文本数据流水线的收尾，解决「把整数 token ID 变成模型可处理的连续向量」这最后一步。核心模块有三个：一是 **token 嵌入层**，`nn.Embedding(vocab_size, emb_dim)` 本质是可学习的查表，输入 ID `i` 返回权重矩阵第 `i` 行，等价于 one-hot 与矩阵乘法，权重经反向传播训练，使语义相近的词在向量空间中靠近；二是**位置嵌入**，因为同一 token 在任意位置都映射为同一向量、且自注意力本身无序，必须额外注入顺序信息，GPT-2 用可学习的**绝对位置嵌入** `nn.Embedding(context_length, emb_dim)`，其行数 `context_length` 决定模型最大序列长度（GPT-2 small 为 1024）；三是**两层相加**，最终输入嵌入 = token 嵌入 + 位置嵌入，靠广播补齐 batch 维，故二者 `emb_dim` 必须一致，真实模型中相加后还过 Dropout。产出张量形状为 `batch × num_tokens × emb_dim`，即下一章注意力机制的直接输入。关键术语包括 `nn.Embedding`、查表、绝对位置嵌入、广播、`emb_dim`、`context_length`，其中可学习绝对嵌入与正弦位置编码、RoPE 的对比为后续讲义埋下承接。
