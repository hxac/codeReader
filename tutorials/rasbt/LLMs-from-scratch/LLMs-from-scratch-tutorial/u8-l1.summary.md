本讲是对支撑全书的 PyTorch 基础的系统回补，不引入新 LLM 知识，而是把前序章节用过但未细究的底层机制讲透，为后续 u8-l2、u8-l3 打底。它组织成四个最小模块：**张量（tensor）**——PyTorch 统一数据容器，按维数分 0D/1D/2D/3D，权重必须是 `float32` 才能求导，矩阵乘法用 `@`，并与 NumPy 有复制与共享内存两种互操作；**自动微分（autograd）**——前向时构建计算图（DAG），反向时用链式法则逐节点累加梯度，区分 `torch.autograd.grad`（显式求导、不写 `.grad`）与 `loss.backward()`（一次性写入所有叶子参数的 `.grad`），并强调 `.grad` 默认累加故须 `zero_grad` 清零；**`nn.Module`**——所有层与模型的基类，自动注册子模块与参数（`nn.Parameter` 默认 `requires_grad=True`），约定只重写 `forward` 而用 `model(x)` 调用，靠 `parameters()`/`state_dict()` 统一管理权重；**通用训练循环范式**——`train() → forward → loss → zero_grad → backward → step → eval()`，配套 Dataset/DataLoader、可微代理损失（交叉熵/MSE）、不可微的 `argmax` 准确率（只汇报不优化）、同设备约束与 `state_dict` 存取（只存权重不存结构）。本书第 5–7 章所有训练均为此骨架的变体。
