本讲是第 2 章文本数据流水线的收尾，负责把前两讲得到的 token ID 序列切成模型能批量吃下去的训练样本。核心围绕语言模型的训练目标「下一个 token 预测」：训练样本必须成对出现，input 取一段上下文，target 是其整体右移一位的序列，使一个 batch 同时编码多个预测关系。本讲引入两个关键实现——GPTDatasetV1 把滑动窗口封装成 PyTorch Dataset，在初始化时一次性切好所有样本存入列表；create_dataloader_v1 是工厂函数，内部写死 GPT-2 BPE 分词器，并用 DataLoader 完成分批、打乱、丢尾。四个关键参数为 batch_size、max_length、stride、drop_last，其中 stride 是核心权衡旋钮：取 1 样本多但高度重叠、易过拟合，取 max_length 样本少但彼此独立，训练时常用后者。本讲产出的 dataloader 输出形状为 (batch_size, max_length) 的 token ID 张量，直接作为下一讲嵌入层的输入接口。
