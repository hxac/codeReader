本讲把 Transformer 块里的稠密前馈层（FFN）替换为**混合专家层（MoEFeedForward）**，核心思想是用「大总参数量换小单步激活量」实现稀疏激活：把一个 FFN 复制成多个并列**专家（expert）**，但每个 token 只激活其中 `top_k` 个，激活参数约占总参数的 `top_k/num_experts`。关键结论：MoE 输入输出形状与稠密 FFN 一致（均为 `(batch, seq_len, emb_dim)`），`TransformerBlock` 仅用一行三元判断即可无缝切换，注意力与残差结构不动。

引入的关键术语：**门控/路由器（gate）**——轻量线性层为每个 token 产出 `num_experts` 个分数；**top-k 选择**——取分数最高的 k 个专家；**softmax 加权**——仅对被选中的 k 个分数归一化（保证权重和为 1）；**按专家分组稀疏计算**——用 `torch.unique` 只遍历被激活专家、收拢同专家 token 一次算完，再用 `index_add_` 累加回原位实现加权求和；**SwiGLU**（`silu(fc1(x))*fc2(x)` 再经 `fc3`，每专家 3 个矩阵，区别于 GELU 版的 2 个矩阵）；以及**共享专家（shared expert）**概念（本讲未实现但结构兼容）。

配置新增 `num_experts` 与 `num_experts_per_tok`（即 top_k）两个字段。权衡上 MoE 以更低的激活显存换取更高的前馈计算开销与负载均衡等训练难题（本讲未涉及负载均衡损失）。
