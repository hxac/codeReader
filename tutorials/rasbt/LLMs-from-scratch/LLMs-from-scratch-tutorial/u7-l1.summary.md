本讲构建指令微调（SFT）的数据流水线，含四个模块。其一采用 Alpaca 风格模板，format_input 将原始 instruction/input/output 三字段拼成结构化输入（空 input 整段省略）。其二 InstructionDataset 在初始化时预分词整条「指令+### Response:+回答」文本，返回不等长 token ID 列表，将对齐推迟到 collate。其三为核心 custom_collate_fn：以 50256（<|endoftext|>）做批内填充对齐，构造右移一位的 target，并用 ignore_index=-100 做损失掩码——保留首个结束符作为「回答结束」信号，纯填充从损失中剔除。其四用 functools.partial 给 collate 预绑定 device 与 allowed_max_length=1024，挂到 DataLoader，数据按 85/10/5 划分为训练/测试/验证。关键结论：cross_entropy 默认跳过 -100（不计入分子分母）；本实现只掩蔽填充、不掩蔽指令段（prompt 段掩蔽留作练习）。训练循环复用第 5 章的 train_model_simple，留到 u7-l2。
