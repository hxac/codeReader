本讲解决「如何给语言模型打分」，承接 u4-l4 那个能跑但只产乱码的模型，将其变成可用标量分数衡量的模型，但不涉及优化（训练循环留给 u5-l2）。核心拆为四个最小模块：(1) **交叉熵损失**：把「下一个 token 预测」的训练目标数值化为可最小化的损失，路径是 softmax 出概率 → 取出模型给目标 token 的概率 → 取对数 → 求平均 → 取负，即 $\mathcal{L}_{\text{CE}}=-\frac{1}{N}\sum\log p_{t^{(n)}}$，PyTorch 的 `cross_entropy` 内部数值稳定地一次算完；取负是为与深度学习「最小化损失」约定一致。(2) **calc_loss_batch / calc_loss_loader**：前者用 `logits.flatten(0,1)` 把 `(batch, num_tokens, vocab)` 压成 `(N, vocab)` 交 `cross_entropy` 一次算出带梯度的标量损失；后者遍历 loader 用 `.item()` 摘成 float 累加求均，仅供汇报、不带梯度。(3) **evaluate_model**：以 `model.eval()`（关 dropout）+ `torch.no_grad()`（不建计算图）营造干净评估环境，靠 `num_batches=eval_iter` 抽样提速，最后 `model.train()` 切回。(4) **困惑度 PPL = exp(损失)**，即「有效词表大小」。最关键结论：未训练模型输出近似均匀，故初始损失 ≈ log V、初始困惑度 ≈ 词表大小（GPT-2 约 50257）。引入的关键术语包括交叉熵损失、展平技巧、干净评估、困惑度，供后续讲义承接。
