本讲把读者从「知道项目长什么样」推进到「跑出第一段模型生成」，覆盖三个最小模块：依赖安装、运行 `gpt.py` 入口、设备与推理模式。依赖安装最快路径是在仓库根目录 `pip install -r requirements.txt`，清单逐行标注了「库 ↔ 章节」对应；仅跑 `gpt.py` 最少只需 `torch` 和 `tiktoken`。第一个可运行入口是 `ch04/01_main-chapter-code/gpt.py`，它把第 2~4 章组件汇总成独立脚本，`main()` 流程为：定义 `GPT_CONFIG_124M` 配置 → `torch.manual_seed` 固定权重 → 构建 124M 的 `GPTModel` → `model.eval()` 关闭 dropout → tiktoken 编码 → `unsqueeze(0)` 补 batch 维 → `generate_text_simple` 自回归生成 → 解码。引入的关键术语包括：依赖、`requirements.txt`、推理模式（eval mode）、dropout、batch 维、贪心解码、token 编码/解码、`torch.no_grad()`。核心结论：该脚本故意只跑在 CPU 上，无 `.to(device)`；GPU/设备选择的 `torch.device(...)` 范式要到第 5 章训练才正式登场；未训练模型生成「语法像 token、语义是乱码」属正常，验证的是流水线跑通而非模型能力。
