本讲讲解 KV Cache 如何加速自回归推理。动机是：朴素生成每步都要重算全部历史 token 的 Key/Value，把它们缓存复用后，K/V 投影的累积开销可由 O(n²) 降为 O(n)。实现包含四个最小模块：(1) 在 MultiHeadAttention 注册两个非持久化缓冲区 cache_k/cache_v（首次初始化、之后沿 token 维 torch.cat 追加）；(2) 通过 use_cache 开关自 GPTModel 经 TransformerBlock 透传至注意力层，并以模型级 current_pos（索引位置嵌入）与每层 ptr_current_pos（切因果掩码）两个位置指针同步推进，使同一段掩码切片同时覆盖 prefill 与 decode 两种形态；(3) 每条新独立序列生成前必须 reset_kv_cache 清理脏缓存，否则结果错乱；(4) 增量生成函数 generate_text_simple_cached 把过程分为 prefill（一次喂整段 prompt）与 decode（每步只喂单个新 token）两阶段，use_cache=False 时退化为朴素路径。核心结论：缓存版须与朴素版输出逐位相同（以 torch.equal 验收），教学版在 CPU 上约 5× 加速；代价是显存随序列线性增长且只能用于推理，其 KV 内存问题由 u9-l3 的 GQA/MLA/SWA 进一步解决。引入的关键术语包括 KV cache、cache_k/cache_v、use_cache、current_pos/ptr_current_pos、prefill、decode、reset_kv_cache。
