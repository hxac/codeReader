本讲讲解如何用多张 GPU 协同训练同一模型，即 PyTorch 的数据并行 DDP（DistributedDataParallel）：每个进程持有一份完整模型副本，仅吃到的数据不同，解决"训练太慢"而非"显存装不下"，这与切开模型的模型并行相对。核心三件套：`init_process_group` 建立进程组通信总线（设会合地址、按系统选 nccl 或 gloo 后端、绑定本进程 GPU）；`DistributedSampler` 把数据按 rank 不重叠切分（需关掉 shuffle、每轮 set_epoch 换种子）；`DDP(model)` 包装模型，在 backward 时自动 all-reduce 对梯度求平均。关键标识为 rank（全局编号）/world_size（总进程数）/local_rank（本机 GPU 号），rank 0 常作主进程负责日志与存盘；启动有脚本内 `mp.spawn` 与外部 `torchrun` 两种方式，后者更适合多机。核心结论：各进程用相同的平均梯度更新相同权重，故模型副本始终逐位一致、无需额外平均模型；各进程训练损失因数据不同而不同，但在同一份测试集上准确率必然相同——这才是验证同步的正确方法。本讲给出把三件套迁移到第 4 章 GPTModel 的骨架，并指向 FSDP、混合精度等进阶主题。
