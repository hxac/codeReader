本讲以 `multi_bins_test` demo 为载体，解决「一个进程同时加载并运行多个网络」的工程问题，是进阶 demo 单元首篇，承接 u2-l3 的初始化三件套与 u6-l2 的检测后处理，聚焦「从单实例到双实例」带来的全部变化点。

讲义拆为四个最小模块：第一，**双 EEPTPU 实例**——从单实例的一个全局指针扩展为两个独立对象，靠可复用的 `eeptpu_init(EEPTPU*& tpu, ..., fg_multi)` 各服务一次，指针引用用于回传工厂方法 `init()` 新建的对象。第二，**多核寄存器 zone**——多核场景需把 core 1 的寄存器窗口（物理地址 `0xA0040000`）连同 core 0（`0xA0000000`）一并注册给运行库，核心结论是「注册哪个 zone，运行库就能驱动哪个核」，PCIe 模式则只注册 core 0。第三，**`fg_multi` 基地址策略**——第一个 bin（`fg_multi=0`）由用户显式 `set_base_address`，后续 bin（`fg_multi=1`）跳过显式设置、交由 `load_bin` 内部协调 DDR，以避免多网络张量在 DDR 互相覆盖。第四，**协同执行与资源管理**——分类与检测在同进程顺序执行，输入按各自 `input_shape` 分别预处理，退出时两实例均须 `close`+`delete`、结果缓冲须 `free`。工具链上 `compile.sh` 不编 npy 文件，`test.sh` 传两个 bin 加一张图。关键术语「待确认」标记反映了闭源库的内部实现边界。
