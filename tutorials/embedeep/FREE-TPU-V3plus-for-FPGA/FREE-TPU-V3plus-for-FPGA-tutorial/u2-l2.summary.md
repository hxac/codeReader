本讲解析 FREE-TPU V3+ demo 共享的图像工具库 **eepimg**，它底层用 stb 系列头文件做 jpg/png/bmp 编解码，在其上提供一层统一封装，解决所有 demo「读图、缩放、画框画字、存盘」的输入输出需求。核心分四个模块：一是数据结构 `image_bytes`，含 `w/h/c/data/layout` 五字段，描述一段 `unsigned char` 像素内存，字节数为 w×h×c，`data` 由库 `calloc` 分配，必须用 `eepimg_free` 成对释放。二是引入两个正交概念——**像素序**（RGB/BGR/GRAY/RGBA/BGRA，决定单像素内分量顺序）与**布局**（HWC 交错/CHW 平面），网络训练用哪种推理就喂哪种，否则颜色或通道错位；`eepimg_rgbgr_image` 做 R↔B 互换，`eepimg_hwc2chw` 做布局转换。三是读图主链路：`eepimg_load_image` 按像素序加载 → `eepimg_resize` 缩放到网络固定输入尺寸 → 喂 `eeptpu_set_input`；`input_shape` 为 NCHW，故传 resize 的是 `(input_shape[3], input_shape[2])` 即（宽,高）。四是可视化三件套 `draw_box/draw_text/save`，均遵循「先 copy 再改、返回新副本」设计，颜色参数顺序为 b,g,r，`save` 默认 `swapRB=true` 把内部 BGR 翻回 RGB 存盘。关键提醒：覆盖式调用会泄漏旧副本，demo 可容忍、实时流不可。本讲为 u2-l3 的 EEPTPU 推理 API 实战打好图像侧基础。
