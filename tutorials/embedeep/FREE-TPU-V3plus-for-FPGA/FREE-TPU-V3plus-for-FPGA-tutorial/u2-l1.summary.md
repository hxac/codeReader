本讲是 SDK 的「地图课」，将 `sdk/` 拆为四部分：`eeptpu_compiler`（编译器）、`libeeptpu_pub`（运行库）、`demo`（Linux 示例）、`standalone`（裸机工程）。

核心主线为：框架模型（caffe/darknet/pytorch(onnx)/ncnn/keras）经 `eeptpu_compiler` 编译成 TPU 硬件专属的 `*.pub.bin`，再由运行库或裸机驱动加载完成推理。bin 不只含权重，还打包了调度表、地址表与输入输出 shape。

引入的关键术语：**TPU bin**（硬件可直接执行的二进制）、**`EEPTPU` 高层类**（init/load_bin/set_input/forward 的调用顺序）、**`EEPTPU_SA`**（裸机版 TPU 对象，直接读写寄存器）、两个魔法地址 `EEPTPU_MEM_BASE_ADDR` 与 `EEPTPU_REG_BASE_ADDR`（由硬件设计定死，两条路线必须一致）。

核心结论：SDK 提供 Linux demo（高层 API、跨 ARM32/ARM64/x86）与裸机 standalone（直接驱动寄存器、串口菜单、延迟更低）两条部署路线；裸机侧用 `eepBinCvt` 把 bin 转成可 include 的 `eepnet.h`/`eepnet.mem`，关键开关集中在 `config.h`（如 `NET_TYPE`）。换网络只需动模型与编译器参数，右端推理与后处理已解耦，demo/standalone 代码基本不动。
