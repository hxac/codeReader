本讲把视线从「FREE-TPU V3+ 是什么」转向「代码长什么样」，建立仓库整体地图。FREE-TPU V3+ 跑在 Xilinx Zynq UltraScale+（ZynqMP）SoC 上，该芯片集成 ARM 处理器（PS）与 FPGA 可编程逻辑（PL）：TPU IP 放在 PL，ARM 软件经地址映射驱动它。仓库顶层分六个目录各司其职：`constr`（引脚与 bitstream 约束，如 `top.xdc`）、`doc`（中英成对手册）、`hardware`（可直接烧录的 BOOT.BIN、image.ub、xsa）、`ip_repo`（加密 TPU IP）、`script`（建 Vivado 工程的脚本）、`sdk`（软件）。其中 `ip_repo`+`script`+`constr` 是「造硬件」的输入，`hardware` 是其产物，`sdk` 是「用硬件」的软件，`doc` 是全程说明书。核心 IP 以 Xilinx ``pragma protect`` 加密形式黑盒交付，看不到内部 RTL；可读源码最集中的是 `sdk`，分 eeptpu_compiler（编译器）、libeeptpu_pub（运行库 API）、demo（classify/yolo/icnet 等）、standalone（裸机 C++ 工程）四部分，对应「框架模型 → eeptpu_compiler 编译成 *.pub.bin → libeeptpu_pub 加载推理（Linux）或 standalone 寄存器驱动（裸机）」的主线。关键术语：bitstream、BOOT.BIN、xsa、IP 核、PS/PL；手册编号 `ug050` 对应编译器、`ug053` 对应 API、`demo_readme` 对应上板。区分「二进制交付物」与「可读源码」是高效学习的关键。
