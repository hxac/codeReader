本讲回答「硬件产物怎么造出来」：一行启动器 `create_prj.sh` 以 `vivado -mode tcl -source $1` 批处理方式拉起 1929 行的 `system_rtl_*.tcl`，后者创建工程（目标芯片 `xczu15eg`、强制 Vivado 2021.1），在 `script/` 下执行以让 `../` 相对路径生效。TCL 的核心是搭 Block Design：用 `create_bd_cell` 放 PS(`zynq_ultra_ps_e_0`)、TPU(`EEP_TPU_0`，VLNV 实例化)、DVP 顶层(`EEP_DVP_Top_0`，module 引用加密 `.v`)、时钟向导(100M→250M/24M)、AXI 互连等约十个实例，再连线和 `assign_bd_address`。两条 AXI 通路是核心概念：控制通路 PS `M_AXI_HPM0_FPD`→互连→各 IP 的 `s00_axi`（TPU 寄存器 `0xA0000000`、DVP 寄存器 `0xA00C0000`，各 256KB）；数据通路 TPU/DVP 的 `M00_AXI`/`M0_AXI`→`HP0`/`HP2`→DDR 低 2GB（高 4GB 被排除，故 `0x31000000` 数据区可达）。加密 IP 的「黑盒集成」由 `pragma protect`(RSA+AES128-CBC) 定义：禁仿真/改/探/导网表/查配置，只许生成 bitstream。`top.xdc` 两条全局属性 `READBACK.SECURITY level2`(禁 JTAG 读回)与 `GENERAL.COMPRESS true`(压缩 bitstream)做纵深防御，引脚约束由 TCL 运行时追加。TCL 末尾 `write_hw_platform` 导出 `system_wrapper.xsa`。核心结论：软件里的「魔法地址」全由 `assign_bd_address` 定死，是贯穿后续讲义的软硬件契约。
