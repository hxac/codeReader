本讲是裸机（standalone）路线入口：从 Linux 的高层 EEPTPU API 切换到直接读写 TPU 寄存器的裸机路径。工程分 `net_model`（x86 编译转换工具链）与 `src`（裸机 C++ 源码）两块，网络元数据 `eepnet.h` 编译期链入 ELF、权重 `eepnet.mem` 运行期从 SD 加载，体现裸机无文件系统的约束。引入术语：BSP（板级支持包）、GIC（中断控制器）、FSBL（一级启动加载器）、DMA 缓存一致性（靠 `Xil_DCacheDisable/Flush` 手动同步，因为裸机无操作系统代管缓存）。`main.cc` 是「上电初始化序列 + 串口 `while` 菜单」结构，选项 1 采集、2 forward、3 存盘（仅 SD）、4 读输入、5 实时循环（`while(1)` 死循环，无法回菜单），把推理链路拆成可分步调试的环节；启动序列线性可追溯（`init_platform`→SD→摄像头 I2C→关 D-cache→DP 帧缓冲与中断→`eeptpu_init`→加载权重）。两条核心结论：一、DP 显示是工程唯一中断使用者，`init_intr_sys` 四步是唯一的 GIC 配置点，关掉 `EEP_DP_ENABLE` 会连带关掉整个中断子系统；二、`config.h` 用 `#define` 在编译期裁剪功能（`NET_TYPE`、`SD_CARD_IS_READY`、`EEP_DVP_CAMERA`、`EEP_DP_ENABLE`），改一个宏即通过条件编译剔除 `main.cc` 整段代码，且关 SD 会使权重/输入回退到编译期烧入的数组。
