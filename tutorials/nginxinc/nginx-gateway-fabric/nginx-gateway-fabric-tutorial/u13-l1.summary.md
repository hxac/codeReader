本讲梳理 NGF 三层测试体系：单元测试（`make unit-test`，纯 Go、无集群、靠 fake 隔离）、conformance 一致性测试（独立 `tests/` 模块、真实 kind 集群、挂载上游 Gateway API 官方套件，`make run-conformance-tests` 驱动）、system 测试（functional 跑 kind、NFR/longevity 跑 GKE）。核心澄清：单元测试用 Ginkgo+Gomega 的 BDD 风格组织，依赖由 counterfeiter 生成的 fake 注入，K8s client 用内存版 `client/fake`，全程零集群依赖；NGF 并未使用 controller-runtime 的 envtest，所谓「集成测试」实为 `tests/` 模块的集群级测试。引入术语：BDD、Ginkgo/Gomega、fake/mock、counterfeiter、表驱动测试、envtest、conformance、build tag、`*_suite_test.go` 引导文件。counterfeiter 经 `//counterfeiter:generate . 接口名` 指令与 `make generate` 在 `<包>fakes/fake_<接口>.go` 产出 fake，提供 `XxxReturns`（控制行为）、`XxxCallCount`/`XxxArgsForCall`（验证交互）三类能力；conformance 入口带 `//go:build conformance` 标签，直接调用上游 `suite.NewConformanceTestSuite` 的 Setup→Run。
