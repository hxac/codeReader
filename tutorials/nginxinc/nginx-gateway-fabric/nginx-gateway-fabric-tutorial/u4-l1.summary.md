本讲下钻 NGF 自研框架层 `internal/framework/controller`，拆解 `controller.Register` 的内部运作，回答「控制器被注册后框架如何把资源变更变成下游可消费的事件」。分三个最小模块：其一，`Register` 是框架层装配控制器的唯一入口，用函数式选项（`WithK8sPredicate`、`WithFieldIndices`、`WithNamespacedNameFilter`、`WithOnlyMetadata`、`WithNewReconciler`）把一个资源类型配置成控制器，框架不关心业务语义，新增维度无需改签名。其二，`Reconciler.Reconcile` 是「笨翻译器」，用一次 `Get` 区分增删：存在发 `UpsertEvent{Resource}`，NotFound 发 `DeleteEvent{Type,NamespacedName}`，真错误交还 controller-runtime 重试，并用 `select` 监听 ctx 防止退出阻塞。其三，事件从 API server 到 channel 穿过多层过滤：cache transform→predicate（入队前最省成本）→namespacedName filter（Reconciler 内、Get 前）；field index 不是过滤器而是缓存上的加速索引（供下游 `MatchingFields` 查询）；predicate 通过嵌入 `predicate.Funcs` 实现，`Update` 常用「old/new 任一命中即放行」防归属迁移丢事件，`AnnotationPredicate` 为现成范本。本讲产出的两种事件是下一讲事件循环的输入。
