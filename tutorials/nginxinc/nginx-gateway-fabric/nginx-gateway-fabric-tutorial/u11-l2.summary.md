本讲剖析 NGF 的「产品遥测」——与上一讲面向运维、高频本地暴露的 Prometheus 指标不同，它是低频（默认 24 小时一次）、主动外发、向项目方汇总集群画像的产品反馈。讲解围绕三块拼图组织：数据模型、数据采集、周期上报。

数据模型以 `data.avdl`（Avro IDL）为唯一契约，所有业务字段均设为 nullable 以保证 schema 演进的前后向兼容；Go 侧 `Data` 结构与 OTLP 属性映射 `Attributes()` 由 `go:generate` 从契约同步产出，三者不可手工拆离。`Data` 含公共元数据、产品/构建信息、集群画像、脱敏 flag、`NGFResourceCounts` 资源与策略计数、snippets 指令统计等。

采集由 `DataCollector` 完成，只读复用 `GetLatestGraph`/`GetLatestConfiguration` 及 API server 快照，任一步失败则整条丢弃。关键隐私点：`parseFlags` 对非布尔 flag 仅上报 `default`/`user-defined`，绝不外泄真实值。

周期上报以 `Leader` 套 `CronJob` 实现——仅 leader、首图就绪后启动、带 10 分钟抖动、失败只打日志不阻塞主链路。

核心结论：运行时唯一能关闭产品遥测的开关是 `--product-telemetry-disable`（默认开启，总闸）；上报周期、端点、TLS 均为构建期 ldflags `-X` 注入，非运行时可调；字段层面无逐项开关。后续讲义将承接契约驱动、Leader/CronJob 模式及脱敏 flag 等概念。
