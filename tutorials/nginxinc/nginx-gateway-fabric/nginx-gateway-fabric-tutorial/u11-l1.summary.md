本讲讲解 NGF 控制面如何采集与暴露 Prometheus 指标，核心模块有三。其一，指标注册：控制面复用 controller-runtime 的全局 `metrics.Registry`，在装配期由 `createMetricsCollector` 用 `MustRegister` 把自定义 `ControllerCollector` 注册进去；关闭指标时改注入 no-op 占位对象以避免 nil 指针，handler 代码零改动。其二，`ControllerCollector` 是控制面唯一自定义 Collector，仅采集一个直方图指标 `event_batch_processing_milliseconds`（命名空间 `nginx_gateway_fabric`，桶 `[500,1000,5000,10000,30000]` ms，常量标签 `class`），在 `HandleEventBatch` 的 `defer` 中每批观测一次，可经 `histogram_quantile` 或超阈值批次占比写处理过慢告警。其三，metrics 端口有两条独立链路：控制面走 flag `--metrics-port`（`config.MetricsConfig` 经 `getMetricsOptions` 翻译为 metrics server 监听地址），数据面 NGINX 走 CRD `NginxProxy.Metrics`（`MetricsEnabledForNginxProxy` 解析）；二者默认均为 9113（`DefaultNginxMetricsPort`），且该端口被图层 `buildProtectedPorts` 列为受保护端口，禁止 Gateway Listener 占用。关键术语：Registry、Collector、`MustRegister`、no-op 占位、`event_batch_processing_milliseconds`、`MetricsConfig`、`NginxProxy.Metrics`、受保护端口。
