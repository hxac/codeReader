本讲是「图构建与数据面配置」单元（u5）收官篇，讲解 `dataplane.Configuration` 这层「渲染模型」：它把领域模型 `graph.Graph` 翻译成面向 NGINX 配置块（而非 K8s 资源类型）切分的中间表示，使生成器与 Gateway API 彻底解耦、可单测。

核心入口 `BuildConfiguration(g *graph.Graph, gateway, plus) Configuration` 采用「早退 → 一组并行 `build*` → 一次性字面量装配」结构：无效 Gateway 返回 `GetDefaultConfiguration` 最小骨架而非空配置；多个 `build*` 函数彼此独立产出不同字段，依赖通过显式传参传递。证书严格分两类：`SSLKeyPairs`（含私钥，前端 TLS 终止用）与 `CertBundles`（仅 CA，验证对端用，用于后端 mTLS/外部鉴权/JWT/OIDC），各有独立 ID 体系，用 map 天然去重、按「需求驱动」过滤落盘。L4（TCP/UDP/TLS）服务落在 `stream` 上下文，由 `buildTLSServers`/`buildL4Servers` 产出 `Layer4VirtualServer`，TLSRoute 区分 Passthrough 与 Terminate。所有会变成配置文本的切片构建后都排序，保证「相同输入→相同输出」以避免无谓 reload。关键术语：`SSLKeyPairID`/`CertBundleID`（可作文件名）、`DeploymentContext`（由 handler 注入而非内部填）、`Layer4VirtualServer`、`SSLListenerHostnames`（misdirected request 检测）。下游交给 `generator.Generate(conf)` 渲染（u6）。
