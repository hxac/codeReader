本讲围绕 NGF 对接 Gateway API Inference Extension 展开三个最小模块：**endpoint-picker shim**（`endpoint-picker` 子命令）是数据面 Pod 内的 HTTP↔gRPC ext_proc 翻译器，因 NGINX 不会发起 ext_proc，由它对外监听 `127.0.0.1:54800`、对内将 HTTP 请求翻译成 ext_proc 流打给 **EPP**，是无状态协议翻译器。**InferencePool 图处理**把该 CRD 当作一类后端：`buildReferencedInferencePools` 反向收集被 Route 引用的池，跑 Accepted/ResolvedRefs 校验，结果沉淀到 `Graph.ReferencedInferencePools` 并双向回写状态；因其属 `x-k8s.io` 非标准 API 组，控制器不走标准 CRD 发现，由 `--gateway-api-inference-extension` flag 单独开关。**NGINX↔EPP 头交互**全链路靠约定头/变量缝合：NJS 用 `X-EPP-Host/Port` 打 shim、收 `X-Gateway-Destination-Endpoint` 写入 `$inference_workload_endpoint`，再经 `map` 解析为 `$inference_backend_<pool>` 供 `proxy_pass` 使用，同时编码 FailOpen/FailClose 兜底。shim 由 provisioner 作为 sidecar 注入，仅推理 Gateway 才有。
