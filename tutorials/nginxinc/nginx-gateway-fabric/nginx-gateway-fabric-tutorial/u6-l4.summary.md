本讲承接 HTTP 上下文配置生成，转向 NGINX 的 stream 上下文，讲清 NGF 如何把 Gateway API 的 TCPRoute/UDPRoute/TLSRoute（L4 路由）翻译成 `stream.conf`。三个最小模块：①stream 上下文定位——L4 配置统一落在 `streamConfigFile`，由静态 `nginx.conf` 的 `stream {}` 经 `include` 拉入，与 http 上下文指令集互不重叠，靠 dest 常量分桶隔离；②TCP/UDP 服务器生成——一条路由产出 `stream server { listen <port>[ udp]; proxy_pass <upstream>; }` 与 `stream upstream { random two least_conn; }`，`(port,protocol)` 用 portProtoKey 去重，多后端加权时 proxy_pass 退化为 `$backend_<port>` 并配 `split_clients`；③TLS Passthrough/Terminate——每个 TLS 端口配一个「只看不摸」的端口级 server（`ssl_preread on` 偷看 SNI 填入 `$ssl_preread_server_name`，再 `pass $dest<port>`）加一张 `主机名→socket` 路由 map，每主机名对应一个 socket server：Passthrough 直接转发加密流，Terminate 先 `ssl_certificate` 终止再代理。关键契约：`getTLSPassthroughVarName`、`getSocketNameTLS`/`-terminate`、map 的 `hostnames;`。生成器注册 `executeStreamServers/Upstreams/Maps` 三个执行函数，输出仍为 `[]agent.File`。
