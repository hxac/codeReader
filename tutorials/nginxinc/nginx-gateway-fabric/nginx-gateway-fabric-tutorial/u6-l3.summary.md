本讲覆盖 NGF 把 `dataplane.Configuration` 翻译成 NGINX `server`/`location`/`upstream` 指令的三个最小模块。其一，server 块（`servers.go`）：将 `HTTPServers`/`SSLServers` 翻译为 `http.Server`，区分业务 server、HTTP default（`return 404`）、SSL default（`ssl_reject_handshake`），并静态写死 503/500 两个兜底 socket。其二，upstream（`upstreams.go`）：逐个决定 zone（OSS 512k / Plus 1m，可被 `UpstreamSettingsPolicy` 覆盖）、LB 方法（默认 `random two least_conn` 而非 round-robin）、keepalive（默认 16，0 则关闭）、state 文件（仅 Plus）；无端点或无效引用时退化为 503/500。其三，location（`createLocations`）：按是否带 HTTP 匹配条件或推理后端分三路——直接 proxy、NJS 内部 location、EPP 推理 location；无尾斜杠 Prefix 拆成双 location 并做去重。多后端加权用 `split_clients`，`percentOf` 向下取整保总和 ≤100，proxy_pass 退化为 `$变量`。关键耦合点 `keepAliveChecker` 使 location 的 Connection 头取值取决于后端 upstream 是否开启 keepalive。所有 server/upstream 模板产物按 dest 分桶合并进同一份 `http.conf`。
