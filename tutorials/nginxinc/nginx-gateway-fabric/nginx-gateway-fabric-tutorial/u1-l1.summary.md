本讲建立对 NGINX Gateway Fabric（NGF）的整体认知。核心结论：NGF 是 Kubernetes Gateway API 的开源实现，以 NGINX 为数据面，把用户声明的 YAML 翻译成 NGINX 配置。三大最小模块：（1）定位——Gateway API 用 CRD 描述流量入口，资源分三层：GatewayClass（声明由哪个控制器实现）、Gateway（监听端口/协议的实例，含 Listener）、HTTPRoute 等路由资源（经 parentRefs 挂到 Gateway，backendRefs 指向 Service）。（2）控制面/数据面分离——Go 控制面 watch 资源并生成配置，NGINX 处理真实流量，NGINX Agent 经 gRPC+mTLS 下发配置并触发 reload，实现控制路径与数据路径解耦；NGF 身份由控制器名 gateway.nginx.org/nginx-gateway-controller 标识，经 --gateway-ctlr-name 传入，须与 GatewayClass 的 controllerName 一致才被认领。（3）版本矩阵——NGF 与 Gateway API、Kubernetes、NGINX(OSS/Plus)、NGINX Agent、F5 WAF 各有兼容版本，选型先定环境再反查 NGF；生产用 latest release，尝鲜用 edge，版本号为构建期注入。
