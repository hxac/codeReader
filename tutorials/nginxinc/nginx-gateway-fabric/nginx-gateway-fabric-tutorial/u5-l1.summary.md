本讲讲解 `Graph` 的构建：把 ChangeProcessor 输出的 `ClusterState`（集群资源快照原料）经唯一入口 `BuildGraph` 加工成内部模型 `Graph`，按依赖拓扑顺序装配，策略类资源必须最后处理。核心结论有三：其一，GatewayClass 经 `processGatewayClasses` 分为 Winner（名字与 controllerName 均匹配，被认领）、Ignored（指向 NGF 但名字不匹配，需回写「不支持」状态）、跳过三类；其二，Gateway 经 `processGateways` 按 `gatewayClassName` 过滤，无效的 Gateway 也进图但仅保留精简结构、不带 Listeners，以保证错误状态可回写；其三，校验采用「分层+早退+渐进兼容」策略——错误不靠 error 抛出，而是沉淀进每个节点的 `Conditions` 字段并由 `Valid` 布尔位决定能否参与配置生成。关键术语包括：节点三元组范式（`Source`+`Valid`+`Conditions`）、数据面无关/相关校验、`EffectiveNginxProxy`（GC 级与 GW 级合并）、`(Type,Status,Reason,Message)` 四元组条件。本讲止于 GatewayClass/Gateway 节点，路由、监听器、后端解析、最终配置表示由后续讲义承接。
