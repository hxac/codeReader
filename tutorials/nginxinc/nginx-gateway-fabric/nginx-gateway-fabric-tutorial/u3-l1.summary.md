`StartManager(cfg)` 是 NGF 控制面总装入口，把 `config.Config` 装配成运行进程，分六阶段：准备底座（healthChecker、controller-runtime Manager、scheme、根 ctx）→ 注册控制器并产出 `discoveredCRDs` → 构造无状态处理器（processor）与下发/状态组件（nginxUpdater、provisioner 等）→ 装配 eventHandler 与事件循环 → 注册 telemetry 并 `mgr.Start`。装配由 `create*/register*` 工厂完成，顺序由依赖关系拓扑排序决定；依赖经结构体字段注入，`eventHandlerConfig` 是依赖最多的集大成者。Runnable 由三种包装器赋予 leader 语义：`LeaderOrNonLeader`（事件循环、gRPC、provisioner，全副本跑）、`Leader`（telemetry，仅 leader）、`CallFunctionsAfterBecameLeader`（当选后打开写状态、建数据面、下发配置三把开关）。关键洞察：非 leader 副本也跑控制器与事件循环、构建图做热备，但改集群动作默认关闭、仅当选 leader 才打开，实现「热备 + 单写」多副本策略。
