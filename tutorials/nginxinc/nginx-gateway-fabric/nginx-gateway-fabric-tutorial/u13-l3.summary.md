本讲是 NGF 学习手册的收官篇，回答「如何给 NGF 新增一个能力（如新策略 CRD）」。它包含三个最小模块：功能开发链路、代码生成、PR 流程。功能开发链路是核心：新增一个策略要依次穿过「API 类型定义→scheme 注册→Kind 常量→控制器 watch→变更存储登记→校验器→生成器→（图附着，通常免改）→（受影响条件，可选）」共约九站，每站都有明确接线点。设计范式是「接口+组合器+注册表」三件套（开闭原则落地）：新增策略只需「写新实现+注册表登记一次」，不动已有代码；`CompositeValidator` 找不到校验器会 panic（fail-fast）。关键结论是图层附着策略无关——`processPolicies`/`attachPolicies` 按 targetRef 的 group/kind 分发，新策略只要实现 `policies.Policy` 接口就免费获得附着、继承、冲突裁决。本讲引入的关键术语包括：功能开发链路、kubebuilder 注解、GVK 路由、`CompositeGenerator`/`CompositeValidator`/`ManagerConfig` 接线、`UnimplementedGenerator` 嵌入补齐、counterfeiter fake 生成、controller-gen 的 `crd`+`object` 产物、`make generate-all` 总钥匙、16 步 PR 流程（先开 issue、配单测、提交前跑生成、squash 单 commit 带 PR 号）。范本策略是 `ClientSettingsPolicy`，其 generator/validator 是「字段→指令」的最简范式。
