本讲纵览 NGF 控制面在高频变更与多副本下的四大横切机制。事件双缓冲用 currentBatch/nextBatch 两切片做无锁批处理：处理期间新事件堆入 nextBatch，处理完合并成一批，把 reload 次数从「每事件一次」（N）压到「每处理窗口至多一次」，下游 EventHandler 需幂等。Leader 选举基于 Kubernetes Lease 实现「热备+单写」：eventLoop 等用 LeaderOrNonLeader 全副本跑做热备，CallFunctionsAfterBecameLeader 在当选时一次性翻开 status/provisioner/eventHandler 三把写开关，ReleaseOnCancel=false 防止周期任务交接重叠。ngfsort 是所有冲突裁决的确定性兜底 tie-breaker，将规范「创建时间→namespace→name」三级规则固化为可复用比较函数，保证「相同输入→相同输出」，消除顺序敏感与无谓 reload。就绪以「首图构建完成」而非进程启动为信号，graphBuiltHealthChecker 用 close channel 一次性广播，既驱动 readyz 摘接流量，也作周期任务就绪门，避免空配置 404 窗口。四者分别管合并、单写、确定性、对外契约，共同支撑系统「快而稳」。
