本讲讲解如何把 NGF 装进集群，含三个模块。其一，Helm chart 结构：由 Chart.yaml（元信息）、values.yaml（默认配置）、templates/（渲染逻辑）组成；values 决定「装成什么样」，templates 决定「渲染出哪些资源」。关键术语有 chart、release、CRD——上游 Gateway API 的 GatewayClass/Gateway/HTTPRoute，与 NGF 自有的 NginxProxy（数据面参数）、NginxGateway（控制面配置）。模板把 gatewayControllerName、gatewayClassName 翻译为 controller 子命令的 flag，并用 parametersRef 把 GatewayClass 指向 NginxProxy；控制器名须为 gateway.nginx.org/nginx-gateway-controller 才能认领 GatewayClass。其二，部署变体（nodeport/nginx-plus/snippets/azure/openshift/experimental/inference 等）本质是唯一 chart 上的最小 values 覆盖，examples/helm 为源、deploy/ 单文件 manifests 为产物，两条路殊途同归。其三，安装顺序铁律：Gateway API CRDs（外部）→ NGF 自有 CRDs → pre-install hook 跑 generate-certs 生成 mTLS 证书 → 控制面 Deployment，因控制器只能 watch 已注册的 CRD。部署后用 kubectl get gatewayclass 验证 Accepted=True 作为成功认领的信号。
