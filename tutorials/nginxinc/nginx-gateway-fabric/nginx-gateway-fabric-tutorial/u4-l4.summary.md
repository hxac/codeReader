本讲下钻事件管线中 `processor` 这一环，覆盖三个最小模块：变更捕获、对象存储、changed predicate。`ChangeProcessor` 只负责登记变更与按需重建图，不碰配置生成与下发；它先逐条 `Capture*`、再一次 `Process`，配合 u4-l2 的批处理把一批事件压缩成一次重建。`Process` 受 `changed` 标志把门：无有效变更时返回 `nil`，整条「生成配置→下发」链路被跳过，这是把 N 次变更压缩成零次 reload 的核心。对象存储按 GVK 分发，`multiObjectStore` 下挂泛型 `objectStoreMapAdapter`、策略专用 `ngfPolicyObjectStore`（按 `PolicyKey` 存）、`convertingReferenceGrantStore`（v1beta1→v1），EndpointSlice 是唯一不持久化的类型。changed predicate 把「资源变了」收窄为「与配置相关的资源变了」，分三类：`nil`（主体资源始终算）、`funcPredicate{isReferenced}`（被引用才算）、`funcPredicate{isNGFPolicyRelevant}`（策略相关才算）、`annotationChangedPredicate`（如 CRD 的 `bundle-version`）。关键概念还包括：`ForceRebuild` 旁路（不动 store 只置真，供 WAF bundle 场景）、predicate 依赖 `latestGraph` 的首图悖论、未注册类型即 panic 的 fail-fast 守门。本讲止于 `Process` 输出的 `graph.Graph`，供 u5-l1 建图与 u5-l3 端点解析承接。
