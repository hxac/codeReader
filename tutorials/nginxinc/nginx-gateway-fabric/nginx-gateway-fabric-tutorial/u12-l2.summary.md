本讲义讲解 NGF 在 NGINX Plus 模式下的三项独有能力。第一是**动态上游（dynamic upstream）**：`UpdateUpstreamServers` 通过 NGINX Plus 运行时 API 原地改写 upstream 共享内存 zone，把后端地址变动从「改配置 + reload」收敛为一次 API 调用，从而跳过 reload；`actionsEqual` 在后端集合未变时直接早退，连 API 调用都省略。其依赖 upstream 的 zone 共享内存与 state file 持久化，且带 resolve 的不可变上游需跳过。第二是 Plus 管理面两段专属配置：`plus-api.conf`（可写 unix socket 供 Agent 改上游 + 只读 8765 dashboard）与 `mgmt` 块（授权令牌、用量上报、TLS），均由 `AllowedAddresses` 或 `g.plus` 开关控制，OSS 不生成。第三是**授权闭环**：`deployment_ctx.json`（标识安装/集群，由 collector 采集、handler 注入、生成器落盘，含 init 容器与 handler 两条互补路径）配合 mgmt 的 `usage_report`/`license_token` 完成计费与鉴权。广播层用 `ConfigApplyRequest`（写文件+reload）与 `APIRequest`（调 Plus API）区分两条下发路径。
