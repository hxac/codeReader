本讲解答「配置如何从控制面送达数据面」，含三个最小模块。其一，`NginxUpdater` 是下发总入口：`UpdateConfig` 只做「算版本、广播元数据、记错误」三步，`configVersion` 未变即早退，实现配置生成与网络通信的解耦。其二，控制面作 gRPC 服务端，注册 `CommandService`（推送主信道）与 `FileService`（反向拉文件），经 mTLS 与 TokenReview 双重校验；`ConnectionsTracker` 用 UUID 追踪每条连接，Agent 上报 InstanceID 后连接才 Ready。其三，`DeploymentBroadcaster` 以「一对多 + 同步确认」扇出消息，`Messenger` 把双向流解耦为发/收两个 channel，`Subscribe` 主循环用 select 统一调度。核心机制是「推文件清单、Agent 反向 `GetFile` 拉内容」的混合模式；整条链路靠 `FileLock` 串成原子事务，保证下发期间文件视图一致。引入的关键术语包括 `NginxUpdater`、MPI 协议、`DeploymentBroadcaster`、`Messenger`、`ConnectionsTracker`、`FileLock` 等。
