本讲讲清了 NGF「从源码到运行」的完整链路，分三个最小模块：Makefile 关键目标、kind 本地部署、测试与 lint 入口。核心结论是 NGF 用唯一的 Makefile 作开发入口，`make help` 即自带说明书；`make build` 经 `go build` 产出 `build/out/gateway` 二进制，并通过 `-ldflags -X main.version=...` 在链接期把版本号、遥测配置注入 `main.go` 同名变量（本地默认版本为 edge），二进制是 cobra 多子命令程序，含 controller、generate-certs、initialize、sleep、endpoint-picker 五个子命令。关键术语：Makefile 变量分可覆盖（`?=`）与不可覆盖两层，默认目标平台为 linux/amd64（macOS 需覆盖 GOOS）；控制面镜像走多阶段 Dockerfile（builder 编译、scratch 运行）；本地用 kind 模拟集群，部署链路为 create-kind-cluster → build-images → load-images → install-gateway-crds → helm-install-local，`install-ngf-local-build` 一键聚合，本地镜像必须配 `pullPolicy=Never`；质量门槛 unit-test（带 -race、-shuffle=on）、vet、lint，`dev-all` 为提交前一键全检。后续讲义将在「运行中的 Pod」内部展开 controller 的行为。
