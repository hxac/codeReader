本讲聚焦 NGF 控制面事件批次的唯一消费者 `eventHandlerImpl`，它只做编排、不做业务。主流程为：逐条 `parseAndCaptureEvent` 调 `processor.Capture*` 登记变更 → 一次 `processor.Process` 构建 `graph.Graph` → 首图构建后由 `graphBuiltHealthChecker` 置就绪 → 进入 `sendNginxConfig` 下发。核心模块 `sendNginxConfig` 按「逐 Gateway：台账登记 → 镜像确定 → 构建 `dataplane.Configuration` → 缓存 → `generator.Generate` 生成文件 → `nginxUpdater.UpdateConfig` 下发 Agent → 入队状态」推进，用 `defer reconcileWAFPollers`、`deployment.FileLock`、WAF fail-closed 保证正确性。关键设计：写状态经多生产者/单消费者的 `statusQueue` 与下发解耦，由常驻 goroutine `waitForStatusUpdates` 消费，用 `NginxConfigPushed` 区分真下发与纯状态变化；多副本单写靠 `isLeader()` 守卫、`LeaderAwareGroupUpdater` 与当选时 `enable` 重发最新图叠加保证；WAF bundle 经 `WAFBundleReconcileEvent` + `processor.ForceRebuild()` 触发重建。引入术语：编排、就绪、状态队列、Capture/Process、objectFilter、QueueObject 等，供后续讲义承接。
