本讲建立 NGF 仓库的顶层目录地图，作为后续所有源码讲义的「坐标系」。顶层目录分四类：入口 `cmd/`（唯一二进制 `cmd/gateway/main.go`，仅装配五个 cobra 子命令，业务全部在 `internal/`）；核心代码 `internal/`（Go 编译器强制仅仓库内可见）；API 类型 `apis/`（可对外 import 的 CRD，按 `v1alpha1`/`v1alpha2`/`waf/v1` 分版本，含自动生成的 `zz_generated.deepcopy.go`）；以及 `deploy/`（kustomize 清单）、`charts/`（Helm chart）、`examples/`、`docs/`、`tests/`（独立 go.mod，跑上游 Gateway API 一致性测试）等部署/示例/文档/测试配套目录。

全仓库最重要的代码边界是 `internal/controller` 与 `internal/framework` 之分：前者写「NGF 这个产品」（Gateway API 语义、NGINX 配置生成、CRD 翻译），按 `state/`、`nginx/`、`status/`、`provisioner/` 等子系统分目录，对应学习手册 u3~u9 各单元；后者写「如何搭一个控制器」（事件循环、控制器抽象、leader 选举、WAF 拉取等通用能力）。两者各自的 `doc.go` 用一句话钉死职责，依赖方向单向：`controller` 依赖 `framework`，不可反向。后续讲义均落在这套结构的某个具体位置。
