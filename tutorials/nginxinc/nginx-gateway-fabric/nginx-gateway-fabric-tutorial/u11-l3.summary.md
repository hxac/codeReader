本讲讲解 NGF 如何为 HTTP/gRPC 路由开启 OpenTelemetry 分布式链路追踪，核心是**两层职责分离**：集群运维在 `NginxProxy.Telemetry` 配置全局 OTLP exporter（决定 trace 发往哪），应用开发者用 Direct 附着策略 `ObservabilityPolicy` 控制每路由的采样与属性；`ValidateGlobalSettings` 强制全局 endpoint 必须就绪，否则策略 NotAccepted。关键术语：`Tracing` 字段（`Strategy` ratio/parent、`Ratio`、`Context` extract/inject/propagate/ignore、`SpanName`、`SpanAttributes`）、渲染模型 `dataplane.Telemetry`、`ngx_otel_module` 指令。渲染分两套模板：全局 `executeTelemetry` 产出 `otel_exporter`/`otel_service_name`/`split_clients`（落 http.conf），每路由 observability 生成器（u8-l3 框架的具体实现，重写 `GenerateForLocation`/`GenerateForInternalLocation`）产出 `otel_trace`/`otel_trace_context`/`otel_span_*`（挂 location）。结论：ratio 采样用 `split_clients` 一致性哈希（同 ratio 去重共享），parent 用 `$otel_parent_sampled`，由 `getStrategy` 决定 `otel_trace` 取值；span 属性按「策略+全局」合并，服务名默认 `ngf:<ns>:<gw>`；当路由需内部 location 时，采样开关置外部、span 属性置内部，分两处渲染。
