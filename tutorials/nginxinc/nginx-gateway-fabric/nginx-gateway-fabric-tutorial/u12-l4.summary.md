本讲讲解 NGF 的两条 ExtensionRef 扩展机制：SnippetsFilter 与 AuthenticationFilter。SnippetsFilter 是把原始 NGINX 指令注入生成配置的「逃逸舱」，仅允许 main/http/http.server/http.server.location 四个上下文、每上下文唯一；main/http 在 Gateway 级全局聚合，server/location 跟随路由规则，最终经 include 按文件名去重落盘，受 SnippetsFilters/Snippets 特性开关门控、默认关闭。AuthenticationFilter 用 spec.type（Basic/JWT/OIDC）三选一，CEL 强制类型与字段互斥：Basic 走 OSS 自带 auth_basic，JWT/OIDC 依赖 NGINX Plus 并在图层判 isPlus。关键术语：ExtensionRefFilter（图层对扩展过滤器的联合包装结构）、Snippet 上下文（NGINX 配置层级）、AuthType 枚举、claim 名脱敏冲突（sanitizeClaimNameForVariable）、Referenced 标记。核心结论：两类过滤器均经 validateExtensionRefFilter 做 Group/Kind 白名单与同 namespace 解析，无效时置 Valid=false 并沉淀进 Conditions；OIDC 因共享 NGINX 路径空间，须在绑定后做 HTTPS 监听器与 URI 冲突校验。整体沿用 Source/Valid/Conditions 三元组 → dataplane 渲染模型 → servers.go 挂指令的统一骨架。
