本讲承接 Graph 构建，下钻 NGF 如何把 Gateway API 的 Route 识别并绑定到 Listener。核心是归一化：HTTPRoute/GRPCRoute 收敛为 L7Route，TLSRoute/TCPRoute/UDPRoute 收敛为 L4Route，由 RouteType 区分；公共构建骨架在 route_common.go，经解析 parentRefs、校验 hostnames/rules、解析 backendRefs、打标 Valid/Attachable 五步完成，枢纽函数为 buildSectionNameRefs（处理 sectionName 缺省时展开到全部 Listener）。Listener 由 listenerConfiguratorFactory 按协议分发校验器，跑 validators、conflictResolvers、externalResolvers 三段，并分离 Valid 与 Attachable；协议到 Route 类型的权威映射见 getValidKindsForProtocol。绑定走 sectionName/port、协议、主机名三重匹配，结果沉淀为 Accepted Condition；TCP/UDP 因无判别条件而一端口一路由，绑定后还做监听器主机名隔离。ListenerSet（GEP-1713）支持监听器跨资源/命名空间复用，需父 Gateway 的 AllowedListeners 授权并复用其 ListenerFactory 以保证冲突检测一致，Route 以 parentRef.kind: ListenerSet 挂载。本讲只把 backendRefs 放入 RouteRule，真正的 Service/Endpoints 解析与 ReferenceGrant 授权留给后续讲义。
