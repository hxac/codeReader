本讲用官方 `cafe-example` 把 NGF 控制面「通电」，端到端跑通一条 HTTP 请求（客户端→NGINX→后端）。核心结论：一个 Gateway API 路由示例由三块拼图组成——后端应用（Deployment+Service）、Gateway（流量入口）、HTTPRoute（路由规则），靠 `gatewayClassName`、`parentRefs.name/sectionName`、`backendRefs` 互相接线，其中 `sectionName` 须匹配监听器名，路径匹配分 `PathPrefix` 与 `Exact`，请求是否放行取决于 HTTPRoute 的 `hostnames` 与监听器 `hostname` 通配是否匹配。验证用固定四步法：apply 后端→apply Gateway 并取 `GW_IP`/`GW_PORT`→apply HTTPRoute→`curl --resolve` 发请求（假域名靠 `--resolve` 在本地指向 NGINX，从而带上正确 Host 头）。NGF 把路由翻译成的 NGINX 配置最终落在数据面 Pod 的 `/etc/nginx/conf.d/*.conf`（HTTP）与 `stream-conf.d`（TCP/UDP）。最后 `examples/` 目录按「HTTP 路由基础、按协议路由、过滤器扩展、NGF 策略 CRD」四类组织，是最高效的用法检索入口，且示例版本要与所用 NGF 版本对应。
