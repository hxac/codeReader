本讲精读 `sync(task)` 调度器与各资源 sync 函数。`sync` 是注册给 taskQueue 的纯调度器，用 `switch task.Kind` 把任务路由到 `syncIngress`/`syncVirtualServer` 等，自身只做横切事务（批量判断、Plus zone-sync、启动/批量收尾）。多数 `syncX` 遵循「三段式」：用 Key 反查 Lister 取资源 → 调 `configuration.AddOrUpdateX/DeleteX` 更新内存模型并返回 `changes/problems` → 分别交 `processChanges`（调 `configurator` 写配置+reload）与 `processProblems`（回写 Invalid/Warning status）处理。核心分层是 Configuration 负责决策与扩散、Configurator 负责执行，syncX 只编排不越层，且 `processChanges` 传入的是组装好的扩展资源（如 `VirtualServerEx`）。批量优化上，队列堆积时 `DisableReloads`、排空后 `EnableReloads` 把 N 次 reload 压成 1 次；启动期另有 `CompleteStartup`+`updateAllConfigs` 一次性路径，二者靠 `isNginxReady` 互斥。特殊 sync 不走三段式：ConfigMap 改全局参数须全量 `updateAllConfigs`（批量期显式跳过、改由收尾统一执行），Service 两段式（外部状态 + 被引用资源），EndpointSlice 在 Plus 下动态更新可免 reload 并据此返回 `resourcesFound`。
