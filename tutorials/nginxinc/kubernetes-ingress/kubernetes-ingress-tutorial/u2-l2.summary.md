本讲精读数据真相源 types.go，定义 5 个 CRD：VirtualServer、VirtualServerRoute、TransportServer、GlobalConfiguration、Policy，统一遵循 TypeMeta+ObjectMeta+Spec+Status 骨架，归属 k8s.nginx.org/v1，且禁止 import 业务层以保持纯净。核心概念是 marker：+kubebuilder 注释驱动 controller-gen 生成 CRD schema（短名、status 子资源、Enum、CEL 跨字段校验 XValidation），+k8s 注释驱动 deepcopy 与 client 生成；改 types.go 后须跑 make update-codegen 与 update-crds。关键结论：VS 的 upstreams 生成 upstream 块、routes 生成 location 块，二者靠 action.pass 用 upstream 的 name 解耦连接，splits/matches 提供流量切分与内容路由；VSR 是被 VS 按名装配的子路由，host 须一致；GlobalConfiguration 是 listener 注册表，供 VS/TS 按名引用；TS 面向四层，结构更瘦，必须引用 GC 登记的 listener，action 只有 pass；Policy 用一排互斥指针表达「一个 Policy 只表达一种策略」。
