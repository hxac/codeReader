本讲讲解 NIC 进入 Kubernetes 集群的两种安装方式，核心结论：二者产出同一组资源，Helm Chart 仅比手写清单多一层参数化与命名模板。手写清单位于 `deployments/`，按部署拓扑分子目录，最小安装需六类资源——命名空间与 ServiceAccount、RBAC、名为 `nginx` 的 IngressClass、全局配置 ConfigMap（data 默认为空、按需覆盖）、运行 NIC 的工作负载（其 `args` 段承接启动 flag，如 `-nginx-configmaps`）、LoadBalancer Service，文件之间靠名字（label / serviceAccountName / configmap 名）互相引用构成闭环。Helm Chart 位于 `charts/nginx-ingress`，由 `Chart.yaml`（区分 chart version 与 appVersion）、`values.yaml`（参数全集）、`templates/`（用 `{{- if }}` 守卫按 values 决定是否渲染，使 Deployment/DaemonSet/StatefulSet 三种工作负载互斥选一）、`_helpers.tpl`（统一命名）、`crds/` 组成。引入的关键术语：manifest、kind、RBAC、IngressClass、downward API。安装完成后还需路由资源：标准 Ingress（`networking.k8s.io/v1`，把后端 service 内联在 path）与 NIC 私有 VirtualServer（`k8s.nginx.org/v1`，把 upstream 显式声明为命名对象、route 用 `action.pass` 按名引用，更解耦、可扩展流量切分）。Cafe 示例演示了二者的等价路由写法。
