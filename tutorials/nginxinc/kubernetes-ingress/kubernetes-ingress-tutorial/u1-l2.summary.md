本讲回答「NIC 源码如何变成可运行二进制」，分三个最小模块展开。其一，Go 模块与依赖：模块名 `github.com/nginx/kubernetes-ingress`，使用 Go 1.26.4，核心依赖 client-go（控制器地基）、nginx-plus-go-client（Plus 动态配置）、cert-manager（证书集成）、aws-sdk-go-v2 与 prometheus/telemetry-exporter（可观测性）。其二，Makefile 目标体系：统一操作入口，默认目标是 `help`，常用目标包括 `build`/`build-local`、`test`、`test-update-snaps`、`lint`（Docker 跑、只查 diff）、`format`、`update-codegen`/`update-crds`、`deps`/`clean`；变量分不可覆盖（`=`）与可覆盖（`?=`）两类。版本号来自 `.github/data/version.txt` 的 `IC_VERSION=5.6.0`，通过 ldflags（`-X main.version=...`）在编译期注入 `main.go` 的 `version` 变量，故必须走 `make build` 而非裸 `go build`。其三，构建标签与本地构建：`make test` 强制带 `-tags=aws,helmunit -shuffle=on`；`tools.go` 用 `//go:build tools` 锁定 controller-gen 等工具版本；本地增量构建依赖 `GO_SRCS`（`git ls-files`），新增 `.go` 文件须先 `git add`。本讲是后续源码阅读的前置条件。
