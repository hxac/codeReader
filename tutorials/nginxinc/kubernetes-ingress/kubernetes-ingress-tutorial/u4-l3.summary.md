本讲解析 Ingress 注解子系统，核心是 `parseAnnotations`：它采用「值拷贝基线 + 逐键探测 + best-effort 降级」模式，基线默认值来自 `NewDefaultConfigParams`，单条注解出错只记日志不中断，字段保持默认。类型转换由 `parsing_helpers.go` 承担——`GetMapKeyAs*` 封装「探测+转换」并统一返回 `(value, exists, error)`，以区分「没写」与「写错」；`ParseTime`/`ParseSize`/`ParseLBMethod` 等则把字符串校验、归一化为合法 NGINX 取值。注解到指令为手工映射，以 `proxy-connect-timeout` 为例的链路是：注解串 → `ParseTime` 归一化 → `ConfigParams` 字段 → `location` 字段 → OSS/Plus 两套模板渲染。mergeable ingress（master/minion）模式用三张表 `masterDenylist`、`minionDenylist`、`minionInheritanceList`，按「过滤 master → 继承到 minion → 过滤 minion」三道工序处理，继承遵循「minion 自带优先 + 白名单限定」，全程作用于深拷贝以免污染 controller 缓存。
