本讲讲解 NIC 如何用 raw client-go 的 **list-watch** 与 **SharedInformerFactory** 感知集群变化：先 LIST 全量建基线，再靠 WATCH 长连接收增量，并每 30 秒 **resync** 兜底重触发事件。Informer 在构造期注册 handler、运行期 `Run()` 中统一 `Start`，且必须 `WaitForCacheSync` 通过后才启动 `syncQueue`，避免用空缓存误调谐。NIC 把「一个命名空间内全部监听」聚合为 **`namespacedInformer`** 结构，内含 core / Secret / CRD / dynamic 四类按需创建的工厂及各资源 Lister 与 `cacheSyncs`。命名空间范围支持三种：精确列表、全局（key `""`）、动态（`-watch-namespace-label`，靠监听 Namespace 自身动态增删 nsi）。**`addXHandler`** 是统一三步模式：取 Informer → 挂 `AddEventHandler` → 取 Lister 并登记 `HasSynced`；Pod 例外，只建 Lister 不挂 handler。事件处理器最终汇聚到 `AddSyncQueue` 把事件转为 task，感知层只负责入队，调谐交由 task queue 与 sync 展开。
