本讲定位 `Configurator` 为五层架构中「配置生成」层的核心：它承接 `Configuration` 的裁决结果（如 `VirtualServerEx`），把决策翻译成 `.conf` 文件并触发 NGINX 行为，**只执行不裁决**。每个资源入口方法遵循「公共/私分离」双层模式：公共 `AddOrUpdateXxx` 调用私有 `addOrUpdateXxx` 完成「生成→渲染→写盘」三段式后再 `Reload`，使批量路径能把 N 次 reload 压成 1 次；三段式即生成器产出模板输入结构体、执行器渲染成文本、`CreateConfig` 写盘并返回 `configChanged`。`Configurator` 编排 **v1**（Ingress + main `nginx.conf`）与 **v2**（VirtualServer / TransportServer / OIDC / TLS Passthrough）两套模板执行器，并支持用户自定义模板的热替换与还原。核心开关 `isReloadsEnabled` 统管三种 reload 模式：启动期累积写盘、批量合并、Plus 端点/权重变更走 API 免重载。安全基石之一是 `escapeNginxString` 转义。
