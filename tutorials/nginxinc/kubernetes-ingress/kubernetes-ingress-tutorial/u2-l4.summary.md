本讲介绍 Policy CRD——一种横切式可复用策略资源。Policy 不定义路由，只封装一种策略（限流、鉴权、访问控制等），被 VirtualServer/VirtualServerRoute/Ingress 按名引用，实现策略与路由解耦。核心不变量是「一个 Policy 恰好表达一种策略」：PolicySpec 用一排互斥指针字段表达，由 Go 校验层 validatePolicySpec 的 fieldCount != 1 强制；而 schema 层（kubebuilder 标记、CEL XValidation）只管格式与跨字段约束，两层校验叠加落地。Policy 没有 selector，目标范围由谁引用它决定——VS/VSR 用 policies 字段（分 server 级与 route 级，route 级覆盖 server 级同类型），Ingress 改用 nginx.org/policies（OSS）/nginx.com/policies（Plus）注解，PolicyReference 含 Name 和可选 Namespace，支持跨命名空间引用。共 12 种策略类型，各有独立结构体与校验函数，统一在 policyFields 数据驱动表里分发；其中 jwt/oidc/waf 需 NGINX Plus，部分还受命令行 feature-gate（gateCheck）控制。适用范围由 IsPolicySupportedOnIngress 单一裁定：全部 12 种支持 VS/VSR，仅 AccessControl/CORS/ExternalAuth/IngressMTLS/EgressMTLS/WAF 6 种支持 Ingress。后续讲义在此基础上承接配置翻译与认证深入。
