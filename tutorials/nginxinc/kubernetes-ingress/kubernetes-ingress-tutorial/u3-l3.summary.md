本讲讲清 Ingress Controller 的「调度层」taskQueue：它把 Informer 事件解耦成可消费的任务，衔接感知层与后续调谐层。核心概念是极简的 task{Kind, Key}——只存资源种类枚举 Kind 和 namespace/name 字符串 Key，不存对象本身，出队后再用 Key 回查 Lister 取最新对象；newTask 用 switch 把任意资源归类为 Kind。taskQueue 是 client-go workqueue.Type 的薄封装，借去重、至少一次投递、并发安全三条语义获得正确性。关键结论：NIC 是单 worker 串行 sync 控制器（Run 的 period 稳态下不生效），全局一条队列一个 worker；入队分 Enqueue/Requeue/RequeueAfter 三种语义。最需破除的误区：本项目无指数退避（未用 RateLimitingInterface）也无最大重试次数，实际失败路径只有 Requeue 立即重入队，靠「单 worker + 去重」天然节流，RequeueAfter 定义却全项目未被调用。配套优化：队列堆积时 DisableReloads 批处理、排空后一次性 reload，Len() 是触发条件；Shutdown 保证优雅关闭，不在 reload 中途被掐断。
