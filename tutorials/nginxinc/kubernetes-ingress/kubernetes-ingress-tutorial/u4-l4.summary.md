本讲讲解 NIC 配置生成层的 version1 链路：如何把标准 Kubernetes `Ingress` 资源翻译成 NGINX 的 `.conf`。核心入口是 `internal/configs/ingress.go` 中的 `generateNginxCfg`，它接收信息完备的输入模型 `IngressEx`（携带裸 Ingress 及 Endpoints、Probe、Secret 引用、Policy 等周边资源），使配置层只执行不查 API。它双层遍历 `Spec.Rules`：每个 host 生成一个 `Server`，每条 path 生成一个 `Location` 和一个 `Upstream`。关键术语包括：`Upstream` 用 `namespace-ingress-host-service-port` 确定性命名并去重排序、无端点时用 `127.0.0.1:8181` 占位优雅降级；`Location` 的 `proxy_pass` 按 ssl 选协议、path 按 pathType 加修饰符；`addSSLConfig` 做 host 级 TLS 匹配，`createHealthCheck` 复用 Pod Probe（Plus 专属），`generateJWTConfig`/`generateBasicAuthConfig` 采用 fail-closed 设计（Secret 无效仍写路径让 NGINX 返 500）。mergeable Ingress 由 master 提供 host 框架、minion 提供 locations，经 `generateNginxCfgForMergeableIngresses` 合并为单 `Server`。最终产物 `IngressNginxConfig` 经模板渲染、写盘、reload 生效。
