本讲是 NIC（nginxinc/kubernetes-ingress）的入门总览，建立整体心智模型。核心结论有三：其一，Ingress 是 K8s 标准内置的声明式路由规则资源，Ingress Controller 才是真正读取规则、将其翻译成负载均衡器配置并持续调谐的执行程序，二者遵循 watch→发现差异→让实际逼近期望的控制器 reconcile 范式。其二，NIC 由 NGINX/F5 官方维护，支持 NGINX 与商业版 NGINX Plus，与社区项目 kubernetes/ingress-nginx 是两个不互通的独立项目；技术选型上使用原始 client-go（SharedInformerFactory + work queue），而非 controller-runtime。其三，NIC 划分为数据模型、校验、控制器、配置生成、进程管理五层架构，职责单向流动；路由资源分两类——标准 Ingress（加 annotations/ConfigMap 扩展）与自定义资源 VirtualServer/VirtualServerRoute（L7）、TransportServer（L4），辅以 Policy、GlobalConfiguration。数据流骨架为：kubectl apply → Informer 监听 → 入队 → sync 调谐 → 解析 Secret → 生成并渲染配置 → reload → 回写 status。引入的关键术语包括：Ingress、Ingress Controller、控制器模式、VirtualServer/VSR、TransportServer、Policy、GlobalConfiguration、五层架构、annotations/ConfigMap、原始 client-go，供后续讲义承接。
