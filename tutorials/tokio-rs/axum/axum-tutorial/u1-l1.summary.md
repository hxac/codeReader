axum 是建在 hyper 之上的 HTTP 路由与请求处理库，强调好用与模块化，本身是「hyper 之上相对薄的一层」，性能接近 hyper。本讲围绕四个最小模块展开：定位（一句话定义与请求生命周期）、四大特性（无宏路由、声明式提取器、可预测的错误模型、最少样板响应）、生态分层（tokio 运行时 → hyper 协议 → tower/tower-http 服务与中间件 → axum 业务层），以及最关键的设计取舍——不自造中间件而直接复用 `tower::Service`。由此引入的关键术语包括：handler、提取器（实现 `FromRequest`/`FromRequestParts`，如 `Path`/`Query`/`Json`）、`IntoResponse`、`tower::Service`/`Layer`、共享的 `http` crate 类型，以及按需裁剪能力的 Cargo feature（`http1`/`http2` 决定 `axum::serve` 是否存在，`json` 决定 `Json` 是否可用）。核心结论是：`tower` 三件套是 axum 的必需依赖，这是它「免费」获得超时、追踪、压缩、鉴权的根本原因。
