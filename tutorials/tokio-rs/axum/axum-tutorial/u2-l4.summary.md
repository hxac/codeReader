本讲解决「如何把多个 Router 组装成可复用的路由树」，覆盖四个最小模块：Router::nest、StripPrefix、Router::merge，以及作为补充的 layer 与 route_layer 区别。核心结论是：nest 在构建期把子路由的相对路径逐条拼上前缀、摊平注册进父路由的同一棵 matchit 树（因此路径冲突会 panic），而非运行时嵌套查找；它只取子路由的 path_router，显式丢弃其 catch_all_fallback 与 default_fallback，这正是嵌套未命中时冒泡到外层 fallback 的根源。nest_service 用于挂任意 tower::Service，靠 {*NEST_TAIL_PARAM} 通配兜住所有子路径。StripPrefix 是 nest 给每条子路由套的层，请求进入子路由前剥掉已匹配前缀并改写 URI，故子路由 handler 只看到相对 URI，需原始 URI 时用 OriginalUri 提取器。merge 不加前缀、按原路径把另一 Router 的路由表搬进来，两个都带自定义 fallback 的 Router 不能 merge，需用 reset_fallback 丢弃一方。layer 包裹所有路由外加 fallback，route_layer 只包裹已注册路由、不碰 fallback 且无路由时 panic；鉴权中间件应用 route_layer，全局追踪类应用 layer 写在外层。两者都只影响已注册路由且运行在路由匹配之后。
