本讲义以官方 `hello-world` 示例为样本，覆盖三个最小模块，把 axum「hyper 之上薄薄一层」的定位落到可运行的程序上。其一是 **Router**：存放「路径 → 方法路由」映射的容器，用 `Router::new().route(path, get(handler))` 链式组装，空 Router 预置默认 404 fallback。关键术语包括 `Router`（路由树根）、`MethodRouter`（某路径上「方法 → handler」的映射，由 `get`/`post` 等宏生成函数产生）与 `Router<S>` 的泛型 `S`（本讲用默认 `Router<()>`，状态留待后续）。其二是 **`axum::serve(listener, app)`**：把 Router 接入 hyper，内部是永不返回的 accept 循环，每条 TCP 连接被 spawn 成独立任务，由 hyper 解析 HTTP 并调用 `app`；网络 IO 与 HTTP 解析全交 tokio/hyper，axum 只做适配与任务 spawn。需注意 `serve` 仅在同时启用 `tokio` 与（`http1` 或 `http2`）feature 时才被编译出来，是 feature 影响能力的直接体现。其三是 **handler 与 Html 响应**：handler 是返回 `IntoResponse` 的 `async fn`，由 `Handler` 的 blanket impl 自动适配；`Html<T>` 是 newtype，复用元组的 `IntoResponse` 实现自动加上 `text/html` 头。核心心智模型：一条请求从 TCP 进入、经 hyper 解析、路由匹配到 handler、返回值经 `into_response` 写回 TCP，这条调用链是后续所有讲义的共同地基。
