本讲承接 POSIX 后端，讲解 libipc 的 Windows 共享内存后端 `shm_win.cpp`。核心差异在于 Windows 采用「内核对象 + HANDLE 句柄」模型替代 POSIX 的「fd + 磁盘文件」模型，映射对象由系统页面文件（page file）支撑，传 `INVALID_HANDLE_VALUE` 给 `CreateFileMapping` 即创建无磁盘文件的纯内存命名映射。`id_info_t` 用 `HANDLE h_` 取代 `fd_`，且不存 `name_`；嵌入式原子计数器 `acc_`（4 字节对齐）与 POSIX 逐字一致。

关键流程：`acquire` 在创建阶段即用 `CreateFileMapping` 把大小焊死，`open` 模式走 `OpenFileMapping`，并用 `GetLastError()==ERROR_ALREADY_EXISTS` 实现 POSIX 的 `O_EXCL`「严格创建」语义；`get_sa()` 提供 NULL DACL 允许任意进程访问。`get_mem` 用 `MapViewOfFile(…,0)` 映射整个对象，再用 `VirtualQuery` 的 `RegionSize` 为打开者探测真实大小，引用计数在此 `fetch_add`；`release` 做 `fetch_sub` + `UnmapViewOfFile` + `CloseHandle`。

两个点睛结论：其一，`remove(name)` 是空操作（`// Do Nothing.`），因内核对象在所有句柄与视图关闭后由系统自动回收，无需 `shm_unlink`；其二，`Global\` 前缀把命名对象放入会话无关的全局命名空间，使 Session 0 的服务与用户会话的客户端能共享同名对象（`win_service` demo 的核心）。两套后端对外签名一致，由 `platform.cpp` 编译期分流。
