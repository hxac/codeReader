本讲为专家层首篇，从「会用」转向「看懂 libipc 的并发可见性与缓存设计」，聚焦 utility.h、circ/elem_def.h、prod_cons.h 三个模板文件。引入四组关键概念：① release/acquire 配对——生产者用 release 发布游标、消费者用 acquire 解开「包裹」使跨核数据可见，读自身独占游标则用 relaxed；② 伪共享（false sharing）与缓存行（cache line，64 字节），靠 alignas(cache_line_size) 把热游标各隔入独立缓存行，共 7 处；③ CAS 内存序按角色选——读来做决策用 acquire、写来发布用 release、读改写都要则用 acq_rel；④ DCLP 双重检查锁，外层 acquire、内层 relaxed、收尾 release 构成最小必要强度。核心结论：libipc 取「最小必要强度，把同步成本集中到少数发布点」的设计哲学——如广播 push 中 f_ct_ 用 release/acquire 扛数据发布、rc_ 用 relaxed 做无害保守的回收决策、epoch_ 用 acq_rel 撑起跨进程预约握手；内存序按使用点而非变量定，强序只留给真正承载可见性的操作。
