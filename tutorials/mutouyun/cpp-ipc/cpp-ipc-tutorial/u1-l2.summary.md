本讲只解决「把 libipc 构建出来并跑通」，不碰库内部实现。它覆盖四个最小模块：①顶层 `CMakeLists.txt` 用 `option()` 提供五个构建开关（`LIBIPC_BUILD_TESTS`/`DEMOS`/`SHARED_LIBS`/`USE_STATIC_CRT`/`CODECOV`），默认全 OFF，开箱只构建库本体；②`src/CMakeLists.txt` 把 `src/libipc` 源码收集成库目标 `ipc`（默认静态库 `lib/libipc.a`），非 Windows 平台自动链接系统库 `pthread` 与 `rt`，并用 `PUBLIC` 暴露 `include/` 头文件目录；③`demo/send_recv` 子目录靠 `target_link_libraries(自己 ipc)` 继承头文件，发/收两个进程用同名字符串 `"ipc"` 连上同一条通道来验证收发；④`vcpkg install cpp-ipc` 一键安装，底层跑同一套 CMake 脚本，产出带 `cpp-ipc::ipc` 命名空间的导出目标。核心结论：开关决定构建什么、`add_subdirectory` 决定进入哪个目录、库产物进 `lib/` 而可执行文件进 `bin/`。本讲引入的关键术语：CMake 构建脚本生成器、构建目标 `ipc`、静态/动态库、`option` 开关、传递式包含目录、生成器表达式、`channel` 的 `sender`/`receiver` 角色与同名通道机制。
