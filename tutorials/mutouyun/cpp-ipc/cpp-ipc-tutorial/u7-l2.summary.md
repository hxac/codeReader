本讲解析 libipc 内存子系统中的「单调缓冲」策略层。核心是 `monotonic_buffer_resource`：一条可增长内存带，用 `head_`/`tail_` 游标做 **bump 分配**（指针前移），靠 `std::align` 一并完成对齐与判满；缓冲不足时**委托上游 `upstream_`（默认堆）增长**，增长因子为 **×3/2**（`next_buffer_size`），新节点经 `make_node` 挂入 `free_list_`，实现摊还 O(1)。单调性决定了它**只能整体释放**：`deallocate` 是空操作，`release` 仅回收自己长出的节点、并复位但不释放借来的初始缓冲。关键结论：`central_cache_allocator()` 单例用一块 1MB `static std::array` 作初始缓冲，外包 `thread_safe_resource`（monotonic + `std::mutex`，靠名字隐藏加锁而非虚函数），再经 `bytes_allocator` 类型擦除对外暴露。这 1MB 装的是 **central cache 的元数据**——`chunk_t`（批量块）与 `node_t`（无锁栈节点），由 `central_cache_pool::construct` 消费；溢出部分回落堆。引入的关键术语：bump 分配、monotonic buffer、upstream 委托、thread_safe_resource、central cache 元数据。central cache 选 monotonic 是「天作之合」：其块级回收走自己的无锁栈，恰好不需要按块释放。下讲 u7-l3 将接续讲清谁调用 central cache（thread-local `block_pool` 的分级路由）。
