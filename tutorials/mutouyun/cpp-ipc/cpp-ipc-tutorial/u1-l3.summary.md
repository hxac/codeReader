本讲建立 libipc 的「目录→模块」导航地图。核心结论：`include/libipc/` 是唯一对外暴露的目录，8 个顶层头各管一个能力域——`ipc.h`（通道收发，聚合门面）、`def.h`（基础类型与全局常量，如 `default_timeout`、`data_length`）、`buffer.h`（消息容器 `buff_t`）、`shm.h`（共享内存）及 `mutex/condition/semaphore/rw_lock` 同步原语，仅 `#include "libipc/ipc.h"` 即获全部能力。`src/libipc/` 是库实现，分核心/circ/mem/sync/platform/utility/imp 七层，其中 `circ/`、`utility/` 为**头文件库**（无 `.cpp`，靠模板在使用处实例化）。判断某目录是否产生机器码的硬证据：是否出现在 `aux_source_directory` 列表中。`platform/` 用 `posix/linux/win` 同名头文件子目录配合编译宏与 include 路径实现**平台分派**，调用方代码不变。公共 API 采用 PIMPL，对外仅暴露 `void*` 不透明句柄 `handle_t`。`test/`（依赖 gtest，开关 `LIBIPC_BUILD_TESTS`）与 `demo/`（无依赖，`LIBIPC_BUILD_DEMOS`）默认 OFF，`archive/` 旧测试被显式排除；`demo/send_recv` 靠同名通道字符串连同一共享内存通道完成收发。
