本讲（u1-l4）以 libipc 公共 API 写出第一个跨进程通信程序，只讲用法、不涉内部实现。核心最小模块有四个：①通道构造与模式位——一条通道由生产者/消费者多重性 `Rp/Rc` 与传输方式 `Ts`（unicast/broadcast）决定，库预设两个类型别名 `route`（单写多读广播）与 `channel`（多写多读广播）；用 `ipc::channel`/`ipc::route` 构造，传 `name` 与 `mode`（`ipc::sender`/`ipc::receiver`）选角色，**同名即同一通道、构造即连接、析构即销毁**（RAII + PIMPL）。②send/recv 调用——`send` 有三重载（指针/`buff_t`/`std::string`），其中 `send(std::string)` 多发 1 字节 `\0`；`send` 超时走 `force_push` 强制发送并返回 true，`try_send` 超时直接返回 false；`recv` 默认 `invalid_value` 无限阻塞，超时返回空 `buff_t`。③`buff_t` 接收——`recv` 返回统一容器 `buff_t`（即 `buffer`），**第一件事必须 `empty()` 判空**，读内容用 `data()`/`size()`、`get<T>()`、`to_vector()`。④优雅退出——发送方 `disconnect()` 会唤醒对端阻塞的 `recv` 使其收到空消息；用 `signal()` 捕获 Ctrl+C、置原子标志并 `disconnect()`。关键术语：`route/channel`、`sender/receiver`、`mode`、`handle_t`、`buff_t`、`default_timeout`、`invalid_value`、`force_push`、`disconnect()`、广播语义。
