本讲承接 u4-l1 的队列抽象层，向下钻入真正躺在共享内存里的对象，核心模块为 `elem_array` 与其继承的 `conn_head` 连接位图，仅涉及 elem_def.h、elem_array.h 两个头文件库。

`elem_array` 是跨进程共享的三段式结构：连接头 `conn_head` + 策略头 `head_`（读写游标 rd_/wt_/ct_，留待 u4-l3/u4-l4）+ 256 槽元素块 `block_`，其 `sizeof` 即共享内存大小。256 槽源于 `elem_max = uint8_max+1`，循环回绕靠 `index_of(c) = static_cast<u1_t>(c)`，等价 `c%256` 且零分支。

`conn_head_base::init()` 用双重检查锁（DCLP）配 `acquire`/`release` 内存序，保证多进程下共享内存对象只被 placement-new 构造一次：锁外 `acquire` 读 `constructed_` 走快路径，锁内 `relaxed` 二次检查防重复构造。

关键术语：`cc_`（全座位图，`atomic<cc_t>`，`cc_t = uint32`）、`cc_id`（本接收者的单一 bit 座位号）。广播特化 `conn_head<P,true>` 用位运算管理座位：`connect()` 以 `curr|(curr+1)` 抢最低空闲位、CAS 循环配 `yield` 退避、返回 `next^curr` 作为 `cc_id`；`disconnect()` 用 `fetch_and(~cc_id)` 原子归还；`conn_count()` 用 Kernighan 算法数位。满座时 `curr+1` 无符号回绕为 0 致 `next==curr` 返回 0，即 32 接收者上限的根因。单播特化 `conn_head<P,false>` 纯计数不分位，故无此限制。

`sender_checker`/`receiver_checker` 按 `relat_trait` 的 `is_multi_producer`/`is_multi_consumer` 偏特化，体现「发送者只占一个 bool、接收者占一个 bit」的不对称设计；`elem_array` 把基类裸 `connect` 私有化，强制每次连接都走 checker 落实多重性约束。本讲不展开 `prod_cons_impl` 的无锁读写算法。
