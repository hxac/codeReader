本讲揭示 libipc 如何用「一份源码覆盖多平台」，核心是四个平台层文件。`detect_plat.h` 是平台「宪法」，用逐级 `#elif` 加兜底 `#error`，把编译器预定义宏（`__linux__`/`_WIN32` 等）统一翻译成自有的 `LIBIPC_OS_*`/`LIBIPC_CC_*`/`LIBIPC_INSTR_*` 宏，保证要么命中已知平台、要么编译失败。`platform.cpp` 是共享内存后端的「编译期分派器」，靠 `#if defined(LIBIPC_OS_*)` 在编译期只 `#include` 一份实现（Windows→`shm_win.cpp`，Linux/QNX/FreeBSD→`shm_posix.cpp`），两份后端对外签名一致、内部各异，落地 u5-l1 的 `shm::handle` 与 `id_t` 不透明句柄。`platform.c` 以 `.c` 后缀强制 C 编译，仅在 Linux 收编内嵌的 a0（AlephZero）纯 C 库，借 futex 提供 robust 互斥量 `a0_mtx_t`，其他平台留空（走 Win32 abandoned mutex 或 pthread robust）。`detail.h` 是公共头兼 C++14/17 兼容垫片，引入 detect_plat.h 并用 `IPC_STBIND_` 等宏抹平版本差异。整套机制的本质是「统一接口、各自实现、编译期分流」。
