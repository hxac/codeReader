本讲聚焦 libipc 跨进程健壮互斥量 `ipc::sync::mutex`，核心解决持锁进程崩溃后其他进程能否继续获锁、避免永久死锁。它经 PIMPL 加编译期宏分派到 Linux(a0)、POSIX(pthread)、Windows 三套后端，API 统一。引入的关键术语：健壮锁、命名互斥量（同名即同一把锁——Linux/POSIX 把锁对象放进命名共享内存，Windows 用内核命名对象）、一致性恢复（consistent），以及三类死亡检测——Linux robust futex 与 POSIX `PTHREAD_MUTEX_ROBUST` 均返回 `EOWNERDEAD`，Windows abandoned 返回 `WAIT_ABANDONED`。核心结论：拿到死亡锁后 Linux/POSIX 必须调 consistent 修复，否则未修复即解锁会使锁永久报废（`ENOTRECOVERABLE`），Windows 无此步骤；恢复路径上 Linux 与 Windows 走"释放后重抢"，POSIX 走"consistent 后直接持锁"。Linux/POSIX 另用进程内 curr_prog 缓存表加原子引用计数做同名句柄去重，Windows 因内核对象自管理无需此结构。
