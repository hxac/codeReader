本讲给出 OpenOFDM 的解码流水线「导航图」，不展开任何模块算法，只讲全局拼图。核心结论：802.11 OFDM 接收是一条 8 步概念流水线（包检测→频偏校正→FFT→信道估计→解调→解交织→卷积解码→解扰），是发射机的严格逆操作，可归为前端同步、频域还原、比特还原三组。关键概念「概念步骤≠模块一一对应」：FFT（第③步）藏在 `sync_long.v` 内部（Xilinx IP `xfft_v7_1 dft_inst`），并非独立模块；频偏校正的 `phase`/`rotate` 原语则在多模块重复出现。顶层 `dot11.v` 几乎不写算法，只按 `power_trigger → sync_short → sync_long(含FFT) → equalizer → delayT → ofdm_decoder → crc32` 连数据平面，并用状态机做控制平面。引入术语：子载波、FFT、信道估计/均衡、星座解调、Viterbi 卷积解码、LFSR 解扰。`ofdm_decoder.v` 是 5 级子流水线（demodulate→deinterleave→viterbi→descramble→bits_to_bytes），位宽呈「32→6→2→1→8」收窄再汇合。全项目统一「数据＋strobe」单向无反压握手；`phase` 模块由 `sync_short` 与 `equalizer` 分时复用，`rot_lut`（双口 RAM）由 `sync_long` 与 `equalizer` 双口共享，以省 FPGA 资源。
