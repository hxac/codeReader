本讲围绕 `dot11.v` 顶层展开，回答「OpenOFDM 在 Spartan 3A-DSP 上如何用最小实例塞下完整 OFDM 解码链」。核心结论是三种 FPGA 资源共享范式：**实例复制**（可并发但费资源）、**双口 RAM 并行共享**（存储类优选，第二端口近乎免费）、**分时复用 TDM**（计算类，用 MUX 在输入端切换、输出端广播，要求消费者时间不重叠）。具体落点在三处：`rot_lut` 旋转因子表用一块**真双口 BRAM** 的 A/B 两端口同时服务 `sync_long` 与 `equalizer`；`phase` 相位流水线（`div_gen` 除法器 + `atan_lut`）因计算型无法双口，改用 `state == S_SYNC_SHORT` 这一个条件做 MUX，在 `sync_short`（粗 CFO）与 `equalizer`（导频细 CFO）间分时复用；`complex_mult` 因消费者需并发而选择复制。共享得以无痛成立，靠全项目统一的「**数据 + strobe**」握手与 `delayT` 延时原语——后者将 strobe「**错相位**」对齐到流水线输出（如 phase 内 `DELAY(36)` 匹配除法延迟）。设计规律是：昂贵且时间不重叠的资源共享，廉价且需并发的资源复制。
