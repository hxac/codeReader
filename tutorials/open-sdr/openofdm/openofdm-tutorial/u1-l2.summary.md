本讲解决「如何把 OpenOFDM 跑起来」，是后续源码精读的前置条件。核心内容覆盖四块：工具链、构建脚本、编译清单、测试台。仿真三件套为 iverilog（编译）、vvp（运行）、gtkwave（看波形），依赖声明在 Readme.rst。关键结论一：Makefile 中真实可用的 target 只有 compile、simulate、all、clean；注释里的 make check / make display 并未实现，需分别用 make compile 和手动 gtkwave 替代，且 make simulate 会自动触发编译。关键结论二：dot11_modules.list 是 iverilog 的 -c 命令文件，含 Xilinx 库搜索路径（unisims/XilinxCoreLib）、手写 RTL、usrp2/ 平台模块与 coregen/ IP 的仿真行为模型；引用的是 coregen IP 的 .v 行为模型（FFT、Viterbi、除法器），故无 Xilinx 综合工具链也能仿真。关键术语：测试台 dot11_tb.v 用 $readmemh 加载 testing_inputs 下的 hex 样本（高 16 位 I、低 16 位 Q），以 #5 翻转产生 100MHz 时钟，并用 clk_count==4 实现 5:1 喂样节拍——这源于 FPGA 时钟 100MHz 与输入采样率 20MSPS 之比；各阶段信号落盘到 sim_out/ 并生成 dot11.vcd 波形。默认样本为 24Mbps dot11a 样本，NUM_SAMPLE 默认 3000。需重点观察的信号链为 power_trigger → short_preamble_detected → dot11_state 状态机。requirements.txt（numpy/wltrace）属 Python 交叉验证依赖，本讲仿真无需安装；注意 scripts/test.py 还隐式依赖 scipy 且为 Python 2 语法，留待后续单元。
