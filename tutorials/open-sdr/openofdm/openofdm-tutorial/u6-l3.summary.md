本讲讲解 OpenOFDM 解码流水线中由 Xilinx IP 核（黑盒）承担的重运算单元——FFT、Viterbi、复乘、除法及查找表——它们集中放在 `verilog/coregen/` 下，由 Xilinx 的 CORE Generator（coregen）工具生成。

核心一是 `dot11_modules.list`：它是 iverilog 的命令文件（`-c`），分四段——Xilinx 库搜索路径（`-y`）、手写 RTL、USRP 平台模块、coregen IP 仿真模型加一个原语；测试台 `dot11_tb.v` 由 Makefile 单独追加。

核心二是一个 IP 的文件族与两种 `.v` 风格：`.xco` 是配方、`.ngc` 是综合/上板网表、`.v` 是仿真模型、`.veo/.vho` 是实例化模板、`.coe/.mif` 是表数据，`.ngc` 与 `.v` 描述同一 IP。风格 A（viterbi_v7_0、xfft_v7_1、div_gen_v3_0、complex_multiplier）是 netgen 扁平网表，由 UNISIM 原语（LUT/FF/进位链/DSP48A）组成、声明不可再综合；风格 B（atan_lut、rot_lut、deinter_lut）是薄封装，在 `translate_off` 内例化 `BLK_MEM_GEN_V4_2` 并标为 black_box。两种风格分别依赖 `unisims` 与 `XilinxCoreLib` 仿真库。

关键结论：用 iverilog 仿真不需跑 Xilinx 综合，但仍需这两套库存在；FFT 由 `xfft_v7_1` 在 `sync_long` 中以正向变换承担；complex_multiplier 用 DSP48A 解释了必须落在 Spartan 3A-DSP（USRP N210）的原因。关键术语：IP 核、coregen、netgen、UNISIM、XilinxCoreLib、黑盒、translate_off/on。
