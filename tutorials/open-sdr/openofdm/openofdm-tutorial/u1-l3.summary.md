本讲是 OpenOFDM 仓库的「地图课」，建立全局导航图。核心结论：仓库硬件源码天然分三类——手写 RTL（作者自写解码逻辑，如 `dot11.v`、`sync_long.v`，是逐行精读对象）、Xilinx IP/库（`coregen/` 的 FFT、Viterbi、除法器等 coregen IP 加 `Xilinx/` 库）、USRP 平台代码（`usrp2/` 的 `setting_reg.v`、`ram_2port.v` 胶水模块），这一划分源于设计目标上板到 USRP N210 的 Spartan 3A-DSP FPGA。关键术语：RTL、IP 核（coregen 黑盒）、命令文件。引入三个关键文件：`dot11_modules.list`（iverilog 编译清单，分 Xilinx 库搜索路径、手写 RTL、usrp2 模块、coregen IP 四段，测试台 `dot11_tb.v` 不在其中）、`scripts/`（Python 工具链：`gen_*_lut.py` 生成查找表、`bin_to_mem.py` 转样本、`decode.py`/`test.py` 做浮点参考解码与逐阶段交叉验证，带修改版 `commpy/`）、`testing_inputs/`（conducted 同轴直连干净样本与 radiated 空口样本，仿真实读经转换的 `.txt`）。核心概念：项目自包含，coregen IP 用 `.v` 行为模型，故无需 Xilinx 工具即可用 iverilog 仿真；`docs/source/overview.rst` 是全仓库目录分工的权威说明。后续讲义将落到 `dot11.v` 顶层模块与 OFDM 解码流水线。
