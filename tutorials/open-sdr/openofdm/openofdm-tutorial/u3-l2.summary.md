本讲介绍 OpenOFDM 接收链路复用的通用原语（积木），不涉及 OFDM 算法本身。`complex_mult` 封装 Xilinx `complex_multiplier` IP（DSP48A 硬核），采用「输入寄存→IP→输出寄存 + `delayT` 延时 strobe」三段式，保证 `output_strobe` 与多级流水线数据对齐，是全项目「数据 + strobe」单向握手的范本。`complex_to_mag` 用 α=1、β=1/4（`max + min>>2`）做无乘法无开方的近似幅值，平均误差约 0.6%、最坏低估约 11.6%（45° 处），仅用于比大小/判门限。`complex_to_mag_sq` 利用「信号乘自身共轭 z·z̄=I²+Q²」复用一个 `complex_mult` 得到精确幅值平方。`delayT` 是每时钟无条件移位的固定延时链，用于周期级对齐，须与按样本推进的 `delay_sample` 区分。`moving_avg` 用双口 RAM 环形缓冲 + 滚动和右移实现 2^k 窗口平均，有窗口预热延迟；`calc_mean` 先各除 2 再相加、可选取反，安全地求两路平均。这些原语支撑后续 sync_short、sync_long、equalizer、rotate 等模块。
