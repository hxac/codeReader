本讲讲解 `demodulate.v`，它在八步解码流水线中是「频域复数 → 比特」的转折点：承接均衡器（equalizer）输出的归一化复数星座点，将其判决为最多 6 比特，再交给解交织（deinterleave），本身不涉及信道或交织。模块用 `case({rate[7], rate[3:0]})` 把上层送来的 `rate` 字段映射到 BPSK / QPSK / 16-QAM / 64-QAM 四种星座（localparam BPSK/QPSK/QAM_16/QAM_64），覆盖 802.11a 全速率与 802.11n MCS 0–7。关键约定是定点刻度 `CONS_SCALE_SHIFT = 10`（故 `MAX = 1024`），由此派生判决门限 `QAM_16_DIV = 682`、`QAM_64_DIV_{0,1,2} = 292 / 585 / 877`；这些门限的比值 2/3 与 2/7、4/7、6/7 正好是星座相邻幅度等级的中点占最外层幅度的比例，背后是 demodulate 与 equalizer「最外层星座点幅度 = MAX」的隐含契约。判决统一为「取反符号位 + 幅值门限比较」：BPSK/QPSK 仅用符号位，16-QAM 每轴加一个比较，64-QAM 每轴加两个比较并自然形成格雷码。模块为两级流水线，输出 strobe 由 `delayT` 延时 2 拍对齐，延续全项目「数据 + strobe」握手风格。本讲确立的 rate 字段、定点刻度与格雷码比特排列，是后续解交织讲义的前置基础。
