本讲聚焦 openwifi 六个自研 IP 共享的「时钟与采样率」配置层，核心是两份分工不同的配置文件：手写的 `board_def.v`（板卡无关的常量契约，定义采样率 `SAMPLING_RATE_MHZ=20` 及派生宏）与构建期由 Tcl 现场生成的 `clock_speed.v`（板卡相关的 `NUM_CLK_PER_US` 与 `SMALL_FPGA`，不进 git）。关键陷阱是 `NUM_CLK_PER_US` 在 `board_def.v` 中仅有注释，真值来自 `clock_speed.v`，IP 源码必须同时 include 两份。

`NUM_CLK_PER_US` 即每微秒时钟周期数（等于基带时钟 MHz 数），由它派生出每样点时钟数 `NUM_CLK_PER_SAMPLE`、1µs 计数上限 `COUNT_TOP_1M`、与软件 10MHz 计数器的换算标尺 `COUNT_SCALE`，改基带时钟时自动同步。

构建链上 `ip_repo_gen.tcl` 先写一版 `clock_speed.v`，随后 `openwifi.tcl` 又覆盖一版，最终生效值由后者顶部 `set NUM_CLK_PER_US` 决定（README 指定的改时钟入口）。板级规模开关 `SMALL_FPGA`（给 tx_intf/rx_intf/xpu）与 `SIDE_CH_LESS_BRAM`（给 side_ch，写在 `fpga_scale.v`）由 `parse_board_name.tcl` 的 `fpga_size_flag` 驱动，把 DMA FIFO 深度从 8192 裁到 4096。这些宏在源码中驱动 1µs TSF 脉冲、SPI 分频、MAC 定时标尺与分数样点自适应。
