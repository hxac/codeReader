本讲深入 block design 内部，讲解 `openwifi_ip` 层级单元如何打包并互连 openwifi 的自研 IP。核心分两块：一是层级本身——它把 WiFi IP 整体封装，对外只暴露 AXI/ADC/DAC/中断接口，本质是 `write_bd_tcl` 导出的层级蓝图（定义过程 `create_hier_cell_openwifi_ip`），可用 `source` 重建、跨板卡迁移；普通 Zynq 版实例化 5 个自研 IP，UltraScale+ 版含全部 6 个（多 `side_ch` 可观测通路）。二是 IP 互连：发射链路 PS→DMA→`tx_intf`(写BRAM)→`openofdm_tx`(生成I/Q)→`tx_intf`→DAC；接收链路 ADC→`rx_intf`(出样点)→`openofdm_rx`(解字节)→`rx_intf`(拼AXIS)→DMA→PS；`xpu` 是控制中枢，`openofdm_rx` 输出常一拖二同时送 `rx_intf` 与 `xpu`。关键术语：层级单元（hierarchical cell）、VLNV（`user.org:user:` 前缀为自研、`xilinx.com:ip:` 为标准件）、AXI-Stream/AXI4-Lite、BRAM、CSI、共享参数文件 `board_def.v`（约定采样率 20MHz 与每采样时钟数，是六 IP 协同的底层契约）。
