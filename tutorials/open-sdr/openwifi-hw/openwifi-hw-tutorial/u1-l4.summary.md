本讲梳理 openwifi-hw 从克隆仓库到产出 system_top.xsa 的四阶段脚本接力链。阶段①三个「只跑一次」的准备脚本：prepare_adi_lib.sh 填充并编译 adi-hdl 通用库（锁定 2022_R2 分支）、prepare_adi_board_ip.sh 把 BOARD_NAME 映射到 adi-hdl 板级参考设计（社区小板多共用 adrv9364z7020/ccbob_lvds 底座）、get_ip_openofdm_rx.sh 填充 openofdm_rx 子模块。阶段②是核心：create_ip_repo.sh 由当前目录名反推 BOARD_NAME 并生成条件编译宏 _pre_def.v，触发 ip_repo_gen.tcl 生成 clock_speed.v/fpga_scale.v/git_rev.v 等参数化文件，循环打包 xpu/tx_intf/rx_intf/openofdm_tx/openofdm_rx/side_ch 六个自定义 IP 进 ip_repo（openofdm_rx 因是外部子模块改用追加模式保留自带定义），末尾 source openwifi.tcl 创建顶层工程 openwifi_$BOARD_NAME 并覆盖 clock_speed.v（基带时钟最终决定点）。阶段③在 GUI 跑 Generate Bitstream 与 Export Hardware。阶段④ sdk_update.sh 把 .xsa/可选 .ltx/git_info.txt 投递到镜像目录供软件仓库消费。关键概念：fpga_size_flag 驱动 SMALL_FPGA/SIDE_CH_LESS_BRAM 宏以适配不同规模器件，这是同一份源码跨 zynq 7020 到 zcu102 的关键。
