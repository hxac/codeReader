本讲覆盖 openwifi 上板后的两类片上可观测手段。其一为 LED/GPIO 映射（慢速、定性、零成本）：`gpio_led.md` 给出「物理引脚 ↔ FPGA 内部信号」的权威对照表。事件型信号（如 `cycle_start0`、`sig_valid`、`phy_tx_started`、`fcs_ok`）采用 **flip** 风格，经 `edge_to_flip`（T 触发器，上升沿翻转，事件率除以 2 后肉眼可辨）；状态型信号（如 `slice_en`、`*_is_ongoing`）采用 **raw** 风格，直接反映电平。其二为 ILA 调试（快速、定量）：通过 `create_ip_repo.sh` 传入 `ENABLE_DBG`，生成 `` `define IP_NAME_ENABLE_DBG ``，使 `` `DEBUG_PREFIX `` 展开为 `(*mark_debug="true",DONT_TOUCH="TRUE"*)`，给选定信号（如 `tsf_pulse_1M`、`backoff_done`、`start_retrans`）打上保留属性，综合后经 Vivado Set Up Debug 手动插核，生成 `.bit` 与 `.ltx` 探针文件，再由 `sdk_update.sh` 随镜像交付，最终在 Hardware Manager 中抓取纳秒级波形。`ENABLE_DBG` 默认关闭，仅调试时开启以节省 BRAM/逻辑。
