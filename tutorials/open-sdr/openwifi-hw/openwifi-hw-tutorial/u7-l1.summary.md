本讲承接 u2-l3 与 u5-l1，讲清 `slv_reg` 在 RTL 中的实现，以及软件 `readl/writel` 如何落到具体寄存器。核心结论：每个自研 IP 的 `*_s_axi.v` 是基于 Xilinx 模板裁剪的 AXI4-Lite 从设备「协议适配壳」——对外讲五通道（AW/W/B/AR/R）握手，对内暴露一组 `slv_regN`，本身不含算法；采用单 outstanding 风格，AW 与 W 同拍握手才产生一拍写脉冲 `slv_reg_wren`，读路径为组合译码加一拍寄存输出。地址解码由 `ADDR_LSB`、`OPT_MEM_ADDR_BITS` 决定，32 位数据下寄存器号取 `addr[ADDR_LSB+OPT_MEM_ADDR_BITS:ADDR_LSB]`、字节偏移 = N×4；xpu 为 8 位地址/64 槽，其余四 IP 为 7 位/32 槽，读未实现槽返回 0。寄存器语义不在壳而在顶层 `xpu.v`：slv_reg0 做按位软复位、slv_reg1[2] 为 RX DMA 过滤总闸、slv_reg9[31] 覆盖 IFS/时隙、slv_reg57/58/59/62/63 聚合状态与版本号，构成软硬件交互契约。引入的关键术语：slv_reg、slv_reg_wren、内存映射、字节选通 WSTRB、地址解码。
