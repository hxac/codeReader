本讲拆解 openwifi-hw 的条件编译体系，含两个最小模块：`_pre_def.v` 生成机制与条件编译宏分类。`_pre_def.v` 是构建期现场生成、不进 git 的配置快照，由脚本把命令行参数翻译成 Verilog `` `define ``，再由源文件 `` `include `` 拉入用 `` `ifdef `` 裁剪代码。注入有两条对齐路径：顶层 `create_ip_repo.sh`（覆盖式、每次干净）与单 IP `create_vivado_proj.sh`→`<ip>.tcl`（append 式、迭代需先清旧文件）。核心约定是宏名按「大写 IP 名加下划线加用户 DEF」加前缀（如 `XPU_ENABLE_DBG`），配合按 IP 区分文件名，解决六 IP 同工程命名冲突；对外部子模块 openofdm_rx 采用追加而非覆盖以保其自带定义。宏分四类：板卡派生（`BOARD_NAME`、`NUM_CLK_PER_US`、`SMALL_FPGA`、`SIDE_CH_LESS_BRAM`）、特性开关（`HAS_SIDE_CH`/`NO_SIDE_CH`）、用户调试（`*_ENABLE_DBG`→`mark_debug` 供 ILA 抓波形）、运行时配置（`OPENWIFI_HW_GIT_REV`、`SPI_*`）。关键结论：基带时钟 `NUM_CLK_PER_US` 真值在 `clock_speed.v` 且被 `openwifi.tcl` 二次覆盖，改时钟唯一正确入口是 `openwifi.tcl`；只有四个自研 IP 真正消费 `*_ENABLE_DBG`，openofdm_* 传了也无害。
