本讲解决「新板卡或 ADI/Vivado 大版本升级时如何把 openwifi 搬过去」这一工程问题，含三个最小模块。板名解析：字符串 `BOARD_NAME`（取自当前目录名而非命令行参数）经 `parse_board_name.tcl` 查表派生出 `part_string` 及两个彼此独立的标志——`fpga_size_flag` 管「砍不砍资源」（小器件时生成 `SMALL_FPGA`、`SIDE_CH_LESS_BRAM` 宏裁剪 DMA FIFO 与 BRAM 深度，不改 IP 列表），`ultra_scale_flag` 管「走 PS7 还是 PS8 那套 Tcl」；该表被顶层工程、IP 仓、单 IP 三处共享 `source`。层级迁移：README 给两法，Method 1 让 Vivado 自动升级旧工程再 diff 回补，Method 2 用 `write_bd_tcl -hier_blks` 导出 openwifi_ip 层级单元，到新 ADI 设计 `source` 后调用 `create_hier_cell_openwifi_ip` 重建，仓库里的 `openwifi_ip*.tcl` 即此 proc 的手工维护版。UltraScale 适配：PS7/PS8 两套 Tcl 差异集中在 IP 数（5 vs 6，关键看 side_ch）、DMA 位宽（64 vs 128）、PS 端口（`sys_ps7/*` 与 `FCLK_CLK2` 对 `sys_ps8/*_FPD` 与 `pl_clk2`）、中断拼接（PS7 由顶层逐位接 `sys_concat_intc`，PS8 层级内自带 xlconcat 拼好故接线行被注释）；`ultra_scale_tcl_gen.sh` 仅 sed 替换器件型号与板 VLNV，不处理结构性差异，不能单靠它生成完整 PS8 核心层级。
