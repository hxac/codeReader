本讲聚焦接收链路的 OFDM 物理层核 `openofdm_rx`：它源自上游 openofdm 项目、经 openwifi 在 `dot11zynq` 分支适配后以 git 子模块挂载于 `ip/openofdm_rx`，克隆后为空，需运行 `get_ip_openofdm_rx.sh`（内部 `git submodule init/update`）填充。其顶层 `dot11` 内含 `ofdm_decoder`、`viterbi`，完成包检测、同步、FFT、信道均衡、解映射与 Viterbi 译码及 FCS 校验，支持 802.11 a/g/n。对外端口由 `openwifi_ip.tcl` 固定：样点 `sample_in` 来自 `rx_intf`；字节流 `byte_out`、`fcs_ok`、`pkt_rate/len`、包头等输出「一拖二」同时送 `rx_intf`（拼包上报 DMA）与 `xpu`（MAC 头解析、地址过滤）；寄存器口 `s00_axi` 经 `axi_interconnect_1` 的 `M05_AXI` 暴露给 PS，并通过共享网络与 `xpu` 协同 RSSI。关键打包差异：`ip_repo_gen.tcl` 对它「打包前不复制板级文件、打包后用 `cat >>` 追加 `_pre_def.v`」，以保留子模块自带定义（这正是提交 `b6a3231` 修复的点），区别于其余五个自研 IP 的覆盖式同步；单独建仿真工程时第三参数为 `SAMPLE_FILE` 而非普通宏。
