本讲建立了 Mayoiuta 仓库的全局导航地图。仓库由两棵源码树组成：`hardware/rtl/`（Verilog RTL 硬件电路）与 `driver/win32/`（Windows WDF 内核驱动），外加 README 与 LICENSE。RTL 源码按六大功能域分目录：top（顶层）、core（计算单元）、memory（存储单元）、power（能效 DVFS）、sparse（稀疏加速）、control（动态形状适配）。关键结论有四：一是 README 的四类模块（Compute/Memory/Interconnect/Control Unit）中，互连网络（IN）没有独立目录，而是由顶层 `npu_soc.v` 中的 `noc_data`/`noc_ctrl` 网线实现；二是文件数不等于模块数，9 个 `.v` 文件含 10 个模块，`pe_array.v` 同时声明了 `PE_Array` 与 `Processing_Element`；三是顶层模块 `NPU_SOC` 是地图原点，通过参数 `CORES` 与 `generate-for` 循环例化多个核，用 `(i+1)%CORES` 连成环形 NoC，体现可扩展性；四是驱动三件套中 INF 的 PCI 厂商/设备 ID（`0x1ACE`/`0xBEEF`）必须与头文件宏一致才能绑定硬件。本讲引入的关键术语包括：RTL、Verilog module、顶层模块、例化、generate-for、环形 NoC、`.inf` 安装清单，以及「待确认」标注——`NPU_SOC` 引用的 `npu_controller`/`npu_core`/`performance_monitor` 源码当前仓库未提供，不臆造。
