本讲是 ICU4X 仓库的导航地图，给出三条核心结论。其一，顶层四大目录各司其职：components 是用户直接调用的国际化算法（日期、数字、排序等），provider 提供可插拔的 locale 数据机制，utils 是零拷贝、低内存的通用 Rust 工具地基，ffi 用 Diplomat 做多语言绑定；主线为「components 用数据、provider 供数据、utils 当地基、ffi 做包装」。其二，整个仓库是一个 Cargo workspace：根 Cargo.toml 用 members 点名近 90 个 crate，以 [workspace.package]、[workspace.dependencies] 集中声明元数据与依赖版本，子 crate 用 workspace = true 继承；共享依赖同时写 version（供 crates.io）与 path（供内部开发），普遍 default-features = false 以支持 no_std 与小体积；examples 被 exclude 以模拟外部用户、验证公开 API。其三，三套命名约定：组件 icu_*、数据 icu_provider_* 与 icu_*_data、工具用短名无前缀。关键术语：crate、workspace、元 crate（icu 仅把各组件 re-export 为 icu::* 模块、自身无逻辑）、baked data（compiled data 实物）；名字带 experimental 或版本为 0.x 表示未稳定。
