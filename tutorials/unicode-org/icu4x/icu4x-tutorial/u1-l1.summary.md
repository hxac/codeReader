ICU4X 是 Unicode 联盟用 Rust 从零重写的模块化国际化组件库，面向客户端与资源受限环境，源自 ICU4C/ICU4J 与 ECMA-402 的设计经验、依赖 CLDR 数据。它不取代服务端的 ICU4C/ICU4J（后者仍是服务端黄金标准），而旨在统一替代零散的客户端国际化方案（如 Closure i18n、Dart Intl）。本讲确立四大设计目标：小而模块化（charter 更强调 fast、low-memory）、可插拔 locale 数据、跨语言易用、由 i18n 专家编写（正确性优先），且这些目标直接映射到仓库目录：components/ 承载模块化组件、provider/ 承载可插拽数据（引入 DataProvider 抽象）、ffi/ 承载跨语言（Diplomat 桥接）、utils/ 承载小体积/零拷贝工具。目标平台覆盖 Web、移动端、穿戴/低功耗（WearOS/WatchOS）、Flutter、Fuchsia；目标语言覆盖 Rust、JS、Objective-C、Java、Dart、C++，通过 crates.io/npm/pub.dev 实际分发，并强调 no_std 子集以支持嵌入式。关键术语：国际化（i18n）、locale、CLDR、ECMA-402、FFI/WASM、零拷贝、no_std。
