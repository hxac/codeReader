本讲聚焦 ICU4X 的 `components/plurals`（对外 `icu::plurals`）模块，回答「一个数在某语言里该用哪种复数形式」。核心结论：Unicode 定义六种 CLDR 复数类别（zero/one/two/few/many/other），但每种语言只用其中一部分——英语基数仅 one/other，俄语用四种，中、日、韩等只用 other。

引入的关键概念：①基数（cardinal，表达「几个」）与序数（ordinal，表达「第几」）是两套独立规则，各有数据 marker `PluralsCardinalV1` / `PluralsOrdinalV1` 与构造函数；②操作数（operands）将数字拆成 `i/v/w/f/t/c` 等零件供规则求值，`v/f` 含末尾零而 `w/t` 不含，正是末尾零信息使 ICU4X 拒绝 `f64`、要求用 `Decimal` 或整数。

调用链为：数字经 `PluralOperands` 拆解 → 加载规则数据 → resolver 的 `test_rule` 求值 → `PluralRules::category_for` 返回一个 `PluralCategory`。本讲承接 u3-l3 的 `Decimal`，并为消息格式化（message formatting）铺垫。
