本讲围绕 `icu_calendar` 补全了「日期本身是什么」这块拼图，拆为三个最小模块。其一，`Calendar` 是被密封的策略 trait，每个日历只负责（年,月,日）与中立中介量 Rata Die / ISO 的互转及若干查询；`Date<A>`（`A: AsCalendar`）把「该日历下的内部日期」与「日历对象」打包，所有用户 API（`try_new`、`month`、`weekday` 等）仅转发，泛型 `A` 而非写死具体日历，是为了允许用 `Rc`/`Arc` 共享非 `Copy` 的日历（如 `Japanese`）。其二，公历族（`Gregorian`/`Iso`/`Buddhist`/`Roc`）共享 `AbstractGregorian` 骨架、月日规则相同、`has_cheap_iso_conversion()` 恒真，差异仅在纪元与 extended year 换算；`to_calendar` 据此走「ISO 快速通道」或「Rata Die 通用通道」，佛历的 `EXTENDED_YEAR_OFFSET = -543` 即「佛历年 = ISO 年 + 543」之由来，BCE 用 `1 - year` 映射（无 0 年）。其三，`AnyCalendar` 以枚举分发实现运行时多态，`to_any()` 做类型擦除，`AnyCalendarKind`（对应 CLDR calendar id）经 `AnyCalendarKind::new(CalendarPreferences)` 解析 locale 的 `-u-ca-` 扩展并失败回退 `Gregorian`；代价是 `Date<AnyCalendar>` 跨变体不可全序（仅 `PartialOrd`）且更重。关键术语：Rata Die、extended year、`AbstractGregorian`/`GregorianYears`、`AnyCalendarKind`、ISO 快速通道。补充：`DateTime<A>` 位于 `icu::time`，仅为 `Date<A>` + `Time`，不带算术与全序。
