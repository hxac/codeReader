本讲讲解 `icu_locale` 里两个把 locale 翻译成可用信息的工具：`LocaleFallbacker` 与 `LocaleDirectionality`。回退（fallback）是 ICU4X 用有限 CLDR 数据覆盖无限 locale 组合的核心机制，从最具体的 locale 逐层退到根 `und`。`LocaleFallbacker` 实现 UTS #35 的 locale 继承，用法为 `new() → for_config() → fallback_for()` 后用手动的 `get()`/`step()` 循环遍历（迭代器不实现标准 `Iterator`，终止条件是 `is_unknown()`）。回退行为由优先级枚举 `LocaleFallbackPriority`（默认 `Language`，另有 `Script`、`Region`）决定先剥离哪个子标签；算法是数据驱动的，依赖 likely subtags 表（推默认 script）和 parents 表（显式父级如 `es-AR → es-419`）。两个阶段：归一化（创建时一次，剥掉冗余 script 如 `zh-Hans → zh`）和步进（每次剥一层）。关键细节：非默认 script 必须显式保留，如 `zh-TW → zh-Hant`、`sr-ME → sr-Latn`，否则会落到错误 script。`LocaleFallbackProvider` 适配器把回退算法自动嵌入数据加载循环。回退迭代器操作的是 `DataLocale`（`Copy` 的精简 locale）。`LocaleDirectionality` 判断 LTR/RTL 用于 UI 镜像布局，核心直觉是方向是 script（书写文字）的属性而非 language 的属性（如 `en-Arab` 是 RTL）：先借 `LocaleExpander::get_likely_script` 推出 likely script，再在已排序的 `ZeroVec<UnvalidatedScript>` LTR/RTL 列表上 O(log n) 二分查找。
