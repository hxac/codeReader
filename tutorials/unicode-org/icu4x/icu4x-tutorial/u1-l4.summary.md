本讲解析 `icu` 元 crate 及其 feature 与数据体系，包含三个最小模块。其一，元 crate 自身不含任何算法，仅以 `pub use` 把 13 个稳定组件（calendar、datetime、collator 等）重导出为统一命名空间下的模块，是编译期的名字重定向、零运行期开销；experimental、pattern 受 `unstable` feature 门控。其二，feature 体系多为转发开关：能力类（compiled_data、serde、datagen）透传给全部组件，行为类（sync、logging）仅穿透到 `icu_provider`，unstable 用 `dep:` 启用可选依赖。其三，数据分两种方式——compiled data（默认开启）在编译期内嵌数据，对应无 provider 参数的 `new`/`try_new`；显式 `DataProvider` 在运行期喂数据，对应 `try_new_with_buffer_provider`（需 serde）与 `try_new_unstable`。核心结论：关闭 compiled_data 后，无 provider 参数的构造函数会从 API 中消失。引入的关键术语：元 crate、转发开关、compiled data、DataProvider，以及 compiled_data/serde/sync/unstable 等 feature。
