本讲讲解 icu_datetime 的现代 API：DateTimeFormatter。它遵循 UTS#35 Semantic Skeletons 范式——先选字段集（field set，决定显示哪些字段），再配 options（决定显示风格）。DateTimeFormatter<FSet> 把字段集编进类型参数，另有固定日历的 FixedCalendarDateTimeFormatter 与无日历的 NoCalendarFormatter 两个瘦身版。字段集分两类：静态字段集（YMD/YMDT/T 等，由宏批量生成 long/medium/short 与 with_* 链式构造器）是类型，编译器可裁掉用不到的数据，体积最小；动态字段集（CompositeFieldSet 配 FieldSetBuilder）用于运行期才确定字段的场景，代价是链接更多 pattern 数据。Length（Short/Medium 默认/Long，#[non_exhaustive]）及 YearStyle/Alignment/TimePrecision 等选项按字段集能力选择性可用。构造期通过 compiled data 加载数据并选定日历与 pattern，返回 Result；format() 则返回惰性的 FormattedDateTime——它实现 Writeable（借此获得 Display），文本在写入时经 try_write_pattern_items 逐项写「字面量字符 + 字段」生成，并支持片段标注（parts，如 YEAR/MONTH/DAY）供富文本高亮。本讲贯穿的核心结论是「把字段需求编码进类型」，由此换来编译期裁体积与输入类型校验。
