本讲是动手讲义，目标是让读者手里有一个能 `cargo run` 的 ICU4X 小程序。它覆盖三个最小模块：① 工具链——区分 MSRV（根 `Cargo.toml` 的 `rust-version=1.88`，给使用者的编译下限承诺）与 `rust-toolchain.toml` 的 `channel=1.95`（贡献者开发用的默认工具链，rustup 自动切换）；② 引入依赖——`cargo new --bin` 后执行 `cargo add icu`，元 crate `icu` 把 `icu_datetime` 等组件 re-export 成 `icu::*` 模块，一行依赖即换回全套组件命名空间；③ 第一个日期格式化示例——最小数据流为 `locale!` 宏（编译期解析，无 Result）→ `DateTimeFormatter::try_new(locale, fieldset)`（运行期加载 compiled data，故返回 Result）→ `Date::try_new_iso` → `dtf.format(&date)` → `println!`。引入的关键术语：元 crate、MSRV、compiled data（默认编译进二进制的 CLDR 数据）、Locale、DateTimeFormatter、fieldset（如 `YMD`，决定显示哪些字段）、length（如 `long`，决定呈现长度）、Writeable（惰性格式化对象，打印时才生成文本）。核心结论：外部用户只需关心 MSRV；compiled data 默认覆盖大部分常用 locale，故常用 `.expect()` 断言成功；fieldset 与 length 共同决定输出形态，同一 fieldset 换不同 locale 会产出完全不同的本地化文本。
