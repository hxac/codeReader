本讲打开 `pcileech_fifo` 模块，确立其为整板卡的「路由器 + 控制中枢」。它以 5 条 interface（dcom、dcfg、dtlp、dpcie、dshadow2fifo）汇聚所有「主机↔PCIe」的数据与控制。接收方向用一组纯组合文本宏做 MAGIC 路由：64 位数据 `[7:0]==0x77` 验明帧身份，`[9:8]` 的 type 字段取 `00/01/10/11` 将其天然互斥地分流到 TLP、CFG、Loopback、Command 四路，其中 TLP/Loopback 取高 32 位（Loopback 另带 2 位 ctx），CFG/Command 保留完整 64 位。发送方向经 `pcileech_mux` 多路复用器按端口优先级把 7 路数据合并成「1 状态字 + 7 数据字 = 256 位」大包回送，端口 tag 与接收 type 完全对称，主机端可无损还原。fifo 还维护 ro（只读状态）/rw（读写控制）两份寄存器文件，命令经「字节地址→位地址」、按 mask 逐位写，或由 `f_shadowcfgspace` 转发至配置空间影子。rw 的控制位经 `_pcie_core_config` 翻译为 `dpcie.pcie_rst_core`、`cfgtlp_en`、DRP 握手等硬件动作，`rw[31]` 更可经 STARTUPE2.GSR 触发整片 FPGA 全局复位。
