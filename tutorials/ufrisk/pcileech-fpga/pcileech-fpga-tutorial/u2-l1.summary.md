本讲是 pcileech-fpga 顶层「接线层」的精读，核心是 SystemVerilog 的 **interface**（把一组总是一起连接的信号打包成「粗电缆」）与 **modport**（从某一模块视角规定信号 input/output 方向）。二者合起来构成模块间的「**契约**」：同一信号在两端 modport 中方向必然互补（一 input 一 output），插错插头会在综合期被报出。所有接口定义集中在前置头文件 `pcileech_header.svh`，顶层 `pcileech_squirrel_top.sv` 声明 5 个 interface 实例把 com、fifo、pcie_a7 三大子系统连起来。

逐个精读四个核心接口：**IfComToFifo**（com↔fifo，64 位下行/256 位上行 + 一个反压位，modport 为 `mp_com`/`mp_fifo`，最简入门样本）；**IfPCIeFifoTlp**（fifo↔PCIe TLP，用数组信号 `rx_*[4]` 表达 4 路并行接收，体现「控制统一、数据分散」，modport 为 `mp_fifo`/`mp_pcie`）；**IfAXIS128**（AXI-Stream 风格的 128 位数据流，`source`/`sink` 生产者—消费者握手，握手条件 `tvalid && tready`，另有省去反压的 `source_lite`/`sink_lite`，`tuser` 9 位同时编码首拍/末拍/命中 BAR）；**IfShadow2Fifo**（最宽的 16 信号契约，承载 fifo 对配置空间影子的读写命令与 `bar_en`/`cfgtlp_en` 等功能开关，modport 为 `fifo`/`shadow`，把「主机写寄存器 → PCIe 行为改变」的控制链打通）。

后续讲义 u2-l2（com）、u2-l3（fifo）将落到这些契约上。
