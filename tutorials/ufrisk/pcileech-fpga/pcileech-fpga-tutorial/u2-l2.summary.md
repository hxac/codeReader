本讲解析 PCIeSquirrel 工程中夹在 USB3 桥接芯片与系统路由中枢 fifo 之间的通信适配层，覆盖三个最小模块。其一 `pcileech_ft601` 是一个 13 状态的物理时序状态机，按 FT245 风格握手信号 `RXF_N/TXE_N/OE_N/RD_N/WR_N` 驱动 FT601，三条核心设计为：RX 优先于 TX（主机命令不被上行流量阻塞）、每次突发后两拍冷却、发送空闲超过 16 拍时填充同步字 `0x66665555`；收发均做字节倒序。其二 `pcileech_com` 下行通路把 32 位流拼成 64 位，靠「连续两个 `0x66665555`」做边界重同步，再用一个很浅的双时钟 FIFO 从 `clk_com` 域搬到 `clk` 域，静态前提为 `2*f_clk_com < f_clk`。其三 `initial_rx[5]` 上电注入机制：在上电第 16~20 拍由计数器 `tickcount64` 触发，com「伪造」5 条 64 位命令喂给 fifo，末条 `64'h00000003_80182377` 经 fifo 的 `CHECK_MAGIC(0x77)`/类型 11 识别为 CMD 写，向 `rw` 寄存器字节 0x18、掩码 `0x0300` 写 0，清掉 `rw[200]`（PCIE CORE RESET），解决「主机软件未启动、PCIe 核须先上线」的先有鸡先有蛋问题。关键术语：FT601、时钟域、字宽转换、`IfComToFifo` 契约、`0x66665555` 同步字、`initial_rx`、`rw` 寄存器、命令字段（MAGIC/TYPE/地址/掩码/值）。
