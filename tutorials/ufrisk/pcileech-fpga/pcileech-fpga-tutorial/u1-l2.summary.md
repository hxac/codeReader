仓库顶层「一设备一目录」：每目录是一个独立可构建的 Vivado 工程，目录名对应 readme 设备表；个别目录（如 CaptainDMA）为家族，内部按 FPGA 型号、封装、PCIe 通道数再分多个子工程。单工程含四类内容：src/ 放源码（.sv/.svh）与约束（.xdc）；ip/ 放 IP 配置（.xci）与初始化数据（.coe）；.tcl 脚本生成工程并构建比特流；另有说明文档。文件分两类：顶层 *_top.sv 与 *.xdc 设备特定，绑定物理引脚；com/fifo/mux/pcie/pcie_cfg/pcie_tlp 等同名文件为靠 interface 解耦、跨设备复用的公共 HDL。通信核心随连接方式而变（USB3→ft601、以太网→eth）。代际分界：较新设备带 tlps128_cfgspace_shadow 与 tlps128_bar_controller，较老设备仅有 pcie_ 前缀旧版配置空间影子、无 bar_controller，此差异决定能否做 BAR 设备仿真。主参考工程为 PCIeSquirrel。
