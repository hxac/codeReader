本讲承接构建流程与约束文件，下沉到 Xilinx IP 核与工程生成脚本内部。核心是「订单/货物分离」的设计哲学：`.xci` 是 IP 实例的 JSON 配置档案（参数订单），`.coe` 是给存储类 IP 的上电初始化数据，真正的可综合产物由 Vivado 在重生成时按订单现场造出。`PCIeSquirrel/ip/` 共 20 个 `.xci`：16 个 FIFO（按命名规律 `fifo_<写>_<读>_<时钟>_<用途>` 分为 `clk2` 双时钟 8 个、`clk1` 单时钟 6 个、默认 2 个）、2 个 BRAM（配置空间影子与 BAR 零页）、1 个分布式 ROM（写掩码）、1 个 PCIe 7x 硬核。3 个 `.coe` 服务前三个存储 IP。`vivado_generate_project.tcl` 五步拼装工程：`create_project` 锁定 `xc7a35tfgg484-2`、导入 11 个源文件、导入 20 个 `.xci` 与 3 个 `.coe`（设 `registered_with_manager`/`synth_checkpoint_mode`）、导入 `.xdc`、`upgrade_ip` 后建 `synth_1`/`impl_1` 两个 run 并设 `bin_file=1`。关键结论是仓库只存订单与数据、坚决不含专有 IP 产物，使项目既遵守 Xilinx 授权、又让任何用户用免费 WebPACK 合法重生成完整比特流。
