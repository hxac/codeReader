本讲以 `pcileech_squirrel_top.sv`（Squirrel 板卡，Artix-7 XC7A35T x1）为对象，讲清「主板」级的系统大图。顶层对外暴露四组物理引脚：系统时钟 `clk` 与通信时钟 `ft601_clk`（两个独立时钟域）、LED/按键、PCIe 金手指（差分对 `pcie_tx/rx/clk_p/n`、`pcie_perst_n`、`pcie_present`）、FT601 USB3 桥（32 位 `inout` 数据线）。全局复位 `rst` 由一个 64 位自由计数器 `tickcount64` 驱动：上电前 64 拍复位 + 按键 SW2 触发复位，长按 SW2 超 5 秒（阈值 500000000，绑定 100MHz）触发 PCIe 配置重载。

核心是三大子系统例化：`com`（USB3 通信）、`fifo`（路由与控制中枢）、`pcie_a7`（PCIe IP 封装）。数据流为 com→fifo→pcie（下行）与反向返回，fifo 永远居中调度。三者通过 5 个 interface 实例相连：`dcom_fifo`（com↔fifo，64 位下行/256 位上行）、`dcfg`（配置读写）、`dtlp`（TLP 收发，4 路并行）、`dpcie`（核复位+DRP）、`dshadow2fifo`（配置空间影子/BAR 控制），后四条均连 fifo↔pcie。每个 interface 用成对 modport（如 `mp_com`/`mp_fifo`、`mp_fifo`/`mp_pcie`、`fifo`/`shadow`）保证方向一致。`led_pcie`/`led_com` 由 pcie 与 com 产生，经 OBUF 驱动到 LD1/LD2。这些术语为后续 u2 单元（interface 语法、com、fifo 内部）提供承接。
