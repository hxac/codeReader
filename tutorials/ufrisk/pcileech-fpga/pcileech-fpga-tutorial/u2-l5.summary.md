本讲打开 `pcileech_fifo` 的控制面，揭示「主机软件→硬件行为」的唯一控制链路。核心是两张寄存器表：只读的 `ro`（320 位，硬件状态镜像，含 MAGIC、UPTIME、DRP 读回、PCIe 复位/在位电平等）与可读写的 `rw`（240 位，主机控制面板，上电由 task 初始化）。字段布局与 PCILeech 主机驱动约定的「控制寄存器映射」一致。`rw[207:200]` 这 8 个核心控制位经 `_pcie_core_config <= rw[207:128]` 整段搬运，翻译成 `dpcie.pcie_rst_core`、`dshadow2fifo.cfgtlp_en`/`bar_en` 等真实输出。读写走 Command 路（MAGIC type=11）的「位寻址+16 位窗口+逐位掩码」协议：64 位命令包中地址字段兼作 `f_rw`/`f_shadowcfgspace` 路由标志，低 15 位左移 3 位得 bit 地址，写要求 `f_rw=1`。DRP 读写为 one-shot 触发位，两段式握手防丢包；`rw[31]` 经 STARTUPE2 触发整片 FPGA 全局复位。
