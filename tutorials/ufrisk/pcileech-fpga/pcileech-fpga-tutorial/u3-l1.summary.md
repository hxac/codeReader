`pcileech_pcie_a7` 是 PCIe 硬核 IP `pcie_7x_0` 与工程其余部分之间的适配/封装层：对外只暴露 4 个 interface（`dfifo_cfg`/`dfifo_tlp`/`dfifo_pcie`/`dshadow2fifo`）加物理引脚，对内把 IP 上百个端口分发给 cfg、tlp 子模块并完成位宽转换。复位分两条线：`rst_subsys`（软复位，只刷 IP 之外的 cfg/tlp/位宽转换逻辑，不动链路）与 `rst_pcie`（硬复位，连根拔起 PCIe 硬核、链路断开重训）；二者分别由 `rst`、IP 的 `user_reset_out`、物理 `pcie_perst_n` 及主机命令 `pcie_rst_subsys`/`pcie_rst_core` 触发。100MHz 差分参考时钟经 `IBUFDS_GTE2` 原语转成单端 `pcie_clk_c`，同时驱动 LED 慢闪。IP 端口归纳为 PHY、s_axis_tx、m_axis_rx、cfg_mgmt、cfg 控制/状态、pl_* 物理层状态、DRP 七组，外加 `user_clk_out`（即整个 PCIe 域的 `clk_pcie`）、`user_lnk_up` 用户接口。接收数据流为：IP `m_axis_rx`(64b) → `tlp_rx` → `pcileech_tlps128_src64`(64→128，统计 DWORD 并抽出 7 位 BAR 命中 `tuser[8:2]`) → `tlps_rx`(128b) → `pcileech_pcie_tlp_a7`；发送方向对称地经 `pcileech_tlps128_dst64`(128→64，用 `d1_*` 暂存后半段) 回到 IP。关键术语 `ctx`(`IfPCIeSignals`)、`IfAXIS128`、`tlps_static` 供后续讲义承接。
