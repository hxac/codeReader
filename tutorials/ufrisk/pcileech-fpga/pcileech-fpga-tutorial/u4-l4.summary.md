本讲是「配置空间影子与 BAR 设备仿真」单元收尾篇，把 PCIe 设备身份定制汇总为四条不等价路径。设备身份由配置空间字段构成：VID/DID（偏移 0x00）、Subsystem VID/DID（0x2C）、Class Code（0x09–0x0B，决定 `lspci` 设备分类）、Revision（0x08）、DSN（扩展配置空间 64 位指纹）；需区分 BDF（主机枚举动态分配的位号）与上述「我是谁」字段。四条路径：①Vivado PCIe 核 `pcie_7x_0` GUI 改 IDs/Class/BAR0，写入 `.xci`，唯一需重生 IP，最权威最贵；②改 `pcileech_pcie_cfg_a7.sv` 中 `rw[127:64]` 经 `assign ctx.cfg_dsn` 驱动硬核输入端口，一行 HDL 即换 DSN，最廉价且无需重生 IP；③改 `pcileech_cfgspace.coe` 配合 `cfgtlp_zero`（即 `rw[203]`，默认 1 读回全零不可见，改 0 才显现影子 BRAM 内容，`lspci -xxxx` 可见）；④`pcileech_fifo.sv` 中 `rw[128:199]` 的 VID/DID/Subsys/Rev 字段标注 NOT IMPLEMENTED，不接线、是死字段，改了无效——这是「改了没反应」陷阱。影子受硬核 in-part override 限制，且 PCIe 侧默认只读（`cfgtlp_wren`=`rw[206]`=0），在线改只能走 USB→`IfShadow2Fifo`。核心结论：改 ID 不等于隐身，检测软件还看能力链、BAR 行为、DSN、链路状态等次级特征；本讲用于防御研究与取证。
