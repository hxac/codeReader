本讲以 PCIeSquirrel 为例，讲清 pcileech-fpga 从源码到可烧录比特流的工程流程。核心是两个 Tcl 脚本：先运行 `vivado_generate_project.tcl` 现场生成 Vivado 工程（仓库不含现成 `.xpr`），它负责导入源码、IP（`.xci`/`.coe`）与 `.xdc` 约束，用 `upgrade_ip` 升级旧版 IP，并创建综合 run `synth_1` 与实现 run `impl_1`，设 `bin_file=1`；再运行 `vivado_build.tcl`，以 `launch_runs` 与 `wait_on_run` 配对先后启动这两个 run，由 `impl_1` 的 `write_bitstream` 产出 `pcileech_squirrel.bin`。关键术语：综合把 SystemVerilog 翻译为逻辑网表，实现完成布局布线并写出比特流，二者是父子 run。构建与烧录相互独立：PCIeSquirrel 用 OpenOCD、CaptainDMA 家族用 CH347，固件须与设备型号/FPGA 工程对应。Windows 下路径过长（约 260 字符上限）会致构建失败，宜放短路径。
