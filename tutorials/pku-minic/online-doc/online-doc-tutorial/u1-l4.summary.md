本讲义聚焦 Docsify 站点入口与导航机制，覆盖三个最小模块。其一是 `index.html`：作为单页应用（SPA）唯一的 HTML 外壳，它只做三件事——预留挂载点 `<div id="app">`、声明全局配置对象 `window.$docsify`、按固定顺序加载核心库与插件脚本；关键结论是配置必须写在核心库加载之前，否则核心读不到配置。其二是 `toc.md`：通过 `loadSidebar: 'toc.md'` 被当作整站侧边栏，它本质是一份普通 Markdown 列表，缩进即层级，链接走 Docsify 路由，而 `subMaxLevel: 3` 会把正文 H1~H3 自动追加为锚点。其三是 `footer.md`：页脚采用「配置 + 插件」两段式——`loadFooter` 仅指名文件，真正渲染由第三方 `docsify-footer` 插件完成，二者缺一不可，这与核心自带的侧边栏机制形成关键区别。后续讲义可承接 `window.$docsify` 其余配置、插件扩展及路由校验。
