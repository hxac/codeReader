本讲聚焦检查器的「输入侧」——把文档里的链接一条条提取出来。由 `extract_links` 按文件扩展名分发到两条路径：Markdown 路径用 `markdown-it-py`（开启 `html`+`linkify`）将文档切成 token 流，从 `link_open` token 取 `href`、从 `image` token 取 `src`，原始 HTML 片段（`html_block`/`html_inline`）转交 HTML 提取器；HTML 路径用 `selectolax` 按「标签→属性」表（`HTML_LINK_ATTRS`→`HTML_LINK_SELECTOR`）批量取节点属性并拆分 `srcset`，因其不保留源偏移，靠单调递增的游标回原文重定位（`find_html_node`/`find_html_value`）来反推行列号。裸 URL 兜底（`BARE_URL_RE`+`trim_bare_url`）只在 `.html` 文件运行，逐行扫描并修剪被误吞的尾部标点与不成对括号；Markdown 中的裸 URL 已由 `linkify` 自动识别，无需此兜底。每条 `Link` 携带行列号，去重以 `(来源文件, 行, URL)` 为键，URL 在构造时经 `html.unescape(strip())` 归一化。本讲产出带行列号的 `Link` 列表，是 u4-l3（本地/Docsify 路由校验）与 u4-l4（远程校验）的前置输入。
