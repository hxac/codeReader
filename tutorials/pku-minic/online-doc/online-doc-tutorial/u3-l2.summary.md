本讲定义了本仓库在标准 Markdown 之外统一使用的四类 Docsify 扩展写法，作为「写内容时」的写作规范。第一是提示框：行首 `?>` 渲染为轻量提示框（补充说明、建议、TODO 占位），`!>` 渲染为重警告框（重要事项、禁止行为、学术诚信）；两者只决定样式，语气轻重由作者自选，且框内正文仍按 Markdown 解析。第二是代码围栏高亮：围栏的 info string（如 `c`、`koopa`、`ebnf`）是「高亮开关」，只有 `index.html` 加载了对应 Prism 组件（c/cpp/ebnf/bash/makefile/bison/rust 及本地 koopa）的语言才会被 Prism 着色，`asm`、`txt` 等无组件的语言仅以纯文本显示。第三是 KaTeX 公式：统一用行内 `$...$`，由 `docsify-katex.js` 插件配合 `katex.min.css` 渲染，仓库从不用块级 `$$...$$`，并应避免在围栏外裸写 `$$`（Bison 动作中的 `$$` 需放进围栏）。第四是 Docsify 路由链接：站内链接写浏览器路由而非相对文件路径，有三种形态——`/dir/` 指向目录 README、`/dir/page` 指向 page.md、`/dir/page?id=锚点` 跳到标题锚点（锚点可直用中文标题），与 `toc.md` 侧边栏写法保持一致。核心结论：围栏语言、KaTeX、路由的渲染能力分别由 index.html 的 Prism 组件、docsify-katex 插件、docsify 路由机制决定。
