本讲引入 Docsify「主动」扩展——插件(plugin)：签名为 `(hook, vm) => {}` 的普通函数，经 `$docsify.plugins.push(fn)` 注册，Docsify 在渲染各环节调用其生命周期钩子。钩子按用途分：改内容用 `beforeEach`/`afterEach`，操作 DOM 用 `doneEach`，只跑一次用 `ready`/`mounted`。因 Docsify 是单页应用，每次切章节正文都会重新渲染，依赖当前 DOM 的逻辑必须放进每次触发的钩子。两个本地插件是核心样本：`sidebar.js` 仅用 `doneEach`，定位侧边栏当前激活项（先查 `.active`，查不到则用 `location.hash` 反查兜底补 `active`），若它在窗口下方之外则 `scrollIntoView()` 滚入可视区；`giscus.js` 用 `ready`+`doneEach`，`ready` 把 Giscus 的 `.giscus` 容器搬进 `.content` 并初始化主题，`doneEach` 用 `body` 的 `data-page` 改写 iframe `src` 的 `term=` 实现每页独立评论，并给主题按钮绑定点击事件同步 `theme=`。两处改写均靠 `String.replace` 重载 iframe 生效；插件须在 Docsify 首次异步渲染触发钩子前完成 `push`，靠同步 `<script>` 保证时序。
