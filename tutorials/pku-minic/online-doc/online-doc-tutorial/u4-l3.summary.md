本讲精读链接检查器 check_links.py 的「本地侧」校验，回答站内 Link 指向的页面/锚点是否存在。docsify_candidates 是「路由→文件」翻译器，按 path 四种形态（空 / 目录 / 带后缀 / 无后缀）返回候选文件集与锚点（取自 #fragment 或 ?id=）；绝对路由挂 docs_root、相对路由挂源文件目录，normpath 防 `../` 绕过。四个 *_candidates 辅助函数用 with_suffix（页面）与 /README（目录）两套约定对 .md/.html 展开候选：无后缀路由试 page+index 四件套，目录路由只试 index，体现 README/index 回退。锚点校验采用「超集匹配」：anchor_forms 造 NFC/小写/空格转连字符/去标点四变体，expand_anchor_forms 再扩为编码/解码维度，链接锚点与目标标题两端集合相交即通过。check_local_link 总装：取首个存在的候选为 target，无 target 报 local-missing-target（带 tried 审计），文件在但锚点不相交报 local-missing-anchor。
