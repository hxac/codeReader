`online-doc` 是北京大学编译原理课程实践的在线文档站（基于 Docsify），面向选课学生，目标是引导学生从零写出一个编译器。本讲覆盖三个最小模块并给出对应核心结论：①项目背景——仓库交付的是「教人写编译器的文档」而非编译器本身，编译器由学生实现、不在仓库内，唯一的程序代码是文档工具 `scripts/check_links.py`；②文档站与编译器本体的区分——仓库里没有 `.c/.cpp/.rs` 源码，根目录 `README.md` 是给 GitHub 访客看的「门牌」，`docs/README.md` 才是站点首页；③编译流水线概览，即 SysY→Koopa IR→RISC-V。引入的关键术语：**SysY**（源语言，精简版 C）、**Koopa IR**（课程自设计的中间表示，形似 LLVM IR 但简化）、**RISC-V**（目标汇编，读作 risk-five）。流水线拆成前端、后端两步，再细分为 **Lv0–Lv9 十个增量阶段**（外加可选 Lv9+），每个 Lv 给编译器递增一项能力。
