本讲承接 u1-l1，解决「如何在本地启动文档站」。核心结论：本地预览只需两条命令——`npm i docsify-cli -g` 全局安装 CLI、`docsify serve docs` 启动，二者均出自根目录 README。

讲义拆为三个最小模块：①docsify-cli 安装；②`docsify serve` 启动；③Python 依赖一览。

引入的关键区分：**docsify-cli ≠ docsify**。docsify-cli 是开发期本地静态文件服务器，提供 `docsify` 命令、默认监听 3000 端口；docsify 才是真正把 Markdown 渲染成网页的库，运行在浏览器端，由 `docs/index.html` 通过 CDN 的 `<script>` 标签加载，无需安装。站点是单页应用（SPA），浏览器只加载 `index.html` 外壳，切章由前端 JS 按需抓取 `.md` 渲染。

核心要点：`docsify serve docs` 的 `docs` 参数不可省——它把 `docs/` 设为服务器根目录，使 `index.html` 中的相对路径配置（挂载点 `id="app"`、`loadSidebar: 'toc.md'`、`loadFooter: 'footer.md'`，以及统一配置对象 `window.$docsify`）正确解析。

第三个模块澄清：浏览文档无需 Python；`requirements.txt` 三个包（httpx、markdown-it-py[linkify]、selectolax）仅服务于链接检查器 `scripts/check_links.py`，分别负责发请求校验远程链接、解析 Markdown 抽取链接、解析 HTML 片段，为第四单元精读该脚本备齐环境。
