本讲教如何从零实现自定义 `fz_device`。核心结论：自定义 device 是 MuPDF 官方一等扩展点，无需注册或改框架，只需三步——定义派生结构体（以 `fz_device super` 作首成员，借「结构体首成员多态」实现 C 手写继承）、实现关心的回调（签名严格照抄 `struct fz_device`）、用 `fz_new_derived_device` 分配清零并填虚表。`fz_calloc` 保证未赋值槽为 NULL，包装函数判空后自动跳过，故只填所需回调即可。关键判据：构造时若 `fz_keep_*` 持有对象则须实现 `drop_device` 配对 `fz_drop_*`（如 list device 借引用录制显示列表），纯借用或只统计则可省略（如 trace device 只借用 `fz_output`）；`close_device` 仅在有缓冲需冲刷时才需要。回调内允许抛异常，框架包装函数会捕获并 `fz_disable_device` 清空整张虚表，使 device「出错即报废」后再 rethrow；容器的 scissor 栈由包装层自动维护。驱动方式与普通 device 一致：`fz_run_page`/`fz_run_display_list` 后 `fz_close_device` + `fz_drop_device`。综合实践示范了一个统计 device（累计 `fill_path` 次数与 `fill_text` 字形数），为后续讲义承接 document writer 的 `begin_page` 返回 device 这一关联埋下伏笔。
