本讲讲解如何用 GNU Make 编译 MuPDF，覆盖构建系统的四个协作文件与裁剪编译机制。

构建逻辑由四个互相 include 的 Make 文件组成：顶层 `Makefile` 负责目标、输出目录与编译命令宏；`Makerules` 是配置中心，翻译编译模式与功能开关并探测系统库；`Makethird` 决定第三方库的集成策略；`Makelists` 纯粹是第三方源码清单。裸敲 `make` 走默认目标 `apps libs`，产物落在 `build/release/`；缺图形依赖时 `make tools` 是最小可用路径，产出 `libmupdf.a`、`libmupdf-third.a` 与 `mutool`。

关键概念包括：输出目录 `OUT` 是由 `build` 模式及 `build_prefix/suffix` 组合而成；编译模式 `build=debug/release/sanitize/memento` 用于不同调试需求，其中 `sanitize`、`memento` 是排查内存问题的首选。功能裁剪是**编译期**的——`html=no`、`mujs=no`、`xps=no` 等开关在 `Makerules` 里被翻译成 `-DFZ_ENABLE_*=0` 宏，从源头排除对应格式 handler 及其第三方库。第三方库遵循统一的「系统版 vs 内置子模块」二选一：`USE_SYSTEM_LIBS=yes` 切系统库，否则把各库源码编进 `libmupdf-third.a`。十几个第三方库以 git 子模块形式置于 `thirdparty/`，克隆后必须 `git submodule update --init`；freetype/libjpeg/zlib/openjpeg/jbig2dec/lcms2 为必装，tesseract/zxing/extract 等可选。
