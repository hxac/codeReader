本讲以 `muconvert.c`（即 `mutool convert`）为例，剖析「读入任意格式 → 逐页转换 → 写出目标格式」的最小转换管线，分三个模块展开。其一，转换主循环：`muconvert_main` 在循环外用 `fz_new_document_writer` 只创建一次写入器，循环内对每文档依次 `fz_open_document` → 鉴权 → `fz_style_document` → `fz_layout_document` → 数页，再由 `runrange`/`runpage` 逐页执行 `fz_begin_page` → `fz_run_page` → `fz_end_page`（含页框选择与原点平移对齐），最后 `fz_close_document_writer` 冲刷 trailer、`fz_drop` 逆序释放。其二，跨格式通用性：全程只用 `fz_document`/`fz_page`/`fz_device`/`fz_document_writer` 四个格式无关抽象，源由 reader 自动识别、目标由 writer 按扩展名分发，device 充当「通用翻译器」；回流源须先排版才能数页。其三，选项传递：`-O` 为逗号分隔 key[=value] 串，经 `fz_new_document_writer` 路由到具体构造函数，由该格式的 `*_parse_write_options`（`fz_new_options` + `fz_lookup_option_*`）解析成专属结构体，并以 `fz_throw_on_unused_options` 防拼写静默失效。结论：转换 = 通用 reader 循环 × 可替换 writer × 可传递选项字符串；muconvert 是最干净的 document-writer 范式，mudraw 则走更底层的 band-writer 路径。关键术语：`fz_document_writer`、`begin_page`/`run_page`/`end_page`、`fz_device` 充当中间语言、`pdf_write_options`、`fz_options`、`fz_write_document` 便捷函数。
