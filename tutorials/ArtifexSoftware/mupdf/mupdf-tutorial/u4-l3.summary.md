本讲补上 MuPDF 渲染的最后一块：位图后端 draw device 与像素容器 fz_pixmap。draw device 是 fz_device 的派生实现，接到 fill_path/fill_image 等指令后用 rasterizer 扫描线填充，把像素写进调用方提供的 fz_pixmap；它不拥有也不清空 pixmap，内部维护 fz_draw_state 状态栈，每层记录当前画布 dest、裁剪框 scissor 与混合模式，裁剪/蒙版/透明组会 push/pop。fz_pixmap 是行优先的连续字节内存：每像素字节数 n = colorspace_n + 专色 s + alpha；stride 是一行字节数（默认 n*w，可带 padding 或为负）；像素 (x,y) 首字节位于 samples + y*stride + x*n，分量顺序固定为「过程色→专色→alpha」。颜色空间 Gray=1/RGB=3/CMYK=4 决定 n，进而决定 stride 与总内存，渲染时自动把内容颜色转换到目标空间。设备变换 dev->transform 与每次调用的 in_ctm 被 fz_concat 相乘，缩放矩阵放 device 还是 run_page 都等价。标准用法四步：清屏 → fz_new_draw_device → fz_run_page → close 后 drop，pixmap 由调用方自行 keep/drop。
