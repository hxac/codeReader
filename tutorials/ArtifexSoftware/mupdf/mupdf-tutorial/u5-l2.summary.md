本讲讲解 MuPDF 的结构化文本（stext）机制——它是文本提取、全文搜索、格式转换的共同底座。核心是 `fz_stext_device`：作为 `fz_device` 的派生实现，它拦截文本绘制指令，用几何启发式把零散字形拼成「段落→行→字」的带坐标结构。PDF 内容流本无「一行字」概念，行/段/空格全由 `fz_add_stext_char_imp` 实时判定——它把字形位移分解为沿基线的 `spacing` 与垂直基线的 `base_offset`，按 0.8、1.5 两阈值区分续行/换行/换段，字间距大时主动补空格。抽取遵循五步流程：`fz_new_stext_page` → `fz_new_stext_device` → `fz_run_page` → `fz_close_device` → 导出；`close` 须补 bbox、做 bidi 重排，不可跳过。产物 `fz_stext_page` 是 `fz_pool` 池式分配的 block 树，节点分 TEXT/IMAGE/STRUCT/VECTOR/GRID 五类，一次分配一次释放。text/html/xhtml/xml/json 五种导出只是同一棵树的不同 walker：HTML 绝对定位还原版面，XHTML 按字号推断标题重排为流式语义，XML 带 quad 供精确分析。抽取顺序取自字形绘制顺序，不保证与视觉阅读顺序一致；释放须先 `drop_run` 归还池外 font/image 引用再 drop 池。`fz_stext_char.quad` 是后续搜索命中与选区高亮的几何依据，导出经 `fz_text_writer` 封装为 document writer 后端。
