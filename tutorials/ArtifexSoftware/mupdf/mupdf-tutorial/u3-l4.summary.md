本讲在 `fz_document` 的「虚表 + 判空转发」模型之上，讲解加密文档的鉴权与元数据读取，分为三个最小模块。第一，密码鉴权是两步流程：`fz_open_document` 只读文档骨架（PDF 的 xref 与 Encrypt 字典）并不验密码；`fz_needs_password` 把关「是否需要密码」（PDF 层判定为未加密或空密码可解即返回 0），需要时再用 `fz_authenticatePassword` 验候选密码。第二，鉴权返回值不是布尔而是位掩码：bit0(1)=无需密码、bit1(2)=用户密码通过、bit2(4)=所有者密码通过，故 `if(!ret)` 当布尔用恰好等价于失败（0）。PDF 层按规范对候选密码做哈希并与 `U`/`O` 值比对得到 `access`。权限通过 `fz_has_permission` 配合 `fz_permission` 枚举查询，以所有者身份（access&4）一律放行，否则按 PDF 权限字 `P` 与 `PDF_PERM_*` 位掩码按位与判定。第三，`fz_lookup_metadata` 按键读取格式无关的元数据：返回含 `\0` 的总长度、找不到返回 -1、进入前预清空缓冲区，判定成功用 `ret>0`；键分 `format`/`encryption` 与 `info:*`（取 trailer 的 Info 字典）两类。鉴权失败默认不抛异常而靠返回值表达，是贯穿本讲的重点。通用层四函数均为判空转发，真实算法在 `pdf-crypt.c` 与 `pdf-xref.c`。
