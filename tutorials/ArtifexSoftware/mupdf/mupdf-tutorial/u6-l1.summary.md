本讲是手册从「读文档」转向「写文档」的转折点，围绕 `fz_document_writer` 导出抽象讲解三个最小模块。核心结论：document writer 是「反向 device」——它在 `fz_device` 之上再加一层「多页 + 文档框架」，其 `begin_page` 回调返回一个 `fz_device *`，使 device 接口在读写两方向被完全复用，这正是 MuPDF 能用一份代码既渲染又导出的根本原因。引入的关键术语包括：document writer 虚表四回调（begin_page/end_page/close_writer/drop_writer）、基类的 `dev` 借出字段、严格生命周期状态机（new → 反复 begin/run/end → close → drop）、dev 归 writer 托管（调用方绝不能自 drop）。派生手法是 C 手写继承：派生结构体以 `super` 起头，用 `fz_new_derived_document_writer` 填虚表，SVG writer 包 svg device、text writer 包 stext device。分发中枢 `fz_new_document_writer` 按「扩展名 → 构造函数」查表，显式 format 优先于 path 推断，PDF/OCR/ODT/DOCX 受 `FZ_ENABLE_*` 宏守卫可裁剪。便捷函数 `fz_write_document` 可一行导出整篇。
