本讲讲解 MuPDF 的多线程渲染，覆盖三个最小模块。其一为 clone_context 与锁：多线程下每个工作线程必须 `fz_clone_context` 出独立 context，而创建 context 时须注入真锁（默认是空函数，clone 会被拒）。其二是主/工作线程分工：`fz_document`/`fz_page` 只能主线程串行访问，自包含不可变的 `fz_display_list` 是跨线程传递页面内容的唯一安全桥梁，工作线程回放 list 产出 `fz_pixmap`。其三是共享 store 设计：一族 clone 通过引用计数共享 store/font/glyph_cache，靠 `FZ_LOCK_ALLOC` 串行化，但各自维护独立异常栈（因 `longjmp` 不可跨线程）。引入的关键术语包括 context 家族、`master`/`context_count`、锁偏序规则（持有锁 i 时只能申请锁 j，i<j，杜绝死锁）、三把锁 `FZ_LOCK_ALLOC`/`FREETYPE`/`GLYPHCACHE`。核心结论：异常是每线程一份、不可跨线程 `fz_rethrow`，须用共享标志回传失败；生产级 mudraw 以常驻 worker 池 + 信号量 + 分带把「每页一线程」升级为工业版。
