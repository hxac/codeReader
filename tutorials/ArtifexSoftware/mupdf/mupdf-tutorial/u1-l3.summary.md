本讲把 u1-l1 的双层架构对应到真实目录，建立「项目地图」。三个最小模块：顶层目录按关注点划分——`include/` 是 API 契约，`source/` 是实现，`platform/` 是 viewer 与语言绑定，`thirdparty/` 是依赖子模块。头文件分两层：通用层 `include/mupdf/fitz/`（约 50 个，定义 `fz_context`、`fz_document`、`fz_page`、`fz_device`、`fz_pixmap`、`fz_stream` 等格式无关抽象）与 PDF 专用层 `include/mupdf/pdf/`（19 个，含 `pdf_obj`、xref、crypt）。引入关键术语「伞形头」：`fitz.h` 按 I/O、资源、设备、文档、输出分组；`pdf.h` 先 include `fitz.h` 再叠加 PDF 头，固化「专用层单向依赖通用层」。`source/` 子目录与格式一一对应（fitz/pdf/xps/html/cbz/svg/reflow 及 tools、helpers）；`platform/` 提供 gl/x11/win32/wasm/java 五类形态；`thirdparty/` 以 18 个子模块（freetype、harfbuzz、zlib 等）支撑字体、图像、压缩等底层能力。定位口诀：找能力看 `include/`、找实现看 `source/`、找平台看 `platform/`、找依赖看 `thirdparty/`。
