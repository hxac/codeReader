本讲承接 u5-l2 的结构化文本树，回答「如何在 stext 上定位关键词的页面位置」，覆盖三个最小模块。其一是命中四边形 `fz_quad`：用 `ul/ur/ll/lr` 四角点描述任意四边形，区别于轴对齐的 `fz_rect`，以贴合旋转、斜体、竖排文字；命中字符的 `quad` 在 `add_quad` 中按基线方向合并，故一次命中可产生多个四边形。其二是单页接口 `fz_search_stext_page`：硬编码大小写不敏感，返回值为四边形数而非命中数，需对 `hit_mark[]` 去重才得真实命中处数；便捷封装 `fz_search_page_number` 还默认开启连字符换行处理，正则/大小写敏感须改用 `fz_match_stext_page`。其三是流式搜索 API（`fz_new_search`/`fz_feed_search`/`fz_search_forwards|backwards`/`fz_drop_search`），它是协作式状态机，搜索器按需索页并维护三页滑动窗口拼成连续草垛，从而命中跨页关键词，喂页时传入的 `seq` 原样回填到命中的 `fz_search_quad.seq`。核心结论：搜索等于把 stext 拍扁为草垛、匹配后再把字节偏移反查回字符取其 `quad`；搜索前对关键词与文本做统一 Unicode 规范化以保证等价匹配；`mugrep`（`mutool grep`）是流式 API 的标准范例，默认将关键词当作正则，字面搜索须加 `-F`。
