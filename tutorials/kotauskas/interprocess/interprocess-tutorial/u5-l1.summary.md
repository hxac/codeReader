本讲介绍 interprocess 最贴近 OS 原语的 unnamed_pipe（匿名管道）模块。核心 API 仅 pipe()，返回 (Sender, Recver)，写端在前，能力分别经标准库 Write/Read trait 暴露。沿用 u2-l3 的「壳/芯」分层：公共 Sender/Recver 是 pub(crate) 字段的 newtype 壳，impmod! 把后端 SenderImpl/RecverImpl/pipe_impl 以统一别名注入公共层，系统调用全部下沉到芯——Unix 走 libc::pipe2 与 FdOps 的 read/write，Windows 走 CreatePipe 经 CreationOptions 构建器，且后端直接返回公共壳类型。multimacro! 借 forward_sync_read/write、forward_handle、forward_debug、derive_raw 等宏批量缝 trait，使 newtype 本体零手写。关键设计是句柄/FD 默认可继承，这是匿名管道适合与子进程通信的根基，并支撑 From<Sender> for OwnedHandle 等句柄转换。平台差异：Windows 写端有 limbo（drop 时后台刷新缓冲），Unix 有 UnnamedPipeExt::set_nonblocking；写端全部关闭后 read 返回 Ok(0) 即 EOF。
