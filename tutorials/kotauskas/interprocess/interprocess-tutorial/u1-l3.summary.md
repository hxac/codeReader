本讲承接项目概览，讲解 interprocess 的构建、Feature Gate 与示例运行。核心结论：async、tokio、doc_cfg 三个 feature 默认全关，其中 tokio 蕴含 async（依赖链 tokio ⇒ async ⇒ futures-core），doc_cfg 仅影响文档构建；依赖按 cfg(windows)/cfg(unix) 互斥选取（Windows 用 windows-sys，Unix 用 libc），[dev-dependencies] 仅对示例/测试生效、不传染下游；因 autoexamples 关闭，示例须用 [[example]] 显式声明，运行用示例名而非文件名；lint 分两套——Cargo.toml 的 [lints] 全 crate 生效，src/lib.rs 的 #![warn] 仅库本体生效。引入的关键术语：MSRV（1.75）、edition 2021、feature gate、dep: 前缀、双 main 门控（未启用 tokio 时给出可编译占位程序）、半双工收发顺序（同步示例刻意一端先发一端先收以规避单线程收发死锁，异步版改用 try_join! 并发收发，从而不必精心安排收发次序）。本讲只把 local socket 当作可运行示例使用，其名称系统与 API 内部实现留待后续单元展开。
