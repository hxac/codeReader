本讲聚焦贯穿全库却常被忽略的基础设施——句柄（Handle）/文件描述符（fd）的所有权与克隆。核心模块有四：一是 `TryClone`（src/try_clone.rs），interprocess 的跨平台可失败克隆抽象，因 `DuplicateHandle`/`dup` 可能失败而返回 `io::Result<Self>`，并对所有 `Clone` 类型提供 blanket impl；newtype 经 `forward_try_clone!`、枚举经 `dispatch!` 转发，最终落到平台 `c_wrappers`。二是所有权形态：`OwnedHandle`/`OwnedFd`（拥有、drop 时关闭）对 `BorrowedHandle`/`BorrowedFd`（借用、不关闭），由 `forward_handle` 宏提供安全转换。三是 Windows 专有 `ShareHandle`：因句柄进程局部，非父子进程无法直接传数值，须经 `DuplicateHandle` 把句柄复制进目标进程表、返回对端才有效的裸值。四是 Windows 的 `AdvOwnedHandle`：用低比特标记 + `NonZeroUsize` 零 niche，在不增内存前提下为句柄叠加布尔状态，由 `Drop`/`as_raw_handle`/`TryClone` 三处保证正确关闭、干净借出、克隆时保留标记。贯穿全库的关键约定：克隆或共享出的副本一律不可继承（Unix 设 `FD_CLOEXEC`、Windows 传 `bInheritHandle=0`），与原始管道句柄「默认可继承」恰相反。
