interprocess 是用 Rust 编写的进程间通信（IPC）工具包，核心设计目标是平衡「暴露平台原生特性」与「跨平台统一接口」。它提供四种原语：跨平台的 local socket（Windows 上由 named pipe 实现、Unix 上由 Unix domain socket 实现）、两端皆有的 unnamed pipe、Unix 专有的 FIFO 文件，以及 Windows 专有的 named pipe。关键术语：进程、IPC、local socket（interprocess 自造的抽象，并非操作系统原语）；最大的易错点是「named pipe」一词在两个平台上指代完全不同，故 interprocess 把 Unix 版本特称为 FIFO 文件。源码组织上，跨平台公共模块（local_socket、unnamed_pipe、error、bound_util）在 lib.rs 直接声明，平台私有后端置于 os::unix / os::windows，用 #[cfg] 互斥做编译期门控。三个 feature 中 tokio 隐含 async、默认关闭，doc_cfg 仅影响文档展示。当前版本 2.4.2，MSRV 1.75。
