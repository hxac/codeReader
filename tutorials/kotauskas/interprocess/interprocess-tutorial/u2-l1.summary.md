本讲围绕三个最小模块展开：`local_socket`（抽象本身）、`local_socket::traits`（接口层）、`local_socket::prelude`（推荐用法）。核心结论有四点。其一，local socket 并非操作系统原生 IPC 原语，而是 interprocess 在 Windows 的 named pipe 与 Unix 的 Unix domain socket（UDS）之上构造的统一抽象，对外提供「按名字寻址、点对点私密连接」的语义；interprocess 绝不插入消息分帧或元数据，字节流完全透明，可与非 interprocess 程序互通。其二，库采用「trait 定义接口 + enum 做派发」的双层设计：`traits` 模块声明「能做什么」，模块顶层的 `Listener`/`Stream` 枚举经 `mkenum` 与 `dispatch!` 宏把调用转发给当前平台后端。其三，当前每个平台只有一个后端，单变体枚举的 match 被编译器消除，派发零开销，且该设计为未来多后端留出余地。其四，prelude 以 `as _` 匿名激活 trait 方法、用 `LocalSocket` 前缀重命名枚举类型，避免命名污染。后续讲义将承接的关键术语包括：local socket、named pipe、UDS、traits/enum 双层设计、对象安全、Sealed 封印、`mkenum`/`dispatch!` 宏，以及 name type。
