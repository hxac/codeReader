本讲聚焦 interprocess 在 Windows 的「安全描述符（SD）」抽象。Windows 用 SD 描述「谁能访问 named pipe 等可安全对象」，由 owner/group SID、DACL（鉴权）、SACL（审计）与控制位组成；关键术语包括 SID（安全标识）、ACE（访问控制项）、ACL（访问控制列表）。头号陷阱是 **null DACL**（present=true、指针为空，授予所有人全部权限）与 **absent DACL**（present=false、走默认安全）语义截然相反，interprocess 用 `unset_dacl()` 与 `set_dacl(null)` 两个方法显式区分。模块用三种类型表示 SD——拥有型 `SecurityDescriptor`（drop 释放 ACL/SID）、`BorrowedSecurityDescriptor`（共享借用、不要求 absolute 格式）、`MutBorrowedSecurityDescriptor`（可变借用、要求 absolute）——统一于 `unsafe trait AsSecurityDescriptor`（实现者须保证背后 SD 合法、内部指针有效且无别名）。SD 经 Windows 专有扩展 trait `ListenerOptionsExt` 注入监听器：采用「字段 `pub(crate)` 私有、trait 暴露」的设计，一路经后端 `from_options` 搬运、`create_security_attributes` 借出，最终作为 `SECURITY_ATTRIBUTES` 传入 `CreateNamedPipeW`。底层 `c_wrappers` 用 `OrErrno` 把 Win32 的 BOOL 翻译为 `io::Result`，`LocalBox` 守住本地堆分配/释放契约，`try_clone` 忠实复刻 null DACL 语义。该模块只提供可组合原语，不替用户构造复杂 SD。
