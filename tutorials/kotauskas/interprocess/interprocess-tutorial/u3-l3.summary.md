本讲（u3-l3）是同步 local socket 使用层收尾，最小模块为 `stream/trait.rs`（接口契约）与 `stream/enum.rs`（枚举本体）。`Stream` 的 supertrait 链 `Read + RefRead + Write + RefWrite + StreamCommon` 同时支持按值与按引用读写；`RefRead`/`RefWrite` 用 GAT 把「`&Self: Read`」编码进类型系统，这是「以 `Arc<Stream>` 替代 `split`」的类型论依据。

引入的关键术语与概念：阻塞控制三件套 `set_nonblocking`/`set_recv_timeout`/`set_send_timeout`——同一签名在 Unix 正常工作，Windows named pipe 后两者恒返回 `Err(Unsupported)`；`StreamCommon` 的 `take_error`（Unix 取暂存错误、Windows 恒 `None`）与 `peer_creds`（留待 u3-l4 详讲），且被 `Sealed` 封印；`split`/`reunite` 因按值消费在枚举层手写 `match`，不走 `dispatch!`；`RecvHalf`/`SendHalf` 与 `ReuniteError`——失败时经 `pub rh`/`pub sh` 原样归还两半所有权，Unix 以 `Arc::ptr_eq` 判同源、`Arc::into_inner` 取回，全程无 `unsafe`。
