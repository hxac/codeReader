本讲建立仓库目录地图，核心结论：本仓库只是 Vivado IP 封装层，功能真身在上游 `psi_multi_stream_daq`。三大模块：其一，顶层目录按「产物来源」划分——`hdl`/`scripts`/`bd`/`refdesign`/`drivers` 为人写、需手工维护，根目录 `component.xml` 与 `xgui/` 由工具生成、改了会被下次打包覆盖。其二，唯一 RTL 文件 `hdl/psi_ms_daq_vivado.vhd` 是封装外壳，内部例化上游实现 `entity psi_ms_daq_axi`，仅做 16 路 Stream 信号重排与「标量参数→数组 generic」聚合，不含任何采集算法与时序逻辑。其三，`scripts/package.tcl`（约 189 行，人写，调用外部工具 PsiIpPackage）是「源」，根目录 `component.xml`（7093 行 IP-XACT）是「产物」，二者关系类似源码与编译产物，改 IP 行为应改源后重新打包。四个必记路径：上述 RTL、打包脚本 `scripts/package.tcl`、驱动头 `drivers/psi_ms_daq_axi/src/psi_ms_daq.h`、参考应用入口 `refdesign/ZCU102/Sdk/app/src/main.c`。本地 `drivers/*.c/*.h` 每次打包都会被上游同名文件覆盖，驱动真身亦在上游。
