本讲承接 u3-l1 的寄存器地址模型，给出驱动「软件状态」的组织方式。对外暴露的两个句柄 `PsiMsDaq_IpHandle`、`PsiMsDaq_StrHandle` 本质都是 `void*`，用不透明指针把真实结构体（藏于 `.c`）隐藏起来，以实现信息隐藏与 ABI 稳定。它们背后是两个私有结构体：`PsiMsDaq_Inst_t` 记「一台 IP」（baseAddr、最大流数/窗口数、流间步进 strAddrOffs、流实例数组、三个访问函数指针）；`PsiMsDaq_StrInst_t` 记「一条流」（身份、配置缓存、中断进度 lastProcWin/irqCalledWin、回调、回指 ipHandle）。`PsiMsDaq_Init` 在堆上一次性分配两者，并建立 IP↔流双向引用——回指指针是流级 API 取得 baseAddr 与访问函数的唯一通道；`GetStrHandle` 零分配，仅返回数组元素地址。内存读写被做成可注入函数指针 `PsiMsDaq_AccessFct_t`，使同一份驱动可在裸机、Linux、仿真间切换。`PsiMsDaq_WinInfo_t` 则是按值传递、栈上瞬时的值类型，非句柄、不可长期持有。配置函数对参数「双写」（寄存器与软件缓存各一份），供后续地址计算直接复用。
