本仓库只是 Vivado IP 封装层，功能 VHDL 在上游 `psi_multi_stream_daq`、基础库在 `psi_common`，自身不带这些源码。本讲核心三点：一是 README「Dependencies」段被两行 HTML 注释标记包裹，是依赖清单的单一事实来源，既给人读也给脚本解析；6 条外部依赖分运行时（`psi_common`、`psi_multi_stream_daq`，无 `for ... only` 标签，综合进 IP）、开发期（`PsiSim`、`PsiIpPackage`、`psi_tb`，仅仿真/打包）、参考设计专用（`vivadoIP_axis_data_gen`）三类。二是 `scripts/dependencies.py` 是 15 行薄包装，用 `Parse.FromReadme` + `Actions.ExecMain` 调用外部包 `PsiFpgaLibDependencies` 把依赖克隆到 PSI 统一布局约定位置，自身无 git 逻辑。三是 `package.tcl` 用 `add_sources_relative` 声明本地唯一 RTL，用基准 `"../../.."`（PsiFpgaLib 根）配 `add_lib_relative` 列出 24 个被综合进 IP 的上游文件（`psi_common` 17 个 + `psi_multi_stream_daq` 7 个）。结论：README 声明的运行时依赖与 `add_lib_relative` 综合清单一一对应、互相印证。引入术语：运行时/开发期/参考设计依赖、PSI FPGA Library 统一布局、薄包装加外部共享包、单一事实来源。
