本讲义讲解 psi_common 的 `generators/` 目录——一层用 Python 在 VHDL 之外预生成「形状可变」薄外壳的代码生成器，因为 VHDL-93/2002 无法在 entity 端口表里循环展开不等宽端口或声明定宽 slv 数组类型。采用三件套结构：snippet 模板（带 `<占位符>` 的合法 VHDL 骨架，描述形状）、Python 脚本（`argparse` 解析参数 → 读模板 → `re.sub` 替换占位符 → 写成品，不理解 VHDL 仅做字符串替换）、bat 示例（记录调用约定）。生成器分两族：**合并族**（`simple_cc_X`/`status_cc_X`）把多个不等宽信号按字典序、低位居前累加拼成单总线喂给核心组件，替换标量占位符 `<WIDTH>`/`<POSTFIX>` 与四个块占位符 `<DATA_IN/OUT/MERGE/UNMERGE>`；**定宽族**（`par_tdm_wX`/`tdm_par_wX`）为 TDM 转换器生成含定宽数组类型端口的 package+entity，仅替换 `<WIDTH>`，通道数留给 generic。关键现实：`generators/` 自首次提交后未再更新，snippet 仍是 v3.0.0 重构前的 PascalCase 命名（`DataWidth_g`/`ClkA`/`DataA`/`VldA`），与现行组件（`width_g`/`a_clk_i`/`a_dat_i`/`a_vld_i`）不符，生成物无法直接编译，需手工刷新命名或借助 u11-l3 的 `migration_from_v2_to_v3_db.json` 迁移脚本。
