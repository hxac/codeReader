本讲介绍 `psi_common_clk_meas`，它用**直接计数法**测频：以主时钟生成 1 秒**闸门时间**，在其中数被测时钟上升沿，计数值即频率（Hz）。组件未例化 `strobe_generator` 或 `pulse_cc`，而是把闸门生成（倒数 `master_frequency_g` 拍到 0 翻转）与 toggle-CDC 都内联实现。

精度为绝对 **±1 Hz（量化误差）**、相对 1/f_test，故测高频很准、测低频很差；主时钟自身相对误差会透传到结果。

核心机制是两个方向相反的 **toggle-CDC 握手**（3 级同步 + 边沿检测）：1 Hz 门控（主→测试域）与结果就绪（测试→主域）。多 bit 计数值 `Result_T` 跨域读回**无需格雷码**，靠握手保证读时数据已稳定，这与异步 FIFO 指针必须用格雷码形成对照。

generic 含 `master_frequency_g`（须等于真实主时钟）、`max_meas_frequency_g`（量程上限兼计数器饱和值）、`rst_pol_g`；输出 `frequency_hz_o` 为 32 位无符号 Hz、同步于主时钟，`vld_o` 为单周期脉冲、约每秒一次，超量程则饱和、被测时钟停摆则超时输出 0。组件定位为**系统自检 / 健康监测传感器**，常用于 SoC FPGA 中以可信 PS 主时钟核对 PLL 输出频率是否配对，结果经 AXI 寄存器送软件读取。
