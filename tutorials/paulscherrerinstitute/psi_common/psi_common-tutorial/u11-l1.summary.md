本讲以 psi_common_pl_stage_tb 为范本，拆解 PSI 库自校验测试平台（self-checking testbench）的通用骨架——这是贡献代码的硬性要求。TB 由 TbGen.py 生成公共脚手架（时钟、复位、停机控制、ProcessDone），人工只填业务（p_stim/p_check 与 testcase/done），呈八段结构。停机协议靠 TbRunning（驱动时钟）与 ProcessDone/AllProcessesDone_c（汇报完成）让仿真自动结束；两个工作进程用 testcase（广播用例号）和 done（应答）按用例对齐，初值 -1/False 防误触发。全库判错约定为 report "###ERROR###: ..." severity error，run.tcl 用 run_check_errors 一次 grep 收口成败；用 error 而非 failure 以便一次跑出尽量多的错。检查代码用 psi_tb 兄弟工具包的 print 与 StdlvCompareInt（比较失败时自动带前缀上报）。assert 常带 "or not <generic>" 守卫使断言仅在对应分支生效；新 TB 须登记进 sim/config.tcl 才会被回归跑到。
