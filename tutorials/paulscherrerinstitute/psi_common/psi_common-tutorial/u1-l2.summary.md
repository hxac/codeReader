本讲义梳理 `psi_common` 仓库的顶层目录组织，只读结构不展开源码。仓库根目录包含七大目录：`hdl/`（可综合源码）、`testbench/`（自校验测试平台）、`sim/`（仿真回归脚本）、`doc/`（文档与组件索引）、`scripts/`（Python 工具与库重构）、`generators/`（按位宽生成实例的代码生成器）、`sigasi/`（IDE 工程）。核心结论有四点。其一，区分「仓库结构」与「工作副本结构」：前者单独克隆即完整，后者因以相对路径引用兄弟仓库 `psi_tb` 和 PsiSim，跑仿真或改库时必须按固定路径摆成兄弟目录。其二，`hdl/` 为扁平目录，遵循「一个文件一个 entity/package」硬性约定，单元统一 `psi_common_` 前缀，当前约 61 个 `.vhd` 文件，按九大类（Memory、FIFO、CDC 等）划分。其三，`testbench/` 一组件一子目录，与 `hdl/` 几乎一一对应，命名规则为 `<源码名>_tb`，少数底层 RAM 与 package 无专属 TB。其四，`doc/README.md` 是全库总索引，串联「源码↔测试↔说明文档」三位一体；新 TB 须登记到 `sim/config.tcl`。引入的关键术语：library、entity、package、testbench、工作副本结构、自校验 TB。
