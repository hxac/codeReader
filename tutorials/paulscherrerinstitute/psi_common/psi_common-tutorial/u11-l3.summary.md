本讲义解读 psi_common 为 v3.0.0「统一代码风格」大重构而设的一次性迁移工具，位于 `scripts/refactoring/`，纯 Python 实现，用正则按行扫描而非构建 VHDL 语法树。核心思路是两阶段解耦：第一阶段 `parse_library.py` 对比 v2 旧库与 v3 新库两份 `hdl/`，利用「v3 是纯改名重构、端口顺序不变」这一关键假设，用 `dict(zip(旧名, 新名))` 按位置配对，自动生成「旧名→新名」映射 JSON；第二阶段 `refactor_library_and_testbench.py` 加载 JSON，对 `hdl/`、`testbench/`、`config.tcl` 做确定性字符串替换。两者以 `migration_from_v2_to_v3_db.json` 这一**数据契约**解耦。`hdlrefactor.py` 是共享工具箱，核心是查表函数 `conv_fun` 的三级回退（组件专属表 → `#ALL#` 全局表 → 原样返回），以及四个作用域不同的改写函数：实体声明、例化、符号（最宽，唯一回退全局表）、TCL generic。关键术语包括 DB/字典、`#ALL#` 全局表、`use_all` 回退开关、注释剥离、原地覆盖。本讲归纳出 v2→v3 五条命名规则（generic 转 snake_case、`_i/_o/_io` 方向后缀取代 `In/Out` 前缀、缩写统一如 `Data→dat`、AXI 信号整体小写、package 函数进 `#ALL#`），并强调同一符号在不同组件映射不同（如 `InData` 在 `pl_stage` 为 `dat_i`、在 `async_fifo` 为 `in_dat_i`），故必须按组件查表，且 DB 需人工复核。
