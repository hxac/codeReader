本讲讲解两个 SPI 主机组件：固定位宽的 psi_common_spi_master 与运行时可配位宽的 spi_master_cfg，二者均沿用 u7-l1 的二进程 record 设计法（r/r_next + p_comb + p_seq）。核心要点：SPI 四种时序模式由综合期 generic spi_cpol_g（SCLK 空闲电平）与 spi_cpha_g（采样沿）组合而成，主机用 GetClockLevel 函数及 ClkInact/ClkAct 两个状态生成 SCLK，频率 = 系统时钟 / clk_div_g，半周期计数阈值 ClkDivThres_c = clk_div_g/2 - 1；CPHA=0 需 SftComp_s 补偿状态预先取首比特。位序由 lsb_first_g 在 ShiftReg 过程中二选一（MSB 先发取高位左移、LSB 先发取低位右移）。并行侧用 start_i/busy_o/done_o 脉冲握手而非 AXI-S 的 VLD/RDY，done_o 恰好一拍，传输途中拉 start_i 被忽略。spi_master_cfg 把位宽下放到运行时端口 trans_width_i（仅在 start_i 拍锁存，BitCnt 与其比较判完成），代价是去掉三线 SPI 与 latch enable，数据端口改为 wr_dat_i/rd_dat_o。三线 SPI 靠 spi_tri_o 配合 read_bit_pol_g、spi_data_pos_g，在命令位移完后把驱动权让给从机。
