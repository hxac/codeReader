本讲建立读任意 psi_common 组件的通用约定，含四块。命名规范：snake_case、端口 _i/_o/_io 方向后缀、接口共同前缀、架构名限 behav/struc/rtl、显式 end entity。AXI-S 握手：传输仅在 VLD 与 RDY 同高一拍发生；源端须自主拉 VLD 且握手前不可撤，宿端可灵活进出 RDY（反压），库以 vld/rdy/dat 简写但不严格遵循标准命名。TDM 约定：等速率多路时分复用不加通道指示，通道按 0-1-2 隐式循环，速率不同才加通道编号，使 TDM 对组合逻辑透明。代表组件 psi_common_pl_stage 把规范落地，采用二进程 record 法（r/r_next 与 p_comb/p_seq）；use_rdy_g 切两套实现——true 走带影子寄存器 DataShad 的完整握手，弥补 rdy_o 寄存导致的反压延迟一拍，false 则省略反压。
