本讲覆盖三个异步时钟域跨越组件，与前置 pulse_cc 构成 pulse_cc→simple_cc→status_cc 依赖链，bit_cc 独立成路。simple_cc 复用 pulse_cc 只跨 valid 一个 bit，数据源域原地锁存、目标域在 valid 还原那拍采样，规避多 bit 总线撕裂；只有 valid 不带 ready，须保证数据率≤f_b/4。status_cc 在其上加 A↔B 请求-应答回环（B 收数据翻转应答位，A 同步后边沿检测再发），自驱动生成节拍，适合慢变状态/配置，变化率≤f_slow/10 且可跳过中间值。bit_cc 自带双级同步器，借 ASYNC_REG 等综合属性免手写约束，无复位端口，各 bit 不保证同拍到达，严禁用于相关多 bit 总线。关键术语：撕裂值、valid-only 握手、请求-应答回环、双级同步器、a_rst_o/b_rst_o 复位透传。选型看信号形态、谁管节拍、是否容忍跳值与是否需反压。
