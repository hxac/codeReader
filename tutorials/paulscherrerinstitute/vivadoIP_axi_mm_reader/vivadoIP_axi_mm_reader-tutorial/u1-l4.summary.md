本讲是 IP 开发闭环中的「打包」环节：用 PsiIpPackage 封装库把 HDL 打包成 Vivado 可识别的自定义 IP，产物为 IP-XACT 格式的 component.xml。package.tcl 是一条线性流水线：init 元信息 → 加三件自有 RTL（wrapper 为顶层）→ 引用 psi_common 九件 → 加 C 驱动 → 声明 GUI 参数 → 清理误检接口 → 设可选端口条件 → package_ip（综合开关打开、器件族 xc7a*）。关键术语：generic（综合时定死的类属）、GUI 参数、可选端口的条件启用（enablement condition）。GUI 参数有两处住所：package.tcl 用 gui_create_parameter/gui_add_parameter 声明，xgui 脚本是 Vivado 原生实现，由 update_MODELPARAM_VALUE.X 按同名把 GUI 值映射到 RTL generic，六个参数与 generic 一一同名。核心结论：m_axis 端口仅在 Output_g=="AXIS" 时出现，由 add_port_enablement_condition 与 add_interface_enablement_condition 两行实现，RTL 侧有 g_axis/g_naxis 两个 generate 块配套，component.xml 条件、GUI 参数、RTL generate 三层须保持一致；默认 Output_g="AXIMM"，故默认打包出来的 IP 没有 m_axis 端口。
