本讲解析 IP 打包产物在 Block Design 与 Vitis 中的集成机制。`component.xml` 是 `package.tcl` 生成的 IP-XACT 总账本，以六大段（busInterfaces/addressSpaces/memoryMaps/model/fileSets/vendorExtensions）自描述接口、地址、参数与文件；端口位宽与 `reg0` 地址范围均为依赖 `MODELPARAM_VALUE` 的表达式，属机器生成产物、不可手改，改行为应改源头再重新打包。`bd.tcl` 借 `init`/`pre_propagate`/`propagate` 三个回调自动传播 AXI4 的 `ID_WIDTH`：`init` 把从机 `C_S00_AXI_ID_WIDTH` 标记为 propagate-only，`propagate` 把上游主机 ID 收进从机参数（`pre_propagate` 因 `m00_axi` 无 ID 而空跑），再经 xgui 的 `update_MODELPARAM_VALUE` 桥接到 RTL generic 落地。`.mdd` 声明外设与驱动绑定（外设名须与 `component.xml` 一致），`.tcl` 用 `xdefine_include_file` 把 `NUM_INSTANCES`/`DEVICE_ID`/`C_BASEADDR`/`C_HIGHADDR` 写入 `xparameters.h`，`Makefile` 把 `.c` 编入 `libxil.a`。核心结论：BD 地址编辑器给 `s00_axi/reg0` 分配的基地址，经 MDD/TCL 链路成为驱动 API `baseAddr` 的来源，将 RTL 寄存器映射与软件 `Xil_Out32` 调用端到端闭合。
