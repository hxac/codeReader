本讲从硬件 RTL 切到嵌入式软件，讲解 `drivers/axi_mm_reader` 这套 C 驱动。它本质是极薄的封装：每个 API 即「按地址常量算字节地址 + 一次 `Xil_In32`/`Xil_Out32`」，地址常量与 RTL `definitions_pkg.vhd` 的字索引严格对应（字节地址 = 字索引 × 4）。核心 API 六个：`SetEnable`/`GetEnable`、`SetRegTable`、`GetLevel`、`ReadFifoEntry`/`ReadFifoPacket`。关键约定：`SetRegTable` 须在 IP 禁用态调用（FSM 仅在 `Idle_s` 采样 `RegCount`），否则返回 `IpMustBeDisabled`；AXIMM 模式取数必须「先读 `RdLast`（peek）再读 `RdData`（pop）」，因读 `RdData` 会弹出 FIFO 一项，顺序反则包尾错位。错误码 `FifoIsEmpty`/`NoCompletePacketInFifo` 描述调用契约违反。驱动无任何软件触发寄存器，读周期由硬件 `Trig` 或内部超时启动，驱动只负责「配置 + 使能 + 取数」。集成三件套 `Makefile`/`.mdd`/`.tcl` 把驱动编进 `libxil.a` 并生成 `xparameters.h` 的 `C_BASEADDR`。
