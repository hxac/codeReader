本讲义围绕 psi_fix 回归测试讲解四件配套自动化：CI 编排、依赖检出、文档生成、原语单测。核心是 ciFlow.py，它采用「文本契约判决」——不信任仿真器退出码，而是扫描 Transcript.transcript：命中 ###ERROR### 判 exit(-1)（功能错/位真不匹配），缺失 SIMULATIONS COMPLETED SUCCESSFULLY 判 exit(-2)（环境错/未跑完）。它分两阶段串行门控：先经 ci.do 薄壳驱动 run.tcl 五步跑 Modelsim VHDL 回归，通过后才跑 Python 单测，前阶段失败即终止进程。dependencies.py 是对外部 PsiFpgaLibDependencies 包的薄封装，把 README.md 依赖段（由 HTML 注释划定边界）当作唯一真相源，自动克隆并摆好 PsiSim/psi_common/psi_tb/en_cl_fix 四个姊妹仓库。hdl2md.py 用正则解析 VHDL entity、借 pandas to_markdown 生成 generics/interfaces 表，确立「接口表为生成产物、描述段为人工产物」的文档分工。psi_fix_pkg_test.py 覆盖原语层（resize/add/mult 等 API），与覆盖组件层的 VHDL 协同仿真互补，以独立硬编码的绝对期望值卡住黄金模型地基。本讲引入的关键术语：CI（持续集成）、batch 模式、两级错误码、文本契约、唯一真相源、生成产物、原语层/组件层。
