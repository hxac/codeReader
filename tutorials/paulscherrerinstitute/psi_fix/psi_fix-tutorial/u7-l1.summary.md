本讲为 psi_fix 的 FIR 家族建立全景图,核心是六字段命名模板 `psi_fix_fir_<decimation>_<calc>_<channels>_<channel-handling>_<coef>_<arch>`,权威定义见 FirNaming.txt,缺省字段直接省略不写。`<decimation>` 标注是否支持抽取(dec);`<calc>` 区分三种计算结构:**ser** 用 1 个乘法器、约 N 周期/样本,**par** 每抽头 1 个乘法器、1 周期/样本,**semi** 用 `multipliers_g` 个乘法器、周期=⌈N/M⌉,是 ser 与 par 之间的可调滑动条;`<channels>` 与 `<channel-handling>` 的 **chpar/chtdm** 直接体现在 dat_i 端口位宽(并行拼总线 vs 时分复用单样本);`<coefficient-handling>` 的 **conf/fix** 由 generic `use_fix_coefs_g` 在综合期二选一,共用同一份 RTL,conf 多出运行时写入接口并常以双口参数 RAM 为载体;`<architecture>` 仅 x7 绑定工艺。贯穿全族的关键结论:所有 FIR 命名变体只描述「RTL 怎么算」,数学上共享同一个位真 Python 模型 psi_fix_fir.py,黄金参考与实现解耦;累加器格式 `accuFmt=(1, outFmt.I+1, inFmt.F+coefFmt.F)` 在模型与 RTL 一致,是位真契约的痕迹。
