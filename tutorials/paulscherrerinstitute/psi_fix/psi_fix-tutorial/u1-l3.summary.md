本讲解读 psi_fix「一键回归」背后的 TCL 脚本框架。回归由**外部 PsiSim 框架**（随 `psi_fpga_all` 发布、经相对路径 `../../../TCL/PsiSim/` 加载，体现并排摆放目录结构）驱动，它把「声明编译/跑什么」与「用哪个仿真器」解耦。Modelsim 入口 `run.tcl` 与 GHDL 入口 `runGhdl.tcl` 仅差一个 `init -ghdl`，二者都 `source` 同一份 `config.tcl`，按 `init → source config.tcl → compile_files -all -clean → run_tb -all → run_check_errors` 五步完成回归；交互式入口只编译不跑，`ci.do` 则把 `run.tcl` 包成 batch 任务。

`config.tcl` 是声明式配置：`add_sources -tag`（lib/src/tb）声明编译清单（`psi_fix_pkg.vhd` 须最先编译），`create_tb_run`/`tb_run_add_arguments`/`add_tb_run` 声明测试台及其**参数矩阵**（同一 TB 换 generics 跑多轮），`tb_run_add_pre_script`/`tb_run_add_time_limit` 挂运行前钩子与超时。

回归判定依赖 `###ERROR###` 字符串约定：每个 VHDL 组件配**位真双模型**（逐位一致的 Python 模型），测试台读 pre_script 生成的整数位表示文本，由 `psi_tb` 的 `ApplyTextfileContent`/`CheckTextfileContent` 逐位比对，失败时打印 `###ERROR###`；`ciFlow.py` 据此判功能错（`exit -1`），另用 `SIMULATIONS COMPLETED SUCCESSFULLY` 区分环境崩溃（`exit -2`），通过后再跑 Python 单测。pre_script 分两类：**数据生成型**在 TB 运行前生成 `Data/*.txt`（FIR/CIC 等，数据量大且固定随机种子可确定性重生），**代码生成型**在编译前生成 `.vhd`（如 `lut_gen`），故必须更早运行。
