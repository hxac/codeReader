本讲围绕 `psi_fix_cordic_vect`（CORDIC 矢量模式）展开三个模块。其一，矢量模式用「移位+加减+查角度表」把向量逐步转到与 x 轴重合（把 y 驱到 0），边转边累加角度，由直角坐标 (I,Q) 算出幅度 r 与相位 θ=atan2(Q,I)，全程不含乘法器；它与旋转模式（u5-l2）共享同一迭代内核，唯一差别是方向判据——矢量看 sign(y)、旋转看 sign(z)，直接体现在 `CordicStepZ` 的函数签名上。其二，因 CORDIC 收敛锥有限（约±99.9°），入口先取绝对值把向量折进第一象限并记下 2 bit 象限，出口再按象限把第一象限相位还原成真实相位；幅度与象限无关，不需补偿。CORDIC 引入固定增益 G_N≈1.6468，由 `gain_comp_g` 控制是否乘 1/G_N 补偿，补偿系数刻意选成 `(0,0,17)` 以落入单个 DSP。其三，4.0.2 把输入流 ready 从误导性的 `rdy_i` 改为 `rdy_o`——因对接收数据的实体而言，TREADY 是由本组件产生、对外输出的反压信号，方向为 `out`；姊妹件 `cordic_rot` 尚未修正。组件提供 PIPELINED（1 样本/时钟、带 `pl_stg_per_iter_g` 旋钮、ready 恒高）与 SERIAL（约 N 时钟/样本、`rdy_o=not XinVld` 反压）两种架构，均配 Python 位真模型与 preScript 协同仿真逐位验证。
