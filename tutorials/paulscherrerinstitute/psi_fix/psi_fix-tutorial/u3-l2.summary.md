本讲把 u3-l1 的「位真双模型」方法论落到滑动平均组件 `psi_fix_mov_avg` 上，给出 psi_fix 协同仿真的完整工程套路，覆盖三个最小模块：自检测试台结构、preScript 数据生成、文本逐位比对。

psi_fix 测试台是**自检测试台**：由 `TbGen.py` 按统一模板生成，含一个 DUT 例化、一个发激励的 `stim` 进程、一个比对输出的 `check` 进程，外加时钟/复位/停机控制进程。`stim` 和 `check` 并发解耦，各自结束时把 `ProcessDone` 向量对应位置 1；`p_tb_control` 仅在所有 done 位凑齐后拉低 `TbRunning` 停机。

**协同仿真**用文本文件桥接 Python 与 VHDL：仿真前 PsiSim 经 `config.tcl` 的 `tb_run_add_pre_script` 调一次 `preScript.py`，它实例化 Python 位真模型（**黄金参考**），用「最坏情况激励（全幅跳变 + `np.random.seed(0)` 固定随机段 + 零段）」跑 `Process()`，再经 **`psi_fix_get_bits_as_int`** 把输入和期望输出转成「二进制位模式的有符号整数」，用 `np.savetxt`(`fmt="%i"`、带 `header=`) 落盘成 `Data/*.txt`。VHDL 侧 `ApplyTextfileContent` 把 `input.txt` 按 `duty_cycle_g` 节拍重放成时序喂 DUT，`CheckTextfileContent` 在每个 `OutVld` 把 `to_integer(signed(OutData))` 与 `output_*.txt` 逐行比对，不符即打印 **`###ERROR###`**（CI 唯一失败判据）。两过程来自外部库 `psi_tb`，中转类型为 `TextfileData_t`，`IgnoreLines => 1` 跳过 header。

核心数学基础是「**整数即位模式**」：同组二进制位按有符号整数解读相等 ⟺ 每一位都相等，故逐位比对可简化为整数比对。`config.tcl` 的 `tb_run_add_arguments` 配合 **命名约定链**（generic `gain_corr_g`、Python 模型常量、文件名 `output_<lower>.txt` 三者经 `to_lower` 自动对齐，无需 `if/else`），实现「同一测试台换 generics 跑多轮」，preScript 只跑一次、多份输出文件供各轮复用。这套「套子」是后续每讲组件复用的协同仿真模板。
