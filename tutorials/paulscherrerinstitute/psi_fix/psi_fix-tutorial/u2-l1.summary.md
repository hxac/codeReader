本讲是进入 psi_fix 源码内部的第一讲，聚焦 psi_fix_pkg 的三个最小模块。它定义全库通用的三个核心类型：格式描述符 psi_fix_fmt_t（record，字段 s 符号位、i 整数位、f 小数位，总位宽 W=s+i+f）、舍入枚举 psi_fix_rnd_t（round/trunc）、饱和枚举 psi_fix_sat_t（wrap/sat），外加数组版 psi_fix_fmt_array_t。psi_fix 采用「壳+内核」分层：不自行实现定点运算，而通过转换桥 psi_fix2_cl_fix/cl_fix2_psi_fix（各 4 个重载）把类型翻译给外部库 en_cl_fix，真正的数学由 cl_fix_* 完成。关键映射为 round→NonSymPos_s（半值向正无穷的非对称舍入）、trunc→Trunc_s、sat→Sat_s、wrap→None_s（不饱和即回绕）。基础查询函数 psi_fix_size、psi_fix_from_real、psi_fix_from_bits_as_int/psi_fix_get_bits_as_int 均为转换桥的瘦封装；from_real 额外加了无符号负数防呆，整数互转函数是协同仿真逐位比对的底座。需注意组件层默认 round/sat（偏安全）与库运算函数默认 trunc/wrap（偏省资源）不同。VHDL 与 Python 两侧转换表同构，是位真双模型的前提，Python 侧另有 53 位精度限制。真正的运算函数留待 u2-l2。
