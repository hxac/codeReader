本讲是 psi_fix 学习手册的收口讲义，把贡献新组件固化为「五件套」流程：Python 位真模型（黄金参考，先于 VHDL）、VHDL 两段式实现、自检测试台、文档与索引登记、config.tcl 回归注册，依赖顺序固定为「设计→Python→VHDL+测试台→文档→注册」。核心纪律：位真模型与自检测试台是不可妥协的入库门槛；模型须镜像硬件每个量化点，默认走手写型，仅「同内核、不同表」的组件族才用代码生成型。测试台由 TbGen.py 生成骨架，preScript 用最坏情况加固定随机种子跑模型，经 psi_fix_get_bits_as_int 写整数文本，测试台逐位比对，不符报 ###ERROR###，这是 CI 判功能失败的唯一判据。config.tcl 分源码、测试台、TB Run 三段，漏 TB Run 则测试台能编译却不被运行；维护阶段禁止只改 VHDL 不更新测试台的脏补丁。后续 u10-l2 讲 CI 自动验收。
