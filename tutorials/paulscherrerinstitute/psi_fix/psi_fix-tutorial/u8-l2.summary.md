本讲承接 u8-l1 的 `psi_fix_lin_approx_calc` 内核，解答 offset/gradient 两张定点表从何而来：psi_fix 用 Python 脚本 `model/psi_fix_lin_approx.py`（位真模型 + 代码生成器二合一）从一个数学函数一次性算出表、生成组件实体、生成自检测试台。表生成算法是：在输入可表示范围上等分 P 段、取段中心采样，offset 直接代值、gradient 用 scipy `derivative` 有限差分求导，再用 `psi_fix_from_real(err_sat=False)` 量化——故意不饱和，超界说明格式选小、应回改配置。有符号输入需把段中心旋转半圈以对齐内核地址映射。模板代码生成由 `GenerateEntity`/`GenerateTb` 用 `str.replace` 把 `snippets/*.vhd` 中的 `<占位符>`（`<ENTITY_NAME>`/`<TABLE_CONTENT>` 等）换成实体名、格式常量与表内容，表行严格按「gradient 高位 | offset 低位」拼接，这是与内核的位真硬约束。四个组件 `sin18b/sqrt18b/inv18b/gaussify20b` 由 `CONFIGS` 配置驱动，共享同一手写内核与同一模板，差异仅在表与格式常量，构成「生成式组件族」：改精度只动配置重跑脚本、手写 RTL 零修改。其测试数据 `stimuli/response.txt` 生成一次提交进仓库、经 generic `stimuli_dir_g` 喂入，不用 pre_script 现场重生，与 FIR 路线对照；`psi_fix_lut.py` 是同套哲学在整表查表场景的平行样例。
