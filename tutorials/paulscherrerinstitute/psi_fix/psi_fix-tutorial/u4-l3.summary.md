本讲在「简单处理组件实战」单元中选取两个算术组件，对照讲解实现与验证策略。`psi_fix_comparator` 是单进程定点窗口比较器，把 `psi_fix_compare` 封装成「阈值窗口（min/max 双标志）+ 三级选通（strobe）延迟对齐」结构；三类输入须同格式 `fmt_g`。`psi_fix_bin_div` 用迭代恢复式除法（restoring division）计算 Num/Denom，策略为「先取绝对值、做无符号除法、最后补符号」；五状态两段式状态机（Idle/Init1/Init2/Calc/Output），`Calc_s` 四步「商左移、比较、条件减、余数左移」，迭代 `out_fmt.I+out_fmt.F+2` 次；内部商 `ResultInt` 虽每拍左移，但存于含 `F_out+1` 小数位的 `result_int_fmt_c`，位模式解释恰为真值。核心结论是验证策略抉择：组件数学归约到可信 psi_fix_pkg 原语（如比较器）→ 纯 VHDL 自检测试台即可；组件自创算术（如除法器）→ 必须 Python 位真模型 + preScript 协同仿真，经 `psi_fix_get_bits_as_int` 把定点值写成位模式有符号整数，VHDL 用 `###ERROR###` 约定逐位比对。关键判据是 `psi_fix_compare` 仅存于 VHDL 侧、无 Python 镜像。
