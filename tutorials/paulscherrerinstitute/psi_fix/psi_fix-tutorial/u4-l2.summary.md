本讲详解 psi_fix 库中纯 VHDL、厂商无关的真双口参数 RAM `psi_fix_param_ram`，它是库内少数「无 Python 位真模型」的存储组件，回归仅靠一个 VHDL 自检测试台覆盖初始内容、零填充与整片覆写。其形态由四个 generic 决定：`depth_g` 定深度、`fmt_g` 定数据位宽、`behavior_g` 定同地址读写语义、`init_g` 定上电初值。A/B 两端口完全对称且可接不同时钟，存储用 `shared variable` 才能让双进程在同一时钟沿并发读写同一阵列。关键术语 RBW（Read-Before-Write，读旧值）与 WBR（Write-Before-Read，读新值）仅在「同端口、同沿、同地址既读又写」时有别，默认 RBW 贴合 FPGA BRAM 的 READ_FIRST 语义，便于综合时推断成硬件块 RAM。`init_g : t_areal` 经 `GetInit` 与 `psi_fix_from_real` 在综合期把前 N 个浮点系数量化为定点、其余补 0，等价于 RAM 上电初值表。它常被 FIR 等可配置组件当作「运行时系数 RAM」：A 口在配置时钟域写系数、B 口在数据时钟域只读取系数，实现不停机换系数。
