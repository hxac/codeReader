本讲承接 u1-l1 的「位真双模型」理念，绘制仓库源码地图，帮助凭命名机械定位任意组件的全部产出物。仓库顶层八个目录分工明确：`hdl`（可综合 VHDL）、`model`（Python 位真模型 + 代码生成）、`testbench`（自检测试台 + preScript）、`sim`（回归脚本）、`scripts`（CI/依赖/文档/重构）、`doc`（文档）、`unittest`、`sigasi`。其中 `hdl`、`model`、`testbench`、`doc` 四目录文件数同量级且命名同构，体现「一组件、五件套」组织方式。核心纪律有二：一是一个 `.vhd` 文件只放一个 package 或 entity；二是测试台子目录 = 组件名 + `_tb`，使文件名即索引。以 `psi_fix_mov_avg` 为样例，其五件套为 VHDL 实现、Python 模型、VHDL 测试台、`preScript.py` 协同仿真脚本、`doc/files` 文档。关键术语包括：preScript 在编译前运行，跑 Python 模型生成 `Data/*.txt`，测试台读回逐位比对形成「Python→文本→VHDL」协同仿真闭环；`model/snippets` 存放代码生成模板、`model/matlab` 提供 MATLAB 复用桥接；`scripts/hdl2md.py` 自动生成文档，README 依赖段被 `dependencies.py` 解析——文档与依赖均为生成产物。`psi_fix_pkg.vhd` 因被所有实体依赖须先编译。
