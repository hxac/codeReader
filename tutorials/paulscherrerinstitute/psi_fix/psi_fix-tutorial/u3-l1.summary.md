本讲为方法论讲义，覆盖三个递进模块。一为七阶段设计流程：算法可行性→设计与文档化→Python 模型→Python 协同仿真→VHDL 协同仿真→硬件验证→维护，整体前重后轻；其中 Phase 2 是唯一禁止编码的阶段，须先把定点格式与资源估算写进"活文档"。二为位真验证思想：Python 位真模型即黄金参考，VHDL 只负责逐位对齐，箭头单向——出现偏差时修 VHDL，而非改 Python 凑结果。三为刺激设计：模型须用最坏情况刺激验证自身正确，样板为 dirac 冲激（输出恰为抽头系数，最易调试）与符号匹配系数的 ±1 序列（逼出理论最大输出 ∑|h[k]| 及饱和）；白噪声符号随机，在合理数据量内几乎命不中最坏情况，会留下饱和死角。关键承接术语：七阶段流程、活文档、黄金参考、最坏情况刺激、协同仿真、逐位比对。
