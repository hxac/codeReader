本讲（u2-l3）讲解 `model/psi_fix_pkg.py` 如何落地位真双模型的 Python 半边，分三个最小模块。

一是类型与 API 镜像：镜像 VHDL 三个核心类型——格式 `psi_fix_fmt_t`（持 `[s,i,f]`）、舍入 `psi_fix_rnd_t`、饱和 `psi_fix_sat_t`，及一组同名运算函数（`from_real/resize/add/sub/mult/shift` 等），构成两侧共同词汇；核心算术 API 严格对齐，但两侧各有专属辅助函数，并非完全对称。

二是位真与精度限制：IEEE 754 双精度仅 53 位有效位，格式总位宽 `W>53` 时无法逐位保真，构造会抛 `BittruenessNotGuaranteed`。检查默认关闭，由 `enable_range_check` 全局打开或用 `with_range_check_disabled` 临时豁免已知安全的大中间格式（如 FIR 累加链）；中间结果如 `cumsum` 同样可能悄悄丢位。

三是与 en_cl_fix 复用：运算函数全是薄封装，经转换桥 `PsiFix2ClFix`（`s==1→Signed`、`round→NonSymPos_s`、`wrap→None_s`）翻译后委托 `cl_fix_*`；通过 `sys.path.insert` 引用并排摆放的同级 `en_cl_fix`，两侧共用单一真相源。psi_fix 仅在 `from_real`（加 `err_sat`）和移位函数（加 `max_shift`）处增值，库级默认 `trunc/wrap`，与组件层 `round/sat` 相反，建模须显式对齐。
