本讲承接 u2-l2 的三数组接口（reg_rdata/reg_wdata/reg_wr），给出 fpga_base 的完整寄存器映射：字节偏移 = 寄存器下标 × 4（即地址高 6 位 A[7:2]），64 个 32 位寄存器覆盖 0x00~0xFF，分版本与日期(0x00~0x28)、项目/设施字符串(0x40~0x5C)、LED 与 DIP(0x60~0x64)、保留区四段，未赋值者读 0（源自复位默认值 ResetVal_g）。核心结论：读写权限并非 AXI 协议层访问保护，而由 reg_rdata 的接法自然形成——直通回显 reg_wdata 即可读写（软件日期、LED 低 8 位），接独立信号源即只读（版本、固件日期、字符串、DIP 开关）。字符串采用大端打包（第 0 字符放最高字节），C 侧读取时按字节倒序还原。HDL 寄存器下标与 C 头文件 C_*_OFS 宏是两份无编译期联动的硬件/软件契约，改一边必须同步另一边（含 EPICS、JTAG 脚本）。软件日期由 fpga_base_version() 在处理器启动时写入 __DATE__/__TIME__，可与固件日期对照判断软硬配套。该映射是 u3（固件日期硬件实现）与 u5（C 驱动、JTAG 调试、EPICS 集成）各篇的共同地基。
