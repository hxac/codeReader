本讲精读日期存储组件 `fpga_base_date`，解答"fpga_base 如何把固件编译时间永久烧进比特流"。最小模块三层递进：第一层用 Xilinx 原语 FDPE（带时钟使能和异步置位的 D 触发器）当一位 ROM——每个 FDPE 的 D/CE/PRE 均接常量 '0'，使运行期永不翻转，输出 Q 永久等于上电初值属性 INIT；年/月/日/时/分各 32 个，共 160 个 FDPE。之所以不用 VHDL `constant`，是因为只有真实原语 cell 综合后才会留下可被 `get_cells` 寻址、再被 `set_property INIT` 改写的单元。第二层用 `dont_touch` 属性绑在每个实例标签上，阻止综合器的常量传播把这批"输入全常量"的 FDPE 当死逻辑删除——它是机制的救命稻草，只挡优化不挡属性改写。第三层用两条互斥的 `if generate` 实现双模式：`g_generics`（开关 `C_USE_GENERIC_DATE=true`）从 generic 常量读日期，由 `update_version.py` 在综合前替换 `$$tag$$` 注入；`g_ngenerics`（默认）从 FDPE 的 Q 读，由 `fpga_base.tcl` 在综合后逐位写 INIT。该开关在顶层统一连到总闸 `C_USE_INFO_FROM_SCRIPT`，同时控制版本号寄存器来源。关键承启点：统一 32 位和可预测的 generate 命名（如 `gen_year[x].year_dfpe_inst`）为下一篇 u3-l2 的 TCL 逐位写 INIT 提供了寻址基础；日期输出直接驱动只读寄存器 `reg_rdata(1~5)`（偏移 0x04~0x14）。
