本讲为 fpga_base 提供两条不写 HDL/C 的系统侧集成路径，均建在 u2-l3 寄存器映射之上。一是人机调试路径：Vivado 硬件管理器经 JTAG-to-AXI Master，用 `create_hw_axi_txn`/`run_hw_axi`/`report_hw_axi_txn` 三件套「无处理器」地读写 AXI 寄存器，每函数首行 `reset_hw_axi`，一次调用即一笔 AXI 事务，纯同步无中断；`-t`/`-w` 解释读回原始字：日期用 `-t u4 -w 32` 直接取整数，字符串用 `-t u1 -w 16`。核心难点是字符串因 HDL 大端打包需「字内字节倒序」循环还原。二是机器集成路径：EPICS 经 regDev 设备支持，用一条 `seq`（`SCAN="1 second"`）单点触发，经 `LNK`/`FLNK` 级联读 5 个日期寄存器，再由 `scalcout` 的 `PRINTF` 逐段格式化、末端拼成可读日期串，单点节流保证一致性与低总线负担。关键结论：同一寄存器偏移在 HDL、C 头文件、JTAG 脚本、EPICS template 四处各自硬编码、无编译期联动，改 HDL 必须同步其余三处——这是本系列反复强调的硬件/软件契约风险。另需注意 `0x84` 的 XADC 函数并非 fpga_base 寄存器（落未分配区读 0），针对的是板上独立 System Monitor IP。关键术语：JTAG-to-AXI Master、regDev、record（longin/stringin/scalcout/seq）、SCAN/FLNK/LNK、SVAL、大端打包与字节倒序、三方/四方重复契约。
