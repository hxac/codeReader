本讲精读 `bd/bd.tcl` 的三个 Block Design 回调与 AXI4 `ID_WIDTH` 跨 IP 自动协商机制。核心是对称结构：master 是 ID 宽度之源，`pre_propagate` 在 BD 传播前把 cell 值向外推到 pin；slave 是汇，`propagate` 在传播后把 pin 值向内拉回 cell；`init` 用 `bd::mark_propagate_only` 声明 `C_S00_AXI_ID_WIDTH` 由传播决定、不可手改，`set_property` 才真正搬运值。触发顺序为 `init` → `pre_propagate` →〔BD 传播〕→ `propagate`。参数名按 `C_<接口名>_<标准参数>` 拼接，与顶层 VHDL generic 逐字符相同，构成 BD hook 驱动 HDL 的命名契约。fpga_base 是 slave-only（仅 `S00_AXI`），故 `init`/`propagate` 生效、`pre_propagate` 空转，master 分支为通用模板的对称性与可复用性而保留，并非冗余。当主机宽度与本 IP 默认值 1 不一致时，`propagate` 用非空守卫把对端协商结果写回 cell，经 `MODELPARAM_VALUE` 流入 generic，重排 `arid/rid/awid/bid` 四端口位宽。诚实观察：`bd.tcl` 未在 `component.xml` 任何 fileSet 或 `package.tcl` 中显式登记，是否在打包产物里真正生效「待本地验证」。
