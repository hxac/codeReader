本讲将 `vivadoIP_fpga_base` 仓库视作「文件柜」，建立目录心智模型，分三模块。一、目录职责按用途分五组：`hdl/` 为 VHDL 电路本体（顶层 `fpga_base_v1_0.vhd`），`component.xml` 为 IP-XACT 元数据入口，`scripts/`、`bd/`、`gui/`、`xgui/` 与根目录 `.tcl` 负责打包构建，`drivers/`、`epics/` 为软件侧接口，`doc/` 及 README 为文档门面。二、区分手写文件（真相之源）与生成文件（产物），引入 fileSet（文件集）、打包（Packaging）概念；PSI 采用「hand-packaged IP」工作流：生成产物 `component.xml` 入库便于分发，而 zip、`package_prj`、`.pyc` 等中间产物被 `.gitignore` 忽略。三、精读 IP-XACT 标准 `component.xml`，涵盖 IP 坐标（psi.ch/PSI/fpga_base/1.4）、端口及 `IMPL_LED` 等可选使能、地址映射（256 字节即 64 个 32 位寄存器）、综合/仿真/xgui/logo/驱动五个文件集及支持器件系列。核心结论：`package.tcl`（输入）与 `component.xml`（输出）互为镜像，理解此对应即把握仓库工程化关键。本讲仅读结构与配置，未涉电路逻辑。
