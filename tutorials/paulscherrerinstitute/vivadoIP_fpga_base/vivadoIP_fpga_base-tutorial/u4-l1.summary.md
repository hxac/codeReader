本讲下沉到命令级，对照输入 package.tcl 与产物 component.xml（IP-XACT 清单），讲清 PsiIpPackage 这层打包 DSL 的三个模块。脚本本身是声明式清单，按加载、基本信息、聚合源码、参数与端口使能、收尾打包的固定顺序调用，自身几乎无控制流。三条 add_relative 命令分别纳入本仓库 HDL、外部 psi_common 库与 C 驱动；相对路径都以脚本目录为锚，框架在产物中统一改写为仓库根锚并减一级，全部 HDL 还编进同一 VHDL 库（logicalName）使顶层 use 语句得以解析，驱动元数据则按 Xilinx 约定自动发现。收尾命令的四参数控制输出目录、不开 GUI、跑综合、目标器件，综合开启方能从 RTL 反推精确端口；打包版本号同时传播至版本字段、库名、显示名、xgui 脚本名与归档标签，而顶层实体名（modelName）源自 entity、属另一独立维度。
