本讲是「源码 → 依赖 → 打包 → 综合产物 → zip」流水线的高层俯瞰，不碰电路逻辑。核心结论是：Python 与 TCL 分工——Python 管 Vivado 外围，TCL 管 Vivado 内部。依赖解析入口 `scripts/dependencies.py` 仅 10 行，是个薄壳，真正解析能力来自须另行安装的外部包 `PsiFpgaLibDependencies`；它把 `README.md` 作为依赖清单的单一真相源，靠两个 HTML 注释标记圈出的「受保护区块」界定机器解析区间，按 `TCL`/`VHDL`/`VivadoIp` 三类声明依赖并克隆到 PSI 约定的兄弟仓库布局。打包主编排脚本 `scripts/package.tcl` 用相对路径 `../../../` 找到这些兄弟库，再调用 PSI 统一打包框架 **PsiIpPackage** 的 DSL 命令（分基本信息、源码聚合、GUI 参数、端口使能、打包收尾五类）完成打包，目标器件 `xc7a200t`（Kintex-7），`package_ip` 的 `Synth=true` 让端口方向/位宽由综合反推落盘。关键对应关系：`package.tcl`（输入声明）与 `component.xml`（产物快照，IP-XACT/IEEE 1685）互为镜像，如 `IP_VERSION 1.4`↔`<spirit:version>1.4>`，`add_sources_relative`↔`xilinx_anylanguagesynthesis_view_fileset`。引入术语：IP 打包、依赖解析、受保护区块、IP-XACT、PsiIpPackage DSL、端口使能条件、用户参数（布尔，裁剪硬件端口）。入库产物仅 `component.xml`，其余 zip、`package_prj`、自动生成 xgui、`*.pyc` 由 `.gitignore` 排除。
