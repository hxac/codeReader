本讲只精读 `fpga_base` 顶层 VHDL 的 `entity` 部分（`architecture` 留给 u2-l2），剖析该 IP 对外暴露的三类接口。一是 AXI4 从机总线：采用完整 AXI4（非 Lite，含 `len/size/burst/last/id`），拆成 AR/R/AW/W/B 五个独立通道，每通道一对 `valid/ready` 握手；站在从机视角，AR/AW/W 的 `valid` 为输入、`ready` 为输出，R/B 反之。二是三个物理端口：`o_led`、`i_sw` 各 8 位，`o_blink` 1 位，分别对应寄存器 24、25 及硬件自由运行的心跳灯（不占寄存器）。三是 6 个泛型（编译期常量，改后需重新综合）：版本组 `C_VERSION` 加两个 16 字符字符串、闪烁组两个频率与信息源开关 `C_USE_INFO_FROM_SCRIPT`、`C_S00_AXI_ID_WIDTH`（决定 `arid/rid/awid/bid` 四个端口位宽）。复位 `s00_axi_aresetn` 遵循 ARM 低有效约定，内部用 `not` 翻成高有效。AXI 地址固定 8 位 → 256 字节寻址空间 → 64 个 32 位寄存器，铺垫下一讲 `C_NUM_REG=64`。
