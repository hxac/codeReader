本讲精读根目录 `fpga_base.tcl`，补完 u3-l1 留下的"FDPE 的 INIT 谁来写"链路。该脚本是一段 `tcl.pre` 钩子，由使用 fpga_base 的消费方工程登记在实现阶段 `opt_design` 之前（即综合之后）——只有此时网表已存在、cell 才可寻址；它并不在打包出的 IP 内（`component.xml` 的 utility 文件集只带 logo）。脚本用 `clock seconds` 取编译机墙上时间，`dec2bin` 把年月日时分转成定宽 32 位、MSB 在前的二进制串；再用 x(31→0)、y(0→31) 双游标循环，保持 `x+y=31`，使 `gen_year[x]` 恰好收到第 x 位（与 VHDL 的 `Q=>year(count)` 一致），由此解释了"串最高位落到 `gen_year[31]`"。写入靠 `set_property -verbose INIT` 回写 FDPE 上电初值，依赖可预测的层次路径 `*/fpga_base_inst/U0/fpga_base_date_inst/gen_year[x].year_dfpe_inst` 与 `dont_touch` 保住的 cell。关键结论：这条"综合后写 INIT"路径与 u3-l3 将讲的 `update_version.py`"综合前改 HDL 写 generic"路径互斥，generic 模式下本脚本无需执行。引入的核心术语：tcl.pre 钩子、INIT 属性回写、定宽 MSB 在前二进制、双游标位对齐、层次路径寻址、双注入路径互斥。
