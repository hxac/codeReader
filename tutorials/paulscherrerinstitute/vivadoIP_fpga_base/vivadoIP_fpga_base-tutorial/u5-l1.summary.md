本讲进入 fpga_base 软件侧，精读 `fpga_base.h`/`fpga_base.c` 两个驱动源文件，拆为三个模块。其一，裸机（bare-metal）程序经内存映射 IO 访问外设：`Xil_Out32`/`Xil_In32`/`Xil_Out8`/`Xil_In8` 本质是带 `volatile` 的指针解引用，一次调用即对应一次 AXI 事务，基地址 `base` 由 `xparameters.h` 的 `XPAR_FPGA_BASE_0_BASEADDR` 提供（详见 u5-l2）。其二，驱动用预处理器三目运算链加字符算术（`c-'0'`），在编译期把 `__DATE__`/`__TIME__` 翻译成 `DATE_YEAR`/`DATE_MONTH` 等整数常量，运行期零开销、不占 RAM；`DATE_MONTH` 须比较到第三个字符才能区分 `Jun`/`Jul`、`Mar`/`May` 等前缀冲突。其三，`fpga_base_version()` 在上电时把软件编译时间写入 `0x18`–`0x28` 五个寄存器，因其 HDL 接成「直通回显」（`reg_rdata <= reg_wdata`）故可读可写，区别于 `0x04`–`0x14` 的固件日期寄存器（FDPE INIT 烧定，只读）；`fpga_base_print()` 汇总版本号与固件/软件两组日期、项目/设施串一并打印，字符串须按字节倒序还原（HDL 大端打包的对称处理）。借此一上电即可对照软硬件编译时间，判断 bitstream 与 elf 是否同次发布配套。
