本讲讲 fpga_base 的脚本化构建信息路径，含三个最小模块。其一，占位符替换：`fpga_base_scripted_info_pkg.vhd` 的 6 个常量带 `$$tag$$` 注释，构成可被脚本注入的合法 VHDL 模板；`update_version.py` 以「tag 定位行、正则改值」两层解耦实现可重复的就地替换。其二，gitpython 读取：从消费方工程仓库取 8 位 git 哈希写入版本寄存器实现可追溯；脏仓库经确认后哈希改为哨兵值 `FFFFFFFF`；再用 `assume-unchanged` 把就地改写的 HDL 对 git 隐藏，保证库内始终是干净的占位模板。其三，双信息源切换：顶层泛型 `C_USE_INFO_FROM_SCRIPT` 作单一总闸，同时切换版本号寄存器与日期组件来源，强制两条链路同源一致。该脚本化 Python 路径与 u3-l2 传统 TCL 综合钩子路径互斥：前者综合前改源码常量并提供哈希加日期，后者综合后改 FDPE 网表只管日期。
