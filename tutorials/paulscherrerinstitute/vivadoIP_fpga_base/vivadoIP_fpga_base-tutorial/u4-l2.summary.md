本讲承接 u4-l1，聚焦 fpga_base 的 GUI 参数与可选端口。引入两类参数：普通参数（gui_create_parameter，对应真实 VHDL generic，会被综合进电路）与用户参数（gui_create_user_parameter，如 IMPL_BLINK/LED/SWITCH，仅存于 GUI、不进 HDL）。判别铁证是自动生成的 xgui 回调脚本——只有真实 generic 才配 update_MODELPARAM_VALUE，把 PARAM_VALUE 桥接到 MODELPARAM_VALUE；另有 init_gui 摆控件、update_PARAM_VALUE 与 validate_PARAM_VALUE 做联动和校验。端口可选由打包级 add_port_enablement_condition 声明，落盘为 component.xml 中 dependency 节点，Vivado 求值决定端口是否暴露，HDL 内无 if generate 守卫，物理裁剪交给消费方综合器。核心结论：裁物理端口用打包级用户参数，裁内部寄存器逻辑必须走 RTL 级 generic 条件生成。
