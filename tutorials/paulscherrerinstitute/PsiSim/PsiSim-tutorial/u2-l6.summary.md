本讲讲解 `run_tb` 如何消费 `TbRuns`、逐个执行每个 run。过滤层沿用 `compile` 套路：库、名字、包含子串三维度各配哨兵值（`All-Libraries`/`All-Names`/`All-regex`），靠 `continue` 串成 AND；`-contains` 实为 `string first` 子串匹配、作用于 `TB_NAME`，非正则。`run_tb` 的 `-all` 同时重置库与名字两维（`compile` 只重置库），防止残留过滤器缩小「全部」范围。执行顺序为过滤通过→打印横幅→skip 检查→pre-script→逐组参数调 `sal_run_tb`→post-script；skip 用 `lsearch $skip $Simulator` 或 `=="all"` 判定，`lsearch` 大小写敏感、`SKIP` 须精确拼写，命中则连 pre/post 与全部仿真一并跳过。pre/post 脚本各只跑一次，把整个 run 的所有仿真整体包起来（两处 `sal_exec_script` 均在 `foreach tbArgs` 循环之外）。仿真次数等于 `TB_ARGS` 长度，默认 `[list ""]` 保证至少跑一次。`run_tb` 以参数形式把 `RunSuppress` push 给 `sal_run_tb`，且不传第 6 个参数 `wave`（回归测试不要波形），这正是与 `launch_tb` 的分水岭。本讲把 `sal_run_tb` 当作启动一次仿真的黑盒，其具体命令翻译留待单元 3。
