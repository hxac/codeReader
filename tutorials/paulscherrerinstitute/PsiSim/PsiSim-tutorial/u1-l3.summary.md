本讲是单元 1 收尾，首次动手跑通一次完整回归测试，核心是 PsiSim 两文件工作流。`config.tcl`（描述）只把库/源/测试运行登记进状态变量（`Libraries`/`Sources`/`TbRuns` 及两个 Suppress 串），不含 `init` 与任何运行命令；`run.tcl`（执行）负责加载框架并跑流程。黄金七步序列为：`source PsiSim.tcl` → `namespace import psi::sim::*` → `init` → `source config.tcl` → `compile_files -all -clean` → `run_tb -all` → `run_check_errors`，归为准备/配置/执行三阶段。关键概念：`init` 是分水岭，它选仿真器并把状态清零，必须是第一个 PsiSim 命令且早于读配置；`config.tcl` 的命令（`add_library`、带 `-tag` 的 `add_sources`、`create_tb_run`/`tb_run_add_arguments`/`add_tb_run` 三段式定义测试运行）只改状态变量，执行命令再消费之——`compile_files` 读 `Sources`、`run_tb` 读 `TbRuns`、`run_check_errors` 读 transcript。两文件拆分的最大回报是嵌套：库级有独立 `config.tcl`/`run.tcl` 可单独跑，项目层 `run.tcl` 只 `init` 一次再连续 `source` 各库 `config.tcl` 汇总回归；铁律为 `config.tcl` 不调 `init`、`init` 在每份被执行的 `run.tcl` 里仅出现一次。多库只需多次 `add_library`（追加并切换 `CurrentLib`）；纯文本行级语义使 `config.tcl` 对 Git 友好，区别于 Modelsim `.mpf`。
