本讲把 CommandRef.md 当作一张「命令地图」来读，只读 CommandRef.md 与 PsiSim.tcl 两个文件，不涉及运行行为。核心结论：PsiSim 暴露的 17 个导出命令被文档分为三大类——General（仅 init，选仿真器并清零，必须最先调用）、Configuration（11 条，登记库/源/测试台/告警，只改状态变量）、Run（5 条，真正执行清理、编译、仿真、检查、调试），满足 1+11+5=17，对应回归测试「准备→配置→执行」的生命周期。本讲引入的关键术语包括：psi::sim 命名空间与 namespace export 导出机制、每条命令对应一个 proc 定义行与 export 行、SAL 抽象层（sal_* 前缀的内部过程）。两条边界规则需承接：一是文档只覆盖被 export 的命令，proc compile 因与 Modelsim 自带命令重名而不导出，对外只暴露包装函数 compile_files（内部 eval 转调 compile）；clean_transcript 是 Internal Function，亦不在文档。二是命令详解统一采用 Usage/Description/Parameters 三段式，「必须夹在 create_tb_run 与 add_tb_run 之间」是两段式编程模型的信号，由源码共享的 ThisTbRun 半成品字典落地。本讲产出一张「命令→行号」映射表，并确立「文档驱动源码阅读」的导航方法：先看目录建全景，再按映射表跳到单个 proc 按需阅读，为单元 2（配置命令内部实现）和单元 3（运行命令经 SAL 调用仿真器）铺垫。
