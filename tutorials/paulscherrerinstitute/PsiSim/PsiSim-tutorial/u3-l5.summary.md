本讲是单元 3（SAL 模拟器抽象层）的收尾篇，专门讲解 PsiSim 的**交互调试入口 `launch_tb`**。它与回归用的 `run_tb` 有三处刻意不同：默认只加载设计而不自动 `run`、跳过前后脚本、命中第一个 `-contains` 匹配即 `return`（人无法同时盯多个波形窗口）。核心参数为 `-contains`（必填，子串匹配选 run）、`-argidx`（选 `TB_ARGS` 第几组泛型，越界即报错；默认长度为 1，故未定义多组泛型时 `-argidx 1` 会触发 `out of range`）、`-wave`（三态：空→`all` 全信号模式、紧跟 `-show`→全信号并开 GTKWave、给文件名→`.do` 波形视图）、`-show`（仅 GHDL，触发 GTKWave 打开）。Vivado 在接口层即被白名单拒绝，无调试路径。

两条调试路径分叉源于仿真器本质差异：**Modelsim 走专用 `sal_launch_tb`**，用 `vsim` 加载设计（仿真时间停 0）、按 `wave` 三态决定是否 `add wave -r /*; run -all; wave zoom full` 或 `do xxx.do`，窗口常驻真正可交互；**GHDL 复用 `sal_run_tb`**（批处理无法停等），跑到结束并把波形落成 `<name>_<argidx>.ghw`（代码实为 `.ghw` 能正确表达 record 等复合类型，CommandRef 写的 `.vcd` 是过时文档），再由 `sal_open_wave` 后台 `gtkwave -f $wave &`（末尾 `&` 不阻塞）打开。GHDL 迭代调试的支点是**文件名稳定**：同 `-argidx` 反复覆盖同一 `.ghw`，配合 GTKWave `File → Reload Waveform` 即可不重开窗口刷新波形；改 `-argidx` 则生成不同文件、需另开窗口对比。
