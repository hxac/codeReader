本讲为单元 4 首篇，把单元 2 的数据模型与过滤逻辑组装成真实工程结构，含三个核心。一、库级与项目级嵌套：凭借 PsiSim 的 namespace 状态（Libraries/Sources/TbRuns/CurrentLib）跨 source 持久、config.tcl 为纯声明，项目 run.tcl 只 init 一次再依次 source 各库 config，即可把各库声明累积成一张大表；铁律是 init 须在 source 任何 config 之前且全工程仅一次，否则会抹掉已累积的声明。二、多库与 -tag：-lib 是物理维度（决定文件编译进哪个库），-tag 是逻辑维度（仅影响 compile 是否选中），二者正交；compile 用三个哨兵加三个 continue 实现「库∧tag∧路径包含」的 AND 过滤，统一 tag 词汇方能跨库切片。三、-contains 局部迭代：编译期匹配源文件绝对路径 PATH、运行期匹配 TB_NAME，同为 string first 子串但数据源不同，须靠命名约定对齐；-clean 缺省传 All-Libraries 会清空全部库，故局部迭代不应带 -clean，仅与 -all 联用。本讲确立的关键概念：跨 source 持久状态、CurrentLib 隐式游标、纯声明 config、哨兵 AND 过滤、-lib/-tag 正交、-contains 不对称、-clean 全局副作用。
