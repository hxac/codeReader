本讲是 PsiSim 学习手册首篇，仅读 `README.md`、`Changelog.md`、`License.txt` 三个文档，不深入 TCL 源码，目标是在读码前建立项目认知。PsiSim 是保罗谢尔研究所（PSI）用 TCL 编写的 VHDL 回归测试框架，核心卖点是：用纯文本脚本描述与运行仿真，对版本控制友好、易于合并，克服了 Modelsim 工程（`.mpf`）与 Vivado 工程难以用 Git 管理的问题。典型用法是 `config.tcl`（描述文件，声明库、源文件、测试台、generics）+ `run.tcl`（执行文件，加载框架→初始化→source 配置→编译→跑测试）的两文件模式，拆分是为了支持库级与项目级嵌套回归。PsiSim 支持三种仿真器，通过 `init` 命令切换：默认 Modelsim（商业，支持最完整）、`init -ghdl` 用 GHDL（开源，快，VHDL-2008 支持好，需独立 TCL 解释器与 PATH）、`init -vivado` 用 Vivado Simulator（商业，慢，VHDL-2008 支持差，仅作备选），选择记入 `Simulator` 状态变量。许可证为 PSI HDL Library License（LGPL 第 2 版或更新 + 固件例外，允许把库综合进 FPGA 比特流后闭源分发）；版本号遵循 `major.minor.bugfix`（破坏兼容性→major，新增功能→minor，仅修 bug→bugfix），当前版本 2.5.0。核心源码全部集中在一个 966 行的 `PsiSim.tcl`，功能封装在 `psi::sim` 命名空间下，后续讲义将反复回到此文件。关键术语：VHDL、Testbench、回归测试、SAL（模拟器抽象层，后续单元 3 展开）、VHDL-2008、独立 TCL 解释器。
