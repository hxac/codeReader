本讲承接 u3-l1 的 SAL 骨架，深入七个工具型 `sal_*` proc 的分支内部，覆盖三大模块：日志与 transcript 抽象、Modelsim 版本探测、版本相关 flag。核心结论：PsiSim 用统一的 `./Transcript.transcript` 文件，把 Modelsim 自动记录与 GHDL/Vivado 手动追加两种日志来源收拢成一条供 `run_check_errors` 正则 grep 判错的链路。关键术语与概念：`sal_print_log` 对 GHDL/Vivado 走「控制台+文件」双写、Modelsim 用内建 `echo`；`sal_init_simulator` 用「文件中转」（`vcom -version` 重定向临时文件+`after 500` 等盘+正则抠版本号写入 `SimulatorVersion`，GHDL/Vivado 仅赋占位串）；`TranscriptFile` 跨 if 分支赋值；`sal_clean_transcript` 借 `batch_mode` 与 Dummy 中转释放 Modelsim 句柄；`sal_version_specific_flags` 仅 Modelsim 且版本<10.7 返回 `-novopt`，供 `sal_compile_file` 消费。整体取向是把仿真器/版本相关脏活收拢在 SAL，使上层 compile/run_tb/run_check_errors 保持仿真器无关，为 u3-l3 编译、u3-l4 运行抽象铺好水电管道。
