本讲义建立 PsiSim 仓库的「架构地图」，承接 u1-l1 对框架的整体认知。仓库扁平化：仅 6 个被 Git 跟踪文件、无子目录——唯一源码 `PsiSim.tcl`（966 行）、三个文档（`README.md`、`CommandRef.md`、`Changelog.md`）、两个许可证文件。全部实现包裹在 `namespace eval psi::sim { ... }` 中。源码内部分三大区块：区块一是 Namespace Variables（状态变量，10 个，如 `Libraries`、`Sources`、`TbRuns`、`Simulator`，仅声明不赋值，由 `init` 初始化）；区块二是 SAL 模拟器抽象层（13 个 `sal_` 前缀内部过程，不导出，屏蔽 Modelsim/GHDL/Vivado 差异）；区块三是 Interface Functions（17 个导出命令构成公开 API）。调用链为用户脚本 → 接口函数 → SAL → 实际仿真器，各层共享状态变量。17 个命令按时间线分三类：General（`init`）、Configuration（建库、加源、定义测试运行、消息抑制）、Run（清理、编译、仿真、检查、调试）。关键术语：接口函数、抽象层 SAL、状态变量；核心结论之一是 `compile` 因与 Modelsim 自带命令重名而刻意不导出，对外仅暴露包装 `compile_files`。
