本讲是 u4 的「实现深潜」，逐行打开 `Generate` 里并发语句段的六个写作方法（`_DutInstantiation` → `_TbControl` → `_Clocks` → `_Resets` → `_Processes`，外加声明区 `_TbControlSignals`），把「标签」与「最终 VHDL」之间的实现补齐。其统一骨架是 `FilterForTag` 原语（`value=None` 只判存在、带值则大小写不敏感地归一成列表匹配），五个方法开头都靠它筛端口/generic。

核心结论：`_Clocks` 把 `TYPE=CLK` 配 `FREQ` 的端口变成自运行进程 `p_clock_<name>`，半周期公式 T_half=0.5/f，缺 `FREQ` 报错；`_Resets` 用 `CLK` 标签（值为另一端口名）把复位归属到时钟域，等两个上升沿再释放到无效值，缺 `CLK` 报错；`_DutInstantiation` 的 `generic map` 收 `EXPORT=true` 与 `CONSTANT` 两类 generic、`port map` 全端口同名相接，靠 `RemoveFromLastLine` 去尾逗号；`_Processes` 单用例走「等复位释放 + 用户占位 + 置完成位」，多用例按 `NextCase` 调度各 case procedure（`PROC` 标签仅多用例经 `GetPortsForProcess` 决定参数）；`_TbControlSignals` 声明 `TbRunning`/`NextCase`/`ProcessDone`/`AllProcessesDone_c`/`TbProcNr_<p>_c` 五类脚手架，位宽由 `tbProcesses` 决定；`_TbControl` 通过 `wait until ProcessDone = AllProcessesDone_c` 汇合后置 `TbRunning <= false`，使时钟进程跳出 `while TbRunning loop` 结束仿真。关键术语 `NextCase` 与 procedure 签名细节留待 u5 承接。
