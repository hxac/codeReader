本讲覆盖四个「文件级标签」（PROCESSES、TESTCASES、DUTLIB、TBPKG）的收集与建模；它们写在独立注释行、描述整个 testbench 骨架，区别于 u2-l2 讲的端口/generic 标签。核心结论：DutInfo.__init__ 遍历 VhdlFile.commentLines，逐行调 _ParseTags，用 update 合并成 fileScopeTags 字典（多块按 key 合并、同名后写覆盖先写、列表须单行逗号分隔、单值返回 str 而非 list）。dutLibrary 是 DUTLIB 的带默认值 "work" 的视图，被 LibraryDeclarations 与 DUT 实例化行共同消费，只替换 work 字样而不新增 library 声明。TbInfo.__init__ 把原始标签翻译成生成参数：tbName=实体名+"_tb"；tbProcesses 源自 PROCESSES、缺省 ["Stimuli"]；isMultiCaseTb 只判 TESTCASES 键是否存在（是模式开关而非数量判断）；testCases 与 tbProcesses 均做 str→list 归一化。TBPKG 按「库.包」拆分、按库聚合成 tbUserPackages 字典，经 UserPkgDelcaration（源码原拼写）展开为 library/use 注入 testbench 顶部。多用例模式生成 2+用例数个文件（主 TB + TB 包 + 每用例一个 case 包），单用例仅 1 个主 TB；多用例内部 procedure 生成细节留待 u5 展开。
