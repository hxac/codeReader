本讲(u5-l1)讲多用例 TB 的触发与生成流程，核心四点。一，模式开关 `isMultiCaseTb`：`TbInfo` 初始化时用「`fileScopeTags` 是否含 `testcases` 键」判定，只看键是否存在、与用例个数无关，单用例也触发整套机制；取出的 `testCases` 经 str→list 归一供下游遍历。二，多用例产物为 `2+N` 个文件（1 主 TB + 1 TB 包 + N 个 case 包），靠两个分流点：库声明区用 `TbPkgDeclaration`/`TbCaseDeclaration` 多 `use` TB 包与各 case 包；末尾调 `WriteTbPkg` 并循环 `WriteCasePkg` 生成包文件。三，`_Processes` 多用例分支把单用例占位替换为内层遍历用例的循环，每用例四步——等 `NextCase=i`、清完成位、调 case 包过程（实参来自 `GetPortsForProcess`、尾加 `Generics_c`）、置完成位。四，`_TbControl` 多用例分支是总调度，循环「`NextCase<=i` + 等所有进程完成」逐用例推进，与单用例唯一差异即此循环。关键术语：`NextCase`（调度器写/进程读，初值 -1 充当未启动哨兵）与 `ProcessDone`/`AllProcessesDone_c`（进程写各自比特、调度器读全向量）构成双向握手的屏障同步。补充：导出 generic 被打包成记录常量 `Generics_c : Generics_t`（`_GenericConstants` 多用例分支），类型定义在 TB 包，故主 TB 必须 `use` TB 包；case 包的 procedure 签名与方向推断留待 u5-l2。
