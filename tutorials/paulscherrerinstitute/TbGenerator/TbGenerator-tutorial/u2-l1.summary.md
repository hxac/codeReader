本讲是「VHDL 注解标签系统」单元第一讲，只讲标签是什么、怎么解析、怎么查询，具体标签如何影响生成留待 u2-l2。覆盖三个最小模块：标签语法、`_ParseTags` 解析器、标签工具方法。

核心结论：`$$ 键=值; 键=值 $$` 注解标签是整个工具的输入契约，写在 VHDL 注释里，由 `Tags` 常量类集中声明所有合法标签名，按作用对象分 Generic、Port、File-scope 三类。值分单值（如 `FREQ=100e6`）与列表（如 `PROCESSES=Input,Output`）两种形态；标签名大小写不敏感，值保留原始大小写。

关键术语：`VhdlFile` 把行尾注释挂到端口/generic 的 `.comment`、独立注释行收集进 `commentLines`；`DutInfo` 的类方法 `_ParseTags` 用 pyparsing（`CharsNotIn`、`Word(alphas)`、`scanString` 等）把注释翻译成 Python 字典，键统一小写，单值留字符串、列表切分成 list，多块汇总成 `fileScopeTags`。隐含约束：标签名只能由字母构成，含数字会被静默忽略。

更高阶的工具方法 `HasTag`/`GetTag`/`GetTagAsList`/`HastTagValue`/`FilterForTag` 封装存在性判定、取值、归一列表、值匹配、批量筛选，值匹配默认大小写不敏感；生成器（`_Clocks`/`_Resets`/`_DutInstantiation`）以「筛选→检查→取值」三连招消费它们。本讲确立了「解析层与使用层分离」的设计原则。
