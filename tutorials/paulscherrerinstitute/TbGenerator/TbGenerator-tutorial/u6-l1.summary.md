本讲逐行拆解 GUI 入口 TbGenGui.pyw 的四个最小模块：TbGenGui(QDialog) 类骨架、LoadSrc/LoadDst、Generate 槽、QErrorMessage。核心是「控件+布局+信号槽」三件套——__init__ 用 QVBoxLayout 嵌套 QHBoxLayout 放复选框，三按钮 connect 到对应槽，控件是否存 self 取决于槽函数是否需读写它。LoadSrc/LoadDst 借 QFileDialog 弹标准对话框，用 self.lastDirectory 做目录记忆并 +"/.." 跳上一级（DUT 源文件与 testbench 分放兄弟目录），靠判空挡住用户取消。Generate 槽是一个「适配层」：从控件读参数喂给同一台引擎 TbGenerator.ReadHdl + Generate，与 CLI 几乎逐行同构——Merge 复选框映射为 ext=".mrg" + overwrite=True，GUI 无 -force 对应物。整体用 try/except Exception 配 QErrorMessage 把任意异常转弹窗，保证出错不杀进程，这是 GUI 错误处理区别于 CLI（print + exit）的根则。综合实践新增 Force Overwrite 复选框，把 overwrite 从 Merge 解耦，贯穿「控件→布局→槽→引擎参数」三段式扩展。
