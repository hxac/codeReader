本讲（u2-l2）从语法层进入语义层，逐一拆解具体端口/generic 标签如何驱动生成器方法。核心围绕五个最小模块：`GetPortValue`、`_Clocks`、`_Resets`、`_GenericConstants`、`_DutSignals`。

`GetPortValue(port, active)` 是初始值的单一真相源，结合 `LOWACTIVE` 标签把「有效/无效」的逻辑意图翻译成 VHDL 字面量：高有效时有效=`'1'`、无效=`'0'`，低有效则成对翻转，向量包装为 `(others => ...)`，未知类型抛 `UnknownVhdlType`；它被 `_DutSignals`、`_Resets`、`_Processes`、`_TbControl` 共同复用，改一处标签即多处一致翻转。

`TYPE=CLK` 配 `FREQ` 驱动 `_Clocks` 生成时钟进程，半周期为 0.5/f；`TYPE=RST` 配 `CLK`（引用其它端口）驱动 `_Resets`，复位先保持有效，等归属时钟两个上升沿后释放到无效，保证至少被采样一次。`_DutSignals` 汇聚两条主线：时钟/复位信号取「有效」初值（对齐上升沿），普通信号取「无效」，陌生类型降级为不给初值。

generic 三分类：`EXPORT=true` 导出（进 TB 实体 generic 与 `generic map`，仅认字符串 `"true"`）、`CONSTANT=值` 固定（用标签值，进 `generic map`）、两者皆无则用 DUT 默认值作内部常量（不进 `generic map`），由 `_GenericConstants`/`_EntityDeclaration`/`_DutInstantiation` 协同实现。`PROC`（端口级、单数）经 `GetPortsForProcess` 把端口绑定到测试过程，但仅在多用例 TB 中决定 procedure 参数列表，单用例 TB 里沉睡不动；须区别于文件级 `PROCESSES`（复数），后者及多用例开关留待 u2-l3。
