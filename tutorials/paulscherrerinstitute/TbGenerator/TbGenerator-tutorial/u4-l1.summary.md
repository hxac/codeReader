本讲聚焦 TbGenerator 的数据模型层，覆盖 DutInfo 与 TbInfo 两个核心模块。DutInfo 在 VhdlFile 之上做三件再加工：提取实体名为 name、按库把 use 语句归类成 libraries 字典、汇总文件级标签成 fileScopeTags 字典；generics/ports 通过 @property 转发保持单一数据源。dutLibrary 是 DUTLIB 标签的带默认值 "work" 视图，LibraryDeclarations 据此写出 testbench 第一个实质段落并对 work 做子串替换。GetPortValue(port, active) 是端口初值的单一真相源，由 LOWACTIVE 标签决定极性、端口类型决定位/向量包装，未知类型抛 UnknownVhdlType。TbInfo 吃一个 DutInfo，翻译出 tbName、tbProcesses（缺省 ["Stimuli"]）、isMultiCaseTb（仅判 TESTCASES 键存在）、tbUserPackages（按"库.包"聚合），并反向持有 dutInfo。三个包声明方法分工明确：UserPkgDelcaration 写用户包、TbPkgDeclaration 供 case 文件引用本 TB 包、TbCaseDeclaration 仅多用例场景供主 TB 引用各 case 包。数据流主线为 VHDL → VhdlFile → DutInfo → TbInfo → 生成器，为后续 u4-l2 的 Generate 主流程提供全部被消费数据。
