本讲承接 u5-l1，拆解多用例 TB 中被主 TB `use` 的两个包文件（`<tbName>_pkg.vhd` 与 `<tbName>_case_<case>.vhd`）的生成细节，共四个最小模块。`GetPortsForProcess`（`TbInfo.py`）是 `FilterForTag(ports, PROC, process)` 的薄封装，按端口级 `PROC` 标签（逗号列表、大小写不敏感）筛出某过程的端口，无 `PROC` 标签的端口不进任何过程。`PortDirectionForProcedure`（`MultiFileTb.py`）用三条规则定参数方向：DUT 输出恒为 `in`；DUT 输入在非 owner 过程里为 `in`；owner 过程（`PROC` 首个名字）里若非 `TYPE=CLK` 则 `inout`、否则 `in`。`WriteTbPkg` 把 `EXPORT=true` 的 generic 打包进 `Generics_t` 记录（空记录塞 `Dummy` 占位），其余 generic 作常量，值取 `CONSTANT` 标签优先、否则取 DUT 默认。`WriteCasePkg` 为每个用例的每个过程写 `procedure` 声明（参数为过程端口加 `constant Generics_c : Generics_t`，类型用 `type.name` 不带范围）与空实现 body（仅一条 `assert false ... warning` 占位，激励留给用户手填）。关键依赖：case 包必须 `use` TB 包以获得 `Generics_t`；主 TB 在 `isMultiCaseTb` 时按 `WriteTbPkg` → 循环 `WriteCasePkg` 顺序生成 `1 + N` 个包文件。引入的关键术语供后续承接：`Generics_t`/`Generics_c`、`PROC` owner、procedure 的 unconstrained 数组参数、`TbPkgDeclaration`。
