本讲是《TbGenerator 学习手册》起点，定位为「读懂文档、理清定位」的导读，覆盖三个最小模块：README（依赖与目录结构）、Changelog（版本史）、License（许可说明）。

核心结论：TbGenerator 是用 Python 编写的 testbench 骨架自动生成器，输入为带 `$$ ... $$` 注解的 VHDL DUT 文件，输出两种形态——单文件 TB（一个 .vhd、无独立用例）与多文件 TB（声明多个 testcase 时每用例一个 package 文件）。引入的关键术语包括 DUT（被测设计）、Testbench（测试台）、VHDL；三类依赖为 Python 库 PsiPyUtils（≥3.0.0，提供文件写入等基础能力）、外部 pip 包 PyQt5（支撑 GUI）与 pyparsing（VHDL 文法解析）。版本演进有两个分水岭：2.0.0（首个开源版本）与 3.0.0（依赖 PsiPyUtils 3.0.0、不向后兼容），当前稳定版 3.0.4 完成 PyQt5 迁移，配套语义化版本标记策略（major.minor.bugfix）。仓库采用扁平布局，入口为 `TbGen.py`（CLI）与 `TbGenGui.pyw`（GUI）。许可为 PSI HDL Library License，本质是 LGPL 2.1 加针对 FPGA/硬件场景的例外条款，允许生成物用于闭源硬件（含位流、flash 镜像）分发，但对工具源码本身的修改与分发仍受 LGPL 约束。后续讲义将转入逐模块读代码，从入口文件与 `TbGenerator` 主类开始。
