本讲绘制 TbGenerator 仓库的「地图」。仓库采用扁平布局，7 个 Python 模块平铺在根目录：核心引擎 `TbGen.py`、GUI 入口 `TbGenGui.pyw`，以及 `DutInfo`（DUT 数据模型）、`TbInfo`（testbench 模型）、`VhdlParse`（基于 pyparsing 的 VHDL 解析）、`MultiFileTb`、`UtilFunc` 等模块。工具分 CLI 与 GUI 两条入口，但最终都汇入同一个有状态类 `TbGenerator`，其对外接口只有两步：`ReadHdl(filePath)` 解析 VHDL 并填充 `self.dutInfo`/`self.tbInfo`，`Generate(tbPath, extension, overwrite)` 按固定段落顺序写出 testbench，且必须先读后写。CLI 用 `argparse` 声明 `-src`/`-dst`/`-clear`/`-mrg`/`-force` 五个参数，全包在 `if __name__ == '__main__':` 守卫内，使文件既可运行也可被导入；GUI 基于 PyQt5 的 `QDialog` 与信号槽机制，将 Merge/Clear 复选框映射到与 CLI 一致的引擎参数。入口文件顶部的 `sys.path` 自举逻辑用于在官方目录结构下定位兄弟库 `PsiPyUtils`。关键结论：外壳（CLI/GUI）与引擎（`TbGenerator`）分离，业务逻辑只有一份。
