本讲引入定点格式三元组 `[S, I, F]`：`S`(Signed) 决定是否有符号位及二进制补码解释，`I`(IntBits) 与 `F`(FracBits) 决定二进制小数点左右各有多少位。三元组唯一决定位串如何解释为实数。核心公式为总位宽 `W = S + I + F`，数值 `V = N × 2^(-F)`，由此推出有符号范围 `-2^I … 2^I - 2^(-F)`、无符号范围 `0 … 2^I - 2^(-F)`。一个重要特性是 `I`、`F` 均可为负：负 `F` 使数值成为某 2 的幂的倍数(粗粒度)，负 `I` 把范围压缩到 0 附近(小数值)，二者都会减小位宽，公式对正零负位一律成立。同一概念在三套实现中语义完全一致但语法外壳不同：VHDL 用 `record FixFormat_t`、Python 用 `FixFormat` 类、MATLAB 用 `cl_fix_format` 函数返回结构体——这是 u1-l1 所述位真一致性的前提。`width()`/`cl_fix_width` 三语言逐字符对应。硬约束 `IntBits + FracBits ≥ 1` 用于避免退化格式：MATLAB 在构造期 `error`、VHDL 在 `cl_fix_width` 内 `assert severity failure` 检查，而 **Python 不检查**，会静默返回退化位宽，是需留意的跨语言差异。本讲只讲格式本身，尚未涉及数值装载、舍入(u1-l4)、饱和(u1-l5)与转换(u3)。
