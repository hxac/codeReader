本讲横向跳到 VHDL 测试基础设施，覆盖四个最小模块：驱动脚本 sim.tcl（清理 transcript → 用 vcom -2008 先编译被测包、再编译 testbench → vsim 加 run -all 运行 → 退出仿真）；Check* 断言过程族（CheckStdlv/CheckInt/CheckReal/CheckStdl/CheckBoolean），分别比对位串、整数、实数、单比特、布尔，统一用 ###ERROR### 前缀加 severity error 报告；为不支持 VHDL-2008 的仿真器（如 xsim）准备的本地 to_string 兼容函数，靠 std_logic'image 逐位提取字符；唯一测试进程 p_control，采用 print 分节标题加一组 Check* 调用的顺序结构，末尾用 wait; 永久挂起。核心结论：成败判定不靠返回码，而靠扫描 transcript 文本，用 regexp -nocase 找 ###ERROR### 与 Fatal: 两个关键字，任一命中即失败，全不命中才算成功。两个关键字分工明确——###ERROR### 由 Check* 产生（severity error 不中止仿真，故一次 run -all 能收集全部失败），Fatal: 由仿真器产生（如被测代码守约束的 severity failure）；CheckReal 另用 10⁻¹² 容差比较浮点以避开严格相等的脆弱性。本讲只讲测试外壳，不涉及定点函数算法本身。
