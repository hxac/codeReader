本讲讲解 en_cl_fix 库的饱和模式 FixSaturate。定点格式 [S,I,F] 表示范围有限，运算结果越界时需决定如何处理溢出。FixSaturate 由两个独立开关——「是否饱和（clip，夹紧到边界）」与「是否告警（warn）」——组合出四种模式：None_s、Warn_s、Sat_s、SatWarn_s；名字带 Sat 的会夹紧，带 Warn 的会告警。

饱和把越界值夹紧到 [min_value, max_value]；回绕用取模（无符号 N mod 2^W，有符号需平移）丢弃高位，可能造成符号反转，近零成本而饱和代价更高、更安全。VHDL、Python、MATLAB 三语言共享 0–3 整数编码，是位真一致性前提；MATLAB 必须先运行 cl_fix_constants 建立 Sat.* 常量。

cl_fix_from_real 默认 SatWarn_s：先 half-up 量化（不受 FixRound 控制），再按模式告警/夹紧，None_s、Warn_s 返回越界值而不夹紧；但 VHDL 版无条件夹紧、不引用 saturate 参数也不告警，与严格按模式分支的 Python 实现不一致。cl_fix_resize 默认模式跨语言不同（Python 为 None_s、VHDL 为 Warn_s），跨语言调用应显式传 sat。告警机制：Python 用 warnings.warn，VHDL 用 assert ... severity warning（并用 translate_off 排除综合）。
