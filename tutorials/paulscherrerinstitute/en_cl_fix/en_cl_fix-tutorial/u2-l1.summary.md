本讲义聚焦 en_cl_fix 的 Python 实现的**工程骨架与测试方法学**，不涉及具体算法细节。最小模块覆盖四块：包结构、numpy 向量化、仿真辅助函数、unittest 测试组织。

Python 端把职责拆成三个子模块——`en_cl_fix_types`（`FixFormat`、`FixRound`、`FixSaturate` 等类型定义，其中 `FixFormat.width()` 直接实现位宽公式 W=S+I+F）、`en_cl_fix_pkg`（主体函数库）、`wide_fxp`（>53 位大位宽实现，详解见 Unit 6）——放进同名包目录，由仅三行的 `__init__.py` 用星号导入构成**统一门面（facade）**，外部一行 `from en_cl_fix_pkg import *` 即可拿到全部名字。核心结论：Python 实现依赖 **numpy**，几乎所有函数按 **ndarray 向量化**设计，用 `np.floor`、`np.where`、`np.max/min` 一次性处理整段数据，而非逐元素循环。

引入的关键术语包括：**门面模式**、**向量化（vectorization）**、两个 **Python 独有、VHDL 不存在**的仿真辅助函数 `cl_fix_random`（在整数网格坐标上 randint 取样，生成均匀覆盖某格式动态范围的随机定点数）与 `cl_fix_write_formats`（把格式清单写成文本，配合 Unit 5 数据写函数构成跨语言位真交换链路）。测试侧：唯一文件 `en_cl_fix_pkg_test.py` 采用**每个被测函数一个 `unittest.TestCase` 子类**的组织，靠 `sys.path.append("../src")` 就地导入，必须在 `python/unittest` 目录运行；星号导入会隐式把 `numpy`（`np`）再导出到测试文件。本讲为后续 u2-l2/u2-l3 三语言测试横向对照、Unit 3 核心函数算法、Unit 6 wide 实现铺垫了骨架。
