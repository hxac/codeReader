本讲讲解 `spi_simple` 内部的两个 `psi_common_sync_fifo`：命令 FIFO（实例 `i_tx_fifo`，缓冲待执行 SPI 命令字）与响应 FIFO（实例 `i_resp_fifo`，缓冲 MISO 读回数据），用于解耦快速 AXI 总线与慢速 SPI 引擎。背压分两层：引擎侧硬件自流控（仅当引擎空闲且 FIFO 非空时置 `SpiStart` 弹一条命令）；软件侧建议性背压（TX 写端口未被 `TxFull` 门控，「别写满」由 C 驱动兜底）。关键术语：Status 寄存器为电平型，含 `TxEmpty/TxFull/RxEmpty/RxFull` 及 `Busy`（=`(not TxEmpty) or SpiBusy`）；运行时可配阈值 `CfgTxAlmEmpty`（`TxLevel ≤ 阈值`）与 `CfgRxAlmFull`（`RxLevel ≥ 阈值`）产生「快空/快满」标志，既置 Status bit 又锁存 IrqVec bit。`TxLevel` 位宽为 `log2ceil(FifoDepth_g)+1`。结论：testbench 用 9 条命令灌深度 8 的 FIFO 验证水位公式（`TxLevel_before(i)=9-i`），印证 FIFO 解耦加软件背压的价值；IrqVec 的锁存与清除后自动重置机制留待 u2-l6 详讲，软件侧背压留待 u2-l7。
