本讲解析刚引入的 3-Wire SPI 扩展。3-Wire 用一根双向数据线替代 MOSI/MISO，IP 额外输出三态控制信号 `spi_tri` 指示 Master「驱动」还是「放手」让从机驱动，物理合线（IOBUF）发生在 IP 之外。引入三个 generic：`ReadBitPol_g`（R/W 位读电平极性）、`TriStatePol_g`（`spi_tri` 有效电平）、`SpiDataPos_g`（有效数据起始位），其真正消费者是外部引擎 `psi_common_spi_master`；`spi_simple` 与 `spi_vivado_wrp` 在此链上仅做同名/改名透传，不加 `generate`。核心结论：`TriWiresSpi_g` 是纯 wrapper 级开关，不进入核心 RTL，仅经 `add_port_enablement_condition`（落地为 `component.xml` 的 `PORT_ENABLEMENT.spi_tri`）控制 `spi_tri` 是否在 IP 边界导出；所谓「`spi_tri` 只在 `TriWiresSpi_g=true` 时综合」属 IP-XACT 端口使能语义，而非 RTL `generate`。陷阱：`SpiDataPos_g` 默认值在 wrapper(8) 与 core(3) 不一致，经 IP 实际生效 8。
