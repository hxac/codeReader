本讲把视线从命令/响应 FIFO 下沉到 SPI 物理时序。`spi_simple` 自身不做移位，而是把时序 generic 透传给外部引擎 `psi_common_spi_master`：`spi_vivado_wrp` 同名透传，`spi_simple` 改名透传（`ClockDivider_g→clk_div_g`、`SpiCPOL_g→spi_cpol_g`），握手三件套为 `start_i/busy_o/done_o`。透传链为 Vivado 实参 → wrapper → spi_simple → 引擎（真正生成 SCK）。两个时序参数：`ClockDivider_g`（须为 2 的倍数）决定 SCK 频率，近似 `f_sck ≈ f_clk/ClockDivider_g`；`CsHighCycles_g`（以 `Clk` 周期计）决定两次传输间 CS_n 最小高电平时长；引擎精确计数待本地验证。CPOL 决定 SCK 空闲电平（0=低、1=高），CPHA 决定采样边沿（0=前导沿、1=后随沿），组合出 Mode 0/1/2/3。testbench 的 `p_spi` 进程用「apply 边沿（驱动 MISO）/ transfer 边沿（采样 MOSI）」两段 if/elsif 编码四种模式；默认 Mode 0 为空闲低、上升沿采样、下降沿切换，首位在首个 SCK 沿出现前就位。
