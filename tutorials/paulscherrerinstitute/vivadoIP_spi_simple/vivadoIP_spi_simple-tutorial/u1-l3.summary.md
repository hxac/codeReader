本讲说明 `vivadoIP_spi_simple` 并非孤立项目，而是复用 PSI HDL Library 家族的四个外部依赖：TCL 层的 `PsiSim`（仿真框架，≥2.1.0）、`PsiIpPackage`（IP 打包，2.0.0），VHDL 层的 `psi_common`（运行期组件，≥3.0.0）、`psi_tb`（测试库，≥3.0.0）。核心结论是区分「最小使用依赖」与「完整开发依赖」：仅 `psi_common` 是使用 IP 时也必需的运行期依赖，其余三者标注 `for development only`，只在仿真或重新打包时需要。项目要求固定的目录结构（文件夹名须精确匹配），因为打包/仿真脚本按相对路径找依赖；聚合仓库 `psi_fpga_all` 用 Git 子模块把这套结构预先搭好，一键 clone 即可。本讲引入的关键术语包括：依赖、最低版本、Git 子模块、单一数据源（SSOT）。README 的依赖段由两行 HTML 注释夹住作为解析边界，既是人类阅读的契约，也是 `scripts/dependencies.py` 的唯一数据源——该 16 行脚本不硬编码任何依赖，通过 `Parse.FromReadme` 解析 README，再调用 `Actions.ExecMain` 拉取，真正逻辑外置于须预先安装的 Python 包 `PsiFpgaLibDependencies`（1.1.0 版本引入）。
