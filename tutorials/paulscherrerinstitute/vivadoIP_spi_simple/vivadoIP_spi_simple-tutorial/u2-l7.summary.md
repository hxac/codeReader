本讲把视线从 FPGA 内部 RTL 转到 PS 侧软件，讲解 `spi_simple.h/.c` 这层 C 驱动。它是一层薄胶水：用 `SPI_SIMPLE_REG_*`、`SPI_SIMPLE_STATUS_*`、`SPI_SIMPLE_IRQ_*` 三组宏把 RTL（`definitions_pkg.vhd`）的寄存器地图镜像到 C 侧，偏移量 = 索引 × 4，两侧须手工同步。返回码枚举 `SpiSimple_ErrCode` 约定 0 为成功、负数为错误，判断须用 `== SpiSimple_Success`。收发 API 在两维正交：TX-only 对 RX+TX（由 `StoreRx` 决定是否把 MISO 存入响应 FIFO）、Blocking 对 NonBlocking（是否轮询 `Status.Busy` 等事务物理完成）。`Data(0x00)` 双语义：写即扣扳机命令入队，读即弹响应 FIFO 取回 MISO。软件背压全靠读 `Status`：`TX_FULL` 门控写入、`RX_EMPTY` 门控读取。配置 API 对应硬件中断/阈值机制：`ClrIrqVec`（W1C）、`SetIrqEna`、两个阈值 setter，ISR 标准模式为 `GetIrqVec` 后 `ClrIrqVec(vec)` 认领即清。
