本讲梳理 `spi_simple` IP 的参数化机制（专家层第一讲）。IP 共暴露 13 个用户可配 generic（另有一个 AXI 内部参数），按时序、数据格式、拓扑、3-Wire 扩展四组划分，权威取值范围由 wrapper 实体 `spi_vivado_wrp.vhd` 的 VHDL 子类型约束规定。

核心结论是「三层值链」：用户在 GUI 填写的 `PARAM_VALUE` 经 `xgui` 的 `update_MODELPARAM_VALUE.*` 回调拷成 `MODELPARAM_VALUE`，再作为 VHDL generic 实参，经 wrapper → 核心 `spi_simple` → 引擎 `psi_common_spi_master` 两层 generic map 透传（部分改名，如 `ClockDivider_g => clk_div_g`）。

关键概念：`scripts/package.tcl` 是参数化唯一数据源，用 `gui_create_parameter`/`gui_parameter_set_range`/`gui_parameter_set_widget_dropdown`/`checkbox`/`gui_add_parameter` 声明控件；`xgui/spi_simple_v1_4.tcl` 与 `component.xml` 均为自动产物，禁止手改。端口条件使能机制使 `spi_tri` 仅在 `TriWiresSpi_g=true` 时显现于外围接口——该 generic 是纯 wrapper 级综合开关，不进核心，内部照常驱动；与之并行的端口宽度依赖使 `spi_cs_n`/`spi_le` 位宽随 `SlaveCnt_g` 变化。阅读陷阱：`SpiDataPos_g` 默认值在 wrapper(8) 与核心(3) 不一致，`CsHighCycles_g`/`FifoDepth_g` 在 GUI 无范围、仅靠 VHDL `positive` 兜底。后续讲义将承接 `ReadBitPol_g`/`TriStatePol_g`/`SpiDataPos_g` 的 3-Wire 行为与 `spi_le` 时序。
