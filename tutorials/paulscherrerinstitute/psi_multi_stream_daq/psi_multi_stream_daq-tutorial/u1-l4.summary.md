本讲从硬件视角转向软件侧，讲解运行在 CPU（如 Zynq PS）的 C 驱动 `psi_ms_daq`：它经 AXI Slave 寄存器配置 IP、经内存拷贝读取 DMA 写入 DDR 的窗口数据。

核心机制是「访问函数注入」：把读/写寄存器、拷贝数据抽象为三个可注入函数指针（`PsiMsDaq_AccessFct_t`），传 NULL 则用默认裸机 MMIO（volatile 解引用）实现，使驱动平台无关、可单元测试；所有寄存器访问最终经 `PsiMsDaq_RegWrite/RegRead`（baseAddr+addr）出口。

引入的关键术语：两层 void* 句柄 `IpHandle`（IP 实例）与 `StrHandle`（单路流），用于信息隐藏；典型调用序列 Init→GetStrHandle→Str_Configure→注册中断回调→使能中断→使能流；`PsiMsDaq_StrConfig_t` 八字段被翻译为 POSTTRIG、MODE、SCFG、BUFSTART、WINSIZE 等寄存器写，其中 WINCNT 字段写 winCnt-1，配置前流必须禁用；记录模式有 Continuous/TriggerMask/SingleShot/Manual 四种；中断处理分窗口方案（每窗口回调一次，需 winOverwrite=false）与流方案两套互斥，靠 lastProcWin 游标与 irqCalledWin 位图保证每窗口恰好回调一次；回调内先 `GetDataUnwrapped` 解环形拷出数据、再 `MarkAsFree` 释放窗口；驱动与硬件的核心约定是 WINCNT=0 即「窗口空闲」。中断去抖、环形解包等算法细节留待 u4 单元展开。
