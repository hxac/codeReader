本讲覆盖 2024 年 8 月合入的两项「配置化」增强。第一项是新增全局寄存器 ACPCFG（地址 0x024，commit 01de9bd），把 AXI Master 写 DDR 时的 AWCACHE/AWPROT/ARCACHE/ARPROT 暴露给软件，按位段 [2:0]/[7:4]/[10:8]/[15:12] 切分，使不同 SoC 平台无需重新综合即可调整缓存与保护属性；该寄存器在 S_Axi_Aclk 域写保持，再由顶层 sync_apc_reg 进程用 2-FF 同步链跨到 M_Axi_Aclk 域驱动 AXI 输出引脚，目的是消除亚稳态，i_memif 的对应引脚改留 open 以避免多驱动。第二项是 commit 16f13e6 把原本写死的 64 位内部数据宽度提取为顶层 generic IntDataWidth_g，并贯穿 input/daq_dma/axi_if，派生出 WconvFactor_c、IntDataBytes_c、BytesWidth_c、DataFifoWidth_c 等常量。其关键机制是将 Input2Daq_Data_t 的 Data/Bytes 字段声明为无约束 std_logic_vector，再由顶层信号、input 端口、dma 端口三处用一致的 record subtype indication 钉死宽度。同名常量 BytesWidth_c 在 input 与 dma 中定义差一。ACPCFG 已在 HDL 实现，但 C 驱动头文件暂无便捷宏。
