本讲在 u5-l1 仿真工程骨架之上，精读了 PSI 项目三个核心模块级 testbench：`psi_ms_daq_input`（输入逻辑）、`psi_ms_daq_daq_dma`（DMA 引擎）、`psi_ms_daq_daq_sm`（控制状态机），三者同属 **TbGen 风格**，共享「顶层 TB（调度机）+ `*_tb_pkg`（公共过程）+ `*_tb_case_*`（场景文件）」三层骨架，靠 `NextCase`（当前用例号）、`ProcessDone`（各进程完成位）、`Generics_c`（参数记录常量）三个机制协同。DUT 实体里的 `$$ testcases` / `$$ processes` 元注解与 case 文件名、`config.tcl` 条目、顶层进程构成机器可读的「多方对应」契约。input TB 用 `ApplyStrData`/`CheckAcqData` 两个过程配 `Generics_t` 记录实现参数化自检，同一份 case 在 8/16/32/64 位流宽下各跑一遍，校验按 qword 拼接的数据、`Last`/`Bytes`/`IsTrig`/`IsTo` 五项元数据。dma TB 拆成 `control`/`input`/`mem_cmd`/`mem_dat` 四进程，用 `EndType_s` 枚举（Trigger/Timeout/NoData/NoEnd）统一描述帧结束方式，8 个 case 覆盖对齐、非对齐、Fifo 满、错误等边界。sm TB 最复杂，四进程外加一整套「Auto」高层过程（`ExpCtxFullBurstAuto`/`CheckIrq` 等）自动跟踪各流指针与窗口号，固定 generic 为 4 流（优先级 1,2,3,1、流宽 8,16,32,64）、8 窗口、512 突发。`irq` case 是本讲精读对象，其 `CheckIrq` 用「只目标流置位 + 窗口号正确 + 单拍撤销」三连断言，配合 Normal/Flipped Order 正例与 Ringbuf Wrap/FIFO 满反例，保证「每个完成的窗口恰好产生一次中断」。这些 TB 的断言本质上是 u2-l3（记录模式/触发）、u2-l4（超时/时间戳）、u4-l1（中断/IRQ FIFO）等硬件机制的可执行规约。
