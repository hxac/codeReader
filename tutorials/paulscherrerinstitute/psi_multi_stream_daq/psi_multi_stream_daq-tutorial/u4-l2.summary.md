本讲从 CPU 侧讲 C 驱动如何消费硬件中断线，仅涉及 `psi_ms_daq.c/.h`。核心入口 `PsiMsDaq_HandleIrq()` 三步：读 `IRQVEC` 查中断源、写回原值（W1C）应答以拉低电平线、遍历各流按注册回调分发。关键寄存器 `IRQVEC`（中断向量，写1清零）、`LASTWIN`（硬件回写最近完成窗口号）、`WINCNT`（0 即窗口空闲）。回调分互斥两种：流方案 `irqFctStr` 每次中断只回调一次、判断全交用户；窗口方案 `irqFctWin` 保证每窗口恰好回调一次，须 `winOverwrite=false`。窗口方案以 `lastProcWin` 游标防漏、`irqCalledWin` 位图防重，do-while 每轮重读 `LASTWIN` 抽干已完成窗口，`break` 在更新游标前执行以保留恢复点。回调拿到栈上临时结构体 `WinInfo_t` 后须调 `PsiMsDaq_StrWin_MarkAsFree()`，同步软件侧清位与硬件侧写 `WINCNT=0` 来释放窗口。驱动整体非线程安全。
