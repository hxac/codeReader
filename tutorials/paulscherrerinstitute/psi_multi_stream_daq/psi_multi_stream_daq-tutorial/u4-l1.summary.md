本讲专讲 DAQ IP 核的硬件侧中断生成机制，全部集中在控制状态机 `psi_ms_daq_daq_sm` 的 IRQ 处理逻辑中。核心围绕 IRQ 信息 FIFO（`i_irq_fifo`，一个 `psi_common_sync_fifo`）展开，它缓存「流号 Stream + 窗口号 LastWinNr + WinDone」三字段记录，宽度为 `StreamBits_c + log2ceil(Windows_g) + 1`，深度取 `2**StreamBits_c*4`、almost-full 阈值 `Streams_g*3`。

三大节拍：写节拍挂在响应路径的 `NextWin_s`，凡 `Size/=0` 的传输都入队，但只有线性缓冲写满窗口或遇到触发才置 `WinDone=1`（环形回绕不发中断）；读节拍由 AXI 主接口的 `TfDone` 驱动，`TfDoneReg` 打一拍、`TfDoneCnt` 计数待发出的完成，与 `IrqFifoEmpty` 双条件成立才弹出一条；只有 `WinDone=1` 的记录弹出时才在对应流发 `StrIrq` 脉冲并输出 `StrLastWin`。

关键结论有二：其一，`TfDoneCnt` 解耦了 `TfDone` 与 FIFO 记录到达的先后顺序，保证发中断当拍传输确已完成；其二，commit 3957ce3 把 `StrIrq` 生成从读判定 `if` 内部移出、改用寄存器版 `r.IrqFifoRead` 门控，消除窗口切换时采样 `dat_o` 瞬态导致「记录被消费但中断永久丢失」的竞态。溢出保护采用源头反压：`IrqFifoAlmFull` 在 `Idle_s` 拦住新命令签发，保证响应一定有处可放。本讲产生的 `StrIrq` 即 `reg_axi` 中 `IRQVEC` 聚合的输入，供后续 C 驱动消费。
