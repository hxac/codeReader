本讲介绍仿真验证单元的收尾内容：顶层多流 testbench `psi_ms_daq_axi_tb`。它与 u5-l2 的模块级 TB 互补，以整个 `psi_ms_daq_axi` IP 为 DUT，端到端验证「AXI 寄存器配置 → 多路流输入 → DMA 写 DDR → 中断回报 → 回读校验」全链路。两大支柱是：公共包 `psi_ms_daq_axi_tb_pkg` 提供的、所有进程共享的字节数组 `Memory`（伪 DDR，字节寻址、64 KiB），以及与硬件 `reg_axi` 地址译码互为镜像的寄存器地址字典和 `Hl*` 高层过程。TB 由若干并发进程扮演系统角色：`p_regacc` 当 CPU、`p_mem` 当带 WLast 自检的 AXI4 写通道 Slave、`p_str0..3` 当 4 路异步数据源、`IrqHandler` 用 VHDL 重写最小中断服务例程，与 C 驱动 `HandleIrq` 逐句对应。每路流封装在独立 `str*_pkg`，提供 Setup/Update/Data/Handler 四过程，刻意覆盖 8/16/32 位宽与环形/线性、覆盖/不覆盖、Continuous/Manual/SingleShot/TriggerMask 四模式组合。关键惯用法包括 `HlIsTrigWin` 利用 WINCNT bit 31 触发位使有符号整数变负、`HlConfStream` 写 wincnt-1，均与硬件和驱动语义对齐。单流变体 `psi_ms_daq_axi_1s_tb` 复用公共包、`Streams_g=1`、AXI 宽度 64、用默认 `IntDataWidth_g`，验证最常见精简配置，骨架与多流版几乎逐行相同。
