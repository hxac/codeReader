本讲聚焦 DMA 引擎 `psi_ms_daq_daq_dma` 的结构与接口（不展开状态机内部细节，留待 u2-l6）。它在 IP 核中「承上启下」：从控制状态机 `psi_ms_daq_daq_sm` 接收命令、从输入逻辑 `psi_ms_daq_input` 接收样本，拼成内存字后经 AXI 主接口 `psi_ms_daq_axi_if` 写入 DDR，再把响应回送状态机。端口分四组：控制、状态机连接（命令/响应/`HasLast`）、输入连接、内存连接（只写不读）。核心是四个缓存：**CmdFifo/RspFifo**（深度 `2**StreamBits_c`，依据「每流最多一条在途命令」）用 `ToStdlv`/`FromStdlv` 序列化记录、解耦命令与响应时序；**DatFifo** 缓冲 FIFO 用 `alm_full`（阈值 16）反压，避免反压逻辑铺满流水线；**Remaining Data RAM**（`psi_common_sdp_ram`，按流号寻址、每流一格）保存一次传输结束时未拼满的残余字节、位置与挂起的 Trigger/Last，使跨传输字边界正确衔接。`FirstDma` 寄存器复位置全 1，使每流首次传输跳过 RAM 未定义初值、零偏移起步，之后清 0（仅复位时重置，软件重使能由状态机 `FirstAfterEna` 处理）。关键术语：`IntDataWidth_g` 字宽、命令/响应记录类型、`DaqDme2DaqSm_Resp_FromStdlv` 拼写、`DatFifo_AlmFull`。
