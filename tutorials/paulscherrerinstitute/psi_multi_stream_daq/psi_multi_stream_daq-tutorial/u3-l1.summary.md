控制状态机 psi_ms_daq_daq_sm 是多流 DAQ IP 的调度"大脑"，统筹输入水位、DMA 命令/响应、上下文读写与中断，把"窗口/优先级"等高层语义翻译成下层"地址+大小"的 DMA 命令。状态机以两进程法（p_comb 算 r_next、p_seq 打入 r）实现，共 14 状态；本讲只覆盖调度决策：Idle_s、CheckPrio1/2/3_s、TlastCheck_s、CheckResp_s。

调度策略为"档间严格优先级、档内轮询"：三个独立仲裁器实例各管一档，输出 one-hot GrantPrio 并压缩为 GrantVld，由 CheckPrio1→2→3 顺序扫描选中。关键 generic 有 StreamPrio_g（数组型，默认 (1,2,3,1)，数字越小优先级越高，仅 1/2/3 三档）、MinBurstSize_g 与 MaxBurstSize_g（单位为内部 64 位字）。

"数据可用"分三层向量：InpDataAvail（使能且数据量达标）→ DataAvailArbIn（再屏蔽 OpenCommand、WinProtected，喂仲裁器）→ DataPending（只屏蔽 WinProtected，用于决定等响应还是降级）。关键设计是 DataPending 不屏蔽 OpenCommand，使高优先级流命令在飞时仍保持优先级，不被低优先级插队；Idle_s 用 IrqFifoAlmFull 做源头反压。MinBurstSize_g 是低优先级流获服务的关键（高优先级数据不达标即让出机会），TlastCheck_s 则绕过该门槛专门冲刷帧末尾（HasLast）残余。GetBitsOfStreamPrio 与 GetStreamNrFromGrant 在"流号编址"与"档内序号编址"两套坐标系间严格互逆转换。后续 u3-l2、u3-l3 讲上下文读取、DMA 地址计算与窗口切换回写，u4-l5 讲 OpenCommand/WinProtected 协议。
