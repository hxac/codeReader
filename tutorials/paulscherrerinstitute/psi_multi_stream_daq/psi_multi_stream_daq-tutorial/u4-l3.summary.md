本讲讲解 C 驱动函数 `GetDataUnwrapped` 的环形缓冲解包。环形缓冲（`winAsRingbuf`）下 DMA 写指针会在窗口内回绕，使时间顺序与地址顺序错位；函数以硬件回写的 `WIN_LAST`（最后写入样本的首字节地址）为锚，把用户请求的「前触发+后触发」样本拷成按时间从旧到新的连续内存。四个辅助函数分工：`GetNoOfSamples` 读 `WINCNT` 样本数，`GetPreTrigSamples` 检查 `IsTrig` 位并算「样本数−配置后触发」，`GetLastSplAddr` 读 `WIN_LAST`，`GetNoOfBytes` 换算字节。关键术语含环形缓冲、前/后触发样本、`trigByteAddr` 与 `lastByteAddr` 的「单次±winSize」模回绕。核心结论：越界返回码 `-6 NoTrigInWin`、`-7 BufferTooSmall`、`-8 MorePostTrigThanConfigured`、`-9 MorePreTrigThanAvailable`；拷贝按 `firstByteLinear` 是否 ≥ `winStart` 分单段直拷与两段 `memcpy` 拼接两支；搬运走注入的 `memcpyFct`（源指针即 IP 视角 DDR 地址，依赖 DMA 与 CPU 物理地址空间一致，cache 维护须先做）；函数只搬数据，窗口应答靠 `MarkAsFree`。
