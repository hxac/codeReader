本讲是 u6 单元的基石，拆解 `top_tb` 仿真平台的三大公共机制。其一，主结构：用两个独立且不可约的时钟——160 MHz 数据域（`Clk`）与 125 MHz AXI 域（`aclk`）——驱动双时钟域 DUT `data_rec_vivado_wrp`，以真实暴露 CDC 行为；`p_control` 顺序调度 case0~case5。借助外部库 `psi_tb` 的 record 类型 `axi_ms_r`/`axi_sm_r`，把整条 AXI 总线压成 `axi_ms`/`axi_sm` 两根信号，过程内整体传给 `axi_single_write`/`axi_single_expect` 操作。其二，激励生成：`InputSamples` 用全局递增共享变量 `PatternCnt_v` 当基底，把**通道号写进最高 3 位**叠加通道偏移，并支持 `dutycycle` 断流与 `trigAt`/`trigIdx` 触发注入；`InputSamplesNoCh` 是 signed、通道间小幅偏移的变体。其三，数据校验：`CheckData` 经 `MemAddr(ch, spl, depth)` 算出线性地址逐样本回读，期望值闭式公式 `ExpVal_v = ch*2**(W-3) + spl + startValue`，其中通道项来自高 3 位编码、`spl + startValue` 是窗口样本的计数器值（`startValue` 即窗口首样本偏移）；`axi_single_expect` 不一致时打印 `###ERROR###`，这是 CI 失败信号源。公式隐含约束：总样本数须 < 2^(W-3)，且 `MemoryDepth_g`（DUT）须与 `MemoryDepth_v`（地址计算）语义一致。
