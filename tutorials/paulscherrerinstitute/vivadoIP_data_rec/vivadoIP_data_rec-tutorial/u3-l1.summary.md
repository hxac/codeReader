本讲拆解 data_rec 核心记录器的 entity（hdl/data_rec.vhd），只读接口、不展开状态机与流水线。四个 generic 决定整块 IP 规模：NumOfInputs_g（1..8 通道）、InputWidth_g（每路位宽）、MemoryDepth_g（采样深度）、TrigInputs_g（0..8 外部触发），其中通道数与外部触发数相互独立。端口位宽几乎都是 generic 的函数：Mem_Data 总宽=NumOfInputs_g×InputWidth_g，地址类端口=⌈log2(MemoryDepth_g)⌉位，而 TotalSpls 比 PreTrigSpls 多 1 位，以允许总样本数超过缓冲深度。引入的关键术语与结论：In_Vld 同时门控数据写入与触发判定（仅在 In_Vld=1 时采到一个样本）；数据端口写死 8 路、循环只用前 NumOfInputs_g 路；Trig_Out 转发内部裁决后的 Trigger_2 而非原始 Trig_In；NonPwr2MemDepth_c（由 log2≠log2ceil 判定）标识非二次幂深度并切换地址处理分支。寄存器侧端口可逐个挂回 u2-l2 地址地图（State→0x0000、PreTrigSpls→0x0008、TrigEna→0x0028），SelfTrigChEna/OnExit/OnEnter 三者共用 0x0018 按位移拼接。存储侧为「外部存储器接口」风格：核心不内置 RAM，仅输出 Mem_Wr/Mem_Adr/Mem_Data，Mem_Data 把所有通道拼成宽字按时间对齐落盘（通道 0 在最低位），FirstSplAddr 给出环形缓冲起点供读出时展开为线性序列。本讲为后续 u3-l2（状态机）、u3-l3（两进程流水线）、u3-l5（非二次幂地址）建立接口与位宽基础。
