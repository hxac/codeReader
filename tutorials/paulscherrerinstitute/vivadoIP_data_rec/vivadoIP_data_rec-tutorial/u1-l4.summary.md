本讲讲解如何把 `data_rec` 打包成 Vivado 可识别的 IP，核心是三个文件：`scripts/package.tcl` 是打包「菜谱」，通过 `PsiIpPackage` 框架的高层命令（`init`/`add_sources_relative`/`gui_create_parameter`/`add_port_enablement_condition`/`package_ip`）声明 IP 身份、源码、参数与可选端口；`component.xml` 是框架自动生成、Vivado 实际读取的 IP-XACT（IEEE 1685/SPIRIT）描述，用 VLNV 四元组 `psi.ch:GPAC3:data_rec:2.4` 唯一标识，编码了 AXI4 Slave（aximm）、时钟、复位、中断接口、端口使能条件、参数默认值与文件集；`xgui/data_rec_v2_4.tcl` 驱动定制面板，其中 `init_gui` 摆放参数，`update_MODELPARAM_VALUE.*` 把 GUI 的 `PARAM_VALUE` 同步为综合用的 `MODELPARAM_VALUE`，最终传给 VHDL generic。关键机制是「可选端口按 generic 使能」：`In_Data0..7` 依赖 `NumOfInputs_g > i`，`Trig_Out` 依赖 `TrigForwarding_g = true`（v2.4 新增）。核心工程教训：同一 generic 在打包脚本、`component.xml`、xgui、VHDL entity 四处镜像，改 IP 必须同步；版本升级还会牵连 xgui 文件名与 VHDL 库名（`data_rec_2_4`）。AXI 默认地址范围由 `C_S00_AXI_ADDR_WIDTH`（默认 14）决定，为 16 KiB（0x4000），该地址空间的寄存器与存储地图留给 u2-l2 展开。
