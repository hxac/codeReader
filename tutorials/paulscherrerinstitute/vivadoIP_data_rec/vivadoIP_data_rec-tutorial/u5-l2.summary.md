封装层 `data_rec_vivado_wrp` 在数据域 `Clk` 与 AXI 域 `s00_axi_aclk`（两域异步）之间设了 4 个跨时钟域（CDC）实例，按「方向 × 性质」构成 2×2 矩阵：`status_cc` 两路（toAxi、fromAxi）传多比特电平/值，`pulse_cc` 两路（fromAxi、toAxi）传单拍事件。核心判据是「值走 status、事件走 pulse」：`status_cc` 经握手在目的域对整条多比特总线做一次整体采样，得到一致快照（避免撕裂），用链式 `subtype` 把 State/TrigCnt/DoneTime 拼成 68 位宽向量、把全部配置拼成随 generics 联动的宽向量；`pulse_cc` 每路占 1 位（generic 为 `num_pulses_g`），靠边沿检测+握手在目的域精确复现一次脉冲。据此，`Arm/Ack/TrigCntClr/Done` 走 pulse（其中 Done 经 toAxi 驱动 `Done_Irq` 中断），sticky 的 `SwTrig` 及所有配置/状态走 status。关键辨析：`reg_cfg_arm` 含单拍 `reg_wr` 是事件必须走 pulse_cc，`reg_swtrig` 仅取 `reg_wdata` 是电平可走 status_cc。另记：核心复位 `RstProc` 取自 status_cc 的 `a_rst_o`，AXI 域复位由 `s00_axi_aresetn` 取反得到 `AxiRst`。
