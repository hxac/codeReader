本讲以 `data_rec.vhd` 为唯一对象，建立记录器核心 RTL 的「骨架」。核心是 **两进程法**：把时序电路拆成组合进程 `p_comb`（计算下一拍 `r_next`）与时序进程 `p_seq`（时钟沿搬入 `r <= r_next` 并同步复位），所有寄存器打包进 `data_rec_r` record，并用 `r`/`r_next` 两信号桥接。`p_comb` 模板为「`v := r;`（默认保持，杜绝锁存器）→ 按 Stage 顺序改 `v` 的字段 → `r_next <= v;`」，用变量 `v` 是为了同拍内顺序计算（如先算 `TrigNow_2` 再推进状态机）。流水搬运靠范围切片 `v.X(low+1 to high) := r.X(low to high-1)` 整体左移，并依字段分输入寄存器、状态、配置、触发输出四类。电路实为 **四级流水线**：Stage0 采入 `In_Vld/Trig_In/Data_0`，Stage1 数据流水与自触发范围判定（`StInRange_1`），Stage2 触发裁决、状态机 `case r.State_2` 与计数器（`Trigger_2`、`Done(2)`、`AdrCnt_2`），Stage3 对齐存储器写端口（`MemWr_3`、`FirstSpl_3`、`AdrCnt_3`）。关键约定：**命名数字后缀即流水级编号**，数据与有效信号均延迟 3 拍到达写端口天然对齐；复位遵循「控制信号复位、数据不复位」。状态机迁移条件见 u3-l2，非二次幂 `FirstSpl` 见 u3-l5。
