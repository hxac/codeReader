本讲是触发机制单元（u4）收尾篇，覆盖三个触发周边机制。最小录制间隔 MinRecPeriod 由倒计数器 LastRecCnt_2 实现"冷却"：触发兑现后重装、每拍递减（不受 In_Vld 门控，单位为数据时钟周期），冷却期内强制 TrigNow_2 为 0 并直接丢弃（而非延迟）外部与软件 pending；其分支含软件调小则钳位、冷却中递减并清 pending、兑现触发则重装，复位为 0 故首次触发必放行。诊断计数器 TrigCnt_3 统计真正兑现的触发次数（被 MinRecPeriod 抑制的不计），由 Cfg 的 TrgCntClr 单拍清零且清零优先；DoneTime_3 在 Done_s 状态每拍 +1 度量主机读出延迟，下次触发归零、到 0xFFFFFFFF 饱和保护。v2.4 新增的 Trig_Out 转发"裁决后兑现"的 Trigger_2，处于数据时钟域、封装层直通，表示"录制开始"；generic TrigForwarding_g（默认 false）仅在 IP 打包层控制端口是否暴露，不影响核心逻辑。它与 Done_Irq（AXI 时钟域、转发 Done、经 pulse_cc 跨域，表示"录制完成"）共同界定一次录制的起讫，体现本 IP 双时钟域分工：数据域管采集与触发，AXI 域管中断与读出。
