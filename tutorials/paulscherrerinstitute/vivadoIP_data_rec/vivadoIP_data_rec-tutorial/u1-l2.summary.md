本讲从「能做什么」转向「代码放在哪、依赖谁、怎么组织」，包含三个最小模块：一是仓库顶层目录结构——按职能分目录，`hdl/` 是核心 RTL（仅 3 个文件：`data_rec.vhd`、`data_rec_register_pkg.vhd`、`data_rec_vivado_wrp.vhd`），`testbench/` 与 `sim/` 负责验证与仿真流程，`scripts/`、`xgui/`、`component.xml` 负责 IP 打包与 GUI，`epics/` 负责控制系统集成，`doc/` 存放数据手册。二是 PSI FPGA 库依赖与文件夹约定——依赖四个外部库：`psi_common`（综合/运行必需）、`psi_tb`（仅仿真）、`PsiSim`（仅仿真流程）、`PsiIpPackage`（仅打包发布）。三是依赖解析脚本 `scripts/dependencies.py`——它通过解析 README 中被 HTML 注释包裹的依赖清单自动 checkout 依赖，README 是依赖的唯一真相源，脚本不含硬编码版本号。核心结论：`component.xml` 是 IP 的「身份证」，最权威地暴露了外部依赖；本仓库必须放在 `<根目录>/VivadoIp/vivadoIP_data_rec/` 下，`component.xml`、`package.tcl`、`config.tcl` 三处相对路径（分别上跳 2/3/3 层）共同印证这一约定，亦可用 `psi_fpga_all` 一键获得合规结构。关键术语包括 IP-XACT/component.xml、AXI、IP 核、psi_common 系列实体（axi_slave_ipif、pulse_cc、status_cc、tdp_ram、pl_stage 等）。
