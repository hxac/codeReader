本讲讲 `data_rec` IP 核如何接入 EPICS。主机 IOC 经 **regDev** 设备支持，把 record（`bo`/`ai`/`mbbo`/`aai`/`calcout`/`seq` 等）绑定到 AXI 寄存器，地址写作 `@base:offset T=type`，靠 `FLNK`/`PP`/`SCAN` 串成处理链。`epics/` 用两个 Python 生成器化解「通道/触发数可变 vs db 模板静态」：`GenerateDataRecTemplates.py` 吃通道数与外部触发数，把带占位符的 `CONTROL.tpl` 母版展开成 `.template`；`GenerateDataRecPanel.py` 吃通道名列表生成 Qt `.ui` 面板（绘图上限 6 通道，而硬件上限 8）。模板心脏是一条 record 闭环：`Done_Irq` 中断（`I/O Intr`）加 0.2 秒轮询双驱动 → `CHECK` 判 Done → `ACQUIRE` 回读各通道 → `POSTREAD` 自动 re-arm。核心结论：13 个寄存器偏移 `0x00`–`0x30` 与 `register_pkg` 的 `Reg_*_Addr_c` 逐一对齐；`TRIGSRC` 枚举 1/2/4 对应三类触发源的 `2^Idx`；存储区 `aai` 偏移 `ch*4*DEPTH` 隐含二次幂深度假设，非二次幂深度会错位；可读通道名仅属面板层，db/硬件层用 `CH` 编号解耦。
