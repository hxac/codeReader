本讲是 `data_rec` 学习手册的收尾篇，主题是二次开发，总纲是「镜像法则」：同一个 generic/端口/地址/参数在仓库里被复制到很多文件，改它必须同步所有镜像。围绕四个模块展开：触发源扩展（咽喉是 `TrigEna` 写死 3 位、三类源各占一位、无空闲，新源须按「边沿+sticky／电平+sticky／边沿+瞬时」三范式之一产生请求，OR 进 `TrigNow_2`，并在 `PreTrig` 与 `MinRecPeriod` 清 pending）；寄存器地图扩展（须三处联动：`register_pkg` 地址常量、封装层解码/回读/CDC、EPICS 模板；空闲仅 `0x34`–`0x7C` 共 19 个，受 `USER_SLV_NUM_REG=32` 与 `Mem_Addr_c=0x80` 约束；值走 `status_cc`、事件走 `pulse_cc`）；样本位宽（沿 `InputWidth_g` 传播，硬约束是 AXI 数据固定 32 位，故样本上限 32 位，窄样本靠符号扩展填满）；打包与版本（改 `package.tcl` 与 RTL 后重跑 `package_ip` 让 `component.xml`/xgui 自动重生成，勿手改；版本号升级有级联效应，牵动 xgui 文件名与 VHDL 库名 `data_rec_2_4`）。以 v2.4 的 `TrigForwarding_g`/`Trig_Out` 作为「一个新特性正确落地」的完整标本，并以「每通道独立自触发阈值」作为综合实践。
