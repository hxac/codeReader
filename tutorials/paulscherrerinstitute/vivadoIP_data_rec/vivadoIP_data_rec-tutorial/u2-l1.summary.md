本讲讲解 IP 对外的「门面」`data_rec_vivado_wrp`（封装层 wrapper），而非核心记录器 `data_rec`。封装层端口分为**算法侧**（数据 `In_Data0..7` + `In_Vld`、触发 `Trig_In`/`Trig_Out`、时钟复位 `Clk`/`Rst`）与**总线侧**（`s00_axi_*`）两大块；五个 generic（`NumOfInputs_g`、`InputWidth_g`、`MemoryDepth_g`、`TrigInputs_g`、`TrigForwarding_g`）决定端口的数量与位宽，数据通道数与外部触发数相互独立，这是参数化与可选端口使能的源头。

总线侧实现一个 **AXI4 Slave**，由 **五个独立通道**组成：AR（读地址）、R（读数据）、AW（写地址）、W（写数据，含 `WSTRB` 字节使能）、B（写响应），每通道靠 `VALID/READY` 握手生效。地址宽度 `C_S00_AXI_ADDR_WIDTH=14` 对应 16 KiB 空间，数据固定 32 位。

核心结论是本 IP 为**双时钟域**设计：数据域 `Clk` 与 AXI 域 `s00_axi_aclk`。是否跨域取决于接收方所在域——`Done_Irq`（中断）经 `pulse_cc` 跨到 AXI 域供 PS GIC 采样，`Trig_Out`（v2.4 新增，由 `TrigForwarding_g` 使能）直通留在数据域供级联逻辑使用。这为后续 u2-l2（地址地图）、u5-l1（AXI 解码）、u5-l2（跨时钟域策略）奠定基础。
