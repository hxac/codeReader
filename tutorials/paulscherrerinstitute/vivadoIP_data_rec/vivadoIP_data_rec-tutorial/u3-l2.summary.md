本讲讲解 `data_rec` 核心记录器的五状态有限状态机（Moore 型）：`Idle→PreTrig→WaitTrig→PostTrig→Done`。状态用枚举类型 `State_t` 定义，枚举位置 0–4 与寄存器包常量 `Reg_Stat_State*_c`、对外 4 位 `State` 端口三者一一对应，须同步维护。迁移逻辑全部写在 `p_comb` 的 `case r.State_2`：Arm 启动、前触发满（`AdrCnt_2=PreTrigSpls-1`，`PreTrigSpls=0` 为显式特例防下溢）进等待、`TrigNow_2` 触发后触发、`SplCnt_2>=TotalSpls` 完成。`Trigger_2` 与 `Done(2)` 是命中迁移时产生的单拍脉冲，分别对外走到 `Trig_Out` 与 `Done` 端口。关键设计是封装层的自动确认：软件在 Done 态读状态寄存器即产生 `AckDone`，经 `pulse_cc` 跨域回核心命中 Done→Idle，无需显式写 Ack。引入的关键术语：`State_t`、`r.State_2`（Stage 2 对齐）、`TrigNow_2`、`Trigger_2`/`Done(2)` 脉冲、`AckDone` 自动确认、`Done_Irq` 中断路径。
