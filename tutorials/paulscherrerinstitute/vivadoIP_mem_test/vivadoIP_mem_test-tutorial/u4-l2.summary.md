本讲解析 wrapper 中的 AXI4 主机实例 i_master：它把核心「地址+beat 数+数据流」的高层命令翻译成标准 burst，负责 burst 拆分、在途事务管理与数据缓冲；其 generic 经 wrapper 透传，分别约束单 burst 上限、在途事务数与缓冲深度。用户接口分写/读命令与写/读数据四组，全为 valid/ready 握手，命令 Size 以 beat 为单位（非字节），核心用右移把字节数换算为 beat 数并清地址低位对齐。四个响应信号中仅写/读错误被 FSM 使用，任一拉高即锁死不可恢复的 AxiError_s 状态、对外报 AXIERR(3)，仅复位可退出；而 Done 信号当前未被引用，核心靠本地计数推进状态。
