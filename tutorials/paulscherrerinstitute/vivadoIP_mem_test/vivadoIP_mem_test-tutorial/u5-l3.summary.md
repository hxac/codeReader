本讲是进阶单元的收尾，把整本手册的交付链闭环为「改 RTL → CI 仿真 → 重新封装 IP → Block Design 集成」。涉及三个文件：`bd/bd.tcl`（BD 钩子）、`scripts/ciFlow.py`（CI 入口）、`sim/ci.do`（桥接 do 文件）。核心结论之一：`bd.tcl` 含 `init`、`pre_propagate`、`propagate` 三个回调——`init` 把 `C_S00_AXI_ID_WIDTH` 标记为「仅传播（propagate-only）」使其在 BD 中无需手填；`pre_propagate` 处理 master（cell→接口，出方向），`propagate` 处理 slave（接口→cell，入方向）。对本 IP 真正生效的是 `propagate` 的 slave 分支，让 `S00_AXI` 的 ID 宽度自动跟随上游主机；master 分支因无对应参数而空操作。核心结论之二：CI 链路为 `ciFlow.py`→`vsim -c -do ci.do`→`ci.do`→`run.tcl`→PsiSim→`top_tb`，其中 `ci.do` 仅 `source run.tcl` 后 `quit` 作薄桥接；`ciFlow.py` 读 Transcript 按两条规则判定退出码——出现 `###ERROR###`（TB 契约串）返回 -1，缺 `SIMULATIONS COMPLETED SUCCESSFULLY` 返回 -2，否则 0。贯穿的设计理念是职责分离：`run.tcl` 只跑不判、`ci.do` 只桥接、`ciFlow.py` 才判退出码；`bd.tcl` 只处理需跨 IP 对齐的总线标准参数（如 ID_WIDTH），普通用户参数走 xgui（u5-l2）。引入的关键术语：Block Design、参数自动传播、TCL 回调、Transcript、`###ERROR###` 契约、退出码。
