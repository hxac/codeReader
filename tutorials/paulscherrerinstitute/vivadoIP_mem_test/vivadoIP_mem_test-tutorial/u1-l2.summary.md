本讲围绕 `vivadoIP_mem_test` 仓库讲解三个最小模块：目录布局、依赖清单、依赖获取脚本。仓库按角色分为八个目录：`hdl`（VHDL RTL 源码，含 `mem_test_pkg.vhd`、`mem_test.vhd`、`mem_test_wrapper.vhd`）、`tb`（testbench）、`sim`（仿真脚本）、`scripts`（构建/CI）、`drivers`（C 软件驱动）、`bd`（Block Design 集成）、`xgui`（IP 参数界面）、`doc`（文档）。

核心结论是：本仓库不能独立运行，必须摆在一个「公共根目录」下的 `VivadoIp/vivadoIP_mem_test/` 位置，这正是 `sim/run.tcl` 与 `config.tcl` 中反复出现的 `../../..` 相对路径的含义。它依赖四个外部库：`psi_common`（VHDL，运行必需，提供 AXI 主从机、FIFO 等组件）、`psi_tb`（VHDL，仿真辅助）、`PsiSim`（TCL 仿真框架）、`PsiIpPackage`（TCL 封装框架），后三者仅开发期使用。依赖清单写在 README 被 HTML 注释包裹的 Dependencies 段，既供人读也供脚本解析。`scripts/dependencies.py` 借助外部包 `PsiFpgaLibDependencies` 读取 README 并自动 clone 依赖到正确位置。本讲为后续仿真讲义（在 `sim/` 跑 `run.tcl`）奠定目录结构与依赖前置条件。
