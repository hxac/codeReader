本讲深入核心实例 `i_logic` 内部的 `mem_test` 实体，建立其对外契约与内部时序骨架。实体端口分三组：控制信号（`Clk/Rst_n`）、寄存器接口（`Reg_*` 四组，类型来自 package）、AXI 主机用户接口（`CmdWr_/CmdRd_/WrDat_/RdDat_` 加 `Wr_Done/Wr_Error/Rd_Done/Rd_Error` 响应）。两个 generic `AxiAddrWidth_g(12~64)`、`AxiDataWidth_g(16~1024)` 用 `range` 钉死合法配置，默认 32。命令与数据端口均为 valid/ready 握手，写侧 `*_Vld` 由核心输出，读侧 `RdDat_Vld` 为输入，唯一核心驱动的 Rdy 是 `RdDat_Rdy`。内部采用 **two-process 设计法**：所有状态打包进记录 `two_process_r`（16 字段，五大类），用 `r`/`r_next` 双信号承载；`p_comb` 纯组合进程用 `v := r` 惯法「只写变化量」算出 `r_next`，`p_reg` 只对时钟敏感每沿抄写并做**同步、部分复位**——只清 `Fsm→Idle_s` 及各握手信号，统计/数据字段留给 START 流程软清零，省下复位触发器。状态机 `Fsm_t` 的流转与 pattern 算法留给后续讲义。
