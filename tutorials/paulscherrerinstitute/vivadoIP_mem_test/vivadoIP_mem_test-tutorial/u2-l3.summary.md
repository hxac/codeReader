本讲讲解 `drivers/mem_test/` 这套 C 驱动如何把 u2-l1 的裸寄存器地址封装成人能读懂的 API，使 CPU 裸机程序能跑完一次完整内存测试。三个最小模块层层递进：**契约镜像**——`mem_test.h` 里的 `MEM_TEST_*_OFFS` 宏是 VHDL `mem_test_pkg.vhd` 中 `REG_*` 的 C 端镜像，全部满足 `字节偏移 = 4 × index`，逐项吻合（含 0x08 留空、状态码 3→6 的分组设计）；三组枚举 `MemTest_Mode/Pattern/Status` 数值也与 VHDL 完全一致。**API 实现**——每个函数都是对 `Xil_Out32`/`Xil_In32` 的薄封装，地址 = `C_BASEADDR + 偏移`，经 AXI-Lite 总线到达硬件寄存器；strobe 型（Start/Stop）固定写 1，配置型（SetMode/SetPattern/SetRange）写参数值，查询型读回枚举或计数。64 位量（ADDR/SIZE/FIRSTERR）因 AXI-Lite 数据宽度仅 32 位而拆成 LO/HI 两个寄存器，读拼接时须先存入 `uint64_t` 再移位，以规避 `u32 << 32` 的未定义行为。**打包元数据**——`mem_test.mdd` 声明驱动绑定 `mem_test` 外设，`mem_test.tcl` 在 BSP 生成时把 Block Design 分配的 `C_BASEADDR` 等写入 `xparameters.h`，应用代码因此无需硬编码地址；Makefile 用 `$(wildcard *.c)` 显式枚举源文件编译进 `libxil.a`。一次完整测试的调用范式为：配置→启动→轮询 Idle→读结果。
