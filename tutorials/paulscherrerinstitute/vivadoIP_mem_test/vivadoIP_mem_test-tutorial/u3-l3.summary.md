本讲义围绕核心实体 `mem_test` 的主状态机 `Fsm_t`（7 个枚举状态）展开，只讲控制流、不讲 pattern 数值。七个状态分两组：主链路五态 `Idle_s`（等 START）、`WrCmd_s`（装配并下达写命令）、`Write_s`（逐拍送写数据）、`RdCmd_s`（下达读命令）、`Read_s`（收数据并比对）；以及两个不可恢复的错误陷阱 `AxiError_s`（总线 `Wr_Error/Rd_Error`）、`IntError_s`（非法 pattern）。主链路靠「注册式 valid 握手」推进：每拍先清零四个 valid/ready，握手判据统一写成 `r.Xxx_Vld='1' and Xxx_Rdy='1'`；命令态先注册一拍 valid 再检测 ready，且用 `PatternCnt = Size-1` 判定最后一拍。四种模式在数据态末尾分流：`READONLY` 从 `Idle_s` 直跳 `RdCmd_s`、`WRITEONLY` 写完回 `Idle_s`、`CONTINUOUS` 按 `ContRunning` 从 `Read_s` 回跳 `WrCmd_s` 形成循环（不经过空闲态、错误累计），`STOP` 仅清标志实现「优雅停止」。纯函数 `FsmToInt` 把 7 个细状态合并成 6 个对外 STATUS 码（写侧合并为 WRITING、读侧合并为 READING），状态码 0/1/2/3/6/7 的跳变为将来预留编码空间。错误陷阱由 case 之后的共享段每拍自锁，只有硬件复位能解锁。本讲频繁触发 `InitPattern_v`/`UpdatePattern_v`，其具体算法留待 u3-l4。
