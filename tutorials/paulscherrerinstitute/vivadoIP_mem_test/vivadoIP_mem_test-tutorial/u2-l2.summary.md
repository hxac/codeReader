本讲阐明 `mem_test` IP 的两类配置语义：**运行模式**与**数据 pattern**，对应寄存器 `REG_MODE`(0x0C) 与 `REG_PATTERN_SEL`(0x20)。四种模式（`C_MODE_SINGLE`/`CONTINUOUS`/`WRITEONLY`/`READONLY`）决定状态机分支：SINGLE 写一遍读一遍即停；CONTINUOUS 由 `ContRunning` 标志驱动循环、`ContIter` 计数迭代轮数，写 STOP 仅清 `ContRunning`，当前轮读完后才回 Idle，即「优雅停止」；WRITEONLY 写完即停、READONLY 跳写直读（要求存储器已含相同 pattern）。四种 pattern 由 `InitPattern_v`（进入命令态）与 `UpdatePattern_v`（每拍握手）两个标志触发：Counter 递增、Walking-1 单 1 循环左移、OwnAddress 数据等于字节地址、PRBN 为 16 位极大长度 LFSR 左移反馈（种子 0x6D3F）。写阶段与读阶段用同一套确定性序列，故硬件正常时 `ERRORS` 必为 0；非法 pattern 被 `when others => IntError_s` 显式拦截。选型直觉：查数据线 stuck-at 用 Walking-1，查地址线用 OwnAddress，长时间压力用 PRBN+CONTINUOUS。错误数在 Continuous 全程累计，仅在 START 时清零。
