本讲只精读 `hdl/mem_test_pkg.vhd`，确立 IP 寄存器地图这一软硬契约，内容分三块。其一，子类型 `rd_t`/`wr_t`/`rdata_t`/`wdata_t` 描述「按寄存器编号访问」的接口模型——选通用每寄存器一位的位向量、数据用 32 元素数组，是 AXI-Lite 从机 IP 译码后给出的友好接口，核心逻辑直接用寄存器常量当下标取数。其二，`REG_*` 地址常量换算固定为 `byte_address = 4 × index`，32 个寄存器空间实际只用 14 个；寄存器分三类：触发型 strobe（START/STOP，只写）、配置型（MODE/SIZE/ADDR/PATTERN_SEL，读写，ADDR/SIZE 各两寄存器拼成 64 位）、状态型（STATUS/ERRORS/FERR_ADDR/ITER，只读）。其三，三组枚举 `C_MODE_*`、`C_PATTERN_SEL_*`、`C_STATUS_*` 是软硬共享的取值词典。关键结论：`STATUS` 由核心函数 `FsmToInt` 把内部 FSM 翻译而成，多个细状态合并为一个对外粗状态；状态码在 3 后跳到 6，有意把总线错误与内部错误分组。一次测试顺序写 MODE→PATTERN_SEL→ADDR→SIZE→START，轮询 STATUS 回 IDLE，再读错误寄存器。pattern 算法、C 驱动封装、从机译码机制留给后续讲义。
