本讲围绕核心实体 mem_test.vhd 回答 u3-l3 遗留的两个问题：每拍写/读数据是什么，读到错误后硬件如何数错并记住首个错误地址。引入关键概念：beat（每拍覆盖 B=AxiDataWidth/8 字节）与字节地址经 s=log2(B) 位移位换算、16 位 LFSR 伪随机序列、two-process「只写变化量」惯例。

Pattern 生命周期由两个触发点驱动：InitPattern 在写/读命令态按四选一播种第 0 拍——Counter 递增、Walking-1 循环左移、OwnAddress 等于所在字节地址、PRBN 为 tap 15/13/12/10 的 16 位 LFSR（种子 0x6D3F），读阶段必须重新播种以保证读写同源；UpdatePattern 在每次 valid/ready 握手后按公式推进序列，使 PatternCnt 与序列项一一对应。

判定全部在读阶段 Read_s 完成：RdDat_Data /= r.Pattern 即错一次、Errors 累加，首个错由 FirstErrFound 锁存，地址按 FirstErrAddr = 基地址 + PatternCnt×B 换算；非法 pattern 编号落入 when others => IntError_s 内部错误陷阱。
