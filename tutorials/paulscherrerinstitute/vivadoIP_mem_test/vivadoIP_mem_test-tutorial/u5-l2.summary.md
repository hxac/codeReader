本讲回答「纯 VHDL 源码如何变成 Vivado IP Catalog 里可拖拽、带 GUI 参数、带 C 驱动、可版本管理的 IP 核」。围绕三个文件展开：`scripts/package.tcl` 是封装配方，调用 PSI 的 PsiIpPackage 框架，按载入框架→init→加源码→加依赖库→加驱动→声明 GUI 参数→`package_ip` 七步把 RTL 描述成 IP，产物落盘到仓库根；`component.xml` 是框架产出的 IP-XACT（IEEE 1685）元数据，分头部四元组、总线接口、地址空间、寄存器 memoryMap、模型（视图/端口/参数）、文件集、用户参数七大块，是 Vivado 真正消费的「IP 契约」；`xgui/mem_test_v1_2.tcl` 定义图形参数界面，含三类过程：`init_gui`（搭界面）、`update/validate_PARAM_VALUE`（联动与校验）、`update_MODELPARAM_VALUE`（GUI 值到综合参数的桥）。核心是参数三级映射 `PARAM_VALUE`→`MODELPARAM_VALUE`→VHDL generic，靠同名自动建立，是新增参数的隐式契约。本 IP 显式声明 4 个用户参数，第 5 个 `C_S00_AXI_ID_WIDTH` 由 AXI4 接口自动提取，交由 `bd/bd.tcl` 在 Block Design 中自动传播，详见下讲 u5-l3。端口位宽与地址空间用依赖表达式动态计算，改参数后无需重新封装。
