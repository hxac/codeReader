本讲义精读 `FileOperations` 模块（约 64 行，4 个无类纯函数）。核心是用**正则 `re.search`** 而非 glob 匹配文件名：`re.search` 是「包含」语义，不锚定首尾；`.` 是「任意字符」而非字面点号，写 pattern 须转义 `\.` 并按需加 `^`/`$`。`FindWithWildcard`（只查）与 `RemoveWithWildcard`（查到即删，不可逆）共享 `os.listdir + re.search` 骨架，均属**非递归浅扫描**，仅作用于单层目录内；因 `os.listdir` 把文件与子目录混判且返回裸名字，误匹子目录会在 `os.remove` 处失败。`OpenWithWildcard` 要求**唯一匹配**，多于一个或零个分别抛通用 `Exception`（仅靠消息文本区分，无自定义异常），内部复用 `FindWithWildcard`，返回 `open(...)` 文件对象故支持 `with as f`，默认 `mode="r"`。`AbsPathLinuxStyle` = `os.path.abspath(path).replace("\\", "/")`，先转绝对再统一正斜杠，服务 Windows 上 Vivado/SDK 等 Linux 风格路径工具，对 Linux 是 no-op。本讲埋下的伏笔（类型注解不统一、通用 `Exception`、变量名 `fileName` 实为列表、`AbsPathLinuxStyle` 无测试、两函数代码重复）将在 u5-l3「批判性读源码」系统回收；`AbsPathLinuxStyle` 的跨平台主题延伸至 u3-l2，正则另一用武之地见 u3-l3 `TextReplace`（`re.DOTALL`、非贪婪 `.*?`）。
