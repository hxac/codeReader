本讲是入门单元收尾，回答「如何确认 PsiPyUtils 的代码能跑」。围绕三个最小模块展开：其一，`RunAll.py` 作为测试套件唯一入口，先 `sys.path.append("..")`，再用 `from Tests.TestXxx import *` 聚合 6 个测试模块，最后在 `__main__` 守卫下调用 `unittest.main()`，由框架扫描所有已加载的 `unittest.TestCase` 子类、执行其 `test*` 方法，一句 `cd Tests && python3 RunAll.py` 即跑完共 18 个用例。其二，`sys.path` 是有序搜索列表，`append("..")` 把仓库根目录加入末尾，使扁平布局下的「裸模块名」`from FileWriter import FileWriter` 成立；`..` 相对当前工作目录而非脚本目录，故必须先 `cd Tests`。关键结论有两点：写了这句的测试文件具备「自举」能力，可脱离聚合器单独运行；没写的（`TestTempFile`、`TestTempWorkDir`）靠 `RunAll.py` 的全局注入跑通却无法单独运行。其三，`setUp`/`tearDown` 在每个用例前后各执行一次（即使断言失败 tearDown 仍跑），用于建拆真实文件与目录，保证干净隔离。另引入「命名空间包」概念：`Tests/` 无 `__init__.py`，靠 Python 3 命名空间包机制 + 根目录在 `sys.path` 中才成立 `from Tests.TestXxx` 写法。本讲只把 `FileWriter` 当「能生成文件的对象」，其缓存-落盘与缩进栈机制留到 u2-l3 揭开。
