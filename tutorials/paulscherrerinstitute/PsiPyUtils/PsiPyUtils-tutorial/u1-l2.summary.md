本讲精读 setup.py 与 __init__.py 两个文件，讲清 PsiPyUtils 的包结构与导入方式。关键概念一为扁平布局：8 个功能模块直接摊在仓库根目录、无同名子目录，靠 setup.py 的 package_dir 将当前目录映射为包根、packages 仅声明顶层包，才成为合法包，且 Tests/ 不进入分发包。关键概念二为 __init__.py 的统一重导出：用相对导入把 8 个子模块的名字搬到包顶层，其中 5 个重导出的是类（from .X import X）、3 个重导出的是模块（from . import X），调用方式有别。核心结论：3.0.0 的 import 改动属增量式，新写法 from PsiPyUtils import FileWriter 与旧写法 from PsiPyUtils.FileWriter import FileWriter 都成立且引用同一类对象；导入只依赖标准库，声明却未用的 lxml 非必需；Changelog 标注的「不向后兼容」不等于旧代码立即报错，需回到源码确认。
