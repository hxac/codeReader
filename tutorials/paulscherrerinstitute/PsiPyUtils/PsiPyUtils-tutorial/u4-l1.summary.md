本讲介绍 PsiPyUtils 的 `ExtAppCall`：把「在指定目录执行外部命令并捕获输出与退出码」封装为一个对象。用法三步——构造（传入只读属性 `work_dir`、`command`，构造时不启动子进程；`ignore_stdout`/`ignore_stderr` 已废弃仅为兼容）、执行、取结果。执行模型只有一个引擎：`run_async`（启动子进程后立即返回）+ `wait`（阻塞到结束并读回输出），`run_sync` 只是二者拼接再加 0.1 秒缓冲。`wait(timeout_sec)` 超时时先 `terminate()` 子进程、清理完再抛 `TimeoutExpired`；`timeout_sec <= 0` 表示无限等待（参数名义为秒、docstring 写毫秒，以实现为准）。`get_stdout`/`get_stderr`/`get_exit_code` 须在执行后调用，约定退出码 0 为成功，非零不自动抛异常、需自行检查。工作目录切换复用 u2-l1 的 `TempWorkDir`（`Popen` 须在其 `with` 块内调用），输出写入临时文件而非管道，`TimeoutExpired` 复用标准库；内部机制留待 u4-l2。
