本讲围绕 PsiPyUtils 中最小的模块 `TextReplace.py` 展开，剖析唯一的对外函数 `TaggedReplace(startTag, endTag, text, file)`：它是无状态的纯函数，负责在文件中两个标签之间做受控替换。核心流程为「读 → 用 `re.search` 查标签 → 用 `re.sub` 替换 → 以 `w+` 写回」；标签之所以「保留」，是因为替换串用 `startTag + text + endTag` 把新文本重新包了回去。关键概念有：非贪婪量词 `.*?`（让同一文件多对标签各自独立匹配）、`re.DOTALL`（使 `.` 跨换行，多行内容可替换），并强调检查与替换两处正则标志位必须一致。异常体系区分自定义的 `TagsNotFoundError`（标签缺失，显式抛出）与内置 `FileNotFoundError`（文件不存在，由 `open` 隐式冒泡）。本讲把 Changelog 3.0.1 的两条 bugfix（commit `2aae7fa` 加 `?`、commit `b33d5db` 补 `re.DOTALL`）与源码、git diff 三者对齐，归纳出「原测试仅覆盖单对标签与单行内容」导致多实例、跨行 bug 潜伏的教训。
