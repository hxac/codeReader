本讲是第 2 单元「上下文管理器三剑客」首篇，也是贯穿 PsiPyUtils 全库的 `with` 协议范式起点，仅围绕 `TempWorkDir.py`（约 28 行）与 `Tests/TestTempWorkDir.py` 两个文件展开。最小模块有二：一是上下文管理器协议本身——`__enter__` 在进入 `with` 时执行、`__exit__` 在退出时执行，走 `finally` 语义故即使体内抛异常也必被调用，且因未 `return True` 而不会吞掉异常；二是 os 路径 API 拼出的「记录-切换-还原」三步：`__enter__` 先用 `os.path.abspath(os.curdir)`（等价 `os.getcwd()`）锁定旧目录绝对路径并存入 `self._prevDir`，再 `os.chdir(self._dir)` 切入；`__exit__` 则 `os.chdir(self._prevDir)` 还原。核心结论：顺序必须「先记录后切换」，缓存进入那一刻的绝对路径才能保证还原目标不受体内目录跳转漂移；`__enter__` 隐式返回 `None`，故用法不带 `as`，靠进程全局副作用生效。关键术语：上下文管理器协议、`__enter__`/`__exit__`、`finally` 语义、`self._prevDir` 不变量。本讲确立了后续 `TempFile`、`FileWriter`、`ExtAppCall` 共用的同一对钩子范式，仅被管理资源不同。
