本讲承接 u3-l1，聚焦跨平台路径处理，覆盖两个最小模块：`FileOperations.AbsPathLinuxStyle` 与 `EnvVariables.AddToPathVariable`。先区分两处核心差异——「斜杠方向」（路径内部连接符，Windows 用 `\`、Linux 用 `/`）与「路径分隔符」（环境变量多条路径间的分隔，Windows 用 `;`、Linux 用 `:`）。`AbsPathLinuxStyle` 即 `os.path.abspath(path).replace("\\", "/")`，无条件吐出正斜杠绝对路径，面向只认 Linux 风格的 EDA 工具，在 Linux 上近乎空操作、Windows 上才生效。`AddToPathVariable` 按「按 `sys.platform.startswith(...)` 选分隔符与斜杠方向 → 转换 → 缺失则建 → 已存在则跳 → 否则追加」五步幂等地修改 `os.environ`，仅影响当前进程及子进程；幂等性源于追加前 `split(varSep)` 的精确成员判断，但该判断大小写敏感，在 Windows 上有语义缺口。两函数斜杠转换方向相反（对外给工具用 `/`、对内给本系统用原生斜杠）。`sys.platform` 分支用前缀匹配换前向兼容，并以 `raise Exception("OS Not Supported")` 拒绝 macOS 等未适配系统。两模块均无测试，留待 u5-l3 盘点。
