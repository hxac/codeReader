本讲围绕 setup.py 讲解 PsiPyUtils 打包机制，覆盖三个最小模块：setuptools.setup 元信息登记表（name/version/install_requires 等字段）；package_dir={"PsiPyUtils":"."} 配合 packages，把扁平布局映射为合法顶层包，使 Tests/ 不进分发包；CustomSdist 经 cmdclass 替换内置 sdist，打包前用 shutil.rmtree 清理 dist/ 与 egg-info/，先清后建。核心结论是两条「声明与实现不一致」：install_requires 声明的 lxml 对当前代码冗余（实际用标准库 ElementTree）；Changelog ≠ git 历史，首个 pip 包实为 2.2.0 而非 Changelog 所述 3.0.0，须以 git tag 为准。引入术语：sdist（源码发行物）、egg-info（中间产物）、cmdclass（命令替换钩子）、git submodule（旧分发方式，仍兼容旧式 import）；版本号用 major.minor.bugfix 语义，setup.py 的 version 须与 git tag 手动对齐。
