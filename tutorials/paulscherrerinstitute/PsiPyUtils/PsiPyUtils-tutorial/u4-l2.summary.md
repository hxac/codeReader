本讲打开 `ExtAppCall` 黑盒，回答三个工程「为什么」。核心结论：**用文件而非管道**——把 stdout/stderr 重定向到临时文件（`Popen(stdout=file)`），绕开管道约 64 KiB 内核缓冲写满导致的父子进程死锁，对应 Changelog 1.1.1。`run_async` 在 `with TempWorkDir(work_dir)` 块内用 `os.path.abspath(".")` 取得临时文件**绝对路径**并启动 `Popen`，故块退出、目录还原后 `wait()` 仍能定位文件。**跨平台 `shell` 分支**：Linux 用 `shell=True`（`/bin/sh` 拆分字符串命令），Windows 用 `shell=False`（`CreateProcess` 自行解析），源自 1.1.0。**删除重试循环**为 Windows 句柄滞留兜底，每文件最多 5 次、`sleep(15)`。关键术语：管道缓冲、文件重定向、绝对路径 vs 相对路径、进程级 cwd。批判性观察：注释「5 秒」与代码不符、裸 `except:` 吞 `KeyboardInterrupt`、`stdin=PIPE` 不闭。复用 u2-l1 的 `TempWorkDir`，与 u2-l2 `TempFile` 共享 Windows 句柄铁律；`ExtAppCall` 无测试，缺口留 u5-l3。
