本讲精读 psi_tb_axi_pkg 中三个把 AXI 单拍事务（len=0）封装为一次调用的过程。axi_single_write 完成 AW→W→B 三次握手，整数重载用 to_signed（负数以补码写入）、字符串重载按 base 选 decimal/hex_string_to_signed 以支持 >32 位；总是整字写（wstrb 全 1），AxSIZE 由 log2(wstrb'length) 自动算，末尾不调 init，并用 StdlvCompareStdlv 校验 bresp，非 OKAY 即打印 ###ERROR###。axi_single_read 完成 AR→R 握手，用 msb/lsb/sex 三参数对 rdata 做「截取窗口 [msb:lsb]→右移到第 0 位→按 sex 零扩展或符号扩展」，返回 integer 或与数据线等宽的 signed（后者不受 32 位限制）。axi_single_expect 即 read 加一次比较：整数重载用 IntCompare（十进制消息），字符串重载调 signed 重载的 read 并用 SignCompare2（十六进制消息、任意位宽）。核心结论：「响应错」（rresp/bresp 非 OKAY，由 StdlvCompareStdlv 抓）与「数据不符」（由 IntCompare/SignCompare2 抓）是两条独立的 ###ERROR### 路径，severity error 不中断仿真，由 CI 的 run_check_errors 统一捕获，一次 expect 最多打印两行。
