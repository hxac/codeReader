本讲承接 u2-l1，把上讲拼出的字符串从「打到屏幕」推进到「写进磁盘文件」，是 u6 位真仿真的前置底座。核心建立在 std.textio 的三件套之上：文件类型 TEXT（output 即预定义实例）、行缓冲指针 line（access string，用完即释放）、以及 readline/writeline/read/write 四个原语；屏幕版 print 与文件版 print(file, string) 同构，唯一差别是 writeline 的目标由 output 换成调用方传入的文件句柄。引入的关键术语：str_read（把文件一行变长文本读进定长字符串，先用空格清空、再用带 is_string 的「安静版」read 逐字符取、行尾平静退出，有效字符数为 min(行长, 缓冲长)，短行尾部填空格）、str_write（逐字符写、遇 LF 停止，因每字符都 writeline 而呈「每字符一行」排版）、文件形参语法 file in_file: TEXT。结论：写一行日志用 print(file, string)；文件 I/O 因 testbench 不可综合而可放心使用。
