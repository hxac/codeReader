本讲承接 u1-l2，讲清「脚本如何被执行、CI 如何判通过/失败」，含三个最小模块。其一，仿真流水线：run.tcl（ModelSim）与 runGhdl.tcl（GHDL）逐字相同，仅差 `init` 与 `init -ghdl`，主线为 `init → source config.tcl → compile_files -all -clean → run_tb -all → run_check_errors`；config.tcl 管内容（`create_tb_run`+`add_tb_run` 注册 TB、`compile_suppress`/`run_suppress` 按 ModelSim 消息编号降噪），run.tcl 管流程，二者分离可被多入口复用；interactive.tcl 只编译不运行，供 GUI 调试。其二，`###ERROR###` 是 psi_tb 全库统一的错误前缀，由比较检查等过程在自检失败时打印，run.tcl 末尾扫描它。其三，CI 双重检查：ci.do（`onerror {exit}`+`source run.tcl`+`quit`）包住 run.tcl，ciFlow.py 启动 vsim、读 Transcript.transcript 后独立判定——见 `###ERROR###` 则 exit(-1)（shell 255，应用错误），未见 `SIMULATIONS COMPLETED SUCCESSFULLY` 则 exit(-2)（shell 254，仿真未完成），都过则 exit(0)。核心结论：CI 成功须同时满足「无 `###ERROR###`」与「有成功标记」，二者分别抓 testbench 自检失败与编译/崩溃等框架级异常，构成纵深防御。PsiSim、Transcript、`###ERROR###`、`SIMULATIONS COMPLETED SUCCESSFULLY`、退出码语义等术语将由 u2（txt_util 输出源）与 u3（compare_pkg 拼接错误消息）承接。
