本讲建立 PsiFpgaLibDependencies 的全局地图，是后续讲义地基：该工具解析各 FPGA 库 README.md 中的「PSI 标准依赖声明」并对其执行三类动作，解决多库间依赖管理难题。依赖声明须含独立 `# Dependencies` 段，bullet 列表罗列依赖，缩进表示文件夹层级，单条依赖写作 `[name](url)(minVersion)`（对应数据模型 libraryName/url/minVersion 字段），当前库用 `**[lib]**` 标记作为参照原点，缺失则解析器抛异常。三类动作对应命令行开关：`-list` 仅打印依赖名/版本；`-check` 切到 rootdir 检查依赖目录是否存在并校验版本，缺失报 ERROR；`-checkout` 对缺失依赖执行 git clone 或 submodule add，已存在则跳过并查版本，后两者均调用 CheckCompatibility。本包自身无依赖（声明为 None），当前版本 2.1.0，核心新特性为对已检出依赖做语义版本检查。
