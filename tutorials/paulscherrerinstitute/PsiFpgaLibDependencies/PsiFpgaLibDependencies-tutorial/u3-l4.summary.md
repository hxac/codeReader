本讲围绕 `Actions.py`，讲清 URL 在交给 git 之前如何被改写。三个最小模块：一是 `PSI_GFA_HTTPS_TO_SSH`，一条带前缀门禁的改写函数，仅对以 `https://git.psi.ch/GFA` 开头的地址做三步变换——去掉协议换 `git@`、主机后分隔符由 `/` 变 `:`、末尾补 `.git`，其余地址原样放行；它隐含假设输入不带 `.git`，否则会叠加成双后缀。二是 `URL_REPLACEMENTS`，一个模块级可变的替换函数列表，普通克隆分支在 `git clone` 前用 `for repl in URL_REPLACEMENTS: url=repl(url)` 链式（管道式）地把列表里每个函数依次套到 `dep.url` 上，复合顺序即列表顺序。三是扩展点：新增规则采用两步法——写一个 `(str)->str` 的前缀门禁函数并登记进列表，登记可改源码字面量（永久）或运行时 `append`（不改源码、按需定制）。核心结论：每个选择性改写函数「只动自己认识的、其余放行」是多规则安全叠加的契约前提；但两条克隆分支不对称——普通克隆遍历注册表是扩展点，子模块分支硬编码 `PSI_GFA_HTTPS_TO_SSH` 不是，新增规则只有普通克隆会自动采用。关键术语：URL 替换、链式替换、前缀门禁、开放注册表。
