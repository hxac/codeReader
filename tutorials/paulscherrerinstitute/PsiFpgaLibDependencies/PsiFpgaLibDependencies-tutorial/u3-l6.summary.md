本讲是专家层（u3）收尾篇，讲「改完代码如何打包、定版本、发版」，含三个最小模块：CustomSdist 打包前清理、sdist 打包流程与产物、Tagging 版本策略。核心结论：CustomSdist 继承 setuptools 的 sdist 命令，在重写的 run() 里用 rmtree(ignore_errors=True) 幂等地删除 dist/ 与 *.egg-info，再调 sdist.run(self) 完成标准打包，是典型的模板方法钩子；cmdclass={"sdist": CustomSdist} 是让其生效的开关，删掉则退回默认实现。归档名由 setup() 的 name+version 拼出（PsiFpgaLibDependencies-2.1.0.tar.gz），version 是全项目唯一版本号来源、需手动维护，不与 git tag、Changelog 自动同步。dist/ 被 git 追踪且必须只含当前版本归档，*.egg-info 被 .gitignore 忽略。Tagging Policy 规定：破坏性变更递增 major、新增功能递增 minor、仅修 bug 递增 bugfix，递增高位段时低位归零。注意 VersionNr（u2-l2）负责解析比较版本号，本节负责决定下一版本号，两者共用三段格式。
