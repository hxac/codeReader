本讲承接 u2-l1 的 Dependency 模型，回答「Dependency 列表从何而来」：它们由 `Parse.FromReadme` 从每个 FPGA 库的 `README.md` 解析而来。本讲只覆盖三个最小模块。其一，依赖段格式：PSI 标准 README 须有独立的 `# Dependencies` 段，用 bullet 列表罗列依赖，缩进表示文件夹层级，每条写成 `[name](url)(minVersion)`；库名取方括号内容并去反斜杠（还原转义下划线），url 取第一个圆括号组（强制存在，缺失即 IndexError），version 取圆括号内以数字开头的段。其二，段落定位：逐行扫描，以「去空格小写后含 `#dependencies` 子串」识别起点，以「原始行以 `#` 开头」遇到下一标题时终止。其三，thisRepo 标记：`[**name**](url)`（`**` 须在方括号内）标示「当前库」，其 version 被设为 `"None"`，它是相对路径换算的坐标原点、并从最终结果中排除自身；整段无此标记则抛 `Active repository not marked with **[repo]**`。文件夹树构建（缩进 → Folder/Repo）与相对路径换算（levelsToRoot、`../../` 前缀、排除 thisRepo）留待 u2-l4、u2-l5 展开。
