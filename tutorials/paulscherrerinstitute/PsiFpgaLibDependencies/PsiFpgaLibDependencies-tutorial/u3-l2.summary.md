本讲拆解 `Checkout` 函数的「检出依赖」逻辑。核心引入 `CHECKOUT_MODE` 枚举（`Master`/`LatestRelease`/`SpecifiedRelease`），命令行字符串 `master`/`latest_release`/`specified_version` 由 `ExecMain` 翻译成枚举，且函数签名默认 `Master`、命令行默认 `latest_release`，两条入口默认行为不同。`Checkout` 采用幂等骨架：依赖已存在则跳过并调用 `CheckCompatibility` 查版本，否则按 `Dependency.GetParentDir()` 确定落点、递归建父目录后克隆，全程用 `os.chdir`+`try/finally` 还原 CWD。`asSubmodule` 切换克隆方式——`False` 走 `git clone --recurse-submodules`，`True` 走 `git submodule add`，两条分支对 `URL_REPLACEMENTS` 的应用不对称（普通克隆遍历替换列表，子模块硬编码 `PSI_GFA_HTTPS_TO_SSH`）。`LatestRelease` 的算法意图为 `git tag` → `VersionNr` 解析 → `max()` 取最大语义版本 → `git checkout`，依赖 `VersionNr.__gt__` 的逐段整数比较（`1.10.0` > `1.2.0`）。
