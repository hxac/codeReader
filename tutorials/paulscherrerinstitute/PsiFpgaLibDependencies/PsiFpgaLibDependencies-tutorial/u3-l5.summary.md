本讲讲解 `ExecMain(repoPath, dependencies)`——本包唯一的命令行入口。它用标准库 `argparse` 定义五个选项：`-list/-check/-checkout/-as_submodule` 四个 `store_true` 开关（在场即真、缺席即假），以及用 `choices` 限定三选一的带值 `-mode`。关键分界：`repoPath` 与 `dependencies` 由客户端以 Python 实参传入，命令行只负责表达「动作意图」，二者不可混为一谈。`parse_args()` 读取 `sys.argv` 返回 `Namespace` 后，用三个独立的 `if`（而非 `elif`）分发到 `ListDependencies/CheckDependency/Checkout`，因此多开关可依次全部执行；`-as_submodule` 是仅在 checkout 块内被读取的从属开关。`-mode` 的字符串经 `if/elif` 翻译成 `CHECKOUT_MODE` 枚举后传给 `Checkout`，末尾的 `else: raise` 因非法值已被 `choices` 在 `parse_args()` 阶段拦截，属「防御性死分支」，且其信息里藏有 `Illegel` 拼写错误。本讲引入并承接的关键术语：`store_true`、`choices`、`dest`、`Namespace`、分发（dispatch）、死分支（dead code），以及「两条入口默认 mode 不一致」（命令行默认 `latest_release`，`Checkout` 函数默认 `Master`）。
