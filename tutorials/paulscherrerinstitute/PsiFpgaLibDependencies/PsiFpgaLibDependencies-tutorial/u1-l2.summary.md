本讲覆盖安装、打包、目录结构三个最小模块。安装上，本包以 `dist/` 里的 sdist 归档随仓库分发（未发布 PyPI），用 `pip3 install <归档>` 安装。打包上，`setup.py` 用 `setuptools.setup(...)` 声明 name/version/install_requires（仅 `typing`），并自定义 `CustomSdist` 在每次 `sdist` 前清理旧 `dist/` 与 `egg-info`；归档名由 name+version 拼出，`version` 是全项目唯一的版本号来源。目录布局极扁平，关键配置 `package_dir = {"PsiFpgaLibDependencies": "."}` 把包名映射到根目录，使「仓库根目录即包目录」，5 个源码文件 `__init__.py/Actions.py/Dependency.py/Parse.py/VersionNr.py` 直接散落根目录，由 `__init__.py` 导出 Actions/Dependency/Parse。须区分两种依赖：README 的 `Dependencies: None` 指 PSI FPGA 库依赖为空，而 install_requires 才是 Python 运行时依赖。版本号按 Tagging Policy 的 major/minor/bugfix 语义化规则递增，以 `Changelog.md` 为依据；`dist/` 未被 `.gitignore` 忽略，故归档随版本库分发。本讲为后续 u1-l3（包入口与集成）与 u2-l2（版本号模型）奠定基础。
