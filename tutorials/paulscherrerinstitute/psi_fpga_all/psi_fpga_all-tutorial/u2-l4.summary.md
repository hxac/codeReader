本讲解读 `scripts/packageAllIp.tcl`——4 个驱动脚本中最干净的 IP 批量打包脚本。它沿用 u2-l1 遍历模板：版权头 → `set myPath [pwd]` → 逐个 `cd` 到 `vivadoIP_*/scripts` 后 `source package.tcl` → `cd $myPath` 收尾。其中 `package.tcl` 与仿真侧的 `config.tcl` 对称，住在各 IP 子模块内部，负责声明该 IP 包含哪些 HDL、暴露哪些参数/端口与版本号，被驱动脚本逐个调用。与仿真脚本的关键结构差异是：打包脚本顶部不统一加载任何框架（`PsiIpPackage` 很可能在每个 `package.tcl` 内部各自加载）。脚本当前打包 8 个 IP，而 `.gitmodules` 共声明 11 个 `vivadoIP_*`，差出 `fpga_base`、`sync_edge_det`、`axi_mm_reader` 3 个未打包（原因本仓库未说明，待确认）。核心洞察是「打包集合」与「仿真集合」相互独立：`power_sink` 三种仿真器全不跑却被打包（功耗不可仿真，但可做 IP 喂功耗分析），`axi_mm_reader` 能仿真却不打包，二者方向相反，共同证明一个 IP 在一处状态不能推断另一处。
