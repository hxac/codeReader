psi_fpga_all 是一个 collection-repo（集合仓库），本身几乎不含业务代码，仅按固定目录结构装配一批独立的 FPGA 库。本讲揭示仓库顶层按用途划分为四大类目录，每类聚合同源库：`VHDL/`（可综合核心硬件描述库）、`TCL/`（仿真与 IP 打包工具框架）、`Python/`（自动化脚本工具链）、`VivadoIp/`（已封装好的 Vivado IP 核成品）。

仓库共 23 个 git submodule，分布为 VHDL 5 / TCL 2 / Python 5 / VivadoIp 11。本讲引入三份清单互相印证：`README.md`（入门说明，库列表仅举例性质，只列 9 个且完全漏掉 VivadoIp）、`.gitmodules`（机器与 git 共同认定的权威清单，每条含 path 与相对形式 url）、`Changelog.md`（按发布版本固定子模块版本，分组印证四大类）。核心方法论是"遇到仓库里有什么，以 .gitmodules 为准"。

关键术语包括 gitlink（父仓库记录子模块 commit 的指针）、四大类成员（PsiSim/PsiIpPackage、psi_common/psi_fix/psi_tb/en_cl_fix/psi_multi_stream_daq、PsiFpgaLibDependencies/TbGenerator 等，以及 11 个 vivadoIP_*）。各库在依赖链中位置明确：VHDL 为底层被依赖方，TCL 为驱动方，Python 覆盖周边环节，VivadoIp 为顶层成品。核心结论：path 每一段都参与库间相对路径引用，四大类目录与子模块目录不可随意改名或移动，这是正确性约束而非风格约定（"目录结构即接口"）。检索还发现 sync_edge_det 的命名演变，说明当前以 .gitmodules 为准、Changelog 是历史快照。
