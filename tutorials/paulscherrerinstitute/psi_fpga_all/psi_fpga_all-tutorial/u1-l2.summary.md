本讲承接「collection-repo」认知，讲清 psi_fpga_all 如何用 git submodule 把各 FPGA 库挂到固定目录上。`.gitmodules` 以 INI 风格声明全部 23 条子模块，每条用 `path` 决定本地目录（即「目录结构即接口」的来源）、用 `url` 决定远程仓库；父仓库只保存子模块的 commit 指针（gitlink），不保存文件内容。url 全部写成相对形式 `../../paulscherrerinstitute/<repo>.git`（提交 774a090 引入），使子模块自动继承克隆父仓库时所用的协议，SSH/HTTPS 皆可，无需单独配置。因此必须用 `git clone --recurse-submodules`（或事后 `git submodule update --init --recursive`）克隆，否则 `VHDL/`、`Python/` 等目录存在却为空。README 的 Cloning 章节给出两条克隆命令，唯一区别是协议前缀。关键术语：submodule、`.gitmodules`、path/url、commit 指针、相对 URL、递归克隆。
