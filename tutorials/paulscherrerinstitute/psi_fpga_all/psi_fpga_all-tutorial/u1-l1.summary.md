本讲是整本手册的起点，只读 README.md，回答「psi_fpga_all 是什么」。核心结论：它是一个 collection-repo（集合仓库），本身几乎不含代码，而用 git submodule 把 PSI 的全部 FPGA 库按固定目录结构聚合；最关键的设计约束是「目录结构即接口」——各库之间用相对路径互相引用，目录一旦改动引用就会断裂。讲义围绕三块展开：仓库定位与 Purpose 章节；Maintainer/Authors/Changelog 指引（维护者为 Benoit Stef，作者为 Oliver Bründler 与 Benoit Stef，变更记录置于独立的 Changelog.md，README 仅放指引）；更新策略——约每 3 个月整体更新一次，两次更新之间不保证各子模块为最新，但可单独更新某一子模块，即「整体快照 + 个别更新」。引入的关键术语：collection-repo、git submodule、submodule 指针、目录结构即接口、相对路径互引、版本固定、Changelog。仓库共分 TCL、VHDL、Python、VivadoIP 四大类库；下一讲 u1-l2 讲解 submodule 机制与克隆方式。
