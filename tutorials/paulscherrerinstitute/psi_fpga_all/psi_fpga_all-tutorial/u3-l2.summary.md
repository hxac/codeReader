本讲覆盖集合仓库维护的三个着力点：`.gitmodules`、README 的 Cloning 章节、`scripts/.gitignore`。核心结论有三：其一，新增 submodule 只有一个模板——`path` 取 `<类别目录>/<仓库名>`（类别限 VHDL/TCL/Python/VivadoIp 四选一，体现「目录结构即接口」），`url` 统一用相对形式 `../../paulscherrerinstitute/<仓库名>.git`，标题名须与 path、url 末尾仓库名三者一致。其二，提交 774a090 把全部子模块 url 从绝对 SSH 改为相对 URL，使子模块自动继承父仓库克隆协议（SSH 或 HTTPS），实现协议无关、双用户零配置可用，相对 URL 只「协议无关」并不「org 无关」。其三，`scripts/.gitignore` 采用「先否后肯」白名单策略：`*` 全屏蔽、`!*.tcl` 与 `!.gitignore` 放行手写驱动脚本、末行 `psi_sim_run.tcl` 用「后行覆盖前行」把 .tcl 结尾的运行产物重新排除。关键术语：相对 URL、gitlink、白名单（ignore-all-except）、insteadOf；承接概念「gitlink ≠ Changelog」要求接入新库后须手工补 Changelog。
