本讲义（u3-l3）是整本手册的收尾，把前三讲拆解的内容装回一条端到端工作流：克隆→仿真→打包。三段顺序固定，不可颠倒。第一段用 `git clone --recurse-submodules`（SSH 与 HTTPS 仅协议前缀不同，子模块用相对 URL 自动继承协议）拉下父仓库与全部子模块，使 `VHDL/`、`TCL/`、`VivadoIp/` 等目录非空——这是后两段的硬前提。第二段仿真自检由三个驱动脚本 `runModelsim.tcl`（无参 `init`）、`runGhdl.tcl`（`init -ghdl`）、`runVivado.tcl`（`init -vivado`）分别对应三种后端，共享 PsiSim 的 init→configure→compile_files→run_tb→run_check_errors 流水线，统一判据为日志中不出现 `###ERROR###` 即通过，「跑完」≠「通过」。第三段由 `packageAllIp.tcl` 驱动，逐个 `source` 各 `package.tcl`，不加载 PsiSim、无统一错误标志、需逐个看日志。关键术语：四个驱动脚本共享隐藏前提 `pwd == scripts/`（由 `set myPath [pwd]` + `$myPath/../` 决定）；打包集合≠仿真集合，`power_sink` 不仿真却打包、`axi_mm_reader` 能仿真却不打包为镜像反例。
