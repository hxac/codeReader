本讲解读 scripts/ 下三个仿真驱动脚本 runModelsim.tcl、runGhdl.tcl、runVivado.tcl。三者共享同一骨架：先 source 加载 PsiSim.tcl 框架、再 namespace import psi::sim::* 使其命令可裸名调用，随后执行五步流水线 init→configure→compile→run→check。init 用参数选仿真器后端（默认即 ModelSim / -ghdl / -vivado）；configure 以 set myPath [pwd] 锚定起点，逐库 cd 到 <lib>/sim 后 source 该库 config.tcl 登记源文件与 testbench，跳过一个库即将其 cd 与 source 两行整块注释。三段执行 compile_files -all -clean、run_tb -all、run_check_errors "###ERROR###" 在三脚本中几乎逐字相同；###ERROR### 是 self-checking TB 自检失败时主动打印的标志，仿真跑完不等于通过，日志不出现它才算通过。三脚本真正差异仅在 init 参数与 configure 阶段启用了哪些库，库与仿真器的兼容性矩阵（含 power_sink 为何不参与仿真）留待 u2-l3 展开。
