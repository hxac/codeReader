本讲义精读 texlab wiki 的 `LSP-Internals.md`「Custom Messages」部分，讲清 texlab 如何在标准 LSP 之上自定义两个请求 `textDocument/build` 与 `textDocument/forwardSearch`。核心机制是：这两个请求是**可选**的、由客户端决定是否支持；它们不占用标准方法名，而是在 `initialize` 握手时通过 `experimental` capability（键名 `textDocumentBuild` / `textDocumentForwardSearch`）声明存在，客户端用标准 JSON-RPC 2.0 报文发起请求，服务器据 `result.status` 回执。本讲引入的关键术语包括 `BuildTextDocumentParams`（含 `textDocument` 与可选 `position`，position 用于与正向搜索联动）、`BuildResult`、`TextDocumentPositionParams`（标准类型、position 必填）、`ForwardSearchResult`，以及两个状态枚举：`BuildStatus`（Success / Error / Failure / Cancelled）与 `ForwardSearchStatus`（Success / Error / Failure / Unconfigured）。核心结论是：两枚举前 3 值对称（0/1/2），第 4 值不同——build 是 Cancelled（编译是长任务可被取消，且 `build.executable` 默认 latexmk 故无需「未配置」态），forwardSearch 是 Unconfigured（正向搜索瞬时无需取消，而 `forwardSearch.*` 默认 null 故需此态）。`Cancelled` 可由 `texlab.cancelBuild` 触发；`Unconfigured` 是排查「PDF 不跳」的第一线索。
