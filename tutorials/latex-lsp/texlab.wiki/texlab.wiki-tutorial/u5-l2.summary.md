本讲覆盖 texlab 中「编辑器内呈现 LaTeX 结构」的四组配置，对应大纲、补全、悬停、内联提示四类 LSP 方法（documentSymbol / completion / hover / inlayHint），它们只改变「看到」的样子、不影响编译结果。三个最小模块：其一，文档符号过滤与自定义环境——`texlab.symbols.allowedPatterns` / `ignoredPatterns` 用正则按「白名单先、黑名单后」过滤符号，且过滤是**递归**的（父节点被移除时子节点仍保留并上提）；`texlab.symbols.customEnvironments`（`SymbolEnvironmentOptions[]`，`name` 必填、`displayName` 默认 title case）把自定义环境纳入大纲，按 Enum 类（`SymbolKind=10`）呈现。其二，补全匹配 `texlab.completion.matcher` 四取值 = {`prefix`,`fuzzy`} × {区分大小写, `-ignore-case`}，默认最宽松的 `fuzzy-ignore-case`；`prefix` 严（从头连续）、`fuzzy` 宽（按序可断），它只决定「留哪些」。其三，`texlab.hover.symbols` 仅作用于符号类命令（如 `\epsilon`），取值 `none` / `glyph` / `image`，默认 `image` 且自动回退到 `glyph`；`texlab.inlayHints.labelDefinitions` / `labelReferences` 默认均为 `true`（`\label`/`\ref` 内联提示默认开），`maxLength` 默认 `null`（不截断）。关键术语：文档符号树、递归过滤、matcher、符号类命令、Inlay Hints。承上呼应 u4-l3 枚举映射，启下指向 u5-l4 的自定义命令/环境扩展点。
