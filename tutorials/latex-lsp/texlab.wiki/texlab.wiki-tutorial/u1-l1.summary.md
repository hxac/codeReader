本讲是整本手册的起点，确立 texlab 定位并导览 wiki。核心结论：texlab 是基于 LSP（语言服务器协议）的 LaTeX/BibTeX 语言服务器，既非编辑器、也非 TeX 引擎或 PDF 阅读器，而是把三者缝合的协调者。LaTeX 工作流由四个角色协作：编辑器（LSP 客户端）、texlab（LSP 服务器）、TeX 引擎、PDF 阅读器。其中只有编辑器↔texlab 走标准 LSP（JSON-RPC）加 texlab 自定义扩展；texlab 与引擎、阅读器之间则靠调用外部命令（executable + args + %f/%p/%l 占位符）交互，由 texlab.build 与 texlab.forwardSearch 配置驱动。关键认知：配置由 LSP 客户端持有、texlab 按需查询。本仓库是纯文档 wiki，共 7 个页面，建议顺序为 Home→Project-Detection→Configuration→Previewing→Tectonic→LSP-Internals→Workspace-commands，其中 Configuration 最厚且横跨多子系统。引入的关键术语包括 LSP、语言服务器、协调者、SyncTeX、tectonic、自定义消息、枚举映射、workspace/executeCommand，供后续讲义承接。
