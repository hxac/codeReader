本讲拆开比较损失的两步黑盒。第一步 `get_comp_loss` 产生候选打分：对每个候选算 token 级 NLL（因果 shift、`reduction="none"`），求和得响应段总 NLL，取负后除以序列全长得「归一化对数似然」`prod`，再 softmax 成和为 1 的预测相对得分 `prod_normalized`；其几何本质是各 token 概率乘积按全长开根的「几何平均似然」，对个别极小概率 token 极敏感。第二步 `compare_loss` 用 `unsqueeze` 广播减法一次性构造预测差矩阵 `diff` 与真实差矩阵 `rw_diff`（均反对称，保证每个无序对至多罚一次），再以掩码 `aval` 挑选可学习对：真实差 `rw_diff>0.2`（显著性门槛，过滤评分噪声的近 ties）且预测差 `diff<0.3`（目标准距，达到即「毕业」停梯度）。损失为对 `aval` 中被选对求和的 `(0.3−diff)`，是带边距的成对 hinge 排序损失。0.2、0.3 分属真实分与 softmax 预测分两个不同尺度，故不相等；两者构成「可学习边距」机制——`aval` 随训练自动收缩，是「还剩多少排序没学好」的实时指标。`aval` 只做选择不参与求导，梯度沿 softmax→对数似然→logits 回传。本讲数学在 u3-l3 显存优化中保持不变。
