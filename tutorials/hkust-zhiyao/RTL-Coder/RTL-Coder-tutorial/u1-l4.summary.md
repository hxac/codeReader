本讲详解 RTL-Coder 流水线三份关键 JSON 的格式。三者均采用逐行 JSON（JSONL）存储，可流式读取；`Response` 统一存成列表（即便单答案也用单元素列表包裹），使生成、训练、评分共用一套读写逻辑。三种格式：生成样本 `data_sample.json` 为 `{Instruction, Input, Response[1]}`，是 GPT 造数据产物；成品训练集 `Resyn27k.json`（约 2.7 万条）刻意丢掉 `Input`，只剩 `{Instruction, Response[1]}`，供标准 SFT 用 `Response[-1]` 取训练目标；评分训练样本为 `{Instruction, Input, Response[N], Score[N]}`，多候选与分数按位置一一对应，末位候选通常为参考答案（Score=1）。核心结论：`Response` 统一为列表是理解三种格式的钥匙；`mle.py` 只取参考答案、`mle_scoring.py` 遍历全部候选并读 Score，两者消费方式不同正是两种数据格式并存的根本原因。
