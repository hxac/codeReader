本讲是 RTL-Coder 训练总览，介绍三种逐级增强的方案：标准 SFT（mle.py，单参考答案、显存最省）、评分训练（mle_scoring.py，一条指令配 N 个带分数的候选，引入域损失与比较损失）、评分训练+梯度切分（mle_scoring_grad_split.py，覆写 training_step 逐候选前向以降显存）。三者共享 DeepSpeed ZeRO-2 与超参，仅 batch 与累积步数不同，有效 batch 对齐在 256。评分方案的共享数据管线有两处关键：ScoreDataset 加载时按「指令 token 数 < model_max_length×0.5」过滤长样本，且 `__getitem__` 返回伪装成 input_ids 的原始字典；DataCollatorForSupervisedDataset 把一条样本展开成 N 条 query+response 序列（指令用 -100 掩码）并产出 idxs 与 scores，由 CompareTrainer.compute_loss 用 idxs 把 logits reshape 回 (batch, N, L, V)。损失公式与显存技巧留作黑盒，交由 u3-l1/u3-l2（损失数学）、u3-l3（梯度切分）展开。
