本讲总览 RTL-Coder 的数据生成流程。因公开 RTL 训练数据稀缺，项目用 GPT-3.5 自动造出约 2.7 万条「需求指令 + 参考 Verilog 代码」配对样本，汇总成 Resyn27k 数据集。流程分三阶段：领域关键词准备 → 指令生成（变异）→ 参考代码生成（对应 README 的 Figure 1 / data_gen_flow）。仓库仅完整 shipped 阶段 2 的驱动 `instruction_gen.py`；阶段 1 用种子 `data_sample.json` 代替，阶段 3 复用 `utils.py` 的 `askGPT35(is_response=True)` 路径。

引入的关键术语：三阶段共用同一个 GPT 调用工具 `askGPT35`，靠 `is_response` 开关切换——False 造题目（空系统提示），True 造代码（「Professional Verilog coder」系统提示）。代码分两层：`instruction_gen.py` 负责编排（加载、循环、响应解析、ROUGE-L 0.7 去重、断点续跑落盘），`utils.py` 提供工具（`askGPT35` 及 `load_json`/`save_json` 逐行 JSONL 读写）。变异提示词模板 `p_example.txt` 通过 `#method#`（功能变但方法/器件相似）、一个原题→改写题示例，以及 `{Instruction}`/`{Input}` 占位符控制产出格式，占位符既是填空位也是切分定位点。

核心结论：GPT 仅作数据标注工具、不参与最终模型；RTLCoder 基于开源底座（DeepSeek-coder/Mistral）微调，推理时不调用任何 OpenAI 接口。
