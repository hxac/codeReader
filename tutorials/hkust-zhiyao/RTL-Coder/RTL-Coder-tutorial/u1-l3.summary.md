本讲围绕 README 的最小可运行脚本，拆解 RTLCoder 推理的三个最小模块：① 用 AutoTokenizer 与 AutoModelForCausalLM 加载模型，统一以 fp16 减半显存，device_map 传整数在多卡中挑选一张卡整模型放置；② 构造专业 Verilog prompt，项目统一范式为「自然语言描述（含端口与行为）+ 模块签名骨架」，复杂电路用分节专业模板、简单电路用一行极简模板；③ 调用 model.generate 采样生成，讲清 temperature 控制随机性、top_p 做核采样裁剪、max_length 是 prompt 加续写的总长上限，并需显式 do_sample=True 才真正采样。引入的关键术语有 CausalLM、Tokenizer、fp16、device_map、采样与贪心解码。核心结论：prompt 质量决定生成质量；Mistral 版会自动停止，Deepseek 版需按 endmodule 关键字做截断后处理；decode(sample[0]) 会回显 prompt，基准脚本用切片跳过。后处理细节留待后续讲义承接。
