本讲解决 RTL-Coder 用 6.7B 底座微调时单卡放不下的问题，核心是一份共用的 DeepSpeed 配置 `ds_stage_2.json`。训练状态在 AdamW+fp16 下每卡需存参数(2Ψ)、梯度(2Ψ)、优化器状态(12Ψ)，共 16Ψ，6.7B 约 107GB，必须切分。引入概念：**ZeRO** 分三阶段切分训练状态，**Stage 2** 只切优化器状态+梯度、保留完整参数，是省显存与少通信的折中，每卡显存降至 2Ψ+14Ψ/Nd（4 卡约 37GB）；Stage 3 全切但通信更重。**offload_optimizer**（device=cpu、pin_memory 页锁定内存）把最胖的优化器状态分片搬到 CPU，GPU 侧再降至约 2Ψ+2Ψ/Nd，属以时间换空间，自动启用 DeepSpeedCPUAdam。**fp16 动态 loss scale**（initial_scale_power=16→S=65536、溢出减半、千步稳定翻倍、hysteresis 防震荡）避免梯度下溢。**torchrun --nproc_per_node=4** 起多进程，配置中大量 `"auto"` 字段由命令行回填，使一份配置同时服务 SFT、评分、梯度切分三种方案，有效 batch 均对齐 256（4×2×32 与 4×1×64）。贯穿结论：评分训练因每样本展开 N 候选、logits 膨胀而比 SFT 更依赖 offload，与 gradient_checkpointing、u3-l3 梯度切分（系统侧/命令行侧/算法侧）共同省显存。
