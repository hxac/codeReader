本讲剖析 `mle_scoring_grad_split.py` 的梯度切分（gradient splitting）方案，解决评分训练因每样本前向 C 条候选而 `batch=1` 仍 OOM 的问题。基线 `mle_scoring.py` 沿用默认 `training_step`，一次前向整批候选、一次反传，所有候选激活与 `(B,C,L,V)` logits 同时驻留，单步峰值正比于候选数 C。切分方案覆写 `training_step`，拆为两阶段：阶段一算「表征梯度」`represent_grad`——在 `no_grad` 下前向得候选得分 `prod`，将其登记为叶子张量后对代理损失反传，得到 C 维方向向量，模型参数不入反传图；阶段二循环 C 次逐候选前向+反传，以 `represent_grad[i]` 为标量权重，每帧计算图用完即弃，使峰值降为与 C 无关（∝1），代价是 C 次前向换显存。关键术语：叶子张量、代理梯度、解耦加权。须注意 shipped 文件是带瑕疵原型，`get_sftscore`、`device`、`amp` 未定义且阶段二 `idxs` 标量化与 `compute_loss` 不兼容，不能直接运行。
