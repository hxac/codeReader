本讲讲解源信息（SourceInfo）：记录每条硬件来自源码何处的元数据，供报错时指回行号。它是 sealed 家族，含 SourceLine（带位置）、UnlocatableSourceInfo（技术限制兜底）、DeprecatedSourceInfo（废弃 API 兜底）。核心是两套编译期机制：机制 A「materialize 隐式宏」挂在 SourceInfo 伴生对象，编译期读 enclosingPosition 把行号焊成 SourceLine 字面量，运行时零开销；机制 B「SourceInfoTransform 黑盒改写宏」把 `+`、`Wire`、`VecInit` 等公开 API 改写成内部 `do_` 方法并插入 `implicitly[SourceInfo]`，以兼容链式 apply。二者协作：B 改写出的 `implicitly[SourceInfo]` 由 A 解析成 SourceLine；而 RegNext 因支持命名/默认参数（Scala 宏盲区）改走普通隐式。源信息最终随 `pushOp`/DefPrim 写进 IR 节点，经 `Builder.error` 用于定位。与 u7-l1 区别：插件管「命名」，本讲管「定位」，二者同为编译期设施却各管一摊。
