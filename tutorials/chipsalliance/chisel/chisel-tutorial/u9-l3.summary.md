本讲辨清 Chisel 中两套同名却独立的 debug 机制。(A) Debug 层（`chisel3.layers.Verification.Debug`）是内建 Extract 子层，`printf`/`flush` 在 `SimLog.printfWithoutReset` 中被 `layer.block`（带 `skipIfAlreadyInBlock`/`skipIfLayersEnabled` 两个开关）自动收容，以实现「断言可综合、打印不可综合」的解耦，默认生效。(B) `chisel3.debug` 包用 Java 反射（`CtorParamExtractor`、`ClassParam`）抽取模块构造器参数并序列化成 JSON，由 `DebugIntrinsics.generate` 遍历内部 IR，发射 `circt_debug_moduleinfo`/`var`/`subfield`/`enumdef` 四类内建函数，经 `addSecretCommand` 秘密命令通道进 CHIRRTL。(B) 受 `EmitDebugIntrinsicsAnnotation`（`--with-experimental-debug-intrinsics`，默认关且 `Unserializable`）控制，由 `AddDebugIntrinsics` Phase 在 Elaborate 后、Convert 前注入并消费注解保幂等；`SuppressDebugParams` marker trait 让跨模块依赖的类型（如 `MixedVec`）跳过冗余 `params=`。两条通路由不同源码路径、不同编译阶段实现，完全独立、互不影响。
