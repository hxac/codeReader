本讲覆盖 chisel3.util 四个位级工具。BitPat 是纯软件对象（非 Data、非 Element），用 value/mask/width 三元组描述带无关位（?）的位模式，? 在 value 与 mask 中均记 0，匹配语义为 value === (x & mask)，因此不能用于 switch/is，需用 when + ===；它经宏桥接改写为 do_===。Cat 把多个信号拼成更宽的 UInt，约定第一个参数为 MSB，靠入口 reverse 抵消 SeqUtils.asUInt「首元素为 LSB」的约定，最终产出单条 ConcatOp。OneHot 家族中 UIntToOH 即 1.U<<in，OHToUInt 用分治 log2 求独热位位置，PriorityEncoder 借 PriorityMux 取最低位 1 的下标（无 1 时未定义），PriorityEncoderOH 多了全 0 兜底。Lookup/ListLookup 是以 BitPat 为键、按列 foldRight 叠成 Mux 树的查表器，靠前映射优先，属遗留 API，建议新代码改用 when 链或 MuxLookup。四者皆遵循「只登记命令、由 firtool 综合」原则。
