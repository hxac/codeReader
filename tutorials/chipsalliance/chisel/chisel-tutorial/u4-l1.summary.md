本讲解析 Chisel 细化（elaboration）期间运行的全局状态机 `Builder`，回答「谁在登记命令、登记到哪里、如何收拢成电路」。核心结论：`Builder` 是一个单例门面，自身几乎不存数据，所有 `Builder.xxx` 访问器都转发到当前 `DynamicContext`，并提供 `build` 入口绑定 context、收尾组装 `Circuit`。关键澄清：命令（`Command`）不存在 `DynamicContext` 中，而存在每个模块的 `_body: Block` 里，`blockStack` 仅是「当前 Block 指针栈」。本讲引入的关键术语：`DynamicContext`（一次细化账本，含 `currentModule`、`components`、`whenStack`、`blockStack`、`errors` 等）、`ChiselContext`（线程级，存 `IdGen`）、`HasId`+`IdGen`（每个硬件对象的全局唯一 `_id` 与 `_parent`）、`Namespace`（压缩 HashMap 命名去重器）、`Builder.State`（快照恢复机制）。完整链路为：`Elaborate` new 出 `DynamicContext` → `Builder.build` 用 `DynamicVariable` 绑定 → `Module.evaluate` 经 `State.guard` 守卫 → `BaseModule` 构造把 `currentModule` 指向自身 → 构造体经 `pushCommand` 灌命令 → `generateComponent` 收口 → 汇入 `components` → 组装 `Circuit`。本讲为后续 u4-l2（内部 FIRRTL IR）与 u4-l3（Binding 系统）奠基。
