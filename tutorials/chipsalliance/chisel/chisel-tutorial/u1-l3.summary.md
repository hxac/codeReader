本讲义讲解 Chisel 仓库的目录结构与子项目划分。Chisel 构建由一个根 `build.mill`（集中 Scala 版本、依赖坐标于 `object v`、公共 trait 如 `ChiselCrossModule`/`HasScalaPlugin`）加每个子目录的 `package.mill`（声明各自 `moduleDeps`）组成，这是 mill 的嵌套构建写法。共有 7 个带源码子项目：`core`（主体源码，含 `Data`/`Module`/`Bundle`/`Vec` 等核心词汇）、`firrtl`（旧 FIRRTL IR 残骸）、`macros`（编译期宏，注入源信息）、`plugin`（scalac 编译器插件，制品名 `chisel-plugin`，按完整 Scala 版本交叉编译）、`src/main`（即 `chisel` 模块，整合层加 `chisel3.util` 生成器）、`svsim`（SystemVerilog 仿真底层库）、`stdlib`（较新的 `chisel3.std`）。核心依赖图：`core → firrtl + macros`，`chisel(src/main) → core + svsim` 并经 `HasScalaPlugin` 的 `-Xplugin` 自动挂载 `plugin`，`stdlib → chisel`；`firrtl/macros/svsim/plugin` 无内部依赖。关键结论是代码归属原则：触及 `chisel3.internal` 私有 API 的代码归 `core`，纯 Chisel（仅用公开 API）的代码归 `src/main` 或 `stdlib`。最终面向用户的 `chisel` 制品由 `unipublish` 聚合而成。后续讲义将按此分层逐子项目深入。
