本讲把 Chisel 的「一行生成 Verilog」打开成一张**四段式架构**心智地图，作为后续 u4（Builder 机制）、u5（Stage 与 CIRCT）的导航图。核心结论：Chisel 编译器是一条单向流水线——①前端 API（`chisel3.*`）→ ②Builder（`chisel3.internal.Builder`）→ ③内部 FIRRTL IR（`chisel3.internal.firrtl`）→ ④发射（现代由 `Converter`+CIRCT/firtool 承担）。

关键术语：**elaboration（细化）**——运行 Scala 构造体长出电路；**IR（中间表示）**——Chisel 私有的 Scala 对象树，语法酷似 FIRRTL；**Phase/Stage**——把编译拆成可组合小步骤，靠注解（Annotation/AnnotationSeq）传递数据；**CIRCT/firtool**——LLVM 硬件编译框架，把 FIRRTL 方言编译成 SystemVerilog。

两个反直觉要点：前端 API「**只登记不施工**」，每条调用经 `Builder.pushCommand` 追加进当前模块命令队列，本身不产硬件；内部 IR 是三层树 `Circuit ⊃ Component ⊃ Command`。Builder 是 elaboration 期的全局状态机，`DynamicContext` 持有 `currentModule` 与 `components`，`build` 收尾时组装成 `ElaboratedCircuit`。`ChiselStage` 用 `PhaseManager` 按 DAG 拓扑序串联 `Elaborate → Convert → CIRCT`，`emitSystemVerilog` 是其用户入口。

重要提醒：README 描述的是**概念模型**（写作 `chisel3.firrtl.*`/`Emitter`），源码事实是 `chisel3.internal.firrtl`，且 `Convert` 阶段自 7.11.0 起已 deprecated，发射路径正从「经 firrtl.ir」迁移到「直接用 `ElaboratedCircuit`」。Verilog 文本最终由第④段 CIRCT 产出。
