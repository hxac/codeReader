本讲讲解 Chisel 如何调用 firtool。自 6.0 起，`circt.stage.phases.CIRCT` 这个 Phase 把 firtool 当作**外部子进程**调用：将 elaboration 出的电路序列化成 CHIRRTL 文本经 stdin 喂入，从 stdout 读回 SystemVerilog，用 `os.proc(cmd).call` 驱动，非 0 退出码抛 `FirtoolNonZeroExitCode`；firtool 二进制由 `firtool-resolver` 库按 `BuildInfo.firtoolVersion` 自动解析或下载，也可手动指定路径。

`CIRCTTarget`（CHIRRTL/FIRRTL/HW/Verilog/SystemVerilog/Btor2 六种）与 `CIRCTTargetAnnotation` 决定 firtool 输出何种产物；目标为 CHIRRTL 时提前返回、根本不启动 firtool。

`firtoolOpts` 的透传分四道关卡：用户 `Array[String]` 参数 → `FirtoolOption` 注解 → `CIRCTOptionsView` 投影成结构化的 `CIRCTOptions.firtoolOptions: Seq[String]` → 原样追加进子进程命令行，字符串保序不改、校验交由 firtool 负责。

需澄清的是，规格中提到的 `CIRCTConverter` 与 `CIRCTPassManager` 已 `@deprecated`，对应旧版「进程内」调用 CIRCT 的路线，当前主路径完全不引用它们；但其方法名仍精确标示 firtool 内部的 lowering 链 CHIRRTL → Low FIRRTL → HW → SystemVerilog。
