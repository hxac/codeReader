本讲讲解 Chisel 面向验证与可观测性的两个特性：Layer（层）与 Probe（探针）。

Layer 将断言、打印、监视器等代码隔离进「编译期可选层」，区别于 `when` 的运行期选择——层块整段代码仅在 Verilog 编译期被启用时才存在，仿真可开、交付可删。层由声明（继承 `chisel3.layer.Layer` 的 object）与层块 `layer.block(L){...}` 构成；`LayerConfig` 分 Extract（落地为独立模块 + SystemVerilog `bind`，靠 include 文件启用）与 Inline（靠 `` `ifdef `` 宏启用）两种约定，可嵌套成树且子层启用必带父层。内建层 `Verification`（含 `Assert`/`Assume`/`Cover`/`Debug` 及 `Temporal` 子层）覆盖常见验证场景，标准 API 自动入层，尤其 `printf` 经 `SimLog` 落入 `Verification.Debug`。

Probe 是对硬件的「引用」，落地为 SystemVerilog 层次名、不占普通端口；`Probe`/`ProbeValue` 只读，`RWProbe`/`RWProbeValue` 读写，探针只能 `define` 恰好一次、`read` 读回，读写探针支持 `force`/`release`，`Data` 上的 `ProbeInfo` 记录可写性与层颜色。层着色探针 `Probe(T, color)` 遵循偏序：可向本层或子层写、从本层或父层读，由 `Layer.canWriteTo` 的祖先判定保证「可选代码不影响主设计」。
