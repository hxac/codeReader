本讲讲清 Chisel 如何构建、测试与本地发布。Chisel 用 mill（而非 sbt）构建：根目录 `./mill` 是自动下载并启动 mill 的 wrapper，`build.mill` 是构建定义本体，其 `mill-version` 头锁定 mill 版本。`object v` 是版本与依赖的单一事实来源，统一管理 Scala 交叉版本（2.13/3）、从 `etc/circt.json` 读取 firtool 版本，并基于 git 距上一个 tag 的提交数算出形如「tag+N-hash-SNAPSHOT」的版本号。仓库拆成 `core`/`firrtl`/`macros`/`plugin`/`src`/`svsim` 等子模块，公共交叉编译逻辑抽到 trait `ChiselCrossModule`，`chisel` 经 `moduleDeps` 依赖 `core`、`svsim`，并由 `HasScalaPlugin` 在编译期自动挂载 `chisel-plugin`。三条核心命令：`chisel[].compile` 编译、`chisel[].test` 跑 ScalaTest、`chisel[].unitTest` 把标记 UnitTest 的电路测试经 firtool 编译、circt-test 执行。firtool 由 `object circt` 按 circt.json 自动下载（或用 `CIRCT_INSTALL_PATH` 指向已解压目录跳过）；`unipublish` 把多模块聚合成名为 `chisel` 的制品，`publishLocal` 同时发布库与插件到本地 Ivy 仓库，供别的项目在 `build.sbt` 引用该版本。
