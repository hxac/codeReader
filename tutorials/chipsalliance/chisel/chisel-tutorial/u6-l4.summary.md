本讲讲解 chisel3.util 的三类条件工具。switch/is 是嵌套 when/elsewhen 的语法糖：Scala 宏把 switch 块改写成对 SwitchContext 的链式 .is 调用，SwitchContext.is 再为每个 is 生成 cond===v 判断；is 的值必须是字面量且互斥。Mux 家族是"表达式版"条件选择：基础 Mux（登记 MultiplexOp 原语）为积木；MuxLookup 按 key 查表、综合成 mux 链，是 switch 的表达式对应物，且在映射覆盖 key 全部取值时省去 default；MuxCase、PriorityMux 按首个命中优先；Mux1H 假设选择信号独热，综合成浅而快的 AND-OR 树。Counter 是内联（非 Module）硬件计数器，返回 (value, wrap) 元组；范围从 0 起始且长度为 2 的幂时，靠寄存器自然溢出回绕而省掉回绕 mux，否则补显式清零。核心结论：switch/is 与 MuxLookup 是同一功能的"语句版/表达式版"；独热用 Mux1H、优先用 PriorityMux；所有工具都遵循"只登记不施工"，最终 RTL 由下游 firtool 产出。
