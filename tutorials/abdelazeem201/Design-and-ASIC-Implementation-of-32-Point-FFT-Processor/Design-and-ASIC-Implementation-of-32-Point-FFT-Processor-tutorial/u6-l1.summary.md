本讲精读 `SYN/scripts/syn.tcl`，讲清 32 点 FFT 从 RTL 到 UMC 130nm 标准单元网表的**综合（synthesis）**全流程。综合把抽象行为算子映射为带面积/延迟/功耗的具体门电路，由 Synopsys `dc_shell` 执行。脚本分五阶段：环境设置、读入、约束、优化、输出。引入的关键术语：`target_library`（映射目标，取 ss 最差角保余量）、`link_library`（可引用库，含 ss/ff 两角做多角分析）、`search_path`（写死的机器相关路径，换机必须改）、工艺角（ss 查 setup、ff 查 hold）、`analyze`（语法分析）/`elaborate`（建层次、推断算子）、`uniquify`（重名实例唯一化）、多轮编译策略 `compile → optimize_registers（重定时）→ compile_ultra`。约束（cons.tcl 的 10 ns 时钟）由 `check_design → source → link` 引入。最终四产物分工：`.v` 门级网表给后端 P&R、`.sdc` 给后端约束、`.sdf` 给门级仿真反标、`.ddc` 给 DC 备份。核心结论：多轮优化使 `clk` 路径 slack=0.00、关键路径 9.68 ns，恰好压线满足 100 MHz。约束细节留给 u6-l2，报告解读留给 u6-l3。
