学完本讲，你应当能够：

本讲只解决一个问题：**最后一级流水线吐出来的 32 个频域样本是乱序的，我们如何把它们恢复成正常顺序？** 答案就是位反转。

## 2. 前置知识

阅读本讲前，请确认你已经理解以下内容（来自前置讲义）：

- **radix-2 DIF 的分治结构**（u2-l1）：32 点 FFT 分成 5 级 stage，每级做「加法分支 + 减法×旋转因子分支」，最后一级（stage 5）旋转因子恒为 1，只剩纯加减。本讲的「乱序」正是这种分治方式带来的副作用。
- **二进制表示**：一个 0~31 的整数可以用 5 位二进制表示，例如 `3 = 00011`、`24 = 11000`。如果不会二进制 ↔ 十进制互转，建议先复习。
- **Python 基础**：字符串切片 `s[::-1]` 表示反转字符串；`int(x, 2)` 表示把二进制字符串转成整数。本讲的代码会用到这两点。

> 提示：位反转是 DIF 算法的「代价」。DIT（时间抽取）算法的乱序出现在**输入端**，DIF 的乱序出现在**输出端**。本项目用的是 DIF，所以乱序在输出端，这正是本讲要处理的位置。

### 4.1 位反转索引的概念

#### 4.1.1 概念说明

所谓「位反转」，就是把一个数的二进制位**从左到右颠倒过来**。对 32 点 FFT 来说，索引用 5 位二进制表示，所以是「5 位反转」。

举几个例子（5 位二进制）：

| 索引 i（十进制） | 二进制 | 反转后二进制 | 反转后（十进制） |
| --- | --- | --- | --- |
| 0 | `00000` | `00000` | 0 |
| 1 | `00001` | `10000` | 16 |
| 2 | `00010` | `01000` | 8 |
| 3 | `00011` | `11000` | 24 |
| 16 | `10000` | `00001` | 1 |
| 31 | `11111` | `11111` | 31 |

数学上，若索引 \(i\) 的 5 位二进制为 \(b_4 b_3 b_2 b_1 b_0\)（即 \(i = \sum_{k=0}^{4} b_k \cdot 2^k\)），那么它的位反转定义为把位顺序倒过来：

\[
\mathrm{bitrev}(i) = b_0 \cdot 2^{4} + b_1 \cdot 2^{3} + b_2 \cdot 2^{2} + b_3 \cdot 2^{1} + b_4 \cdot 2^{0}
\]

更紧凑地写成：

\[
\mathrm{bitrev}(i) = \sum_{k=0}^{4} b_k \cdot 2^{\,4-k}
\]

位反转有一个重要性质：**它是对合的（involution）**，即对同一个值做两次位反转就回到原值：

\[
\mathrm{bitrev}(\mathrm{bitrev}(i)) = i
\]

这意味着「把位置 `i` 的数搬到位置 `bitrev(i)`」和「把位置 `bitrev(j)` 的数搬到位置 `j`」是同一件事，方向无所谓——这一点在后面理解硬件实现时很有用。

#### 4.1.2 核心流程：为什么 DIF 输出是位反转的？

这是本模块最关键的理解点。我们用「分治」的视角看 radix-2 DIF（回顾 u2-l1）：

- **第 1 级**：把 32 个样本按「加法分支 / 减法分支」一分为二。加法分支恰好产生**偶数频率** \(X[0], X[2], X[4], \dots\)，减法分支产生**奇数频率** \(X[1], X[3], X[5], \dots\)。也就是说，第 1 级决定了结果的**最低位（LSB，bit 0）**：减法分支的数，其频率索引的 bit0 = 1。
- **第 2 级**：在每一半内部再分治，决定频率索引的 **bit 1**。
- **第 3 级**：决定 **bit 2**。
- **第 4 级**：决定 **bit 3**。
- **第 5 级**：决定 **bit 4（MSB）**。

问题来了：**第一级（最早算出来的）决定的是 bit 0（最低位），最后一级（最晚算出来的）决定的是 bit 4（最高位）。** 而数据在数组里是按「先放加法分支、再放减法分支」的顺序排的——也就是「先高位后低位」地堆放。

最终结果就是：**数组里第 `i` 个位置存的，其实是频率索引 `bitrev(i)` 的结果**。换句话说，输出顺序是「位反转」的。这是 radix-2 DIF 的固有特性，不是 bug。

用一个伪代码描述这个现象：

```
# stage5 算完后，stage5_r[i] 里实际放的是 X[bitrev(i)]
# 要得到正常顺序，需要做一次"反向搬运"：
for i in 0..31:
    正常顺序结果[bitrev(i)] = stage5_r[i]
```

### 4.2 Python 位反转还原实现

#### 4.2.1 概念说明

知道了位反转是什么，下一步就是看懂 `FFT.py` 是怎么用它把乱序结果整理成正常顺序的。核心思路非常简单：

> **遍历 stage 5 的每一个输出位置 `i`，把它的值搬到「正常顺序数组」的第 `bitrev(i)` 个位置上。**

`FFT.py` 用了一种很 Pythonic 的写法，把「格式化成 5 位二进制 → 反转字符串 → 转回整数」三步压在一行里完成。

#### 4.2.2 核心流程

还原流程可以分成三步：

1. **格式化**：`'{:05b}'.format(i)` —— 把整数 `i` 格式化成宽度为 5、不足补 0 的二进制字符串。例如 `1 → '00001'`、`3 → '00011'`。
2. **反转**：`[::-1]` —— Python 切片，把字符串整体反转。例如 `'00001' → '10000'`。
3. **转回整数**：`int(..., 2)` —— 以 2 为基数把二进制字符串解析回整数。例如 `'10000' → 16`。

三步合起来就是 `bitrev(i)`，然后执行搬运 `final_ans_r[bitrev(i)] = stage5_r[i]`。

伪代码：

```
for i in 0..31:
    r = bitrev(i)                         # 位反转后的目标位置
    final_ans_r[r] = stage5_r[i]          # 把乱序值搬到正常位置
    final_ans_i[r] = stage5_i[i]
```

#### 4.2.3 源码精读

下面这段就是项目里位反转还原的全部实现。注意它是在 stage 5 算完之后才执行的：

```python
print('-------------final answers------------- ')
for i in range(len(stage5_r)):
    r = int('{:05b}'.format(i)[::-1], 2)
    final_ans_r[r] = stage5_r[i]
    final_ans_i[r] = stage5_i[i]
```

逐行解释这段 [FFT.py:189-196](https://github.com/abdelazeem201/Design-and-ASIC-Implementation-of-32-Point-FFT-Processor/blob/89ce7665eca8b77a0c48e610dab11928108566b6/SIM/FFT.py#L189-L196)：

- 第 191 行 `r = int('{:05b}'.format(i)[::-1], 2)`：算出 `i` 的 5 位位反转值 `r`。这就是 4.1 讲的 `bitrev(i)`。
- 第 192~193 行 `final_ans_r[r] = stage5_r[i]` / `final_ans_i[r] = stage5_i[i]`：把 stage 5 第 `i` 个位置的实部、虚部，搬到 `final_ans` 的第 `r` 个位置。`final_ans_r / final_ans_i` 是在文件开头就初始化好的 32 元素数组 [FFT.py:169-170](https://github.com/abdelazeem201/Design-and-ASIC-Implementation-of-32-Point-FFT-Processor/blob/89ce7665eca8b77a0c48e610dab11928108566b6/SIM/FFT.py#L169-L170)。
- 循环走完 32 次，`final_ans` 里就是按频率索引 \(0,1,2,\dots,31\) 正常排列的 FFT 结果。

随后第 195~196 行只是把整理好的正常顺序结果打印出来：

```python
for i in range(len(stage5_r)):
    print(i,' : ',int(final_ans_r[i]),int(final_ans_i[i]),'i')
```

> 一个常见疑问：为什么是 `final_ans_r[r] = stage5_r[i]`，而不是反过来？因为 stage 5 的第 `i` 个位置存的是频率 `bitrev(i)` 的结果（见 4.1.2），我们要把它放回频率 `bitrev(i)` 对应的正常位置 `r = bitrev(i)`。又因为位反转是对合的，这里写成 `final_ans[bitrev(i)] = stage5[i]` 和「正向」理解完全一致。

#### 4.2.5 小练习与答案

**练习 1**：把 `bitrev5` 改写成「不使用字符串、纯整数运算」的版本（用按位与、移位）。

> **答（示例代码）**：
> ```python
> def bitrev5_int(i):
>     r = 0
>     for k in range(5):
>         r = (r << 1) | ((i >> k) & 1)   # 每次把 i 的第 k 位接到 r 的低位
>     return r
> ```
> 思路：从 `i` 的最低位（k=0）开始取，依次「挤」进 `r` 的低位，最终 `i` 的低位变成 `r` 的高位。结果与字符串版完全相同。

**练习 2**：`FFT.py` 里 `'{:05b}'` 中的 `05` 能不能改成 `4`（即 `'{:04b}'`）？为什么？

> **答**：不能。32 点索引用 5 位二进制表示（因为 \(2^5 = 32\)）。若改成 4 位，索引 16~31 会被截断或格式化错误，位反转结果也就错了。位反转的「位宽」必须等于 \(\log_2 N\)，本项目里固定是 5。

### 4.3 正常顺序输出的需求

#### 4.3.1 概念说明

到目前为止，我们都在讲软件 `FFT.py`。但你可能会问：**硬件 `FFT.v` 也需要做位反转吗？**

答案是肯定的。硬件流水线的最后一级（第 5 级）吐出来的 32 个样本，和软件 stage 5 一样是乱序的。如果直接把这个乱序结果送出芯片，下游模块就完全没法用了——频域样本必须按频率 \(0,1,2,\dots,31\) 的正常顺序排列才有意义。

所以硬件里也必须有一个「排序模块」，把乱序样本整理成正常顺序。在本项目里，这个模块叫 **SORT**，它的核心就是 `FFT.v` 里那张大型 `case(y_1)` 查找表。

> 这里只建立「概念对应」，不展开硬件实现细节——因为硬件那张表的映射关系并不简单等于 `bitrev(y_1)`，需要结合流水线的反馈结构仔细验证。**这件事留给后续讲义 u4-l2 完成。**

#### 4.3.2 核心流程

软件和硬件做「同一件事」的对照如下：

| 维度 | 软件 `FFT.py` | 硬件 `FFT.v` |
| --- | --- | --- |
| 触发时机 | stage 5 算完后，单独一段循环 | 第 5 级每吐出一个样本，就立刻查表写入 |
| 位反转计算 | 一行公式 `int('{:05b}'.format(i)[::-1], 2)` | 一张 `case(y_1)` 查找表，把到达样本写到对应数组下标 |
| 暂存结构 | `final_ans_r[0:31]` 一维数组 | `result_r[0:31]` 寄存器数组（二维存储） |
| 输出 | 按顺序打印 | 等 32 个样本写满后，再按 `count_y` 顺序读出 `dout` |
| 还原后 | 正常顺序的频域结果 | 正常顺序的频域结果 |

#### 4.3.3 源码精读（概念性对应）

硬件排序模块的核心位置：

[FFT.v:323-453](https://github.com/abdelazeem201/Design-and-ASIC-Implementation-of-32-Point-FFT-Processor/blob/89ce7665eca8b77a0c48e610dab11928108566b6/RTL/FFT.v#L323-L453) —— 这段 `case(y_1)` 就是硬件版的「位反转还原表」。每当第 5 级输出一个样本（用计数器 `y_1` 标识它是第几个到达的样本），就根据 `y_1` 的值把样本写到 `result_r_ns[某个下标]`。

例如表里第一项（[FFT.v:324-329](https://github.com/abdelazeem201/Design-and-ASIC-Implementation-of-32-Point-FFT-Processor/blob/89ce7665eca8b77a0c48e610dab11928108566b6/RTL/FFT.v#L324-L329)）：

```verilog
case((y_1))
5'd0 : begin
    result_r_ns[31] = out_r[23:8];
    result_i_ns[31] = out_i[23:8];
end
```

它的含义是：**第 0 个到达的样本（`y_1 == 0`）被写到存储数组的第 31 号位置**。这和软件里「stage 5 第 `i` 个样本搬到 `final_ans[bitrev(i)]`」的**目的**是一样的——把乱序还原成正常顺序。

> ⚠️ 注意：不要误以为硬件表里「`y_1 == 0` 写到 [31]」就等于 `bitrev(0) = 0`。实际映射并非简单的 `bitrev(y_1)`（你可以自己核对：`y_1==1` 写到 [15]、`y_1==2` 写到 [7]……）。这是因为硬件的「样本到达顺序」与软件 `stage5` 数组的下标顺序并非一一相同，还受到 SDC 反馈式流水线结构的影响。**这张表是否「整体等价于一次 5 位位反转」，需要结合 u4 的流水线数据流才能证明，本讲不展开，留给 u4-l2。**
>
> 本讲你只需要记住结论：**软件用一行位反转公式还原顺序，硬件用一张查找表做同样目的的事，二者在「把乱序整理成正常顺序」这一目标上完全对应。**

#### 4.3.5 小练习与答案

**练习 1**：如果把这个 32 点设计扩展成 64 点（见 u7-l3），位反转的「位宽」应该改成多少？

> **答**：改成 6 位。因为 \(64 = 2^6\)，索引用 6 位二进制表示，所以 `FFT.py` 里要改成 `'{:06b}'.format(i)[::-1]`，对应做一次 6 位反转。

**练习 2**：位反转为什么是「无损」的整理操作（不丢失任何频域信息）？

> **答**：因为位反转是一个**排列（permutation）**——它只是把 32 个样本的位置重新编排，每个样本都被搬到唯一的新位置，既不增加也不减少样本。它是对合的（\(\mathrm{bitrev}(\mathrm{bitrev}(i))=i\)），所以还原是可逆的，所有频域信息完整保留。

## 5. 综合实践

把本讲知识串起来，完成下面这个贯穿任务：

**任务**：用 Python 复现 `FFT.py` 的「stage 5 → 正常顺序」这一段，但**不要照抄** `'{:05b}'.format(i)[::-1]`，而是用你自己写的、纯整数运算的位反转函数（4.2.5 练习 1 的版本）。

**步骤**：

1. 假装你手上已经有「乱序」的 stage 5 结果——你可以手动构造一个长度为 32 的列表 `stage5 = list(range(32))`（值等于下标，方便观察搬运效果）。
2. 用你写的 `bitrev5_int(i)` 把它还原成正常顺序 `final_ans`。
3. 打印 `final_ans`，验证它是否与「直接用 `FFT.py` 原始公式」得到的结果完全一致。

**参考实现（示例代码）**：

```python
# 示例代码：用纯整数位反转复现 FFT.py 的输出整理
def bitrev5_int(i):
    r = 0
    for k in range(5):
        r = (r << 1) | ((i >> k) & 1)
    return r

stage5 = list(range(32))                 # 假装的乱序结果（值=下标，便于观察）
final_ans = [0] * 32
for i in range(32):
    final_ans[bitrev5_int(i)] = stage5[i]

# 与 FFT.py 原始公式对照
ref = [0] * 32
for i in range(32):
    ref[int('{:05b}'.format(i)[::-1], 2)] = stage5[i]

print('final_ans == ref ?', final_ans == ref)   # 预期: True
print('1 搬到了位置', final_ans.index(1))        # 预期: 16
print('3 搬到了位置', final_ans.index(3))        # 预期: 24
```

**预期结果**：程序打印 `final_ans == ref ? True`，且值 `1` 被搬到了位置 `16`、值 `3` 被搬到了位置 `24`——这正对应 `1→16`、`3→24` 的位反转映射。

> 待本地验证：在你的 Python 环境运行上述脚本，确认输出与预期一致。

## 6. 本讲小结

- **位反转**是把索引的二进制位颠倒；32 点 FFT 用 5 位反转，例如 `1(00001)→16(10000)`、`3(00011)→24(11000)`。
- radix-2 **DIF 的输出天然是位反转乱序的**，因为最早算的第一级决定的是频率索引的最低位，最后算的第五级决定最高位，导致「先低位后高位」地堆放。
- `SIM/FFT.py` 用一行 `r = int('{:05b}'.format(i)[::-1], 2)` + `final_ans_r[r] = stage5_r[i]` 还原正常顺序，位反转在项目里**只出现这一次**。
- 位反转是**对合的排列**，可逆、无损；位宽必须等于 \(\log_2 N\)（本项目为 5）。
- 硬件 `RTL/FFT.v` 里有一张大型 `case(y_1)` 查找表（SORT 模块）做**目的相同**的事，但其映射并非简单的 `bitrev(y_1)`，详细验证留给 u4-l2。
