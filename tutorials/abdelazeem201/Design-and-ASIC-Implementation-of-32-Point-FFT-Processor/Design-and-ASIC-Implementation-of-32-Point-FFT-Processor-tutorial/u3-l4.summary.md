本讲拆解 ROM_16/8/4/2 四个模块，阐明它们身兼两职：一是用计数器分段生成 2 位 `state` 信号（`2'b00` 等待、`2'b01` first half、`2'b10` second half）去驱动 radix2 蝶形单元的三态机；二是用 `case` 查表在 second half 输出 24 位定点旋转因子供复数乘法。核心结论是：四级阈值 N 逐级减半为 16/8/4/2，且 N 同时等于等待/前半/后半各段拍数、该级 shift 延时深度、旋转因子个数与蝶形对距离，是 SDC 架构的关键参数。ROM_16 为单计数器风格（`count` 直接分段，含 `2'b11` 禁止态与 valid 自保持锁存，count=47 收尾）；ROM_8/4/2 为双计数器风格（`count` 判等待、`s_count` 判前半后半并查表），其 `next_valid` 属未驱动的死代码。各级 valid 借助 valid 菊花链（上一级蝶形 `outvalid` 当下一级 ROM 的 `in_valid`）自动对齐数据流。第 5 级无 ROM，旋转因子退化为常数 256+j0，`state` 由顶层 `no5_state` 生成。
