# radix-2 DIF FFT 算法原理

## 1. 本讲目标

本讲是算法基础篇的第一讲。学完后你应当能够：

- 说清楚 **DFT 与 FFT 的关系**，以及为什么 FFT 能把 \(O(N^2)\) 的运算降到 \(O(N\log_2 N)\)。
- 说清楚 **radix-2 DIF（Decimation-In-Frequency，频率抽取）** 蝶形运算的两个分支：加法分支与"减法 × 旋转因子"分支。
- 逐级看懂 Python 参考模型 `SIM/FFT.py` 中 stage1 ~ stage5 的 **数据分组规律**（32→16→8→4→2→1）。
- 理解每一级末尾的 **复数乘旋转因子** 结构，并能手算一个小例子。

本讲只讲算法，不碰 Verilog 硬件实现（那是 u3、u4 的事）。读者只要懂复数运算和"分治"思想即可。

## 2. 前置知识

### 2.1 离散傅里叶变换（DFT）

把一段长度为 \(N\) 的离散信号 \(x[0], x[1], \dots, x[N-1]\) 变成它的频谱 \(X[0], \dots, X[N-1]\)，标准定义是：

\[
X[k] = \sum_{n=0}^{N-1} x[n] \cdot e^{-j\frac{2\pi}{N}kn}, \quad k=0,1,\dots,N-1
\]

记 **旋转因子**：

\[
W_N = e^{-j\frac{2\pi}{N}} = \cos\!\Big(\tfrac{2\pi}{N}\Big) - j\sin\!\Big(\tfrac{2\pi}{N}\Big)
\]

则 \(X[k] = \sum_{n} x[n]\, W_N^{kn}\)。

直接按定义算，每一个 \(X[k]\) 要做 \(N\) 次复数乘法，\(N\) 个 \(X[k]\) 一共 \(N^2\) 次。本项目 \(N=32\)，\(32^2=1024\) 次复数乘法——还能接受；但 \(N=1024\) 时就要 100 万次，这就是 FFT 要解决的问题。

### 2.2 复数乘法

两个复数 \((a_r + j\,a_i)\) 与 \((c + j\,d)\) 相乘：

\[
(a_r + j\,a_i)(c + j\,d) = (a_r c - a_i d) + j\,(a_i c + a_r d)
\]

即：**实部 = 实·实 − 虚·虚**，**虚部 = 虚·实 + 实·虚**。这是 4 次乘法、2 次加减（俗称"4 乘 2 加"）。`FFT.py` 用的就是这种标准形式；硬件 `radix2.v` 后面会优化成"3 乘 5 加"，那是 u3-l2、u7-l1 的内容，本讲先按下不表。

### 2.3 分治思想

把一个大问题拆成两个结构相同的"半规模"小问题，递归下去，再把结果合并——这就是分治。FFT 的核心就是：把一个 \(N\) 点 DFT 拆成两个 \(N/2\) 点 DFT。

> 术语提示：DIF（频率抽取）是把 **输出** \(X[k]\) 按 \(k\) 的奇偶拆分；DIT（时间抽取，Decimation-In-Time）是把 **输入** \(x[n]\) 按 \(n\) 的奇偶拆分。本项目用的是 DIF。

## 4. 核心概念与源码讲解

### 4.1 DFT 与 FFT 的关系

#### 4.1.1 概念说明

DFT 的"贵"在于：每一个输出 \(X[k]\) 都要遍历全部 \(N\) 个输入，所以总共 \(N^2\) 次复数乘法。FFT 的发现在于：旋转因子 \(W_N^{kn}\) 有极强的周期性与对称性，例如 \(W_N^{kN/2} = -1\)，可以让"一半的项"和"另一半的项"共享计算。

radix-2 算法要求 \(N\) 是 2 的幂。它把一个 \(N\) 点 DFT 拆成两个 \(N/2\) 点 DFT，递归 \(\log_2 N\) 层后就只剩 1 点 DFT（1 点 DFT 就是它本身）。于是总运算量降为：

\[
\text{复数乘法} \approx \tfrac{N}{2}\log_2 N
\]

对 \(N=32\)：\(\frac{32}{2}\times 5 = 80\)，比 1024 小了一个数量级。这就是"快"的来源。

#### 4.1.2 核心流程

`FFT.py` 把这个递归过程 **展开成 5 个顺序的串行 stage**（\(\log_2 32 = 5\)），每个 stage 就是一次"全数组扫描 + 蝶形运算"，不递归、不调用函数。这是硬件友好的写法——硬件流水线正是按这 5 级来搭的。

整体流程伪代码：

```
读入 32 个复数样本 x[n] = mem[n] + j·img[n]
预置 16 个旋转因子 W_32^k = w[k] + j·w_i[k]
stage1: 32 点 → 两组 16 点
stage2: 每组 16 点 → 两组 8 点（共 4 组 8 点）
stage3: 每组 8 点 → 两组 4 点（共 8 组 4 点）
stage4: 每组 4 点 → 两组 2 点（共 16 组 2 点）
stage5: 每组 2 点 → 两组 1 点（共 32 个 1 点，即最终频谱，但乱序）
位反转还原 → 正常顺序的 X[k]
```

### 4.2 radix-2 DIF 蝶形运算

#### 4.2.1 概念说明

DIF 的关键一步：把 \(N\) 点 DFT 的输出 \(X[k]\) 按 \(k\) 的 **奇偶** 拆成两组，每组恰好是一个 \(N/2\) 点 DFT。把求和式按 \(n\) 拆成前半（\(0 \dots N/2-1\)）与后半（\(N/2 \dots N-1\)）两段，并利用 \(W_N^{kN/2} = (-1)^k\)，可以得到（令 \(a[n]=x[n]+x[n+N/2]\)，\(b[n]=x[n]-x[n+N/2]\)）：

\[
\begin{aligned}
X[2r]   &= \sum_{n=0}^{N/2-1} a[n]\, W_{N/2}^{rn} \\
X[2r+1] &= \sum_{n=0}^{N/2-1} \big(b[n]\cdot W_N^{n}\big)\, W_{N/2}^{rn}
\end{aligned}
\]

物理含义非常清晰：
- **偶数频率输出**（\(X[0],X[2],X[4],\dots\)）由 **加法** \(a[n]=x[n]+x[n+N/2]\) 再做 \(N/2\) 点 DFT 得到。
- **奇数频率输出**（\(X[1],X[3],X[5],\dots\)）由 **减法后再乘旋转因子** \(b[n]\cdot W_N^n\) 再做 \(N/2\) 点 DFT 得到。

这就是 **radix-2 DIF 蝶形**，画成图就是一个交叉的"X"形：

```
 x[n] ────────●─────────────→  a = x[n] + x[n+N/2]      （加法分支，→ 偶数频率）
        ╲     │     ╱
         ╲    │    ╱  乘旋转因子 W_N^n
          ╲   │   ╱
 x[n+N/2] ───●──→  b = x[n] - x[n+N/2]  →  b·W_N^n      （减法×旋转分支，→ 奇数频率）
```

注意与 DIT 的区别：DIF 是 **先加减、后乘旋转因子**；DIT 是先乘旋转因子、后加减。本项目选 DIF，是因为它的"先加减"天然适配硬件的延时寄存器（shift）。

#### 4.2.2 核心流程

对一级蝶形，设跨度为 \(S=N/2\)（即配对的两个样本相距 \(S\) 个位置），旋转因子序列为 \(W_N^0, W_N^1, \dots, W_N^{S-1}\)：

```
对 i = 0 .. S-1：
    add_r[i]   =  x[i]     +  x[i+S]          # 加法分支（实部）
    add_i[i]   =  xi[i]    +  xi[i+S]         # 加法分支（虚部）
    sub_r[i]   =  x[i]     -  x[i+S]          # 减法分支
    sub_i[i]   =  xi[i]    -  xi[i+S]
    mul_r[i]   =  sub_r[i]·wr[i] - sub_i[i]·wi[i]   # 减法 × 旋转因子（实部）
    mul_i[i]   =  sub_i[i]·wr[i] + sub_r[i]·wi[i]   # 减法 × 旋转因子（虚部）
本级输出 = [add_0..add_{S-1}, mul_0..mul_{S-1}]      # 前半是加法，后半是减法×旋转
```

把这一级的输出前半（加法结果）当作下一级的一个 \(N/2\) 点输入、后半（减法×旋转结果）当作另一个 \(N/2\) 点输入，再各做一次同样的蝶形，就递归下去了。

读到这里你会发现，stage1 的输出 32 个数的排列是 `[16 个加法结果, 16 个减法×旋转结果]`，这正是 DIF "偶数频率在前、奇数频率在后"的排列约定，下一级会把这两半各自当作一个独立的 16 点输入继续拆。

#### 4.2.4 代码实践：手算一个 4 点 DIF 蝶形

**目标**：用一个最小的 4 点例子，验证 DIF"加法分支 / 减法×旋转分支"的正确性。

设输入（实数，虚部为 0）：\(x[0]=1,\ x[1]=2,\ x[2]=3,\ x[3]=4\)。4 点 DIF 的旋转因子：\(W_4^0=1\)，\(W_4^1=e^{-j\pi/2}=-j\)。

**Stage 1（跨度 \(S=2\)，配对 (x[0],x[2]) 与 (x[1],x[3])）**：

\[
\begin{aligned}
A[0] &= x[0]+x[2] = 1+3 = 4 \\
A[1] &= x[1]+x[3] = 2+4 = 6 \\
A[2] &= (x[0]-x[2])\cdot W_4^0 = (1-3)\cdot 1 = -2 \\
A[3] &= (x[1]-x[3])\cdot W_4^1 = (2-4)\cdot(-j) = 2j
\end{aligned}
\]

**Stage 2（跨度 \(S=1\)，配对 (A[0],A[1]) 与 (A[2],A[3])，旋转因子均为 \(W_2^0=1\)）**：

\[
\begin{aligned}
B[0] &= A[0]+A[1] = 4+6 = 10 \\
B[1] &= A[0]-A[1] = 4-6 = -2 \\
B[2] &= A[2]+A[3] = -2+2j \\
B[3] &= (A[2]-A[3])\cdot 1 = (-2-2j)
\end{aligned}
\]

**位反转还原（2 位反转索引）**：\(0(00)\to0,\ 1(01)\to2,\ 2(10)\to1,\ 3(11)\to3\)，所以：

\[
X[0]=B[0]=10,\quad X[1]=B[2]=-2+2j,\quad X[2]=B[1]=-2,\quad X[3]=B[3]=-2-2j
\]

**验证**（用 DFT 定义手算 \(X[1] = x[0] + x[1](-j) + x[2](-1) + x[3](j) = 1-2j-3+4j = -2+2j\)）✓ 完全一致。

**操作步骤**：
1. 在纸上按上面流程推一遍，自己任选一组输入（例如 \(x=[2,0,0,0]\)，结果应为四个都等于 2）。
2. 把 4 点 DIF 的两步画成蝶形图，标出加法分支和减法×旋转分支。

**预期结果**：手算结果与 DFT 定义直接代入的结果完全相同，说明"加法分支→偶数频率、减法×旋转分支→奇数频率"的拆分是正确的。

#### 4.2.5 小练习与答案

**练习 1**：DIF 蝶形中，"乘旋转因子"发生在加/减之 **前** 还是之 **后**？

**答案**：之后。DIF 是先做 \(x[n]\pm x[n+N/2]\)，再对减法结果乘旋转因子；而 DIT 是先乘旋转因子再加减。

**练习 2**：为什么 stage1 的输出要排成"前 16 个加法、后 16 个减法×旋转"，而不是交错排列？

**答案**：因为前 16 个对应偶数频率 \(X[0],X[2],\dots,X[30]\)，后 16 个对应奇数频率 \(X[1],X[3],\dots,X[31]\)；下一级把这两半各当作一个独立的 16 点 DFT 继续拆，"连续分组"最方便用循环（`for x in range(2)`）顺序处理。

### 4.3 五级 stage 的数据分组

#### 4.3.1 概念说明

DIF 把 \(N\) 点逐级对半拆，本项目 \(N=32=2^5\)，所以恰好 5 级。每一级有三个会变化的参数：

- **组数 \(G\)**：当前一级把数组分成多少个独立的"小 DFT"。组数每级翻倍：1 → 2 → 4 → 8 → 16。
- **组的大小 \(L\)**：每个小 DFT 有多少点。组大小每级减半：32 → 16 → 8 → 4 → 2。
- **跨度 \(S=L/2\)**：蝶形配对两点的距离。跨度每级减半：16 → 8 → 4 → 2 → 1。
- **旋转因子索引步进**：每级蝶形用的旋转因子从 \(W_N^0\) 开始，步进随 \(L\) 变化。

| Stage | 组数 \(G\) | 组大小 \(L\) | 跨度 \(S\) | 旋转因子索引 | 对应 |
|-------|--------|---------|--------|----------------|------|
| 1 | 1 | 32 | 16 | `w[i]`         (步进 1) | \(W_{32}^i\) |
| 2 | 2 | 16 | 8  | `w[i*2]`       (步进 2) | \(W_{16}^i = W_{32}^{2i}\) |
| 3 | 4 | 8  | 4  | `w[i*4]`       (步进 4) | \(W_{8}^i = W_{32}^{4i}\) |
| 4 | 8 | 4  | 2  | `w[i*8]`       (步进 8) | \(W_{4}^i = W_{32}^{8i}\) |
| 5 | 16| 2  | 1  | `w[i*16]`      (步进 16)| \(W_{2}^i = W_{32}^{16i}\) |

这张表是本讲最重要的"地图"——看懂它，就能看懂 `FFT.py` 的全部循环。

#### 4.3.2 核心流程

每一级的循环模板（以 stage2 为例）：

```
for x in range(G):              # 遍历每个组，组基址 = L*x
    for i in range(S):          # 加法分支：组内前 S 个 = 组内第 i 个 + 第 i+S 个
        add[i + L*x] = in[i + L*x] + in[i+S + L*x]
    for j in range(S):          # 减法分支
        sub[j + S*x] = in[j + L*x] - in[j+S + L*x]
    for i in range(S):          # 减法 × 旋转因子
        out[L*x + S + i] = sub[i + S*x] * W[步进*i]
```

唯一的"魔法"是旋转因子索引 `W[步进*i]`：随级数加深，步进翻倍（1→2→4→8→16），因为越往后的小 DFT 用到的旋转因子越稀疏，本质都是在引用同一组基础旋转因子 `w[0..15]`（即 \(W_{32}^0 \dots W_{32}^{15}\)）的不同子集。

#### 4.3.3 源码精读

**stage2**（2 组 × 16 点，跨度 8，旋转因子 `w[i*2]`）：

### 4.4 复数乘旋转因子结构

#### 4.4.1 概念说明

每一级蝶形的"减法 × 旋转因子"分支，做的就是复数乘法 \((b_r + j\,b_i)\cdot(W_r + j\,W_i)\)。展开：

\[
(b_r + j\,b_i)(W_r + j\,W_i) = \underbrace{(b_r W_r - b_i W_i)}_{\text{实部}} + j\underbrace{(b_i W_r + b_r W_i)}_{\text{虚部}}
\]

这是 **4 次实数乘法 + 2 次实数加减**（俗称 4 乘 2 加），`FFT.py` 各级都是这么写的。注意：脚本里的 `w_i` 已经带好了负号（\(W_i = -\sin(\cdot)\)），所以式子里直接写 `+` 即可，不再额外加负号。

旋转因子本身从哪来？\(W_{32}^k = \cos(\pi k/16) - j\sin(\pi k/16)\)，即 `w[k] = cos(πk/16)`、`w_i[k] = -sin(πk/16)`，共 16 个（\(k=0\dots 15\)），存满了 32 点 DIF 第一级需要的全部旋转因子。后续各级只是按步进复用它们的子集。

#### 4.4.2 核心流程

复数乘法的计算流程：

```
输入: 减法结果的实部 sub_r、虚部 sub_i，旋转因子 wr、wi
实部 mul_r = sub_r * wr - sub_i * wi
虚部 mul_i = sub_i * wr + sub_r * wi
```

注意乘法项里 `sub_r/sub_i` 与 `wr/wi` 的交叉：实部用"实·实 − 虚·虚"，虚部用"虚·实 + 实·虚"。背成口诀就是"实实减虚虚，虚实加实虚"。

#### 4.4.3 源码精读

**旋转因子定义**（32 点的全部 16 个旋转因子）：

[SIM/FFT.py:L51-L61](https://github.com/abdelazeem201/Design-and-ASIC-Implementation-of-32-Point-FFT-Processor/blob/89ce7665eca8b77a0c48e610dab11928108566b6/SIM/FFT.py#L51-L61) —— 显式给出 `w[0]=1` 到 `w[8]=0`（即 \(\cos 0 \dots \cos\pi/2\)），再用对称性 `w[8+i] = -w[8-i]` 生成 `w[9..15]`（即 \(\cos\) 在第二象限的负值）。

[SIM/FFT.py:L63-L73](https://github.com/abdelazeem201/Design-and-ASIC-Implementation-of-32-Point-FFT-Processor/blob/89ce7665eca8b77a0c48e610dab11928108566b6/SIM/FFT.py#L63-L73) —— 同理给出虚部 `w_i[0]=0` 到 `w_i[8]=-1`（即 \(-\sin 0 \dots -\sin\pi/2\)），再用 `w_i[8+i] = w_i[8-i]` 生成后半段。验证：`w[1]=0.98078≈cos(π/16)`，`w_i[1]=-0.1950≈-sin(π/16)`，完全符合 \(W_{32}^1\) 的定义。

## 6. 本讲小结

- **DFT → FFT**：利用旋转因子 \(W_N^{kN/2}=(-1)^k\) 的对称性，把 \(N^2\) 次复数乘法降到 \(\tfrac{N}{2}\log_2 N\)，32 点从 1024 降到 80。
- **radix-2 DIF 蝶形**：每个蝶形有加法分支（→ 偶数频率）和"减法 × 旋转因子"分支（→ 奇数频率），先加减、后乘旋转。
- **五级分组规律**：组数每级翻倍（1→2→4→8→16），组大小每级减半（32→16→8→4→2），旋转因子索引步进每级翻倍（1→2→4→8→16）。
- **复数乘旋转因子**：标准 4 乘 2 加，实部 \(=b_r W_r - b_i W_i\)，虚部 \(=b_i W_r + b_r W_i\)；`FFT.py` 各级结构同构，仅旋转因子下标不同。
- **stage5 特殊**：旋转因子恒为 1，只剩纯加减；DIF 输出天然乱序，需位反转还原。
- **算法与硬件的对应**：`FFT.py` 的五级顺序结构，正是硬件五级流水线（u3、u4）的蓝本；4 乘 2 加在硬件中会被优化为 3 乘 5 加（u7-l1）。
