本讲打开解密通路控制层的时钟级黑盒，即 `aes_decipher_block.v` 的三个最小模块：`round_logic`（纯组合，按 `update_type` 在 INIT/SBOX/MAIN/FINAL 四种写回中现算对应逆运算，仅初始轮读输入端口密文，其余读状态寄存器）；`round_ctr`（4 位**递减**计数器，由 `round_ctr_set` 一次性置 N——AES-128 为 10、AES-256 为 14，每轮 `round_ctr_dec` −1 至 0）；`decipher_ctrl`（4 状态总控 FSM：IDLE/INIT/SBOX/MAIN）。核心结论：解密按倒序使用轮密钥 K_N→K_0，故计数方向与加密相反，却能共用同一句 `assign round=round_ctr_reg` 与同一份 `key_mem`。关键不对称是 `round_ctr_dec` 仅在 CTRL_SBOX 末拍出现一处，而加密 `round_ctr_inc` 出现在 INIT、MAIN 两处。`CTRL_FINAL` 系死代码，最终轮由 CTRL_MAIN 的 else 分支用 FINAL_UPDATE 实现，并采用等价逆密码执行序。一次 AES-128/256 解密耗 51/71 拍，与加密完全相同。
