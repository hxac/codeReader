本讲只覆盖顶层 wrapper `aes.v`（不进入 aes_core），讲清主机如何通过总线驱动 AES 核。三个最小模块：顶层端口——7 个信号（`clk`、`reset_n`、`cs`、`we`、`address[7:0]`、`write_data`/`read_data[31:0]`）构成内存映射的 32 位同步总线接口，无专用 start/done 引脚，靠片选 `cs` 门控读写，`read_data` 为组合读、当拍出值；地址映射 `0x00~0x33` 分五区段——身份 NAME/VERSION（只读）、控制类 CTRL/STATUS/CONFIG、密钥 KEY0..KEY7（256 位，只写）、明文 BLOCK0..BLOCK3（128 位，只写）、结果 RESULT0..RESULT3（128 位，只读）；`api` 组合块（`always @*`）做命令译码、`reg_update` 时序块（`posedge clk`）落地寄存器，落地 u1-l3「两段式」约定，KEY/BLOCK 数组用地址低位作下标。关键概念：CTRL 的 `init`(bit0)/`next`(bit1) 为单拍脉冲触发位，STATUS 的 `ready`(bit0)/`valid`(bit1) 为状态回读，CONFIG 的 `encdec`(bit0，1=加密 0=解密)/`keylen`(bit1，0=128 位 1=256 位) 为持久配置，位定义始终以 `aes.v` 为准（testbench 同名 parameter 为死代码不可信）。一次加密的访问主线：写密钥→写配置→写 CTRL.init→写明文→写配置→写 CTRL.next→轮询 STATUS→读 RESULT。
