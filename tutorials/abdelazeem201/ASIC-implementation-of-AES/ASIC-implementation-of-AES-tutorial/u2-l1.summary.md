本讲打开 `aes_core` 黑盒，它是顶层 wrapper 与算法子模块之间的「调度中枢」，自身不做运算，只例化 4 个子模块（加密车间 `enc_block`、解密车间 `dec_block`、轮密钥仓库 `key_mem`、唯一的正向 `sbox_inst`）并用 3 个组合逻辑块调度。核心模块为 `aes_core_ctrl`（三状态 FSM `IDLE`/`INIT`/`NEXT`，粗粒度，不含轮循环细节）：`init` 脉冲触发密钥扩展，`next` 脉冲触发加/解密；忙态拉低 `ready`，完成拉高，且仅 `NEXT` 完成置 `result_valid`。`encdec_mux` 依 `encdec` 在加/解密通路间二选一，统一分发 `next`、轮号 `muxed_round_nr`、结果 `muxed_new_block`、完成 `muxed_ready`，使两路共享同一份 `key_mem` 轮密钥存储。`sbox_mux` 依 `init_state` 把唯一正向 S-box 分时复用于密钥扩展与加密；解密走内部私有逆 S-box，不占共享 S-box（解密期间其空闲）。关键结论：三块逻辑环环相扣——FSM 产 `init_state` 喂 `sbox_mux`，`encdec_mux` 的 `muxed_ready` 回喂 FSM 判断，`muxed_round_nr` 决定 `key_mem` 吐哪把轮密钥。子模块内部细粒度轮 FSM 留待后续讲义。
