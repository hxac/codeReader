本讲是专家篇收尾，不引入新模块，而把已有 AES 核当作可改造对象，围绕测试、接口、核心三层源码讲授"改得动、改得对"的二次开发能力。涵盖四个主题：一是新增 NIST/FIPS 测试用例，得益于 tb_aes.v 的"数据与流程分离"，只需在 aes_test 末尾追加模板调用 ecb_mode_single_block_test，加密/解密靠输入与期望是否对调区分；二是改地址映射与总线接口属高风险操作，地址表在 aes.v（localparam，权威）与 tb_aes.v（parameter，镜像）重复定义，且含 CTRL_ENCDEC_BIT/KEYLEN_BIT 死参数陷阱，须两边同步；三是流水线化用面积换时间，方向 A（SubBytes 全并行）性价比最高，但与"全核共享单个 S-box"的核心取舍冲突；四是回归测试，借 5 个分层 testbench 与 NIST 金标准向量把 bug 锁定到具体模块。结论：加测试是最低风险入口，改接口风险最高，改对与否的唯一判据是 error_ctr 为 0。
