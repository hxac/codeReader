本讲解析 `aes_encipher_block.v` 的加密控制状态机 `encipher_ctrl`，即把上一讲的四个变换函数串成按时钟节拍流动的「指挥层」。它由四个最小单元组成：`round_logic` 是纯组合数据层，用 `update_type`（INIT/SBOX/MAIN/FINAL_UPDATE 四种编码）选择把哪种运算结果写回 4 个字寄存器，关键细节是初始轮读输入明文端口、其余轮读状态寄存器 `old_block`；`round_ctr`（4 位）是轮计数器，输出轮号送给 `key_mem` 组合选 round key，同一轮的 SubBytes 拍与 MAIN 拍共享同一轮号 k；`sword_ctr`（2 位）是字计数器，因全核仅一个共享 S-box，128 位 SubBytes 被拆成 4 拍逐字替换（广播 `new_sboxw` 配选择性写使能），直接体现「用时间换面积」；`encipher_ctrl` 是 4 状态 FSM（IDLE/INIT/SBOX/MAIN），`CTRL_FINAL` 是从未被 case 引用的死代码，最终轮由 `CTRL_MAIN` 的 else 分支用 `FINAL_UPDATE` 实现——数状态看 case 分支而非 localparam。核心结论：一次 AES-128 加密耗 51 拍（1+10×5），AES-256 耗 71 拍；全程沿用 reg/_new/_we 两段式风格。
