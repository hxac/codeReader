本讲是入门篇收尾，让整个工程真正跑起来，仅读 rtl/tb_aes.v 与 rtl/aes.v，分三个最小模块。其一，时钟与复位：clk_gen 用无敏感列表的 always 块自驱动产生周期为 2 的方波，sys_monitor 以 cycle_ctr 做周期计数并在 DEBUG 时转储状态，复位封装在 reset_dut 里拉低两周期再释放，对应 DUT 的异步低有效复位。其二，测试驱动模型：init_sim 给所有引脚定初值，dump_dut_state 用层次化引用 dut.xxx 偷看内部寄存器（仿真特权，不可综合代码中不可用），write_word/read_word 两个 task 落地 u1-l4 的总线访问主线，关键信号是 STATUS 的 ready/valid 与 RESULT。其三，仿真运行：ModelSim 的 simulation.mpf 默认 work 库、ns 分辨率、顶层 tb_aes，但文件路径写死成作者 Windows 绝对路径换机即失效，宜用 vlog/iverilog 命令行编译。核心结论：跑 tb_aes 最终打印 16 组 NIST 用例全通过，该数字可从源码直接数出。
