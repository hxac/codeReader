本讲追踪一次 AES 加解密从主机写寄存器到读回密文的端到端数据流，覆盖四个最小模块：aes.v（顶层译码与寄存器落地）、aes_core（调度中枢）、aes_key_mem（密钥扩展与轮密钥仓库）、aes_encipher_block（加密车间）。核心结论：一次加解密必须分两次主机触发——init 阶段一次性生成并存储全部轮密钥（写满 key_mem[0..num_rounds]），next 阶段才真正做加/解密，二者因共享同一正向 S-box 而必须分时复用，无法合并。本讲引入的关键概念：init/next 为单拍脉冲（区别于电平开关），工程靠 ready/valid 状态位接力、各 FSM 自驱完成，无阻塞握手引脚；init 信号不经 encdec_mux 直达 key_mem（密钥扩展与方向无关）；next 阶段由 encdec_mux 统一分发脉冲、轮号、结果与完成信号，轮密钥经 round 端口纯组合回送；core 的 result 直接等于车间实时工作寄存器而非独立结果寄存器，顶层 result_reg 每拍无条件镜像，故全程在变，只有 result_valid=1（仅 next 完成时置位）后读 RESULT 才有意义；BLOCK 写入与 RESULT 读出采用大端字序自洽。本讲为专家篇总纲，后续验证、分层测试与架构取舍均建立于对这条数据流的掌握之上。
