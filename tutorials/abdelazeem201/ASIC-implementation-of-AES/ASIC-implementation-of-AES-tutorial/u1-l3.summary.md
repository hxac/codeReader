本讲（u1-l3）以 `rtl/aes.v`（顶层 wrapper）和 `rtl/aes_core.v`（核心状态机）为样本，讲清贯穿全工程的 Verilog 编码风格，是后续阅读所有子模块的通用钥匙。核心结论有三：其一，所有寄存器统一写在 `always @(posedge clk or negedge reset_n)`，采用「上升沿触发 + 异步低有效复位」模板，复位分支置已知初值（如 `aes_core_ctrl_reg` 置 `CTRL_IDLE`，core 的 `ready_reg` 置 `1`），时序块一律用非阻塞赋值 `<=`。其二，最核心的「两段式」约定：组合逻辑块算出下一个值 `_new` 和写使能 `_we`，时序逻辑块在时钟沿执行 `if (_we) _reg <= _new` 机械搬运。关键术语是 `reg/_new/_we` 伴随信号的三种变体——完整三件套（如 `aes_core_ctrl`）、数据寄存器 `reg/_we`（新值即 `write_data`，无 `_new`，如 `block_reg`）、脉冲寄存器 `reg/_new`（每拍都搬，形成单拍脉冲，如 `init_reg`）。其三，组合逻辑用 `always @*` + 阻塞赋值 `=`，且块开头先给所有输出写默认值以防生成锁存器（latch）；`aes.v` 的 `api` 块做命令译码，`aes_core.v` 的 `sbox_mux`/`encdec_mux`/`aes_core_ctrl` 块做多路选择与算下一状态。状态机采用「组合块算下一状态 + 时序块搬状态」的两段式写法，状态本身存在 `_reg` 触发器里。综合实践追踪了主机写 `ADDR_CTRL` 使状态机从 `IDLE` 进入 `INIT` 的链路，并指出该过程跨越两个时钟沿，为 u1-l4（地址映射）与 u1-l5（仿真波形）承接埋下伏笔。
