本讲是整本学习手册的「索引页」，解决 clone 仓库后「先打开哪个文件、某知识点去哪找」的导航问题。核心结论有四点：第一，仓库按 EDA 工具与知识领域分目录——`IC Compiler II/`(当前 ICC2 主流程，含 PnR.tcl)、`IC Compiler/`(旧版 ICC)、`PrimeTime/`(STA)、`mentor_scripts/` 放脚本；`MY-Design/`、`cmsdk/`(ARM Cortex-M0)、`memories/` 放可读 RTL 样例；可综合 HDL 规范、HDL Compiler 分册、yosys 手册与在线资源清单分目录存放 PDF 参考资料。每个目录可对应到 RTL→GDSII 流程的某一环。第二，引入关键术语：文件后缀(`.tcl/.pl/.v/.upf`)、目录树、GitHub 永久链接、HDL Compiler、CI 与 GitHub Actions。第三，特别提醒几个需自行核对的异常：根目录与 `LEF2FRAM/` 下重复的 `.pl` 是同文件副本、README 中 `Tutorials and Courses` 链接已失效、配图链接指向另一名字相近的仓库(看图以 `Figures/` 本地文件为准)。第四，`.github/workflows/blank.yml` 仅为 GitHub 起步模板，只 `echo Hello, world!`，并未真正构建或测试，故 CI 绿灯不等于代码被验证，正确性须靠 EDA 报告。后续讲义的素材取材均回到本讲这张地图。
