本讲把项目三块定位落到目录，划分五大目录：`rtl/`（可综合核心源码）、`Pre-Synthesis_Simulation/`（自包含仿真目录）、`syn/` 与 `pnr/`（综合→布局布线后端两阶段，产物递进）、`rtl/RTL_modified/`（可上 SoC 的独立扩展支线）。核心设计含六个 `.v`：顶层 `tpu_top` 例化五子模块 `systolic`（8×8 脉动阵列本体）、`systolic_controll`（主控状态机）、`addr_sel`（地址解码制造输入歪斜）、`quantize`（21bit 饱和量化为 16bit）、`write_out`（按反对角线写回三组 SRAM），数据流为「控制器指挥→地址喂入→阵列计算→量化→写回」。三条关键结论：其一，`rtl/` 与仿真目录的核心模块逐字节相同，后者仅多带 testbench、SRAM 行为模型与测试数据；其二，综合脚本 `syn.tcl` 以 `../rtl/${design}.v` 读源码，`rtl/` 是权威综合输入；其三，文档与代码矛盾时以源码为准，顶层参数 `ARRAY_SIZE=8`、`DATA_WIDTH=8`、`SRAM_DATA_WIDTH=32`、`OUTPUT_DATA_WIDTH=16`，乘加中间位宽 `ORI_WIDTH=21bit`。另 `rtl/systolic array/` 教学版副本与无后缀 `rtl/tpu_top` 死副本均非综合输入。
