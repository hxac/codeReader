本讲精读 Synopsys Design Compiler 综合主脚本 syn.tcl，讲清 RTL 如何被翻译并优化成某工艺的标准单元门级网表。流程分四段：①设库——由 search_path（库搜索目录）、target_library（映射终点的目标库）、link_library（解析引用、开头 `*` 指当前设计内存）三变量控制，选用 Nangate 45nm 的 ss0p95vn40c 坏 PVT 角做保守综合留余量。②读入展开链接——analyze 解析语法，elaborate 把 RTL 展成与工艺无关的 GTECH 中间网表并冻结参数（模块名带 _ARRAY_SIZE8 后缀为证），uniquify 给多实例唯一名，link 解析悬空引用。③编译——compile_ultra 把 GTECH 映射到标准单元；选项是「授权」非「强制」（开 -gate_clock 却零门控单元），-no_autoungroup 等保守组合保住 systolic（占 89.6% 面积）等模块层次。④输出——report_* 给人看（.rpt），write_* 交下游工具：门级网表 .v、约束 .sdc、延迟 .sdf、数据库 .ddc。关键结论：优化开关须以报告实际效果为准。时序约束细节留待 u5-l2。
