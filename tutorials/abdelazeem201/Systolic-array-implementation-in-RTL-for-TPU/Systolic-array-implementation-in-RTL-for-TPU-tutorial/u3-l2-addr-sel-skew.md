# 地址选择与输入歪斜 addr_sel

## 1. 本讲目标

学完本讲，你应当能够：

- 说清 `addr_sel` 模块在 TPU 数据通路里的位置：它把控制器送来的单一串行号 `addr_serial_num` 解码成四路 SRAM 读地址。
- 手算出任意 `addr_serial_num` 对应的 `sram_raddr_w0/w1/d0/d1` 四个值，包括「零扩展」与「−4 偏移」两条规则。
- 解释「截断到 127」为什么等价于向阵列里灌入全零操作数，以及它如何制造输入流开头/结尾的时序填充。
- 理解输出端那一拍寄存器（flip-flop）如何与外部 SRAM 的读延迟一起，凑成「addr_serial_num 领先 cycle_num 2 拍」的两级流水。
- 把「地址歪斜（skew）」和上一讲 u2-l1 讲过的「反对角线波前」联系起来：地址偏移是让 weight 与 data 在正确 cell、正确节拍对齐相乘的命门。

## 2. 前置知识

在进入源码前，先建立三个直觉。

**(1) 脉动阵列需要「输入歪斜」。** 回顾 u2-l1：阵列里 weight 每拍下移一格、data 每拍右移一格。同一个 cell(i,j) 要等到 weight 注入后第 i 拍、data 注入后第 j 拍才会同时收到这对操作数。为了让「该相乘的那一对」在同一拍相遇，进入阵列边界的输入必须在时间上被错开（skew）。本讲讲的 `addr_sel`，就是制造这种错开的硬件部件之一。

**(2) 读 SRAM 是有延迟的。** 给 SRAM 一个读地址，数据要等到下一个时钟沿才出来。本设计里 SRAM 读延迟是 1 拍。`addr_sel` 内部还有 1 拍输出寄存器。两段延迟叠起来是 2 拍——这正是 u3-l1 里控制器让 `addr_serial_num` 领先 `cycle_num` 整整 2 拍的原因：现在提前 2 拍发地址，2 拍后数据刚好到阵列，与 `cycle_num` 对齐。

**(3) 「127 号地址存 0」。** 代码注释写明 `max = 126, setting all of the addr127 = 0`：SRAM 的 127 号地址被预先填成全零。所以「让读地址等于 127」就等于「这一拍向阵列灌零」，零对乘加没有贡献，相当于一个无害的空拍。`addr_sel` 正是用这个技巧来填充输入流的开头和结尾。

> 术语：本讲里「歪斜 / skew」指同一批数据在时间上的相对错位；「队列 / queue」指 SRAM 的一条读取通道，每路 32bit 含 4 个 8bit 字节通道。

## 4. 核心概念与源码讲解

本讲对应三个最小模块：**四组读地址的组合解码**、**上界截断到 127**、**输出打一拍寄存器**。我们按这三个模块拆开讲。

### 4.1 四组读地址的组合解码（raddr_nx assign）

#### 4.1.1 概念说明

`addr_sel` 是一个**纯地址译码器**：输入只有一个 7bit 的串行号 `addr_serial_num`，输出是四路 10bit 的 SRAM 读地址。这四路分别驱动四块 SRAM：

- `sram_raddr_w0`：驱动 weight SRAM 的「前半」（队列 0~3，即阵列左侧 4 列的权重）。
- `sram_raddr_w1`：驱动 weight SRAM 的「后半」（队列 4~7，即阵列右侧 4 列的权重）。
- `sram_raddr_d0`：驱动 data SRAM 的「前半」（队列 0~3，即阵列上侧 4 行的数据）。
- `sram_raddr_d1`：驱动 data SRAM 的「后半」（队列 4~7，即阵列下侧 4 行的数据）。

核心设计就两条规则：

1. **w0/d0 直接读**：地址 = `addr_serial_num`（零扩展到 10bit）。
2. **w1/d1 滞后 4 读**：地址 = `addr_serial_num - 4`。

也就是说，w1/d1 永远在读 w0/d0 「4 个地址之前」的内容。换个角度看：w0 在第 T 拍读到的 SRAM 内容，w1 要等到第 T+4 拍才读到——**w1/d1 的数据流整体比 w0/d0 晚 4 拍到达阵列**。这 4 拍的相对延迟，就是脉动阵列的输入歪斜。

为什么是 4？因为 `ARRAY_SIZE = 8`，每路 SRAM 的 32bit 正好喂 4 个 cell（4 = ARRAY_SIZE/2，半个阵列宽度）。把每个输入行/列拆成两半后，让远端那半晚半个阵列宽度进入，才能让 weight 与 data 在正确的 cell、正确的节拍上对齐相乘。这一点会在 4.1.4 与综合实践里展开。

#### 4.1.2 核心流程

四条 `assign` 的译码规则可总结为下表（`asn` 代表 `addr_serial_num`）：

| 输出 | 有效条件 | 取值 | 含义 |
|---|---|---|---|
| `sram_raddr_w0_nx` | `asn <= 98` | `asn`（零扩展） | 前半 weight，立即启动 |
| `sram_raddr_w1_nx` | `4 <= asn <= 102` | `asn - 4` | 后半 weight，晚 4 拍启动 |
| `sram_raddr_d0_nx` | `asn <= 98` | `asn`（零扩展） | 前半 data，立即启动 |
| `sram_raddr_d1_nx` | `4 <= asn <= 102` | `asn - 4` | 后半 data，晚 4 拍启动 |

注意两个对称性：

- **w0 与 d0 完全相同**，**w1 与 d1 完全相同**。weight 通道和 data 通道用的是同一套地址译码逻辑——因为 weight 和 data 都需要同样形式的歪斜。
- 两条有效窗口的长度都是 99 个地址（w0: 0..98；w1: 4..102 映射到 0..98），说明两块 SRAM 存的有效数据量一样，只是读取时刻错开 4 拍。

伪代码描述一条通道的译码：

```
if (前半)  addr = (asn <= 98)  ? zero_extend(asn)     : 127;  // 越界读 127(=全零)
if (后半)  addr = (4<=asn<=102)? zero_extend(asn - 4) : 127;
```

#### 4.1.3 源码精读

先看模块端口：单输入 `addr_serial_num`（7bit，注释标注最大用到 126，127 留作全零地址），四路 10bit 输出地址。

[rtl/addr_sel.v:4-16](https://github.com/abdelazeem201/Systolic-array-implementation-in-RTL-for-TPU/blob/6a93418faefac866eeab6349c8adfc91576099fb/rtl/addr_sel.v#L4-L16) —— 模块与端口声明：注意 `addr_serial_num` 是 `[6:0]`（7bit），输出地址是 `[9:0]`（10bit），所以译码时需要把 7bit 零扩展到 10bit。

四条组合译码是整个模块的灵魂：

[rtl/addr_sel.v:33-37](https://github.com/abdelazeem201/Systolic-array-implementation-in-RTL-for-TPU/blob/6a93418faefac866eeab6349c8adfc91576099fb/rtl/addr_sel.v#L33-L37) —— 四组 `assign`：w0/d0 直接取 `addr_serial_num`，w1/d1 取 `addr_serial_num - 7'd4`；越界则给 127。

其中零扩展写法值得拆开看：

```verilog
{ {3{1'd0}} , addr_serial_num }
```

`{3{1'd0}}` 是 3 个 0 比特（`3'b000`），拼上 7bit 的 `addr_serial_num`，正好 10bit。它就是把 7bit 的串行号高位补零，变成 10bit 地址——这就是「零扩展」。

而 `addr_serial_num - 7'd4` 里的 `7'd4` 是显式标注位宽的常量 4，做 7bit 减法，得到「滞后 4」的地址。

> 设计取舍：这里 `4` 与 `98/102` 都是写死的字面量，没有写成 `ARRAY_SIZE/2` 之类的参数表达式。这与 u1-l4 指出的「半参数化」一致——位宽骨架随参数走，但偏移、边界仍按 8×8 硬编码。

#### 4.1.4 代码实践

这是本讲的主实践任务。**目标**：亲手把 `addr_serial_num` 译成四路地址，验证「−4 偏移」与「127 填充」两条规则。

**操作**：对 `addr_serial_num = 0、4、50、100` 四个取值，套用 4.1.2 表格的两条规则，填出四路 `_nx`（组合输出）的值。

**手算过程**（以 `asn=4` 为例）：
- w0：`4<=98` 成立 → `4`。
- w1：`4<=4<=102` 成立 → `4-4 = 0`。
- d0 同 w0 → `4`；d1 同 w1 → `0`。

**完整结果表**：

| `addr_serial_num` | `sram_raddr_w0` | `sram_raddr_w1` | `sram_raddr_d0` | `sram_raddr_d1` | 现象解读 |
|---|---|---|---|---|---|
| 0 | 0 | **127** | 0 | **127** | 后半还没启动，w1/d1 读全零地址 |
| 4 | 4 | 0 | 4 | 0 | 后半刚启动（读地址 0）；前半读 4，正好相差 4 |
| 50 | 50 | 46 | 50 | 46 | 稳态区，w1/d1 = w0/d0 − 4 |
| 100 | **127** | 96 | **127** | 96 | 前半已结束读全零；后半仍在读（96） |

**需要观察的现象**：w0/d0 与 w1/d1 在稳态区恒差 4；在开头（asn=0..3）只有前半有数据、后半灌零；在结尾（asn=99..102）只有后半有数据、前半灌零——两半的「有效窗口」刚好错开 4 拍。

**预期结论**：w1/d1 之所以比 w0/d0 滞后 4 个地址，是因为它们喂给阵列的「远端半」（队列 4~7）。读地址减 4，等价于数据流晚 4 拍到达；这 4 拍（= ARRAY_SIZE/2，半个阵列宽度）就是让远端半的操作数与近端半错相进入、从而在对角线波前上正确对齐的输入歪斜。若去掉这个 −4，两半会同时喂入，操作数错位 4 拍，MAC 累加出错误结果。

> 说明：上表给出的是组合 `_nx` 值；由于输出端有一拍寄存器（见 4.3），真正出现在 `sram_raddr_*` 端口上的值要再晚 1 个时钟，但数值不变。

#### 4.1.5 小练习与答案

**练习 1**：`addr_serial_num = 7` 时，四路地址各是多少？此时 w0 与 w1 的地址差是多少？

**参考答案**：w0 = 7、w1 = 7−4 = 3、d0 = 7、d1 = 3。地址差仍是 4（w0 读 7，w1 读 3，w1 落后 4 个地址）。

**练习 2**：为什么 w0 的代码和 d0 一模一样、w1 和 d0…不，w1 和 d1 一模一样？这反映了 weight 通道与 data 通道之间什么关系？

**参考答案**：因为 weight 和 data 在阵列里走的是对称的歪斜方案（一个自上而下、一个自左而右），需要的地址错相规则完全相同，所以两套通道共用同一份译码逻辑。

### 4.2 上界截断到 127：用全零地址做时序填充

#### 4.2.1 概念说明

光有「−4 偏移」还不够。考虑输入流的开头：第 0 拍 w0 已经在读地址 0，可 w1 此时的「自然地址」是 `0 − 4 = −4`——没有负地址。怎么办？设计者的做法是：**凡是不在有效窗口里的拍，就让地址等于 127**，而 127 号地址存的是全零。于是 w1 在开头几拍读到全零，等于「还没开始真正喂权值」，自然地把后半的启动推迟了 4 拍。

结尾同理：当 `addr_serial_num` 超过 98，前半 w0/d0 的有效数据读完了，但后半 w1/d1 因为 −4 偏移，还要再读 4 拍（读到 asn=102）。这 4 拍里前半就读 127（全零），相当于「前半先收尾，后半后收尾」。

一句话：**截断到 127 = 用全零操作数填充输入流的头部和尾部，把两半的起止时刻错开**。这正是歪斜在地址层面的体现。

#### 4.2.2 核心流程

把四条 `assign` 的条件分支画成时间线（`asn` 从 0 递增）：

```
asn:   0 ... 3 | 4 ........ 98 | 99 .. 102 | 103 ... 126 | 127(保持)
w0/d0: [ 有效 asn ...............] [ 127全零 ......................)
w1/d1: [127全零 ] [ 有效 asn-4 ............] [ 127全零 .........)
        ↑后半空等   ↑两半都有效(稳态)        ↑前半已空等
        4拍                                    4拍
```

可以读出三个阶段：

1. **爬坡（asn 0..3）**：前半已出数据，后半读 127（全零）。后半被刻意压住 4 拍。
2. **稳态（asn 4..98）**：两半都有效，地址恒差 4。
3. **收尾（asn 99..102）**：前半读完读 127，后半还在读最后 4 个有效地址。前半先空 4 拍。

边界值 98 与 102 之差正好是 4，与偏移量一致——前后半的有效数据量相同，只是时间上整体平移了 4 拍。

#### 4.2.3 源码精读

截断条件直接写在三元运算符里：

[rtl/addr_sel.v:33-34](https://github.com/abdelazeem201/Systolic-array-implementation-in-RTL-for-TPU/blob/6a93418faefac866eeab6349c8adfc91576099fb/rtl/addr_sel.v#L33-L34) —— 前半 `<=98` 否则给 127；后半 `>=4 && <=102` 否则给 127。

关键细节：

- 前半的有效上界是 **98**：`asn <= 98 ? asn : 127`。当 `asn` 到 99 及以后，w0/d0 输出 127。
- 后半的有效窗口是 **`[4, 102]`**：下界 4（再减 4 才不会变成负/无效），上界 102（减 4 后等于 98，与前半上界对齐）。
- 字面量 `127` 是无位宽十进制常量，赋给 10bit 线网时被解释为 `10'd127 = 10'b0001111111`，即地址 127。
- 注释 [rtl/addr_sel.v:7](https://github.com/abdelazeem201/Systolic-array-implementation-in-RTL-for-TPU/blob/6a93418faefac866eeab6349c8adfc91576099fb/rtl/addr_sel.v#L7) 点明了这个约定：`max = 126, setting all of the addr127 = 0`——127 号地址预填全零，专门用来当「空拍」。

#### 4.2.4 代码实践（源码阅读型）

**目标**：从源码里找出「两半有效窗口的精确边界」，并验证它们长度相等。

**操作步骤**：
1. 在 [addr_sel.v:33-37](https://github.com/abdelazeem201/Systolic-array-implementation-in-RTL-for-TPU/blob/6a93418faefac866eeab6349c8adfc91576099fb/rtl/addr_sel.v#L33-L37) 中分别找到 w0 与 w1 的上界条件。
2. 计算 w0 有效地址数 = `98 - 0 + 1 = 99`。
3. 计算 w1 有效地址数 = `102 - 4 + 1 = 99`。

**需要观察的现象**：两条窗口长度都是 99，说明两块 SRAM 存的有效数据量一致，只是读取时刻整体平移了 4 拍。

**预期结果**：99 == 99，验证「前后半等量、错相 4 拍」的结论。

#### 4.2.5 小练习与答案

**练习**：如果把后半的下界从 4 改成 6（即 `asn>=6 && asn<=102`），w1 的有效窗口长度变成多少？这会对歪斜造成什么影响？

**参考答案**：窗口变成 `102-6+1 = 97`，比前半的 99 少 2，两半数据量不再相等，会造成末尾 2 拍错位；同时后半启动被推迟到第 6 拍，歪斜量从 4 变成 6，阵列里 weight/data 的对齐节拍也会被打乱——这会直接导致 MAC 结果错误。可见 `4` 这个值是和 ARRAY_SIZE/2 绑定的，不能随意改。

### 4.3 输出打一拍寄存器与两级读流水（flip-flop）

#### 4.3.1 概念说明

`addr_sel` 的输出地址不是组合直出的，而是先算出 `_nx`（next，下一拍值），再用一个 `always@(posedge clk)` 把它打进寄存器。源码注释把这叫做 "fit in output flip-flop"——给输出加一拍触发器。

为什么要多这一拍？两个好处：

1. **改善时序**：地址译码是纯组合，若直接拉到芯片外部的 SRAM，关键路径会很长；打一拍寄存器把「译码」和「SRAM 读」切成两级，每级都短，利于满足 3ns 周期目标（见 u5-l2）。
2. **凑齐流水延迟**：这一拍 + 外部 SRAM 的 1 拍读延迟 = 2 拍，正好对上控制器「`addr_serial_num` 领先 `cycle_num` 2 拍」的设计。

#### 4.3.2 核心流程

把地址从产生到变成阵列输入数据，画成一条两级流水：

```
拍 T    : addr_serial_num = N            (控制器给出)
拍 T    : raddr_*_nx   = decode(N)        (组合, 当拍算出)
拍 T+1  : sram_raddr_* = decode(N)        (addr_sel 输出寄存器, 打一拍)  ← 第 1 级延迟
拍 T+2  : sram_rdata   = SRAM[decode(N)]  (外部 SRAM 读, 再 1 拍)        ← 第 2 级延迟
```

也就是说，`addr_serial_num` 在第 T 拍发出的地址，要到第 T+2 拍才变成真正进入阵列的数据。与此同时，控制器保证 `addr_serial_num = cycle_num + 2`（见 4.3.3），于是第 T 拍发出的地址对应的数据，在第 T+2 拍到达阵列，而那拍的 `cycle_num` 正好等于 N——**数据与节拍号严格对齐**。这就是为什么 u3-l1 反复强调「`addr_serial_num` 领先 `cycle_num` 2 拍」。

用公式表达这条对齐关系：

\[
\text{addr\_serial\_num}(t) \;=\; \text{cycle\_num}(t) + 2
\]

而数据到达阵列的时刻为 \(t+2\)，此时 \(\text{cycle\_num}(t+2) = \text{cycle\_num}(t) + 2 = \text{addr\_serial\_num}(t)\)，即「数据 = 该地址对应的内容」与「当前节拍号」一致。

#### 4.3.3 源码精读

输出寄存器只有一个 `always` 块，把四路 `_nx` 同步打进四路输出：

[rtl/addr_sel.v:25-31](https://github.com/abdelazeem201/Systolic-array-implementation-in-RTL-for-TPU/blob/6a93418faefac866eeab6349c8adfc91576099fb/rtl/addr_sel.v#L25-L31) —— `always@(posedge clk)` 把 `_nx` 打一拍送到 `sram_raddr_*`，注释写明 "fit in output flip-flop"。

注意它**没有复位端**：`addr_serial_num` 在复位期间若被控制器保持为 0，则 `_nx` 会算出有效地址 0，打一拍后输出 0；这不算问题，因为复位期间 `alu_start=0`，阵列不会真正做乘加（见 u3-l1）。

再看控制器那边如何维持「领先 2 拍」。在 `WAIT1` 状态，控制器把 `addr_serial_num` 设为 2 而 `cycle_num` 仍为 0：

[rtl/systolic_controll.v:114-115](https://github.com/abdelazeem201/Systolic-array-implementation-in-RTL-for-TPU/blob/6a93418faefac866eeab6349c8adfc91576099fb/rtl/systolic_controll.v#L114-L115) —— `WAIT1` 态把 `addr_serial_num_nx` 置 2，此时 `cycle_num` 还是 0，建立了 2 拍的领先。

进入 `ROLLING` 后两者每拍各 +1，领先关系得以保持：

[rtl/systolic_controll.v:117-122](https://github.com/abdelazeem201/Systolic-array-implementation-in-RTL-for-TPU/blob/6a93418faefac866eeab6349c8adfc91576099fb/rtl/systolic_controll.v#L117-L122) —— `ROLLING` 态 `addr_serial_num` 每拍 +1（到 127 后保持），与 `cycle_num` 同步递增，故恒有 `addr_serial_num = cycle_num + 2`。

> 一致性检查：`addr_serial_num` 是 7bit，最大 127；`cycle_num` 是 9bit。当 `addr_serial_num` 在 127 饱和后不再增长，而 `cycle_num` 仍会继续增长——但这发生在所有有效地址都已读出之后（前半早停、后半也停），阵列靠 127 号全零地址维持空拍，不影响结果。

#### 4.3.4 代码实践（时序跟踪型）

**目标**：亲手验证「领先 2 拍」如何让数据与节拍对齐。

**操作步骤**：
1. 假设某拍 `addr_serial_num = 10`，写出当拍 `cycle_num` 的值（由 4.3.2 公式）。
2. 算出 w0 的 `_nx` 值（= 10），并指出它经过输出寄存器后，在哪一拍出现在 `sram_raddr_w0` 端口。
3. 再经过 SRAM 1 拍读延迟，数据在哪一拍到达阵列？那拍的 `cycle_num` 是多少？

**需要观察的现象**：数据到达阵列的那拍，`cycle_num` 应该正好等于当初发出的 `addr_serial_num`（10）。

**预期结果**：
- `cycle_num = 10 − 2 = 8`。
- `sram_raddr_w0` 端口在下一拍（`addr_serial_num` 已变 11 时）出现 10。
- 数据再过 1 拍到达阵列，此时 `cycle_num = 10`，与发出的地址号一致。对齐成功。

（如需在仿真器里确认，可在 testbench 里给 `sram_raddr_w0` 与 `cycle_num` 各加一行 `$display` 监控，本讲不假设已运行。）

#### 4.3.5 小练习与答案

**练习 1**：如果删掉 `addr_sel` 里那个 `always@(posedge clk)` 输出寄存器、让地址组合直出，控制器的「领先 2 拍」还成立吗？应改成领先几拍？

**参考答案**：不成立。少了这一拍，从地址发出到数据到达只剩 SRAM 的 1 拍延迟，因此控制器应让 `addr_serial_num` 只领先 `cycle_num` 1 拍（即 `WAIT1` 里把 `addr_serial_num` 设为 1）。可见这拍寄存器是整条流水延迟预算里的一环，不能随意增删。

**练习 2**：输出寄存器块为什么没有写复位分支？复位期间输出地址会是什么？

**参考答案**：因为复位期间 `addr_serial_num` 被控制器保持为 0，`_nx` 会算出 w0/d0=0、w1/d1=127，打一拍后端口即为此值；而此时 `alu_start=0`，阵列不做乘加，所以这些地址读到的数据不会被使用，无需专门复位地址寄存器。

## 5. 综合实践

把本讲三条规则串起来，重建 `addr_serial_num` 从 0 到 7 的「爬坡」过程，亲眼看到歪斜是如何建立起来的。

**任务**：

1. 列一张表，`addr_serial_num` 取 0,1,2,3,4,5,6,7，分别填出 `sram_raddr_w0` 与 `sram_raddr_w1`（d 通道同理，可省略）。
2. 在表旁标注每一拍 w0、w1 各自读到的「是有效数据还是全零（127）」。
3. 回答：w1 从第几拍才开始读到有效数据？这与「w1 滞后 w0 4 拍」是否一致？
4. 进阶：结合 4.3 的两级流水，说明如果控制器没有让 `addr_serial_num` 领先 2 拍，这个爬坡表里的数据会在哪一拍错位地进入阵列。

**参考答案要点**：

| `asn` | `w0` | `w1` | w0 状态 | w1 状态 |
|---|---|---|---|---|
| 0 | 0 | 127 | 有效 | 全零 |
| 1 | 1 | 127 | 有效 | 全零 |
| 2 | 2 | 127 | 有效 | 全零 |
| 3 | 3 | 127 | 有效 | 全零 |
| 4 | 4 | 0 | 有效 | 有效（刚启动） |
| 5 | 5 | 1 | 有效 | 有效 |
| 6 | 6 | 2 | 有效 | 有效 |
| 7 | 7 | 3 | 有效 | 有效 |

- w1 从 `asn=4`（即第 4 拍）才读到有效地址 0，正好比 w0 晚 4 拍——与「滞后 4」一致。
- 这 4 拍的「前半单独工作、后半灌零」就是输入歪斜的建立期；之后两半同步推进、地址恒差 4。
- 若控制器不领先 2 拍（例如只领先 1 拍或 0 拍），那么 `asn=N` 对应的数据会在 `cycle_num = N−1` 或 `N−2` 时进入阵列，整套 `cycle_num` 门控的写回时序（u3-l1 里 `cycle_num >= 9` 才写回）都会错位，导致写到错误地址、与 golden 比对失败。

## 6. 本讲小结

- `addr_sel` 是纯地址译码器：单一输入 `addr_serial_num`（7bit）→ 四路输出 `sram_raddr_w0/w1/d0/d1`（10bit）。
- 两条核心规则：w0/d0 直接读 `addr_serial_num`（零扩展）；w1/d1 读 `addr_serial_num − 4`，使后半的数据流整体晚 4 拍（= ARRAY_SIZE/2）到达阵列，构成脉动阵列的输入歪斜。
- 越界时输出地址 127，而 127 号 SRAM 预填全零，等于用空拍填充输入流的头部（后半晚启动）和尾部（前半先收尾），把两半的起止时刻错开。
- 输出端的一拍寄存器把译码与 SRAM 读切成两级流水；它加外部 SRAM 的 1 拍读延迟 = 2 拍，正是控制器让 `addr_serial_num` 领先 `cycle_num` 2 拍的根因，保证数据与节拍严格对齐。
- w0 与 d0、w1 与 d1 逻辑完全相同，反映 weight 与 data 通道共用同一套歪斜方案。
- 偏移量 `4` 与边界 `98/102` 都是按 8×8 写死的字面量，属于本项目「半参数化」的又一处体现。
