本讲聚焦扩展架构 `top.v` 输出侧三个模块：累加表 `accumTable`、读写控制 `accumTableWr_control`/`accumTableRd_control`、激活阵列 `reluArr`。`accumTable` 是分块累加存储，以子矩阵块行号 `submat_m`、块列号 `submat_n`、块内行号 `mat_row` 三元索引定位地址（写地址总线 160 位＝16 个 bank × 10 位），使 16×16 脉动阵列通过多次 `tpu_mat_mult` 把内维条带的部分和累加，拼出最大 128×128 的大矩阵乘。写控制把阵列列有效信号 `mmu_col_valid_out[0]` 译成写使能与写地址，读控制是纯组合地址发生器；该列有效信号还或归约成 `data_mem_calc_done` 反馈主控。`reluArr` 为 16 路 16 位并行 ReLU，串联在累加表读出侧与 `outputMem` 之间，由 `relu_en` 门控（en=0 透传、en=1 钳负到 0）。关键约束：`accum_clear` 仅在初始化与读后清理两阶段生效，与累加过程互斥；`relu_en` 仅在 `tpu_store_outputs` 取结果阶段生效。二者均对应主机 `instr_set.h` 的 `activate`/`clear` 参数。
