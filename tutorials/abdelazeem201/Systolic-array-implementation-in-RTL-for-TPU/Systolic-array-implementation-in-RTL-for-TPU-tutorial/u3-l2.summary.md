本讲解析 `addr_sel` 地址译码模块（rtl/addr_sel.v），它是制造脉动阵列输入歪斜（skew）的硬件部件。模块为纯组合译码器：7bit 串行号 `addr_serial_num` 经零扩展与「−4 偏移」生成四路 10bit 读地址 `w0/w1/d0/d1`，分别驱动 weight 与 data SRAM 的前后半。核心规则有二：w0/d0 直接读串行号；w1/d1 读「串行号−4」，使远端半的数据流晚 4 拍（= ARRAY_SIZE/2，半个阵列宽度）到达阵列，这正是脉动阵列所需的输入歪斜。越界则输出地址 127——该地址预填全零，用空拍填充输入流头部（后半晚启动 4 拍）与尾部（前半先收尾），把两半的起止时刻错开。输出端一拍寄存器加外部 SRAM 1 拍读延迟合计 2 拍，正是控制器令 `addr_serial_num` 领先 `cycle_num` 2 拍的根因，保证数据与节拍号严格对齐。w0 与 d0、w1 与 d1 逻辑完全相同，反映 weight 与 data 通道共用同一套歪斜方案。
