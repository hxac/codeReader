本讲讲解 testbench 在 `tpu_start` 前的预处理任务 `data2sram`：把磁盘上三批 8×8 矩阵（`mat1`/`mat2` 各 24 行）整形后预填进四块读 SRAM（a0/a1 装 weight，b0/b1 装 data）。数据流经三层临时数组：`mat`(24×64bit) 经水平拼接成 `tmp_c_mat`(8×192bit)，再按 `i%4` 字节级错位成 `tmp_mat`(8×216bit)，最后由 `char2sram` 打包写入。核心是 `i%4` 错位：每路左移 0/1/2/3 字节，等价于在时间轴上延后 0/1/2/3 拍，将脉动阵列所需的输入歪斜（skew）预先"烧"进 SRAM 布局；之所以用 `%4`，是因 8 路分给 a0/a1 两块并行 SRAM 同址读出。`char2sram` 是绕过时钟的仿真预载后门，把 4 路各 1 字节拼成 32bit 字（路 0 在最高字节），与 `systolic.v` 切片严格对应；地址 27~127 填零以模拟尾部空拍。
