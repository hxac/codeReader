本讲解析扩展架构 `rtl/RTL_modified/top.v` 的数据通路，它是 `rtl/` 核心（8×8 一次批次）的放大版，面向 SoC 集成、能算超过阵列尺寸的大矩阵。核心结论：`master_control` 是中心调度者，吃进主机 `opcode`、矩阵尺寸 `dim_*`、子矩阵索引与两路反馈（`fifo_to_arr_done`、`data_mem_calc_done`），向 input/weight/output 三侧存储、两条 FIFO、`accumTable` 与 `reluArr` 统一下发使能与地址。主数据流为 `inputMem → sysArr → accumTable → reluArr → outputMem`，权重走 `weightMem → weightFifo → sysArr`。引入的关键术语：分块累加（tiling）——用 3 位子矩阵索引让 16×16 阵列分块算到 128×128 大矩阵；`sys_arr_active2`（`inputMem_rd_en[0]` 经两级寄存器延迟 2 拍，用于激活阵列补偿读延迟）；`data_mem_calc_done`（16 条列有效 `mmu_col_valid_out` 按位或，语义为「有列在产出」而非「全部完成」）。位宽自洽：输入 128 位（8 位/元素），结果链 256 位（16 位/元素）。需注意：仓库仅收录 `top.v`，所例化的多数模块源码不在库内，内部行为均靠端口、注释与软件 API（`instr_set.h`）推断。
