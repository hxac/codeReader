本讲收口 u4 端到端验证闭环，回答「硬件结果是否正确」，核心模块是仿真顶层 test_tpu.v。引入的关键概念：golden 黄金参考按**行优先**存放于 golden1/2/3，每个 128 位字即结果矩阵的一行，由 `$readmemb` 载入；硬件输出 SRAM（c0/c1/c2）则按**反对角线**存放，地址 0..14 对应第 addr 条反对角线的全部有效元素、不足 8 个补 0，由 systolic 的 gather 与 write_out 共同决定。因两种存储顺序不同、无法直接逐地址比，`golden_transform` 任务按 `i+j==k` 把行优先重排成反对角线顺序的 `trans_golden[k]`（k=0..14）加以对齐。验证即逐地址比较 128 位整字 `trans_goldenN[i]==sram_cN.mem[i]`：相等判 PASS，不等打印明细并 `$finish`（首处错即停），比对走的是 SRAM 内部数组 `mem` 这一仿真后门。c0/c1/c2 分别对应 golden1/2/3 及写回端口 a/b/c。此外 `cycle_cnt` 在 `while(~tpu_finish)` 循环内对下降沿计数，统计一次三批矩阵乘的端到端周期数，作为性能探针。
