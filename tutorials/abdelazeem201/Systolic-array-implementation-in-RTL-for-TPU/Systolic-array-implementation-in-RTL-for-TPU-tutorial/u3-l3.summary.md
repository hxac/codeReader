本讲精读 `write_out.v`，这是 TPU 数据通路的最后一站，负责把上游 `quantize` 输出的 128 位量化结果搬运到外部输出 SRAM，不参与任何计算。模块结构为「一时序块 + 三组合块」：三块组合 `always@(*)` 并行算出 a/b/c 各自的下一拍值（用 `_nx` 后缀表示 next），唯一的时序块在时钟上升沿统一寄存到输出端口；复位时写使能默认置 1（即「不写」，安全默认）。贯穿全模块的关键约定是**「0 表示写」的低有效约定**，对接 SRAM 的 `wsb`（write strobe bar）端口。

a/b/c 三组通过 `data_set` × `matrix_index` 互斥仲裁决定本拍写哪块 SRAM：a 专属第 0 趟、c 专属第 1 趟、b 跨两趟承接交界三角形；同拍最多 a+b 或 b+c 双写，a 与 c 永不共存。每拍还受控制器总闸 `sram_write_enable`（仅 `cycle_num>=9` 为 1）门控。

数据摆位遵循两条规则：一是按 `MAX_INDEX`（=ARRAY_SIZE-1=7）做**顺序反转**（元素 i → 位段位置 7-i），使读回顺序与矩阵自然行列一致；二是按 `matrix_index` 分正常型（不断变长）/混合型（不断变短）做**补零**。地址生成上，a/c 直接用 `matrix_index`（0..15），b 用 `matrix_index∓8` 跨半寻址，保证同地址一趟内只写一次。「为何需要三组 SRAM」= 同拍双写需两端口 + 两趟/跨界需三个独立地址空间，是最小配置。
