本讲承接逻辑综合，讲解 PnR（布局布线）如何将门级网表转化为可制造版图。须先指出本项目实为经典 ICC 流程（基于 Milkyway 库与 mw_cel 存档点），并非标题所写的 ICC2。核心是 0~6 七阶段流水线：导入网表与约束、floorplan 画边界（参数 core_utilization、flip_first_row）、搭电源、摆标准单元、CTS 建时钟树、布线、校验导出，每阶段用 save_mw_cel 存一份快照。核心结论有三：其一，CTS 须先于布线，因其插入时钟缓冲器会改变网表与布局，而布线要求网表稳定；其二，CTS 内部 clock_opt 须先于 derive_pg_connection，以使新插入的缓冲器连上电源，否则 LVS 报 PG 悬空；其三，布线遵循「时钟优先」——先 route_zrt_group 布时钟网再布信号网。收尾阶段用 verify_drc/verify_lvs 校验并插填充单元，再由 extract_rc 产出 SPEF，取代综合期 WLM 粗估，成为签字版时序依据，最终导出 GDSII、后版图网表、SDF、SDC、SPEF 等交付物。
