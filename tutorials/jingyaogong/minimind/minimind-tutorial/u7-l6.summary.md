本讲是强化学习后训练单元的收官篇，把 u7-l4 的 GRPO/CISPO 损失接到「多轮工具轨迹 + 延迟可验证奖励」上。核心结论是：Agentic RL 不是新算法，而是同一套 PO 更新骨架喂不同的 rollout/reward。五个最小模块均在 `trainer/train_agent.py`：`AgentRLDataset` 以三元组 \((x,\mathcal{T},gt)\) 组织样本，丢弃黄金 assistant 回答、仅留 gt 供奖励校验；`parse_tool_calls`/`execute_tool` 用正则容错解析 `<tool_call>` JSON，沙盒 `eval` 加 `SIGALRM` 1 秒超时执行；`rollout_single` 把单 prompt 展开成多轮轨迹，关键设计是 `response_mask`——策略生成 token 打 1 参与 loss，环境注入的 observation token 打 0 仅作上下文；`calculate_rewards` 在整条轨迹结束后一次性结算延迟奖励 \(R(\tau)\)，分无工具分支（RM+格式分，RLAIF 式）与有工具分支（gt 命中+工具合法性，RLVR 式），总奖励截断到 \([-3,3]\)；`rl_train_epoch` 的优势、ratio、KL、损失分支与 u7-l4 逐行相同。引入的关键术语：轨迹 \(\tau\)、延迟奖励、response_mask、左截断保近期上下文、首个 EOS 后清零、`*0.0` 防 DDP 挂起。
