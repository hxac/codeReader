本讲以 `trainer/train_pretrain.py` 的 `train_epoch` 为模板，讲解 MiniMind 项目的统一训练底座：DDP 分布式、混合精度与梯度累积。共四个最小模块：(1) `init_distributed_mode` 通过检测环境变量 `RANK` 是否存在来自动判别单卡/多卡模式，`torchrun --nproc_per_node N` 注入 `RANK/WORLD_SIZE/LOCAL_RANK` 并起 N 个进程，`LOCAL_RANK` 决定进程绑定的 GPU；(2) `DistributedSampler` 按 rank 切分数据并需 `set_epoch` 重洗，`DistributedDataParallel` 在 `backward` 时自动 all-reduce 平均梯度使副本同步，保存权重须剥 `.module`/`._orig_mod`，装配顺序为先 `torch.compile` 后 DDP；(3) `autocast` 让矩阵乘降精度，fp16 动态范围小需 `GradScaler` 放大 loss 防梯度下溢，bf16 不需要——靠 `enabled=(dtype=='float16')` 兼容两者；(4) 梯度累积将 loss 除以 N、累加到 `.grad`，凑够 N 步才更新，有效 batch ＝ batch_size × accumulation_steps × world_size。核心结论：更新块顺序严格为 `unscale_` → `clip_grad_norm_` → `scaler.step` → `scaler.update` → `zero_grad`，尾部不足 N 的梯度由补更新处理。该写法是后续 SFT/LoRA/DPO/PPO 等所有训练脚本的通用骨架。
