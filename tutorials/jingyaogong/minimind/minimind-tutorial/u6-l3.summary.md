本讲是参数高效训练与模型压缩单元的收尾，聚焦 MiniMind 中 `train_distillation.py` 这一**白盒知识蒸馏**参考实现。核心区分：**黑盒蒸馏**只见教师最终输出文本、用普通 CE 训练（主线 `full_sft` 即其载体）；**白盒蒸馏**额外拟合教师在输出层的 token 分布 logits。损失为 \(\mathcal{L}=\alpha\,\mathcal{L}_{\text{CE}}+(1-\alpha)\,T^2\,\mathrm{KL}(p_t^T\|p_s^T)\)。引入关键术语：**温度缩放**使教师 logits 除以 \(T>1\) 软化，暴露非 top-1 token 间的相对偏好即 Hinton 所称的**暗知识（dark knowledge）**；前置 \(T^2\) 补偿软化导致的梯度缩小；\(\alpha\)（默认 0.5）平衡硬标签 CE 与软标签 KL。`distillation_loss` 用 `F.kl_div(student_log_probs, teacher_probs, reduction='batchmean')` 计算 \(\mathrm{KL}(\text{teacher}\|\text{student})\)，参数顺序与 reduction 均易错。工程要点：`init_model` 两次调用按 `{weight}_{dim}{_moe?}.pth` 各自定位文件实现**教师/学生双模型加载**，教师须 `eval()`+`requires_grad_(False)` 并师生共用同一分词器保证词表对齐；`train_epoch` 不传 labels、手动从 `res.logits` 切位移，用同一份 `loss_mask`（assistant 段）同时驱动 CE 掩码平均与 KL token 筛选，日志同时输出 loss/ce/aux_loss/distill，导出权重名为 `full_dist_{dim}{_moe?}.pth`。
