本讲义讲解 `serve_openai_api.py` 如何把 MiniMind 的 `model.generate` 包装成兼容 OpenAI Chat Completions 协议的 HTTP 服务，补齐流式、思考、工具调用三件原生 generate 缺失的能力。核心模块有四：`ChatRequest`（Pydantic 请求模型，`get_open_thinking()` 兼容顶层与 `chat_template_kwargs` 两种开启思考的入口）；`CustomStreamer`（继承 `TextStreamer`，用后台线程跑生成、`Queue` 传片段、`None` 作结束哨兵的生产者-消费者桥）；`parse_response`（正则把 `<think>` 解析为 `reasoning_content`、`<tool_call>` 解析为标准 `tool_calls`，`arguments` 为 JSON 字符串，并对半截 `</think>`、非法 JSON 容错）；`/v1/chat/completions` 端点（FastAPI + SSE，流式用 `full_text`+`emitted` 双指针在 `</think>` 处切换 `reasoning_content`/`content`，末尾补发 `tool_calls` 与 `finish_reason`，非流式则一次性 `parse_response`）。关键术语：SSE、`StreamingResponse`、`reasoning_content`、`tool_calls`、`open_thinking`、`enable_thinking`。注意流式会把 `<tool_call>` 原文随 `content` 发出，非流式则清理干净。
