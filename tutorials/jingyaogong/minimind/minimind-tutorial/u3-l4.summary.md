本讲拆解 MiniMind 的 Transformer 前馈段。Dense 模型用 `FeedForward`（SwiGLU）：三个无偏置投影 `gate_proj/up_proj/down_proj`，算式为 `down(silu(gate(x)) * up(x))`，门控支路过 SiLU 后与线性支路逐元素相乘，比朴素 FFN 多一条门控支路。MoE 模型用 `MOEFeedForward`：`gate` 路由器对每个 token 打分，经 `softmax→topk→norm_topk_prob` 选出 k 个专家，再用 `for` 循环加 `index_add_` 把 token 分桶送入对应专家、加权回填。默认 4 专家/top-1，总参数约 4 倍但每 token 仅激活 1 个专家（稀疏激活换大容量），多出的约 50% 耗时来自路由与 kernel 调度开销。`aux_loss` 采用 Switch Transformer 公式 `N·c·Σf_i·P_i`，靠可导的平均概率 P_i 把梯度引回 gate 以均衡专家负载，仅训练时计算、推理为 0；DDP 兼容则靠 `0*` 占位项让没收到 token 的专家参数仍进入计算图。`MiniMindBlock` 用一行三目 `FeedForward if not use_moe else MOEFeedForward` 收敛 Dense/MoE 差异，保持 Pre-Norm 加双残差；`aux_loss` 经 `MiniMindModel` 聚合后，在训练脚本中与交叉熵主损失相加。
