本讲拆解 CausalLM 一次 forward 从 input_ids 到 loss 的完整链路。MiniMindModel（躯干）执行 embed → 逐层 Block → 末层 RMSNorm，产出 hidden_states、KV Cache 及聚合后的 aux_loss，不算损失也不出 logits；Dense/MoE 共用此代码，aux_loss 用一行 sum(..., new_zeros) 兼容两者——Dense 返回与 hidden_states 同设备同 dtype 的标量 0，MoE 返回各层负载均衡损失之和，故无需 if 分支。MiniMindForCausalLM（外壳）额外接 lm_head 把 hidden_states 映射成词表 logits，并提供效率钩子 logits_to_keep：默认 0 因 slice(-0) 等价 slice(0) 表示「保留全部位置」（反直觉），训练必须如此，推理 prefill 可传 1 省 lm_head 矩阵乘。传入 labels 时用位移交叉熵（logits[:-1] 对 labels[1:]）实现 next-token prediction，ignore_index=-100 屏蔽提问/padding，只对回答算 loss；不传 labels 则 loss=None。全部产物打包进 MoeCausalLMOutputWithPast 统一返回容器；训练侧 loss = res.loss + res.aux_loss，Dense 自动退化为纯 CE，日志中 logits_loss = loss − aux_loss 为纯语言建模损失。
