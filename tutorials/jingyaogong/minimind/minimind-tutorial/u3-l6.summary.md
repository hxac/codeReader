本讲拆解 MiniMind 自研的 `generate` 与 `TextStreamer` 两个模块，讲清推理时模型如何逐字生成。核心是自回归循环配合 KV Cache 增量推理：首步 prefill 全量缓存 prompt 的 K/V，之后每步 decode 只送入新 token，使单步计算量与序列长度无关，显著快于朴素全量重算。采样四件套按温度、重复惩罚、top_k、top_p 的顺序改写最后一位 logits，再由 `do_sample` 切换 multinomial 采样与 argmax 贪心，前者多样、后者确定可复现。batch 生成时，先吐 EOS 的序列被标记 `finished`，并在后续每步强制补 EOS 以维持张量对齐，全部结束则提前终止。streamer 作为可注入回调实现逐字流式输出，降低首字延迟；扩展点 `num_return_sequences` 可一次复制 batch 出多条候选，`return_kv` 额外返回缓存，供多轮续聊或后续 rollout 复用。
