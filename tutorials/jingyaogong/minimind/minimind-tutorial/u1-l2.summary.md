本讲只讲环境与目录，不涉模型原理。MiniMind 按「数据流方向」分四层：`dataset/`（jsonl→张量）、`model/`（模型结构定义，核心为 `model_minimind.py`）、`trainer/`（8 个 `train_*.py` 对应预训练/SFT/LoRA/蒸馏/DPO/PPO/GRPO/Agent 各阶段，须 `cd trainer` 执行）、`scripts/`（推理/部署/评测/转换），根目录 `eval_llm.py` 为 CLI 推理入口；`out/`、`checkpoints/` 训练时才自动生成。`requirements.txt` 中 `torch`/`torchvision`/`peft` 均被注释——torch 须按本机 CUDA 版本单独安装，peft 注释因 LoRA 由项目从 0 自实现。第0步即 `git clone --depth 1` 配 `pip install`，国内推荐阿里云镜像。用 `torch.cuda.is_available()` 一行确认后端（cuda/mps/cpu），返回 False 多为装成 CPU 版。所有 `.jsonl` 须放 `./dataset/`，最快复现 Zero 模型的最小组合为 `pretrain_t2t_mini.jsonl` 与 `sft_t2t_mini.jsonl`。
