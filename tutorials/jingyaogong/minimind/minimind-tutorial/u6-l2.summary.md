本讲承接 u6-l1（LoRA 模块实现），讲解如何把 LoRA 跑成一个完整训练任务，核心结论是「LoRA 训练 = Full SFT 训练骨架 + 只放开旁路参数 + 换更大学习率」。讲义拆为五个最小模块：其一，通过「双重保险」确保只更新 LoRA 参数——把不含 lora 的参数 `requires_grad` 置 False（冻结 base，省显存），并把仅含 lora 的 `lora_params` 列表喂给 `AdamW`（限定更新对象）；其二，`train_epoch` 与全参 SFT 几乎逐行同构，实质差异仅是梯度裁剪对象改为 `lora_params`、保存改调 `save_lora`；其三，权重遵循 `lora_xxx_{dim}{_moe?}.pth` 命名规则，`save_lora` 只存旁路 A/B 权重（fp16、剥外壳），文件极小；其四，垂域适配（`lora_medical`/`lora_identity`）复用 `SFTDataset`，推理时由 `eval_llm.py --weight full_sft --lora_weight xxx` 做「基模 + LoRA 增量」组合加载；其五，因 `apply_lora` 的 monkey-patch 实例级 forward 与 `torch.compile` 的类级静态追踪冲突，脚本会自动禁用 compile。关键术语：`requires_grad`、垂域适配、monkey-patch、冻结/过滤双重保险、`lora_params`、组合推理。
