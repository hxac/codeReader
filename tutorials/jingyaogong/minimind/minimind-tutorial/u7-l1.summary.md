本讲围绕 MiniMind 从 0 实现的 DPO（直接偏好优化），覆盖四个最小模块：偏好数据与参考模型、`logits_to_log_probs`、`dpo_loss`、以及贯穿后续 RLAIF 的 PO 统一框架。核心结论：DPO 用「一问 + chosen（好答）+ rejected（差答）」的偏好数据，直接最大化 chosen 优于 rejected 的对数几率，无需训练任何 Reward / Value 模型。关键术语包括：参考模型（π_ref，与策略模型同结构同初始权重但被冻结、在 no_grad 下前向，充当参照系）、对数概率比（log-ratio = logπ_θ − logπ_ref）、温度系数 β（控制策略偏离 ref 的程度）。实现上，`logits_to_log_probs` 用 log_softmax + gather 抽出逐 token 对数概率；`dpo_loss` 用 mask 把回答段对数概率求和成序列对数似然，再按前一半 chosen / 后一半 rejected 切开算 logits，损失为 −logsigmoid(β·logits)。工程上 chosen/rejected 被 cat 成翻倍 batch 一次前向，故函数内 batch_size 为 2 × 数据 batch。最后立起统一视角：所有 xxPO 都在优化「策略项 f(r_t) × 优势项 g(A_t) − 正则项 h(KL_t)」；DPO 的策略项是 log r_w − log r_l、优势项无显式、正则隐含在 β 中，为后续 PPO/GRPO/CISPO 的「三项填法」对比奠定基础。
