本讲讲解 ringbuf 写端的统一抽象 `Producer` trait（继承自 `Observer`，写端必先会观测）。方法分两层：底层直接写入（`vacant_slices_mut`、`advance_write_index`，多为 `unsafe`）与高层便捷写入（`try_push`、`push_slice`、`push_iter`）。所有写入遵循同一范式：观测 → 写入 `MaybeUninit` 空闲槽 → 推进 write 索引提交；只有 write 索引前进后数据才对消费端可见，在 `SharedRb` 上这次推进就是一次 `Release` 原子 store。

`vacant_slices_mut` 把 `[write, read+capacity)` 区间切成最多两段线性切片，写入后必须配一次 `advance_write_index(count)`，且 `count` 不得超过实际写入数——这是整套 `unsafe` 的核心安全前提。`try_push` 是该范式最小实例，满则 `Err(elem)` 原样退回、不阻塞，体现 lock-free 语义；`push_slice`（要求 `Copy`）与 `push_iter`（不要求 `Copy`）采用「一次提交」：写完所有能写的元素后只调用一次 `advance_write_index`，使写 N 个元素的跨核同步开销仅为 `try_push` 循环的 1/N。委托机制上，`DelegateProducer`（空标记 trait）+ blanket impl 让包装器零成本复用全部写入方法；`impl_producer_traits!` 宏则为任何 `Item = u8` 的 `Producer` 自动实现 `std::io::Write` 与 `core::fmt::Write`。
