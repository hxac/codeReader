本讲是理解 ringbuf 的基石，只讲索引算术，覆盖两个最小模块。

模块一：双索引与 2*capacity 模数。ringbuf 用两个落在 0..2*capacity 的整数索引描述状态：read 指向最旧元素，write 指向下一个空槽，物理槽位 = 索引 % capacity。经典难题是仅凭对 capacity 取余无法区分「空」与「满」。ringbuf 的解法是把模数从 c 改为 2c（由 modulus() 提供），于是空 ⟺ read==write，满 ⟺ (write-read) % (2c) == c，既不牺牲槽位、也不加额外字段。索引推进一律 %(2*capacity)；occupied_len/vacant_len 用「先加模数再取余」避免绕圈下溢；合法状态要求占用不超过 c。Observer trait 暴露 read_index/write_index 及 is_empty/is_full 等。

模块二：ranges()。它对环形区间 [start,end)，用「索引除以 capacity 的商」的奇偶判断是否绕过数组末尾，把区间拆成最多两段连续线性切片，是所有批量与切片访问（push_slice、as_slices、unsafe_slices）的根基。这些算术对 LocalRb（Cell）与 SharedRb（原子）完全相同，差别仅在存储介质与是否跨核同步。
