本讲聚焦 ringbuf 的 SPSC 不变量及其运行时守卫——hold 标志。核心结论是：ringbuf 作为无锁 SPSC FIFO，「至多一个生产者、至多一个消费者」是其无锁安全的前提，由此保证写者独占 write 索引与 vacant 槽、读者独占 read 索引与 occupied 槽，两端集合永不相交，故不会并发触碰同一元素，仅需 Acquire/Release 同步两个索引。

为实现该不变量，`SharedRb` 用两个 `AtomicBool` 字段 `read_held` / `write_held` 记录两端占用状态。关键术语包括：`RingBuffer::hold_read` / `hold_write`（两个 unsafe 方法，采用 test-and-set 语义，底层用 `swap(flag, AcqRel)` 置位并返回旧值）；`Observer` trait 的只读查询 `read_is_held` / `write_is_held`（Acquire load）。`AcqRel` 因 `swap` 是读改写操作而成为正确内存顺序。

包装器（`Direct` / `Frozen` / `Caching`）把 unsafe 的 `hold_*` 绑定到安全生命周期：`new` 时 `assert!(!hold_*(true))` 置位，`close` / `Drop` / `into_rb_ref` 时 `hold_*(false)` 复位；对已 split 缓冲区再次拆分会在置位断言处 panic，从而强制不变量。

本讲还澄清两点：trait 层不写 `T: Send`，跨线程时约束由 auto-trait 推导浮现，安全性正源于 SPSC 的单向 hand-off；`push_overwrite` 因同时动 read/write 两个索引必须 `&mut self` 独占，仅在未拆分缓冲区可用，并发使用需外加 mutex。
