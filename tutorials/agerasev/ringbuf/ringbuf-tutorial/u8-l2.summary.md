本讲承接 u2-l2（Storage 抽象）与 u2-l3（LocalRb/SharedRb），讲二次开发：当内置的 Array/Slice/Heap/Ref 四种存储后端不够用时，如何接入外部内存（内存映射文件、DMA 缓冲、静态区、裸内存）。全讲由三个最小模块构成。其一，Storage 契约：自定义存储只需实现 `unsafe trait Storage` 的两个必需方法 `as_mut_ptr` 与 `len`，骨架照抄内置 `Ref`（裸指针+长度+生命周期幻影）；它须满足三条安全契约——不与自身数据别名（这才使 `slice_mut` 能在 `&self` 上返回 `&mut`，靠 `#[allow(mut_from_ref)]` 放行）、`as_mut_ptr` 指向有效数据、`len` 恒定；跨线程用 SharedRb 时须手写以 `T: Send` 为条件的 `Send`/`Sync`，因元素是跨线程搬运而非并发共享。其二，`from_raw_parts(storage, read, write)` 用外部内存构造缓冲区，read/write 是 `[0, 2*capacity)` 内的原始索引，划定初始化边界（区间内已初始化、外未初始化），容量为 0 会 panic；构造器宏中 Default 传 (0,0)、From<[T;N]> 传 (0,N)。其三，`into_raw_parts()` 是其逆运算，靠 `ManuallyDrop`+`ptr::read` 把带数据的存储原样交还，析构责任随之转移给调用者。引入的关键术语：二次开发、裸指针+长度、ManuallyDrop、ptr::read、原始索引、初始化边界、析构责任转移。
