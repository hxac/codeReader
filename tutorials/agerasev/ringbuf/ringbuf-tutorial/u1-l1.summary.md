ringbuf 是无锁（lock-free）、单生产者单消费者（SPSC）、先进先出（FIFO）的环形缓冲区，支持直接访问内部内存，用于在不加锁、不阻塞的前提下于两个执行流间高效传递一连串元素。本讲围绕"定位、特性、crate 结构"三模块完成项目认识。

关键术语：缓冲区用于解耦生产与消费的速度差异；环形缓冲区以 read/write 两个索引反复复用固定内存；SPSC 约束"至多一个写端、一个读端"，由 hold 标志运行时强制；无锁即读写"立即成功或失败"——try_push 满时返回 Err(item)、try_pop 空时返回 None。典型用法四步：创建 → split 拆为 Producer/Consumer → try_push → try_pop。

核心特性：元素类型任意（不要求 Copy）、支持单/批量、可经切片直接访问内部内存、实现 Read/Write、overwrite 覆盖写入（满时挤掉最旧元素，因会移动 read 索引故需独占访问）、支持 no_std/no-alloc；单线程用 LocalRb（无跨核缓存同步、稍快），多线程用 SharedRb（常用实例 HeapRb、StaticRb）。

项目是 Cargo workspace：核心 crate ringbuf 仅提供非阻塞 try_* 接口，派生 crate async-ringbuf 加 async 同步、ringbuf-blocking 加阻塞同步，三者共用同一核心；核心泛型 SharedRb<S> 更换存储后端 S 即得不同缓冲区（HeapRb=SharedRb<Heap>、StaticRb=SharedRb<Array<N>>），内部由 Storage、Indices、Hold flags 三部分组成。
