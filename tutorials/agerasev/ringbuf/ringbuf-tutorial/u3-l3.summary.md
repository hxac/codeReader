本讲覆盖 `Consumer` trait（读端统一抽象，`Consumer: Observer`），核心是读端镜像写端的统一三步范式：**观测 → 从 `MaybeUninit` 槽 `assume_init_read` 移出元素 → `advance_read_index` 推进 `read` 索引提交**（对 `SharedRb` 为 Release 原子 store）。只有第 3 步发生后元素才算真正离开缓冲区。方法分三层：索引层（`set_read_index`/`advance_read_index`，unsafe）、切片层（`occupied_slices(_mut)`）、便捷层（带默认实现的派生方法）。

引入的关键术语与概念：`try_pop`（移除、FIFO、空返 `None`）对照 `try_peek`/`first`/`last`（只读不移除，差在第 3 步）；批量方法 `pop_slice`（要求 `Copy`，一次性推进索引以摊薄同步开销）与 `peek_slice`（不推进）；三件套迭代器——`iter`/`iter_mut`（只读）、`PopIter`（**惰性移除、延迟提交**，仅 `commit()` 或 `Drop` 时推进，提前退出不误删）、`IntoIter`（每次立即提交）。`skip`/`clear` 先 `drop_in_place` 再推进索引，安全回收非 `Copy` 资源。另提及 `DelegateConsumer` 委托机制（详见 u3-l5）与 `impl_consumer_traits!` 宏生成的 `std::io::Read`（`Item = u8`）。
