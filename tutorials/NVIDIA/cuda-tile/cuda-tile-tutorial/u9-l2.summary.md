本讲剖析 LoopSplit Pass（循环分裂）：它把循环体内条件为「归纳变量 iv 与循环不变量比较」的 if 沿翻转点切成两段 for，每段条件恒定，从而删除每迭代一次的分支判断并消除 GPU 分支发散。判定总函数 isSplittableCondition 依次过五关：阈值非 0、条件是 cmpi、normalizeForOpCmp 通过、rhs 是循环不变量、谓词合法且收益达标。normalizeForOpCmp 把任意方向的比较统一规范化为「iv op value」并翻转等价谓词，同时强制 cmp 符号性与 for 的 unsignedCmp 一致。isSplitProfitable 用阈值（默认 1）短路，否则看分支操作数是否达阈值或是否含九类「重操作」（访存、MMA、归约、嵌套控制流）——重操作无视阈值触发分裂。performLoopSplit 计算分裂点（部分谓词 +1、非 1 步长做 lb+⌈(split-lb)/step⌉ 向上取整对齐、mini/maxi 夹逼到 [lb,ub]），copyLoop 用 IRMapping 克隆两段并把 cmpi 替换为常量、仅保留恒定一侧分支体。阈值来源优先级为 if 属性 > for 属性 > entry 属性 > 命令行 --split-threshold，0 表示禁用；属性名 cuda_tile.loop_split 是 discardable 属性。该 Pass 经 TableGen 声明为绑定 FunctionOpInterface 的 InterfacePass，属语义保持的结构变换（不改计算结果、仅改控制流），与 u9-l1 的 FuseFMA（非数值保持的算术重写）相对照。
