本讲讲解 CUDA Tile 方言的**调试信息（DI）建模与校验**。核心是 DI 属性类层级 `DINodeAttr → DIScopeAttr → DILocalScopeAttr`：`di_file`/`di_compile_unit` 是顶层作用域，`di_subprogram`/`di_lexical_block` 是函数内局部作用域，`di_loc` 是把「行号列号 + 局部作用域」打包而成的自定义 Location 属性。关键类型约束是 `di_loc` 的 scope 限定为 `DILocalScopeAttr`，从类型层禁止把指令直接挂到文件/编译单元上；词法块可任意嵌套，形成以子程序为根的作用域树。官方范例 `debuginfo_attr.mlir` 演示了「文件→编译单元→子程序→词法块→位置」的完整声明，并用 `--mlir-print-debuginfo` 开关查看 DI 信息。`DebugInfoVerifier` 通过 6 条一致性规则约束函数级与指令级 DI：核心是 Rule 2（函数名须等于子程序 linkageName）、Rule 3（函数无 scope 则体内操作也不得有）、Rule 4（操作 scope 经词法块沿 scope 链归约后须与函数同属一个子程序）。两个辅助函数 `getDILoc`（递归剥除各类 Location 包装）与 `getSubprogram`（将词法块归约到子程序）保证校验鲁棒。校验分两阶段挂载于 `EntryOp::verify()`（函数自身）与 `verifyRegions()`（函数体）。后续讲义将承接 DI 属性的 `sinceVersion="13.1"` 标记、字节码 Debug 段序列化、以及 `cloneWithNewLinkageName` 在变换中的使用。
