本讲是「进阶·计算类操作」首篇，精读 `cuda_tile` 方言 Core 组的 9 个操作。它们派生自 `CudaTileCoreOpDef`、多带 `Pure` trait，是无副作用的数据搬运与形状变换工具，按职责分四模块：构造用 `constant`（标量广播态与稠密列表态）和 `iota`（一维 `[0,n-1]` 整数序列，元素数不得超类型上限）；形状用 `reshape`（元素数与类型不变，按行主序改形状）和 `broadcast`（只拉伸大小为 1 的维、不改秩，改秩须先 reshape）；重组用 `extract`（按整除大小切完整切片）、`cat`（沿 `dim` 拼接）、`permute`（按排列重排维度）；选择与地址用 `select`（元素级 `cond?x:y`，cond 须为 i1 Tile）和 `offset`（按 pointee 位宽把整数偏移换成指针 Tile 的地址增量）。核心结论是 README 的标准地址构造链 `iota→reshape+broadcast→offset→load_ptr_tko`：把单个基址扩成指针 Tile 再逐元素加偏移供访存。本讲只构造地址，真正读写留待 u5 访存讲义，算术运算则见 u4-l2。
