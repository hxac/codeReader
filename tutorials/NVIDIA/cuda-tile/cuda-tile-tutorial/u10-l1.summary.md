本讲讲解 CUDA Tile 的 C API 集成接口（目录 `cuda_tile-c`，实现于 `lib/CAPI/`），它是 C++ 库面向跨编译器、跨语言互操作的稳定 C 边界，Python 绑定等也建立在 MLIR C API 之上。其实现遵循 MLIR 的「不透明句柄 + `wrap`/`unwrap`」桥接套路：C 句柄经 `unwrap` 拆出 C++ 对象、调用真实实现、再用 `wrap` 装回。讲义覆盖三个最小模块：注册入口（`Registration.cpp` 把 `mlirCudaTileRegisterAllDialects`/`mlirCudaTileRegisterAllPasses` 翻译为方言 `insert` 与 `registerCudaTilePasses`）；方言能力清单（`CudaTileDialect.h` 暴露类型、属性、字节码写入与逐个 Pass 注册，其中 `GetChecked` 版本在类型非法时返回空句柄而非抛异常）；优化器 C API（`mlirCudaTileOptConfig` 用结构体+位掩码+诊断回调表达配置，经 `toCpp` 翻译为 `TileIROptimizerOptions`，由 `mlirCudaTileApplyOptimizations` 转调 `optimizeTileIRModule`）。CMake 集成部分将 C API 编为 `CudaTileCAPIRegistration`/`CudaTileCAPIDialects`/`CudaTileCAPIOptimizer` 三个公共库，受 `CUDA_TILE_ENABLE_CAPI` 控制，README 给出预编译库与源码 `FetchContent` 两种集成方式。
