本讲深入 `cuda_tile` 方言属性层，含四个模块：属性定义机制、`OptimizationHintsAttr` 结构、逐操作校验、告警/错误开关。属性分两类：带参数自定义属性（由 `CudaTileAttrDef` 包装，比 MLIR 原生 `AttrDef` 多 `sinceVersion` 等字段，用于字节码版本兼容与规范生成）与枚举属性（`CudaTileI32EnumAttr`/`CudaTileEnumAttr`，承载舍入、整数溢出、原子模式、内存序、内存作用域等），二者均经 `.td` 声明、`tblgen` 生成 `AttrDefs.inc`、`registerAttributes` 注册。主角 `OptimizationHintsAttr` 采用「外层按 SM 架构（`sm_80`..`sm_121`）或 `default` 分桶、内层逐提示给值」的嵌套字典，底层仅一个 `DictionaryAttr`，语义由 `extraClassDeclaration` 与 `Attributes.cpp` 赋予；五类提示分挂两类操作：`entry` 接 `num_cta_in_cga`/`num_worker_warps_per_cta`/`occupancy`，访存接 `latency`，视图访存再加 `allow_tma`；逐操作支持矩阵由 `verifyWithOp` 用 `isa<...>` 动态决定，查询遵循「精确架构优先、`default` 兜底」。最关键且反直觉的设计是：非法/不支持的提示**默认完全静默**——`verifyParamWithContext` 开头即 `if (!getWarnUnsupportedHints()) return success()` 早返回，以保证前端 lowering 的前向兼容；`-Wunsupported-hints` 把静默升级为 per-param warning，`-Werr-hints`（依赖前者）使 `verifyWithOp` 返回失败、由 `verifyOptHintsCommon` 发出 op 级 error，但 per-param 诊断级别不变。
