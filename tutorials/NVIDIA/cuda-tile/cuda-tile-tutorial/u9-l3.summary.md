`cuda-tile-optimize` 是 CUDA Tile IR 工具链中「字节码进、字节码出」的独立优化器，与操作 MLIR 文本的 `cuda-tile-opt`、做 MLIR↔字节码翻译的 `cuda-tile-translate` 各司其职。本讲围绕三个最小模块展开：命令行工具入口（`Options` 结构定义 `cl::opt` 选项，`main` 极薄，仅翻译配置再调库）、配置契约（`TileIROptimizerConfig` 聚合 `TileIROptimizerOptions`/`Input`/`Output`，输出模式 `TileIROptOutputMode` 为位掩码可组合）、管线执行（`optimizeTileIR` 带 I/O、`optimizeTileIRModule` 只跑管线）。核心结论：默认 `-O3` 管线为 FuseFMA→Canonicalize→CSE→LICM→LoopSplit→Canonicalize，整条经 `pm.nestAny()` 嵌套在 `cuda_tile::EntryOp` 上，故 `--before`/`--after` 文本不可带锚点；FuseFMA 为「非数值保持」变换，独立于 optLevel、须显式 `--fuse-fma`；`loopSplitThreshold` 工具固定为 1 不暴露；`loadInputModule` 凭 magic 自动识别字节码或 MLIR 文本；命令行与 C API 在 `optimizeTileIRModule` 汇合，构成「单一管线、两个入口」。
