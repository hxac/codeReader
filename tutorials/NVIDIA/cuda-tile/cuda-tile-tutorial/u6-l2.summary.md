本讲介绍 `cuda_tile.assume` 操作及其静态假设谓词机制。核心结论是：`assume` 是零运行时开销的「贴标签」操作，它原样透传输入 SSA 值（`AllTypesMatch` 保证结果与输入类型一致），仅把一条谓词属性附加到结果上供编译器生成更优代码；它不检查谓词正确性，错误假设会导致未定义行为，这与运行时 `assert` 截然不同。本讲引入的关键术语包括：静态假设、`AssumePredicateAttrInterface` 接口（唯一方法 `verifyWithAssumeOp`，使 `assume` 的 verifier 把合法性校验完全委派给谓词自身，实现解耦）。三种内置谓词各有适用范围：`div_by`（整除性，divisor 必须为正 2 的幂，支持整数/指针 tile 与 tensor_view，可选 `every`/`along` 分组但必须成对）；`same_elements`（逐维同元素组大小，仅整数/指针 tile，数组长度须等于秩，全 1 为安全默认值）；`bounded`（有符号值域 `[lb,ub]`，两侧均可用 `?` 省略，仅整数 tile 不支持指针）。此外还涵盖谓词的简写解析/打印语法（`custom<AssumePredicate>`）与 `assume` 的 fold 规范化（仅当连续 assume 谓词完全相同时折叠冗余链），以及 Python 端因缺专门绑定而采用拼接文本属性字符串再 `Attribute.parse` 的变通实现。
