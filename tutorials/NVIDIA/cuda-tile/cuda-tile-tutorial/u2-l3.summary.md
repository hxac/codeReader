本讲承接方言定义，说明 `.td` 声明如何经代码生成（codegen）变为可编译产物，核心是工具 `cuda-tile-tblgen` 与其驱动的生成流水线。最小模块有四：一是工具入口——`cuda-tile-tblgen` 是上游 `mlir-tblgen` 的超集，通过链接 `MLIRTblgenLib` 复用全部标准后端，并用 `mlir::GenRegistration` 静态自注册机制额外挂入 `-gen-op-spec`、字节码后端等自定义动作，最终委托 `MlirTblgenMain`。二是两类生成并存的区分：`mlir_tablegen(...)` 走上游工具产出 C++ 胶水（`Ops.h.inc`、`Types.h.inc`、`AttrDefs.h.inc`），手写头文件以 `#define GET_OP_CLASSES` + `#include "Ops.h.inc"` 的 include 技巧拼接；`tablegen(CUDA_TILE ...)` 走本项目工具产出规范 `ops.rst` 与字节码胶水。三是 `gen-op-spec` 后端：`generateSpec` 经 `splitBySections` 按 `cudaTileSections[]` 硬编码顺序分箱排序、收集 `AttrDef`、对每个操作 `emitOpDoc` 渲染，并跳过 Testing 分组。四是 `SpecEmitter` 与 `Emitter` 值类型（`Header`/`Table`/`Badge`/`Code`）封装 RST 排版，`--examples-directory` 让示例落盘为独立 `.mlir` 并经 `literalinclude` 引用。贯穿全讲的「单一数据源」思想：同一份 `.td` 记录同时驱动 C++、规范、字节码三类产物。关键术语：TableGen 记录（Record）/ 后端（Backend）/ `.inc` 产物、`GenRegistration`、`MlirTblgenMain`、include 技巧、`CudaTileOp` 元数据中间层、`sinceVersion`/`cudaTileSpecGroup` 分组、`SpecEmitter`、`ops.rst` 规范文档。
