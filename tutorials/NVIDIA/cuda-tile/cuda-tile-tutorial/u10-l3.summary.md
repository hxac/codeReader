本讲介绍 CUDA Tile IR 测试体系，复用 LLVM 的 lit + FileCheck 范式。lit 只负责跑 RUN 行、看退出码，本身不判对错；FileCheck 按期望行匹配输出。测试按被测对象分目录存放，白名单后缀 `.mlir`/`.c`/`.py` 各走一条运行路径：`.mlir` 经 `cuda-tile-*` 工具管道、`.c` 编为可执行程序、`.py` 走 pytest，由 `lit.cfg.py` 的后缀白名单与排除名单管控，子目录可用 `lit.local.cfg` 做局部开关（如未启用绑定则跳过 `python/`）。CMake 目标 `check-cuda-tile` 先构建依赖（`FileCheck`/`not`/三个 `cuda-tile-*` 工具/Python 模块/CAPI 程序）再跑 lit；`check-cuda-tile-build-only` 只构建不跑，服务 CI 的构建/测试分离。配置分两层：`lit.site.cfg.py.in` 是 CMake 模板注入机器相关路径，`lit.cfg.py` 定义替换符，`%round_trip_test`（Windows 用 `.py`、Linux 用 `.sh`）、`%PYTHON`、工具名均在后者注册。核心机制是 round-trip 测试：MLIR→字节码→MLIR 后与用 `cuda-tile-opt` 同源生成的参考文本去空行比对，一次性验证字节码写入器与读取器。此外，仅测试构建注册的测试 Pass（`TestPasses.cpp` 经 `#ifdef CUDA_TILE_ENABLE_TESTING` 挂到 `cuda-tile-opt`）为缺乏命令行入口的编译器内部工具函数提供可测入口。
