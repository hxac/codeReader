本讲阐述 CUDA Tile 的 Python 绑定是一套「三层金字塔」架构：底层为 C API（u10-l1 的 wrap/unwrap 桥，作为稳定 ABI 边界），中层是两个 nanobind 原生扩展模块，上层是手写高层 API。

最小模块有三：一是 `SiteInitializer.cpp` 编译出的 `_site_initialize_1`，依赖 MLIR 的 site-initialization 机制——它按约定名字在 import 期自动调用 `register_dialects` 完成方言注册，而 `register_passes` 不是约定钩子、必须用户手动调用；二是 `DialectCudaTile.cpp` 编译出的 `_cuda_tile`，用 `mlir_type_subclass`/`mlir_attribute_subclass` 适配器把 C API 包装成 `PointerType`/`TileType`/`TokenType` 等类与 `applyTileIROptimizations`/`writeBytecode` 函数，遵循「isa 判定＋get 类方法＋upcast_type＋只读属性」四件套模板，`TileType.get` 用 `GetChecked` 版本、失败返回 `None`；三是 `cuda_tile_ops.py`，靠 `Tile`/`Pointer`/`TileView` 包装类与运算符重载、`cuda_tile_op` 装饰器、`return_results` 智能分派，把构造 SSA 值做成 NumPy 式写法。核心结论：高层 API 校验更友好（`None` 升级为异常、MMA 注册表前置校验），与 C++ verifier 形成纵深防御。
