本讲拆解 CUDA Tile 方言的 Memory 分组操作，核心是「显存访存」与「访存顺序」两件事。最关键的心智模型是：名字带 `_tko` 后缀的 token-ordered 操作（如 load_ptr_tko、store_ptr_tko）默认不受程序顺序约束，编译器可自由重排，需要排序时只能用 token 显式串接。token 是一种不携带运行时数据的纯依赖边，由 `make_token`（造新 token）和 `join_tokens`（汇合≥2 个 token，合成依赖全部输入的新 token）操纵。`load_ptr_tko` 基于指针 tile 做 gather 读取（返回数据 tile + token），`store_ptr_tko` 做 scatter 写入（只返回 token），二者都支持可选 `mask`（tile<i1> 屏蔽）与 load 专属 `padding`（屏蔽处填充值）。内存序取值 weak/relaxed/acquire/release/acq_rel 与作用域 tl_blk/device/sys 配合：load 允许 weak/relaxed/acquire、store 允许 weak/relaxed/release，且 weak 与 memory_scope 互斥、其余必填 scope，由 verifyMemoryModelLoad/Store 强制。类型校验抽成可复用三件套 verifyLoadStoreType（source 是 result 的指针版）/verifyLoadStoreMask（同形状）/verifyLoadPadding（同形状同类型），以 trait 挂在 LoadOpBase/StoreOpBase 上。`alloca`（13.3 引入）做块内临时分配，返回标量指针 tile，alignment 须为 2 的幂且不小于元素自然大小，生命周期限于所在 block。本讲是 u4-l1 地址构造链的终点，所引入的 token、mask、padding、内存序等概念会被后续 u5-l2（视图访存）与 u5-l3（原子操作）直接承接。
