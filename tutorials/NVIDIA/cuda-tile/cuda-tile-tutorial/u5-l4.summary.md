本讲讲解 cuda_tile 方言 Control Flow 组的 8 个操作，按角色分三类：容器（for/loop/if）、终止符（yield/continue/break/return）、诊断（assert）。核心设计原则是「父操作决定子终止符语义」，由 TableGen 的 ParentOneOf 在语法层约束「谁能装在谁里」。for 是结构化范围循环，归纳变量按左闭右开区间 [lb,ub) 以正步长 step 步进，支持 iter_values 循环携带值且入口与返回类型相同，体块以 continue 收尾，不能提前退出。loop 是无限循环，靠 break 退出、continue 进入下一轮，因 break 独立交还最终值，返回类型可与入口不同。if 接受标量 tile<i1> 条件，then/else 分支用 yield 交还值，带返回值时两分支 yield 类型须一致且 else 不可省，结果不能是 tensor_view 等视图类型。关键工程工具 ControlFlowImplicitTerminatorOpType 是 MLIR 隐式终止符机制的多终止符增强版，使 if 块接受 yield/break/continue/return、loop 块接受 continue/break，缺省时分别补 yield 与 continue；break/continue 穿透 if 回到最近循环的行为由 verifyEarlyExitOp 在语义层收紧，形成「语法宽松、语义严格」的双层校验。本讲承接 u4-l2 的 tile<i1> 条件类型；其 for/if 结构与 yield 语义将分别被后续优化讲义（LoopSplit、归约扫描）承接。
