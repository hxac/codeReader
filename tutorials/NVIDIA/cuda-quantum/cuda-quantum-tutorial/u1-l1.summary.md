CUDA-Q 是面向「混合量子-经典计算」的编程平台，让 CPU、GPU、QPU 在同一系统内被统一编程与调度，同时提供 C++ 与 Python 两套前端。它本质上是一个「编程模型 + 编译器」项目，模拟器与真实 QPU 都是它能瞄准的目标，而非其自身。混合算法的典型执行形态是「量子侧跑参数化线路采样，经典侧计算目标函数、梯度并更新参数」的低开销交替循环，采样概率与期望值是其中反复出现的基本量。

仓库由四大子系统组成：`cudaq/`（基于 MLIR 的编译器）、`runtime/`（C++ 运行时与 nvqir 模拟器，是公共底盘，Python 前端最终复用它）、`python/`（Python 前端）、`realtime/`（FPGA 与 GPU 紧耦合的低延迟实时控制子系统）。

本讲引入的关键术语：Quake（CUDA-Q 自定义的量子方言，有内存语义与值语义两种比特建模）、CC（建模 Quake 函数中经典逻辑的方言）、QIR（基于 LLVM 的开放量子中间表示规范，是最终 lowering 目标）、nvq++（编排编译四步流程的 bash 驱动脚本：映射到 Quake、注册到运行时、改写内核入口、降低到 QIR 并链接）。核心设计结论：后端在链接期切换，源码不指定具体模拟器；量子语义（如不可拷贝）由类型系统在编译期强制守护。
