本讲承接 C++ 内核（u1-l4），改用 Python 前端实现首个量子内核，覆盖三个最小模块。核心机制：`@cudaq.kernel` 装饰器把普通 Python 函数变成内核，本质是 `PyKernelDecorator` 捕获函数源码、解析 AST，首次调用时惰性编译为 Quake MLIR。关键认知：Python 内核函数体从不被解释器执行，`h`、`x`、`mz` 等是 AST 桥按名字识别的内建门，而非普通函数。比特分配用 `cudaq.qubit()` / `cudaq.qvector()` 类型桩，主机域实例化会抛 `KernelTypeError`，仅在内核 AST 中有意义，翻译为 `quake.alloca`；受控门用 `.ctrl` 修饰符（如 `x.ctrl(control, target)`）。`cudaq.sample(kernel, shots_count=1000)` 返回 `SampleResult`，内核必须含测量，否则触发 0-shot 警告。核心结论：Python 与 C++ 双前端共享同一套 C++ 运行时，等价内核的采样结果统计一致，这是 CUDA-Q 双前端设计的核心收益。
