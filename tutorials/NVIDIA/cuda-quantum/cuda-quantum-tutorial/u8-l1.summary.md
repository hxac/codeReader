本讲梳理 CUDA-Q 四类测试的分工与平台跳过机制。cudaq/test 用 lit+FileCheck 做编译器回归，靠 RUN/CHECK 按序匹配 IR 文本，极快；unittests 用 GoogleTest 测运行时库，因定义 CUDAQ_LIBRARY_MODE、不走 AST bridge，可直接调用 cudaq::sample，经 gtest_discover_tests 注册为独立 ctest 条目；targettests 走 nvq++ 完整链路做端到端，用 XFAIL/REQUIRES/UNSUPPORTED 控制环境；python/tests 用 pytest 测前端。选位口诀：编译器输出配 FileCheck、运行时库配 gtest、端到端配 targettests、Python 配 pytest。

关键认知是 skip 与 xfail 的语义差异：前者不执行，后者执行并预期失败。ARM64 上 C++ 异常穿过 LLVM JIT 帧会 std::terminate 杀死 pytest worker（上游 llvm-project#49036），使 xfail 失效，只能用 skip 规避。2026 年 7 月 PR #4790 新增 skip_arm64_jit 标记，用 platform.machine() in ('arm64','aarch64') 同时覆盖 macOS 与 Linux aarch64，把 test_adjoint_bug 从仅限 macOS 的 skip_macos_arm64_jit 迁移过来，修复了 Linux aarch64 CI 的 worker 崩溃；其余用例维持原标记不动。
