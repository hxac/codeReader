本讲打开 CuTe 算法层的「黑盒」，回答一条 Tensor Core 指令如何被封装复用。核心是 Operation → Traits → Atom 三段式：Operation 描述裸 PTX 与寄存器数组，Traits 补充逻辑形状 Shape_MNK 与「线程-值→坐标」布局，Atom 合体后提供面向 Tensor 的 `.call()`。关键术语包括 MMA（矩阵乘加）、fragment（每线程持有的寄存器片段）。`MMA_Atom` 封装单条指令，只吃 rank-1 寄存器张量，经 `mma_unpack` 拆包触发 PTX；`TiledMMA` 用 `make_tiled_mma` 沿 M/N/K 铺开 atom，产出线程布局 `ThrLayoutVMNK`。`Copy_Atom`/`TiledCopy` 设计完全对称，多出 Src/Dst/Ref 三套布局与 ValType，`make_tiled_copy_A/B/C` 让拷贝划分与 MMA 对齐。最后 partition 通过 `get_slice` + `partition_A/B/C` 把张量切到每个线程，底层范式是 `zipped_divide + compose`。arch 关联靠按架构 `#include` 各 `*_traits_smXX.hpp` 实现。
