本讲以官方示例 example 49 为蓝本，串联起一个 Hopper warp-specialized GEMM 从声明到启动的全过程。核心组装链路是四步 typedef：epilogue `CollectiveBuilder` → mainloop `CollectiveBuilder` → `kernel::GemmUniversal` → `device::GemmUniversalAdapter`；builder 吃少量高层参数（类型、布局、TileShape、Cluster、Schedule）自动推断 TiledMma、TMA copy atom 与共享内存布局，`Auto` 标签让它连 schedule/stage 也替你选，且主循环与 epilogue 必须同时用 `Auto`。先用 epilogue builder、再用其 `SharedStorage` 大小喂给 `StageCountAutoCarveout` 才能算准主循环缓冲级数。Hopper basic 内核采用「1 个 producer warp group（实际仅 1 个 DMA warp 发 TMA，兼搬主循环 A/B 与 epilogue C）+ 1 个 consumer warp group（发 wgmma 并 store）」的角色分流，靠 `PipelineTmaAsync<Stages>` 多级缓冲实现搬算重叠；FP16 默认走 SS 主循环（A、B 都以 shared memory descriptor 喂 wgmma）。Epilogue 有两条路：预定义 `LinearCombination`（命名 α/β 字段）与自定义 EVT（计算图、嵌套匿名 tuple）。主机端三步 API（`can_implement → initialize → run`）将 `Arguments` 翻成含 TMA descriptor 的 `Params`，并通过 cluster launch API（cluster 形状取自 `DispatchPolicy::ClusterShape`）发射内核。硬性要求是 `CUTLASS_NVCC_ARCHS=90a`。后续讲义将承接 `PipelineTmaAsync`（u3-l1）、TMA descriptor（u3-l2）与 EVT 访客树（u2-l10）。
