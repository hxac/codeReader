本讲把读者从「了解 CUTLASS」推进到「能编译运行」。核心结论有四点：一是 CUTLASS 是 header-only 库，在 CMake 中被声明为 `INTERFACE` 目标，能力全在 `include/` 头文件模板里，编译发生在使用者实例化内核时。二是版本门槛：CMake ≥ 3.19、CUDA ≥ 11.4（推荐 12.8）、GCC ≥ 7.3，并全局强制 C++17/CUDA 17 与默认 Release。三是关键变量 `CUTLASS_NVCC_ARCHS` 决定为哪些 SM 架构生成代码，受支持架构随 CUDA 版本动态计算；带 `a` 后缀（如 `90a`/`100a`）才允许使用 Tensor Core 等加速指令，填错会运行时失败。四是构建开关：`CUTLASS_ENABLE_TESTS`/`CUTLASS_ENABLE_HEADERS_ONLY` 等控制子项目参与，作为子项目引入时单元测试默认关闭。标准流程为 `cmake .. -DCUTLASS_NVCC_ARCHS=<sm> && make <target> -j`，其中 `test_unit` 为环境体检、最小示例 `00_basic_gemm`（用 `cutlass::gemm::device::Gemm` 跑一次自校验 SGEMM，成功打印 `Passed.`）为冒烟测试。关键术语供后续讲义承接：header-only、`INTERFACE` 库、`CUTLASS_NVCC_ARCHS`、架构加速特性后缀、`cutlass::gemm::device::Gemm`。
