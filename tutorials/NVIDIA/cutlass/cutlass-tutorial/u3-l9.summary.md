本讲是专家层「CuTe DSL」系列首讲，将 C++ CuTe 两大基石 `Layout=(Shape,Stride)` 与 `Tensor=(Engine,Layout)` 迁移到 Python。核心结论：CuTe DSL 是 CUTLASS 4.x 的 Python 前端，采用「Python 前端 + MLIR 后端」运行模型，`@cute.kernel` 装饰设备端内核、`@cute.jit` 装饰主机端入口，首次调用时惰性构造 CuTe MLIR 再下沉为 PTX/CUBIN。布局用 `cute.make_layout(shape, stride=...)` 构造、`cute.crd2idx` 算下标，整套代数（`composition`/`*_divide`/`*_product`/`local_tile` 等）与 C++ 同名同义；张量沿用 `T=E∘L`、`T(c)=*(E+L(c))`，由 `cute.make_tensor(iterator, layout)` 组合，Engine 可为指针、坐标元组或 SmemDesc，`from_dlpack` 让 torch/numpy 直接接入。DSL 特有「静态 vs 动态」表达：字面量编进 IR 类型，`cutlass.Int32(x)` 记为 `SymInt`、打印为 `?`，`print` 看静态类型、`cute.printf` 看运行时值。验证了 `(4,4):(1,4)` 映射为 `i+4j`（列主序），与 C++ CuTe 逐位一致，为下一讲 u3-l10「CuTe DSL GEMM 实践」铺路。
