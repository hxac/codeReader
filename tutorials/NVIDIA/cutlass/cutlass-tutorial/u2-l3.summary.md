本讲让 CuTe 张量真正「动起来」，建立其标准算法库。四个最小模块：① **`cute::copy` 家族**（`copy`/`copy_if`/`copy_aligned`）是统一数据搬运入口，靠重载分发自动挑硬件指令——无策略时走 `AutoVectorizingCopy`（按对齐与公共向量 recast 向量化），`AutoCopyAsync` 在 SM80+ 对 gmem→smem 自动选 `cp.async`，SM90 还可走 TMA bulk copy；`copy_if` 用谓词掩码处理边界 tile。② **`cute::gemm`** 采用 `D = A*B + C` 三/四元组模型，不传 MMA 时默认用软件 `UniversalFMA`（host 可跑），并按各张量 **rank + 内存空间** 分发到 5 种 canonical 形态（`V`/`M,N`/`M,K`/`V,M`/`V,M,K`），其中 dispatch [4] 用蛇形遍历最大化寄存器复用，smem→rmem 变体内部自带「先 copy 再 gemm」小流水线。③ **`fill`/`clear`** 用 `prefer<1>/prefer<0>` 优先匹配更优实现、否则逐元素兜底。④ 核心结论：**算法与张量彻底解耦**——同一份源码因 Engine 内存空间（gmem/smem/rmem）与静态 Layout 不同而在编译期自动选指令，零运行时开销，这正是把 Layout/Engine 做成编译期类型的回报，也为下一讲 `MMA_Atom`/`Copy_Atom` 与 TiledMma 埋下伏笔。
