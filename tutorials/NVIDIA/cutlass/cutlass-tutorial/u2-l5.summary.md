本讲打开 u2-l4 留下的 Operation 黑盒，专讲 CUTLASS 的 arch 层。核心结论：arch 层离硬件最近，唯一职责是把一条 Tensor Core MMA 的 PTX 指令包装成 C++ 可调用对象，不管分块调度。CUTLASS 有两套并行体系——cutlass/arch（2.x，模板全特化+operator()，服务经典分层 GEMM）与 cute/arch（3.x，struct+寄存器数组别名+static fma，即 MMA_Atom 真正消费的 MMA_Operation），二者封装同一条 PTX，新架构（Hopper/Blackwell）只在后者实现。关键术语：fragment（每线程持有的寄存器段）；三代指令演进 mma.sync（warp、操作数在寄存器）→ wgmma.mma_async（warp group 128 线程、操作数可用共享内存描述符）→ tcgen05.mma/UMMA（累加器落专用张量内存 TMEM）。衔接层上，MMA_Traits 是指令的元信息说明书，提供 Shape_MNK、ThrID 与 (tid,vid)→坐标 的线程-值布局；mma_unpack 按 traits 把 rank-1 寄存器张量 recast、explode 后调 Operation::fma 发出 PTX。完整链路为 Operation→Traits→Atom→make_tiled_mma→TiledMMA，arch 经 mma_atom.hpp 末尾 include mma_traits_smXX.hpp 按架构挂载，为 u2-l6/7/8、u3-l2、u3-l7 铺垫。
