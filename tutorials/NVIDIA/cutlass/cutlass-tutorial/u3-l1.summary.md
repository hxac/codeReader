本讲深入 Hopper（SM90）内核最核心的并发机制——异步流水线（async pipeline）与 warp specialization（线程束特化），目的是让 TMA 数据搬运与 wgmma 计算在时间上重叠，消除串行主循环中的搬算空转。核心模块有三：一是 `PipelineState<Stages>` 用 `(index, phase, count)` 三元组索引环形缓冲，每绕一圈用 `phase ^= 1` 翻转相位（正确性命脉），Producer 以反相起步（缓冲初始为空）；二是 `PipelineTmaAsync<Stages>` 的双屏障模型，每块缓冲配满门 `ClusterTransactionBarrier`（按字节、由 TMA 硬件翻转）与空门 `ClusterBarrier`（按到达数、由 Consumer 软件翻转），四个原语 `producer_acquire`（等空门）、`producer_get_barrier`+TMA（翻满门）、`consumer_wait`（等满门）、`consumer_release`（翻空门）两两配对；三是 warp specialization 在内核入口按 `warp_group_idx` 分派，warp group 0 为 Producer（仅 warp 0 发 TMA）、warp group 1 为 Consumer（发 wgmma），共 256 线程。关键结论：Producer 唯一等待点为 `producer_acquire`，Consumer 另有 `consumer_wait` 与 `warpgroup_wait<K_PIPE_MMAS>` 两处，释放游标 `smem_pipe_release` 滞后读取游标一步，确保 wgmma 真正读走数据后才把缓冲还给 Producer；「try+finalize」两段式等待让等待与计算交错。本讲把 TMA 视为下单后硬件自动 complete_transaction 的黑盒，其描述符构造细节留待后续讲义承接。
