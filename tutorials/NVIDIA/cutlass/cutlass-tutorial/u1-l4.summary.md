本讲讲清 CUTLASS 如何用 C++ 类型表达多种数值精度。核心是四类数值类型：16 位半精度 `half_t`（1+5+10，精度高、易溢出）与 `bfloat16_t`（1+8+7，动态范围大、与 float 互转快）；占用 32 位存储的 `tfloat32_t`（19 位有效）；8 位 FP8（`float_e4m3_t` 精度优先、`float_e5m2_t` 范围优先）；以及不足整字节的「子字节类型」（如 `float_e2m1_t`、`int4b_t`）。这些类型都用 `alignas` 加整数存储实现为 host/device 通用，并遵循「硬件优先、软件兜底」。引入两个关键元函数：以位为单位的 `sizeof_bits<T>`（子字节类型靠偏特化给值）及 `is_subbyte<T> ≡ sizeof_bits<T>::value < 8`。容器 `cutlass::Array<T, N>` 按第三参数 `RegisterSized` 默认值分派两条偏特化分支：元素位宽 ≥32 走 array.h 的连续存储分支，否则走 array_subbyte.h 的位打包分支（`Array<half_t,4>` 走此分支、占 8 字节）。跨类型转换由 `NumericConverter`（标量）与 `NumericArrayConverter`（数组）统一处理，舍入方式由 `FloatRoundStyle` 指定，常见组合有硬件向量化特化。统一入口为 `numeric_types.h`。
