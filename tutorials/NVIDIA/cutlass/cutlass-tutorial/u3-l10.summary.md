本讲是 CuTe DSL 系列第二讲，把上一讲迁移到 Python 的 Layout/Tensor 抽象落成一篇完整 GEMM，蓝本为 `dense_gemm.py`。覆盖四个最小模块：一是 atom 与 TiledMma/TiledCopy，DSL 的 `MmaAtom`/`TiledMma`/`ThrMma` 与 C++ 同名同义，差别仅在类型活在 MLIR IR 里，靠 `atom_layout_mnk` 沿 M/N/K 铺开线程；二是 mainloop，用 `cute.copy(tma_atom)` 发 TMA、`cute.gemm(tiled_mma)` 发 wgmma，借 `PipelineTmaAsync` 三条游标实现 producer-consumer 搬算重叠，释放游标刻意滞后读取游标一步；三是 launch 与 shared memory，`@cute.jit` 主机入口装配 tile/TMA/SharedStorage 并 `.launch(grid,block,cluster,stream)`，`@cute.kernel` 跑设备主循环，stage 数由 smem 容量反推；四是 autotune，`autotune_jit` 与 `tune` 两条路均建立在「多次编译+benchmark+缓存」之上，候选须为 `Constexpr`。核心结论：DSL 算法调用只生成 IR op、不在 Python 里计算，由后端按张量 rank 与内存空间在编译期下沉为 `wgmma`/`cp.async.bulk.tensor`。
