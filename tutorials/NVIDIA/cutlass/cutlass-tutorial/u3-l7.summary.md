本讲承接 Hopper（SM90）warp-specialized GEMM，转向 Blackwell（SM100），核心围绕第五代 Tensor Core 指令 **UMMA（`tcgen05.mma`）** 与新存储 **TMEM（Tensor Memory，每 SM 128×512×32 位=256KB）** 展开。关键结论是：SM90 把累加器放在 consumer warp 的寄存器中，MMA 与 epilogue 串行于同一组 warp；SM100 把累加器搬到 TMEM，由独立 warp 读写，从而把 MMA 与 epilogue **解耦并发**。UMMA 分 SS、TS、2x1SM 三种风味（命名按「A 来源 + B 来源」），累加器一律落 TMEM，由 `ScaleOut::Zero`/`One` 控制首拍清零或累加。SM100 collective（`sm100_mma_warpspecialized.hpp`）采用**两条流水线**：`MainloopPipeline` 搬 A/B、`AccumulatorPipeline` 双缓冲 TMEM 累加器（ACC_PIPE=2）；内核将 warp 划为 **MMA / Sched / MainloopLoad / EpilogueLoad / Epilogue** 五类角色。**Cluster（簇）** 提供分布式共享内存（DSM）与 TMA multicast，**CLC（Cluster Launch Control）** 是硬件辅助的动态 tile 调度器（对应 `Sched` warp），取代 SM90 软件游标领活。簇形状为偶数时可启用 **2x1SM UMMA**（`cta_group::2`），配 `Allocator2Sm` 与 `SM100_TMA_2SM_LOAD` 实现跨 SM 分布式 MMA。blockscaled/sparse/mixed/array 等变体均复用此「UMMA+TMEM+簇+CLC」底座。从 Hopper 迁移仅需改 `ArchTag` 与 tile/cluster 形状，TMEM 分配与五类 warp 接线由 CollectiveBuilder 自动完成。
