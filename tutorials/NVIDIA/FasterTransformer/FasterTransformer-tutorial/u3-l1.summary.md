本讲精读 FT `kernels/` 目录下三个「层内最小算子」：layernorm、activation、add_residual，是进入 layer 之前的最后关卡。先确立贯穿全库的 `invokeXxx` 约定：每个算子由「`__global__` 设备 kernel + host 启动函数」两层组成，layer 只调 `invokeXxx`、从不直接写 `<<<>>>`；启动异步、立刻返回。LayerNorm 采用「一行一 block」模型，靠 `warpReduceSum`（warp shuffle 蝴蝶归约）与 `blockReduceSum`（warp 归约+共享内存+首 warp 再归约，仅 thread 0 结果有效）把求均值/方差从 O(n) 压到 O(log n) 步，shuffle 走寄存器不占带宽。激活 kernel 用「模板的模板参数」做编译期策略模式，`GeluActivation`/`SiluActivation`/`IdentityActivation` 等提供 `apply`，换激活零运行期开销，并原地读写、用 nullptr 运行期开关覆盖 bias/gate/INT8 多种语义。add_bias_residual 把「加偏置+残差+可选 INT8 反量化」融进单个 elementwise kernel，以 `RESIDUAL_NUM`（1/2，对应标准与 GPT-J 并行残差）和 `T2`（int32）两个模板参数适配多场景，显存读写近乎减半。三者共同体现 FT 的融合与模板特化哲学；数据类型分派（FP32 标量 / FP16 走 half2 / INT8 带 scale）套路承接 u1/u2，并作为 u3-l2 注意力 kernel 的直接前置。
