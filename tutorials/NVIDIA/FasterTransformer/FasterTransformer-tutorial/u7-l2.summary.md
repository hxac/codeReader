本讲承接 u7-l1 张量并行，讲解 FT 推理在「层数过多、单卡放不下」时如何把不同层纵向分配到不同 GPU，并把推理组织成跨多机的多进程。

核心分工是「MPI 当组织者、NCCL 当搬运工」：`mpi_utils` 仅提供 `initialize`/`getCommWorldRank`/`barrier`/`bcast` 等薄封装，受 `BUILD_MULTI_GPU` 宏守卫，宏关闭时退化为空壳返回 0，实现单卡零成本降级；它只负责启动进程、给全局编号 rank、做少量标量广播，真正的张量传输全靠 NCCL 走 NVLink/Infiniband。

硬约束 `world_size = tensor_para_size × pipeline_para_size` 在 `init_multiprocessing` 校验，同时要求 `decoder_layers % PP == 0`、`head_num % TP == 0`。每个进程按 `cudaSetDevice(rank % device_count)` 绑定本机 GPU，使多节点下每节点从 GPU 0 重新编号。

最关键机制是 `ftNcclInitialize` 中的二维笛卡尔拓扑：用 `MPI_Cart_create(dims={PP,TP})` 配合 `MPI_Cart_sub` 把一维全局 rank 折成 `(tp_rank, pp_rank)`——行构成 TP 组切 head、列构成 PP 组切层，对应公式 `global_rank = pp_rank·TP + tp_rank`。TP、PP 各建一个独立 NCCL communicator（`ncclUniqueId` 由 `MPI_Bcast` 引导散播后 `ncclCommInitRank` 握手），通信范围严格隔离。

PP 按 `layers_per_group = decoder_layers/PP` 把层均分成连续段，每个阶段只为自己那段层分配 KV cache 与加载权重。部署原则为 TP 宜走节点内 NVLink、PP 可跨节点走 IB，OpenMPI 默认 by-slot 让连续 rank 先填满单节点，使 TP 组自然落节点内、PP 组自然跨节点。后续讲义 u7-l3 将介绍用 CUDA 直连替代 NCCL 的自定义 all-reduce kernel。
