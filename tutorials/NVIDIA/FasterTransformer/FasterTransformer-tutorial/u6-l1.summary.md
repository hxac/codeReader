本讲解析 FasterTransformer（FT）的 `ParallelGpt` 架构。核心结论是：FT 将 GPT 一次生成拆为 **context 阶段**（一次处理整段 prompt、写满初始 KV cache）与 **decoder 阶段**（逐 token 自回归、向 cache 追加），根本动机在于两阶段 query 序列长度不同导致最优 kernel 选择不同——context 的 query 长 = `max_input_length`，走 cuBLAS 批量 GEMM（`TensorParallelGptContextAttentionLayer`）；decoder 的 query 恒为 1，走融合 masked MHA kernel（`TensorParallelDecoderSelfAttentionLayer`）。`ParallelGpt` 作为编排者持有 `gpt_context_decoder_`、`gpt_decoder_`、`dynamic_decode_layer_` 三子组件，其构造参数（`beam_width`、`tensor_para`、`int8_mode`、`attention_type`、`shared_contexts_ratio` 等）与派生量 `vocab_size_padded_`（TP 下按 8 对齐）决定运行期行为。`forward` 主循环 `for (step_ = step_start; step_ < gen_len)` 每步做 embedding lookup → 单步 decoder → logits GEMM（TP>1 时 all-gather）→ dynamic decode → early stop。KV cache（形状 `[num_layer/PP, batch*beam, local_head_num, ..., memory_len, ...]`）是两阶段交接缓冲：context 写满 `0..max_input_length-1`，decoder 从 `step_start` 起追加。关键术语：context、session_len/memory_len、step_start、gen_len、last_token_hidden_units、remove padding。ContextDecoder 比 Decoder 多 remove padding 与 last token 提取两步，二者因输入张量形状与自注意力实现不同而不可互换。后续 u6-l2 将深入 cache 的 beam 重排，u6-l3 展开交互式续写。
