本讲是 FT 源码的「导航地图」，阐明 `src/fastertransformer/` 的目录组织与命名规范。核心库分为「核心引擎」与「框架外壳」两大块：引擎包括 `kernels`、`layers`、`models`、`utils`、`cutlass_extensions`（与框架无关的纯 C++/CUDA）；外壳 `th_op`（PyTorch）、`tf_op`（TensorFlow）、`triton_backend`、`tensorrt_plugin` 仅做框架张量与内部 `Tensor` 的零拷贝转换，不参与计算。全库骨架是 kernel/layer/model 三层抽象：kernel 是 GPU 上单件事的并行函数（以 `invoke` 起头），layer 把 kernel 与 GEMM 组合成 attention、FFN 等语义模块并自管临时显存，model 串联各层完成前向，严格自底向上、下层不感知上层。`utils` 提供贯穿全库的 `Tensor`/`TensorMap`、显存分配与 cuBLAS 封装，`Tensor.where` 标记数据在 CPU 还是 GPU。命名约定：单类文件用大驼峰、工具文件用小写下划线、函数小驼峰、变量小写下划线。新增模型须复用现有组件新建类，绝不改老模型。外壳目录按模型名再分子目录，覆盖范围 th_op 最全、tensorrt_plugin 最少，对应支持矩阵差异。
