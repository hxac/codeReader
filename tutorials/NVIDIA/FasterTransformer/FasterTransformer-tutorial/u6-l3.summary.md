本讲围绕 KV cache 的跨请求/跨轮次复用展开三个正交模块。交互式生成用输入张量 `continue_gen` 与 `session_len` 实现续写：续写轮把起始步 `initial_step=step_`，只处理新增 token 并复用首轮按整段会话预分配的 cache 缓冲。共享上下文用 `invokeFindContextDups` 建立 `shared_contexts`/`compact_to_batch`/`batch_to_compact` 三张索引表，把 batch 中完全相同的 prompt 去重为 `compact_size` 行只算一次 context 阶段，再由 `invokeCompactInputs` 紧凑、`invokeUnCompactOutputs`/`invokeUnCompactCaches` 散开，由 `shared_contexts_ratio` 按比例门槛决定启用；核心 kernel 用三角对枚举配合 `atomicMin` 与 CUB 前缀扫描建表。流式生成在 `GptStreamer` 中以 `std::async` 跑 forward、主线程轮询 `sequence_length` 取新 token 并触发 `streamHook`/`stopCriteria`，仅流水并行最后 rank 异步且强制 `beam_width=1`。三者可叠加，但流式与 beam search 互斥；这些能力统一在 `ParallelGpt::forward` 的输入张量集合暴露，是 FT 从一次性 batch 推理走向真实服务的关键一步。
