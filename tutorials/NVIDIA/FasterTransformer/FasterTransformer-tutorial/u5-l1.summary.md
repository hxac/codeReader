本讲精读 FT 的 `Decoder` 类——自回归解码的「单步」最小计算单元：`forward` 一次只吃一个 token 的隐状态 `[batch, hidden]`（无序列维），靠 CPU 标量 `step` 定位 cache 写入槽位，外层循环反复调用它即生成序列。

一个 Decoder block 比 BERT 多了 **cross-attention**，结构为 `pre-LN → masked self-attn → cross-attn → FFN`，采用 Pre-LN，每个子层后跟一次残差共三次。三类子层全部复用 BERT 件：FFN 用同源 `FfnLayer`（默认 `ReluFfnLayer`）、self-attn 用 u3-l2 融合 `masked_multihead_attention` kernel、layernorm/残差用通用 kernel——Decoder 自身没有 `.cu` 文件。

关键概念：**encoder memory**（源句浓缩向量）、**K/V cache** 与 **memory cache**——前者每步追加新 token、随步增长，后者仅在首步由 encoder memory 计算并写入、之后只读。self 的 QKV 因输入相同可合并成一次 GEMM，cross 的 Q/K/V 来源不同必须分开。还区分了 Decoder（单层块）/ Decoding（端到端生成流程）/ GPT（无 cross-attn）三个易混名字。`AddBias+残差+下一步 pre-LN` 三合一融合 kernel 是核心省访存优化，所有层 cache 共用一块连续大 buffer 按层偏移切分。
