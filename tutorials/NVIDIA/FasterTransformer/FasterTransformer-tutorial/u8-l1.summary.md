DynamicDecodeLayer 是 FasterTransformer（FT）解码流程的统一外观（Facade）：对外只暴露一个 forward(TensorMap*, TensorMap*)，对内聚合 online beam search、（已废弃）朴素 beam search、Top-K、Top-P 四个后端，均于 initialize() 一次构造，按 beam_width（取自 logits 第二维、全批固定）分发——大于 1 走 online beam search，等于 1 走 Top-K 与 Top-P 串行。本讲引入关键概念 runtime_arg_names_：六组「运行期可变、但 beam search 无批量 kernel、同批须一致」的参数（temperature、len_penalty 等）；hasDiffRuntimeArgs 在 CPU 上逐元素探测同批不一致，结果驱动 beam search 是否退化为逐请求。setup 与 forward 必须配对：setup 将运行期参数拷至 device buffer（仅 sampling 需要），forward 做实际解码，二者借 has_diff_runtime_args_ 隐式通信。核心结论：同一次 forward 内 beam search 与 sampling 不可混用；能按请求混用的只有 Top-K 与 Top-P，靠 top_k==0 / top_p==0 触发 skip_decode 串行接力，并借 runtime_logits_buf_ 避免污染原始 logits。构造期参数均填 deprecated 占位值，真正生效的值由每次 setup 注入，体现 FT 从构造期向运行期配置的演进。
