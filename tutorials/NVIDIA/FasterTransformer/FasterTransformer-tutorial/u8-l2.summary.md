本讲解析 FasterTransformer 的 beam search 解码后端。它采用模板方法：`BaseBeamSearchLayer::forward` 固化「penalty → `invokeSoftMax` → 间接表更新」三步骨架，子类 `BeamSearchLayer` 与 `OnlineBeamSearchLayer` 仅重写纯虚函数 `invokeSoftMax`。penalty 阶段由 `invokeAddBiasApplyPenalties` 按条件对 logits 做 bias/temperature、repetition/presence、min_length 三类修正，默认值相等则跳过。朴素版显式物化 `[batch*beam, vocab]` 的 log 概率缓冲，累加进 `cum_log_probs` 后做两阶段 topk：每条 beam 取 top-k 再跨 beam 收缩回 `beam_width` 条（`beam_width=4` 即 4×4 候选缩为 4），并用 length penalty 归一化分数配合 `BeamHypotheses` 收集已完成路径。在线版用 `invokeTopkSoftMax` 融合 softmax+topk+收集，工作区不依赖 vocab_size，是大词表大 batch 的默认后端。多 beam 时经 `update_indir_cache_kernelLauncher` 写入 `tgt_cache_indirection`，衔接 KV cache 重排。
