本讲精读 FT 中性能收益最大的「融合多头注意力」kernel `decoder_masked_multihead_attention`。核心结论：「融合」即把 `QK^T→softmax→PV` 压进单个 CUDA kernel，让分数矩阵、logits、部分和始终留在共享内存与寄存器，朴素实现里 `[B,H,L,L]` 分数矩阵需多次显存往返、融合后归零，是访存密集型注意力大幅加速的根本。文件分三层：`.h` 定义 POD 参数结构 `Multihead_attention_params`（按值传入 kernel，含 KV cache、`inv_sqrt_dh`、`timestep`、`masked_tokens`，用 `CROSS_ATTENTION` 模板参数区分 self/cross）；顶层 `.cu` 按 `Dh`（12 档）与数据类型 switch 分发；子目录 12 个 `_<N>.cu` 按序列长度动态选 block 配置并显式实例化；`template.hpp` 为真正的融合 kernel。causal mask 在 decoder 单 query 场景结构性免费（循环上界 `tlength` 天然不看未来），`masked_tokens` 才显式处理 padding/finished。多精度共用一份模板：`T` 控制存储精度、`kernel_type_t<T>` 推导计算精度（FP8→float），INT8(`int8_mode==2`)/FP8 靠运行期反量化/量化分支接入，体现「低精度存储 + 高精度计算」。并行模型为「一个 block 负责一个 (序列,头)」，内部用 warp shuffle + 两级共享内存归约求 max/sum。本讲承接 u3-l1 的 elementwise kernel，是 u3-l3 注意力层与 u6-l2 KV cache 的衔接基础。
