本讲承接 u6-l1 的 context/decoder 两阶段分裂，彻底讲透贯穿两阶段的交接缓冲——KV cache。核心模块有四：一是 cache 的内存布局。它是按"层×序列×头×每头维度"组织的巨型 GPU 缓冲，K 与 V 尾维不同：K 重排为 [dim/x, time, x] 以便沿时间轴做 16 字节向量化扫描，V 保留 [time, dim] 自然布局；二者共享一块连续显存，常量 x = 16/sizeof(T)，TP 切头、PP 切层。二是 cache 的写入节奏：context 阶段一次性写满整段 prompt，decoder 阶段每步追加一个 token，两阶段通过层偏移与 iteration 偏移定位到同一块层 buffer，step 决定时间槽。三是 beam search 下的间接寻址机制：cache 不做物理拷贝，而是用 cache_indirection 间接表配合 src/tgt 双缓冲完成"逻辑重排"，构建 tgt 表的 kernel 是 update_indir_cache_kernel，真正按表读 cache 的是融合 MHA kernel。四是 gpt_kernels.cu 中的辅助 kernel，其中 invokeUnCompactCaches 是唯一真正搬运 cache 的 kernel，但它服务于共享上下文去重而非 beam search。引入的关键术语包括 KV cache、x 因子、cache_indirection、双缓冲、compact/uncompact，供后续讲义（u6-l3 共享上下文、u3-l2 注意力 kernel）承接。
