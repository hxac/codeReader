本讲承接 u10-l1（th_op），讲解 FT 如何把 BERT/GPT/Decoder 等封装成 TensorFlow 自定义 OP（tf_op），供原生 TF 图调用。核心要点：所有 TF OP 继承抽象基类 `BaseOp<T>`，它统一管理 cuBLAS/cuBLASLt handle 与互斥锁，并提供零拷贝转换方法 `convert_tensor`（把 `tf::Tensor` 的 GPU 指针直接包装成 `ft::Tensor`，不复制数据）与 `get_tensor`。一个 OP 的三件套为 `REGISTER_OP`（声明签名）→ `OpKernel` 子类（重写 `Compute`）→ `REGISTER_KERNEL_BUILDER`（注册 GPU kernel），`TFTraits` 负责 `Eigen::half` 到 FT `half` 的类型映射。`Compute` 流程为取 stream 并 `cublasSetStream` → 建 `Allocator<AllocatorType::TF>`（把临时显存委托给 TF 的 `context->allocate_temp`，全程零 `cudaMalloc`）与 `cublasMMWrapper` → 按下标装权重 → 构模型 → `forward`。接口演进上，Bert/Encoder/Decoder 用旧式 `std::vector<Tensor>`，Gpt/Decoding 用新式 `TensorMap`（按名字索引，便于插入可选运行期采样参数）。编译需二选一开关 `BUILD_TF`（TF1，旧 ABI）或 `BUILD_TF2`（TF2，新 ABI），且必须配 `TF_PATH`，否则 cmake 报错。下接 u10-l3（Triton backend）。
