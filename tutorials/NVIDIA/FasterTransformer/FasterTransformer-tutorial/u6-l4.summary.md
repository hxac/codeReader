本讲对比 FT 中五类 transformer 变体，核心结论是只有两套编排骨架：decoder-only（GPT/GPT-J/GPT-NeoX）走「context + decoder 两阶段」——先整段处理 prompt 写满 KV cache，再逐 token 生成；encoder-decoder（BART/T5）把源序列编码外包给独立 encoder，`*Decoding` 只剩单阶段生成循环（从 step=1 起）。两者生成主循环（embedding→decoder→logits→dynamic_decode→early stop）同构，差异集中在位置编码、是否有 encoder、激活函数、layernorm 与残差结构四维。引入的关键术语：rotary 位置编码有 GPT-J 交错式与 GPT-NeoX rotate-half 两种风格，数学等价但权重排列不同，由 `neox_rotary_style` 区分 kernel 分支；并行残差（单 LayerNorm + 一次性合并 attn/ffn/残差）由 GPT-J 固化、GPT-NeoX 用 `use_gptj_residual_` 开关在并行/串行间切换；encoder-decoder 同时持有 self KV cache 与 memory cache（cross-attention 仅首步计算）是区别于 decoder-only 的判据；T5 较 BART 多 relative position bias、RMSNorm（`invokeGeneralT5LayerNorm`）与 MoE/adapter 扩展。整体体现「同一套基础设施 + 差异点开关」的设计哲学。
