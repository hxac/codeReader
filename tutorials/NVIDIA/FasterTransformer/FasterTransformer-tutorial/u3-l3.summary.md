本讲讲解 `layers/attention_layers/` 层抽象。核心结论：FT 注意力层是一棵继承树，统一接口为 `forward(TensorMap*, TensorMap*, const AttentionWeight<T>*)`，走哪条路径由 `AttentionType` 枚举（FUSED/UNFUSED × PADDED/UNPADDED 四值）与 `getAttentionType()` 在运行期按 `(sm, size_per_head, remove_padding, causal_mask)` 自动决策。UnfusedAttentionLayer 用 cuBLAS GEMM + `invokeMaskedSoftmax` 显式展开注意力，通用、全数据类型，但需物化 `[B,H,L,L]` 分数矩阵 `qk_buf_`；FusedAttentionLayer 把 QK^T→softmax→PV 交给 TensorRT `MHARunner` 融合 kernel，不物化分数矩阵、更快但仅支持 FP16 且 `size_per_head∈{32,64}`。GptContextAttentionLayer 处理整段 prompt 并写 KV cache，DecoderSelfAttentionLayer 逐 token 解码（单 query、1 次合并 QKV GEMM + 复用 `masked_multihead_attention` kernel），两者不可互换。TensorParallel 注意力层 = 继承单卡层 + 构造时传入 `head_num/world_size` 列切分 + 末尾一次 `ftNcclAllReduceSum`；QKV 投影列并行无通信，输出投影行并行需 all-reduce，全层唯一通信点在输出投影之后，`do_all_reduce_` 开关可延迟通信。
