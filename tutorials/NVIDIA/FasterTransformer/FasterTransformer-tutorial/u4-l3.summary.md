本讲把 FasterTransformer 的编码器从 NLP 推进到视觉，覆盖 ViT 与 Swin 两个模型。ViT 的核心是用 stride=kernel=P 的卷积做 **patch embedding**，把图像切成 token 序列（序列长度 N 在构造期按 (img/P)²+cls 算定），随后原样复用 BERT 的 transformer block（Fused/Unfused 注意力与 GeluFFN），唯一新增依赖是 cuDNN 卷积。Swin 则通过**窗口注意力**与**移窗（shifted window，shift_size 在 stage 内随 block 奇偶于 0 与 window_size/2 间交替）**把注意力压到对图像尺寸近似线性，并采用**层级式 patch merge**（每级空间分辨率减半、通道加倍）输出多尺度特征；调用链为 SwinTransformer→BasicLayer→Block 三层。Swin 相比 ViT 多两个专属 kernel：`image_shift_partition`（循环移位+窗口重排，单遍融合）与 `image_merge`（patch merge 的 2×2 空间→通道重排）。GEMM 调优上，vit_gemm 因序列与 BERT 同构而复用 `generate_encoder_gemm_config`，swin_gemm 因 seq_len 退化为窗口大小、batch 被窗口数放大、需枚举 4 级形状而拥有独立的 `generate_swin_gemm_config`。结论：分类且分辨率不高选 ViT，高分辨率密集预测（检测/分割）选 Swin。
