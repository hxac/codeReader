本讲讲解 FasterTransformer 中 FFN（前馈网络）层的实现，与 u3-l3 注意力层共同构成一个 transformer block。FFN 落地为「两段 GEMM + 一段激活」：GEMM1 将 hidden 升维到 inter_size、GEMM2 降回 hidden，第二段偏置 b2 融合进下游残差 kernel 而非显式相加。激活差异用模板方法模式解决：基类 FfnLayer 写完整 forward，子类 Gelu/Relu/SiluFfnLayer 仅重写虚函数 getActivationType()，由 genericActivation 按类型分发到对应 kernel，门控变体 GeGLU/ReGLU/SiGLU 共用同一分支。张量并行 FFN 采用「GEMM1 列切分 + GEMM2 行切分 + 末尾一次 all-reduce」：构造时 inter_size /= world_size，整层仅在末尾做一次原地 ftNcclAllReduceSum。与单卡等价的数学根基是「激活必须逐元素」，故行并行的部分和求和恰好还原完整结果。TensorParallelGelu/Silu/ReluFfnLayer 三个变体逐行同构，说明并行逻辑与激活逻辑正交解耦；do_all_reduce_、enable_custom_all_reduce_ 两开关分别支持延迟通信与 NCCL/自定义 kernel 二选一。一个 transformer block 在张量并行下恰好两次 all-reduce（注意力层与 FFN 层各一次）。
