本讲讲清 FasterTransformer 的 PyTorch 集成外壳层 th_op：职责单一，只做 torch 张量与 ft::Tensor 之间的零拷贝翻译，不做任何 GPU 计算。每个模型 Op 采用三段式封装——对外类 XxxOp（继承 CustomClassHolder）、抽象接口 IFXxx、模板实现 FTXxx<T>，构造时用 switch(scalar_type) 把运行期 dtype 分发到对应模板实例以实现类型擦除。零拷贝核心是 th_utils 的 get_ptr（取 data_ptr）与 convert_tensor（组装 {where,type,shape,data} 描述符），前提是 tensor 连续，由 CHECK_INPUT 宏把关。Op 借文件末尾的全局静态 torch::jit::class_ 在 .so 加载时自动注册，Python 端经 torch.classes.load_library 实例化调用；无状态工具函数则用 torch::RegisterOperators。forward 内复用 PyTorch 的 CUDA stream、cuBLAS handle 与 caching allocator（AllocatorType::TH），输出张量也由 torch 预分配，整条链路 GPU 数据零拷贝。开启 BUILD_PYT 编译出单一 libth_transformer.so（聚合全部 th_<model> 与引擎），加载即注册所有 Op。
