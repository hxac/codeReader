本讲是量化单元收尾篇，聚焦「权重重排（repack）」与「sidecar」两类贴近位运算的细节，承接 u2-l4 的 `load_weights → apply_all_repacking` 主线。核心分四个模块。第一，int4 检查点布局与重排：AWQ（沿输出轴列打包 int32）、GPTQ（沿输入轴行打包 int32）、ModelOpt 预打包 uint8 三种格式，经 `apply_all_repacking` 固定六趟顺序翻译成 int4 GEMM 插件的 swizzled int8；其中「零点折叠」`adjusted = nibble − qzero + 8` 把非对称约定对齐到 kernel 的 `(nibble − 8)` 对称约定，MoE 专家堆叠必须先于 GPTQ repack。第二，NVFP4（E2M1）浮点 4-bit 采用「per-16 block FP8 scale + per-tensor super-scale」两级缩放：普通线性层仅 `uint8→int8` view，MoE 专家需 decode→encode→6D MMA swizzle 并区分 SwiGLU FC1 的 interleave/concat layout。第三，FP8 embedding sidecar 用 per-128 分块 scale 压成 FP8 E4M3 省一半显存，但 `--fp8-embedding` 仅对 LLM thinker 生效，TTS 的 talker/code_predictor 回退 FP16。第四，EAGLE3 与 DFlash 因非 transformers 原生而各有自包含校准实现：EAGLE3 默认 fp8、借 base 的 embedding，DFlash 默认 nvfp4、借 base 的 lm_head 且强制排除精度敏感的 `fc`。关键术语包括 repacking、sidecar、零点折叠、swizzle、两级缩放、6D MMA、per-row 分块量化，这些构成 Python 导出端与 C++ 运行时的权重契约。
