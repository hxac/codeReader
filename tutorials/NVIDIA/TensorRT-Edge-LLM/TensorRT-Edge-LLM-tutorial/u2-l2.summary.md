本讲讲解 `AutoModel.from_pretrained` 如何把检查点目录变成带权重的 `nn.Module`。核心机制是两张模块级注册表：`_MODEL_REGISTRY`（`model_type → 模型类`）与 `_ATTENTION_SCALE_DEFAULT_REGISTRY`（`model_type → 注意力缩放函数`），在导入时由 `register_model` 填充；未注册的架构回退到默认 `CausalLM`（依据 `config.layer_types` 逐层搭建）。选模型类分两层：变体优先（draft 模型硬编码）、注册表兜底（普通 LLM 查表，查不到回退默认 `CausalLM`）。`_resolve_model_variant` 先做互斥校验，再按优先级裁决唯一变体：EAGLE3 draft 依据检查点自带的 `draft_vocab_size` 自动检测，其余变体（mtp / dflash / gemma4_mtp 的 base 与 draft）均由布尔开关驱动。权重加载阶段用 key remapping（`_eagle3_key_remap` / `_mtp_key_remap` / `_dflash_key_remap`）把检查点张量名翻译为模型期望的参数名。所有上层调用（CLI、Python API、VLM 子模型、各 draft 导出）最终汇聚于同一 `from_pretrained` 入口，角色完全由布尔开关编码。
