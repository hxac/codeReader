本讲承接 u2-l2「建空壳 nn.Module」，讲解把 safetensors 检查点真实权重灌进空壳的完整链路，覆盖三个最小模块：loader（权重加载主线）、repacking（量化权重格式重排）、embedding_quantization（FP8 词嵌入 sidecar）。核心结论：load_weights 不用 PyTorch 的 load_state_dict，而用自写的 _set_tensor 直接写 buffer/parameter，以原样保留 fp8/uint8/bfloat16 等量化与特殊 dtype，并顺手做 bf16→fp16 转换；找不到的 key 只记 debug 日志不报错。主流程为：_build_shard_map 定位分片 → 剥离 VL 多模态前缀（_detect_key_prefix）→ key_remap 重命名 draft 模型 key → _set_tensor 赋值（融合权重 qkv_proj/gate_up_proj 由 _try_split_fused_tensor 兜底拆分）→ pre_repack_hook → apply_all_repacking → tie_weights。引入的关键术语包括：buffer/parameter、shard_map、key remap、tp_split_dim（张量并行在赋值时就地切分）、零点折叠（把各量化方案的零点烤进 nibble）、repack、sidecar。apply_all_repacking 按固定顺序跑六趟扫描，将 AWQ 列打包、GPTQ 行打包、NVFP4、ModelOpt 预打包翻译成 C++ 插件期望的 swizzle 布局；MoE 专家堆叠须在 GPTQ repack 之前。embedding 表不走线性层 repack，而是在导出阶段经 quantize_embedding_to_fp8 量化成 FP8 E4M3（128 列分块 per-block scale），单独写成 embedding.safetensors，由运行时反量化查表；draft 模型复用 base embedding 故跳过。本讲产出的被 repack 过、被 TP 切分过的 buffer 正是下一讲 u2-l5（ONNX 导出）trace 的对象。
