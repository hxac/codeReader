本讲剖析 EdgeLLM「默认 CausalLM」——即 u2-l2 中 AutoModel 注册表查不到时回退的标准 decoder-only 实现。核心三模块：modeling_default.py 手写解码器（组装层次 CausalLM→Transformer→DecoderLayer→Attention/MLP，含 GQA、SwiGLU、RMSNorm，RoPE 在 attention_plugin 内施加），其子模块名刻意与 safetensors 权重 key 一一对齐以支持按名加载；关键取舍是「从检查点权重直接构建 nn.Module，而非 trace HuggingFace 的 FX 图」，确保导出图确定、最小且与 C++ 运行时一一对应。linear.py 的 make_linear 是量化统一入口，先由 module_quant_type 决定每模块精度，再分发到 FP16/FP8/AWQ/GPTQ/INT8-SQ 线性层或 NVFP4 组合式（LinearMethodBase 配列/行并行外壳），前向分「DQ+MatMul 依赖 TRT Q-DQ 融合」与「专用量化 GEMM」两类。ops.py 的自定义算子是 trace 期占位符：导出时不执行函数体、由 register_fake 做形状传播，真值交 C++ 插件；算子名（trt::/trt_edgellm::）即 Python 与 C++ 间的契约，承接 u2-l4 加载与 u2-l5 导出。
