本讲围绕 tensorrt_edgellm 的命令行入口与公共 API，分三个模块。其一，pyproject.toml 的 [project.scripts] 以入口字符串「模块:函数」定义 6 个命令（export、quantize、lora 系列、reduce-vocab），映射到 scripts 下各自无参的 main；命令名用连字符、模块名用下划线，是否成命令只看是否登记在表，与有无 main 无关。其二，以 export 为样本追踪执行链：main 经 argparse 解析 sys.argv，构建「阶段表」stages 遍历各 _export_*，最终统一调用 export_onnx。其三，__init__.py 借 __all__ 暴露 9 个公共 API（如 AutoModel、export_onnx），并在导入时用 register_model 填充模型注册表。核心结论：命令行工具与 Python API 底层同源，最终都落到 export_onnx。
