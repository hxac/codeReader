`tensorrt_edgellm.quantization` 是与 ONNX 导出彻底解耦的独立量化包，唯一产物是「带量化元数据的统一 HuggingFace 检查点」，不产出 ONNX/engine；底层依赖 NVIDIA ModelOpt 做训练后量化（PTQ）与校准——跑前向统计激活 `amax`，按 `scale=amax/maxbound` 算出量化比例。一次量化由 backbone、lm_head、kv_cache、visual、audio、cp 六轴组成；backbone 支持 fp8/int4_awq/nvfp4/mxfp8/int8_sq，可与 lm_head 任意搭配实现混合精度。`build_quant_config()` 把方法名翻译成 ModelOpt 配方（recipe），靠「深拷贝基底→删补 lm_head→禁用非 LLM 组→叠加覆盖」有序合并，依赖 ModelOpt「后写覆盖先写（last-writer-wins）」的 pattern 语义。`quantize_and_export()` 编排 load→校准→量化→写出检查点→拷贝 sidecar 全流程，是 CLI 与 Python API 的总入口。draft 量化分两路：EAGLE3/DFlash 走 `draft` 子命令（由 `dflash_config` 区分，独立校准模型实现），MTP 非独立检查点、在 `llm` 命令内部先于 base 量化。本包产出的方法是导出侧 `QuantConfig`（u2-l1）所读全集的子集（不含 `int4_awq_modelopt`、`int4_gptq`）。
