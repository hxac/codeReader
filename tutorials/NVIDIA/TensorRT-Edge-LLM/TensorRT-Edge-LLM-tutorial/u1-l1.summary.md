TensorRT Edge-LLM 是 NVIDIA 面向边缘设备（Jetson、DRIVE、DGX Spark）的高性能、纯 C++ 的 LLM/VLM 推理运行时，只做推理部署，不做训练。其核心是一条三段式流水线：第一步在 x86 开发机上用 Python（tensorrt_edgellm 包）读取 HuggingFace 检查点、可选量化后导出 ONNX 中间产物；第二步在边缘设备上用 C++ 引擎构建器把 ONNX 编译成优化的 TensorRT engine；第三步用纯 C++ 运行时加载 engine 做推理。C++ 运行时以统一的 LLMInferenceRuntime 类与 handleRequest() 方法为入口，靠可插拔的 DecodingStrategy 层切换普通自回归解码与投机解码（base + draft 双引擎）。关键特性围绕「在边缘上又快又省地跑」展开：4-bit 量化与 FP8 KV 缓存省显存、纯 C++ 无 Python 依赖保证生产就绪，并支持 LoRA、系统提示缓存、多模态与 EAGLE3/MTP/DFlash 投机解码。平台精度各异：Jetson Orin 走 FP16/INT8/INT4，Thor/Spark 等较新平台走 FP8/NVFP4。典型场景为汽车、机器人、工业 IoT 与端侧设备，共同诉求是离线、低延迟、低内存、可进终端产品，与面向云端高并发的 vLLM 形成定位差异。
