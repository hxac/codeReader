本讲建立 TensorRT Edge-LLM 的全局地图，拆为两个最小模块。**仓库目录**遵循「按流水线阶段分目录」原则：Python 导出前端在 `tensorrt_edgellm/`，C++ 构建器在 `cpp/builder/`，运行时在 `cpp/runtime/`，示例入口在 `examples/`，原始 CUDA/CuTe-DSL 算子在 `kernelSrcs/`；最大的 `cpp/` 又细分为 runtime、builder、plugins、kernels、common、tokenizer、sampler、multimodal 等子包。**三段式流水线**主轴为：HuggingFace 检查点 → Python 导出（ONNX）→ C++ 引擎构建（engine）→ C++ 运行时（推理），四类产物文件类型依次是 `.safetensors`、`.onnx`、`.engine`、文本 token。关键结论：量化是导出的前置可选步骤，其产物仍是 HF 风格检查点而非 ONNX；导出在 x86 开发机进行，构建与运行必须在边缘设备完成，因 engine 绑定特定 GPU 型号与 TensorRT 版本。引入的关键术语：检查点、ONNX、engine、`LLMInferenceRuntime`、`DecodingStrategy`、sidecar。权威参考为 `AGENTS.md` 架构小节与 `overview.md` 流水线图。
