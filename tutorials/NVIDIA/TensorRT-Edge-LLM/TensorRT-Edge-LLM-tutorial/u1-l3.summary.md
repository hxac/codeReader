本讲围绕 TensorRT Edge-LLM 如何在本机被构建出来，分三个最小模块展开。**C++ 构建系统**：分层管理，顶层 `CMakeLists.txt` 管全局选项与依赖定位，`cpp/CMakeLists.txt` 产出三个静态库（`edgellmKernels`/`edgellmCore`/`edgellmBuilder`）加一个**共享**插件库 `NvInfer_edgellm_plugin`。关键变量 `TRT_PACKAGE_DIR` 是 `cmake/FindTensorRT.cmake` 消费的 TensorRT 搜索根提示，多版本 x86 环境须显式指定；CUDA 版本必须用 `CUDA_CTK_VERSION`（默认 12.8），传 `CUDA_VERSION` 会被主动拒绝；运行时靠 `LD_LIBRARY_PATH` 定位动态库。插件库须共享，因 TensorRT 运行期动态加载插件符号。**Python 包依赖**分三档：基础（导出 ONNX 必需，7 个锁定包）、`tools`（量化/LoRA/词表/音频，含 `nvidia-modelopt`）、`server`（FastAPI/Uvicorn/pybind11），核心包版本被严格 pin；并注册了六个 `tensorrt-edgellm-*` 命令入口。**Git 子模块**：`3rdParty/` 下 nlohmann/json（常规构建即引用）、googletest（仅单测）、NVTX（仅 profiling），`git clone` 后必须 `git submodule update --init`。
