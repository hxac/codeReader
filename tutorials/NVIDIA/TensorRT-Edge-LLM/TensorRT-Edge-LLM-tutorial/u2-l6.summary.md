本讲是导出流水线的「编排层」：`scripts/export.py` 的 `main()` 充当项目经理，不亲自做 dynamo 导出，而用一张「阶段表 stages」`(enabled, component, callable)` 驱动所有组件导出，主循环仅 `for ... if enabled: fn(out_dir)`，新增组件只需加一行。编排主键是检查点的 `model_type`，由五组分类常量（`_VLM_MODEL_TYPES`/`_AUDIO_MODEL_TYPES`/`_CODE2WAV_MODEL_TYPES`/`_ACTION_MODEL_TYPES` 与 dict 形态的 `_LLM_COMPONENTS`，默认 `{"thinker"}`）决定带哪些组件（thinker/talker/code_predictor/visual/audio/code2wav/action）；`--components` 白名单与 `--skip-*` 可进一步收窄，权重与 ModelConfig 惰性加载。输出子目录由 `_DEFAULT_LAYOUT` 与 `_LAYOUT_OVERRIDES`（Qwen3-Omni/Qwen3-TTS 定制）决定。视觉/音频编码器经 `export_encoder.py` 的三张「家族注册表」路由到各自 `build_*` 函数，复用与 LLM 同一套 dynamo 底座与 custom_translation_table。核心结论是：sidecar（`config.json`/`embedding.safetensors`/tokenizer/`processed_chat_template.json`、经 `_patch_multimodal_token_ids` 注入的 `image_token_id` 等、以及 `--externalize-weights` 抽出的外部权重）是 Python 导出端与 C++ 运行时之间的契约文件。
