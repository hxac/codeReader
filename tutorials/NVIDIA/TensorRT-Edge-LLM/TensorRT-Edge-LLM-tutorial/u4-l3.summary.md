本讲讲解把 ONNX 编译成 engine 的两个 CLI：llm_build 与 visual_build。二者均为「薄封装」，统一流程为 getopt_long 解析参数 → 校验 config.json 是否存在 → 填充 Config → 构造 Builder → 调用 build()，真正的八阶段构建逻辑全在 builder 内。llm_build 的核心参数为 maxInputLen（prefill 一次最多喂多少 prompt token）、maxKVCacheCapacity（整条序列长度上限，即 KV 缓存与 RoPE 可寻址位置）、maxBatchSize，须满足 maxKVCacheCapacity ≥ maxInputLen，可生成余量为两者之差，显存随二者线性增长。--specBase/--specDraft 标记投机解码 base/draft 角色，二者必须共用同一 engineDir，靠 spec_base.engine/spec_draft.engine、base/draft_config.json 及 draft_ 前缀分流防冲突（--eagleDraft/--eagleBase 为其弃用别名）。visual_build 用图像 token 三元组 minImageTokens ≤ maxImageTokensPerImage ≤ maxImageTokens 刻画视觉编码器范围，引擎落到 engineDir/visual/ 子目录，无 prefill/decode 双 profile、无 KV 缓存。VLM 场景下语言侧 maxInputLen 需把图像 token 也计入。
