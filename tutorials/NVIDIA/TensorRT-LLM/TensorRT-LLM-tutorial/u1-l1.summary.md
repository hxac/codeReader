本讲是 TensorRT-LLM 学习手册第一讲，建立「高空视图」。核心定位：它是 NVIDIA 开源库，用专用 kernel、高效运行时与可扩展的 Python 框架，优化 LLM 与 Visual Gen 在 NVIDIA GPU 上的推理；强调 PyTorch 原生架构——模型用原生 PyTorch 写，运行时可改可扩展，设计口诀是「Python 调度、C++ 加速」。

请求全链路为 HF Model → LLM API → Executor → Scheduler → Model Forward → Decoder → Sampling → Tokens，由 PyExecutor 单步循环推进（取请求→调度→前向→decoder→追加 token），这是 in-flight batching 的基础；调度器分 CapacityScheduler 与 MicroBatchScheduler 两步。推理分 Prefill（算力密集）与 Decode（带宽密集）两阶段。

后端矩阵：PyTorch 为默认后端（入口 TorchLlmArgs，路径 `_torch/pyexecutor/` → PyExecutor），AutoDeploy 为 Beta（通过 shim 适配 PyExecutor，做 FX 图变换 + torch.export），二者共享 nanobind 暴露的 C++ 核心，含调度流水线（Scheduler→BatchManager→KVCacheManager）与解码流水线（Decoder→Sampling）。资源管理遵循 prepare/update/free 三段式生命周期。

三条产品线：LLM API（离线推理）、trtllm-serve（OpenAI 兼容在线服务，支持分离式服务）、VisualGen（DiT 图像/视频生成，非 LLM 后端）。关键术语：KV cache、in-flight batching、nanobind、分离式服务、投机解码、MoE 与专家并行。
