本讲介绍 `axi/simlink/` 模块：它通过 **VHDL 空壳实体 + `FOREIGN` 属性 + C 实现** 的技巧，把 VCS 仿真器内的 AXI-Lite / AXI-Stream 接口经 **ZeroMQ（PUSH/PULL 套接字，本机相邻两端口）** 桥接到 Python 的 **Rogue** 框架，实现软硬件协同仿真（co-simulation），让 Python 像驱动真实硬件一样驱动仿真中的 RTL。其中 VHPI（IEEE 1076-2008，GHDL 不支持）是 VHDL 的 C 接口。

三套桥共享 **VhpiGeneric** 公共框架：在时钟信号上注册值变化回调，每拍执行“读端口 → std_logic 枚举转 int（九值压成 0/1，不建模 X/Z）→ 调业务 `Update` → int 转回枚举 → 写回端口”，业务代码只需实现 `Update` 并自行检测上升沿。

两条核心数据通路：**RogueTcpMemory** 是 AXI-Lite 主机（与 u8-l3 的 I2C/SPI 从机桥方向相反），用 7 态状态机逐拍驱动五通道，收 4/5 帧请求（id/64 位 addr/size/type[/data]）、回 6 帧响应，每拍搬 4 字节、地址按字节偏移自增；**RogueTcpStream** 是 AXI-Stream 帧桥，内部固定 64 位、理解 SSI 的 SOF/EOFE 侧带，inbound 逐拍按 TKEEP 拼帧发 Python（4 帧：flags/chan/err/data），outbound 反向逐拍拆帧驱动。工程里实际例化含位宽整形与多通道路由的 `*Wrap` 封装。`ruckus.tcl` 据 GHDL 标志在 `sim/`（带 VHPI）与 `ghdl/`（占位）间分流，真正运行需 VCS 编出 `libAxiSim.so`。
