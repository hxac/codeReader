本讲讲解 SURF 回归测试工具链的四块拼图：ruckus 负责读遍所有 ruckus.tcl 并把全仓库源码展平成清单，GHDL 负责编译与仿真，cocotb 用 Python 协程注入激励与断言，pytest 负责发现与并行调度，中间靠 cocotb-test 的 cocotb_test.simulator.run 把调用方向从「pytest 反过来启动 GHDL」，这是整条链的钥匙。

讲义围绕三个最小模块展开。其一是 ruckus 源缓存：make MODULES=$PWD import 把源码收集与仿真运行解耦，生成 build/SRC_VHDL/{surf,ruckus} 缓存，测试侧由 build_vhdl_sources 读取，缺失时报错并提示先 import。其二是翻译核心 run_surf_vhdl_test：它推算 Python 模块名、合并 import 缓存与测试自带 wrapper 源、用 _sim_build_path 生成参数专属构建目录、钉死与根 Makefile 一致的 GHDL 编译参数，最终调用 cocotb_test.simulator.run。其三是 pytest 并行回归：PARAMETER_SWEEP 配合 @pytest.mark.parametrize 把每个参数组合展开成独立仿真，pytest -n auto --dist=worksteal 用工作偷取动态均衡负载。

关键术语包括：HDL 泛型与环境变量的分工——以 _G 结尾的键经 hdl_parameters_from 进 VHDL 泛型，其余键经 extra_env 进环境变量并由 env_flag 读回；构建目录按参数隔离是并行安全的前提；-n 0 用于串行聚焦调试。CI 中 lint 线用 analysis、test 线用 import 加 pytest，两条线共用同一套 Makefile/GHDL/ruckus 基础设施。
