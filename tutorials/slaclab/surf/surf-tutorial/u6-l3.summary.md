本讲承接 u6-l2 的 IPv4 引擎，引入填占协议号 0x11（UDP）槽位的 `UdpEngine` 与跳过 IP 的二层成帧器 `RawEthFramer`。`UdpEngine` 顶层纯结构，例化按本地端口号路由的 `UdpEngineRx`、服务器/客户端各一份 `UdpEngineTx`（靠 `IS_CLIENT_G` 区分）、可选 `UdpEngineDhcp` 与客户端 ARP 表，末级 `AxiStreamMux` 合流。关键差别在远端 MAC 来源：服务器被动从入向帧源字段学到远端 MAC/IP/端口三元组，客户端配置远端 IP、用应用流 `tDest` 作索引经 ARP 表翻成 MAC。收发两端沿用 u6-l2 的「非标准 IPv4 伪头部」，UDP 长度与校验和留 0 交 `EthMacCore` 补算，端口与 IP 均按 big-Endian 配置。`UdpEngineDhcp` 走 Discover→Offer→Request→ACK 四步握手，解析 option 53/51，收 ACK 后落实本机 IP 并启动续约（=租约/2）与租约两个定时器。`RawEthFramer` 给数据加/剥「目的 MAC + 源 MAC + EtherType（默认私有 0x1000）」二层头，用 LutRam 缓存与 `minByteCnt` 处理短帧末拍 `tKeep`、用 TUSER 的 BCF 位支持广播，共享的 remoteMac 由顶层小 FSM 仲裁；`UdpEngineWrapper` 则把客户端远端端口/IP 暴露为 big-Endian 的 AXI-Lite 寄存器。
