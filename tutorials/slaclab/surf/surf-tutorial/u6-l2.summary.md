本讲承接 u6-l1 的以太网 MAC 核心，介绍纯结构顶层 IpV4Engine：在 MAC 之上搭一层 IPv4 协议栈，数据通路为 DeMux→Rx→协议引擎→Tx→Mux。IpV4EngineDeMux 按 EtherType 分流 ARP/IPv4；IpV4EngineRx 剥掉 20 字节标准 IP 头、按协议号用 tDest 路由到对应子引擎；IpV4EngineTx 用 tDest 反查协议号、重建 IP 头并支持本地回环；AxiStreamMux 合并回 MAC 发流。Rx/Tx 共享一份「非标准 IPv4 伪头部」（remote MAC、源/目的 IP、协议号、长度），使子引擎无需重复解析 IP 头；协议表由编译期函数把 ICMP/IGMP 追加到用户协议尾部，索引位置确定。ArpEngine 是三合一的 5 状态机：响应入向请求（目标为本机时原地改写应答）、客户端 IP→MAC 缓存带 1 秒超时防重发、本机 IP 直接短路。IcmpEngine 就地回 ping：交换源/目的 IP、Echo Request(0x08)→Reply(0x00)、用 Internet 校验和增量法（+0x0800，含进位回卷）修正，不缓存整包。可选 IgmpV2Engine 完成 IGMPv2 组成员管理（上电与收到 Query 时上报 Report，收到同组 Report 自我抑制）。关键术语：协议号解复用、非标准伪头部、tDest 路由与反查、ARP 1 秒缓存超时、增量校验和。
