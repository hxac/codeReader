本讲在 SSI 侧带（SOF/EOF/EOFE/VC）语义上展开两件事：链路自检与帧保护。自检由 `SsiPrbsTx`/`SsiPrbsRx` 这对「同款 LFSR + 事件号种子」核完成——Tx 发三段式帧（种子字＝事件号、长度字、LFSR 数据字），末拍据 `forceEofe` 或 FIFO 溢出写 EOFE；Rx 用同一种子重建 LFSR、逐拍比对，输出丢包/长度/数据总线/EOFE/字错五类错误，结果跨域上报。种子与步数相同使序列逐比特吻合，故可逐拍校验而无需缓存期望数据。帧保护由三类核构成：`SsiIbFrameFilter`/`SsiObFrameFilter` 在 FIFO 两侧强制帧不变量（单 SOF、单 VC、不交错），把散字丢弃、重复 SOF 或 VC 交错截断成 EOFE 帧、溢出时补 EOFE 收尾；`SsiFrameLimiter` 用 `FRAME_LIMIT_G`（字数）与 `TIMEOUT_G` 两道闸门把超长/挂死帧截断成 EOFE。核心结论是：EOFE 是贯穿全链路的统一错误语义总线，可由 Tx 注入、过滤器/限制器补盖、Rx 汇总报错；这对 PRBS 核是几乎所有数据链路回归测试的底层激励/校验源。
