本讲解析 SURF 的 FIFO 构建块：一个 FIFO 由「写 FSM（FifoWrFsm）+ 读 FSM（FifoRdFsm）+ 双口 RAM」三件套拼成，同步版为 FifoSync、异步版为 FifoAsync。同步 FIFO 单时钟域、指针直接用二进制；异步 FIFO 用 Gray 码指针配合 SynchronizerVector 跨时钟域，因相邻值仅差 1 比特，同步延迟只会让对端指针「滞后」而永不为乱码，从而保证写侧偏满、读侧偏空的单向保守安全。两个 FSM 各维护 count = 本地指针 − 对端同步后指针：满侧标志（full/almost_full/prog_full）归写域，空侧标志（empty/almost_empty/prog_empty）归读域，有效容量为 2^ADDR_WIDTH_G − 1。读侧由 FWFT_EN_G 在标准读与首字直通（FWFT）间切换，FWFT 用 tValid 小流水消化 BRAM 2 拍 / LUTRAM 1 拍读延迟，FifoOutputPipeline 再做可选 skid 与空洞压缩。统一封装 Fifo 用 SYNTH_MODE_G（inferred/xpm/altera_mf）与 GEN_SYNC_FIFO_G 在编译期分流后端，ruckus.tcl 按 Vivado 版本门控 FifoXpm 是否进构建（否则用 dummy 占位）；同步模式下 wr_data_count 与 rd_data_count 别名到同一内部信号。
