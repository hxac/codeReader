sv-elab（yosys-slang）只翻译 SystemVerilog 的可综合子集。面对未支持的 SV 构造，翻译器有两条路：**软诊断**（`add_diag`，攒进队列后综合继续，常用于「有意不支持」）与**硬中止**（`require`/`unimplemented`/`ast_invariant`/`ast_unreachable` 四个宏，经 `abort_helpers.cc` 的 `unimplemented__` 自动打印 AST 的 JSON 与源码行 dump 后 `log_error` 退出，错误信息直接带触发点的文件:行号）。关键判断：预期构造会出现但有意不综合用软诊断，确信不该发生或尚未编写用硬中止。

扩展点挂在 slang **ASTVisitor** 上的三大访问器：`EvalContext::operator()`（表达式，产出 `RTLIL::SigSpec`，出口有位宽不变量）、`StatementExecutor`（语句，产 case 树）、`PopulateNetlist`（符号，产 RTLIL 模块内容）。前两者已有软诊断兜底（`LangFeatureUnsupported`），符号层未覆盖则默认硬中止。加支持的套路：跑一次读 dump 定位访问器与 kind → 判断是否对应电路（生成硬件 or 软诊断跳过）→ 加 `case`/`handle` → 必要时按「五处对齐」加诊断码 → 补测试。生成硬件配等价性测试（`equiv_make`/`equiv_induct`/`equiv_status`），软诊断配 `test_slangdiag -expect`。`wait`（Warning）与 `covergroup`（硬中止）是一对对照范例。贡献前须先与维护者沟通，且项目有严格 AI 政策：禁止提交 LLM 生成的代码到 PR，但允许用 LLM 辅助搜索、调试与机械编辑。
