本讲讲解 sv-elab 同时维护的 CMake（主线）与 Bazel（封闭可复现）两套构建系统。CMake 顶层负责搭舞台：设 C++20、定义 `BUILD_AS_PLUGIN` 选项、用 FetchContent/子模块装配 fmt 与 slang。核心约束是 PIC——slang 与 fmt 因被静态链接进共享库插件，必须在 `BUILD_AS_PLUGIN=ON` 时强制位置无关编译。`cmake/FindYosys.cmake` 把外部 `yosys-config` 包装成 `yosys::yosys` INTERFACE IMPORTED 目标，只透传 `-I`/`-D` 以免污染编译选项。`GitRevision.cmake` 配合 `version.h.in` 在配置期把 git 哈希注入 `version.h`，供 `slang_version` 打印，失败回退 `UNKNOWN`。`src/CMakeLists.txt` 据 `BUILD_AS_PLUGIN` 分叉：插件分支产出 `slang.so`（隐藏符号、安装到 `${YOSYS_DATDIR}/plugins`），静态库分支产出自包含的 `libyosys-slang.a`。Bazel 线经 `MODULE.bazel`、`dependency_support/*.bzl` 与根 `BUILD.bazel` 产出同一插件，但只构建插件、依赖按固定哈希拉取，修订号取自 `module_version()` 而非 git。关键术语：BUILD_AS_PLUGIN、INTERFACE IMPORTED、PIC、FetchContent、use_extension、alwayslink。
