本讲承接 u8-l1 单元等价性测试，讲 croc_boot 这条端到端功能验证流水线：把真实 RISC-V SoC（croc）整体喂给 `read_slang` 综合，用 CXXRTL 仿真到 CPU 真正启动执行固件，最后校验一个金丝雀内存值（`0x03000008` 写 7）证明整条链路无误。核心源码在 `tests/croc_boot/`：`prepare.ys_`（`read_slang` 综合 + `clk2fflogic`/`chtype` 把触发器统一到自定义 `__ff` + `write_cxxrtl` 导出 + `g++` 就地编译）、`main.cc`（CXXRTL testbench，复位后播种 `wfi` 死循环 idle，并化身虚拟 JTAG 在 Unix 套接字上服务 OpenOCD）、`run.sh`（串起 main 与 OpenOCD）、`debugger.tcl_`（经 remote_bitbang/DMI halt→灌固件→校验金丝雀）。关键术语：differential validation by execution（不比网表比行为，无需参考实现）、CXXRTL、remote_bitbang、DMI、金丝雀。结论：croc_boot 走深度路线、CI 中并列的 compat-suite（6 个开源 IP）走广度路线，二者互补；croc_boot 不在 `ctest`，源码由固定哈希 tar 在 CI 注入，且只跑首尾 Yosys 版本。
