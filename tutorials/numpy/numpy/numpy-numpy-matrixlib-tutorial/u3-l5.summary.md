本讲是专家层，讲解 matrix 与 numpy 通用生态机制的交互，覆盖五个最小模块：subok、*_like、nditer、nanfunctions、MaskedArray 多重继承。核心结论有四：其一，`subok` 是子类型传染总开关，`astype`/`zeros_like` 等默认 `subok=True` 保 matrix 子类，`subok=False` 则降级为 ndarray，subok 管「类型」、copy 管「内存」，二者正交。其二，`nditer` 自动分配输出按 `__array_priority__` 选子类型（matrix=10.0 胜 ndarray=0.0），但 matrix 二维约束遇三维广播输出会抛 RuntimeError，用 `'no_subtype'` 标志可绕回基类。其三，`nan*` 函数保形靠「子类安全慢路径」：matrix 因 `type(a) is not np.ndarray` 必走慢路径，其 `np.amin/np.sum` 经 `_wrapfunc` 委托给 matrix 自己 `keepdims=True` 的归约方法，从而得到 `(1,N)`/`(N,1)` 朝向——单靠 `__array_finalize__` 无法给出列向量。其四，`MMatrix(MaskedArray, np.matrix)` 多重继承因 MRO 只自动调用一个 `__array_finalize__`，必须手动双调两个基类的 finalize 才能同时守住「二维」与「掩码传播」两套不变量。贯穿全讲的两个协议是 subok（构造时选不选子类型）与 `__array_priority__`（混合运算/分配时谁赢）。关键术语：subok、`__array_priority__`、慢路径/快路径、`_wrapfunc` 委托、`'no_subtype'`、`__array_finalize__` 手动双调。
