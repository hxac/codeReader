本讲精读 `numpy.matrixlib.defmatrix.py` 中 `matrix.__new__` 构造函数。核心结论是：`matrix` 类不写 `__init__`，所有构造工作在 `__new__` 里完成，最终通过 `N.ndarray.__new__(cls, shape, dtype, buffer=arr, order=order)` 一次性定下形状、dtype 与内存布局，并用 `buffer=arr` 借用已规整好的内存。

`__new__` 入口无条件发出 `PendingDeprecationWarning`（`stacklevel=2` 指向调用者，故 `asmatrix`/`bmat` 也触发），随后按四路分发：分支 A 处理 `matrix` 输入、分支 B 处理 `ndarray` 输入、分支 C 处理字符串（交 `_convert_from_string`）、分支 D 处理通用 array_like。判断顺序上 `matrix` 必须先于 `ndarray` 判，否则因子类关系会被吞掉。

关键概念与术语：`copy` 标志语义随输入而异——仅「matrix 输入＋同 dtype＋`copy=False`」返回原对象（`is` 为真），ndarray 输入 `copy=False` 得共享内存的视图、`copy=True` 得副本，通用输入则把 `copy=False` 转为 `None` 交给 `np.array`。二维强制把 0 维补成 `(1,1)`、1 维补成 `(1,N)`、超二维拒绝；但报错有两条消息：通用输入抛 `matrix must be 2-dimensional`，ndarray 输入经 `data.view(cls)` 触发的 `__array_finalize__`（会先挤掉长度为 1 的维度再判定）抛 `shape too large to be a matrix.`。`__array_finalize__` 的完整机制留给下讲。
