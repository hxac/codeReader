本讲聚焦 `np.matrix` 的字符串构造入口（如 `np.matrix('1 2; 3 4')`），解析其核心实现——模块级私有函数 `_convert_from_string`。该函数定义于 `defmatrix.py` 顶部，由 `matrix.__new__` 的字符串分支调用；它不返回 ndarray 或 matrix，而是把字符串翻译成普通 Python 嵌套列表（`list[list]`），随后在 `__new__` 中与通用 array_like 分支汇合，统一交给 `np.array` 定形。

讲义拆为四个最小模块：①函数全貌与角色，②分隔符语法，③元素求值，④行列校验。关键结论是：字符串语法有三层分隔符，优先级为 `;`（分行）> `,`（分块）> 空格（分元素），`,` 与空格在「列」层等价可混用；方括号 `[` `]` 在解析起始即被无条件剥离，仅作装饰。每个元素用标准库 `ast.literal_eval` 求值而非 `eval`，前者只接受安全的 Python 字面量（数字、布尔等），拒绝变量名与函数调用，从根本上杜绝代码注入，这也是 `matrix("invalid")` 抛 `ValueError` 的根因。行列校验以第一行长度为基准 `Ncols`，任意后续行列数不等即抛 `ValueError("Rows not the same size.")`。

引入的关键术语与概念（供后续承接）：字面量、`ast.literal_eval` 与 `eval` 的安全边界、嵌套列表、分隔符优先级、`__new__` 中字符串分支与通用分支的汇合点。本讲特指解析字面量的 `_convert_from_string`，与下一讲 `bmat` 中解析变量名的 `_from_string` 名称相近但职责不同，切勿混淆。
