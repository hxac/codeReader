本讲是 matrixlib 专家层收官篇，方法不是「读 `defmatrix.py`」而是「读测试、反推被测代码契约」。它覆盖 `numpy/matrixlib/tests/` 下 7 个文件的职责划分：自身行为（构造/索引/属性/代数）归 `test_defmatrix.py`，跨模块交互（sort/kron/nan/r_/nditer 是否退化）归 `test_interaction.py`，linalg 数值正确性归 `test_matrix_linalg.py`，历史 bug 回归归 `test_regression.py`，视图/点积/对角归 `test_multiarray.py`/`test_numeric.py`，与 MaskedArray 多重继承归 `test_masked_matrix.py`。核心结论：matrix 是「被弃用但要永久兼容」的子类，测试重心在回归保护而非功能扩展。引入的关键术语包括 pytest 收集约定与节点 ID（`路径::类名::方法名`）、`MatrixTestCase` 混入复用（数据 `LinalgCase`/`TEST_CASES` 与逻辑 `XxxCases` 基类分离，一行 `pass` 跑几十条）、`apply_tag` 标记、`@pytest.mark.thread_unsafe`、`PytestTester`（自动过滤 matrix 的 `PendingDeprecationWarning` 并返回布尔）、以及 Ticket #71/#125/#473/#83 四条回归契约。本讲把测试定位为三条核心不变量（恒二维、朝向正确、运算后保 matrix 子类型）的「可执行说明书」。注意 `like_function` 因缺 `test_` 前缀不被收集，是 pytest 约定的真实反面教材。
