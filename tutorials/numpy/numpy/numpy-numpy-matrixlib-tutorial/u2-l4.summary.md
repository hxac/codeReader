本讲聚焦 `np.matrix` 作为 `ndarray` 子类的两条「生命线」：`__array_finalize__` 与 `__array_priority__`。核心结论是：`__array_finalize__` 是 NumPy 在每次派生新数组（视图、切片、copy、astype、ufunc 输出等）时自动调用的收尾钩子，`matrix` 借此把可能掉维的中间结果重新补回二维，是「永远二维」不变量的真正守护者；改形状一律用私有方法 `_set_shape`，而非 `.shape =`，以绕开 NumPy 2.5 的弃用警告并就地重塑、避免再次回调。本讲引入四个最小模块与关键术语：`__array_finalize__` 的五条分支（含「索引短路」与「补行向量」）、`_set_shape` 内部通道、`_getitem` 标志（`__getitem__` 与 `__array_finalize__` 的握手，用于区分行向量与列向量）、`__array_priority__`（基类默认 0.0，matrix 设 10.0，高者优先决定混合运算的输出子类型）。关键直觉：`__new__` 仅显式构造时跑一次，`__array_finalize__` 才是每次派生都跑的约束守护者；优先级决定「结果的类型」，`__array_finalize__` 决定「该类型的收尾」，两者互补。
