`numpy.matrixlib` 是一个极小的子包，全部业务代码集中在 `defmatrix.py`，`__init__.py` 仅做装配与转发导出。子包对外只暴露三个名字——`matrix`、`asmatrix`、`bmat`，公开清单由 `defmatrix.py` 的 `__all__` 定义并经 `__init__.py` 原样转发，构成单一数据源。`np.matrix` 是一个继承自 `ndarray`、强制保持二维的子类：标量补成 `(1,1)`、一维补成 `(1,N)`、超过二维直接抛 `ValueError`；它还把 `*` 重定义为矩阵乘，与普通数组的逐元素语义冲突。正因如此，官方不再推荐使用 `matrix`，类文档明确建议改用普通 `ndarray` 配合 `@` 运算符和 `numpy.linalg`。每次 `matrix(...)` 构造都会在 `__new__` 开头无条件发出 `PendingDeprecationWarning`，该类别表示「计划弃用但尚无移除时间表」，比 `DeprecationWarning` 更软，且默认被 Python 过滤，需用 `warnings.catch_warnings(record=True)` 配合 `simplefilter("always")` 才能捕获；`asmatrix` 与 `bmat` 因最终调用 `matrix(...)`，同样会触发该警告。本讲引入的关键术语包括子包边界（`__all__` 转发）、强制二维约束、`PendingDeprecationWarning` 警告机制，为后续构造函数内部、字符串解析、运算符重写及子类化机制讲义奠基。
