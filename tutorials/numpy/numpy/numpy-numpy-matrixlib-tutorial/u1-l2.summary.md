本讲从「名字搬运」视角解析 `numpy.matrixlib` 的导出关系。核心结论：业务实现全集中在 `defmatrix.py`，`__init__.py` 仅做转发装配；公开清单 `__all__=['matrix','bmat','asmatrix']` 作为**单一数据源**定义在 `defmatrix.py`，子包门面与顶层 `__all__` 均引用它以防漂移。关键机制：`from . import defmatrix` 的**导入副作用**令 `np.matrixlib.defmatrix` 三层路径可达；顶层以显式 `from .matrixlib import ...` 将三名提升为 `np.matrix` 等 API，并用自定义 `__dir__()` 对 `matrixlib` 做「**软隐藏**」（属性在、目录不显示）；`@set_module('numpy')` 装饰器把 `matrix`/`asmatrix` 的 `__module__` 改写为 `'numpy'`，真实出处存入 `_module_source`，从而解释「三路径同源、`__module__` 全为 numpy」现象。另以 `PytestTester(__name__)` 为子包装配一行式 `test()` 入口。需澄清：`defmatrix`（def+matrix 拼接）及顶层别名 `_mat` 均非私有模块。
