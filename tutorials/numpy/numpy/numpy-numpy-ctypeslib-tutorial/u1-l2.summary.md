本讲是 numpy.ctypeslib 的「全景图」，只讲六个公共对象的职责，不展开实现细节。公共 API 由 `__all__` 白名单导出，恰好六个：`load_library`、`c_intp`、`ndpointer`、`as_ctypes_type`、`as_array`、`as_ctypes`，共同服务于「用 NumPy 数组安全地调用 C 函数」这一目标，可按「加载库 → 声明类型契约 → 准备/转换数据 → 调用」四阶段组织。其中 `load_library` 跨平台加载 C 共享库得到一个 CDLL 库对象；`ndpointer` 是工厂函数，按 dtype/ndim/shape/flags 约束动态造出一个数组类型，写入 argtypes 后 ctypes 在调用前自动调其 from_param 校验，不合规即抛 TypeError/ArgumentError，根本进不了 C；`c_intp` 是匹配平台指针大小整数（即 NumPy 的 intp）的 ctype 类型常量；`as_ctypes_type` 把 dtype 翻译成 ctype（翻译规则）；`as_array`/`as_ctypes` 则是零拷贝互转，返回对象与原对象共享同一段内存、改一边另一边跟着变，`as_ctypes` 借 `__array_interface__` 取地址并用 `__keep` 保活原数组防止悬空。本讲引入的关键术语：CDLL、argtypes/restype、零拷贝（zero-copy）、from_param 调用前校验、动态建类与缓存、内存共享。
