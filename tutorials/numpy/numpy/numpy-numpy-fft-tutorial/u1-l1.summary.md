本讲是 `numpy.fft` 学习手册首篇，定位「跑起来、看懂入口」，精读 `__init__.py` 与 `_pocketfft.py`（`fft`/`ifft`），仅对照 `_helper.py`。核心定位结论：`numpy.fft` 是随 NumPy 自带的「基础款」傅里叶变换子包，功能上是 `scipy.fft` 的子集，胜在零额外依赖；其能力分四类——Standard FFTs、Real FFTs、Hermitian FFTs、Helper routines，本讲只动手用 Standard 类的 `fft`/`ifft`。引入的关键术语：DFT（离散傅里叶变换，数学定义）、FFT（O(n log n) 的快速算法）、时域/频域（频谱）。DFT 定义中正变换指数取负号、反变换取正号且默认带 1/n 归一化，故 `ifft(fft(a)) ≈ a` 成立。源码层面：`fft`/`ifft` 的 Python 层极薄——`asarray(a)` 转数组、`n is None` 时取 `a.shape[axis]`，再统一委托给 `_raw_fft`；二者唯一差异是 `is_forward` 标志（`fft` 为 `True`、`ifft` 为 `False`），由其选用 C++ 扩展 `_pocketfft_umath` 的正向/反向 ufunc，真正的 FFT 计算落在 C++。`n` 参数控制输出长度：短则截断、长则补零。导出装配上，入口 `__init__.py` 以 `from ._pocketfft import *` 与 `from ._helper import *` 收集名字，并用 `_pocketfft.__all__.copy()` 再 `+= _helper.__all__` 拼成共 18 个名字（14 变换 + 4 helper）的 `numpy.fft.__all__`；`test` 函数不在其中。后续讲义将承接：目录与构建（Meson 编译 `_pocketfft_umath`）、数学约定与 `fftfreq`/`fftshift` 等 helper、以及 `_raw_fft` 内部逐行拆解。
