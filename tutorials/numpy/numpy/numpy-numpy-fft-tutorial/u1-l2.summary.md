本讲绘制 `numpy/fft/` 的文件地图，确立其**三层架构**：入口层 `__init__.py`（用 `from ._pocketfft import *` 与 `from ._helper import *` 聚合，`__all__` 经 `.copy()` 拼成 18 个公开名字）→ Python 逻辑层 → C++ 后端层。核心结论是**接口与实现分离**：`_pocketfft.py` 定义全部 14 个变换但自身不算 FFT，只做参数处理、归一化与轴编排，经统一入口 `_raw_fft` 委托给 C++ 扩展 `pfu`（即 `_pocketfft_umath`）；`_helper.py` 的 4 个 helper（fftfreq/rfftfreq/fftshift/ifftshift）是纯 Python，与算法解耦。引入的关键术语包括子包、C 扩展模块、gufunc、类型存根（`.pyi`，本包 3 个，运行时不参与）、git 子模块（`pocketfft/`，header-only，经 `pocketfft_hdronly.h` 被 `#include` 缝合，缺失则编译失败，`meson.build` 负责检查）。`meson.build` 区分编译、直接安装、带 `tests` 标签安装三类文件。
