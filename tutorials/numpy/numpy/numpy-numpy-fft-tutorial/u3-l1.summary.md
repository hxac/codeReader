本讲精读 `_pocketfft.py` 中 `fft`、`ifft`、`_raw_fft` 三个函数，打通一维复数变换主链路。核心结论：`fft`/`ifft` 是极薄 Python 外壳，仅做 `asarray`、`n` 默认取轴长、委托 `_raw_fft`，自身不碰算法；二者唯一本质差异是传给 `_raw_fft` 的 `is_forward` 标志（正变换为 `True`、反变换为 `False`）。`n` 小于输入长度则取前 `n` 点（截断），大于则末尾补零；`axis` 选被变换轴，默认 `-1`。引入的关键术语：`_raw_fft` 是所有一维变换的统一入口，用 `(is_real, is_forward)` 两个布尔压成四象限决策表选定 C++ 后端 gufunc（本讲落点 `pfu.fft`/`pfu.ifft`，输出长度恒为 `n`）；归一化系数 `fct`（即 `inv_norm`）在此计算，反变换经 `_swap_direction` 翻转 `norm` 语义；最后 `ufunc(a, fct, axes=[(axis,),(),(axis,)], out=out)` 缝合 Python 与 C++。`norm` 三模式、`rfft`/`irfft` 的 Hermitian 减半分别留待后续讲义展开。
