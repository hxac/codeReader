本讲以 `numpy/fft/meson.build`（38 行）为核心，讲清 NumPy 用 Meson 构建 fft 子包的完整链路，可分为三个最小模块：守卫检查、C++ 扩展编译、纯 Python 源安装。

核心结论是：整个子包只有 1 个文件 `_pocketfft_umath.cpp` 需要编译，其余 9 个文件（6 个包源/存根 + 3 个测试）全部原样安装——这正是「接口（纯 Python）与实现（C++）分离」在构建层面的落点。Meson 读 `meson.build` 生成 Ninja 脚本，再由 Ninja 调用 C++ 编译器把 `.cpp` 编成平台相关的动态库（`.so`/`.pyd`）。

引入的关键术语：**Meson/Ninja**（构建系统与底层脚本）；**`py.extension_module`**（编译 C/C++ 扩展的方法，要求扩展名、`PyInit_` 函数名、Python 导入名三者严格一致）；**`py.install_sources`**（原样拷贝纯 Python 源/存根）；**`np_core_dep`**（提供 NumPy C API 头文件包含路径的依赖对象）；**`install_tag: 'tests'`**（给测试文件贴标签，打包时可排除出发行 wheel）；**git 子模块**（`numpy/fft/pocketfft`，来源 `mreineck/pocketfft`，header-only 库，被 `#include` 缝合进 `.cpp`，clone 后默认为空，需 `git submodule update --init`）；**哨兵文件** `pocketfft/README.md`（守卫检查它来把子模块缺失转成清晰报错）。编译产物里注册了五个 gufunc，留待 u5-l1 详讲。
