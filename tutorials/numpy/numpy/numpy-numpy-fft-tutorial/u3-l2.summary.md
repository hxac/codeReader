本讲义聚焦 `_pocketfft.py` 中归一化机制，拆开上一讲遗留的「`fct` 如何算出」黑盒，覆盖三块：`norm` 三模式与 `fct` 计算、`_swap_direction` 方向对调、`real_dtype` 精度保护。

核心结论：`norm` 取 `"backward"`（默认，等价于 `None`）、`"ortho"`、`"forward"` 三选一，描述正反变换这一对里谁被缩放。`_raw_fft` 用一个以「正变换视角」书写的三路 `if/elif` 把 `norm` 映射为乘性系数 `fct`：`backward/None → 1`、`ortho → reciprocal(sqrt(n))` 即 1/√n、`forward → reciprocal(n)` 即 1/n，非法值抛 `ValueError`。因 `fft`/`ifft` 共用此入口，反变换进分支前先用 `_swap_direction`（依据 `_SWAP_DIRECTION_MAP`）把 `"backward"↔"forward"` 对调、`"ortho"` 为不动点，使一套分支对正反方向都正确；该函数还被 `hfft`/`ihfft` 复用以修正「用相反方向算子实现」的方向错位。

关键术语：`fct`（乘性归一化系数，因 C++ ufunc 用它做乘法而非除法，兼避零长度轴除零）、`real_dtype = result_type(a.real.dtype, 1.0)`（将 `fct` 计算精度兜到至少 `float64`，并复用为 `irfft` 输出 dtype）。由此 `float32`/`complex64` 输入也得到 double 族输出，任意 `norm` 下 `ifft(fft(a, norm=X), norm=X) ≈ a`。
