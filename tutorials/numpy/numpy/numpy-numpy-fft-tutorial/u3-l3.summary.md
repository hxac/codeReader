本讲聚焦 `numpy.fft` 的 `out` 参数透传链路与输出 dtype 决策，仅涉及 `_pocketfft.py` 的 `_raw_fft` 后半段与 `_pocketfft.pyi` 的存根。核心结论：四个公开函数 `fft`/`ifft`/`rfft`/`irfft` 只把 `out` 原样转交，不解析不校验；所有逻辑集中在统一入口 `_raw_fft`。`out=None` 时按「实/复」分流自动分配输出——`irfft` 用 `real_dtype = result_type(a.real.dtype, 1.0)` 输出实数，其余变换用 `result_type(a.dtype, 1j)` 输出复数；新数组形状仅把 `axis` 维替换为 `n_out`。`out` 已给时，Python 层在形状校验处仅检查维数与 `axis` 维长度是否等于 `n_out`，失败抛 `ValueError("output array has wrong shape.")`；dtype 兼容性则交给 ufunc 在 `ufunc(..., out=out)` 时把关，形成「形状 Python 把关、dtype ufunc 把关」的分层分工。类型提升遵循 NEP 50 弱类型：`float32→complex64`、整数/`float64`→`complex128`、`longdouble→clongdouble`。`.pyi` 把提升逐 dtype 写成 `@overload`，并为显式 `out` 单列返回类型跟随 `out` 的重载。关键术语：`out` 输出缓冲区、类型提升、NEP 50 弱类型、`real_dtype`、`n_out`、`_raw_fft`、`@overload` 存根。
