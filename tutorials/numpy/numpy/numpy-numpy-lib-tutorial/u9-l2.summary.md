本讲覆盖 `_nanfunctions_impl.py` 中「需要做除法或排序」的六类 NaN 感知统计函数：`nanmean`、`nanvar`、`nanstd`、`nanmedian`、`nanpercentile`、`nanquantile`。核心结论是：与 `nansum`/`nanprod` 仅靠「替换-聚合」不同，这组函数必须**先数清有效个数（`cnt`）或先物理剔除 NaN**，否则分母与排序结果都会被污染。`nanmean` 走「替换 NaN 为 0 → 求和 ÷ `cnt`」并用 `_divide_by_count` 安全除，整数输入经 `np.mean` 自动升 `float64`，全 NaN 切片靠 `0/0` 天然得 NaN。`nanvar` 在此基础上增加就地「偏差-清零-平方」三步，关键是用 `_copyto` 把 NaN 位置减均值产生的虚假偏差清零，`dof<=0` 时显式写回 NaN；`correction` 是 `ddof` 的 Array API 别名，`nanstd` 仅为开方薄包装。`nanmedian` 经 `_ureduce` 套壳，内核 `_nanmedian` 分 1D／小数组（<600 复用 `np.ma.median`）／大数组三路径，底层用 `_remove_nan_1d` 剔除 NaN。`nanpercentile`/`nanquantile` 几乎不做分位计算，只规整 `q`（÷100 或 `[0,1]`）后委托 u7-l2 的 `_quantile_unchecked`，体现「清洗外壳 + 复用算法内核」的分层设计。引入的关键术语：自由度 `ddof`/`correction`、虚假偏差清零、`_divide_by_count`、`_remove_nan_1d`、`_quantile_unchecked`、`_ureduce` 复用。
