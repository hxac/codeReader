本讲义覆盖 `numpy/lib/_stride_tricks_impl.py` 中三个广播公开函数与两个内部内核。核心直觉是：广播等于插入步长为 0 的维度，因此广播结果是零拷贝视图且天然只读（多处指向同一内存，写入不安全）。`broadcast_to` 把单数组广播到目标形状，返回只读视图（`readonly=True`）；`broadcast_arrays` 把多数组对齐到公共形状，返回可写但带弃用警告的视图（`readonly=False`）；`broadcast_shapes` 只返回形状元组。内部内核分工：`_broadcast_to` 用 `np.nditer(itershape=...)` 真正生成视图，并做非标量→标量、负尺寸两道校验；`_broadcast_shape` 用旧 `np.broadcast` 算公共形状（因 nditer 对 size-0 处理不一致），并按 63 分块绕开 64 参数上限。关键术语：0 步长、广播视图、`itershape`、`itviews`、`_warn_on_write`、零字节占位数组（`np.dtype([])`）、NEP-18 `array_function_dispatch`。导出上，三个函数经 `module='numpy'` 加顶层直接导入挂到 `np.`，不经薄模块。承接 u5-l1 的步长/视图/`__array_interface__` 概念，为 u6 数值函数的形状预测与只读视图直觉奠基。
