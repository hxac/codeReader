本讲深入 `numpy/lib/_stride_tricks_impl.py`，拆解视图构造工具 `as_strided` 与 `sliding_window_view`，核心认知是：**`as_strided` 是 `sliding_window_view` 的内核，二者只是「手工调参」与「自动算参」之别**。关键概念包括：步长（stride，沿某维前进一格的字节位移，可为 0、负数或大于 itemsize）、视图（view，换 shape/strides 读同一块内存、零拷贝）、`__array_interface__` 协议。最小模块含 `DummyArray`（仅挂载接口字典并保活 base 的替身对象，是注入任意 shape/strides 的官方后门）、`as_strided`、`_maybe_view_as_subclass`（subok 时经 `.view(type=...)` 还原子类并调用 `__array_finalize__`）、以及复用的 `byte_bounds`（越界校验尺子）。核心流程：复制接口字典 → 改写 shape/strides → `asarray(DummyArray)` → `_set_dtype` 补回结构化 dtype → 子类还原 → 可选只读与 `check_bounds` 校验（沿 `.base` 链下钻到 owner 后比对字节边界）。`sliding_window_view` 则把窗口翻译为 `out_strides = x.strides + 参与轴 strides`、`out_shape = 各轴缩(w-1) + window_shape`，默认 `writeable=False` 因滑窗天然自重叠。承接后续 u5-l2（广播即插入 0 步长）。
