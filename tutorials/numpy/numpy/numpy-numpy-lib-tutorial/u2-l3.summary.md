本讲串讲 numpy.lib 三类「从数组外部观察或接入其内部」的工具，共同主题是用协议（`__array_interface__`、`__array_ufunc__`、CPU 分发表）接管数组行为。

`np.lib.introspect.opt_func_info` 从 C 层 `__cpu_targets_info__` 读取每个 ufunc 实际选中的指令集（`current`）与全部可选目标（`available`），支持按函数名与签名（类型字符或类型名）双重正则过滤，揭示 numpy「为何快」的 CPU 分发黑盒。

`byte_bounds`（实现藏于 `_array_utils_impl.py`，经薄模块 `array_utils.py` 再导出，遵循 `_impl` 分层）借助 `__array_interface__` 的 `data/strides/shape` 算出数组字节区间 `(low, high)`：逐维累加，正步长抬高 `high`、负步长拉低 `low`，末尾补 `itemsize`；连续时 `high-low` 恰为 `size*itemsize`。

`mixins.py` 用 `_binary_method`/`_numeric_methods` 等工厂把「调用某 ufunc」封装成 Python 双下方法（正向/反射/就地三形态），使 `NDArrayOperatorsMixin` 覆盖约 40 个运算符。该 mixin 自身不实现 `__array_ufunc__`，仅做「运算符→ufunc」接线，子类补上 `__array_ufunc__` 即可让运算走自定义逻辑并保持类型不变。`_disables_array_ufunc` 实现 NEP-0013 的 opt-out：对方设 `__array_ufunc__ = None` 时返回 `NotImplemented`，交还 Python 反射机制。引入的关键术语包括 `__array_interface__`、ufunc、双下方法三形态、NEP-0013 与 `__array_ufunc__`、`_impl` 再导出分层。
