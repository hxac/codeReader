本讲精读 `_function_base_impl.py` 中 16 个「条件选择与数组编辑」函数，共享同一范式：用 `asarray`/`asanyarray` 规整输入，再用布尔掩码或下标搬运数据，多数返回新数组。

`select(condlist, choicelist, default)` 靠「反向烙印」实现优先级——先铺满 default，再倒序 `copyto` 覆盖，靠前条件最后写、最终胜出；`piecewise` 在条件子集上调用对应函数，多出一个即为「否则」分支。`extract` 与 `place` 互逆：前者用 `take(ravel(arr), nonzero(cond))` 抽出真值为一维数组，后者就地写入，语义为「前 N 个 + 循环复用」，区别于 `copyto` 的按位置对齐。

工具型函数中，`copy` 默认 `order='K'`（保持布局，区别于 `ndarray.copy()` 的 `'C'`）；`average` 按加权公式计算，整数输入强制升 float64，权重和为 0 抛 ZeroDivisionError；`asarray_chkfinite` 仅对浮点类型查 NaN/Inf；`iterable` 对 0 维数组返回 False。`append`/`delete`/`insert` 均返回新数组，`append` 是 `concatenate` 封装，`delete` 掩码分支等价 `arr[~obj]`，`insert` 多点插入用稳定排序加 `arange` 累加补偿下标位移。`meshgrid` 的 `xy`/`ij` 差异源于前两轴交换；`digitize` 封装 `searchsorted`，`side` 与 `right` 因参数对调而反向；`sort_complex` 总返回复数；`rot90` 由 `flip`+`transpose` 组成、返回视图；`trim_zeros` 用 `argwhere` 的 min/max 找边界裁零。
