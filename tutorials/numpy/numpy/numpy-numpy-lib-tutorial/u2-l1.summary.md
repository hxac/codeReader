本讲义唯一覆盖 `numpy/lib/_version.py` 的 `NumpyVersion` 类，专门解决字符串逐字符比较的陷阱（如 `'1.10.0'` 会被误判小于 `'1.9.0'`）。核心机制：`__init__` 用三条正则把版本串解析成五个字段——`major/minor/bugfix`（还原为整数）、`pre_release`（取值 `'final'`、`'aN'/'bN'/'rcN'`，纯开发版为空串而非 final）、`is_devversion`（布尔）。比较经 `_compare` 三层级联完成：第一层 `_compare_version` 按整数比主体并短路返回 1/0/-1；第二层 `_compare_pre_release` 借 ASCII 序实现 a<b<rc，但 `'final'` 因 `'f'<'r'` 必须特判为最大、空串排最低；第三层比 `is_devversion`，dev 旧于非 dev。硬规则是同一预发布下所有 dev 版本（不同 git hash）比较相等。六个比较运算符全部一行委托给 `_compare`，据此可推得 `2.0.0rc1 < 2.0.0` 成立。
