本讲聚焦 numpy/lib/__init__.py，确立 numpy.lib 的整体定位与三大机制。核心结论：lib 是 numpy 的「杂项函数库」，收纳不属于 core、也不属于 random/fft/linalg/ma 等明确用途子包的函数。讲义覆盖三个最小模块：__init__.py 总入口、__all__ 公开命名空间、__getattr__ 兜底报错。引入的关键术语：__init__.py 是子包总入口，决定导入后能拿到哪些名字；__all__ 仅有 12 个名字，且 lib.__all__ ≠ 顶层 np. 命名空间——pad、histogram 等实现写在私有 _xxx_impl 里，再由顶层 numpy/__init__.py 往上搬到 np. 暴露；模块级 __getattr__（PEP 562）对 NumPy 2.0 移除/私有化的旧别名（如 emath、arraypad）抛出带迁移指引的 AttributeError，把模糊失败变成有指导的失败。此外区分两类源码：对外的薄再导出模块与藏实现的私有 _impl 模块，构成「实现藏 _impl、对外只露薄模块」的分层。
