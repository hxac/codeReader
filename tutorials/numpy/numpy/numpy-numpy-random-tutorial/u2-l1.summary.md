本讲讲解 `numpy.random` 两层架构（BitGenerator 产比特流、Generator 转分布）间的对接契约——C 结构 `bitgen_t`。它极简：1 个 `void *state`（类型擦除，容纳任意生成器内部状态）加 4 个 `nogil` 函数指针 `next_uint64`/`next_uint32`/`next_double`/`next_raw`，声明于 `bit_generator.pxd`，镜像自 C 头文件 `bitgen.h`，相当于「手写 vtable」。具体生成器（如 PCG64）在 `__init__` 中把 `state` 指向自身状态、把四个回调填入内嵌的值类型成员 `_bitgen`；基类只置空 `state` 并建好 `PyCapsule`，函数指针交由子类填——即「基类定骨架、子类填实现」。`next_double` 建立在 `next_uint64` 之上：`uint64_to_double(rnd)=(rnd>>11)/2^53`，正好用满 `double` 的 53 位尾数。`_bitgen` 经 `capsule`（名为 "BitGenerator"）、`ctypes`、`cffi` 三套接口对外暴露同一份底层数据。核心结论：分布层 `distributions.c` 全程只调函数指针取数，不出现任何具体生成器名，故生成器可自由替换、两层彻底解耦。
