本讲义定位 `numpy.random` 子系统，核心结论是官方主推新 API，旧 API 仅作向后兼容保留。它引入三套关键术语：一是推荐入口 `default_rng()`，返回 `Generator` 实例，其内部按 `seed` 类型派发（BitGenerator/Generator/RandomState/其他），默认路径为 `Generator(PCG64(seed))`，不维护任何全局状态。二是新 API 的两层架构——`BitGenerator`（如 PCG64、MT19937、Philox 等）只负责产出原始随机比特流，`Generator` 作为容器把比特流转成各种概率分布，实现关注点分离、内核可替换。三是旧 API 的遗留层，`RandomState` 基于 MT19937，所有模块级全局函数（`rand`/`randn`/`seed`/`random_sample` 等）均绑定到模块加载时创建的共享单例 `_rand`，导致隐式全局状态、难复现难并行。两者最本质差异在于兼容承诺：旧 API「冻结」、保证跨版本位级别一致；新 API「不保证」，可随算法改进改变比特流。因此「同种子⇒同输出」仅在相同 API 与算法内成立，`default_rng(42)` 与 `RandomState(42)` 结果完全不同。这些术语与结论为后续讲义（构建系统、`bitgen_t` 结构、`SeedSequence` 等）奠定大局观。
