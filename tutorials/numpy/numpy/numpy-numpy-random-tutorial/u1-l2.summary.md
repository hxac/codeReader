本讲绘制了 `numpy/random` 子系统的「骨架」：目录结构、构建系统与扩展模块依赖图。

目录按两条线索组织：一是「语言三层」——`.py`/`.pyx` 接口层、`.pxd` Cython 声明层、`src/**/*.c` 纯 C 实现层，上层调用下层；二是「生成器分目录」——各 BitGenerator 内核（mt19937/pcg64/philox/sfc64）各占 `src/` 下一子目录，而所有分布采样算法集中放在 `src/distributions/`。

构建中枢 `meson.build` 先把 `src/distributions/` 下 5 个 C 文件归档成 `npyrandom` 静态库（安装到 `numpy/random/lib`，供第三方 Cython/CFFI/Numba 扩展链接复用）；再用 `foreach` 把 `random_pyx_sources` 声明的 9 个 Cython 扩展模块逐个编译，每个生成器包装模块都是「一个 `.pyx` + 一个同名 C 内核」并统一链接 `npyrandom`。其中 `_bounded_integers.pyx` 由 `.pyx.in` 经 Tempita 模板展开生成，是代码生成的典型例子。

关键例外是遗留模块 `mtrand`：它不链接 `npyrandom`，而是直接编译 `distributions.c` 与 `legacy-distributions.c`，并用 `-DNP_RANDOM_LEGACY=1` 启用遗留语义，从而把旧 API 比特流与新 API 隔离开，兑现「冻结遗留比特流」的承诺。

`c_distributions.pxd` 作为 `npyrandom` 的 Cython 侧「函数清单」，所有分布函数都以 `bitgen_t *` 为首参且多为 `nogil`——这预告了下一讲 `bitgen_t` 作为「生成器与分布层契约」的核心地位。
