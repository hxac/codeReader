本讲围绕 `numpy.random.default_rng()` 展开三个最小模块。其一，`default_rng` 是官方推荐的统一入口，默认返回 `Generator(PCG64(seed))`，且不维护任何全局实例——每次调用都是独立的新对象，与旧 API 的全局单例本质不同；它按 `seed` 类型做四路派发：BitGenerator 直接包装、Generator 透传（幂等）、RandomState 转换迁移、其余用 `PCG64(seed)` 兜底。其二，`Generator` 是「持有 BitGenerator 的容器」，自身不产比特，而是把底层比特流翻译成分布样本；构造时通过 `capsule`（PyCapsule）校验并解引用出 C 层 `bitgen_t` 结构，这座桥在后续讲义详述。其三，`seed` 是伪随机数列的起点：`seed=None` 拉取 OS 熵不可复现，固定整数则可复现；可复现性的精确表述为「相同 seed + 相同调用顺序 + 同一 NumPy 版本 ⇒ 相同输出」，而 `Generator` 声明了 No Compatibility Guarantee，跨版本不保证比特流一致。本讲引入的关键术语有 `default_rng`、`Generator`、`BitGenerator`、`PCG64`、`capsule`、`bitgen_t`、`SeedSequence` 与「不维护全局实例」「No Compatibility Guarantee」。需注意「同 seed 重建」可逐位复现，但「同一生成器连续抽两次」会因内部状态推进而得到不同结果。
