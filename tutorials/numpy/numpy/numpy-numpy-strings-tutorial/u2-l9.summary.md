本讲覆盖 numpy.strings 的 center/ljust/rjust/zfill/expandtabs 五个对齐填充函数。核心结论：它们在 Python 包装层共享「校验参数 → 变长 StringDType('T') 快速分流 → 预算输出宽度 → empty_like 预分配 out → 交给 C 层私有 ufunc」的四步套路，即 u2-l5 归纳的「输出 dtype 路径 A」。根本动因是 C 层定长 ufunc 无法从输入 dtype 推断输出宽度，故由 Python 层负责量尺寸、开缓冲区。关键术语：str_len 充当「尺子」，用 np.maximum(str_len(a), width) 取逐元素宽度、再 .max() 拼出全数组统一宽度的定长 out_dtype；center/ljust/rjust 是共用 _just_dispatcher 的三胞胎，仅调用的 C 层 ufunc 名不同，填充策略全在 C 层；变长分流分两种写法——三者因含 fillchar 用 result_type(a, fillchar).char=="T"，zfill/expandtabs 用 a.dtype.char=="T"；fillchar 须恰好一字符；zfill 的「符号 + / - 后补零」与 expandtabs 的两遍式（_expandtabs_length 先量长、_expandtabs 再填充，因输出宽度强数据相关）均下沉 C 层、Python 层不可见。
