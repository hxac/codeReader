本讲剖析 numpy.strings 七个查找函数（find/rfind/index/rindex/count/startswith/endswith）高度同构的原因：其输出宽度固定（int64 或 bool），Python 包装因此极薄，仅用 `@set_module` 装饰、无 dispatcher，函数体仅两行——把 `end=None` 归一化为 `MAX` 哨兵（`np.iinfo(np.int64).max`）后转交对应 `_xxx_ufunc`。C 层不理解 None，靠照搬自 CPython 的 `adjust_offsets` 把 MAX 收敛到字符串实际长度，并统一规范化 start/end 的切片语义。find 与 index 的差异同样发生在 C 层：string_index 在找不到时设置 ValueError 并返回特殊状态码 -2，共用循环 string_findlike_loop 见 -2 立即中止整个 ufunc，故数组中任一元素缺失都会使 index 整体失败。find/rfind/index/rindex/count 五件套共用同一循环靠函数指针区分，startswith/endswith 共用另一循环靠 FRONT/BACK 起点区分。
