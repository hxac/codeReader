本讲讲解 `numpy.strings` 的三个拆分函数：`partition`、`rpartition`、`slice`，核心代码集中在 `numpy/_core/strings.py`。

`partition`/`rpartition` 把每个字符串切成三段并返回三元组数组。其难点是输出宽度取决于分隔符位置这一运行时数据，无法从输入 dtype 推断，故走 u2-l5「路径 A」的变体：Python 层先用 `_find_ufunc`（`partition`，找首个 sep）或 `_rfind_ufunc`（`rpartition`，找最末 sep）定位，用 `str_len` 量尺寸，再拼出结构化 dtype 字符串（如 `"U5,U1,U8"`），用 `empty_like` 开一个含 `f0/f1/f2` 三字段的缓冲区，委托 C 层 `_partition_index`/`_rpartition_index` 写满后拆成元组。二者复用同一 `_partition_dispatcher`；`rpartition` 仅在「找不到分隔符时整段归后半」上与 `partition` 相反。引入的关键概念是**结构化 dtype**、复用的 **`MAX` 哨兵**，以及中间段在「全找不到」时优化为宽度 1。

`slice` 引入关键概念 **`np._NoValue` 全局唯一单例哨兵**：把 `stop` 默认值设为它，以区分「没传」与「显式传 `None`」，模仿 CPython `slice_new` 的「单参数当 stop」语义；`None` 起止值按 `step` 正负用 `np.where` 转成 `intp.max/min` 边界，模仿 `PySlice_Unpack`，并支持逐元素数组 `step`，最终委托 C 层 `_slice`。注意 `slice` 仅有 `@set_module`，不参与 NEP-18 分发。StringDType（`'T'`）输入在三者中均走快速分流，直接委托 C 层、不预算宽度。
