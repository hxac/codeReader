本讲讲解 NumPy 中那些暂无专用 C ufunc 循环、转而委托 `_vec_string` 的字符串函数。`_vec_string` 是连接数组与 Python 字符串方法的通用桥：C 侧按元素类型挑方法对象（`S` 取 bytes、`U`/`T` 取 str），按有无参数分派 `_vec_string_no_args`（快路径）与 `_vec_string_with_args`（带广播）。大小写族（upper/lower/swapcase/capitalize/title）输出宽度不变，直接复用输入 dtype，不经 object 中间态；mod/decode/encode 输出宽度数据相关，须先用 `np.object_` 承接、再用 `_to_bytes_or_str_array` 测宽还原，mod 用 dunder 名 `__mod__`。translate 对 str_/bytes_ 双分支，是因 `str.translate`（一参）与 `bytes.translate`（两参）方法签名不同，而非 dtype 还原。关键术语：`_vec_string` 及快慢两条路径、object 中间态还原、`__all__` 以「输出宽度能否仅由输入 dtype 决定」判断函数能否演化为 ufunc 的分界准则。
