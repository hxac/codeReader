本讲讲解 StringDType（`'T'`，变长 UTF-8 存储）专用的 ufunc 循环，主体文件为 `stringdtype_ufuncs.cpp`。其元素是「定宽句柄 `npy_packed_static_string` + 堆上变长字节」，输出宽度无法在循环前预知，故定长 dtype 的「预算宽度 + 预分配 out」套路失效，须由独立循环逐元素 `load` 解包句柄 → 算宽度（含溢出检查）→ `load_new_string`/`pack` 打包。循环统一骨架为：抢实例自带 `allocator` 锁 → 逐元素处理（含 NA 传播/替身/报错三分支）→ 放锁。注册总入口 `init_stringdtype_ufuncs` 与 `init_string_ufuncs` 平行，往同一批 ufunc 挂含 StringDType 的循环，复用「一族函数共用一个 loop + static_data 区分行为」的手法。Python 层 `char=='T'` 分支把宽度预算、溢出检查、开 out 整体下放给 C 层（如 `multiply` 命中 `multiply_loop_core`）；无直接循环的 dtype 组合由 promoter 提升到 StringDType，再经 `casts.cpp` 转码命中循环。三件配套各司其职：`dtype.c` 管 dtype 本体与 setitem/getitem/NA，`casts.cpp` 提供双向转换是 promoter 能工作的前提，`utf8_utils.c` 提供 UTF-8 编解码原语支撑按码点计数/切片的语义。
