本讲承接 u1-l1，深入 `numpy/_core/strings.py` 的最基础部分：`numpy.strings` 处理的三种字符串 dtype 及字符计数机制。核心结论：`numpy.strings` 的输入有三种 dtype——变长 `StringDType`（`dtype.char == 'T'`，内部 UTF-8 动态存储）、定长 `bytes_`（`'S'`，1 字符 = 1 字节）、定长 `str_`（`'U'`，UCS4，1 字符固定 4 字节，故 `itemsize` 是字符数的 4 倍）。代码中常以 `dtype.char == 'T'` 为变长类型走专用快速路径（如 `multiply` 的 `return a * i`）。本讲引入两个最小模块：其一 `_get_num_chars` 是纯 Python 私有辅助函数，面向定长 dtype，用 `issubclass(a.dtype.type, np.str_)` 判断后对 `str_` 返回 `itemsize // 4`、否则返回 `itemsize`，给出的是 dtype 字段宽度；其二 `str_len` 是真正的 ufunc（由 `numpy._core.umath` 导入，`__module__` 被改写为 `numpy.strings`），逐元素返回字符串实际字符数，对三种 dtype 语义统一。关键区分：`str_len` 统计运行时实际长度，`_get_num_chars` 读取类型字段宽度，二者不可混淆。「按 dtype.char 分发 + 用 str_len 统一字符数」是后续讲义反复出现的套路。
