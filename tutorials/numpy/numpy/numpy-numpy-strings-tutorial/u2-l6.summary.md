本讲覆盖 numpy.strings 三类运算：比较类六件套（equal/not_equal/less/less_equal/greater/greater_equal）、add（字符串拼接）、multiply（字符串重复）。核心结论：比较类与 add 属「直接复用型」ufunc，Python 层零包装、无 dispatcher，不在 `_override___module__` 名单内，处理字符串全靠 C 层把循环挂到既有顶层 ufunc 上。multiply 是本组唯一特例：带 `@set_module` 与 `@array_function_dispatch` 的 Python 包装函数，内部调用改名为 `_multiply_ufunc` 的顶层 ufunc；其双分支为 StringDType（`dtype.char=='T'`）直接 `return a * i`（溢出检查委托 C 层），定长 `str_`/`bytes_` 则走「`str_len` 预算输出宽度 + `sys.maxsize` 溢出保护（判据 `L·i>sys.maxsize`）+ 显式 `out`」三步——因底层 ufunc 无法凭输入 dtype 推断输出宽度，`resolve_descriptors` 强制要求 `out`。引入术语：直接复用型 ufunc、Python 包装函数、双分支、缓冲区预算、out 必填。
