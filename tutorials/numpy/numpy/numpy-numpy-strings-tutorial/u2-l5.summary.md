本讲梳理 `numpy/_core/strings.py` 三个私有辅助函数与它们支撑的 dtype 分发套路。`_get_num_chars` 封装「str_ 需 `itemsize // 4`、其余直接用 itemsize」这一规则，从 dtype 反推定长字段「容量」（区别于运行时长度），且全文件仅被 `_to_bytes_or_str_array` 调用。`_to_bytes_or_str_array` 是 object 中间态还原桥梁：用「模板」定输出 dtype 类、用实际数据定宽度，含空数组保 shape 与 StringDType 跳过宽度计算两处特判。`_clean_args` 为委托 Python 字符串方法清理不能传 `None` 的可选参数及其后续参数。核心结论是四条输出 dtype 路径：C（复用输入 dtype）、A（str_len 预算宽度拼定长 out 交 C 层 ufunc）、B（object 中间态+还原）、D（object 不还原，故未公开），凭是否出现 `_to_bytes_or_str_array`、`f"{char}{宽度}"` 等即可一眼归类。
