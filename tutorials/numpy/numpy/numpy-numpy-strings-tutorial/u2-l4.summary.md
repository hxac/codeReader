本讲讲清 `numpy.strings` 包装层的两件套装饰器。`@set_module`（定义于零依赖的 `numpy/_utils/__init__.py`）只把函数 `__module__` 改写成门面模块名、不创建新对象，管「身份」；`@array_function_dispatch`（`numpy/_core/overrides.py`）用 C 对象 `_ArrayFunctionDispatcher` 包住实现函数，启用 NEP-18 `__array_function__` 分发，管「行为」——调用时先由 dispatcher 取「相关参数」，无覆盖则走快速路径直接执行实现，有覆盖则整段交给该类型。dispatcher 只声明哪些参数相关，签名须与主函数一致且可选默认值只能是 `None`。`strings.py` 另用 `functools.partial` 预绑 `module='numpy.strings'`，与 `@set_module` 对 `__module__` 形成双保险。`str_len`、`isalpha` 等裸 ufunc 无法被装饰器装饰，改由 `_override___module__()` 在 import 时就地改写 `__module__` 与 `__qualname__`。后续 l5–l11 各具体函数均套用这套模子。
