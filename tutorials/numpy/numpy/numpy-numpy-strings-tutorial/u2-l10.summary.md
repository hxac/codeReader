本讲聚焦裁剪族 strip/lstrip/rstrip。Python 包装极薄，仅按 chars 是否为 None 在两条 C ufunc 间分发：缺省剥空白（1 入 1 出），给定剥字符集（2 入 1 出）。核心结论：strip 族走「路径 C」——输出不超输入长度，resolve_descriptors 直接令输出 dtype 等于输入 dtype，不预算宽度、不预分配 out，与对齐函数的「路径 A」形成反差；且无 dispatcher、不触发 NEP-18。C 层 6 个 ufunc（3 方向 × 2 模式）只对应 2 个循环：方向靠 STRIPTYPE 静态数据（LEFTSTRIP/RIGHTSTRIP/BOTHSTRIP）共用，模式因输入个数不同拆成两个。chars 是「字符集合」而非子串，从两端向内剥离、遇非目标即停，故被保护字符夹住的目标剥不掉（如 strip('  aA  ','a') 原样不变）。StringDType('T') 与定长 S/U 共用同名 ufunc，靠 dtype 自动分发到各自循环，无需 Python 层 T 分流。
