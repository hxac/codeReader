本讲阐明 `numpy.char` 与 `numpy.strings` 的关系：二者共享同一份底层实现（`numpy/_core/strings.py` 与 `numpy/_core/defchararray.py`），`char` 通过 `from numpy.strings import *` 复用全部公共函数，再以覆盖、改名包装、借用私有函数三种方式补充历史接口。核心结论有三：其一，`char.__all__` 比 `strings.__all__` 多 8 个成员（切分族 `join/split/rsplit/splitlines` 与 chararray 专属 `array/asarray/compare_chararrays/chararray`），`strings` 仅多一个 `slice`；其二，`_join/_split/_rsplit/_splitlines` 因返回长度不一的 object 列表数组、行为未定型，故以私有名存于实现层，只被 `char` 借为公共接口；其三，六个比较运算在 `char` 中先 `rstrip` 尾部空白（`compare_chararrays(..., rstrip=True)`）以兼容 numarray，`strings` 则原样比较。此外 `char` 对 `multiply`（TypeError 转 ValueError）、`partition/rpartition`（三元组转二维数组）做了薄包装，且 `chararray` 类及 `array/asarray` 自 NumPy 2.5 起已废弃。引入的关键术语：门面 `__all__`、numarray 兼容、`rstrip`、`compare_chararrays`、`chararray`、`_vec_string`、`set_module`/`array_function_dispatch`，为后续装饰器与分发机制讲义铺垫。
