本讲从 Python 包装层下探到 C++ 实现层，讲解定长字符串（bytes_/str_）的 ufunc 循环注册，核心文件为 string_ufuncs.cpp。注册总入口 init_string_ufuncs 采用三层架构：底层 add_loop 按名取出已存在的 ufunc、塞入循环指针并调 PyUFunc_AddLoopFromSpec_int 完成挂载（不创建 ufunc）；中层 init_ufunc 与 init_mixed_type_ufunc 组装 spec，顶层枚举函数。关键术语：ENCODING（ASCII/UTF32）模板借 NPY_OBJECT 哨兵按编码替换为真实 dtype，使一份循环同时服务 bytes_ 与 str_；priv=1 是携带 _NPY_METH_static_data 私有 slot（如 JUSTPOSITION、STRIPTYPE）的必要条件；比较族用 add_loops 变参模板批量注册。核心结论：_center/_ljust/_rjust 共用同一双编码模板循环、靠 static_data 区分行为，从而支持 ASCII fillchar 配 UTF32 输入；resolve_descriptors 有三种典型写法——A 类按输入宽度求和、B 类强制要求 out、C 类输出等于输入。
