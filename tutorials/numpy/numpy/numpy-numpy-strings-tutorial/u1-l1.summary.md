本讲是 numpy.strings 系列首篇，回答「`np.strings` 这个名字从哪来」。核心结论：`numpy.strings` 是 NumPy 向量化字符串运算的官方公开命名空间，它本身是纯门面——`numpy/strings/__init__.py` 仅两行，靠 `from numpy._core.strings import *` 转发全部 46 个公共符号（另显式补回 `__all__`/`__doc__` 两个 dunder），自身不实现任何函数；转发不复制不包装，门面函数与 `_core.strings` 中的函数是同一对象。类型存根 `__init__.pyi` 逐个显式列出接口清单，改门面须同步改 stub。`np.strings` 经顶层 `numpy/__init__.py` 的模块级 `__getattr__` 懒加载，首次访问才 import。三者分工为 `_core.strings`（实现，下划线表私有）→ `numpy.strings`（门面，新代码入口）→ `numpy.char`（`defchararray.py`，复用门面并补回 `join/split` 等行为未定型的历史函数）。引入的关键术语：门面、命名空间、向量化、懒加载、ufunc、`_vec_string`、`set_module`、`array_function_dispatch`、`.pyi` 类型存根，为后续讲义铺垫实现层与装饰器机制。
