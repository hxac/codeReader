本讲说明 numpy.core 垫片如何用 PEP 562 实现惰性转发。关键概念一：模块级 `__getattr__(name)`（无 self）仅在正常属性查找失败（先查 `__dict__`、再查 `ModuleType` 未命中）时被回调，且只在缺失时触发、每次缺失访问都触发（不自动缓存）、可用 `AttributeError` 表示属性真不存在。关键概念二：惰性转发把真模块的 `import` 写进 `__getattr__` 函数体，使垫片极轻、真模块按需加载，避免 import 时一次性拉起底层 C 扩展。核心区分：模块顶层代码因 sys.modules 缓存只执行一次，而 `__getattr__` 每次访问都跑——这正是垫片「既惰性、又每次打印 DeprecationWarning」的根因。主范例 numeric.py 四步：惰性 import 真模块、用 sentinel 哨兵取属性并判存在、调 _raise_warning 报警、正常返回；其中「先判存在再报警」使真缺失时不报警。包入口 `__init__.py` 与子模块垫片分两层转发，仅 import 放置位置不同。
