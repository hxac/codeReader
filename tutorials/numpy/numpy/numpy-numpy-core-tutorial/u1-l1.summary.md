本讲确立 `numpy.core` 的身份：它早已不含任何实现，只是 NumPy 2.0 为向后兼容保留的「垫片（shim）」。原 `core` 被改名为私有的 `numpy._core`，真正的数组、ufunc 等底层实现都搬到了那里，而 `numpy.core` 未来将被移除——这一结论的第一手依据是 `numpy/core/__init__.py` 开头的模块文档字符串。垫片靠「转发」工作：先以 `from numpy import _core` 引入实现包，再用 PEP 562 的模块级 `__getattr__`，对找不到的名字执行 `getattr(_core, name)` 取出同名对象，并经 `_utils.py` 的 `_raise_warning` 抛出 `DeprecationWarning`，提示改用 `numpy._core` 或公开 API。`__all__` 列出子模块名是为强制惰性加载、确保每次访问都打印警告。子模块属性（如 `numeric.asarray`）的转发分两层完成，本讲只覆盖包级第一层；`__getattr__`、惰性加载、`stacklevel` 等细节留待第 2 单元。验证手段是检查返回对象的 `__module__` 是否含 `numpy._core`。
