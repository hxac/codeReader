本讲只读 `numpy/core/_utils.py`（全目录唯一不做转发的 `.py`），核心是 `_raise_warning(attr, submodule=None)`：一个「纯副作用」函数——先用两个局部变量拼出旧/新模块名，再用「相邻字面量隐式拼接」拼出一段弃用长消息，以 `DeprecationWarning` 类别调用 `warnings.warn` 后返回 `None`，不抛异常。引入的关键术语：「单一信息源」（把 17 个垫片的弃用文本、类别、stacklevel 集中到一处，避免文本漂移）、可选参数 `submodule`（不传为包级 `numpy.core`、传 `"numeric"` 等为子模块级，取值约定即「垫片文件名去掉 .py」）、以及 `stacklevel`（从 `warn` 所在帧往上数层，决定警告归因到哪个文件的哪一行）。numpy 硬编码 `stacklevel=3`，因为包级与子模块级两条调用链形状相同——「用户 → `__getattr__` → `_raise_warning` → `warn`」恒为 3 帧，恰好把警告归因到用户代码。该值是脆弱的：调用链多包一层就须同步加 1。
