本讲聚焦「观测」：把访问 `numpy.core` 时触发的 `DeprecationWarning` 真正跑出来并抓住。最小模块有三个：标准库 `warnings`、`_utils._raise_warning`、`numeric.__getattr__`。核心结论：警告与异常不同——它不中断程序，「报警」与「正常返回」同时发生；Python 默认警告过滤器会静默第三方来源的 `DeprecationWarning`，故「没看到」不等于「没触发」。可靠捕获的唯一姿势是 `with warnings.catch_warnings(record=True)`，且 `simplefilter("always")` 必须写在 `with` 内部（因进入块时过滤器会重置）。关键术语：警告过滤器、`WarningMessage`（用 `.category`、`.message`、`.filename`/`.lineno` 断言）、`stacklevel=3`（把警告归因到用户代码而非 numpy 内部）、sentinel（哨兵写法）。`_raise_warning` 把类别固定为 `DeprecationWarning`、消息一定含 `numpy._core`，并通过 `submodule` 参数为各垫片复用。`numeric.__getattr__` 体现「转发+报警+仍返回」：属性真不存在时抛 `AttributeError`，废弃但存在才走警告。
