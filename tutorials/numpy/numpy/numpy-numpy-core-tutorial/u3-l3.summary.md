本讲回答「废弃模块的 `.pyi` 怎么写」。核心结论：类型存根只供类型检查器读取、运行时不执行，故永不触发 `DeprecationWarning`——弃用警告是 `.py` 的 `__getattr__` 产生的运行时副作用，与存根完全解耦。

`numpy/core` 存根按「可验证性原则」分三类：上游有 `__all__` 时走完整再导出（`numeric.pyi`/`umath.pyi` 用 `import *`，并补 `import __all__ as __all__`，因 `__all__` 以 `_` 开头不会被 `import *` 自动带入）；上游无 `__all__`/`__dir__` 且废弃非公开时走省略再导出（`overrides.pyi` 仅留 NOTE，因不可验证、且标废弃需逐字复制签名才能套 `@deprecated` 装饰器，即 PEP 702）；私有内部模块用近空/空存根（`_internal.pyi` 一行注释、`_dtype.pyi` 零字节）。包入口 `__init__.pyi` 走与运行时相反的「全量 import」路线以保类型精度，对上游无存根的 `_multiarray_umath` 用 `ModuleType` 兜底。策略选择本质是在类型精度、维护成本、模块是否公开三者间权衡。
