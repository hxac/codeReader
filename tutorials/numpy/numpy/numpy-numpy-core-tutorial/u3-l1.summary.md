本讲聚焦 numpy/core/ 三个特殊垫片（_internal.py、multiarray.py、_multiarray_umath.py）与包入口 __init__.py 的 pickle 向后兼容机制。核心是 pickle 的 find_class(module, name) 契约：反序列化靠「模块路径+名字」定位重建函数，且该路径被硬编码进字节流；NumPy 2.0 把 core 搬到 _core 后，旧地址必须仍能解析同名对象，否则旧 pickle 作废。关键结论是「pickle 反序列化等不起警告」：重建函数若走惰性 __getattr__ 会触发 DeprecationWarning，且 stacklevel 因调用链多帧而错位，故 _reconstruct、scalar、所有 ufunc、_dtype_from_pep3118 必须在垫片顶部 eager 绑定（写进 __dict__），绕开 __getattr__ 与警告。引入的关键术语：find_class 契约、eager 绑定、_reconstruct 两层历史绑定（_internal 的 Python 复刻兼容 <1.0 pickle、multiarray 的 C 实现兼容 ≥1.0）、_ufunc_reconstruct 守护 1.20 前 ufunc pickle、_multiarray_umath 用 isinstance 遍历批量绑定全部 ufunc。这些「免警告」机制区分特殊垫片与纯转发垫片，并为下一讲 _ARRAY_API「抛 ImportError」守卫铺垫。
