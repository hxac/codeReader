本讲聚焦 `numpy/core` 纯转发垫片在「判断属性是真缺失还是废弃但存在」时的两种写法，对照 `numeric.py`（哨兵写法）与 `umath.py`、`records.py`（None 写法）。两者共享同一骨架：在模块级 `__getattr__` 里惰性导入真模块，用 `getattr(真模块, name, 默认值)` 把「缺失」映射为拿到默认值，再据此分流——真缺失抛 `AttributeError` 且不报警，废弃但存在则调 `_raise_warning` 报弃用警告并正常返回。关键术语：**None 写法**用 `None` 作默认值、`if ret is None` 判断，无法区分「属性缺失」与「属性存在但值恰为 `None`」，是其唯一盲点；**sentinel 写法**用 `sentinel = object()` 作默认值，因哨兵全程序唯一，对所有取值（含 `None`）都安全。核心结论：numpy 因用 `is None` 而非真值判断，`0`、`False` 等假值不会被误判，仅值为 `None` 的真属性才会翻车；选型取决于被转发模块是否可能出现值为 `None` 的公开名字。两种写法的 `AttributeError` 分支逐字相同，统一以 `_raise_warning` 报警，其 stacklevel 细节留待 u2-l3 展开。
