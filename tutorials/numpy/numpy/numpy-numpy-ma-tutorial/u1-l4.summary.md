本讲解决「掩码数组创建之后如何读出内部信息」的问题，覆盖四个最小模块：三大属性、`filled`、`compressed`、以及 `getdata/getmask/getmaskarray` 函数族。核心结论：一个 `MaskedArray` 由「三件套」组成——`.data` 是把自身按 baseclass 看的普通 ndarray 视图（含坏值，非拷贝）、`.mask` 返回的是视图（保护 shape/dtype 但允许改值）、`.fill_value` 是按 dtype 懒加载的标量（int 默认 `999999`、float 默认 `1e20`）。引入的关键术语：`nomask`（即 `False`，表示无掩码的省内存单例）。读取「干净」数据有两条路：`filled()` 用填充值替换屏蔽位、保持原形状、返回纯 ndarray，无屏蔽位时零拷贝；`compressed()` 直接丢弃屏蔽位、按 C 顺序压成一维、返回纯 ndarray，且与仍带掩码的 `compress(condition)` 不同。模块级 `getdata/getmask/getmaskarray` 用鸭子类型对任意输入安全取值，其中 `getmask` 返回内部 `_mask` 的原始引用（可能为 `nomask`），而 `getmaskarray` 永远返回同形状布尔数组，适合后续做布尔索引。三者返回值都不再是 `MaskedArray`。
