本讲从结构视角解析 numpy.ma 包，回答「写下 `np.ma.xxx` 时 Python 去哪找」。核心结论：np.ma 的公开名字并非在 `__init__.py` 逐个定义，而是经 `from .core import *` 与 `from .extras import *` 两个星号导入「批发」搬运而来，对外清单由 `['core','extras']` 拼上两子模块各自的 `__all__` 组成。引入的关键术语：包与命名空间、re-export（再导出）、`__all__`（公开 API 清单）、类型桩（`.pyi`）。四模块分工：`core.py` 是地基（`MaskedArray`、`masked`/`nomask`、掩码 ufunc、fill_value），`extras.py` 是依赖 core 的上层工具（统计/集合/拼接），`mrecords.py` 提供字段级屏蔽记录数组，`testutils.py` 提供掩码感知断言。关键区分：仅 core 与 extras 被自动 re-export，mrecords、testutils 须显式 import——目录有文件不等于在 `np.ma.` 命名空间。`tests/` 与源码一一对应，由 PytestTester 提供 `np.ma.test()` 入口。`.pyi` 仅给静态类型检查器看、运行时不执行，因 `import *` 难追踪而手写 `__init__.pyi` 显式标注每个公开名归属。判别名字来源：运行时看 `__module__`，静态看 `.pyi`。
