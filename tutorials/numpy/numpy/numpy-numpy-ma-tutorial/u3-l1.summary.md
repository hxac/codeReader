本讲（专家层首讲）讲解 `MaskedArray` 的三个内部开关。硬掩码 `_hardmask` 为 `True` 时，赋值只能"加屏蔽不能减"：`__setitem__` 用 `mask_or` 合并掩码、用 `copyto(where=~mindx)` 跳过屏蔽位写数据，被屏蔽值无法被赋值还原；软掩码（默认）则直接覆盖 data 与 mask，赋确定值即解除屏蔽。`harden_mask`/`soften_mask`/`shrink_mask` 均就地修改并返回 `self`，可链式调用，`hardmask`、`sharedmask` 为只读 property。`shrink_mask` 把全 `False` 的真实掩码压缩回 `nomask` 单例，属纯内存优化、无语义影响，结构化 dtype 不可 shrink。`_sharedmask` 表示"掩码可能被他人共享"，为 `True` 时改前需先 copy，`unshare_mask()` 仅在确实共享时才复制、零开销。ufunc 钩子 `__array_wrap__` 无条件执行 `result._mask.copy()`，切断与输入掩码的引用以防结果就地修改污染输入。关键术语：硬/软掩码、`nomask` 单例、`mask_or`、`_sharedmask`、`unshare_mask`、构造时共享判定。
