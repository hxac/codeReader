本讲覆盖 `numpy.ma.extras.py`——这是建立在 `core.py` 三件套（data/mask/fill_value）之上的「上层工具带」，依赖 core 而不被 core 依赖，以 `ma.concatenate`（对 data 与 mask 分别拼接、保留屏蔽位）与 `unique`（输出恒为 MaskedArray）为地基。核心结论：`apply_along_axis` 只是沿轴切片、逐条喂给 func1d、再拼回的「搬运工」，本身不做掩码运算，屏蔽是否跳过完全由传入的 func1d 决定，实现上用 object dtype 数组中转、最后取最大 dtype 统一 cast 以防意外降级；集合运算家族（unique/union1d/intersect1d/setdiff1d/setxor1d/in1d/isin）共享「输出恒为 MaskedArray、屏蔽值视为同一个元素」的契约，多由「拼接+去重+排序后比较相邻」组合而成，其中 `in1d` 用稳定 mergesort、`isin` 是其保形版本（新代码推荐）；`compress_rowcols`（删掉含屏蔽值的整行列、返回更小 ndarray）与 `mask_rowcols`（把含屏蔽值的整行列染黑、返回同形 MaskedArray）方向相反，背后 N 维实现是 `compress_nd`；`clump_masked`/`clump_unmasked` 用异或 `mask[1:] ^ mask[:-1]` 找跳变点，把连续同质区段切成 `slice` 列表；`ndenumerate` 是 `np.ndenumerate` 的掩码版、默认跳过屏蔽元素；`_fromnxfunction_single/seq/allargs` 三套模板固化「对 data 与 mask 各跑一次同一函数」的套路，批量生成 vstack/hstack/atleast_1d 等；`mr_` 是 `numpy.r_` 的掩码版快速拼接器。
