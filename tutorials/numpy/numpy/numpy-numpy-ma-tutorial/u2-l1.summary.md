本讲聚焦 `numpy/ma/core.py` 中掩码的内部表示与构造。核心模块与结论：`nomask` 是代表「无屏蔽」的省内存单例（`MaskType(0)`），全库以 `is nomask` 做 O(1) 身份判断，避免为无屏蔽数组分配全 False 布尔数组；`make_mask_descr` 递归地把任意 dtype 的每个字段（含嵌套与子数组）替换为布尔，得到结构化数组 `_mask` 的合规格码 dtype；`getmask` 忠实返回内部 `_mask`（无屏蔽时为 `nomask`/`False`），而 `getmaskarray` 永远用 `make_mask_none` 展开为同形状全 False 数组，适合布尔索引；`make_mask` 按「0→False、非 0→True」规整掩码，`shrink=True` 时全 False 压回 `nomask`；`mask_or` 用 logical_or 合并两掩码，含 nomask 短路（共享视图）、dtype 一致性检查、结构化逐字段合并三条路径。设计主线是「单例加身份判断」省内存，配以安全展开函数在需统一形状时兜底。
