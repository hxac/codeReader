本讲厘清三个核心概念。字段（field）是结构化数组的一「列」，由名字、子dtype、字节偏移（及可选标题）组成，dtype.names 给字段名、dtype.fields 给 (子dtype, 偏移)。普通结构化数组本就支持字典式取列 arr['x']，是 ndarray 原生能力，无需 record array。取单条记录时，普通数组返回 numpy.void 标量（只能 arr[0]['x']），record array 返回 numpy.record 标量——record 是 nt.void 的子类，多了属性访问能力。recarray 是 ndarray 的子类，与标准数组「几乎一样」，区别仅在于属性查找字段并以 record 构造。arr.view(np.recarray) 不拷贝数据，仅换视图类型；__array_finalize__ 会把 void 结构化 dtype 自动提升为 record，使数组级 r.x 与标量级 r[0].x 同时生效。本讲只建立直觉，__getattribute__ 的字段查找实现细节留待进阶篇 u3-l2。
