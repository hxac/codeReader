本讲聚焦 `numpy/_core/records.py`，阐明一条贯穿始终的核心不变量：结构化 recarray 的 `dtype.type` 恒为 `numpy.record`。引入的关键术语与结论有：视图 `arr.view(np.recarray)` 不拷贝数据，仅改 Python 类型与（经钩子补救的）标量类型，与原数组共享内存；二元 dtype `np.dtype((np.record, descr))` 保留字段结构、偏移、itemsize 不变，只把标量从 `void` 换成 `record`，是整个 record 机制的原子操作（因 `record` 是 `void` 的子类而合法）；两个守门钩子分工协作——`__array_finalize__` 拦截 view/切片/取字段等派生操作，`__setattr__` 拦截直接给 `dtype` 赋值，二者都用同一个 `(record, ...)` 把 `void` 贴回 `record`。该提升单向且有条件：仅对「带字段的 void」结构生效，裸 void（如 `'V8'`）和非 void 标量（如 `int64`）不动，且 `view(ndarray)` 不可逆还原。此外 `np.rec.array(..., copy=...)` 的 `copy` 仅对 ndarray/recarray 输入生效，`True` 复制数据、`False` 共享内存，ndarray 分支靠末尾 `.view(recarray)` 统一贴身份。
