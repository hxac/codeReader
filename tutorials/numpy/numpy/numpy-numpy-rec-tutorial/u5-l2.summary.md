本讲讲透 record array 的 `__repr__` 打印逻辑，实现集中在 `numpy/_core/records.py`，打印引擎在 `arrayprint.py`。核心是 `recarray.__repr__` 按 `dtype.type` 分发前缀：`is record` 或非 void 子类走 `rec.array(...)`；void 子类但非 record（源码称 "strange games"）走 `array(...).view(numpy.recarray)`。正经 record array 打印前做 dtype 反转 `sb.dtype((nt.void, repr_dtype))`，把 record 剥回 void 使输出可粘贴回 `rec.array()`，这是 u5-l1 二元 dtype 的逆操作。数据串由 `array2string(separator=', ', prefix, suffix)` 渲染——逗号分隔源于 separator，续行缩进由 prefix 长度决定；多维空数组（size==0 且 shape≠(0,)）走特例打印 `[], shape=`。legacy 打印模式内部用整数表示，由 `_get_legacy_print_mode()` 取出，records.py 阈值统一 `<=113` 仅命中 `'1.13'`，此时 `record.__str__` 退化为 `str(self.item())`、`__repr__` 委托 `__str__`，且换行多一个尾随空格以逐字节兼容旧版 doctest。
