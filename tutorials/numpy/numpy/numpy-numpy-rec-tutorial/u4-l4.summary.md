本讲讲解 `np.rec.array` 作为 numpy.rec 的统一调度入口：自身几乎不构造数据，按 `obj` 类型把请求转发到 `recarray`/`fromstring`/`fromrecords`/`fromarrays`/`fromfile`。

执行分两段。阶段一做类型描述预处理，三选一，优先级 `dtype > formats > 延迟下推`：当 dtype 与 formats 都缺省时，把 `formats/names/titles/aligned/byteorder` 打包成 kwds 原样下推，让被调函数据实数据自动推断类型；而 `None`/`str`/文件因无法推断类型，被前置守卫强制要求给出 formats 或 dtype。

阶段二是七条顺序敏感的 `if-elif` 分支：None→`recarray`、bytes→`fromstring`、list/tuple 看首元素（tuple/list 走 `fromrecords` 行式，否则 `fromarrays` 列式）、recarray→view+copy、文件（具备 readinto）→`fromfile`、ndarray→view+copy 再 `.view(recarray)`、else 经 `__array_interface__` 兜底。

引入的关键术语：调度（dispatch）、视图与拷贝、kwds 延迟下推、首元素的行式/列式判定。要点：recarray 判定须先于 ndarray（避免被父类吞并）；str 无成功分支，二进制须用 bytes；recarray 与 ndarray 两分支唯一差别是末尾的 `.view(recarray)`，copy=False 时都可能返回与原对象共享内存的结果。
