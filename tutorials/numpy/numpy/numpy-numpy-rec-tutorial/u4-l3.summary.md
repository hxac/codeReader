本讲讲解 `numpy.rec.fromfile`，它是二进制 record array 读取的最后一块拼图，承接 u4-l2（fromstring）。核心结论：`fromfile` 与 `fromstring` 形参同构，但内存语义相反——fromstring 用 `buf=` 共享缓冲（零拷贝、可能只读），fromfile 则 `recarray(shape, descr)` 新分配数组再 `readinto` 拷贝，结果独立可写。实现位于 `numpy/_core/records.py`，涉及三个要点：其一，用 `hasattr(fd, 'readinto')` 一句探测区分「已打开文件对象」与「路径」，前者走 `nullcontext(fd)`（不负责关闭），后者走 `open(os.fspath(fd),'rb')`，统一进同一 `with` 块；其二，模块级辅助函数 `get_remaining_size` 用「记位→`seek(0,2)` 到尾→差值→`finally` 还原」的 seek/tell 三明治量出从当前位置到文件尾的剩余字节 `size`，要求文件支持随机访问；其三，`shape` 缺省 `(-1,)`，含一个 `-1` 时靠「连乘积为负」探测，按 `size // -shapesize` 推断未知维。两道安全闸：`nbytes>size` 抛 `ValueError`（字节不足），`readinto` 实读字节不足抛 `OError`（读取截断）。关键术语：readinto 探测、nullcontext、get_remaining_size、`-1` 自动推断、offset 相对 seek（whence=1）。其调度入口将在 u4-l4（array 总调度）串联。
