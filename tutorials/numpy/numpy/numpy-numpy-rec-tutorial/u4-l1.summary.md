本讲深入 `numpy.rec.fromrecords`，讲解如何从「行式」数据（外层每项一条记录，内层为各字段值）构建 record array，并与列式的 `fromarrays` 互为转置。核心结论：函数唯一的路径开关是 `if formats is None and dtype is None:`——二者均为 None 走**路径 A（自动探测）**，否则走**路径 B（直接构造）**。路径 A 先将数据铺成二维 object 数组，再逐列 `tolist()` 加独立类型推断，类型提升仅在列内发生，最后委托 `fromarrays` 装填，故较慢。路径 B 用 `sb.array(recList, dtype=descr)` 一次性构造，行是 tuple 走成功分支 `.view(recarray)`，行是 list 走降级分支逐行 `tuple()` 填写并发 `FutureWarning`。关键澄清：list-of-list 仅在指定 dtype/formats 时才告警，不指定时行为与 list-of-tuple 一致。本讲还引入二元 dtype `(record, dtype)`、`_deprecate_shape_0_as_None` 守卫（把 shape=0 译为 None，stacklevel=3 指向用户代码），为后续 `fromstring`/`fromfile`/`array` 调度讲义承接铺垫。
