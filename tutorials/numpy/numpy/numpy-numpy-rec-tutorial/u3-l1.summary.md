本讲聚焦 NumPy 中 `recarray` 类的构造机制，核心源码位于 `numpy/_core/records.py`。`recarray` 是 `ndarray` 的子类，`record` 是 `nt.void` 的子类，二者在内存布局上与普通结构化数组完全一致，区别仅在于支持属性式取列且标量类型被改写为 `numpy.record`。关键概念是「二元 dtype」`dtype((record, descr))`——它保持 `descr` 的结构不变，仅把 `.type` 从 `void` 换成 `record`，从而使数组级 `r.x` 与标量级 `r[0].x` 同时可用。

构造分两条路径：`recarray.__new__` 负责从头创建，先按 `dtype` 优先于 `format_parser` 解析出 `descr`，再按 `buf=None`（新分配，类似 `empty`）或 `buf` 已给（复用缓冲）两条内存路径，统一以 `(record, descr)` 调用 `ndarray.__new__`。对于绕过 `__new__` 的派生操作（如 `arr.view(recarray)`、切片），则由 `__array_finalize__` 收尾钩子补救：当且仅当满足「`is not record`、`issubclass(void)`、`names is not None`」三条 AND 条件时，调用 `ndarray._set_dtype` 就地把 `void` 提升为 `record`。类属性 `_set_dtype = None` 是向 NumPy C 层声明 dtype 变化交由此钩子处理。这两个方法分工合作，保证无论新建还是派生，`dtype.type` 都稳定为 `record`。
