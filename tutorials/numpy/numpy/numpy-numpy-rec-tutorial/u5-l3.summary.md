本讲是 numpy.rec 专题收尾篇，以官方测试套件 test_records.py 作「行为说明书」，拆解五大模块。核心结论：①字段名与 ndarray 属性/方法（shape、mean、field 等）同名时，属性式读取会被遮蔽而返回方法，但写入仍落到字段，须用 ra['字段名'] 或 ra.field() 作逃生口——这是读写路径不对称的直接后果；②fromstring 共享 bytes 得只读 recarray，写字段（含 setfield）会抛 ValueError，须先 .copy() 或改用 bytearray；③给定 dtype 时 fromrecords 可一次性解析二维乃至多维记录网格，外层形状即数组形状，给 names 则走较慢的自动探测；④recarray 与 record 标量复用 ndarray/void 的 pickle 机制，往返后数值相等、类型保持为 numpy.record，且标量是独立可写的新内存。异常区分：字段不存在抛 AttributeError，字段存在但内存只读写抛 ValueError，根因不同需区分。引入的关键术语有：遮蔽（shadow）、逃生口、只读缓冲、pickle 往返。
