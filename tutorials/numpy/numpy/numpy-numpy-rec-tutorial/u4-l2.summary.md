本讲聚焦 `numpy.rec.fromstring`：把一段 bytes-like 二进制缓冲按结构化 dtype 解释成 record array。它虽名含 "string"，却只吃 `bytes`/`bytearray`/`memoryview`，明确拒绝 `str`（Python 3 中 str 是 Unicode 文本，字节布局不可直接切分）——且拒绝并非来自函数体内的显式检查，而是 `str` 被当作 `buffer` 传给 `ndarray.__new__` 时由 C 层抛出 `TypeError`。

核心结论拆为三块。其一，硬约束：必须给出 `dtype` 或 `formats` 之一（二者同给时 `dtype` 优先），否则直接报错。其二，shape 自动推断：不指定或传 `-1` 时按 `(len(datastring) - offset) // itemsize` 计算记录数，尾部不足一条记录的余数被整除静默丢弃；`shape=0` 经 `_deprecate_shape_0_as_None` 译为 `None` 并发 `FutureWarning`。其三，零拷贝复用：最后一步 `recarray(shape, descr, buf=datastring, offset=offset)` 走 `__new__` 的 buf 分支，数组与原缓冲共享内存，并用 `(record, descr)` 二元 dtype 保证标量类型为 `numpy.record`；只读语义继承自缓冲——`bytes` 只读、`bytearray` 可写且改数组会写回原对象，与 `numpy.frombuffer` 一致。上层 `np.rec.array` 遇 `bytes` 输入即分发至此。本讲承接 u4-l1 `fromrecords`，为 u4-l3 `fromfile`（二进制文件）铺路。
