本讲深入 `format_parser` 流水线的第二步 `_setfieldnames`，讲透字段「上户口」的命名与查重细节。`names` 有三种合法形态（list / tuple / 逗号字符串），源码用 `type()` 精确判定而非 `isinstance`（故 list 子类被拒），非法类型抛 `NameError`（非 `TypeError`）。长度对齐靠截断 `names[:_nfields]`（给多静默丢弃）与补齐 `range(len(_names), _nfields)`（给少补 `f{i}`，且编号从停下的地方继续、非从 0）；`titles` 走相同截断逻辑但补 `None`。`find_duplicate` 基于 `collections.Counter` 返回重复元素列表，是公开 API（`np.rec.find_duplicate`），但 `format_parser` 只对 `names` 查重、不对 `titles` 查重——这是关键不对称，重名字段抛 `ValueError`，因为 `dtype.fields` 是字典、重名会导致字段静默丢失。`_byteorderconv` 是 14 项查表，把 `'b'/'l'/'s'/'>'` 等各种写法归一化；`_createdto` 只取 `byteorder[0]` 再查表（故 `'big'`/`'little'` 可用，未知字符抛 `KeyError`、空串抛 `IndexError`）。`newbyteorder` 区分绝对（`>`/`<`/`=`）与相对交换（`s`），只改字节序解释、不搬运内存字节。本讲仍只产出 `dtype`，装数据留待 u2-l3。
