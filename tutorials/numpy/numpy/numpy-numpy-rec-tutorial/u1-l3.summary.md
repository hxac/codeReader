本讲承接 u1-l2 的直觉，把 record array 落到「能跑」层面，覆盖三个最小模块。其一，`np.rec.fromrecords` 接收**行式数据**（list of tuple，外层行内层列），不指定 `dtype`/`formats` 时走自动探测路径：先转 object 二维数组再逐列单独推断类型，最后经 `.view(recarray)` 把普通结构化 ndarray 升级为 record array，此视图不拷贝数据。其二，`names` 参数支持逗号字符串或列表两种等价写法，缺省时自动补 `f0,f1,...`，并由 `find_duplicate` 拦截重名字段；record array 的属性访问 `r.字段名` 由 `recarray.__getattribute__` 实现——先查对象属性，找不到再去 `dtype.fields` 里按名取列。其三，`np.rec.array` 是**总调度入口**，按 `obj` 类型分发到 `recarray/fromstring/fromrecords/fromarrays/fromfile` 等函数；其中 list/tuple 的分支看首元素是否为 tuple/list 来区分行式（fromrecords）与列式（fromarrays）。本讲引入的关键术语包括：行式与列式数据、`dtype`/`formats`/`names` 三套类型描述参数、`format_parser`（组装 dtype）、`record` 标量类型、`copy` 参数控制内存共享。注意真实实现位于 `numpy/_core/records.py`，`numpy/rec/__init__.py` 仅是再导出垫片。二进制分支（fromstring/fromfile）与 list of lists 兼容分支留待 u4 展开。
