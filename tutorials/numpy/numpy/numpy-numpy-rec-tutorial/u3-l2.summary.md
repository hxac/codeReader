本讲剖析 `recarray` 的属性访问魔法（实现于 `numpy/_core/records.py`）。核心结论：`r.x` 与 `r['x']` 走两条不同代码路径——前者经 `__getattribute__` 两级回退，后者经 `__getitem__`。读写魔法的底层引擎是 `dtype.fields`（字段目录，元组取 `[:2]` 得 `(子dtype, 偏移)`）配合 `ndarray.getfield`/`setfield` 按字节偏移切/写内存。

关键术语：两级回退、字段名冲突、逃生口。`__getattribute__` 先用 `object.__getattribute__` 找对象属性，失败才查 `dtype.fields`，故字段名与 ndarray 内置属性（如 `shape`、`T`）同名时属性永远赢。`__setattr__` 先 `object.__setattr__`，若发现是字段名则撤销实例字典写入并改走 `setfield`，从而禁止动态创建与字段同名的实例属性；对 `dtype` 赋值时还会把 void 结构化类型提升为 record，延续「保 record 语义」主线。冲突字段及非法标识符字段名需用逃生口 `r.field(name)` 或 `r[name]` 访问，二者绕开属性查找。
