本讲精读 `defchararray` 中 `multiply`、`partition`、`rpartition` 三个「本地薄包装」。它们并非 `numpy.strings` 同名函数的纯再导出（`is` 判定为假），而是仅挂 `@set_module("numpy.char")`、不直接参与 NEP-18 分发的普通函数，真正的分发发生在被委托的 strings 层。三函数之所以能调到原版，靠顶部「别名导入」（如 `as strings_multiply`）在本地 `def` 同名之前先留一份原版引用，否则会陷入无限递归。归纳出两种包装手法：错误类型转换——`multiply` 用 `try/except` 把 strings 在乘数非整数时抛的 `TypeError` 翻译成 char 历史上的 `ValueError`，仅为向后兼容；结果重组——`partition`/`rpartition` 用 `np.stack(..., axis=-1)` 把 strings 返回的三元组（三个形状为 `S` 的数组）堆成单个形状 `S+(3,)` 的数组，故 char 版比 strings 版多出一条长度为 3 的末维。关键术语：薄包装、别名导入、适配器模式、单数组返回 vs 元组返回。
