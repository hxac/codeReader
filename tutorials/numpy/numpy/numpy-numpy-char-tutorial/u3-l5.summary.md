本讲从测试工程视角收束 u3 三层关系（char 门面→defchararray→strings 现代层），覆盖三个最小模块。其一，测试组织：`test_defchararray.py` 按功能簇切分为若干 `TestXxx` 类，规律是「凡直接用 `chararray`/`array`/`asarray` 的类都挂 `@ignore_charray_deprecation`，只用自由函数或 `_vec_string` 的类不挂」，这反向印证 u2-l1 的弃用边界——只有这三个名字进 `__DEPRECATED` 才报警。其二，弃用过滤标记：`ignore_charray_deprecation` 是模块级 `pytest.mark.filterwarnings` 常量，正则 `\w+ (chararray|array|asarray) \w+` 用 `re.match` 锚定起始匹配警告开头；它稳定的前提是告警文案写死为 "The chararray class is deprecated..."，与名字无关，三选一实为防御性写法。其三，`_vec_string` 是 C 层逐元素调度器，按 dtype 在 `bytes`/`str`/用户类型上按方法名查并调用，`numpy.strings` 中尚未 ufunc 化的函数（`upper`、`mod`、`decode` 等）仍走它；`TestVecString` 是唯一直接测该 C 入口的类，专覆盖其错误契约（`AttributeError`/`TypeError`/`ValueError`），且无需弃用过滤。测试即机制文档。
