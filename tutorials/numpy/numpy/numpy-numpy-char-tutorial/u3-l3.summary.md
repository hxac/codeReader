本讲精读 `numpy/_core/defchararray.py` 中的 `array()` 与 `asarray()` 两个工厂函数，它们是创建 `chararray` 的官方入口（均属 `__DEPRECATED`，访问时经 `numpy/char/__init__.py` 的 `__getattr__` 发 `DeprecationWarning`）。核心结论：`asarray` 仅是一行包装 `return array(obj, itemsize, copy=False, ...)`，二者唯一实质差异是默认是否拷贝。

引入的关键概念：**工厂函数**、**输入归一化**、**零拷贝**。`array()` 用五条 `isinstance` 分支归一化输入：①标量 `str/bytes`（含「切块」语义 `shape=len//itemsize`）；②`list/tuple` 转 `ndarray` 后穿透；③`character` 型 `ndarray` 主路径（`view(chararray)` 后推断与拷贝判定）；④`object` 型 `ndarray` 先 `.tolist()` 再穿透；⑤默认兜底（`narray` + `view`）。`itemsize`/`unicode` 为 `None` 时自动推断：`str`/`str_` → `unicode=True`、`bytes`/`bytes_` → `False`；object 与混合列表落到默认分支时 `unicode` 仍为 `None`，被当假值 → 产出 `bytes_`。

关键细节：`str_` 需做 `itemsize //= 4`，把字节数换算回 `chararray.__new__` 期望的字符数。拷贝由四条件判定（`copy`、宽度不等、`str_→bytes_`、`bytes_→str_`），`asarray` 的零拷贝承诺在 `bytes_` 数组上可靠兑现，但因「字符数 vs 字节数」单位不一致在 `str_` 数组上常失效。
