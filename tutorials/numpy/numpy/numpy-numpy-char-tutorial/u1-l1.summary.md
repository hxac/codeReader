本讲梳理 `numpy.char` 的定位与三层调用关系。核心结论：`numpy/char/` 目录本身没有任何业务逻辑，仅由 31 行的 `__init__.py` 充当「门面（facade）」，真正实现在 `numpy/_core/defchararray.py`，而后者通过 `from numpy.strings import *` 把多数函数委托给现代层 `numpy.strings`，形成 **char（门面）→ defchararray（实现/遗留）→ strings（现代 ufunc）** 的三层转发链。

本讲引入的关键概念：向量化字符串操作、模块级 `__getattr__` 的惰性再导出、`__all__`/`__doc__` 继承、`__DEPRECATED` 弃用集合（`chararray`/`array`/`asarray`）、`@set_module` 装饰器、本地包装函数（`multiply`/`partition`/`rpartition`）。关键事实：`chararray` 类及 `array`/`asarray` 工厂在 NumPy 2.5（2026-01-07）被弃用，访问时触发 `DeprecationWarning`；官方推荐改用普通 `str_`/`bytes_` dtype 数组搭配自由函数，或直接使用 `numpy.strings`。`defchararray` 内多数自由函数与 `numpy.strings` 是同一对象，仅 `chararray` 等少数名字为本地独有。
