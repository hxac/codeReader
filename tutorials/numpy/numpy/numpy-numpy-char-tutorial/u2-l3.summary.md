本讲揭示 `numpy.char` 六个比较函数 `equal`、`not_equal`、`greater_equal`、`less_equal`、`greater`、`less` 的统一实现模式：它们都由 `array_function_dispatch` 装饰，函数体仅一行 `return compare_chararrays(x1, x2, 比较码, True)`，彼此只差比较码字符串（`'=='`、`'!='`、`'>='`、`'<='`、`'>'`、`'<'`）。

真正干活的是 C 层 `compare_chararrays(a1, a2, cmp, rstrip)`：它把比较码映射成 Python 富比较常量 `Py_EQ`/`Py_NE`/`Py_LE`/`Py_GE`/`Py_LT`/`Py_GT`，校验字符串 dtype 后调用底层 `_umath_strings_richcompare`。该函数的 `__module__` 被改写为 `numpy.char`，`chararray` 的六个比较运算符也直接委托给这些自由函数。

核心结论是「比较前先剥尾部空白」语义：六个函数固定传 `rstrip=True`，故 `np.char.equal('aa', 'aa ')` 返回 `True`，与 `numpy.equal` 的 `False` 相反。这是为兼容已停更的 numarray 而保留的历史包袱，C 源码注释已标记其「应当弃用」，也是官方推荐新代码改用 `numpy.strings`（不剥空白）的根本原因。本讲引入的富比较、比较码、rstrip 开关等术语将供后续讲义承接。
