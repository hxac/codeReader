本讲覆盖两个最小模块：NEP-18 分发与模块归属。`array_function_dispatch` 是 NumPy 落实 NEP-18 的入口，把 implementation 包成 C 类 `_ArrayFunctionDispatcher` 并登记进全局集合 `ARRAY_FUNCTIONS`，使带 `__array_function__` 的第三方数组（Dask 等）能接管这些函数。调用时先经 dispatcher 取「相关数组参数」，用 `_get_implementing_args` 挑出实现该协议的类型按优先级询问，无人接管才回退到 implementation。char 的 `_binary_op_dispatcher` 只返回 `(x1, x2)`，因比较码与 rstrip 开关是写死常量；`verify_matching_signatures` 强制 dispatcher 与 implementation 签名一致。`set_module` 则就地改写 `__module__` 为 `'numpy.char'`（对类额外存 `_module_source`），仅影响文档归属与内省，不参与分发判定。两条路径殊途同归：比较函数经 `module=` 偏函数、`multiply`/`partition`/`chararray`/`array` 等经 `set_module`，最终 `__module__` 均为 `numpy.char`，把定义在 `_core` 的实现伪装成 char 公开成员。
