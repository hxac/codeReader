本讲锚定 `numpy/_core/defchararray.py`，拆解 `chararray` 作为 ndarray 子类「最像 Python 字符串」的一面：运算符重载与方法委托。核心结论有三：其一，六个富比较运算符（`__eq__`/`__ne__`/`__ge__`/`__le__`/`__gt__`/`__lt__`）各写一行，委托给同名自由函数 `equal`/`not_equal`/`greater_equal`/`less_equal`/`greater`/`less`，最终落到 C 层 `compare_chararrays(..., rstrip=True)`，故比较会**先剥尾部空白**，返回布尔 ndarray，不包裹。其二，算术运算符 `+`/`*`/`%` 提供元素级字符串语义（拼接、重复、旧式格式化）；关键不对称在于 `__add__` 直接返回（`add` 是 ufunc，靠 `__array_wrap__` 自动 view 回 chararray），而 `__mul__`/`__mod__` 需显式套 `asarray(...)` 以稳定返回 chararray；`__rmod__` 返回 `NotImplemented` 致反向 `%` 抛 TypeError。其三，几十个字符串方法采用「方法委托」设计；源码顶部 `IMPLEMENTATION NOTE` 定下规矩：返回字符串数组的方法须用 `asarray` 包回 chararray 以支持链式调用，返回 int/bool 的方法（`find`/`count`/`isalpha`）则刻意不包，否则撞 `__array_finalize__` 的字符串 dtype 校验。此处 `asarray` 作用是「重新认定类型」而非复制。
