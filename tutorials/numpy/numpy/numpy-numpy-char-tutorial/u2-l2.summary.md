本讲揭示 `numpy.char` 的数十个字符串函数绝大多数并非自实现，而是 `defchararray` 顶部用 `from numpy.strings import *` 整体再导出（re-export）而来——`import *` 绑定同一对象，故 `np.char.capitalize is np.strings.capitalize` 为真。`defchararray` 命名空间由三组 import 拼装：从私有 `numpy._core.strings` 捞 `join`/`split`、从 `numpy.strings` 整批导入、用 `strings_multiply` 等别名留存原版。引入「`is` 身份」与「strings 是否有此名」两个判据，把 char 的名字分四类：①纯再导出（同源）；②本地覆盖（六个比较函数，同名异体、char 版先剥尾部空白再比较）；②′本地包装（`multiply`/`partition`/`rpartition`）；③char 独有（`chararray`/`array`/`asarray`/`compare_chararrays`）及③′私有捞取（`split` 等）。底层机制为「后定义覆盖先导入」。`numpy.strings` 是现代 ufunc 层：公开包仅 2 行、再导出实现模块 `numpy._core.strings`，后者 `__all__` 把行为尚未「结晶」的 `join`/`split` 等注释在外，这正是 char 必须从私有模块捞它们的原因。官方推荐新代码改用 `numpy.strings`，甩掉 `chararray` 等历史包袱。
