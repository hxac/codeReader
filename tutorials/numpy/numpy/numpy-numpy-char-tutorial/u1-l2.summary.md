本讲围绕 `numpy/_core/defchararray.py` 下沉讲解字符串在 NumPy 内存中的真实形态，承接上一讲的三层模型。核心结论：`numpy.char` 面向**定宽字符串数组**，其 dtype 家族包括 `str_`（`<U`，Unicode，UCS-4，每字符 4 字节）、`bytes_`（`|S`，每字符 1 字节）、`object_`（元素为 Python 对象指针，`itemsize≈8`）以及二者共同抽象基类 `character`。引入的关键术语与概念：dtype、itemsize（每元素字节数）、定宽、UCS-4、字节序前缀。关键公式 `itemsize = 字符数 × c`（`str_` 取 4，`bytes_` 取 1），故同字符数下 `str_` 字节数为 `bytes_` 的 4 倍。源码精读了两处：`chararray.__new__` 用 `(dtype, itemsize)` 把类型与**字符宽度**打包传给底层 ndarray，并对 `itemsize` 强制 `int()` 以规避 NumPy 整数陷阱；`array()` 工厂在 `itemsize is None` 时对 `str_` 输入执行 `itemsize //= 4`，把底层字节数换算回字符数——这正是「除以 4」注释的来源。`object` 数组无固定宽度，需 `tolist()` 后由底层按最长元素定宽。最后强调：`chararray`/`array`/`asarray` 在 NumPy 2.5 已弃用，新代码应直接用 `str_`/`bytes_` 普通数组搭配 `numpy.char` 或 `numpy.strings` 自由函数。
