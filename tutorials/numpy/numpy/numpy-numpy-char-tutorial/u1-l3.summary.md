本讲梳理 numpy.char 的公共 API 全貌并完成首次向量字符串操作。核心结论：numpy.char 只是门面，其 53 个公共名字（`__all__`）完全由 `numpy._core.defchararray` 决定，`__init__.py` 仅整体转交。这些函数可归为比较、查询、变换、拼接、对齐、修剪、拆分、编码、工厂九大类，绝大多数是 Python str/bytes 方法的元素级（element-wise）数组版。来源上，多数函数经 `from numpy.strings import *` 再导出，真正函数体在 `numpy/_core/strings.py`；仅 `multiply`、`partition`、`rpartition` 在 defchararray 本地包装。本讲精读 `upper`、`add`、`center`、`multiply` 四例，确立返回 dtype 三规则：保持原宽（`upper`，长度不变）、宽度求和（`add`，实为 C 级 ufunc）、取最大宽度（`center`/`multiply`）。其中 `multiply` 是 `numpy.strings.multiply` 的薄包装，仅把乘以非整数的 `TypeError` 翻译为 `ValueError`。本讲引入的关键术语：元素级、定宽数组、itemsize、dtype 重算、再导出委托、`chararray` 弃用。结论强调新代码应使用普通 `str_`/`bytes_` 数组配合自由函数，避免已弃用的 `chararray`。
