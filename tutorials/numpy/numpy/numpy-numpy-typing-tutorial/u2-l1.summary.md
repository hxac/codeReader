本讲拆解 NumPy 公共别名 `ArrayLike` 的构成（实现藏在私有 `numpy/_typing`，经公共壳 `numpy/typing` 转发）。其顶层为 `Buffer | _DualArrayLike[np.dtype, complex | bytes | str]`，由四块积木拼成：缓冲区、带 `__array__` 的对象、内置标量（`complex` 隐含 `int/float/bool`）、以及表达任意深度嵌套的递归协议 `_NestedSequence`。引入的关键术语有三：`_SupportsArray`，把运行时 `__array__` 鸭子协议翻译成带 `DTypeT` 参数的 `@runtime_checkable` 泛型协议，使 dtype 信息可沿类型传递；`_NestedSequence`，用协变 TypeVar 与递归返回类型接纳 `list`/`tuple` 及多层嵌套；`_DualArrayLike`，DTypeT/BuiltinT 双轨参数化引擎，同一引擎同时服务最宽松的公共 `ArrayLike` 与最严格的私有 `_co` 收窄别名。核心取舍：`ArrayLike` 刻意避开 object 数组——生成器、裸对象、dict 运行时能跑出 object 数组却遭类型系统拒绝；object 数组支持被隔离到私有 `_ArrayLikeObject_co`，需要时以 `# type: ignore` 或 `Any` 逃生。
