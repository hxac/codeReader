本讲拆解 NumPy 类型子系统的「公共壳 + 私有实现」两层分层，涵盖三个最小模块：`__all__`（公共 API 契约）、`numpy.typing`（极薄公共壳）、`numpy._typing`（私有聚合中枢）。核心结论：`numpy.typing` 对外仅暴露四个名字 `ArrayLike`、`DTypeLike`、`NDArray`、`NBitBase`，由 `__all__` 圈定；公共壳靠一行 `from numpy._typing import …` 搬来全部名字，再用 PEP 562 的模块级 `__dir__`/`__getattr__` 收窄对外暴露（连真实存在的 `test` 都被藏起）；真正的实现集中在私有包，其 `__init__.py` 用 8 个 `from ._子模块 import X as X` 块汇聚上百个内部别名。引入的关键术语：再导出与 `import X as X`（PEP 484 显式再导出标记）、公共/私有命名约定、`@set_module` 装饰器（把私有实现「化妆」为公共产物，如 `NBitBase.__module__` 报告 `"numpy.typing"`）、双轨制（类型检查器读 `.pyi`、运行时跑 `.py`，两者 `__all__` 一致但内容不同）。
