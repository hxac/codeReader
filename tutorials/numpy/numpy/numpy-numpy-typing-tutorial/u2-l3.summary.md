本讲从单元2的输入侧别名（ArrayLike、DTypeLike）转向输出侧，讲清如何标注「NumPy 数组」。核心结论：numpy.ndarray 是接收两个类型参数的泛型类——形状（int 元组，如 tuple[int, int]）与元素类型（np.dtype[...] 子类型），二者正交独立、可分别已知或未知。公共别名 NDArray 展开为 np.ndarray[_AnyShape, np.dtype[ScalarT]]：把形状钉死为彻底留白的 _AnyShape，只把元素类型 ScalarT（上界 np.generic）暴露给用户参数化，适合只关心元素类型的通用签名。私有文件 _shape.py 给出三个形状别名：_Shape＝tuple[int, ...]（忠实于运行时形状）、_AnyShape＝tuple[Any, ...]（完全留白）、_ShapeLike＝SupportsIndex | Sequence[SupportsIndex]（能转成形状的输入，如 axis 参数）。前两者描述「是什么形状」，后者描述「能变成什么形状」。需精确跟踪形状时用 PEP 695 type 语法自定义带形状的别名。关键限制：类型系统只跟踪维数与元素类型，无法承载运行时具体数值（如恰好 3×3 或 Literal int），故静态类型比运行时更粗。
