本讲围绕 NumPy 私有文件 `numpy/_typing/_nested_sequence.py` 中的 `_NestedSequence` 协议展开，拆解四个最小模块：递归类型主体、协变 TypeVar `_T_co`、旧式泛型协议语法、`TYPE_CHECKING` 双分支。

核心结论是：`_NestedSequence` 用**递归类型**刻画「任意深度嵌套的序列」——在 `__getitem__`/`__iter__`/`__reversed__` 的返回类型里出现 `_T_co | _NestedSequence[_T_co]`，使一层、多层嵌套在类型上「坍缩」为同一个 `_NestedSequence[T]`，这是 `np.array([[1.0]])` 这类输入能被精确静态标注的关键。其类型参数 `_T_co` 声明为**协变**（`covariant=True`，`_co` 后缀约定），成立前提是它只出现在输出位置，故 `__contains__`/`count`/`index` 的输入参数特意写成 `object`/`Any` 而非 `_T_co`；协变规则 `B<:A ⇒ Nested[B]<:Nested[A]` 让 `Sequence[bool]` 可匹配 `_NestedSequence[int]`。

本讲引入的关键术语包括：递归类型、变型（协变/逆变/不变）、`_co` 协变约定、旧式 `Protocol[_T_co]` 与新式 PEP 695 `class Proto[T](Protocol)` 两套等价泛型协议语法、`TYPE_CHECKING` 双分支（静态侧见带 `default=Any` 的 PEP 696 `typing_extensions.TypeVar`、运行时见不带 `default` 的 `typing.TypeVar`，兼顾裸用合法、免运行时三方依赖与老 Python 兼容）。已知限制：该协议不能与函数级 TypeVar 搭配以传出叶子类型。它是 `ArrayLike`（经 `_DualArrayLike`）及若干 `.pyi` 桩文件描述嵌套输入的核心拼图。
