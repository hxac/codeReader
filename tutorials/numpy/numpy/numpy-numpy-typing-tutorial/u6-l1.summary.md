本讲聚焦 NumPy 类型测试的最小模块 `numpy/typing/tests/test_typing.py` 及其 `data/` 夹具目录，核心结论是：NumPy 通过 `mypy.api.run(...)` 把 mypy 当作库在 pytest 进程内调用，对四类夹具批量检查。整套测试默认关闭，由环境变量 `NPY_RUN_MYPY_IN_TESTSUITE` 与「是否安装 mypy」两道闸门控制。

引入的关键术语：`run_mypy` 模块级 autouse fixture（只跑一次，要求 mypy 不崩溃即 stderr 空、exit∈{0,1}）、`OUTPUT_MYPY` 按文件归类的错误字典、四类夹具目录的期望语义——`pass`（真 `.py`，期望零错误且运行时可执行）、`fail`（`.pyi` 桩，用 `# type: ignore[code]` 钉死必有错误）、`reveal`（用 `assert_type` 固化返回类型）、`misc`（仅冒烟、不逐条断言）。强调 `reveal_type`（产出 `note:` 行，供人看、被框架丢弃）与 `assert_type`（产出 `error:` 行、可机器校验）的区别——目录虽叫 reveal，现代夹具几乎全用 `assert_type`。`mypy.ini` 的 `strict` 与 `ignore-without-code` 是 fail 夹具精确预言错误码的前提。`get_test_cases` 配合 `pytest.param(id=文件名)` 生成可读用例，`test_pass`/`test_reveal` 以「文件不在 `OUTPUT_MYPY` 即通过」为捷径，`test_code_runs` 用 importlib 对 pass 做纯运行时回测。
