本讲围绕 `numpy/_typing/_nbit_base.py` 及其桩文件，讲解 `NBitBase` 精度刻度体系。`NBitBase` 把数值精度抽象为一组有序刻度（`_128Bit→…→_8Bit`），作为标量泛型类（如 `np.floating[T]`）的**不变（invariant）类型参数**，使精度提升（promotion）能在静态层面被表达；它仅服务类型检查，运行时是空标记类，方向上越靠近根越高精度，子类表示「更窄精度」，是对继承语义的非常规复用。封闭性靠三道机制维持：`@final`（静态禁继承）、`@set_module`（改写 `__module__` 对齐公共 API）、`__init_subclass__`（运行时按名字白名单拒绝非法子类）。桩文件比运行时多出纯类型实体 `_256Bit`、`_80Bit`，体现「静态可比运行时更丰富」的双轨哲学。自 NumPy 2.3 起 `NBitBase` 弃用：静态用 `@deprecated`、运行时用模块级 `__getattr__` 懒发 `DeprecationWarning`，推荐改用以标量类为上界的 `TypeVar` 或 `@overload`。引入的关键术语：精度刻度、不变类型参数、精度提升、三道封闭保护、桩独有精度实体，供后续讲义承接。
