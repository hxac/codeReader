本讲是手册首篇，覆盖两个最小模块：PEP 484 与 numpy.typing，仅做全局认知。核心结论：静态类型检查（mypy/pyright）在运行前按注解推演，与运行时行为可能不一致，注解不影响运行时执行。numpy.typing 是 NumPy 自 1.20 引入、遵循 PEP 484 的官方类型子系统，对外仅暴露 4 个公共名字——ArrayLike、DTypeLike、NBitBase、NDArray（由 `__all__` 声明），实现藏于私有的 numpy._typing，呈「公共壳 + 私有实现」分层。核心理念是「类型化 API 比运行时更严格」：主动拒绝生成器造 object 数组、字段字典 dtype、timedelta64 当 signedinteger 子类、recarray 中 dtype 与 formats 混用等合法但不推荐写法；数值精度被当作不变（invariant）泛型参数处理。时效提醒：NBitBase 自 2.3 弃用，改用以标量类为上界的 TypeVar 或 @overload。本讲为后续讲义承接了静态类型检查、运行时、类型别名、`__all__`、公共壳/私有实现、不变泛型参数等术语。
