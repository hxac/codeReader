本讲义解析 numpy.polynomial 的核心抽象基类 ABCPolyBase，揭示 Polynomial/Chebyshev/Legendre 等六大便捷类接口高度一致的根源。ABCPolyBase 是「mixin + abc.ABC」抽象基类，自身不可实例化，集中承载算术、求值、拟合、求根、打印等共用外壳行为。

核心契约是 12 个用 `@staticmethod @abc.abstractmethod` 声明的虚函数（`_add`/`_sub`/`_mul`/`_div`/`_pow`/`_val`/`_int`/`_der`/`_fit`/`_line`/`_roots`/`_fromroots`，分属算术、微积分求值、构造拟合三组），外加 3 个抽象属性 `domain`/`window`/`basis_name`。设计精髓为「父类管流程、子类管算法」：外壳在内部调用 `self._xxx` 传入系数数组并取回新系数，具体基函数算法由子类决定。

子类以 `_add = staticmethod(polyadd)` 一行直接赋值完成绑定，构成「外壳（_polybase.py）→ 函数式 API（polynomial.py）→ polyutils」三层委托链，命名遵循「前缀=基、后缀=功能」规律。子类漏实现任一虚函数时，`abc` 会在实例化时抛 `TypeError` 点名缺失方法；而 `__array_ufunc__ = None`（退出 NumPy ufunc）与 `__hash__ = None`（声明不可哈希）则同为把潜在错误转为清晰报错的护栏。
