本讲深入 `numpy.polynomial.polynomial` 的「创建」与「算术」两组函数式 API，并把它们与上一讲的 `polyutils` 通用工具相联系。创建方面：`polyline(off,scl)` 构造一次多项式，斜率为 0 时退化为常数；`polyfromroots(roots)` 经 `pu._fromroots` 返回首一多项式，内部用 `polyline` 生成一次因子后组织成平衡二叉乘法树（深度 O(log n)）配对相乘。算术方面：`polyadd`/`polysub` 是 `pu._add`/`pu._sub` 的薄委托，套路为「`as_series` 规整 → 以长数组为容器对齐 → `trimseq` 收尾」；`polymul` 特化为 `np.convolve`（多项式乘法即系数卷积），`polymulx` 乘以 x 时对零多项式做边界保护。除法有两套等价实现：通用模板 `pu._div` 靠注入 `mul_f` 构造移位基逐次消元、服务六大基；幂级数特化 `polydiv` 归一化除式后下标原地消元更高效。`polypow` 注入 `np.convolve` 委托 `pu._pow`，后者为迭代连乘（非快速幂），函数式默认 `maxpower=None` 不限指数，便捷类的 `maxpower=100` 才是安全阀。通篇印证「能复用就复用、有性能优势就特化」的设计哲学。
