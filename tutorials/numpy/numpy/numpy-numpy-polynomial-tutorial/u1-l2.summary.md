本讲梳理 numpy/polynomial/ 子包的目录分层与导入方式。源码按单向依赖分层：底层 polyutils.py 提供共享工具（as_series、trimseq、getdomain 等）；其上 _polybase.py 定义抽象基类 ABCPolyBase，集中承载六大便捷类的算术、打印、domain/window 等共同行为；六个家族模块各含一套函数式 API 与一个便捷类，便捷类继承 ABCPolyBase，通过「虚函数委托」（如 _add=staticmethod(polyadd)）把操作交给本模块的同名函数；__init__.py 作门面，把六大类提到顶层命名空间。核心结论：便捷类 API（np.polynomial.Polynomial，官方首选）经 __all__ 在顶层导出，而 polyval 等函数式 API 仅留在子模块、必须带路径；两条路径取到的是同一个类对象，只是导航深浅不同。此外 set_default_printstyle 通过改写类属性 ABCPolyBase._use_unicode 一次性切换全部类的打印风格，np.polynomial.test 则是一个 PytestTester 对象。
