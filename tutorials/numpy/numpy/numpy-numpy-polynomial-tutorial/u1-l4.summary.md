本讲聚焦 `polynomial.py` 的函数式 API 与内置常数，揭示便捷类只是糖衣、函数式 API 才是模块本体。核心覆盖三个最小模块：一是 `poly*` 命名规律——公开函数全以 `poly` 开头（类用大写 `Polynomial`），后缀表功能（`add/val/der/fit/roots/vander/line/fromroots`）与维度（`2d/3d/grid`），「前缀=基、后缀=功能」的规律通用于六大正交族；`__all__` 列出 29 个公共名字，既是 API 清单也是文档目录。二是四个内置常数：`polydomain=[-1.,1.]`（浮点默认区间）、`polyzero=[0]`、`polyone=[1]`、`polyx=[0,1]`（整型，为兼容 `Decimal`）；`polyline(off,scl)` 是一次多项式系数工厂，`polyx`/`polyone` 是其特例，且便捷类默认 `domain`/`window` 即 `polydomain` 的拷贝。三是 `polytrim=pu.trimcoef` 别名所揭示的三层委托链：便捷类（`_polybase.py`）→函数式 API（`polyadd` 等）→通用工具（`pu._add` 等）；委托是「能复用就复用、有性能优势就特化」的实用策略，`polyval`（Horner）、`polymul`（`np.convolve`）、`polydiv`（手写长除法）即为特化。关键术语：首一多项式、模块级别名、薄委托、虚函数绑定（`_add=staticmethod(polyadd)`）。
