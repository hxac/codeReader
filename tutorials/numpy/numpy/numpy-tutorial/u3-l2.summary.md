本讲讲清 ndarray 形状操作的统一原理：只改写 shape 与 strides 两个小数组，底层 data 缓冲区不动，故尽量返回视图。基于「三件套」（data/shape/strides）与偏移公式 Σ i_k·strides[k]，视图即共享 data 的新数组对象。reshape 改各维长度而不改元素顺序，免拷贝判定由 _attempt_nocopy_reshape 完成——目标序与缓冲区布局一致则返回视图，否则按 copy 策略拷贝或报错，-1 由 _fix_unknown_dimension 推断。transpose 改维的顺序，永远返回视图：PyArray_Transpose 复用 data 指针，仅按排列重排 dimensions/strides，再 UpdateFlags 重算连续性标志。swapaxes、matrix_transpose(mT)、moveaxis、rollaxis 均归约为一次 transpose；moveaxis 在 Python 层用 normalize_axis_tuple 构造排列。atleast_* 用 newaxis 切片拼装视图，证明任何改写 shape/strides 的手段都算形状操作。引入 C/F 连续、视图与拷贝、轴置换等术语，承接上一讲索引机制，衔接下一讲 strides 技巧。
