本讲剖析 NumPy multiarray 模块全貌，涉及三个文件：纯 Python 薄壳 multiarray.py、C 扩展入口 multiarraymodule.c、装配与诊断层 _core/__init__.py。核心结论：_multiarray_umath 才是真正的 C 扩展，numpy._core.multiarray 仅是复刻旧命名空间以保向后兼容的薄壳，通过星号再导出、改写 __module__ 为 numpy、对少数函数绑定 __array_function__ 调度器来暴露名字——带调度装饰器者是「调度器＋C 实现」复合体，其余即 C 函数本体。引入关键术语：PyMethodDef 函数注册表与调用约定（METH_FASTCALL/METH_VARARGS/METH_KEYWORDS）、PEP 489 多阶段初始化（PyInit 仅建骨架，_multiarray_umath_exec 注册类型并导出常量）、PyCapsule 封装的 _ARRAY_API/_UFUNC_API 两张 C-API 表，以及「集中注册、分散实现」模式（set_typeDict 注册与实现分居 multiarraymodule.c 与 descriptor.c）；此外 _core/__init__.py 还承担区分未编译/不兼容的导入失败诊断与 set_typeDict 灌注 dtype 字典两项职责。
