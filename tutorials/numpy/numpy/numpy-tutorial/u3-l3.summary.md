本讲承接「形状操作只改 shape/strides、不动 data 缓冲区」的结论，把视图推到极致：用任意 shape 与 strides 构造视图。覆盖 `_stride_tricks_impl.py` 的 `as_strided`、`sliding_window_view`、`broadcast_to` 及 C 层 `getset.c` 属性 getter。核心结论：视图与拷贝的根本区别是是否共享 data 缓冲区，判据为 `np.shares_memory`，构造视图仅复制 shape/strides 两个小数组、代价 O(ndim)；`as_strided` 伪造 `__array_interface__` 挂于 `DummyArray` 保活 base，默认不查越界且绕过 `_view_is_safe`，最自由也最危险；`sliding_window_view` 以「裁剪原轴、追加窗口轴、strides 追加窗口轴原 stride」公式造安全只读窗口；`broadcast_to` 借 nditer 将长度 1 的轴 stride 置 0 实现广播视图，结果只读且常不连续。关键术语：视图、拷贝、strides（可取 0 或负）、`__array_interface__`、DummyArray、byte_bounds、自重叠、stride=0 广播、`NPY_ARRAY_WRITEABLE`。
