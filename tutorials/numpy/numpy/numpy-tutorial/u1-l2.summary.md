本讲讲解 NumPy 源码如何变成可 `import` 的扩展模块。核心结论:NumPy 的 PEP 517 构建后端是 `meson-python`,链路为 pip → meson-python → Meson(描述)+ Cython(.pyx→.c)→ Ninja(调度)→ C/C++ 编译器 → 扩展模块。关键术语:构建后端、构建依赖(区别于运行依赖)、`vendored-meson`(仓库自带 Meson 以锁版本,子模块未初始化时 `.spin/cmds.py` 会在最早期报错)。两条安装路径:终端用户用 `pip install`(装 PyPI 预编译 wheel),开发者用 `spin`(命令在 `pyproject.toml` 的 `[tool.spin.commands]` 注册、实现在 `.spin/cmds.py`,每组都「扩展」底层 `spin.cmds.meson.*`);`spin build` 产物落 `build-install/` 并自动设 `PYTHONPATH`,`spin test` 默认 `-m 'not slow'`,`spin bench` 封装 asv 支持 `--compare`。BLAS/LAPACK 选择集中在 `meson.options`(`blas`/`allow-noblas`/`use-ilp64`/`blas-order`),PyPI wheel 强制 `-Dallow-noblas=false` 以保证链接真 BLAS。涉及最小模块文件:pyproject.toml、meson.build、meson.options、.spin/cmds.py、INSTALL.rst、building_with_meson.md。
