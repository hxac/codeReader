本讲剖析 NumPy 索引机制。所有 `arr[key]` 都从同一 C 入口 `array_subscript` 进入，经 `prepare_index` 把 key 解析为 `npy_index_info[]` 并产出 `HAS_*` 位掩码（INTEGER/SLICE/ELLIPSIS/NEWAXIS/FANCY/BOOL）进行分流。核心结论：基础索引（整数/切片/省略号/None）只改写 shape/strides/data 指针，由 `get_view_from_index` 返回视图；花式索引（整数数组、非整组布尔）经 `PyArray_MapIterNew`+`mapiter_get` 逐元素搬运返回拷贝；整组布尔掩码（形状与被索引数组相同）是特例，走 `array_boolean_subscript` 返回 1-D 拷贝。`PyArray_MapIterSwapAxes` 配合 `consec` 字段决定连续/分离的高级索引在结果中的轴位置。赋值 `arr[key]=val` 共用同一套解析，最终落到 `mapiter_set`；PR #31975 修复了其搬运跨缓冲区块（默认 8192 元素）时，类型转换错误被 `return 0` 静默吞掉的缺陷，改由调用方 `array_assign_subscript` 的 `goto fail` 正确抛出。Python 层辅助器 `np.ix_`（开放网格造笛卡尔积）、`np.r_`/`np.c_`（基于 `__getitem__` 的拼接构造器）、`np.s_`/`np.index_exp`（切片语法对象化）亦一并介绍。
