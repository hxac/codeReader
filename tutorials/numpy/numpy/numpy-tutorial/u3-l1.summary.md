本讲深入 NumPy 的 `arr[key]` 实现，核心是区分两大类索引并判定返回视图还是拷贝。所有形式的索引都从同一个 C 入口 `array_subscript` 进入：它先用 `prepare_index` 把 `key` 解析成 `npy_index_info[]` 并返回一个 `HAS_*` 位掩码（`HAS_INTEGER`/`HAS_SLICE`/`HAS_ELLIPSIS`/`HAS_NEWAXIS`/`HAS_FANCY`/`HAS_BOOL` 等），再按掩码用 `==` 判定唯一形态、用 `&` 判定是否含某类索引来分流。**基础索引**（整数、切片 `start:stop:step`、省略号 `...`、`None`/`np.newaxis`）走 `get_view_from_index`：只改写 `shape`/`strides`/`data` 指针、不搬数据，结果恒为视图（纯整数走 `get_item_pointer` 返回标量）。**花式索引**（整数数组、非整组布尔）走 `PyArray_MapIterNew`+`mapiter_get`，新分配内存逐元素搬运，结果恒为拷贝；单个一维整数数组有 `mapiter_trivial_get` 快速通道。**整组布尔掩码**（形状与原数组相同）是特例，走 `array_boolean_subscript` 返回 1-D 拷贝。多个高级索引连续时由 `PyArray_MapIterSwapAxes`+`consec` 决定结果轴位置。引入的关键术语：视图/拷贝、基础/花式索引、位掩码分流、`strides` 偏移公式、子空间。Python 层辅助器：`ix_`（用 `(1,..,n,..,1)` 开放网格造笛卡尔积）、`r_`/`c_`（基于 `__getitem__` 的构造器）、`s_`/`index_exp`（把切片语法转成可复用 `slice` 对象），以及 `nonzero`/`flatnonzero`/`argwhere`/`normalize_axis_tuple`/`indices`。这些概念为后续形状操作、`as_strided` 与广播（stride=0）讲义承接。
