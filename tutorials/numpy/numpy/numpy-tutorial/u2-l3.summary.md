本讲区分「类型转换」与「类型提升」：前者用 astype/can_cast 显式指定目标 dtype，由五级 casting（no/equiv/safe/same_kind/unsafe）评判转换安全性；后者在运算时自动求能同时容纳双方的最小公共 dtype，函数 promote_types/result_type/min_scalar_type 均为「Python 壳 + C 核心」，真正算法在 convert_datatype.c 的 PyArray_PromoteTypes/PyArray_ResultType，按 kind 顺序（bool<整数<浮点<复数）取高者、精度取足，两段式求上确界（如 int64+uint64→float64）。关键术语 NEP 50 弱类型（weak type）：Python 标量参与运算只贡献 kind、不贡献精度（C 层退化为抽象 DType、descriptor 置空），故低精度数组与 Python 标量运算保持数组 dtype，值装不下时报 OverflowError 或溢出告警。NumPy 2.2 起取消提升策略开关，精度配置改由 seterr/errstate 承担；_dtype.py 的 _kind_to_stem 将 kind 映射为名称词根。本讲为后续 ufunc 类型解析奠基。
