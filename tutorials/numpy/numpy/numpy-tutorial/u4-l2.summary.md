本讲义下沉到 C 层拆解 `np.ndarray`。结构体 `PyArrayObject_fields` 关键字段为 `data`（`char *` 数据指针）、`nd`/`dimensions`（形状）、`strides`（字节步长）、`descr`（dtype）、`flags`（属性位掩码）、`base`（视图保活，支撑零拷贝）；元素地址由偏移公式 `data + Σ iₖ·strides[k]` 决定，`PyArray_GETPTR1..4` 是其逐字实现。直接字段访问已弃用：推荐用法下 `PyArrayObject` 为仅含 `PyObject_HEAD` 的不透明结构体，须用 `PyArray_DATA`/`PyArray_STRIDES` 等内联访问宏以保 ABI 稳定；`flags` 经 `PyArray_CHKFLAGS` 做「全部命中」位判定（`C_CONTIGUOUS`/`OWNDATA`/`ALIGNED`/`WRITEABLE`）。官方「Memory model」将数组归纳为「内存块 + 步长解释」，步长以字节为单位、不必是元素尺寸整数倍（广播 stride=0、结构化字段视图），0 维时 strides/dimensions 为 NULL。核心结论：转置、切片、广播不复制数据，只改写 `dimensions`/`strides`；本讲为 u4-l3（ufunc）与 u8-l2（C-API）地基。
