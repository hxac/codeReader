本讲只读三个文件（`numpy/__init__.py`、`numpy/_core/__init__.py`、`numpy/lib/__init__.py`），讲清 NumPy 顶层命名空间的「装配」机制。核心结论有三：其一，顶层子包严格分层、单向依赖——`_core` 是用 C 实现的底层核心（`ndarray`/`ufunc`/`dtype`，经 C 扩展 `_multiarray_umath` 提供），`lib` 是构建在 `_core` 之上的纯 Python 工具集，`linalg`/`fft`/`random`/`ma`/`polynomial`/`f2py` 则是面向领域的功能子包，全部依赖 `_core`。其二，`numpy/__init__.py` 通过再导出（re-export）与 `__all__` 并集，把数百个名字汇聚到 `np.` 命名空间。其三，懒加载（lazy import）借助模块级 `__getattr__`（PEP 562）实现：`linalg`/`fft` 等仅在首次访问时才 `import`，加快启动并按需付费。正因如此，`np.array` 作为直接构造 C 层数组的底层原语必然来自 `_core` 而非 `lib`。
