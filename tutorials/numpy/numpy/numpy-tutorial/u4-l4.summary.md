本讲承接 u4-l3 的 ufunc 内部实现，主题是归约（reduction）——把数组沿某轴压扁成聚合结果，以及累积（accumulation，保留中间过程的归约）。核心结论：归约本质就是「反复套用同一个二元 ufunc 的内层循环」，`sum=add.reduce`、`prod=multiply.reduce`、`max=maximum.reduce`、`any/all=logical_or/and.reduce`。

全讲拆成三个最小模块。其一 `_methods.py`（方法层）：C 方法尽量薄，算法落在 Python 中间层；`sum/prod/max/min` 只是一行参数透传，`mean/std/var` 则是「先 `add.reduce` 求和，再用 `_count_reduce_items` 的计数做 `true_divide`」的复合运算。其二 `fromnumeric.py`（顶层函数分发）：`_wrapreduction` 用 `type(obj) is ndarray` 判定——ndarray 直通 `ufunc.reduce` 绕过 `_methods`，非 ndarray 优先调对象自己的方法以尊重子类与第三方数组（NEP-18 精神）；positional-only 签名与 `_NoValue` 哨兵服务热路径。其三 `reduction.c`（C 引擎）：`PyUFunc_ReduceWrapper` 用 `NpyIter` 驱动，靠 `result_axes` 落实 keepdims（轴删除或留长度 1），靠 `get_reduction_initial` 取幺元（identity）——加法为 0、乘法为 1、最大值无幺元，故空数组 `sum` 返回 0 而 `max` 报错。引入的关键术语包括 identity、reorderable、wheremask、`try_reduce_contiguous` 快速路径、accumulate；`axis=None` 展开为全部轴，不可重排序 ufunc 拒绝多轴归约。
