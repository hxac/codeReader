本讲深入 NumPy ufunc 的 C 层实现，核心对象是 `PyUFuncObject` 结构体：它是一张「类型→循环」分发表加执行框架，由 `functions[]`/`types[]`/`ntypes` 描述已注册的逐元素 C 内层循环，由 `nin`/`nout`/`identity`/`type_resolver`/`vectorcall` 描述执行方式；Python 层的 `nin`、`types`、`identity` 等属性即对这套字段的薄封装 getter。一次调用是五步流水线：vectorcall 入口 → `ufunc_generic_fastcall` 主控（参数解析与重载检查）→ `promote_and_get_ufuncimpl` 挑循环 → `resolve_descriptors` 定 descr → 内层循环执行。旧式类型解析（`PyUFunc_DefaultTypeResolver`）在 `functions[]`/`types[]` 上线性搜索，`xx->x` 模式 ufunc（如 `np.add`）走 `PyUFunc_SimpleUniformOperationTypeResolver` 快速通道，用 `PyArray_ResultType` 按提升后的结果类型而非输入字面类型选循环。新式 ArrayMethod 机制（NEP 43）以 `_loops` 字典做 DType 多重分派、用 promoter 处理提升、用 `_dispatch_cache` 缓存，旧表经 `legacy_promote_using_legacy_type_resolver` 包装桥接，新旧共存。`identity` 枚举（Zero/One/None/ReorderableNone）决定归约可行性，为后续归约、SIMD 分发、自定义 dtype 讲义铺垫。
