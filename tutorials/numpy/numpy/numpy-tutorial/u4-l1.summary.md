本讲从源码层面讲解 NumPy 的「通用函数」（ufunc）：它是对数组逐元素作用、自带广播的 C 对象，并非 Python 函数。`np.add`、`np.sin` 都是 `numpy.ufunc` 实例，由 C 扩展 `_multiarray_umath` 构建时批量生成，`numpy/_core/umath.py` 仅做再导出，`a + b` 之所以触发 `np.add`，是 `_PyArray_SetNumericOps` 把它注册成了 ndarray 的 `__add__`。

执行模型为「逐元素 + 广播」：给定输入 dtype，ufunc 经类型解析从 `functions[]`/`types[]` 表中选出一条 C 内层循环（kernel，即 `PyUFuncGenericFunction`），在广播后的形状上按 strides 推进指针完成计算。

每个 ufunc 在 C 层是 `PyUFuncObject` 结构体，其字段直接映射到 Python 属性 `nin`、`nout`、`nargs`、`ntypes`、`types`、`identity`。其中 `identity`（加法为 0、乘法为 1、减法为 None，对应头文件枚举 `PyUFunc_One`/`PyUFunc_Zero`/`PyUFunc_None`）决定能否做 `reduce`。

`np.frompyfunc(func, nin, nout)` 把任意 Python 可调用对象包装成 ufunc：它在 `umathmodule.c` 中用载体 `PyUFunc_PyFuncData` 持有该函数，唯一内层循环 `PyUFunc_On_Om`（定义于 `loops.c.src`）逐元素回调该函数，并由 `object_ufunc_type_resolver` 强制所有输入输出 dtype 为 `NPY_OBJECT`——故其输出永远是 object dtype，价值在接口统一而非提速。

此外，`reduce`/`accumulate`/`outer` 等方法挂在 ufunc 类型的 `ufunc_methods` 表上，本质是反复应用同一条 ufunc；本讲指出新一代机制为 `ArrayMethod`，留待后续讲义深入。
