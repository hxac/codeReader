本讲区分 Module（模块）与 Component（组件）：组件是引擎流水线零件、注册进 Registry；模块是「一批组件的打包注册单元 + initialize/finalize 生命周期钩子」、注册进 ModuleManager。RimeModule 为跨 C 边界 POD（含 module_name/initialize/finalize/get_api），ModuleManager 用 map_ 按名查找、loaded_ 指针集合做幂等去重，LoadModule 只触发 initialize、不查找。库加载时借 `__attribute__((constructor))`/`.CRT$XCU` 段自动登记，`RIME_REGISTER_MODULE` 约定 rime_<name>_initialize/finalize 函数名，`RIME_REGISTER_CUSTOM_MODULE` 额外可挂 get_api。模块组是「假模块」：default=core+dict+gears，deployer=core+dict+levers。加载链为 RimeInitialize→LoadModules(kDefaultModules)→default.initialize→依次 core/dict/gears→组件进 Registry→StartService。四个内置模块分工：core 配置系统、dict 词典与 db 后端、gears 流水线四类组件、levers 部署任务；方案 YAML 的 engine 段只采购 gears 中的组件名。
