本讲讲解 librime 的方案切换器与开关配置模型。Switcher 多重继承 Processor 与 Engine，是跑在主引擎内的迷你引擎：作为 Processor 插在处理器链首位截获 hotkey，作为 Engine 复用组词机制生成方案/开关菜单。需区分 engine_ 指主引擎、context_/schema_ 指切换器自身。激活时由 set_active_engine(this) 实现 active_engine 旁路，前端经 Session::context() 读到切换器菜单，切换器对多数键返回 kAccepted 独占按键。菜单候选由 schema_list_translator 与 switch_translator 产出，选中经 OnSelect 触发 SwitcherCommand::Apply，以命令模式统一「选方案/翻开关」两类动作。Switches 是 switches 段的只读解析视图，按有无 name 键区分 toggle（单 option）与 radio group（互斥多 option），归一成 SwitchOption。开关状态有两独立机制：reset 字段经 InitializeOptions 控制启动/换方案时复位（优先级高于恢复值），save_options 名单经 user.yaml 的 var/option/* 控制用户切换后的持久化（RestoreSavedOptions 恢复、Switch::Apply 写回）。
