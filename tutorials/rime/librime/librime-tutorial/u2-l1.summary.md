本讲拆解 librime 对「一次按键」的内部表示，涉及 `key_event.h`/`key_event.cc` 与 `key_table.h`/`key_table.cc`。核心结论：`KeyEvent` 用两个 `int` 表示一次按键——`keycode`（键值/keysym，复用 X11 定义）记录「按了哪颗键」，`modifier`（位掩码）记录「当时状态」，二者相互独立，故大写字母键名本身不带 Shift 位，表达 Shift 修饰须显式写 `Shift+a`。modifier 中 `kShiftMask`/`kControlMask`/`kAltMask`/`kLockMask`/`kSuperMask`/`kReleaseMask` 各占一位，组合用按位或、检测用按位与，`kModifierMask` 圈定所有有效位。`key_table.h` 提供「名字↔数值」四个双向查表函数，查不到修饰名返回 `0`、键名返回 `XK_VoidSymbol`，是 `Parse` 的判错依据。`repr()`/`Parse()` 完成文字编解码：单字符走字节值快路径，多字符按 `+` 切分（前缀为修饰名、末段为键名）；`KeySequence` 是按键序列，用 `{...}` 转义不可打印或组合键以保证往返一致。C API `process_key(keycode, mask)` 在边界把两个 int 封装成 `KeyEvent`，交会话与引擎流水线处理（u2-l2 承接）。
