本讲聚焦 librime 流水线中的 Translator（翻译器）族，衔接 u6-l3 的 Segment/Segmentation：Segmentor 把输入切段打 tag，Translator 则把「带 tag 的段」翻译成候选词。Translator 基类契约极简，仅一个纯虚 `Query(input, segment) -> an<Translation>`，靠 `segment.HasAnyTagIn(tags_)` 做 tag 门控，`name_space_`（来自 Ticket 处方串 `klass@alias` 的 `alias`）决定读方案哪一节配置。共享积木集中在 `translator_commons`：`TranslatorOptions`（delimiters、tags 默认 `{"abc"}`、enable_completion、preedit/comment_formatter 等）、`Phrase`（`DictEntry` 的薄包装候选）、`Sentence`（多词整句候选）。两大主力翻译器 `ScriptTranslator`、`TableTranslator` 均多重继承 `Translator + Memory + TranslatorOptions`：前者面向音码，先建 SyllableGraph 把输入切成所有音节序列再沿图查词典，支持纠错、造句、用户词优先；后者面向形码，把输入当直接编码做字符串前缀匹配，`LazyTableTranslation` 按需分批扩展，并带 `UnityTableEncoder` 为用户自造词生成编码。辅助翻译器族包括 `reverse_lookup_translator`（双词典反查）、`punct_translator`（标点）、`echo_translator`（quality=-100 兜底）、`history_translator`（历史复用）、`schema_list_translator`/`switch_translator`（用 `dynamic_cast<Switcher*>` 门控，仅切换器引擎生效）。核心思想是数据驱动装配：注册一次、实例多次，靠 `@alias` 覆盖命名空间与认领不同 tag 实现同方案多翻译器并存。
