本讲讲解 librime 的 C API 入口。核心结论：库只导出唯一符号 `rime_get_api()`，它返回指向静态单例 `RimeApi` 的指针，所有能力都是这张「方法指针表」里的槽位，借此兼顾版本兼容与符号卫生。`RimeApi` 按功能分组：setup（全局环境/日志）、initialize/finalize（启停引擎与 Service）、会话管理、input、output（`get_*`/`free_*` 成对，体现「谁分配谁释放」的内存契约）、options、方案、config；会话级方法均以 `session_id` 为主键。最关键的机制是「自版本化结构体」：每个跨边界结构体首字段为 `data_size`，由调用方填入 `sizeof(T)-sizeof(int)`，库用 `RIME_STRUCT_HAS_MEMBER`/`RIME_PROVIDED` 判断某字段是否存在，从而安全地新增字段而不破坏老前端；`RIME_API_AVAILABLE` 把同样思路用于方法指针探测。主干调用顺序为 setup→initialize→create_session→process_key：setup 偏全局环境，initialize 才加载模块并 StartService，process_key 将 C 层的 keycode+mask 包装成 `KeyEvent` 交给会话。本讲引入的关键术语：`rime_get_api`、`RimeApi` 方法表、`data_size` 自版本化、`RIME_STRUCT_INIT`/`HAS_MEMBER`/`PROVIDED`、`RIME_API_AVAILABLE`、`RimeTraits`、`session_id`、`KeyEvent(keycode, mask)`。
