本讲拆开 u3-l1 中被视为黑盒的 `Context::composition_` 第一层，讲解 `Segmentation` 与 `Segment` 两个数据结构（源码在 `segmentation.h`/`segmentation.cc`）。`Segmentation` 是 `public vector<Segment>` 外加一份输入串引用 `input_` 与维护方法，用一串首尾相接的 `Segment`（第 i 段 `end` == 第 i+1 段 `start`）无遗漏、无重叠地覆盖整段输入。`Reset(new_input)` 采用增量策略：只找新旧输入首个不同位置 `diff_pos`，丢弃受影响尾部段而保留已确认前缀。`Segment` 用 `[start, end)` 半开区间定位，字段含 `length`、`tags`、`menu`、`selected_index`、`prompt`；`Clear()` 重置候选但保留位置。关键术语：`Status` 四态状态机 `kVoid/kGuess/kSelected/kConfirmed` 按成熟度递增数值化，故源码多用 `status >= kSelected` 区间判断；`Reopen` 让已选段回退编辑、`Close` 处理部分选中拆段并写入 `partial` tag。`tags`（`set<string>`，可叠加多值）是 segmentor 与 translator 的契约：常见 `abc`/`raw`/`punct`/`phony`/`xxx_prefix`/`xxx_suffix`/`partial` 分别由对应 segmentor 写入。`AddSegment` 三规则——左对齐、贪心最长、等长合并 tag——把各 segmentor 表态归并；`Forward` 腾新格、`Trim` 撤空格，由 `Engine::CalculateSegmentation` 驱动，并以「end 未推进即 break」防死循环。候选如何挂载到 `Segment::menu` 留待 u3-l3。
