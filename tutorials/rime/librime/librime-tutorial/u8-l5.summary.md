本讲讲运行期 `Dictionary` 如何把音节图翻译成词条候选。`Dictionary` 是 Prism（音节索引）与 Table（码表）的门面组件，同名产物经 `weak_ptr` 缓存共享以避免重复 mmap。它对外暴露三接口：`Lookup(SyllableGraph)` 正向查询（音码主力）、`LookupWords(字符串)` 反查（形码/反查翻译器，经 Prism 把字符串翻成 syllable_id 且只留 `kNormalSpelling`）、`Decode(Code)` 把 syllable_id 还原成拼写。正向链路：`Table::Query` 用「位置 + TableQuery 游标状态」队列沿音节图做 BFS，在 `indices` 分叉处 `Access` 取词、`Advance` 下钻，产出按 `end_pos` 分组的 `TableQueryResult`；`lookup_table` 再翻成 `DictEntryCollector`，并用 `match_extra_code` 把 TailIndex 长词（>3 音节）的 `extra_code` 接回音节图确定真实结束位置。`DictEntryCollector = map<size_t, DictEntryIterator>`，迭代器由若干 `Chunk` 组成，采用 `partial_sort` 增量排序 + 拉模型 `Peek`，权重为 `e.weight - log(1e8) + credibility`（对数空间逐边累加）。引入的关键术语：`syllable_id`（Prism 与 Table 间的硬通货）、`end_pos`（按消费字节分桶）、`credibility`/`quality_len`、`remaining_code`、拉模型。本讲是 u6-l4 翻译器族调用的底层接口，亦为 u8-l6 用户词典的姊妹篇（后者沿用同一 collector 模型，底层换为 LevelDB 并动态更新权重）。
