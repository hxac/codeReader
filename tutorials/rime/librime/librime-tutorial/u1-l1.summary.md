本讲是《librime 学习手册》首篇，通过精读 README 建立项目全局地图，不涉及 C++ 细节。核心结论：librime 是一个用 C++17 写的**跨平台、模块化、可扩展的输入法引擎**，只负责「按键 → 文字」的核心计算（接收按键、查词典、生成候选、决定提交），不画界面。与之分工的是**前端**（Squirrel/macOS、Weasel/Windows、ibus-rime/Linux 等），负责显示候选窗与对接 OS 输入法框架。引入的关键术语包括：引擎/前端边界、音码与形码、**输入方案（schema）**（用 YAML 写的 DSL，换方案即换输入法，引擎不变）、**拼写代数（Spelling Algebra）**（派生变体拼写的机制）、OpenCC（繁简/字形转换）、和弦输入。依赖区分为**构建依赖**与**运行时依赖**（后者为前者子集，gtest 仅构建期、glog 可选），核心库含 Boost、LevelDB（用户词典）、marisa（音节索引）、OpenCC、yaml-cpp。扩展分两层：前端（对外接入平台）与**插件**（对内增强引擎，如 librime-lua），另有 plum 等配套工具。
