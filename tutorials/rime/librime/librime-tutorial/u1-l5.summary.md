本讲以 `tools/rime_api_console.cc` 这个最小但完整的「终端前端」为样例，把入门层 u1 串成可运行整体，拆为五个模块：①程序生命周期 `setup→set_notification_handler→initialize→start_maintenance(+join)→会话循环→destroy_session→finalize`；②按键主循环——靠 `find_session`+`create_session` 实现会话自愈，用 `simulate_key_sequence` 把一行文本当作一串按键喂入；③三种输出结构 `RimeCommit`（提交文字）/`RimeStatus`（模式标志）/`RimeContext`（preedit+候选菜单），`get_*` 与 `free_*` 必须成对调用，否则内存泄漏；④会话级交互 API：`select_schema`、`select_candidate_on_current_page`、`set_option`、`candidate_list_*` 迭代器、`sync_user_data`；⑤通知回调 `on_message` 的「推」式事件（schema/option/deploy）约定与 `RIME_API_AVAILABLE` 版本兼容用法。核心结论：console 覆盖了 `RimeApi` 最常用槽位，是学习 C API 的最佳样例，并为 u2 的 KeyEvent、Service/Session、Schema、Engine 深入奠定直觉。
