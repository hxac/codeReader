本讲精读 `Menu` 类（源码在 `src/rime/menu.cc`），它是 Translator 与前端之间的**惰性缓冲层**：内部维护一条候选流 `result_` 和一份已确认候选数组 `candidates_`，做到「显示多少、算多少」。核心设计是**共享候选区**——`candidates_` 同时被 `MergedTranslation`（按引用）和各 `Filter`（按指针）读写，使归并选举与去重都能感知「已吐出的候选」。装配阶段由 `Engine::TranslateSegments` 驱动：`AddTranslation` 把翻译器塞进 k 路归并器，`AddFilter` 以 `result_ = filter->Apply(result_, &candidates_)` 将链头层层包裹，在**装配期**就嵌套成一棵洋葱式装饰器树（如 `uniquifier(simplifier(Merged(...)))`）。消费阶段中，`Prepare(count)` 是拉模型的心脏：循环里 `Peek` 取候选（可能被 Filter 吞空不入列）、`Next` 无条件推进，直至凑够有效候选或链耗尽。`CreatePage(page_size, page_no)` 在切片前按需 `Prepare`，用「链耗尽且本页贴末尾」双条件判定 `is_last_page`；页号与高亮索引由调用方按 `selected_index / page_size`（整除）与 `selected_index % page_size`（取模）计算，最终经 C API 翻译为前端读到的 `RimeMenu`。关键术语：拉模型、共享候选区、洋葱式装饰器树、`is_last_page`。
