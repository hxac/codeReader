本讲讲解 librime 会话管理骨架。Session（会话）为每个输入焦点提供隔离状态，SessionId 取自 Session 对象地址（uintptr_t），0 表无效，是所有会话级 C API 的主键。Service 为全局单例，用 SessionMap（id→shared_ptr<Session>）统一管理会话，disabled() 在未启动或部署维护时拒绝会话操作。生命周期含 CreateSession（造并续命）、GetSession（查并续命）、DestroySession（点名删）、CleanupStaleSessions（5 分钟过期批量删）。Session 用 the<Engine>（unique_ptr）独占一个引擎，按键转发给它，状态经 active_engine() 透传（支持切换器旁路）。引擎与前端间有两条信号链：提交链为「拉」模型（引擎 sink_→Session::OnCommit 累加进 commit_text_→前端 get_commit 取走并 Reset）；通知链为「推」模型（引擎 message_sink_→Service::Notify 加锁→前端 notification_handler），部署器事件以 session_id==0 标记为全局。本讲只立骨架，Engine 内部流水线与 Context 结构留待后续讲义。
