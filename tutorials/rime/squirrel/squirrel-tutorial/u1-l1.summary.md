本讲建立「前端 vs 引擎」这一贯穿全册的认知地基：Squirrel（鼠鬚管）是 Rime 输入法在 macOS 上的官方前端发行版，本身不做中文转换，按键→候选词的转换交给引擎 librime 完成；Squirrel 经 InputMethodKit（IMK）与 macOS 对接，程序入口为 Main.swift。关键术语：前端（收键盘事件、画面板、上屏）与引擎（做转换）；输入方案（schema，按键到汉字的规则集合，如朙月拼音）。Squirrel 依赖三个以 git submodule 形式引入的组件：librime（引擎 C++ 库，经 dylib 调用）、plum/東風破（方案与词库的配置管理器）、Sparkle（自动更新框架），另有 OpenCC（简繁转换）。授权为 GPL v3（强 copyleft），仅支持 macOS 13.0+。Rime 以「共享 librime 引擎 + 各平台专属前端」实现跨平台：macOS 用 Squirrel、Linux 用中州韻、Windows 用小狼毫。主干按键链路为：NSEvent → Squirrel 收事件 → rimeAPI.process_key（前端→引擎分界点）→ 取回候选 → 画面板/上屏。
