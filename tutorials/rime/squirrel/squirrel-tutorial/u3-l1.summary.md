本讲深读 `sources/SquirrelConfig.swift`，它是 librime C 配置 API 之上的 Swift 门面，把零散的 `config_open/schema_open/config_get_*` 等函数包装成 `getBool/getDouble/getString/getColor/getAppOptions` 五个类型化接口，返回值统一用 Optional 表达「键不存在」。核心结论有三：其一，base config（`squirrel.yaml`）与 schema config（输入方案）是父子回退关系，`getXxx` 在当前配置未命中时递归调 `baseConfig?.getXxx`，使用方另有「整配置级回退」（方案无 `style` 节则整体退回 base config）；其二，四个 `getXxx` 共用「查缓存 → 调 librime 写缓存 → 回退」三段式模板，回退命中的值缓存在 base config 实例，故共享 base 能被多个 schema 复用以压减 C 调用；其三，颜色字节序为 Rime 历史约定的 `0xAABBGGRR`（6 位省略 Alpha 默认 255），解析后按 `display_p3` 或 `sRGB` 色空间构造 NSColor。`getAppOptions` 用迭代器三段式遍历 `app_options/<bundleID>`，返回的布尔开关经 `set_option` 写入引擎，驱动 `ascii_mode`、`no_inline` 等运行时行为。本讲引入的门面、父子回退、缓存、色空间等概念将由 u3-l2（配置文件结构）、u3-l3（主题加载）、u3-l4（亮暗双加载）承接。
