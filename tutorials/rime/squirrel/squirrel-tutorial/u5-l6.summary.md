本讲为第五单元收尾，聚焦 Squirrel 的本地化与持续集成，含四个最小模块。一是 xcstrings 字符串目录本地化：`Localizable.xcstrings` 经 `NSLocalizedString` 供代码查表取文案，`InfoPlist.xcstrings` 由系统读 `Info.plist` 时自动替换（如输入源可读名）；源语言为 en，程序化键须为含英文在内的所有语言提供译文，缺失则回退源语言。二是 SwiftLint：`.swiftlint.yml` 以 `included: sources` 限扫自家代码、阈值规则（圈复杂度等）约束规模，三类抑制注释（`:next` 豁免下一行、`disable`/`enable` 成对豁免、单规则豁免）显式声明「故意违规」。三是三条 GitHub Actions 工作流（commit/PR/release）共享「固定 Xcode→checkout 子模块→swiftlint→action-build.sh→periphery scan」骨架，差异仅在触发器与产物出口（artifact/草稿 Release/nightly）。四是 `action-build.sh` 作 CI 构建总入口，注入 `ARCHS`、`BUILD_UNIVERSAL` 产出 universal 二进制，调 `action-install.sh` 下载预编译依赖（librime、Sparkle），最后 `make` 落到 `release`/`package`/`archive` 目标。
