本讲义剖析 Squirrel 前端的默认配置 `data/squirrel.yaml`，将其拆为四个最小模块：顶层全局项（`config_version` 部署哨兵、`keyboard_layout` 西文布局回退、`chord_duration` 和弦节奏、`show_notifications_when` 通知开关、`status_icon` 状态栏图标）、`style` 全局样式节（布局、字体、几何、候选格式）、`preset_color_schemes` 配色仓库、`app_options` 应用级特化。核心结论有三：其一，同一个文件的键被分散在 `SquirrelApplicationDelegate` 与 `SquirrelInputController` 两个文件、三个生命周期阶段（`loadSettings`/`activateServer`/和弦定时器）读取；其二，颜色遵循 Rime 历史字节序 `0xAABBGGRR`（6 位省略 Alpha 默认 255），`native` 配色是跟随系统亮暗外观的占位方案；其三，`app_options` 按 bundle ID 把布尔选项 `set_option` 写进引擎，`handle` 每次进门检测应用切换并重灌，且只接受布尔值。本讲引入 `config_version` 哨兵、配色方案叠加覆盖、`candidate_format` 的具名占位符 `[label]/[candidate]/[comment]`、`inline_preedit` 与 `no_inline`/`inline` 的联合判定等概念，为 u3-l3 主题加载与 u3-l4 亮暗特化做词汇铺垫。配置改动需同步到 `~/Library/Rime/squirrel.yaml` 并重新部署方生效。
