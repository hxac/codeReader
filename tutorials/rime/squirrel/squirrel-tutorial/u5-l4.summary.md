本讲集中讲解 Squirrel（Swift）与 librime（C API）之间的桥接约定，主角是 `sources/BridgingFunctions.swift`。包含四个最小模块：①`rimeStructInit()` 定义在 `DataSizeable` 协议扩展上，用 `memset` 把整块 C 结构（含 padding、指针字段）清零，并按 librime 的 ABI 约定回填 `data_size` 这个「尺寸书签」字段——引擎据此判断结构体版本与可安全访问的字节范围，缺一则拒绝填充，Swift 默认 `init()` 做不到这两点。②`setCString` 用 `strdup` 把 Swift 字符串拷成 C 堆长期副本写进字段（因 `withCString` 临时缓冲出闭包即失效），所有权归字段，覆盖同字段时自带 `free` 回收；它与 `get_*`/`free_*`（引擎 malloc、前端替引擎释放）构成一对镜像方向的所有权约定。③自定义 `?=` 可选赋值运算符（`AssignmentPrecedency`），右侧非 nil 才赋值，两个重载分治非可选左值（配置覆盖默认）与可选左值（刷新 weak client），永不把左值清成 nil。④`NSRange.empty`（location=NSNotFound）作「无区间」哨兵、`RimeStringSlice.asString` 严格按 `.length` 截取避免误读截断点后的文本、`NSPoint` 几何运算符简化二维向量运算。关键术语：`rimeStructInit`、`data_size`、`setCString`、`?=`、`NSRange.empty`、`asString`，供后续讲义（如 u5-l3 的 `@convention(c)` 回调、u5-l5 打包）承接。
