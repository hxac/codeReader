本讲精读 Squirrel 主循环 `handle(_:client:)`，即 IMK 键盘事件回调入口。系统只送 `recognizedEvents` 登记的两类事件：`.keyDown` 与 `.flagsChanged`。每次进门先做 session 自愈（失效即重建，连重建也失败则透传），用 `?=` 刷新弱引用 `client`，切 App 时重载应用级选项，再按 `event.type` 三路分发。`flagsChanged` 分支把修饰键状态变化翻译成带按下/松开标记的按键送入引擎：capslock 因锁状态时序错位需异或回拨 `kLockMask`，并靠「释放 insert 队首、按下 append 队尾」的 buffer 实现释放优先，无变化时直接吞掉。`keyDown` 分支一进门即放行所有带 Command 的快捷键，其余按键经 `osxKeycodeToRime` 翻译后送入 `processKey`。返回值 `handled` 默认 `false`，true 表吞掉、false 表透传，统一在 `return handled` 收口；普通按键最终吞放由 librime `process_key` 决定，映射不出 Rime 键码者一律透传。处理末尾统一调用 `rimeUpdate()` 刷新面板。
