本讲聚焦 `SquirrelInputController`（继承 `IMKInputController`）的生命周期与会话管理。核心结论：一个 controller 实例与一个 librime session 一一对应——`init` 经 `createSession()` 申请会话（`RimeSessionId`，0 为无效哨兵），`deinit` 经 `destroySession()` 释放。

关键术语：client（`IMKTextInput`，目标应用文本框）必须用 `weak` 弱引用以防与 IMK 形成引用循环泄漏，并借助项目自定义 `?=` 运算符（仅非 nil 才赋值）稳健刷新；marked/commit text 即 `setMarkedText`/`insertText`。

生命周期链为 `init → activate ⇄ deactivate → deinit`：`activateServer` 做上岗准备（覆盖键盘布局、重置 preedit、刷新状态栏）；`deactivateServer` 三步收尾——隐藏面板、`commitComposition` 提交原始输入、置空 client，是避免「悬挂组合（stranded composition）」的关键。`commitComposition` 作「安全网」，用 `get_input` 取原始码原样上屏再 `clear_composition`，与正常选词的 `get_commit` 路径复用同一 `commit(string:)`；`handle` 开头还有 session 失效即重建的自愈逻辑。
