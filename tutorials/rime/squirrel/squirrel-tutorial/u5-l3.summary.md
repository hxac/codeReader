本讲讲的是 Squirrel 的「保留属性（Reserved Property）」机制——一条让 librime 插件在不改前端源码的前提下指挥前端显示的约定通道，由 PR #1143 引入。核心契约在 `ReservedProperty.swift`：键名必须以 `_` 开头，集中登记在 `ReservedPropertyKey` 枚举里，三个键分别为 `_comment_highlight`、`_comment_warning`（携带候选索引集合，给非高亮候选行的注释上色）、`_refresh_ui`（纯重绘触发器）。值由 `ReservedPropertyValue.parse` 解析，支持历史逗号列表（`0,2`）与 URL 风格 query-string（`value=0,2`）两种语法，统一归一到默认字段 `value`，`indices()` 访问器只读该字段、对空格与重复值宽容。应用器 `handleReservedProperty(key:value:for:)` 经三重会话守卫后，把索引写入会话级状态 `specialCommentIndices: [ReservedPropertyKey: Set<Int>]`；`SquirrelPanel.update` 拼注释富文本时反查该字典，命中强调键用 `accentCommentTextColor`、警告键用 `warningCommentTextColor`（分别来自 YAML 的 `accent_text_color`/`warning_text_color`，漏配则静默降级为 nil 不改色）。「转瞬即逝」靠 `rimeUpdate(clearReservedComments: Bool = true)` 实现：普通按键触发的 `rimeUpdate()` 默认清空索引，只有 `_refresh_ui` 调用的 `rimeUpdate(clearReservedComments: false)` 保留索引，使插件着色随下次按键自动消失。消息路由经 `notificationHandler`：C 回调在 librime 线程切出 key/value（只在第一个 `=` 切一刀），再经 `Task.detached { @MainActor in ... }` 唯一一次跨线程跳到主线程，按 `delegate.panel?.inputController` 寻址后执行应用，避免 AppKit 线程违规与数据竞争。
