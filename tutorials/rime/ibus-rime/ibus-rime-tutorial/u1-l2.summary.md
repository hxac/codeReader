本讲回答「ibus-rime 怎么被编译出来、怎么跑起来」。它拆成四个最小模块：CMake 构建脚本与整体流程、pkg-config 依赖发现、configure_file 模板生成、Makefile 包装层。核心结论是：ibus-rime 以 **CMake** 为构建系统，根目录的 `Makefile` 只是对 `cmake` 命令的薄封装，让用户用 `make`/`make install` 一行命令完成 out-of-source 构建（产物隔离在 `build/`，源码树保持干净）。四组依赖用三种机制发现：IBus、libnotify 用 `pkg_check_modules`（pkg-config 读 `.pc` 文件）；librime 用 `find_package(Rime)`（依赖 librime 自带的查找模块，注意变量是单数 `Rime_INCLUDE_DIR`）；rime-data 用项目自定义的 `cmake/FindRimeData.cmake` 按候选目录清单探测，且可被 `-DRIME_DATA_DIR` 覆盖。关键术语包括：`configure_file(@ONLY)` 把模板 `rime_config.h.in` 加工成 `build/rime_config.h`，注入三个宏——`IBUS_RIME_VERSION`（来自 `1.6.1`）、`IBUS_RIME_ICONS_DIR`、`IBUS_RIME_SHARED_DATA_DIR`（值来自 `RIME_DATA_DIR`）；同理生成 `rime.xml`。最终产物是可执行文件 **`ibus-engine-rime`**，构建期位于 `build/`，安装期位于 `/usr/lib/ibus-rime/`，并通过 `--ibus` 参数被 IBus 启动。四条 install 规则分别安装 `rime.xml`、可执行文件、图标、`ibus_rime.yaml`。这些路径与产物是后续 U2 启动流程阅读的索引。
