本讲拆解 ibus-rime 核心「投影」函数 `ibus_rime_engine_update`——所有外部事件（按键、焦点、按钮、翻页）的唯一出口，采用三段式结构：`get_status`→`get_commit`→`get_context`，每段配 `free_xxx` 释放。状态段调用 `ibus_rime_update_status`，先以 `is_disabled`、`is_ascii_mode`、`schema_id` 三字段去重，相同则 return；否则三态切换状态栏 `InputMode` 属性：禁用（disabled.png/維護）、英文（abc.png/Abc）、中文（zh.png/方案名或回退中文），并由 label 首字符派生 symbol。提交段经 `ibus_engine_commit_text` 上屏，所有权转交 IBus，须先于 context 处理。context 段设早退分支：取不到上下文或 `composition.length==0` 时隐藏全部 UI。本讲为 U4 总纲，预编辑文本（u4-l2）与候选表（u4-l3）均在 context 段后续展开。
