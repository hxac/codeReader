本讲是手册收束篇，从架构视角收束前五单元，涉及六个最小模块：rime_main.c（入口与全局 rime_api 取得）、rime_engine.c/.h（引擎对象与 UI 投影）、rime_settings.c/.h（配置加载）、ibus_rime.yaml（用户配置）。核心结论：ibus-rime 是「薄前端」，只做 IBus 原语与 librime API 间的翻译转发，不实现任何输入法算法；所有事件汇聚到唯一投影函数 ibus_rime_engine_update，UI 是 librime 状态的单向、幂等、可重入投影。RimeApi 是稳定边界，经「全局指针 + extern」共享，配合 RIME_STRUCT、RIME_API_AVAILABLE 做版本与能力协商，使前端能独立于核心升级。GObject 虚函数表（class_init 重写）即扩展点清单，G_DEFINE_TYPE 一行完成注册；配置驱动 UI 遵循「内置默认 + 读到才覆盖」的幂等模式。引入的关键术语：薄前端、投影（projection）、稳定边界、虚函数表即扩展点清单、三类扩展点（配置项 / 状态栏按钮 / 按键事件）、改造模板（加 YAML 字段 → 加结构体字段 → 加默认值与读取分支 → 渲染处使用）、长期偏好与瞬时动作之分。
