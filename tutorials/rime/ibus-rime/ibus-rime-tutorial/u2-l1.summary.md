本讲围绕 `rime_main.c` 讲清 ibus-rime 进程「怎么起来、怎么活着、怎么安全退下」的主线，拆成四个最小模块。其一，`main()` 故意写得很短，只做三件事：注册 `SIGTERM`/`SIGINT` 回调、用 `rime_get_api()` 拿到 librime 句柄存入全局 `rime_api`、调用 `rime_with_ibus()` 进入真正主流程，体现「入口层只编排、不做业务」。其二，`sigterm_cb` 是优雅退出的关键，遵循异步信号安全原则：信号回调里不直接清理，只调 `ibus_quit()` 请求主循环退出，真正的 `finalize` 留给主线程常规流程，避免与正在操作 librime 的主线程并发冲突。其三，全局指针 `rime_api`（类型 `RimeApi*`）是前端与核心引擎之间的稳定边界，初值 `NULL`，在 `main` 中赋值、在引擎层与设置层只读复用。其四，`ibus_rime_start()` / `ibus_rime_stop()` 是对 librime `initialize()` / `finalize()` 的前端封装：前者计算用户目录 `~/.config/ibus/rime`、填充 `RimeTraits`、初始化并按需 `deploy_config_file`；后者带 `if (rime_api)` 防御。核心结论是完整调用链：`main → setup → start(initialize) → ibus_main 阻塞 → 信号触发 ibus_quit → stop(finalize) → 返回`，其中 `ibus_main()` 是阻塞分水岭，`finalize` 必发生在其返回之后。
