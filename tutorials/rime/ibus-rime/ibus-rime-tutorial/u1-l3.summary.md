本讲为 ibus-rime 建立「目录结构 + 源码地图」。仓库体量很小：根目录三对 `.c/.h`，外加 `icons/`、`cmake/`、`package/` 资源目录，以及 `librime/`（核心引擎）、`plum/`（方案与词库）两个 git 子模块。核心是三个 C 文件的职责划分：`rime_main.c`（入口层，约 150 行）负责进程启动、IBus 总线与 librime 生命周期；`rime_engine.c`（引擎层，约 610 行，最重）定义 `IBusRimeEngine` 类型、处理按键、把 librime 状态渲染成 IBus 的预编辑文本/候选表/状态栏按钮；`rime_settings.c`（设置层，约 90 行）读取 `ibus_rime.yaml` 填入全局 `g_ibus_rime_settings`。三者经 `aux_source_directory` 编进同一个 `ibus-engine-rime`，通过全局 `rime_api` 与 `extern` 协作。关键术语：组件描述模板 `rime.xml.in` 分组件级与引擎级，组件名 `im.rime.Rime`（D-Bus 总线名）与引擎名 `rime`（引擎标识）属不同命名空间，须与代码字符串严格对应；`--ibus` 仅为约定标记，`main()` 不解析 argv。由此确立「入口/引擎/设置」三层心智模型，为 U2 启动流程、U3 引擎对象精读奠基。
