本讲聚焦 `ibus_rime_engine_update` 的 context 段，把 `RimeContext` 翻译成两类 UI 出口：**内联预编辑（inline_text）**与**辅助文本（auxiliary_text）**。渲染由三组配置共同决定：`embed_preedit` 总开关（来自 `style/inline_preedit`）为真才进入内联分支，否则 preedit 交 IBus 弹窗；`preedit_style`（`preview`/`composition`）决定内联文本来源——`preview` 取 `commit_text_preview`（转换后汉字），`composition` 取 `composition.preedit`（原始编码）；`cursor_type`（`insert`/`select`）决定光标停在插入点还是选中片段起点，并反向控制 librime 的 `soft_cursor`。关键概念含**选中片段**（`sel_start`/`sel_end` 标记）与字节偏移到字符位置的换算（`g_utf8_strlen`）。核心差异：辅助文本仅在 **PREVIEW + 有选中片段** 时出现，显示未转换编码尾巴并用固定灰高亮；`IBusText` 属性系统提供下划线与前景/背景色，`color_scheme`（aqua/azure/ink/luna）只影响 `inline_text` 选中片段配色，可用 `~` 关闭。
