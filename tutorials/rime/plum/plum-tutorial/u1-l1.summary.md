plum（東風破）是 Rime 引擎的配置管理器，而非引擎本身；它从 GitHub 下载并更新输入方案，安装到 Rime 用户目录，并能适配不同前端（ibus/Squirrel/Weasel/fcitx）的不同路径。本讲只读 README.md，不涉及任何脚本逻辑。

引入四个易混淆的核心概念：schema（输入方案，`.schema.yaml`，定义按键如何被解释）、dictionary（词典，`.dict.yaml`，提供编码到候选词的映射）、package（数据包，对应一个 GitHub 仓库，可含多个互相关联的方案与词典）、recipe（配方，符号 ℞，描述如何安装或加工一个 package，数据包本身即可作为最常见的 recipe）。关系链为：schema 引用 dictionary，package 打包二者，recipe 描述安装动作。

重要事实边界：plum 只安装源文件，不会自动启用新方案（不改动 `default.yaml` 的 `schema_list`），启用需用户手动操作。

官方包索引由 Rime Developers 维护，共 22 个包，按内容分 4 大类：Essentials（基础，prelude、essay）、Phonetic-based（基于读音，12 个，含现代标准官话、拼音衍生、其他方言、中古汉语四组）、Shape-based（基于字形，6 个）、Miscellaneous（杂项，emoji、ipa）。另有按安装范围划分的横向集合：`:preset` / `:extra` / `:all` 三档，其中 `:all` 为前两者合集。
