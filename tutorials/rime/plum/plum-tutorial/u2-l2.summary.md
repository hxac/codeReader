本讲聚焦 `scripts/resolver.sh`，解决「命令行 target 字符串如何被拆解理解」这一前置问题，是 u2-l3「安装主循环」的直接前置。

核心概念是**配方字符串（recipe order）**，统一形态为 `user/repo@branch:recipe:key=value,...`，其中 `user/`、`@branch`、`:recipe:options` 三段皆可省略，最简形式只需一个裸包名（如 `luna-pinyin`）。它携带五个字段：user（默认 `rime`）、repo（`rime-` 前缀可省）、branch（默认分支）、recipe（空则不跑配方）、options（键值参数）。

拆解由 **6 个 `resolve_*` 函数**承担，采用「一事一函数」设计，全靠 Bash 参数展开逐字段切片：`${1%%[@:]*}` 取「仓库定位段」是核心技巧（`[@:]` 是字符集，匹配单个 `@` 或 `:`）；`resolve_recipe` 与 `resolve_recipe_options` 通过两次 `${rx#*:}` 组合分别取「两冒号之间」和「第二冒号之后」的内容，选项再经 `sed` 把 `,`/`:` 换成空格，配合调用方 `$(...)` 拆分成数组。

归约由 **`load_package_list_from_target`** 完成，把四类 target（远程清单 URL、本地 `.conf`、预设集合 `:name`、裸包名）统一归约成 `package_list` 数组；前三类靠 `source` 清单文件（清单本质是可执行 shell 片段，用 `+=` 追加，故 `all-packages.conf` 需先清空再 source），第四类直接构造单元素数组。关键设计是**职责分离**：归约（哪些包）与拆解（每个包怎么装）分属 `load_package_list_from_target` 与 `install_package`。
