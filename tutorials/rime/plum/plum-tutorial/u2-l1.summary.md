本讲剖析 plum 最底层的两个脚本——也是整个 `scripts/` 的地基：`bootstrap.sh` 实现模块系统，`styles.sh` 提供统一输出函数。

核心是用纯 Bash 自实现的模块系统：`provide '名字'` 在模块文件末尾把名字追加进全局数组 `loaded_modules`（登记）；`require '名字'` 在使用前先查该数组，已有则直接返回，否则 `source` 对应的 `.sh` 文件，source 之后再查一次，仍查不到就报 `ERROR: failed to load module`。模块查找根 `module_root_dir` 由 `BASH_SOURCE` 定位到 `bootstrap.sh` 所在目录，模块名即去掉 `.sh` 的文件名。去重时给「数组拼接串」和「模块名」两侧各补一个空格、再用 `grep -F` 匹配，把子串匹配升格为整词匹配，避免模块名互相包含（如 `style` 与 `styles`）误判；这套「source 后再查」的设计把「模块是否正确收尾」变成了可即时发现的错误。

`styles.sh` 是被 `require` 最多的叶子模块：定义 ANSI 颜色变量（`esc='\x1b'` 经 `echo -e` 解释为真正的 ESC 字节）及 `info`/`warning`/`error`/`highlight` 等输出函数，结构均为「起始色 + `$@` + reset」。

关键约定：模块文件统一写作 `source bootstrap.sh` → `require 依赖` → 定义函数 → `provide 自身名`。本讲是后续 `resolver`、`install-packages` 等讲义中所有 `require` 调用的理解前提。
