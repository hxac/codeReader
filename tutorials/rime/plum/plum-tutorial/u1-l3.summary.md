本讲承接 u1-l2，把焦点从「脚本如何启动」转到「用户如何用」，回答两个问题：告诉 `rime-install` **装什么**、**装到哪**。关键术语：**target**（命令行每个空格分隔参数）、**rime_dir**（Rime 用户目录，文件最终装进这里）。

核心结论：一行 `curl | bash` 等价于克隆 plum 后运行 `bash rime-install`；无参数时默认装 `:preset`，并按 OSTYPE 猜 `rime_dir`。`rime-install` 只负责收参数、循环、转发，不做语法解析；它把每个 target 交给 `install-packages.sh`，唯一特例是 target 为 `plum` 时在子 shell 里 `git pull` 更新 plum 自身。

target 有六种写法：预设集合 `:preset`/`:extra`/`:all`（冒号开头，由对应 `*-packages.conf` 定义）、纯包名（自动补全成 `rime/rime-<name>`）、`user/repo`、`@branch`、远程 `*-packages.conf` 清单，以及特殊的 `plum`。`:preset` 当前含 prelude、essay、luna-pinyin、terra-pinyin、bopomofo、stroke、cangjie、quick 共 8 个包。

环境变量 `rime_dir`（装到哪）、`rime_frontend`（给哪个前端，映射 ibus-rime/squirrel/weasel 等）、`plum_repo`/`plum_dir`（plum 自身来源与落点）都能猜默认、也能用前缀赋值覆盖。

重要边界：plum 只把源文件拷进 `rime_dir`，**不启用**方案（不写 `default.yaml` 的 `schema_list`）；target 的具体解析（`@master` 切分、`:preset` 展开）发生在 u2 的 resolver，本讲不展开。
