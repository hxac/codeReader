本讲拆解 install 主循环里 `fetch_or_update_package` 之后的「下载/更新」两个分支：`fetch-package.sh` 与 `update-package.sh`。前置前提是 `package/` 目录是一次性源码缓存（真正交付的是拷进 `rime_dir` 的文件），所以 plum 不保留其中的本地改动——这是硬重置逻辑成立的基础；`fetch_or_update_package` 按「目录不存在则下载、已存在则更新」二分调用两个脚本。`fetch-package.sh` 作为独立子进程运行，用自带（与 resolver.sh 同名但反向）的 `resolve_package_name` 把裸名、`rime-` 前缀名或 `user/repo` 统一还原成 GitHub URL，再以 `--depth 1 --recurse-submodules`（git≥2.9 追加 `--shallow-submodules`）做浅克隆。`update-package.sh` 先用 `git_current_branch`/`git_default_branch` 以返回码 2/1/0 三态判断当前与目标分支，不一致则经 `switch_branch`（内部 `fetch_all_branches` 补救单分支浅克隆缺 ref）切换。同分支更新走 `git fetch --recurse-submodules` + `git merge --ff-only origin/<branch>`，ff-only 失败即 `git reset --hard` 硬重置回 origin。`no_update` 置 1 则跳过整个 fetch+merge。本讲为后续 u2-l6 配方执行引擎做衔接。
