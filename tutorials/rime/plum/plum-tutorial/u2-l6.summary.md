本讲解析 `recipe.sh`——plum 的配方执行引擎。配方（recipe）是对「需加工」的包执行的动作；配方文件是「顶层键+缩进段」组织的 YAML，plum 不引解析器，靠 `print_section`（一个 sed 范围过滤器）按段名切出属下缩进内容。`install_recipe` 是总入口，按固定顺序执行 `check_recipe_info` → `apply_download_files` → `apply_install_files` → `apply_patch_files`，四段皆可选、缺则跳过。`check_recipe_info` 用 `recipe` 段里的 `Rx:` 行核对配方名，仅对用户显式点名配方的场景生效，默认 `recipe.yaml` 路径跳过校验。`apply_download_files` 与 `apply_install_files` 共用「`print_section` 提段 + `sed` 列表化」模式：前者用 `download_file`（curl `-fRL`、支持 `本地名::URL` 改名、`-z` 增量下载）把文件下到包缓存目录；后者把 glob 展开结果喂给 u2-l3 的 `install_files`，享受 `diff` 增量拷贝与 `files_updated` 计数。最精妙的是 `apply_patch_files`：它是一台代码生成器，用一段 sed 把 `patch_files` 段的 YAML 转写成合法 bash 脚本（每个目标文件 → 一句 `patch_file 名 <<EOF ... EOF`），再 `| bash` 执行；`patch_file` 往目标文件追加 `__patch:` 段（只追加、不覆盖整份用户文件），用三段式 `# Rx: package:recipe:options` 标记裹住每块补丁，先删旧块再插新块，从而实现幂等——重复执行同一配方，补丁永远至多一份。
