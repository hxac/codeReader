本讲以「颜色」为线索把配置端与绘制端串联。核心是 `UIStyle` 这份「服务端决策、前端执行」的样式契约：23 个 int 颜色字段加圆角、阴影、字体等几何字段，命名遵循「对象＋`hilited_` 状态」规律。因绘制依赖 Windows COLORREF（`0x00BBGGRR`），内部统一用 `0xAABBGGRR`（`COLOR_ABGR`），用户写 ARGB/RGBA 时由 `ARGB2ABGR`/`RGBA2ABGR` 做字节重排；`_RimeGetColor` 是宽容解析器（支持前缀、简写、6 位补 alpha、8 位保留，写错走缺省值不报错），`_UpdateUIStyleColor` 用 `COLOR(键,字段,回退)` 宏表把 yaml 键映射到字段并构成回退继承链，`blend_colors` 负责前后景 alpha 混合。定制走 levers 覆盖层：Deployer 用 `customize_string` 写 `weasel.custom.yaml` 的 `__patch`，合并进 `weasel.yaml` 后由 `Initialize` 与 `_LoadSchemaSpecificSettings`（区分 `color_scheme` / `color_scheme_dark`）读入；`__synced` 惰性重发使大块样式仅在切方案、切深色或建会话时整块序列化重发一次，前端 `Styler` 反序列化后由 `_TextOut`/`_HighlightText` 落到 DirectWrite 文字与 GDI+ 圆角阴影绘制。
