本讲是「候选窗口 UI 渲染」单元收尾，补上真正把像素画上屏幕的绘制层。核心是 DirectWriteResources（DWR）资源池：把 D2D/DWrite 工厂、DC 渲染目标、画刷与四种文本格式（候选正文、拼音、序号、注释）收为一处，在样式或 DPI 变化、渲染失败时由 `_InitFontRes` 整体重建；字体回退表故意用无效主名 `_InvalidFontName_`，强制所有字形按 Unicode 区段走回退分配。`DoPaint` 严格遵循「先 GDI+ 画背景（圆角、阴影、边框、高亮），再 BindDC 用 DirectWrite 画文字」的顺序，两套 API 共享同一张 memDC，最后 `UpdateLayeredWindow` 一次性合成；BindDC/EndDraw 失败即重建 DWR 自愈。形状三细节：圆角由 `GraphicsRoundRectPath` 配 `IsToRoundStruct`，阴影用盒滤波近似高斯（4 次可分离 box blur），配色由 `_RimeGetColor` 把 ARGB/RGBA 统一转成 Windows 的 ABGR(COLORREF)。绘制层只经 Layout 的 getter 取矩形，布局与绘制解耦。
