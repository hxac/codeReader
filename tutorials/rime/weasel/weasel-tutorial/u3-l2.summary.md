本讲聚焦 WeaselTSF 前端「抓键」环节，承接 u3-l1 的 _InitKeyEventSink、向下打通 u2 的 IPC，覆盖三个最小模块。一是去重：用 _fTestKeyDownPending/_fTestKeyUpPending 两组对称标志，把 TSF 的 OnTestKeyDown（询问）与 OnKeyDown（通知）两阶段回调幂等化为每个物理按键只发一次 IPC，兼容 WORD 多次 Test、QQ 缺 Test 两类不规范应用。二是翻译：ConvertKeyEvent 用 keyState 算修饰键 mask、TranslateKeycode 字典查特殊键（查不到走 ToUnicodeEx 解码 Unicode 字符），产物 weasel::KeyEvent{keycode:16, mask:16} 作为 32 位整数经 WEASEL_IPC_PROCESS_KEY_EVENT 零拷贝发往 Server。三是放行与补丁：_ProcessKeyEvent 设键盘关闭/禁用、连不上 Server 两道显式闸门，加 _SendMessage 异常退化为返回 0 的隐式兜底，确保断线不卡键；Caps Lock 因 Windows 与 Rime 状态时序不一致，靠翻译时反转 LOCK_MASK、抬起时 SendInput 模拟两次击键（配 keyCountToSimulate 防重发）弥补。核心出参 *pfEaten 决定按键被吃（拦截）或放行（透传给应用）。
