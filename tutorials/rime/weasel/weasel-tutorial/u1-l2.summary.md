本讲绘制 Weasel 的「目录地图」。weasel.sln 统管 10 个编译子工程：WeaselTSF（DLL，产物 weasel.dll，前端瘦客户端）、WeaselServer（EXE，全局服务，托管 librime 与 UI）、WeaselDeployer/WeaselSetup（设置与安装 EXE）、4 个静态库（WeaselIPC、WeaselUI、WeaselIPCServer、RimeWithWeasel）及 2 个 Google Test 工程。产出类型由 .vcxproj 的 ConfigurationType 决定，依赖关系由 ProjectReference 决定；WeaselUI/WeaselIPC/WeaselIPCServer 是不依赖他者的叶子库，WeaselServer 依赖最重，WeaselTSF 同时链接 IPC 客户端与 UI。include/ 公共头经 $(SolutionDir)\include 被各工程共享，核心数据契约为 WeaselIPCData.h（定义 Context/Status/UIStyle，用 boost::serialization 序列化以跨命名管道传输）、WeaselUI.h（weasel::UI 抽象接口与 DirectWriteResources）。GUI 基于外部头文件库 WTL/ATL，经各工程 stdafx.h 引入。辅助目录 test/、resource/、output/、update/、arm64x_wrapper/ 分别承担测试、图标资源、安装打包、自动更新、ARM64X 跨架构 DLL；librime 与 plum 为 git 子模块。
