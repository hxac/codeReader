本讲建立 Weasel 的全局地图，分三个最小模块。首先定位生态：Weasel（小狼毫）是跨平台引擎 librime 在 Windows 上的前端，与 macOS 鼠须管、Linux 的 ibus-rime/fcitx5-rime 共用同一引擎与配置；Rime 的精髓是「引擎与平台前端分离」，引擎交互通过 C 接口 rime_api.h 收口于 RimeWithWeasel 子工程。其次讲多进程架构：WeaselTSF（DLL）作为瘦客户端驻留每个应用进程，只负责抓键与上屏；WeaselServer（EXE）全局唯一，托管 librime、候选窗口 UI 与托盘。二者经命名管道 IPC 通信，管道名形如 \\.\pipe\<用户名>\WeaselNamedPipe 实现按用户隔离，命令集定义在 WEASEL_IPC_COMMAND 枚举中，核心是 WEASEL_IPC_PROCESS_KEY_EVENT。IPC 层定义抽象基类 RequestHandler，由 RimeWithWeaselHandler 继承并把命令转交 librime，实现 IPC 与引擎解耦；WeaselServerApp 则把 Server、Handler、UI、托盘组装成完整服务。最后串成主链路：应用进程抓键 → 管道发命令 → Server 派发 → librime 算字 → UI 画候选并回传上屏文字 → WeaselTSF 写回应用文档。该数据流图是后续所有讲义的导航索引。
