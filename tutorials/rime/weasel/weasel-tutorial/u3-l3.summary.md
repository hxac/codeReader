本讲聚焦 WeaselTSF 前端的「上屏」环节，覆盖三个最小模块。其一，**EditSession 机制**：TSF 规定任何文档修改都必须封装在 `ITfEditSession` 对象里，经 `RequestEditSession` 提交，由 TSF 在读写锁（`TF_ES_READWRITE`）保护下回调 `DoEditSession(TfEditCookie ec)`，`ec` 是持锁修改文档的「编辑凭据」。Weasel 有两套会话：`WeaselTSF` 自身实现 `ITfEditSession` 作为每按键的「主会话」，以及一组继承 `CEditSession` 的「专项会话」（开始/结束写作串、插字、取坐标）。其二，**Composition 的 preedit 与 commit**：主会话读 Server 响应后按「有 commit 则开→插→关上屏、用 `_status.composing` 对齐写作串状态、重定位候选窗」编排。核心结论是 `inline_preedit` 开关只改变三处——是否塞 CUAS 占位空格、是否把 preedit 写进文档、选区塌缩方向——commit 上屏流程两种模式一致。其三，**中止与清理**：焦点离开与写作串被清空走 `_AbortComposition`（前后端同清且幂等），光标移出走 `OnEndEdit`→`_EndComposition`（仅前端收尾），由 `ITfTextEditSink`/`ITfTextLayoutSink` 经 `_InitTextEditSink` 挂载接收。本讲只讲前端如何写 TSF 文档，Server 侧 `commit`/`preedit` 的计算留给 u4。
