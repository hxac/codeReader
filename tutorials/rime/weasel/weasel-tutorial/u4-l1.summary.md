`RimeWithWeaselHandler` 是抽象基类 `weasel::RequestHandler` 的唯一实现，也是 Weasel 全仓库唯一直接调用 librime C API 的类，由此把 IPC 传输层与引擎层彻底解耦。本讲覆盖三个最小模块：其一，librime 的 C API 是一张函数指针表 `RimeApi`，经 `rime_get_api()` 取得并存为文件级静态变量；`RimeTraits` 向引擎登记共享/用户数据目录、发行版身份与日志目录，`setup()` 必须在 `initialize()` 之前调用。其二，初始化呈三段式——构造函数取句柄并 `_Setup()`（`setup` + 挂 `OnNotify` 回调）、`WeaselServerApp::Run()` 显式调 `Initialize()`（`initialize` + 维护 + 读 `weasel.yaml`）、退出时 `Finalize()`；其中维护模式（maintenance）让按键在数据重建期间禁用，命名互斥体 `WeaselDeployerMutex` 保证 Server 与独立进程 `WeaselDeployer.exe` 不同时改写用户数据。其三，`OnNotify` 可能在部署后台线程触发，Weasel 以静态消息缓冲配 `m_notifier_mutex`，由 `OnNotify` 写、`_ShowMessage` 读、`_UpdateUI` 清，UI 更新严格留在主线程。关键术语 `RIME_STRUCT` 与 `RIME_API_AVAILABLE` 负责 ABI 版本协商，供后续讲义承接。
