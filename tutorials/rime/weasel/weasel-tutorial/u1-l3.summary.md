本讲讲解如何把 Weasel 从源码构建并运行调试，覆盖三个最小模块。第一，构建总入口 `build.bat` 是一条流水线：加载 `env.bat`、计算版本号、校验 `BOOST_ROOT`（缺失即报错退出），再按命令行参数（`all`/`boost`/`rime`/`weasel`/`installer` 等）驱动 boost→librime→数据→weasel→安装包。第二，版本号注入：用 `render.js` 把 `weasel.props.template` 渲染成 `weasel.props`，经各 `.vcxproj` 引用后作为宏进入资源编译器与 C 源码；非发布构建还会用 git 提交数与短 hash 补全 `PRODUCT_VERSION`。第三，仓库并存两套构建体系——官方 `weasel.sln`+MSBuild（`build.bat` 实际驱动，覆盖 ARM/ARM64/安装包）与替代的 `xmake.lua`（更精简，仅 x86/x64），两者共用 `BOOST_ROOT` 与 `VERSION_*`，产物布局一致。最后，`output/install.bat` 负责向 TSF 注册并启动 `WeaselServer.exe`，含权限提升与按 Windows 版本分支的逻辑；调试时须区分进程——服务端/引擎/UI/IPC 服务端代码附加到 `WeaselServer.exe`，前端按键抓取与上屏代码（`weasel.dll`）附加到正在接收输入的应用进程。
