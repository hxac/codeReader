本讲聚焦 Dock 布局系统的容器与布局树。DockArea 是顶层 Render 视图，用「左｜中心列｜右」的 flex 三段式拼出窗口：左/下/右三个 Dock 是固定停靠位，仅可整体开合与调宽，尺寸受 PANEL_MIN_SIZE=100px 下限约束；中心区是可任意分屏的主舞台，底部 dock 藏在中心列内部。中心区与停靠位内容由 DockItem 布局树描述，四种节点为 Split（分屏，背后 StackPanel，唯一可递归）/Tabs（标签页，TabPanel）/Panel（单面板叶子）/Tiles（自由拼贴）；任意 IDE 式布局可由 Split 嵌套加 Tabs 组合表达，Dock 内部不支持 Tiles。构造 DockItem 需传入 &WeakEntity<DockArea> 以建立实体并订阅事件（用 window.defer 推迟）。DockArea 实现 EventEmitter<DockEvent>，充当中继把底层 PanelEvent 翻译成对外的 LayoutChanged 与 DragDrop，并处理 ZoomIn/ZoomOut 全屏放大（zoom_view）。LayoutChanged 是布局自动存盘钩子，触发频繁需防抖。布局经 dump/load 与 DockAreaState 序列化往返，靠全局 PanelRegistry 按面板 panel_name 重建实体，未注册名会变 InvalidPanel。
