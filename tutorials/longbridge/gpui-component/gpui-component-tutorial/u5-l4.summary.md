本讲剖析 gpui-component 的菜单系统，核心结论是「一个引擎 + 两个触发器」：`PopupMenu` 是唯一的有状态引擎实体（实现 `Render + Focusable + EventEmitter<DismissEvent>`），负责菜单渲染、键盘导航与 Action 派发；`ContextMenu`（右键触发）与 `DropdownMenu`（点击触发）只是不同的触发外壳，自身不画菜单。三类菜单项统一由 `PopupMenuItem` 枚举表达（分割线、Label、普通项、自定义元素项、子菜单）。菜单项行为推荐绑定 GPUI 的 `Box<dyn Action>`（可自动显示快捷键提示），亦可回退到 `on_click` 回调，两者二选一、`handler` 优先。关键术语包括：`key_context("PopupMenu")`、`action_context`（关闭后焦点归还来源）、`DismissEvent`（上层据此收起）。`ContextMenu` 是自定义 Element，靠 hitbox 命中检测右键、用 `ContextMenuState`（`Rc<RefCell<...>>`）跨帧保持状态，被挂元素必须有 `.id()`；`DropdownMenu` 是 `RenderOnce` 组件，复用 `Popover` 并以 `use_keyed_state` 只 build 一次菜单、靠监听 `DismissEvent` 关闭并清缓存。`DropdownButton` 本身不实现该 trait，而是内部拼装箭头按钮再挂菜单。键盘导航（上下左右/回车/ESC）在 `init` 中绑定；限制是开启 `scrollable(true)` 后子菜单不可用。
