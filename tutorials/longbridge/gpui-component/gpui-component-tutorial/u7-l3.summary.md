本讲精读 Tree 树形组件，延续 u7-l1「虚拟化+状态实体+无状态外壳」范式，覆盖数据模型、状态引擎、键盘导航、定位坐标四个模块。核心结论：用户写嵌套的 TreeItem，内部按深度优先、仅含已展开节点地扁平化为 Vec<TreeEntry>，再经 uniform_list 虚拟滚动渲染，展开/折叠的本质是 rebuild_entries 重建扁平数组，缩进靠 entry.depth()。TreeState 是 EventEmitter<TreeEvent> 实体，集中托管选中（扁平下标 selected_ix）、展开折叠与右键，toggle_expand 时广播 Expanded/Collapsed 事件，payload 用稳定 id 而非易变下标。键盘导航用 key_context("Tree")+actions 体系，把方向键绑定到 SelectUp/Down/Left/Right/Confirm，仅在 Tree 获焦时生效。澄清 IndexPath{section,row,column} 是 List/ListDelegate 体系的坐标，Tree 内部并不直接用它而用扁平 usize，二者等价为 IndexPath{0,n,0}，且都随展开状态不稳定，程序内稳定定位应优先用 TreeItem::id。render_item 闭包若要回调外层视图，需用 view.update 把 &mut App 升级为 Context 才能调用 cx.listener。
