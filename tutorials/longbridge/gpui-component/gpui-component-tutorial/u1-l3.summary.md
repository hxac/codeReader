本讲从「结构」视角剖析 gpui-component 仓库。它是一个多 crate workspace：根 Cargo.toml 的 members 含 6 个 crate（ui 核心库、story/story-web 展示库、macros 过程宏、assets 资源库、webview）加 12 个 example，default-members 决定裸 cargo run 编译谁；依赖单向：story → ui → {macros, assets}。核心库 crates/ui/src/lib.rs 用三类声明组织代码：私有 mod（基础设施）、pub mod（56 个组件模块，按输入/展示/布局/浮层/数据渲染分类，复杂组件用目录、简单组件用单文件）、pub use 再导出常用类型到 crate 根；init(cx) 列表揭示需全局初始化的子系统（theme、root、dock、input 等），无状态组件不在其中。最后讲清编译期协作：assets 通过 links + build.rs 发布图标目录为 DEP_ 环境变量，macros 的 icon_named! 宏扫描 SVG 自动生成 IconName 枚举，IntoPlot 为 derive 宏。引入了 workspace、crate、再导出、proc-macro、links/DEP_ 机制等供后续讲义承接的术语。
