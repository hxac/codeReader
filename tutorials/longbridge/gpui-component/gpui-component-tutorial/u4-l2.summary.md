本讲精读 Switch / Checkbox / Radio 三个选择控件，三者均为无状态 `RenderOnce`，真值外置到外层 View，经 `.checked(...)` 下发、`on_click(&bool,...)` 上报，闭环必须配 `cx.notify()`。Switch 是即时生效的单布尔开关（轨道 + 滑块），用 `on_mouse_down` 回调取反后的新状态 `&!checked`，配 0.15s 滑动动画。Checkbox 实现独立多选，额外提供 `Selectable`（选中即勾选，可被 ButtonGroup 等通用容器调度）与 `ParentElement`（可塞富文本/Markdown 标签）能力，并经 `track_focus` + `focus_ring` 接入焦点系统支持 Tab 聚焦。Radio 视觉与 Checkbox 对称（圆框 + 复用的勾选图标），但本身不内置互斥：单选语义靠 `RadioGroup` 容器实现。核心结论是单选互斥无需手写「取消其他项」——`RadioGroup` 用「外置 `selected_ix` + `checked == Some(ix)` 下帧重绘 + 索引回调 `&usize`」自动完成。切换动画统一复用 u3-3 的 `use_keyed_state` + `with_animation` 补间。注意本版本 Switch 暂未接入焦点系统。
