本讲是学习手册的开篇，不涉及代码，只建立对 `gpui-component` 的整体认知。它围绕三个最小模块展开：项目定位、核心特性（Features）以及与同类框架的对比。核心结论是：`gpui-component` 是一个基于 GPUI（来自 Zed 编辑器团队）、用 Rust 编写的跨平台桌面 UI 组件库，提供 60+ 现代化组件，设计风格源自 shadcn/ui、图标来自 Lucide，许可证为 Apache-2.0。

本讲引入并要求后续承接的关键术语包括：GPUI（底层 GUI 渲染框架，而非 wgpu）、RenderOnce（无状态组件设计，类似 React 函数组件）、Sizable（统一实现 xs/sm/md/lg 四档尺寸）、Rope（ropey 库的文本底层，支撑 20 万行编辑）、Tree Sitter（语法高亮方案）、Dock/Tiles（复杂面板布局）、虚拟化 Table/List（同时虚拟化行与列）。其差异化优势集中在三块：现代 UI 风格与内置主题、Markdown/HTML 富文本混排、基于 Rope+Tree Sitter 的编辑器与大数据表格，这使其尤其适合编辑器类、数据密集型与富文本类桌面应用。阅读 README 时应把握「定位 → 特性 → 用法 → 运行 → 对比」的主线，对比表是理解选型理由的最佳入口。后续讲义将分别精读 init/Root 入口、构建运行流程、workspace 结构等细节。
