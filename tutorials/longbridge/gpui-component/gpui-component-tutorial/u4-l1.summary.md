本讲讲解 gpui-component 的表单模块（仅 `crates/ui/src/form/` 一个目录，三个文件）。核心结论：`Form` 与 `Field` 分工明确——`Form` 是无状态的 CSS grid 容器，负责整张表的方向（垂直/水平）、列数、标签宽度与尺寸；`Field` 负责单行字段的「标签 + 输入 + 描述」方向感知排版，并提供 `required`、`visible`、`col_span` 等配置。

关键术语与机制：`FieldProps` 是表单级共享参数包，`Form` 在渲染时通过 `field.props(ix, props)` 统一下发给每个字段，使配置「设一次、全局生效」；`label`/`description` 均接受 `FieldBuilder`（字符串/闭包元素/视图三态）。最重要的概念是——`Field` 没有内置校验状态字段，校验遵循「状态外置 + 描述反馈」约定：必填靠 `required(true)` 画红色 `*`，错误提示则把外层 View 持有的错误文案经 `description_fn` 渲染成 `cx.theme().danger` 红色文字，复用描述槽位实现。
