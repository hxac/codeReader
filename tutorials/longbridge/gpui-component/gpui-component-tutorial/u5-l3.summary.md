本讲承接 u5-l1 的 Dialog，讲 Sheet 与 Notification 两类窗口级浮层如何由 Root 统一承载。核心结论：两者的画面出口是 render_sheet_layer 与 render_notification_layer 两个关联函数，必须由根视图 render 手动 .children 挂载，Root::render 自身不画这些浮层，漏挂即「调了也看不见」。

三类浮层管理模型不同：Dialog 是栈（可嵌套）、Sheet 是单个（Option，新开替换旧的、不可嵌套）、Notification 是常驻实体列表（Entity<NotificationList>，可堆叠、去重、各自倒计时）。Sheet 为无状态 RenderOnce，沿用「描述+闭包重建」范式，经 window.open_sheet 打开，支持四方向（默认 Right）、遮罩、滑入动画，ESC、点遮罩、关闭按钮三种关闭，on_close 仅在用户主动关闭时触发，程序化 close_sheet 不触发。Notification 为有状态 Render 实体，经 window.push_notification 推送，四种类型（Info/Success/Warning/Error），默认 5 秒自动消失，随机 id 追加、id::<T>() 去重替换、.action 会关闭自动消失。Root 还会读取当前 Sheet 的方向尺寸给通知加 margin 避让。
