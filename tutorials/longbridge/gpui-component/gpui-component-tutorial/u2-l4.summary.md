本讲讲解 gpui-component 中三块「给 GPUI 元素加挂通用能力」的基础设施，核心手法统一为「扩展 trait + blanket impl」——在不动 GPUI 源码的前提下，让 `div()` 等元素直接获得新方法。其一，`InteractiveElementExt`（event.rs）提供 `on_double_click`，复用 GPUI 的 `on_click` 并用 `click_count() == 2` 过滤，仅授予状态化元素 `Stateful<E>`（即带稳定 id 的元素）。其二，`ElementExt`（element_ext.rs）的核心 `on_prepaint` 往容器塞一个铺满透明的 `canvas` 子元素作「隐形探针」，在绘制后回调出像素矩形 `Bounds<Pixels>`，约束为 `ParentElement`，是 tooltip/popover/slider 定位与命中测试的底座。其三，`FocusTrapElement`（focus_trap.rs）的 `focus_trap` 通过全局 `FocusTrapManager`（存 `WeakFocusHandle`）登记容器，再由 `Root` 拦截 `Tab`/`TabPrev` 动作，借 `find_active_trap` 与 `contains_focused` 实现模态容器的「焦点不逃逸、绕回首尾」循环；该全局必须在 `init(cx)` 里注册。三者合称 Dialog/Sheet/Popover/Tooltip 等组件交互能力的底座。
