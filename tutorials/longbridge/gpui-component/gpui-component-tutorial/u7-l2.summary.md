本讲区分 gpui-component 中两个同名但定位不同的表格组件。`Table` 是一组无状态、可组合的 `RenderOnce` 积木（`TableHeader`/`TableBody`/`TableRow` 等），靠 flex 布局与 `flex_basis(relative(col_span))` 比例宽度排版，不做虚拟化，仅适合几十行的小表。`DataTable` 才是大数据表格，遵循「状态实体 `TableState` + 无状态外壳 `DataTable` + 数据契约 `TableDelegate`」范式，支持行/列/单元格选择、列宽拖拽、列重排、左侧固定列、点击排序、右键菜单与滚动加载。

`DataTable` 对行、列用两套虚拟化：行等高，用 GPUI 的 `uniform_list`（整数除法求可见区间）；列宽各异，表头用 `calculate_visible_leaf_col_range` 手动做前缀和扫描并配 spacer 撑总宽，表体每行复用本库的 `virtual_list`（横轴，前缀和），200px overdraw 防闪烁。左侧固定列不进虚拟列表、常驻渲染。

关键术语：`Column`（静态列定义）与 `ColGroup`（运行时状态，多了拖拽后的真实 `width` 与 `bounds`，渲染一律用运行时宽度）；列宽在 `resize_cols` 中 clamp 到 `[min_width, max_width]`，排序在 `perform_sort` 中做三态循环。接入数据靠实现 `TableDelegate`：必填 `columns_count`/`rows_count`/`column`/`render_td`，可选 `render_th`/`render_tr`/`group_headers`/`perform_sort`/`context_menu`/`load_more`/`visible_rows_changed` 等。
