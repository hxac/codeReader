本讲为「高性能数据展示」单元首讲，讲解虚拟化原理及两个核心组件。虚拟化的本质是每帧只渲染可见区间 `[first, last)` 内的几十项，其余项不进入元素树，可见区间由「前缀和 + 滚动偏移」的线性扫描求得（核心工作发生在 Element 的 prepaint 阶段），从而把可见项数与总数据量解耦。关键术语包括：视口 viewport、滚动偏移（负数）、可见区间、前缀和。本讲覆盖两个最小模块：一是通用虚拟滚动 `Element`「VirtualList」，支持横/纵双轴与异构尺寸（区别于 GPUI 的 uniform_list），但要求调用方预先给出 item_sizes，跨轴尺寸由测第 0 项得到，渲染以回调闭包按 visible_range 按需调用；二是基于代理（delegate）的成品「List」与状态实体「ListState」，在 VirtualList 之上封装分段（section）、键盘导航、内置搜索、懒加载，但假定全员等高，它通过 `RowsCache` 把分段数据拍平为一维 RowEntry 喂给 VirtualList。使用者需实现 `ListDelegate` 契约的 `items_count`、`render_item`、`set_selected_index` 三方法，关联类型 `Item: Selectable + IntoElement`。设计范式沿用「无状态 RenderOnce 外壳 + 有状态 Entity」（如 List/ListState 与 Input/InputState 同构）， IndexPath{section,row,column} 为定位坐标。选型结论：等高大数据用 List，不等高或更自由的虚拟化用 VirtualList。
