本讲是 rust-i18n 学习手册的起点，覆盖两个最小模块并确立其核心定位。第一模块讲定位：rust-i18n 把翻译文件（YAML/JSON/TOML）在编译期（cargo build 阶段）解析、合并、扁平化并代码生成（codegen）进二进制，运行时零文件 IO、解析器默认不进二进制，由全局 `t!` 宏在静态后端里查找取文本。其 API 灵感来自 ruby-i18n 与 Rails I18n。核心特性包括：编译期 codegen、全局 `t!`、多格式多文件合并、fallback 回退链（含 v2.4.0 的 territory 自动回退）、minify_key 短键（v3.1.0）、变量插值与 std::fmt 格式化、log-miss-tr、load-path feature。第二模块讲 `src/lib.rs` 中的宏导出：`t!` 与 `available_locales!` 都是 macro_rules! 声明宏且为转发壳——`t!` 转发到由 `i18n!` 在编译期生成的内部宏 `crate::_rust_i18n_t!`（故未调用 `i18n!` 则编译报错），`available_locales!` 转发到内部函数 `crate::_rust_i18n_available_locales()`。根 crate 通过 `pub use` 暴露 `rust_i18n_macro`（过程宏，含 `i18n`、`_tr`）与 `rust_i18n_support`（运行时类型）。全局 locale 由 `CURRENT_LOCALE`（`LazyLock<AtomicStr>`，默认 "en"）承载，`set_locale`/`locale` 读写。关键术语：i18n、l10n、locale、编译期代码生成、转发壳。
