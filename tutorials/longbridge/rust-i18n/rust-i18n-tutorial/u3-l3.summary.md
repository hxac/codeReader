本讲承接 `_tr!` 的解析与代码生成，专门讲清 `t!("key", count = 7 : {:05})` 中插值参数与格式说明符如何在编译期处理、运行期填入译文 `%{name}` 占位符，由三个最小模块构成。其一，运行时纯函数 `replace_patterns(input, patterns, values)`：用一个三态（地面态／已见 `%`／在占位符内）字节级状态机定位 `%{name}`，按名字在 `patterns` 查找、用同下标 `values` 替换，未命中则原样保留，并用 `SmallVec` 与 `chunks_exact(2)` 兼顾性能与对畸形输入的鲁棒性。其二，编译期 `Argument::parse`：将 `count = 7 : {:05}` 中的 `: {:05}` 读成 `Argument.specifiers = Some(":05")`，只存花括号内部内容、逐 token 拼接、不做语义校验。其三，桥梁 `into_token_stream`：把 `specifiers` 包回 `format!("{:05}", value)`，按相同顺序收成 `keys`/`values` 两数组交给 `replace_patterns`。核心结论是：替换按名字匹配而非下标，参数名须与 `%{name}` 同名；即使译文未命中，原始消息也会做一次插值，因此无翻译文件时字面消息加参数仍能工作；格式化发生在调用方、替换发生在 `replace_patterns`。
