本讲聚焦 `crates/support/src/lib.rs` 中三个 `#[cfg(feature="codegen")]` 函数，承接 u2-l2 的文件解析，完成同一 locale 多份内容的合并与嵌套树拍平，产物交给 u2-l4 代码生成。核心是「先合并、后扁平化」两阶段：合并产物为 `Translations`（locale → 嵌套 JSON 树），扁平化产物为 `BTreeMap`（locale → 点号键 → 译文）。先在嵌套树层面合并，才能清晰表达「对象下钻、叶子覆盖」的语义，避免扁平键覆盖不可预测。`merge_value`：双方皆对象则逐键递归下钻，缺失键以 `Value::Null` 占位，其余整体覆盖（last-write-wins）。`flatten_keys`：深度优先、用点号拼前缀，叶子（含数字、布尔）字符串化，`Null` 与 `Array` 压成空串。`format_keys` 是 v2 解析专用的「跳过空段、用点连接」工具，与前者拼接语义等价。最终扁平表因采用 `BTreeMap` 而按键名字典序排列，即下讲 `generate_code` 的输入；点号键形态与运行时 `t!` 查询完全对齐。
