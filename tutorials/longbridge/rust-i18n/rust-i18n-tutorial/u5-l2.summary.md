本讲承接「编译期加载与解析本地化文件」，回答「改 yml 后为何翻译不更新」的问题，核心结论是：翻译更新依赖 Cargo 增量编译感知，而 `locales/*.yml` 非 Rust 源文件，默认不计入 crate 指纹，故需构建脚本显式登记。涉及最小模块有四个：根 crate 的 `build.rs`（仅扫描工程内所有 `locales` 目录、逐个发出指令，自身不解析任何翻译内容）、`cargo:rerun-if-changed` 指令、`workdir()` 定位策略与 `RUST_I18N_DEBUG` 调试开关。关键术语与概念：构建脚本（build script，编译前置钩子）、Cargo 指纹与增量重编译、过程宏展开依赖 crate 重编译、`cargo:rerun-if-changed=<路径>`（Cargo 标准指令，将文件纳入发出该指令的 crate 的指纹）、`[build-dependencies]` 隔离（`globwalk`/`regex` 仅存于构建期）、`workdir()` 两级回退（首选 `CARGO_MANIFEST_DIR`，回退用正则切掉 `OUT_DIR` 中 `/target/.../build/` 反推工程根）、`is_debug()`（仅 `RUST_I18N_DEBUG` 精确等于 `"1"` 生效），以及 `cargo:i18n-*`（rust-i18n 自定义日志前缀，非 Cargo 标准指令，仅供检索排错）。分工上，`build.rs` 只负责告知 Cargo「关注哪些文件」，真正读取解析翻译由 `i18n!` 中的 `try_load_locales` 完成。
