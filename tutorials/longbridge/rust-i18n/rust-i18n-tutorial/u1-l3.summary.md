本讲围绕真实示例 `examples/app`，讲清 rust-i18n 的最小可用闭环：初始化、切语言、取值、插值。引入三个核心 API：`i18n!("locales")` 是编译期执行的过程宏，负责扫描翻译目录、合并同名语言多文件、把嵌套键拍平成点号键，并生成静态后端 `SimpleBackend` 与内部宏 `_rust_i18n_t!`；它是总开关，不调用则 `t!` 无法编译。`set_locale`/`locale()` 操作全局变量 `CURRENT_LOCALE`，默认值 `"en"`，基于 `arc-swap` 做线程安全的原子替换，`locale()` 返回守卫而非裸引用以防悬空。`t!` 是转发壳，转发到 `_rust_i18n_t!` 查表；支持按 key 取值、用 `name = "..."` 填充翻译里的 `%{name}` 占位符（占位符名须与参数名一致）、以及临时指定 `locale = "fr"`。关键区分：`set_locale` 改全局、影响后续所有 `t!`；`t!(..., locale=)` 仅影响当次、不改全局。关键结论：加载路径相对本 crate 的 `Cargo.toml` 解析；嵌套 YAML 编译期拍平；修改 yml 后无需额外操作即可重编译。术语如「编译期代码生成」「点号键」「守卫」「临时 locale/全局 locale」供后续讲义承接。
