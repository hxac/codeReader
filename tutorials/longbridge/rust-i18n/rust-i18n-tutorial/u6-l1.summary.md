本讲讲解 rust-i18n 短键（minify_key）算法本身，源码集中于 `crates/support/src/minify_key.rs` 一个文件。短键为解决「文案即 key」导致长句反复存储、增大二进制体积与哈希开销的问题而生：把长文案压缩成定长短串当 key，译文不变。

入口函数 `minify_key(value, len, prefix, threshold)` 返回 `Cow<str>`：当 `value.len() <= threshold` 时短路返回 `Cow::Borrowed` 原文（零分配）；否则走「哈希 + base62 + 截断 + 加前缀」。`threshold` 控制「是否压缩」，`len` 控制「压缩后留多长」，二者相互独立。

生成链路分三步：`hash128` 用固定密钥的 `SipHasher13`（经 `LazyLock` 全局复用）对字符串 UTF-8 字节算出确定性 128 位哈希；`base62::encode` 将其编码成可读短串；最后用 `len.min(encoded.len())` 安全截断（128 位 base62 最多 22 字符，故默认 `len=24` 几乎总被钳到 22，默认不损信息）并拼接 `prefix` 做命名空间隔离。

四个 `DEFAULT_MINIFY_KEY_*` 常量（总开关默认 `false`、`len=24`、`prefix=""`、`thresh=127`）是 macro crate 与 support crate 共享的兜底默认值，默认关闭以向后兼容。关键术语「确定性哈希」「base62」「阈值/截断上限」为后续 u6-l2（`_minify_key!`/`tkv!` 编译期调用）与 u6-l3（`_tr!` 与提取器协作）承接埋下伏笔。
