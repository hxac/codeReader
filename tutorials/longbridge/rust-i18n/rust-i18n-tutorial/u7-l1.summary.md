本讲聚焦 cargo i18n 命令行入口（crates/cli/src/main.rs），拆解四个最小模块。其一，cargo i18n 是 cargo 子命令：二进制须命名为 cargo-i18n，程序内用 clap derive 的 CargoCli 枚举（顶层 name/bin_name 设为 cargo）配 I18n 变体将 i18n 识别为子命令；I18nArgs 用 #[derive(Args)] 定义 -t/--translate（可重复，类型 Vec<(String,String)>）与位置参数 SOURCE（默认 ./，last=true 须在 -- 之后）。其二，translate_value_parser 用 split_once("=>") 兼容纯文本（返回 (s,s)）与键值对写法（返回 (k,v)，经 trim 与去引号），且永不报错。其三，add_translations 遵循贯穿提取器的关键约定——HashMap 的 key 是查找键（开 minify_key 则哈希成短键）、Message.key 是译文内容——用 entry().or_insert() 实现「提取器优先、手动翻译不覆盖」。其四，main 按固定顺序串联主流程：解析 → I18nConfig::load（读 source_path 所指项目的 Cargo.toml，无配置回落 default）→ iter/extract → add_translations → 按 index 排序 → generator::generate（写入 load-path）→ 返回 Err 则 exit(1)，使 CI 在有未翻译文案时失败。引入的关键术语：cargo 子命令、clap derive、translate_value_parser、查找键与 Message.key、Message 结构、I18nConfig，供 u7-l2（遍历与 t! 提取）和 u7-l3（TODO.yml 生成与去重）承接。
