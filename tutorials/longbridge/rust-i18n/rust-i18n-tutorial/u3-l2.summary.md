本讲义逐模块剖析 `_tr!` 过程宏的解析与代码生成，全部集中在 `crates/macro/src/tr.rs` 一个文件，五个最小模块构成自底向上的数据类型与自顶向下的控制流：`Value` 是参数「值」的统一枚举（`Empty`/`Expr`/`Ident`），靠 `fork` 探测先试表达式再试标识符，`to_string()` 仅对字符串字面量返回 `Some`，这限定 minify_key 的编译期短键只能对字面量生效；`Argument` 统一解析 `key = value`、`key => value`、`key = value : {:spec}` 三种形态，名字可取标识符或字符串字面量，`=` 与 `=>` 等价，specifiers 仅存花括号内部内容；`Tr` 是把 `Messsage` 与 `Arguments` 打包、并承载 `locale`/`minify_key*` 的总容器；`filter_arguments` 把 `locale` 及 `_minify_key*` 五个系统参数分流并用 `retain` 删除，保证它们绝不进入 `replace_patterns` 的 `keys`/`values`，从而 `locale` 只作为查找参数、不被当占位符替换；`into_token_stream` 按 `args.is_empty()` 分无参数（命中直接返回）与有参数（命中/miss 均调 `replace_patterns`）两条路径，`values` 各元素经 `format!("{spec}", v)` 统一为 `String` 并应用格式说明符。两条路径最终都收敛到同一查表入口 `crate::_rust_i18n_try_translate(locale, &msg_key)`。占位符替换的字节级实现留待 u3-l3，回退链留待 u3-l4，minify_key 另三条 codegen 分支留待 u6 系列。
