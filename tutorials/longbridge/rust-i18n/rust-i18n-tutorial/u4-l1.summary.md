本讲义围绕 `crates/support/src/backend.rs`，拆解 rust-i18n 的「翻译存储抽象」。首先定义 `Backend` trait：三个方法 `available_locales`、`translate`、`messages_for_locale`，均返回 `Cow<'_, str>`（借用零拷贝、需生成时则 Owned），并带 `Send + Sync + 'static` 超职约束以放入全局静态量供多线程访问。核心结论一：`Backend::translate` 只做精确命中返回 `None`，**不负责回退**，回退链由外部 `_rust_i18n_try_translate` 编排，实现了「存储」与「回退策略」的解耦。其次 `SimpleBackend` 是默认实现，用两层嵌套 `HashMap<locale, HashMap<key, value>>` 存翻译，`translate` 为两次哈希查找，命中路径经 `Cow::Borrowed` 零拷贝。`add_translations` 用 `entry().or_default()`+`extend` 实现「合并而非覆盖整表」，同名键 last-write-wins，使 codegen 能逐语言安全灌入。最关键结论：`i18n!` 生成的 `_RUST_I18N_BACKEND` 本质就是装进 `Box<dyn Backend>`、由 `LazyLock` 延迟初始化的 `SimpleBackend`；二者仅灌数据方式不同。这两个类型是纯运行时类型，可脱离 `i18n!` 独立使用，为后续 `BackendExt`/`CombinedBackend` 组合与自定义后端铺垫。
