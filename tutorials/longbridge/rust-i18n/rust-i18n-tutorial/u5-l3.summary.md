本讲围绕 rust-i18n 如何用 Cargo 的 feature flags 与可选依赖机制，把翻译文件解析器「默认挡在用户二进制之外、按需放回」展开。核心涉及两个面向用户的根 feature（`load-path`、`log-miss-tr`）和 support crate 的内部 feature `codegen`：根 feature 在 `[features]` 中通过 `crate/feat` 语法把开关接力转发到子 crate。`codegen` 是关键总开关，用 `optional = true` 配合 `dep:` 前缀把 serde/serde_yaml/serde_json/toml/globwalk/normpath/itertools 七个解析器依赖做成「编译期独占」，并配合 `#[cfg(feature="codegen")]` 守卫所有加载/解析源码（含 `try_load_locales`）；macro crate 无条件开 `codegen`，使 `i18n!` 在编译期能读文件，而解析器默认不进运行时二进制。`log-miss-tr` 是空 feature，仅在编译期给每次 `t!` 未命中处注入 `log::log!` 告警，因 macro 不依赖 `log`，生成代码注入用户 crate，故用户须自备 `log`。`load-path` 接力打开 `codegen`，让运行时函数 `try_load_locales` 被根 crate 条件 `pub use` 暴露，实现改翻译文件无需重编译；其解析逻辑与编译期 `i18n!` 复用同一管线。需警惕撞名陷阱：cargo feature `load-path` 与 `[package.metadata.i18n]` 配置项 `load-path` 是两回事。两条线（`dep:` 控依赖、`#[cfg]` 控源码）必须配套。
