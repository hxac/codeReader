本讲聚焦 rust-i18n 本地化文件的两种写法。核心开关是文件顶层的 `_version` 字段：它是「翻译文件格式版本」，与 crate 版本号无关，不写时默认 `1`，只有显式 `_version: 2`（数字 2，非字符串）才走 v2 分支，其它任何值都按 v1 处理（机制由 `get_version` 的 `unwrap_or(1)` 与 `parse_file` 的 match 分发决定）。**v1 风格**是一种语言一个文件，locale 完全由文件名推导（`file_stem` 再 `split('.').last()`，支持「模块名.语言.yml」前缀命名），整个文件原样挂到该 locale 下（`parse_file_v1`），适合多人/多团队各负责一种语言。**v2 风格**是多语言合并到一个文件，采用「键 → { 语言: 译文 }」的**嵌套 locale 结构**，解析时忽略文件名、locale 由内容中的语言子键决定，由 `parse_file_v2` 用 `format_keys` 拼接键名并按 locale 重组，适合单人配合 AI 批量补全。两种风格解析后内部数据结构殊途同归，都可被同一套 `t!` 查找。`yml/yaml/json/toml` 四种扩展名在两风格下均受支持。
