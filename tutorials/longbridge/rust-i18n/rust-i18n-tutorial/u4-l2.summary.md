本讲围绕 `crates/support/src/backend.rs` 一个文件，讲清如何把任意两个后端组合成一个。`BackendExt` 是 `Backend` 的子 trait，只多一个 `extend(self, other)` 方法：按值消费两个后端，返回元组结构体 `CombinedBackend<A, B>`，其中 `A=self.0`（被扩展者）、`B=self.1`（后加入者）。`where Self: Sized` 约束排除了 `dyn Backend`，因此组合必须发生在「装箱」之前；任何 `Backend` 实现加一行空 impl 即可免费获得该能力，`SimpleBackend` 正是如此。`CombinedBackend` 本身也实现 `Backend`，故组合可链式嵌套，最外层 `self.1` 优先级最高。三方法优先级方向不一致：`translate` 用 `or_else` 实现「`self.1` 优先、`self.0` 兜底」；`messages_for_locale` 按 key 去重、`self.1` 胜，与 translate 一致；`available_locales` 则以 `self.0` 为基底用 `contains` 显式去重取并集，先后只影响列表顺序。核心结论：`i18n!(backend = ...)` 将本地文件的 `SimpleBackend` 作 `self.0`、用户自定义后端作 `self.1`，故自定义后端的同名键覆盖本地文件，其根因就在 `CombinedBackend::translate`。
