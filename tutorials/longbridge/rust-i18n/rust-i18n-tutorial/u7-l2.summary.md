本讲深入 `cargo i18n` 提取器内部，覆盖三个模块：`iter_crate` 用 `ignore::WalkBuilder` 遍历源码树，天然尊重 `.gitignore`，只挑 `.rs` 文件把 `(路径, 全文)` 喂给回调，实现遍历与处理分离；`extract` 先用 `syn::parse_file` 解析源码（剔除注释）再展平为 token 流，`invoke` 以 `Ident + "!"` 模式递归识别宏调用，名字须在白名单 `METHOD_NAMES = ["t","tr"]` 中；`take_message` 只取宏首参字面量，非字面量（动态键、数字）即放弃，行号取自字面量 `span`。核心数据模型 `Results = HashMap<查找键, Message>`，`Message` 含译文内容 `key`、首次出现序号 `index` 与多个 `Location`；`format_message_key` 把连续空白归一化为单空格，使文字相同空白不同的调用去重成一条 Message、收集多个 Location。本讲产出 `Results`，落盘 `TODO.yml` 留给 u7-l3。
