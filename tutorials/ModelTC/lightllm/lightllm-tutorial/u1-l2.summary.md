本讲解决"装好 LightLLM 并跑起推理服务"。环境要求 Linux、Python≥3.9.16（推荐 3.10）、GPU 算力≥7.0。三种安装方式：官方 Docker 镜像、源码构建镜像、源码安装（conda 建环境→pip install -r requirements.txt→python setup.py install）。Docker 须加 --shm-size（纯文本≥2GB、多模态≥16GB），因多进程靠共享内存传数据，否则启动失败。依赖分两套：setup.py 的 install_requires 是 10 个最小依赖（不含 torch）；requirements.txt 是含 torch、transformers、flashinfer、pyzmq、rpyc 的完整锁版依赖。启动服务一行命令 python -m lightllm.server.api_server --model_dir <目录>，--model_dir 为唯一必填参数，默认监听 127.0.0.1:8000，直接读 HuggingFace 格式权重。验证用 curl POST /generate，body 含 inputs 提示文本与 parameters（max_new_tokens、frequency_penalty）。本讲引入概念：LLM 推理服务、HTTP API、GPU 算力(CC)、共享内存、conda 环境、最小依赖与锁版依赖之分。
