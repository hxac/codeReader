本讲打开「从 logits 挑下一个 token」的黑盒，覆盖采样、惩罚项、top-k/top-p 三个最小模块。统一入口 `sample()` 的执行顺序固定为：组装批参数 → 应用惩罚项 → 非法 token 屏蔽 → 温度缩放 → softmax → 分支采样。惩罚项在 softmax 前就地改写 logits，含三类：repetition（按比例，正值除法/负值乘法）、frequency（按出现次数线性递减）、presence（固定值），外加作用于 eos 的指数长度惩罚；token 计数分 cpu 稀疏与 gpu 稠密两种模式。过滤采样在 softmax 后作用于概率分布，top-k 与 top-p 均采用「清零」语义取两者交集，随机采样借助 Gumbel-max 技巧。关键结论：默认 `top_k=-1` 被归入贪心快路径，默认参数下走 argmax 确定性解码，需显式设 `top_k>1` 或带 seed 才会随机采样。
