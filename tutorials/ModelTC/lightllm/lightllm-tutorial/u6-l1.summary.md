本讲讲解 LightLLM 中 CUDA Graph 的捕获（capture）与重放（replay）机制：它把 decode 整条前向录成一张「图」，重放时 CPU 仅发一次命令，把几十次 kernel launch 摊销为一次，主要降低 decode 单 token 延迟而非吞吐。机制受两大约束：形状固定、地址固定。形状靠 `gen_cuda_graph_batch_sizes` 生成的两段式档位表（小 batch 逐 1 密集、大 batch 按 `graph_grow_step_size` 稀疏）配合 `find_closest_graph_batch_size` 向上 padding 解决；地址靠留存捕获时的 infer_state 与 model_output 作固定容器，重放前用 `copy_for_cuda_graph` 灌入新数据，即「地址不变、内容可变」。`_decode` 决策链为：`graph is not None` → `can_run`（batch 与序列长度双把关）→ `find_closest` → `need_capture`（首次则捕获、否则重放）；warmup 从大到小逐一录图并共享 mempool。prefill 图则另起炉灶：以 `handle_token_num` 为键、复用 decode 的 mempool、要求精确命中档位（不 padding）、默认关闭且仅支持部分模型。关键术语：kernel launch 开销、capture/replay、mempool、档位表、warmup。
