本讲讲解 LightLLM 的模型注册机制，回答「框架如何凭 `--model_dir` 自动选出推理类」这一前置问题，是第五单元「模型适配实践」的入口。核心源码集中在 `lightllm/models/registry.py` 与 `lightllm/models/__init__.py` 两个文件。

引入的关键术语：`_ModelRegistries` 注册中心（以 `model_type → List[ModelConfig]` 为结构，用 `defaultdict(list)` 允许同名多候选）；`@ModelRegistry(...)` 装饰器（副作用是登记，可接收单名或列表实现多别名）；`ModelConfig`（封装 `model_class`、`is_multimodal`、`condition` 三字段）；以及查询入口 `get_model`（查表并实例化）与 `get_model_class`（只返回类，供带 `lru_cache` 的轻量预查询）。`condition` 是 `dict → bool` 的条件谓词，用于同名多候选消歧。

三个最小模块与核心结论：（1）登记发生在 import 期——`__init__.py` 逐行 import 各 `model.py` 触发装饰器，故「被登记」等价于「被 import」；（2）匹配以 `config.json` 的 `model_type` 查表，按「condition 为 None 或对当前 config 为真」过滤候选，0 命中报不支持、1 命中直接用；（3）条件分发在多命中时只保留带 condition 的候选并断言恰剩 1 个，要求重名候选满足「一个无条件默认 + 若干互斥特例」或「全部互斥特例」。内置谓词工厂 `is_reward_model()` 与 `llm_model_type_is(name)` 分别按 `architectures` 与嵌套 `llm_config/text_config` 分流。
