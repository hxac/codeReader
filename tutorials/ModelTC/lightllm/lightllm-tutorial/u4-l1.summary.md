本讲打开 LightLLM token 级 KV Cache 显存管理，含三个最小模块。一是 MemoryManager，作为 KV 显存总容器，拥有一块 4 维张量 kv_buffer，形状为 (layer_num, size+1, 2*head_num, head_dim)，其中 token 维 +1 的 HOLD_TOKEN_MEMINDEX 槽位永不外借、专供 padding；K、V 沿 head 维拼接，可一次切片取出。二是 KvCacheAllocator，以连续区间 [mark_start, mark_end) 配可重写栈实现 alloc 顺序弹出、free 压回栈顶、free_all 全量复位，全程不申请新显存；它返回的是「索引张量」而非显存，登记进 req_to_token_indexs 映射表。三是 profile_size，用户未指定 --max_total_token_num 时按可用显存与单 token 全层字节数 cell_size 估算出 max_total_token_num，并跨 TP rank 取 MIN 对齐。运行时 can_use_mem_size 经 SharedInt 写入共享内存供 Router 做调度准入；select_mem_manager_class 则按模型族与 --llm_kv_type 在普通管理器与 FP8/INT8 量化变体间选型。本讲为后续 RadixCache 复用、KV 量化、PD 分离迁移奠定「索引 + 大 buffer」地基。
