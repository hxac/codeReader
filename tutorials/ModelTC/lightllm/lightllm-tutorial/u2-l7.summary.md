本讲覆盖 Detokenization 进程：它处于 ModelBackend 之后、HttpServer 之前，专职把共享内存中的 token id 还原为文本。它用 PULL 收 Router 的请求登记，从共享内存读新 token，做增量解码后写回共享内存，再用 PUB 向 HttpServer 发空通知。核心是增量解码：靠 prefix_offset 与 read_offset 两个游标，对含安全余量的 token 段 decode 后裁掉前缀得到新增文本，并以替换符检查避免多字节字符乱码。每请求由 DecodeReq 追踪，经登记、解码、回收三阶段，文本片段写入嵌于 Req 的 out_tokens_queue 环形队列，由 HttpServer 的 SUB 唤醒后批量搬运。关键术语还有 decode_token、eos 与 special 抑制、all_special_ids，以及 PD 分离模式下游标适配的 decode_mode_fix。整体为数据走共享内存、zmq 只传空通知的零拷贝流式模式。
