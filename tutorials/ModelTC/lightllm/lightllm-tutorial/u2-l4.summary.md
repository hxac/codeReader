本讲剖析每个 GPU 上独立运行的模型推理后端进程，核心结论是「rpyc 做控制、共享内存做数据」的分工。启动期，`ModelRpcServer`/`ModelRpcClient` 基于 Unix socket 用 rpyc 一次性下发 `init_model` 与 `get_max_total_token_num`，并借 `async_wrap`（`rpyc.async_`+`asyncio.to_thread`）把同步调用桥入 Router 的 async 循环，实现多 rank 并行初始化。`ModeBackend` 作为模板基类，在 `init_model` 中依次组装分布式环境、`TpPartBaseModel`、KV 内存管理器、`RadixCache` 并注册到全局上下文 `g_infer_context`，末尾启动两个 `infer_loop` 线程做双 batch overlap。运行期 backend 完全自驱：`infer_loop` 主动轮询共享内存 `shm_reqs_io_buffer`，自行决定 prefill/decode/pass，不再经 rpyc；Router 经 `_add_batch` 写入并 `set_ready`，backend 用 NCCL broadcast 协调各 rank 读取。单步前向（`prefill_normal`/`decode_normal`）为四阶段 overlap，由 `InferReqUpdatePack` 解耦延迟更新，结果经 master rank 写回共享内存中的 `Req`。
