本讲用策略模式把注意力算子抽象成可替换后端，分三块讲解。一是后端抽象：BaseAttBackend 是进程级单例，常驻并持有大块显存，充当工厂生产两个一次性状态对象 BasePrefillAttState 与 BaseDecodeAttState（每次前向新建），由 AttControl 承载 alibi、滑动窗口、mla、nsa 等入参开关。二是后端选择：create_utils 按 KV 数据类型映射表、prefill/decode 各自的优先级列表，再加 backend_validator 在独立子进程用 PyTorch SDPA 做真值校验，三层共同挑选；prefill 默认偏 fa3、decode 偏 flashinfer，用户可用 --llm_prefill/decode_att_backend 显式指定绕过自动选择，全部失败则兜底 triton。三是状态生命周期：状态对象每拍新建并由 init_state 现算派生张量，decode 的 copy_for_decode_cuda_graph 保证 CUDA Graph 重放所需「地址不变、内容可变」。普通、MLA、NSA 三类模型各用一套映射表与后端实现，是后续 MLA 与 FP8 量化讲义的前置知识。
