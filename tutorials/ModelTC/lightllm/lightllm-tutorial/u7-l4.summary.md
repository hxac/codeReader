本讲讲解 LightLLM 的分布式通信体系。核心结论是：为满足 CUDA Graph 对地址与 stream 固定的硬约束，LightLLM 用纯 Python `ctypes` 直接绑定 `libnccl.so`（`pynccl_wrapper.py`→`pynccl.py`），而非 `torch.distributed.all_reduce`。`PyNcclCommunicator` 通过 rank 0 取 `ncclUniqueId` 广播、各 rank `ncclCommInitRank` 握手建域，提供 `all_reduce/send/recv`，主要用于 PD 分离跨节点 KV 传输；`StatelessP2PProcessGroup` 基于 TCPStore 做无状态元数据交换，支撑按需配对建域。日常 TP 通信则走 `communication_op.py` 统一接口：`CustomProcessGroup` 持有 NCCL 子域并按消息大小派发 `FlashInfer→SymmMem→NCCL`，`DistributeGroupManager` 按 `group_size` 建多套域以支撑 microbatch overlap，单组短路。关键术语：NCCL 通信域、`ncclUniqueId`、rank/world_size、all-reduce/all-gather/reduce-scatter、TPSP。TP 下 all-reduce 跟在行并行层（`o_proj`、`down_proj`）之后，TPSP 时替换为 reduce_scatter 并配 allgather/sp_split。
