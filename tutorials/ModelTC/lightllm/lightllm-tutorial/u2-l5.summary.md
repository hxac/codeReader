本讲讲解 LightLLM Router 进程的调度主循环。Router 是 uvloop 驱动的心跳循环 `loop_for_fwd`：每拍先执行 `_step()` 做决策，再 `await asyncio.sleep`（默认约 30ms）让出节拍以攒批、避免忙等。一拍 `_step` 严格按五步执行：收新请求并调度、写 profiler 命令、把新 batch 合并进 running_batch 并经共享内存命令管道下发 backend、过滤已完成请求、处理中止/停止串请求。核心是三段式迁移：`req_queue`（等待区）→ `schedule_new_batch`（中转区，跨拍累积）→ `running_batch`（在跑区），下发受命令管道空闲制约，评估是否塞新请求时合并 running_batch 与 schedule_new_batch 避免重复计显存。激进与保守调度由 `router_token_ratio` 与 `is_busy` 切换：显存占用率超阈值则保守（少塞、估更长），否则激进；比值为 0 即纯保守的 `is_safe_schedule`。Router 只做决策，真正推理由 ModelBackend 的 infer_loop 完成，二者通过命令管道解耦。
