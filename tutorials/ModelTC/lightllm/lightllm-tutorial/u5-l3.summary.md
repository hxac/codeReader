本讲拆解 LightLLM 新增模型的三个最小模块：新增模型流程、模板复用、注册与校验。核心结论是「新增模型几乎不是写新逻辑，而是把算子填进既有骨架的钩子里」，工作量与模型偏离标准 transformer 的程度成正比。

新增模型即在 `lightllm/models/<model_type>/` 下建「三件套」：`model.py`（继承 `TpPartBaseModel`，填六个插槽——两类权重类、三类推理类、一类推理状态类）、`layer_weights/`、`layer_infer/`，可选 `infer_struct.py` 与 `triton_kernel/`；`model_type` 取自 config.json，既是目录名也是注册键。

模板复用基于「模板方法模式」：`TransformerLayerInferTpl` 厚模板写死 prefill/decode 两条对称残差骨架与 KV 落池，只留 7 个 `need to impl` 钩子，pre/post 层为薄模板；权重用「填 HF 名 + 选元权重类型」的现代写法取代手写 TP 切片，构造函数统一 `__init__(self, kvargs)`。

注册靠 `@ModelRegistry` 装饰器且「注册即 import 副作用」，必须在 `models/__init__.py` 加 import 才生效，`get_model` 按 `model_type` 查表分发；校验分三层：启动期 `_verify_params` 断言、加载期元权重 `verify_load`、运行期 `skills/test_model` 的 lm_eval 回归。
