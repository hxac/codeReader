RadixCache 用一棵基数树（radix tree）按公共前缀共享存储「token 序列 → KV 槽位索引」的映射，相同前缀的 KV 只算一次、存一份。TreeNode 以 `token_id_key`（连续 token 序列，作 key）和 `token_mem_index_value`（对应槽位索引，作 value，即 u4-l1 的索引而非 KV 张量本身，复用即抄索引）为核心字段，`ref_counter`/`time_id`/`node_prefix_total_len` 则服务于冷热淘汰与命中长度判断。三大操作：`match_prefix(update_refs=True)` 在 prefill 前查最长公共前缀，命中则把索引抄进请求映射表并将 `cur_kv_len` 置为命中长度，跳过该段 prefill 计算；请求释放时 `insert`（用 `split_node` 处理部分匹配）把新算的 token→索引写回树，供未来请求复用；`evict` 按 `(ref==0, 子节点数, time_id)` 排序，自底向上只删引用为 0 的叶子。`tree_total_tokens_num - refed_tokens_num` 即当前可淘汰量。RadixCache 经 `free_radix_cache_to_get_enough_token` 衔接 MemoryManager（决定留谁、腾谁），经共享计数与只读客户端衔接 Router 调度（决定还能再塞多少请求）。
