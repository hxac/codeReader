本讲紧接 TpPartBaseModel 初始化框架，讲解模型初始化完成后一次前向推理的真实执行路径。核心结论：模型对外只有单一入口 `forward(ModelInput) → ModelOutput`，内部由 `ModelInput.is_prefill` 这一布尔字段分发到两条路径。`_prefill` 负责"读题"——按 token 数补齐到 CUDA Graph 档位、构造 `InferStateInfo`、批量登记 KV 槽位进 `req_to_token_indexs`、调用 `_context_forward` 跑层；`_decode` 负责"逐 token 作答"——现场 `gather_token` 取上一步 token、按 batch size 补齐、每请求登记 1 个槽位、走 CUDA Graph 录制/重放 `_token_forward`。两阶段层模板结构对称（embedding → N 层 transformer → post 层 logits），唯一差别是注意力核（context attention 处理一段 token，token attention 让 1 个新 token 复用全历史 KV）。关键术语：`ModelInput`/`ModelOutput`/`InferStateInfo` 数据流、padding/unpadding（CUDA Graph 形状对齐前置条件）、`req_to_token_indexs` 映射表、`b_`/`b1_` 前缀命名约定、`b_ready_cache_len`（前缀缓存）与 `b_position_delta`（阶段硬约束）。后端在 `forward` 后用 `_sample_and_scatter_token` 采样出新 token 写回 `req_to_next_token_ids`，形成闭环。
