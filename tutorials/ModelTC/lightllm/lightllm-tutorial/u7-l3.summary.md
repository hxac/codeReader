本讲讲解 LightLLM 数据并行（DP）模式下的请求分发与负载均衡，沿三个层层包裹的模块展开：**数据并行 → DP 队列 → 负载均衡**。数据并行把机器上的 GPU 切成多个**DP 组**（dp rank），各组独立跑不同请求、各有独立 KV Cache 与队列，与张量并行（TP）正交；本机组数 `dp_size_in_node = max(1, dp // nnodes)`，DeepSeek-V2 因 MLA 压缩 KV 常用 `dp == tp` 提升吞吐。`DpQueue` 是 DP 模式的外壳，仅在组数 ≥ 2 时创建，内部持有 N 个 `inner_queues` 和一个均衡器，对 Router 暴露与单队列一致的接口；请求进入时 `suggested_dp_index == -1` 的先入暂存区，每拍调度由均衡器分流、各组各自挑批、再 `_merge_batch` 合成含多组请求的单一 batch，靠 `suggested_dp_index` 区分归属。均衡器有两种策略：`round_robin` 在等待队列最短的候选组间轮询；`bs_balancer`（默认）以「在跑+等待」总请求数最小为目标，防止某 DP 组空转 padding 浪费算力。最后须分清两个同名概念：Router 层的 `--dp_balancer` 管**请求往哪送**，Backend 层的 `--enable_dp_prefill_balance` 管 prefill 时 **token 怎么摊**（依赖 all-to-all 重分布，前置条件为 `--enable_tpsp_mix_mode` 且 `--dp > 1`）。
