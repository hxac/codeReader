本讲建立 open-r1 仓库的「目录地图」，回答「功能在哪个目录」。顶层五类目录职责互不重叠：`src/` 是可被 import 的核心包 `open_r1`，`scripts/` 是独立命令行工具（评估、去污染、路由服务），`recipes/` 存 YAML 配方，`slurm/` 存集群作业脚本，`tests/` 存测试（分快速单元测试与需沙箱的慢测试两层，`make test` 故意忽略后者）。三大入口脚本中，`sft.py` 与 `grpo.py` 是带 `__main__` 的可运行入口，`generate.py` 则是提供 `build_distilabel_pipeline()` 的函数库；它们都「薄」，只做组装，训练能力委托给 TRL。关键约定有二：其一，包根 `open_r1/__init__.py` 为空，真正的对外接口在 `utils/__init__.py`，集中导出 `get_dataset`、`get_model`、`get_tokenizer` 等核心零件，故脚本写 `from open_r1.utils import ...`；其二，三元组配置——每个脚本由 `TrlParser` 解析脚本参数、训练参数（`SFTConfig`/`GRPOConfig`）、模型参数（`ModelConfig`）。GRPO 比 SFT 多一行 `from open_r1.rewards import get_reward_funcs`。复现链为：`sbatch slurm/train.slurm --model/--task/--config/--accelerator` → `accelerate launch` → `src/open_r1/{task}.py` → `recipes/<模型>/<sft|grpo>/config_<后缀>.yaml`，其中 `--task` 同时决定入口脚本与配方子目录。
