本讲讲解 open-r1 的四个 accelerate 配置（ddp / zero2 / zero3 / fsdp）。这些 YAML 不被 Python 脚本读取，而由 accelerate launch 在脚本启动前消费、搭建分布式环境；train.slurm 用 --accelerator 选配置，并用命令行覆盖 num_processes 等多机字段，故多机下改 YAML 不生效。核心差异在于分片粒度：DDP 每卡持完整副本，只适合小模型；ZeRO-2 分片优化器与梯度（参数仍完整），ZeRO-3 再分片参数，最省显存但通信最重，特有字段为 zero3_init_flag 与 zero3_save_16bit_model；FSDP 的 FULL_SHARD 等价 ZeRO-3、SHARD_GRAD_OP 等价 ZeRO-2。四配置共享 bf16 混合精度。结论：大模型须分片参数，open-r1 在 32B 实测选用 FSDP 配合 paged AdamW 8-bit 以装下最大上下文。
