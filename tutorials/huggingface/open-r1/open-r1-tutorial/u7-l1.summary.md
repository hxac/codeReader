本讲义讲解 open-r1 如何在 Slurm 集群上编排大规模训练（`slurm/train.slurm`），并额外拉起 vLLM 服务节点来喂给 GRPO 的在线采样，覆盖三个最小模块。其一为参数解析与配置定位：把 `--model/--task/--config/--accelerator` 拼成 YAML 路径，再 grep 出 `gradient_accumulation_steps`、`model_name_or_path`、`model_revision`，注意 `MODEL` 变量会被 YAML 中的真实模型 id 覆盖。其二为 `use_vllm` 分支触发的 **N+1 节点拓扑**：脚本用纯文本 grep 检测 `use_vllm: true`，把最后一台节点拨给 `trl vllm-serve`（其 `--dp`/`--tp` 应满足 TP×DP=8 填满单机），前 N-1 台跑训练，训练侧 `WORLD_SIZE` 相应减去单节点卡数，并通过 `--vllm_server_host` 把 vLLM 主机名注入训练进程。其三为 **srun（外层，把进程派发到物理节点）与 accelerate launch（内层，组建分布式进程组）** 的嵌套启动，以 `--machine_rank $SLURM_PROCID` 标识各节点 rank。核心结论是：train.slurm 只是编排器而非训练实现；把 vLLM 物理隔离到独立节点是为避免与训练争显存导致 OOM。脚本中亦包含若干 HF 集群专有细节（如 Weka 预热），迁移到自有集群时需按 README 调整。
