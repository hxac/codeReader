本讲承接 u3-l2（奖励注册表与数学正确性奖励），逐个拆解 open-r1 中与「输出格式 / 推理结构」相关的四个奖励函数，全部位于 `src/open_r1/rewards.py`，均为纯函数加正则或字符串计数，无模型、无沙箱。背景动机：`accuracy_reward` 是稀疏二值信号，模型早期几乎全得 0 分，缺乏可优化梯度；且不保证答案可被下游脚本解析。格式奖励正是补这两点，提供密集的非零塑形信号，其前提是 open-r1 的 `<think>...</think><answer>...</answer>` 两段式标签约定。

四个函数：`format_reward` 是严格二值裁判，用一条 `^...$` 正则要求整段输出完全合规，差一换行即 0 分；`tag_count_reward` 是其温和版，把结构拆成四个子串检查，每命中一个加 0.25，给出 0~1 连续部分得分，为训练早期提供梯度；`reasoning_steps_reward` 向内看推理内容，用多选一正则数步骤标记（`Step \d+:`、过渡词等），得分 `min(1.0, count/3)`，魔数 3 鼓励 3 步及以上推理，且未启用 `re.MULTILINE`；`code_format_reward` 是工厂函数 `get_code_format_reward(language)` 产出的闭包，要求答案段含指定语言的围栏代码块，并通过 `kwargs["language"]` 支持混合语言训练。

核心结论：二值奖励守住「可解析」底线、部分得分奖励提供上升梯度，二者常搭配；`format` 与 `tag_count` 是 GRPO 默认奖励函数（默认 `["accuracy", "format", "tag_count"]`），格式塑形是 open-r1 GRPO 标配。
