本讲打开 u5-l1 留下的「沙箱黑盒」：抽象基类 `CodeExecutionProvider` 定下契约 `execute_scripts(scripts, languages) -> List[float]`，工厂 `get_provider(provider_type)` 按 YAML 字符串分发到具体实现，使奖励函数 `code_reward` 与沙箱后端彻底解耦，换后端只需改一个字段。两个内置实现共享「同步入口 + `asyncio.run` 桥接 + `Semaphore` 限流 + `gather` 并发 + 异常兜底为 0.0」的同构骨架，确保奖励函数永不抛错、不拖垮 GRPO。`E2BProvider` 精华是三层超时（28/30/32 秒），因 E2B 自带超时失效而靠 `asyncio.wait_for` 硬兜底，末行表达式靠沙箱 Notebook 式捕获回收；`MorphProvider` 因 SDK 同步阻塞而用 `asyncio.to_thread` 扔进线程池，结果需 `text末行→整段→stdout` 多级 fallback 解析。`configs.py` 承诺的 `"local"` 实为空缺，由 `LocalProvider`（subprocess 跑脚本、末行改 print）综合实践补全。可选依赖条件 import 收口于 `import_utils.py`；Router 批量分支留待 u5-l3。
