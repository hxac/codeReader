本讲讲解 open-r1 蒸馏流水线最上游的「数据工厂」`src/open_r1/generate.py`：它不训练模型，而是用 Distilabel 把 prompt 喂给推理模型、收集带思维链的回答并打包成数据集。核心模块有三：一是装配函数 `build_distilabel_pipeline`，它用 Python 代码（而非 YAML）拼出一条「只有一步 `TextGeneration`」的流水线，所有旋钮做成函数入参，供 CLI 与 Slurm 注入；二是 `TextGeneration` 步骤，通过 `OpenAILLM`（`base_url` 指向本地 vLLM 的 OpenAI 兼容服务、`api_key` 仅占位）连外部 vLLM，用 Jinja2 `template` 渲染、用 `input_mappings` 把数据集列改名为 `instruction`，并由 `num_generations` 配合 `group_generations=True` 把每条 prompt 的多份回答打包成列表（输出行数等于输入行数 N）、`StepResources(replicas=...)` 控制并行副本；三是 `Pipeline().ray()` 强制走 Ray 后端，这是它与 README 进程内 vLLM 示例的根本区别。CLI 执行链路为 `argparse` → `load_dataset` → 装配流水线 → `pipeline.run(use_cache=False)` 产出 `distiset` → 可选 `push_to_hub`；集群上由 Slurm 把 vLLM 服务与 `generate.py` 分别作为两个 Ray job 提交。
