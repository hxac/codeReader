本讲是强化学习单元首篇，只讲 `grpo.py` 训练入口的「主流程」骨架，奖励函数实现留给后续讲义。核心结论：`grpo.py` 与 `sft.py` 共用「薄脚本」外壳，`main()` 各阶段（日志、checkpoint、加载数据/分词器/模型、训练、保存、评估、推送）几乎逐字相同；GRPO 的增量只有三段——`get_reward_funcs` 取奖励函数、`make_conversation` 构造 prompt 对话、`GRPOTrainer(reward_funcs=...)` 注入奖励。

`make_conversation` 把数据集某列（默认 `problem`）包成 `[system?, user]` 消息列表写入新列 `prompt` 并删除多余 `messages` 列（GRPO 只需「问」，由模型自己采样回答再打分）。`reward_funcs` 走「YAML 字符串 → `get_reward_funcs` 查注册表 → 可调用函数」的链路注入 Trainer，多函数按 `reward_weights` 加权求和，注册表中既有现成函数也有读取超参的工厂函数。`use_vllm` 用 vLLM 连续批处理加速 GRPO 最昂贵的在线采样；`num_generations`(G) 决定每个 prompt 采样的回答数即「组」大小，并约束 `per_device_train_batch_size` 能被 G 整除，且有 unique_prompts = batch / G。

引入的关键术语：GRPO（组内相对策略优化）、组（group）、优势（advantage，组内归一化，免去价值模型）、奖励函数（reward function）、reward_funcs / reward_weights、注册表模式、use_vllm、num_generations。
