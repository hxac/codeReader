本讲拆解 ms-swift 数据加载链路的三块最小模块。`DATASET_MAPPING` 是全局注册表，靠「导入即注册」填满：包入口的 `register_dataset_info()` 读 JSON 清单，`import dataset` 触发 `llm.py`/`mllm.py` 的代码注册，key 有字符串（带 `dataset_name`）与三元组两种形态；`DatasetMeta` 描述来源/子集/切分/预处理器。`DatasetLoader.load` 是分派器：本地文件走 `_load_dataset_path`（复用 `hf_load_dataset` + 扩展名映射），仓库走「选子集、逐子集下载、拼接」，`safe_ddp_context` 防多卡重复读盘。顶层 `load_dataset` 编排整链路：`DatasetSyntax.parse` → 匹配或构造 meta → `loader.load` → `post_process` 采样切分。`DatasetSyntax` 用 `::`（Hub）、`:`（子集）、`/`（多子集）、`#`（采样条数）四符号，`#500` 表示采样 500 条而非训练步数；为绕开 key 形态不一，另建 `(来源, id)` 二级索引反查，未注册 id 兜底为空 `DatasetMeta` + `AutoPreprocessor`。本讲止于「数据进门」，字段清洗/编码/打包归后续预处理器与编码讲义。
