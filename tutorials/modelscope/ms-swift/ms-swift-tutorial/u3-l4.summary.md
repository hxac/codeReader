本讲承接 u3-l3 纯文本 Template，扩展到多模态，核心是「像素如何变成 token」。三大模块：一是**占位符机制**，区分框架切分文本用的 `special_tokens`（`<image>`/`<video>`/`<audio>`）与模型词表真实占位 `placeholder_tokens`（如 `<|image_pad|>`）；经四阶段——`remove_messages_media` 解析 content、`replace_tag` 替换为单个占位 token、`_extend_tokens` 按图像网格 `image_grid_thw` 扩展成 N 个、前向 `masked_scatter` 用视觉 embedding 填充；占位段 `loss_scale=0` 且截断时受 `protected` 保护，关键约束是占位 token 数须严格等于视觉 embedding 数。二是 `vision_utils` 媒体加载，`load_file`/`load_image` 统一本地/HTTP/base64 三源为 PIL 图，`rescale_image` 配合 `max_pixels` 控显存；grounding 用 `norm_bbox='norm1000'` 使坐标与分辨率解耦。三是 `support_padding_free` 约束：多模态位置编码重算复杂，默认不支持 padding_free/packing，靠该属性显式声明，`None` 时按 `not is_multimodal` 推断，强行开启则报错。
