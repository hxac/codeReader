本讲讲解 ms-swift 的部署与服务化，含三个最小模块。①启动链路：`swift deploy` 经 CLI 路由到 `SwiftDeploy(args).main()`，它继承 `SwiftInfer` 复用模型/模板/引擎装配，`__init__` 构建 FastAPI 并由 `_register_app` 注册端点，`run()` 用 `uvicorn.run` 常驻成 HTTP 服务——与跑完即退的 `swift infer` 的区别仅在 `run`；`SwiftDeploy` 仅覆写 `get_infer_engine`（vLLM data-parallel 校验）与 `run`，是钩子式扩展的范例。②OpenAI 兼容协议为双层结构：外层 `ChatCompletionRequest`/`CompletionRequest`/`EmbeddingRequest` 经 `parse()` 拆成内层 `(InferRequest, RequestConfig)` 交 `infer_async`，响应 `ChatCompletionResponse` 字段与 OpenAI 对齐并扩展；completion/embedding 复用 chat 主链路。多 LoRA 路由以 `request.model` 作 `adapter_mapping` 的 key 选出 adapter 包成 `AdapterRequest`，一份基座可同时服务多个 LoRA；流式输出用 SSE。`InferClient` 是上述过程的逆操作，使本地推理与调远端服务对上层透明。③`swift app` = `run_deploy`（用 spawn 起子进程拉部署、轮询就绪、退出终止）+ Gradio，由 `--base_url` 决定自带/外接模式。部署模式的可观测性靠 `strict=True`、`InferStats` 与可选 `JsonlWriter`。
