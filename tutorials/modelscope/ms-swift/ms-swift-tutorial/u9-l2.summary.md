本讲讲解 ms-swift 的序列并行（Sequence Parallel）能力，专治长文本训练中注意力显存随序列长平方爆炸的问题；它切的是序列/头维，与 DDP/DeepSpeed（切 batch/参数）正交。核心由三个最小模块构成：其一，`--sequence_parallel_size` 开关触发 `sequence_parallel.prepare()`，完成 DP×RP×SP 三维 device mesh 划分，并对 flash-attn、forward hook、MoE aux loss 打全局猴子补丁，模型代码无需改动；其二，**Ulysses** 在注意力头维切分，靠 all-to-all 把「序列满/头分」转置为「序列分/头满」，反向近乎免费，硬约束是头数须能被 SP 卡数整除；其三，**Ring-Attention / zigzag** 在序列维切分，靠环形 P2P 传 K/V 配合在线 softmax（`update_out_and_lse`）累加，无头数约束但强制依赖 `padding_free` 的 `cu_seqlens`。关键结论是 `sp_world_size = gcd(num_heads, world_size)`，多余并行度交由 `rp_world_size`，从而自动解除「头数必须被卡数整除」的约束——这也是 GQA（KV 头数少）模型能组合使用两者的根因。引入的关键术语包括 SP/RP/DP 组、`cu_seqlens`、zigzag、`num_key_value_heads`、all-to-all。
