本讲是「分布式与高性能训练」单元首篇，仅覆盖数据并行家族（DDP/DeepSpeed ZeRO/FSDP）与朴素模型并行 device_map，张量/流水/序列并行留待后续讲义。三大核心模块：其一，torchrun 多卡启动——swift 命令本质是发射器，由 NPROC_PER_NODE/NNODES 环境变量经 use_torchrun 决定是否套 torchrun，且仅 pt/sft/rlhf/infer 走多进程；进程拓扑统一由 get_dist_setting() 从 torchrun 注入的环境变量解析，is_dist/is_master/is_mp 均建立其上。其二，DeepSpeed 接入——_init_deepspeed 把 zero2/zero3/*_offload 简写展开为 swift/config 下 JSON dict，并与 device_map 互斥；ZeRO 遵循显存换速度连续谱：zero2 参数常驻最快、zero3 连参数切分更省、*_offload 卸载至 CPU 最省最慢。其三，FSDP2/device_map——FSDP2 是 ZeRO-3 的 PyTorch 原生替代，与 DeepSpeed、device_map 均互斥；device_map 按层切模型，is_mp 判定为每进程分到≥2 卡，框架用 patch_mp_ddp 让其与 DDP 混用并修补优化器设备错位。
