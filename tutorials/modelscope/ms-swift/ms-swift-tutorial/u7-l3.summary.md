本讲承接 u7-l2，下沉到「奖励分数从哪里来」。ms-swift 以统一契约收敛三类异质奖励：规则型奖励函数（ORM/AsyncORM，纯 Python 打分）、判别式奖励模型（带 value head 的分类网络）、生成式奖励模型（LLM-as-judge）。三者均返回等长 List[float]，由 compute_rewards_per_func 统一调度成 [N, n_funcs] 张量，再以 reward_weights 加权 nansum 成单列汇入 advantage。关键术语与注册表：规则奖励查 orms（基类 ORM；PRM 仅用于采样、不支持 GRPO）；奖励模型查 rm_plugins，含 default（取 logits[:,0] 当 reward）与 genrm（裁判模型按「Reward: x」格式输出后正则抽分）。核心结论：数据集列经 to_reward_row→rows_to_batched 自动铺成 kwargs 流入奖励函数；奖励返回 None 会被替换为 NaN 并在 nansum 中被忽略，全 None 才告警；判别式 RM 经 _prepare_rewards 追加进 reward_funcs 末尾，故 reward_weights 顺序为 [reward_funcs, reward_model]；自定义奖励只需写进 orms 并用 --external_plugins 加载。
