本讲是「推理引擎」单元首篇，阐明 ms-swift 用统一抽象屏蔽多后端差异。核心是三层继承：纯抽象基类 BaseInferEngine 只声明同步批量 infer() 与异步单条 infer_async() 两接口；InferEngine 提供吃 template、批处理、metrics 等共享实现，子类只需实现 infer_async 即获 infer；Transformers/Vllm/Sglang/Lmdeploy 等本地引擎与走 HTTP 调 OpenAI 兼容服务的远端 InferClient 共守同一契约，使本地推理与调远端服务对上层透明。输入拆成正交两件套：InferRequest 承载 messages 与多模态资产（描述「问什么」），RequestConfig 承载 max_tokens/temperature/stream 等采样参数（描述「怎么答」），HTTP 层的 ChatCompletionRequest 经 parse() 拆解回二者。输出 ChatCompletionResponse 结构对齐 OpenAI Chat Completions，并扩展 prompt_token_ids、token_ids 等字段，为评测、GRPO rollout 与服务化部署奠基。
