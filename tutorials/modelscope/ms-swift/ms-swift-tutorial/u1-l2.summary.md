本讲聚焦「如何把 ms-swift 装起来以及安装背后发生了什么」，承接上一讲对项目能力矩阵的全局认知。它拆成三个最小模块：一是 `setup.py` 的依赖解析与入口注册，自定义函数 `parse_requirements` 将 `requirements.txt`（通过 `-r` 递归指向 `framework.txt`）解析为核心依赖 `install_requires`，并经 `console_scripts` 注册出 `swift`（指向 `swift.cli.main:cli_main`）与 `megatron` 两个命令；二是 `requirements/` 的「核心最小 + 可选分组」策略，extras 分组对应 `megatron/eval/ray/swanlab` 等文件，其中 `[all]` 等于核心加 eval/swanlab/ray，但**不含 megatron**，后者需单独安装或参照 `install_all.sh` 手动编译；三是运行环境版本矩阵，版本约束散见于 `setup.py` 的宽松下限、`framework.txt` 的硬窗（如 transformers<5.13、peft<0.20、trl<1.0）与官方文档推荐值，冲突时以 `framework.txt` 硬约束为准。本讲引入的关键术语包括 `parse_requirements`、`extras_require`、`entry_points`/`console_scripts`、`ROUTE_MAPPING` 路由、`framework.txt` 核心依赖集，以及核心结论：验证安装应使用 `swift sft --help`（子命令 + argparse）而非 `swift --help`（后者因 `--help` 非 `ROUTE_MAPPING` 合法键而抛 `KeyError`）。这些为后续讲义拆解 CLI 命令分发奠定基础。
