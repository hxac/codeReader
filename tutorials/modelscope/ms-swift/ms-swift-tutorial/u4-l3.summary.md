本讲承接 u4-l2，讲清规范化后的 `messages` 如何变成模型可用的 token 序列，以及如何消除无效 padding、提升数据效率。

核心模块有三：其一，`EncodePreprocessor` 把编码能力挂到 `RowPreprocessor` 流水线上，统一入口 `template.encode(row, return_length=True)` 产出 `input_ids`/`labels`/`loss_scale` 及 `lengths`；`AddLengthPreprocessor` 则只追加 `lengths`、不固化 token。其二，Packing 在数据预处理阶段用装箱函数 `calculate_matched_group` 把多条短样本拼成接近 `packing_length` 的长样本，提供 `binpack`（best-fit-decreasing，打乱顺序）与 `sequential`（保序 next-fit）两种策略；`PackingDataset` 多进程装箱并广播到各 rank。其三，padding_free 在 collator 阶段用 `packing_row` 把 batch 压平、`position_ids` 每条从 0 重置，依赖 flash-attn varlen 实现因果隔离，消除 batch 内 padding。

关键结论：packing 必然强制开启 padding_free，二者绑定；`LazyLLMDataset` 是运行时容器，取数时延迟编码并跳过 `MaxLengthError` 坏样本；packing/streaming 与 `lazy_tokenize` 互斥。
