本讲讲解 ms-swift 两条 Gradio 可视化入口。`swift web-ui` 是覆盖训练/RLHF/GRPO/推理/导出/评测/采样的全功能控制台，二者均不走 torchrun 多进程；`swift app` 是围绕已部署模型的对话试用台。核心机制是 `BaseUI` 基类的「元数据驱动 + 猴子补丁」：以 `elem_id`（=参数名）从 `choice_dict`/`default_dict`/`arguments` 三表查候选值、默认值与文案，表单直接镜像 `RLHFArguments` 等 dataclass 字段，避免 UI 与参数定义漂移。`build_ui` 用模板方法编排：设全局 `builder`/`base_builder` → `do_build_ui` 布局 → 递归 `sub_ui` → 根 Tab 绑 `after_build_ui`；`update_input_model` 据模型路径区分本地 checkpoint（`args.json` 全字段回填）与 Hub 模型（仅回填 `template`/`model_type`/`system`）。提交时 `LLMTrain.train` 把表单值翻译成 `swift sft/rlhf` 命令以子进程后台启动，UI 与训练解耦。`swift app` 则用 `run_deploy` 拉起 OpenAI 兼容服务，再以 `InferClient` 跑流式聊天，是 u8-l2 部署能力的前端壳。
