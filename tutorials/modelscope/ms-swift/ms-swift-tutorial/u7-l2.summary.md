本讲拆解 ms-swift 中 GRPO 算法核心，覆盖数据结构、奖励聚合、训练循环三大支柱。GRPO 对同一 prompt 采样 K 条回答，用组内均值/方差代替 PPO 的 critic，省去价值网络，这是其免 critic、省显存的根本原因。引入 `OnPolicySample`/`GRPOSample`/`GRPOBatch` 三类数据结构作为 RL 信号载体，子类在通用基类上追加 `rewards`/`advantages` 字段。奖励侧由 `compute_rewards_per_func` 统一处理同步函数、async 协程与判别式奖励模型三类异质打分者，产出 `[N, n_funcs]` 矩阵，经 `reward_weights` 加权（`None→NaN`+`nansum` 容错）、再组内去均值除 std 得到 advantage。训练循环为固定五阶段骨架 `_generate_and_score_completions`（rollout→score→prepare batch→postprocess→log），advantage 因依赖全局奖励和编码 batch 延迟至 postprocess 计算；loss 为 PPO 风格 clip 策略梯度 `-min(ρA, clip(ρ)A)`，叠加可选 KL 惩罚。核心设计是算法层（纯 PyTorch）与训练器层解耦，支持 HF/Megatron/Ray 多后端复用。
