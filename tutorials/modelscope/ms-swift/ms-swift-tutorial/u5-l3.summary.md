本讲深入 `swift/tuners/` 实现层，讲清轻量微调（PEFT）方法的实现与注册。核心是 `SWIFT_MAPPING` 注册表：每类方法由「配置类 + 实现类」配对，以 `config.swift_type` 为 key 派发；`SwiftModel._prepare_model` 先冻结全模型再挂载增量。`SwiftConfig`/`SwiftAdapter`/`SwiftOutput` 三基类规定统一契约，后者经保存、标记可训练、优化器分组三个回调回传。重点剖析 LoRA：用低秩分解 ΔW=(α/r)BA 构造增量，B 清零保证初始输出不变，核心参数 `lora_rank`、`lora_alpha`、`target_modules` 在 `prepare_adapter` 翻译为 config。ms-swift 有两个 LoRA 配置类——`LoraConfig`（peft 后端，默认）与 `LoRAConfig`（swift 后端），按是否为 `SwiftConfig` 分流到 `PeftModel` 或 `SwiftModel`；两套 checkpoint 可经 `swift_to_peft_format` 互转，`hot_patch_peft_module` 在导入时给 peft 打增强补丁。另对比 LLaMAPro（插入新层并零初始化 o_proj/down_proj）与 Adapter（猴子补丁 forward 加瓶颈 MLP），与 LoRA 代表三种干预粒度。
