本讲解析 ms-swift 的 Template（对话模板）体系，即 messages 对话与 token 序列之间的翻译层。核心模块有四：一是 `TEMPLATE_MAPPING` 注册表与 `register_template`，以「导入即注册」范式收录各模型对话格式，重名默认报错防呆；二是 `TemplateMeta`「格式配方」，用 prefix/prompt/chat_sep/suffix/system_prefix 等字段描述纯文本拼接规则，`__post_init__` 派生 support_system/support_multi_round 能力标记，`init(tokenizer)` 把 `'eos_token_id'` 等字符串占位解析为真实 token id；三是 `get_template`/`get_template_meta`，按优先级决出模板名（显式 `--template` > 模型目录 args.json > `ModelMeta.template` > candidate_templates，歧义时报错）并实例化 Template；四是 `Template` 基类的 encode 主链路（encode → _encode_truncated → _encode → _swift_encode → _encode_context_list），把对话切成上下文段，由 `LossScale` 按 `ContextType`（RESPONSE/SUFFIX 为回答与结尾、OTHER 为 system/外壳/分隔）赋 0/1，据此对非回答段填 `-100`、回答段保留真 label，从而实现「只在 assistant 回答上算 loss」，并通过 `_add_dynamic_eos` 让结尾停止符也参与训练使模型学会停止；非训练模式下 labels 为 None。关键术语：TemplateMeta、TEMPLATE_MAPPING、ContextType、loss_scale、base_strategy（default/last_round/all）、data_collator、padding_free。
