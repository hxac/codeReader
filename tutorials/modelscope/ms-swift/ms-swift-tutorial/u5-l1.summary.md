本讲讲清 ms-swift 训练器体系的派发与装配。核心是 `TrainerFactory`：它用 `TRAINER_MAPPING` 与 `TRAINING_ARGS_MAPPING` 两张键对齐的字典，以 `task_type`（RLHF 时优先取 `rlhf_type`）为路由键，靠字符串路径加 `importlib` 动态导入派发出 Trainer 类与 TrainingArguments 对象，以此保护懒加载。生成式 `causal_lm` 用 `Seq2SeqTrainer`（继承 `SwiftMixin + DataLoaderMixin + HfSeq2SeqTrainer`），判别式 `seq_cls`/`embedding`/`reranker` 用 `Trainer` 及其子类，共享 `SwiftMixin`+`DataLoaderMixin` 公共底座。Trainer 以 template 为中心装配：`data_collator`、前向上下文与 `compute_sft_loss` 均委托给 template，自身仅管 MoE aux loss、acc、loss_scale 错位等上层逻辑。`task_type` 由 `get_model_info_meta` 决出（显式优先于 `num_labels`，默认 `causal_lm`），经 `_init_model_info` 回填到 `args.task_type`，构成「注册→加载决出→参数回填→查表派发」一条链路。`patcher.py` 则在 `swift.trainers` 包 import 时以猴子补丁全局替换 HF 默认回调，为所有 Trainer 注入显存/速度日志与「最后一步强制 save」行为。
