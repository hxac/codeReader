本讲讲解 ms-swift 三类训练扩展点——Callbacks、Optimizers、Metrics，三者均遵循 u1-l3 的「三件套」范式：抽象基类 + *_map 注册表 + 一个 CLI 开关，基类构造统一为 (args, trainer)。Callbacks 继承 HF 的 TrainerCallback，钩子名与 HF 完全一致，由 TrainArgumentsMixin._init_callbacks 根据其它参数自动注入 lisa/adalora/early_stop/activation_cpu_offload，其余在 SwiftMixin._get_callbacks 查表实例化后经 callbacks= 交 HF 调度。Optimizers 由 OptimizerCallback 用 create_optimizer_and_scheduler/create_optimizer/create_scheduler 三方法定义契约，SwiftMixin 把建优化器责任整体委托给它，子类按需覆写（multimodal/lorap/muon 覆写建优化器，galore 同时定制调度器）。Metrics 关键区分两条通路：opt-in 的 eval_metrics_map（EvalMetrics 产出 HF 的 compute_metrics 与 preprocess_logits_for_metrics，仅验证时调用，限 sft/pretrain/reranker/embedding）与始终在线的 custom_metrics（MeanMetric 累加器，跨 rank all_reduce，产出 train_acc/eval_acc）。--external_plugins 是不改源码即可把自定义类塞进任意 *_map 的二次开发统一入口；三类扩展都挂在 SwiftMixin 上，与 TrainerFactory、patcher 共同构成 Trainer 可扩展骨架。
