本讲讲解如何把业务数据接入 ms-swift，含三个核心模块。一是格式规范：下游统一为 messages 列表，每条 message 仅允许 role/content/loss/loss_scale 四键（由 _check_messages 强制）；AutoPreprocessor 按列名嗅探，自动把 messages、sharegpt、query-response、alpaca 四种原始格式清洗成标准形态，故常可用 --dataset 直接喂文件零注册。二是注册机制：嗅探不出或需别名复用时，用 --custom_dataset_info（JSON 清单）或 --external_plugins（Python 代码，可挂自定义预处理器）；register_dataset_info 经 _preprocess_d_info（摘 columns、选预处理器、解析路径、递归 subsets）构造 DatasetMeta 写入 DATASET_MAPPING，内置清单在导入时自动注册一次。三是多轮训练控制：loss（布尔，仅 assistant 生效）与 loss_scale（浮点权重）提供逐轮逐段的 loss 控制，字段优先级高于命令行 --loss_scale 策略，loss_scale>1 须配 --is_binary_loss_scale false。
