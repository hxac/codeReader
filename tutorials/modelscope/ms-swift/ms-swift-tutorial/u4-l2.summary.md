本讲讲 ms-swift 预处理器（Preprocessor）：位于数据加载（u4-l1）与编码成 token（u4-l3）之间，职责是把千奇百怪的原始列统一成标准 messages 结构，这是 Template 体系唯一认识的输入。核心是模板方法基类 RowPreprocessor，编排列映射→逐行调用子类钩子 preprocess(row)→四道清洗→容错丢弃坏行，子类只覆盖此钩子。列映射分两层：用户显式 columns（优先）与子类别名表（仅 enable_auto_mapping=True 时生效）；safe_rename_columns 大小写不敏感、对多源同目标放弃重命名。三大内置预处理器产出同构 messages：ResponsePreprocessor（单轮问答/老式 history，靠 history_to_messages）、AlpacaPreprocessor（拼接 instruction+input 后继承 Response）、MessagesPreprocessor（角色名归一，兼容 sharegpt/DPO）。AutoPreprocessor 凭列名按 messages→alpaca→response 优先级自动选型，是 DatasetMeta.preprocess_func 默认值，故未注册数据也能被清洗；standard_keys 为允许带走的列白名单，remove_useless_columns 在 loader 收尾清列。
