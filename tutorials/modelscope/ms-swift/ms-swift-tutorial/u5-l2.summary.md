本讲聚焦 ms-swift 中「如何把加载好的基座模型改造成可训练的微调模型」，即 `tuner_plugin` 模块。核心是 `Tuner` 基类用三个静态方法 `prepare_model`/`save_pretrained`/`from_pretrained` 定义「挂载—保存—加载」统一契约，使上层 pipeline 与 trainer 能用同一套代码处理各种微调方式；`PeftTuner` 作为中间层提供通用 peft 序列化默认实现，子类只需覆写 `prepare_model`。关键概念包括：`tuners_map` 注册表与「双轨制」分派——仅 `ia3`/`lora_llm`/`dummy` 三个需自定义序列化的 tuner 进表，其余（lora/llamapro/vera 等）走 `prepare_adapter`→`Swift.prepare_model` 复用 peft 默认逻辑；`TunerMixin.prepare_model` 为真正指挥者，依 `is_adapter`（即 `tuner_type != 'full'`）在「先冻结再挂载增量」与「全量解冻后选择性冻结」两条主线间分流；adapter 微调省显存的根基是「先 `requires_grad_(False)` 全冻再只放开增量」，`--target_modules all-linear` 经 `find_all_linears` 或 `get_multimodal_target_regex` 展开。trainer 保存时据 `tuner_type in tuners_map` 分派，解释了 LoRA checkpoint 为何只产 `adapter_model.safetensors`，而 `lora_llm` 额外产 `vit.safetensors`。
