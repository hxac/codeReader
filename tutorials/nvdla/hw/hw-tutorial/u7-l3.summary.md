`cmod/` 是 NVDLA 用 SystemC/TLM 写的参考模型（reference/golden model），与 `vmod/` RTL 描述同一份规格，但用事务级 `b_transport` 代替逐拍信号翻转，更快而舍精确时序，编译成 `libnvdla_cmod.so`。它承接 trace-player 测试台（u7-l1/l2 验时序与可运行性），作为并行软件模型在数值层为 RTL 提供期望值。`NV_NVDLA_core` 是中央骨架：例化全部引擎（bdma/cdma/csc/cmac_a·b/cacc/sdp/pdp/cdp/mcif/cvif/glb/rubik）并用 TLM socket 连线，拓扑与 RTL 分区同构；顶层 `NV_nvdla` 包住 core 与 CSB/AXI 适配器（`NvdlaCsbAdaptor` 替代 apb2csb），工厂函数为 `extern "C" NV_nvdlaCon`。各引擎统一为「base（端口）+ reg_model（寄存器）+ 手写功能核」三层，用 `SC_THREAD/SC_METHOD` 与 `sc_fifo` 描述行为。黄金比对由宏 `NVDLA_REFERENCE_MODEL_ENABLE` 切换：开启时挂 `internal_monitor` 探针，并接所有 DMA 及卷积/后处理数据通路，与 RTL 逐事务比对存储写。能做位精确比对，靠寄存器（SystemRDL 同源）与浮点（同一 HLS 源）双重同源。诚实边界：cmod 源码完整可构建，但 monitor 实现与 RTL↔cmod 自动比对框架未随仓库开源。
