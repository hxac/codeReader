PDP（平面数据处理器）是 NVDLA 的池化引擎，在每个通道平面内做空间池化（average/max/min），不跨通道、不跨 batch，承接 CACC→SDP→PDP 数据链，支持 int8/int16/fp16。其核心是把二维池化可分离地拆成水平 cal1d＋垂直 cal2d 两级：对 max/min 严格等价，average 两段只求和，除法（实为乘软件预算的定点倒数 RECIP_KERNEL_*）仅在 cal2d 末端做一次，从而省去除法器。算子由 pooling_MAX/MIN/SUM 经 pooling_fun 按 pooling_type 派发，靠精度使能位复用同一硬件；cal2d 用行缓冲缓存 kernel_height 行再做垂直聚合。配置字段为 0-based，即 pooling_size_v=cfg+1，故 2×2 池化字段写 1。数据通路为 PDP_rdma（ig→cq→eg）读入 → nan/preproc → cal1d → cal2d → PDP_wdma 写回 MCIF/CVIF；flying 模式下输入由 SDP 直送、不进 RDMA。RDMA 拥有独立 CSB 端口（基址 0xC000），WDMA 与 core 共用主配置（0xD000）。层完成时 WDMA 以 reading_done_flag 等 RDMA 收尾后认定 wdma_done，再经 intr_fifo 按 interrupt_ptr 交替点亮 pdp2glb_done_intr_pd[1:0] 双脉冲上报 GLB 的 done_source，对应 producer/consumer 两个影偶组。
