GLB 是 NVDLA 的「中断汇总台」，位于中央枢纽 partition_o，顶层 NV_NVDLA_glb.v 例化三个子模块：u_csb（CSB 接口适配，内嵌自动生成的寄存器文件 u_reg）、u_fc（故障控制器，回 0 占位）、u_ic（中断控制器，核心）。8 类引擎（sdp/cdp/pdp/bdma/rubik/cdma_dat/cdma_wt/cacc）各带 2 个影偶组，共 16 个中断源，被 ic 拼成 16 位 done_source[15:0]；mask/set/status 三类寄存器按此 16 位一一对应。三者语义：INTR_MASK（0x4，写 1 屏蔽，复位全 0 即默认放行）、INTR_SET（0x8，只写、写 1 置位、读回 0）、INTR_STATUS（0xc，读得 pending、写 1 清除 W1C）。核心算式 core_intr = OR(~mask & status)。status 是软硬件双源寄存器（硬件 done 置位、软件 SET 置位、软件 W1C 清除），故留在手写的 ic 而非自动生成的 reg 文件中，避免双写冲突。core_intr 先在 core 域寄存一拍，再经 NV_NVDLA_sync3d_c 三级同步链跨到 falcon 域输出，最终接顶层 dla_intr；为电平有效信号设计。两条断言守护：单引擎不得同周期报两组完成、不得同周期既 set 又 clear。
