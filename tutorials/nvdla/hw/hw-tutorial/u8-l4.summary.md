本讲为手册收尾篇，讲清两件事：如何把各引擎串成一次真实网络层推理，以及如何把 IP 集成进 SoC。其一是端到端编程序列：一层生命周期为「预装数据→配置各引擎→逆流水线点火→轮询并清 GLB done 中断」，CPU 仅写寄存器；点火须逆流水线——下游 SDP 先就绪、源头 CDMA 最后点火，使整条流水一次点亮无空泡，各引擎寄存器基址由 csb_master 地址译码决定。其二是影偶（shadow）配置与 done 中断协作：操作参数分 group0/group1 两份，CPU 写 producer 组不影响运行中的 consumer 组；done 为 2 位、对应两个影偶组，完成即无缝切组、双缓冲不停顿；8 类引擎×2 组共 16 个 done 源在 GLB 的 u_ic 聚合为 core_intr=OR(~mask & status)，顶层仅一根 dla_intr。其三是 SoC 集成：需接配置口（CSB 或经 apb2csb 桥的 APB）、两组 AXI memif（DBB 片外、CVSRAM 片上）、两个时钟域、复位、按分区 SRAM 电源及单根中断，其余计算由 IP 内部自理。
