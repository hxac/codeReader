本讲下钻到 NVDLA 的 RTL 积木库 vmod/vlibs/，覆盖四类基础原语。sync3d 同步器家族用于单比特跨时钟域（core 域与 falcon 域之间），采用 3 级触发器链降低亚稳态失败率，经四级抽象（叶子 p_SSYNC3DO → 行为 sync3d → 工程 NV_NVDLA_sync3d）实现，并以占位实例名 NV_GENERIC_CELL 供综合统一替换；变体按复位安全默认值选用：_s 默认 1、_c 默认 0（如 GLB 中断 core_intr 跨域即用 _c）。FIFO 配套原语不提供通用 FIFO 单元，而是 nv_assert_fifo（上溢/下溢断言监视器，仿真专用、综合剔除）与 oneHotClk_async_*（异步 FIFO 两侧的 DFT 时钟门控限定符），真正的 FIFO 主体由各引擎自行拼装。MUX2D4 与 BLKBOX（BUFFER/SRC0/SINK）等一句话单元，价值在于为综合约束、DFT 替换与 lint 清理提供具名锚点。库化使综合与时序收敛保持一致，换工艺库只改 vlibs 而引擎层不动；sync3d 在全工程约 38 处、13 文件，集中在分区边界与中断汇总台。
