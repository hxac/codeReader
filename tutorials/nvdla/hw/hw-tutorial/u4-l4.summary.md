本讲介绍 NVDLA 中唯一横跨两套存储的「桥 DMA」引擎 BDMA（Bridge DMA）。它不参与计算，专做存储间批量搬运，把 CPU 从内存拷贝中解放出来。其源端、目的端可各自经 MCIF（接片外主存 DBB）或 CVIF（接片上 CVSRAM）二选一，由 `src_ram_type`/`dst_ram_type` 各一比特决定；本仓库 RTL 中 0 对应 CVIF、1 对应 MCIF，以源码译码为准。BDMA 寄存器页基址为 0x4000。

BDMA 内部是「取数—缓冲—写数」三段流水：`u_load` 消费描述符、游走源端 3D 地址并发读请求；`u_cq`（20 深、161 位上下文 FIFO）解耦取数与写数，缓存已开始搬运但未写完命令的目的端上下文；`u_store` 配合读返回数据发写请求并产生 done；`u_csb` 是寄存器与控制中枢，另含 20 深、289 位 `csb_fifo` 缓存尚未开始的完整描述符快照。

它支持三维搬运模型 cube←surface←line←transaction，`line_size` 以 32 字节为单位，`line_stride`/`surf_stride` 以字节为单位且源、目的可独立配置，从而搬运时可重排布局。

编程模型为「描述符＋双组启动」：写配置寄存器→写 `OP_ENABLE` 把快照入 `csb_fifo`→写 `LAUNCH0/1` 启动并归属 group 0/1。完成中断逐级上报：store 的 `st2csb_grp0/1_done` → `bdma2glb_done_intr_pd[1:0]` → GLB `done_source[7:6]`；两个 group 实现双缓冲不停顿搬运，`free_slot` 指示剩余队列容量。
