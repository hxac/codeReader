本讲拆解 NVDLA trace-player 测试平台围绕 DUT（NV_nvdla）搭起的外围 RTL。TB 顶层是 `tb_top.v` 中名为 `top` 的模块，负责造时钟、发复位、并把 DUT 与外围模型连起来。TB 自产三路时钟：`half_speed_clk`（喂 DUT 的 core/csb 域，仿真里同频简化）、`mem_clk`（喂 AXI slave）、`mem_clk_fast`（喂存储数组）；复位为 active-low，上电 1000 拍后释放。激励分两级：`csb_master_seq`（sequencer）用 `$readmemh` 读入 `input.txn.raw` 逐条回放 trace 命令、输出逻辑请求，`syn_csb_master` 用 5 状态 FSM 将其翻译成符合 CSB 握手的 63 位请求包（含 addr/wdat/write/nposted 字段）。存储模型由 `syn_slave_mem_wrap` 包住 2 个 `syn_axi_slave` 与 2 块 `slave_mem`，分别对应 DBB（`0x8000_0000`）与 CVSRAM（`0x5000_0000`）；`syn_axi_slave` 用 7 个 FIFO 把 AXI 五通道事务解耦、配对、按 id 还原响应，是存储模型"可回压"（throttle）的关键；`slave_mem` 为纯行为存储数组，内置百分比式带宽节流，使仿真逼近真实带宽约束。`zemi3_tb.sv` 则是面向 ZeBu/Cadence 加速后端的 DPI-C 监控变体，默认 VCS 流程的等价职责（结束仿真、存储加载/转储）由 `csb_master_seq` 承担。
