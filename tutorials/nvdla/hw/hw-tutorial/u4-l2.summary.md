本讲承接 u4-l1，深入 NVDLA 主存接口 **MCIF**（Memory Controller InterFace，挂 DBB 的 AXI 接口）的源码实现。核心结论：MCIF 每个方向（读/写）都是 **IG → cq → eg** 三级结构，把多个引擎的 DMA 请求仲裁成一组对外 AXI 事务，再按响应 id 把数据送回正确引擎。读通路有 10 个客户端、写通路有 5 个（只有产出数据的引擎才写）。关键术语：**IG（ingress，入口）**内部分四级——**bpt**（按 256B 行边界切首/中/尾事务并做信用反压）、**arb**（10 选 1 加权轮询）、**spt**（按 AXI beat 拆拍）、**cvt**（生成 AXI AR 并写上下文，受 `os_cnt` outstanding 反压）。**cq（命令队列）**是一块单写口、多读口的 256×7 共享 RAM，靠 255-bit 空闲位图分配/归还，存放 `{ldrop,fdrop,ltran,odd,swizzle,lens}` 上下文，按 `thread_id=axid` 归属。**eg（egress）**按 `rid` demux 到各引擎 lat_fifo，经**纯 RR 仲裁**用上下文做半行 swizzle/drop 重排，输出 514-bit（2 mask + 512 data）读响应。区分 bpt 与 spt 两类拆分、以及 `tieoff_lat_fifo_depth=0` 关闭信用（如 cdma_dat/cdma_wt）是本讲要点。
