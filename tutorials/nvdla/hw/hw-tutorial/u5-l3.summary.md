CDP（通道数据处理器）是 NVDLA 后处理四件套中的“通道维”引擎，核心算子是跨通道 LRN（局部响应归一化），与 SDP（逐点）、PDP（空间池化）的根本区别在于沿通道 C 方向滑动窗口。硬件不直接实现含实数幂的 (k+αΣ)^−β，而是将其拆为三段：sum 求邻域平方和 Σ、lut+intp 用软件在线编程的查表加线性插值逼近 f(Σ)、mul 把 f(Σ) 乘回原始激活 a——sum 与 mul 构成 LRN 的“左右手”。计算数据通路为 cvtin → syncfifo 扇出三路（sum / sync2mul 原始激活旁路 / 旁路输出）→ lut → intp → mul → cvtout。因平方和动态范围大、非线性函数难定点实现，sum/lut/mul 在内部用 fp17 浮点（HLS_fp17_mul）运算，两头由 cvtin/cvtout 按 (in+offset)×scale>>shifter 的 RSC 模型完成定点↔浮点桥接并统计饱和。CDP 占两个 4KB 寄存器页：0xe000 配独立 RDMA、0xf000 配核心/WDMA/LUT，done 中断分两位影偶组上报 GLB。SQSUM_BYPASS 可把 CDP 退化为纯逐点 LUT 激活引擎，体现其复用性。
