本讲拆解 NVDLA 硬件构建系统，回答分散的源码如何被拧成可仿真 RTL。核心结论：构建采用「顶层 Makefile + tmake 编排器 + 每 sandbox 共享 Makefile + 三个生成器」的分层结构。四个最小模块：(1) 首次敲 `make` 交互生成环境配置文件 `tree.make`（含 `PROJECTS` 与工具路径），全树共享；(2) `tools/etc/build.config` 是 YAML 写的依赖 DAG，登记每个 sandbox 与依赖关系，是仓库构建骨架；(3) Perl 脚本 `tmake` 读 `build.config`，用「反复摘叶子」的深度优先拓扑排序逐个 sandbox 喊 `make`，本身不编译；默认 `--build verif` 走 VCS，不构建 `verilator`；(4) sandbox 内由 defgen→vcp→eperl 流水线加工 Verilog：defgen 把 `%define` 翻译成多后端宏头 `project.h/.vh`，vcp 借 cpp 安全展开 `#ifdef`，eperl 执行注释里内嵌 Perl 生成重复 RTL。关键术语：TOT、sandbox、target、tree.make、build.config、defgen/vcp/eperl。根本规律：模块例化关系决定构建顺序——cdma/csc/cmac/cacc 必须先于 `vmod_nvdla_top` 编译。
