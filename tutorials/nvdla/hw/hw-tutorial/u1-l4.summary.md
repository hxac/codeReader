本讲聚焦 `verif/sim` 这个仿真 sandbox，讲清楚如何跑通第一个 sanity 仿真。核心结论：仿真标准两步为 `make build`（用 VCS 把 DUT 顶层 `NV_nvdla` 加测试平台编译成 `simv`，并用 `checkcompile.pl` 校验）和 `make run TESTDIR=...`（准备结果目录、生成激励、跑 `simv`、检查结论），结果落在 `verif/sim/<testname>/test.log`。引入的关键术语：DUT、Testbench（默认 `synth_tb`）、激励/trace（文本 `input.txn`）、CSB 配置总线、VCS、trace-player——即 `inp_txn_to_hexdump.pl` 把 `input.txn` 转成定宽十六进制 `input.txn.raw` 供 sequencer 回放，支持 `write_reg/read_reg` 等七类命令；`sanity0` 是最小冒烟 trace（读默认值→写魔数→读回校验）。`DUMP=1 DUMPER=VERDI` 开 fsdb 波形（`debussy.fsdb`）。`tools/bin/run_sanity` 是 `tmake` 的薄包装，一键构建整树并从 `build.log` 汇总。`checktest` 系列把日志判为 PASSED/FAILED/RUNNING/NOTRUN：`checktest.pl`（sv_tb）认 UVM 消息，`checktest_synthtb.pl`（默认 synth_tb）认 `ERROR` 关键字加 dump 比对。
