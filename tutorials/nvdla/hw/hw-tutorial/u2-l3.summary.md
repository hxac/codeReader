本讲以 CDMA 为主、GLB 为对照，解析 NVDLA 引擎寄存器口的内部结构。核心结论：由 RDL/Ordt 自动生成的寄存器文件（`_CSB_reg/_single_reg/_dual_reg.v`）都遵循统一四段式模板（地址译码、输出拼装、读多路选择、触发器声明）与统一读写接口 `reg_offset/reg_wr_en/reg_wr_data/reg_rd_data`；真正调度影偶（shadow）切换的大脑是手写的 `_regfile.v`。

引入的关键术语：`single_reg` 装即时配置（POINTER/STATUS/ARBITER），`dual_reg` 装逐层操作参数并被例化两份（d0/d1）轮换；POINTER 的 bit0 = producer 是 CPU 写入哪组的开关，bit16 = consumer 是引擎当前使用哪组的只读镜像。

影偶机制：引擎跑第 N 层时 CPU 把第 N+1 层参数填入另一组并写 OP_ENABLE 点火（op_en）；完成时引擎拉 done，regfile 同步翻转 consumer、清本组 op_en、按 consumer 切换输出选通，引擎无缝接跑、无空泡。写保护（已点火组禁写）与 STATUS 三态（空闲/运行/待命）防止 CPU 覆盖运行中的配置。
