本讲解决松散对象「文件数爆炸」问题：git 把大量对象拼接进单个 `.pack` 文件，配 `.idx` 索引按哈希定位，并用 **delta 压缩**消除对象间冗余。三个最小模块：

**pack 文件格式**：三段式——12 字节文件头（`PACK` 签名 + 版本 + 对象数）、一串对象记录、末尾哈希校验尾。每条记录是「变长头（类型占 3 位 + 变长 size）+ zlib 压缩内容」，对象身份是它在文件中的字节 **offset**；写入侧在 `pack-write.c`，经 `hashfile` 边写边算哈希。

**`.idx` 索引（v2）**：由魔数头、256 项 **fanout 表**、排序哈希表、CRC32 表、32 位偏移表（高位为 1 则跳 64 位大偏移表）、pack 哈希与 idx 哈希组成。fanout 表把二分范围缩到同一首字节桶，使查找接近 O(log n)；读取走「哈希→序号→偏移」两步。

**delta 压缩**：`OFS_DELTA`（相对偏移、省空间、base 须同 pack 在前）与 `REF_DELTA`（哈希、灵活、宜 thin pack/网络）。还原一个 delta 对象须沿 **delta 链**向下钻孔到完整 base、再向上逐层 `patch_delta` 打补丁，并有 `delta_base_cache` 与 `pack.depth` 平衡体积与速度。pack 与 idx 经末尾校验哈希相互绑定，打开时交叉校验。
