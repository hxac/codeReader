本讲讲命令执行前「找到所在仓库」的逻辑（全在 setup.c）。入口 setup_git_directory（找不到即 die）与温和版 setup_git_directory_gently，均委托私有主循环 setup_git_directory_gently_1：从 cwd 向上逐层探测，每层依次试 .git 是否为 gitfile、.git 目录、或当前目录本身 bare，并以 GIT_CEILING_DIRECTORIES 与文件系统设备号设两道边界；结局统一编码进 enum discovery_result，正数成功、负数失败。.git 为文件时由 read_gitfile_gently 解析「gitdir: 路径」格式并校验目标；is_git_directory 以合法 HEAD 加可访问 objects/refs 为仓库最低判据。bare 仓库受 safe.bareRepository 策略约束，ensure_valid_ownership 配 safe.directory 防御可疑属主。结果经 set_git_dir_1 写入 GIT_DIR、纳入 local_repo_env 供后续命令继承。核心术语：cwd、gitdir、bare 仓库、worktree、prefix、gitfile。本讲只解决 gitdir 定位，struct repository 内部字段留待下讲。
