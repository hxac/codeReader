本讲承接 u5-l1 的 `ref_store` 虚表抽象，回答「引用以何种格式存盘」。它覆盖三个后端模块：**files** 后端用「松散引用（`.git/refs/` 下的文件）+ `packed-refs`」两层结构，读路径遵循**松散优先、packed 兜底**，遍历时用 `overlay_ref_iterator` 合并两层；**packed** 后端只管单一文本文件，靠特性头（`peeled/fully-peeled/sorted`）与 `^` 剥皮行描述内容，用 mmap **快照**配 `stat_validity` 缓存，能力受限（无 symref、无 reflog），且并非用户可选后端，而是 files 内嵌的 `packed_ref_store` 子组件；**reftable** 后端是为超大仓库设计的二进制块格式，靠排序分块、前缀压缩、不可变文件、**栈式追加**（`tables.list`）、**tombstone 逻辑删除**与自动压缩，实现近常数时间查找与原子更新。三者经 `struct ref_storage_be` 虚表接入，注册表 `refs_backends[]` 仅 files/reftable 两槽，按 `enum ref_storage_format` 选，选择发生在 `git init`（`--ref-format`/`init.defaultrefformat`/`GIT_DEFAULT_REF_FORMAT`/`feature.experimental`），写入 `extensions.refstorage`，默认 files，Git 3.0 计划默认 reftable。事务与更新机制留待 u5-l3。
