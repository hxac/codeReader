本讲讲解 git diff 的文件级引擎 diffcore。diff 分两层：diffcore（本讲）决定文件如何配对与状态判定，xdiff（下一讲 u8-l2）在已配好的对上生成行级 `@@` 补丁。核心数据模型为三件套：`diff_filespec`（单文件 oid/path/mode）、`diff_filepair`（新旧一对 one/two）、`diff_queue_struct`（队列，全局工作台 `diff_queued_diff`）。整条管线为：上游经 `diff_addremove`/`diff_change` 喂料入队，`diffcore_std` 依次跑 skip-stat-unmatch、break、rename、merge-broken、pickaxe、order、rotate、resolve、apply-filter，再由 `diff_flush` 按 raw/name-status/stat/patch 输出。`-B` 用「拆分（删+增）」与「合并（仅删）」两套阈值把彻底重写先拆后合；`-M`/`-C` 按 OID 精确、基名、delta 模糊三步检测重命名，以 `rename_limit` 等护栏控成本。状态码 A/D/M/R/C/T/U 与相似度分数（内部 0–60000，显示如 `R100`）由 `diff_resolve_rename_copy` 统一赋值。
