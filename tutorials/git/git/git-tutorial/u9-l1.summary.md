本讲以「`git add`/`update-index` 如何把工作树变更同步进索引」为核心，回答暂存的底层机制。建立的关键概念有：git 三层数据模型（工作树、索引、对象数据库）；「暂存」等价于两步原子操作——把内容写成 blob 对象，再在索引中新增/替换一条 `cache_entry`（记录路径、模式、哈希与 stat 快照）；以及 porcelain 与 plumbing 的区分。`git add` 是 porcelain，分两条路径：已跟踪文件的修改/删除由 `add_files_to_cache` 经 diff 引擎（`run_diff_files` + `update_callback`）驱动；新文件由 `fill_directory` 遍历后经 `add_files` 处理；二者最终都落到 `add_file_to_index → add_to_index`。`git add <pathspec>` 隐式等价 `-A`，默认自动处理新增与删除；而 `update-index` 是 plumbing，逐路径处理（链为 `update_one → process_path → add_one_path`），默认不自动加新文件、不自动删。pathspec 是带 magic 语法的小语言（`struct pathspec`/`pathspec_item`），`parse_pathspec` 解析 `glob`/`icase`/`exclude`/`top`/`attr` 与 prefix，`nowildcard_len` 用于廉价预筛。交互式方面：`git add -i`（`run_add_i`）是文件级菜单，跑两次 diff 展示 staged/unstaged 两栏；`git add -p`（`run_add_p`）是 hunk 级裁决，靠 `patch_update_file` 的三态循环（UNDECIDED/USE/SKIP）逐块选择，最终用 `git apply --cached` 应用只含选中 hunk 的补丁，从而实现「半个文件地暂存」。所有暂存都包在索引锁与 `odb_transaction` 事务里以保证原子性。
