本讲解答 git 对象以何名、何格式落盘，含三个最小模块。其一，对象哈希：git 哈希的不是裸内容，而是「类型 长度\0」头部与原始内容拼接的整段字节流，故与 sha1sum 不同；头部既自描述类型与大小，又防对象类型混淆。算哈希与落盘分离，前者只产 OID，后者经对象数据库分发到后端。其二，松散对象存储：一对象一文件，文件名即 OID；目录分片（哈希前 2 位作子目录）防单目录膨胀，内容为「头部+内容」的 zlib 压缩；写入走临时文件、原子硬链接、边压边重哈希自检三保险，读取为逆过程并可重哈希校验完整性。其三，哈希算法抽象：虚表 git_hash_algo 封装 SHA-1/SHA-256 的属性与方法指针并登记于全局数组，上层全经指针间接调用，换算法无需改业务代码；默认仍 SHA-1，底层默认用带碰撞检测的 sha1dc。关键术语：内容寻址、OID、松散对象、目录分片、zlib 压缩、哈希算法虚表，供后续 pack 打包讲义承接。
