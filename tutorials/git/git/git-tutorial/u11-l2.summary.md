本讲承接 u11-l1，只讲协议 v2 的**服务端分发**，含三个最小模块。**协议 v2 命令模型**：v2 改「服务」为「命令式（command-oriented）」，一条连接顺序执行多个命令，ref 通告从无条件全量改为 `ls-refs` 显式按前缀请求，能力从藏于 NUL 后改为独立成段。版本协商双向各算一次：客户端默认 v2（`get_protocol_version_config`），服务端从 `GIT_PROTOCOL` 取客户端声明的最大版本、默认 v0 兜底（`determine_protocol_version_server`），由 `upload-pack.c` 的 switch 分流。**serve.c 能力通告与命令分发**：以 `struct protocol_capability` 注册表做数据驱动分发，三回调 `advertise`/`command`/`receive` 分别决定通告、标记命令、处理普通能力；主循环 `process_request` 仅查表、校验、调 `command->command()`。命令与能力语法分离：命令须用 `command=` 选且仅一个，带 `command` 字段的能力不能当普通能力收。**ls-refs 与 object-info**：`ls-refs` 用 `ref-prefix` 配合 `refs_for_each_ref_in_prefixes` 按需过滤、以 `TOO_MANY_PREFIXES` 防 DoS，附加特性（如 `unborn`）以能力值 `ls-refs=unborn` 通告；`object-info` 只返回对象大小不传内容，默认关闭。关键术语：capability advertisement、command、stateless-rpc、protocol_capability 注册表、object-format 协商。客户端协商算法留待 u11-l3。
