本讲解析 `revision.c` 的历史遍历引擎，它是 `git log`/`git rev-list`/`git shortlog` 等命令的共享底层。核心结论：遍历是在 commit DAG 上做搜索，用户给出的端点分 positive（要展示）与 negative（标 `UNINTERESTING` 作裁剪边界，经 `mark_parents_uninteresting` 向祖先传染）。引擎对外暴露标准四步曲：`repo_init_revisions`（装填 `REV_INFO_INIT` 安全默认值）→ `setup_revisions`（解析参数）→ `prepare_revision_walk` → 循环 `get_revision`，全程状态挂在 `struct rev_info` 上。遍历有五种 walk mode，默认 streaming：用 `prio_queue` 二叉小顶堆「弹一个最新 commit、压回其父亲」滚动维持时间倒序，内存仅与历史前沿宽度成正比；带路径过滤时切到 limited 模式，由 `limit_list` 预先吃完整段历史，靠 `SLOP=5` 吸收时间戳错位做提前终止。关键术语包括对象标志位 `SEEN`/`UNINTERESTING`/`SHOWN`/`ADDED`、`prio_queue`（`compare` 为 NULL 时退化为 LIFO 栈的双模式设计）、generation number（拓扑代数，拓扑遍历入度法的主键，u7-l2 详讲）。最终「是否展示」由 `get_commit_action` 层层 AND 裁决。路径过滤的 `TREESAME` 判定留给 u9-l4/u8-l1。
