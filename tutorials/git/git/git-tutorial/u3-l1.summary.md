本讲（u3-l1）只讲对象在内存中如何表示与分类（哈希与磁盘存储见 u3-l2）。

四种一等对象：blob 记文件内容、tree 记目录清单、commit 记快照（引用 tree 与 parent）、tag 是带说明的指针。类型以 enum object_type 编号，pack 中压成 3 位（TYPE_BITS）故枚举受限；另有 OBJ_NONE（未知，源于零初始化）、OBJ_BAD、及 pack 专用 delta OBJ_OFS_DELTA/OBJ_REF_DELTA（非内存一等对象）。

struct object 是统一基类，载 parsed、type(3位)、flags(29位)、oid；四种结构体把它置于首成员实现 C 风格继承，任一对象指针即可当 struct object*。四个 lookup_* 同构：查表命中则 object_as_type 强转，否则 create_object(alloc_*_node)。object_as_type 三分支为「一致直用、从 OBJ_NONE 升级、不符报错」，commit 升级经 init_commit_node 额外分配全局序号 index；union any_object 按最大尺寸分配，支持类型未知时原地重解释。

flags 是 29 位操作级临时资源，靠 object.h 注释表协调分配，用完由 clear_object_flags 归还、不持久化。
