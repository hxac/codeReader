本讲承接 u2-l2 的 struct repository，回答其 config 指针指向的 config_set 如何被填满，含四个最小模块。一、回调驱动解析：git_parse_source 逐字符线性扫描配置文件，每凑齐一个键值对就回调 config_fn_t，键名规范为「节.子节.变量名」（节与变量名小写、子节保留大小写），解析器经 struct config_source 与数据源解耦，出身信息记入 key_value_info(kvi)。二、多级加载：do_git_config_sequence 按 system→xdg→user→local→worktree→command 顺序喂入，遵循「最后读到的获胜」，include/includeIf 由回调包装层 git_config_include 拦截。三、config_set 缓存：以 hashmap 与有序列表缓存整仓库配置，懒加载于首次查询，单值取末项、多值返回全部，每个值都带 kvi 出身。四、parse-options 独立解析命令行选项，靠 struct option[] 表直接把参数写进 C 变量，与配置文件解析是两套体系；流式与缓存两种用法共用同一套解析器。
