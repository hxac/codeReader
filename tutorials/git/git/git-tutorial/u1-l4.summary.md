本讲只读 `git.c` 一个文件，讲清「命令行字符串如何被翻译成 C 函数」的分发链路。git 的 C 入口是 `common-main.c` 里极薄的 `main()`，它把控制权交给 `git.c` 的 `cmd_main()`，后者依次完成：剥离程序名、处理 `git-xxxx` 硬链接特例、用 `handle_options` 吃掉 `-C`/`-c`/`--git-dir` 等全局选项、把 `--version`/`--help` 改写为子命令，最后进入 `run_argv` 主循环。

核心是一张函数指针表 `commands[]`，每项为 `{命令名, cmd_函数, 选项位图}`，查表靠线性 `strcmp`（`get_builtin`）。表项第三个字段是选项位图，本讲引入 `RUN_SETUP`（须在仓库内）、`RUN_SETUP_GENTLY`、`NEED_WORK_TREE`（须有工作树，禁止 bare 仓库）、`USE_PAGER`、`NO_PARSEOPT`、`DEPRECATED` 等标志位；它们把「需要仓库/工作树/分页器」等横切关注点集中到 `run_builtin` 统一处理，再调用 `p->fn`。

关键术语：`cmd_struct`、函数指针表、`run_builtin`、`prefix`（子目录相对工作树顶的路径）、builtin 内置命令、dashed external 外部命令、alias 别名（含 `!` 开头的 shell 别名）。分发是「内置 → 别名 → 外部命令」三路回退循环：`handle_builtin` 查到即执行退出，查不到静默让位；`handle_alias` 借 `split_cmdline` 拆参数并带环路检测；`execv_dashed_external` 去 `PATH` 找 `git-<cmd>`；全落空时由 `help_unknown_cmd` 用编辑距离给拼写建议。新增 builtin 须同步改实现、`builtin.h` 声明、`commands[]`、`Makefile` 四处。仓库发现 `setup_git_directory` 与全局选项如何写入配置栈被有意留到后续讲义。
