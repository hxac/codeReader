本讲是「索引 index」单元首讲，讲清索引在 git 三层数据模型（工作树／索引／对象数据库）中的桥梁表示与读写。核心结论：索引本质是一张「路径 → 对象哈希 + stat 快照」的表，提交时被冻结为 tree 对象。内存里由 `struct index_state` 表示（持有 `cache_entry *` 指针数组为核心，外加 name_hash/dir_hash、cache_tree、split_index、resolve_undo、untracked、fsmonitor 等辅助缓存）；每条记录 `struct cache_entry` 的三大信息字段为：`ce_mode`（经 `create_ce_mode` 规范化为 reg/link/gitlink 三类，只保留属主可执行位）、`oid`（指向 blob，子模块则指向 commit）、`ce_stat_data`（缓存的 stat 快照，支持「先 stat 后内容」的快速判变路径）。关键术语：cache_entry、stage（0/1/2/3 标识合并冲突阶段）、racy clean（将 size 抹零以强迫下次重比内容）。磁盘格式 `.git/index`（签名 DIRC）= 文件头 + 条目区 + 扩展区 + 校验尾，全大端序固定 32 位字段以保跨平台/NFS 一致；扩展区为 TLV 设计，按 4 字母签名分派（TREE/REUC/link/UNTR/FSMN/sdir 等），大写未知扩展被安全忽略以实现前后向兼容。读写分别经 `do_read_index`、`do_write_index`，条目统一从内存池分配、整池回收，写时落临时文件算完校验再原子改名。版本 v2~v4，v4 靠路径前缀压缩省体积。cache-tree、split-index、sparse-index 的深入机制留待 u4-l2、u4-l3 承接。
