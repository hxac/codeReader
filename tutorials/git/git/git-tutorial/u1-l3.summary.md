本讲在已编译的 git 源码树里建立目录地图。核心结论：git 是单一二进制加一堆硬链接，子命令靠 git.c 内部命令表分发。源码分两层——顶层扁平核心库（blob.c、commit.c、object-file.c 等，以文件名前缀划模块，经 Makefile 的 LIB_OBJS 汇总为 libgit.a）与 builtin/ 子命令目录（每文件导出一个 cmd_* 入口）。关键陷阱：Makefile 用 patsubst 自动把 builtin/xxx.o 转成 git-xxx，但「命令名 ≠ 文件名」（如 git status 的 cmd_status 实住在 builtin/commit.c）需在 BUILT_INS 手动登记。引入术语 porcelain（瓷器/高级命令）与 plumbing（底层命令），登记于 command-list.txt，驱动 git help 分组与 man 手册生成。refs/ 是引用后端适配层（files/packed/reftable 三种实现），reftable/ 为可独立二进制格式子库，二者定位不同。t/ 为约千个 shell 端到端测试，Documentation/ 为 .adoc 手册与 CodingGuidelines；新增一个 builtin 命令须同步改 builtin/、Makefile、git.c 命令表、command-list.txt 四处。
