本讲承接 u6-l1，讲清如何不改配置文件而临时覆盖配置。git 环境变量分两类：上下文型（如 GIT_DIR）在仓库发现阶段读取，决定读哪个仓库，不进配置链、属更底层覆盖；配置型（如 GIT_CONFIG_PARAMETERS）在配置加载阶段读取，优先级最高。由此得到完整优先级链 system→global→local→worktree→command，铁律是后读覆盖先读；每条值由 struct key_value_info 携带 scope 与 origin_type 标签，命令级即 CONFIG_SCOPE_COMMAND。核心结论：`-c` 并非直接写配置表，而是转义后追加进 GIT_CONFIG_PARAMETERS、留待加载最后一步读取，故命令级最高；`--config-env` 殊途同归，但值取自环境变量，避免明文出现在命令行（ps 可见），适合放令牌。local_repo_env[] 是「仓库局部」清单，子进程切换仓库时清除以防泄漏。
