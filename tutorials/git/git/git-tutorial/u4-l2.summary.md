本讲承接 u4-l1 的索引三层数据模型，深挖索引层三个「用空间换时间」的性能缓存机制。**cache-tree**（对应 `TREE` 扩展）挂在 `index_state` 上的目录树形缓存，每节点存 `entry_count`（覆盖条目数）与 `oid`（折叠后的 tree 哈希）；核心不变式是 `entry_count>=0` 为有效、`-1` 为无效。索引改动经 `cache_tree_invalidate_path` 惰性失效（沿父链置 `-1`，极廉价），`write-tree` 时 `update_one` 命中即短路、无效才重建，并支持 `prime_cache_tree` 预填。**split-index**（对应 `link` 扩展）把索引拆成只读共享基 `sharedindex.<oid>` 加增量 `.git/index`，增量由 `delete_bitmap`/`replace_bitmap` 两个 EWAH 位图表达；`cache_entry.index` 字段（1-based，0 表新条目）区分基/增量，非共享占比超 20% 则换新基，且与 sparse-index 互斥。**name-hash** 维护 `name_hash` 与 `dir_hash` 两个 hashmap 懒加载，支撑大小写不敏感查找，仅 `ignore_case` 且索引≥4000 条且多核时用两阶段多线程构建。三者统一遵循「读按需加载、写按需持久化、改动按需失效」的缓存套路，由 `cache_changed` 位标志与 `EXTMASK` 协调是否只需写扩展。
