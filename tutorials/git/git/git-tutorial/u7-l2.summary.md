本讲承接 u7-l1，从「算法」与「数据结构」两层解决历史遍历的代价问题。算法层引入**可达性**与 **merge-base（合并基）**：`commit-reach.c` 的 `paint_down_to_common` 用「涂色」心智模型——从两端同时向下走，分别刷 `PARENT1`/`PARENT2`，碰头处标记 `RESULT`，其 parent 标 `STALE` 排除，从而只走分叉以来的小段历史。剪枝依据是**世代号（generation number）**的单调性引理：`gen(c)=1+max(gen(parents))`，祖先世代号严格更小，故队列弹到低于下界即可停；世代号只加速、不影响正确性。数据结构层引入 **commit-graph**：只读缓存文件（签名 `CGPH`，chunk 分块格式），`CDAT` 块每行 `rawsz+16` 字节存 tree/两个 parent 下标/打包世代号与日期；parent 存为**图内行号下标**而非 OID，使取 parent 变为 O(1) 数组访问。它是**加速器而非真相**，`repo_parse_commit_internal` 先查图、查不到回退对象库。底层设施 **commit-slab** 用 `c->index` 作主键的两级稀疏数组，在不改 `struct commit` 的前提下挂临时/缓存数据，`_at` 分配、`_peek` 只读、零初始化。
