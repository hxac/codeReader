本讲讲解 git 行级 diff 库 xdiff（源自 LibXDiff），它在 u8-l1 diffcore 的文件级配对之上，为一对已配好的文件算出逐行增删补丁。其数学本质是在两行序列间求最长公共子序列（LCS），等价于最短编辑脚本（SES），编辑距离 D = N+M−2|LCS|，Myers 复杂度 O((N+M)D)。三个最小模块：①算法核心——xprepare.c 把字节切成行记录 xrecord_t，按内容归类赋 minimal_perfect_hash（实为行类下标，使两行比较退化为整数比较），并剔除无匹配行、裁掉公共首尾；xdiffi.c 的 Myers 用分治（xdl_recs_cmp）与对角线扫描（xdl_split 前后向 snake 交汇）填出布尔数组 changed[]，mxcost 是启发式收尾上限，故默认算法不保证绝对最小。②桥接层 xdiff-interface.c——xdi_diff 加 MAX_XDIFF_SIZE 安全闸与尾部裁剪后转交 xdl_diff，xdiff_outf 用 remainder 把字节缓冲拼成完整行回调 git。③userdiff.c——内置约 30 种语言的 funcname/word-regex 驱动表，由 .gitattributes 的 diff= 属性按路径选择，未匹配回退 default。四种算法（默认 Myers、--minimal、--patience、--histogram）与默认开启的 --indent-heuristic 编码在 xpparam_t.flags，于 xdl_do_diff 分派。
