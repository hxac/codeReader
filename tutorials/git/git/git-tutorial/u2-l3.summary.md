本讲拆解 `git init` 如何在磁盘上「无中生有」地建出 `.git`，核心逻辑在 `setup.c` 的 `init_db`（`builtin/init-db.c` 的 `cmd_init_db` 仅做参数解析与目录准备）。涵盖三个最小模块：一是目录骨架，由 `safe_create_dir` 建 `.git` 本身、`create_object_directory` 建 `objects/` 及其 `pack`/`info` 子目录，而 `hooks/`、`info/` 并非显式 mkdir，而是随模板复制出现；二是模板复制，来源有四层优先级 `--template` → `$GIT_TEMPLATE_DIR` → `init.templateDir` → 编译期默认目录，命中即止不合并，`copy_templates_1` 递归复制时跳过隐藏文件、不覆盖已存在项；三是 `create_default_files` 动态写 `config`（先复制模板、清缓存重读，写 `core.repositoryformatversion` 及探测的 `core.filemode`/`core.symlinks`/`core.ignorecase`、`core.bare`/`core.logallrefupdates`），`create_reference_database` 则经引用后端建 `refs/` 并把 `HEAD` 写成指向 `refs/heads/<初始分支>` 的符号引用，初始分支名取自 `-b`、`init.defaultBranch` 或默认 `master`。核心结论：`git init` 幂等，`is_reinit` 依据 `.git/HEAD` 是否存在决定哪些步骤跳过，重复初始化安全。引入关键术语：符号引用（symref）、common dir、引用后端（files/reftable）、能力探测，供后续对象、索引、引用单元承接。
