本讲剖析 `git checkout`/`switch` 的源码实现，覆盖三个最小模块：①命令分发——`checkout_main` 依是否带 pathspec 分为分支模式（`merge_working_tree`，把旧/新两棵 tree 交给 `unpack_trees`）与路径模式（`checkout_paths`，带 overlay 叠加语义，用 `read_tree_some` 读 tree 进索引）；②`unpack-trees.c` 的两阶段引擎——阶段一多路遍历合并（checkout 用 `twoway_merge`），回调只产决策与内存标志；阶段二 `check_updates` 物理执行，遵循「先删 `CE_WT_REMOVE`、后写 `CE_UPDATE`」以协调新旧内容，且 `do_add_entry` 把 `CE_REMOVE` 自动升级为 `CE_WT_REMOVE`；③`entry.c` 的单文件写出——读 blob、`convert_to_working_tree` 转换、按 reg/lnk/gitlink 模式落盘、`update_ce_after_write` 回填 stat。核心结论：checkout 不误伤未提交修改靠两道闸——合并阶段的 `verify_uptodate`/`verify_absent` 与 entry 层的 `ie_match_stat`+`!force`。引入的关键术语有：`unpack_trees_options`、`tree_desc`、`struct checkout` 状态结构、`CE_UPDATE`/`CE_WT_REMOVE`/`CE_MATCHED` 等内存标志，以及 `merged_entry`/`deleted_entry`/`keep_entry` 决策函数。
