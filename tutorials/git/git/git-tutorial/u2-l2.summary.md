本讲围绕 `struct repository`（定义于 repository.h）展开，它是 git 对「一个仓库」的运行时上下文表示，承接上一讲找到的 `gitdir` 路径字符串，将其固化为持有路径、子系统指针、配置开关与状态标志的聚合结构。讲义覆盖三个最小模块：一是字段总览，区分路径字段（`gitdir`/`commondir`，多工作树下二者不同，由 `different_commondir` 位域标记）、子系统指针（`objects`/`refs_private`/`index`/`config`）、行为算法字段（`hash_algo`/`settings`/`ref_storage_format`）与状态标志（`initialized`），并点出「急切初始化 vs 懒加载」的设计取舍；二是对象库与引用存储指针，`objects` 在格式校验阶段由 `odb_new()` 急切创建，`refs_private` 则经 `get_main_ref_store()` 首次访问时懒加载；三是 `enum ref_storage_format`（`UNKNOWN`/`FILES`/`REFTABLE`），当前默认 `FILES` 由 `REF_STORAGE_FORMAT_DEFAULT` 宏经条件编译决定，未来 Git 3.0 可能切到 reftable。此外引入全局指针 `the_repository`（指向主仓库实例）及 git 正在推进的「显式传 `struct repository *` 参数」迁移机制。
