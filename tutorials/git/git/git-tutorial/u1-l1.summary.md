本讲是 Git 源码学习手册的入门第一讲，不写配置、不深入 C 细节，只从项目自身文档确立「Git 是什么」的认知地基，覆盖三个最小模块：项目定位与许可、内容寻址设计思想、文档与社区入口。核心结论是：Git 是一个 fast/scalable/distributed 的版本控制系统，由 Linus Torvalds 于 2005 年开创，以 GPL v2 开源；其设计灵魂是「内容寻址的对象数据库」——以内容哈希为一切对象命名，记录快照而非逐行变更，故天然去重、历史不可篡改。对比传统 CVS/SVN 的集中式、基于变更、按名寻址，Git 为分布式、基于快照、按内容哈希寻址。最小源码示例由 version.h（extern 声明 git_version_string）与 version.c（赋值构建期注入的 GIT_VERSION 宏）串成「文档版本号 ↔ 源码符号」的链路。关键术语包括：仓库、对象、内容寻址、哈希（SHA-1 为 40 字符，SHA-256 为 64 字符）、快照、blob、索引文件。协作上 Git 以邮件列表 git@vger.kernel.org 为中心，安全漏洞私邮 git-security@googlegroups.com。真正的对象读写源码留待第 6 单元精讲。
