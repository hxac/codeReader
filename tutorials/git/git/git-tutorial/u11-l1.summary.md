本讲拆 git 远程「建连+握手列 ref」前置环节的三个最小模块。pkt-line 是线协议分包格式：每段数据前加 4 字节十六进制长度前缀（长度含前缀自身），0000/0001/0002 复用为 flush/delim/response_end 控制包，单包载荷上限 65516，packet_reader 是「读一行+偷看下一行」的状态机。connect 的 git_connect 按 URL scheme 建连：git:// 开 TCP 把请求行作为首个 pkt-line 发出，ssh/file spawn 子进程、版本号经 GIT_PROTOCOL 环境变量传递，结果统一归一成 fd[2]；连上后 discover_version 用 peek 偷看首行协商 v0/v1/v2，并解析 capability（含 object-format 决定哈希算法）。transport 用 struct transport 加虚表 transport_vtable，把 git/ssh/file/bundle/helper 统一成「列 ref/抓取/推送/断开」接口，transport_get 按 URL 选后端，连接懒建立、disconnect 发 flush 收尾。
