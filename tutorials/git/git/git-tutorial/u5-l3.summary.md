本讲覆盖 files 后端两模块：多引用原子修改与读路径内存缓存。ref 事务以 OPEN→PREPARED→CLOSED 三态机约束生命周期，PREPARED 态必须显式提交或回滚，直接 free 会泄漏 .lock。原子性靠两阶段提交：prepare 抢全部 .lock 锁、用 old_oid 做 CAS 校验、新值写入锁文件，任一失败全量回滚；finish 写 reflog 后用 commit_ref 把 .lock 原子改名公开。读路径上 ref-cache 把 .git/refs/ 镜像成内存目录树，REF_INCOMPLETE 驱动懒加载、bsearch 做 O(log n) 查找、写后整树失效；overlay/merge 迭代器把松散与 packed 合并为有序流。debug.c 用装饰器模式加 trace。术语：ref_transaction、三态机、两阶段提交、CAS、commit_ref、ref-cache、迭代器组合。
