本讲讲解 BabelDOC 如何把同步翻译主链路包装成「边翻译、边吐进度」的异步事件流，覆盖四个最小模块：async_translate、ProgressMonitor、AsyncCallback 与 create_progress_handler。

核心机制：async_translate 是异步生成器，用 loop.run_in_executor 把同步的 do_translate 丢进线程池，主线程事件循环一边从 AsyncCallback 取事件一边 yield 给调用方，从而非阻塞地报进度。需注意另有一个纯同步入口 translate()，只返回结果、不发事件。

引入的关键术语：事件协议含五种核心事件 progress_start / progress_update / progress_end / finish / error，外加 ProgressMonitor 构造时发的 stage_summary；ProgressMonitor 用阶段权重（TRANSLATE_STAGES 第二列归一化后即占比，ILTranslator 最重、对应调 LLM 瓶颈）计算 overall_progress，progress_update 还带 report_interval 节流，分片(part)翻译时再加 part_offset（已完成片 × 100/总片数）；AsyncCallback 用 call_soon_threadsafe 跨线程把事件写入 asyncio.Queue，遵循「先入队、再置 finished」以防空读丢事件；cancel_event（threading.Event）与 finish_event（asyncio.Event）配合实现取消与收尾，由 on_finish 跨线程置位、保证工人清理临时文件与线程后再退出；create_progress_handler 则把事件渲染成 rich（每阶段一条子条加总条）或 tqdm（单条按 overall_progress 差量更新）进度条，供 CLI 与下游 UI 复用。
