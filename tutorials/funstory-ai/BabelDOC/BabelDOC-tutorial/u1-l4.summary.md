本讲建立 BabelDOC 的「三层配置汇聚」模型：命令行参数与 TOML 文件（语义等价，命令行优先）经 `configargparse` 与 `TomlConfigParser(["babeldoc"])` 解析合并为 `args`，再搬入中心配置对象 `TranslationConfig`，下游流水线只消费这一个 `config`。涵盖四个最小模块：一是 configargparse/TOML，TOML 键名用连字符、须置于 `[babeldoc]` 节下；二是 `TranslationConfig` 作为「装配盘」，除收纳参数外还做派生计算（`pool_max_workers` 默认取 `qps`）与副作用联动（`enhance_compatibility`、`ocr_workaround`、`auto_enable_ocr_workaround` 连带改写多个字段），新参数须追加在 `__init__` 末尾以保向后兼容，废弃字段如 `table_model` 会被强制清零；三是常用项 `WatermarkOutputMode` 枚举（`Watermarked`/`NoWatermark`/`Both`）、`parse_pages`/`should_translate_page` 页面范围解析、`--max-pages-per-part` 触发的分片策略；四是 `const.py` 全局常量 `CACHE_FOLDER`、`TIKTOKEN_CACHE_FOLDER`、`WATERMARK_VERSION`，在 import 时即完成 tiktoken 缓存目录的副作用初始化。核心结论：`TranslationConfig` 是 CLI 与 TOML 的唯一汇聚点，是后续翻译主流程编排读取全部设置的来源。
