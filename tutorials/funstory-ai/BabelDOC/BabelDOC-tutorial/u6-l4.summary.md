本讲聚焦 midend 流水线中权重第二高的 stage「自动术语抽取」，对应类 `AutomaticTermExtractor`，它在 ILTranslator 翻译前先跑一遍，专门提炼全文术语并统一译法，解决逐段独立翻译时术语前后不一致的问题。核心流程分三步：首先是**切批**，按累计 token 超 600 或段落数超 12（先到先触发）将段落组成批次，用 tiktoken（gpt-4o 分词器）本地估算 token，批次投进优先级线程池，token 多的优先调度；其次是**LLM 抽取**，每批调一次 `BaseTranslator.llm_translate`（模板方法：查缓存→限流→`do_llm_translate`），让 LLM 扮演术语学家，输出 JSON 数组 `[{src,tgt}]`，并把命中当前文本的用户术语表作为「参考术语表」拼进 prompt；最后是**多数投票汇总**，各批零散的 `(src,tgt)` 二元组流式追加进跨分片共享上下文 `shared_context_cross_split_part.raw_extracted_terms`（加锁并发安全），在 `finalize_auto_extracted_glossary` 里按 src 分组、用 `Counter.most_common(1)` 选出频次最高译文，汇总成去重的 `Glossary`，再由翻译阶段 `get_glossaries_for_translation` 取用（开启时替代用户术语表）。该阶段受三重门控：`--no-auto-extract-glossary` 开关、`skip_translation`/`only_parse_generate_pdf` 强制关闭、以及翻译引擎必须支持 LLM 模式（`do_llm_translate`）。关键术语包括：自动术语表、多数投票、参考术语表、`shared_context_cross_split_part`、优先级线程池、`llm_translate` 模板方法，这些将由后续 u6-l5（术语表系统）与 u8-l2（分片翻译）承接。
