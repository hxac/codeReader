本讲是 BabelDOC 的「全局地图课」，建立 **frontend（解析）/ midend（处理）/ backend（渲染）** 三段式心智模型。核心结论：BabelDOC 在 Parsing 和 Rendering 之间引入带坐标的中间表示 **IL（Intermediate Language）**，从而能「贴着原版面」画回译文，而非像 mathpix 那样推倒重排。三段以数据形态区分——frontend 输入 PDF 字节、输出一个 `il_version_1.Document`；midend 输入并输出同一个 IL、原地加工；backend 把加工后的 IL 渲染成 mono/dual PDF。IL 是一棵「带坐标的对象树」，承上启下，全程复用同一个 `docs` 变量，使各阶段天然解耦。`high_level.py` 中的 `TRANSLATE_STAGES` 是流水线「节目单」，元组第二列为相对耗时权重，顺序即真实执行顺序；其中翻译（ILTranslator, 46.96）和自动术语抽取（AutomaticTermExtractor, 30.0）权重最大，是调 LLM 的耗时瓶颈；`get_translation_stage` 会按配置剔除不运行的阶段。源码目录与三段高度对齐：解析住 `new_parser/` 与 `document_il/frontend/`，中端住 `document_il/midend/`（一文件一 stage），渲染住 `document_il/backend/pdf_creater.py`，IL 模型与共用工具置于顶层和 `utils/`；`high_level.py` 顶部 import 路径即为目录镜像。关键术语：frontend/midend/backend、IL、stage、`docs` 变量、`TRANSLATE_STAGES`、stage_name。三段式解耦的好处是可单独替换解析器、翻译引擎或渲染方式。
