本讲打开解析前端黑盒的第一层——PDF 字节如何被「读懂」。PDF 文件由九类基本对象（名字、整数、实数、布尔、null、字面/十六进制字符串、数组、字典）外加「流」容器拼成，还有 JSON 没有的「名字」与「间接引用」类型。核心主角有二：`object_parser.py` 是手写递归下降解析器，唯一入口 `parse_object_bytes` 按「首字节分派」逐字节翻译，返回普通 Python 值（字典是 `dict` 而非 `PdfObjectDict`）；它用 `_peek(2)` 区分 `<` 十六进制串与 `<<` 字典，用 `_maybe_indirect_ref` 的「投机读取 + 回退」区分孤立整数与 `整数 整数 R` 引用，并以 `MAX_OBJECT_NESTING_DEPTH=128` 防爆栈。`object_model.py` 定义四个类型：`PdfObjectDict`（带 `objid` 的字典）、`PdfIndirectRef`（`objid`+`generation` 的 frozen 引用）、`PdfObjectStream`（字典属性+流字节）、`ActiveLiteral`（名字包装）；它们由下游在普通值之上「装配」贴标签，非解析器直接产出。间接引用生命周期是两段的：解析器仅识别成 `PdfIndirectRef`，真正物化（查 xref、解析、按需升级为流、缓存）由 `resolve_xref` 在被访问时惰性完成。本讲是 u4-l3 内容流解释器的前置——操作符的操作数正是这些对象。
