本讲梳理 BabelDOC frontend（解析前端）把 PDF 变成 IL 的整体调用链。frontend 有两条入口：产品路径 `parse_prepared_pdf_with_new_parser_to_legacy_ir`（被 `high_level._do_translate_single` 调用，复用已打开的 `doc_pdf`）与独立会话路径（自开 PDF，供脚本测试用），主流程走前者。产品入口四步：造 sink、装页、造解释器、跑页。关键术语：**prepared page**（页视图 + 对象视角合并的解析原料，页面范围过滤在此阶段生效）、**page interpreter**（把内容流解释成事件、交页面几何给 sink）、**text run positioner**（`NativeTextRunPositioner`，把文本事件展开成带坐标字形 `AWLTChar`）、**sink**（`ActiveILCreater`，收 `on_*` 事件、用 `project_native_*` 把字符/曲线/字体投影成 `PdfCharacter`/`PdfCurve`/`PdfFont`，最终 `create_il()` 返回完整 `Document`）。注意 `_NativePageInterpreter.end_page` 是 no-op，避免重复计数。数据形态沿链演变：PDF 路径 → `PreparedPdfPage` → 事件 → 带坐标字形 → IL 实体 → `Document`。
