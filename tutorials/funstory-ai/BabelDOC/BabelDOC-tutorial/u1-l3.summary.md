本讲梳理 `babeldoc` 包的目录结构与两条核心入口。顶层目录按「一条主线 + 两个可换部件 + 底层支撑」切分：主线为 `format/pdf/`（编排入口 `high_level.py`、解析前端 `new_parser/`、IL 处理 `document_il/`）；可换部件为 `docvision/`（版面/表格识别）与 `translator/`（翻译服务）；底层支撑含 `assets/`、已 vendor 的 `pdfminer/`、`babelpdf/`。两条入口须分清：CLI 壳 `main.cli()` 供终端使用，库 API `high_level.async_translate()` 供下游调用。引入术语：IL（中间表示）、`TranslationConfig`（中心装配盘）、`TRANSLATE_STAGES`（阶段全景表）、frontend/midend/backend 三段式。核心结论：`new_parser` 负责「PDF→IL」造 IL，`document_il` 定义 IL 模型并加工、渲染；`do_translate` 经 `run_in_executor` 把同步主链路塞进线程池，真正一条龙在 `_do_translate_single`；docvision 与 translator 装配进 config，替换它们不动主链路；命令入口由 `pyproject.toml` 注册到 `babeldoc.main:cli`。
