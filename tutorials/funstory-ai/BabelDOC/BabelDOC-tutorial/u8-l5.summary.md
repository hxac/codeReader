本讲覆盖 BabelDOC 流水线的「防护层」，分四个最小模块：异常体系、PDF 修正与容错、保存回退、翻译回退，取向是「尽力修、尽量保、保不住软失败、仍不行才硬报错」。异常仓库 BabelDOCException.py 并非基类：ScannedPDFError/ExtractTextError/InputFileGeneratedByBabelDOCError/ContentFilterError 各自直接继承 Exception（仅后者带 `.message`），前三者致命、第四者单段可恢复。主链路解析前跑修正三件套 fix_null_page_content/fix_filter/fix_null_xref，fix_media_box 返回 mediabox_data 供后端还原裁剪框，全程包 try/except、修不好也继续。保存侧四级回退 safe_save→rebuild_pdf_by_inserting_pages→open_pdf_with_save_fallback→save_pdf_with_same_path_fallback 保住产物；add_metadata 与 check_metadata 借 producer 防伪标记构成防二次翻译闭环。翻译侧 ILTranslator 单段异常只跳过本段不中断，ContentFilterError 经 add_content_filter_hint 注入提示；LLM-only 质量校验三关中 token 比例永远生效，「同文」与「编辑距离」两关受 disable_same_text_fallback 控制。致命异常经 do_translate 外层 except 与 translate_error 落地为 error 事件，finally 无条件 on_finish 并清理。
