本讲是「训练工程」单元首篇，讲清 llm.c 中与数值精度有关的三个工程开关。其一，精度是编译期选项：Makefile 的 `PRECISION`（默认 `BF16`）经 `-DENABLE_*` 宏把 `floatX` 类型别名定为 `float`/`half`/`__nv_bfloat16`，三种格式中 BF16 因与 FP32 共享 8 位指数、动态范围相同而成为主力，FP16 缺 gradient scaler 实际不可用。其二，混合精度训练保留一份 **FP32 master weights**（`-w` 控制，默认开），以解决 BF16 尾数太短、吞掉微小 AdamW 更新的问题：前向/反向用 BF16，优化器在 FP32 master 上更新，再以无偏的 **随机舍入（stochastic rounding）** 写回 BF16 工作副本。其三，须区分**数据精度**（`CUBLAS_LOWP`，随 `PRECISION` 变）与**计算精度**（`cublas_compute`，默认 FP32 累加，是混合精度稳定的关键）；**TF32** 只是 FP32 数据在 Ampere+ tensor core 上「19 位乘、FP32 累加」的加速计算模式（非存储格式），由 `PRECISION_MODE==FP32 && major>=8 && -f` 三条件与决定，BF16/FP16 训练时永不介入。
