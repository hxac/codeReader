本讲以 llm.c 的 CPU 参考实现 `train_gpt2.c` 为对象，建立「骨架地图」，只看模型定义与训练主循环，各层算法细节留待后续讲义。

核心覆盖三个模块：其一是 `GPT2Config`——仅用 6 个超参数（`max_seq_len`、`vocab_size`、`padded_vocab_size`、`num_layers`、`num_heads`、`channels`，GPT-2 124M 为 1024/50257/50304/12/12/768）就决定了全部张量形状，由此展开 16 个参数张量（`NUM_PARAMETER_TENSORS`）与 23 个激活张量（`NUM_ACTIVATION_TENSORS`），分别封装为 `ParameterTensors`/`ActivationTensors`，参数总量约 1.24 亿。其二是关键工程技巧：一次性 `malloc` 一整块内存，再经指针排布把各张量「钉」进不同偏移，从而分配次数少、内存连续、可用一次 `fread` 装满全部权重；并引入「懒分配」概念——参数在 `gpt2_build_from_checkpoint` 时即分配，而激活、梯度、优化器状态因依赖运行时的 B、T 而推迟到首次前向/反向/更新时分配。其三是 `main` 训练主循环：按「验证（每 10 步）/ 采样生成（每 20 步）/ 训练（每步）」调度，训练即经典「前向 → 清零梯度 → 反向 → AdamW 更新」四行；生成时用 `eot_token`（`<|endoftext|>`）作新文档起点「开场」，且每生成一个 token 都重算整段前向——作者坦承此为代码简洁而忍受的浪费。
