本讲把 PyTorch 参考 `train_gpt2.py` 与 C 版 `train_gpt2.c` 逐模块对照，确立其在 llm.c 中的三重角色：算法参考、正确性标尺、`.bin` 权重生产者。核心直觉是「PyTorch 的 forward 即 C 的 forward，PyTorch 的 `backward()` 即 C 全部手写的 `*_backward`」——autograd 自动完成反向计算。覆盖模块：配置类字段一一对应，唯一差别是 PyTorch 侧无 `padded_vocab_size`（填充发生在导出边界）；`Block.forward` 两行展开为 C 版层循环 10 行函数调用；权重绑定令 `lm_head` 共享 `wte`；因果 mask 在 PyTorch 用 `masked_fill(-inf)`、C 版用循环上界 `t2<=t` 实现；`configure_optimizers` 按 `dim` 分组设权重衰减（矩阵衰减、bias/LayerNorm 不衰减），C 版 `gpt2_update` 统一处理、靠默认 `wd=0` 掩盖差异。关键差异：两版 `beta2` 不同（0.95 vs 0.999），但 `test_gpt2.c` 自带超参与 `expected_losses` 自洽闭环。引入术语：autograd、权重绑定、debug state（优化器介入前导出，与超参无关）、分组权重衰减、梯度累积。
