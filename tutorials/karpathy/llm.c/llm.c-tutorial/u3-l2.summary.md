本讲讲解 llm.c CPU 参考实现里的 `gpt2_update` 函数，即训练四步（前向→清零梯度→反向→更新）的最后一步，它实现了 **AdamW 优化器**：对长度约 1.24 亿的扁平参数数组逐个独立循环更新，不涉及前向/反向如何算梯度。核心更新规则为：先用当前梯度更新一阶动量 `m`（梯度 EMA，平滑方向）和二阶动量 `v`（梯度平方 EMA，估计波动），再做**偏差修正** `m̂=m/(1-β₁ᵗ)`、`v̂=v/(1-β₂ᵗ)`，最后 `θ -= lr·(m̂/(√v̂+ε) + wd·θ)`。关键概念：`m`、`v` 提供「信噪比」式的**每参数自适应步长**；偏差修正在冷启动时抵消 0 初值造成的低估，使第 1 步更新量近似为 `-lr·sign(g)`；本工程超参为 lr=1e-4、β₁=0.9、β₂=0.999、ε=1e-8、wd=0。**AdamW 与经典 Adam 的本质区别是解耦权重衰减**：衰减项不混进梯度、不被 `√v̂` 缩放，而是作为独立加法项只受 lr 缩放，保证「等比例收缩」语义。`m_memory`/`v_memory` 采用懒分配（首次 calloc 清零），`main` 传 `step+1` 使 `t` 从 1 起避免除零。CUDA 版 `llmc/adamw.cuh` 公式一致，额外含 master weights 与 `grad_scale`，留待 u6 展开。
