本讲只讲编译与运行，不涉及模型算法。三个训练目标对应三份不同源码：train_gpt2（train_gpt2.c，CPU 参考）、train_gpt2fp32cu（train_gpt2_fp32.cu，GPU fp32 legacy）、train_gpt2cu（train_gpt2.cu，CUDA 主线，支持混合精度与多卡）。Makefile 用 `$(shell)` 做**环境自动探测**，逐一检查 OpenMP、NCCL、MPI、cuDNN、nvcc、GPU 算力，找到才追加 `-l` 库与 `-D` 宏，使同一份 Makefile 适配纯 CPU 到多节点多卡的机器。关键 make 变量 `PRECISION`（默认 BF16，合法值 FP32/FP16/BF16）转为 `-DENABLE_*` 编译宏，仅主线目标接受，决定源码中 `floatX` 类型别名；`GPU_COMPUTE_CAPABILITY` 由 nvidia-smi 查询并取所有卡的最小值；`USE_CUDNN`（默认 0）控制是否编译 Flash Attention。CPU 版运行流程：download_starter_pack.sh 下载 .bin 权重与数据 → `make train_gpt2` → `OMP_NUM_THREADS=N` 运行，观察 loss 从约 5.3 逐步下降。
