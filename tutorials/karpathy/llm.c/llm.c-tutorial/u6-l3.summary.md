本讲覆盖训练过程中三个「不改变模型数学、但决定训练成败」的辅助子系统。**学习率调度**：`LearningRateScheduler` 以 `step` 为自变量返回当步 lr，先经线性 **warmup**（分子用 `step+1` 保证首步非零），再进入衰减；支持 `cosine`/`linear`/`wsd`/`constant` 四种形状，衰减终点由 `final_learning_rate_frac` 控制（`min_lr = lr×frac`），其默认值 `1.0` 等于不衰减，必须显式传 `-q` 才会真正衰减。**MFU 估算**：用理论式 `flops_per_token = 6*N + 6*L*C*T`（`6*N` 权重主项、注意力项通常是小头）估分子，`get_flops_promised` 按型号+精度查表并按核心数/频率线性缩放得分母，二者之比即利用率；它是单卡指标，注意与乘了 `num_processes` 的全局吞吐口径区分。**日志**：无状态追加式 `Logger` 仅在 rank 0 激活，每次 `fopen("a")`/`fclose` 保证崩溃不丢数据，与 `printf0` 宏共享「只 rank 0 输出」的多卡安全思路；三种 `logger_log_*` 分别记验证损失、评测准确率、训练 loss/lr/norm，`resume=0` 清空旧文件、`resume=1` 追加。关键术语 warmup、MFU、final_learning_rate_frac、rank 0/printf0、NVML 供后续讲义承接。
