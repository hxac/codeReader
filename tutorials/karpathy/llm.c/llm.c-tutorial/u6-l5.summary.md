本讲承接单机多卡的 ZeRO/NCCL，把训练扩展到多节点。核心要解决的是 NCCL 通信子的「引导」问题：在任何集合通信前，rank 0 必须先调 `ncclGetUniqueId` 生成一枚邀请码 `ncclUniqueId`，可靠分发给所有进程，各进程再凭同一枚 ID 加自身 rank 调 `ncclCommInitRank` 组建 `ncclComm_t`。`multi_gpu_config_init` 提供的三条路径 `mpi`/`tcp`/`fs`，唯一区别就在于分发这枚 ID 所借的外部信道不同：`mpi`（默认）复用 `MPI_COMM_WORLD`，用 `MPI_Bcast` 广播、`MPI_Comm_rank/size` 提供拓扑，并以主机名哈希归类算出本地 GPU 号；`tcp` 让 rank 0 在端口 12345 自建 TCP 服务器、其余连入取 ID；`fs` 把 ID 写入共享文件 `<fs_path>/ncclUniqueId.sync`、其余轮询读取。后两者不依赖 MPI，拓扑信息 `num_processes`/`process_rank`/`gpus_per_node` 全部来自命令行（脚本再灌入 SLURM 环境变量），本地卡号取 `rank % gpus_per_node`。`tcp`/`fs` 存在的根源是新版 slurm-wlm 关闭了 PMIx 导致 MPI 跨节点引导失败。实际命令行标志为 `-pn/-pr/-pg/-pi/-ps/-pf`（帮助文本里的 `-pm/-pp` 与代码不符）。三个脚本分别对应裸 `mpirun`、SLURM+TCP、SLURM+共享文件三种运行时，并共享 NCCL/RDMA 优化环境变量。
