本讲讲解 llm.c 的数据管线，含三个最小模块。一是 tokenized `.bin` 文件：1024 字节头（256 个 int32：magic、version、ntok）加 `uint16` token 流，大小满足 `1024+ntok×2`；Python 端 `write_datafile` 写、C 端读，靠同一魔数 20240520 形成跨语言格式契约。二是 `DataLoader`：用 `process_rank`/`num_processes` 做多卡分片、`glob` 接管多分片并支持 shuffle；核心 `next_batch` 读 `B*T+1` 个 token，令 `targets[i]=inputs[i+1]` 错位一位，把下一个 token 预测编进数据。三是 `tokenizer.h` 只做解码：加载 `gpt2_tokenizer.bin`（魔数 20240328，区别于数据文件），用 id→字符串查表把生成 token 打印成文字，`safe_printf` 过滤 BPE 词表的控制字节。关键术语：token、uint16、B/T、EOT(50256)、BPE、下一个 token 预测、错位一位。核心结论：分词离线完成以提速训练，C 端只读不解码，数据流为「文本→token→.bin→batch→前向/生成→解码→文字」，生成时将 `gen_tokens` 填 EOT 无条件续写。
