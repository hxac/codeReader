本讲是 Unit 7 收尾篇，从「读懂」转向「使用」llm.c，含三模块。其一，复现脚本 scripts/run_*.sh 是自包含训练配方，用 while true + done_file 实现断点续训、`-y 1` 从最新 checkpoint 恢复；最易踩的坑是「`-d` 是全局 batch 整数，`-e` 才是模型来源」，梯度累积步数 = total_batch_size / (B·T·num_processes)，显存吃紧时调 `-r`/`-b`/`-t`。其二，模型描述符：`-e` 既传 `.bin` 也传描述符（GPT-2 的 `dX` 按层数、`gpt3:cX` 按通道数），由 gpt_build_from_descriptor 查硬编码表定 L/C/NH/T，main 靠 ends_with_bin 分流两条路径，随机初始化用 mt19937 + seed42 与 PyTorch 逐位对齐。其三，Llama3 扩展：train_llama3.py 相对 GPT-2 改 RoPE/GQA/SwiGLU 三处核心，加 RMSNorm、去 bias、独立 lm_head，并配套新 `.bin` 权重契约（魔数 20240803、新张量顺序与新 config 字段），C 端 kernel 留待补全。GPT-2 124M 与 GPT-3 125M 形状几乎相同（L=12, C=768, NH=12），唯一差别是上下文 1024→2048，故 GPT-3 把 micro-batch 减半以维持显存预算。引入术语：能力度 6ND、repeat_kv、freqs_cis、键名适配。
