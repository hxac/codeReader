本讲讲解 GPU 注意力的两条编译期互斥路径。默认手写路径 `attention.cuh` 并非纯手写：两次矩阵乘复用 cuBLASLt（batched，`B*NH` 为批次），自己只写 permute/unpermute、online softmax 与 softmax 反向 in-place kernel。核心是 online softmax——边流式读取边维护运行最大值与运行和，遇更大值时用 `exp(old−new)` 缩放累加和，一遍扫描完成数值稳定 softmax，一个 warp 管一行、倒序遍历 block 做缓存优化；它物化完整 `att` 方阵 `(B,NH,T,T)`。另一条 Flash Attention 路径 `cudnn_att.cpp` 用 cuDNN frontend graph API 调用 SDPA 算子，靠 stride 描述 `(B,T,3,NH,HS)` 交错布局省掉 permute，用 `std::map` 缓存计算图，只输出 `stats(B,NH,T)` 而非大方阵，且不支持 FP32。两路由编译宏 `ENABLE_CUDNN` 决定（`make USE_CUDNN=1`，默认关闭，原因是编译时间膨胀与代码较新），无运行时开关。核心结论：两者数学等价，差异纯为显存对工程复杂度的取舍。
