本讲以 `train_gpt2_fp32.cu`（fp32 legacy 版）为入口，把 `train_gpt2.c` 的 CPU 算子翻译成 CUDA 代码，覆盖三个最小模块。一是 kernel 与 grid/block 划分：CUDA 用 grid→block→thread 三层模型，长度为 N 的任务标准写法是 `idx=blockIdx.x*blockDim.x+threadIdx.x; if(idx<N)`，grid 用 `CEIL_DIV` 计算，多余线程靠守卫挡掉；元素级算子（残差、GELU、encoder 前向）套用「算 idx、加守卫、原样写公式」的四步模板。二是显存管理与数据搬运：host/device 内存分离，需用 `cudaMalloc/cudaMemset/cudaFree` 管显存、`cudaMemcpy` 按 H2D/D2H 搬运——权重 H2D 一次常驻，每步 inputs/targets H2D，损失与采样 logits D2H 回 CPU。三是各层 CUDA 化的三种策略：手写元素级 kernel、手写分块/归约 kernel、调用 cuBLAS；并引入两个要点——scatter-add 型反向（encoder 反向）须用 `atomicAdd` 防竞争，cuBLAS 的 `beta`（0=覆盖写、1=累加）编码了「激活梯度用 `=`、参数梯度用 `+=`」的约定。核心结论：本讲只换「执行器」不换「数学」，与 CPU 版数值等价，并通过 scratch 复用与懒分配省显存。
