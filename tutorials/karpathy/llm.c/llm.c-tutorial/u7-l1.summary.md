本讲带读者进入 llm.c 的「CUDA 实验场」`dev/cuda/` 目录，定位为「kernel 选型试验田」：与主线 `train_gpt2.cu`/`llmc/` 只保留胜出版本不同，这里把同一算子写出多版（通常 3~6 个），版本号越大越快也越复杂，跑出最快者经微调（如写死 block_size）后粘进主线。

讲解以 `layernorm_forward.cu` 的 kernel1~6 为主线标本。关键演进是「减少对输入 x 的重复显存读取」：kernel1 读三遍、kernel3 用 cooperative groups 读两遍、kernel4 用**方差巧算** var=E[x²]−E[x]² 把统计量两遍扫描压成一遍（数值代价是均值大时的灾难性抵消）、kernel5 把协作粒度从 warp 升级到 block（共享内存归约各 warp 局部和）、kernel6 进一步用动态共享内存预载 weight/bias 与 x、配合 `Packed128` 128 位向量化访存（`__ldcs`/`__stcs` 流式缓存提示）达到最快。

引入的关键术语包括：cooperative groups（cg，`cg::tiled_partition`/`cg::reduce` 封装 warp 归约）、方差巧算、访存带宽 vs 算力。每个 `.cu` 文件遵循统一五段式结构（文件头说明→CPU 参考→多版 GPU kernel→启动器+版本分发→统一 main），并配 `common.h` 三件套测试脚手架：`validate_result`（绝对+相对组合容差）、`benchmark_kernel`（每次计时前 flush L2 cache）、napkin math（访存受限算子用 GB/s、计算受限算子用 TFLOPS，对照硬件标称值判断优化空间）。`matmul_forward.cu`、`attention_forward.cu` 作为同思路旁证。
