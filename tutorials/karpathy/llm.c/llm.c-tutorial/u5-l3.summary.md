本讲讲解 llm.c 用 cuBLASLt 替代手写 GEMM 的最终形态，核心是 `matmul_cublaslt` 封装：它把转置、`m/n/k` 维度、`alpha/beta`、bias、gelu 全参数化，前向、反向与 batched attention 复用同一函数。关键约定有二：`transA=true` 利用「行主序权重等价列主序转置」的恒等式，把行主序权重喂给列主序 cuBLAS；`beta` 编码覆盖（激活梯度 `=`）还是累加（权重梯度 `+=`）。epilogue（尾声）融合是 cuBLASLt 的杀手锏，bias/GELU 在一次 GEMM 内完成省一次显存往返，`GELU_AUX` 还把 pre-gelu 存到辅助指针供反向使用，是否融合由 `gelu_fusion`（默认 0）控制。反向 `matmul_backward` 求三路梯度：dinp 覆盖、dweight 累加，二者复用同一 GEMM；dbias 不走 GEMM 而用手写归约 kernel `matmul_backward_bias_kernel9`。最后厘清三类精度：数据精度 `CUBLAS_LOWP`（随 `PRECISION` 切换 floatX）、计算精度 `cublas_compute`（默认 FP32 累加，含 BF16/FP16）、TF32（FP32 数据走 tensor core 的加速模式，仅 FP32+Ampere+ 才启用，BF16/FP16 不涉及）。
