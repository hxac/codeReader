本讲讲解 llm.c 的 GPU 性能剖析工具链。为避免直接剖析 train_gpt2cu 导致 ncu 报告巨大且混入数据加载等噪声，llm.c 维护独立入口 profile_gpt2.cu：通过 `#define TESTING` 加 `#include` 复用全部训练代码（保证剖析的是真实 kernel），但 main 只跑单步、把 num_layers 临时改 1、用假数据填充、关闭多卡，使 `.ncu-rep` 报告小而干净；解析时再把单层耗时乘 N_LAYERS=12 还原。引入的关键术语包括：剖析（profiling）、ncu（NVIDIA Nsight Compute CLI，靠 kernel 重放采集硬件计数器）、`.ncu-rep` 报告、`-lineinfo` 编译选项（让 ncu 把汇编关联回源码行）、以及耗时（gpu__time_duration）、访存带宽（dram__bytes、lts__t_sectors）、Tensor Core 利用率三类核心指标。profile_gpt2cu.py 自动探测 ncu 路径与权限、生成并解析报告，按 encoder/前向/分类器/反向/优化器五段归类。核心判据：理想 kernel 应被显存带宽或 Tensor Core 算力之一限死，整体 efficiency 越接近 100% 越好，效率最低的 kernel 即优化候选。
