本讲深入 llm.c 手写的 GPU kernel（layernorm / encoder / gelu / adamw），核心是把 CPU 参考实现里的串行循环翻译成三种并行套路：元素级切片、行内 warp 归约、全局 block 归约。引入的关键概念包括三层执行模型（线程/warp/block/grid）、shuffle 指令 `__shfl_xor_sync`、`Packed128`/`x128` 的 128 位向量化 load/store 及 `cs`/`cg` cache hint。核心结论：LayerNorm 用「一个 warp 管一行 (b,t)」加 `warpReduceSum`（5 步二叉归约）替代 CPU 内层 `for i` 求 mean/var，统计量恒存 fp32；gelu/residual/encoder 前向共用元素级模板；encoder 反向的 scatter-add 不用 `atomicAdd`（不可复现），改用 CPU 分桶 + 每 block 归约一桶实现确定性，wpe 因地址天然唯一而简单；AdamW 是一线程一参数的完全并行，二维 grid 兼顾跨层与 ZeRO 分片，BF16 下维护 fp32 master weights 并用 stochastic rounding 无偏写回；global_norm 用 grid-stride + 两阶段 `blockReduce` 算梯度平方和供 `grad_scale` 裁剪。`warpReduceSum`/`blockReduce`/`stochastic_rounding`/`Packed128` 是所有 kernel 的公共地基。
