本讲解析 GPT-2 中最频繁、最耗时的 matmul 算子。一个线性层即 out=inp@weight^T+bias，输入 (B,T,C)、权重 (OC,C)、输出 (B,T,OC)，OC 为本讲引入的「输出通道数」。朴素版为四重循环 b→t→o→i，沿 i 做内积，瓶颈是访存——大权重被重复扫 B*T 次。cache blocking 版一次处理 8 个 (b,t)，让权重读入寄存器复用 8 次，把权重内存流量降到原来的 1/8，不满足 B*T%8==0 时回退朴素版。反向 matmul_backward 求三路梯度：dinp 沿 o 求和（用前向 weight）、dweight 与 dbias 沿 (b,t) 求和（用前向 inp），全部用 += 累加并依赖每步清零；为避免多线程写冲突，拆成两轮循环、分别并行 (b,t) 与 o。最终 logits 投影复用 wte 作权重、bias 传 NULL（权重绑定）。
