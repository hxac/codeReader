本讲只讲训练完成后的「写文字」，聚焦三个最小模块。一是极简 PRNG `random_f32`，基于 xorshift* 算法：单个 64 位状态经三次异或移位加一次常数乘法，再用 `(u>>8)/2^24` 映射到 [0,1) 的 float，可复现、统计够用。二是多项分布采样器 `sample_mult`，采用逆累积分布函数（inverse CDF）法：累加概率直到超过一枚 [0,1) 的「硬币」，选中所落区间对应的 token，因每段长度恰为 p_i，故 P(返回 i)=p_i，能还原原分布；末尾兜底返回 n-1 防浮点误差，且只在真实词表 V（非填充后的 V_p）上采样。三是自回归（autoregressive）生成循环：先将 gen_tokens 填满 EOT 作种子，再 `for t=1..genT-1`，每步对整段 B×T 重算 `gpt2_forward(..., NULL, ...)`（无 KV cache，故计算量平方增长，但够 sanity check），取 `probs[0, t-1, :]`（即用位置 t-1 的分布预测位置 t 的下一个 token，且只跟 b=0 条序列），抛硬币采样写回。需区分：`llmc/rand.h` 的 Mersenne Twister 是为与 PyTorch 逐位一致、服务数据 shuffle 与测试，不参与生成采样。CUDA 主线的 `sample_softmax` 直接吃 logits，与 CPU 版统计等价。
