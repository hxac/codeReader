本讲讲解 llm.c 权重 `.bin` 文件的跨语言契约。模型文件由「1024 字节自描述头 + 按 C 端布局排好序的张量流」组成：头存魔数 20240326、版本号、6 个 config（block_size、vocab_size、n_layer、n_head、n_embd）及填充词表 Vp。写端 `write_model` 把 16 类参数按「类型分组、跨层连续」重排写出，顺序与 C 端 `ParameterTensors` 指针排布逐位对应，使读端 `gpt2_build_from_checkpoint` 一次 fread 即可让所有指针自动就位。引入的关键概念：fp32→version 3（CPU 参考实现）、bf16→version 5（CUDA 主线，借 `view(int16)` 绕过 numpy 无 bf16）；`pad_vocab` 把真实词表 50257 上取整到 128 倍数 50304，算法 no-op、仅为 GPU 矩阵对齐；权重绑定使 `wte` 兼任 `lm_head`，故只有 16 类参数而非 17 类。仓库用不同魔数区分权重、state、tokenizer、数据四类 `.bin`，同一套「1024 字节头」思路贯穿全局。
