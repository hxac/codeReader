本讲以 `test_gpt2.c` 为 llm.c 建立数值正确性判据。测试用 `#define TESTING` 后 `#include "train_gpt2.c"` 复用全部模型代码（训练版 main 被 `#ifndef TESTING` 守卫排除），永不与被测代码分叉。判据是 PyTorch 导出的「标准答案」`gpt2_124M_debug_state.bin`（魔数 20240327、版本 2、一律 fp32，由 `write_state` 在优化器介入之前生成，与 lr/wd 无关；含 x、y、logits（B·T·V 未填充）、loss 及 16 个梯度）。三道关卡：step 0 逐元素比对——logits 用 V/Vp 双下标、容差 1e-2、首错即停，loss 用 1e-2，16 个梯度用 `check_tensor` 以 2e-2 比对（梯度因多层累加误差更大）；10 步训练用同一固定 batch 过拟合，把硬编码 `expected_losses` 曲线作「指纹」回归，每步 |Δloss|<1e-2，能抓单步比对漏掉的误差指数放大。测试 `gpt2_update` 用 wd=0.01（正式训练用 0.0）以逐位复现曲线，故仅 loss 曲线依赖优化器超参。任一关、任一步失败都经 `allok && ...` 累积，`overall okay: 1` 为唯一全过信号，且失败不提前退出。引入的关键术语：debug state、回归测试、容差、check_tensor、双下标比对、分层正确性策略。
