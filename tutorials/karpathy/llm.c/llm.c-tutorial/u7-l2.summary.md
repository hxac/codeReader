本讲讲解 llm.c 的评测体系。核心是两条互补路径：一是训练中内联 HellaSwag 评测（completion-style，快、周期性），二是训练后用 EleutherAI lm-evaluation-harness 跑 6 个基准（multiple-choice style，权威可横评）。关键术语是 completion-style：把上下文与每个候选续写拼接，用模型对整个续写的平均下一个 token 损失打分，avg_loss 最小者胜出，对应 acc_norm。两种口径数字接近但不相等，是方法论差异。数据侧，write_evalfile 把四选一题打包成专用 .bin（魔数 20240522、上下文只存一份、EXAMPLE_BYTES 支持 fseek 跳过、65535 作哨兵），由 C 端 EvalLoader 展开成 batch 的 4 行，targets 错位一位、mask 标记续写区，evalloader_stat_losses 在 mask 区求平均损失判正确并多卡 all-reduce。标准流程为 export_hf.py（.bin 转 HuggingFace 模型，承接 u4-l2 协议）→ run_eval.sh → summarize_eval.py 按各任务主指标汇总。最后，sampler.h 的 sample_softmax 用 coin *= norm 优化省掉一次归一化遍历；训练中评测依赖 gpt2_validate 产出的 per-token cpu_losses 作为判分基础。
