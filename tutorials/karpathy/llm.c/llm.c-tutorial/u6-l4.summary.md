本讲讲解 llm.c 的 CUDA 主线如何从单卡扩展到多卡数据并行。核心模块有四：其一，DataLoader 用 `process_rank`（rank r）/ `num_processes`（进程数 P）把 token 流切成互不重叠的分片，每卡按 `local_batch_offset_bytes = r*B*T*2` 错位取数，实现数据并行。其二，跨卡梯度合并靠 NCCL 三种集合通信原语：`ncclAllReduce`（每卡得完整平均梯度，对应 DDP）、`ncclReduceScatter`（每卡只得本卡分片，对应 ZeRO-1）、`ncclAllGather`（散播更新后的新参数）；这些调用被 `multi_gpu_async_reduce_gradient` 用 `ncclGroupStart/End` 合并提交，靠 event 做异步流同步。其三，ZeRO 当前只支持 stage 0 和 stage 1：`set_zero_configs` 会把 stage 2/3 或不可整除的 stage 1 静默降级为 0；stage 1 仅分片优化器状态（AdamW 的 m、v、master weights 按 N/P 分配），参数和梯度仍完整，数学等价性由「reduce-scatter 合并梯度 + 各卡更新本卡分片 + all-gather 散播新参数」三步保证，通信量与 DDP 相当，收益纯来自显存（8 卡下每卡训练显存约压到原来的 1/3）。其四，梯度累积用 `total_batch_size = grad_accum_steps * B * T * P` 定义全局 batch，`dloss = 1/(B*T*grad_accum_steps)` 保证累积加跨卡平均后恰为全体 token 均值梯度，跨卡通信只在最后一个 micro-step 发生。关键术语：process、rank、shard、reduce-scatter 后非本卡段梯度过期（故 grad norm 只算本卡分片再 `ncclSum`）。
