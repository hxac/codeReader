本讲（u6-l2）讲 CUDA 主线 `train_gpt2.cu` 三类训练工程手段：用「显存-算力」换取训练稳定与高效。第一，**recompute（重计算，gradient checkpointing）** 是一个 0/1/2 整数开关：`0` 不重算、`1` 只重算 GELU（把最占显存的 `fch_gelu` 从每层存一份缩成全局一份，反向时用保留的 `fch`/pre-gelu 现算，是默认甜点）、`2` 再重算 `ln1`/`ln2`；省显存的落点是 `fill_in_activation_sizes` 里的尺寸三元判断，反向必须「先重算、再消费」。第二，**融合算子** 收益来自减少显存往返而非 FLOP：`fused_residual_forward5` 把残差相加与 LayerNorm 合一，相加结果留在共享内存接力做归一化；`fused_classifier` 用 online softmax 只取目标处概率算交叉熵，再用招牌公式 `dlogit = (prob − onehot)·dloss` 把 logits 原地改写成 dlogits，**从不物化 probs 向量**，前后向融合靠 `__syncthreads()` 防数据竞争。第三，**global_norm** 用「每个 block 写唯一槽位 + 单独确定性归约」避开 `atomicAdd`，换取 grad norm 的逐位可复现；**梯度裁剪**为按全局范数（阈值 1.0），仅传一个标量 `grad_scale`，真正作用点在 AdamW kernel 首行 `grad = grad_scale * grads[idx]`。三者共同把训练推向显存够用、kernel 少、数值稳定且可复现。
