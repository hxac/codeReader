本讲建立 PyTorch 的「整体地图」，分四个最小模块。**双重定位**：PyTorch 同时是带强 GPU 加速的张量计算库与基于 tape 的自动求导框架，两条能力在源码上相对分离（前者落在 `aten/`、`torch/`，后者落在 `torch/autograd/` 与 `torch/csrc/autograd/`）。**顶层组件**：README 列出 6 个——`torch`、`torch.autograd`、`torch.jit`、`torch.nn`、`torch.multiprocessing`、`torch.utils`；`torch.distributed` 未列在表中但确为真实子包。**分层架构**（Python 前端 + C++ 后端）：`torch/` 是 Python API 层，`torch/csrc/` 与 `torch/_C/` 构成 pybind11 绑定层，C++ 后端中 `c10/` 是最小跨平台核心（`TensorImpl`、`Storage`、`DispatchKey`），`aten/` 是不含 autograd 的算子现代实现并按后端分目录，`torchgen/` 从 `native_functions.yaml` 生成绑定代码，`caffe2/` 为遗留库。**import 流程**：`torch/__init__.py` 经 `_load_global_deps()` 用 `ctypes.CDLL(RTLD_GLOBAL)` 预加载 `libtorch_global_deps`，再 `from torch._C import *` 把 C++ 后端注入 Python 命名空间。关键术语：tape-based autograd、binding、DispatchKey、`torch._C`、`USE_GLOBAL_DEPS`。算子如何从 Python 分发到 C++ 留待后续讲义。
