本讲承接 u10-l1，讲建连之后进程间如何交换张量。集合通信原语分归约型（all_reduce/reduce/reduce_scatter）与交换型（broadcast/all_gather/all_to_all）。对比两套 API：经典 dist.* 是 in-place、可选 async_op、归约算子用 ReduceOp 枚举、返回 Work 或 None；functional collectives 是纯函数式、天然异步（eager 返回 AsyncCollectiveTensor、tracing 插 wait_tensor）、算子用小写字符串、底层落到 torch.ops._c10d_functional.* 真实算子。桥接二者的是 REDUCE_OP_TO_STR 与 *_inplace 包装（functional 结果 copy_ 回原张量）。核心更新是新增 _remap_traceable_collective 与 _LegacyToFunctionalCollectiveMode，在 make_fx 的 compile_on_one_rank 路径与非 strict export 路径，把旧 in-place 集合通信改写为 functional 形式。动机是 in-place c10d.* 把 ProcessGroup 这个不可序列化的 torchbind 常量烤进图，致 GraphPickler 失败；functional 把 group 当算子参数（字符串 name 或 mesh_get_process_group 节点），使 group 成为可序列化数据流，编译出的图可在任意 rank 加载。
