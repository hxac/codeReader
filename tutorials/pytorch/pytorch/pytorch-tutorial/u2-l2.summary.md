本讲揭示 `torch.Tensor` 由「数据」与「视图」两部分分离拼成：数据由 Storage 持有，视图由 `sizes`/`strides`/`storage_offset` 描述。核心定位公式为 `offset = storage_offset + Σ i_d · stride_d`，单位是元素，再乘 `itemsize` 才得字节偏移。

Python 侧推荐用 `UntypedStorage`（经 `tensor.untyped_storage()` 获取）；`TypedStorage` 已废弃，只是 `UntypedStorage` 加 dtype 标签的薄壳。C++ 层 `StorageImpl` 真正只持有 `data_ptr_` 与 `size_bytes_`，不感知形状语义，所有视图与解释职责留在 `TensorImpl`，二者靠侵入式引用计数共享。

核心结论：只改 sizes/strides/storage_offset 的操作（转置、切片、`view`）零拷贝；一旦现有布局表达不了目标形状（非连续 `view`、扩容），就必须真实拷贝（`contiguous()`、`reshape()` 的退化路径）。注意单位坑：`stride`/`storage_offset` 是元素单位，`nbytes`/`data_ptr` 偏移是字节单位。
