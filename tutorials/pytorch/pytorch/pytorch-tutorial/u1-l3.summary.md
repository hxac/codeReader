本讲把 PyTorch 的分层架构细化到目录级别，建立一张五目录源码导航地图。`torch/` 是用户 import 的 Python 前端包（外加编译扩展 `_C`），含 nn、autograd、optim、distributed、_dynamo、_inductor 等子包，强调「深度集成进 Python」而非对单体 C++ 框架的绑定。`torch/csrc/` 是 Python↔C++ 绑定的 C++ 源码，编译出 `torch._C`，入口为 `Module.cpp` 的 `initModule`。`aten/` 是 ATen 张量算子库（C++ 后端，不含 autograd），其算子接口的「单一事实来源」是 `native_functions.yaml`，按 native/ops/core 及 cuda/cpu 等后端分目录。`c10/` 是最底层跨平台核心，定义 `TensorImpl`、`StorageImpl`、`DispatchKey`，依赖方向为 aten→c10 单向。`torchgen/` 是构建期代码生成器，读 YAML 产出各层声明、绑定与类型桩，不参与运行时。一次 `torch.add` 调用穿越路径为 torch/→torch/csrc/→aten/→c10/。本讲引入的关键术语：前端/后端、绑定、算子、代码生成、DispatchKey、native_functions.yaml、单一事实来源、运行时目录 vs 构建期目录。
