本讲剖析 `import torch` 执行链路，含四个最小模块：顶部纯 Python 工具与版本加载；全局依赖预加载；`torch._C` 的 C++ 初始化；顶层符号组装。核心结论：`__init__.py` 先导入纯 Python 工具（`_utils`/`_utils_internal`）与初始 `__all__`，再按 `USE_GLOBAL_DEPS` 开关用 `ctypes.CDLL(..., RTLD_GLOBAL)` 预加载 `libtorch_global_deps.so` 使 C++ 依赖全局可见（`USE_RTLD_GLOBAL_WITH_LIBTORCH` 是互斥的另一条 Hard way 路线），然后 `from torch._C import *` 触发 `stub.c` 的 `PyInit__C` 转发到 `Module.cpp` 的 `initModule()`，后者创建模块对象、注册 dtype/Device/Layout/Tensor/autograd 类型、绑定 jit/dynamo 等子系统并 `at::init()`。导入后用 `_initExtension` 当哨兵验证 `_C` 可用性，遍历 `dir(_C)` 把公开符号补进 `__all__`、修正 `__module__`、子模块入 `sys.modules`，最后调 `_C._initExtension()` 完成 layout/dtype/storage 的 Python 侧二次初始化。关键术语：`_running_with_deploy`、`USE_GLOBAL_DEPS`、`USE_RTLD_GLOBAL_WITH_LIBTORCH`、`RTLD_GLOBAL`、`PyInit__C`、`initModule`、`_initExtension`、`at::init()`、`THPVariable_initModule`。
