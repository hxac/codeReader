本讲讲解 PyTorch 「pip + CMake」混合构建体系。核心结论：PyTorch 本质是 C++ 项目套 Python 外壳，`setup.py`（pip）只是入口与打包层，真正编译几千个 `.cpp/.cu` 交给 CMake。构建产物分三类：codegen（torchgen 读 `native_functions.yaml` 生成的绑定与类型桩）、C++ 编译（`libtorch_python.so`、`torch._C`）、Python 包。规范安装命令为 `pip install --no-build-isolation -v -e .`，其中 `--no-build-isolation` 使构建复用当前已装好依赖的环境。关键概念：`torch._C` 不在 setuptools `ext_modules`，而由 CMake `--target install` 编译到 `torch/lib`，经 `BinaryDistribution` 标记为二进制包。`CMakeLists.txt` 用 `option`/`cmake_dependent_option` 定义 `USE_CUDA/USE_ROCM/USE_XPU` 三大后端开关及 `USE_CUDNN/USE_NCCL` 等依赖型选项，`USE_CUDA=0` 会级联关闭整排 CUDA 选项；shell 环境变量经 `cmake/EnvVarForwarding.cmake` 自动转为 `-D` 选项。`MAX_JOBS` 控并行度，`DEBUG`/`REL_WITH_DEB_INFO` 控构建类型，`CMAKE_ONLY`/`CMAKE_FRESH` 控配置阶段。`Makefile` 是薄包装：`make` 仅走 CMake 适合增量重编 C++，`make triton` 装匹配版本 triton。
