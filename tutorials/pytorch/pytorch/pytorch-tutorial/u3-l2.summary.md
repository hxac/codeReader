本讲承接「native_functions.yaml 算子模式」，讲单一事实来源 YAML 如何经 torchgen 生成全部 C++。它是构建期工具，不参与 import 运行时，分三模块：model.py 用 frozen dataclass 把 YAML 建模成强类型对象，核心为 NativeFunction 与 FunctionSchema，以「无损往返（lossless roundtrip）」断言保证 parse 与 print 完全一致，dispatch 信息外置到 BackendIndex，SchemaKind 把算子归为 functional/inplace/out/mutable 等类；gen.py 的 main() 分 headers/sources/declarations_yaml 三阶段，用柯里化生成器（ComputeFunction、ComputeOperators、ComputeTensorMethod）配 FileManager 做模板替换，仅内容变化时写盘（_write_if_changed）；native_function_generation.py 用 signature() 归一化识别同族算子聚成 NativeFunctionsGroup，自动补出缺失的 functional/out 变体及合成 kernel。核心结论：人类只维护 YAML，机器保证所有派生产物一致；公开 C++ API 最终经 at::_ops::xxx::call 转入 Dispatcher，留待下一讲。
