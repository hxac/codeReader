本讲阐明 torch.Tensor 本质是一层薄薄的 Python 包装，继承自 C++ 后端的 torch._C.TensorBase（内部类型 THPVariableType），真正的数据与计算在底层 TensorImpl/StorageImpl 及 C++ kernel 中。核心结论：_tensor.py 中没有 def add，上千个算子由 torchgen 生成并注册到 TensorBase，Python 经 MRO 继承获得。算子有两套等价入口——方法式 x.add(y) 源自 TensorBase，函数式 torch.add(x,y) 源自 _C._VariableFunctions（由 __init__.py 在 import 时循环复制进 torch 命名空间），二者最终汇合于同一 C++ 算子。_tensor.py 里显式手写的方法分三类：转发到 _VariableFunctions（如 __rsub__）、直接别名到 TensorBase（如 __neg__）、仅补 docstring（如 detach）；必须 Python 手写的是协议方法（__repr__/__len__/__iter__）、backward 转发、hook 与序列化。shape/dtype/device/grad/grad_fn 等属性均为 C++ 通过 tp_getset 注册的 getter，Python 不重复定义。引入的关键术语：TensorBase、THPVariableType、_VariableFunctions、MRO、TensorImpl、StorageImpl，为 u2-l2（Storage 与内存布局）、Unit 3（torchgen 代码生成）与 Unit 4（autograd）作承接。
