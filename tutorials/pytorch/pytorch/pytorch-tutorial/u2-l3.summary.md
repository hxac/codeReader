本讲承接「张量 = 数据 + 视图」，解析张量除形状外的三大正交属性及 C++ 聚合器 TensorOptions。dtype 决定 Storage 裸字节按何种类型解释，单一事实来源是 ScalarType 枚举（底层 int8_t），Python 的 torch.float32 等只是「枚举值 + 名字」的薄包装，默认 float32 且仅影响浮点推断。device 决定数据物理位置，由 (类型, 索引) 二元组构成，设备字符串经 c10::Device 的状态机解析，缺省索引 -1 表示当前设备，第三方后端经 PrivateUse1 接入。layout 决定内存存储结构类别（Strided/Sparse 等），由 c10::Layout 枚举列出、默认 Strided，须与 stride 步长区分。c10::TensorOptions 把三轴打包为全可选、按值传递的配置包（压在 128 位内），同时存「值」与「是否显式设置」以区分默认与未指定，右偏 merge_in 实现单轴继承；其 computeDispatchKey 把三轴折算为 DispatchKey，构成通向 Unit 3 Dispatcher 选 kernel 的桥梁。关键术语：ScalarType、DeviceIndex、c10::Layout、TensorOptions、merge_in、computeDispatchKey、DispatchKey、DeviceContext；Python 的 dtype/device/layout 三属性继承自 C++ TensorBase，set_default_device 实为 DeviceContext(TorchFunctionMode)。
