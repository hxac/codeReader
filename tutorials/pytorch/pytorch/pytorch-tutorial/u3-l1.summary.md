本讲讲解 `native_functions.yaml`——PyTorch 全部公开算子的单一事实来源，所有 C++ 声明、Python 绑定、类型桩均由 torchgen 从它派生。每条算子即一段 schema「名字.重载(参数列表) -> 返回」，支持 `?` 可选、`=默认值`、`*` 关键字分隔、`Tensor(a!)` 别名标注等语法。核心要点：三件套模式——functional（无标注）/ inplace（`self` 标 `(a!)`）/ out（`out` 标 `(a!)`），其中 out 常为 `structured: True` 真身，另两者经 `structured_delegate` 委托以共享形状推导逻辑。`dispatch` 是 dispatch key 到 kernel 的映射：`CompositeImplicitAutograd` 实现通吃所有后端且隐式可微（自动拼梯度），`CompositeExplicitAutograd` 仅走推理、反向公式须去 `derivatives.yaml` 找，真正的求导公式不在本文件。`variants` 决定生成函数式还是方法式。本讲止于 schema 与字段语义，代码生成（u3-l2）与 Dispatcher 运行时查表（u3-l3）留待后续。
