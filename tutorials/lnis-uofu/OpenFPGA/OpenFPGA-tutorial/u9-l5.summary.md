本讲讲解 OpenFPGA 如何对大阵列 GSB 去重压缩与缓存。`DeviceRRGSB` 用三张 unique id 矩阵（SB/CBX/CBY）记录每个坐标归属的 unique 模块，`rr_gsb_` 存内容、id 矩阵存归类，`is_compressed_` 标志标记归类完成。压缩分三层：SB/CB 各用 `is_sb_mirror`/`is_cb_mirror` 做拓扑镜像判定（依赖 RRGSBEdges 入边信息），GSB 层用「三 id 全等」汇总，`build_gsb_unique_module` 是 `is_compressed_` 唯一置位点。`--compress_routing` 与 `is_compressed()` 联动决定现场压缩或跳过比较。`read/write_unique_blocks` 用 XML 或 capnp 二进制缓存 unique 划分，preload 无条件相信文件、不做比较，把镜像识别变为廉价回灌，但缓存须与架构严格配套，换架构需重新生成。核心更新：GSB 版本（V1/V2）不再为命令选项，而由 VPR device context 在 RR graph 生成时写入、link 阶段读出，新增 `gsb_version_set_` 守卫使「未设置即读取」从静默错误变为快速失败，V2 走 side-agnostic 的 `build_rr_gsb2` 路径。注意区分 `write_unique_blocks`（去重缓存）与 `write_gsb`（逐坐标调试 dump，不做去重）。
