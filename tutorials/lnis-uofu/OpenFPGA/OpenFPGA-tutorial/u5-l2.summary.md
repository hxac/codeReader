本讲打开 `openfpga_arch.xml` 的「解析」黑盒，追踪 `read_openfpga_arch` 命令如何把 XML 文本变成内存中的 `openfpga::Arch` 对象，覆盖四个最小模块。其一，命令遵循「命令注册 → 执行模板 → 库实现」三层：编排在引擎层，真正解析在 `libarchopenfpga` 的总入口 `read_xml_openfpga_arch()`。其二，总入口是编排器，按「定义类（circuit_library、technology_library）→ 绑定类（configuration_protocol、布线段、pb_type 注解）」固定顺序调用各 `read_xml_*` 子解析器，并在每段后穿插 `link_*`/`bind_*` 链接动作。其三，「解析存名字 → 链接查 ID」有三个落点：模型间引用（`read_xml_buffer` 存名 + `build_model_links` 查 ID）、配置协议内存模型（`set_memory_model_name` 存名 + `link_config_protocol_to_circuit_library` 查 ID）、电路-工艺绑定。其四，XML 字符串与 C++ 枚举靠两类常量桥接：`circuit_types.h` 的字符串表（与枚举同下标）和 `*_xml_constants.h` 的标签/属性名常量，转换函数统一用「表查找 + 哨兵 `NUM_XXX_TYPES`」模式。核心结论：子解析器只填字段不做跨对象对账，对账交给链接器；解析后执行模板用 `check_circuit_library`/`check_config_protocol`/`check_tile_annotation` 三道只读闸门做语义校验，任一失败即返回 `CMD_EXEC_FATAL_ERROR`；所有 XML 错误被 `try/catch` 捕获后经 `archfpga_throw` 带行号重抛。关键术语：pugixml/pugiutil（带行号的 XML 访问）、强类型 ID（`CircuitModelId`）、`loc_data`（行号上下文）。本讲的 `link_*` 只解决 OpenFPGA 架构内部链接，下一讲 u5-l3 将讲 `link_openfpga_arch` 如何把 `Arch` 与 VPR device context 跨工具关联。
