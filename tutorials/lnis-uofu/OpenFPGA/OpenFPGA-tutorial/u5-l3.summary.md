link_openfpga_arch 是 vpr 与 build_fabric 之间的桥梁：它把 read_openfpga_arch 解析出的悬空名字字符串，在 VPR 已建好的 device 里反查对账，落实成 VPR 指针与 CircuitModelId。编排器 link_arch_template 按固定顺序串起十余个标注子函数，顺序即依赖：先标注 pb_type 与 pb_graph，再绑定布线电路模型、构建 DeviceRRGSB、收集 mux 库与 tile 直连，最后做聚类布局标注及可选的 activity、仿真与比特流设置。链接结果几乎全部汇入 VprDeviceAnnotation——一个不修改 VPR 源码、以 VPR 对象指针为键平行挂载电路级信息的中央账本，体现 annotation 模式。其中 annotate_pb_types 最复杂，分 physical mode、physical pb_type、circuit model、mode bits 等段，各走显式反查加隐式推断加 check 闸门：单模式 pb_type 可自动推断 physical mode，多模式必须显式声明。关键术语：annotation 模式、operating 与 physical pb_type、GSB 版本（由 VPR device context 注入而非命令选项，gsb_version_set_ 守卫防未设即读）、activity_file 与 sort_gsb_chan_node_in_edges。该命令被命令依赖机制列为 build_fabric 等下游命令的硬前置。
