本讲承接 device 级比特流，讲 `build_fabric_bitstream` 命令如何将其重组为可下载的 fabric 级比特流（`FabricBitstream`）。核心是沿模块层级做 DFS「只读重组」：用「块名==实例名」把配置位按物理访问顺序重排，位值不重复存（仅保留 `ConfigBitId` 引用，额外附加寻址信息）。重组方式由 `config_protocol.type()` 分派：standalone 直接收集、scan_chain 在区域内部反转位序（链式头进尾出）、memory_bank 逐位算 (BL,WL) 地址、frame_based 算层级地址、ql_memory_bank 委托专用函数。关键术语：紧凑表示 `FabricBitstreamMemoryBank` 用 `datas`/`masks` 两张 (region,wl) 位图存储（每 `uint8_t` 装 8 个 BL），把逐位存地址的几十 GB 压到几 MB，且仅对 ql_memory_bank 的「双 flatten」启用；只要 BL 或 WL 用 decoder/shift_register，就退回逐位字符串地址（decoder 为二进制、flatten/shift_register 为 one-hot）。ql_memory_bank 的 BL/WL 可各自选 decoder/flatten/shift_register 三种子协议，由此决定「跳过几个尾部译码器子模块」与输出形态（逐位地址 / 按 WL 行 / 字式）。教科书级命令依赖链 `repack → build_architecture_bitstream → build_fabric_bitstream → write_fabric_bitstream` 在本讲落地，文件格式与 `fast_configuration` 留待下一讲。
