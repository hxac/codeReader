本讲讲的是如何从空仓库编出 `openfpga` 二进制并配置运行环境。核心结论：OpenFPGA 是一套工具，依赖 yosys（综合）、vtr（布局布线）、yosys-slang 三个 git 子模块；clone 后这些目录为空，必须先 `make checkout`（即 `git submodule update --init --recursive`）填满，这是编译的绝对前置条件。关键概念：构建是两层结构，外层 Makefile 仅作封装、内层 CMake 才是内核。三个核心目标为 checkout（拉子模块）、compile（背后是 prebuild 的 cmake 配置加 `make -C build`）、all（前两者串行，全新环境首选）。可调参数有 BUILD_TYPE（release/debug）、cmake_goals（缩小编译范围，如只编 openfpga）、CMAKE_FLAGS（传 `-D` 给 CMake）。CMakeLists.txt 通过一组 `OPENFPGA_WITH_*` 选项（如 YOSYS、INSTALLER）控制构建范围，版本号从 VERSION.md 读取（当前 1.2.4307），子工程按 vtr→libs→openfpga 顺序纳入，要求 C++20。最后 `source openfpga.sh`（须在仓库根目录）在当前 shell 设置 OPENFPGA_PATH 等变量并定义 run-task、goto-task 等函数，为下一讲跑设计流铺路。
