本讲是把上一讲编出的 `openfpga` 二进制真正跑起来，完成从 Verilog 到比特流的端到端流程。核心是三层调用链：`openfpga.sh` 定义的快捷函数（`run-task`/`run-flow`/`goto-task` 等）→ 批量调度器 `run_fpga_task.py` → 单次流程执行器 `run_fpga_flow.py` → `openfpga -batch -f` 执行 `.openfpga` 脚本。`openfpga.sh` 供 `source` 加载，导出 `OPENFPGA_PATH` 等环境变量与 bash 函数（必须用 source 而非 bash 执行）。一个任务即一个含 `config/task.conf` 的目录，`task.conf` 用 INI 段落（`[GENERAL]`/`[OpenFPGA_SHELL]`/`[ARCHITECTURES]`/`[BENCHMARKS]`/`[SYNTHESIS_PARAM]`）描述「架构 × 基准 × 脚本参数」，job 数为三者笛卡尔积。调度器用 `runXXX` 编号 + `latest` 软链接组织多次运行，多线程经 `Semaphore(maxthreads)` 限流，汇总 `task_result.csv`。关键结论：fabric 网表与比特流并非 Python 生成，而是由 `openfpga` 二进制执行 `.openfpga` 模板（如 `write_full_testbench_example_script.openfpga`）的命令链产出，`run_fpga_flow.py` 仅做 Yosys 综合、模板变量替换（`safe_substitute`）与编排。引入的关键术语：任务/job、`task.conf`、配置协议 `cc`（scan_chain）、运行模式 `-batch -f`，这些为 u2（shell 命令）、u3（架构文件）承接。
