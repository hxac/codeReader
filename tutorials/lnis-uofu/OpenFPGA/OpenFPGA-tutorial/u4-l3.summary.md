本讲打开 pb_type_annotations 节点，说明如何把 VPR 的 pb_type 绑定到 circuit_model。关键概念：operating pb_type（可打包模式，VPR 打包目标）与 physical pb_type（不可打包的物理实现模式，OpenFPGA fabric 实现目标）。每条 `<pb_type>` 注解解析为一个 PbTypeAnnotation 对象，只存名字字符串、不存 ID，承载三类绑定：operating↔physical、pb_type↔circuit_model、interconnect↔circuit_model。判定类型的唯一依据是 physical_pb_type_name 的有无：有即 operating，无即 physical；当某 pb_type 只有一个 mode 时该 mode 默认即 physical 模式。路径语法 `a[mode].b.c` 中方括号是下一级 pb_type 所在的 mode 名。mode_bits 写入物理电路的 mode_select sram 端口，以一个物理模型加少量配置位实现多种工作模式（典型如 inpad/outpad 共享同一 GPIO，靠 mode_bits 1/0 区分输入输出方向）；x（don't-care）位只允许出现在 operating 注解里，physical 必须完全确定。这些名字字符串如何翻译成 CircuitModelId 并与 VPR 的 pb graph 对账，留待 u5。
