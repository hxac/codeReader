本讲从「引擎视角」揭示 `openfpga_arch.xml` 在内存中凝结成的 C++ 对象。核心是 `openfpga::Arch`——电路级架构的只读聚合根，含 10 个成员（`circuit_lib`、`tech_lib`、派生的 `circuit_tech_binding`、`config_protocol`、布线三段绑定 `cb/sb/routing`、`arch_direct`、`tile_annotations`、`pb_type_annotations`），分别由各顶层节点解析填充，构建后全引擎仅以 `const` 访问以支撑模块化。其生命周期遵循统一范式：**解析存名字 → 链接查 ID（强类型 ID）→ 只读冻结**。配套讲解三个辅助模型：`TechnologyLibrary`（SoA + 强类型 ID 组织晶体管/RRAM 器件与工艺偏差，供 SPICE 展开）、`ArchDirect`（CLB 直连的电路模型 + 几何类型/方向）。最关键的纠正：`SimulationSetting` 与 `BitstreamSetting` **并不在 `Arch` 内**，而是 `OpenfpgaContext` 里与之平级的兄弟结构，各有独立根标签、读取命令与文件；旧注释「simulation parameters 属于 Arch」已过时，以结构体定义为准。
