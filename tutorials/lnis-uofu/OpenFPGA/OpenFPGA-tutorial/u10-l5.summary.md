本讲是手册收官篇，讲解如何验证改动、CI 把关与贡献规范。OpenFPGA 采用端到端回归测试三层体系：run-task 跑单个含 task.conf 的任务，basic_reg_test.sh 串起几百个任务，CI 用 matrix 并行跑多个回归脚本。其「定盘星」是脚本结尾对黄金产出做 git diff 字节级比对，抓住「流程跑通但产物退化」的隐蔽回归。黄金产出即被提交进 git 的正确答案；VPR 以 MACRO 命令（不注册选项、整行参数透传给 VPR 自带解析器）经 vpr_wrapper 嵌入 shell，分 vpr（堆分配、保留结果供后续命令读取）与 vpr_standalone（栈分配、跑完即释放）两种包装。代码风格用 make format-cpp/xml/py 三件套（clang-format、xmllint、black），由 dev/check-format.sh 的「format 后 git diff」格式闸门强制。新增回归的标准范式是「一个功能任务 + 一个 no_time_stamp 黄金任务 + 回归脚本加两行 run-task」，Bus Based MUX（总线型 mux 共享配置位）即其教科书式实例。
