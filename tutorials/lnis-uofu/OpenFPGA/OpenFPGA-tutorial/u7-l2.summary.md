本讲讲解 `build_architecture_bitstream` 如何生成 device 级、协议无关的 `BitstreamManager`：入口 `build_device_bitstream` 先建顶层 `ConfigBlock` 并预估容量，再分交 grid 与 routing 两路解码，最后断言块数/位数与预估一致。grid 比特流沿 `pb_graph` 递归，三类来源——LUT 真值表（经 2^K:1 mux_graph 反查）、primitive 的 mode_bits、pb_graph 内部互连 mux；routing 比特流遍历各 GSB 的 SB/CB，用「输出 net + 前驱 rr_node」双重匹配定出每个布线 mux 的 path id（fan-in>1 即为 mux）。两路最终都把 path id 交给共用引擎 `build_mux_bitstream`：`decode_memory_bits` 反查 raw 位、local encoder 再逐级二进制编码，未使用 mux 按 `unused_mux_config`(first/last/auto/unused_input) 选默认路径。本讲重点引入**总线型 mux（`bus="true"`）**：W 位 N:1 mux 共享单个选择器，配置位由 W 份压成 1 份；强约束是「整条 bus 各位须选同一输入端口」否则致命错；共享存储块以最低代表位命名以与 fabric 模块侧对齐。关键术语：path id、`DEFAULT_PATH_ID=-1`、`PhysicalPb`、`VprRoutingAnnotation`、代表位。device→fabric 的重组、文件格式与 fast_configuration 留给 u7-l3、u7-l4。
