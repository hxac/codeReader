本讲讲解 OpenfpgaContext——OpenFPGA shell 中命令间交换数据的唯一全局中枢。它由 OpenfpgaShell 唯一持有，只创建一次、不可拷贝，继承自 VPR 的 Context，并聚合架构（arch_）、Fabric 模块图（module_graph_）、两级比特流（bitstream_manager_/fabric_bitstream_）、多路选择器库（mux_lib_）等核心数据。关键设计是 const/mutable 两套访问器：arch() 只读返回 const T&，mutable_arch() 可写返回 T&，命名即意图，并与 set_command_const_execute_function（只读）/set_command_execute_function（可写）两种执行函数一一对应，使「命令是否改数据」在注册时即声明。FlowManager 记录跨命令的流程级开关（如 compress_routing），由产生副作用的命令写入、供下游查询。核心结论：命令的先后顺序本质就是 Context 各分区的读写依赖；本讲将 arch_ 视为黑盒，留待 u3 打开。
