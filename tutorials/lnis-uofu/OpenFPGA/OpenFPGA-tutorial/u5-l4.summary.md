本讲系统讲解 `openfpga/src/annotation/` 子系统。核心是 **annotation（标注）模式**：因 VPR 是不可侵入的第三方代码，OpenFPGA 不改其类，而是以 VPR 对象的指针或强类型 id 为键，在 `OpenfpgaContext` 上平行挂「副表」叠加 OpenFPGA 专属信息，从而隔离升级、分离关注点；写入方用 mutable 访问器、读取方用 const 访问器。目录内容分三类：一是标注 VPR 结果的六大便利贴——`VprDeviceAnnotation`（pb_type/RRSwitchId→电路模型、physical mode 等）、`VprNetlistAnnotation`（block/net 改名）、`VprClusteringAnnotation`（net 重映射、修正真值表、`PhysicalPb` 打包）、`VprPlacementAnnotation`（含未映射格子）、`VprRoutingAnnotation`（rr_node 的 net 与前驱）、`VprBitstreamAnnotation`（比特流来源）；二是 OpenFPGA 自有设备结构——`DeviceRRGSB` 收集每个坐标的通用开关块（SB+CBX+CBY），用「三者 unique id 全同即互为镜像」做 **unique module 压缩**，由 `is_compressed_` 标记完成，并通过 `read/write_unique_blocks` 以 preload 跳过比较做缓存加速；`FabricTile` 把 grid+sb+cb 打包成可复用 tile，其 `equivalent_tile` 依赖 `DeviceRRGSB`。三是填充/检查动作。`gsb_version` 由 VPR device context 在 link 阶段注入并用 `gsb_version_set_` 守卫，保证先设置后读取。`check_netlist_naming_conflict --fix` 把非法标识符字符替换为下划线、改名写入 `VprNetlistAnnotation` 而不动 VPR 原始 netlist。
