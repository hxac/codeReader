本讲解析 OpenFPGA 的命令分组与注册机制。OpenFPGA 在 `OpenfpgaShell` 构造函数里按固定顺序注册七大命令类别：VPR、OpenFPGA setup、FPGA-Verilog、FPGA-Bitstream、FPGA-SPICE、FPGA-SDC、Basic，对应七个 `add_*_commands` 函数。顺序不可调换，因为后续组要靠 `shell.command("名字")` 反查前置命令（如 setup 反查 `vpr`、其余组反查 `build_fabric`）来构造依赖边。

引入的关键术语：**命令类别**（仅用于 `help` 分组显示的标签）、**Shell<T>**（自带命令框架，`T` 为 `OpenfpgaContext`）、**MACRO 命令**（自带解析器、shell 透传整行参数的黑盒，如 `vpr`）、**执行函数类型**（`set_command_const_execute_function` 只读 context、`set_command_execute_function` 可写 context）、**命令依赖**（运行时硬约束，非注释）。

核心结论：每条命令遵循**四步注册法**——建 `Command` 加选项 → `add_command` 取 id → `set_command_class` 归类 → 绑定执行函数并可选声明依赖。setup 组主链为 `read_openfpga_arch → link_openfpga_arch → build_fabric`；bitstream 组有教科书级依赖链 `repack → build_architecture_bitstream → build_fabric_bitstream → write_fabric_bitstream`。`shell.tpp` 的 `execute_command` 在调用执行函数前遍历依赖列表，前置命令未运行或失败即报错返回，依赖只检查一级、不递归。**Basic 组必须最后注册**，且组内 `help` 必须最后添加，因其执行函数要对已注册命令集做快照来打印帮助。
