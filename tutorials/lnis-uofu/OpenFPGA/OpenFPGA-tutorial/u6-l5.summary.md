本讲是 u6（Fabric 构建）收尾，讲 build_fabric 之后 OpenFPGA 导出/缓存哪些 fabric 元信息供下游使用，覆盖四类产物。fabric key 并非密钥，而是可配置存储器实例的「地址编排清单」，记录每个配置位实例的模块名、实例号、别名、坐标与所属 config region；由 build_fabric 的 --write/--load/--generate_random_fabric_key 选项或独立 write_fabric_key 命令生成，底层汇入 write_fabric_key_to_xml_file() 落盘 fabric_key.xml，其中 --generate_random_fabric_key 靠打乱存储器地址实现混淆/安全。fabric hierarchy 由 write_fabric_hierarchy 经 DFS 输出带缩进的纯文本模块树，服务层次化 PnR。io_location_map 在 build_fabric 内部构建，建立顶层 GPIO pin 与 (x,y,z) 坐标的反查表，供 testbench 与 PCF 互译，可由 write_fabric_io_info 导出。FabricGlobalPortInfo 从 circuit_library 与 tile_annotation 收集全局端口（clock/set/reset/prog/BL/WL 等）属性，仅在内存供 testbench/bitstream 读取，无写文件命令。三类写命令均属 OpenFPGA setup 组、绑定 const 只读执行函数、依赖 build_fabric。另引入 module_name_map（由 rename_modules 填充，为 fpga_core wrapper 自定义命名基础）；blwl_sr_banks 与 BL/WL 特判衔接后续移位寄存器 bank。
