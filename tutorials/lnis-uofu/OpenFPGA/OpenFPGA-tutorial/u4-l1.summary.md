本讲解析 canonical 脚本 example_script.openfpga，将其分为「输入→处理→输出」三阶段，命令顺序由 OpenfpgaContext 的读写依赖决定。脚本中 ${} 是 Python string.Template 占位符，由 run_fpga_flow.py 的 safe_substitute 替换，openfpga 本身不解析，直接 -f 模板会跑不通。vpr 作为首命令把设备数据写入共享 context，link_openfpga_arch 是 vpr 与 build_fabric 的桥梁。处理阶段串行链为 build_fabric→repack→build_architecture_bitstream→build_fabric_bitstream→write_fabric_bitstream，其中 repack 把网表从逻辑 pb 重打包到物理 pb，须在比特流生成前完成；比特流分 device 级与 fabric 级。验证产物三种：write_full_testbench 含编程阶段（慢）、write_preconfigured 系列内嵌比特流跳过编程（快）。依赖检查仅覆盖 context 数据依赖、只查一级不递归，文件级依赖需人工保证。
