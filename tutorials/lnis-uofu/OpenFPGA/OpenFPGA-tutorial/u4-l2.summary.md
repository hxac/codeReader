本讲打开任务执行黑盒，拆为四个最小模块。任务即含 config/task.conf 的目录，其 INI 段中 GENERAL/BENCHMARKS/ARCHITECTURES 必需，另有 OpenFPGA_SHELL、SYNTHESIS_PARAM、SCRIPT_PARAM_*；调度器 run_fpga_task.py 按「架构×基准×脚本参数集」笛卡尔积切 job，总数为三者乘积，脚本参数集靠段名含 SCRIPT_PARAM 识别。每次运行落递增编号的 run001 与 latest 软链接，单 job 产出在 <runXXX>/<arch>/<top_module>/<参数标签>/ 下，并行由 --maxthreads 配信号量限流，--test_run 可只切 job 不执行。基准位于 benchmarks/，micro_benchmark/and2 为最简微基准，靠 benchN_ 前缀在 BENCHMARKS 与 SYNTHESIS_PARAM 间对齐。回归测试分两层：basic_tests/ 按特性分任务，regression_test_scripts/*.sh 串成脚本，build.yml 矩阵式触发，并用 git diff 守护 no_time_stamp 黄金产出。核心结论是新特性落地回归的标准范式：「一个能跑通的功能任务 + 一个锁定关键产出的 no_time_stamp 任务（用 .gitignore 白名单只锁关键文件）+ 回归脚本各加一行 run-task」，本次「Bus Based MUX」（bus 互连共享单个配置位）即其最新实例。
