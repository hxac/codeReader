本讲是 en_cl_fix 库的地基，专讲定点格式 `[S,I,F]`，不涉及舍入、饱和、运算（留待 u2-l2/l3/l4）。三元组中 S 为符号位数（只能 0 或 1），I 为整数位，F 为小数位，I、F 均可为负，从而把「小数点位置」与「存储宽度」解耦，是库的核心设计。核心结论：总位宽 W=S+I+F（Python 的 `width` 属性与 VHDL 的 `cl_fix_width` 公式逐字一致）；位权重以 LSB 为锚，最低位恒为 2^-F，符号位为 -2^I（补码），其余逐位翻倍；可表示范围最大值 2^I-2^-F（任意 S），最小值有符号为 -2^I、无符号为 0，对应 VHDL `max_real`/`min_real`。承接 u1-l2 的三语言镜像架构：Python 用 `FixFormat` 类（断言 S∈{0,1}、I+F>=0 以拒绝边角格式），VHDL 用 `FixFormat_t` record（`natural range 0 to 1` 约束），二者语义完全镜像。关键术语：定点（fixed-point）、补码、位权重、锚点模型，以及哨兵常量 `NullFixFormat_c=(0,0,-1)` 表示「未指定格式」——它是 Python 会拒绝的越界值，属两语言刻意差异。这些概念将贯穿后续舍入（FixRound）、饱和（FixSaturate）与结果格式预测讲义。
