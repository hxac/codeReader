本讲承接 u7-l1，讲清 VHDL 测试台如何把 Python cosim 落盘的黄金数据读回、逐拍喂给被测器件并比对，主讲范例为 `cl_fix_round_tb`。核心模块有四：一是 VUnit 测试台运行骨架——`test_runner_setup` → `while test_suite`/`run("test")` → `test_runner_cleanup`，辅以 `test_runner_watchdog` 防卡死，测试名由 `run.py` 的 `get_tests` 与 VHDL 的 `run(...)` 字符串对齐。二是 `en_cl_fix_fileio_pkg`，一层「格式感知糖衣」，通过 `cl_fix_to_signedness` 把 `FixFormat_t` 翻译成 (位宽, 符号性) 再委托 `en_tb` 的 `read_file`，免去手算位宽。三是黄金参数读取：格式串用 `FixFormatArray_t`（经 `cl_fix_read_format_file` → `cl_fix_format_from_string` 解析 `"(S,I,F)"`），舍入模式用 `integer_vector`，在架构顶层一次性读入，用例数取自 `a_fmt_c'length`，并靠 `tb_path(runner_cfg)` 定位数据目录。四是用 `for ... generate` 在编译期把测试台展开为多个并行子测试，每个子测试是「输入进程 + UUT + 检查进程」三件套，支持各异信号位宽与独立 generics。检查进程以 `out_valid` 握手（非固定延迟）逐拍比对，失败打印精确诊断，并用「同种子双随机源」验证 meta 透传。引入的关键术语：VUnit、UUT、`runner_cfg`、`tb_path`、`generate` 展开、`SlvArray_t`、`FixFormatArray_t`、`RegisterMode_t` 轮换，为 u7-l3 的 `run.py` 装配作铺垫。
