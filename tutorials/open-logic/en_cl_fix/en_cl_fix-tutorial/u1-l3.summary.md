本讲属「环境与运行」层，回答 en_cl_fix 仓库如何跑起来。承接前置「三语言镜像架构」（VHDL 金标准、Python 参考模型、MATLAB 薄封装）与「验证闭环」（Python cosim 先生成黄金数据，测试台再逐拍比对），本讲给出：依赖分三类——Python（numpy 与 vunit-hdl，由 requirements.txt 精确锁定，经 python -m pip install -r 安装）、MATLAB（R2023b）、VHDL 仿真器（GHDL/NVC/Modelsim/Questa，由 VUnit 驱动），并对应三类独立测试入口。

两个脚本构成仿真骨架：sim/common.py 为 VUnit 挂上 --simulator、--simulator-path 等五个自定义参数（命令行优先、环境变量兜底），翻译为 VUNIT_SIMULATOR 环境变量，落实 RTL 用 VHDL-93、TB 用 VHDL-2008 双标准；sim/run.py 的 create_test_suite 把 OSVVM、en_tb 子库、RTL 与各测试台装入 VUnit 工程，并为每个测试台绑定 pre_config=cosim.run 回调，仿真前先由 Python cosim 生成黄金数据；cosim_runner 以全局锁加双重检查与自禁用，保证同名 cosim.py 线程安全且最多执行一次。关键术语 VUnit、UUT、cosim、pre_config 回调供后续讲义承接。
