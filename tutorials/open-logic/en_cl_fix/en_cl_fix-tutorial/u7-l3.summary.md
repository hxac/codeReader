本讲是 U7「Python↔HDL 协同仿真验证流程」的收口段，讲解把 cosim 黄金数据生成端与 VHDL 测试台比对端粘合起来的 Python 装配层，即 `sim/` 下三个脚本。`common.py` 是「公共底座」，在 VUnit 命令行上挂载 `--simulator`、`--simulator-path`、`--vendor-lib`、`--coverage`、`--disable-cosim` 五个自定义参数（每个都有环境变量兜底），校验后翻译成 `VUNIT_SIMULATOR` 等环境变量，并按仿真器产出两个关键常量 `vhdl_standard_rtl` 与 `vhdl_standard_tb`：Modelsim/Questa 下 RTL 用 93、TB 用 2008，GHDL/NVC 下都用 2008——分两套标准是为了保证可综合 RTL 严格兼容 VHDL-93。`run.py` 是「装配总指挥」，其核心函数 `create_test_suite` 把 OSVVM 库、共享的 `en_tb` 子库（用 `try/except` 防重复创建）及项目自用 `lib` 库（RTL、TB 工具、TB 主体分批装入）组织起来，再为每个测试台通过 `add_config` 把「cosim 脚本路径、generics、pre_config 回调」三件套绑在一起；round/saturate/resize 因声明了 `meta_width_g`，分别用 0 和 8 两种边带宽度各配一遍。`cosim_runner.py` 是「cosim 执行器」，用「全局锁保护 sys.path、局部双重检查锁加自禁用」保证所有同名 `cosim.py` 安全导入且其 `run()` 在并行测试下最多执行一次，`--disable-cosim` 通过初始 `enable=False` 短路。末尾按 GHDL/NVC/Modelsim 分别设置编译仿真选项，键名互不干扰，并为 GUI 模式自动绑定波形脚本，使批跑与调试共用同一份 `run.py`。关键术语：装配层、vhdl_standard_rtl/tb、create_test_suite、add_config/pre_config 回调、meta_width_g、cosim_runner、双重检查锁、单次执行保证。
