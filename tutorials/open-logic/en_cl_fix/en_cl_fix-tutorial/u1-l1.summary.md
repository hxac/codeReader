本讲解析 en_cl_fix 的 Python 主接口 `en_cl_fix.py`，核心结论是其每个 `cl_fix_*` 函数都与 VHDL 包同名同语义，构成「镜像」，使 Python 参考模型可直接对照验证 HDL。

最小模块覆盖四点。其一，格式预测函数（如 `cl_fix_add_fmt`、`cl_fix_mult_fmt`）只是 `FixFormat` 静态方法（`for_add`/`for_mult` 等）的直接别名，无额外开销。其二，算术函数（加、减、乘、绝对值、取反、移位）统一遵循「预测全精度中间格式 `mid_fmt` → 在 `mid_fmt` 下做真实无损运算 → 用 `cl_fix_resize` 收敛到目标格式」三段式；运算本身无损，精度损失集中在 resize。其三，`r_fmt=None` 时目标格式等于 `mid_fmt`，返回无损全精度结果；显式指定更窄的 `r_fmt` 才触发舍入与饱和。其四，`cl_fix_resize` 结构为「先 round、后 saturate」不可交换，因为饱和阶段要求小数位 F 不变，必须先 round 对齐。

引入的关键术语供后续承接：全精度中间格式 `mid_fmt`、结果格式 `r_fmt` 的无损开关语义、`cl_fix_resize = cl_fix_round ⟶ cl_fix_saturate` 的精度收敛器、以及 `cl_fix_is_wide` 以 53 位为界把内部计算分发到 `NarrowFix`（≤53 位，双精度浮点，快）或 `WideFix`（任意精度整数，慢），遵循「任一相关格式 wide 即全程 wide」的提升规则。接口对外只暴露裸数据，narrow 返回 float64、wide 返回 object 整数数组。
