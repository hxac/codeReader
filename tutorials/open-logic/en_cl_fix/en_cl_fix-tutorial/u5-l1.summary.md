本讲只读 `en_cl_fix_pkg.vhd` 包头（声明），不进函数体。它定义四个核心类型：`FixFormat_t`（record，表达定点格式 `[S,I,F]`，子类型约束在编译期挡住非法值）与三个枚举 `FixRound_t`（舍入七态）、`FixSaturate_t`（饱和四态）、`RegisterMode_t`（寄存器策略三态，只控制流水线组件是否插寄存器，不参与数值计算）。公共 API 按职责分五组：格式函数、字符串转换、类型转换、舍入饱和、数学函数。默认参数三条约定：`round` 取 `Trunc_s`、`saturate` 取 `Warn_s`、`result_fmt` 取 `NullFixFormat_c`；两个破例为 `cl_fix_from_real`（默认 `SatWarn_s`）与 `cl_fix_shift`（无 `result_fmt` 默认）。`NullFixFormat_c=(0,0,-1)` 位宽为负、属非法格式，充当「结果格式未指定」哨兵：数学函数检测到它便回退到全精度中间格式做无损运算、再经 `cl_fix_resize` 收敛，缺省即无损返回，与 Python 侧 `None` 语义镜像。全包遵循 VHDL-93 合规，故 `maximum`/`minimum` 走私有包、`to_string` 不依赖 `'image`。包体实现留待后续讲义。
