本讲讲解 Open Logic `fix` 区域的纯 Python 工具 `olo_fix_pkg_writer`，它是一个把 Python 里算好的常量、定点格式、向量自动渲染成 VHDL 包或 Verilog 头文件的代码生成器，核心动机是建立「Python 单一真相源」，避免参数在 Python 与 HDL 两处各写一份而失配。

四个最小模块：其一，writer 是「累加→渲染」的有状态对象，构造时建两个空字典 `_constants`/`_vectors`；`add_constant`/`add_vector` 把成员以 `MemberData(type, value, as_string)` 存入（含命名查重与类型白名单，fail-fast），`write_vhdl_pkg`/`write_verilog_header` 用 Jinja2 模板渲染成 `<pkg_name>.vhd`/`.vh`。其二，`as_string` 开关决定按原生类型还是 `string` 生成；喂给 `olo_fix` 实体的定点格式推荐 `as_string=True`，对 Verilog 则为强制（Verilog 无 `FixFormat` 类型，原生写法抛 `ValueError`）。其三，VHDL 模板会 `use <olo_library>.olo_base_pkg_array.all` 与 `en_cl_fix_pkg.all`，常量在包头直接赋值、包体为空；浮点经 `_float_str` 统一格式化。其四，关键时序约束：`sim/run.py` 必须在 VUnit 扫描源文件之前调用 `codegen_generate()`，否则生成包不存在、TB 编译失败；生成包被 `.gitignore` 的 `*_pkg.vhd` 忽略，作为可重建构建产物。验证闭环由 `codegen.py` 生成 `pkg_writer_test_pkg` 供自动发现的 TB 以四组用例（native/string × constant/vector）逐项 `check_equal`。

关键术语：代码生成（codegen）、`as_string`、`MemberData`、单一真相源、Jinja2 模板，承接 u8-l1/l2 的定点三元组 `(S,I,F)` 与字符串泛型模式，并为下一讲 u8-l5 协同仿真铺垫数据级位真验证体系。
