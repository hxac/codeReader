本讲讲解 Open Logic 中支撑「可信代码」的三段 Python 脚本，构成「仿真→解析→阈值/徽章→CI 门禁」的质量闭环。`AnalyzeCoverage.py` 调 `vcover` 把覆盖率数据库导出为文本，按 `File:/Branches/Statements` 解析每个 `olo_*` 实体的语句与分支覆盖率，过滤 `*_tb` 后用 `--min_coverage` 判定，任一不达标即 `sys.exit(1)`；PR 到 main 时传 95%，配合分支保护必需检查阻止合并。`AnalyzeIssues.py` 依「实体名即 GitHub label」约定用 PyGithub 关联 issue，`potential-bug`/`confirmed-bug` 两标签决定风险等级。`Badge.py` 是被另两者调用的底层库，把数据组装成 shields.io 端点 JSON 上传到 GCS 公开桶：覆盖率徽章实际仅 ≤90 红 / >90 绿两档（橙色 `elif` 不可达，为死代码），issue 徽章按「确认 bug 红 > 疑似 bug 橙 > 无绿」分色。关键设计：先生成徽章后判阈值，绝不掩盖差覆盖率；CI 按成本分级——覆盖率仿真（昂贵，AWS runner）只在 PR/push 到 main 触发，issue 徽章（免费 runner）每日更新，贡献阶段由 HDL-Check 用开源仿真器兜底。本讲还指出 `AnalyzeCoverage.py` 的兜底守卫变量误用列表导致恒真。
