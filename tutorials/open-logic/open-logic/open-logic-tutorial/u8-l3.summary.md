本讲把定点运算统一分解为 operation → round → saturate 三段。olo_fix_round 只减小数位、olo_fix_saturate 只减整数/符号位，olo_fix_resize 经中间格式 RoundFmt 把二者串联为通用格式转换器。每段是否有真实逻辑由 LogicPresent_c 判定（round 看小数位、saturate 看整数/符号位及饱和模式），再由 fixImplementReg 的 YES/NO/AUTO 三态结合私有可选寄存器 olo_fix_private_optional_reg 决定是否插寄存器。add/sub/mult 与 neg/abs 结构同构：先用 cl_fix_*_fmt 算出全精度自然格式、硬编码 Trunc/Warn 不丢精度，再接 OpRegs 与 resize 收敛到目标格式，默认延迟 3 拍，用户的 Round/Saturate 仅在 resize 段生效。compare 输出 1 位布尔、无 round/sat，靠字符串泛型 Comparison_g 选择运算符。from_real/to_real 是唯一无寄存器实体对，real 仅存于综合/仿真期，to_real 限 testbench 使用。所有实体与对应 en_cl_fix 函数位真等价，可用协仿真测试台验证。
