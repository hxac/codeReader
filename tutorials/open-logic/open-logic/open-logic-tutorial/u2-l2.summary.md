本讲以 `olo_base_pl_stage` 为载体讲解流水线阶段与 AXI-S 握手。其核心价值不是简单寄存数据，而是切断长组合路径——尤其 Ready 反压路径易成为关键路径。代码采用 Open Logic 全库通用的「两进程法 + record」：组合进程 `p_comb` 只算下一拍状态 `r_next`，时序进程 `p_seq` 只打拍并复位，状态收进 record，复位以进程末尾覆盖实现、仅复位状态位。为把 Ready 寄存一拍又不丢数据，带反压的一级实质是「双-entry FIFO」，由主寄存器与 shadow（影子）寄存器组成；当 `IsStuck`（主寄存器满且下游不收且有数据来）成立时，多余数据落入影子，下游腾位时再提升为主寄存器输出。两个泛型：`Stages_g` 用 `for generate` 串联单级、取 0 退化为纯直通；`UseReady_g=false` 关闭反压、退化为最简寄存器，并以一组跨厂商综合属性强制做成独立 FF，换取面积但不可接会反压的下游。引入的关键术语：两进程法、record、反压、shadow 寄存器、IsStuck、Stages_g、UseReady_g。
