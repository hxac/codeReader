本讲讲解 Open Logic 如何用 VUnit 搭建仿真流程。总入口 `sim/run.py` 是一段配置脚本，依次完成「构建库、收集源文件、注册测试台配置、交给 `vu.main()` 运行」：生产源码进 `olo` 库、测试台（Testbench，TB）进 `olo_tb` 库，二者分离便于覆盖率统计。关键术语：仿真即在 PC 上模拟硬件以验证 RTL；Generic（泛型）是实例化时确定的参数；VUnit 自动发现源文件，并按 generic 把同一 TB 注册成多个测试用例、调度仿真器逐个运行。

仿真器由一组互斥开关选择——`--ghdl`（默认）/`--nvc`/`--modelsim`/`--rivierapro`，经环境变量 `VUNIT_SIMULATOR` 传递；`--coverage` 仅与 `--modelsim` 或 `--nvc` 兼容。TB 配置按区域（base/axi/intf/fix）拆到 `test_configs/*.py`，由 `named_config()` 把 generic 组合翻译成具名配置，单个运行实例全名为「库.TB.配置.用例」。核心时序约束：仿真启动前必须先 `codegen_generate()` 生成 fix 区域依赖的 VHDL 包，且须在 VUnit 扫描文件之前完成，否则编译失败。
