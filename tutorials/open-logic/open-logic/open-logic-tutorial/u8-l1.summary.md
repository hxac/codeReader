本讲是 Open Logic `fix` 区域（定点运算）的地基课。核心结论：fix 区域建立在第三方库 en_cl_fix 之上，Open Logic 把 en_cl_fix 的 VHDL 函数包装成 `olo_fix_*` 实体、并用字符串泛型对外暴露，根本动机是让同一套组件既可被 VHDL、也可被 Verilog 实例化。

引入的关键术语：定点格式用三元组 `(S,I,F)`（符号位/整数位/小数位）描述，位宽 W=S+I+F、分辨率 Δ=2^-F；对应类型为 `FixFormat_t`（record）、`FixRound_t`（舍入枚举，如 `Trunc_s`、`NonSymPos_s`）、`FixSaturate_t`（饱和枚举，如 `Sat_s`、`None_s`、`Warn_s`）。一个基本运算自然分为 operation→round→saturate 三段，分段旨在段间插入流水线寄存器以提升主频；寄存控制泛型取 `"YES"`/`"NO"`/`"AUTO"`，默认 `"YES"`。

字符串泛型模式：接口上用 `string`（如 `"(1,8,23)"`、`"Trunc_s"`）而非自定义类型，因后者跨语言实例化无统一支持；字符串在实体内部经 `cl_fix_format_from_string` 等编译期函数转回 typed 常量，并经 `fixFmtWidthFromString` 推导端口位宽。编译上 en_cl_fix 必须先于 olo_fix 编进同一 `olo` 库（即必须 `--recursive` 克隆）。配套的 Python 单一真相源（`olo_fix_pkg_writer`）与协仿真（`olo_fix_cosim` 等）留待 u8-l4/u8-l5 详讲。
