本讲讲解 Open Logic base 区四类时序操控实体。延迟方面，olo_base_delay 以泛型固定延迟、按深度自动选 SRL/BRAM；olo_base_delay_cfg 由端口运行期可配、仅用 BRAM，切换后有 <5 拍未定义窗口。关键概念：延迟单位是「数据拍」（仅 In_Valid=1 时前进，非时钟周期），且 RAM/SRL 不可复位，须用 RstStateCnt 掩盖复位残留。脉冲方面，olo_base_strobe_gen 按频率产生单周期脉冲（整数等距/小数高精度两种模式），olo_base_strobe_div 做整数分频（In_Ratio=期望比−1）。olo_base_rate_limit 用 AllowSample 门控同时作用于 Out_Valid 与 In_Ready，把流限为每 Period_g 拍最多 MaxSamples_g 个，SMOOTH 用信用计数器均铺、BLOCK 用周期配额先到先得，平均速率相同。olo_base_latency_comp 非侵入式地对齐旁路与处理通路，DYNAMIC 用 FIFO 吸收可变延迟、FIXED_CYCLES 用固定延迟线，以粘性 Err_Overrun/Err_Underrun 报告对齐失配。
