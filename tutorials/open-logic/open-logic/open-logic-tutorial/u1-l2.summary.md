本讲走查 Open Logic 仓库目录布局，建立“代码、测试、文档、脚本各放哪里”的全局地图。

`src/` 存放全部可综合源码，按四个区域（area）划分：`base`（地基，无外部依赖）、`axi`（AXI 总线）、`intf`（UART/SPI 等外部接口）、`fix`（定点运算）；后三者单向依赖 base，fix 额外依赖第三方子模块。实体命名严格遵循 `olo_<area>_<function>`，由文件名即可反推其区域与所在目录。

`test/` 结构镜像（mirror）`src/`，约定为 `test/<area>/<entity>/<entity>_tb.vhd`，每个实体都有专属测试台（testbench）；`test/tb/` 则存放跨实体复用的验证组件（VC，Verification Component），统一以 `olo_test_*_vc` 命名，相当于软件测试中的 mock。`doc`/`sim`/`lint`/`tools` 四个目录分别支撑文档浏览、VUnit 仿真、VSG 风格检查、厂商集成与综合资源评估。

`compile_order.txt` 按依赖（而非区域）排序全部源文件，配合“单库编译”策略把全库串进名为 `olo` 的 VHDL 库；`3rdParty/en_cl_fix` 是 MIT 许可的 git 子模块，仅被 fix 区域依赖，克隆时须加 `--recursive`，否则 fix 无法编译。
