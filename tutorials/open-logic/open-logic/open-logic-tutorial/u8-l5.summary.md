本讲是 fix 区域收尾课，讲解如何用「Python 黄金模型 + 文件中介 + HDL 逐拍比对」实现定点运算的位真（bit-true）协仿真。涵盖三个最小模块：`olo_fix_cosim` 把定点数组写成 `.fix` 文件（首行格式串 `(S,I,F)`，其余每行一个定宽大写十六进制，负数转无符号补码）；`olo_fix_sim_stimuli` 读文件回放激励到 DUT 输入；`olo_fix_sim_checker` 读文件作期望值逐拍比对 DUT 输出，不一致即 `assert` 报错（带文件行号），二者都先用 `fixFileCheckHeader` 校验格式、再以 AXI-S 握手推进。关键概念 timing-master/slave 解决多路同拍配对：每个方向一个 master（stimuli 管 `Valid`、checker 管 `Ready`），其余 slave 共享握手信号。端到端链路依赖 VUnit `pre_config`——在每个用例仿真前先跑 Python 生成文件到 `output_path`，TB 再读回，必须「先生成、再仿真」。同套 `.fix` 格式也兼容 VUnit VC 变体。
