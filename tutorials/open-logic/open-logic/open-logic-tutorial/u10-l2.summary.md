本讲聚焦测试台之外、驱动整个仿真工厂运转的工程层。核心三件：sim/run.py 是约200行的 VUnit 配置脚本（非测试代码），按「导入→备料→解析CLI→选仿真器→建库收源码→加配置→交棒 vu.main()」线性流水线组织；sim/codegen.py 是前置代码生成器；sim/test_configs/ 按区域（base/axi/intf/fix）组织 generic 组合。最关键的时序约束是：codegen_generate() 必须在 VUnit 扫描源文件之前把生成包写盘，否则引用它的 TB 编译失败。仿真器选择靠四个互斥开关（默认 --ghdl）翻译成 VUNIT_SIMULATOR 环境变量；--coverage 仅兼容 modelsim/nvc；--compile_list 与 --vhdl_ls 为导出工件即 exit(0) 的旁路。工具函数 named_config() 把 generic 字典拼成具名配置名并调用 tb.add_config，使一个 TB 配合多组 generic 展开成大量仿真实例，配置选择背后是对内部实现分支的覆盖意图。
