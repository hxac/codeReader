本讲进入 Open Logic 的 intf 区域，专责 FPGA 与外部信号打交道，覆盖三个层层叠加的最小模块。olo_intf_sync 是 N 级（默认 2 级）同步器，将外部异步单比特串过多级触发器以降低亚稳态风险，并配齐防 SRL、防优化、async_reg 等综合属性，保证工具实现成独立触发器；它只能处理相互独立的单比特，不可传多位总线（后者须用 base 区域的 olo_base_cc_*）。olo_intf_debounce 在内部实例化同步器，再用共享 Tick 预分频器加每比特稳定计数器一步完成消抖，分 LOW_LATENCY（边沿近实时转发、会把短脉冲拉宽）与 GLITCH_FILTER（须稳定满 DebounceTime_g、能吃掉短毛刺）两种模式。olo_intf_clk_meas 用已知主时钟产生 1 Hz 秒脉冲，经 cc_pulse 跨时钟域后在待测域计 1 秒周期数即得频率，再经 cc_simple 跨回主域，并能检测待测时钟停摆报 0。核心结论：外部信号处理须电路与时序约束配套，AMD 靠 scoped tcl（含关键的 IOB FALSE）自动覆盖，其他厂商需手写 set_max_delay。
