本讲专讲同步包 FIFO 实体 `olo_base_fifo_packet`，它在同步 FIFO 基础上把数据从「字」级提升到「包」级。核心结论是该实体工作在存储转发（store-and-forward）模式：整包写完才允许读出，从而实现「包压缩」；放不下的包会被自动丢弃，且不支持直通。结构上是「大 RAM（`olo_base_ram_sdp`）存负载 + 小 FIFO（`olo_base_fifo_sync`）存包尾地址」：每写完一包把尾地址推入小 FIFO，读侧据此算出包大小并界定边界。包边界完全由 `Last` 信号决定，不使用字节使能；读侧额外给出 `Out_Size` 与 `Out_Last`。引入的关键术语与概念包括：word/packet、Last、反压、DropLatch（写侧丢弃锁存位）。两条核心机制——写侧丢包（`In_Drop`/`In_IsDropped`，含超长包自动丢弃、丢弃时写指针回卷且 `In_Ready` 强制拉高以抽干剩余字）与读侧跳过/重复（`Out_Next` 提前收尾跳到下一包、`Out_Repeat` 拨回包首重读且不消耗 FIFO 空间）。功能子集 `FeatureSet_g`（FULL/DROP_SKIP_ONLY/DROP_ONLY，越低越省资源，DROP_ONLY 用单寄存器替代小 FIFO 并把 Last 并入 RAM）与优化档 `Optimization_g`（SPEED/THROUGHPUT，后者与 FULL 互斥）决定可用功能与吞吐取舍。
