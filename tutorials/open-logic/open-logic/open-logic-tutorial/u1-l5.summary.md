本讲以 `olo_base_pl_stage.vhd`（带反压的流水线寄存器）为样本，对照 `doc/Conventions.md` 讲解 Open Logic 全库统一的编码规范，并示范如何按结构阅读一个实体。覆盖四个模块：命名规范——用后缀标注标识符身份（泛型 `_g`、常量 `_c`、类型 `_t`、变量 `_v`、状态 `_s`），实体统一为 `olo_<area>_<function>`，端口不加 `_i/_o` 方向后缀；握手与复位约定——数据传递一律用 AXI-S 的 Valid/Ready 握手（数据信号按功能命名，如 `In_Data`），复位为同步、高有效，写成进程末尾的覆盖而非开头分支，且只复位状态寄存器，以降低复位扇出；默认值约定——所有可选泛型/端口都带默认值，不需要的功能可直接省略；实体文件结构——按版权/描述/库/实体/架构分段。本讲引入的关键术语包括 AXI-S 握手、反压（Back-pressure）、复位扇出、两进程法（two-process method，用 record 收纳寄存器，分组合进程 `p_comb` 与时序进程 `p_seq`）、shadow（影子）寄存器（吸收反压卡住瞬间到达的数据，做到不丢数据）、私有实体（`olo_private_` 前缀）以及综合属性常量（来自 `olo_base_pkg_attribute` 包）。核心结论：规范由 linter（VSG）强制执行，PR 中 lint 的 error 与 warning 必须清零。
