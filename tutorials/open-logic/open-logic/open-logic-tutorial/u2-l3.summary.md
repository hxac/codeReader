本讲讲解 Open Logic 三种 RAM 拓扑的最小模块：单端口 `olo_base_ram_sp`、简单双端口 `olo_base_ram_sdp`、真双端口 `olo_base_ram_tdp`。三者端口逐级增加——SP 读写共用一组地址，SDP 拆为专职的写口与读口（可选双时钟 `IsAsync_g`），TDP 则给出对称、皆可读写的 A/B 双口；端口越多资源越贵，需按需选最便宜的拓扑。

每个实体采用「公共实体 + 私有实体（`_nobe`，不带字节使能）」拆分：公共实体负责字节使能拆分（把宽 RAM 切成多个 8-bit 窄 RAM）、`RdValid` 流水线与断言；私有实体持有真正的存储阵列。共享泛型包括 `Depth_g/Width_g/RdLatency_g/RamStyle_g/RamBehavior_g/UseByteEnable_g/InitString_g/InitFormat_g`。

核心难点是**读时写歧义（read-during-write）**：同一地址同周期读写时返回旧值还是新值，对应两种行为——**RBW（读前写，返回旧值）**与**WBR（写前读，返回新值）**。实现上靠 `shared variable` 存储阵列，让同一时钟进程内「读、写语句的先后顺序」直接体现这两种行为，并由 `RamBehavior_g` 开关切换。不同器件块 RAM 原生行为不统一，RTL 行为须与之吻合才能被推断成块 RAM，否则退化为触发器/分布式 RAM，故不确定时两种都试、看综合报告。

其他关键结论：读延迟由 `RdPipe`（数据）与 `RdValidPipe`（有效）两条流水线实现，并用 `shreg_extract=suppress` 阻止其被抽成移位寄存器以稳定推断；复位**只清 `RdValid`、不清存储内容**，初值只能靠 `InitString_g` 指定。
