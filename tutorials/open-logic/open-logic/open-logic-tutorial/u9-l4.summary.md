本讲为 fix 区 DSP 滤波器专题收尾。首先确立 FIR 命名约定：实体名编码三个正交维度——速率变换（dec/int）× 抽头计算（ser/par/semi）× 通道组织（chpar/chtdm），仓库现实现 dec_ser_chpar 与 dec_ser_chtdm。串行抽头（ser）指一个乘法器每拍算一个抽头、N 拍产出一输出；实体不产生反压，输入速率受带宽上限约束、需 olo_base_rate_limit 限速。chpar 每通道配独立乘加、数据 RAM 宽字浅、N 拍出一组；chtdm 共享一个乘加、RAM 窄字深且通道在地址高位、N×通道拍出一组；系数存储与通道数无关。两者共享格式推导：乘法取全精度、累加器留 GuardBits 保护位、末级 resize 收敛，并以 ReplaceZero 保证与 Python 模型位真。olo_fix_coef_storage 支持实数初始化自动量化、ROM/RAM 双模式、Coef/Cfg 双读口，靠 shared variable 读写语句顺序实现 RBW/WBR。olo_fix_mov_avg 用「加新减旧」把 O(N) 累加降到 O(1)，增益校准 NONE/SHIFT/EXACT 三档，N 为 2 的幂时 EXACT 退化为纯移位、免乘法器。
