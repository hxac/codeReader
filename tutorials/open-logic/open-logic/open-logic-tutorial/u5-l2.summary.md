本讲讲三种仲裁器，解决多请求者争抢单一共享资源、每拍至多授权一个获胜者（one-hot）的问题，约定最高位优先级最高。固定优先级 arb_prio 用前缀或 ppcOr 加边沿检测选出最高位请求者，时序最优但有饥饿风险，无握手、Latency_g 控制寄存级数。轮询 arb_rr 用两个 arb_prio 配掩码，仅当下游 Out_Ready 接受才推进掩码，无饥饿、最坏延迟 O(N)。加权轮询 arb_wrr 在 arb_rr 上加权重计数器，请求者连续拿 wi 拍才换人，带宽份额为 wi/Σwj，权重 0 者禁用。三者层层复用，选型口诀为主次分明用 prio、均分公平用 rr、按比例用 wrr。
