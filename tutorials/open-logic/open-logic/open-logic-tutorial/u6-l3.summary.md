本讲介绍 `olo_axi_master_simple`，它是 open-logic AXI「主机」线的第一讲，承接 u6-l1（AXI4 五通道、`olo_axi_pl_stage`）与 u2-l4（同步 FIFO）。其核心价值是用一个极简的**命令接口**（起始地址 + 拍数 Size + 延迟模式 LowLat）封装 AXI4 五通道，命令与数据完全解耦，内部用多个 `olo_base_fifo_sync` 缓冲，完成后以单周期脉冲 `Wr_Done`/`Rd_Done`（或 `Wr_Error`/`Rd_Error`）回报。

「simple」有两层含义：**不做非对齐访问、不做位宽转换**。用户数据宽度恒等于 AXI 数据宽度，起始地址必须字对齐，由常量 `UnusedAddrBits_c` 与掩码函数 `addrMasked()` 强制清零低位保护。任意长度的用户命令由 `WriteTfGen`/`ReadTfGen` 状态机**自动切分**成不超长（≤`AxiMaxBeats_g`）、不跨 4KB 边界的多个 INCR 突发。

关键术语：**高/低延迟模式**——高延迟等数据齐/读 FIFO 有空间再发命令、不卡总线（默认）；低延迟立即发命令、可能卡总线；读操作官方建议永远用高延迟。完成判定靠 `wr_resp`/`rd_resp` 两个小 FIFO 跟踪「是否最后一段」，在途命令数受 `AxiMaxOpenTransactions_g` 限制。`olo_axi_pkg_protocol` 是包体为空的「协议字典」，集中定义响应码 `AxiResp_*`、突发类型 `AxiBurst_*`、传输位宽 `AxiSize_*`，是主机与测试 VC 的单一真相源。下一讲 u6-l4 的 `olo_axi_master_full` 将放开非对齐与位宽转换。
