本讲拆解 lmdeploy/pytorch/nn/ 下三个积木：Attention、RMSNorm/LayerNorm、RotaryEmbedding/ApplyRotaryEmb。核心结论：所有 nn 积木统一采用「薄包装+委托」桥接模式——构造时 get_backend()→get_layer_impl_builder(OpType.X)→self.impl，前向时 self.impl.forward(...)，把接口与设备实现解耦；OpType 枚举是接口树与实现树唯一的共同词汇，换设备时 nn/ 与 models/ 无需改动。Attention 用 num_heads/num_kv_heads/v_head_size 三参数统一表达 MHA/GQA/MLA，经 _update_num_heads 按 TP 切分头数，前向对接 Paged Attention 的 k_cache/v_cache 与 AttentionMetadata，并为 FP8 KV cache 预留 k_scale/v_scale。RMSNorm 较 LayerNorm 省去减均值与 bias，支持残差融合（Add+Norm 合一）与 W8A8（smooth_quant）分支，以 weight_loader+chunk_aligned 做 TP 权重切分。RoPE 拆为「建表」（RotaryEmbedding，整模型一次）与「施加」（ApplyRotaryEmb，每层一次），build_rotary_params 用策略表分发 default/linear/dynamic/yarn/llama3 等变体，可叠加 FoPE/MRoPE 多模态扩展。这些积木是 models/*.py 重写类被拼装的零件，其 impl 真身住在 backends/，线性层积木见 u5-l2。
