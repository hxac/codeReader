本讲建立 LMDeploy 的全局目录与架构地图。仓库根分四类：Python 源码仅在 lmdeploy/，TurboMind 的 C++ 源码在 src/，其余为按设备拆分的构建脚本、文档与测试。lmdeploy/ 包由基础设施文件（messages.py、model.py、archs.py、pipeline.py）与五个核心子包组成：pytorch/（PyTorch 引擎）、turbomind/（TurboMind 引擎）、lite/（量化压缩）、serve/（服务化）、vl/（多模态）；__init__.py 的 __all__ 即公开 API 表。核心架构主线是「两条后端，一个 Pipeline」：用户只调 pipeline()，Pipeline.__init__ 经 archs.py 的 autoget_backend_config 选 pytorch 或 turbomind，经 get_task 选 AsyncEngine（纯文本）或 VLAsyncEngine（多模态），最终落入 PyTorch 或 TurboMind 两引擎之一；api.py 的 serve/client 已废弃，应改用 CLI。
