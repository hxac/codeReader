本讲讲解 LMDeploy PyTorch 后端如何让单实例同时挂载并服务多个 LoRA 适配器，按「加载→计算→选择」三模块组织。加载：`patch.add_adapters` 为每个目标线性层在 `LinearBase.lora_adapters` 上建打包 `LoRA` 子模块，所有适配器的低秩矩阵 A、B 沿 rank 维拼接成 `(sum_rank,*)` 单组缓冲，由 `AdapterInfo.rank_offsets` 标注区间，按 `adapter_id` 灌权重；核心约定是空适配器恒占 id=0（纯 base 推理），`add_adapters` 与 `AdapterManager` 须各自「排序+头插 None」保持一致。计算：`LinearBase` 先算 base 输出，再委托 `TritonLoRAImpl` 的 `fused_lora` Triton kernel，用两段 matmul 算出 s·B(Ax) 增量并原子累加回 base，权重按 Megatron 式 TP 切分（列并行切输出、行并行切输入）。运行时：`AdapterManager` 提供 name→id 映射，`InputsMaker._set_adapter_ids` 每步把每条序列的 `adapter_name` 翻成 `local_adapter_ids` 张量，kernel 据此为同 batch 不同请求取各自 scaling/rank 段，实现单 kernel 多适配器；未挂适配器时两道短路保证零开销。
