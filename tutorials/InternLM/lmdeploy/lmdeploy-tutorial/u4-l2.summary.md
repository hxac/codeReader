本讲拆解 `EngineLoop`——由 `Engine.async_loop()` 创建的「多协程总管」。它在同一 asyncio 事件循环里并发运行四条核心协程：`preprocess_loop`（消费外部请求喂给调度器）、`main_loop`（调度+prefill/decode+forward 的心脏）、`send_response_loop`（把推理产出送回客户端）、`migration_loop`（PD 分离时迁移 KV cache），各自节奏不同却互不阻塞。核心结论有三：其一，`main_loop` 单步完成「取输入→激活→预取下一批+阻塞等当前批输出→回收」，预取写在等待之前，实现 CPU 准备输入与 GPU 计算 forward 的重叠；其二，引入两个自定义同步原语——`CounterEvent`（带计数器，N 次 clear 配 N 次 set 才点亮，追踪在途 forward）与 `RunableEventAsync`（set 时主动查 `has_unfinished()` 决定亮灭，避免假唤醒）；其三，`scheduler.tick()` 不在 `engine_loop.py`，而在 `inputs_maker.py` 的 `_send_next_inputs_impl`，每次 forward 派发（含预取）让 `scheduler_tick` 自增，即「派发即计步」。关键术语还包括 `forward_async`（非阻塞投递）与 `get_output_async`（阻塞等输出）的区分，以及响应回送链路 `resp_queue → _send_resps 去重 → resp.event.set()` 点亮事件唤醒客户端等待。
