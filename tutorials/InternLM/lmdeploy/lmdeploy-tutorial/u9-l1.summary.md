本讲讲解 lmdeploy 中视觉语言模型（VLM）的完整处理链路。核心结论是：VLM 由与后端无关的视觉编码基础设施（`lmdeploy/vl/`）与语言模型引擎（PyTorch / TurboMind）两部分组成，视觉编码器相对独立、被单独抽出，由 `ImageEncoder` 统一调度。引入的关键术语包括：`ImageEncoder`（视觉预处理与前向的总调度器，通过 `max_workers=1` 线程池串行执行以免抢占 GPU）、`VisionModel` 抽象基类及其子类（通过 `VISION_MODELS` 注册表与 `_arch` 匹配被选中，存在新式与旧式两套 preprocess）、占位 token 思想、以及引擎侧统一数据载体 `MultiModalData`。两条注入路径是本讲核心：PyTorch 后端传 `pixel_values`，视觉前向推迟到 patched 模型 `forward` 内、用 `masked_scatter` 把占位 token 的 embedding 替换为图像 embedding；TurboMind 旧式则在 Python 侧预算 `input_embeddings` 与 `input_embedding_ranges`，引擎按 `[begin, end)` 区间替换。此外引入多模态模态枚举（IMAGE/VIDEO/AUDIO/TIME_SERIES）、MRoPE 三维旋转位置编码，并指出多模态前缀缓存依赖 `content_hash`（对像素字节做 SHA-256），仅「PyTorch 后端 + 新式 preprocess」路径可稳定支持。
