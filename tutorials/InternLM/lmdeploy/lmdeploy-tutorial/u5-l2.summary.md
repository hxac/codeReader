本讲承接 u5-l1 的「薄包装 + 委托」桥接模式，讲清线性层与量化的结合。核心模块有四：`LinearBase` 是所有线性层的公共基座，统一处理张量并行（列切 colwise / 行切 rowwise + all_reduce）、LoRA 挂载、DP-TP 分轮 GEMM 三种前向派发，但把真正的矩阵乘 `_forward_default` 留给子类，不碰量化的数学。运行时按量化策略选层是**两段式派发**：`build_linear` 先按 `quant_method`（None/awq/smooth_quant/fp8）选 Python 类（决定权重打包与加载），每个类再用 `OpType`（Linear/LinearW4A16/LinearW8A8/LinearBlockedF8）在 `backends` 选设备 kernel；`quant_method` 由 `QuantizationConfig.get_quant_method(prefix, module_kind)` 决定，受 `ignored_layers` 与 `fp8_quant_scope='moe_only'` 影响，故同模型不同层可走不同类。三种量化变体的权重布局是重点对比：`AwqLinear`（W4A16）打包 int32、布局 `(in, out//elem)`、有 scales+qzeros、激活仍 FP16 只需反量化权重；`W8A8Linear`（SmoothQuant）int8、`(out,in)`、逐输出通道 scale、走 INT8 GEMM；`BlockedF8Linear` 分块 FP8、`(out,in)`、逐 128×128 块的 `weight_scale_inv`，且独有**在线量化**（dtype 不匹配时加载期压成 fp8）与 `dp_gather` 能力。LoRA 则是挂到 `LinearBase.lora_adapters` 的独立低秩模块，按「base + Σadapter」路径与任意量化叠加。本讲只讲「怎么用」，权重来源（auto_awq/smooth_quant/quant_blocked_fp8）与 kernel 实现留给 u7 与 u5-l4/u5-l5。
