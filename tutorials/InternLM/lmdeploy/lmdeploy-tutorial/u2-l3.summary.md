本讲区分了 LMDeploy 的两层配置：`GenerationConfig` 管每次推理的采样（在调用时确定），而 `TurbomindEngineConfig` 与 `PytorchEngineConfig` 管「引擎本身长什么样」（创建 pipeline 时配置一次）。二者均定义在 `lmdeploy/messages.py`，用 `@dataclass` + `__post_init__` 在构造阶段做字段校验，提前暴露参数错误；TurboMind 额外采用 pydantic dataclass 以增强序列化与类型校验。核心字段包括并行度 `tp`/`dp`/`ep`（其中专家并行 `ep` 仅 PyTorch 后端有）、会话长度 `session_len`、批大小 `max_batch_size`，以及 KV 缓存旋钮 `cache_max_entry_count`（默认 0.8，表示「权重加载后剩余显存中划给 KV cache 的比例」）。关键差异：TurboMind 的 `cache_max_entry_count` 可传整数表示 KV block 总数，PyTorch 严格要求 (0,1) 小数；`QuantPolicy` 描述的是 KV cache 量化，TurboMind 拒绝 FP8，PyTorch 支持（限 cuda/ascend）。PyTorch 配置更前沿，多出 `eager_mode`/CUDA Graph、LoRA `adapters`、PD 分离 `role` 等字段。配置经 `pipeline()` 的 `backend_config` 传入。
