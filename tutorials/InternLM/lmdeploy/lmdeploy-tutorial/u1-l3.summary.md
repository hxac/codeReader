本讲讲解 lmdeploy 的两种安装路线:普通用户用 `pip install lmdeploy`(下载预编译 wheel),贡献者或特殊设备用户用 `setup.py`+CMake 从源码构建。内容分三个最小模块。其一,**按设备拆分的 requirements**:lmdeploy 跨 cuda/ascend/rocm/maca/camb 多设备,运行时依赖各不同,故按设备拆成多套;根目录 `requirements_<device>.txt` 是 `[all]` 全套聚合,`requirements/runtime_<device>.txt` 是默认 `install_requires`,共用 `common.txt`,并用 `-r` 递归引用与环境标记控制平台。其二,**setup.py 构建入口**:核心是 `LMDEPLOY_TARGET_DEVICE`(选设备)与 `DISABLE_TURBOMIND`(是否编译 C++)两个开关,编译 TurboMind 的条件为设备为 cuda 且未禁用;`parse_requirements` 递归解析 `-r`。其三,**CMakeLists.txt 构建 TurboMind C++ 扩展**:用 FetchContent 拉 cutlass/fmt/yaml-cpp/xgrammar,按 CUDA 版本选定 GPU 算力架构(70→100a),编译 `src/` 源码,产出 `_turbomind` 装入 `lmdeploy/lib/`。核心结论:PyTorch 后端仅靠 Python 依赖、安装轻;TurboMind 需编译 C++/CUDA、对工具链有要求,二者通过 `cmake_configure_options` 的 `-D` 变量耦合。后续讲义的承接点:编译出的 `_turbomind` 由 `lmdeploy/lib` 桥接(U6 讲),`entry_points` 注册的 `lmdeploy` 命令行将在 u1-l5 讲解。
