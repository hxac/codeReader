本讲是 LMDeploy 手册首篇，仅读 README、README_zh-CN、CLAUDE.md 三份门面文档建立全局认知。核心结论：lmdeploy 是 LLM「压缩—部署—服务」一体化工具包，三职责分别对应 lite/、turbomind/ 与 pytorch/、serve/ 子包。四大特性为高效推理（吞吐约为 vLLM 1.8 倍）、可靠量化（4bit 约为 FP16 2.4 倍）、便捷服务、卓越兼容（KV Cache 量化、AWQ、自动前缀缓存可同时启用）。引入的关键术语：持续批处理（continuous batching/persistent batch）、分块 KV 缓存（blocked KV cache，即 Paged Attention 思想）、张量并行、VLM、PD 分离。架构主线是「两条后端一个 Pipeline」：用户经 pipeline(...) 统一入口，内部在 TurboMind（C++，追求极致性能，2023/07 首发）与 PyTorch（纯 Python，降低开发门槛，2024/01 首发）两套并存互补的引擎间选择。README 的 Latest News 既是特性演进时间线，也是后续各单元的学习导航。遇到表述分歧以英文 README 为准。
