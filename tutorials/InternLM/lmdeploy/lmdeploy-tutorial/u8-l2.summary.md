本讲拆解 `lmdeploy serve api_server` 的启动链路，覆盖三个最小模块。首先明确分叉点：CLI 中 `dp==1` 或 TurboMind 后端走 `serve()`（单进程），否则走 `launch_server()`（多进程 DP 编排）。`launch_server()` 只编排不干活——按 `dp/tp/ep` 推导 GPU 切片（`num_devices=max(dp,tp,ep)`）、为每个子进程分配连续端口（自 `server_port` 起，不超 65535），用 `spawn` 起子进程，每个子进程内部仍调 `serve()`，父进程轮询保证「一个挂全挂」。`serve()` 是 FastAPI 装配工厂：造引擎挂到进程级单例 `VariableInterface.async_engine`、`include_router` 三套路由、加四类中间件（CORS/鉴权/睡眠拦截/并发上限）、`uvicorn.run` 启动；推理循环在 `startup_event` 的 `start_loop` 中真正运转。最后用纯 `requests` 实现的 `APIClient` 演示 OpenAI 兼容端点调用与 SSE 流式解析（剥 `data:` 前缀、跳 `[DONE]`）。
