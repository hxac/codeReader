本讲承接 u4-l4，专攻 PyTorch 后端 KV cache 的分块管理，引入 Paged Attention 的三类地址：逻辑块、物理块、block table（逻辑→物理映射）。`LogicalTokenBlocks` 以预分配计数数组记录序列占用的逻辑块号；`PhysicalAllocator` 用数组加游标做栈式分配回收；`LogicalAllocator` 将 GPU/CPU 双物理池缝进统一逻辑空间，靠 `ref_count` 实现共享块归 0 才回收，支持 swap 换页。`DefaultBlockManager` 实现 `num_required_blocks`（算差额）、`allocate_msg`（增量补块）、`free`、`try_swap_out/in` 四类方法；驱逐非其职责，由调度器配合 `RecomputeEvictionHelper` 触发。每块 KV 字节数由层数、block_size、KV 头数估算，引擎据此反推 `num_gpu_blocks`。`ref_count` 共享为后续前缀缓存 `BlockTrie` 奠定基础。
