本讲讲清 PyTorch 引擎的并行度如何从用户配置落到 GPU 真实通信。核心有三：其一，DistConfig 作为「拓扑求解器」，在 `__post_init__` 中由 tp/dp/ep 与分层 attn_tp/mlp_tp/moe_tp 解出 `world_size`（取 max(mlp_tp, moe_tp) 或 ep）与各级 TP，并用 assert 拦截非法组合，TPMode 区分 DEFAULT 与 DP_TP。其二，distributed.py 用 `dist.new_group` 把拓扑切成 attn/mlp/moe/ep 四套通信子组，每组同时建 NCCL 与 gloo，由 DistManager 单例挂入全局 DistContext 供任意模块无参查询。其三，初始化链为 build_executor → worker.init_process_group → `dist.init_process_group(nccl)` → DistContext.build，`world_size==1` 走 uni 且不建组，`dp>1` 与 empty_init 强制走 ray。核心结论：TP 通信全部发生在 LinearBase，遵循 Megatron「列并行不通信、行并行 all_reduce」规则，每层 o_proj、down_proj 各做一次 all_reduce，QKV、gate_up 列并行不通信。
