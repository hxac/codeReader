本讲剖析 lmdeploy 自带反向代理 proxy（`serve/proxy/`）：它站在多个 `api_server` 实例前，自身不做推理，仅做节点管理与 HTTP 转发。核心是模块级单例 `NodeManager`，维护节点清单（含 `unfinished`/`latency`/`speed`/`role`/`models`）、后台心跳探活剔除失联节点，并用三种路由策略 `random`、`min_expected_latency`、`min_observed_latency` 选目标节点，请求前后经 `pre_call`/`post_call` 更新负载。Hybrid 模式用 aiohttp 裸字节透传、最高保真，流式收尾靠 `BackgroundTasks` 触发 `post_call`。`ProxyStreamingResponse` 重写 `stream_response`，在发响应头前先取首块，首块异常即透传后端真实状态码（`APIServerException`），破解 SSE「头不可改」难题。DistServe（PD 分离）模式将 prefill 与 decode 分到不同引擎，proxy 经 `PDConnectionPool` 建立 RDMA/NVLink 通道，用 `MigrationRequest` 携带 KV 定位信息实现跨机迁移；关键枚举 `ServingStrategy`/`EngineRole`，且仅 PyTorch 后端支持。
