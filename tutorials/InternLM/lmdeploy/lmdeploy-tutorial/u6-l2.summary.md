本讲精读 `turbomind.py`，拆解 TurboMind 后端的 Python「翻译层」。三模块：`TurboMind` 是 C++ 引擎门面，构造链 `from_pretrained→__init__→_from_hf`——解析 HF 配置，把用户面 `TurbomindEngineConfig` 逐字段搬进 `_tm.EngineConfig`，调 `_tm.TurboMind.create` 建 `model_comm`（C++ 调用总入口），由 `ModelLoader` 灌权重、`_create_engine` 定稿，再用 `create_instance` 发轻量句柄；`TurboMindInstance` 是单次推理句柄，核心 `async_stream_infer` 完成「准备输入→翻译生成配置→进 C++ forward→流式增量切片 yield」四步，靠 `StreamingSemaphore` 把 C++ 信号线程产出喂回 asyncio 协程；DLPack 零拷贝两函数为张量翻译官：`_np_dict_to_tm_dict`（`_tm.from_dlpack`）进、`_tm_dict_to_torch_dict`（`torch.from_dlpack`，含 `uint32→int32` 视图）出。关键术语：`input_meta` 当前仅承载 MRoPE 多模态三维位置编码（`mrope_position_ids`/`mrope_position_delta`，由 VL 前端塞入，纯文本为 None）；`errcode_map` 把 C++ 整数 status 译为用户面 `ResponseType`。核心结论：它是用户面配置/张量与 `_tm` 对象、C++ 流式 token 与 `EngineOutput` 之间的双向翻译层，全程靠 DLPack 零拷贝与信号量跨线程同步。
