本讲介绍 `AsyncEngine`：服务层与引擎层（PyTorch `Engine` / TurboMind `TurboMind`）之间的**后端无关异步封装层**。它替上层接管会话管理、chat 模板、增量去词表化、统计上报与睡眠/唤醒，对下通过 `EngineBase`/`EngineInstanceBase` 鸭子类型契约屏蔽两套后端差异，构造期按 backend 用工厂方法延迟 import 构造 `self.engine`。

核心是请求转发链路 `generate() → safe_run() → handle.async_stream_infer()`。`generate()` 是 async generator，用 `yield GenOut` 一段段产出；真正跨进引擎边界的是 `safe_run` 内 `handle.async_stream_infer(session_id, **kwargs)`，`handle` 为从 `RequestHandlePool` 借来的轻量句柄；`safe_run` 用 `asyncio.shield` 保护清理协程，并用 `SafeRunException` 区分已处理取消。

`VLAsyncEngine` 以最小增量继承 `AsyncEngine`：加 `vl_encoder`、换多模态 `prompt_processor`、加约束，**不重写 `generate()`**，多模态能力靠依赖注入获得。健康检查由 `EngineHealthMonitor`（缓存+后台轮询）、`AsyncEngine.health_probe`（有界、防重入、校验调度进展）、`Engine.get_health_status` 三层协作；`/health` 读缓存、不健康时再即时确认。
