本讲讲解 LMDeploy 的 PD 分离部署（DistServe，仅 PyTorch 后端支持）：将计算密集的 prefill 与访存密集的 decode 拆到两组引擎，prefill 后把已算好的 KV cache 迁移到 decode，避免重复计算与互相干扰。代码集中于 disagg 目录，分三层：config 层用枚举固化词汇表——ServingStrategy（Hybrid/DistServe）、EngineRole、MigrationBackend（DLSlime/Mooncake）、MigrationProtocol（仅 GDR RDMA 真正实现），DistServeEngineConfig 握手时交换并行度与缓存布局以算偏移、推 worker 连接对；backend 层是数据面，实现 MigrationBackendImpl 接口，按逐层逐块的 AssignmentInstruct 搬 KV 字节，由 decode 端发起 RDMA read 主动拉取；conn 层是控制面，PDConnectionPool 编排惰性三步握手（engine_info→p2p_initialize→p2p_connect）与容错回收，engine_conn.py 提供 zmq 信道并处理缓存释放通知。控制面（HTTP+zmq）与数据面（RDMA/NVLink）完全分离，端到端闭环为 proxy 分流→prefill 产 KV 定位→decode 的 migration_loop 搬运→吐首 token。
