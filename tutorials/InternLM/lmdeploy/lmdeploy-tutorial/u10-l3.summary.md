本讲讲解 lmdeploy 的测试体系与运行方式。测试统一放在 `tests/test_lmdeploy/` 下，采用 pytest 框架，约 42 个 `test_*.py` 文件，按功能分为顶层、`test_lite/`、`test_vl/`、`serve/` 四组，与功能子包一一对应。关键事实是 `pyproject.toml` 无自定义 pytest 配置，完全走默认发现规则。运行上，可用路径、节点 ID、`-k` 过滤串、`--collect-only` 实现从粗到细的定向运行，其中参数化用例经笛卡尔积展开为节点（形如 `文件::类::方法[参数]`）。三大代表性文件各有侧重：`test_messages.py` 为纯函数级单元测试，验证 `GenerationConfig`（构造期容错夹断）与 `ChatCompletionRequest`（Pydantic 严格报错）对同字段的不同校验策略；`test_model.py` 采用对照测试，以 transformers 官方 `apply_chat_template` 为标准答案，参数化跨约 30 个模型逐字符比较 chat 模板产出；`test_pipeline.py` 为端到端集成测试，借 `TestBackendInference` 加双后端参数化让同一套用例跑遍 PyTorch 与 TurboMind，靠 `scope='class'` 的 fixture 复用昂贵的 pipeline 实例。贯穿始终的分界线是「离线轻量逻辑」与「需 GPU 加联网下载模型」两类用例的区别，运行前须据此甄别。关键术语：pytest 收集规则、节点 ID、参数化、fixture、对照测试、端到端集成测试。
