本讲承接 u5-l2，讲 LMDeploy PyTorch 后端对 MoE（混合专家）模型的支持，由三个最小模块组成。其一是融合 MoE：把 N 个专家权重堆成三维张量，用 `FusedMoEBase` 的「dispatch（分发，多卡为 all-to-all）→ gemm（各专家 GEMM）→ combine（加权聚合）」三段流水线一次算完，由 `MoeType` 枚举区分 `Default`（同步）与 `DSAsync*`（异步、用 handle/event 重叠通信与计算）。其二是门控路由（gate/router）：小线性层产 `router_logits`，`NoauxTCRouter` 实现 DeepSeek-V3 的 `noaux_tc`——偏置 `e_score_correction_bias` 仅参与选 `top_k` 专家，权重取自不带偏置的 sigmoid 得分，达成无辅助损失负载均衡。其三是 MoE 量化变体：`build_fused_moe` 按 `quant_method` 派发 `FusedMoE / FusedMoEW8A8 / FusedMoEBlockedF8`，共享同一骨架、仅权重容器与 impl 不同（不含 AWQ）。最后是 EPLB（专家级负载均衡）：离线用「复制热点专家＋贪心装箱」建「逻辑专家→物理专家」映射，在线在 gate 后把逻辑 ID 随机映射到某物理副本，使各卡 EP all-to-all 收发量均衡。桥接模式（`OpType`→`backends` impl）贯穿全讲。
