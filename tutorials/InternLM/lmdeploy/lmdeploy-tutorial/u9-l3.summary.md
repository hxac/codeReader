本讲讲解 lmdeploy PyTorch 引擎在 Paged Attention 分块 KV 之上的前缀缓存机制（默认关闭，需 `enable_prefix_caching=True` 开启），通过复用 system prompt 等公共前缀省 prefill 算力、降首 token 延迟。核心数据结构 `BlockTrie` 用前缀树按「token 块 + 多模态 content_hash」为键组织已缓存块，`Node` 为一条边，持有物理块号、tokens、子节点字典与父指针。三大动作：`match` 逐块下走 trie 复用命中块并 +1 引用、推进 step；`allocate` 把新满块挂回 trie，重复前缀则复用并释放冗余块；`evict` 按 LRU 踢 `ref_count==1` 的叶子。安全性源自引用计数：`free` 仅在 ref 归零时真正回收物理块，故多序列共享安全。`PrefixCacheStats` 用命中/查询 token 数算命中率，因 `match` 为试探性，统计配 snapshot/restore 支持调度器回滚，recompute 路径用 `suppress_match_stats` 防虚高。调度器固定按 match → 驱逐检查 → block_manager.allocate → block_trie.allocate → 发布 的顺序协作。SSM（Mamba 类）另走 `_match_state_checkpoint` 路径，需 KV 与循环状态快照同时命中。
