本讲讲解投机解码（speculative decoding），其核心思想是「草稿模型先提议 N 个 token、目标模型一次 forward 并行验证、拒绝采样决定接受前缀」，从而把访存开销摊薄到多个 token，加速推理且数学上保证输出分布与纯目标采样一致。关键术语包括：目标模型、草稿模型（proposer，支持 eagle/eagle3/deepseek_mtp/qwen3_5_mtp）、草拟 token 数、奖励 token、拒绝采样、结构化解码。

本讲围绕 `lmdeploy/pytorch/spec_decode/` 三个最小模块展开：编排器 `SpecModelAgent` 负责草稿自回归生成（输入相对目标左移 1 位、草稿恒取 argmax，随机性全由目标侧拒绝采样承担），关闭时退回空桩 `BaseSpecModelAgent`；`RejectionSampler` 分贪心（首处不匹配即停）与随机（按 target/draft 概率比接受、拒绝则用 Gumbel-max 从残差分布重采样）两路；`GuidedSpecHelper` 解决结构化解码与投机解码的冲突——跨 N+1 位串行 bitmask、fork 保护原件 matcher、接受由拒绝采样驱动、Eagle3 草稿词表翻译。设计上采用空对象模式与 Triton kernel 加速。用户通过 `--speculative-algorithm` 等参数启用。
