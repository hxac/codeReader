本讲建立 TurboMind 后端「Python→C++」全局地图。TurboMind 是 C++/CUDA 高性能后端，与纯 Python 的 PyTorch 后端分工互补：前者追求吞吐与延迟，只认 `SUPPORTED_ARCHS` 白名单（由 `supported_models.py` 的 `is_supported` 把关，不在表内则提示改用 PyTorch）；后者靠加载 HF 模型并动态 patch 层，无需白名单。二者共用同一个 `pipeline()` 入口。

C++ 源码集中在 `src/turbomind/`，按 `core/engine/models/kernels/comm/python` 单向分层，顶层入口类 `TurboMind`（`turbomind.h`）以 pimpl 惯用法隐藏实现。两侧靠 pybind11 扩展 `_turbomind`（`bind.cpp` 编译为 `_turbomind.so`）桥接：把 C++ 类绑成 Python 类，张量经 dlpack 零拷贝互传，`EngineConfig`/`GenerationConfig` 逐字段手写搬运，`forward` 进入 C++ 时释放 GIL 以支持异步流式。

分块 KV 缓存（Paged Attention）在 C++ 侧由 `models/llama/BlockManager.h` 的 `BlockManager` 管理（注意路径，不在 `engine/` 下），块有 free/active/cached 三态，靠 `use_count` 与 `timestamp` 区分，`unique_id` 做分配安全校验，支撑前缀缓存复用与 LRU 驱逐。后续讲义将展开 Python 包装精读（u6-l2）、模型转换（u6-l3）与权重构建器（u6-l4）。
