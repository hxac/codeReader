本讲剖析 `lmdeploy` 命令行从敲键到执行的骨架。核心设计模式是「类体即注册 + `set_defaults(run=...)` 派发 + 延迟导入」：`setup.py` 把命令钉到 `lmdeploy.cli:run`，`entrypoint.run()` 三步走（装配→解析→派发），派发全靠统一调用 `args.run(args)`，新增子命令无需改入口、符合开闭原则。三类分挂 `check_env`/`chat`、`serve`（api_server/proxy）、`lite`（auto_awq/auto_gptq/calibrate/smooth_quant）；一级命令 `serve`/`lite` 在各自模块类体导入时挂入，二级命令由各 `SubCli.add_parsers()` 挂入。共享积木有 `ArgumentHelper`（统一参数词汇表）、`convert_args`（CLI 参数与底层 Python API 形参一一对应）、`FlexibleArgumentParser`（下划线/连字符互通、点号嵌套字典）、`DefaultsAndTypesHelpFormatter`（帮助自带默认值与类型）。`serve api_server` 以「共享 action」让参数在 PyTorch 与 TurboMind 双引擎分组同时显示，并经 `autoget_backend` 选 `PytorchEngineConfig`/`TurbomindEngineConfig`；`lite` 子命令统一遵循「延迟导入 + convert_args + 转发」模板。
