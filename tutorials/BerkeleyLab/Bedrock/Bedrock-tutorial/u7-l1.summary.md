本讲介绍 Bedrock 的 PicoRV32 RISC-V 软核 SoC（`soc/picorv32`），按 gateware（可综合硬件）/firmware（C 驱动库）/project（上板工程）/test（仿真）/common（启动脚本与 Python 工具）分层，CPU 核为上游开源 RV32IMC 软核。核心模块有四：一是软核，`pico_pack.v` 固定参数（开压缩/乘除法/中断）并经 `mpack`/`munpack` 把原生 valid/ready 总线打包成 69/33 位宽线，`mem_wstrb` 区分读写字节使能；二是总线仲裁，CPU 与网络侧 localbus 共享一条总线，`lb_bridge` 配 `lb_merge` 采「localbus 永远优先、CPU 用 `& ~busy` 让路重试」策略，根因是 PicoRV32 拉高 `mem_valid` 后可任意拍才回 `mem_ready`、而 localbus 无握手且固定 `READ_DELAY` 读无法暂停，CPU 用 `addr[31:24]==BASE_ADDR` 解码外设；三是固件构建，`rules.mk` 用模式规则串起交叉编译链 `.c/.S→.o→.elf→8.hex→32.hex→32.dat`，`hex8tohex32.py` 将字节流按小端重排成 32 位字，`make system_load` 经 `boot_load.py` 与片上 `bootloader.S` 走握手灌 RAM，`byte_offset 0xe0` 保留 bootloader 区以实现免重新综合换程序；四是 `spimemio`，把外部 SPI Flash 翻译成 CPU 内存接口、靠预取缓冲加速顺序取指。其中网络侧 localbus master 即 Packet Badger，`gateware/*_pack.v` 外设是后续外设驱动讲义的承接点。
