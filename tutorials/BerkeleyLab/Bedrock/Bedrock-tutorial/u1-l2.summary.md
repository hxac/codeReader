本讲解决在干净机器上跑起 Bedrock 的问题，聚焦两个最小模块：依赖清单 dependencies.txt 与测试编排脚本 selftest.sh。Bedrock 把工具分为必需（GNU Make、Icarus Verilog、python3-numpy、pip、flake8）与可选（Verilator、Yosys、scipy、matplotlib、cocotb 等）：numpy 被列为必需，因为多数校验脚本仅需其数组运算即可完成数值比对，而 scipy/matplotlib 仅在画图与高级信号处理中用到；这种分层降低了上手门槛。selftest.sh 是 GitLab CI test 阶段的本地翻版，本质是按字母顺序逐个子目录执行 make -C <dir>，外加环境版本巡检与 flake8 风格检查，建议用 sh -ex 运行使缺工具尽早暴露。核心结论：缺可选工具会让对应测试被 skip 而非整体失败；需要 migen、RISC-V 工具链或 SymbiYosys 的测试由 more 参数或注释门控；脚本故意不自动 git clean，clean 由各子目录 Makefile 负责，仅全程无错才打印 selftest OK。「make -C <dir> [target]」是后续各讲统一的手势。
