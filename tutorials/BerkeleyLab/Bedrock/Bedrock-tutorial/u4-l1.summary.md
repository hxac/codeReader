本讲讲时钟域跨越（CDC）基础。CDC 本质是处理异步时钟域间信号传递的亚稳态问题：单比特用两级同步器（MTBF 可达数千年以上），多位数据绝不能逐位打两拍（各比特跌落时刻不同会撕裂），须靠「源域稳定锁存 + 跨域 gate 资格认证」。

最小积木层层嵌套：`reg_tech_cdc` 是带 `ASYNC_REG` 与 `magic_cdc` 属性的同步器原子，也是 u6-l1 形式化检查工具 `cdc_snitch` 的锚点；`flag_xdomain` 用 toggle + 两级同步 + XOR 边沿检测把单比特事件脉冲搬到目的域，假设输入为稀疏脉冲（连发会被吞并告警）；`data_xdomain` 处理多位数据：源域在 `gate_in` 有效时锁存数据并保持，用 flag_xdomain 把 gate 跨过去，数据位仅取一级同步，亚稳态防护集中在 gate 上（前提是 clk_out 比 gate_in 速率快两倍以上）；`async_to_sync_reset_shift` 实现「异步复位、同步释放」的复位同步器，并把脉宽撑到 LENGTH 拍。

核心结论：localbus 写侧是单向跨域，用一行 `data_xdomain`（如 cryomodule.v 的 lb_to_1x，size=49=17 位地址+32 位数据）即可搬过时钟域，延迟可忽略；而读侧需请求-响应握手，localbus 刻意无握手，故读侧难——这正引出 u4-l2 存储网关（用固定延迟绕开握手）与 u4-l3 jit_rad（预警前把数据预快照入 dpram）存在的根本动机。
