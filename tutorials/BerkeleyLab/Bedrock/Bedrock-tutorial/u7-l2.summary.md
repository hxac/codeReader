本讲精读 `peripheral_drivers/` 子目录，核心是外设驱动的统一哲学：芯片驱动文件只是「壳」——用 `passthrough` 透传控制线、用 `` `ifndef SIMULATE `` 守卫包裹厂家原语（IOBUF/ISERDES/ODDR）、配 `*_sim.v` 模型闭环仿真，真正的引脚时序由可移植引擎产生。四大模块：`spi_master` 是参数化 SPI 主机（地址+数据移位、用 `sdio_as_sdo` 在数据相位切双向 SDIO 完成读回），`spi_mon` 在其上做可编程序列器并双缓冲回读；`ad9653`（源同步 LVDS 接收，ISERDES/IDELAY/MMCM、逐 lane CDC 同步）、`ad9781`（DDR LVDS 发送）、`ad5662`（无原语纯逻辑）展示驱动复杂度的三台阶；I2C 桥是分层典范——`i2c_bit`（位→电平，每 bit 14 相位）、`i2c_prog`（字节指令→协议时序）、`i2c_chunk`（内存接口与集成壳），外加 Python `assem.py`（人话意图→字节指令），`i2c_chunk` 用 4 KB DPRAM 四分（程序/逻辑分析仪/结果/后台缓冲）+ ping-pong + freeze 提供自洽的主机读出范式。核心结论：「驱动」不等于「时序发生器」，壳与引擎分离是贯穿全讲的设计关键。
