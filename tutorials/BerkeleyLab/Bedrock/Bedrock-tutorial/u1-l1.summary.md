本讲是 Bedrock 学习手册的开篇，回答三个问题：Bedrock 是什么、由哪些部分组成、从哪里看起。结论是：Bedrock 是 LBNL 多年积累的、平台无关的 Verilog 代码库，用于 DSP 与 RF（含 LLRF）控制，并附带到 FPGA（如 Xilinx）所需的全部适配层。全讲只涉及仓库根目录两个最小模块：README.md 的顶层说明，以及 dir_list.mk 的目录常量定义。Bedrock 是一个多子系统聚合体，主要子系统包括 dsp（数字信号处理算法）、cordic（坐标旋转计算）、rtsim（RF 系统实时仿真）、cmoc（RF 控制器）、badger（响应 Ethernet/IP/UDP 的硬件核）、localbus（片上总线）、fpga_family 与 board_support（厂家与板卡适配）、projects（具体上板工程）等；soc/picorv32（RISC-V 软核 SoC）、serial_io、peripheral_drivers、build-tools 等仅见于 dir_list.mk。关键术语有 HDL/Verilog、FPGA、DSP、RF/LLRF、LBNL、GNU Make。核心结论有三：工具链地基为 Make + iverilog + Python；设计哲学是"命令行批处理优先、厂家图形工具并存"，厂家生成物统一放入 _<VENDOR_NAME>（如 _xilinx/）目录；dir_list.mk 通过 $(MAKEFILE_LIST) 自定位根目录并集中定义子目录绝对路径变量，使工程稳健可移植。完整顶层地图需把 README 子系统清单与 dir_list.mk 拼在一起看。
