本讲承接项目概览，解决「拿到 AERIS-10 雷达仓库后去哪里找东西、往哪里放东西」。第一模块讲**编号目录体系**：顶层用 `1_` 到 `9_` 编号，数字小偏文档/规划，数字大偏代码/实现；代码层集中在 `9_Firmware/`，内部再按 MCU、FPGA、GUI 分 `9_1`/`9_2`/`9_3`，其中 `9_1` 又含驱动库、算法、入口三层，跨层系统级测试归到 `tests/cross_layer/`。第二模块讲**文件放置策略**：区分入库（tracked）与忽略（gitignored），人写的源、需审查或正式发布的报告入库，机器生成、可重新产出的**生成产物（artifact）**则忽略；四类落点为——发布报告进 `docs/`（tracked）、仿真输出进 `5_Simulations/generated/`、FPGA 产物进 `9_2_FPGA/reports/`、可复用 TCL 脚本进 `9_2_FPGA/scripts/`（tracked），铁律是「生成产物不要留在仓库根目录」。`.gitignore` 忽略 `*.vcd`/`*.vvp`、各类模块仿真 CSV、`__pycache__`、`reports/` 等，是该策略的执行层。第三模块讲**docs 站点**：用 GitHub Pages 托管的工程文档，首页有架构、实现日志、bring-up、报告、发布说明五个入口及工程状态指标卡片，承担「工程看板」角色；正式 PDF 与里程碑 `.bit` 在此入库。硬件目录（`4_`）用 CERN-OHL-P，软件目录（`9_`）用 MIT，呼应双许可证约定。
