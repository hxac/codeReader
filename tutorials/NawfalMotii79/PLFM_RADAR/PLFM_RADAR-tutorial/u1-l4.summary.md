本讲补上「把代码跑起来」的最后拼图：工具链与本地运行方式。最小模块四块：(1) uv 管理依赖——开发依赖（ruff/pytest/numpy/h5py）置于 pyproject.toml 的 dev 组，GUI 运行依赖单列 requirements_v7.txt 并分核心必需与可选降级；(2) Python 三连检：ruff 静态检查（规则集针对 LLM 生成代码的常见坏味道）、py_compile 全仓语法编译、pytest 单元测试；(3) GitHub Actions 设四个并行 CI job，分别盯 Python、MCU、FPGA、跨层契约；(4) MCU 用 make test，借 Makefile 的 -Ishims 让 mock HAL 覆盖真 HAL，从而在 PC 上编译运行嵌入式固件；FPGA 用 run_regression.sh 走 iverilog 仿真。引入术语：工具链、依赖管理器、linter、测试框架、CI、shim/mock、优雅降级。核心结论：每条 CI 命令皆可在本地复现，CI 非黑盒；读配置文件本身比读其上方的注释更可靠，因为注释会随代码演进而滞后。
