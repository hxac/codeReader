本讲承接 u10-l1 的 FPGA 自检，转向整块物理板的 bring-up 流程与构建产物。核心是三个最小模块：一是 bring-up 检查清单三段式（板前门禁→板日冒烟测试→首次上电可观测目标），由 `bring-up.html` 操作总则与 `board-day-worksheet.html` 可填写工作表驱动，遵循「风险随能量上升，RF 与 PA 永远最后」原则，证据缺失即停。二是开发板（TE0713+TE0701）两颗递进镜像：心跳镜像（`radar_system_top_te0713_dev`，WNS +17.863 ns）只验证 FPGA 配置/时钟/I/O；FT601 集成开发镜像（WNS +0.059 ns）用合成测试数据验证 USB 枚举与主机命令协议，须「先心跳后 FT601」。三是调试手段：`adc_clk_mmcm.v` 用 MMCME2 对 ADC 400 MHz 时钟做 1:1 抖动清洗（输出 BUFG 加 DONT_TOUCH 防止时序被毁），`ila_capture.tcl` 的四探针（adc@400M、ddc/mf/doppler@100M）对应接收链四阶段，`health` 场景快速体检。关键术语：bring-up、.bit 流文件、WNS/WHS/WPWS 时序三件套、.ltx 探针、MMCME2、ILA、FMC LPC、时序收敛、逐级可证伪链。`program_fpga.tcl` 以 DONE 引脚统一判定 FPGA 就绪，`diag_log.h` 提供纯观测层 USART3 日志。
