本讲覆盖 AERIS-10 雷达 Python 侧的质量门禁，即 CI `python-tests` job 的三道关卡：ruff（风格与常见缺陷）→ `py_compile`（全仓语法编译）→ pytest（行为验证），遵循从便宜到昂贵的分层拦截原则。ruff 规则集在 `pyproject.toml` 的 `select` 中针对 LLM 遗留代码异味定制：`T20` 抓散落 `print`、`ARG` 抓无用参数、`ERA` 抓注释代码、`A` 抓遮蔽内置名、`BLE` 抓盲 `except` 等；`per-file-ignores` 对 `test_*.py` 放宽 `ARG/T20/ERA`，对再导出门面 `v7/hardware.py` 放宽 `F401`。`py_compile` 用 `doraise=True` 全仓编译、跳过 `.git/__pycache__/.venv/docs`，只查语法、不解析 import，与 ruff 的豁免机制相互独立。`test_v7.py` 与 `test_GUI_V65_Tk.py` 用 `unittest.TestCase` 写、由 pytest 驱动，承担两类守卫：协议契约守卫（`Opcode` 枚举、命令/状态包 round-trip 与 RTL 对齐）与死代码守卫（断言 `chirp_duration`、`crcmod`、`multi_prf_unwrap` 等被删物未复活）。可选依赖缺失用 `@unittest.skipUnless` / `self.skipTest` 跳过而非失败，是 u8-l1 优雅降级在测试侧的镜像。`smoke_test.py` 是独立板级 bring-up 脚本，用 `logging` 而非 `print` 通过 `T20`，不属 CI 回归。关键术语：ruff、select、per-file-ignores、py_compile、协议契约守卫、负测试/死代码守卫、SoftwareFPGA、round-trip、skipUnless。
