本讲义解析 FPGA 顶层模块 `radar_system_top.v`（「核心」）与 `radar_system_top_50t.v`（量产物理包装层）的关系，确立顶层是「接线员而非计算员」的定位。其核心职责是声明物理端口、用 `BUFG` 缓冲时钟、用 `ASYNC_REG` 同步复位，并例化五大子模块串成数据流水线：发射机 `tx_inst`（120MHz DAC 域，产 chirp 与帧计数）、接收机 `rx_inst`（100MHz 域，内部 DDC→匹配滤波→MTI→Doppler）、CFAR `cfar_inst`（吃 DC notch 后的数据）、自测试 `self_test_inst`（借 ADC debug tap）、USB `usb_inst`（双向收发）。引入的关键术语：时钟域（100MHz 系统/120MHz DAC/USB 接口时钟）、跨时钟域 CDC、generate 块、tie-off 悬空处理、opcode 操作码、`host_*` 配置寄存器。三个最小模块为：顶层例化接线、`USB_MODE` 编译期 generate 二选一（FT601 32 位 USB3.0 vs FT2232H 8 位 USB2.0，未用引脚 tie-off）、命令译码（4 字节命令经 toggle-CDC 进 100MHz 域，由 `case(usb_cmd_opcode)` 写入 `host_*`）。核心结论：opcode 编号与 Python `Opcode` 枚举构成跨层硬契约；关键 opcode 如 `0x03→host_detect_threshold`、`0x21→host_cfar_guard`、`0x30→host_self_test_trigger`（自清零脉冲），`0x15` 设 chirp 数会被钳制到 `DOPPLER_FRAME_CHIRPS=32` 并置 `chirps_mismatch_error`。CDC 实现细节留待 u3-l2。
