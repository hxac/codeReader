本讲打通雷达数据 FPGA→上位机的「最后一公里」USB 接口，覆盖四个模块。其一，FT601（32 位 USB 3.0，200T 开发板）与 FT2232H（8 位 USB 2.0，50T 量产板）两套模块经顶层 `parameter USB_MODE` 配 `generate` 在编译期二选一，共用同一组内部信号（`usb_cmd_*`、`status_*`），未选物理引脚 tieoff，换板只改一个参数。其二，裸字节流靠成帧区分：11 字节数据包（`0xAA` 帧头 + range_q/range_i + doppler 实虚部 + detection + `0x55`）承载一帧 2048 个距离-多普勒单元；26 字节状态包（`0xBB` + 6×32 位状态字 + `0x55`）回读配置。关键术语：`detection` 字节 `bit0` 为 CFAR 检测、`bit7` 为帧起始标志（复用一字节实现帧同步，且注释与 RTL 不符应以代码为准）；`rxf_n/txe_n/oe_n` 流控、245 同步 FIFO 模式、`ft_clk`（60/100 MHz）。其三，Read FSM 以「读优先」仲裁共享总线，按 FIFO 时序逐字节（FT601 则一次 32 位）收齐 4 字节命令 `{opcode, addr, value_hi, value_lo}`，拆出 `cmd_valid` 交顶层 CDC 译码（opcode 语义留待 u6-l2）。其四，时钟活动看门狗用 100 MHz 域监视 `ft_clk` 心跳，约 0.655 ms 无跳变即置 `ft_clk_lost` 并 OR 进 USB 域复位实现断线自愈，恢复时用 2 级同步器做异步复位同步释放以避免亚稳态。
