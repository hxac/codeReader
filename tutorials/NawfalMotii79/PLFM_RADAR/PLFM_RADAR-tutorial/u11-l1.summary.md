本讲讲 AERIS-10 的 FPGA 回归测试如何用开源 iverilog 在无 Vivado 许可证下跑起来。入口脚本 `run_regression.sh` 划分为 Phase 0 lint 加四个测试阶段（单元/集成/信号处理/基础设施），由 `run_test` 函数统一完成「编译→运行→数 `[PASS]/[FAIL]` 标记」，退出码 0/1 供 CI 判定，未知输出按 FAIL 保守处理。核心一是「Vivado 级 lint」：用 iverilog `-Wall` 输出加正则/awk，把位选择越界、端口位宽不匹配、`case` 缺 `default` 等 Vivado 综合才报的问题在秒级拦截，lint 失败即中止回归。核心二是真实数据协同仿真 `tb_fullchain_realdata.v`：用真实 ADI CN0566 雷达数据跑「距离抽取＋Doppler」全链路，与冻结的 Python 位精确黄金值逐比特比对，`MAX_ERROR=0` 要求 2048 点全精确相等。讲义区分三类校验：自生成黄金值（`gen_mf_golden_ref.py`，合成信号自生自比，易循环论证）、真实数据 exact-match（`golden_reference.py` 独立实现＋真实数据锚点）、数学定义校验（`validate_mem_files.py`）。关键结论是真实数据因含 staggered PRI、距离-多普勒耦合等结构，能抓合成信号「太干净」掩盖的架构错误，如 32 点↔双 16 点 Doppler 改动。引入术语：回归测试、iverilog/vvp、Vivado 级 lint、黄金值、协同仿真（cosim）、exact-match、定点运算、自生成 vs 真实数据黄金值，承接 u4-l3/u4-l4 并为 u11-l3、u14-l1 铺路。
