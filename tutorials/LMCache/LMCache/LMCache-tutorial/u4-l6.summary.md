本讲拆开 LMCache 内存底座。`memory_management.py` 是词汇表：`MemoryFormat` 用形状约定描述 KV 张量物理布局，`token_dim()` 定位 token 所在维度；`MemoryObj` 携带 `MemoryObjMetadata`，由 `ref_count` 与 `pin_count` 两本账驱动自动回收；`MemoryAllocatorInterface` 规定 allocate/free 契约，分配失败返回 `None` 而非抛异常。`AddressManager` 用排序空闲链加 4096 对齐与相邻合并三招抑制外部碎片，只管虚拟地址不管字节。两种切分策略：`TensorMemoryAllocator`（变长、加锁、适合大小混杂）与 `PagedTensorMemoryAllocator`（定长页、deque 出入队、无锁无碎片，适合等大 chunk）。生产默认的 `MixedMemoryAllocator` 按 `MemoryFormat` 路由 pinned 张量池与字节 buffer 池，并改写 `parent_allocator` 保证「从哪条池来、回哪条池去」的回收对称。物理内存经 `_allocate_cpu_memory` 取得 pinned 页锁定内存，这是异步 DMA 传输与跨进程零拷贝的物理基础。
