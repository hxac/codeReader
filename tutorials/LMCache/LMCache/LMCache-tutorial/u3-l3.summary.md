本讲聚焦 MP Coordinator：LMCache 中独立的 FastAPI/uvicorn 舰队级进程，维护任何单台 cache server 都无法独自知道的舰队信息（谁在线、谁缓存了什么内容、用量是否超标）。核心模块位于 lmcache/v1/mp_coordinator/。会员（membership）采用三段式分工：registry.py（纯内存会员表、单调时钟判活）、instances_api.py（REST 门面）、registrar.py（mp server 侧客户端，404 自愈）。blend_directory.py 即 GlobalBlendMatcher，用多项式滚动哈希 + 直接定址表实现舰队级内容指纹目录，遵循「coordinator 做所有哈希」原则，查询为 numpy gather + 稀疏复检，碰撞与过期条目安全降级为重算、永不返回错误 KV。registrar 与 blend_client 均为 opt-in、best-effort 的 no fate-sharing 客户端：coordinator 挂掉或未配置则推理照常运行。关键约束：全舰队 chunk_size 必须一致；cache_salt 不进指纹 key，靠 retrieve 时用请求方自己的 salt 展开实现租户隔离。
