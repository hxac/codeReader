本讲打开 u4-l2 中作为「黑盒」的 L2（远端异步存储层），讲解 `lmcache/v1/distributed/l2_adapters/` 包。核心是统一契约 `L2AdapterInterface`：它把 store / lookup_and_lock / load 三件事抽象为「批量、非阻塞、调用方管理内存生命周期」的原语，统一走「submit → poll event fd → query/pop 结果」三步模式。最硬的不变量是每个适配器必须暴露三个互不相同的 event fd（store / lookup_and_lock / load），否则控制器 fd→adapter 派发表会被静默误导；store 整批成败、lookup/load 返回逐 key 的 Bitmap。基类内置字节账本，子类调用 `_notify_keys_stored/_deleted` 即可由 `get_usage()` 返回只读 `AdapterUsage` 快照，`max_capacity_bytes` 决定 `supports_global_eviction` 与 `usage_fraction` 是否给出有效淘汰信号（`-1.0` 表示无信号）。接入靠「两张注册表（配置类 + 工厂）+ pkgutil 惰性自动发现」：新增 `*_l2_adapter.py` 文件并底部自注册即可零改动接入，缺第三方依赖的模块被静默跳过以实现依赖隔离。两大代表性实现：RESP（Redis/Valkey）属「薄配置 + 委托 C++ 连接器」流派，经 `NativeConnectorL2Adapter` 桥接；S3 属「纯 Python + asyncio + awscrt 全自研」，单守护线程事件循环、GET 用 `ctypes.memmove` 直写缓冲、带熔断器与 HEAD 缓存。`serde_config` 触发 `SerdeL2AdapterWrapper` 包装（接 u4-l4），`supports_global_eviction` 为淘汰/配额铺路（接 u4-l5）。
