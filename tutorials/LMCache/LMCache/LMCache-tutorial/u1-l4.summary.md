本讲梳理 LMCache 的进程入口与启动方式，追踪 `pyproject.toml` 的 `[project.scripts]` 到具体 `main()`。三个 console script：`lmcache`→`cli/main.py`（带子命令的诊断 CLI）、`lmcache_server`→`v1/server`（原始 socket 服务）、`lmcache_controller`→`v1/api_server`（cache-controller，非本讲重点）。关键结论：mp coordinator 不是 console script，需用 `python -m lmcache.v1.mp_coordinator` 或 `lmcache coordinator` 启动。核心术语包括：console script/module 入口、daemon、子命令自动发现（`discover_subclasses()` 扫描 `BaseCommand` 子类，坏命令「大声失败」）。两种后台进程风格对比：`v1/server` 用位置参数 + 裸 TCP + connection-per-thread + 自定义二进制协议（`ClientMetaMessage`/`ServerMetaMessage`）；`v1/mp_coordinator` 用 `LMCACHE_MP_COORDINATOR_*` 环境变量（`MPCoordinatorConfig.from_env()`，非法值告警回退）+ FastAPI app 工厂 `create_app` + uvicorn。`BaseCommand` 契约含 `name/help/add_arguments/execute` 四方法，通过 `set_defaults(func=execute)` 实现分发。
