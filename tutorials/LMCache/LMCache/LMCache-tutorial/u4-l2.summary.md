本讲解析 LMCache 分布式存储体系 `v1/distributed/`：它是 MP 架构下 cache server daemon 的存储大脑，与单机 `v1/storage_backend/` 并存（legacy 目录现仅剩 `serde/`）。核心是 L1（本地内存池）/ L2（远端异步适配器）两级模型——读写都在 L1 落脚，L2 全程非阻塞，将慢速 I/O 踢出 GPU 热路径。引入的关键术语：`ObjectKey`（内容寻址 + `cache_salt` 租户隔离 + `kv_rank` 并行编码）及其 JSON 形态 `EncodedObjectKey`；`Tier` 枚举（L1/L2/ALL）作为跨进程层级词汇；`StorageManager` 门面暴露写/读/预取 API；三类后台控制器 StoreController（L1→L2 复制）、PrefetchController（L2→L1 预取，仅加载连续前缀）、EvictionController（水位淘汰）；`L2AdapterInterface` 的 store/lookup_and_lock/load 非阻塞原语；`QuotaManager` 双语义多租户配额。后续讲义将展开 L2 适配器实现、SERDE 变换与淘汰策略。
