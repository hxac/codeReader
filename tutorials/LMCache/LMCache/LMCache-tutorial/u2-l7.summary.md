本讲打开 u2-l3 中作为黑盒一带而过的 RemoteBackend 序列化/反序列化。SERDE（序列化与反序列化）是「离开本机内存前的最后一道边界变换」，只服务于 LocalDiskBackend 与 RemoteBackend，本地热缓存零拷贝绕过；抽象契约仅 to_bytes/from_bytes，由工厂 CreateSerde 按配置字符串（naive/kivi/cachegen）选具体实现。

讲义对比两套编码路径。legacy CacheGen 采用「量化 + 算术熵编码」两步压缩：按层组用不同 bins 对称量化成 int8（x_q=round(x·C/m)，反量化需逐 token 存 max_tensors 还原尺度），再按通道统计 CDF、用 C 内核 encode_fast_new/decode_fast_prefsum 做算术编码；压得狠但实现重、K 亦有损。v1 的 AsymK16V8Codec 走「不对称量化」：K 保留原生 dtype 原样拷字节（因 K 直接参与 QK^T 点积，量化误差会被放大），V 量化到 FP8 e4m3fn（s=m/qmax 并 clamp 饱和），不做熵编码——更轻、更快、K 无损；scale 粒度分 PER_TENSOR/PER_LAYER_HEAD/PER_PAGE_HEAD，分页 KV 推荐最细。

v1 产物是自描述 blob：头部含 magic「LMCKV」、dtype/shape、六项识别哈希（防跨模型 cache poisoning，不匹配抛 CodecMismatchError）与 CRC32（防损坏，失败抛 CorruptEncodedKVError），解码前可完整自检。codec 内核由 MultiSerializer 粘合层接入分布式 L2，留待 u4 展开。
