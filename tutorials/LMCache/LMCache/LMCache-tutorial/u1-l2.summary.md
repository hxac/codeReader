本讲解决 LMCache 如何安装、构建与运行。其一，包元信息集中在 pyproject.toml：依赖与版本号均为动态生成，运行依赖刻意不锁 torch，避免与 vLLM、SGLang 自带的 torch 冲突；构建期则锁死 torch==2.11.0，仅用于发布可复现的 wheel。其二，注册三个命令行入口：lmcache 是带子命令的诊断 CLI，lmcache_server 是原始 socket 的 KV 存储 TCP 服务，lmcache_controller 是基于 FastAPI 的 HTTP 编排服务。其三，源码构建采用策略模式：setup_extensions 的 BuildPolicy 按环境开关或自动探测硬件选出 BuildProfile，CUDA profile 用 nvcc 把 csrc 源文件编译成 c_ops 扩展。最后是安装速查：开关 NO_NATIVE_EXT、NO_GPU_EXT、BUILD_WITH_HIP 分别对应纯源码、CPU-only、ROCm，并推荐 --no-build-isolation 以保留 torch 版本控制。
