本讲是学习手册首篇，仅精读 README.md 与 docs/design/README.md 两份文件建立整体认知。先说明 KV cache 是 attention 为历史 token 缓存的 Key/Value 向量，显存随上下文线性增长（一条 8K 请求约 1 GiB），而传统引擎请求结束即丢弃造成浪费。核心结论：LMCache 是横跨推理引擎与存储之间的「KV cache 管理层」，厂商中立而非引擎；它把算完即扔的 KV cache 变成可持久复用资产，降低 TTFT、提升吞吐，支持跨请求/会话/引擎实例复用，主线数据流为 store→lookup→retrieve，对应后续 LMCacheEngine 三大 API。引入术语：prefill/decode、TTFT、吞吐、offloading、分层存储、no fate-sharing、CacheBlend 非前缀复用、PD 分离与 KV 传输、SERDE。并约定 docs/design 镜像 lmcache/ 包树，读代码前先读对应 design doc。
