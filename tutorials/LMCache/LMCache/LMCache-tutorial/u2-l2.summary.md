本讲承接设备抽象层，深入 GPU 连接器：它在引擎分页 KV cache 与 LMCache 连续 MemoryObj（KV_2LTD 布局）之间充当双向翻译器，真正的内存搬运由 C 内核 multi_layer_kv_transfer 完成。抽象接口 GPUConnectorInterface 仅声明 to_gpu / from_gpu 及其批量版、get_shape，连接器本身不存数据。工厂 CreateGPUConnector 按 EngineType × torch_device_type × 配置开关（layerwise / blending / v3）路由到具体实现类；vLLM 仅覆盖 cuda/xpu/musa/hpu，cpu 等类型落入 else 抛 RuntimeError，体现早失败设计。默认主力 VLLMPagedMemGPUConnectorV2 用 slot_mapping 把连续 token 写入引擎分页槽，靠 store_stream/load_stream 两条 CUDA Stream 提升并发；V3 额外支持异构层组（如 DeepSeek V4 类压缩层）。KV 布局检测集中于 kv_format/ 子包：normalize_kv_and_discover_format 是全项目唯一解析入口，attempt_permute_to_contiguous_view 零拷贝还原物理视图，VLLM_Detector 结合 LayoutHints 判定 HND/NHD；核心不变量是布局只能在此一处解析。HND 与 NHD 仅 NH、BS 两维顺序不同，EngineKVFormat 枚举名即形状说明书。
