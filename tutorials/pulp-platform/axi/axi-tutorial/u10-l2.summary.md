本讲精读 `axi_id_serialize`：当下游 master ID 位宽卡死、连 `axi_id_remap` 都装不下（即 `AxiSlvPortMaxUniqIds > 2**AxiMstPortIdWidth`）时，它以**放弃 ID 独立性**为代价，允许两个不同 slave ID 映射到同一 master ID，使这两组事务被迫相互保序——故称「序列化」。核心是三段式通路：`axi_demux` 按 `SlvIdMap` 把事务**分桶**（桶号即目标 master ID）→ 每桶一个 `axi_serializer` 在桶内把 ID 抹零、并用存全宽 slave ID 的 FIFO 在响应方向还原原上游 ID → `axi_mux` 合并并把桶号前置进 master ID 高位。映射由编译期函数 `map_slv_ids` 一次性算成 `localparam`：默认 `(id+base)%N` 取模，可用显式 `IdMap` 覆盖（后写胜），`MstIdBaseOffset` 不影响被覆盖项。输出端 `id>>1`/`id<<1` 丢弃并补回 serializer 留下的恒 0 最低位；N=1 退化时改用 `axi_id_prepend` 零扩展。选型口诀：能保留独立性就用 `remap`，被迫放弃才用 `serialize`；面积无绝对优劣。它由 `axi_iw_converter` 在降 ID 分支自动选用，且无专属测试台。
