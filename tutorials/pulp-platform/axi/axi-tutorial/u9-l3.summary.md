`axi_serializer` 是一个「事务降级器」：它把上游发往下游的所有 AW/AR 的 `id` 强制抹零（写为 `'0`），使只认单一 ID 的能力受限下游（如 AXI-Lite 桥、单 outstanding 外设）正常工作；同时在上游响应返回前把原 id 还原，因此上游无感。其核心数据通路是读写两个独立的 **ID FIFO**（基于 `fifo_v3`）：请求握手时押入原 id 当备忘录，响应握手时按到达顺序弹出还原——因为所有下游 id 都是 0，唯一能区分事务的就是顺序。FIFO 的 `full` 反压请求方向（防原 id 丢失），`empty` 屏蔽响应方向（防错配）；其深度即各方向在途事务上限，决定了吞吐。读 FIFO 仅在 `r.last` 弹出（一次读突发多拍 R），写 FIFO 握手即弹（单 B）；`FALL_THROUGH=0` 使输出晚一拍，恰好对齐响应。AXI5 原子操作（ATOP）由 **AtopIdle / AtopDrain / AtopExecute** 三态机处理：先排空所有在途事务，再独占执行，并为 `ATOP_R_RESP`（无 AR 却有 R 的原子写）人为向读 FIFO 补一次押栈以还原其 R 的 id。序列化的代价是吞吐——去掉 ID 级并行与乱序空间；它与降突发长度的 `axi_burst_splitter` 正交，可串联喂养「单 ID + 单拍」最简下游，并为后续 u10-l2 的 `axi_id_serialize`（每 master 端口 ID 一个 serializer）提供积木。
