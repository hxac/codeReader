本讲介绍四件 AXI 等价性检查与事务可观测性工具，分两条路线：可综合的 axi_bus_compare、axi_slave_compare，与仅仿真的 axi_chan_compare、axi_dumper。共同内核是「请求方向入队、到达方向出队比对」：chan_compare 用 SV 无界队列最直白实现、失配即 `$error` 并打印逐字段对照表；bus_compare 用每 ID 一个 stream_fifo、两侧同 id 都 valid 才弹的同步机制做成硬件，输出 mismatch/busy 信号；slave_compare 是「一主驱双从」包装，请求 fork 给参考/测试两个从端、master 只取参考响应、测试响应握手后丢弃，内部复用 bus_compare。IgnoreId、AllowReordering、UseSize、FifoDepth 分别处理 ID 被改写、响应乱序、窄传输、时序差四类合法差异。axi_dumper 把每个握手 beat 落盘为类 Python 字典日志，整体被 ifdef TARGET_SIMULATION 包住（仓库脚本不自动定义，需手动 +define），配 axi_dumper_interpret.py 用 ast.literal_eval 解析、重拼事务并校验 AR↔R、AW↔W↔B 的 last 自洽。这些工具与 u3-l2 的 scoreboard 互补，覆盖在线自检到事后离线重放，核心术语包括等价性检查、抽头式比对、逐拍比对与黄金队列模型。
