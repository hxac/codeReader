本讲以 `tb_axi_xbar` 为载体讲解定向随机验证：在随机生成地址、ID、突发长度、停顿拍数的基础上施加约束以保证激励合法，再由旁路监听的黄金模型自动判对错。随机主端 `axi_rand_master` 用三道闸门——地址区间登记、在途计数上限反压、ID 合法化——确保合法性，闸门之内随机探索；从端 `axi_rand_slave` 提供随机响应与背压。测试台由时钟复位、随机激励、自检器、停止控制四类并发块组成，仿真长度由激励量而非固定时长决定。核心结论：xbar 只路由、不碰数据，故不用比对数据值的通用 scoreboard，而用专用的 `axi_xbar_monitor` 比对路由契约（端口、ID、addr、len、last 标志），以 `rand_id_queue` 容忍同 ID 保序下的乱序，并用 `Expected/Conducted/Failed` 三计数加信号量判通过（`Conducted==0` 同样判失败，防「空跑误通过」）。回归借 `run_vsim.sh` 的 `SEEDS` 数组与 `-sv_seed` 控制随机种子：种子 0 作可复现基线，`--random-seed` 追加真随机，同一 TB 跑多种子即可成倍扩展覆盖。
