本讲义讲解 `axi_burst_splitter` 与 `axi_burst_splitter_gran`：前者是把 `len_limit_i` 钉死为 `8'h00` 的单拍薄外壳，后者（Level 2 引擎）才是真正的拆分核心。引入关键概念「粒度」：以 AXI `len` 值 `len_limit_i` 表达每段子突发最大拍数，`max_beats = len_limit + 1`，段数为 `⌈N/(len_limit+1)⌉`，`len_limit=0` 即全拆单拍。动机是很多下游（AXI-Lite 桥、窄外设、单拍存储接口）吃不下长突发。顶层先用 1:2 `axi_demux_simple` 按 `txn_supported` 分流，不支持者（WRAP、ATOP、部分非 modifiable 短突发）交给 `axi_err_slv` 回 `RESP_SLVERR`。AW/AR 拆分由 `ax_chan` 的 `Idle`/`Busy` 状态机完成，仅 `BURST_INCR` 用 `aligned_addr` 推进地址，`BURST_FIXED` 地址不变，WRAP 不支持；并发在途由 `delta_counter` 阵列经 `id_queue` 按 ID 跟踪。三个响应通道配套改写：W 在段边界插入 `last`，B 把多个响应合并为一个并累积错误改写 SLVERR，R 抑制中间段 `last` 仅保留末拍——实现「地址方向拆、响应方向合」且对数据透明。
