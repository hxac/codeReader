本讲将 `axi_demux` 与 `axi_mux` 组装成全连接交叉开关 `axi_xbar`：顶层只例化 1 个 `axi_xbar_unmuxed`（demux 阵列 + cross 矩阵）加 `NoMstPorts` 个 `axi_mux`，体现「组合优于配置」。`axi_xbar_unmuxed` 负责路由：每个 slave 端口配一对 `addr_decode`（读写各自译码）和一个 `axi_demux`，多出一路接 `axi_err_slv` 吸收译码失败并返回 `RESP_DECERR`，再用 `axi_multicut` 摆成 cross 矩阵交给 mux。核心结论是 master 端口 ID 宽度 = slave 端口 ID 宽度 + ⌈log₂(NoSlvPorts)⌉，多出的高位是 mux 用 `axi_id_prepend` 前置的来源端口标签，使响应路由无需在途计数器。配置收口于 `xbar_cfg_t`：`LatencyMode` 高 5 位给 demux、低 5 位给 mux，`PipelineStages` 控 cross 处流水线级数；外加 `ATOPs`（原子操作开关）与 `Connectivity`（链路剪裁矩阵）两个独立开关。地址匹配、保序停顿与死锁证明留给后续讲义。
