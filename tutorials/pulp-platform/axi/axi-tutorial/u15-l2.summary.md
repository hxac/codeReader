本讲围绕单一核心模块 `axi_inval_filter` 展开，解决无硬件缓存一致性（cache coherence）系统中「写旁路」带来的陈旧数据问题：当 DMA 或非缓存主端绕过 cache 直写内存时，让 L1 cache 中对应的旧副本同步失效。核心结论是：它与上讲的 `axi_atop_filter` 同属「监听 AW + 产生副作用」家族，但本质区别在于——`inval_filter` 对 AXI 流量完全透明（不改地址/数据/响应码），仅从一个独立的非 AXI 端口（`inval_addr_o / inval_valid_o / inval_ready_i`）旁路通知 L1。结构分为三块：受 `en_i` 控制的 AW FIFO（深度 `MaxTxns`，存整条 `aw_chan_t`）、默认全通道直通（仅 FIFO 满时反压 AW）、两态无效化 FSM。关键术语包括：写旁路、cacheline、`L1LineWidth`（行内字节，默认 2 的幂）、突发总字节 `B_burst=(len+1)·2^size`。FSM 用行内偏移判定单行/多行跨越，多行时按行宽步进逐行发地址，恰好覆盖被写到的行。无效化由 AW 触发、与 W/B 解耦。该模块为叶子模块，无官方 testbench 与内部例化。
