本讲承接「仓库结构与构建系统」，把 Level 0–6 编译层级落到可执行操作，讲解如何用一条 `make` 命令完成编译、仿真、综合，不涉及 AXI 协议细节或任何 RTL 修改。最小模块包括：`Makefile`（总指挥，把三大动作包装成 `compile.log`/`sim-<tb>.log`/`elab.log` 文件目标，并以日志内容而非返回码判成败）、`scripts/compile_vsim.sh`（用 `bender script vsim -t test -t rtl` 生成按 Level 0–6 排序的编译脚本，仅对全库根基 `axi_pkg` 开启 `-lint -pedanticerrors`）、`scripts/run_vsim.sh`（围绕多 `sv_seed` 种子的定向随机回归）、`scripts/run_verilator.sh`（Verilator lint）与 `scripts/synth.sh`（Synopsys DC elaborate），后两者都针对综合用例 `axi_synth_bench`。核心结论有三：一是仓库用日志内容当通过判据——脚本层 `grep "Errors: 0,"` 判仿真器无错，Makefile 层 `grep "Error:"`/`"Fatal:"` 兜底；二是测试台遵循 `tb_<dut>.sv` 命名约定，`make sim-<dut>.log` 只跑指定 TB；三是 `axi_synth_bench` 用 `for` 循环一次性实例化大量宽度/参数配置，让 elaborate 快速回归可综合性。引入的关键术语：Bender target（rtl/test/simulation/synthesis/synth_test）、`sv_seed` 随机种子、`Errors: 0` 统计行、directed random verification（定向随机验证）与回归（regression）、综合 elaborate 与 lint，以及 DUT/TB 概念——这些为后续讲义拆解 `axi_pkg`/`axi_intf` 及手写测试台提供方法学基础。
