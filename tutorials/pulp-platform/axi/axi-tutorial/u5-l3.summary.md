本讲精读 `axi_mux`，它是 `axi_demux` 的对偶模块，做 N 合 1：把 `NoSlvPorts` 个 slave 端口汇聚成 1 个 master 端口。其核心机制是 ID 扩展路由——每个 slave 端口 `i` 用 `axi_id_prepend` 把端口号拼进 AXI ID 的最高 `MstIdxBits = $clog2(NoSlvPorts)` 位，因此 master 端口 ID 宽度 = slave 端口 ID 宽度 + MstIdxBits。请求方向 AW/AR 各用一棵 `rr_arb_tree`（`LockIn=1`）轮询挑一路；W 不仲裁，而是用一个深 `MaxWTrans` 的 W FIFO 按队头端口号转发 W 拍，`last` 拍弹队，杜绝跨端口 W 交错；`lock_aw_valid` 锁存切断 AW 与 W FIFO 的组合依赖以防死锁。响应方向 B/R 不需仲裁，只解码 ID 高位得到 `switch_id`，用 one-hot 仅点亮目标端口的 valid。由于 ID 高位天然隔离各端口，mux 无需 demux 那样的同 ID 在途计数器，这是它更简单的根本原因。在 `axi_xbar` 中每个 master 端口配一个 mux，与每个 slave 端口的 demux 通过内部矩阵对称相连。
