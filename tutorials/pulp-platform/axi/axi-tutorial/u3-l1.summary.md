本讲剖析 `axi_test` 包中的两个**底层逐拍驱动器**类：`axi_lite_driver` 与 `axi_driver`，它们把「在一个 AXI 通道上完成一次合法握手」封装成 `send_aw`/`recv_b` 等 task，免去手写握手与清零。核心机制是**虚接口（virtual interface）绑定**：`new()` 只保存 `AXI_LITE_DV`/`AXI_BUS_DV` 句柄，`reset_master()`/`reset_slave()` 严格按 modport 方向清零本侧输出。时序灵魂是两个参数：**TA（application time）**用 `<= #TA` 在沿后延迟施加激励（模拟 clock-to-Q），**TT（test time）**用 `#TT` 在沿后延迟采样输入（满足 setup/hold），约定 `0≤TA<TT<T_clk`。每个通道提供对称的 `send_*`（驱动 valid 逐拍等 ready，满足「valid 不撤」铁律）与 `recv_*`（驱动 ready 等 valid 后采样载荷）。`axi_driver` 改用 **beat 对象**（`axi_ax_beat` 等）承载 AXI4 十几根信号，其中 `rand` 字段为后续随机化奠基；另增 `mon_*` 被动监听任务，不驱动信号、只旁观 `valid && ready`，供 scoreboard 使用。最小范式为 `reset_*` → `send_aw/send_w` → `recv_b`（写）或 `send_ar` → `recv_r`（读）。
