本讲解析 `axi_demux` 的核心模块 `axi_demux_id_counters`。AXI 要求同 ID 同方向事务按序返回，demux 把同 ID 拆到不同端口后 B/R 响应会跨端口乱序，故须为每个（截断后的）ID 跟踪在途（outstanding）事务数与目标端口。该模块是 \(2^{AxiLookBits}\) 元计数器阵列，每元含一个 `delta_counter` 与端口选择寄存器 `mst_select_q`，提供 lookup（只读占用与端口）、push（+1 并登记端口）、pop（-1）、inject（ATOP 注入 +1）四操作，用 `unique case` 算同周期多操作命中时的净增量。`axi_demux_simple` 例化两份：AW 方向（AW 决策时 push、B 握手时 pop）与 AR 方向（AR 握手 push、最后一拍 R pop）。ATOP 原子读改写无 AR 却有 R，故 AW 握手时向 AR 计数器 inject 一笔 +1 抵消未来 pop，其安全性依赖原子事务 ID 在途唯一。三参数：`AxiLookBits` 权衡面积与误冲突，`MaxTrans` 决定计数器位宽与并发上限，`UniqueIds=1` 在「同方向在途 ID 唯一」或「同 ID 必去同端口」时可整组删除计数器阵列（面积 O(2^I)→O(I)）。需澄清：W 通道保序由独立的 W 突发队列（`w_select_q`）负责，id_counters 只服务 B/R 响应的保序。
