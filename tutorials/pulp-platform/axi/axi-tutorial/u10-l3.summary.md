`axi_iw_converter` 是 AXI 库中处理两端 ID 宽度不一致的统一入口模块。它自身几乎不写逻辑，本质是一个编译期「分发器」：按 slave/master 两端 ID 宽度的关系，从四种实现中编译期选一种。决策树两级判定——先看 master 端是否更窄（严格 `<`），再用判据 `AxiSlvPortMaxUniqIds ≤ 2^AxiMstPortIdWidth` 区分：成立则例化 `axi_id_remap`（保留 ID 独立性），否则例化 `axi_id_serialize`（放弃独立性换更窄 ID）；master 端更宽时用 `axi_id_prepend` 高位补零；等宽则 `assign` 直通。

关键术语与概念：「降宽/升宽」「在途唯一 ID 数」容量判据、「按分支条件生效」的 Max* 参数并集、编译层级 Level 4（= max(L2 remap, L3 serialize, L2 prepend)+1）作为「纯组合、无新协议逻辑」的证据、结构体内核与零逻辑接口外壳 `axi_iw_converter_intf`。

核心结论：该模块是 U10 单元的统一验证与封装入口；唯一测试台 `tb_axi_iw_converter` 用「除 ID 外逐字段相等」加「同上游 ID 映射到同下游 ID」两条不变式自检，靠 `run_vsim.sh` 参数矩阵（其 if 分支与 DUT 同构）覆盖全部四条分支。它与 `axi_dw_converter`（数据宽度转换）职责正交，并将在异构网络中充当 ID 子网胶水。
