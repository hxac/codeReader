本讲介绍两个会主动改写 AXI 字段的纯组合连接器。axi_modify_address 是地址替换器：请求方向把 AW/AR 的 addr 换成外部喂入的新地址（mst_aw_addr_i / mst_ar_addr_i），其余字段（含 W、valid/ready、atop）原样复制，响应方向整块直通。新地址由外部计算、模块本身不算地址，体现「组合优于配置」，可支持任意翻译规则与地址宽度转换；关键约束是新地址须在对应握手完成前保持稳定，以符合 AXI「valid 期间载荷稳定」规则。axi_id_prepend 是 ID 前置/剥离器：请求方向将来源标签 pre_id_i 拼到 ID 最高位前，响应方向利用「id 位于 struct packed 最高位」的排列，借位流类型转换丢弃高位完成剥离。派生宽度 PreIdWidth = AxiIdWidthMstPort − AxiIdWidthSlvPort，为 0 时走零开销直通分支。其真实调用方 axi_mux 把 slave 端口号作为标签拼入 ID，这正是 xbar「master 端口 ID 比 slave 端口更宽、用于路由响应」的物理实现。两模块字段正交（一个只改 addr、一个只改 id），可背靠背串联且无死锁、无冒险。
