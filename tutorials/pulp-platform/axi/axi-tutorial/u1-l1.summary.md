本讲为手册起点，定位 pulp-platform/axi：一个用 SystemVerilog 编写、遵循 AXI4 / AXI4-Lite 的片上通信网络 IP 库，既给拼搭网络的积木，也自带端点（DMA、存储控制器）与验证组件。核心哲学是「组合优于配置」——把 Unix 哲学搬进硬件，宁可把职责单一的小模块背靠背串起来，也不靠堆参数；axi_xbar 本身就是 unmuxed（demux 阵列）加 axi_mux 阵列拼成，是其活样本。引入术语：Master/Slave 主从、片上互联、五通道 valid/ready 握手、in flight（在途）与 pending（挂起）、AXI4+ATOPs（AXI4 叠加 AXI5 原子操作）。ATOP_R_RESP 位的原子写会同时产生 B 和 R 响应，不支持 ATOP 的从端前须加 axi_atop_filter。模块可按互连 / 转换 / 端点 / 验证四类建立心智地图。关键结论：xbar 的 master 端口 ID 比 slave 端口宽 ⌈log2(NoSlvPorts)⌉ 位，用于路由响应。本讲不要求懂协议细节。
