本讲讲解 AXI 的「1 拆 N」路由器。`axi_demux_simple`（Level 2）是纯组合内核：它不译码地址，而是接收外部喂入的 `slv_aw_select_i`/`slv_ar_select_i`（位宽 ⌈log2 N⌉），把 AW/AR 路由到 `NoMstPorts` 个 master 端口之一，体现了「译码在外、路由在内」的分工。请求方向采用「全广播载荷 + 选择性拉 valid」；W 通道没有自己的 select，靠 `w_open` 计数器加 `w_select_q` 寄存器组成的「W 突发队列」跟随其 AW，并用放行条件杜绝跨端口 W 交错。响应方向用两棵独立的 `rr_arb_tree`（`LockIn=1`）把 N 路 B/R 轮询合并成 1 路回收。`axi_demux`（Level 3）= 内核 + 五个可选 `spill_register`（默认仅 AW/AR 开），用于切断组合路径换时序；五开关全关即退化为 simple 行为。为满足 AXI 同 ID 同方向保序，内核例化 `axi_demux_id_counters` 追踪每个低位 ID 在途事务去的端口，同 ID 去往不同端口即停顿；`AxiLookBits` 折中面积与误冲突，`UniqueIds` 在保证唯一时可省掉整组计数器。demux 支持 ATOP，靠 AW 向 AR 计数器 inject 原子 ID 防止 R 通道下溢。需注意文档的 `FallThrough` 参数与源码不一致，以源码为准。
