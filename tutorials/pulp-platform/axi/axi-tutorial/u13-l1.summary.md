本讲讲解 AXI4 与 AXI4-Lite 之间的双向协议转换桥，核心结论是两方向难度不对称——因为 Lite 是 AXI4 的严格子集。`axi_lite_to_axi`（Lite→AXI4）只需补默认值（id=0、len=0、burst=FIXED、size=$clog2(数据宽度/8)、w.last=1、cache 由外部输入提供），是纯组合、无损的一段 assign，无需时钟。`axi_to_axi_lite`（AXI4→Lite）则需丢信息，构成三级流水线：`axi_atop_filter` 过滤原子操作、`axi_burst_splitter` 拆突发为单拍、`axi_to_axi_lite_id_reflect` 裁剪信号并完成 ID 回射，顺序不可调换。关键机制「ID 回射」：Lite 无 ID，模块用两个 fifo_v3（深度即最大在途事务数）暂存 AW/AR 的请求 id，在 B/R 响应返回时拼回，并以 FIFO 的 full/empty 做请求反压与响应门控；id_reflect 用 assume property 强制输入满足 atop==0、len==0、w.last==1，把「必须先接前两级」的依赖显式化。N 拍突发在 Lite 侧会被压扁为 N 个单拍事务，吞吐受 FIFO 深度限制。
