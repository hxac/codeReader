本讲（u9-l1）聚焦 FFPA 三个测试文件：前向 `test_ffpa_fwd.py`、反向 `test_ffpa_bwd.py`、用法锁定 `test_monkey_patch.py`。核心结论：前向测试把覆盖拆成两层——`CORRECTNESS_SHAPES`（少量精选形状 + `assert_close`）管精度，`DISPATCH_SHAPES`（HEADNUMS×HEADDIMS 笛卡尔积，只查 finite/shape/dtype）管「能跑通」；因小 D（D≤256）默认走 `fallback()` 回退，同一矩阵同时校验「回退路径」与「FFPA kernel 路径」。参考实现分工明确：`_sdpa_ref` 调 Python 层 SDPA，`_sdpa_fallback` 刻意调底层绑定 `torch._C._nn.scaled_dot_product_attention` 以镜像 FFPA 那条避免 monkey-patch 递归的 `# HACK` 回退分支；`_is_sdpa_fallback_shape` 是生产 `fallback()` 的测试侧镜像，决定该用哪个参考。`IS_ROCM` 探测驱动跨厂商处理：dropout 因 Triton-AMD RNG 不可对齐而 skip，反向大归约因 FMA/codegen 差异而放宽容差或标 xfail（附 gfx1101 等 reason）。monkey-patch 测试用「抛异常的桩」`_block_native_sdpa` 同时锁定大 D 真走 FFPA、小 D 回退不递归。
