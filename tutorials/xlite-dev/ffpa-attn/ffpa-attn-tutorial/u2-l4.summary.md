本讲讲解 GQA/MQA 分组查询注意力在 FFPA 中的实现。核心是让多个 Query 头共用一组 K/V：定义 group_size = Nh_q / Nh_kv，MQA 即 Nh_kv==1 的特例，MHA 即 group_size==1 的特例。FFPA 采用「连续分组」约定 h_kv = h_q // group_size，在 Triton kernel 中直接以指针偏移（off_hkv = off_hq // group_size）实现，无需复制 K/V。`enable_gqa` 默认 `False`，与原生 SDPA 严格对齐；头数不等时必须显式传 `True`，否则在 `normalize_inputs` 抛 `ValueError`。输入硬约束为 K/V 头数一致、`Nh_q` 能被 `Nh_kv` 整除、K/V 序列长度一致、三者 head_dim 一致。正确性参考实现用 `repeat_interleave(group_size, dim=1)` 沿头维连续复制 K/V 后跑 MHA，可与 FFPA 逐头对齐（不能用整体平铺的 `repeat`，否则头排列交错）。GQA 本身不触发 SDPA 回退，是否回退仍由 D、Nq、Nkv 决定。
