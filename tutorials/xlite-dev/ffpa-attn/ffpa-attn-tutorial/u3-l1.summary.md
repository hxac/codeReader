本讲是 FFPA 分发层（dispatch）总览，建立四个可替换后端的全局地图与选型指南。四后端定位：SDPA 为基线与回退目标（sm≥75、全 head_dim）；CUDA 为手写 kernel，仅前向、需单独编译 `_C`；Triton 是默认后端，前向+反向、跨架构含 AMD ROCm、支持 autotune；CuTeDSL 基于 CUTLASS，sm90 专用路径在 H200 最快。分发层 `functional.py` 直接 import 核心依赖 triton/aten，用 try/except 挂可选后端 cute/cuda，缺失置 None。运行时能力探测分两套：CUDA 用编译期常量 `CUDA_FWD_AVAILABLE`，CuTeDSL 用运行时函数 `cute_forward_available()`（判 major≥8）。分发决策树：D≤256 优先走 aten，再按 CUDABackend/TritonBackend/CuTeDSLBackend 分支。核心结论是默认选 Triton 而非 CUDA：前者前向反向齐全、纯 Python JIT、跨厂商，后者无反向且需编译；前向↔反向为多对多，`_ffpa_apply` 用 `torch._dynamo.disable` 断图。引入术语：backend、dispatch、fallback、Backend 配置子类、autotune、acc、小 D 边界 256。
