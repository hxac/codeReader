本讲剖析 FFPA Triton 后端的在线自动调优机制，核心是两件事：生成候选 config 与定义缓存 key。引入关键概念：config 是 tile 大小（BLOCK_M/BLOCK_N/BLOCK_HEADDIM_QK/V）与 num_warps/num_stages 的组合；autotune_mode 分 fast/max 两档——max 在候选更多（前向 generic 大 D 下 fast=8、max=32）与 seqlen 分桶更细两方面都更重。autotune_seqlen_key 把 seqlen 分桶，使相近长度复用同一次调优、避免逐值重搜，缓存 key 统一为四元组（seqlen_q/k 桶值、causal、HEADDIM）。同时梳理端到端链路：TritonBackend 的 autotune/autotune_mode 旋钮经 op 边界将 mode 编码为 int、再按 num_splits 分流到 generic/decode 启动器；autotune=False 时改走持久化配置或默认 config，不在线搜索。须区分 full-D 候选的 docstring 动机与代码实际判定（仅 headdim==256，且大 D 路径不可达）。本讲引入的 exact_autotune_seqlen_keys 为后续 u8-l2 离线生成器、u8-l3 运行期查找铺垫。
