本讲介绍 FFPA 的端到端基准套件 `python -m ffpa_attn.bench`，经 `bench.py` → `_bench.main` → `_benchmark_rows` → 前向/反向 runner 的链路，对八类标准用例（self/cross/decode/gqa/causal/attn-mask/dropout/non-aligned）跑出延迟、TFLOPS、对 SDPA 的 speedup 并校验正确性。关键术语：八类用例由基准序列长度 `N` 派生固定形状，CuTeDSL 后端会剔除不支持的 attn-mask/dropout/decode。TFLOPS 只统计注意力的两次主 GEMM（dominant GEMM FLOPs）以保证跨实现可比：前向 FLOPS = `4·B·Hq·D·valid_pairs`，反向 = `2.5×` 前向，`TFLOPS = FLOPS / (latency_ms × 1e9)`；valid_pairs 在因果时按「query 对齐 KV 尾部」逐行累加。延迟测量采用 warmup + iters + `synchronize` 流程：前向用 CPU 端 `perf_counter`，反向默认用 CUDA event 只隔离 `backward()`。正确性用 `torch.allclose` 对比 SDPA，容差随 dtype/方向变化（fp16=2e-2、bf16=5e-2、bf16+causal 反向=7.5e-2），dropout 时强制 SDPA 走 efficient attention 后端以对齐 RNG。`--tune` 仅在前后向后端为 triton 时生效，`--show-fallback` 可在无 GPU 环境出图。
