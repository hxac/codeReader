本讲是「手写 CUDA 后端」首篇，跨过 Python 边界进入 `csrc/cuffpa/` 的手写前向 kernel（需 `ENABLE_FFPA_CUDA_IMPL=1` 编译出 `ffpa_attn._C` 才被 `CUDABackend` 选中，只实现前向）。核心结论：前向按形状分三套模板——大 D 走 `split_d` 的 **Split-D**、小 D 走 `persistent`（不切 D）、decode 走 `stage1/stage2` 两阶段（split-KV + log-sum-exp 合并），由 `launch_templates.cuh` 在 host 端按 head_dim、Nq 与占用率选定。大 D 主线是 KV 块主循环的四阶段：算 score（Split-D 归约，累加进同一块）→ mask + online softmax → 算 O（每 D 片段独立累加器）→ 滚动重缩放并更新 m/l，使 SRAM 占用 O(1) in D、寄存器 O(d/4)。引入关键术语：`g2s/s2r/r2g`、`mma.sync.m16n8k16` 与 `ldmatrix` 的 fragment 布局、`MMAMode::kInplaceUpdate`、`cp.async` 多级缓冲（`kStageQK/kStagePV`、`commit/wait_group`）、Hopper **TMA** bulk-copy 与 mbarrier 相位等待、XOR **swizzle** 消 SMEM bank 冲突（cp.async 与 TMA 共用同一 swizzled 布局）；并强调 bf16 强制 fp32 累加的硬件约束。
