本讲区分三种注意力模式：self（Nq==Nkv）、cross（Nq!=Nkv，常 Nq<Nkv）、decode（Nq 极小、常为 1）。因 QK^T 形状为 Nq×Nkv，Q 定行、KV 定列，故 Nq 可不等于 Nkv，输出 O 的序列维始终跟 Nq。三者共用同一 ffpa_attn_func，仅凭传入张量形状区分，无独立 API。硬约束：key 与 value 必须共享相同的 Nh_kv 与 Nkv，否则 normalize_inputs 抛 ValueError；is_causal=True 还需 Nkv>=Nq。另需注意 8<=Nq<512 或 Nkv<512 会回退 SDPA，小 Nq 用例默认未必真跑在 FFPA kernel 上。核心结论：decode 因 query 行太少、prefill 主 kernel 并行度不足以填满 SM，改走专门 split-KV 路径——沿 KV 切 chunk、用 LSE 在对数域合并各段部分输出以补回并行度，切分与否由 0.8×num_sms 占用率启发式判定；Nq==1 时还会进一步走 USE_GEMV 向量归约特化。
