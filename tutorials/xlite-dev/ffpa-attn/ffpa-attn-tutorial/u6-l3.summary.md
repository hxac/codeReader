本讲聚焦 CuTeDSL SM90 kernel 的「调度与同步」层，分三条主线。其一，tile scheduler 把线性 `block_idx` 解码成 4 轴工作 tile `(m_block, head_idx, batch_idx, split)`，统一封装在 `WorkTileInfo` 中，共享 CUTLASS 的 `initial_work_tile_info` / `advance_to_next_work` / `producer_tail` 协议；三种在用实现对应不同场景：`SingleTileScheduler`（恒等映射、dense 非因果）、`SingleTileLPTScheduler`（L2 swizzle + LPT 块序反转、dense 因果）、`SingleTileVarlenScheduler`（warp 级 prefix-sum 定位 batch、varlen），外加预留的 `SingleTileLPTBwdScheduler`。L2 swizzle 让连续 CTA 共用 K/V 以命中 L2，宽度由 50MB 预算除单头 KV 体积取 2 的幂；LPT 则让因果掩码下最重 tile 先跑以均衡负载。其二，Pipeline 借 `PipelineTmaAsync`（TMA 加载、带 `extra_tx_count`）与 `PipelineAsync`（SMEM 中转、elect_one 提交）经 mbarrier 的 empty/full 双向信用实现多级缓冲背压，把 g2s 与 WGMMA 重叠以隐藏显存延迟。其三，`NamedBarrier` 是 0–15 号 PTX barrier ID 的可读别名，与 Pipeline 互补：一次性事件用裸命名屏障，带数据流转的握手用 Pipeline。三者抽象均源自 flash-attention 的 SM90 精简版。
