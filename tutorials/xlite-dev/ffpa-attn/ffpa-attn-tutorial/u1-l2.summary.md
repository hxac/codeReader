本讲讲清 FFPA 的三种落地方式：PyPI 预编译 wheel（零编译）、源码 Triton-only（默认，不生成二进制扩展 `ffpa_attn._C`）、源码 + CUDA 前向扩展（设 `ENABLE_FFPA_CUDA_IMPL=1` 才编译 `_C`）。核心结论：是否编译 `_C` 由 `setup.py` 的 `BUILD_CUDA_EXT` 决定，等于 `(not FFPA_SKIP_CUDA_EXT) and ENV.enable_fwd_cuda_impl()`；默认为 Triton-only，手写 CUDA 仅覆盖前向，反向仍走 Triton/SDPA。职责分工上 `setup.py` 决定「要不要编」，`env.py` 的 `ENV` 类决定「编什么、怎么编」（head_dim 集合、源文件、nvcc 参数、目标 SM）。关键术语：构建隔离与 `--no-build-isolation`、`CUDA_FWD_AVAILABLE` 运行时优雅降级、按 head_dim 拆翻译单元、`FFPA_DEV_HEADDIMS` 子集、`build_fast.sh` 用 ccache shim + `MAX_JOBS` + `FFPA_NVCC_THREADS` 提速；`pyproject.toml` 的 `packages.find` 把 `csrc*` 排除在 Python 包之外，故 Triton-only 产物不含任何 C++ 源码。
