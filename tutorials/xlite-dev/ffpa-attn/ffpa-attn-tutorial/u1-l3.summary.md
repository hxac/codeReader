本讲是 FFPA 源码地图篇。核心结论：仓库呈双轨制——Python 轨 `src/ffpa_attn` 与 CUDA 轨 `csrc/cuffpa` 经 `pyproject.toml` 包发现规则隔离，`csrc*` 不进 wheel，Triton-only 产物无任何 C++ 源码。`src/ffpa_attn` 采用「公共入口 `ffpa_attn_interface.py`（定义 `ffpa_attn_func`）+ 分发层 `functional.py`（Backend 配置类、autograd.Function）+ 六子包」分层：`aten`（SDPA 小 D 回退，D≤256）、`cuda`（手写 CUDA，需 `_C`）、`triton`（默认，前向+反向）、`cute`（CuTeDSL，目录名与对外后端名 `"cutedsl"` 不一致）、`ray`（多卡自动调优）、`cli`（基准）。关键术语：翻译单元 TU、`.cuh` 模板、按 head_dim 代码生成（文件名仅含「存储 dtype+head_dim」，函数名多编码累加器精度，故每 head_dim 2 文件 3 声明，bf16 必须 fp32 累加、无 bf16f16 变体）、pybind 入口 `ffpa_attn._C`。辅助目录 `bench/tests/tools/docs` 不进 wheel，配置疑问查 `docs/env.md`。
