本讲为 Split-D 专题：在 Triton 前向 kernel 内部对 head_dim（D）做 MMA 级精细分块。核心是 D 在两次矩阵乘里身份不同——QK^T 中为归约维（可拆分相加），PV 中为输出维（只能各存各的）。由此引入两个对偶计数：num_qk_d_chunks 控制 Phase 1 的 D 归约循环，NUM_V_GROUPS=cdiv(D,BLOCK_HEADDIM_V) 决定 Phase 3 的独立累加器个数，二者因 tile lockstep 数值相等而语义相反。Phase 1 用 tl.dot(acc=scores) 把各 D 片段累加进同一块 score；Phase 3 为每个 V-group 配独立累加器 o_accs[g]，按 R_g=α·R_g+P@V_g 更新，同一块 P 被所有 V-group 复用以免重算 QK/softmax。结论：SRAM 工作集与 D 无关（O(1) in D），压力转移到寄存器，o_accs 随 D 线性增长（O(D)），这是 FFPA 支持 D=320/512/1024 的根本；o_accs 元组对应手写 CUDA 的 R_D 寄存器。
