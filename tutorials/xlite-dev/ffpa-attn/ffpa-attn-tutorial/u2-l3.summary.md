本讲讲解 `ffpa_attn_func` 限制可见范围的两种机制。因果掩码 `is_causal=True` 采用 query 对齐 KV 尾部的约定：query 行 r 只看 key 列 k≤r+(Nkv−Nq)，要求 Nkv≥Nq，否则 `normalize_inputs` 抛 ValueError；由 kernel 内部行/列比较实现，零开销、不物化掩码。`attn_mask` 有布尔（True 参与、False 映射为 −∞，不可导）与可加偏置（直接加到 score）两种语义，`normalize_attn_mask` 将二者统一归一为紧凑 4 维 `attn_bias`，广播维度不展开、零 stride 广播在 kernel 内完成。二者互斥，同时设置抛 RuntimeError。`_validate_attn_mask_shape` 按 SDPA 约定校验广播：最后两维恒为 [Nq,Nkv]，3D 首维是 batch（非 head），4D 才能按 head 区分。掩码处理仅在非回退路径（大序列 Triton 后端）生效，否则回退原生 SDPA。
