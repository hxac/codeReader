本讲说明如何用一行 monkey-patch（`F.scaled_dot_product_attention = ffpa_attn_func`）把 FFPA 接入任意使用 SDPA 的现成模型。前提是 `ffpa_attn_func` 与 SDPA 签名对齐（前三个位置参数 q/k/v 加同名关键字）。关键概念：monkey-patch 指运行时替换模块函数；`torch._C._nn.scaled_dot_product_attention` 是 SDPA 的底层 C++ 绑定，区别于 Python 符号 `F.scaled_dot_product_attention`。核心结论：`ffpa_attn_func` 内部先调用 `FFPAAttnMeta.fallback()` 做短路判定，默认 Triton 后端下 `D≤256`、`D>1024`、`8≤Nq<512`、`Nkv<512` 任一成立即静默回退原生 SDPA，否则进入 FFPA kernel（判定只看前向、不看反向）。回退分支必须调底层绑定而非被 patch 过的 Python 符号，否则会无限递归——源码以 `# HACK: Avoid recursive for monkey-patch usage.` 标注；测试用「把底层绑定换成抛异常的桩」同时锁定「大 D 走 FFPA」与「回退不递归」。
