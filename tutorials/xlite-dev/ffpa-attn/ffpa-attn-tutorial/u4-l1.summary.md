本讲只覆盖 FFPA Triton 前向的**通用路径**（generic path，`num_splits==1`），主角是 kernel `_ffpa_fwd_kernel_impl` 与宿主启动器 `_ffpa_attn_forward_generic_impl`，文件为 `src/ffpa_attn/triton/_ffpa_fwd.py`。它用 FlashAttention v2 风格的**二维网格**：`program_id(0)` 是 Q 行块 `start_m`，`program_id(1)` 是「batch×query 头」拍扁索引 `off_hb`，再用 `off_hkv = off_hq // group_size` 直接换算 GQA 的 KV 头，无需复制 K/V。每个 program 维护逐行状态 `m_i`（行最大值，初值 -∞）、`l_i`（行求和，初值 0）和按 V 片段分组的 fp32 累加器 `o_accs`（片段数 `NUM_V_GROUPS`）。KV 主循环分四阶段：Phase 1 沿 D 分片累加出完整 score（Split-D 的 QKᵀ）；Phase 2 用 `m_new/alpha/p/l_new` 做 **online softmax** 逐行重缩放；Phase 3 把同一 `p` 复用到每个 V 片段累加 PV（省算力）；Phase 4 收尾 `out = o_accs/(l_i+1e-10)`、`LSE = m_i+log(l_i)`，除以 `l_i` 恰等价于 `softmax(s)V`。LSE 保存自然对数约定且 dropout 只作用在 O 累加的 `p` 上。启动器负责算 grid 并在 autotune / 持久化配置 / 默认 config 三条路径间选择；更上层的 `_ffpa_attn_forward_impl` 用 split-KV 占用启发式决定何时走本路径，decode 两阶段留给 u4-l3。
