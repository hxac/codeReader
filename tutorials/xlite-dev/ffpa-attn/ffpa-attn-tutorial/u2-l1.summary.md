本讲聚焦 FFPA 密集注意力公共入口 `ffpa_attn_func` 的正确调用，其签名与 `torch.nn.functional.scaled_dot_product_attention`（SDPA）对齐：`(query, key, value, attn_mask=None, dropout_p=0.0, is_causal=False, scale=None, enable_gqa=False, **kwargs)`，末尾 `**kwargs` 承载 FFPA 专有后端配置。函数体仅三步：`fallback` 短路回退到原生 SDPA（调底层绑定以免递归）、`normalize` 做校验归一化、交 `FFPAAttnFunc.apply` 进入 autograd 边界。

核心约束：`query/key/value` 为四维 `[B, Nh, N, D]` 布局；`Nq` 可不等 `Nkv`（cross/decode）；`Nh_q` 必须整除 `Nh_kv`（GQA/MQA），`key/value` 共享相同 `Nh_kv` 与 `Nkv`，三者 `head_dim` 一致；dtype 须为 `fp16` 或 `bf16` 且一致，否则报 `TypeError`；`scale=None` 时默认取 `1/sqrt(D)`；`enable_gqa` 默认 `False` 以匹配 SDPA，头数不等须显式置 `True`；`is_causal=True` 要求 `Nkv>=Nq`。返回单个张量 `O`，形状 `[B, Nh_q, Nq, D]` 与 query 同形、dtype 一致，内容为 `softmax(scale·QKᵀ)V`，与 SDPA 数值接近。本讲引入并供后续承接的关键术语：SDPA、`[B,Nh,N,D]` 布局、GQA/MQA、group_size、enable_gqa、is_causal 尾对齐约定、normalize_inputs、FFPAAttnFunc.apply。
