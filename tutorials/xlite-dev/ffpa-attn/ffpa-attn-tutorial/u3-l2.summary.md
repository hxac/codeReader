本讲集中在 `functional.py` 一个文件，讲解 FFPA 的 Backend 配置类体系。核心是 `Backend` 基类用 `forward` / `backward` 两个 `bool | None` 字段表达「管不管某一段」——`None` 表示「未声明」，由 `__post_init__` 自动补全为 bool（都 None 则都置 True，单边 None 取另一边的非），由此实现前向与反向后端的独立解耦，配置对象本质是一对后端。

四个子类各加专有字段与硬约束：`SDPABackend` 近空壳（前向恒短路回退，主要作反向）；`CUDABackend` 仅前向，带 `acc`/`stages` 旋钮；`TritonBackend` 旋钮最多（autotune、enable_tma、enable_ws、persist_dkdv、split_launch 等），反向专用开关需 `backward=True`；`CuTeDSLBackend` 仅暴露 SM80 反向存储精度。`_coerce_backend` 把字符串或实例统一成 Backend 实例，同一字符串随 `source` 不同生成 forward/backward 标志不同的对象，传实例则原样返回。`FFPAAttnMeta.from_kwargs` 按优先级「显式 forward_backend/backward_backend > backend > 默认 Triton」解析三参数，`backend` 仅在两侧都未显式时充当两段，否则被静默忽略；配置层强制 cutedsl 前后对称并为单边自动补全，未知 kwarg 立即拒绝以 fail-fast。
