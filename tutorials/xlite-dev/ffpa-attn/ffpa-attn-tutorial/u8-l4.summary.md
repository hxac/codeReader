本讲讲解如何把离线持久化调优从单卡串行变成多卡并行。`@ray.remote(num_gpus=1)` 实现「一卡一 actor」的物理 GPU 独占，保证 benchmark 计时准确；worker 按物理卡的 PCI bus id 建私有 Triton cache 目录，既避免多进程并发 JIT 编译的文件竞争，又支持跨 session 复用。调度引擎 `run_ray_autotune` 用 actor pool + 工作队列两阶段：Phase A 给每个 worker 塞一个任务填满，Phase B 用 `ray.wait(num_returns=1)` 完成即派发，把下个任务交给最先空闲的同一 actor，实现动态负载均衡。两层容错：worker 吞 OOM 转空结果、引擎吞 `RayActorError`/`RayTaskError` 记 warning 继续，单个失败不拖垮整批。Ray 运行时懒加载，仅 `--num-gpus>1` 走多卡路径，输出格式与单卡完全一致。
