本讲义是 FFPA 项目的入门概念篇，覆盖四个最小模块：大 head_dim 的困难、Split-D 方案、复杂度结论、适用边界。

FFPA（Faster Flash Prefill Attention）是专为**大 head_dim（D ∈ [320, 1024]）**设计的 Flash Prefill Attention，相对 PyTorch SDPA 快 1.5~3 倍。经典 FlashAttention-2 沿序列长度 N 方向分块、每块宽仍为完整 D，SRAM 占用 ∝ (Bᵣ+2Bᵦ)·D，随 D 线性增长；当 D>256 时撑爆单 SM 的 SRAM 预算，故 FA-2 上限止于 256。

核心创新 **Split-D** 指在 **MMA 指令层**对 QKᵀ 与 PV 两次矩阵乘做**精细分块（fine-grained tiling）**：沿 head_dim 方向切成宽 16 的片段，使 SRAM 任意时刻只驻留 16 宽的 Q/K/V 切片。

由此得到两条硬核结论：**SRAM 复杂度 O(Bᵣ×16)≈O(1)**（与 D 无关）；**寄存器复杂度 O(d/4)**（随 D 线性，1/4 折扣来自 MMA fragment 打包）。即 Split-D 把 D 方向的压力从 SRAM 转移到更能扛的寄存器。

适用边界：FFPA 主攻 **prefill + 大 D + 长序列（N≥512）**；当 **D≤256 或 N<512** 时未必比 SDPA 快，会**自动回退（fallback）到 SDPA**，二者是互补而非替代关系。
