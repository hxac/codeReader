本讲回答 decode 形状（Nq 极小、Nkv 长）下为何弃用 generic 路径：Q 行不足使网格塌缩、SM 跑不满。FFPA 用 FlashAttention 风格的 `_get_decode_num_splits` 占用度启发式，并行度不足 80% SM 时沿 KV 切成 `num_splits` 个 chunk，以 KV 维并行补 Q 维不足。前向变两阶段：stage1 每个 program 算一个 KV chunk 的局部 softmax 输出 `partial_out` 与 chunk 级 logsumexp `chunk_lse`，Nq=1 走 GEMV 向量归约（一维累加器、不切 D），Nq>1 走多行 MMA（保留 Split-D）；stage2 用 log-sum-exp（w_c=exp(LSE_c−m)）在对数域合并各 chunk 成最终 O 与全局 LSE，数学上严格等价于整体 softmax·V。启动器 `_ffpa_attn_forward_decode_impl` 分配两块 fp32 scratch、运行期注入 CHUNK_SIZE 并串起两 kernel，上层以 `num_splits==1` 判定是否回退 generic。引入术语：split-KV、chunk、num_splits、占用度启发式、GEMV 路径、partial_out/chunk_lse、LSE 对数域合并。
