本讲聚焦 CuTeDSL 后端的「入口胶水层」，含四个最小模块。① 布局转换：dense 用 `_bhnd_to_bnhd`/`_bnhd_to_bhnd` 在 SDPA 的 `[B,H,N,D]` 与 FlashAttention 原生 `[B,N,H,D]` 间互转（须 `.contiguous()` 物化；`lse` kernel 出口已头在前故不转）；varlen 的 packed `[T,H,D]`（THD 布局）本即原生布局，无需转置。② dense 算子 `_fwd_cute`/`_bwd_cute` 走 `torch.library` 三件套（define/impl/register_fake）却刻意不绑 `register_autograd`，反向留在 `_FFPAAttnFunc.backward` 按运行时 meta 分发，以支持「一前向多反向」。③ `_check_supported_options` 遵循「无静默回退」原则，把 dropout/window/softcap/mask 等 11 类扩展选项一次性点名拒绝，指向 triton 后端。④ varlen 用 `custom_op` 加 `register_autograd` 自管 autograd、整链绕开 `_FFPAAttnFunc`，可选 int 以哨兵 `-(2**31)` 编码过 schema；并引入 `SeqlenInfoQK`（batch 边界切片）与 `pack_gqa`（GQA 头折叠进序列维）。两路径共用张量级校验 `_require_cute_supported` 与 SM 分发谓词 `_use_sm90_specialized`（选 SM90 专用或 SM80 通用 Split-D）。
