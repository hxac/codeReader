本讲（u7-l3，依赖 u7-l1）退回 host 侧，讲清 FFPA 如何把「多种 head_dim × dtype × 累加精度」的组合编译得快、分发得对并暴露成 Python 可调用函数。最小模块有四：env.py 的 per-headdim 代码生成器 `generate_split_headdim_sources`（按 dtype/acc 生成声明头 + 每 head_dim 两份 `.cu` TU + 一份 dispatch，共约 27 份文件）；C++ 三级分发链（统一入口 `ffpa_attn_forward` → `(dtype,acc)` 入口 `ffpa_attn_fwd_{fp16f16,fp16f32,bf16f32}` → per-headdim 符号 `..._d{D}` → `launch_ffpa_attn_fwd_template`）；`PYBIND11_MODULE` 把 C++ 函数绑成 `ffpa_attn._C.ffpa_attn_forward`；上层再包 `torch.library` 算子 `_fwd_cuda` 以支持 `torch.compile` 与统一分配输出。核心结论：拆 TU 是为把并行度从文件内提到文件间，加速 nvcc 编译；bf16 激活必须 fp32 累加（GPU 无 bf16-acc 的 MMA PTX），在 Python、C++ 入口、代码生成三层纵深强制。关键术语：acc 编码（0=f16/1=f32）、翻译单元、模板特化、`CUDA_FWD_AVAILABLE` 编译期能力位。
