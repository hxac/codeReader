本讲讲解 FFPA 的持久化调优配置生成器——离线 CLI `python -m ffpa_attn.autotune`，把目标显卡上对各种形状调出的最优 launch config 写成一张「设备 JSON」，供运行时零调优复用，以解决在线 autotune 启动慢、结果不持久的问题。四个最小模块：`_parse_args` 暴露 `--mode`（须与运行时 `autotune_mode` 严格一致）、`--directions`、`--dtypes`、`--B/--H`、`--full-tasks`、`--enable-*-tma/ws`（新旧两套开关等价）等开关；任务网格由 `DEFAULT_HEADDIMS`（仅 ≥320 大头维）与 `DEFAULT_SEQLENS` 张成，分 prefill（`Nq≥512`）与 decode（`Nq=1`）两支，生成期切「精确 key」让每个形状各自 benchmark，`16384` 需 ≥48 GiB 才纳入；`--full-tasks` 追加 attn-mask/dropout/gqa/mqa 四种单特性变体（非笛卡尔积），GQA 头数由 `_resolve_gqa_heads` 取 `heads//4` 向下整除值；调优结果经 `_record_entry` 用元组 key 去重、`_build_payload` 套上 `schema_version` 等元数据后，按 `sanitize_device_name(显卡名)` 落盘（如 `NVIDIA_L20.json`），默认写包内 `configs/` 并双重防覆盖。一条 entry 含 direction、kernel、causal、dtype、headdim、seqlen_q/k、has_attn_bias、has_dropout、config 等字段，外加各 kernel 专属字段。后续 u8-l3 讲运行时如何查找这张 JSON。
