本讲承接 u5-l1 的 delta 预处理，进入 FFPA 反向主路径（Nq≥8）计算 dQ/dK/dV 三个梯度的 Triton kernel，核心是 **shared program-id（共享 pid）网格设计**：同一个 `pid` 在 `_ffpa_bwd_dkdv` 中被解释为 K 列块（外层遍历 Q 块），在 `_ffpa_bwd_dq` 中被解释为 Q 行块（外层遍历 K 块），网格第 0 维取 `max(cdiv(Nk,BN), cdiv(Nq,BM))`，越界角色由守卫跳过。关键结论是**所有权（ownership）决定是否需要原子**：每个输出 tile 由唯一 program 单写、累加发生在 program 内部串行循环，故非融合路径中 dQ/dK/dV 全部用 load-add-store、无 `atomic_add`，并以「首块 store、其余 load+add+store」替代原子。代价是 score 矩阵被算两遍。融合变体 `_ffpa_bwd_dkdvdq` 只算一次 dS 并复用到三个累加，但 pid 改为只索引 K 块，使 dQ 失去唯一拥有者而必须用 `tl.atomic_add`。融合由 `FFPA_TRITON_BWD_FUSE_DKDVDQ=1` 触发（默认关闭），bf16 DQ 在 SM<90 上原子退化为 CAS 循环，可用 `grad_q_storage_dtype=fp32` 缓解。Split-D 的 D 分片循环贯穿三种 launch 模式（单 launch 共享 pid / split 双 launch / dkdvdq 融合）。decode 反向（Nq<8）与 SM90/TMA/WS 开关切至 u5-l3、u5-l4。
