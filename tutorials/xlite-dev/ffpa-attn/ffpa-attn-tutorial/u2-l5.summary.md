本讲讲解 FFPA 的变长注意力入口 `ffpa_attn_varlen_func`，承接密集版 `ffpa_attn_func`。它解决训练中序列不等长时密集 `[B,H,N,D]` 布局需 padding 到 `N_max` 造成的算力与显存浪费，改用 **packed THD 布局**：把整个 batch 的 token 首尾相接成 `[T,H,D]`，无 padding；batch 边界用 **`cu_seqlens`（长度 B+1 的 int32 累计偏移，首元素为 0、末元素等于 T）** 描述，kernel 内部按边界做分段注意力，同一条序列的 Q/K/V 才互相 attend。其**仅 CuTeDSL 后端**可用，受限于 SM8x/SM90、大 head_dim（D∈[320,1024] 且对齐 32）、fp16/bf16，不满足直接报错、不回退 SDPA。关键概念：`max_seqlen_q/k` 填单条最大长度而非 T；`return_lse=True` 时返回 `(out, lse)`，`out` 为 `[T_q,H_q,D]`，`lse` 为 `[H_q,T_q]` 头在前的 fp32 张量。调用链为：公共函数透传 → `FFPAAttnVarlenFunc` 经 `@torch._dynamo.disable` 断图 → `_ffpa_attn_varlen_cute` 逐层校验与 SM90/SM80 kernel 分发 → `torch.ops.ffpa_attn._varlen_fwd_cute`。varlen **自己管 autograd**，用 `custom_op`+`register_autograd` 绑定专属反向，不复用密集路径的 `FFPAAttnFunc`。守门员 `_check_supported_options` 对 `dropout_p`/`window_size`/`softcap` 等几乎所有 FlashAttention 扩展选项遵循「**无静默回退**」原则，一次性点名拒绝并建议改用 `forward_backend='triton'`。
