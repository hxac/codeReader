本讲梳理 LeetCUDA 仓库结构与 kernel 模块约定。顶层按「学习内容」而非软件分层组织：核心是 kernels/（约 30 个算子子目录），外围有 third-party/（CUTLASS 依赖）、others/（PyTorch 分布式与 TensorRT 扩展）、slides/、docs/，以及 cutlass、HGEMM、ffpa-attn 三个 git 子模块；clone 后须执行 git submodule update --init --recursive 拉取，否则子模块目录为空。三条核心约定：① kernels/ 下多数算子遵循「三件套」README.md + <算子>.cu + <算子>.py，阅读顺序固定为 README → .cu → .py；② 一个 GPU kernel 到 Python 入口须经「四层接力」——__global__ kernel、host 启动函数（用 <<<grid,block>>> 启动 kernel）、PYBIND11_MODULE 注册、lib.xxx 调用，Python 调用的是启动函数而非 kernel 本身；③ 仓库有两种绑定范式：简单算子（如 relu）用 .cu 内联绑定 + load() JIT 即时编译，大型算子（如 hgemm/flash-attn）用独立 pybind/*.cc 集中绑定 + setup.py（CUDAExtension）预编译，并按优化路线再分 naive/、wmma/、mma/、wgmma/ 等子目录，体现「同一算子多版本对照」的学习方法论。引入的关键术语：git 子模块、三件套约定、四层接力、JIT 即时编译、CUDAExtension 预编译、优化路线。
