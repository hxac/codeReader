本讲下沉到 PTX 汇编层，用手写内联汇编实现 Ampere（sm_80+）Tensor Core 的 HGEMM TN kernel，包含四个最小模块。第一，介绍 MMA `m16n8k16` 指令：一条 warp 指令完成 C[16×8]=A[16×16]×B[16×8]+C[16×8]，是 Tensor Core 原生粒度，2 条它等价于 WMMA 的一条 `m16n16k16`；寄存器配比为 A 4 个、B 2 个、C/D 2 个 uint32，可由规模反推。引入关键术语：MMA、fragment、op(A)/op(B)、`row.col` 后缀。第二，TN 布局即 A row-major、B col-major（等价 Bᵀ row-major），使 `mma...row.col` 天然匹配，B 的 ldmatrix 无需 `.trans`；并讲清 ldmatrix 线程参与规则——一个 8×8 需 8 线程给行地址，故 x4 用 32 线程、x2 仅前 16 线程。第三，统一循环版 multistage：迭代 k 既算 tile k 又预取 tile k+K_STAGE-1，prefetch 预载 K_STAGE-1 个 stage 后省掉独立尾循环；warp 2×4 layout 按 col-major 排列 MMA0~MMA7，8 warp×4×4 VAL Tile 产 128×128 输出。第四，C fragment 排布：RC0/RC1 是 col-major 两个 8×8 子矩阵的相邻行（实际跨 8 行），epilogue 用 `__shfl_sync` 收集后由 lane%4==0 的线程做 128-bit store 写回。末尾预告 swizzle（`SWIZZLE_32B`）与 TMA 的联系，承接后续讲义。
