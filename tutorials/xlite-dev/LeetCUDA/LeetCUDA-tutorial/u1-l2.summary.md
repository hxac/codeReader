本讲讲解如何编译运行单文件自包含程序 notes-v2.cu。其一，无需构建系统，一条 nvcc 命令即可编译，关键参数含 -std=c++20、-O2、目标架构 -arch=sm_89（Ada）或 -gencode arch=compute_90a,code=sm_90a（Hopper）、-I 指向 CUTLASS 头文件、-lcublas 与 -lcuda 链接库。其二，引入两个条件编译宏：NOTES_V2_ENABLE_CUTE 门控 CuTe 段（依赖 CUTLASS 头文件），NOTES_V2_ENABLE_WGMMA 门控 Hopper 的 WGMMA/TMA 段（需 sm_90a 与 Driver API 的 cuInit），使一份源码适配不同 GPU。其三，main() 即 verification harness，逐个跑 kernel 与 CPU/cuBLAS 参考实现比对，求逐元素最大绝对误差 Max Err，按算子相关阈值判 PASS/FAIL。核心结论：阈值随精度放宽（转置 1e-6、f32 算子 1e-4~1e-2、f16 GEMM 1.0、FlashAttention 1e-1），「可接受」指 Max Err 远小于对应阈值而非为零；以无宏命令编译的输出表会比 README 示例少 HGEMM CuTe 与 HGEMM WGMMA 两行，属条件编译的预期结果。
