本讲是《LeetCUDA 学习手册》开篇，不涉及具体源码，只建立项目全景图。核心结论：LeetCUDA 面向「会用 PyTorch 但没写过 CUDA」的初学者，提供从 naive 实现一路优化到逼近官方库性能的可对照源码学习路径，其五大支柱为：CUDA Cores 与 Tensor Cores 双覆盖、TF32/FP16/BF16/FP8 多精度、200+ 个 kernel、手写 HGEMM 达到 cuBLAS 98%~100% TFLOPS、以及纯 MMA PTX 实现的 FlashAttention。引入的关键术语包括：kernel（核函数）、Tensor Cores/CUDA Cores、各类精度、算子大类（element-wise/reduce/softmax/norm/RoPE/GEMV/GEMM/attention）、「输入精度/累加精度/难度」三栏范式。学习分两条主线：按算子目录深入（README+.cu+.py 三件套）与按 `notes-v2.cu` 的 8 个 Phase 通读（Phase 0 为纯注释的面试框架速查，含 GPU 架构、内存层次、优化清单与 Roofline；Phase 1–8 含 kernel，按 naive→向量化→分块→Tensor Core→流水线→swizzle→warp specialization 递进）。贯穿全书的判断依据是算术强度 AI=FLOPs/Bytes：GEMM 属 compute-bound，GEMV/Softmax 属 memory-bound；优化本质是把数据尽量留在更快的存储层（寄存器>SMEM>L2>HBM）。正确性验证采用「与 PyTorch 参考实现比对 Max Err 是否小于算子相关阈值」的范式，不要求逐比特相同。
