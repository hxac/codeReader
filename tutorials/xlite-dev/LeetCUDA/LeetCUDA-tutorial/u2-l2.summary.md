本讲是 CUDA 性能优化第一性原理，承接线程层次。引入三组核心概念：一是**内存层次**——GPU 存储由远到近分为 HBM（global memory）、L2、Shared Memory（SMEM）、Register 四层，越近越快越小越贵，HBM 带宽最低延迟最高；优化总纲是「把热点数据搬到更快层复用，减少对最慢 HBM 的访问」。二是**算术强度（AI=FLOPs/Bytes）**，用以判断 kernel 是否 memory-bound。三是**合并访问（coalesced access）**：针对整个 warp 而非单线程，当 32 个线程访问连续、对齐的 128B segment 时合并为 1 次**内存事务**，否则最坏拆成 32 次；判定口诀是「相邻线程读相邻元素」。本讲用 elementwise_add、dot_prod 两个一维 kernel 验证：连续 idx 公式使 warp 内 idx 连续，故读 a/b、写 c 全部天然合并，已是合并访问范本；dot 的归约阶段在寄存器与 SMEM 中完成，不走 HBM。核心结论：这类 kernel 的 AI≈0.083–0.25，严重 memory-bound，已无改访问模式的空间，后续只能靠向量化（float4）减少访存指令数。
