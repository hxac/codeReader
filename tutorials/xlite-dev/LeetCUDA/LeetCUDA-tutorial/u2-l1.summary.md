本讲建立 CUDA 线程骨架地基，覆盖三个最小模块：线程层次与 SIMT 模型、全局索引与边界保护、elementwise 多输入推广。核心结论：GPU 用四级层次 grid→block→warp→thread 组织线程，warp 固定 32 个线程（WARP_SIZE=32），是 SIMT 最小不可分割执行单位；同一 warp 内 32 线程锁步执行同一条指令，故 kernel 函数体无循环，并行由硬件铺开海量线程体现，分歧分支会浪费算力。引入关键术语：kernel、host/device、launch、SM，内置变量 threadIdx/blockDim/blockIdx/gridDim，以及 branch divergence。灵魂公式 idx=blockIdx.x*blockDim.x+threadIdx.x 为每线程分配唯一且连续的下标；host 用向上取整 grid=(N+block-1)/block 保证线程数≥N，device 用 if(idx<N) 兜底越界线程。以 relu 与 elementwise_add 两个 naive kernel 为载体证明：所有一维 elementwise 算子共享同一线程骨架，改算子只需改核心计算那一行；block 尺寸属 host 端编排决策，影响性能而非正确性。
