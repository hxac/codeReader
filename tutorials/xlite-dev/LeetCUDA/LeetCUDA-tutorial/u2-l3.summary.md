本讲承接 u2-l2 合并访问，主题为向量化访存（vectorized memory access）。核心结论：向量化不改变总搬运字节数与算术强度 AI，只减少访存指令条数，省的是指令发射开销而非带宽；GPU 单条 load/store 指令上限 128 bit。关键术语：FLOAT4/HALF2/LDST128BITS 等向量宏均用 reinterpret_cast 把地址重解释为宽类型，其中 LDST128BITS 与 FLOAT4 定义相同但语义类型无关、固定 128 bit，是任意精度 pack 的通用工具。最小模块有三块：relu_f32x4 演示 float4 向量化（索引乘 p 对齐、block 缩为 256/n_elements、grid=ceil(N/256) 不变）；elementwise_add_f32x4 以「向量主体 idx+3<N + 标量尾部」两段式安全处理非对齐 N；relu_f16x8_pack 用 LDST128BITS 配合 half2 的 __hmax2 实现 SIMD-within-a-register，访存与计算同时最大化，预演后续 HGEMM/FlashAttention 的 pack 范式。另需注意 grid/block 的真相看 `<<<>>>` 而非上方注释；fp16 向量化相比 fp32 既省访存又省计算，性价比最高。
