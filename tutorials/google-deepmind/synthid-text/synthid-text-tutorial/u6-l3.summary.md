本讲收尾单元六，覆盖数据加工、训练入口、端到端打分三个模块。数据加工由 `process_raw_model_outputs` 与 `process_outputs_for_training` 完成：把参差不齐的水印正样本（用同一把 keys 生成）与非水印负样本（维基百科真实文本）划分 train/CV，逐分片截断、右填 eos、重算 g 值与掩码、左填充对齐、打标签并 shuffle，产出形状统一的六元组（g 值/掩码/标签）。要点：正负样本用不同截断长度（`pos/neg_truncation_length`），过短样本被整条丢弃；g 值与 `context_repetition_mask` 天然短 `ngram_len-1`，靠左填充补 0 与 `eos_token_mask` 对齐并被屏蔽；g 值/掩码用 torch 算、训练用 jax，以 `.cpu().numpy()` 桥接。`train_best_detector` 做「双重选择」：在 `l2_weights` 网格上每个候选独立训练，内部再由 `train` 选最优 epoch，留 CV loss 最低者。打分用包装类 `BayesianDetector.score(outputs)`（单参传 token 序列，内部重算 g 值与掩码），区别于模块级 `BayesianDetectorModule.score(g_values, mask)` 双参；`depth=len(keys)` 体现密钥定制。核心原则：文档与源码冲突时以源码为准。
