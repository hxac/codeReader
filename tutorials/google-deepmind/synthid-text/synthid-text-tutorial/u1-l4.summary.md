本讲是单元一的收尾，沿主线 Notebook 把「输入 prompt → 生成带水印文本 → 输出 [0,1] 检测分数」的端到端数据流串通，拆为三个最小模块：水印化生成、g 值与掩码重算、打分输出。核心节奏是「生成时埋水印、检测时重算」。生成阶段在 `model.generate` 内部由 `SynthIDLogitsProcessor` 完成，真正入口是带状态的 `watermarked_call`（标准 `__call__` 被禁用），它逐 token 计算 ngram keys→取哈希低位得到 g 值→偏置 scores，调用方只需把模型类换成带 `SynthID` 前缀的空子类（能力来自 Mixin）。检测阶段不信任生成时状态，只用输出 token 序列加密钥重算 `compute_g_values`，并用 `compute_eos_token_mask` 与 `compute_context_repetition_mask` 相乘得到 `combined_mask`，序列维统一对齐到 `output_len-(ngram_len-1)`。打分有两条路线：免训练的 `mean_score`/`weighted_mean_score`，与需按密钥单独训练的 `BayesianDetector`（真实训练入口是 `train_best_detector`，非 README 的 `optimize_model`）。关键术语：g 值（二进制指纹，`[batch, seq, depth]`，depth=层数）、掩码、watermarked_call、DEFAULT_WATERMARKING_CONFIG（`ngram_len=5` 对应论文 H=4）。PyTorch 施加与 JAX 检测通过 g 值衔接——「框架即分水岭」。
