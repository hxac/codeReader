本讲聚焦 SynthID Text 全项目最小却最常被调用的工具函数 `hashing_function.accumulate_hash`，它把一段 token 序列与水印密钥「搅拌」成一个 64 位整数，是后续 g 值计算、上下文去重与掩码生成的共同基础。本讲覆盖三个最小模块：其一，LCG 原理与参数——函数本质是改编自线性同余生成器（LCG）的混合函数，每步对 64 位状态执行「加数据 → 乘乘子 → 加增量」，默认乘子 `6364136223846793005`（`0x5851F42D4C957F2D`）与增量 `1` 取自 newlib/musl 参数，模数 \(2^{64}\) 由 int64 自然回绕提供；乘法前的「先加数据」一步使输出依赖于输入。其二，可累积性——`f(x, data) = f(f(x, 前缀), 后缀)`，因模运算成环而严格成立，使水印施加能「上下文只哈希一次、候选 token 续哈希一步」，并被 `torch.vmap` 并行复用。其三，非密码学安全——LCG 线性可逆，不提供密码学保证；水印保密性依赖密钥 `keys` 的保密与用 SHA-256 导出的不可预测初始向量 `hash_iv`。关键术语：LCG、乘子、增量、模数、可累积性、`hash_iv`。
