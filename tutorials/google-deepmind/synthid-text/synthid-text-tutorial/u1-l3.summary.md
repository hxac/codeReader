本讲建立 SynthID Text 的源码地图：全部代码位于 `src/synthid_text/` 下，共 9 个 `.py` 文件，目录下没有 `__init__.py`，依靠 PEP 420 命名空间包工作，统一以 `from synthid_text import` 形式导入，构建走 setuptools 的 src 布局自动发现。核心结论是**框架即分水岭**：水印施加侧用 PyTorch（`logits_processing.py` 负责水印内核与 g 值/掩码计算，`synthid_mixin.py` 提供接入 HuggingFace 的 Mixin），检测侧用 JAX/Flax（`detector_mean.py` 免训练打分、`detector_bayesian.py` 贝叶斯检测与训练）。判断文件归属应看顶部 import 而非文件名。两侧通过**g 值**（二进制指纹）衔接，其计算依赖公共工具 `hashing_function.py` 的 `accumulate_hash`（LCG），配合纯 Python 的 `g_value_expectations.py`；`detector_bayesian.py` 中出现 torch 仅用于预处理施加侧张量。测试以 `_test.py` 结尾、与被测模块同目录，由 pytest 收集；`torch_testing.py` 因被测试 import 而位于包内；项目主入口是 `notebooks/` 下的 Notebook。贯穿原则：**当文档与源码冲突时以源码为准**——README 中的 `train_detector_bayesian.optimize_model` 已不存在，真实训练入口是 `BayesianDetector.train_best_detector` 类方法。
