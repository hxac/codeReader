本讲介绍 MLC LLM 的**微服务（Microserving）部署模式**与**路由器（Router）**。路由器本身不做推理，而是编排层：它用 `PopenServer` 拉起 N 个完整的 `mlc_llm serve` 子进程，对外仅暴露一个 OpenAI 兼容入口 `/v1/completions`，再按调度策略把请求分发到各子进程。这与单一 serve「单进程内 prefill+decode 一体调度」有本质区别，旨在支撑横向负载均衡与 prefill-decode 分离式推理。

代码分三层：`cli/router.py`（薄壳 argv 解析）→ `interface/router.py` 的 `serve()`（组装 FastAPI+uvicorn）→ `router/router.py` 的 `Router` 类。关键扩展点是 `interface.serve(router_type=...)`，可注入 `Router` 子类并覆写 `translate_request` 实现自定义跨引擎编排；CLI 不暴露此参数，故自定义路由只能写 Python 脚本。

两种内置调度：`round-robin`（实为最少连接负载均衡，整请求转发到子进程的 `/v1/completions`）与 `disagg`（端点 0 恒为 prefill、其余为 decode）。分离式依赖三个 RISC 风格子请求原语 `prep_recv`/`remote_send`/`start_generate`，它们复用 completion 请求加 `DisaggConfig`，KV 地址借响应的 `usage.extra` 回传。两个工程要点：子进程必须**并发启动**（nvshmem world 需所有 GPU 同时在场）；decode 返回 `preempt` 时 `yield None`，触发外层 `while` 循环整体重跑。C++ 侧的 KV 跨机收发留待 u12-l3。
