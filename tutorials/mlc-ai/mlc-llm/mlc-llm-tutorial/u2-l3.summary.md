本讲承接 u2-l1，拆解 `mlc_llm` 三条运行/部署期子命令，它们均遵循「cli 入口解析参数 → interface 接口实现」两层结构，背后共用同一个 C++ 引擎。**chat** 是交互式终端 REPL：CLI 极薄，接口层固定创建 `JSONFFIEngine(mode="interactive")` 单并发，靠 `ChatState` 的前缀分发实现 `/set`、`/stats` 等特殊指令；其中 `/set`（覆盖 `ChatCompletionOverride` 生成参数，`;` 分隔且不可有空格）区别于启动期 `--overrides`（覆盖 `ModelConfigOverride` 模型结构参数）。**serve** 把模型封装为 OpenAI 兼容 HTTP 服务，编排 `AsyncMLCEngine` + `ServerContext` + FastAPI 路由 + uvicorn，靠 `--mode`（local/interactive/server）与 `EngineConfigOverride` 调整并发显存，注意 `context_window_size` 映射为引擎的 `max_single_sequence_length`。**package** 是面向 iphone/macabi/android 的部署期打包命令，经 `build_model_library`、`validate_model_lib`（合并静态库并按 `___tvm_ffi__library_bin` 符号校验）、按 device 构建 binding，产出移动端静态库与 `mlc-app-config.json`。
