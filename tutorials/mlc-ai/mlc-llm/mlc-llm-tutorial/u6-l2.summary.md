本讲回答了「字符串名如何变成 `Conversation` 对象」。核心有二：其一，`ConvTemplateRegistry` 是全局注册表（类变量字典），把模板名映射到 `Conversation`，提供 `register_conv_template`（默认防覆盖，`override=True` 才覆盖）与 `get_conv_template`（查不到返回 `None`、返回共享引用而非副本），与 `MODELS`/`QUANTIZATION`/`LOADER` 同构。其二，「导入即注册」：注册写成模块顶层语句，`__init__.py` 用 import 元组批量导入子模块，副作用即填表；新增模板等于新建 `.py` 并在元组加一行。消费方 `gen_config` 查表命中则序列化写入 `mlc-chat-config.json`，未命中则降级为自定义模板字符串。模板演化上，Llama2 用 `[INST]` 方括号、双分隔符、`add_role_after_system_message=False`；Llama3 引入 `<|start_header_id|>role<|end_header_id|>\n\n` 特殊 header/role token、单分隔符 `<|eot_id|>`、开关置 `True`；`llama-3_1` 补 tool 角色与 `<|eom_id|>`，`llama-4` 改名 header token。关键术语：注册表、导入即注册、特殊 header/role token、`add_role_after_system_message`。
