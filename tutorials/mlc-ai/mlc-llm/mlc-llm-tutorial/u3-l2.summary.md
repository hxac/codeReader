本讲以 Llama 为例，讲解如何用 Relax nn（tvm.relax.frontend.nn）这套「类 PyTorch」写法定义 MLC 模型。模型由两部分组成：Config（读 HF config.json，在 __post_init__ 里推断 head_dim、context_window_size、rope_theta 等派生字段）与一组层层嵌套的 nn.Module——顶层 LlamaForCausalLM 经 LlamaModel、LlamaDecoderLayer 下钻到 LlamaAttention 与 LlamaFFN，写出来的是计算图 IR 而非即时计算。注意力层做 QKV 融合（单个 qkv_proj）并支持 GQA，FFN 用 gate/up 融合的 SwiGLU，两者以 tp.ShardSingleDim 标注张量并行分片。关键设计：RoPE 不在 attention 模块显式出现，而是在 create_paged_kv_cache 时把 rope_theta/rope_scaling 写进 PagedKVCache，由 attention_with_fused_qkv 施加，以便编译器把 RoPE、attention、KV 写回融成单 kernel。顶层对外暴露一组阶段方法：embed/get_logits/prefill（只取最后 token）/decode（单步）/batch_verify/create_paged_kv_cache 等；get_default_spec() 作为张量接口契约，声明各方法的形状、类型与 param_mode，在 compile.py 的 export_tvm(spec=...) 处被消费，决定最终导出哪些 Relax 函数。上一讲 MODELS 注册表里的构造器由此落地为完整的配置、建图、阶段方法与 spec 契约闭环。
