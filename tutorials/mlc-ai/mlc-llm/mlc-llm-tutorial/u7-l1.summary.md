本讲打开 `mlc_llm compile` 接口层黑盒。`compile()` 是薄编排层：解析 `mlc-chat-config.json` 成 `model_config`、装进 `CompileArgs` 信封、转发给 `_compile`，CLI 与 Python/JIT 共用此路径。`CompileArgs.__post_init__` 调 `opt.update(target, quantization)`，把 `--opt` 按 target 矫正为实际生效项（如 flashinfer 需 CUDA≥80）。`_compile` 五步：建模+量化、导出 Relax IRModule、preprocs 参数预处理、组装 metadata/pass_config、`build_func` 挂 `mlc_llm` pass 流水线导出。核心是 preprocs「编译期记配方、运行期真切」：为带 `shard_strategy` 且 `tensor_parallel_shards>1` 的参数，用 `gen_shard_info` 写配方、`gen_tir` 生成分片 TIR，配方进 metadata、TIR 进 IRModule，运行期由 C++ `multi_gpu_loader` 现场切分并 scatter 给各卡。`ShardSingleDim` 是 preprocs 与 convert_weight preshard 的共享底座，前者只记配方、后者真切落盘，二者须读同一 `tensor_parallel_shards`。`build_func` 由 `--device` 决定，封装「IRModule → 平台产物（`.so`/`.tar`/`.wasm`）」的最后一跳；metadata 是编译期与运行期的共享契约。各 pass 算法细节留待 U8。
