本讲是 MLC LLM 量化体系的入口，只讲「注册表 + 接口 + 工厂」三层抽象，不展开具体算法。核心模块有三：其一，`QUANTIZATION` 全局注册表把量化名字符串（如 `q4f16_1`）映射到携带完整参数的配置对象，命名编码了权重位宽（`q4`=int4）、计算精度（`f16`）与变体编号（`_1` 对应 NK 布局）。其二，`Quantization` 对象遵循统一接口（鸭子类型）：`name`、`kind`，以及两个对立方法——`quantize_model` 负责图重写（把 `nn.Linear` 等替换成量化层并填充 `QuantizeMapping`），在 `convert_weight` 与 `compile` 两处执行；`quantize_weight` 负责数值压缩，仅在校重载阶段按 shape 缓存回调。其三，`make_quantization_functions` 工厂用 `supports_*` 能力开关为每个模型类生产以 `kind` 为键的 `Model.quantize` 字典。关键术语 `kind`（量化家族，如 `group-quant`）是连接「名字查配置」与「`kind` 查函数」两级查表的桥。下讲将深入 `quantize_weight` 的张量布局。
