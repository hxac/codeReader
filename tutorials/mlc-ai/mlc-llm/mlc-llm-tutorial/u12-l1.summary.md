本讲讲清 MLC LLM 张量并行（TP）的「声明→切分→跨卡分发」完整链路。模型代码只按单卡形状写一遍，在权重上挂 `shard_strategy` 标签（数据类 `ShardSingleDim(name, dim, segs)`）即声明分片：`dim=0` 切输出维（qkv/gate_up，column-parallel，配 `segs` 按段均分以保 GQA 比例），`dim=1` 切输入维（o_proj/down_proj，row-parallel，需 all-reduce）。`gen_tir` 用 reshape+transpose 生成切分 TIR，`gen_shard_info` 生成运行期配方。切分执行有两条互斥路径：路 A 编译期 preprocs（默认，写进 model lib，运行期现场切，加载器 `LoadMultiGPU` 由 worker 0 切后 `ScatterFromWorker0`）；路 B 转换期 preshard（`MLC_INTERNAL_PRESHARD_NUM` 触发，提前切好落盘为 `param_shard-{i}`，`LoadMultiGPUPresharded` 各取一份），两端切数必须一致。运行期跨卡协同由 disco 分布式会话承担：`CreateDiscoSession` 按 `num_workers=num_shards*num_stages` 拉起 `ProcessSession`/`SocketSession` 并 `InitCCL`，worker 进程入口统一为 `mlc_llm.cli.worker`。核心结论：切分知识集中于一处，执行被编译期/加载器吸收，分布式协同交给 disco，故无需手写一行分布式代码。
