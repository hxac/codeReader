本讲讲解 MLC LLM 的分离式推理（disagg / PD 分离）：把一次推理的 prefill 与 decode 拆给 P、D 两个实例，用 NVSHMEM 把算好的 KV cache 从 P 跨机搬到 D，使算力相与带宽相独立扩缩容、消除长 prefill 对 decode 的阻塞、提升显存利用。关键术语有三类特殊请求枚举 `DisaggRequestKind`（`kPrepareReceive`/`kRemoteSend`/`kStartGeneration`）与贯穿三段的元数据信封 `DisaggConfig`（含 `kv_window_begin/end`、`kv_append_metadata`、`dst_group_offset`）。核心源码：P 端 `DisaggRemoteSend` 复用普通 prefill，在 `AddNewSequence` 后调 `DisaggMarkKVSend` 打发送标记、不采样、显式 `StreamSync`；D 端 `DisaggPrepareReceive` 不跑前向，只调 `DisaggPrepareKVRecv` 预留页面并把地址经 `usage.extra` 回传。两动作被插在动作表最前，按 kind 从 `waiting_queue` 分流。底层 disco 会话在引擎启动时集合初始化 NVSHMEM world，模型层经 `FunctionTable` 转发到 `vm.builtin.kv_cache_disagg_*` 全局函数完成 RDMA 传输；Python `Router` 编排 prep_recv→remote_send→start_generate 三步，HTTP 传控制面元数据、NVSHMEM 传 KV 数据面，二者分离。
