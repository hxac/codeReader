本讲回答「IRModule 到模型库之间编译器做了什么」，打开 MLC LLM 的编译流水线 `_mlc_llm_pipeline`。它用 `@register_pipeline("mlc_llm")` 注册，由 `compile.py` 经 `relax.get_pipeline("mlc_llm", ...)` 装配、在 `PassContext` 中驱动；装配期参数（flashinfer/cublas_gemm/metadata 等）塑形流水线，运行期开关（use_cuda_graph）控制内置 pass。整条 `Sequential` 划为五阶段：Phase 0 附加元数据与运行时函数并 `BindTarget`；Phase 1 高层算子图融合与派发（FuseAddRMSNorm、BLASDispatch 等）；Phase 2 复用 TVM 官方 zero 流水线把 Relax 降级到 TIR；Phase 3 做 TIR 级融合与死代码清理；Phase 4 做底层调度、`StaticPlanBlockMemory` 内存规划、`RewriteCUDAGraph` 及 VM 字节码降级。各 pass 遵循 `@tvm.transform.module_pass` 统一外形，靠「外层三元/内部读 target 早退/`OptimizationFlags.update` 矫正」三层机制实现 target 自适应；`_LogProgress`/`_DebugDump` 为不改图的哑 pass。关键术语：IRModule、Pass、PassContext、target、五阶段、`OptimizationFlags` 与 `OPT_FLAG_PRESET`。本讲只讲编排与设计意图，单个 pass 算法留待 U8 展开。
