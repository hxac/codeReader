本讲将推测解码落地为可插拔的 Action 链：用廉价草稿模型先猜 K 个 token，再让大模型一次前馈打包校验，以访存受限下的空闲算力换吞吐。校验用拒绝采样（接受概率 min(1,p/q)，拒绝则从残差分布补采）保证输出分布无偏。引擎支持 small_draft（独立小模型链式起草）、eagle（吃大模型 hidden states、共享 lm_head、verify 尾部前瞻起草）、medusa 三种模式，由 CreateEngineActions 按 speculative_mode 装配不同动作链。起草动作产出 draft_output_tokens 并经 ScatterDraftProbs 把概率写入 DraftTokenWorkspaceManager 跨动作 GPU 槽位池；校验动作 GatherDraftProbs 取回做拒绝采样，接受 token 经 CommitToken（逻辑）与 CommitAcceptedTokenTreeNodesToKVCache（物理）两步落进分页 KV cache，拒绝部分用 PopNFromKVCache 回退。AutoSpecDecode 作为「动作套动作」外壳，按并发批量查表决定是否推测，受 effective_batch 不超 max_num_sequence 的安全闸约束。
