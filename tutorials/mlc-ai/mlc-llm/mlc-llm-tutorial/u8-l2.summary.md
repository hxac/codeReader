本讲讲解 MLC 编译流水线里的派发 pass：派发即在编译期把模型代码中的占位/通用调用改写为某具体后端实现，从而解耦模型定义与后端选择。重点剖析三个 pass：BLASDispatch 用声明式 FuseOpsByPattern 把未量化或 FP8 的大 GEMM 派发给 cuBLAS/hipBLAS，并主动排除单 batch decode；DispatchTritonKernel 用手动 PyExprMutator 拦截 mlc.triton.* 占位，按形状即时生成去重的 Triton TIR kernel，仅 cuda 生效；DispatchKVCacheCreation 用 BlockBuilder 把 KV cache 创建占位展开为「TIR 必有 + FlashInfer 条件生成」两个实现，并把维度写进 metadata，因此位于 Phase 0，而前两者在 Phase 1。三者都靠 target 守卫加 OptimizationFlags 矫正实现优雅降级。本讲引入术语：占位调用、external_mods、metadata、FuseOpsByPattern、PyExprMutator、call_tir。
