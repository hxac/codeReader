本讲围绕 Python↔C++ FFI 桥，讲解配置信封、引擎封装、服务上下文三块设施。

EngineConfig 是贯穿引擎生命周期的纯数据信封，字段分模型定位、并发上下文、显存分页、高级特性、并行五组，经 asjson/from_json 跨 FFI 往返；三档 mode（local/interactive/server）仅填充未显式指定字段的默认值，显式值优先。_check_engine_config 是进 C++ 前的一致性闸门，规则为「None 表示不指定、由对方决定」，并硬约束 kv_cache_page_size 必须为 16；真正覆盖发生在 MLCEngineBase 构造函数：构造参数覆盖 engine_config 同名字段后经 reload 下发用户意图，再由 get_complete_engine_config 回读 C++ 在显存预算内推断完整的配置；detect_device("auto") 按 cuda→rocm→metal→vulkan→opencl→cpu 优先级探测设备。

三种引擎以「谁驱动 Step」分野：AsyncMLCEngine/MLCEngine 包 ThreadedEngine、由后台线程自动驱动并提供 OpenAI API；SyncMLCEngine 包裸 Engine、调用者手动 while:step()，无后台线程、无 OpenAI API，仅供测试调试。

ServerContext 是进程级单例与多模型注册中心，靠 __enter__/__exit__ 管理 chat 与 embedding 引擎的注册、查询与退出时统一 terminate，current() 供端点取上下文，单引擎托管时 get_engine 贴心地忽略请求里的 model 名。
