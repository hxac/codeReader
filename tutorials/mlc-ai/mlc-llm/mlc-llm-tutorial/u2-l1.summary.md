本讲剖析 `mlc_llm` 命令行总入口，覆盖三个最小模块：入口点注册、子命令分发、cli/interface 分层。

`mlc_llm` 并非可执行程序，而是由 `setup.py` 的 `console_scripts`（规则 `mlc_llm = mlc_llm.__main__:main`）注册、pip 生成的包装脚本；`python -m mlc_llm` 也进入同一 `main()`。

分发采用「两段式解析」：`__main__.py` 仅用 `sys.argv[1:2]` 取子命令名（compile 等，共 compile/convert_weight/gen_config/chat/serve/package/calibrate/router 八个），再把 `sys.argv[2:]` 交给对应模块；映射靠 `if/elif` 加「懒加载」（import 写在分支内），避免无关子命令被 TVM 等重型依赖拖慢，故弃用 `add_subparsers`。

架构分两层：`cli/` 是命令行入口层，做 argv 解析与 `detect_*` 自动探测（把字符串/auto 翻译成结构化对象）；`interface/` 是 Python 接口层，含真正实现（如 `CompileArgs` 与 `compile()`），两层一一对应，使同一套实现既能被命令行也能被 Python 代码直接调用。
