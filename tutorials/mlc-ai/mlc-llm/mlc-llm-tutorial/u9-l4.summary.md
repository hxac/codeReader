本讲打开引擎的「计算黑盒」，含三个最小模块。**ModelObj** 是 C++ 引擎看到的「一台模型」——一层纯虚接口，按模型计算、KV cache 管理、信息查询、工具四类组织能力，其实现 **ModelImpl** 持有 FunctionTable 把每个虚方法翻译为函数句柄调用，并用 **ModelWorkspace** 复用动作间的共享张量。**FunctionTable** 是「model lib 函数名→可调用句柄」的查表容器：Init 把 `.so`/`.tar` 加载成 TVM VM，`_InitFunctions` 按名字符串解析出计算、采样、KV cache 三类函数（必需用 ICHECK、可选用 `value_or` 容错、GPU 采样需先过 `SupportGPUSampler`），单卡走本地 VM、多卡走 disco 会话广播。核心结论是**「三层同名」契约**：C++ 方法→FunctionTable 字段→model lib 函数名字符串，这条名字符串正是 Python 编译器与 C++ 引擎之间的胶水。**EngineConfig**（`EngineConfigNode`）以 `gpu_memory_utilization` 驱动 `InferrableEngineConfig::InferForKVCache` 反推容量上限，把 page_size、max_num_sequence 等 6 个字段透传给 CreateKVCache；后者按 `ModelMetadata` 的 **kv_state_kind**（`kKVCache`/`kRNNState`/`kHybrid`/`kNone`）分四种建法，RNN 与 Hybrid 需为前缀缓存回收序列额外预留槽位。承接术语：packed function、model lib、disco、`SessionFuncAsPackedFunc`、编译期↔运行期名字符串契约。
