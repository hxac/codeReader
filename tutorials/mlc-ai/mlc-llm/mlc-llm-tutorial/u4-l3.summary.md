本讲打开 `mlc_llm convert_weight` 接口层黑盒，把 u4-l1（Loader）与 u4-l2（ExternMapping/QuantizeMapping）串成一条完整流水线：定位权重（`detect_weight` 自动探测 torch/safetensor）→ 解析配置并建图拿 MLC 期望参数 → 加载+映射+量化 → 可选预分片 → 流式落盘。核心是 `ConversionArgs` 信封与 `_convert_args` 主流程，全程用 `_param_generator` 生成器驱动以压低峰值内存，`_check_param` 前置校验并将已匹配项从 `named_params` 删除，从而精确定位缺失参数。

关键概念：**preshard（预分片）**由环境变量 `MLC_INTERNAL_PRESHARD_NUM` 触发，把带 `shard_strategy` 的参数（如 Llama 的 qkv/o/gate_up/down_proj）在转换期切成 N 份落盘（`_shard-0…_{N-1}`），运行期用 `LoadMultiGPUPresharded` 直接各取一份，启动快但绑定卡数；不设则走 `LoadMultiGPU` 现场切。`ShardSingleDim.gen_tir` 用 reshape+transpose 让 head 均匀交错分布，`apply_preshard` 将其包成 Relax 函数（call_tir→split→squeeze）编译成 VM，由 loader 在 yield 前调用。它与编译期 `preprocs` 共享 `ShardSingleDim`，区别是 preshard 真切、preprocs 只记配方待运行时切；两端读同一环境变量须一致。

权重以 `tvmjs` ndarray cache 跨平台存储：`params_shard_*.bin` 分片 + `tensor-cache.json` 索引 + 元数据（`ParamSize/ParamBytes/BitsPerParam`，后者衡量量化效果），`encode_format="f32-to-bf16"` 仅压缩 fp32。加载顺序在 `loader.load` 内嵌套：ExternMapping 取组合 → QuantizeMapping 量化（一生多）→ preshard_funcs 切分（一生 N）。
