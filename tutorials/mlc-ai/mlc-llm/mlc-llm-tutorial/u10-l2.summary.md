本讲讲解 KV cache 之上的「前缀缓存」层：让共享前缀（如 system prompt）的请求复用已算好的 KV、省去重复 prefill，底层用基数树（radix tree）按 token 前缀索引。抽象接口 `PrefixCacheObj` 有 `PrefixCacheImpl`（默认）与 `NoPrefixCache`（禁用）两个实现，它只管 token 拓扑，物理 KV 由构造时注入的 `remove_callback` 联动释放。`PagedRadixTree` 用固定大小 page（容量 64）+ 循环缓冲 + 左孩子右兄弟存序列，`ForkSequence` 是写时复制——只挂 seq_id 不拷 token，故 O(1) 且不占 KV 预算。`InsertSequence` 走「匹配 → 复用 → 分叉」三段决策，先 pop_back 保证至少 prefill 一个 token，再按「最短回收序列 + 0.9 阈值」接管或 fork 活跃序列，返回 `PrefixCacheMatchedResult` 四字段（prefilled_offset、forked/reused_seq_id 等）。请求结束后 KV 经 `RecycleSequence(lazy=true)` 进 LRU 队列，显存紧张时 `TryFreeMemory` 按最旧优先驱逐、且优先于抢占运行请求；`ExtendSequence` 为 lazy commit，由 `CommitSequenceExtention` 批量落树。
