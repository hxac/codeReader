本讲拆解 MLC LLM 中 Python↔C++ 的两条跨界「桥」：`JSONFFIEngine`（字符串桥，chat CLI 默认，仅支持流式）以 JSON 字符串为唯一跨界载荷，拼 prompt、分词、去分词、序列化全在 C++ 侧完成；`MLCEngine`/`AsyncMLCEngine`（类型化对象桥，Python 主推 API，REST 服务器底层）直接传 TVM 结构化对象，分词/去分词与 OpenAI 响应组装在 Python 侧完成。两桥最终都创建同一个 C++ `ThreadedEngine`，靠 PackedFunc 按名字符串取函数互通。`ThreadedEngine` 跑两个后台循环：`RunBackgroundLoop` 跑引擎心跳 `Step()` 产 delta，`RunBackgroundStreamBackLoop` 把 delta 批量拍平后经 `request_stream_callback_` 回送 Python，二者解耦使生成与回送并行；回送链跨越两个线程边界与两个缓冲队列。同步 `MLCEngine` 用 `queue.Queue` 阻塞 `get()`，异步 `AsyncMLCEngine` 用 `AsyncRequestStream` 配合 `call_soon_threadsafe` 线程安全投递，契合 FastAPI 异步服务。关键术语：JSON FFI、PackedFunc、ThreadedEngine、后台双循环、契约字符串、`process_chat_completion_request`、`CreatePrompt`/`GetResponseFromStreamOutput`。
