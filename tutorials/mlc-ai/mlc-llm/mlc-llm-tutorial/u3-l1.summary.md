本讲讲解 MLC LLM 的 Model 注册表。核心是用全局字典 `MODELS`（注册表模式）将「架构名」绑定到 `Model` dataclass 实例——一个装满线索的「信封」，打包七个字段：`name`（架构名）、`config`（存配置类本身，带 `from_file` 类方法，运行时按 config.json 求值）、`model`（吃配置吐 `nn.Module` 的构造器）、`source`（以来源格式为 key，返回 `ExternMapping` 参数改名规则）、`quantize`（以量化 kind 为 key，返回量化模型与 `QuantizeMapping`）、`model_task`（chat/embedding）、`embedding_metadata`。`detect_model_type` 把 `--model-type`（或 auto 推断）解析成 `Model` 实例并校验合法性。引入的关键术语与概念：注册表模式、`EmbeddingMetadata`（编码器/解码器、池化策略、归一化）、独立的 `QUANTIZATION` 注册表及衔接二者的 `Quantization.kind`、工厂函数 `make_quantization_functions` 的 `supports_*` 开关。核心结论：拿到 `MODELS[model_type]` 这个信封即获得配置、加载、量化、任务类型的全部入口，后续 convert_weight/compile 的分支选择源头都在此；`__post_init__` 在导入期校验 `model_task` 与 `embedding_metadata` 一致性。
