本讲是纯阅读理解型讲义，为 MLC LLM 仓库建立「目录地图」，覆盖三个最小模块：顶层目录职责映射、Python 包与 C++ 库的边界、第三方依赖。核心结论：顶层目录可按编译、引擎、部署、文档、依赖、工程六类贴标签——`python/` 同时承载编译器、CLI 与引擎封装，`cpp/` 是高性能推理引擎，`android/ios/web` 是各平台部署，`3rdparty/` 是依赖地基。关键边界判断：编译期逻辑（model/quantization/compiler_pass/loader/interface）几乎全在 Python 侧，运行期快路径在 C++ 侧（`cpp/serve/`），两者经 **JSON FFI** 衔接——Python 把请求序列化成 JSON 调进 C++，`MLCEngine` 只是这层边界的薄封装。引入的关键术语：编译期/运行期、JSON FFI、MLCEngine、console_scripts 入口点、scikit-build-core（在 pip install 时调 CMake 一并构建 C++ 产物 `libmlc_llm.so`）、git submodule。三大依赖地基：`tvm`（编译能力来源，用的是 `mlc-ai/relax` 定制 fork）、`tokenizers-cpp`（文本↔token 转换）、`xgrammar`（结构化约束解码）。推理引擎心脏在 `cpp/serve/threaded_engine.cc`，JSON 桥在 `cpp/json_ffi/`。
