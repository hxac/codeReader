本讲是 U12「扩展部署与工程化」收官，覆盖四个模块：多端部署、package 打包、bench 基准测试、tests 测试体系。核心结论是「多端 = 同一产物换外壳」：模型权重跨平台共享，模型库按平台专用（Android/iOS 用 `.tar/.a`、Web 用 `.wasm`），外壳分别套 Java(mlc4j)、Swift(MLCSwift)、TypeScript(WebLLM)，三端对外均为 OpenAI 兼容 API。`mlc_llm package` 读 `mlc-package-config.json`，经 build_model_library（按需 JIT）→ validate_model_lib（合并成单静态库并按 `___tvm_ffi__library_bin` 符号校验）→ 按 `device`（合法值 iphone/macabi/android）分发。bench 是服务端基准测试客户端，支持 7 种数据集、6 种后端、固定并发/泊松到达/回放三种模式。测试以 pytest 为主、按子系统分目录，用 5 类 marker 区分是否需 GPU，C++ 功能多经 TVM FFI 暴露给 Python 测。关键术语：MLCEngine、ModelRecord、`required_features`、`system_lib_prefix`、JIT 兜底。
