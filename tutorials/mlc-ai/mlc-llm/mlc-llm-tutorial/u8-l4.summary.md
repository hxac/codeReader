本讲是 U8 编译优化 pass 单元收官篇，精读三个面向运行期、分处不同层级、可正交叠加的 pass。`LowBatchGemvSpecialize`（TIR 调度层）服务 decode：小 batch 权重乘算术强度极低、属访存受限，宜用 GEMV 调度；编译期不预测 batch，而用 Dlight 的 `LowBatchGEMV` 与通用 `Matmul` 生成多版本，按阈值（≤2/≤8/else）用 `IfThenElse` 派发，并以 `is_scheduled` 防被通用调度覆盖。`AttachMetadataWithMemoryUsage`（Relax 元数据层）须在 `StaticPlanBlockMemory` 之后，遍历各函数 `alloc_tensor`/`alloc_storage` 累加字节，符号维度保守跳过，写入返回 JSON 的 `_metadata`；运行期取各函数临时 workspace 最大值 ×2 得 `temp_buffer_bytes`，配合权重反推 KV cache 容量。`PipelineParallelRewrite`（Relax 结构层）对 `pipeline_parallel_stages>1` 的函数，把 `pipeline_stage_boundary` 当剪刀，剪成 N 个 stage 子函数，接缝插入 disco 的 `recv_from_prev_group`/`send_to_next_group`，入口改为按 group id 派发的 `DispatchFunctionByGroup`。三者共现「编译期为运行期服务」。
