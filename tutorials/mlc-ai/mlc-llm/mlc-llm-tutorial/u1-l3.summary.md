本讲把前两讲的「读」转化为「跑」，回答「如何安装 mlc_llm 并最快地与真实大模型对话一次」。核心内容：一是两种安装方式——预编译 wheel（快）与源码编译（灵活），验证分 Python 层（`import mlc_llm`）和 C++ 层（找到 `libmlc_llm.so`），由 `libinfo.py` 跨平台定位动态库。关键结论：跑模型只需随包附带的 TVM runtime，无需完整 TVM 编译器，后者仅用于编译自己的模型。二是三种运行入口对应三种 mode：chat CLI（`interactive`，走 `JSONFFIEngine`）、Python `MLCEngine`（`local`，走 `MLCEngineBase`）、REST serve（`server`，`AsyncMLCEngine`+FastAPI）。关键术语：`HF://` 前缀、JSON FFI 桥、两扇「门」（`CreateJSONFFIEngine` 与 `create_threaded_engine` 最终都创建同一 C++ ThreadedEngine）、速度统计同源于 `usage.extra` 的 `prefill/decode_tokens_per_s`。
