本讲把 u11-l1 建立的 Python↔C++ JSON FFI 桥封装为对外暴露 HTTP 的 REST 服务器，是协议层（u6-l3）与引擎层（u11-l1）之间的 HTTP 适配层，本身不实现推理逻辑。核心结论：`mlc_llm serve` 的启动编排集中在单个 `serve()` 函数——构造 `AsyncMLCEngine`、进入 `ServerContext` 登记全局单例、组装 FastAPI app、注册异常处理器、最后 `uvicorn.run` 阻塞服务。引入的关键术语：`ServerContext`（进程级全局单例兼多模型注册中心，端点靠 `current()` 取上下文、`get_engine(model)` 取引擎，单引擎时忽略请求 model 字段）；四个 OpenAI 兼容端点（`/v1/chat/completions`、`/v1/completions`、`/v1/embeddings`、`/v1/models`）；两段式 async generator（手动 `await anext` 拉第一块以在端点作用域捕获早期异常，再用嵌套 generator 包装成 SSE 文本流）；以及三层横切关注点机制——CORS 中间件、`Depends` 注入的 API key 鉴权、`BadRequestError` 异常处理器。
