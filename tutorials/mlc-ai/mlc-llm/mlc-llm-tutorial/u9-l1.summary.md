本讲读 cpp/serve 下三模块建立 C++ 引擎心智模型：ThreadedEngine 是线程安全外壳，EngineImpl 是跑模型的内核，EngineState 是二者共享的可变状态。Engine 抽象接口分引擎管理、请求管理、动作循环三类；Step 是引擎心跳，依次询问按优先级排序的动作表 actions_，命中首个有事的动作即执行并返回。ThreadedEngine 用「单消费者后台线程 + 加锁指令队列」把非线程安全的 Engine 包成多线程服务入口——前台仅投递指令，Step 由后台线程驱动；生成结果先入送回队列、再由独立线程回调用户，避免阻塞主循环。EngineState 含等待/运行两级队列、每请求细粒度状态、内部 id 管理器、前缀缓存与 metrics；状态声明为 mutable，只能由唯一后台线程访问。
