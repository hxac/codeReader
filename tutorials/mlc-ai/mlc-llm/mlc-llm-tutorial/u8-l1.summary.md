本讲深入 MLC 编译流水线 Phase 1/Phase 3 的算子融合类 pass。融合收益有二：针对 memory-bound 算子减少显存往返、针对 decode 阶段小算子减少 kernel 启动开销，以算术强度（运算量/访存字节）衡量。涉及四个 pass：FuseAddRMSNorm 以手动 PyExprMutator 注入手写带 warp 归约的 kernel，把 rms_norm(add) 融合，并用 set_var_remap 把残差第二输出重绑给下游；FuseDequantizeMatmulEwise 以声明式 FuseOpsByPattern 参数化扫描，靠 PrimFunc 命名前缀（dequantize/matmul/NT_matmul）恢复算子语义；FuseTransposeMatmul 把 matmul(permute_dims) 改写成 NT_matmul，转置烘进读取索引；FuseDequantizeTranspose 折叠 dequantize 内部 T_transpose 与外层 permute_dims 的冗余转置。关键承接概念：两种改图机制（声明式 vs 手动 mutator）、call_tir/PrimFunc、NT_matmul 跨 pass 命名接力、tirx 新版 TIR DSL。
