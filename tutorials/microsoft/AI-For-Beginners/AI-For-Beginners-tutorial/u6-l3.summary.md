本讲讲解如何用 Docsify 把全 Markdown 课程仓库变成在线文档站点，覆盖三个最小模块。Docsify 是「运行时文档生成器」，与 Jekyll/Hugo 等静态站点生成器的本质区别是零构建：浏览器加载唯一的 index.html 后，用 fetch 现取 Markdown 经 marked 渲染为纯前端 SPA；故站点必须跑在 HTTP 服务（如 python -m http.server）下，file:// 会被跨域拦截，默认入口是各目录的 README.md。其行为集中在 index.html 的 window.$docsify 配置，仅 5 字段：name（站名）、repo（GitHub 角标）、relativePath:true（按当前文档目录解析相对链接，适配深层嵌套）、auto2top:true（切换回顶）、routerMode:'history'（干净 URL，依赖服务器回退 index.html，与 u6-l1 同源）。部署靠零字节 .nojekyll 让 GitHub Pages 跳过 Jekyll、原样发布文件，部署配置在网页 Settings 而非源码。最小可运行组合为 index.html + .nojekyll + README.md。
