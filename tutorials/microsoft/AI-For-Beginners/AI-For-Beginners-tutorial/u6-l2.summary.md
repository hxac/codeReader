本讲补全测验子系统的生成端，贯通"源文本→脚本→JSON→前端渲染"全闭环，含三个最小模块。其一，`questions-en.txt` 是唯一内容源（single source of truth），以行首标记法书写：`Lesson` 开新测验、`*` 题干、`+` 正确选项、`-` 错误选项，后缀 `B`/`E`（Before/End）区分课前/课后。其二，`qzmkjson.py` 是状态机式转换器：逐行解析，用 `mk_id` 把 `B`/`E` 换算成 100/200 段数字 id（课前 101–124、课后 201–224），用 `deepcopy` 模板按课号归并课前课后，生成 24 份 `lesson-N.json` 与聚合用的 `index.js`。其三，`index.js` 按 0 基下标聚合、`translations/index.js` 按 locale 分发，`Home.vue` 与 `Quiz.vue` 双层 `v-for` 遍历并按 `quiz.id` 匹配路由，故 id 即 URL。核心结论：`quiz.id` 是贯穿全链路的钥匙；源格式、脚本、`Quiz.vue` 的 `nextQuestion < 3` 三方共持"每套测验 3 题"的隐式契约。另需注意脚本写死相对路径（须从 `etc/quiz-src/` 内运行）与未使用的 matplotlib 死依赖。本流水线仅管英文，多语言留待后续讲义。
