本讲是「配套工具与维护机制」单元首篇，转向支撑课程运转的工具链，精读 `etc/quiz-app/`——一个负责渲染每课课前/课后测验的 **Vue 2 单页应用（SPA）**。覆盖三个最小模块：项目结构（`package.json` 声明 `vue`、`vue-router`、`vue-i18n` 三件套依赖与 `serve`/`build`/`lint` 脚本，由 `@vue/cli-service` 打包）、路由与启动链路（`index.html` 的 `<div id="app">` → `main.js` 用 `new Vue({ i18n, router, render }).$mount('#app')` 挂载根组件 `App.vue` → `<router-view>` 按路由表渲染），以及测验组件与多语言机制（`Quiz.vue` 用两层 `v-for` 加 `v-if="route == quiz.id"` 过滤出当前测验，`handleAnswerClick` 实现答对前进、答错重试）。核心结论有三：其一，路由含 `/`、`/quiz/:id`（动态参数）与通配 404 三条，`mode:'history'` 让 URL 干净但部署需 `routes.json` 回退；其二，每套测验硬编码 3 题，`nextQuestion<3` 是隐式契约，题数不符会出 bug；其三，多语言**未**用 `vue-i18n` 的 `$t()`，而是手动按 `locale` 键索引嵌套题库 `messages[locale][课号].quizzes[套].quiz[题]`，`vue-i18n` 仅被借用为全局响应式 locale 状态，由 `App.vue` 下拉框与 `watch` 写入 `$root.$i18n.locale`，`Quiz.vue` 经 `currLocale` 计算属性读取响应。当前 `en` 完整（24 课）、`es` 仅第 1 课。引入的关键术语：**单文件组件（SFC）、SPA、路由、动态路由参数、history 模式与部署回退、i18n、手动语言索引与全局 locale 状态**，供后续讲义承接。
