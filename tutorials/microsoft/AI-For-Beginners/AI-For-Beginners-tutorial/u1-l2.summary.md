本讲是仓库的「目录地图」，含三个最小模块。其一，lessons/ 采用两级编号：一级「数字-主题」大单元对应 README 的罗马数字（如 4-ComputerVision 即第 IV 单元），二级「两位数字-主题」为具体小课程，数字前缀即学习顺序，文件系统自动排序；另有 0-course-setup 与 X-Extras 两个特殊目录。其二，单课文件约定：典型课含 README.md 讲义、.ipynb 可执行 Notebook（常分 PyTorch/TensorFlow 双版本）、可选的 lab/ 代码实验或 assignment.md 写作作业及 images/ 配图，并非每课齐备。其三，顶层辅助目录各司其职：examples/ 新手示例、etc/ 测验应用与题库、translations/ 多语言镜像、data/ 数据集、binder/ 云端运行等。核心结论：找东西口诀为「学课程→lessons、跑示例→examples、看测验→etc、要中文→translations」；translations 体积巨大是克隆慢的主因，可用 sparse-checkout 跳过。
