本讲聚焦 `scipy/io/_idl.py`，讲解 `scipy.io.readsav` 如何**只读**解析 IDL 的 `.sav` 二进制文件。IDL `.sav` 是一条 record 流：以 `SR` 签名 + 2 字节 recfmt 开头，每个 record 自带「下一个 record 的 64 位绝对偏移」，靠 `f.seek(nextrec)` 跳转读取；压缩文件（recfmt `0x06`）需逐 record 解压到临时文件并重写偏移。`readsav` 主流程为读 record 列表 → 收集 `HEAP_DATA` 成堆字典 → 对每个 `VARIABLE` 调 `_replace_heap` → 装进容器。关键术语包括：record/rectype、typedesc（类型描述符，用位标志区分标量/数组/结构，`&4` 为数组、`&32` 为结构）、heap（堆）、VARSTART 魔数 7。数据重建由 `_read_data`/`_read_array`/`_read_structure` 完成，紧凑数值用 `np.frombuffer`，16 位类型非紧凑存储（读 2 倍字节取 `[1::2]`），多维数组 `dims.reverse()` 把列主序转行主序，结构体产出 NumPy `recarray` 而非 dict。指针机制为两趟设计：`Pointer`/`ObjectPointer` 只存堆索引占位符，由堆字典替换，使指向同一索引的两个指针成为同一 Python 对象；空指针（index 0）与坏索引均置 `None`。返回容器默认为大小写不敏感、支持下标/属性/调用三种访问的 `AttrDict`，`python_dict=True` 时返回普通 dict。本格式为大端序、列主序。
