本讲围绕 scipy.io 的 `FortranFile` 类（私有实现于 `_fortran.py`，经顶层 re-export 暴露），讲解 Fortran 无格式顺序文件的读写。核心是 record 结构：每条记录为「前长度标记 + 纯数据 + 后长度标记」三段，前后标记须相等，标记默认用 4 字节 `uint32`（`header_dtype`）解释，字节序须与生成平台匹配。读写靠 `write_record` / `read_record` / `read_ints` / `read_reals`；`read_record` 以「前标记 ÷ dtype 总尺寸」做整除校验，多 dtype 混合记录须精确给出各变量尺寸。最易踩坑的是列序：Fortran 列主序与 NumPy 行主序不同，写多维数组要 `.T`、读回要 `reshape(..., order='F')`，两者配套缺一不可。异常区分两类——`FortranEOFError` 表示在记录边界干净结束（正常，用于终止读取循环），`FortranFormattingError` 表示记录内部被截断或标记残缺（文件损坏）；二者均同时继承 `TypeError` 与 `OSError`，其中继承 `TypeError` 仅为向后兼容。
