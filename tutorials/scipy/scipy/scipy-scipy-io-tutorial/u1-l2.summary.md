本讲只讲 `scipy.io` 的骨架，不展开任何格式的读写细节。`scipy/io` 顶层文件分三类：私有实现模块（`_` 前缀，如 `_mmio.py`，真正干活）、弃用包装模块（同名无前缀，如 `mmio.py`，十几行壳）、以及特例 `wavfile.py`（实现就在无前缀文件里，非包装）。Meson 构建靠两个原语：`py3.install_sources([...], subdir: ...)` 安装 `.py`，`subdir(...)` 递归子包；实现与包装都必须安装，否则旧用法会直接找不到模块。公共 API 只从顶层包暴露，私有实现藏在 `_` 模块，由 `__init__.py` 的 re-export 提升上去，`__all__` 用列表推导式把「不带下划线即公共」的约定代码化。核心是弃用命名空间包装：靠 PEP 562 的模块级 `__getattr__` 把属性访问委托给共享函数 `_sub_module_deprecation`，合法名字「警告 + 转发」，非法名字「带弃用提示的报错」，并保留历史实现（如顶层 `mmread` 已迁到 C++ 后端 `_fast_matrix_market`，而旧路径仍指向 `_mmio`）。引入的关键术语：私有实现模块、弃用包装、re-export、模块级 `__getattr__`、`_sub_module_deprecation`，为后续逐格式精读私有实现模块奠基。
