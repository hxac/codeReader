本讲是 MATLAB `.mat` 子系统入口第一讲，只打通「认识子包→打开文件→判版本→选 reader」链路，不展开具体读写细节。涉及三文件：`__init__.py`（汇聚公共 API 与手写 `__all__`、转发弃用命名空间）、`_mio.py`（提供 `loadmat`/`savemat`/`whosmat`、文件打开工具 `_open_file`/`_open_file_context` 及工厂 `mat_reader_factory`）、`_miobase.py`（版本判定 `matfile_version`/`_get_matfile_version`、异常类、抽象基类 `MatFileReader`/`MatVarReader`）。关键术语：`.mat` 是一族格式而非一种——v4 无全局头、v5/v7 含 128 字节全局头、v7.3 实为 HDF5（SciPy 不实现，需 h5py）；工厂模式即由工厂函数按探测到的文件特征返回合适类型的 reader。核心结论：`_open_file` 用鸭子类型把文件名与已打开文件对象统一为字节流，以 `opened` 标志区分关闭责任，`_open_file_context` 包成安全 `with`；`_get_matfile_version` 三段式判定——读前 20 字节查截断/全零损坏、前 4 字节含 0 即 v4、否则读偏移 124 的版本整数与字节序探针，借 `'I'` 的位置无分支解出主次版本号（1=v5，2=v7.3）；`mat_reader_factory` 按主版本号分发到 `MatFile4Reader`/`MatFile5Reader`，v7.3 抛 `NotImplementedError` 并建议 h5py。`loadmat`/`whosmat` 统一经由工厂，使「打开+判定+选择」只实现一次。
