本讲承接 u3-l1 的入口链路，精读 `.mat` 子系统所有 reader 共有的基类基础设施，覆盖 `_byteordercodes.py` 与 `_miobase.py` 两个文件，不展开具体版本读写。

核心结论：`_byteordercodes.py` 是字节序归一化层，`to_numpy_code` 先对输入做 `lower()` 再按 `aliases` 同义词表，把 `'native'`/`'BIG'`/`'<'` 等写法统一映射成 numpy 的 `<`/`>`，并导出 `native_code`/`swapped_code`。`_miobase.py` 顶部定义四个轻量类 `MatReadError`/`MatWriteError`（继承 `Exception`）与 `MatReadWarning`/`MatWriteWarning`（继承 `UserWarning`），构成错误词汇表，`__all__` 仅导出这四个。

引入的关键术语：`docfiller = doccer.filldoc(doc_dict)` 文档复用装饰器，用 `%(load_args)s` 占位符在定义时展开共享参数说明；`MatVarReader` 是变量读取器纯接口（方法体 `pass`），约定 `read_header`/`array_from_header`，由 `VarReader4` 与 Cython 版 `VarReader5` 实现；`MatFileReader` 是根基类，集中存储读取选项，开关间存在覆盖优先级 `simplify_cells > matlab_compatible > 单独入参`；`guess_byte_order` 是模板方法钩子，基类返回本机序，v4 从 `MOPT`、v5 从 `'IM'/'MI'` 探针各自覆写真实探测；架构注释点明「文件级/变量级(header)/元素级」三组参数分层。
