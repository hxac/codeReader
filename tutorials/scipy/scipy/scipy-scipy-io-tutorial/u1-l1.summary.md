本讲是 scipy.io 学习手册的概览篇，只涉及一个核心文件 `scipy/io/__init__.py`，目标是建立全局地图，不深入任何格式的二进制细节。核心结论是：scipy.io 是 SciPy 提供的「多格式科学数据读写器集合」，不发明新格式，而是为 MATLAB、NetCDF、Fortran、Matrix Market、IDL、Harwell-Boeing、WAV、ARFF 等已有外部格式各提供一个 Python 读写器，统一挂在 scipy.io 命名空间下；它与 NumPy 内置 IO（`.npy`/`.npz` 与通用文本）互补，源码 docstring 中以 `seealso` 指向 NumPy IO。

本讲引入的关键术语包括：文件格式（文本与二进制之分）、docstring（既是注释也是官方文档源）、re-export（子模块名字经 import 暴露到顶层）、私有实现模块（带下划线前缀如 `_netcdf`、`_idl`）与弃用命名空间包装（不带前缀的 `mmio`/`idl`/`netcdf`/`harwell_boeing`/`arff`/`wavfile`，将在 SciPy 2.0 移除）。其中六类格式的读写函数被直接 re-export 到顶层（如 `loadmat`、`netcdf_file`、`FortranFile`、`mminfo`/`mmread`/`mmwrite`、`readsav`、`hb_read`），WAV 与 ARFF 则通过子模块访问（`scipy.io.wavfile.read`、`scipy.io.arff.loadarff`）。

`__init__.py` 本质是「总目录」，用若干行 `from ... import ...` 汇聚公共 API；`__all__` 由 `[s for s in dir() if not s.startswith('_')]` 自动生成，体现「不带下划线即公共」的 SciPy 约定，因此定义在第 117 行之后的 `test` 对象虽存在却不进入 `__all__`。后续讲义将逐格式深入源码，并展开公共/私有模块与构建方式的细节。
