本讲为理解 kd-tree 加速最近邻查询奠定几何基础，核心是闵可夫斯基距离与轴对齐超矩形两个工具。闵可夫斯基距离 \(d_p\) 是一族距离：\(p=1\) 为曼哈顿、\(p=2\) 为欧氏、\(p=\infty\) 为切比雪夫；`_kdtree.py` 中 `minkowski_distance_p` 返回 \(d_p^p\) 以省去开方、便于比较大小，`minkowski_distance` 再补开根，二者自 1.18.0 起弃用，新代码应改用 `scipy.spatial.distance.minkowski`。`Rectangle` 类表示轴对齐超矩形：构造时自动摆正上下界，`volume()` 求边长积，`split` 沿某维切分（靠 `np.copy` 不污染原对象）。点到矩形的最近/最远距离有逐维闭式解，最近距离即超出量 \(\delta_i=\max(0,\mathrm{mins}_i-x_i,x_i-\mathrm{maxes}_i)\) 取 \(L_p\) 范数；注意 `Rectangle` 的距离方法调用的是会开根的 `distance.minkowski`。剪枝本质：用 \(d_{\min}(x,R)\) 作下界，若其已不小于当前最优 \(d_{\text{best}}\)，整棵子树可安全跳过，这正是查询压缩至约 \(O(\log n)\) 的根因。本讲 Python 版 `Rectangle` 为参考实现，热路径上的剪枝由 C++ 同名类完成，留待 u8 详讲。
