本讲义只讲 `scipy.spatial` 的目录结构与公共 API 导出机制，不涉及算法。`spatial/` 目录由公共入口 `__init__.py` 与一批以 `_` 开头的私有实现模块（如 `_kdtree.py`、`_ckdtree.pyx`、`_qhull.pyx` 等）组成，外加 `distance`/`transform` 子模块和 `tests`/`src` 子目录。`__init__.py` 用 7 行 `from ._xxx import *` 把私有模块经各自 `__all__` 放行的符号搬到顶层，形成稳定的公共命名空间，顶层 `__all__` 再以 `[s for s in dir() if not s.startswith('_')]` 动态推导并追加子模块名。由此引入的关键概念包括：模块与包、`import *` 与 `__all__` 的「公共出口清单」作用、私有实现（可变）与公共接口（稳定）的分层，以及两层 `__all__` 共同决定可见符号。历史子模块 `kdtree.py`/`qhull.py`/`ckdtree.py` 被改造为弃用影子模块：自身不定义任何符号，只靠模块级 `__getattr__`（PEP 562）把属性访问转发到对应私有模块，并由共用引擎 `_sub_module_deprecation` 做白名单校验、发出指向用户代码（`stacklevel=3`）的 `DeprecationWarning` 后返回私有模块真身；取模块本身（`sp.kdtree`）不触发警告，只有取属性才会，整套机制计划在 SciPy 2.0.0 移除。后续讲义将基于「Python 写易读逻辑、Cython/C++ 写性能路径」这一分层继续展开。
