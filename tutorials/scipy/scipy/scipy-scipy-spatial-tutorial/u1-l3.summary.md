本讲从「Python 层」下沉到「构建层」，回答两个工程问题：扩展模块怎么编译出来、测试怎么跑。核心结论：`scipy.spatial` 的性能内核被编译成六个扩展模块（`_qhull`、`_ckdtree`、`_distance_wrap`、`_distance_pybind`、`_voronoi`、`_hausdorff`），全部由 `spatial/meson.build` 里的 `py3.extension_module` 声明，语言栈涵盖 Cython→C、Cython→C++、纯 C 与 pybind11 C++。引入的关键概念：Meson 声明式构建系统、`extension_module`/`generator` 两个原语、`.pyx` 经 Cython 生成器转 `.c`/`.cpp`（项目级 `cython_gen`/`cython_gen_cpp` 与本包自定义的 `spt_cython_gen`）、`.pxd` Cython 头文件、`_cython_tree` 欺骗相对导入的技巧、`version_link_args` 与外部库 `qhull_r_dep`。测试侧：`scipy.spatial.test` 是 `PytestTester` 实例，默认 `label="fast"` 等价 `-m "not slow"`；`tests/__init__.py` 是空文件，仅作包标记，真正的测试清单与 data 基准由 `tests/meson.build` 声明。
