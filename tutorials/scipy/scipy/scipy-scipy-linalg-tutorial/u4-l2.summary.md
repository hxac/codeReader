本讲专讲 `scipy.linalg` 中面向对称/Hermitian 矩阵的特征值入口 `eigh` 与 `eigvalsh`，与 u4-l1 的一般 `eig` 形成对照。核心结论是：对称/Hermitian 矩阵满足谱定理——特征值全为实数、特征向量可取为正交归一，因此 `eigh` 调用专用驱动（实数前缀 `sy`、复数前缀 `he`，如 `syevr`/`heevr`），相比 `eig` 返回实特征值与正交特征向量，更快更稳，且支持只求部分特征值。

本讲拆为三个最小模块：`eigh`（标准与广义问题的主入口）、`eigvalsh`（`eigh(..., eigvals_only=True)` 的一行薄封装，docstring 自述为 legacy convenience）、`_check_select`（带状/三对角家族的旧式 subset 校验器，并非 `eigh` 使用）。

引入的关键术语与概念：对称/Hermitian 矩阵、谱定理、`lower`/`uplo`（'L'/'U' 只存一半且不校验对称性）、subset 体系（`subset_by_index` 映射为 `range='I'`、`subset_by_value` 映射为 `range='V'`，0 基转 Fortran 1 基 `il/iu`）、driver 选择（默认 `evr`/`gvd`/`gvx`，`ev`/`evd`/`gv`/`gvd` 不支持 subset）、`@_apply_over_batch` 的 Python 层逐片批处理（区别于 `eig` 的 C++ 原生批处理）、`info` 错误翻译（`info>n` 表示 B 非正定）。此外，旧式 `select`+`select_range` API 及 `_conv_dict` 映射为下一讲 u4-l3 的带状/三对角特征值问题埋下伏笔。
