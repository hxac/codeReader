本讲是 scipy.linalg 的「阅读导航篇」，不涉及算法，只建立源码地图，核心结论有三：其一，目录按「算法主题」拆文件，约定以 `_` 开头者才是真正的私有实现模块，文件名暗示主题——`_decomp_*.py` 为各类矩阵分解、`_matfuncs*.py` 为矩阵函数、`_special_matrices.py` 构造特殊矩阵、`_basic.py` 为基础求解；同名但无下划线的文件（如 basic.py）是将于 v2.0.0 移除的废弃垫片，应忽略。其二，`__init__.py` 本身不含算法，靠 21 条星号导入 `from ._xxx import *` 把各实现文件 `__all__` 里的函数汇聚到顶层命名空间，再用 `[s for s in dir() if not s.startswith('_')]` 自动生成 `__all__`，无需手写、自动跟随导入更新。其三，实现文件顶部的 `__all__` 决定对外公开哪些函数，拼接即得「文件→函数」索引（如 svd→`_decomp_svd`、expm→`_matfuncs`），可用 `函数.__module__` 验证定位。关键术语：子包、命名空间、星号导入、`__all__`、私有实现模块、废弃垫片。
