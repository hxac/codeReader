本讲是 scipy.linalg 的首个可运行程序实践，覆盖 solve、inv、det、norm 四个核心函数。solve 解线性方程组 Ax=b，默认自动检测矩阵结构并分派最优 LAPACK 路径；inv 求逆、det 求行列式，二者底层都依赖 LU 分解（getrf），奇异时 inv 抛 LinAlgError、det 优雅返回 0；norm 是 numpy 版的增强，多了 check_finite，热门路径（一维欧氏、二维 1/inf 范数）走 BLAS/LAPACK、其余回退 NumPy。本讲引出贯穿全包的「公共参数」体系：check_finite 用 asarray_chkfinite 挡 NaN/Inf，overwrite_a 仅在 2D 且 Fortran 列主序连续时才真正生效；异常复用 NumPy 的 LinAlgError，并自定义 LinAlgWarning 报告病态。核心结论：解方程用 solve 而非 inv，更稳更快。整体架构为「Python 薄层做校验与错误聚合 + 编译后端 _batched_linalg 做数值计算」，错误经 _format_emit_errors_warnings 统一翻译。涉及的 assume_a 结构编码、PLU 分解、_datacopied 等留待后续讲义展开。
