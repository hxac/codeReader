本讲专讲 `_decomp_qr.py` 中三个正交三角分解函数：`qr`、`qr_multiply`、`rq`。核心结论：`qr` 是「校验+委派」薄壳，将 `mode` 字符串编码为整数（`full=1/r=11/raw=21/economic=31`，与 C++ 后端 `QR_mode` 枚举对齐），把算力交给 C++ 批量后端 `_batched_linalg._qr`；四种 mode 决定返回粒度——`full` 给方阵 Q、`economic` 给瘦长 Q、`r` 只给 R、`raw` 给 LAPACK 内部 `(Q, tau)`。`pivoting=True` 切换列主元 QR（LAPACK `geqp3`），满足 `A[:,P]=QR` 且 R 对角非增，用于秩揭示。`qr_multiply` 复用 `qr(mode='raw')` 拿紧凑 Householder 反射，再用 `ormqr/unmqr` 把 Q 隐式乘到 c 上，省内存省算力。`rq` 直接走 f2py 的 `gerqf`/`orgrq`，得 `A=RQ`，R 在右下角上三角带。关键术语：Householder 反射与 `tau`、`safecall`（封装 `lwork=-1` 查询最优工作长度两步法 + 翻译 `info<0` 错误）、`_NoValue` 哨兵（`qr` 的 `lwork` 已在 1.18.0 废弃，`rq` 仍可用）、`jpvt` 列置换。本讲承接 u3-l1 LU 分解，是矩阵分解基础积木。
