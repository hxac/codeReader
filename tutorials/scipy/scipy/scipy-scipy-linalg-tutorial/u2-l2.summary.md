本讲义钻进 `scipy.linalg.solve` 的内部，讲清「同一个 `solve(A,b)` 为何对正定阵与普通稠密阵走不同求解器」。覆盖三个最小模块：`solve`、`_format_emit_errors_warnings`、`_datacopied`。

核心结论：`solve` 的 Python 层只做「校验→归一化→编码→委派→汇报」，把 `assume_a` 字符串查表翻译成整数 `structure`，再交给 C++ 后端 `_batched_linalg._solve` 真正认结构、选 LAPACK 例程。关键概念：`assume_a=None` 编码为 `-1` 触发**逐片自动检测**（按带宽→对称性顺序，并把实对称/复 Hermitian 乐观先试 Cholesky，失败回退 sym/her）；显式指定（如 `'pos'`、`'gen'`）则跳过检测、更快但说错结构会得错误答案而非报错。`structure` 取值与 C++ 枚举 `St` 一一对应。

`_format_emit_errors_warnings` 是 `solve/inv/lstsq` 共用的批量错误聚合范式：按「奇异>LAPACK 内部错误>病态」优先级，分别抛 `LinAlgError`、`ValueError` 或发 `LinAlgWarning`。`_datacopied` 用 `.base is None` 判断输入是否已拷贝，据此决定能否安全原地覆写；`solve` 主函数则用更严的「二维+F 列主序连续」门控约束 `overwrite_*`，以保证不污染用户数据。本讲引入的 `structure` 编码、错误聚合函数与覆写门控，将被后续求逆/最小二乘、结构化求解器等讲义承接复用。
