本讲义讲解 scipy.constants 如何接入 Python Array API 标准，使 `convert_temperature`、`lambda2nu`、`nu2lambda` 三个「数组进、数组出」的函数支持 NumPy 之外的后端（CuPy/PyTorch/JAX/Dask）。改造思路是函数体不再写死 `np.xxx`，改用 `array_namespace(val)` 取输入所属的命名空间 `xp`，再用 `_asarray(val, xp=xp, subok=True)` 规整输入，输出类型由输入决定、全程零分支。整套机制受环境变量 `SCIPY_ARRAY_API` 全局开关控制，默认关闭时一律返回 NumPy 命名空间，保证老用户零感知。

核心结论：必须区分两件事——**运行时的后端分发**靠 `array_namespace` 与 `_asarray`；而 `@xp_capabilities()` 是**纯标注装饰器**，不包装函数、不改运行逻辑，仅登记能力元数据到全局 `capabilities_table`（供测试框架 `make_xp_test_case` 自动生成 skip/xfail 标记）并追加 docstring 说明。

关键术语：命名空间 `xp`、Array API 标准、`SCIPY_ARRAY_API` 开关、`capabilities_table`、`out_of_scope`。`_codata.py` 中四个标量查询函数 `value`/`unit`/`precision`/`find` 因输入字符串、返回标量、根本不接触数组，用 `@xp_capabilities(out_of_scope=True)` 显式声明不参与，该标志会短路能力表生成并产出「not in-scope」文档段落。一句话：**「能跑多后端」靠函数体写法，「声明/文档/测试支持哪些后端」靠装饰器**，两者分离。
