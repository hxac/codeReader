本讲聚焦 `scipy.constants` 子包的目录结构、构建与导出机制，覆盖五个核心文件：`meson.build`、`__init__.py`、私有实现 `_codata.py`/`_constants.py`，以及弃用垫片 `codata.py`/`constants.py`。核心结论有四：其一，构建树与源码树同构，父级 `scipy/meson.build` 用 `subdir('constants')` 挂载子包，`py3.install_sources` 显式列出待安装文件，`install_tag: 'tests'` 区分测试；其二，`__init__.py` 作为门面，用 `from ._xxx import *` 将私有模块的公开符号汇聚为顶层 API，下划线即私有是全包约定；其三，体现单一数据源哲学——文档表格由 `physical_constants` 运行时生成并经 `__doc__ % dict(...)` 填入 `%(constant_names)s` 占位符，`__all__` 由 `[s for s in dir() if not s.startswith('_')]` 自动派生；其四，`codata`/`constants` 垫片不含实现，借助 PEP 562 模块级 `__getattr__` 与 `_sub_module_deprecation` 发警告并转发到私有模块，`# noqa: F822` 压制 lint 告警，计划 v2.0.0 移除。关键术语：门面、构建树、`import *`、动态 `__all__`、弃用垫片、单一数据源，供后续讲义承接。
