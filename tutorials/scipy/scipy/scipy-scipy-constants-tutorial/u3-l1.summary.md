本讲围绕 `scipy.constants` 中 `convert_temperature` 这唯一的非线性（仿射）物理量换算函数展开。核心结论：温度换算不能像长度那样只乘因子，因为温标零点不一致，必须用 `T_new = a·T_old + b`（a 为刻度比、b 为零点偏移）。该函数采用「中转设计」——先四进一出地把 Celsius/Kelvin/Fahrenheit/Rankine 统一换到以绝对零度为真零点的 Kelvin（中间变量 `tempo`），再一进四出地换到目标温标，将 N² 条公式压缩为 2N 条。公式以水的冰点（0 °C=273.15 K=32 °F=491.67 °R）为锚点：摄氏仅平移零点、华氏既移零点又换刻度、兰氏仅换刻度。实现上复用常量 `zero_Celsius`，但华氏/兰氏分支用字面量 5/9、9/5（数值恰等于 `degree_Fahrenheit` 及其倒数，因后者注释为「仅用于温度差」）。函数头戴 `@xp_capabilities()` 装饰器，配合 `array_namespace`/`_asarray` 适配 NumPy 之外的数组库，装饰器只登记后端能力、不改运行逻辑。两个方向各设独立 `else` 抛 `NotImplementedError`，错误信息分别带 `old_scale=`/`new_scale=` 前缀；新增温标须同时改「→Kelvin」与「Kelvin→」两处。该装饰器为下一讲「Array API 集成与 xp_capabilities」埋下伏笔。
