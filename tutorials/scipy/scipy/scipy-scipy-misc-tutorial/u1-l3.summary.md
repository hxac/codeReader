本讲回顾 `scipy.misc` 的历史职能并解释其退役原因。历史上的 `scipy.misc` 是个**杂物箱（catch-all）模块**，承担四类互不相干的职能：示例数据集（`face`/`ascent`/`electrocardiogram`）、数值工具（`derivative`/`central_diff_weights`）、通用工具子模块 `common`、文档辅助子模块 `doccer`。退役时按主题分流：数据集迁往新建的 `scipy.datasets`（用 `pooch` 实现 `os_cache` + SHA256 registry + 按需下载，取代旧的打包 `.dat` 方式）；`doccer` 迁往私有 `scipy/_lib/doccer.py`（连同测试）；`common` 的 `factorial`/`comb` 早年已迁往 `scipy.special`；`derivative`/`central_diff_weights` 被直接移除、无内置替代。退役根本动机是杂物箱三大毛病：职责不清、依赖与体积膨胀、难以独立演进。本讲引入的关键工具是 `git show <commit>:<path>`（用 `<移除提交>^` 取「移除之前」的版本，移除提交为 `43fc97efa8`，对应 PR #21864），借此还原被删除的旧 `__init__.py` 与 `_common.py`（后者才是数值工具的真身）。旧入口曾通过 PEP 562 模块级 `__getattr__` 按名字拦截访问并给出弃用提示，该机制将在后续讲义逐行精读。
