本讲覆盖 `scipy/_lib/deprecation.py` 与 `scipy/misc/__init__.py` 两个核心文件，讲解 SciPy 版本驱动的弃用政策：先标记弃用（挂 `DeprecationWarning`）→ 跨版本保留 → 在公告版本删除，其中「闹钟」就是写死在警告消息里的版本字符串（属性级 `dep_version`、命名空间级 `2.0.0`）。引入的关键术语包括：哨兵对象 `_NoValue`（用 `is _NoValue` 区分「未传参」与「传了 None」，范例为 `loadmat` 的 `spmatrix`）、装饰器 `_deprecated`（`functools.wraps` 包装、对外公开、拒绝装饰类）、`_sub_module_deprecation`（`stacklevel=3`，软弃用＋重定向，旧入口仍可用）、`_deprecate_positional_args`、`deprecate_cython_api`（靠 `_DeprecationHelperStr.__eq__` 实现「比对即警告」）。核心结论是：`scipy.misc` 因 PR #21864 删空内容，无法走「软弃用」路线，退化为只剩一个裸 `warnings.warn` 的墓碑桩文件，闹钟为 `2.0.0`。
