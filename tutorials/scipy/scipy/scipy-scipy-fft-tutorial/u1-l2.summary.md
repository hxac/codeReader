本讲建立 scipy.fft 的目录地图，提出「四层架构」核心模型：公共 API 层（`_basic.py`、`_realtransforms.py`、`_fftlog.py`、`_helper.py`）只放对外签名与 docstring；分派层由独立库 uarray 按 domain 路由；后端层（`_basic_backend.py` 等 `*_backend.py`）桥接 numpy 与核心；计算核心层是子包 `_duccfft/`，最终落到 C 扩展 `pyduccfft` 真正算 FFT。`__init__.py` 不写算法，仅作搬运清单。命名规律：`_xxx.py` 是对外签名，`_xxx_backend.py` 是同名实现，二者汇入 `_duccfft`。一次 `fft(x)` 调用以「domain+Dispatchable / 方法名查找 / 参数传入」三种方式逐层穿透、彼此解耦。`meson.build` 用 `install_sources` 拷贝 Python 文件、用 `extension_module` 编译 `pyduccfft.cxx`。关键术语：`_dispatch` 装饰器、`Dispatchable`、`domain="numpy.scipy.fft"`、`_ScipyBackend`、`_execute_1D`、`functools.partial` 派生 `fft`/`ifft`、`c2c` 内核、`is_numpy` 分流。该分层使换算法、换数组库、换后端彼此独立，是后续分派机制与自定义后端讲义的基石。
