本讲专讲复数一维变换 fft/ifft，逐个讲透其七个参数（x、n、axis、norm、overwrite_x、workers、plan）的含义与实现。核心结论：公共函数体仅 `return (Dispatchable(x, np.ndarray),)`，是「分派协议声明」而非计算代码，真正计算在四层架构深处。关键概念包括：四层调用链 `_basic.fft → _basic_backend.fft → _execute_1D → _duccfft.c2c → pfft.c2c`；fft/ifft 由同一内核 `c2c` 经 `functools.partial` 派生，靠 `forward` 参数区分方向。参数 n 通过 `_fix_shape` 实现截断（取前 n 个、返回视图）或补零（末尾补 0、新建数组）；axis 选变换轴。norm 三模式 backward/ortho/forward 由 `_NORM_MAP` 映射为整数 0/1/2，再用 `2 - inorm` 翻转实现方向相关缩放，正反配对恒可逆，区别仅在 1/N 缩放归属；默认 None 等价 backward。overwrite_x 仅在复数路径可能原地写，workers 由 `_workers` 解析，plan 立即被拒。C 扩展内 Bluestein 算法保证任意长度都不差于 O(N log N)。
