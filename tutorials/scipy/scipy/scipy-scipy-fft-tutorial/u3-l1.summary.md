本讲剖析 `scipy.fft` 中 `dct`/`idct`/`dctn`/`idctn` 四个余弦变换函数，覆盖四个核心模块：公共签名与四层分派、四种 type 的数学定义与类型翻转、norm 三模式、orthogonalize 正交化。

核心结论：四个公共函数体仅做 `return (Dispatchable(x, np.ndarray),)` 分派声明，无计算逻辑；真正计算沿「公共 API → uarray 分派 → `_execute` 数组桥接 → ducc 核心 `_r2r`/`_r2rn` → C 扩展 `pyduccfft.dct`」四层链路完成。`dct` 是 `dctn` 的单轴封装，8 个函数均由 `_r2r`/`_r2rn` 经 `functools.partial` 绑定 `forward` 与 `transform` 派生。

关键术语：DCT 四种 type（I~IV，默认 type 2，差别在余弦采样位置/边界对称延拓）；type II 与 III 互逆、type I 与 IV 自逆，`idct` 不独立而是对 type 做 2↔3 翻转复用内核；`norm` 三模式 `backward`/`ortho`/`forward` 由 `_NORM_MAP` 映射成整数 `inorm`，用 `2 - inorm` 在正逆间翻转，保证任意模式下 `idct(dct(x))==x`；`orthogonalize` 修正首尾元素（乘/除 √2）使矩阵满足 OᵀO=I（type IV 天然正交）。最关键的源码细节：`orthogonalize` 默认值（`norm="ortho"` 时 True 否则 False）并非 Python 判断，而由 C 层 `bool ortho=inorm==1` 决定，Python 仅把 None 透传。
