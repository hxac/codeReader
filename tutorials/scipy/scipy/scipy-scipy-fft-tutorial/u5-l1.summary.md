本讲打开 `scipy.fft` 四层架构的最底层 `_duccfft`，揭示真正执行 FFT 的计算核心。核心结论有三：其一，全部 14 个公共基础变换仅由 3 个 Python 内核 `c2c`（复→复）、`r2c`（实→复半谱）、`c2r`（复半谱→实）按输入/输出实复属性分工承担；关键洞见是「哪边是实数」由内核决定，而布尔参数 `forward` 只决定 DFT 方向与归一化归属，故同一内核靠 `forward` 的 True/False 复用出正逆两个变换。其二，`functools.partial(内核, True/False)` 派生出 `fft/ifft`、`rfft/ihfft`、`hfft/irfft`，消除预处理逻辑重复；因 `partial` 对象无 `__name__`，派生后必须手动赋名，否则按名字路由的分派链断裂。其三，真正的浮点运算发生在 pybind11 编写的 C++ 扩展 `pyduccfft`（别名 `pfft`）中，Python 侧的 `forward/norm/workers` 经 `m.def` 翻译为 C 层的 `forward/inorm/nthreads`，底层 ducc 库用 Bluestein 算法保证任意长度不低于 O(N log N)。N-D 内核与 1-D 内核调用同一 C 函数，仅 `axes` 列表长度不同。
