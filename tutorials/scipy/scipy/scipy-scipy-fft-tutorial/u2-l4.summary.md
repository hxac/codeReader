本讲聚焦 `scipy/fft/_helper.py`，覆盖三个最小模块：频率刻度、零频居中、最优补零长度。

频率刻度模块由 `fftfreq(n, d)` 与 `rfftfreq(n, d)` 实现，将 FFT 的 bin 下标换算为物理频率：前者返回全谱（先正后负，偶数长度中间为 Nyquist 负频），后者返回长度 `n//2+1` 的非负半谱（Nyquist 视为正）。关键术语：采样间距 `d`、奈奎斯特频率。

零频居中模块中，`fftshift` 沿轴循环移位 `⌊n/2⌋` 把零频从开头搬到中间，`ifftshift` 是其严格逆（移位 `⌈n/2⌉`）。核心结论：偶数长度二者等价，奇数长度差一个样本，但 `ifftshift(fftshift(x))==x` 恒成立。

补零模块的 `next_fast_len`/`prev_fast_len` 返回最优平滑数长度：`real=False` 允许因子 2/3/5/7/11，`real=True` 仅 2/3/5。核心结论是 FFT 在平滑数长度上最快，素数长度最慢，最优补零长度未必是 2 的幂，而是最小 n-smooth 数。

底层 `next_fast_len` 是纯 docstring 占位函数，真正计算由 C 扩展 `pyduccfft.good_size` 完成，经 `lru_cache` 缓存后用 `update_wrapper` 嫁接占位函数的 docstring 与签名，再以 `inspect.signature` 恢复签名。所有 helper 均遵循 array API 约定：优先调用数组库自带实现，否则回退 NumPy。
