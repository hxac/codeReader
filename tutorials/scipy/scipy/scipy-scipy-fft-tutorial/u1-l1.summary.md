本讲只剖析 `scipy/fft/__init__.py` 一个文件，确立 `scipy.fft` 的定位并绘制其公共 API 全景。核心结论是：`scipy.fft` 是 SciPy 1.4 起引入的现代、统一傅里叶变换子包，是 `numpy.fft` 的超集（同名同语义函数齐全，另多出实变换、Hankel、可插拔后端与 `workers` 并行），写新代码应优先选用它，而 `scipy.fftpack` 是历史遗留。引入的关键术语包括：离散傅里叶变换 DFT/快速算法 FFT、模块文档字符串、`__all__` 公共 API 契约、后端（backend）分派机制。入口文件仅做两件事——一段把公共函数按数学性质分成五大类（FFT 18 个、DCT/DST 8 个、Hankel 2 个、helper 9 个、backend 4 个，合计 41 个名字）的文档字符串，以及一组把子模块函数搬入命名空间的 `import` 加 `__all__` 声明。需牢记：文档功能分类与代码物理出处是两个独立维度（如 `fhtoffset` 归 helper 却来自 `_fftlog`，`set_workers` 归 helper 却来自 `_duccfft.helper`）；`__init__.py` 只是索引，真正算法藏在 `_duccfft`（C 扩展 `pyduccfft`）中，分派逻辑在 `_backend`，留待后续讲义展开。
