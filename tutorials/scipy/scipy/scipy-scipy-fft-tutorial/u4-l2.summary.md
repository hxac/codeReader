本讲义围绕 scipy.fft 的 `_backend.py`，讲解 uarray 的后端管理四个 API：`set_backend`/`skip_backend` 是**临时上下文管理器**（仅在 `with` 块内生效，局部优先级最高，`skip_backend` 还可屏蔽全局/注册后端）；`set_global_backend`/`register_backend` 是**永久全局设置**（全局唯一、可覆盖；注册可多个）。核心是优先级模型：候选后端按「**局部（后进先出）→ 注册（正序）→ 全局**」尝试，全局后端位置由 `try_last` 决定。关键结论：scipy.fft 在 import 时执行 `set_global_backend('scipy', try_last=True)`，使默认 scipy 后端**反而最后**被尝试，把优先位置让给注册进来的加速后端（如 CuPy）。三个标志：`only`（该后端返回 `NotImplemented` 即终止回退并抛错）、`coerce`（允许昂贵类型转换且隐含 `only`）、`try_last`（仅全局后端，scipy 默认开）。关键术语：作用域、回退、`NotImplemented`（让位信号）、`BackendNotImplementedError`（全部后端拒绝才抛）、`_backend_from_arg`（经 `_named_backends` 把字符串翻译成后端并强制校验 `__ua_domain__ == "numpy.scipy.fft"`，快速失败）。权威实现位于 C++ 的 `for_each_backend_in_domain`。
