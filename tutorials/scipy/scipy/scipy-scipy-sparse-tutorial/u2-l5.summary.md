本讲介绍三种「构造型」稀疏格式 LIL、DOK、DIA，它们都定位为「先把矩阵搭起来，算之前统一 `tocsr()`」。

**LIL**（List of Lists）用 `rows`/`data` 两个 object 数组、每行一个有序 Python list 实现按行增量写入，`lil_insert` 内核保持每行列号升序，适合按行顺序搭骨架；列切片与算术都慢，构造完即转 CSR。

**DOK**（Dictionary Of Keys）继承内置 `dict` 但真实数据存在 `self._dict`，键为 `(row,col)`（1 维时为 int），支持 1-D/2-D。关键不变量：写零自动删键，故 `nnz == count_nonzero` 恒成立，适合散点 O(1) 读写；出口多为 `tocoo()`。

**DIA**（DIAgonal）用 `offsets`（对角线编号）+ 二维 `data`（每行一条对角线）专为带状矩阵设计，上下对角线靠填充零对齐，`_data_mask` 标出有效位置；它继承 `_data_matrix`，不支持切片，降序 offsets 时等价于 BLAS 带状格式。

核心结论：三者皆非运算器；选型口诀为「散点随机写用 DOK，按行顺序写用 LIL，天然带状用 DIA，建好都转 CSR/CSC」。三种格式的 `nnz` 语义各异：DOK 不存零，LIL 可能含显式零，DIA 仅计对角线有效长度。
