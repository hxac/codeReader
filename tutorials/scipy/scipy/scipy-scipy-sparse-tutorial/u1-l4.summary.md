本讲是稀疏学习手册的第一次动手实践，聚焦七种格式中最常用的两种：用于构造的 COO 和用于运算的 CSR，贯穿「COO 用来造，CSR 用来算」这条设计主线。讲义覆盖四个最小模块：构造方式、输入分发、核心属性、矩阵-向量乘。

第一模块指出 `coo_array`/`csr_array` 共用 `coo_array(arg1, shape, dtype, copy)` 签名，按 `arg1` 不同区分为四类输入：稠密数组、其它稀疏数组、形状元组、坐标三元组；CSR 还多一种原生 `(data, indices, indptr)` 形式。第二模块拆解 `_coo_base.__init__` 用 `isinstance(tuple)` + `isshape` + `issparse` 三层判断将输入统一翻译为 `coords + data + _shape`，并强调三元组构造时 `has_canonical_format=False`、稠密构造时为 `True`。第三模块明确 `nnz` 是「已存储元素数（含显式零）」，区别于真正非零的 `count_nonzero`；CSR 的 `nnz` 读自 `indptr[-1]`，`indptr` 长度为行数加一。第四模块讲解 `@` 经 `__matmul__`、`_matmul_dispatch`、`_matmul_vector` 最终调用 C++ 内核 `csr_matvec`/`coo_matvec`，标量乘法须用 `*`。

核心结论：COO 允许重复坐标，转 CSR 时由 `sum_duplicates` 自动求和，是有限元矩阵组装的标准套路；稀疏数组不可直接丢给 NumPy 逐元素函数（如 `np.sin`），否则会静默出错，应先 `toarray` 或用 scipy.sparse 自带方法。
