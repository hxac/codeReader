本讲围绕 SciPy「构建系统」展开，解释为何多语言混合项目（Python + C/C++/Cython/Fortran/Pythran）需要真正的构建系统。核心构建链为：`pip` → `meson-python`（PEP 517 后端）→ `meson`（读 `meson.build`）→ `ninja`（执行编译）。三个关键配置文件：`pyproject.toml` 声明构建后端、构建依赖（Cython/pythran/pybind11）与运行依赖（numpy≥2.0.0、Python≥3.12）；`meson.build` 顶层编排，声明语言、默认链接 OpenBLAS、检查编译器与五个 submodule；`meson.options` 暴露 `blas`/`lapack`、`use-ilp64`、`use-system-libraries` 等可配置开关。引入术语：BLAS/LAPACK（默认 OpenBLAS）、ILP64、构建隔离、submodule。核心结论：构建不仅编译，还动态生成 `version.py` 与 `__config__.py`（即 `show_config()` 的数据来源）。从源码构建有两种方式：`spin build` 与 `pip install -e . --no-build-isolation`。
