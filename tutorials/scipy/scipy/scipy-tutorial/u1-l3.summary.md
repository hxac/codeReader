本讲精读顶层包入口 `scipy/__init__.py`，解析 `import scipy` 时的执行序列：加载构建配置、取版本号、发行商钩子、NumPy 版本校验、扩展模块金丝雀检查、挂上 `test`、定义延迟导入设施。核心是 PEP 562 模块级 `__getattr__`：子包名只登记在 `submodules`，首次访问时才经 `importlib.import_module` 真正加载，`__dir__` 配合保证补全与 `dir()` 正常，从而实现「用到才加载」的延迟导入。两道安全闸门为 NumPy 版本须落在 `[2.0.0, 9.9.99)`（不满足仅发警告）与故意导入编译扩展 `LowLevelCallable` 的金丝雀检查。关键术语：延迟导入（lazy import）、`__getattr__`、`submodules`、金丝雀检查（canary）、构建产物（`version.py`/`__config__.py`）、`PytestTester`、`show_config`。核心结论：`__version__` 与 `show_config()` 数据均来自构建产物（前者由 `tools/gitversion.py` 在构建时生成，dev 版追加 git 后缀）；`PytestTester` 是按绑定模块名跑 pytest 的可调用对象，统一了 `scipy.test()` 与各子包 `.test()` 入口；docstring 子包清单、`submodules`、`__all__` 三处必须保持一致。
