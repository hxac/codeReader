本讲精读 `scipy/_lib/_util.py`（辅以 `_sparse.py`、`_array_api.py`），它是 SciPy 把各子包重复样板提炼成的「私有共享工具箱」，拆成五个最小模块：① RNG 工具——`check_random_state` 把 None/int/RandomState/Generator 四种 seed 统一为实例（None 返回全局单例），`rng_integers` 统一两套整数接口，`_transition_to_rng` 装饰器按 SPEC 7 把旧参数名平滑迁移到 `rng` 并动态修补签名与文档；② 惰性分支——`_lazyselect` 用函数取代 `np.select` 的数组分支，仅对满足条件元素求值，省算且更安全；③ 输入校验——`_asarray_validated` 串联稀疏/掩码/有限性/object/浮点五道关卡，`_contains_nan` 用 `xp.max` 技巧高效判 NaN 且兼容 Array API，`_sparse.issparse` 以 `isinstance(x, SparseABC)` 一行识别稀疏；④ 可调用封装——`MapWrapper` 把 int/-1/map-like 统一为并行 map 并规避 POSIX fork 死锁，`_call_callback_maybe_halt` 复用 `StopIteration` 表达「用户请求终止」；⑤ 结果与反射——`_RichResult`（`OptimizeResult` 的基类）兼顾 dict 灵活性与属性访问，`getfullargspec_no_self`/`_rename_parameter`/`_apply_over_batch` 维护包装函数的签名、文档与批量语义。关键结论：复用热点集中在 optimize/stats/linalg/integrate/ndimage/signal 等 Python 代码量大的子包，而 special 因核心是编译 ufunc 几乎不直接用 `_util`。本讲引入的术语 `_lazyselect`、`check_random_state`、`_transition_to_rng`、`_asarray_validated`、`_contains_nan`、`_RichResult`、`MapWrapper`、`array_namespace`、SPEC 7 将由后续 uarray、Array API 等讲义承接。
