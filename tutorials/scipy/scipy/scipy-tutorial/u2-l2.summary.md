本讲讲解 SciPy 内嵌的 uarray 协议——「用户显式选后端」式分发，与下一讲 Array API「代码看数组」相对。四个最小模块：①多方法与后端：`generate_multimethod(extractor, replacer, domain, default)` 生成只占位、不实现计算的多方法；后端需提供 `__ua_domain__`（层级点分域，如 `"numpy.scipy.fft"`）与 `__ua_function__(method, args, kwargs)`，可选 `__ua_convert__`；返回 `NotImplemented` 即礼貌拒绝、触发回退，全部拒绝则抛 `BackendNotImplementedError`。②激活方式：`set_backend`（临时、栈式最高优先）、`set_global_backend`（全局、可 `try_last`）、`register_backend`（全局、最低优先）、`skip_backend`（临时黑名单），`only`/`coerce` 控制短路。③`scipy.fft` 落地：`_basic.py` 用 `@_dispatch` 把变换函数做成域为 `"numpy.scipy.fft"`、只对输入 `x` 分发的多方法；`_backend.py` 的 `_ScipyBackend` 借 `getattr(_basic_backend, name)` 委托真实计算，模块加载时 `set_global_backend('scipy', try_last=True)` 使 SciPy 永远兜底、用户注册的后端优先。④边界：`fft`/`dct`/`fht` 是可被后端拦截的多方法，而 `fftfreq`/`fftshift` 等走 `array_namespace` 的普通函数后端拦截不到——这正是 uarray 与下一讲 Array API 的分界线。
