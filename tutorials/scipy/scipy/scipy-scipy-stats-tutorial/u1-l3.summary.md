本讲承接 u1-l2「`__init__.py` 是纯路由层」的结论，拆解三个最小模块：命名空间聚合、首次函数调用、模块测试入口。核心结论：scipy.stats 用一系列 `from ._xxx import *` 把分散在 `_stats_py.py`、`_continuous_distns.py`、`distributions.py` 等子领域文件里的公开名字搬运进统一命名空间，再以一行 `__all__ = [s for s in dir() if not s.startswith("_")]` 自动生成公开 API，新增函数无需手工登记；一个名字（如 `norm`）需经 `_continuous_distns.py → distributions.py → __init__.py` 三级搬运才抵达 `scipy.stats`。调用方面：`describe` 返回命名元组 `DescribeResult`（兼顾属性访问与解包），`norm.pdf` 落到 `_norm_pdf` 与常量 √(2π)，分布被当作「对象、pdf 为方法」的黑盒。`_common.py` 的 `ConfidenceInterval` 命名元组是带推断结果对象的共享砖块。`stats.test()` 由三行 `PytestTester` 接线提供，因接线发生在 `__all__` 冻结之后，故 `test` 可调用却不在公开 API——印证「`__all__` 只管 `import *`、不管属性访问」，可用 `extra_argv=['-k','关键词','-q']` 跑测试子集。`norm_gen`、`rv_continuous`、`loc/scale` 等留待 u3/u4 展开。
