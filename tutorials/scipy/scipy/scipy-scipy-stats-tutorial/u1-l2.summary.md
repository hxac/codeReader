本讲讲清了 `scipy/stats/` 目录的组织规则。目录文件归为四类：公共入口/聚合壳（如 `__init__.py`、`distributions.py`，本身不写算法只搬运）、子领域实现（下划线前缀的 `_xxx.py`，如 `_stats_py.py`、`_continuous_distns.py`，是真正的算法所在）、编译扩展源（`.pyx`/`.c`/`.cpp`，编译成 `.so`）、以及自带 `meson.build` 的子包（`_levy_stable`/`_unuran`/`_rcont`）。核心结论有二：其一，`__init__.py` 是纯路由层，靠 `from ._xxx import` 聚合、再用 `__all__ = [s for s in dir() if not s.startswith('_')]` 自动生成公开 API，因此读导入区即读 API 清单；其二，`meson.build` 用 `extension_module` 编译扩展（主目录 7 个）、用 `install_sources` 原样安装纯 Python 与数据文件，搜文件名落点即可判断其形态。引入的关键术语包括聚合壳、遗留 shim（如 `stats.py`，v2.0 将移除）、`_result_classes.py`（文档专用聚合，不定义类）、以及「C/C++ 内核 + Cython 外壳」的子包分层模式。
