本讲将 u1-l1 的功能与 u1-l2 的源码分层缝合，讲清 `scipy.ndimage` 公开 API 如何自底向上经四层架构装配：第 0 层 C 内核 `_nd_image`，第 1 层 Python 实现（5 个 `_*.py`），第 2 层裸 API 聚合 `_ndimage_api.py`（用 `import *` 抄函数、`dir()` 生成 `__all__`，作单一事实来源），第 3 层后端委托 `_support_alternative_backends.py` 配 `_delegators.py`，第 4 层 `__init__.py` 仅重新导出并 `del` 私有名。核心是第 3 层 `delegate_xp` 装饰循环：按数组命名空间分派——CuPy（黑名单外全转交 cupyx）、JAX（仅 map_coordinates）、其余回退裸 NumPy 实现后转回 xp。环境变量 `SCIPY_ARRAY_API`（默认关闭）为总开关，关闭时跳过整层委托外壳直抵 NumPy。另保留 filters/morphology 等不带下划线的弃用桩（v2.0.0 移除）。引入的关键术语：四层装配链、裸 API、delegate_xp、委托/分派、capabilities_dict、`*_signature`、弃用桩、单一事实来源。
