本讲承接 u1-l1，把视角从「能做什么」转向「源码在硬盘上如何组织」。核心结论是：`scipy/ndimage/` 采用「薄 Python 层 + 厚 C 层」结构，目录被刻意组织成五类文件：Python 包装层（`_*.py` 私有实现与弃用桩）、C/C++/Cython 扩展源码（`src/`）、构建脚本（`meson.build`）、测试（`tests/`）和工具（`utils/`）。

引入的关键术语与概念：**扩展模块**（编译成 `.so`/`.pyd` 的动态库，名字带前导下划线表示私有）；**Meson 构建系统**及其 `py3.extension_module`、`py3.install_sources`、`python_sources` 安装清单等函数；三个生产级扩展——`_nd_image`（纯 C，8 个源文件，承载滤波/插值/形态学/测量内核）、`_ni_label`（Cython，连通区域标记）、`_rank_filter_1d`（C++，一维秩滤波）；**`methods[]` 分发表**（Python 名到 C 函数指针的单一事实来源）与 **`PyInit_<模块名>` 初始化函数**；**C 包装函数三段式**（解析参数 → 调用 `NI_*` 内核 → 清理返回）；`ni_` 前缀与「一个内核模块一对 `.h`/`.c`」约定；以及「C 层比 Python 公开层更聚合」（少数 C 原语支撑数十个公开函数）。此外区分了弃用桩文件（无下划线同名文件，v2.0.0 移除）与真正的私有实现，并指出 `nd_image.c` 是把各功能域头文件串联起来的入口胶水。
