本讲专讲 `scipy.ndimage.label`，它把数组中所有相互连通的非零像素（前景）各打上唯一整数标签，背景固定为 0。涉及三个源码文件：Python 入口 `_measurements.py`、结构元生成器 `_morphology.py` 的 `generate_binary_structure`、以及独立于 `_nd_image` 之外的 Cython 内核 `src/_ni_label.pyx`。

核心结论有四点。其一，`label` 有两种返回契约：`output=None` 或 dtype 时返回 `(labeled_array, num_features)` 二元组；`output` 为已存在数组时则就地写入、只返回 `num_features`，分叉由 Python 端 `caller_provided_output` 控制。其二，「连通」由结构元定义：默认十字元对应 `generate_binary_structure(ndim,1)` 的 4-连通；`connectivity=2` 给出全连通的 8-连通，本质是「到中心的 L1 距离上限」。其三，结构元必须中心对称（centrosymmetric），否则连接单向、判定失效，Python 与 Cython 双重校验。其四，内核用 fused type 对多 dtype 特化，采用逐行扫描的两遍算法：第一遍用 union-find 等价类合并（`mergetable`）记录「实为同一物体」的标签冲突，第二遍把跳号标签紧缩为 1,2,3…。此外引入 `NeedMoreBits` 机制处理用户指定过窄 dtype 导致的溢出，靠异常升级位宽重算并拒绝真实截断。

关键术语包括：连通分量、结构元、connectivity、中心对称、union-find/等价类合并、fused type、caller_provided_output、NeedMoreBits。这些概念为后续 `find_objects`、按标签统计及形态学单元（更大结构元）承接。
