本讲是 ndimage 入门单元的收尾，聚焦被几乎所有函数共享、位于 `_ni_support.py` 与 `_ni_docstrings.py` 的四个「支撑工具」，它们构成一切函数的公共骨架：先填文档，再备输出，再规范参数，最后把 mode 编码交给 C 内核。核心结论如下：`_extend_mode_to_code` 把 8 个边界模式字符串翻译成 C 内核所需的 7 个整数码（0–6），`is_filter` 参数仅在 `grid-wrap`/`grid-constant` 上起作用，让纯滤波场景归并复用 C 内核、插值场景保持独立语义。`_get_output` 统一处理 `output` 的三种形态——`None` 新建同类型数组、dtype 类/字符串按指定类型新建、已存在数组就地复用，并管理复数输出的提升与校验。`_normalize_sequence` 把标量广播成等长序列（先排除字符串以免误拆），`_check_axes` 把 `None`/标量/序列统一成合法、唯一、非负的轴元组。`docfiller` 基于 `scipy._lib.doccer`，用 `%(key)s` 占位符加 `docdict` 字典让数十个函数共享参数文档。掌握这些后，读后续滤波、插值、测量、形态学讲义时可跳过千篇一律的参数预处理，直击各自算法本身。
