本讲承接 u4-l1 的连通区域标记，介绍 `scipy.ndimage` 的两个互补定位工具。`find_objects` 接收标签数组，为每个标签返回最小包围盒切片（多维 slice），结果列表按 `label-1` 索引；其内核是单遍全数组扫描逐点扩张包围盒（start 取 min、end 存 coord+1 以匹配排他切片），复杂度 O(N)。关键参数 `max_label` 同时决定搜索范围、返回列表长度与 C 端 `regions` 缓冲区大小，默认取 `input.max()`，标签缺失处用 `None` 占位，越界标签被静默忽略。`value_indices` 接收任意整数数组，返回「值→坐标元组」字典，内核用三遍扫描（求 min/max、建直方图、填坐标）把多值定位降到常数次扫描，并因「值减最小值作下标」的设计而仅接受整数。两者分工：前者粗（连续标签→矩形包围盒），后者细（任意整数→逐像素精确坐标），常与 `label` 配合完成「标记→定位→分析」，并为 u4-l3 按区域统计（sum/mean/var 等）奠定基础。
