本讲是「测量与连通区域」单元（u4）收尾篇，覆盖 `_measurements.py` 最后四个公开函数：`labeled_comprehension`、`extrema`、`center_of_mass`、`histogram`，以及唯一有独立 C 内核的 `watershed_ift`。核心结论有三：其一，`labeled_comprehension` 是通用聚合引擎，语义等价于 `[func(input[labels==i]) for i in index]`，通过「按 labels 排序 + `searchsorted` 二分夹段」将复杂度从 O(N·K) 降到 O(N log N)，靠 `func`/`out_dtype`/`default`/`pass_positions` 四参数协同，`histogram` 即其最典型客户（闭包包装 `np.histogram`，`out_dtype=object` 容纳数组）。其二，`extrema` 复用 u4-l3 的 `_select`「正/反向 fancy indexing」一次性返回每区域 min/max 值与位置，扁平下标经 `dim_prod` 整除取模还原为 N-D 坐标；`center_of_mass` 无新内核，质心 Σx·v/Σv 的分子分母均委托 `sum_labels`。其三，`watershed_ift` 基于 IFT 算法，以「路径上最大弧权 |I(p)-I(v)|」为代价，用桶优先队列（代价 0..maxval 各一桶）从 marker 松弛传播；正/负 marker 区分对象/背景且平局时对象优先，input 限 uint8/uint16 正是桶队列的设计约束。关键术语：通用聚合、扁平下标、unravel、加权质心、IFT、桶优先队列、路径代价函数、marker、对象/背景 marker。
