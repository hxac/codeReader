本讲义系统讲解 scipy.ndimage 的测试分层策略，对应 u7 单元第 2 篇，承接 u6-l1 的 `_nd_image` 扩展模块与 u7-l1 的数组 API 后端委托。核心观点是：ndimage 测试是一套分层策略，而非零散断言。它按四个维度组织：一是按功能域分文件，五大域（filters/interpolation/measurements/morphology/fourier）各对应一个 `test_*.py`，且 `tests/meson.build` 的 `python_sources` 是文件清单的单一事实来源，未注册则不会安装。二是 `test_datatypes.py` 用「数据 dtype × 坐标 dtype × order」的笛卡尔积矩阵（约 864 组合）覆盖插值函数的全部 12 种数值类型，对已知平台 bug 用 `@pytest.mark.xfail` 显式登记。三是本次（提交 `ce1f6477`）新增的 `test_array_likes.py`，是一份 numpy-only 冒烟回归，用「双调对比」（list 输入对 ndarray 输入）加递归多态的 `_assert_same_result` 比对器，一次性守住全 74 个公开 API 对 array_like 的接受度；其中数组位参数（weights/coordinates/matrix/labels/index）需双份化，非数组参数（回调、标量、sigma）只传一次。四是 `test_c_api.py` 用「参照 Python 实现 vs 四种回调入口」（纯 Python、C 指针、Cython、`LowLevelCallable` 两变体）两两比对，验证 C 扩展回调桥接的正确性。此外，支撑测试 `test_ni_support` 穷举 `_get_output` 分支、`test_splines` 用样条结点矩阵反推 `spline_filter1d`；冻结回归向量链则由 `generate_label_testvectors.py` 生成 `data/*.txt`，由 `test_label_structuring_elements` 消费，作为保护 Cython 内核的黄金标准。关键术语包括横切测试、dtype 笛卡尔积矩阵、双调对比、`_assert_same_result`、`LowLevelCallable` 回调四入口、冻结回归向量链。
