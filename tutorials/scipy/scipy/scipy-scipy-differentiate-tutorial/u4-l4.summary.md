本讲讲解 scipy.differentiate 的 Array API 跨后端实现，覆盖三个核心模块。其一，array_namespace(x) 据全局开关 SCIPY_ARRAY_API 把输入数组映射为命名空间 xp：默认走 NumPy 零开销快路径，开关开启后分发到 torch/jax/cupy/dask；配合 xp_promote 统一 dtype 并可强制浮点、xp_copy 屏蔽各后端拷贝差异。其二，xpx.at(...)[idx].set/multiply(...) 以返回新数组的函数式写法替代 NumPy 原地索引赋值，从而兼容 JAX 等不可变后端；索引可为布尔掩码或花式索引。其三，@xp_capabilities 装饰器集中声明各函数支持或跳过的后端，同时驱动 docstring 支持矩阵表注入与测试 skip_xp_backends 标记自动生成。核心结论：array_api_strict 与 dask.array 被跳过，根因正是 xpx.at 依赖的花式与布尔索引赋值在二者上缺失。
