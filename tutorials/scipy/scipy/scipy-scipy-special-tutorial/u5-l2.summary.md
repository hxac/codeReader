本讲对比 `scipy.special` 中三类正交多项式接口：`eval_*`（逐元素求值的 NumPy ufunc，住在编译产物 `_ufuncs`，C 内核用三项递推求数值稳定的 \(P_n(x)\)）、`roots_*`（返回高斯求积节点与权重的纯 Python 函数，住在 `_orthogonal.py`）、以及 `legendre`/`jacobi` 等（返回存系数的 `orthopoly1d` 对象）。关键结论有三：(1) `roots_*` 表面与 `eval_*` 并行，实则依赖它——通用引擎 `_gen_roots_and_weights` 的牛顿抛光与 Christoffel 权重靠 `f`/`df` 回调，回调内部就是调 `eval_*`（求导则用解析恒等式换参数降阶实现）；默认走 Golub-Welsch，但 `roots_chebyt`（解析闭式）和 `roots_hermite` 大 \(n\) 分支（Townsend 渐近算法）是反例。(2) 因输出长度由标量 \(n\) 决定且需跨元素加权和，`roots_*` 注定不能是 ufunc，只能返回数组在 Python 层求和。(3) 由根反乘系数受 Wilkinson 病态污染，高阶系数 Horner 求值失稳；`orthopoly1d.__call__` 通过 `_eval_func` 优先路由到 `eval_*` 缓解，但算术会丢弃该路由而退回病态系数法。选型口诀：积分用 `roots_*`，求值用 `eval_*`，碰系数对象留意算术后退化。承上启下：下一讲 u5-l3 将过渡到 `_multiufuncs.py` 的 `MultiUFunc`（一次取所有阶，如 `legendre_p_all`）。
