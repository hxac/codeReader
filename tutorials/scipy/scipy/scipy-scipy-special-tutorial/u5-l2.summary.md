本讲区分 scipy.special 两组来源不同的正交多项式接口：`eval_*` 是 `_ufuncs` 编译产物里的求值 ufunc，逐元素广播、C 内核走三项递推、数值稳定；`roots_*` 住于 `_orthogonal.py`，输出高斯求积节点与权重，因输出长度由标量 n 而非输入形状决定，故是普通 Python 函数而非 ufunc。核心结论：`roots_*` 通用引擎 `_gen_roots_and_weights` 的牛顿抛光与 Christoffel 权重均靠回调 f/df，而回调即直接调用 `eval_*`，故 roots 内部依赖 eval；`roots_chebyt` 闭式公式、`roots_hermite`(n>150) 渐近算法则绕开 Golub-Welsch。第三组 `orthopoly1d` 存系数，但系数由根反乘得来（Wilkinson 病态），高阶 Horner 求值失稳；其 `__call__` 优先路由到 `_eval_func`（即 eval_*）保稳定，算术会丢弃它而退化。选型：积分用 roots_*，求值用 eval_*。
