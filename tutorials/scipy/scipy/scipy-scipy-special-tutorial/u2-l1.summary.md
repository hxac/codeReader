本讲义精读 scipy.special 的 ufunc 机制，确立核心结论：命名空间里几乎所有函数都是 NumPy ufunc。涵盖四块：(1) ufunc 本质是按类型分发、逐元素求值、可批量的 C 级对象，而非 Python 函数，标量与数组同源；(2) 类型系统用单字符类型码（f/d/g 实数、F/D/G 复数）描述 loop（类型环），`.types` 返回 `d->d`、`dddD->D` 形式的输入→输出签名；(3) 多类型分发——运行时按输入 dtype 自动选 loop，如 erf 挂 f/d/F/D 四环以支持复数，支不支持复数可从 `.types` 一眼看出；(4) 三大通用能力——广播、`out=`（多输出 ufunc 如 sici/airy 需传数组元组）、`where=`，且必然逐元素，故跨元素聚合的 logsumexp 做不成 ufunc。引入的关键术语：ufunc、loop（类型环）、类型码、多类型分发、广播、`out=`、`where=`、必然逐元素、类型桩 `_ufuncs.pyi`。佐证源码为 `_ufuncs.pyi`（静态身份与 `__all__` 清单）、`_generate_pyx.py`（类型码定义）、`_special_ufuncs.cpp`（loop 注册）、`functions.json`（多输出签名）。
