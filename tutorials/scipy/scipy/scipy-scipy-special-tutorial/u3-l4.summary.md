本讲梳理 `scipy.special` 背后的 C/C++ 数学后端版图，核心结论是：`functions.json` 中层的「头文件」字段是后端调度的总开关——文件名指明后端库，后缀（`.h`/`.h++`/`.pxd`）指明语言，并据此决定内核编进 `_ufuncs` 还是 `_ufuncs_cxx`。引入的关键术语包括：**xsf**（SciPy 自研的现代统一 C++ 库，位于 `scipy/_lib/xsf/`，已收编 Cephes 与 Zhang&Jin specfun，经 `xsf_wrappers.h` 的 `extern "C"` 桥接给 ufunc，并存在「生成式 `.pyx`」与「`_special_ufuncs.cpp` 直接 C++ 注册」双轨）；**Boost.Math**（承担高精度/强健壮需求，靠 `SpecialPolicy` 关闭类型提升、靠 `user_*_error` 把 C++ 异常桥接成 Python 告警/异常，提供 float/double 双内核）；**Cephes**（已迁入 `xsf/cephes/`，另有 `_legacy.pxd` 处理历史浮点截断兼容）；**cdflib**（编为静态库 `cdflib_lib`，经 `_cdflib_wrappers.pxd` 调用，擅长非中心分布与反解参数）；**Carlson 椭圆积分**（轻量 C++ 包装 `ellint_carlson_wrap.hh++`）。同一函数可按 dtype 分发到不同后端（如 `hyp1f1` 实数走 Boost、复数走 xsf），后端正整体向直接 C++ 注册路径迁移。
