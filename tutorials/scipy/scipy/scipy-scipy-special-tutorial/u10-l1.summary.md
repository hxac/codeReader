本讲剖析 `_support_alternative_backends.py` 如何以「元数据驱动」方式把 80 多个逐元素 ufunc 改造成多后端函数,分三块:(1)数据类 `_FuncInfo` 把每个函数在各后端上的差异(参数数、后端别名 `alt_names_map`、回退实现 `generic_impl` 等)抽成结构化字段,使分发逻辑只写一份;(2)全局开关 `SCIPY_ARRAY_API`(读自同名环境变量)默认关闭,此时 `_FuncInfo.wrapper` 直接返回裸 ufunc,仅借 `xp_capabilities` 原地改写文档串与能力表(`assert cap_func is func` 是契约保证);开启时则返回分发包装,先 `array_namespace(*args)` 推断后端,再 `_wrapper_for(xp)` 按六级优先级解析内核:NumPy 直通 → 后端原生 → `generic_impl` → marray 解包 → dask 分块 → NumPy/SciPy 兜底(JAX 另有 JIT 支路);(3)`generic_impl` 是用纯 Array API 原语现写实现的工厂,可返回 `None` 表达「此路不通」,构成优雅退化链。命名空间覆盖在模块加载时由 `globals().update` 配合 `__init__.py`「后导入覆盖先导入」完成。整套 marray/dask 兜底依赖「函数逐元素」这一前提,故 `logsumexp` 等非逐元素函数不在清单内。`@xp_capabilities` 装饰器留待 u10-l2 详讲。
