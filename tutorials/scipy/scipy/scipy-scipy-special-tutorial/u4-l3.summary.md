本讲解析 `scipy/special/_logsumexp.py`，覆盖 `logsumexp`、`softmax`、`log_softmax` 三个函数。它们是纯 Python 实现，但因跨元素求和破坏了 ufunc「逐元素」前提，故不是 ufunc，构成「为何不用 ufunc」的另一类范本。核心动机是数值稳定性：直接 `log(sum(exp(a)))` 会在指数阶段溢出（如 `exp(1000)=inf`）。贯穿全讲的 **max-shift 技巧**利用恒等式先减去 `a_max` 再取指数，使所有指数非正、永不溢出，最后加回 `a_max`。`softmax` 借同除 `e^m` 约去、`log_softmax` 写成 `tmp - log(sum(exp(tmp)))` 避免饱和失真；内部 `_logsumexp` 进一步采用 **Blanchard–Higham 精化**，把最大项单独提出用 `log1p` 高精度计算。进阶特性包括：可负权重 `b`、`return_sign`（符号/幅度分离，复数下返回单位复数辐角）、复数输入相位包裹 `_wrap_radians`（把辐角折回 `(-π,π]` 并保留区间内精度）。跨后端能力由 `array_namespace`、`xp_promote(force_floating=True)` 与 `@xp_capabilities()` 装饰器三件套提供，默认 `SCIPY_ARRAY_API` 关闭即纯 NumPy，开启后分发到 PyTorch/JAX/CuPy/Dask。引入的关键术语有：max-shift、log1p、符号分离、complex phase、Array API 命名空间与 `force_floating`，为 U4-l4 薄包装与 U10 多后端分发讲义作承接。
