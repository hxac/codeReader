本讲承接 u1-l2 的「Python 包装层 → Cython 层 → C/C++ 内核层」三层地图，只精读 `meson.build` 与 `__init__.py` 两个文件，回答这三层如何被构建并变成可运行代码。核心结论：`scipy.special` 是编译型子模块，函数体住在 Cython/C/C++ 编译出的共享库里而非纯 `.py`。Meson 构建流水线为：`subdir('special')` 纳入构建 → `custom_target` 跑 `_generate_pyx.py` 配合 `functions.json` 做代码生成（产出 `_ufuncs.pyx`）→ `generator` 把 `.pyx` 翻成 `.c`/`.cpp` → 编译链接成 `.so` → 安装纯 `.py`。构建目标分六类：静态库（`cdflib`）、扩展模块（共 7 个）、代码生成、Cython 生成器、安装、子目录递归。7 个扩展模块为 `_special_ufuncs`、`_gufuncs`、`_specfun`、`_ufuncs`（主体，承载绝大多数 ufunc）、`_ufuncs_cxx`（承载 Boost.Math 重型 C++ 函数，与 `_ufuncs` 拆分以隔离编译成本）、`_ellip_harm_2`、`cython_special`（供外部 `cimport`）。注意同名混淆：`cython_special` 既是代码生成 `custom_target`，又是成品扩展模块。加载侧：`import scipy.special` 时 `__init__.py` 用 `from . import _ufuncs` 触发共享库加载，再 `from ._ufuncs import *` 把 ufunc 转发进命名空间；`_support_alternative_backends` 会覆盖部分函数以加 Array API 支持。测试侧有两种等价入口：`special.test()`（`PytestTester` 实例而非类，薄封装走 `--pyargs` 验证已安装包）与直接 `pytest`（跑源码树）。引入的关键术语：编译型扩展模块、Cython、Meson、`subdir`、`static_library`、`extension_module`、`custom_target`、代码生成管线、ufunc、命名空间拼装、PytestTester。
