本讲义是 U4「纯 Python 包装层」的第四篇,提出第四种「为何不用 ufunc」的模式——薄包装(thin wrapper):函数本身逐元素、由底层 ufunc 干活,只需在调用前后做少量参数预处理。仅涉及 `_lambertw.py` 与 `_spherical_bessel.py` 两个最小模块。核心结论有三:其一,`lambertw` 把分支索引 `k` 用 `np.asarray(k, dtype="long")` 强制成 long 整数,因底层 `_lambertw` 的 loop 串 `Dld_D`/`Flf_F` 第二输入位固定为 `l`,这是一道类型护栏;其二,`spherical_jn/yn/in/kn` 通过布尔参数 `derivative` 在底层 `_xxx`(函数值)与 `_xxx_d`(导数)两组 ufunc 间二选一,阶数 `n` 同样强制为 long。引入的关键术语包括薄包装、带参数的装饰器工厂 `use_reflection`、DLMF 10.47(v) 反射公式。`use_reflection` 按 `sign_n_even` 处理负实轴:复数放行、正实轴直算、负实轴用 `f(-z)*sign` 还原,`spherical_jn`(+1)与 `spherical_yn`(-1)符号差异源于 `(-1)^n` 与 `(-1)^{n+1}` 差一负号,导数分支再多一链式法则负号;`spherical_kn` 因 `K_ν` 支割线改用复数反射 `spherical_kn_reflection`,`test_negative_real_gh14582` 为共同回归护栏。
