本讲精读 `scipy.special._orthogonal.py` 单一文件，串起正交多项式与高斯求积的三块主线。首先，正交多项式族满足三项递推 \(P_{n+1}=(x-A_n)P_n-B_nP_{n-1}\)，其中系数 \(A_n\) 进入矩阵主对角线、\(\sqrt{B_n}\) 进入次对角线。其次，把这对系数摆成对称三对角的 **Jacobi 矩阵**，其特征值即为高斯求积的节点——这是 **Golub-Welsch 算法** 的核心；本实现为改良版，节点取特征值并经牛顿抛光，权重用 **Christoffel 公式** \(w_i\propto 1/(P_{n-1}(x_i)P_n'(x_i))\) 配对数稳定化求出，省去求特征向量，最后以 `w *= mu0/w.sum()` 归一到权函数积分 \(\mu_0\)。每个 `roots_*` 都遵循同一模板：填好四个回调 `an_func`/`bn_func`/`f`/`df` 交给通用引擎 `_gen_roots_and_weights`。`orthopoly1d` 继承自 `numpy.poly1d`，额外携带节点、权重、权函数与区间，并通过 `_eval_func`（底层 `eval_*` ufunc）优先求值以规避系数法高阶失稳。要点：`roots_*` 是纯 Python 求积编排，而 `eval_*` 来自 `_ufuncs` 的 C 内核 ufunc，二者分工明确；闭式解（`roots_chebyt`）与大 \(n\) 渐近法（`roots_hermite` 的 \(n>150\) 路径）说明 Golub-Welsch 是默认通用引擎而非唯一引擎。
