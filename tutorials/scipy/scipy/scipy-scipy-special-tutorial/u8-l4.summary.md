本讲聚焦 `scipy.special` 中两套风格迥异的专项 C/C++ 内核。其一为 Carlson 对称椭圆积分：支撑 `elliprf/rd/rg/rj/rc` 五个 ufunc，以对称函数 \(R_F,R_D,R_G,R_J,R_C\) 为核心，采用「复制定理（duplication theorem）」迭代实现二阶收敛，并以 7 阶 Taylor 展式一次性收尾，底层由 header-only C++ 模板库 `ellint_carlson_cpp_lite` 提供，经 `ellint_carlson_wrap.*` 的 `extern "C"` 壳实例化为实/复双版本。其二为 cdflib：Fortran 翻译而来的经典分布库，关键结论是它在当前 SciPy 中**仅服务 4 个「对自由度求逆」的函数**（`fdtridfd`/`ncfdtridfd`/`ncfdtridfn`/`stdtridf`），正向 CDF 走 Boost；它返回 `(结果, 状态码, 边界)` 元组，由 `_cdflib_wrappers.pxd` 的 `get_result` 翻译为 `sf_error` 信号。构建侧形成鲜明对照：Carlson 是 `declare_dependency(sources=...)` 的头文件源依赖，编进 `_ufuncs_cxx`；cdflib 是 `static_library` 预编译后 `link_with` 进 `_ufuncs` 与 `cython_special`。两者的错误最终都汇入同一 `sf_error` 管线，承接 u7-l1 的 C 层错误机制。
