本讲是 U3 代码生成管线首讲,精读 `functions.json`(声明表)与 `_generate_pyx.py`(生成器),建立声明式 ufunc 生成的语法基础。核心是三个层层递进的模块:其一,JSON 三层嵌套映射——顶层是 ufunc 的 Python 名,中层是头文件(决定后端与语言:`.h` 为 C、`.h++` 为 C++、`.pxd` 为 Cython),内层是内核函数名与类型签名;判一个 ufunc 是否用 C++ 只看头文件名是否以 `++` 结尾。其二,单字符类型码 `f/d/g`(实浮点)、`F/D/G`(复数)、`i/l/p`(整数,`p` 为指针宽度)、`v`(仅返回占位),由 `CY_TYPES/C_TYPES/TYPE_NAMES` 三张字典翻译成 Cython/C/NumPy 拼写;签名分单输出 `input->retval`(走返回值)与多输出 `input*output->*retval`(输出走指针、`*` 开头的返回值被丢弃,通常是 sf_error 状态码)两式,由 `_parse_signature` 正则解析。其三,多内核分发:显式多内核按「列在前者优先 + `cast_order` 稳定排序」匹配,`iter_variants` 自动派生变体(`i→l` 恒派生,`d→f/D→F` 仅在无整数参数时派生以避 NumPy dtype bug),`DANGEROUS_DOWNCAST` 禁止有损转换。另引入融合类型特化写法 `[double]`、`_unsafe`/`_l` 命名约定,以及 `special_ufuncs` 名单走另一条 C++ 直注册路径(`_special_ufuncs.cpp`),`main()` 闸门避免重复生成。生成器如何把声明编译成 `.pyx` 源码留待 u3-l2。
