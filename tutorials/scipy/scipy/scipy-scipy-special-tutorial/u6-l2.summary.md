本讲承接 u6-l1，从「读源码」转向「写代码」，教你在自己的 `.pyx` 中使用 `cython_special`。三大模块：(1) 通过 `cimport scipy.special.cython_special` 做编译期导入，引入随包安装的 `.pxd` 中 `noexcept nogil` 的 C 函数声明；融合类型（如 `gamma` 用的 `Dd_number_t`）函数在调用时必须特化——用确定类型实参让编译器推断，或 `gamma[double](x)` 显式特化，否则编译报错。(2) 在 `with nogil:` 热循环里逐标量调用：单返回值函数直接赋值；多返回值函数（如 `airy`）是 `cdef void` 加指针输出，需先声明栈标量、传地址取回，块内绝不构造 Python 元组、不碰 Python 对象。(3) 核心结论：ufunc 路径内部本就是 C 级逐元素循环、与 cython_special 同源同内核，孤立单函数批量两者同量级；cython_special 真正优势在于「嵌在更大 nogil 计算流中、避免中间数组与 Python 边界往返」，并可配合 `prange` 多线程。错误语义上，cython_special 因 `nogil` 契约只能返回 `nan`、不能发告警，循环后须自行 `np.isnan` 复查。
