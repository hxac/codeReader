本讲解读 `scipy.special` 的类型化 Cython 层 `cython_special`（`.pxd` 声明 + `.pyx` 实现 + `.pyi` 桩），它是供 Cython `cimport` 的「标量直通版」函数集合，与命名空间里的 ufunc 版本（如 `airy`、`betainc`、`voigt_profile`）同名但签名迥异：ufunc 吃数组、按 dtype 运行时分发、可发 `SpecialFunctionWarning`；cython_special 吃 C 标量、按融合类型编译期特化、出错误只能返 `nan`。二者最终落到同一批 C/C++ 内核（`special_airy`、`xsf_*`、Boost 导出符号）。三条核心结论：(1) 签名双轨约定——单返回值函数 `cpdef ... return`，多返回值函数 `cdef void` 配末尾指针输出（如 `airy` 的 4 个指针），因为 C 只能返回单值且 nogil 下无法构造 Python 元组，多输出函数还需 `_pywrap`（如 `_airy_pywrap`）才能被 Python 触达。(2) 融合类型（`number_t`/`Dd_number_t`/`df_number_t`/`dfg_number_t`/`dlp_number_t`）让一个名字在编译期展开为多份特化代码，函数体里 `if X is double:` 是编译期分支、运行时零开销，成员集精确对应底层提供的内核类型组合。(3) `noexcept nogil` 是「可在无 GIL 多线程段调用」的安全契约，要求函数体绝不碰 Python 对象，`_complexstuff` 的纯 C 值拷贝（`npy_cdouble` ↔ `double complex`）即为此而存在。本讲引入的术语融合类型、`cdef`/`cpdef`、`.pxd`/`.pyx`、nogil、noexcept、指针输出、`_pywrap` 桥、`_complexstuff` 桥接器，将供后续讲义（u6-l2 实战、u7 `sf_error` 体系、u8 后端 `xsf_wrappers`/`boost_special_functions`）承接。
