本讲义聚焦 `boost_special_functions.h`，讲解 `scipy.special` 如何把 header-only 的 Boost.Math 特殊函数/概率分布接入 ufunc 体系，含三个最小模块。其一，**policy（策略）机制**：Boost.Math 用模板参数把「数学公式」与「工程行为」解耦；SciPy 定义两套策略——`SpecialPolicy`（服务 `special.*`，离散分位数取实数、域错误抛异常）与 `StatsPolicy`（服务 `scipy.stats`，离散分位数向上取整以保证 `ppf(cdf)` 往返一致、域错误静默、溢出/求值错误走 `user_error`）。两者均关闭 `promote_float`/`promote_double`，以配合 ufunc 的 `fff->f`/`ddd->d` 类型环，保证输入输出同 dtype。其二，**错误策略桥接**：把 Boost 的 C++ 错误译成 Python 信号有两条路径——路径 A（`SpecialPolicy`）走 `try/catch` 捕获 C++ 异常再调 C 层 `sf_error`（复用 u7-l1 的 GIL 桥）发 `SpecialFunctionWarning/Error`；路径 B（`StatsPolicy`）由 `user_overflow_error`/`user_evaluation_error` 自己抢 GIL 调 `PyErr_SetString`/`PyErr_WarnEx`，发 `OverflowError`/`RuntimeWarning`。其三，**float/double 双内核设计**：维度一是类型双内核（`xxx_float`/`xxx_double` 薄壳，配类型环），维度二是策略双实例（如 `bdtrik_wrap` 同一模板配两套策略各实例化一遍，衍生 `bdtrik_*` 与 `binom_ppf_*` 共四个入口、两个 ufunc 名）。所有 Boost 内核编译进 `_ufuncs_cxx` 扩展模块（链接 `boost_math_dep`），经函数指针导出与 name override 被 `_ufuncs` 引用，从而隔离 C++ 重型代码。
