本讲剖析 `scipy/special/_basic.py` 中组合与阶乘类函数的设计，核心动机是回答「为何这些函数只能用纯 Python 实现，而非做成 NumPy ufunc」：ufunc 要求固定输出 dtype 且逐元素同构，而精确组合数/阶乘的结果位数随输入无界增长，只能用 Python 任意精度整数（object 数组）承载。由此引出贯穿全讲的 **exact 双模式**：`exact=True` 走纯 Python 整数路径（`math.comb`、`_range_prod` 分治连乘、动态规划），返回任意精度整数；`exact=False` 委托底层 ufunc（`binom`/`poch`/`gamma`/`_stirling2_inexact`）返回 float64。

关键术语包括：`comb`/`perm`/`stirling2` 三个计数函数的精确与近似分工；`factorial`/`factorial2`/`factorialk` 三者共享分发器 `_factorialx_wrapper`、仅参数 `k` 不同的多重阶乘家族；以及预存的 `_FACTORIALK_LIMITS_64BITS`/`_FACTORIALK_LIMITS_32BITS` 两张溢出边界表，据此在 `long`/`int64`/`object` 三档间为数组精确计算自动选 dtype（如 k=2 时 33!! 刚好不溢出 int64）。另含便利函数 `sinc`/`softplus`（转手 numpy，借 `logaddexp` 保数值稳定）与 `bernoulli`/`euler`（委托 `_specfun` 取序列）。
