本讲精读 `_special_ufuncs.cpp` 与 `_gufuncs.cpp`,阐明 `scipy.special` 的「新 ufunc 注册路径」。核心结论:现存两条并存路径——旧的生成式路径(`functions.json`→`_generate_pyx.py`→`_ufuncs.pyx`)与新的直注册路径(`.cpp` 中调用 `xsf::numpy::ufunc`/`gufunc`,产出 `_special_ufuncs`、`_gufuncs` 两个扩展模块);二者靠 `_generate_pyx.py` 的 `special_ufuncs` 闸门名单防止重复注册,被跳过函数经生成产物末尾 `from ._special_ufuncs import (...)` 重新并入 `_ufuncs` 命名空间与 `__all__`。

引入的关键术语:直注册语法 `xsf::numpy::ufunc({static_cast<类型别名>(xsf::内核)},"name",doc)` 以括号初始化列表枚举各 dtype loop,`static_cast` 在编译期挑出 C++ 重载,类型别名(`f_f`/`d_dddd`/`F_FFFF`)与 u3-l1 类型码同源。区分普通 ufunc(逐元素、定长输出)与 gufunc(带「核心维度」、输出长度随整数参数变化,需签名串如 `"()->(np1),(np1)"` 与 `map_dims` 回调);`xsf::numpy::compose{autodiff(),...}` 包装使内核一次返回值与各阶导数。文档外置:长 docstring 独居 `_*_docs.cpp`,靠 `extern const char *` + 同模块链接对接,注册文件只关注注册逻辑;两模块各自带 `_set_action` 与 `-DSP_SPECFUN_ERROR`,承接每模块一份 TLS 错误动作表。`erf`/`airy`/`voigt_profile` 等已整体迁出 `functions.json`,新函数默认走直注册路径。
