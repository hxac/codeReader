本讲拆解 `scipy/interpolate/` 目录，把目录里的文件分为四类：公共私有模块（`_xxx.py`，真正的实现）、Cython 源（`.pyx/.pxi`）、原生 C/C++ 源（`src/`），以及几乎不含实现、仅转发旧导入路径的弃用命名空间垫片（不带下划线的 `fitpack.py`、`interpolate.py` 等，计划在 v2.0 移除）。核心结论是：扁平公共命名空间由 `__init__.py` 顶部一连串 `from ._xxx import *` 拼装而成，`__all__` 用 `dir()` 自动收集；任何公共 API 都可借各私有模块的 `__all__` 追溯回定义文件，后导入的同名符号会覆盖先导入者，故 import 顺序很重要。垫片靠模块级 `__getattr__` 实现延迟弃用，仅在被访问时才触发警告。引入的关键术语包括：私有模块、`__all__` 自动收集、垫片与延迟弃用、扩展模块、Cython/Pythran、静态库与 `declare_dependency`。`meson.build` 用三种模式构建原生扩展：纯 Cython（`_ppoly`/`_interpnd`/`_rgi_cython`）、C++ 加静态库（`_dierckx` 链接 `__fitpack`）、可选 Pythran。最重要的架构事实是子包内并存两套 FITPACK 原生后端——`_dierckx`（现代 C++）与 `_fitpack`（f2c 旧实现）。后续讲义将以本讲的「公共名→私有模块→原生扩展」对照表作为定位坐标。
