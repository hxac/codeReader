本讲围绕 `hierarchy/_hierarchy_impl.py` 的 `linkage()` 总入口展开，主题是「方法分派」。关键常量 `_LINKAGE_METHODS` 把七种链接方法名（single/complete/average/weighted/centroid/median/ward）映射为整数编码，供 Cython 后端的 Lance-Williams 函数指针表使用；`_EUCLIDEAN_METHODS=('centroid','median','ward')` 标记必须配欧氏度量的方法。`linkage()` 的 Python 层完成输入规整校验、把 1D/2D 输入统一归约成压缩距离矩阵后，由内嵌 `cy_linkage` 闭包分派到三路 Cython 后端：single→`mst_single_linkage`，complete/average/weighted/ward→`nn_chain`，centroid/median→`fast_linkage`。七个便捷函数只是固定 method 转发 `linkage` 的薄包装。核心结论：分组依据是方法的「可还原性」而非方法名——centroid/median 不可还原（可能距离倒挂）故只能用 O(n³) 的 fast_linkage；ward 虽属欧氏方法但可还原，仍走 nn_chain。欧氏度量约束只在 2D 观测输入时校验，1D 距离矩阵不校验、由用户自负。
