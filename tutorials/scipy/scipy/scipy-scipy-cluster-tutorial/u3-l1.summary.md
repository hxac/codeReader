本讲是 hierarchy 子模块的地基课，只讲清两件事：凝聚式聚类在做什么、linkage matrix Z 这一数据结构。凝聚式（agglomerative，自底向上）聚类让每个观测先自成一簇，每步合并当前最相似的两簇，共 n−1 次合并，产物为一棵二叉树。引入的关键术语 linkage matrix Z 形状恒为 (n−1)×4，四列依次为被合并两簇编号、合并距离、新簇的原始观测数；按 n+i 编号约定，原始观测占 0…n−1、第 i 步（从 0 起）新簇编号为 n+i，整树共 2n−1 个节点；其形状、Z[i,0]<Z[i,1]、簇 size=两孩子 size 之和等性质由后端 label 与 LinkageUnionFind.merge 硬性保证。linkage 接受两种输入：一维压缩距离矩阵（pdist 输出，长度 n(n−1)/2）与二维观测矩阵，由 num_obs_y 反推 n、is_valid_y 校验合法性；centroid/median/ward 三法只允许欧氏距离。本讲刻意不展开七种距离方法的数学细节与 Cython 算法复杂度，留待后续讲义。
