本讲进入 `scipy.cluster.hierarchy` 的 Cython 后端，讲解被所有聚类算法复用的三个底层数据结构。第一，`condensed_index`：用公式 `n·i − i(i+1)/2 + (j−i−1)` 将簇对 (i,j)（i<j）映射成一维压缩距离矩阵的下标，该矩阵长度为 n(n−1)/2，是 pdist 输出的「按行拍平上三角」格式；它是所有算法「把一维数组当二维表用」的地址翻译层，出现频率最高。第二，`Heap`：位于 `_structures.pxi`，是支持 O(log n) 按键改值（change_value）的最小二叉堆，靠 `index_by_key` 与 `key_by_index` 两张互逆反查表实现，所有元素交换都必须同步维护二者；它弥补了 `heapq` 无原生改值接口的缺口，仅被 `fast_linkage` 使用。第三，`LinkageUnionFind`：一种「合并时新建父节点 + 路径压缩」的并查集，仅被 `label()` 使用，而 `label()` 仅被 `nn_chain` 和 `mst_single_linkage` 调用，用于给无序 dendrogram 重新标号（find 将旧 id 翻译为活跃代表）并算簇大小。核心结论：三者各司其职——地址翻译、选最小、标号；`fast_linkage` 用 Heap 但不用 label，其余两算法用 label 但不用 Heap，三者共用 `condensed_index`。本讲是 u4-l2（MST 单链接）与 u4-l3（nn_chain / fast_linkage）的共同地基。
