本讲是阅读 `scipy.cluster` 的「地图课」，只讲结构与构建，不涉及具体算法。核心结论：`scipy.cluster` 是一个容器包，顶层 `__init__.py` 不含算法，仅通过 `__all__=['vq','hierarchy']` 暴露两个子模块。构建由 Meson 驱动，靠 `subdir(...)` 递归进入 `vq/` 与 `hierarchy/`；每个子目录的 `meson.build` 用 `extension_module` 把 `_*.pyx` 编译成可导入的扩展模块（`.so`），用 `install_sources` 原样安装纯 Python 源。关键术语：包、扩展模块、Cython、双层架构。双层架构指纯 Python 的 `_*_impl.py`（封装/实现层，负责 docstring、输入校验与类型分发）调用 Cython 的 `_*.pyx`（性能层，跑热点循环）。公共 API 经 `__init__.py` 重导出，故 `kmeans`、`linkage` 的真身都在 `_*_impl` 中，运行时再通过 `from . import _vq` 等触达编译后端，类型不匹配时回退纯 Python。追踪口诀：沿 `from ._xxx_impl import` 找实现层，再沿 `from . import _xxx` 找 Cython 后端。
