本讲义围绕 `scipy.cluster.hierarchy` 的「最优叶序（optimal leaf ordering）」展开，承接 u6-l1 的树表示与 `leaves_list`。核心问题是：固定已建好的层次树结构不动，只交换各内部节点的左右孩子，使所有相邻叶子的两两距离之和最小。关键结论是它**只重排叶子排列、绝不改变聚类**——距离列、簇大小列、合并关系与 `fcluster` 结果均不变，纯属 dendrogram 的可视化美化。

涉及两个最小模块：一是纯 Python 封装层 `optimal_leaf_ordering(Z, y)`，做输入校验（`y` 可为压缩距离矩阵或观测矩阵，后者内部走 `pdist`）并通过 `xpx.lazy_apply` 惰性桥接到 Cython 后端；二是 `linkage` 的 `optimal_ordering=True` 参数，本质是「建树 + 重排」的语法糖，默认 False 因算法在大数据集上较慢。

底层 Cython `identify_swaps` 实现 Bar-Joseph 等人 2001 年的动态规划算法，用递推 U_V[u,w]=min{U_{V_l}[u,m]+D[m,k]+U_{V_r}[k,w]} 自底向上求解，配合「簇=连续区间」编码与「排序+提前剪枝」两个优化。最终 `must_swap` 经 `is_descendant` 沿树传递并对 2 取模，生成只对调部分行左右孩子的 `swapped_Z`。引入的关键术语：最优叶序问题、相邻距离和、`optimal_ordering`、`identify_swaps`、连续区间编码、提前剪枝、`final_swap`。
