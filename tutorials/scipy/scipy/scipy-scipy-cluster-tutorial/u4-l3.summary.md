本讲义精读 `scipy.cluster.hierarchy` 中除 `single` 外的两条 Cython 聚类后端：`nn_chain`（最近邻链）与 `fast_linkage`（Müllner Generic Algorithm），并回答核心命题：分派依据为何是**可还原性**而非方法名。`cy_linkage` 闭包按 `method` 字符串三路分发：`single`→MST，`complete/average/weighted/ward`（可还原）→`nn_chain`，`centroid/median`（不可还原）→`fast_linkage`。`nn_chain` 用「最近邻链 + 互为最近邻（RNN）检测」替代全局找最小对，省去 `Heap`，稳定 \(O(n^2)\)，但产出无序需 mergesort + `label()`（`LinkageUnionFind`）收尾。`fast_linkage` 用 `Heap` 缓存每簇「最近邻下界」，靠惰性校验 `dist == D[x,y]` 决定是否重算，最优 \(O(n^2)\)、最坏 \(O(n^3)\)，产出已排序、簇号合法无需 `label`；其基本动作 `find_min_dist` 是 \(O(n)\) 线性扫描，只看高序号方向、返回 `Pair` 结构体。关键结论：`centroid/median` 在平方距离上递推含被减项，可能 \(d(x\cup y,i)<\min(d(x,i),d(y,i))\)，破坏可还原性，故无法用 `nn_chain`。两条算法都通过 `linkage_methods[method]` 函数指针表复用同一套 Lance-Williams 公式，朴素 `linkage` 函数仅作测试 ground truth。
