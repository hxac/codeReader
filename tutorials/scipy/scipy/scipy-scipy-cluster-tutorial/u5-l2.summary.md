`fclusterdata` 是 `scipy.cluster.hierarchy` 的纯 Python 便捷函数，把「`pdist → linkage → inconsistent → fcluster`」三/四步流水线打包成一次调用，对标 MATLAB 的 `clusterdata`，只返回扁平簇标签 `T`（一维 int32、从 1 起）。它与 `linkage` 的根本区别是：`linkage` 既吃压缩距离矩阵又吃观测矩阵，而 `fclusterdata` 只接受二维观测矩阵 `X`（强制 `X.ndim==2`，内部用 `_asarray` 转 C 连续 float64），自己用 `pdist` 算距离。函数体四步依次为 `pdist(X, metric)`、`linkage(Y, method)`、`inconsistent(Z, d=depth)`（仅当 `R=None`）、`fcluster(Z, criterion, depth, R, t)`。关键参数透传映射：`metric→pdist`、`method→linkage`、`depth→inconsistent 与 fcluster 两处`、`criterion/t/R→fcluster`，默认配置为 `euclidean + single + inconsistent + depth=2`。核心结论：`fclusterdata` 与「相同参数的手动三步」结果逐位相同，无任何额外魔法；但它丢失三项能力——不支持 `monocrit`/`maxclust_monocrit` 准则、不暴露 `optimal_ordering`、且不校验 `method` 与 `metric` 兼容性（`ward/centroid/median` 配非欧氏度量会静默跑过）。关键术语：便捷函数、参数透传、不一致矩阵 `R`。选择原则：只想要标签、用常见配置、不在乎 `Z`/`R` 时用 `fclusterdata`；需要画 dendrogram、用 monocrit、在同一棵树上切多次或细控距离度量时，必须手动走三步。本讲承接 u5-l1（fcluster 五种 criterion），并为 u5-l3（`inconsistent` 与 `maxRstat`/`maxinconsts` 生成 monocrit）铺垫了「为何 monocrit 准则只能手动实现」。
