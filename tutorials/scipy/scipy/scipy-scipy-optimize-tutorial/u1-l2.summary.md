本讲只回答「scipy.optimize 代码如何摆放、如何编译成可导入模块」这一底层问题，分三个最小模块。一是目录三层分工：纯 Python 负责算法逻辑与接口、子包按主题归类、C/C++/Cython 源码（集中于 src/）承担性能核心，心智模型为「算法逻辑在 .py、性能核心在 C」。二是 meson.build 编译清单：以 extension_module 声明扩展、static_library 声明被链接的静态库、install_sources 原样安装纯 Python 文件、subdir 递归纳入子目录构建。三是 src/ 下 C 内核多为经典 Fortran 数值库（MINPACK、L-BFGS-B、SLSQP、NNLS）的 C 移植，全大写函数名即 Fortran 命名痕迹。本讲确立的关键术语有：三件套命名约定（`_xxx_py.py` 包装 ↔ `_xxx` 编译扩展 ↔ `_xxxmodule.c` 胶水层 + `src/xxx.c` 内核）、反向通信（C 不调用 Python，而返回 FG 等状态码请 Python 喂回函数值/梯度，再继续迭代）、两个同名 `_zeros` 分别属顶层与 cython_optimize 子包。这套「扩展名→算法→Python 包装」对照表将作为后续算法讲义定位 C 后端入口的索引。
