本讲承接 u2-l2「`root_scalar` 只是调度器」的结论，往下钻入 `_zeros_py.py` 与 `Zeros/*.c`，讲解 6 个一维求根算法的迭代策略与落地实现。核心模块为：结果封装底座 `RootResults` 与状态码、4 个 C 括号法（`bisect`/`brentq`/`brenth`/`ridder`）、点法 `newton` 及其向量化 `_array_newton`、纯 Python 括号法 `toms748`。关键结论：括号法靠「区间两端始终异号」这一不变量保证收敛（介值定理），点法不保证收敛但更快且支持复数；收敛阶依次约为 1（bisect）、1.62（Brent 系列）、2（ridder）、≥2.7 至 ~4.6（toms748）。`brentq`/`brenth` 的 C 代码几乎逐行相同，仅外推公式不同（逆二次插值 vs 双曲）。关键术语：括号法、点法、收敛阶、状态码（`CONVERGED=0` 等，C/Python 双份）、`_wrap_nan_raise`、步长判据（newton）与区间宽度判据（括号法）、`failures`/`nz_der` 掩码、逆三次插值与 Newton-二次步、`OptimizeResult` 子类双形态访问。架构上区分纯 Python 实现（`newton`/`_array_newton`/`toms748`）与 C 后端薄包装。
