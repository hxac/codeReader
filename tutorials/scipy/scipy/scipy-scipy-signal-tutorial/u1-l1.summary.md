本讲是 scipy.signal 学习手册的纲领篇，仅涉及 `__init__.py` 与 `docs/signaltools.README` 两个文件。核心结论有四：其一，scipy.signal 是 SciPy 中面向数字信号处理（DSP）的子包，处理对象为离散数值序列，在 NumPy 之上提供滤波器设计、谱估计、STFT 等专业能力；其二，其能力被组织为 12 个功能分组（卷积、B 样条、滤波、滤波器设计、Matlab 式 IIR、LTI 系统、表示转换、波形、窗函数、峰值检测、谱分析、CZT/Zoom FFT），这份「能力目录」写在模块文档字符串里，同时驱动 Sphinx 文档生成，也是后续单元划分的依据；其三，公开 API 经「私有实现模块 → `_signal_api` 聚合 → `_support_alternative_backends` 装饰（支持 CuPy/JAX 后端）→ `__init__.py` 暴露」四层接力产生，`__all__ = _support_alternative_backends.__all__`，`_signal_api`、`_delegators` 等中转脚手架会被 `del` 掉；其四，子包保留了 `signaltools` 等弃用 stub 模块以兼容旧代码，计划 v2.0.0 移除。历史源流方面，scipy.signal 脱胎于 1999 年 Travis Oliphant 的 SIGTOOLS 模块，起步仅含 4 个 C 例程（N-D 相关、排序滤波、线性滤波 lfilter 前身、remez）与 3 个 Python 例程（convolveND、wiener、medfilt），「Python 为主、热点路径用 C/Cython/Pythran」是沿用至今的工程原则。引入的关键术语包括：子包、模块文档字符串、能力目录、功能分组、`__all__`、弃用 stub 模块、SIGTOOLS、命名空间暴露链。
