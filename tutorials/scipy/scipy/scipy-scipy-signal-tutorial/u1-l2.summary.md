本讲义建立 `scipy/signal/` 的「文件地图」。源文件分三类：以 `_` 开头的**私有实现模块**（`_xxx.py`，算法真正所在）；无下划线前缀的**弃用 stub**（小文件，经模块级 `__getattr__` 调 `_sub_module_deprecation` 转发到私有模块并发弃用警告，v2.0.0 移除）；**编译扩展源**（`.cc`/`.pyx`/Pythran `.py`）。

`meson.build` 走两条路径：`extension_module` 编译出 6 个二进制扩展（`_sigtools`、`_max_len_seq_inner`、`_peak_finding_utils`、`_sosfilt`、`_upfirdn_apply`、`_spline`，其中 `_max_len_seq_inner` 按 Pythran/Cython 二选一编译）；`install_sources` 则原样安装 34 个纯 Python 文件。

`_signal_api.py` 是私有实现的**聚合层**，用 `dir()` 自动生成 `__all__` 汇拢各 `_*` 模块的公开函数，交 `_support_alternative_backends` 统一做多后端装饰，再经 `__init__.py` 暴露为 `scipy.signal.*`，自身私有且最终被 `del` 清除。`windows/` 子包为 `signal/` 的微缩版（`_windows` + stub `windows` + `__init__` 三件套，无编译扩展）；`tests/` 的 `test_*.py` 与实现模块基本一一对应，以 `install_tag:'tests'` 单独安装，`mpsig.py`、`_scipy_spectral_test_shim.py` 为测试辅助工具。
