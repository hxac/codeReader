本讲义剖析 `scipy/datasets/_download_all.py` 这个注册表驱动的批量下载脚本，围绕三个模块展开。第一，`download_all(path)` 遍历 `_registry.registry`（文件名→SHA256）与 `_registry.registry_urls`（文件名→URL），用**无状态**的 `pooch.retrieve` 把所有数据文件下载到**任意指定目录**（默认才回退到 `pooch.os_cache('scipy-data')` 缓存目录），返回 `None`，只搬运不解析。这与单个数据集函数（`ascent`/`face` 等）用**有状态**的 `data_fetcher.fetch`、固定下到缓存、返回解析后的 `ndarray` 形成对照。第二，`if __spec__.parent is None or __spec__.parent == '':` **双导入分支**：依据 `__spec__.parent`（父包名）判断当前是包内模块还是独立脚本，分别走相对导入 `from . import _registry` 与绝对导入 `import _registry`，使同一文件既能被 import 又能被 `python xxx.py` 运行。第三，`main()` 用 `argparse` 暴露可选位置参数 `path`，并通过 `if __name__ == "__main__": main()` 守卫挂载脚本入口，保证导入时不触发下载。关键细节：`main()` 中 `default=pooch.os_cache(...)` 是立即求值的，pooch 缺失时会先抛 `AttributeError`，抢先于 `download_all` 内部统一的 `ImportError`，是 CLI 降级处理的一处小瑕疵。
