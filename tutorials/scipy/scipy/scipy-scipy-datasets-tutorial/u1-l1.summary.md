本讲义聚焦 `scipy.datasets` 子模块的入口文件 `__init__.py`，将其定位为整个子模块的「门面」——自身几乎不含业务逻辑，只做三件事：用相对导入汇集私有模块的函数、用 `__all__` 声明公开 API、用模块文档字符串说明工作原理。它对外暴露五个公开函数，分为两类：数据集方法 `ascent / face / electrocardiogram`（调用后返回 ndarray），以及工具方法 `download_all / clear_cache`（负责批量下载与清理缓存，不返回数据本身）。核心结论是：「名字对外可见」由 `from ._xxx import` 引入与 `__all__` 登记共同决定，两者缺一不可；其中 `__all__` 才是控制 `import *` 行为与文档工具识别公开接口的依据。关键术语包括：`__all__` 公开合约、相对导入、模块文档字符串、可选依赖 Pooch、registry（文件名→SHA256+URL 的注册表）、SHA256 校验。整体机制为：数据集文件存放在遵循 `dataset-<name>` 命名的独立 GitHub 仓库，首次调用时由 Pooch 联网下载并做 SHA256 校验，缓存到本地 `scipy-data` 目录，之后命中缓存即返回 ndarray，缓存路径随平台而异，断网时可手动放入文件。函数实现在带下划线的私有模块 `_fetchers / _download_all / _utils` 中，体现「内部私有 + 门面公开」的组织方式。
