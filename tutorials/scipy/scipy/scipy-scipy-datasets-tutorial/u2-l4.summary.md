本讲专讲 `scipy.datasets` 的缓存清理机制，覆盖三个最小模块：公开入口 `clear_cache`、内部实现 `_clear_cache`、以及路径兜底库 `platformdirs`。核心结论：`clear_cache` 是一层纯转发薄壳，公开签名只有 `datasets` 一个参数（可传单个可调用对象、其列表/元组、或 `None` 清全部），函数体仅一行转发；复杂度全部下沉到带下划线的 `_clear_cache`，后者多出 `cache_dir`、`method_map` 两个可注入参数，专为单元测试隔离真实缓存而设。`_clear_cache` 是一棵五步决策树：补 `method_map` 默认值 → 补 `cache_dir` 默认值 → 目录不存在则提示返回 → `datasets is None` 用 `shutil.rmtree` 整目录删除 → 否则归一化为列表、按 `method_map` 逐文件 `os.remove`。关键设计：清理侧不 `import pooch`，而是用 `platformdirs.user_cache_dir("scipy-data")` 重建 pooch 用过的同一条路径，保证「写到哪」与「删哪」对齐；清理幂等友好，目录或文件缺失只打印不报错，仅方法名查不到才抛 `ValueError`。引入的关键术语：薄壳/实现分层、可注入参数、分支决策树、`method_files_map` 作为方法名→文件名的唯一查表消费点（方法名取自函数 `__name__`，故须传函数对象本身）。
