本讲是专家层收官篇，把前八讲的零件串成一个综合实战：给 `scipy.datasets` 新增一个数据集时该改哪几处、不改哪几处、为什么。核心结论是「注册表驱动」——新增数据集本质上是「填三张表 + 写一个解析函数 + 公开导出 + 一条测试」。必改清单为 `_registry.py` 的 `registry`/`registry_urls`/`method_files_map` 各加一条、`_fetchers.py` 写一个戴 `@xp_capabilities(out_of_scope=True)` 的 fetcher 函数、`__init__.py` 补导入与 `__all__`、`tests/test_data.py` 补 shape 与哈希断言；而 `data_fetcher`/`fetch_data`/`clear_cache`/`download_all`/autouse fixture 因运行时才读表而自动受益。本讲引入两个生态集成点术语：`@xp_capabilities(out_of_scope=True)`（声明退出数组 API 跨后端适配，效果为置 `np_only=True`、登记能力表并短路、向 docstring 注入说明）与 meson 白名单安装（`python_sources` 列表 + `py3.install_sources`，测试文件加 `install_tag: 'tests'`，仅新建独立 `.py` 文件时才需登记）。同时强调 `_registry.py` 纯数据地基与 `_fetchers.py` 逻辑层的职责分离。
