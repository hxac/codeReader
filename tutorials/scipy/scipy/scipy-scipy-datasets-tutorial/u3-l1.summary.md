本讲剖析 `scipy.datasets` 对可选依赖的「降级处理模式」，核心是 **try-import-or-None** 惯用法：用 `try: import DEP / except ImportError: DEP = None` 在导入时静默吞错，把失败推迟到真正调用时再以友好的 `ImportError` 抛出，从而让不下载数据的用户对缺依赖无感。讲义引入的关键术语包括：可选依赖、传递依赖、哨兵值 `None`、导入段（静默）与使用段（报错）的两段式设计，以及「导入即失败」与「用到才报错」两种失败策略的对比。

讲义精读三处降级：`_fetchers.py`（pooch，多一个 `else` 分支创建 `data_fetcher` 单例）、`_utils.py`（try 的是 platformdirs 却报 pooch，因前者是后者的传递依赖）、`_download_all.py`（pooch）。三处共同合约：导入静默、只抓 `ImportError`、哨兵用 `None`、判断用 `is None`、报错文案逐字相同且统一提示「用 pip/conda 装 pooch」——这是有意设计。同时指出 `_download_all.py` 的 `main()` 中 argparse 默认值 `default=pooch.os_cache(...)` 会在 pooch 缺失时先抛 `AttributeError`，是一处「漏网」细节。
