本讲从「硬盘文件」视角剖析 `scipy.integrate` 的物理结构与构建方式，拆解四个最小模块：顶层 `meson.build`、`_ivp` 子包、`_rules` 子包、`src/` C 源目录。核心结论：构建（build）是把 C/Fortran 数值核心编译成 `.so` 扩展、供 Python `import` 的装配流程，由 Meson 构建系统驱动。顶层 `meson.build` 用三指令完成装配——`py3.extension_module` 编译 C 源为扩展模块、`subdir` 递归构建子包、`py3.install_sources` 原样拷贝纯 Python 文件。引入的关键术语：扩展模块（`_quadpack`/`_odepack`/`_vode`/`_dop` 四个核心扩展，其中 `_odepack`、`_vode` 依赖 `lapack_dep`）、胶水层、子包、加载桩（弃用垫片）、`src/` 数值核心（由 Fortran 翻译而来）。核心解耦思想：纯 Python 子包负责易变逻辑与 API，C 扩展承担性能敏感的古老数值核心，二者经运行时 `import` 解耦。
