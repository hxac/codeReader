本讲聚焦 fftpack 包入口 `__init__.py`，讲解 Python 模块导出机制如何塑造公共 API。核心要点：`__all__` 是模块对外公开名字的白名单，既是 `from module import *` 的取名单，也是作者承诺维护的公共契约；fftpack 的包级 `__all__` 共 31 项，不含任何 `_` 开头的私有名字。这 31 个名字通过四条聚合导入（`from ._basic import *` 等）从 `_basic`、`_pseudo_diffs`、`_helper`、`_realtransforms` 四个私有子模块搬运到包顶层，实现门面模式（Facade）；各子模块自带 `__all__`（8+10+5+8=31）决定其贡献，与包级 `__all__` 严格同步。公共 API 归为五大功能分组：FFT、实变换 DCT/DST、伪微分算子、辅助函数为包顶层扁平名字；卷积一组地位特殊，作为独立子模块 `scipy.fftpack.convolve`（Cython 编译）存在，需带模块前缀访问且不在 `__all__` 中。最后引入再导出（re-export）概念：`fftshift`、`ifftshift`、`fftfreq` 经 `_helper.py` 从 numpy 转手而来，可用 `__module__` 属性识破其真实出处，且与 numpy 中对象为同一引用。承接术语：`__all__`、聚合导入、门面模式、五大分组、再导出、`__module__`。
