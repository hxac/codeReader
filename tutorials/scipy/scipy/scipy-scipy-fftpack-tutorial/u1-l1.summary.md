本讲义是 `scipy.fftpack` 学习手册首篇，聚焦该包入口文件 `__init__.py`，阐明 `fftpack` 作为**遗留（legacy）模块**的身份定位。核心结论有三：其一，模块通过文档字符串中的 `.. legacy::` 指令声明遗留身份，官方明确建议**新代码改用 `scipy.fft`**，但 legacy 标记仅属文档层、不影响运行行为；其二，`fft`、`dct` 等公共函数实际定义在 `_basic.py` 等私有子模块，经 `from ._basic import *` 聚合导出，因此 `from scipy.fftpack import fft` 是合法且**不报警**的路径；其三，报警的真正来源是 `basic.py`、`helper.py` 等不带下划线的遗弃命名空间 shim——它们借助模块级 `__getattr__`（PEP 562）委托 `_sub_module_deprecation` 发出 `DeprecationWarning`，将在 SciPy v2.0.0 移除。由此引入的关键术语：legacy 模块、模块文档字符串与 `.. legacy::` 指令、聚合导入（`__all__` 与 `from ._xxx import *`）、shim 垫片、模块级 `__getattr__`、`_sub_module_deprecation`，以及「遗留（文档层）≠ 弃用警告（运行层）」这一核心区分，供后续讲义承接。
