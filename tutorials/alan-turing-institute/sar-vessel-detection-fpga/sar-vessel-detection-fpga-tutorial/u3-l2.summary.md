本讲覆盖训练侧流水线「图像入网」的最小闭环：TIFF 加载与多通道归一化。核心模块有三：其一，SAR 芯片是 int16、3 波段的 GeoTIFF，像素为 dB（VV/VH 约 [-50,20]）与水深米值（bathymetry 约 [-6000,2000]），必须用 cv2.IMREAD_UNCHANGED 读取，否则默认 cv2.imread 会降位并破坏通道，悄悄掉精度，且需覆盖每一处 imread 调用。其二，normalize() 对每通道做「线性映射到 [0,1] + clip + 放大取整」三步，压成 uint8 [0,255]；clip 不可省，否则 astype(uint8) 会回绕。其三，特选 uint8 而非 mean/std 浮点，是为了零改动复用 Ultralytics 默认增强（Mosaic/HSV/翻转均假设 [0,255]），normalize 必须插在 BaseDataset.get_image_and_label 中 load_image 之后、增强之前。关键术语：dB、波段/通道、int16、uint8、广播、通道反转——cv2 读取 3 波段 TIFF 时按 BGR 约定反转通道，故 min_values=[-6000,-50,-50] 对齐的是内存顺序 (bathymetry,VH,VV) 而非磁盘写盘顺序 (VV,VH,bathymetry)。本讲归一化属 u1-l3「训推一致性」暗线一环，推理侧 C++ 需用同公式映射至 signed int8（见 u6-l1）。
