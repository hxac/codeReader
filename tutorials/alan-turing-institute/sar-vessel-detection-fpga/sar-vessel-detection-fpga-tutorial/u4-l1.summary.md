本讲是第四单元「Vitis AI 量化」首篇，负责把训练好的 float32 模型转为 int8。KV260 的 DPU（DPUCZDX8G）是 int8 定点加速器，不量化无法运行；量化换来体积缩为 1/4、更高吞吐与更低功耗，代价是可控的精度损失。int8 对称量化核心是缩放因子 scale，单点误差上界为 s/2；为每层激活确定动态范围、选出最优 scale 的过程即校准（calibration）。

量化两条路线的分水岭是「是否改权重」：PTQ（训练后量化）不改权重，仅用校准集前向观测激活统计量，分钟级、适合快速摸底；QAT（量化感知训练）跑完整训练循环让权重适应量化噪声，精度更高但更慢。PTQ 流程为 calib→test/export→compile：quant_mode=calib 算 scale 落盘为 quant_info.json，再以 quant_mode=test dump_xmodel=True 固化导出，最后 vai_c_xir 编译。本项目位宽 bitwidth=8，由 pytorch_nndct 库解释所有 nndct_* 参数，quant/dequant stub 包在网络前后。

两个激活替换标志把 sigmoid→hsigmoid、silu→hswish 换成 MobileNetV3 分段线性近似，以适配 DPU 定点单元；由于 scale 依赖激活分布，imgsz=800 与两个标志必须在 calib、导出、QAT 全部命令中逐字一致。后续 u4-l2 讲 QAT 实现、u4-l3 讲算子适配、u4-l4 讲导出与 DPU 编译。
