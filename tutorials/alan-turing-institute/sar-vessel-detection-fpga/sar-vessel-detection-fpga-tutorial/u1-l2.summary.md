本讲为项目仓库建立「地图」。仓库根 README 列出六大核心组件：framework/vitis_ai（推理框架补丁）、platform/kv260（硬件平台与固件）、platform/post_processing（HLS 解码核）、software/training（训练）、software/quantization（量化）、software/inference_app（板载推理），外加 dataset/ 数据准备。数据流主线为 dataset 芯片→training 的 .pt→quantization 的 int8 .xmodel→inference_app 板载预测，vitis_ai 补丁需先贴到 Vitis AI 源码。讲义区分软件侧（CPU/GPU 可完成）与平台侧（FPGA 底座及加速核），引入产物（.pt/.xmodel/.xsa/固件三件套）、训推一致（TIFF 加载、归一化、PIoU2 在 Python 与 C++ 两侧重复实现）、signed 8-bit 输入、DPU（DPUCZDX8G_B4096/325MHz）与 HLS 后处理解码核的软硬对应等关键概念。quantization 因许可证无源码。
