本讲为「FPGA 硬件平台构建」首篇，建立 KV260 与 DPU 的硬件认知底座。KV260 核心是一颗 Zynq UltraScale+ MPSoC，物理上分为 **PS**（4 核 Cortex-A53 ARM，跑 Linux 与 C++ 推理程序）与 **PL**（FPGA 可编程逻辑，承载 DPU），二者经 **AXI** 总线互联并共享 DDR，形成「PS 下指令、PL 干重活、共享内存省搬运」的卸载模型。推理算力来自 PL 里的 **DPU 4.1（DPUCZDX8G）**，板载架构名 `DPUCZDX8G_ISA1_B4096`，工作频率 325 MHz，开启 softmax 引擎（SFM_ENA=1），URAM_N_USER=40；这些参数在「TCL 配置 → 资源表 → `xdputil query`」三处自洽闭环。资源利用率方面：LUT 53%、寄存器 45% 较宽裕，BRAM 75.69% 最紧（剩 35 块），URAM 62.5% 几乎全归 DPU。核心结论：**BRAM 是瓶颈资源**，是评估能否再加 PL 加速核（如 u8 解码核）的决定性约束。关键承接概念：DPU 为 int8 定点加速器、ISA1 不含非线性指令（解释 u4 算子替换动机）、BRAM 紧张，共同构成后续量化、推理、HLS 加速的物理根源。
