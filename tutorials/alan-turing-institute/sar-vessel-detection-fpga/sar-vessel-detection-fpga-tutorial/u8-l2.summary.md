本讲（u8-l2）精读 HLS 解码内核 `decode_kernel.cpp` 的算法主体（不含接口与优化指令），把 DPU 输出的 int8 定点特征图解码为 NMS 前候选框。内核被 host 逐层调用，每层处理一个检测头（P2/P3/P4/P5，`layer_size` 分别为 200/100/50/25，`layer_stride` 为 4/8/16/32）。

核心结论围绕三个最小模块：第一，**按字节提取 int8**。每个 anchor 有 67 个 int8 特征，排列为「4 段距离×16 bin + 3 类别」，即类别下标为 `n*67+64+m`，距离 bin 下标为 `n*67+t*16+m`；从 `ap_uint<64>` 字取值用固定三连位运算 `idx>>3`、`idx&0x7`、`(int8_t)(word>>(byte_idx*8))`，须与 testbench 小端打包对应。第二，**整型阈值剔除**。利用 sigmoid 单调性，把 `sigmoid(0.1·q)>0.9` 预解为整数比较 `q>conf_thresh_inverse`（由 `det_scale=0.1` 算出 floor 值 21），在 softmax 之前用一次整数比较整批 `continue` 掉背景 anchor，`expf` 只对真正输出的框计算。第三，**DFL 距离 softmax 与边框还原**。对左/上/右/下四条边各做一次减最大值的 16-bin 数值稳定 softmax，取期望值得 `distances[0..3]`（DFL 解码），再以格子中心为参考点用「参考点 ± 距离」拼框、乘 `layer_stride` 还原为像素坐标的 `box_cx/cy/w/h`（dist2bbox）。该算法与 u6-l3 软件后处理逐行同构，是软硬一致性交叉验证对象；优化指令（UNROLL/PIPELINE/ARRAY_PARTITION）留待 u8-l3。关键术语：anchor-free、DFL、`det_scale`、`conf_thresh_inverse`、softmax 期望值、dist2bbox。
