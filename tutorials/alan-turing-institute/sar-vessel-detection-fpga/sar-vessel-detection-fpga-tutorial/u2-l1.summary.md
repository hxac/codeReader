本讲是第二单元「xView3-SAR 数据集准备」首篇，承接端到端工作流的「数据切片」阶段，只讲「原始数据长什么样」，不讲裁剪细节。核心内容三块：一是磁盘目录约定——数据集分为 `data/`（场景影像）、`labels/`（检测 CSV）、`shoreline/`（海岸线 `.npy`）三大顶层子目录，其中 `data` 和 `shoreline` 内部再按 training/validation/public 三划分，每个场景是一个以 `scene_id` 命名的文件夹；`labels` 则是一个划分一个大 CSV，靠 `scene_id` 列区分归属（`scene_id` 末字母 t/v/p 暗示划分）。二是 SAR 三通道输入——每个场景含 `VV_dB.tif`、`VH_dB.tif`（极化回波强度，dB）、`bathymetry.tif`（水深）三个 GeoTIFF，三者叠加构成模型输入；bathymetry 需重采样到 VH 网格，芯片波段顺序固定为 1=VV、2=VH、3=bathymetry，须贯穿训练↔推理一致。三是标注字段——核心字段为 `scene_id`、`detect_scene_row/column`（场景全局像素坐标）、`is_vessel`、`is_fishing`，类别编码 `label = is_vessel + is_fishing`（0/1/2）；评估阶段另用 `confidence`、`near_shore`、`length`。关键术语：SAR、极化 VV/VH、bathymetry、GeoTIFF、rasterio、scene_id、dB、NODATA(`-32768`)。仓库不含数据，需用 `aria2` 自行下载。
