本讲解析解码 HLS 内核的对外「门面」——函数签名、数据打包与接口分工，不含解码算法（u8-l2）与综合优化（u8-l3）。要点有三：(1) 入口用 `ap_uint<64>*` 将 8 个 int8 特征图打包进一个 64 位字，恰好填满 KV260 的 64 位 AXI 通道，带宽利用率由 1/8 升至 8/8，内部以 `idx>>3`、`idx&0x7` 拆分字/字节索引。(2) INTERFACE pragma 令大数组走 `m_axi offset=slave`（连 DDR、支持突发、基地址由主机下发），小标量 layer_size/layer_stride/out_box_num 与 `return` 握手走 `s_axilite`，为 Vitis 加速器标准范式。(3) 读输入独占 bundle `gmem0`、6 个输出共享 `gmem1`，成「1 读 1 写」两条独立物理通道，读写并行且省 HP 端口；可调常量 `MAX_BOXES=2048`、`NUM_CLASSES=3`、`OUTPUT_DIM=67=4×16+3`，`depth=335000` 为最大层 200×200 打包后的字数；主机以 `int8_t*` 写、内核以 `ap_uint<64>*` 读同一片内存，借 AXI「只搬字节」实现跨边界类型双关。
