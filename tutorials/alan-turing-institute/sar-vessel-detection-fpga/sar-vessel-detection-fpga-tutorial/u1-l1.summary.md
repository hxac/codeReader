本讲建立项目全局认知，分三个模块：项目动机、xView3-SAR 数据集、KV260 功耗约束。项目让卫星在轨道上就地分析 SAR（合成孔径雷达）图像、自动检测船舶，即「星载 ML」，以规避卫星与地面站断续连接导致的数小时传输延迟，服务海事安全等时间敏感场景；障碍是 SOTA 模型太大、太耗电。数据采用最大开源 SAR 船舶数据集 xView3-SAR（Sentinel-1），每场景由 VV、VH、bathymetry（水深）三通道组成，按 data/labels/shoreline 三类组织，须先用 generate_xview3.py 裁剪成固定大小芯片（如 800×800）方可训练。硬件选 Kria KV260 MPSoC：PS 为 ARM CPU、PL 为 FPGA，神经网络推理跑在 PL 侧的 DPU 硬件 IP 上，功耗压在 <10W 卫星红线内。核心结论：相比 SOTA GPU 模型，检测与分类精度仅低约 2% 与 3%，计算效率却高出约 50× 与 2500×。<10W 功耗约束是后续量化、剪枝、HLS 加速等所有工程决策的总根源。
