本讲是「Vitis AI 推理框架补丁」单元首篇，目的是把训练侧的数据约定在 C++ 推理侧复刻，解决训推不一致。核心是 `xview3_yolov8_v3.5.patch`：它用 unified diff（含 hunk 头、`+`/`-` 行）修改 Vitis AI 的四个源文件，分别校正数据格式、数值范围、IoU 度量、网络结构四类不一致。其中 `demo_accuracy.hpp` 与 `yolov8_imp.cpp` 是本讲精读的「输入端」改动，`apply_nms.cpp`（PIoU2 NMS）与 `yolov8.cpp`（P2 头）属「输出端」，留待 u6-l2、u6-l3。

关键结论：读 SAR 的 `int16` TIFF 必须给 `cv::imread` 加 `cv::IMREAD_UNCHANGED`，否则位深被压缩甚至读空，与训练侧同一教训。`image_preprocess` 用 `subtract/divide/min/max` 四步把三波段（bathymetry、VV、VH）线性归一化到 `[0,1]`，参数 min=`(-6000,-50,-50)`、量程=`(8000,70,70)`，与训练侧 `normalize()` 完全一致；再 `convertTo(CV_8S,255,-128)` 转 signed int8 `[-128,127]`，因 DPU（DPUCZDX8G）是有符号 8 位定点加速器。训练侧 uint8 与推理侧 signed int8 仅差 −128 平移，真正一致性由共同的 `[0,1]` 归一化保证。补丁还删掉默认 resize+letterbox（写死 `scale=1.0/left=0/top=0`），因芯片在 u2 已切成 800×800；当 README 摘要与补丁冲突时以补丁为准。
