本讲精读 `xview3_benchmark.cpp`，拆解 Vitis AI `demo_accuracy` 四段多线程流水线：1 个 `TiffReadImagesThread` 读图 → N 个 `DpuRunThread` 跑 DPU → 1 个 `SortDpuThread` 按 `frame_id` 重排 → 1 个 `YolovXAcc` 输出。段间用 `BoundedQueue`（有界队列）衔接、靠反压控速；N 个 DPU 线程的真正收益是把主机前后处理与 DPU 执行在时间上重叠以隐藏延迟，而非同时算 N 张图。两处子类化是核心：`TiffReadImagesThread` 注入全局 `g_last_frame_id` 作为收尾扳机（`IMREAD_UNCHANGED` 读 int16 TIFF 已由补丁在基类解决），`YolovXAcc` 把 `YOLOv8Result` 逐框写成 JSON 数组，靠 `g_last_frame_id` 命中加 `seekp(-2L)` 覆盖末尾逗号闭合数组并 `exit(0)`，坐标为芯片局部像素坐标。已知脆弱点：若最后一帧零检测，`seekp(-2)` 会误伤收尾。
