本讲讲解 KV260 硬件设计如何从源码编译成比特流并交接给软件，分三块。一是 TCL 批处理构建：`main.tcl` 用一条 `vivado -mode batch` 命令串起建工程、导入约束、注册 IP 仓库、引入块设计 `dpu_kv260.tcl`、综合、实现写比特流、导出硬件平台全流程，使硬件设计成为可复现、可审查的代码；`${board}` 参数仅决定 xdc 与块设计脚本的文件名，工程板型已写死为 KV260。二是 xdc 三类约束：`BITSTREAM` 比特流压缩、`PACKAGE_PIN` 引脚绑定、`IOSTANDARD/SLEW/DRIVE` 电气属性；KV260 作为 SOM 把外设收进 PS，故 PL 约束极简，仅约束风扇使能引脚 `fan_en_b`。三是 XSA 导出：`write_hw_platform -include_bit` 将比特流、PS 配置、地址映射打包为 `.xsa`，是硬件到软件（u5-l3 PetaLinux、`createdts` 固件）的唯一正式交接物，下游据此生成设备树与驱动。
