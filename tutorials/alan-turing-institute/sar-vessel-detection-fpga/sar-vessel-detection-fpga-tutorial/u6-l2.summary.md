本讲聚焦 Vitis AI 补丁对 `apply_nms.cpp` 的改动，覆盖三个最小模块：apply_nms 的三种 IoU 实现、PIOU2_NMS 运行时环境变量、训练-推理一致的 IoU 度量。

补丁在 NMS 阶段保留三种可切换度量：`cal_iou`（标准 IoU，不重叠即归零）、`cal_new_iou`（DIoU，IoU 减中心距平方对外接框对角线平方之比）、`cal_piou2`（PIoU2，新增核心）。NMS 按分数降序贪心抑制，用相似度 `ovr >= nms_thresh` 判冗余，故度量选择直接决定「多近算冗余」、影响最终保留框集合。`cal_piou2` 先算标准 IoU 并加不重叠守卫，再叠加非单调聚焦惩罚：以真值框宽高归一化中心偏移 P，得 L_v1 与聚焦权重 f(x)=3xe^(-x²)，Λ 默认 1.2。

环境变量开关通过 `DEF_ENV_PARAM(PIOU2_NMS,"0")` 编译期注册、`ENV_PARAM` 运行时读取实现「编译一次、shell 改一行即可切换度量」，契合板载交叉编译二进制难重编译的约束；`yolov8.cpp` 默认传 `iou_type=0` 走标准 IoU，需 `export PIOU2_NMS=1` 才切到 PIoU2。

核心结论是训推一致暗线：C++ 侧 `cal_piou2` 与训练侧 Python `compute_piou` 在 dw/dh、P、L_v1、聚焦组合、Λ 上逐式一致，唯一差异为 C++ 多不重叠守卫；启用 PIoU2 使推理判重口径与训练框回归损失对齐，为本项目推荐配置，承接 u3-l3 并为 u6-l3 的后处理流程铺垫。
