本讲剖析 nano-vllm 承载请求生命周期的核心数据结构 Sequence。它维护四组关键字段：token 计数（num_tokens 总长、num_prompt_tokens 为 prompt 长度定值、num_cached_tokens 表示已算 KV 的进度水位、num_scheduled_tokens 为本步增量）、三态状态机 WAITING/RUNNING/FINISHED、物理块编号列表 block_table、分块视图（num_blocks 向上取整、block(i) 切片）。状态迁移均由调度器驱动：schedule 在 prefill 累计算完后推至 RUNNING，preempt 显存不足时回退至 WAITING 并复位 is_prefill 以重做 prefill，postprocess 命中 eos 或达 max_tokens 时终结为 FINISHED；postprocess 同时累加 num_cached_tokens、清零 num_scheduled_tokens。__getstate__/__setstate__ 为多进程张量并行广播而精简序列化：prefill 带整张 token_ids、decode 仅带 last_token，丢弃采样参数等 worker 不需字段。block_size 为类级属性，引擎启动时由 Config.kvcache_block_size 统一覆盖。本讲是 u2 调度单元与 u3 块管理的共同数据基础。
