本讲讲解 `ModelRunner.run` 作为调度器（Sequence 对象）与模型（GPU 张量）之间的「张量桥梁」，五步编排为 prepare → sample → run_model → sample → reset_context，末尾清空保证 Context 的每步语义。prefill 用 **varlen 打包**把多条不等长序列首尾相接拼成一维，以 `cu_seqlens_q`（新 token 数）与 `cu_seqlens_k`（含缓存前缀的全部 key 数）标记边界：二者相等即无前缀缓存，后者更大则命中前缀、需构造 `block_tables`。`slot_mapping` 按公式 `slot=block_table[p//B]*B+p%B` 把新 token 的 K/V 落入 paged cache 物理槽位（prefill 按块逐段生成，decode 每序列一个值）。decode 每序列每步只送 1 个 token，用 `context_lens`（=序列总长）指明 attention 读取的历史长度，每步必构造 `block_tables`。核心直觉：`slot_mapping` 管「K/V 写到哪里」，`block_tables` 管「历史 K/V 从哪读」；`positions` 用绝对位置以与缓存前缀的 RoPE 一致。除 `input_ids`、`positions` 外，所有注意力元数据经全局 `Context` 传递，为后续 Attention 内核、Context 机制与 Qwen3 结构讲义奠定接口基础。
