本讲建立 nano-vllm 的全局代码地图，覆盖三个最小模块。其一，包的对外接口：`__init__.py` 仅导出 `LLM` 与 `SamplingParams` 两个名字，构成瘦出口；内部按 `engine / layers / models / utils` 四子包分层，子包目录无 `__init__.py`，靠 Python 3 命名空间包机制工作，打包范围由 `pyproject.toml` 的 `include = ["nanovllm*"]` 决定。其二，继承入口：`LLM` 是 `LLMEngine` 的空子类（复刻 vLLM 命名），能力完全等同；`LLMEngine.__init__` 装配引擎「三件套」——`model_runner`（执行）、`tokenizer`（编解码）、`scheduler`（调度），并经 `Config`、`Sequence` 协作初始化。其三，四子包职责：`engine` 管调度与执行、`layers` 管可复用神经网络层、`models` 管具体模型结构、`utils` 管跨模块工具。核心结论是一条贯穿主链路：`__init__ → llm.LLM → engine.llm_engine.generate → scheduler + model_runner → models.qwen3 → layers.*`，其中 `engine` 在上驱动、`models/layers` 在下被驱动、`utils.context` 横向传递注意力元数据。关键术语：`LLMEngine`、`Config`、`Scheduler`、`ModelRunner`、`Sequence`、`BlockManager`、`Qwen3ForCausalLM`、`Context`、命名空间包、空子类、引擎三件套。
