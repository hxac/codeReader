本讲介绍 nano-vllm 的注意力元数据传递机制：用一个模块级全局 `Context` 对象，把每步推理所需的注意力元数据从写端 `ModelRunner` 送到读端 `Attention`/`ParallelLMHead`，避免污染整条 `forward` 调用链签名。`Context` 是 `@dataclass(slots=True)` 轻量数据类，含八个字段：阶段开关 `is_prefill`、varlen 边界 `cu_seqlens_q/k` 与 `max_seqlen_q/k`、写槽位的 `slot_mapping`、读历史长度的 `context_lens`、读历史块号的 `block_tables`；prefill 与 decode 各用其中互补子集。生命周期由三个函数管理：`set_context` 整体替换全局对象，`get_context` 读取，`reset_context` 清空，`ModelRunner.run` 严格按「prepare→set→forward(get)→reset」封口，每步不留残留。读端方面，`Attention.forward` 消费几乎所有字段：用 `slot_mapping` 写 K/V，靠 `is_prefill` 在 `flash_attn_varlen_func`（prefill）与 `flash_attn_with_kvcache`（decode）间切换，命中前缀缓存时改读整张 cache 并配 `block_tables`；`ParallelLMHead.forward` 只用 `is_prefill` 与 `cu_seqlens_q`，prefill 时按 `cu_seqlens_q[1:]-1` 抽取每序列最后一个 token 再算 logits，把计算量从总 token 数降到序列数。该设计依赖推理主循环单步串行，多进程张量并行时每个 rank 各持一份等价 `Context`，无需跨进程同步。后续讲义将承接此机制：CUDA Graph 回放时 `get_context` 在 `graph.replay()` 前把元数据拷进静态 `graph_vars`；张量并行下 `ParallelLMHead` 露出词表分片与 `gather` 尾巴。
