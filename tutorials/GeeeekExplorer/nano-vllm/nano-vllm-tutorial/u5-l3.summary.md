本讲讲清张量并行（TP）的运行时落地：当 `tensor_parallel_size > 1` 时，nano-vllm 如何切模型到多卡并让多进程同步跑同一次前向。核心结论是它采用「**rank 0 住主进程、spawn 出 rank 1..n-1 worker**」的拓扑，子进程入口即 `ModelRunner.__init__`，末尾 `loop()` 使其常驻；选 `spawn` 而非 `fork` 以避免 CUDA 上下文污染。初始化有两个同步点：`init_process_group`（NCCL 会合，端口 2333）与 `barrier`（保证 rank 0 先建好名为 `"nanovllm"` 的 1 MiB 共享内存）。关键设计是**控制面/数据面分离**：NCCL 只传 GPU 张量（数据面），方法名与参数这类 Python 对象走「共享内存 + Event」广播（控制面）。`write_shm`/`read_shm` 采用「4 字节长度前缀 + pickle 载荷」的单向协议，`call` 作为分发层由 rank 0 广播后所有 rank 各自执行。因 `run` 内含 `all_reduce`/`gather`，NCCL collective 充当**隐式屏障**，故单向 Event 无需 ack。最后**只有 rank 0 采样**：`ParallelLMHead.gather` 把完整 logits 汇到 rank 0，且 rank 0 即主进程，采样结果天然回到引擎，无需结果回传 IPC。
