本讲承接 Scheduler 的 prefill 优先调度，解决两类卡死。其一，超长 prompt 单步算不完时启用 chunked prefill（分块预填充）：由 max_num_batched_tokens 不足触发，每步实际算 min(待算, remaining) 个 token，分块序列卡在 waiting 队首续算，算完才晋升 RUNNING；「只允许首条分块」守卫保证每 step 至多一条半截 prefill 序列，以简化 varlen 打包。其二，KV Cache 装不下所有 decode 序列时 preempt（重算式抢占）：deallocate 归还物理块、置 WAITING 与 is_prefill=True、appendleft 插回 waiting 队首，受害者选队尾以保护老序列；恢复后整条重算而非 swap 换出。decode 跨块检查仅一行 can_append：len(seq) % block_size == 1 时需新空闲块，否则恒真，返回 False 即触发抢占。两机制共同保证调度器在请求极大、显存极紧下不卡死。
