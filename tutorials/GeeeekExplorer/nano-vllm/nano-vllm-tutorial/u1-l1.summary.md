本讲是 nano-vllm 学习手册的第一篇，定位为面向零基础读者的总览与快速上手。核心结论是：nano-vllm 是一个约 1200 行 Python 的 vLLM 极简复刻，聚焦**离线推理**（一次性给一批 prompt、算完返回），不做在线服务，主打代码可读而非功能齐全。它声明四大特性——Prefix Caching（前缀缓存）、Tensor Parallelism（张量并行）、torch.compile、CUDA Graph，原理留待后续讲义展开。安装需 GPU 环境，依赖 flash-attn 与 triton，默认用 Qwen3-0.6B 跑通 example.py。

本讲覆盖三个最小模块。**项目本身**：定位、特性、安装与源码地图，只看用户视角最外层文件。**SamplingParams**：采样参数数据类，仅 temperature / max_tokens / ignore_eos 三字段，温度越小分布越尖锐输出越确定；关键设计是**不允许 greedy 采样**（temperature=0 会触发断言报错），因其采样器走指数分布路线需非零温度。**LLM 入口**：LLM 只是 LLMEngine 的空子类，理解它即理解构造函数与 generate 方法；构造参数经 **kwargs 过滤为 Config 字段（enforce_eager、tensor_parallel_size、max_model_len 等），拼错会被静默忽略；generate 返回 [{"text": str, "token_ids": list[int]}, ...]，进度条用 Prefill/Decode 后缀展示两类阶段吞吐。建立的关键术语包括 token、prefill、decode、KV Cache、张量并行，供后续讲义承接。
