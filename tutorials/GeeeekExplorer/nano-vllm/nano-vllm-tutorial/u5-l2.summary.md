本讲拆解 nano-vllm 中贴有 @torch.compile 装饰器的四个层：RMSNorm、RotaryEmbedding、SiluAndMul、Sampler，全部集中在 nanovllm/layers/。核心结论是：算子融合（TorchDynamo 捕获计算图、Inductor 生成代码）同时削减 kernel launch 次数与中间张量在 HBM 显存的往返带宽；逐元素算子是融合主力，而规约（mean、sum、softmax）会构成融合边界，故 RMSNorm 大致融成两个 kernel。引入的关键概念包括 kernel launch overhead、pointwise/reduction、fp32 规约防溢出、pre-norm 残差须在归一化前定值。四层各有要点：RMSNorm 拆出纯归一化与融合残差两条编译路径；RotaryEmbedding 融合 gather 与 rotate-half 旋转，全模型经 lru_cache 共享一份 cos_sin_cache；SiluAndMul 将 chunk、silu、乘融成单 kernel，中间 sigmoid 不落盘；Sampler 用 argmax(probs/exponential) 一步实现 Gumbel-max 采样，全程 GPU 并行、免 CPU 往返，这也是禁止 temperature=0 的原因。融合降 kernel 数，与 CUDA Graph 降 launch 数叠加；Sampler 在图外调用、仅 rank 0 运行。
