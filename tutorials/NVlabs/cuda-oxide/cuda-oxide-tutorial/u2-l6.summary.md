本讲回答泛型核与闭包如何上 GPU，含四个最小模块。**单态化**：由 rustc 完成，宏以 `#[inline(never)]` + 易失指针取地址生成「强制单态化」胶水，使每个类型实例被 backend 收集；泛型核在 `load()` 时不预加载，启动时按名现取并缓存。**闭包标量化**：闭包即匿名捕获结构体，整体作单条 byval `.param` 穿过边界（`push_kernel_scalar`），host/device 两侧均不展平捕获，零捕获闭包为 ZST 两端一致丢弃。**type_id_u128**：用 `core::intrinsics::type_id`（边界 `T: ?Sized` 而非 `'static`）算出与 backend `tcx.type_id_hash` 同一的 128 位稳定哈希，格式化为定长 `<base>_TID_<hex32>`。**核心结论**：稳定哈希擦除生命周期区域，故借用不同生命周期的闭包不泛滥伪 PTX 变体；Fn/FnMut/FnOnce 派发路径不同，各成入口家族；`Copy` 让每线程持闭包副本；`move` 闭包按值传值，非 `move` 闭包捕获 host 引用、依赖 HMM 才能在 GPU 解引用。
