本讲回答两个问题：泛型内核如何变成多个 PTX 入口，闭包捕获如何穿过 host/device 边界。其一，单态化（monomorphization）由 rustc 完成，cuda-oxide 只用宏生成「强制单态化」胶水（`#[inline(never)]` 与易失指针取地址），让每个用到的类型实例被 backend 收集器发现；泛型核在 `load()` 时不预加载，启动时按名现取并缓存。其二，闭包本质是匿名捕获结构体，被当作单条 byval `.param` 整体压栈（`push_kernel_scalar`），host 与 backend 两侧都不展平捕获；零捕获闭包作 ZST 两端一致丢弃。其三，`type_id_u128` 用 `core::intrinsics::type_id`（边界 `T: ?Sized`、非 `'static`）算出与 backend `type_id_hash` 完全相同的 128 位稳定哈希，渲染为定长 `<base>_TID_<hex32>` PTX 名；哈希擦除生命周期，故借用不同 region 的闭包不泛滥伪变体。其四，`Fn`/`FnMut`/`FnOnce` 走不同设备侧派发即不同 PTX 入口家族；`Copy` 让每线程持闭包副本；`move` 闭包按值传值，非 `move` 闭包捕获 host 引用依赖 HMM 才能在 GPU 解引用。
