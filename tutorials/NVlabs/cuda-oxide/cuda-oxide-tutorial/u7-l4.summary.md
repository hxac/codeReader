本讲是「测试、调试与工程化」收束篇，串联三块让 cuda-oxide 长 codegen 流水线可信赖的基础设施。一是保留符号命名契约：`reserved-oxide-symbols` crate 作为唯一真相源，用 `sha256` 截断哈希后缀 `246e25db` 防用户意外撞名，用「常量/构建器/谓词」三层 API 防多处重复，宏在展开期借 `RESERVED_ROOT` 拒绝越界命名，谓词用 `contains` 子串匹配统一处理裸名与 FQDN，并依赖哈希保证的互斥子串免写排除逻辑。二是 `.oxart` 制品嵌入与锚符号防裁剪：bundle 包进宿主目标文件嵌入二进制，对库 crate 用三道防线对抗链接器丢弃——段首全局锚符号（`SymbolScope::Linkage`）配宏侧 `link_name` 引用触发归档提取、`SHF_GNU_RETAIN` 防 `--gc-sections`、弱遗留别名保新旧宏混用兼容。三是差分模糊测试：`fuzzer` crate 用 `no_std` 的 FNV-1a 单 `u64` 轨迹把 CPU 与 GPU 执行的中间值折叠成指纹，「两 u64 相等」等价于「轨迹逐位一致」，配 rustlantis 风格生成器，产出 `PASS`/`MISMATCH`/`COMPILE_FAIL`/`UNSUPPORTED` 四态自动捕获 codegen 回归。
