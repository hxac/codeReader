本讲是手册起点，仅涉及 README.md 与 rustc-codegen-cuda/src/lib.rs 两个文件。核心结论：cuda-oxide 是一个自定义 rustc 后端，把纯 Rust 的 `#[kernel]` 函数编译成 CUDA PTX，支持 host/device 单源编译，当前处于 alpha 阶段。引入的关键术语包括 kernel（GPU 内核）、PTX、MIR（rustc 中级 IR）、codegen backend（可插拔代码生成后端）、单源编译、Pliron（类 MLIR 的 IR 框架，含 dialect-mir 与 LLVM dialect 两种方言）、mem2reg、llc。架构上"包装而非重写"：CudaCodegenBackend 内持标准 LLVM 后端，经入口符号 `__rustc_codegen_backend` 由 rustc 动态加载，分流发生在 `codegen_crate`。device 与否由保留命名空间 `cuda_oxide_kernel_*` 加调用图可达性判定，无需 `#[cfg]`。端到端流水线为 Rust → MIR（共享）→ dialect-mir → mem2reg → LLVM dialect → LLVM IR → llc → PTX，host 路径走标准 LLVM。硬约束：编译必须 `-Z mir-enable-passes=-JumpThreading`，否则屏障被复制致 GPU 死锁，且需 llc-21 以上。
