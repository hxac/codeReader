本讲全部发生在宿主端 cuda-core，聚焦 GPU 显存的分配、释放与主机↔设备搬运。核心引入 `DeviceCopy`——比 `Copy` 更严的 unsafe trait，要求「字节级复制合法且全零比特模式合法」，把 `String`、`bool` 等挡在设备内存之外，错误在编译期暴露。`DeviceBuffer<T>` 类比设备端 `Vec`，靠 RAII 与 `dealloc_stream` 字段区分同步分配（`cuMemAlloc`）与流序分配（`cuMemAllocAsync`），`Drop` 按来源配对释放，混用即 use-after-free。搬运分两层：底层 `memory.rs` 是 `cuMemcpy*`/`cuMemset*` 的 unsafe 薄封装；高层 `from_host`/`to_host_vec`/`zeroed` 等安全 API 因接收借用的 host slice，会在入队拷贝后 `synchronize` 再返回，`*_async_unchecked` 变体则不同步、由调用方管理 host 数据生命周期。最后，`PinnedHostBuffer` 用 `cuMemAllocHost` 分配锁页内存，省掉驱动对可分页内存的中转拷贝，换得更高带宽与真正的异步重叠，代价是资源稀缺、只宜给高频搬运数据使用。
