本讲深潜 `mir-lower` crate，解释 dialect-mir/dialect-nvvm 的 op 如何逐条降级为 LLVM dialect op。派发骨架是 pliron 的 `DialectConversion`：driver 在 `rewrite` 中把需要跨函数状态的少数 op（Func/SharedAlloc/GlobalAlloc/ExternShared）特判分流，其余 op 经 `op_cast` 做 O(1) 虚表派发到各自的 `MirToLlvmConversion`；三张跨函数状态表（shared_globals、device_globals、dynamic_smem_alignments）经 Context aux_data 传递，`LoweringOptions` 同样挂在 Context 而非全局环境变量。算术 op 的核心难点是符号性恢复：LLVM 整数 signless，`div/rem/shr/cmp/checked_*` 必须靠 `operands_info` 查回转换前的 MIR 类型，才能在 `sdiv/udiv`、`ashr/lshr`、`slt/ult` 间正确选择。FMA 收缩契约是本讲关键结论：浮点 `add/sub/mul/div/rem` 经 `add_fastmath_flags` 只挂 `contract` 一个 fast-math 标志，由 `allow_fma_contraction`（受 `CUDA_OXIDE_NO_FMA` 控制）开关，绝不挂其他标志以保持逐位可比；`f*_fast` 子族另挂全套 `fast`，禁用 FMA 时再抠掉 `contract`。`mir.call` 是漏斗：依次拦截 rust bit/saturating/bigint/float-math 占位调用，分别落到 LLVM intrinsic、libdevice `__nv_*` 调用或 `f*_fast` 浮点 binop，普通调用经参数展平与 addrspace 桥接发出 `llvm.call`。GPU intrinsic 走 `call_intrinsic` 或 convergent/sideeffect 内联 PTX 两条路，原子因 NVPTX 后端 bug 采用 `atomicrmw Monotonic + 前后 fence` 栅栏拆分变通；本轮新增的 f16/tf32/s8 三条 mma 统一走「建 struct 结果 → convergent inline PTX → ExtractValue 拆回」范式，差别仅在约束串与 PTX 模板。引入术语：signless、operands_info、fast-math flags（contract/reassoc/nnan 等）、libdevice `__nv_*`、占位 callee、inline_asm convergent/sideeffect、栅栏拆分变通。
