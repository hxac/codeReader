本讲是 U2「编写 GPU 内核」首讲，拆解 `#[kernel]` 与 `#[cuda_module]` 两个过程宏在编译期的展开逻辑，共四个最小模块。第一，命名契约：`reserved-oxide-symbols` 是宏与 codegen 后端间唯一的对话真源，核心常量 `KERNEL_PREFIX = cuda_oxide_kernel_246e25db_`，其中 `246e25db` 是防误撞的固定哈希后缀；分三层——常量、构造器（`kernel_symbol`）、谓词/提取器（`is_kernel_symbol`、`kernel_base_name`），后者用 `contains` 兼容跨 crate 路径限定。第二，`#[kernel]` 作用于单个函数：改名为 `KERNEL_PREFIX + 原名` 并加 `#[no_mangle]` 供后端按全名收集，同时生成 marker 结构体实现 `CudaKernel` trait，其 `PTX_NAME` 为去掉前缀的原始名。第三，`#[cuda_module]` 扫描模块内所有 `#[kernel]`，生成 `LoadedModule` 结构体（每个 kernel 一个 `CudaFunction` 字段）、类型安全的 `module.<核>(...)` 启动方法（通过设备端类型到宿主端 `DeviceBuffer` 的映射保证编译期类型安全），以及 `load()`/`load_named()`/`from_module()` 加载器。第四，`load()` 运行时三步走：anchor 握手保住内嵌 `.oxart` 段、`load_embedded_module` 解析 PTX、对每个 kernel 用 `PTX_NAME` 调 `cuModuleGetFunction` 拿到句柄。核心结论：后端收集用带前缀全名，而 PTX 入口与驱动解析用剥前缀的原始名——两个名字服务两个阶段。引入的关键术语包括 `CudaKernel` trait、`LoadedModule`、`DisjointSlice`、`thread::index_1d()`、anchor 握手等，为 u2-l2 线程索引、u2-l4 启动配置等后续讲义承接。
