本讲介绍 cuda-oxide 宿主运行时地基 `cuda-core` crate，把以 `CUcontext`/`CUstream`/`CUmodule`/`CUevent` 为中心的 CUDA Driver C 接口封装为带生命周期的 Rust 类型。核心类型四个：`CudaContext` 用 RAII + `Arc` 管理进程级共享的主上下文，构造走 `cuInit`→`cuDeviceGet`→`cuDevicePrimaryCtxRetain`，释放走对应 release；`CudaStream` 以非阻塞流 + fork/join 模型搭建多流依赖图，同流严格 FIFO、跨流靠显式 event 建序；`CudaModule`/`CudaFunction` 经 `Arc` 链式保活（function→module→context）防止模块被提前卸载；`CudaEvent` 作跨流同步与 GPU 端计时原语。贯穿全部类型的两条暗线：各持一份 `Arc<CudaContext>` 保活，调用驱动前先 `bind_to_thread()` 满足「线程绑定」隐式契约。关键概念包括主上下文引用计数、线程绑定、粘性错误（Drop 中无法返回 Result 故延迟补报）、`launch_host_function` 跨 C 边界需 `catch_unwind`、计时必须用 `CU_EVENT_DEFAULT`（默认 `DISABLE_TIMING` 不可计时），以及 `IntoResult` 统一把裸 `CUresult` 转为 `DriverError`。本讲为后续模块加载与异步执行模型奠定基础。
