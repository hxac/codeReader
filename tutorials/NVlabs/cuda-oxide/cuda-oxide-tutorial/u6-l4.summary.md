本讲是 u6-l2（mir-importer）与 u6-l3（mir-lower）的合龙篇，把「端到端新增一条 GPU intrinsic」做成一条可复制的五阶段流水线：`cuda-device → mir-importer → dialect-nvvm → mir-lower → llvm-export`，五工位各管一段、缺一不可。以 f16 张量核 `mma_m16n8k16_f32_f16` 和 int8 `mma_m16n8k32_s32_s8` 为样板：设备层只写 `#[inline(never)]`+`unreachable!()` 的**设备桩**与安全文档；方言层定义带 `Verify` 的 `nvvm.*` op 并 `register`；派发层用 **FQDN** 在 `try_dispatch_intrinsic` 加 `match` 臂；emitter 走「拆 fragment 数组→建 op→装回」三段范式，用常量下标 `MirExtractFieldOp` 保持片段不进栈；lowering 用 **LLVM 内联汇编约束串**（如 `"=f,...,r"`）的「先输出后输入」编号把操作数从 `C,A,B` 重排成 PTX 的 `D,A,B,C`，输出靠 struct 聚合再 `ExtractValue` 拆回。关键区分：走内联 PTX 的 convergent 指令用 `inline_asm_convergent`（convergent 已焊标记，可跳过导出层）；走 LLVM intrinsic 路径则用 `call_intrinsic`，且 convergent 指令必须在 `is_convergent_intrinsic` 登记，否则 LLVM 可能 hoist 导致结果错误或死锁。同一形状的变体（如 `.satfinite`）几乎是复制粘贴。
