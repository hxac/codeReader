本讲聚焦 Hopper sm_90+ 引入的 Thread Block Cluster 层级与分布式共享内存（DSMEM）。cluster 在 Grid 与 Block 之间新增第五级，把至多 8 个 block 绑定到同一 GPC 协同调度，使其私有共享内存拼接成可跨块寻址的 DSMEM，省去绕道全局显存。核心 API 包括坐标 intrinsic（cluster_ctaidX/Y/Z、cluster_nctaidX/Y/Z）与派生量 block_rank()、cluster_size()；`#[cluster_launch(x,y,z)]` 仅注入编译期标记，mir-importer 据此产出 `.reqnctapercluster`。DSMEM 须用 dsmem_read_u32（mapa + ld.shared::cluster 一条汇编），map_shared_rank 后 generic 解引用会触发 ILLEGAL_ADDRESS；其正确用途是取远程 mbarrier 地址。cluster_sync() 是要求所有线程到达的集群阻塞屏障，与 u5-l3 的 mbarrier（能感知异步搬运）互补，二者叠加构成 mcast 协议。集群形状是 launch contract 一等公民：宏自动 .with_cluster 并把 min CC 抬到 (9,0)，prepare_* 在活设备上逐项校验，失败映射为带 kernel 名的 Cluster* 错误。本讲为 u5-l5（TMA 多播）铺路。
