本讲讲解 instant-ngp 如何通过 CMake 把源码组织、编译、链接成可运行程序。构建分**配置**（`cmake -B build`，检测环境、生成工程文件）与**编译**（`cmake --build build`，真正调用 nvcc/g++）两阶段；README 的两条命令即对应这两步。核心构建链是：所有核心源码先编成静态库 `ngp`，再链接出两个产物——可执行文件 `instant-ngp`（入口 `src/main.cu`）与 Python 共享库 `pyngp`，核心逻辑只编译一次。配置阶段最早期会检查 git 子模块，缺失则 `FATAL_ERROR`，需 `git submodule update --init --recursive` 修复。关键概念包括：6 个 `NGP_BUILD_WITH_*` 开关控制 GUI、OptiX、Python 绑定、RTC、Vulkan 等附加能力，且 Vulkan/DLSS/OpenXR 均嵌套在 GUI 分支内，关 GUI 即一并禁用，条件编译由 `NGP_GUI` 宏驱动；CUDA 目标架构通过环境变量 `TCNN_CUDA_ARCHITECTURES` 指定（检测逻辑委托给 tiny-cuda-nn，取值为 GPU 计算能力数字，如 RTX 3090 为 86）；项目用 C++14、CUDA14，并启用 `--extended-lambda`、`--use_fast_math` 等 nvcc 选项。`requirements.txt` 仅服务于 `scripts/` 下 Python 脚本，与 C++ 编译无关。
