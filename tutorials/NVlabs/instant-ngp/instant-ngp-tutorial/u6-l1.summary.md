本讲拆解 instant-ngp 渲染管线的「最后一公里」。核心是 CudaRenderBuffer 缓冲对象体系：渲染内核只写 frame_buffer，accumulate 将多帧运行平均进 accumulate_buffer，tonemap 再经色调映射写到 surface，构成三级解耦流水线；CudaRenderBufferView 是传给内核的轻量 POD 视图。SurfaceProvider 抽象出 CudaSurface2D 与 GLTexture（内嵌 CUDAMapping）两类渲染目标。CUDAMapping 用 cudaGraphicsGLRegisterImage 把 GL 纹理注册成 CUDA 可写 surface，实现零拷贝上屏，WSL 等环境退化为 CPU 拷贝。动态分辨率按 factor=sqrt(pixel_ratio/render_ms*1000/target_fps) 缩放 in_resolution 追目标帧率，带 ±20% 防抖。注视点渲染用 Foveation 分段二次扭曲让中心满采样、边缘稀疏，blit 用 GL_LINEAR 填边；累积采样 spp 做运行平均去噪，相机移动即重置。in_resolution 与 out_resolution 之分是理解 DLSS、动态分辨率、注视点三者各自省算力的核心。
