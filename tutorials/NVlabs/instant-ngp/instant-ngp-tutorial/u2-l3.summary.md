本讲串联文件加载的「入口→判定→分发」链。`mode_from_scene` 是纯函数，按路径形态短路判定模式：目录或 `.json`→Nerf、`.obj`/`.stl`→Sdf、`.nvdb`→Volume、其余兜底为 Image；路径不存在返回 None。`load_training_data` 是数据内核：先用 `mode_from_scene` 定模式→`set_mode` 清旧状态并切换→`switch` 分发到 `load_nerf/load_mesh/load_image/load_volume`，「先切模式后加载数据」顺序不可换。`load_file` 是用户文件统一入口（CLI 位置参数、GUI 拖拽、pyngp 汇于此），用扩展名加 JSON 字段嗅探把文件分流到快照 / 网络配置 / 相机路径 / 训练数据四类，训练数据为兜底分支，首次拖入时自动置 `m_train=true`。异常分层：`mode_from_scene` 只返回 None 不抛错，`load_training_data` 升级为异常，`load_file` 用 try/catch 降级为日志。
