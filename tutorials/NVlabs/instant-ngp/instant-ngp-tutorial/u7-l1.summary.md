本讲讲解 instant-ngp 的 Python 绑定模块 pyngp 的构造方式，覆盖三个最小模块：pybind11 模块注册、Testbed Python API、numpy/json 互转。

核心结论：pyngp 是项目的第二条构建产物，由 CMake 开关 NGP_BUILD_WITH_PYTHON_BINDINGS（对应宏 NGP_PYTHON）控制，全部绑定集中在 src/python_api.cu 的单一 PYBIND11_MODULE(pyngp, m) 巨型块中，按「枚举 → 自由函数 → 辅助类 → Testbed 及嵌套类」顺序注册。由于 Testbed 是承载项目全部能力的「上帝对象」，整类暴露后 Python 即获得无 GUI 但功能更全的 instant-ngp。

关键术语与概念：pybind11（纯头文件胶水库）、`def`/`def_readwrite`/`def_readonly`/`def_property` 四种成员暴露写法（后者经 lambda 处理需转换/计算/校验的属性如 loss、dlss、root_dir）、`py::overload_cast` 选择重载、`implicitly_convertible` 实现字符串到 fs::path 的隐式转换、`call_guard<gil_scoped_release>()` 在耗时 GPU 操作中释放 GIL。同时指出直接调 load_training_data 不会自动开启训练，需手动置 shall_train=True。

数据互转三套机制：numpy 经 py::array_t 与 request() 搬运图像/网格；JSON 经 pybind11_json 让 nlohmann::json 形参直接接 Python dict；vec/mat 经 type_caster 对接，矩阵需在导入时转置以消除行/列主序差异。典型控制流为：构造 Testbed → load_training_data → shall_train=True → frame() 循环训练 → render 出 numpy 图 → save_snapshot 存 .ingp 快照。
