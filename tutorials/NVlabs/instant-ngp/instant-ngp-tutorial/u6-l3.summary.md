本讲讲解 instant-ngp 的快照（snapshot）机制——把训练好的模型连同重建所需的全部信息写进一个文件。核心结论：`save_snapshot` 把已摊平的完整网络配置 `m_network_config` 与 `m_trainer->serialize()` 序列化的权重（可选含优化器动量）合并在同一个 JSON 对象里写盘，并附加版本号、模式、训练步数、相机内外参，以及 NeRF 专有的密度网格（`density_grid_binary`）等运行时状态，构成自包含的训练现场。

磁盘格式上，`.ingp`（msgpack 外套 zlib 压缩，推荐）与 `.msgpack`（裸 msgpack）都属快照；GUI 的 Compress 复选框仅在 `.ingp` 时可用。`load_network_config` 按后缀分三路：`.ingp`/`.msgpack` 走 msgpack 反序列化且**不做 parent 合并**，`.json` 走 `json::parse` 加 `merge_parent_network_config`——这正是快照必须携带完整配置（而非依赖 parent 链）的原因。

还原时 `load_snapshot` 先用配置 `reset_network(false)` 重建拓扑一致的网络（不归零训练状态），再 `deserialize` 灌回权重、恢复 `training_step`，从而接着训练而非从头训。关键开关 `include_optimizer_state` 决定是否保存 Adam 动量：带上可平滑续训，不带则文件更小、适合纯推理。
