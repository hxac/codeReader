本讲阐释 instant-ngp 的 JIT 融合（JIT fusion）机制：在运行时用 NVRTC（RTC）把「位置编码 + 密度 MLP + 方向编码 + 颜色 MLP + 体渲染合成」整条管线编译成单个融合内核，使中间特征留在寄存器、只 launch 一次，以节省显存带宽与启动开销。核心载体是 tiny-cuda-nn 的 `CudaRtcKernel`，它按需编译、源码由 `fmt::format` 动态拼接、头文件经 cmrc 内嵌、编译产物缓存于 `rtc/cache`。关键设备函数 `eval_nerf` 由 `NerfNetwork::generate_device_function` 递归生成，靠 params offset 与 fwd_ctx offset 把四块子组件权重首尾相接拼成端到端函数，密度取自密度头第 0 维。三类融合内核为 `render_nerf`（渲染快路径，仅 Shade 模式）、`train_nerf`（含 Rfl/RflRelax 训练唯一实现）、`trace_sdf`（SDF 球面追踪）；Volume/Image 不走此路。`m_jit_fusion` 开关受 CUDA≥11.8、算力≥75 限制，编译失败被 try/catch 捕获后自动降级回非融合 `NerfTracer` 路径，关 JIT 时 Rfl/RflRelax 训练强制降回普通 NeRF 模式。
